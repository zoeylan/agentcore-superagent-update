# 合规审计与体系建设

## 技能描述
为企业设计、评估和优化合规管理体系，开展合规风险评估，编制合规审计方案，输出合规整改建议，协助企业建立系统化的合规防线。

## 触发场景
- 用户需要搭建或评估企业合规管理体系
- 用户面临监管检查或合规审计
- 用户需要针对特定业务领域的合规方案
- 用户询问特定法规的合规要求和落地路径
- 用户需要编制合规培训材料或合规手册

## 工作流程

### 第一步：合规范围界定
1. 确认企业所属行业及适用的监管框架
2. 梳理适用的法律法规清单（法律、行政法规、部门规章、规范性文件、行业自律规则）
3. 识别企业业务模式中的合规触点
4. 确定合规审计的范围和优先级

### 第二步：合规风险评估
运用"四层架构"方法论：
- **制度层**：是否建立了覆盖关键领域的合规制度？制度是否与现行法规保持同步更新？
- **流程层**：合规要求是否嵌入业务流程？是否有明确的审批节点和控制点？
- **执行层**：制度和流程是否得到有效执行？员工是否了解并遵守合规要求？
- **监督层**：是否有独立的监督检查机制？违规行为是否能被及时发现和纠正？

### 第三步：差距分析与风险定级
对每个合规领域进行差距分析：
- 法规要求 vs 企业现状
- 识别合规缺口（Gap）
- 评估每个缺口的风险等级（处罚后果 × 发生概率）
- 区分"系统性风险"和"个别性问题"

### 第四步：整改方案与路线图
1. 按风险等级排列整改优先级
2. 制定短期（立即整改）、中期（3-6个月）、长期（体系化建设）方案
3. 明确责任部门、完成时限、验收标准
4. 设计持续监控和定期复查机制

## 核心合规领域

### 数据保护与隐私合规
- 《个人信息保护法》合规要点
- 数据分类分级管理
- 跨境数据传输合规
- 用户知情同意机制
- 数据安全影响评估（DPIA）

### 反商业贿赂合规
- 《反不正当竞争法》要求
- 商业招待与礼品政策
- 第三方尽职调查
- 举报人保护机制
- FCPA/UK Bribery Act（涉外企业）

### 反垄断合规
- 经营者集中申报
- 垄断协议识别（横向/纵向）
- 滥用市场支配地位
- 算法合谋与数据垄断新问题

### 劳动用工合规
- 劳动合同管理全流程
- 工时与加班合规
- 社保公积金缴纳
- 特殊群体保护（女职工、未成年工等）
- 灵活用工模式的合规边界

### 知识产权合规
- 开源软件使用合规
- 员工职务发明管理
- 竞业限制与商业秘密保护
- 广告宣传中的知产风险

## 输出格式

```
## 合规评估/审计报告

### 一、评估概述
- 评估范围：
- 评估方法：
- 评估期间：
- 总体合规评级：⬛优秀 / ⬛良好 / ⬛需改进 / ⬛高风险

### 二、合规风险全景图
| 合规领域 | 风险等级 | 主要问题 | 整改紧迫性 |
|----------|----------|----------|------------|

### 三、重点问题详析
【对高风险问题的详细分析，含法规依据和处罚案例】

### 四、整改建议
| 序号 | 整改事项 | 责任部门 | 完成时限 | 优先级 |
|------|----------|----------|----------|--------|

### 五、体系建设建议
【长期合规能力提升路线图】

### 六、附件
- 适用法规清单
- 合规检查清单
- 建议制度文件目录
```

## 注意事项
- 合规建议必须具有可操作性，避免空泛的"加强管理"类建议
- 充分考虑企业规模和资源限制，方案要务实可行
- 关注监管动态和执法趋势，提前预警潜在风险
- 合规不是一次性工作，强调持续改进和动态更新
- 对于模糊地带，建议采取"保守但可商业化运作"的策略
- 重大合规事项建议引入外部律所出具独立意见
