# 合同审查与风险识别

## 技能描述
对各类商业合同进行全面审查，识别法律风险点，提出修改建议，确保合同条款的合法性、完整性和可执行性。

## 触发场景
- 用户提交合同文本要求审查
- 用户询问合同条款的法律含义或风险
- 用户需要合同修改建议或替代条款
- 用户要求对比分析两个版本的合同差异

## 工作流程

### 第一步：合同基本面审查
1. 确认合同类型（买卖、服务、许可、合作、投资等）
2. 审查合同主体资格（是否具有签约能力、是否需要授权）
3. 确认合同标的的合法性和可行性
4. 检查合同形式要件（是否需要书面/公证/审批/备案）

### 第二步：核心条款逐项审查
按照"三看"原则系统审查：
- **一看主体与资格**：签约主体是否适格、代理权限是否充分、是否存在关联关系
- **二看权利义务对等性**：付款条件与交付条件是否匹配、风险转移时点是否合理、单方权利条款是否过度
- **三看违约救济与退出**：违约责任是否明确可执行、解除条件是否合理、争议解决机制是否有利

### 第三步：风险评估与分类
使用风险矩阵对识别的问题分级：
- 🔴 红色风险：条款违反法律强制性规定，或可能导致合同无效/不可执行
- 🟠 橙色风险：条款严重偏向对方，可能导致重大经济损失
- 🟡 黄色风险：条款存在模糊或遗漏，可能引发争议
- 🟢 绿色提示：建议优化但不影响合同效力的改进点

### 第四步：修改建议输出
对每个风险点提供：
1. 问题描述（哪里有问题、为什么有问题）
2. 法律依据（适用的法律条款或司法解释）
3. 修改建议（具体的替代条款文本）
4. 谈判策略（如何与对方沟通修改理由）

## 审查要点清单

### 通用条款审查
- [ ] 合同主体信息完整准确
- [ ] 合同标的描述清晰具体
- [ ] 价款/对价条款明确（含税、币种、支付条件）
- [ ] 交付/验收标准和程序
- [ ] 知识产权归属和许可
- [ ] 保密义务（范围、期限、例外）
- [ ] 陈述与保证条款
- [ ] 违约责任（违约金/赔偿/解除权）
- [ ] 不可抗力条款
- [ ] 合同解除/终止条件
- [ ] 争议解决（管辖/仲裁、准据法）
- [ ] 通知条款
- [ ] 合同生效条件和期限

### 特殊关注点
- 限制竞争/竞业禁止条款的合理性
- 无限责任/连带责任条款
- 自动续约/默示同意条款
- 单方变更/解释权条款
- 格式条款的效力问题

## 输出格式

```
## 合同审查意见书

### 基本信息
- 合同名称：
- 合同双方：
- 审查日期：
- 风险等级：【整体评价】

### 风险识别清单
| 序号 | 条款位置 | 风险等级 | 问题描述 | 修改建议 |
|------|----------|----------|----------|----------|

### 重点问题分析
【对重大风险点的详细分析】

### 修改建议汇总
【按优先级排列的修改清单】

### 审查结论
【总体评价和签约建议】
```

## 注意事项
- 始终基于现行有效法律法规进行分析
- 对于法律存在争议的问题，说明不同观点和司法实践倾向
- 区分"必须修改"和"建议修改"，不过度放大风险
- 考虑商业可行性，不提出对方不可能接受的修改要求
- 重要合同建议附带谈判策略备忘录
