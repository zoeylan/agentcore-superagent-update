# 合同修改建议 (Contract Revision Suggestion)

## 技能概述

本技能基于风险识别结果，为每个需要修改的条款提供具体的替代条款文本和修改说明。输出的修改建议必须是可直接应用的条款文本（而非笼统的方向性意见），并附带法律依据和修改理由，使业务部门和对方能够快速理解修改意图。

## 适用场景

- 对审查中识别的风险条款提供具体修改方案
- 为合同谈判提供多套备选方案（强势版/折中版/底线版）
- 对缺失条款提供补充条款建议
- 对表述模糊的条款提供明确化改写方案

## 执行步骤

### 步骤1：确定修改优先级
根据风险等级确定修改的紧迫性：
- 高风险条款：必须修改，否则建议不予签署
- 中风险条款：强烈建议修改，如对方不接受需获得管理层确认
- 低风险条款：建议修改，可作为谈判筹码灵活处理

### 步骤2：设计修改方案
对每个需修改的条款，提供分层方案：

**方案A（最优方案）**：完全保护我方利益的条款表述
**方案B（折中方案）**：平衡双方利益的妥协表述
**方案C（底线方案）**：我方可接受的最低保护标准

### 步骤3：编写替代条款文本
- 使用规范的法律术语和合同语言
- 确保修改后条款与合同其他部分的逻辑一致性
- 标注修改部分（新增用下划线，删除用删除线）
- 确保条款可独立理解、表意完整

### 步骤4：撰写修改说明
每条修改建议附带：
- **修改理由**：简明说明为何需要修改（法律风险/权益失衡/表述模糊）
- **法律依据**：支持该修改的法律条文或司法案例
- **谈判要点**：与对方沟通时的建议说辞和论据
- **让步空间**：标注该条款的谈判灵活度（硬性要求/可协商/锦上添花）

### 步骤5：整体协调性检查
- 确认所有修改建议之间不产生内部矛盾
- 验证修改后的条款与未修改条款的衔接
- 评估全部修改的整体谈判可行性（避免修改过多导致对方拒绝）

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| risk_findings | 列表 | 是 | 法律风险识别的输出结果 |
| original_clauses | 文本 | 是 | 需修改条款的原文 |
| negotiation_position | 枚举 | 否 | 我方谈判地位（强势/对等/弱势） |
| industry_context | 文本 | 否 | 行业特殊背景信息 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| revision_list | 列表 | 逐条修改建议（含原文、修改方案A/B/C、理由、法律依据） |
| supplementary_clauses | 列表 | 建议新增的补充条款 |
| negotiation_guide | 文本 | 整体谈判策略建议和优先级排序 |
| revision_summary | 摘要 | 修改建议总览（必改项/建议改项/可选改项数量） |

## 最佳实践

1. **替代条款即用即改**：提供的条款文本应可直接复制到合同中使用，无需再加工
2. **理由充分但简洁**：修改理由对内详细对外精炼，方便业务部门向对方解释
3. **考虑谈判实际**：避免提出不切实际的修改要求，考虑商业关系和行业惯例
4. **标注谈判策略**：明确哪些是"必须坚持"的底线，哪些可以作为交换筹码
5. **保留原始权利**：修改建议应尽量保留我方在原条款下已有的权利

## 约束条件

- 每个修改建议必须提供至少一个具体的替代条款文本
- 不得仅输出"建议修改"或"需要调整"等模糊意见
- 修改方案需考虑对方接受的可能性，不能全部极端有利于我方
- 涉及合同金额、期限等商业条款的修改需标注"需业务部门确认"
