# 合同签署管理 (Contract Signing Management)

## 技能概述

本技能用于管理合同审批通过后的签署流转过程，确保签署主体适格、授权有效、印章使用合规、签署流程可追溯。支持电子签署和纸质签署两种模式，确保签署完成的合同具有完整的法律效力。

## 适用场景

- 合同审批通过后发起签署流程
- 验证签署方的主体资格和授权代表权限
- 管理电子签章的发起和确认
- 管理纸质合同的用印和快递流转
- 处理签署过程中的异常情况（签署逾期、对方拒签等）

## 执行步骤

### 步骤1：签署前准备
- 确认合同已获得相应层级审批（审批单号/流程记录）
- 确认合同文本为最终定稿版（版本号核对、修订痕迹已清除）
- 确认合同份数（通常各方各持等份）
- 选择签署方式（电子签/纸质签）

### 步骤2：签署主体验证
**我方签署验证：**
- 确认签署授权人与合同约定一致
- 确认授权书有效期覆盖签署日期
- 确认授权金额范围覆盖合同金额
- 确认印章类型正确（合同章用于一般合同，公章用于重大合同）

**对方签署验证：**
- 确认对方签署代表的身份和授权
- 要求对方提供授权委托书（非法定代表人签署时）
- 核对对方印章（名称与主体一致、印章备案有效）

### 步骤3：签署流转执行
**电子签署流程：**
1. 在电子签约平台创建签署任务
2. 上传最终版合同PDF
3. 设置签署位置和顺序（通常对方先签，我方后签）
4. 发送签署邀请
5. 监控签署进度
6. 双方签署完成→自动生成带时间戳的签署版本

**纸质签署流程：**
1. 打印合同（确认打印版与电子版一致）
2. 我方用印（登记用印台账）
3. 快递至对方（使用可追溯的快递服务）
4. 跟踪对方签署进度
5. 对方签署完成后寄回我方留存联
6. 收到后核对（签章完整、无篡改痕迹）

### 步骤4：签署异常处理
- **签署逾期**：审批通过后5个工作日内未完成签署→发出第一次提醒；10个工作日→升级至业务负责人；15个工作日→升级至法务负责人
- **对方拒签**：记录拒签原因，反馈至业务部门评估是否需要重新谈判
- **签署瑕疵**：发现印章模糊、签名不规范等问题→要求补正
- **合同篡改**：发现签署版本与审批版本不一致→立即终止签署流程并报告

### 步骤5：签署确认与归档
- 确认所有方签署完毕
- 扫描纸质合同生成电子存档件
- 录入签署完成信息（签署日期、签署方式、合同编号）
- 触发后续台账建设和履约监控流程

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| approved_contract | 文件 | 是 | 已审批通过的合同最终版 |
| approval_record | 结构化数据 | 是 | 审批记录（审批人、日期、层级） |
| signing_method | 枚举 | 是 | 签署方式（电子签/纸质签） |
| signing_parties | 列表 | 是 | 签署方信息（名称、代表人、联系方式） |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| signed_contract | 文件 | 签署完成的合同文本 |
| signing_record | 结构化数据 | 签署过程记录（时间、方式、各方确认） |
| signing_status | 枚举 | 签署状态（进行中/已完成/异常） |
| next_actions | 列表 | 后续需要执行的动作 |

## 最佳实践

1. **先对方后我方**：通常让对方先签署，确认对方签署完毕后我方再签，降低风险
2. **留痕可追溯**：签署全过程留下完整记录，支持日后争议时的举证需要
3. **电子签优先**：在对方接受的情况下优先使用电子签署，效率更高且防篡改
4. **时效管控**：设置明确的签署期限，超期未签视为异常并及时升级
5. **原件管理**：纸质原件必须统一保管，复印件盖"副本"章区分

## 约束条件

- 签署前必须确认合同已获得正式审批，未经审批不得发起签署
- 电子签署平台必须具有合法CA认证资质
- 纸质合同原件扫描上传时限为签署完成后24小时内
- 签署过程中不得对合同内容做任何修改（任何修改需回到审批环节）
