# 交易对手风险评估 (Counterparty Risk Assessment)

## 技能概述

本技能用于对合同交易对手进行系统性的信用和法律风险评估，为合同风险评级提供交易对手维度的输入数据。评估范围包括主体资格核验、财务状况分析、诉讼记录审查、历史合作表现和行业声誉等多个方面。

## 适用场景

- 新交易对手的首次合作风险评估
- 已有合作方的年度/续约风险重评
- 交易对手发生重大变化（股权变更、高管变动、被列入失信名单）时的紧急评估
- 大额合同（≥100万）签约前的交易对手深度尽调

## 执行步骤

### 步骤1：主体资格核验
- 工商登记信息核实（企业名称、统一社会信用代码、注册资本、成立日期、经营范围）
- 法定代表人信息确认
- 经营状态确认（正常营业/吊销/注销/迁出）
- 经营异常名录检查
- 严重违法失信企业名单检查
- 实际控制人穿透识别

### 步骤2：财务状况评估
- 注册资本与实缴资本对比
- 近三年营收和利润趋势（如有公开信息）
- 重大资产抵押/质押情况
- 被执行信息（执行金额/数量）
- 税务处罚记录
- 行政处罚记录

### 步骤3：诉讼风险评估
- 作为被告的诉讼案件数量和金额
- 案件类型分析（合同纠纷/劳动争议/侵权/其他）
- 是否存在被限制高消费或列入失信名单
- 涉及的仲裁案件
- 终本案件数量和金额（代表执行不能的风险）

### 步骤4：合作历史回顾（老客户）
- 历史合同履约表现（按时交付率、质量合格率）
- 付款信用（是否有拖欠记录、平均付款周期）
- 合同纠纷/争议记录
- 合作中发现的风险事件

### 步骤5：行业与外部评估
- 行业地位和市场评价
- 是否属于制裁名单/出口管制黑名单
- 是否为关联方（需要关联交易合规审查）
- 母公司/集团信用状况（如为子公司）

### 步骤6：综合评估结论
输出交易对手风险等级（优/良/一般/差）及核心风险标注：
- **优**：主体资格完备、财务健康、无诉讼风险、合作记录良好
- **良**：主体正常、财务一般、少量诉讼但非恶意、合作基本正常
- **一般**：存在一定风险信号需关注、建议加强合同保护条款
- **差**：存在重大信用风险、建议要求担保或拒绝合作

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| company_name | 字符串 | 是 | 交易对手企业名称 |
| credit_code | 字符串 | 否 | 统一社会信用代码 |
| contract_amount | 数字 | 是 | 本次合同金额（万元） |
| cooperation_history | 结构化数据 | 否 | 历史合作记录 |
| assessment_depth | 枚举 | 否 | 评估深度（快速/标准/深度） |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| counterparty_grade | 枚举 | 交易对手风险等级（优/良/一般/差） |
| entity_verification | 结构化数据 | 主体资格核验结果 |
| risk_signals | 列表 | 识别出的风险信号 |
| recommendation | 文本 | 建议措施（如要求保证金、银行保函等） |
| assessment_report | 文本 | 完整评估报告 |

## 最佳实践

1. **多源交叉验证**：不依赖单一数据源，至少交叉验证2-3个信息渠道
2. **动态监控**：重要交易对手应设置持续监控（如诉讼新增提醒）
3. **关注实质风险**：区分"形式上有瑕疵但实质无风险"和"形式合规但实质有风险"
4. **时效性**：评估结果有效期为3个月，超期需重新评估
5. **穿透识别**：对复杂股权结构进行穿透，识别实际控制人的风险

## 约束条件

- 信息收集仅通过合法公开渠道（国家企业信用信息公示系统、裁判文书网、天眼查等）
- 评估结论必须基于可查证的事实，不得依据传闻或未经核实的信息
- 涉及个人隐私信息的收集和使用需符合《个人信息保护法》要求
- 评估报告中的负面信息需标注信息来源和获取时间
