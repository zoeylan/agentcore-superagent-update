# 履约监控 (Performance Monitoring)

## 技能概述

本技能用于对签署生效的合同进行持续的履约进度跟踪和异常预警。通过监控合同约定的关键节点（付款、交付、验收等），及时发现逾期履行和潜在违约风险，确保合同双方按约定履行各自义务，在问题萌芽阶段即进行干预。

## 适用场景

- 日常履约进度的定期检查和状态更新
- 关键节点到期前的提前预警
- 逾期履行的检测和升级处理
- 对方违约行为的识别和记录
- 履约过程中的变更和调整跟踪

## 执行步骤

### 步骤1：监控规则配置
根据合同台账中的关键节点设置监控规则：
- **付款监控**：应付日期前7天→提醒准备付款；到期日→确认是否已付；超期1天→发出预警
- **交付监控**：交付日前14天→确认对方准备进度；交付日→确认交付状态；超期→预警
- **验收监控**：交付完成后→启动验收倒计时；验收期限前7天→提醒完成验收
- **里程碑监控**：里程碑日期前7天→确认进度；到期→确认完成状态
- **到期监控**：合同到期前60天→触发续约/终止评估

### 步骤2：状态采集与更新
定期（每日/每周）获取履约状态：
- 与业务部门确认交付和验收进度
- 与财务部门确认付款和收款状态
- 核查对方的履约表现（交付物是否符合合同要求）
- 记录履约过程中的沟通和变更（会议纪要、邮件确认等）

### 步骤3：异常识别与分级
当检测到履约异常时，按严重程度分级处理：

**一级预警（提示级）**：
- 节点即将到期的正常提醒
- 轻微延迟但在合理范围内（如±3天的付款浮动）
- 处理方式：通知业务部门关注

**二级预警（警告级）**：
- 逾期≤15天，经催告后对方承诺补救
- 交付物存在轻微质量问题但不影响使用
- 处理方式：通知业务部门催告，记录异常

**三级预警（严重级）**：
- 逾期>15天，对方未积极回应
- 交付物存在重大质量问题
- 对方出现经营异常信号（如被列入异常名录）
- 处理方式：升级至法务部门评估违约责任和应对策略

**四级预警（危急级）**：
- 对方明确表示无法履行
- 对方进入破产或被吊销营业执照
- 发现对方欺诈行为
- 处理方式：立即启动紧急响应，法务介入评估法律行动

### 步骤4：预警响应与跟踪
- 发出预警通知（邮件/系统消息/即时通讯）
- 记录预警的响应措施和进展
- 跟踪问题是否得到解决
- 如问题升级则触发下一级处理流程
- 问题解决后更新状态并归档记录

### 步骤5：履约数据分析
定期（月度）生成履约监控报告：
- 在履合同数量和金额统计
- 本月到期节点执行情况
- 异常预警数量和处理结果
- 按交易对手统计履约表现
- 识别系统性风险趋势

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| contract_id | 字符串 | 是 | 合同编号 |
| monitoring_nodes | 列表 | 是 | 需监控的关键节点清单 |
| status_update | 结构化数据 | 否 | 最新的履约状态更新 |
| alert_threshold | 结构化数据 | 否 | 自定义预警阈值 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| monitoring_status | 枚举 | 当前监控状态（正常/预警/严重/危急） |
| alerts | 列表 | 已触发的预警清单（含级别和建议） |
| performance_report | 文本 | 履约状态报告 |
| escalation_required | 布尔 | 是否需要升级处理 |

## 最佳实践

1. **预防胜于补救**：通过提前预警和主动跟进，将问题消灭在萌芽阶段
2. **证据留痕**：所有催告和沟通记录保留书面证据，为可能的违约追责做准备
3. **区分主次**：C/D级风险合同的监控频率和力度应高于A/B级
4. **商业判断**：逾期不等于违约，需结合商业惯例和双方关系综合判断
5. **闭环管理**：每个预警必须有明确的处理结果和关闭记录

## 约束条件

- 监控系统需保证至少每日一次的状态检查频率
- 三级及以上预警必须在识别后2小时内发出通知
- 监控数据不得用于不当目的（如恶意终止合同的借口）
- 履约监控报告每月至少出具一次
