# 法律意见书编写 (Legal Opinion Drafting)

## 技能概述

本技能用于编写正式的合同法律意见书，综合条款审查发现和风险评级结论，以结构化的法律文书格式输出完整的法律分析意见。法律意见书是法务团队向管理层和业务部门传递法律建议的正式载体，要求专业严谨、逻辑清晰、结论明确。

## 适用场景

- 合同审批前的法律意见出具（常规流程必需环节）
- 重大合同（金额≥200万）的专项法律分析
- 复杂交易结构（关联交易、对赌、跨境）的法律可行性论证
- 管理层决策支持（合同纠纷风险评估、诉讼可能性分析）
- 审计和合规检查中的法律意见留档

## 执行步骤

### 步骤1：确定意见书类型和格式
根据使用场景选择适当的文书格式：
- **标准审查意见**：常规合同审批随附，1-3页
- **专项法律分析**：复杂交易或重大金额，3-8页
- **法律可行性论证**：新业务/新模式的法律路径分析，5-15页

### 步骤2：编写基本情况概述
- 合同基本信息（名称、编号、交易方、金额、期限、标的概述）
- 审查背景（因何触发本次法律审查）
- 审查范围和限制声明（审查所依据的信息和文件清单）

### 步骤3：核心法律分析
**3.1 条款审查结论汇总**
- 引用条款审查专家的核心发现（高风险/中风险条款）
- 对关键风险条款进行补充法律分析

**3.2 合规性分析**
- 合同是否违反法律强制性规定
- 是否满足特定交易的合规要求（如招投标程序、国资审批）
- 关联交易合规性（信息披露、回避决议）

**3.3 效力分析**
- 合同成立要件是否满足
- 是否存在无效、可撤销事由
- 附条件/附期限条款的效力判断

**3.4 风险评估**
- 各风险点的发生概率和影响程度评估
- 最坏情形分析（worst case scenario）
- 风险量化（可能的最大损失金额）

### 步骤4：结论与建议
- 明确的结论性意见（建议签署/附条件签署/不建议签署）
- 如建议签署：前提条件和注意事项清单
- 如附条件签署：具体的风险缓释措施要求
- 如不建议签署：主要风险点和替代方案建议

### 步骤5：法律依据附录
- 列出意见书中引用的所有法律法规条文
- 引用的司法解释和指导案例
- 参考的学理观点（如有）

### 步骤6：质量校验
- 确认所有法条引用准确无误（条文号、内容核对）
- 确认分析逻辑链完整无跳跃
- 确认结论有充分的前述分析支撑
- 确认语言表达专业、无歧义

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| risk_grade_result | 结构化数据 | 是 | 风险评级结果 |
| clause_review_findings | 列表 | 是 | 条款审查的风险发现 |
| contract_basic_info | 结构化数据 | 是 | 合同基本信息 |
| opinion_type | 枚举 | 是 | 意见书类型（标准/专项/可行性） |
| special_concerns | 文本 | 否 | 管理层或业务部门的特别关注点 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| legal_opinion | 文本/文件 | 完整的法律意见书文本 |
| opinion_summary | 文本 | 意见摘要（一段话概括核心结论） |
| action_items | 列表 | 需要落实的行动事项清单 |
| legal_references | 列表 | 引用的法律法规清单 |

## 最佳实践

1. **结论前置**：在意见书开头即给出明确结论，详细分析放在后续章节
2. **法条准确**：每个法律观点必须有准确的法条依据，引用到具体条款号
3. **风险量化**：尽可能将法律风险转化为商业语言（可能损失金额、时间成本）
4. **假设明示**：明确声明意见书所基于的假设和限制条件
5. **受众适配**：管理层版本简洁商业化，法务留档版本详细专业化

## 约束条件

- 法律意见书引用法条准确性要求100%，不得模糊引用
- 结论必须明确，不得给出模棱两可的意见
- 意见书出具时效：标准审查意见2个工作日，专项分析5个工作日
- 涉及诉讼风险评估时需声明"仅供参考，不构成诉讼结果预测"
- 法律意见书需经法务负责人审签后方可正式出具
