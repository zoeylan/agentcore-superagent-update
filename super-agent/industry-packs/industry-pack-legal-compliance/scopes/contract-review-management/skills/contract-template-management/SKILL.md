# 合同模板管理 (Contract Template Management)

## 技能概述

本技能用于管理和维护企业标准合同模板库，包括模板的版本控制、适用范围匹配、更新迭代和质量维护。标准模板是合同起草效率和质量的基石，确保企业在常规交易中使用经过法律审查的规范化合同文本，减少重复审查工作量。

## 适用场景

- 根据业务需求匹配最适合的标准合同模板
- 新增合同模板（新业务类型需要新模板）
- 更新现有模板（法规变更、审查反馈驱动的修订）
- 模板使用情况分析和优化建议
- 模板版本管理和废弃模板标记

## 执行步骤

### 步骤1：模板分类体系维护
按照以下维度建立模板分类索引：
- **交易类型**：采购、销售、技术开发、委托服务、租赁、投资、劳动、保密等
- **合同金额档位**：<50万标准版、50-200万加强版、>200万高规格版
- **适用主体**：与国企合作、与外企合作、与个人合作
- **行业适配**：通用版、制造业版、互联网版、金融版
- **地域特征**：境内合同、涉港澳台合同、跨境合同

### 步骤2：模板匹配与推荐
根据业务部门提交的需求要素，执行匹配逻辑：
1. 确定交易类型 → 筛选对应类型模板集
2. 确定合同金额 → 选择对应规格版本
3. 确定交易对手特征 → 调整适配版本
4. 确定特殊需求 → 标注需要定制的部分
5. 输出匹配结果（含匹配度评分和使用建议）

### 步骤3：模板更新流程
- 触发条件：法规更新、审查高频修改反馈、业务模式变化、管理层要求
- 更新流程：起草修订→法务审核→法务总监审批→发布新版→废弃旧版
- 版本控制：主版本号.次版本号（如V2.3），记录每次修订内容和原因
- 通知机制：模板更新后通知所有使用该模板的业务部门

### 步骤4：模板质量评估
定期（季度）评估模板质量指标：
- 模板使用率（被选用次数/同类合同总数）
- 模板偏离率（使用模板但需要大量修改的比例）
- 审查通过率（使用模板生成的合同一次审查通过的比例）
- 纠纷关联率（使用该模板的合同发生纠纷的比例）

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| business_requirement | 结构化数据 | 是 | 业务需求要素（交易类型、金额、对手等） |
| operation_type | 枚举 | 是 | 操作类型（匹配/新增/更新/查询） |
| update_reason | 文本 | 否 | 模板更新原因（更新操作时必填） |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| matched_templates | 列表 | 匹配的模板（含匹配度、版本号、适用说明） |
| customization_notes | 文本 | 使用该模板时需要定制的部分标注 |
| template_metadata | 结构化数据 | 模板元数据（版本、更新日期、审批状态等） |

## 最佳实践

1. **80/20原则**：确保标准模板覆盖80%以上的常规合同需求
2. **渐进式更新**：避免大规模重写模板，采用增量修订方式
3. **审查反馈闭环**：将审查专家的高频修改意见定期回馈到模板更新
4. **多级模板**：同一类型合同按复杂度/金额分设多级模板，避免小合同使用过重模板
5. **条款模块化**：将通用条款（保密、争议解决、不可抗力）模块化，支持灵活组合

## 约束条件

- 模板更新必须经过法务审核和审批，不得未经审批直接发布
- 废弃模板需标记废弃原因和替代模板，不得直接删除（历史合同可能引用）
- 模板中的占位符必须使用统一标记格式（如【甲方名称】），便于自动填充
- 涉及重大法规变更的模板更新需在法规生效前完成
