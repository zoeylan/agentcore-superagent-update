# 条款定制化 (Clause Customization)

## 技能概述

本技能专注于处理超出标准模板范围的非标准条款定制需求。当业务场景的特殊性使标准模板无法满足要求时，需要针对特定交易结构、行业惯例或商业诉求，从零开始设计和编写合同条款。此技能要求深厚的法律功底和对商业逻辑的准确把握。

## 适用场景

- 新业务模式下的创新交易结构条款设计（如SaaS服务条款、数据授权条款）
- 对赌协议、回购条款等复杂投资协议条款
- 跨境交易中的特殊条款（如FCPA反腐败承诺、数据跨境条款）
- 技术开发合同中的源代码托管和知识产权分阶段归属条款
- 业务部门提出的创新性商业安排的法律表达

## 执行步骤

### 步骤1：需求深度理解
- 与业务部门沟通确认定制需求的商业目的
- 分析需求的法律本质（创设权利、设定义务、设置条件、风险分配）
- 确认该需求在法律上的可实现性（是否与法律强制性规定冲突）
- 调研类似交易中的行业惯例做法

### 步骤2：条款结构设计
- 确定条款的逻辑结构（主规则→例外→例外的例外）
- 设计触发条件和法律后果的对应关系
- 考虑正常场景和异常场景的处理
- 设置兜底条款和争议处理机制

### 步骤3：条款文本编写
- 使用精确的法律术语，关键概念需在合同定义条款中明确界定
- 条件关系使用规范的法律逻辑表达（"如...则..."、"除非...否则..."）
- 确保主语明确（"甲方应..."、"乙方有权..."）
- 量化标准具体明确（时间、金额、比例、次数等避免模糊表达）
- 权利义务对等考量（单方权利条款需有合理基础）

### 步骤4：合规性验证
- 检查条款是否违反法律强制性规定
- 确认格式条款是否满足《民法典》第496条的提示说明义务
- 评估条款被认定为无效/可撤销的风险
- 检查与合同其他条款的一致性

### 步骤5：风险标注
- 标注该非标条款的风险等级
- 说明潜在的法律风险和应对措施
- 标记需要条款审查专家重点审查的方面
- 如涉及对方利益重大让步，标注谈判难度预期

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| customization_need | 文本 | 是 | 业务部门的定制需求描述 |
| business_purpose | 文本 | 是 | 定制条款的商业目的 |
| contract_context | 文本 | 是 | 该条款所在合同的整体背景 |
| reference_cases | 列表 | 否 | 参考的类似条款或案例 |
| constraint_conditions | 文本 | 否 | 定制条款的约束条件 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| custom_clause_text | 文本 | 定制条款完整文本 |
| design_rationale | 文本 | 条款设计思路和法律依据 |
| risk_annotation | 结构化数据 | 风险标注（等级、描述、缓释建议） |
| integration_guide | 文本 | 该条款在合同中的位置建议和衔接说明 |

## 最佳实践

1. **目的导向**：始终围绕商业目的设计条款，避免过度法律化导致偏离业务需求
2. **可执行性优先**：条款必须具备可操作性，避免写出"正确但无法执行"的条款
3. **参考先例**：优先参考经过法院或仲裁确认有效的类似条款表述
4. **风险对冲**：高风险定制条款配套设计保护性条款（如对赌配套回购）
5. **简洁明了**：避免过长复杂的句式，一个条款只表达一个完整的法律关系

## 约束条件

- 定制条款不得违反法律法规的强制性规定
- 涉及对方重大权利限制的条款需评估格式条款风险
- 定制条款须明确标注为非标条款，提醒审查专家重点关注
- 首次使用的创新条款需附具设计说明备查
