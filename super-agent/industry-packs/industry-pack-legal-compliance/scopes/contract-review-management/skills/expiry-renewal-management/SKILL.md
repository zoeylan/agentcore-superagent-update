# 到期续约管理 (Expiry & Renewal Management)

## 技能概述

本技能用于管理合同到期前的续约/终止决策流程，以及合同终止后的归档保存。确保每份到期合同都经过充分评估并做出明确的处理决定（续约/终止/不续约），避免合同默认续期带来的非预期义务，同时保证归档合规。

## 适用场景

- 合同到期前60天的续约评估启动
- 自动续约合同的终止窗口期管理
- 续约谈判中的条件评估
- 合同终止后的尾款结算和资料退还
- 合同档案的长期保存和到期销毁

## 执行步骤

### 步骤1：到期预警触发
- 合同到期前60天，系统自动生成到期预警
- 预警信息发送至：业务负责人、法务对接人、合同管理员
- 预警内容包含：合同基本信息、履约表现回顾、续约/终止建议

### 步骤2：续约评估信息收集
收集续约决策所需信息：
- **业务需求评估**：业务部门是否仍有合作需求
- **履约表现回顾**：对方在本合同期内的履约评分
- **市场对标**：当前市场条件与原合同条件的对比
- **风险变化评估**：交易对手信用状况是否有变化
- **法规变化影响**：是否有新法规要求调整合同条款

### 步骤3：续约/终止决策
根据评估信息形成建议方案：

**建议续约（满足以下条件）：**
- 业务需求持续存在
- 对方履约表现良好（评分≥80分）
- 市场条件未发生重大不利变化
- 无重大法律合规风险

**建议终止（满足以下任一条件）：**
- 业务需求不再存在
- 对方履约表现差（评分<60分或存在重大违约）
- 市场条件重大变化（可获得更优条件的替代方）
- 法律合规风险不可接受

**建议调整后续约（满足以下条件）：**
- 业务需求存在但条件需要更新
- 原合同存在需改进的条款
- 需要根据法规变化更新合规条款

### 步骤4：续约/终止执行

**续约执行：**
1. 业务部门确认续约意向
2. 确定需要调整的条款（如有）
3. 触发合同起草助手启动新一轮合同流程（续约版）
4. 新合同走完整审查和审批流程
5. 在原合同到期前完成新合同签署

**终止执行：**
1. 按合同约定发出不续约/终止通知（注意通知期限要求）
2. 确认双方权利义务是否已全部履行完毕
3. 处理尾款结算（应收/应付确认）
4. 处理资料退还/销毁（保密信息、知识产权材料等）
5. 出具合同终止确认函

### 步骤5：归档管理
合同终止后：
- 确认所有相关文件完整归档（合同文本、审批记录、履约记录、终止文件）
- 标记合同状态为"已终止"并记录终止日期和原因
- 设置归档保存期限：合同终止后10年
- 保存期满前3个月评估是否需要延长保存
- 保存期满且无延长需要→按保密等级执行销毁程序

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| contract_id | 字符串 | 是 | 即将到期的合同编号 |
| expiry_date | 日期 | 是 | 合同到期日期 |
| performance_record | 结构化数据 | 是 | 履约表现数据 |
| business_feedback | 文本 | 是 | 业务部门的续约意向反馈 |
| market_conditions | 文本 | 否 | 当前市场条件信息 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| renewal_recommendation | 枚举 | 续约建议（续约/调整续约/终止） |
| assessment_report | 文本 | 续约评估报告 |
| action_plan | 列表 | 需要执行的行动清单（含时限） |
| archive_status | 枚举 | 归档状态（如为终止） |

## 最佳实践

1. **提前启动**：60天提前量确保有足够时间完成评估和谈判
2. **避免被动续约**：对含自动续约条款的合同特别关注终止窗口期
3. **成本效益分析**：续约决策不仅看价格，还要考虑切换成本和关系价值
4. **平滑过渡**：决定终止时确保有替代方案，避免业务中断
5. **完整闭环**：合同终止不等于管理结束，归档和保存期管理同样重要

## 约束条件

- 到期预警必须在到期前60天触发，不得遗漏
- 终止通知必须在合同约定的通知期限前发出（通常为30天）
- 合同归档保存期限为终止后10年，不得提前销毁
- 涉及商业秘密的合同，终止后需确认保密条款存续
- 续约合同必须走完整的审查审批流程，不得简化
