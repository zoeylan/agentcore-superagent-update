# 侵权监控

## 概述

侵权监控是知识产权维权的前端环节，通过建立系统化的线上线下监控机制，主动发现可能侵犯企业专利、商标、著作权的行为，为后续的证据保全和维权行动提供及时的线索输入。有效的侵权监控能够将发现侵权到启动应对的时间窗口压缩到最短。

## 适用场景

- 日常侵权巡查（电商平台、市场、展会）
- 新产品上市后的仿冒监控
- 专利侵权的技术方案比对
- 商标侵权和假冒商品监控
- 软件著作权盗版监控
- 竞品新产品的FTO风险反向评估

## 执行步骤

### 步骤1：监控范围与策略设定
- 确定需要重点保护的IP资产清单：
  - 核心专利（TOP20，按商业价值排序）
  - 注册商标（所有有效商标，重点关注知名品牌）
  - 主要软件著作权
- 设定监控渠道：
  - 线上：淘宝/天猫、京东、拼多多、1688、亚马逊、独立站
  - 技术：竞品产品拆解、技术文档分析、专利引用追踪
  - 线下：行业展会、区域市场、经销商反馈
  - 舆情：社交媒体、行业论坛、新闻报道
- 设定监控频率：线上每周/线下每月/技术按竞品发布节奏

### 步骤2：线上侵权监控
- **商标侵权监控**：
  - 电商平台关键词搜索（品牌名+常见拼写变体）
  - 图片反向搜索（商标图样、产品外观）
  - 店铺名称包含品牌名的排查
  - 广告投放中使用品牌关键词的监控
- **专利侵权监控**：
  - 竞品新产品的技术参数与我方专利权利要求比对
  - 行业新品发布会/展会上展示的技术方案分析
  - 学术论文和技术博客中的技术方案追踪
- **著作权侵权监控**：
  - 应用商店中的软件抄袭（功能/界面/代码相似性）
  - 网络平台上的内容盗用

### 步骤3：初步侵权判定
- **专利侵权判定**（全面覆盖原则）：
  - 提取我方专利独立权利要求的全部技术特征
  - 对照疑似侵权产品/方法的对应技术特征
  - 逐一比对：相同特征标记"√"，等同特征标记"≈"，缺少特征标记"×"
  - 全部特征被覆盖（相同或等同）→初步判定侵权
  - 缺少任一必要技术特征→不构成字面侵权（但需评估等同侵权）
- **商标侵权判定**：
  - 使用相同/近似标识
  - 在相同/类似商品/服务上
  - 容易导致混淆
- **判定结论分级**：高度疑似侵权/可能侵权/存疑/不构成侵权

### 步骤4：线索记录与上报
- 对每条侵权线索建立记录卡：
  - 发现时间、发现渠道、侵权方信息
  - 涉嫌侵权的产品/行为描述
  - 对应的我方IP权利编号
  - 初步判定结论和依据
  - 证据截图/链接（临时性证据保全）
- 按严重程度分级上报：
  - 紧急（大规模/知名平台/核心专利）→立即上报，24小时内启动证据保全
  - 重要（中等规模/一般渠道）→48小时内上报
  - 一般（小规模/个别行为）→周度汇总上报

### 步骤5：监控效果评估
- 月度统计：发现侵权线索数、确认侵权数、误判率
- 渠道效果：各监控渠道的线索产出比
- 响应时效：从发现到上报的平均时间
- 覆盖率评估：是否存在监控盲区
- 持续优化监控策略和关键词

## 输入

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 重点保护IP资产清单 | 需要监控的专利/商标/著作权列表 | 是 |
| 监控关键词 | 品牌名、产品名、技术术语 | 是 |
| 竞品/可疑企业名单 | 重点关注对象 | 否 |
| 历史侵权记录 | 已知的侵权方和侵权模式 | 否 |

## 输出

| 输出项 | 说明 |
|--------|------|
| 侵权线索报告 | 新发现的疑似侵权信息 |
| 初步侵权判定意见 | 技术比对结论 |
| 监控月度统计 | 各渠道发现情况汇总 |
| 预警通知 | 紧急侵权事件的即时通知 |

## 最佳实践

1. **全面覆盖**：线上+线下+技术多渠道并行，避免监控盲区
2. **持续迭代**：根据侵权方的规避手段更新监控策略
3. **证据及时性**：线上侵权信息可能随时被删除，发现后立即临时保全
4. **误判控制**：初步判定需要谨慎，避免错误指控影响商业关系
5. **技术手段**：善用图片识别、爬虫工具等技术手段提高效率
6. **内部协同**：建立研发、销售、客服等部门的线索上报机制

## 约束条件

- 核心资产的线上监控频率不低于每周一次
- 发现紧急侵权线索后24小时内完成上报
- 初步判定准确率目标≥85%
- 监控手段必须合法合规（不得黑客入侵、不得侵犯隐私）
- 所有监控记录保留至少3年备查
