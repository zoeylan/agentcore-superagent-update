# 异常行为检测分析 (Anomaly Detection Analysis)

## 概述

本技能用于对企业内部数据源进行持续扫描和模式分析，识别可能指示合规违规的异常行为信号。采用基于规则的检测与统计异常检测相结合的方法，覆盖反商业贿赂、数据合规、反洗钱等多个合规领域的异常模式识别。

## 适用场景

- 费用报销系统中的异常礼品招待支出检测
- 财务系统中的异常交易模式识别（反洗钱）
- 数据访问日志中的越权访问和异常行为
- 供应商/客户与制裁名单的关联比对
- 利益冲突场景的关联关系发现
- 合同执行中的异常模式（如频繁变更、超标采购）

## 执行步骤

### 步骤1：数据源接入与预处理
- 确定监控数据源：费用报销系统、ERP交易记录、CRM操作日志、数据访问日志、HR系统（利益冲突声明）、邮件系统（关键词预警）
- 数据标准化处理：统一时间格式、金额币种换算、实体名称标准化
- 建立基线数据模型（正常行为基线）

### 步骤2：规则引擎检测
- 反商业贿赂规则：
  - 单次礼品/招待支出 ≥ 500元（触发审批确认）
  - 同一对象累计季度支出 ≥ 2000元（触发关注）
  - 敏感时间点支出（招标期间、审批期间）
  - 现金等价物（购物卡、礼品卡）支出
- 反洗钱规则：
  - 单笔交易 ≥ 大额交易报告标准
  - 短期内频繁小额交易（拆分交易嫌疑）
  - 与高风险国家/地区的资金往来
  - 新开户后短期大额交易
- 数据合规规则：
  - 非工作时间的敏感数据批量访问
  - 超出权限范围的数据导出
  - 未经审批的跨境数据传输
  - 离职预告期内的异常数据操作

### 步骤3：统计异常检测
- 计算个体行为偏离其历史均值的程度（Z-score）
- 识别群组异常（某部门整体偏离公司平均水平）
- 时序分析（突变检测、趋势异常）
- 关联分析（多个低级别异常的组合可能指示高级别风险）

### 步骤4：预警生成与分级
- 预警等级判定：
  - 紧急：明确违反强制性法规（如制裁名单交易），需48小时内处置
  - 高：高置信度异常模式，需5工作日内调查
  - 中：需进一步确认的可疑行为，15工作日内处置
  - 低：偏离基线但可能有合理解释，纳入观察项
- 预警信息结构化输出（含证据链）

### 步骤5：误报管理与模型优化
- 记录预警处置结果（确认违规/误报/存疑）
- 分析误报原因并调整检测规则阈值
- 补充白名单（已确认合规的特殊业务场景）
- 定期回测检测模型的准确率和召回率

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 交易/行为数据 | 结构化数据记录 | 是 |
| 检测规则集 | 规则定义和阈值参数 | 是 |
| 制裁/黑名单 | 外部制裁名单数据 | 是 |
| 历史基线数据 | 正常行为模式基线 | 是 |
| 白名单 | 已确认合规的例外情况 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 风险预警通知 | 结构化消息 | 含预警编号、等级、证据、建议 |
| 异常行为分析报告 | 文档 | 详细的异常模式分析 |
| 监控仪表板数据 | 数据集 | 预警数量、分布、趋势 |
| 模型效果报告 | 统计报告 | 准确率、误报率、覆盖率 |

## 最佳实践

1. **分层检测**：先用低成本规则引擎过滤明确违规，再用统计模型检测隐蔽异常
2. **持续校准**：基于调查结果定期调整阈值，平衡灵敏度和误报率
3. **隐私保护**：监控过程本身须合规，遵循最小必要原则，不过度监控员工
4. **证据链完整**：预警输出应包含可追溯的数据证据，便于后续调查
5. **关联分析**：单一弱信号可能无意义，多个弱信号的组合可能是强风险信号

## 约束条件

- 检测结果为"疑似"而非"定性"，最终判定需人工调查确认
- 监控数据的获取和使用必须有合法授权基础
- 不得将监控结果直接用于员工处分，需经正式调查程序
- 涉及个人隐私的监控须进行合法性评估并告知员工
- 预警推送范围严格按需知原则控制
