# 合规制度编写 (Compliance Policy Drafting)

## 概述

本技能用于编写和修订企业合规管理制度文件，将法律法规要求和行业最佳实践转化为企业内部可执行的管理制度。确保制度语言明确、可操作、可衡量，覆盖从总纲到操作指引的三级架构。

## 适用场景

- 新合规制度的起草（新法规要求或新业务需求）
- 现有制度的定期修订（法规变更、审计发现、业务变化）
- 操作指引的编写（将制度细化为具体操作步骤）
- 制度框架的调整（组织架构变化或治理模式升级）
- 制度间一致性的检查和修正

## 执行步骤

### 步骤1：需求分析与调研
- 明确制度编写/修订的触发原因（法规变更/审计发现/业务需求/管理层要求）
- 梳理适用的法律法规和监管要求
- 研究行业标杆企业的实践（对标分析）
- 了解企业当前业务流程和管理现状
- 征询相关部门意见（合规需求和执行可行性）

### 步骤2：制度结构设计
- 确定制度层级：总纲/专项制度/操作指引
- 设计制度框架（标准章节结构）：
  1. 总则（目的、依据、适用范围、基本原则）
  2. 定义与术语
  3. 组织与职责（各层级角色和责任）
  4. 管理要求（实质性条款）
  5. 流程规定（操作步骤和审批流程）
  6. 监督与考核
  7. 违规处理
  8. 附则（生效日期、解释权、修订记录）
- 确定与其他制度的引用关系和层级逻辑

### 步骤3：条款编写
- 将法律义务转化为内部管理要求：
  - 使用"应当""不得""必须"等明确义务性表述
  - 量化标准（如"500元""3个工作日""100%覆盖"）
  - 明确责任主体（具体到部门/岗位）
  - 规定违反后果
- 确保条款SMART化：具体(Specific)、可衡量(Measurable)、可实现(Achievable)、相关(Relevant)、有时限(Time-bound)
- 避免"原则上""一般情况下""视情况"等模糊表述

### 步骤4：征求意见与修改
- 向相关部门发放征求意见稿
- 收集反馈并逐条回应
- 平衡管控要求与业务效率
- 法务审核条款合法性和表述准确性
- 形成送审稿

### 步骤5：审批发布
- 按制度管理流程提交审批
- 审批通过后正式发布（含版本号、生效日期）
- 制定宣贯计划（配合培训督导）
- 归档并更新制度清单/知识库
- 设定下次复审日期

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 触发原因 | 为何编写/修订 | 是 |
| 法规依据 | 适用的法律法规条款 | 是 |
| 业务流程 | 相关业务的运作方式 | 是 |
| 现有制度 | 当前版本（修订时） | 修订时必需 |
| 标杆参考 | 行业最佳实践资料 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 制度正文 | 正式文档（含版本号） | 完整的制度文件 |
| 修订对照表 | 表格 | 新旧条款对比（修订时） |
| 条款说明书 | 文档 | 关键条款的背景解释 |
| 宣贯方案 | 计划表 | 培训和宣传安排 |
| 合规映射表 | 矩阵 | 法规→制度条款对照 |

## 最佳实践

1. **用户视角**：制度的最终读者是一线员工，确保表述清晰易懂
2. **可执行性优先**：宁可要一个执行到位的简单规则，不要一个无法落地的完美制度
3. **风险导向**：高风险领域的制度应更详细具体，低风险领域可适当概括
4. **一致性检查**：新制度发布前检查与现有制度体系的一致性
5. **生命周期管理**：制度至少每两年复审一次，确保与时俱进

## 约束条件

- 制度不得与上位法（法律法规）冲突
- 制度修订不得降低原有的合规管控标准（除非风险已消除）
- 重大制度变更需经合规委员会审批
- 制度条款不得侵犯员工合法权益
- 涉及跨部门的制度须获得所有相关部门的会签
