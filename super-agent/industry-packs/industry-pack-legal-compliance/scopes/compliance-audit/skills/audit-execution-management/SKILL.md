# 审计执行管理 (Audit Execution Management)

## 概述

本技能涵盖合规审计从通知下达到现场执行完成的全过程管理，包括审计准备、资料收集、现场检查、证据固定和问题确认等关键环节。确保审计工作按照专业标准高质量完成。

## 适用场景

- 定期合规审计的现场执行
- 专项审计（风险预警触发或管理层指令）
- 跟踪审计（验证前次审计整改情况）
- 远程审计（非现场文档审阅和数据分析）
- 监管检查的内部预演（模拟检查）

## 执行步骤

### 步骤1：审计准备（审计通知前）
- 编制具体审计方案：明确审计目标、范围、方法、时间安排
- 设计审计检查清单：按合规领域细化到具体检查要点
- 准备资料需求清单：列明需被审计部门提供的文件/数据
- 组建审计团队：确认人员分工和时间安排
- 了解被审计领域背景：上次审计发现、风险扫描预警、最新法规要求

### 步骤2：审计通知下达
- 提前5个工作日向被审计部门下达正式审计通知
- 通知内容：审计目的、范围、时间安排、配合要求、资料清单
- 明确联络人和沟通机制
- 强调审计纪律和保密要求

### 步骤3：资料收集与审阅
- 被审计部门3个工作日内按清单提供资料
- 对提交资料进行完整性检查
- 文档审阅：制度文件、流程记录、审批记录、培训记录等
- 数据分析：交易数据、日志数据、统计报表等
- 识别需要进一步现场核实的疑点

### 步骤4：现场检查执行
- 人员访谈：关键岗位人员的合规意识和制度知晓度
- 控制测试：模拟业务场景测试控制措施是否按设计运行
- 穿行测试：选取若干交易从头到尾追踪流程执行情况
- 抽样检查：按统计抽样方法选取样本进行详细核查
- 实地观察：办公环境、系统操作、日常工作流程

### 步骤5：证据固定与管理
- 所有审计发现必须有书面证据支撑
- 证据分类：文件证据、数据证据、访谈记录、观察记录、外部确认
- 证据编号并建立索引
- 确保证据的充分性（足够证明结论）、相关性（与审计目标相关）、可靠性（来源可信）
- 工作底稿整理：审计程序→执行结果→证据引用→初步结论

### 步骤6：问题确认与沟通
- 将初步发现与被审计部门进行事实确认
- 允许被审计部门解释和申辩
- 区分事实问题和判断问题
- 对有争议的发现进行补充调查
- 达成事实共识（意见分歧应记录并在报告中反映）

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 审计方案 | 目标、范围、方法 | 是 |
| 检查清单 | 逐项检查要点 | 是 |
| 被审计资料 | 部门提交的文件数据 | 是 |
| 前次审计记录 | 历史发现和整改情况 | 否 |
| 风险预警信息 | 扫描引擎的预警详情 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 审计工作底稿 | 结构化文档集 | 完整的审计执行记录 |
| 证据文件包 | 编号档案 | 支撑发现的全部证据 |
| 初步发现清单 | 表格 | 待确认的问题列表 |
| 检查清单（已完成） | 填写完毕的清单 | 每项标注结果 |
| 问题确认记录 | 会议纪要 | 与被审计部门的确认结果 |

## 最佳实践

1. **风险导向**：将审计资源集中在高风险区域，而非均匀分配
2. **证据为王**：任何发现没有证据支撑不得写入报告
3. **充分沟通**：过程中保持与被审计部门的专业沟通，避免对抗
4. **灵活调整**：审计过程中发现新风险线索应适当扩展范围
5. **及时记录**：现场发现当天整理工作底稿，避免事后遗忘

## 约束条件

- 审计人员不得干预被审计部门的正常业务运行
- 访谈记录应经受访人确认签字
- 审计期间获取的信息严格保密
- 审计结论必须基于证据，不得凭印象或推测
- 超出审计范围的重大发现应及时报告但不自行扩大调查
