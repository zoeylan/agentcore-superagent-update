# 法规变更监控 (Regulatory Change Monitoring)

## 概述

本技能用于持续追踪法律法规、监管政策、行业标准的颁布和修订动态，评估变更对企业合规体系的影响，并生成结构化的法规变更影响报告。适用于合规风险扫描引擎在日常监控和专项法规追踪场景中使用。

## 适用场景

- 新法律法规颁布后的企业影响评估
- 现有法规修订后的制度差距分析
- 监管政策文件（征求意见稿、正式稿）的追踪
- 行业标准更新后的合规要求变化识别
- 跨法域法规变更的协同影响分析

## 执行步骤

### 步骤1：法规信息采集
- 监控官方法规发布渠道（全国人大、国务院、各部委官网、地方政府）
- 追踪监管机构公告（市场监管总局、网信办、央行、证监会、银保监会等）
- 关注行业协会标准发布和修订
- 订阅国际法规动态（GDPR更新、OFAC制裁名单、BIS出口管制清单等）

### 步骤2：变更内容解析
- 提取法规核心变化点（新增条款、修订条款、废止条款）
- 识别关键合规义务变化（新增义务、义务加重、时限变更、处罚加重）
- 标注生效日期和过渡期安排
- 整理配套实施细则和解释性文件

### 步骤3：企业影响评估
- 将法规变更映射到企业现有制度和业务流程
- 评估影响等级：高（需立即修改制度/流程）、中（需在过渡期内调整）、低（仅需知晓，暂无实质影响）
- 识别受影响的部门和岗位
- 评估不合规风险（行政处罚、刑事风险、商业损失）

### 步骤4：输出法规变更报告
- 法规基本信息（名称、发文机关、文号、生效日期）
- 核心变化摘要（对比新旧规定）
- 企业影响评估结论（影响等级+具体影响点）
- 需修订的内部制度清单
- 建议响应时间表和责任分工
- 后续跟踪事项

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 法规信息源 | 官方发布渠道URL或法规全文 | 是 |
| 企业制度清单 | 现有合规制度目录及版本 | 是 |
| 业务范围描述 | 企业业务类型和涉及行业 | 是 |
| 关注领域 | 重点监控的合规领域标签 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 法规变更监控报告 | 结构化文档 | 包含变更摘要、影响评估、行动建议 |
| 影响等级标签 | 高/中/低 | 用于触发不同响应流程 |
| 制度修订建议清单 | 列表 | 需更新的制度文件及修订要点 |
| 响应时间表 | 甘特图/时间轴 | 各项修订任务的时间安排 |

## 最佳实践

1. **时效性优先**：高影响法规变更应在发布后24小时内完成初步评估
2. **全面性覆盖**：不仅关注正式法规，还需追踪征求意见稿（预判立法趋势）
3. **关联性分析**：评估单一法规变更时，考虑与其他法规的交叉影响
4. **历史对比**：维护法规版本变更历史，便于追溯制度修订依据
5. **30天响应标准**：法规变更后30天内必须完成影响评估和制度修订方案

## 约束条件

- 不得对法规条文做超出字面含义的扩大解释
- 影响评估结论需注明不确定性和前提假设
- 涉及重大法规变更（可能导致业务模式调整），必须标记为需法务总监确认
- 国际法规变更需考虑中国法域的适用性和冲突解决
