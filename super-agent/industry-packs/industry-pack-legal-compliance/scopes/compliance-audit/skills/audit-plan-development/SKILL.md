# 审计计划编制 (Audit Plan Development)

## 概述

本技能用于基于年度合规风险评估结果编制合规审计计划，合理配置审计资源，确定审计频次和优先级。审计计划是合规审计工作的纲领性文件，指导全年审计活动的执行。

## 适用场景

- 年度合规审计计划编制（每年Q1完成）
- 专项审计计划制定（风险预警触发）
- 审计计划年中调整（重大风险变化时）
- 新业务/新法规导致的审计范围扩展
- 监管要求的强制性审计安排

## 执行步骤

### 步骤1：输入信息收集
- 获取最新年度合规风险评估报告及风险登记册
- 回顾上年度审计发现及整改情况（未关闭项优先）
- 了解管理层关注点和合规委员会指示
- 确认可用审计资源（人员、时间、预算）
- 收集外部环境信息（监管重点、行业处罚趋势）

### 步骤2：审计范围确定
- 基于风险等级确定审计频次：
  - 高风险领域：半年审计一次
  - 中风险领域：年度审计一次
  - 低风险领域：两年审计一次（轮换制）
- 确定审计类型：全面审计 vs 专项审计 vs 跟踪审计
- 明确每次审计覆盖的合规领域和部门
- 识别强制性审计要求（监管要求、管理层指令）

### 步骤3：审计方案设计
- 每个审计项目的审计目标和关键关注点
- 审计方法选择（文档审阅、人员访谈、控制测试、抽样检查、数据分析）
- 时间安排（开始日期、持续时间、里程碑）
- 资源分配（审计负责人、团队成员、外部支持需求）
- 被审计部门的配合要求

### 步骤4：计划审批与沟通
- 编制年度审计计划书（含季度分解）
- 提交合规委员会/管理层审批
- 与被审计部门提前沟通年度安排（仅告知时间窗口，不透露具体重点）
- 建立计划执行监控机制

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 风险评估报告 | 年度合规风险评估结论 | 是 |
| 上年审计结果 | 发现事项及整改状态 | 是 |
| 可用资源 | 人员、时间、预算 | 是 |
| 管理层指示 | 特别关注领域 | 否 |
| 监管要求 | 强制性审计义务 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 年度审计计划书 | 正式文档 | 含目标、范围、时间、资源 |
| 季度审计排期表 | 甘特图/表格 | 具体审计项目的时间安排 |
| 审计资源分配表 | 矩阵 | 人员与项目的对应 |
| 审计通知模板 | 文档模板 | 标准化审计通知格式 |

## 最佳实践

1. **风险驱动**：审计资源向高风险领域倾斜，避免平均主义
2. **灵活性预留**：年度计划预留15-20%资源用于计划外专项审计
3. **周期轮换**：低风险领域实行轮换审计，确保3年内全覆盖
4. **提前沟通**：让被审计部门了解年度安排，但保持审计重点的独立性
5. **持续监控**：每季度回顾计划执行进度，必要时调整

## 约束条件

- 审计计划的制定必须独立于被审计部门的干预
- 高风险领域的审计安排不得推迟或取消（除非风险等级经正式评估下调）
- 计划调整须经合规委员会批准
- 审计时间安排应避免与被审计部门业务高峰期严重冲突
