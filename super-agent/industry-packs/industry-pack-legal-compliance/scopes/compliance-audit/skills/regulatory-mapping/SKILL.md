# 法规映射与合规义务管理 (Regulatory Mapping)

## 概述

本技能用于建立和维护法律法规到企业内部制度/流程的映射关系，确保每一项外部合规义务都有对应的内部控制措施。通过合规义务登记册和映射矩阵，实现对合规覆盖度的可视化管理和差距识别。

## 适用场景

- 构建企业合规义务全景图
- 新法规发布后的义务提取和映射
- 合规制度覆盖度差距分析
- 监管检查前的自查准备（快速定位对应制度）
- 多法域合规义务的整合管理
- 合规体系认证（ISO 37301）的文档准备

## 执行步骤

### 步骤1：合规义务识别
- 梳理企业适用的法律法规清单（按领域分类）
- 逐条提取法规中的合规义务（"应当""必须""不得"条款）
- 识别义务类型：强制性义务、禁止性义务、许可性义务
- 标注义务的约束对象（企业整体/特定部门/特定岗位/特定行为）
- 记录违反后果（行政处罚/民事责任/刑事责任）

### 步骤2：内部控制措施梳理
- 梳理现有合规制度清单（制度名称、版本、覆盖领域）
- 识别各制度中的具体控制措施（制度条款、流程节点、审批机制、系统控制）
- 对控制措施进行分类：预防性控制/检测性控制/纠正性控制
- 评估控制措施的设计有效性

### 步骤3：映射关系建立
- 建立"法规条款→合规义务→内部制度→控制措施"的四层映射
- 使用矩阵格式展示映射关系
- 标注映射状态：
  - 完全覆盖：有明确对应的制度和控制措施
  - 部分覆盖：有制度但控制措施不充分
  - 未覆盖：无对应制度或控制措施（差距）
- 对于多法域义务，标注冲突或重叠情况

### 步骤4：差距分析与建议
- 识别所有未覆盖或部分覆盖的合规义务
- 评估差距的风险程度（结合违反后果和发生可能性）
- 提出覆盖建议：新建制度/修订制度/增加控制措施/系统改造
- 按优先级排序并制定实施路线图

### 步骤5：持续维护
- 法规变更时更新义务清单和映射关系
- 制度修订时更新控制措施映射
- 定期（至少年度）全面复核映射准确性
- 生成合规覆盖率报告

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 适用法规清单 | 企业须遵守的法规目录 | 是 |
| 现有制度清单 | 内部合规制度目录 | 是 |
| 组织架构 | 部门和岗位设置 | 是 |
| 业务描述 | 主要业务类型和流程 | 是 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 合规义务登记册 | 数据库/表格 | 全部合规义务的结构化记录 |
| 法规-制度映射矩阵 | 矩阵表 | 可视化映射关系 |
| 差距分析报告 | 文档 | 未覆盖义务和建议 |
| 合规覆盖率报告 | 统计报告 | 覆盖比例和趋势 |

## 最佳实践

1. **颗粒度适当**：义务拆解不宜过粗（遗漏）或过细（管理成本过高），以可独立验证为标准
2. **动态维护**：映射关系是活文档，需随法规和制度变化即时更新
3. **双向核验**：不仅从法规→制度方向验证覆盖度，也从制度→法规方向验证依据充分性
4. **风险优先**：差距整改按风险程度排序，不追求一次性完美覆盖
5. **工具化**：建议使用数据库或GRC系统管理映射关系，避免纯文档维护的滞后性

## 约束条件

- 法规义务的提取应忠实于条文原意，不做扩大或缩小解释
- 映射结论不构成法律意见，涉及争议条款的解释需法务确认
- 多法域义务冲突时须标注并升级决策，不自行取舍
- 映射工作须覆盖企业所有业务实体和地域
