# 审计报告编写 (Audit Report Writing)

## 概述

本技能用于将审计执行过程中的发现、证据和分析整合为专业的合规审计报告。报告是审计成果的最终体现，需要做到事实准确、分析深入、建议可行，在5个工作日内完成编写。

## 适用场景

- 定期合规审计完成后的报告编写
- 专项审计/调查的结论报告
- 跟踪审计的整改验证报告
- 年度合规报告（全年汇总）
- 向监管机构提交的合规报告

## 执行步骤

### 步骤1：审计发现梳理
- 汇总所有已确认的审计发现
- 按合规领域分类整理
- 区分系统性问题和个别问题
- 确认每项发现的证据充分性
- 排除已通过申辩程序消除的发现

### 步骤2：根本原因分析
- 对每项发现进行根本原因分析（5-Why、鱼骨图等方法）
- 原因分类：制度缺失/制度不清/执行不到位/系统缺陷/人员能力/管理层重视度
- 识别共性原因（多个发现可能源于同一根因）
- 评估原因的可控性和改善可行性

### 步骤3：风险影响评估
- 每项发现标注风险等级：
  - 高风险：可能导致重大经济损失（>100万）、监管处罚或刑事风险
  - 中风险：可能导致一定经济损失（10-100万）或监管关注
  - 低风险：控制缺陷但未造成实质影响
- 评估已造成的实际影响和潜在风险暴露
- 量化评估（尽可能用数字说明风险规模）

### 步骤4：整改建议制定
- 针对每项发现提出具体、可操作的整改建议
- 建议应解决根本原因而非表面症状
- 明确建议类型：制度修订/流程优化/系统改造/人员培训/加强监督
- 建议应考虑实施成本和可行性
- 设定整改时限：高风险立即/中风险15天/低风险30天

### 步骤5：报告编写与审核
- 报告结构（FCIA格式）：
  1. 执行摘要（面向管理层的一页摘要）
  2. 审计概述（目标、范围、方法、时间）
  3. 审计发现详述（按严重程度排序）
  4. 总体评价（合规体系有效性的总体判断）
  5. 整改建议汇总（含时间表和责任人）
  6. 附件（证据索引、检查清单、被审计部门意见）
- 内部质量审核（确保逻辑一致、引用准确）
- 征求被审计部门意见（事实准确性确认，非结论协商）

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 审计工作底稿 | 完整的执行记录 | 是 |
| 证据文件 | 支撑发现的全部证据 | 是 |
| 问题确认记录 | 与被审计方的确认结果 | 是 |
| 被审计方意见 | 申辩和解释内容 | 否 |
| 前次审计报告 | 用于对比和趋势分析 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 审计报告正文 | 正式文档 | FCIA格式的完整报告 |
| 执行摘要 | 一页文档 | 面向管理层的关键信息 |
| 整改跟踪表 | 表格 | 问题编号、建议、责任人、时限 |
| 审计评级 | 等级判定 | 合规/基本合规/不合规 |

## 最佳实践

1. **事实导向**：报告只陈述已证实的事实，不加入主观推测
2. **清晰表达**：避免审计术语堆砌，确保非专业读者能理解
3. **建设性语气**：报告目的是推动改进，而非指责追责
4. **可追溯性**：每项发现必须标注对应的证据编号
5. **时效性**：现场工作结束后5个工作日内出具报告

## 约束条件

- 审计报告一经发出不得修改事实部分（可补充说明）
- 报告分发范围严格按需知原则控制
- 高风险发现不得等报告完成才通报，应在发现时立即上报
- 报告中不得包含可识别举报人身份的信息
- 涉及法律定性的结论需标注"建议法务确认"
