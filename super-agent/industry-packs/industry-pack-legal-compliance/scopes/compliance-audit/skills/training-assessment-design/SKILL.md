# 培训考核设计 (Training Assessment Design)

## 概述

本技能用于设计合规培训的考核评估体系，包括题库建设、考试组织、通过标准设定和补考机制管理。确保培训效果可衡量、可追踪，关键岗位考试通过率达到≥95%的目标。

## 适用场景

- 合规培训课程配套考核设计
- 考核题库的开发和维护
- 关键岗位合规资格认证考试
- 新员工入职合规考核
- 定期复训效果验证
- 合规知识竞赛和演练活动设计

## 执行步骤

### 步骤1：考核目标确定
- 与培训目标对齐：明确考核要验证的知识/技能/态度
- 确定考核层次（参照布鲁姆分类法）：
  - 知识层：记忆合规规则和制度要求
  - 理解层：理解合规要求背后的原理
  - 应用层：能在具体场景中做出正确的合规决策
- 设定通过标准：通识培训≥80分，关键岗位≥90分
- 明确考核与岗位任职资格的关联（不通过的后果）

### 步骤2：题库开发
- 题型设计：
  - 判断题：测试基本规则知晓度
  - 单选题：测试规则理解和辨别能力
  - 多选题：测试综合理解能力
  - 案例分析题：测试场景应用能力（关键岗位）
  - 情景模拟题：测试决策能力（管理层）
- 题库规模：每门课程至少50-100题，随机抽取20-30题组卷
- 难度分级：基础(40%)、中等(40%)、进阶(20%)
- 按合规领域分类标签：反贿赂/数据合规/反洗钱/出口管制/通用

### 步骤3：考试组织方案
- 考试形式：在线考试（主要）、纸质考试（无系统条件时）
- 防作弊措施：随机抽题、乱序选项、限时答题、IP监控
- 考试窗口期：培训完成后7天内完成考试
- 补考安排：首次不通过者15天内安排一次补考
- 二次不通过处理：启动岗位适配评估，安排一对一辅导

### 步骤4：结果分析与反馈
- 个人维度：成绩通知、错题解析、薄弱点提示
- 组织维度：部门通过率对比、易错题统计、知识盲区识别
- 趋势维度：年度通过率变化、重复错误率
- 反馈应用：低通过率领域作为下次培训重点

### 步骤5：题库维护与更新
- 定期分析题目质量（区分度、难度系数）
- 淘汰过时题目（制度修订后）
- 补充新题目（新法规、新案例）
- 确保题库与最新制度同步
- 每年至少更新20%的题目

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 培训课程大纲 | 考核内容范围 | 是 |
| 合规制度文件 | 考核依据 | 是 |
| 合规案例库 | 案例分析题素材 | 是 |
| 岗位合规要求 | 关键岗位的特定要求 | 是 |
| 历史考试数据 | 用于题目质量分析 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 考核题库 | 题库数据 | 分类、分级的题目集 |
| 考试方案 | 文档 | 考试规则和组织方案 |
| 成绩报告 | 统计报告 | 个人和组织维度 |
| 题目质量分析 | 分析报告 | 区分度和难度分析 |
| 薄弱点分析 | 报告 | 组织合规知识盲区 |

## 最佳实践

1. **场景化考核**：多用案例分析和场景模拟，少用死记硬背的知识题
2. **及时反馈**：考试后立即显示错题解析，强化学习效果
3. **差异化标准**：不同岗位风险等级匹配不同考核深度
4. **激励机制**：优秀成绩给予正向激励，避免纯惩罚导向
5. **数据驱动**：基于考试数据优化培训内容，形成闭环

## 约束条件

- 考核内容不得超出培训范围
- 题库严格保密，防止泄题
- 考核结果用于改进培训，不得作为唯一的员工处分依据
- 为特殊情况（出差、病假等）提供补考安排
- 考核记录保存期限不少于3年
