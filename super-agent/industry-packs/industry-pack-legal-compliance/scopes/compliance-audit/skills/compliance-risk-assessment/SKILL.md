# 合规风险评估 (Compliance Risk Assessment)

## 概述

本技能用于对企业面临的合规风险进行系统性识别、分析和评级，生成风险热力图和风险登记册。适用于年度合规风险评估（Q1集中评估）和触发式专项风险评估场景，是合规管理PDCA循环的"识别-评估"环节核心能力。

## 适用场景

- 年度合规风险全面评估（覆盖所有合规领域）
- 新业务/新市场准入前的合规风险评估
- 法规重大变更后的专项影响评估
- 合规事件发生后的系统性风险重新评估
- 并购交易中的合规尽职调查支持

## 执行步骤

### 步骤1：风险识别（Risk Identification）
- 梳理企业面临的合规义务全景（法规清单+监管要求+合同义务+行业标准）
- 按合规领域分类：反商业贿赂、数据合规、反洗钱、出口管制、劳动用工、环保安全、税务合规、行业特定要求
- 识别每个领域的具体风险事件（Risk Event）和风险场景
- 考虑固有风险（Inherent Risk）和剩余风险（Residual Risk）

### 步骤2：风险分析（Risk Analysis）
- 评估风险发生可能性（Likelihood）：5级评分（极低/低/中/高/极高）
  - 参考因素：历史发生频率、控制措施有效性、外部环境变化、行业通常水平
- 评估风险影响程度（Impact）：5级评分（轻微/较小/中等/重大/严重）
  - 影响维度：经济损失、监管处罚、声誉损害、业务中断、刑事风险
- 计算风险等级 = 可能性 × 影响程度（25分矩阵）

### 步骤3：风险评级与排序（Risk Rating）
- 按风险得分划分等级：
  - 高风险（16-25分）：需要立即关注，半年审计一次
  - 中风险（9-15分）：需要监控和控制，年度审计一次
  - 低风险（1-8分）：维持现有控制，两年审计一次
- 生成风险热力图（横轴为影响，纵轴为可能性）
- 编制风险登记册（Risk Register）

### 步骤4：控制措施评估
- 识别现有控制措施（制度、流程、系统控制、人员）
- 评估控制有效性（有效/部分有效/无效）
- 识别控制缺口（无控制措施或控制失效的风险点）
- 提出控制改进建议和优先级

### 步骤5：输出风险评估报告
- 执行摘要（关键发现和建议）
- 风险热力图和TOP 10风险清单
- 各领域风险详细分析
- 控制措施评估结论
- 年度审计计划建议（基于风险等级分配审计资源）
- 风险监控指标（KRI）建议

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 企业业务描述 | 行业、规模、地域、业务模式 | 是 |
| 法规义务清单 | 适用的法规和监管要求 | 是 |
| 历史合规事件 | 过去发生的违规事件记录 | 是 |
| 现有控制措施 | 制度、流程、系统控制清单 | 是 |
| 外部环境信息 | 监管趋势、行业事件、处罚案例 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 风险评估报告 | 结构化文档 | 包含完整的风险评估结论 |
| 风险热力图 | 矩阵图 | 可视化展示风险分布 |
| 风险登记册 | 表格 | 所有已识别风险的详细记录 |
| 审计计划建议 | 计划表 | 基于风险的审计资源配置 |
| KRI指标清单 | 列表 | 关键风险指标及阈值 |

## 最佳实践

1. **全员参与**：风险评估过程应征询业务部门意见，避免纯合规视角的盲区
2. **动态更新**：风险评估不是年度一次性活动，重大变化应触发即时更新
3. **证据导向**：风险评级应基于数据和事实，避免主观臆断
4. **对标外部**：参考同行业公开处罚案例校准风险评级
5. **管理层参与**：高风险项目需管理层确认并签署风险接受或缓释决策

## 约束条件

- 风险评估结果不构成法律意见，不替代专业律师判断
- 评估覆盖范围应与企业实际业务范围一致，不遗漏也不过度扩展
- 风险评级方法论应保持年度一致性，便于趋势对比
- 涉及高度主观判断的风险项，须注明评估假设和不确定性
