# 合规框架设计 (Compliance Framework Design)

## 概述

本技能用于设计企业整体合规管理体系框架，包括治理结构、三道防线模型落地、制度架构规划和持续改进机制。基于ISO 37301合规管理体系标准，为企业构建系统化、可持续的合规管理基础设施。

## 适用场景

- 企业合规管理体系初始建设
- 合规体系升级（从被动响应转向主动预防）
- ISO 37301合规管理体系认证准备
- 集团化企业合规治理架构设计
- 新业务/新市场的合规体系扩展设计
- 合规体系有效性评估后的架构优化

## 执行步骤

### 步骤1：现状诊断
- 评估企业当前合规管理成熟度（5级模型：初始级/可重复级/定义级/管理级/优化级）
- 诊断现有治理结构和职责分工
- 识别制度体系的覆盖范围和质量
- 评估合规文化现状（管理层承诺、员工意识）
- 分析资源投入（人员、预算、系统）

### 步骤2：治理架构设计
- 合规治理层级：
  - 董事会/合规委员会：最高合规决策和监督
  - 首席合规官(CCO)/合规部：日常合规管理职能
  - 业务部门合规联络人：一线合规执行
- 三道防线落地方案：
  - 第一道：业务部门自我合规（岗位合规职责、嵌入式控制）
  - 第二道：合规职能部门监督（制度建设、培训、监控、咨询）
  - 第三道：内部审计独立评价（独立性、客观性）
- 报告路线和升级机制设计
- 合规与其他风险管理职能（法务、内控、风险管理）的协同

### 步骤3：制度体系架构
- 三级制度架构设计：
  - 第一级：合规管理总纲（纲领性文件，明确基本原则和治理框架）
  - 第二级：专项合规制度（按领域：反贿赂、数据保护、反洗钱、出口管制等）
  - 第三级：操作指引/工作手册（具体操作步骤、表单模板、常见问答）
- 制度间层级关系和引用规则
- 制度生命周期管理规则（制定、审批、发布、培训、评估、修订、废止）

### 步骤4：运行机制设计
- PDCA循环落地：
  - Plan：年度合规工作计划、风险评估
  - Do：日常合规管理、培训、咨询、审批
  - Check：监控、审计、绩效评估
  - Act：整改、制度优化、持续改进
- 合规监控机制（指标监控、预警、报告）
- 合规考核与激励机制
- 合规沟通与报告机制（定期报告频率和内容）

### 步骤5：实施路线图
- 分阶段实施计划（通常18-24个月）
- 优先级安排（高风险领域先行）
- 里程碑和验收标准
- 资源需求预估（人员/预算/系统）
- 成功指标定义

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 企业基本信息 | 行业、规模、组织架构 | 是 |
| 合规成熟度评估 | 现状诊断结论 | 是 |
| 风险评估结果 | 企业面临的合规风险 | 是 |
| 管理层期望 | 合规建设目标和资源承诺 | 是 |

| 输出项 | 格式 | 说明 |

## 最佳实践

1. **管理层驱动**：合规体系建设必须有高管层的明确承诺和资源支持
2. **业务融入**：合规要求应嵌入业务流程而非叠加在业务之上
3. **渐进式建设**：不追求一步到位，优先覆盖高风险领域
4. **文化为本**：制度和流程只是基础，合规文化才是长期有效性的保障
5. **持续改进**：合规体系是动态演进的，至少年度进行有效性评估

## 约束条件

- 框架设计须符合所在行业的特定监管要求
- 不可照搬其他企业框架，须结合自身规模和风险特征定制
- 治理架构设计须尊重企业现有公司治理结构
- 资源需求建议须考虑企业承受能力，提供分档方案
- 框架设计不替代具体制度编写，二者分步进行
