# 举报调查处理 (Whistleblower Investigation)

## 概述

本技能用于管理合规举报的接收、评估、调查和处理全流程，确保举报渠道有效运作，举报人权益得到保护，所有举报在15个工作日内完成处理闭环。是合规体系"监控-检测"环节的重要组成部分。

## 适用场景

- 匿名/实名合规举报的受理和初步评估
- 举报事项的调查计划制定和执行
- 举报调查结论的形成和处理建议
- 举报人保护措施的落实和监控
- 举报处理的全程记录和档案管理

## 执行步骤

### 步骤1：举报受理（Day 1）
- 通过合规举报渠道（热线/邮箱/系统/信函）接收举报
- 登记举报基本信息（时间、渠道、类别、摘要）
- 确认举报渠道的匿名性保护机制有效
- 向实名举报人发送受理确认（24小时内）
- 分配举报编号并建立调查档案

### 步骤2：初步评估（Day 1-3）
- 评估举报事项是否属于合规举报范围
- 判断举报内容的具体性和可调查性
- 评估事项严重程度和紧迫性
- 决策：
  - 启动正式调查（有具体线索、可能存在重大违规）
  - 转交其他部门处理（非合规事项，如人事纠纷）
  - 存档备查（信息过于笼统、无法调查）
- 涉及利益冲突回避评估（调查人员与被举报人的关系）

### 步骤3：调查执行（Day 3-12）
- 制定调查计划（调查范围、方法、时间表）
- 证据收集：
  - 文档调取（交易记录、审批记录、邮件通信）
  - 人员访谈（被举报人、相关证人）—注意调查的保密性
  - 系统数据分析（日志、报表）
  - 外部验证（对手方确认、公开信息核实）
- 证据固定（原件保全、时间戳记录）
- 过程中如发现涉及犯罪，立即暂停并上报法务总监

### 步骤4：调查结论（Day 12-14）
- 基于证据形成调查结论：
  - 举报属实（违规事实成立）
  - 部分属实（部分事实核实）
  - 不属实（证据不支持举报内容）
  - 无法确认（证据不足以判定）
- 编写调查报告（事实认定→证据引用→结论→建议）
- 提出处理建议（纪律处分/制度整改/移送司法/其他）

### 步骤5：处理与反馈（Day 14-15）
- 调查结论经合规负责人审批
- 重大事项报管理层/合规委员会决策
- 向举报人反馈处理结果（保护调查细节和被调查人隐私）
- 实施处理决定
- 评估是否需要系统性整改（举一反三）
- 关闭举报档案并归档

## 输入规格

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 举报信息 | 举报内容、渠道、时间 | 是 |
| 举报人信息 | 实名/匿名，联系方式 | 视情况 |
| 相关数据 | 涉及的交易/人员/系统记录 | 调查阶段必需 |
| 利益冲突声明 | 调查人员回避确认 | 是 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 举报登记表 | 标准表格 | 受理时填写 |
| 调查报告 | 正式文档 | 事实认定和结论 |
| 处理建议书 | 文档 | 处置方案和依据 |
| 举报人反馈函 | 函件 | 结果通知（脱密） |
| 调查档案 | 文件集 | 全流程记录和证据 |

## 最佳实践

1. **保密优先**：调查信息严格按需知原则控制，防止打击报复
2. **时限刚性**：15个工作日闭环是硬性要求，仅在特殊情况下可申请延期
3. **程序正义**：给予被举报人陈述和申辩的机会
4. **证据标准**：调查结论基于"优势证据"标准，而非刑事的"排除合理怀疑"
5. **系统改进**：每个确认的举报都应评估是否存在系统性问题需要整改

## 约束条件

- 举报人身份信息严格保密，未经本人同意不得向任何第三方透露
- 调查过程中不得采取任何可能构成对举报人报复的行为
- 调查人员必须与被调查事项无利益冲突
- 涉嫌犯罪的案件不得自行继续调查，必须上报并考虑移交司法
- 调查期间被调查人的合法权益须予保障
