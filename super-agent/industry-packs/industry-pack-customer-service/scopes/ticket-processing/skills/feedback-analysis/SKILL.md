# 反馈数据分析 (Feedback Analysis)

## 概述
对收集到的客户满意度数据进行多维度统计分析，识别服务质量的薄弱环节、趋势变化和系统性问题，为管理决策和改进行动提供数据支撑。

## 适用场景
- 日报/周报/月报中的满意度数据分析
- 坐席绩效评估中的满意度维度分析
- 问题类型维度的满意度对比分析
- 满意度异常波动的归因分析
- 改进措施效果的前后对比分析

## 执行步骤

### 1. 数据聚合
按以下维度聚合满意度数据：
- **时间维度**：日/周/月/季度趋势
- **坐席维度**：个人/组别平均满意度
- **问题类型维度**：各分类的满意度分布
- **渠道维度**：各接入渠道的满意度对比
- **客户维度**：不同等级客户的满意度差异

### 2. 趋势分析
- 计算环比/同比变化率
- 识别持续下降趋势（连续3个周期下降即告警）
- 识别突变点（单日/单周突然下降超过0.5分）
- 关联外部事件（系统故障、政策变更、大促等）解释突变

### 3. 归因分析
对低分评价进行文本分析：
- 提取高频负面关键词
- 聚类分析不满原因TOP5
- 对比高分/低分工单的处理差异（响应时间、转派次数、处理步骤）
- 识别系统性因素（如某类问题普遍低分说明方案本身有问题）

### 4. 坐席表现分析
- 各坐席CSAT均值、中位数和标准差
- 识别持续低分坐席（月均<3.8分连续2月）→ 辅导建议
- 识别高分标杆坐席 → 经验萃取
- 坐席满意度与工单类型交叉分析（某坐席处理特定问题低分）

### 5. 洞察输出
生成结构化分析报告：
- 关键指标总览（CSAT均值、各维度得分、回访响应率）
- 亮点与问题（TOP3改善点、TOP3恶化点）
- 根因分析结论
- 改进建议（可操作的、有负责人的、有时限的）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| time_range | object | 是 | 分析时间范围（start_date, end_date） |
| dimensions | array | 是 | 分析维度列表 |
| comparison_period | object | 否 | 对比时间段 |
| filter_conditions | object | 否 | 过滤条件（渠道/技能组/客户等级） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| summary_metrics | object | 关键指标汇总 |
| trends | array | 趋势数据序列 |
| top_issues | array | TOP低分原因列表 |
| agent_rankings | array | 坐席满意度排名 |
| anomalies | array | 异常波动事件 |
| recommendations | array | 改进建议列表 |

## 最佳实践
- 分析要结合样本量考虑，低样本量数据波动大不宜过度解读
- 同比优于环比（消除季节性波动影响）
- 改进建议需具体可落地，避免"提升服务意识"等空泛建议
- 定期跟踪前期建议的执行效果，形成PDCA闭环

## 质量指标
- 分析报告按时输出率 100%
- 洞察建议被采纳率 >= 60%
- 改进措施实施后满意度提升幅度可量化
- 数据准确性 100%（不允许计算错误）
