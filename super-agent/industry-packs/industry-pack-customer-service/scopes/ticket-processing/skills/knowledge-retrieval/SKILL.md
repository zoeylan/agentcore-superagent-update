# 知识库检索与匹配 (Knowledge Retrieval)

## 概述
从企业知识库中高效检索与工单问题相关的知识内容（FAQ、操作指南、排查手册、话术模板），为工单处理提供解决方案的知识支撑。该技能是提升首次解决率（FCR）的关键能力。

## 适用场景
- 工单处理Agent查找问题解决方案时
- 坐席需要参考操作指南处理复杂问题时
- 生成客户回复前的知识验证
- 多关键词联合检索定位精准答案

## 执行步骤

### 1. 查询构建
- 从工单内容中提取核心检索词（产品名+问题类型+关键症状）
- 扩展同义词和相关术语（如"登不上"→"登录失败/无法登录/账号异常"）
- 构建多维度查询向量：语义向量 + 关键词 + 标签过滤

### 2. 多路召回
- **语义检索**：基于向量相似度召回TOP10相关文章
- **关键词检索**：基于BM25算法召回TOP10精确匹配文章
- **标签过滤**：根据工单关联的产品线和问题类型过滤
- 合并去重，生成候选文章池

### 3. 相关性排序
- 综合相似度评分、文章质量评分（点赞率/解决率）、时效性（发布时间/更新时间）进行排序
- 过期文章（超出有效期）降权但不完全排除（可能仍有参考价值）
- 输出TOP3最相关文章及其匹配片段

### 4. 知识验证
- 检查检索结果的时效性（是否在有效期内）
- 检查适用条件（产品版本、地区限制等）是否匹配当前工单
- 如果无满足条件的结果，标记为"知识盲区"并记录

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query_text | string | 是 | 检索查询文本 |
| product_line | string | 否 | 产品线过滤 |
| issue_category | string | 否 | 问题分类过滤 |
| content_type | enum | 否 | 内容类型（faq/guide/troubleshoot/template） |
| max_results | int | 否 | 最大返回数量，默认3 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| results | array | 检索结果列表 |
| results[].article_id | string | 文章ID |
| results[].title | string | 文章标题 |
| results[].relevance_score | float | 相关性评分 |
| results[].matched_snippet | string | 匹配的关键片段 |
| results[].is_valid | boolean | 是否在有效期内 |
| knowledge_gap_flag | boolean | 是否为知识盲区 |

## 最佳实践
- 检索无结果时，尝试放宽过滤条件或使用更通用的查询词
- 高频检索无结果的query应定期汇总提交给知识库运营团队
- 对于"部分匹配"的结果，明确标注哪些内容适用、哪些需要人工判断
- 结合客户具体情况对通用知识进行定制化适配

## 质量指标
- 检索命中率 >= 90%（有结果返回的比例）
- 检索相关性 >= 85%（返回结果确实相关的比例）
- 平均检索延迟 < 3秒
- 知识盲区发现并上报率 100%
