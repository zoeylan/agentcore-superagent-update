# 低分挽回 (Low Score Recovery)

## 概述
对满意度评分<=3分的客户进行主动二次触达和挽回，通过分析不满原因、提供补充解决方案或适度补偿，将不满意客户转化为满意客户。挽回成功率目标>=40%。

## 适用场景
- CSAT评分<=3分的客户自动触发挽回流程
- CSAT=2分且包含投诉关键词的紧急挽回
- CSAT=1分的VIP客户专项挽回
- 连续多次低分的同一客户深度挽回

## 执行步骤

### 1. 低分原因分析
基于客户评分和文字反馈，判断不满原因类别：
- **响应慢**：回复时间不满意、等待过久
- **方案无效**：问题未真正解决、解决方案不适用
- **态度问题**：坐席语气不佳、缺乏耐心
- **流程复杂**：需要多次联系、被反复转接
- **未解决**：工单关闭但客户认为问题仍然存在
- **期望差距**：客户期望的补偿/结果未达到

### 2. 挽回策略制定
根据不满原因和客户等级选择挽回策略：
| 不满原因 | 挽回动作 | 补偿建议 |
|----------|----------|----------|
| 响应慢 | 致歉+说明原因+下次优先承诺 | 小额代金券 |
| 方案无效 | 升级处理+提供新方案 | 视情况 |
| 态度问题 | 致歉+确认改进+换人跟进 | 适度补偿 |
| 流程复杂 | 致歉+一站式解决 | 体验补偿 |
| 未解决 | 重新开单+专人跟进 | 视问题而定 |
| 期望差距 | 解释政策+在权限内最大化满足 | 有限补偿 |

### 3. 挽回沟通执行
- **开场**：感谢反馈 + 真诚致歉 + 表达重视
- **中段**：说明已了解的问题 + 提出补救方案
- **结尾**：确认方案是否接受 + 后续保障承诺
- 沟通语气要求：真诚、不推诿、不辩解、有行动力

### 4. 结果评估
- 客户接受方案并表示满意 → 挽回成功
- 客户修改评分至4分以上 → 挽回成功
- 客户仍不满意 → 记录原因，升级至组长介入
- 客户无响应 → 3天后再次尝试一次，仍无回应则关闭

### 5. 投诉预防
- CSAT<=2分且评语中含投诉意向关键词 → 推送投诉管理预警
- 同一客户连续2次低分 → 标记为高流失风险客户
- 同一坐席被连续3个客户低分评价 → 推送坐席培训建议

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| survey_id | string | 是 | 关联调查编号 |
| csat_score | float | 是 | 客户评分 |
| text_feedback | string | 否 | 客户文字评价 |
| ticket_history | object | 是 | 原始工单处理记录 |
| customer_level | enum | 是 | 客户等级 |
| agent_id | string | 是 | 原处理坐席ID |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| recovery_strategy | string | 选择的挽回策略 |
| root_cause | enum | 不满根因分类 |
| recovery_message | string | 挽回沟通内容 |
| compensation_proposed | object | 补偿方案（类型+金额） |
| recovery_result | enum | 结果（success/failed/no_response/escalated） |
| complaint_risk | boolean | 是否有投诉风险 |

## 最佳实践
- 挽回沟通需在低分产生后24小时内发起（时效性很重要）
- 补偿不是唯一手段，很多客户需要的是被认真对待的感觉
- 挽回应由不同于原处理坐席的人进行（避免客户抵触）
- 记录每次挽回的成功/失败案例，持续优化挽回话术

## 质量指标
- 低分挽回触发率 100%（所有低分都尝试挽回）
- 挽回成功率 >= 40%
- 挽回响应时效 <= 24小时
- 挽回后投诉转化率 < 10%
