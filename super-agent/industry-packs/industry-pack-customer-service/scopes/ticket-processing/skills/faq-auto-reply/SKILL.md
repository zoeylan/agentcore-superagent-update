# FAQ自动应答 (FAQ Auto Reply)

## 概述
对于明确匹配知识库FAQ的标准咨询类问题，由AI直接生成回复并发送给客户，实现零人工干预的自动化解决。该技能是提升AI自动处理率（目标>=40%）的核心能力。

## 适用场景
- 高频标准咨询问题的自动回复（如营业时间、密码重置、物流查询等）
- 客户自助服务引导
- 非工作时间的即时响应

## 执行步骤

### 1. FAQ匹配判定
- 将客户问题与知识库FAQ条目进行语义匹配
- 匹配阈值：相似度 >= 0.90 方可触发自动回复
- 多条FAQ命中时，取相似度最高的一条
- 如果最高匹配度在 0.80-0.90 之间，仅推荐给人工坐席参考，不自动发送

### 2. 回复内容生成
- 基于匹配的FAQ标准答案，结合客户具体情况进行个性化适配
- 插入客户相关变量（如订单号、产品名、具体金额等）
- 根据渠道调整格式：
  - 在线聊天：简洁、分步骤、可加emoji
  - 邮件：正式开头结尾、完整结构化内容
  - 微信：控制在300字以内、使用客户友好语气

### 3. 安全校验
在发送前执行以下校验：
- **敏感操作排除**：涉及退款、账户注销、合同变更等操作不做自动回复
- **情绪校验**：客户情绪为negative/angry时不自动回复，转人工
- **连续自动限制**：同一客户连续2次收到自动回复未确认解决，第3次自动转人工
- **内容时效校验**：FAQ内容是否在有效期内

### 4. 发送与跟踪
- 发送自动回复并附带"是否解决您的问题？"确认按钮
- 客户确认解决 → 自动关单 → 记录为AI自动解决
- 客户选择"未解决" → 立即转人工坐席，附带对话上下文
- 超过24小时无反馈 → 发送一次提醒后自动关单

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customer_message | string | 是 | 客户问题原文 |
| matched_faq_id | string | 是 | 匹配的FAQ条目ID |
| match_score | float | 是 | 匹配相似度分数 |
| channel | enum | 是 | 客户接入渠道 |
| customer_context | object | 否 | 客户上下文（订单信息等） |
| sentiment | enum | 是 | 客户情绪状态 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| reply_content | string | 生成的回复内容 |
| auto_reply_eligible | boolean | 是否满足自动发送条件 |
| block_reason | string | 不满足条件时的拦截原因 |
| confidence | float | 回复置信度 |
| follow_up_action | enum | 后续动作（wait_confirm/transfer_agent） |

## 最佳实践
- 自动回复准确率必须>=95%，宁可不回复也不回复错误内容
- 回复开头先确认理解客户问题（"您好，关于您咨询的XXX..."）
- 涉及操作步骤的回复需包含完整步骤，不省略
- 定期分析"客户选择未解决"的case，优化FAQ内容

## 质量指标
- FAQ回复准确率 >= 95%
- 客户确认解决率（自动应答满意确认率）>= 80%
- 自动处理占比 >= 40%（占总工单量）
- 误发送率 < 0.5%（发送了错误或不相关内容）
