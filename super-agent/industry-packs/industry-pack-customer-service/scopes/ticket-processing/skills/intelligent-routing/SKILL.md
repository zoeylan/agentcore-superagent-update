# 智能路由分配 (Intelligent Routing)

## 概述
基于坐席技能矩阵、实时工作负载和客户特征，将工单分配给最合适的处理资源。该技能是客服运营效率的核心杠杆，直接影响首次解决率、平均处理时长和客户满意度。

## 适用场景
- 新工单的首次路由分配
- 坐席状态变化后的队列重分配
- 工单转派时的重新路由
- 高峰期的溢出路由策略

## 执行步骤

### 1. 路由需求分析
解析工单要求的处理能力：
- 所需产品线知识（如A产品、B产品、C服务）
- 所需技能标签（退款处理、技术排查、合同变更等）
- 所需语言能力（普通话、粤语、英语等）
- 所需权限等级（普通坐席、高级坐席、专家坐席）

### 2. 候选坐席筛选
从坐席池中筛选满足基本条件的候选人：
- 技能标签匹配（必须覆盖工单所需的核心技能）
- 当前状态为在线可用（排除离线、小休、会议状态）
- 权限等级满足要求
- 未超过最大并发工单数限制

### 3. 最优匹配算法
对候选坐席进行评分排序：
```
综合评分 = 技能匹配度×0.3 + 负载空闲度×0.25 + 历史服务关系×0.2 + 处理能力评分×0.15 + 客户等级匹配×0.1
```
- **技能匹配度**：技能标签覆盖率及熟练度评分
- **负载空闲度**：当前处理工单数 / 最大并发容量
- **历史服务关系**：是否曾服务过该客户（优先熟悉的坐席）
- **处理能力评分**：该类型工单的历史FCR和AHT表现
- **客户等级匹配**：VIP客户匹配高级坐席

### 4. 分配执行
- 选择综合评分最高的坐席进行分配
- 如果最优坐席5分钟内未接受，自动分配给次优坐席
- 如果所有候选坐席负载超过80%，进入排队等待队列
- 排队超过预设等待阈值，触发溢出路由（跨组支援或外包坐席）

### 5. VIP专属策略
- VIP客户优先分配专属VIP坐席组
- VIP专属坐席全忙时，分配技能评分TOP20%的高级坐席
- 绝不让VIP客户进入普通排队队列

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ticket_id | string | 是 | 工单编号 |
| required_skills | array | 是 | 所需技能标签列表 |
| priority | enum | 是 | 工单优先级 |
| customer_level | enum | 是 | 客户等级 |
| preferred_agent | string | 否 | 客户偏好坐席ID |
| language | string | 否 | 所需语言 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| assigned_agent_id | string | 分配的坐席ID |
| assigned_agent_name | string | 坐席姓名 |
| routing_score | float | 匹配评分 |
| routing_reason | string | 分配原因说明 |
| estimated_wait_time | int | 预计等待时间（秒） |
| fallback_strategy | string | 备选策略说明 |

## 最佳实践
- 避免"最忙坐席越忙"的马太效应，设置负载均衡保护阈值
- 新坐席给予保护期（首30天降低分配量，逐步加压）
- 对转派工单增加权重，优先分配给更高级的坐席
- 实时监控路由准确率（分配后不被转派视为准确），目标>=88%

## 约束条件
- 路由决策必须在15秒内完成
- 同一工单最多路由3次，超过需人工干预
- 系统故障时启用简单轮询兜底策略
