# 满意度调查 (Satisfaction Survey)

## 概述
在工单关闭后自动触发并执行客户满意度（CSAT）调查，通过多渠道适配的调查方式收集客户对服务体验的定量评分和定性反馈。回访触达率目标>=70%。

## 适用场景
- 工单关闭后的自动满意度回访
- 重要客户的主动满意度关怀
- 升级工单/投诉工单解决后的专项回访
- 定期服务满意度抽样调查

## 执行步骤

### 1. 回访触发判定
工单状态变为"已关闭"后进入回访队列：
- 常规工单：关闭后24小时内触发
- 升级/投诉工单：关闭后12小时内触发（更需关注满意度）
- 排除条件：同一客户30天内已回访2次、客户已设置免打扰、测试工单

### 2. 渠道选择
根据工单接入渠道和客户偏好选择回访方式：
| 原始渠道 | 回访方式 | 备选方式 |
|----------|----------|----------|
| 在线聊天 | 聊天窗口弹窗评价 | 短信 |
| 邮件 | 满意度调查邮件 | - |
| 微信 | 微信模板消息 | 短信 |
| 电话 | IVR按键评价 | 短信 |
| APP | APP内推送通知 | 短信 |

### 3. 调查内容生成
标准调查模板包含：
- **开场语**：感谢使用服务 + 说明调查目的（不超过2句）
- **定量评分**：
  - 总体满意度（1-5分CSAT量表）
  - 响应速度满意度（1-5分）
  - 解决效果满意度（1-5分）
  - 服务态度满意度（1-5分）
- **定性反馈**：开放式文本框（"您还有什么建议吗？"）
- **结束语**：感谢配合 + 服务承诺

### 4. 发送与收集
- 按选定渠道发送调查邀请
- 设置回复截止时间（发送后72小时）
- 首次发送后24小时无响应，发送一次温和提醒
- 超过72小时无响应，标记为"未响应"并归档

### 5. 结果记录
- 收到评分后立即记录并关联工单
- 计算加权总分（总体满意度×0.4 + 解决效果×0.3 + 响应速度×0.15 + 服务态度×0.15）
- 标记为低分（<=3分）或高分（>=4分）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ticket_id | string | 是 | 关闭的工单编号 |
| customer_id | string | 是 | 客户标识 |
| channel | enum | 是 | 原始接入渠道 |
| agent_id | string | 是 | 处理坐席ID |
| ticket_type | enum | 否 | 工单类型（普通/升级/投诉） |
| close_time | datetime | 是 | 工单关闭时间 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| survey_id | string | 调查编号 |
| survey_channel | enum | 实际使用的调查渠道 |
| sent_time | datetime | 发送时间 |
| response_received | boolean | 是否收到回复 |
| csat_score | float | CSAT总分 |
| dimension_scores | object | 各维度分数 |
| text_feedback | string | 文字反馈内容 |
| is_low_score | boolean | 是否为低分 |

## 最佳实践
- 调查内容简洁，完成时间控制在1分钟以内
- 避免在客户非活跃时间发送（深夜/凌晨）
- 对VIP客户可使用更个性化的回访话术
- 升级/投诉工单的回访由组长级别review结果

## 约束条件
- 绝不在调查中夹带营销或推销内容
- 单客户30天内最多接收2次回访
- 调查链接/入口有效期72小时
- 客户评价数据隐私保护，不公开具体评语关联到坐席姓名
