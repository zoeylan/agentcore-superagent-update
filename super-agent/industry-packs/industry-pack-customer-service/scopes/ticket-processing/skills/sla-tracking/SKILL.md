# SLA追踪 (SLA Tracking)

## 概述
对所有活跃工单的SLA时限进行持续追踪计算，监控各优先级工单的时间消耗进度，为预警和升级决策提供实时数据支撑。该技能是确保95%以上SLA达标率的基础能力。

## 适用场景
- 工单创建后的SLA倒计时启动
- 实时监控所有活跃工单的SLA进度
- SLA暂停/恢复计算（如等待客户回复期间）
- SLA达标率统计和趋势分析

## 执行步骤

### 1. SLA时钟初始化
工单分配成功后立即启动SLA时钟：
- 记录SLA起始时间点
- 根据优先级确定SLA总时限（P1:4h, P2:8h, P3:24h, P4:72h）
- 计算首次响应和最终解决两个维度的截止时间
- 考虑工作时间规则（是否7x24还是5x8）

### 2. 实时进度计算
持续计算每个工单的SLA消耗状态：
```
SLA消耗比 = 已用时间 / 总时限 × 100%
剩余时间 = 总时限 - 已用时间
预计达标状态 = 基于当前进展和历史同类工单处理时长预测
```

### 3. SLA暂停规则
以下情况暂停SLA计时：
- 状态变为"待客户确认"（等待客户反馈）
- 状态变为"待协作回复"（等待跨部门回复）
- 客户主动要求暂缓处理
- 注意：暂停需有上限（最长48小时自动恢复计时）

### 4. 阈值监控
| SLA消耗比 | 状态标识 | 触发动作 |
|-----------|----------|----------|
| 0-60% | 绿色/正常 | 正常监控 |
| 60-80% | 黄色/关注 | 进入关注列表 |
| 80-100% | 橙色/预警 | 触发预警通知 |
| 100-150% | 红色/超时 | 触发升级至组长 |
| >150% | 深红/严重超时 | 触发升级至主管 |

### 5. 数据输出
- 实时SLA看板：各优先级当前达标率
- 临期工单清单：未来2小时内即将到期的工单
- 超时工单清单：已超时工单及超时程度
- 趋势数据：日/周/月SLA达标率变化

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ticket_id | string | 是 | 工单编号 |
| priority | enum | 是 | 优先级 P1-P4 |
| created_at | datetime | 是 | 工单创建时间 |
| assigned_at | datetime | 是 | 分配时间 |
| current_status | enum | 是 | 当前工单状态 |
| pause_periods | array | 否 | 暂停时间段记录 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| sla_percentage | float | SLA消耗百分比 |
| remaining_time | int | 剩余时间（分钟） |
| status_color | enum | 状态颜色标识 |
| response_sla_met | boolean | 首次响应是否达标 |
| resolution_sla_met | boolean | 解决时限是否达标 |
| predicted_outcome | enum | 预测结果（on_track/at_risk/breached） |

## 最佳实践
- SLA计算必须考虑工作日历（法定节假日、公司休息日）
- P1工单按7x24计算，P2-P4按工作时间计算
- 暂停期间仍需监控（防止无限期暂停逃避SLA）
- 优先级变更时需重新计算SLA基准

## 约束条件
- SLA计算精度到分钟级别
- 数据刷新频率：P1每1分钟、P2每5分钟、P3/P4每15分钟
- 历史SLA数据保留12个月用于趋势分析
