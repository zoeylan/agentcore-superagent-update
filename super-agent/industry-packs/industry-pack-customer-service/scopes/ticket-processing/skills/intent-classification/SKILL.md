# 意图识别与分类 (Intent Classification)

## 概述
基于NLP技术对客户请求进行问题类型识别和意图分类的核心能力。该技能是工单智能路由的前置基础，直接决定后续处理流程的正确性和效率。

## 适用场景
- 客户通过任意渠道提交新请求时的实时分类
- 批量历史工单的重新分类和标签校准
- 分类模型准确率监控和迭代优化

## 执行步骤

### 1. 文本预处理
- 提取客户消息核心内容（去除寒暄、重复信息）
- 识别关键实体：产品名称、订单号、金额、时间等
- 检测语言和情绪基调（正面/中性/负面/愤怒）

### 2. 一级分类判定
将请求归入四大类别之一：
- **咨询类**（约60%）：产品功能、价格、物流查询等信息获取型问题
- **问题类**（约25%）：故障报告、bug反馈、账户异常等需要修复的问题
- **变更类**（约10%）：退换货、地址修改、套餐变更等需要操作的请求
- **投诉类**（约5%）：明确表达不满、要求赔偿或升级处理的请求

### 3. 二级意图标注
在一级分类下进一步细化具体意图，例如：
- 咨询类 → 物流查询 / 产品功能咨询 / 价格咨询 / 开通流程
- 问题类 → 登录异常 / 支付失败 / 功能故障 / 数据丢失
- 变更类 → 退款申请 / 退货申请 / 地址修改 / 套餐升降级
- 投诉类 → 服务态度投诉 / 产品质量投诉 / 虚假宣传投诉

### 4. 置信度评估
- 输出分类置信度分数（0-1）
- 置信度 >= 0.85：直接确认分类结果
- 置信度 0.6-0.85：标注为"低置信度"，建议人工复核
- 置信度 < 0.6：标记为"待人工确认"，不做自动分类

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message_content | string | 是 | 客户原始消息内容 |
| channel | enum | 是 | 接入渠道（phone/chat/email/wechat/app/social） |
| customer_id | string | 是 | 客户唯一标识 |
| conversation_history | array | 否 | 多轮对话历史 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| primary_category | enum | 一级分类（inquiry/issue/change/complaint） |
| secondary_intent | string | 二级意图标签 |
| confidence_score | float | 置信度分数 0-1 |
| detected_entities | object | 识别的实体信息 |
| sentiment | enum | 情绪（positive/neutral/negative/angry） |
| keywords | array | 提取的关键词列表 |

## 最佳实践
- 投诉类关键词词库需定期更新（如新增网络流行投诉用语）
- 对于包含多个意图的复合消息，取主要意图进行分类，次要意图作为补充标签
- 情绪为"angry"的消息即使内容是咨询，也需标注投诉风险flag
- 分类结果需附带原文关键证据片段，便于后续审核

## 质量指标
- 分类准确率 >= 92%
- 平均处理延迟 < 2秒
- 投诉类召回率 >= 98%（宁可误判为投诉也不能漏判）
