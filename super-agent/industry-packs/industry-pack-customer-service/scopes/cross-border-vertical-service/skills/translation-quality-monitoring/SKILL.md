# 翻译质量监控 (Translation Quality Monitoring)

## 概述

翻译质量监控技能负责通过多维度信号（客户直接反馈、人工抽检评分、翻译后行为指标）持续评估翻译引擎的输出质量，识别特定语言/术语/场景下的系统性翻译偏差，推动翻译引擎的持续优化。翻译质量直接影响跨境客服的服务体验——翻译不准确可能导致客户误解、订单错误甚至投诉。

## 适用场景

- 翻译准确率的日常监控与趋势追踪
- 特定语言翻译质量问题的根因定位
- 术语库覆盖率和准确性评估
- 翻译引擎版本更新后的效果验证
- 新市场/新语言接入时的翻译质量基线建立

## 详细执行步骤

### 步骤1：质量信号采集
1. **客户反馈信号**（实时）：
   - 客户表达困惑："I don't understand"、"什么意思"、"？？？"
   - 客户要求重新说明："Can you say that again in simpler words"
   - 客户直接指出翻译错误："That's not what I meant"
   - 对话中客户切换语言（可能表示翻译不可用）
2. **人工抽检信号**（定期）：
   - 每月随机抽检200+条翻译记录
   - 按语言×消息类型分层抽样
   - 专业翻译团队评分（5分制：1-不可用、3-基本准确、5-完全准确）
3. **行为指标信号**（统计）：
   - 翻译后客户流失率（vs未翻译场景对比）
   - 翻译后首次响应后客户沉默率
   - 需要多轮澄清的对话比例

### 步骤2：质量指标计算
1. **整体翻译准确率**：人工抽检评分>=4分的比例（目标>=90%）
2. **分语言准确率**：按每种语言单独统计，发现短板语言
3. **分场景准确率**：按消息类型（咨询/投诉/营销/系统通知）分别评估
4. **术语翻译一致性**：同一术语在不同上下文中的翻译一致率
5. **置信度校准率**：翻译引擎自报置信度与实际准确率的吻合程度
6. **客户因翻译问题投诉率**（目标<1%）

### 步骤3：问题模式识别
1. 聚类分析低分翻译样本，识别系统性问题模式：
   - 特定语言整体偏低 → 可能需要该语言的模型微调
   - 特定术语反复翻译错误 → 术语库缺失或冲突
   - 特定句式/语法结构翻译不佳 → 语言模型局限
   - 特定平台内容翻译差 → 平台特有表达未覆盖
2. 按影响面和严重度对问题排优先级：
   - P0：导致客户投诉或订单错误的翻译问题
   - P1：高频出现但尚未导致投诉的翻译偏差
   - P2：低频或非关键信息的翻译瑕疵

### 步骤4：优化建议输出
1. **术语库更新建议**：列出需新增/修正的术语映射
2. **模型优化建议**：针对特定语言提供微调数据方向
3. **流程优化建议**：如某语言准确率持续低于85%，建议增加该语言的人工校验比例
4. **质量门槛调整**：基于实际数据建议翻译置信度阈值调整

### 步骤5：效果验证闭环
1. 追踪已提交的优化建议的落地情况
2. 优化措施实施后，对比前后指标变化
3. 验证翻译引擎版本更新后各语言质量是否提升
4. 维护翻译质量改进台账（问题→措施→效果）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| monitoring_type | string | 是 | 监控类型（realtime/scheduled/adhoc） |
| language_filter | array | 否 | 指定监控的语言（不指定则全部） |
| time_range | object | 否 | 分析时间范围（定期报告用） |
| sample_data | array | 否 | 人工抽检评分数据（定期导入） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| quality_scores | object | 各维度质量评分 |
| problem_patterns | array | 识别的问题模式列表 |
| optimization_suggestions | array | 优化建议 |
| trend_analysis | object | 质量趋势分析 |
| alert_level | string | 当前翻译质量告警级别 |

## 最佳实践

1. **分层监控**：重要场景（金额确认、政策说明）的翻译执行更严格的质量标准
2. **快速反馈**：P0级翻译问题24小时内完成术语库热修复
3. **正向样本**：不仅关注错误，也收集优秀翻译案例用于模型强化学习
4. **客户感知优先**：技术指标（BLEU分数等）仅供参考，最终以客户感知为准
5. **季节性考虑**：大促期间新商品术语激增，需提前准备术语库

## 约束条件

- 月度人工抽检>=200条
- 术语库月更新率100%
- 客户因翻译问题投诉率<1%
- P0级问题24小时内修复
- 翻译质量月报按时输出（每月5号前）
- 新语言接入前必须完成500条样本的基线测试
