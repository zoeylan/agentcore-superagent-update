# MCP工具编排 (MCP Tool Orchestration)

## 概述

MCP工具编排技能负责通过MCP（Model Context Protocol）Server调用各类外部系统工具，实现跨境电商和行业垂直场景中的端到端自动化业务操作。该技能能够理解客户意图后选择正确的工具组合、编排调用顺序、处理工具间的数据传递，并在工具调用失败时执行优雅降级。

## 适用场景

- 物流轨迹查询（4PX、云途、极兔、DHL等跨境物流商）
- 订单状态查询与修改（各平台订单API）
- 退换货申请的自动提交
- 优惠券/折扣码的生成与发放
- 航班查询、退改签计算、座位锁定（航空场景）
- 金融账户余额查询、交易授权（金融场景）
- 支付状态核验与退款执行

## 详细执行步骤

### 步骤1：意图解析与工具选择
1. 接收客户意图识别结果（intent_type + intent_parameters）
2. 根据意图类型查询工具映射表，确定需要调用的MCP工具：
   - 物流查询 → logistics_tracking_tool
   - 订单查询 → order_status_tool
   - 退换货 → return_exchange_tool + (refund_calculation_tool)
   - 优惠券发放 → coupon_generation_tool
   - 航班查询 → flight_search_tool
   - 退改签 → ticket_change_tool + fare_calculation_tool + seat_lock_tool
   - 余额查询 → account_balance_tool
3. 判断是否需要多工具编排（单工具直接调用，多工具制定调用计划）

### 步骤2：参数准备与验证
1. 从客户对话和系统上下文中提取工具所需参数
2. 对必要参数进行验证：
   - 订单号格式校验（各平台格式不同）
   - 物流单号有效性检查
   - 金额数值范围合理性
3. 缺少必要参数时，生成追问话术向客户补充获取
4. 对敏感参数（如支付信息）确认客户身份验证已通过

### 步骤3：工具调用执行
1. 单工具调用：直接调用并等待响应
2. 多工具串行编排（有依赖关系）：
   - 示例：退改签 = 航班查询(获取航班信息) → 票价规则查询(获取退改条件) → 费用计算(计算退改费) → Policy Engine验证 → 座位锁定(如改签)
3. 多工具并行调用（无依赖关系）：
   - 示例：物流查询 + 订单状态查询（同时发起，减少等待时间）
4. 每个工具调用设置超时限制（默认10秒），超时则进入重试逻辑

### 步骤4：结果处理与组装
1. 接收工具返回结果，进行数据清洗和格式化
2. 多工具结果的关联组装：
   - 将物流状态与订单状态关联展示
   - 将退改费用计算结果与Policy Engine决策关联
3. 将技术性返回结果转化为客户友好的自然语言描述
4. 对关键结果信息进行高亮标注（如预计到达时间、退款金额等）

### 步骤5：异常处理与降级
1. 工具调用超时：重试1次(间隔3秒)，仍超时则降级处理
2. 工具返回错误：
   - 4xx错误（参数错误）→ 检查参数并修正后重试
   - 5xx错误（服务端错误）→ 记录错误，通知运维，告知客户系统繁忙
3. 第三方系统维护中：查询系统维护公告，告知客户预计恢复时间
4. 所有降级场景：提供人工转接选项，确保客户问题不被中断

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| intent_type | string | 是 | 客户意图类型 |
| intent_parameters | object | 是 | 意图关联参数 |
| customer_context | object | 是 | 客户上下文（ID、平台、历史） |
| identity_verified | boolean | 是 | 身份验证状态 |
| tool_chain | array | 否 | 预定义工具调用链（如有） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| execution_status | string | 成功/部分成功/失败 |
| tool_results | array | 各工具调用结果列表 |
| customer_response | string | 面向客户的自然语言回复 |
| next_action | object | 后续动作建议（如需Policy Engine验证） |
| error_details | object | 错误详情（如有失败） |

## 最佳实践

1. **工具预热**：对高频使用的工具保持连接池，避免每次冷启动连接
2. **结果缓存**：同一订单的状态查询结果缓存5分钟，避免重复调用
3. **参数模板**：为常见场景预设参数模板，减少参数组装时间
4. **调用链断路**：多步编排中某步失败时，评估后续步骤是否仍有意义
5. **版本兼容**：工具API版本变更时，通过适配层保证向后兼容

## 约束条件

- 单次工具调用超时限制：10秒
- 工具编排总耗时限制：30秒（含所有串行步骤）
- 工具调用失败率>5%时触发降级告警
- 涉及资金操作的工具调用前必须通过Policy Engine验证
- 所有工具调用记录写入审计日志，保留180天
