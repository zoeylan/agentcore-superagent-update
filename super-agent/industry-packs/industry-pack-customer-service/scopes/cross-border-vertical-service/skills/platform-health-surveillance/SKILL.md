# 平台健康度监控 (Platform Health Surveillance)

## 概述

平台健康度监控技能负责实时追踪各跨境电商平台（Shopee/Lazada/TikTok Shop/Amazon）店铺的核心运营指标，包括店铺评分、回复率、违规记录、账号状态等，在接近平台处罚阈值时提前预警，避免因指标不达标导致的店铺限流、降权或封禁等严重后果。

## 适用场景

- 各平台店铺评分和回复率的实时监控
- 平台考核周期节点的指标预警
- 平台规则变更后的合规性检查
- 店铺违规风险的提前识别和应对
- 多店铺运营时的健康度对比和资源调配

## 详细执行步骤

### 步骤1：平台指标采集
1. 定期（每小时）从各平台API采集关键指标：

   **Shopee**：
   - 聊天回复率（Chat Response Rate）：目标>=80%，低于则限流
   - 店铺评分（Shop Rating）：1-5分，<4.5分影响搜索排名
   - 迟发率（Late Shipment Rate）：<5%
   - 订单取消率（Cancellation Rate）：<2%

   **Lazada**：
   - 卖家响应率（Seller Response Rate）：目标>=85%
   - 卖家评分（Seller Rating）：<4.0分有降权风险
   - 物流准时率

   **TikTok Shop**：
   - 消息回复率：目标>=80%
   - 店铺体验分：<4.0分限制活动报名
   - 违规积分：累计达到阈值封店

   **Amazon**：
   - 消息24小时回复率：目标>=90%
   - 订单缺陷率（ODR）：<1%
   - A-to-Z索赔率
   - 账号健康评级（Account Health Rating）

### 步骤2：阈值评估与预警
1. 三级预警机制：
   - **绿色（安全）**：指标>平台要求的110%
   - **黄色（警告）**：指标介于平台要求的100%-110%
   - **红色（危险）**：指标<平台要求或即将低于
2. 动态预测：基于最近7天的指标趋势，预测未来3天是否会降至阈值以下
3. 考核周期感知：在平台月度/季度考核节点前7天加强监控频率

### 步骤3：风险分析与根因定位
1. 指标下降时自动分析根因：
   - 回复率下降 → 检查消息量激增/坐席不足/系统故障
   - 店铺评分下降 → 分析近期差评关键词和来源
   - 违规增加 → 定位违规类型和触发原因
2. 关联分析：多指标联动变化的根因定位（如系统故障导致回复率和评分同时下降）

### 步骤4：应急策略生成
1. **回复率告急**：
   - 优先处理即将超时的消息
   - 启动AI快速回复模式（先回复再处理）
   - 调配额外人力资源
2. **评分下降**：
   - 分析近期差评集中的问题类型
   - 对差评客户进行挽回联系
   - 向运营团队推送改进建议
3. **违规风险**：
   - 立即审查违规行为的根源
   - 暂停可能触发进一步违规的操作
   - 准备申诉材料（如误判）

### 步骤5：报告与追踪
1. 每日输出各平台健康度面板数据
2. 预警事件追踪至解除（指标恢复安全区间）
3. 月度输出平台合规性报告
4. 平台规则变更时及时更新监控规则和阈值

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| platform | string | 否 | 指定平台（不指定则监控全部） |
| shop_id | string | 否 | 指定店铺ID |
| check_type | string | 是 | 检查类型（routine/alert_response/rule_update） |
| platform_metrics | object | 是 | 平台最新指标数据 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| health_status | object | 各平台健康状态（green/yellow/red） |
| alerts | array | 当前预警列表 |
| risk_forecast | object | 未来3天风险预测 |
| emergency_actions | array | 建议的应急措施 |
| compliance_report | object | 合规性报告 |

## 最佳实践

1. **提前量充足**：在指标可能触碰阈值前至少24小时预警，留足应对时间
2. **多店铺优先级**：当多店铺同时告警时，优先处理GMV最高的店铺
3. **规则更新感知**：订阅各平台的规则更新通知，第一时间同步新标准
4. **申诉准备**：对平台可能误判的违规，提前准备申诉模板和证据
5. **季节性基线**：大促期间指标波动属正常，使用大促期间的历史基线对比

## 约束条件

- 指标采集频率：每小时（预警状态时每15分钟）
- API故障检测时间<=1分钟
- 红色预警必须在5分钟内通知运营负责人
- 平台规则更新后24小时内完成监控规则同步
- 数据采集失败时使用最后已知值+趋势外推，标注数据来源
