# 跨境数据分析 (Cross-Border Data Analytics)

## 概述

跨境数据分析技能负责聚合来自多平台、多语言、多时区的全维度客服运营数据，进行多维交叉分析，生成日/周/月度运营报告，识别服务短板和异常指标，输出数据驱动的优化建议。该技能是跨境客服运营决策的数据基础，支撑管理层的资源调配和策略调整。

## 适用场景

- 日度运营快报：关键指标实时监控和异常预警
- 周度趋势分析：服务质量趋势、效率变化和问题追踪
- 月度战略报告：ROI评估、资源优化建议、竞争态势分析
- 大促复盘：活动期间服务承载能力和客户体验分析
- 专项分析：特定平台/语言/品类的深度分析

## 详细执行步骤

### 步骤1：数据采集与聚合
1. 从各数据源采集原始数据：
   - 多平台网关Agent：消息量、响应时效、平台分布
   - 垂直业务处理Agent：操作类型分布、Policy Engine决策日志、工具调用成功率
   - 商机识别Agent：商机识别数、评分分布、转化漏斗
   - 翻译引擎：翻译量、语言分布、置信度分布、人工校验率
   - 客户记忆系统：STM命中率、LTM覆盖率、隐私操作日志
2. 数据清洗：去重、补缺、格式统一、时区对齐（统一为UTC+8）
3. 数据入仓：按维度模型存储（平台/语言/时段/品类/处理方式）

### 步骤2：核心指标计算
1. **效率指标**：
   - 各平台响应时限达标率（SLA Compliance Rate）
   - 平均响应时间（ART）/ 平均处理时间（AHT）
   - AI自动处理率（整体/夜间/按难度级别）
   - 夜间漏单率
2. **质量指标**：
   - 客户满意度（CSAT）按平台/语言/品类维度
   - 首次解决率（FCR）
   - 翻译准确率（人工抽检+客户反馈）
   - Policy Engine决策准确率
3. **商业指标**：
   - 商机识别数和转化率
   - 商机带来的增量GMV
   - 客服ROI（成本vs创收）
4. **健康指标**：
   - 各平台店铺评分和回复率
   - 坐席利用率和负载均衡度
   - 系统可用性（API故障频率和持续时间）

### 步骤3：多维交叉分析
1. 平台×时段分析：识别各平台的流量高峰时段和对应的人力缺口
2. 语言×满意度分析：发现特定语言的翻译质量导致的满意度差异
3. 品类×问题类型分析：识别高频问题集中的品类，推动产品改进
4. 处理方式×效果分析：对比AI自动处理和人工处理的效率/质量差异
5. 时间趋势分析：周环比/月环比变化，识别改善或恶化趋势

### 步骤4：异常检测与预警
1. 基于历史基线的统计异常检测：
   - 指标偏离历史均值2个标准差 → Warning
   - 指标偏离历史均值3个标准差 → Critical
2. 业务阈值预警：
   - 平台店铺评分<4.5 → Critical（有被限流风险）
   - 平台回复率<80% → Critical（有被处罚风险）
   - 夜间漏单率>5% → Warning
   - 翻译投诉率>1% → Warning
3. 趋势预警：连续3天某指标持续下降 → 提前预警

### 步骤5：报告生成与建议输出
1. **日报**（自动生成，每日09:00发送）：
   - 昨日核心指标概览（红/黄/绿状态灯）
   - 异常事件摘要
   - 今日重点关注项
2. **周报**（每周一生成）：
   - 周度趋势变化分析
   - TOP3改善项和TOP3恶化项
   - 上周优化措施效果验证
   - 本周优化建议
3. **月报**（每月3号生成）：
   - 月度全景数据概览
   - 战略级洞察（市场变化、竞争态势）
   - ROI分析和资源优化建议
   - 下月重点工作方向

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| report_type | string | 是 | 报告类型（daily/weekly/monthly/adhoc） |
| time_range | object | 是 | 分析时间范围 |
| dimensions | array | 否 | 分析维度（platform/language/category等） |
| metrics | array | 否 | 指定分析指标（不指定则计算全量） |
| comparison_period | object | 否 | 对比周期（环比/同比） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| report | object | 完整报告内容 |
| summary | string | 报告摘要（<200字） |
| alerts | array | 预警列表 |
| recommendations | array | 优化建议列表 |
| data_visualizations | array | 图表数据（供前端渲染） |

## 最佳实践

1. **数据时效性**：日报数据延迟不超过2小时，确保管理决策的时效性
2. **可操作性**：每个发现都要附带"谁应该做什么"的具体建议
3. **趋势优先**：单点数据波动不值得关注，持续趋势变化才需要行动
4. **分层呈现**：管理层看摘要和趋势，运营团队看详细数据和action item
5. **自动vs手动**：日常报告自动生成，专项分析支持手动触发和自定义维度

## 约束条件

- 日报每日09:00自动生成并发送
- 异常预警实时触发（检测延迟<5分钟）
- 报告数据准确性>=99%（经抽样验证）
- 分析样本量<30时标注"样本不足"
- 数据保留策略：原始数据90天、聚合数据2年、报告永久
