# 跨会话记忆管理 (Cross-Session Memory Management)

## 概述

跨会话记忆管理技能负责维护客户在多次咨询间的上下文连续性，包括STM（Short-Term Memory，短期记忆）和LTM（Long-Term Memory，长期记忆）两个层次。STM确保同一客户24小时内再次咨询时无需重复提供背景信息；LTM学习并记录客户的长期偏好和行为模式，支撑个性化服务策略。

## 适用场景

- 客户24小时内再次咨询时，自动加载上次对话上下文
- 长期客户的语言偏好、常购品类识别与应用
- 基于历史问题模式的主动服务预判
- 客户VIP等级升降的长期行为跟踪
- 跨平台同一客户的身份关联与记忆合并

## 详细执行步骤

### 步骤1：客户身份识别与记忆加载
1. 通过customer_id（统一客户标识）检索客户记忆库
2. 跨平台身份关联：同一客户在不同平台的账号关联（通过手机号/邮箱/收货地址匹配）
3. 加载策略：
   - 判断距上次交互时间：<24小时 → 加载STM+LTM；>24小时 → 仅加载LTM
   - STM内容：上次对话摘要、未解决问题、进行中的操作状态
   - LTM内容：语言偏好、常购品类TOP3、历史问题类型分布、VIP等级、特殊标记

### 步骤2：STM管理（24小时短期记忆）
1. **记忆写入**：每次对话结束时生成STM快照：
   - conversation_summary: 本次对话核心内容摘要（<200字）
   - unresolved_issues: 未解决的问题列表
   - pending_actions: 进行中的操作（如等待审批、等待物流更新）
   - emotional_state: 客户情绪状态标记（正常/不满/焦虑）
   - last_intent: 最后一个意图类型
2. **记忆读取**：下次对话开始时注入上下文，免去重复询问
3. **记忆过期**：24小时后STM自动降级为历史记录，不再主动加载
4. **记忆更新**：同一客户多次交互时，STM增量更新而非覆盖

### 步骤3：LTM管理（长期偏好记忆）
1. **偏好学习**：基于多次交互数据，自动提取和更新长期偏好：
   - language_preference: 偏好语言（可能不同于平台默认语言）
   - communication_style: 沟通风格偏好（正式/轻松/简洁）
   - category_interests: 常购品类及价格区间
   - issue_patterns: 历史问题模式（如经常咨询物流、偏好退款而非换货）
   - satisfaction_trend: 满意度趋势（近30天/90天的CSAT变化）
2. **偏好准确率验证**：定期（每30天）验证LTM偏好的准确性：
   - 如果客户行为与LTM记录显著不符 → 标记需重新学习
   - LTM偏好准确率目标>=85%
3. **偏好应用**：基于LTM进行个性化服务：
   - 自动使用客户偏好语言回复
   - 推荐商品基于常购品类
   - 服务策略基于历史满意度调整（低满意度客户给予更高关注）

### 步骤4：隐私合规管理
1. 数据最小化原则：只存储服务所必需的信息
2. 严禁存储的信息：完整证件号码、支付密码、银行卡完整号码、生物特征数据
3. 可脱敏存储的信息：手机号（保留后4位）、收货地址（保留区级以上）
4. 客户要求删除记忆时：48小时内完成STM和LTM的完全清除
5. 符合GDPR/PDPA等数据保护法规：跨境场景特别注意各国数据本地化要求

### 步骤5：记忆质量维护
1. 定期清理无效记忆（如已关闭店铺的订单信息、过期优惠券记录）
2. 合并冲突处理：同一客户跨平台记忆出现冲突时（如不同平台记录不同语言偏好），以最近30天高频行为为准
3. 容量管理：单客户STM上限5条（保留最近5次交互），LTM上限20个字段
4. 异常检测：当客户行为突然剧烈变化时（如突然改变语言或品类偏好），标记为可能的账号被盗风险

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customer_id | string | 是 | 统一客户标识 |
| platform | string | 是 | 当前交互平台 |
| action | string | 是 | 操作类型（load/save/update/delete） |
| conversation_data | object | 否 | 当前对话数据（save时必填） |
| memory_type | string | 否 | 指定操作的记忆类型（STM/LTM/both） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| memory_loaded | object | 加载的记忆内容（STM+LTM） |
| context_summary | string | 上下文摘要（供Agent使用） |
| personalization_hints | array | 个性化服务建议列表 |
| privacy_flags | object | 隐私相关标记（数据保护级别、合规要求） |
| memory_health | object | 记忆健康度（覆盖率、新鲜度、准确率） |

## 最佳实践

1. **渐进式加载**：先加载LTM核心偏好（<50ms），再异步加载完整STM
2. **置信度标注**：LTM每个偏好字段标注置信度，低置信度偏好不用于自动决策
3. **显式确认**：对重要偏好变更（如VIP等级变化），在应用前向客户确认
4. **失忆兜底**：当记忆加载失败时，退化为无记忆模式，不应因此拒绝服务
5. **记忆继承**：客户主动告知偏好变化时，直接更新LTM不需要等待学习

## 约束条件

- STM有效期：24小时（可配置）
- LTM偏好准确率>=85%
- 记忆加载延迟<200ms
- 隐私数据合规存储100%
- 客户要求删除记忆时48小时内执行
- 跨境场景严格遵守数据本地化要求（欧盟数据不出欧盟、东南亚数据按PDPA要求）
