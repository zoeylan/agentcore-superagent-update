# 智能调度 (Smart Scheduling)

## 概述

智能调度技能负责跨境客服场景下7×24小时的服务资源调度，根据时段、消息量、坐席可用性和平台SLA要求，动态决定每条消息由AI自动处理还是分配给人工坐席。核心目标是在保证各平台响应时限达标率>=95%的前提下，最大化夜间AI自动处理率（>=60%）并将漏单率控制在<5%。

## 适用场景

- 工作时段的人工坐席负载均衡分配
- 夜间时段（22:00-08:00 UTC+8）的AI自动处理模式启动
- 高峰时段的动态扩容与溢出策略
- 节假日和大促期间的特殊调度方案
- 坐席技能匹配与客户偏好对齐

## 详细执行步骤

### 步骤1：时段判定与模式切换
1. 获取当前时间（UTC+8），判断所属时段：
   - 工作时段：08:00-22:00（人工为主+AI辅助）
   - 夜间时段：22:00-08:00（AI为主+值班人工兜底）
   - 特殊时段：节假日/大促期间（预设特殊调度方案）
2. 工作时段检查在线坐席数量和各技能组的空闲状态
3. 夜间时段确认AI自动处理系统可用性和值班人工的响应通道

### 步骤2：消息分类与处理能力匹配
1. 根据消息预分类结果判断处理难度：
   - L1（简单）：FAQ类问题、物流查询、订单状态查询 → AI可独立处理
   - L2（中等）：退换货申请、优惠咨询、商品推荐 → AI处理+人工复核
   - L3（复杂）：投诉、纠纷、高额退款、特殊诉求 → 必须人工处理
2. 夜间时段：L1自动处理，L2由AI处理并标记待次日复核，L3缓存待值班人工或次日优先处理
3. 工作时段：L1由AI处理释放人力，L2/L3按技能匹配分配人工

### 步骤3：坐席匹配与分配
1. 获取在线坐席列表及其属性：
   - 技能标签（语言能力、品类专长、平台熟悉度）
   - 当前负载（正在处理的会话数）
   - 当前等待队列深度
   - 历史绩效（CSAT评分、解决率）
2. 执行匹配算法：
   - 优先匹配：语言能力对应 > 品类对应 > 平台对应
   - 负载均衡：优先分配给负载低于均值的坐席
   - VIP客户：优先分配给高绩效坐席
3. 无匹配坐席可用时：进入等待队列，设置SLA倒计时监控

### 步骤4：队列深度监控与负载均衡
1. 每30秒检查各平台/各技能组消息队列深度
2. 触发负载均衡的阈值：
   - 单坐席排队>=5条 → 尝试分流到同技能组其他坐席
   - 单技能组积压>=20条 → 触发跨技能组支援
   - 全平台总积压>=200条 → 触发负载均衡预警+请求管理层增派
3. 预警升级链：自动分流 → 跨组支援 → 管理层预警 → 应急扩容

### 步骤5：SLA保护机制
1. 对所有未响应消息维护SLA倒计时
2. 到达SLA 80%时间时：提升优先级，发送坐席提醒
3. 到达SLA 90%时间时：触发自动接管（AI先发送安抚消息）
4. 确保各平台响应时限达标率>=95%

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | object | 是 | 标准化消息体 |
| message_classification | object | 是 | 消息预分类结果（难度级别、类型） |
| available_agents | array | 是 | 当前可用坐席列表及状态 |
| queue_status | object | 是 | 当前各队列深度信息 |
| time_context | object | 否 | 时间上下文（是否节假日/大促） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| scheduling_decision | object | 调度决策（AI处理/指定坐席/排队等待） |
| assigned_agent_id | string | 分配的坐席ID（如分配人工） |
| processing_mode | string | 处理模式（auto/human/hybrid） |
| sla_countdown | object | SLA倒计时信息 |
| load_balance_alerts | array | 负载均衡预警（如触发） |

## 最佳实践

1. **渐进式切换**：工作时段/夜间时段切换时设置30分钟过渡期，避免服务断档
2. **弹性阈值**：大促期间动态调整队列告警阈值（放宽50%），避免频繁误报
3. **坐席预热**：夜间值班坐席上线前10分钟推送当日热点问题摘要
4. **AI兜底**：所有分配到人工但5分钟无响应的消息，AI自动发送安抚消息
5. **历史规律利用**：基于过去30天同时段数据预测消息量，提前安排资源

## 约束条件

- 夜间漏单率<5%（有消息进入但无任何响应的比例）
- AI自动处理率夜间>=60%
- 各平台响应时限达标率>=95%
- 调度决策延迟<1秒
- 坐席单人同时处理会话数上限：8个
