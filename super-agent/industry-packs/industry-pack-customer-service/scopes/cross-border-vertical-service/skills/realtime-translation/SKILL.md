# 实时多语言翻译 (Realtime Translation)

## 概述

实时多语言翻译技能为跨境客服场景提供高质量、低延迟的双向翻译能力，支持8+种语言与中文之间的实时互译。区别于通用翻译，本技能针对电商领域进行了深度优化，包括商品术语、促销话术、物流用语、平台规则描述等专业领域的翻译准确性，同时保留目标语言的文化习惯和敬语体系。

## 适用场景

- 客户消息从源语言翻译为中文（供AI或人工坐席处理）
- 中文回复翻译回客户的语言（发送前的最后环节）
- 翻译置信度评估与低置信度消息人工校验标记
- 电商专业术语和本地化表达的优化翻译
- 批量历史消息翻译（用于数据分析和质量抽检）

## 详细执行步骤

### 步骤1：语言检测
1. 对输入文本执行自动语言检测（Language Detection）
2. 支持检测的语言列表：
   - 东南亚：泰语(th)、越南语(vi)、印尼语(id)、马来语(ms)、菲律宾语(tl)、缅甸语(my)
   - 欧美：英语(en)、西班牙语(es)、法语(fr)、德语(de)
   - 东亚：日语(ja)、韩语(ko)
3. 当检测置信度<80%时，结合客户历史语言偏好和平台目标市场辅助判断
4. 对混合语言消息（如中英混合、马来语夹杂英语），识别主体语言并保留外语专有名词

### 步骤2：术语库匹配
1. 加载对应语言的电商专业术语库：
   - 物流术语：tracking number、last mile delivery、customs clearance等
   - 商品术语：SKU、variant、pre-order、flash sale等
   - 客服术语：refund、return、exchange、voucher等
   - 平台专属术语：Shopee Guarantee、Lazada Wallet、FBT等
2. 对消息中命中术语库的词汇进行锁定翻译（确保一致性）
3. 未命中但疑似专业术语的词汇标记为待确认

### 步骤3：翻译执行
1. 调用翻译引擎执行主体翻译
2. 应用术语库锁定词替换翻译引擎的通用翻译
3. 对目标语言执行文化适配：
   - 泰语：添加适当的礼貌词尾 (ค่ะ/ครับ)
   - 越南语：使用正确的人称代词体系
   - 日语：使用适当的敬语级别
   - 印尼语：注意正式/非正式语体选择
4. 保留原始消息中的数字、订单号、链接等不可翻译元素

### 步骤4：置信度评估
1. 对翻译结果进行质量评分（0-100%）：
   - 句法完整性检查
   - 术语翻译一致性检查
   - 语义保留度评估（回译对比）
2. 置信度分级：
   - >=90%：高质量，直接使用
   - 70%-89%：可接受，标记供后续抽检
   - <70%：需人工校验，暂不发送

### 步骤5：输出与反馈
1. 输出翻译结果及元数据（源语言、目标语言、置信度、术语命中数）
2. 低置信度翻译进入人工校验队列
3. 人工校验/修正结果回流至翻译引擎进行模型微调
4. 新发现的术语映射自动添加至术语库待审列表

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| text | string | 是 | 待翻译文本 |
| source_language | string | 否 | 源语言代码（不提供则自动检测） |
| target_language | string | 是 | 目标语言代码 |
| context | object | 否 | 上下文信息（商品类目、平台、对话历史摘要） |
| customer_preference | object | 否 | 客户语言偏好（正式/非正式、敬语级别） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| translated_text | string | 翻译后文本 |
| source_language_detected | string | 检测到的源语言 |
| confidence_score | float | 翻译置信度(0-1) |
| terms_matched | array | 命中的术语列表 |
| needs_review | boolean | 是否需要人工校验 |
| review_reason | string | 需要校验的原因（如有） |

## 最佳实践

1. **上下文传递**：翻译时传入前3轮对话上下文，提升代词指代和语境理解准确性
2. **术语一致性**：同一会话中相同术语的翻译必须保持一致
3. **文化敏感性**：避免直译可能产生冒犯的表达，尤其是涉及宗教（东南亚）和性别敬称
4. **回译验证**：对重要消息（如退款金额、政策说明）执行回译验证确保准确
5. **增量更新术语库**：每月基于人工校验数据更新术语库，确保月更新率100%

## 约束条件

- 单条消息翻译延迟<2秒
- 翻译准确率>=90%（以人工抽检为准）
- 翻译置信度<70%的消息100%进入人工校验
- 不翻译客户提供的订单号、追踪号、链接等结构化数据
- 月度人工抽检>=200条，确保质量持续达标
