# 意向评分引擎 (Intent Scoring Engine)

## 概述

意向评分引擎技能负责将商机信号检测技能识别出的原始购买意向信号进行多维度综合评分，输出0-100的标准化意向度分数，并根据分数等级（高>=80/中50-79/低<50）触发不同的后续处理策略。评分引擎考虑信号强度、客户价值、产品利润率、转化窗口等因素，确保有限的销售资源聚焦在最高价值商机上。

## 适用场景

- 对已检测到购买意向信号的客户进行意向度量化评分
- 高意向商机的快速识别与优先推送
- 中意向商机的自动化营销策略触发
- 评分模型的持续校准与优化
- 促销活动期间的评分阈值动态调整

## 详细执行步骤

### 步骤1：多维度特征提取
1. 从商机信号中提取评分特征：
   - **信号维度**（权重30%）：
     - 信号类型数量（检测到几种不同信号）
     - 最强信号的强度等级
     - 信号出现的频次和密度
   - **客户维度**（权重25%）：
     - 历史购买金额（客户价值分层）
     - 历史转化率（该客户过去的咨询→购买比例）
     - VIP等级和忠诚度
     - 客户生命周期阶段（新客/活跃/沉默/流失预警）
   - **产品维度**（权重20%）：
     - 产品利润率（高利润产品加权）
     - 产品库存状态（有库存vs缺货）
     - 产品热度（热卖品vs冷门品）
   - **时效维度**（权重15%）：
     - 是否在促销窗口期内
     - 客户表达的时间紧迫度
     - 产品促销即将到期
   - **竞争维度**（权重10%）：
     - 是否提到竞品
     - 竞品比价压力大小
     - 独家产品vs通用产品

### 步骤2：评分计算
1. 对每个维度计算子分数（0-100）
2. 按权重加权计算综合得分：
   ```
   total_score = signal_score×0.30 + customer_score×0.25 + product_score×0.20 + timing_score×0.15 + competition_score×0.10
   ```
3. 应用修正因子：
   - 客户明确表示"免打扰" → 分数归零
   - 当前对话中有拒绝表态 → 分数×0.5
   - 大促期间 → 分数×1.2（上限100）
4. 输出最终评分及分级：高(>=80) / 中(50-79) / 低(<50)

### 步骤3：策略匹配
1. **高意向（>=80分）**：
   - 立即标记为高优先级商机
   - 1小时内推送至对应区域/品类销售人员
   - 附带完整客户画像、对话摘要、推荐跟进策略
   - 设置跟进超时监控（销售2小时未响应 → 升级提醒）
2. **中意向（50-79分）**：
   - 标记为潜在商机，进入自动化营销队列
   - 根据产品类型选择营销策略（优惠券/限时折扣/相关推荐）
   - 48小时后重新评估（如未转化则降级）
3. **低意向（<50分）**：
   - 记录存档但不主动跟进
   - 在常规服务完成后附加轻量推荐（"您可能还感兴趣..."）
   - 纳入后续批量营销池

### 步骤4：评分反馈与模型优化
1. 跟踪每个评分商机的最终转化结果（7天/14天/30天窗口）
2. 计算评分模型的关键指标：
   - 高意向转化率（目标>=30%）
   - 中意向转化率（目标>=10%）
   - 低意向转化率（基线对照）
3. 基于转化反馈，按月调整各维度权重和子分数规则
4. 检测评分偏移：如果某段时间高意向转化率持续<20%，触发模型校准

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| signals | array | 是 | 检测到的商机信号列表 |
| customer_profile | object | 是 | 客户画像（历史购买、VIP等级等） |
| product_info | object | 否 | 相关产品信息（利润率、库存等） |
| campaign_context | object | 否 | 当前促销活动上下文 |
| scoring_config | object | 否 | 评分配置（如自定义权重） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| total_score | integer | 综合意向得分(0-100) |
| score_level | string | 意向等级(high/medium/low) |
| dimension_scores | object | 各维度子分数明细 |
| recommended_strategy | object | 推荐的后续处理策略 |
| estimated_amount | float | 预计成交金额 |
| confidence | float | 评分置信度 |

## 最佳实践

1. **动态阈值**：大促期间将高意向阈值从80降至70，捕获更多转化机会
2. **冷启动处理**：新客户缺乏历史数据时，客户维度使用群体平均值替代
3. **多次评分**：同一客户同一对话中信号增加时，允许重新评分并升级
4. **A/B测试**：新的评分规则变更通过A/B测试验证效果后再全量上线
5. **分品类校准**：不同品类的转化周期不同，评分模型按品类分别校准

## 约束条件

- 评分计算延迟<500ms
- 商机标记整体准确率>=80%
- 高意向商机推送率(1小时内)>=95%
- 评分模型按月进行一次校准
- 转化跟踪窗口：高意向7天、中意向14天、低意向30天
