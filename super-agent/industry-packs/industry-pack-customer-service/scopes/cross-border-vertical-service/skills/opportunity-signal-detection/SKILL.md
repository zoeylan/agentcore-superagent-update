# 商机信号检测 (Opportunity Signal Detection)

## 概述

商机信号检测技能负责从跨境电商客户咨询对话中实时识别购买意向信号，通过NLP语义分析和对话上下文理解，发现客户对话中隐含的购买、复购、批量采购等商业机会。该技能支持跨语言检测（翻译后的中文+原始语言的情感和语气特征），是将客服场景从成本中心向利润中心转化的关键能力。

## 适用场景

- 客户咨询过程中实时检测购买意向信号
- 促销活动期间客户反应的意向度快速判断
- 老客户复购意向的识别与标记
- B2B批量采购需求的早期识别
- 竞品比较场景中的转化机会捕获

## 详细执行步骤

### 步骤1：对话流监听与预处理
1. 实时接收客户对话消息流（已翻译为中文的版本）
2. 维护当前会话的对话上下文窗口（最近10轮对话）
3. 提取对话中的关键实体：
   - 产品/品类提及
   - 价格/数量相关表述
   - 时间紧迫性表述
   - 情感和语气信号

### 步骤2：信号模式匹配
1. 识别以下购买意向信号类型：

| 信号类型 | 关键表达示例 | 信号强度 |
|----------|------------|----------|
| 直接询价 | "多少钱"/"什么价格"/"最低多少" | 高 |
| 批量需求 | "100件有折扣吗"/"批发价多少" | 高 |
| 紧迫购买 | "今天能发吗"/"还有货吗" | 高 |
| 竞品比较 | "你们比XX好在哪"/"和XX对比" | 中 |
| 犹豫信号 | "让我想想"/"有点贵但..."/"再看看" | 中 |
| 复购意向 | "上次买的不错"/"再来一个" | 高 |
| 推荐询问 | "有没有类似的"/"推荐一下" | 中 |
| 关注细节 | 反复询问材质/尺寸/使用方法 | 低-中 |

2. 跨语言特征增强：
   - 泰语/越南语中的语气词强化判断（积极/消极）
   - 英语中的条件句型分析（"If the price is right..."）
   - 印尼语中的议价惯用表达识别

### 步骤3：上下文关联分析
1. 结合客户历史行为（来自LTM）增强信号判定：
   - 曾经购买过同品类 → 复购信号权重+20%
   - 近7天浏览过同产品 → 购买意向权重+15%
   - 历史高客单价客户 → 商机价值评估上调
2. 结合当前对话流分析：
   - 对话轮次多（>5轮）且持续询问同一产品 → 意向度提升
   - 对话中有明确拒绝表述 → 降级或排除
   - 对话中提到具体使用场景 → 意向度提升

### 步骤4：信号输出与标记
1. 检测到有效信号时，输出信号报告：
   - signal_type: 信号类型（直接询价/批量需求/竞品比较等）
   - signal_strength: 信号强度（high/medium/low）
   - key_quotes: 触发信号的关键对话句子（原始语言+翻译）
   - product_context: 相关产品/品类信息
   - timing: 检测时间点
2. 将信号传递给意向评分引擎进行综合评分
3. 单次对话可能产生多个信号，按时间序列记录

### 步骤5：反模式过滤
1. 排除以下情况的误判：
   - 客户明确表示"只是问问"/仅收藏不购买
   - 售后咨询中的产品提及（不是购买意向）
   - 投诉场景中的重复询价（是不满而非购买）
   - 机器人/自动消息的干扰
2. 对已标记"免打扰"的客户不输出商机信号

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | object | 是 | 当前消息（含翻译） |
| conversation_history | array | 是 | 对话上下文（最近10轮） |
| customer_ltm | object | 否 | 客户长期记忆信息 |
| product_catalog | object | 否 | 相关产品信息 |
| campaign_context | object | 否 | 当前促销活动信息 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| signals_detected | array | 检测到的信号列表 |
| signal_type | string | 信号类型 |
| signal_strength | string | 信号强度（high/medium/low） |
| key_quotes | array | 关键触发句 |
| confidence | float | 检测置信度 |
| recommended_action | string | 建议后续动作 |

## 最佳实践

1. **实时检测**：每条消息都进行信号检测，不等对话结束
2. **信号叠加**：同一对话中多次出现信号，累加计入意向评分
3. **时效性**：大促期间降低信号检测阈值，因为客户决策更快
4. **多信号交叉验证**：单一信号可能误判，多个不同类型信号交叉出现可信度更高
5. **持续学习**：基于最终转化结果反馈，持续优化信号识别模型

## 约束条件

- 信号检测延迟<1秒（不影响对话体验）
- 商机标记准确率>=80%（减少销售团队的无效跟进）
- 误报率<20%（避免过度打扰客户）
- 尊重客户"免打扰"意愿，标记后30天内不再检测
- 不向客户暴露任何商机标记信息
