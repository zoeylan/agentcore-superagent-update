# 客户沟通与情绪管理

## 概述
客户沟通技能用于投诉处理全过程中与客户的专业沟通，包括首次联系、进展通报、方案协商和最终确认。该技能聚焦于在管理客户情绪的同时有效传递信息、推动问题解决，是投诉挽留率的关键决定因素。

## 适用场景
- 投诉受理后的首次联系客户
- 调查过程中的进展通报
- 补偿方案的说明和协商
- 客户拒绝方案后的二次沟通
- 投诉解决后的确认回访

## 执行步骤

### 1. 沟通准备
- 回顾完整投诉信息和调查进展
- 了解客户情绪状态（参考情感分析结果）
- 确认可承诺的范围和底线
- 选择最适合的沟通渠道（电话/在线/邮件）
- 准备沟通话术框架

### 2. 开场与共情
标准沟通结构（LAST模型）：
- **L（Listen倾听）**：让客户充分表达，不打断
- **A（Acknowledge确认）**：复述客户核心诉求确认理解无误
- **S（Sorry道歉/Sympathize共情）**：对客户的不良体验表示歉意
- **T（Take action行动）**：说明将要或已经采取的行动

### 3. 信息传递
- 事实陈述：客观描述调查发现，不推卸不辩解
- 方案说明：清晰说明补偿内容和执行方式
- 预期管理：明确告知时限、流程和下一步
- 选择提供：给出选项让客户有掌控感

### 4. 异议处理
客户常见异议和应对策略：
| 异议类型 | 应对策略 |
|----------|----------|
| "补偿太少" | 解释依据 + 询问期望 + 尝试组合方案 |
| "要求道歉" | 诚恳道歉 + 说明改进措施 |
| "要找领导" | 表示理解 + 尝试自身解决 + 必要时升级 |
| "要投诉到监管" | 不阻拦 + 说明配合态度 + 继续推进处理 |
| "要求书面回复" | 确认格式 + 约定时限 + 准确执行 |

### 5. 沟通闭环
- 总结本次沟通达成的共识
- 明确下一步行动和时间承诺
- 获取客户确认（接受/需考虑/不接受）
- 完整记录沟通内容入系统

## 输入规范
```json
{
  "complaint_id": "投诉编号",
  "communication_purpose": "首次联系|进展通报|方案协商|确认回访",
  "customer_emotion_level": 1-5,
  "available_solutions": ["可提供的方案选项"],
  "constraints": ["沟通约束条件"],
  "previous_communications": ["历史沟通记录摘要"]
}
```

## 输出规范
```json
{
  "communication_record": {
    "time": "沟通时间",
    "channel": "沟通渠道",
    "duration": "时长",
    "content_summary": "内容摘要",
    "customer_attitude": "客户态度描述",
    "outcome": "accepted|considering|rejected|escalation_requested",
    "commitments_made": ["做出的承诺列表"],
    "next_action": "下一步行动",
    "follow_up_time": "下次跟进时间"
  }
}
```

## 最佳实践
- **首次联系时效**：一般投诉4小时内、重大投诉2小时内、监管投诉1小时内
- **情绪优先**：客户情绪未平复时不急于推方案，先安抚后解决
- **承诺管理**：只承诺能做到的事，承诺了必须按时兑现
- **记录完整**：每次沟通后立即录入系统，包括客户原话的关键表述
- **渠道一致**：尽量通过客户投诉的原始渠道进行后续沟通

## 约束条件
- 不得承诺超出审批权限的补偿方案
- 不得使用激化矛盾的措辞（如"这是您的问题"）
- 不得向客户透露内部流程细节或其他客户信息
- 不得在未经授权的情况下对外承认企业系统性过错
- 沟通记录必须如实记录，不可美化或隐瞒客户的负面反馈
