# 舆情监控与风险评估

## 概述
舆情监控与风险评估技能用于实时追踪外部公开平台上与品牌相关的投诉和负面信息，评估舆情传播风险，在问题扩大前进行预警和介入建议。该技能是企业声誉风险管理的前哨系统。

## 适用场景
- 日常外部平台投诉信息抓取
- 社交媒体品牌舆情实时监控
- 投诉舆情扩散风险评估
- 舆情危机预警和应对建议
- 竞品投诉舆情对标分析

## 执行步骤

### 1. 监控范围设定
重点监控平台及频率：
| 平台 | 监控频率 | 关注内容 |
|------|----------|----------|
| 12315平台 | 实时（API对接） | 新增投诉工单 |
| 黑猫投诉 | 每30分钟 | 新增投诉、已有投诉互动量变化 |
| 微博 | 每15分钟 | 品牌关键词+投诉/维权/曝光相关词 |
| 抖音/小红书 | 每30分钟 | 品牌相关维权视频/笔记 |
| 知乎 | 每小时 | 品牌相关负面问答 |
| 微信公众号/视频号 | 每小时 | 品牌相关维权文章 |

### 2. 信息抓取与录入
- 抓取投诉原文、发布时间、发布者信息
- 记录当前传播数据（阅读量、转发量、评论数）
- 识别是否为已有投诉的同一客户
- 评估发布者影响力（粉丝数、认证状态、历史维权经历）

### 3. 风险等级评估
| 风险等级 | 判定条件 | 响应要求 |
|----------|----------|----------|
| 低风险 | 普通用户、传播量<100、情绪温和 | 正常流程处理，24小时内响应 |
| 中风险 | 有一定传播（100-1000）或内容敏感 | 优先处理，4小时内响应 |
| 高风险 | 大V发布（粉丝>10万）或传播量>1000 | 紧急处理，1小时内响应+通知管理层 |
| 危机级 | 热搜/头条/媒体报道/多平台扩散 | 立即启动危机应急，管理层直接参与 |

评估维度：
- 传播力：发布者影响力 × 内容引爆潜力 × 平台算法加持概率
- 伤害力：投诉内容严重程度 × 是否涉及公众利益 × 是否有证据（截图/录音）
- 持续性：是否为系列维权 × 发布者态度（可协商/不可调和）

### 4. 预警发布
- 按风险等级触发对应通知渠道
- 预警内容：事件摘要、风险评级、传播现状、趋势预判、建议行动
- 通知对象：根据风险等级逐级扩大（客服主管→PR团队→管理层→CEO）

### 5. 应对建议
- 线上回应策略建议（是否公开回复、回复口径）
- 线下处理优先级调整（加速该客户的投诉处理）
- PR协调建议（是否需要官方声明、媒体沟通）
- 持续监控计划（加大该事件的监控频率）

```json
{
  "monitoring_config": {

    "negative_keywords": ["投诉/维权/曝光等负面关键词"],

```

## 最佳实践
- **速度第一**：外部平台发现到内部录入不超过30分钟
- **主动出击**：高风险舆情不等客户联系，主动联络客户协商解决
- **口径统一**：公开回复内容经PR审核，确保措辞专业得体
- **证据保全**：对外部投诉内容及时截图存档，防止删除后无法追溯

## 约束条件
- 不得在未经授权的情况下代表公司公开回复
- 不得对外透露内部调查进展和处理方案
- 不得试图通过非正当手段删帖或消除舆情
- 监控数据仅用于风险管理，不得用于打击报复消费者
