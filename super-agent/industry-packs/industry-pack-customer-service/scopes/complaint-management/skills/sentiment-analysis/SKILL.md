# 情感分析与情绪识别

## 概述
情感分析技能用于评估投诉客户的情绪状态和激烈程度，为投诉分级、处理优先级判定和沟通策略选择提供情绪维度的决策依据。该技能结合文本语义分析和行为模式识别，输出量化的情绪评估结果。

## 适用场景
- 投诉受理时评估客户情绪强度
- 投诉处理过程中监控客户情绪变化
- 识别高风险客户（情绪极端、有舆情扩散倾向）
- 辅助沟通策略选择（安抚优先 vs 方案优先）

## 执行步骤

### 1. 文本情感特征提取
- 识别情绪关键词：愤怒词汇（欺骗、无耻、投诉到底）、威胁词汇（法院、律师、曝光、监管）、绝望词汇（无奈、求助、没人管）
- 分析标点符号密度：大量感叹号、问号的使用
- 评估大写/加粗/重复使用（如"！！！"、重复投诉多次）
- 识别讽刺和隐性不满

### 2. 行为模式分析
- 投诉频次：短时间内多次联系
- 渠道扩散：从单一渠道扩展到多渠道
- 社交媒体活跃度：是否已在公开平台发声
- 历史交互态度变化：从温和到激烈的演变趋势

### 3. 情绪强度量化评分
| 等级 | 特征描述 | 处理建议 |
|------|----------|----------|
| 1级-轻微不满 | 语气平和，理性表达问题 | 标准流程处理 |
| 2级-明确不满 | 有抱怨情绪但可控 | 优先响应，主动沟通 |
| 3级-强烈愤怒 | 用词激烈，要求升级 | 高级专员处理，先安抚后解决 |
| 4级-极度愤怒 | 威胁投诉到监管/媒体 | 主管关注，快速响应 |
| 5级-危机状态 | 已扩散至公开渠道/监管 | 管理层介入，危机应对 |

### 4. 风险预判
- 舆情扩散概率评估（基于客户影响力、平台粉丝数）
- 升级概率预测（基于情绪强度趋势）
- 流失风险评估（基于客户价值和满意度历史）

## 输入规范
```json
{
  "text_content": "客户投诉文本内容",
  "interaction_history": ["历史交互记录数组"],
  "customer_profile": {
    "social_influence": "社交影响力指标",
    "historical_sentiment_scores": [历史情绪分数]
  }
}
```

## 输出规范
```json
{
  "emotion_intensity": 1-5,
  "primary_emotion": "anger|frustration|disappointment|desperation|neutral",
  "escalation_risk": "low|medium|high|critical",
  "public_spread_risk": "low|medium|high",
  "recommended_approach": "standard|priority|senior_handler|manager_intervention",
  "key_triggers": ["触发情绪的关键因素列表"],
  "sentiment_trend": "escalating|stable|de-escalating"
}
```

## 最佳实践
- 情绪分析与事实判定分开：客户情绪强烈不等于投诉事实成立
- 动态跟踪：在处理过程中持续监控情绪变化，及时调整策略
- 文化语境理解：理解中国消费者常用的表达习惯和隐含含义
- 多源验证：结合文字、语音语调（如有录音）多维度判断

## 约束条件
- 情绪分析结果仅作为辅助决策参考，不作为唯一分级依据
- 不对客户进行心理诊断或人格标签化
- 分析结果不对外展示给客户
- 高情绪分数（4-5级）必须触发人工复核确认
