# 合规文档编制与报送

## 概述
合规文档编制与报送技能用于按照监管机构的格式和内容要求，准备投诉处理的各类合规材料，包括消费者回复函、监管报送材料、整改报告等。该技能确保企业与监管机构的沟通专业合规，避免因材料问题导致的合规风险。

## 适用场景
- 12315平台投诉的官方回复编制
- 银保监/行业监管部门的报送材料准备
- 消费者协会调解的书面说明
- 投诉整改情况的定期报告
- 合规台账的维护和更新

## 执行步骤

### 1. 确定文档类型和要求
| 文档类型 | 格式要求 | 内容要素 | 提交时限 |
|----------|----------|----------|----------|
| 消费者回复函 | 正式公函格式 | 事件经过+处理结果+联系方式 | 7工作日内 |
| 监管情况说明 | 监管机构指定模板 | 事实描述+责任认定+处理措施+防复发措施 | 按要求时限 |
| 整改报告 | 正式报告格式 | 问题分析+整改措施+时间计划+效果承诺 | 按要求时限 |
| 合规台账 | 表格+附件形式 | 案件全部信息+处理过程+结果证明 | 实时维护 |

### 2. 材料收集与整理
- 投诉原文和客户诉求
- 调查结论和证据摘要
- 处理方案和执行结果
- 客户确认记录（满意/不满意）
- 系统性改进措施（如有）

### 3. 文档编写
**消费者回复函模板结构**：
1. 称呼和投诉确认（确认收到投诉并重视）
2. 调查结果（客观陈述调查发现）
3. 处理方案和结果（已执行的补偿/整改）
4. 后续改进承诺（避免类似问题再次发生的措施）
5. 联系方式（如有疑问的沟通渠道）

**监管报送材料要点**：
- 语言严谨，不使用含糊表述
- 事实部分有据可查
- 认错态度诚恳但不过度承诺
- 整改措施具体可行可验证
- 时间承诺必须能够兑现

### 4. 内部审核
- 法务合规审核（措辞、承诺、法律风险）
- 业务部门确认（事实准确性、可执行性）
- 管理层审批（重大投诉的对外回复）
- 修改反馈整合和终稿确认

### 5. 提交与归档
- 按要求渠道提交（平台在线回复/邮件/快递）
- 保存提交凭证（截图/回执/快递单号）
- 录入合规台账并更新案件状态
- 设置后续跟踪提醒（如需要进一步报送）

## 输入规范
```json
{
  "document_type": "consumer_reply|regulatory_report|rectification_report|compliance_ledger",
  "complaint_id": "关联投诉编号",
  "regulatory_source": "监管来源",
  "investigation_result": "调查结论",
  "resolution_details": "处理方案和结果",
  "deadline": "提交截止时间",
  "special_requirements": "特殊格式或内容要求"
}
```

## 输出规范
```json
{
  "document": {
    "type": "文档类型",
    "title": "文档标题",
    "content": "文档正文内容",
    "attachments": ["附件列表"],
    "review_status": "draft|under_review|approved|submitted",
    "submission_channel": "提交渠道",
    "submission_deadline": "提交截止时间",
    "submission_proof": "提交凭证（提交后补充）"
  }
}
```

## 最佳实践
- **提前准备**：不要等到截止日期才开始编写，预留至少2天审核时间
- **模板标准化**：维护各类文档的标准化模板，确保输出质量一致
- **用语审慎**：避免使用"绝对不会再发生"等过度承诺的表述
- **事实为本**：所有陈述必须有据可查，不编造或美化事实
- **台账实时**：合规台账随案件进展实时更新，不积压补填

## 约束条件
- 对外文件必须经法务审核后方可提交
- 不得在文件中包含其他客户的个人信息
- 不得在未确认事实的情况下对外书面承认过错
- 所有提交的文件必须保留完整副本存档
- 不得擅自修改已提交给监管机构的材料
