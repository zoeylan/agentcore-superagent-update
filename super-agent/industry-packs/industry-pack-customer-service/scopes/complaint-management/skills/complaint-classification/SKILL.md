# 投诉分类与分级

## 概述
投诉分类与分级技能用于对接收到的投诉进行标准化分类和严重程度判定。该技能是投诉管理流程的入口环节，分级结果直接决定后续处理通道、时限要求和审批权限。

## 适用场景
- 新投诉受理时的初始分级
- 工单升级为投诉时的等级判定
- 外部平台投诉录入时的分类
- 重复投诉的自动升级判定

## 执行步骤

### 1. 信息提取
- 从投诉内容中提取：投诉对象（产品/服务/人员）、事件描述、客户诉求、涉及金额
- 识别关键敏感词：资金损失、隐私泄露、人身安全、欺诈、监管、律师、媒体
- 获取投诉来源渠道信息

### 2. 客户历史关联
- 查询客户ID关联的历史投诉记录
- 判断是否为重复投诉（同一客户+同一问题类型，30天内）
- 获取客户价值等级（CLV分级）

### 3. 严重程度判定
按以下规则执行分级：

| 等级 | 判定条件 | SLA时限 |
|------|----------|---------|
| 一般投诉 | 服务态度、响应慢、流程不便等体验问题，无实质损失 | 3个工作日 |
| 重大投诉 | 资金损失>100元、隐私泄露、安全事故、重复投诉、VIP客户投诉 | 24小时响应+5个工作日 |
| 监管投诉 | 12315/银保监/黑猫投诉等外部平台受理、律师函、媒体曝光 | 2小时上报+7个工作日法定时限 |

### 4. 分类标注
- 投诉类别：产品质量/服务态度/流程问题/系统故障/价格争议/隐私安全/其他
- 涉及部门：客服/产品/技术/财务/物流/法务
- 情绪强度：1-5级（基于用词激烈程度、威胁程度）

### 5. 分派决策
- 一般投诉 → 普通处理队列
- 重大投诉 → 高级处理专员 + 通知主管
- 监管投诉 → 合规专员 + 即时通知管理层

## 输入规范
```json
{
  "complaint_source": "渠道来源(direct/escalation/external_platform/regulatory)",
  "customer_id": "客户唯一标识",
  "complaint_content": "投诉原文内容",
  "attachments": ["附件列表"],
  "received_time": "接收时间ISO格式"
}
```

## 输出规范
```json
{
  "complaint_id": "系统生成的投诉编号",
  "severity_level": "general|major|regulatory",
  "category": "投诉类别",
  "core_demand": "核心诉求摘要(100字内)",
  "emotion_intensity": 1-5,
  "is_repeat": true/false,
  "sla_deadline": "SLA截止时间",
  "assigned_channel": "分派通道",
  "escalation_required": true/false,
  "regulatory_report_deadline": "监管上报截止时间(仅监管投诉)"
}
```

## 最佳实践
- **就高原则**：当分级判定存疑时，按较高等级处理，后续可降级但不可因低判延误
- **关键词组合判定**：单一敏感词不一定触发升级，需结合上下文（如"退款"是正常诉求，"退款+欺诈+12315"则需升级）
- **时间敏感性**：分级动作必须在15分钟内完成，超时自动预警
- **证据保全**：原始投诉内容必须完整保存，不可编辑或摘要替代

## 约束条件
- 不得在分级阶段对客户做任何承诺
- 不得修改客户原始投诉内容
- 监管投诉分级不可降级，只可维持或升级
- 重复投诉判定后必须关联历史案件编号
