# 数据趋势分析与报告

## 概述
数据趋势分析与报告技能用于对投诉数据进行周期性的多维度统计分析，识别趋势变化、异常波动和新兴模式，输出结构化的分析报告为管理决策提供数据支撑。

## 适用场景
- 每周投诉数据周报编制
- 每月投诉趋势深度分析
- 投诉量异常波动时的快速诊断
- 管理层数据汇报准备
- 改进措施效果的量化评估

## 执行步骤

### 1. 数据采集与清洗
- 从投诉管理系统提取指定时间范围的全量数据
- 数据字段：投诉编号、时间、类型、等级、来源、处理时长、结果、满意度
- 清除无效数据（测试数据、重复录入、误操作）
- 补充缺失标签（未分类投诉的归类）

### 2. 多维度统计分析
**时间维度**：
- 日/周/月投诉量趋势图
- 环比（上周/上月）、同比（去年同期）变化率
- 工作日vs周末分布、时段分布（高峰时段识别）

**类型维度**：
- 各投诉类型占比和排名变化
- 新增投诉类型识别
- 各类型解决率和满意度对比

**渠道维度**：
- 各来源渠道的投诉量分布
- 外部平台投诉占比变化趋势
- 渠道间转化路径分析

**结果维度**：
- 整体解决率、满意度趋势
- 各等级投诉的SLA达标率
- 补偿金额分布和总成本趋势

### 3. 异常检测与预警
- 设置投诉量波动阈值（环比增长>30%触发预警）
- 新增高频关键词监控（可能的新问题信号）
- 特定产品线投诉集中爆发检测
- SLA达标率连续下降预警

### 4. 洞察提炼
- 从数据中提取3-5条关键洞察
- 每条洞察包含：现象描述+数据支撑+可能原因+建议行动
- 区分确定性结论和需要进一步验证的假设
- 标注需要管理层关注的重点事项

### 5. 报告生成
按标准模板输出：
- 执行摘要（管理层1分钟速览）
- 核心指标看板（KPI达成情况）
- 趋势分析（图表+解读）
- 异常预警（如有）
- 改进进展（上期建议的执行情况）
- 本期建议（需要采取的行动）

## 输入规范
```json
{
  "report_type": "weekly|monthly|ad_hoc",
  "time_range": {"start": "开始日期", "end": "结束日期"},
  "comparison_period": "对比周期",
  "focus_areas": ["重点关注的维度"],
  "audience": "management|operations|all"
}
```

## 输出规范
```json
{
  "report": {
    "summary": "执行摘要(200字内)",
    "kpi_dashboard": {
      "total_complaints": "投诉总量",
      "change_rate": "环比变化率",
      "resolution_rate": "解决率",
      "satisfaction_score": "满意度评分",
      "sla_compliance": "SLA达标率"
    },
    "trends": [{"dimension": "维度", "finding": "发现", "data_support": "数据支撑"}],
    "alerts": [{"level": "预警等级", "description": "异常描述", "recommendation": "建议"}],
    "insights": ["关键洞察列表"],
    "recommendations": [{"action": "建议行动", "priority": "优先级", "owner": "建议责任人"}]
  }
}
```

## 最佳实践
- **定时产出**：周报周一上午产出，月报次月3个工作日内产出
- **对比分析**：任何数字都不是孤立的，需要有参照系（环比/同比/目标值）
- **可视化优先**：数据用图表呈现，结论用文字描述，两者配合
- **行动导向**：报告的价值在于推动行动，每份报告至少有1条明确的行动建议

## 约束条件
- 报告数据必须与系统一致，不可人为调整
- 涉及具体客户的数据在报告中必须脱敏
- 对外共享的报告需经管理层审批
- 异常预警必须在发现后24小时内输出，不等待定期报告周期
