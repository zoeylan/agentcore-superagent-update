# 监管时限追踪与合规管理

## 概述
监管时限追踪技能用于管理投诉处理过程中涉及的各类法定时限、监管报送截止日期和合规要求。该技能确保企业投诉处理全程在法律法规允许的时间框架内完成，避免因超时导致的监管处罚和合规风险。

## 适用场景
- 监管投诉接收后的时限计算与倒计时
- 各类投诉的SLA时限监控
- 监管报送材料的截止日期管理
- 法定节假日对工作日时限的影响计算
- 合规审计时的时限达标率统计

## 执行步骤

### 1. 时限规则库匹配
根据投诉类型和监管来源，匹配适用的时限规则：

| 监管来源 | 法定回复时限 | 内部处理时限 | 上报时限 |
|----------|-------------|-------------|---------|
| 12315平台 | 7个工作日 | 5个工作日内处理完毕 | 2小时内上报管理层 |
| 银保监投诉 | 15个工作日 | 10个工作日内处理完毕 | 2小时内上报管理层 |
| 黑猫投诉 | 无法定时限，建议48小时 | 24小时内首次回复 | 4小时内通知相关人员 |
| 消协转办 | 7个工作日 | 5个工作日内处理完毕 | 2小时内上报管理层 |

### 2. 时限计算
- 确定起算时间点（以系统接收时间为准）
- 排除非工作日（法定节假日、周末，按最新国务院办公厅通知）
- 计算各关键节点的截止时间
- 设置多级预警时间点（50%时限、80%时限、90%时限）

### 3. 预警与通知
- 50%时限：系统内提醒，确认处理进展
- 80%时限：邮件+IM预警，通知主管关注
- 90%时限：短信+电话预警，升级至总监
- 超时：立即上报管理层，生成合规事件记录

### 4. 台账维护
- 记录每个监管投诉的完整时间线
- 标记各节点实际完成时间
- 计算时限消耗百分比
- 归档合规完成证据（回复截图、系统日志）

## 输入规范
```json
{
  "complaint_id": "投诉编号",
  "regulatory_source": "监管来源",
  "received_time": "接收时间",
  "complaint_type": "投诉类型",
  "industry_sector": "所属行业(影响适用法规)"
}
```


## 最佳实践
- **宁早勿晚**：实际处理完成时间应预留至少1个工作日的缓冲
- **多日历管理**：维护动态更新的工作日日历，考虑调休、补班等特殊情况
- **证据链完整**：每个时间节点都保存系统时间戳作为合规证据
- **定期对账**：每周核对监管平台状态与内部系统状态是否一致

## 约束条件
- 时限计算必须以最严格的适用标准为准
- 不得人为修改接收时间以延长时限
- 时限预警不可被静默或关闭
- 超时事件必须在合规台账中如实记录，不得隐瞒
