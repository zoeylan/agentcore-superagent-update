# 证据调查与事实还原

## 概述
证据调查技能用于投诉处理过程中的事实调查和证据收集，通过多源数据交叉验证还原投诉事件的完整经过，为责任判定和补偿方案制定提供客观事实依据。

## 适用场景
- 投诉事实存在争议时的调查取证
- 涉及资金损失的投诉责任判定
- 服务态度投诉的录音/记录回溯
- 系统故障导致的问题复现和时间线梳理
- 为监管回复准备事实证据

## 执行步骤

### 1. 证据源识别
根据投诉类型确定需要调取的证据来源：

| 投诉类型 | 主要证据源 | 辅助证据源 |
|----------|-----------|-----------|
| 服务态度 | 通话录音、聊天记录 | 坐席工作日志、质检评分 |
| 资金损失 | 交易流水、支付记录 | 系统操作日志、接口调用记录 |
| 产品质量 | 订单信息、物流记录 | 产品批次记录、质检报告 |
| 隐私泄露 | 数据访问日志、授权记录 | 系统安全审计日志 |
| 系统故障 | 系统日志、告警记录 | 变更发布记录、影响范围评估 |

### 2. 证据调取
- 通话录音：按投诉涉及的时间范围调取完整录音
- 聊天记录：导出完整对话上下文（含系统消息）
- 系统日志：按时间线抽取相关操作记录
- 交易记录：调取涉及的全部交易明细和状态变更

### 3. 时间线重建
- 按时间顺序排列所有证据
- 标注关键时间节点和状态变更
- 识别证据之间的逻辑关系和矛盾点
- 生成事件时间线图

### 4. 事实判定
- 交叉验证多源证据的一致性
- 识别证据缺失点和无法验证的部分
- 基于证据形成事实结论
- 标注结论的确定程度（确认/高度可能/存疑）

### 5. 责任归属
- 企业责任：服务失误、系统故障、产品缺陷
- 客户责任：操作失误、理解偏差、违规使用
- 第三方责任：物流损坏、支付系统故障
- 混合责任：双方均有过失的情况

## 输入规范
```json
{
  "complaint_id": "投诉编号",
  "complaint_summary": "投诉摘要",
  "time_range": {"start": "调查起始时间", "end": "调查结束时间"},
  "customer_id": "客户ID",
  "related_order_ids": ["相关订单号"],
  "investigation_focus": "调查重点"
}
```

## 输出规范
```json
{
  "investigation_report": {
    "timeline": [{"time": "时间", "event": "事件描述", "evidence_source": "证据来源"}],
    "findings": "事实发现摘要",
    "responsibility": "enterprise|customer|third_party|mixed",
    "confidence_level": "confirmed|highly_probable|uncertain",
    "evidence_list": [{"type": "证据类型", "source": "来源", "summary": "内容摘要"}],
    "gaps": ["证据缺失点"]
  }
}
```

## 最佳实践
- **时效性**：证据调取应在投诉受理后4小时内启动，部分证据（如录音）有保留期限
- **完整性**：调取证据时不可选择性采集，必须获取完整上下文
- **客观性**：事实判定基于证据而非推测，存疑部分明确标注
- **保全措施**：关键证据及时备份，防止自动清理或覆盖

## 约束条件
- 证据调取必须在授权范围内，涉及个人信息需遵守《个人信息保护法》
- 不得篡改、删除或隐瞒已获取的证据
- 录音调取需符合内部数据安全管理规定
- 调查结论应区分事实和推断，不可混为一谈
