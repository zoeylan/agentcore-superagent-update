# 补偿审批流程管理

## 概述
补偿审批流程管理技能用于处理超出一线权限的补偿方案的审批流转，确保审批过程高效、合规，在权限控制和处理时效之间取得平衡。该技能管理从审批发起到最终决策的完整流程。

## 适用场景
- 补偿金额超出一线客服权限时发起审批
- 审批驳回后的方案调整和重新提交
- 紧急投诉的加急审批流程
- 监管投诉的特事特办审批通道
- 批量投诉的集中审批处理

## 执行步骤

### 1. 审批材料准备
- 投诉案件摘要（投诉原因、客户诉求、调查结论）
- 补偿方案详情（工具选择、金额计算、依据说明）
- 客户信息（价值等级、历史投诉、流失风险评估）
- 风险评估（不处理的后果预估：舆情风险、监管风险、流失损失）
- 对比参考（同类投诉历史处理标准）

### 2. 审批层级确定
```
补偿价值 ≤100元   → 系统自动审批
补偿价值 101-500元 → 提交组长审批
补偿价值 501-2000元 → 提交主管审批
补偿价值 >2000元   → 提交总监审批
特殊情况（监管投诉/舆情风险）→ 可跳级至总监直接审批
```

### 3. 审批流转
- 生成审批工单并推送至审批人
- 设置审批时限提醒（普通2小时内、紧急1小时内）
- 提供一键审批操作（通过/驳回/调整金额后通过）
- 审批人不在线时自动转发至代理审批人

### 4. 审批结果处理
- **通过**：通知投诉处理Agent执行方案，记录审批人和审批时间
- **驳回**：反馈驳回原因，触发方案调整流程
- **有条件通过**：审批人调整金额后通过，按新金额执行
- **超时未审批**：自动升级至上一级审批人

### 5. 审计追踪
- 记录完整审批链（申请人→审批人→决策→时间）
- 标记异常审批（加急审批、跳级审批、超时审批）
- 月度审批数据统计（通过率、平均审批时长、驳回原因分布）


```json
    "rejection_reason": "驳回原因（如被驳回）",
```

## 最佳实践
- **材料完整一次通过**：审批材料不完整是驳回的主要原因，首次提交力求完整
- **预判升级**：如客户诉求明显超出自身权限，早期即可准备审批材料
- **紧急通道善用**：监管投诉和舆情风险投诉使用紧急通道，日常投诉不滥用
- **驳回不等于终止**：驳回后应分析原因，调整方案重新提交

## 约束条件
- 不得拆分补偿金额以规避审批层级（如将600元拆为两笔300元）
- 审批人不得审批自己发起的补偿方案（利益回避）
- 紧急审批事后必须补完完整审批材料
- 审批记录不可删除或修改
