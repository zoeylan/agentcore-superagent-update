# AI准确率监控 (AI Accuracy Monitoring)

## 技能概述

AI准确率监控技能专注于追踪AI客服系统基于知识库内容生成回答的准确性和有效性，快速发现AI回答偏差，区分偏差原因（知识内容问题vs AI理解问题），并推动针对性修正。确保AI回答准确率>=95%。

## 适用场景

- AI自动回复准确率日常监控
- AI回答被客户否定或被坐席覆盖后的归因分析
- 知识库内容更新后的AI效果验证
- 新增知识内容上线后的AI调用效果观察
- AI准确率下降的专项排查
- 知识库内容与AI回答一致性校验

## 执行步骤

### 步骤1: 监控数据采集
- 实时采集AI回答效果数据：
  - AI回答采纳率（客户确认解决/坐席采用）
  - AI回答后人工接管率（Handoff Rate）——越高说明AI回答越不满意
  - 客户对AI回答的评价（满意/不满意/未评价）
  - 坐席对AI推荐答案的采纳/修改/否定操作
- 关联AI回答与其引用的知识库文章ID
- 按知识类型、产品线、问题类型分维度统计

### 步骤2: 偏差检测与预警
- 设定预警阈值：
  - 单篇文章相关AI回答准确率<90%时触发预警
  - 单日某类型问题AI接管率突增>10%时触发预警
  - 单篇文章连续3次被坐席否定时立即预警
- 偏差分类：
  - **内容偏差**：AI回答引用了过时/错误的知识内容
  - **理解偏差**：知识内容正确，但AI理解/摘要不当
  - **匹配偏差**：AI匹配了不相关的知识文章
  - **缺失偏差**：客户问题在知识库中无对应内容，AI强行回答

### 步骤3: 根因分析与定位
- 对预警事件进行逐案分析：
  - 调取AI回答原文 vs 知识库源文 进行对比
  - 分析客户原始问题 vs AI匹配的知识条目 是否相关
  - 检查知识库内容本身是否存在准确性/完整性问题
- 归因判定：
  - 知识内容问题 → 推送给内容修正流程（内容生命周期Agent）
  - AI理解问题 → 生成AI调优工单给AI团队
  - 知识匹配问题 → 推送给搜索优化流程（标签/关键词优化）
  - 知识缺失问题 → 推送给内容需求分析流程

### 步骤4: 效果验证与报告
- 修正措施实施后追踪效果：
  - 修正前3天 vs 修正后3天的准确率对比
  - 确认偏差不再复现
  - 如效果不达预期需二次分析
- 周度/月度输出AI准确率报告：
  - 整体准确率趋势
  - 各维度偏差分布（内容/理解/匹配/缺失）
  - TOP问题文章清单
  - 已修正问题效果验证结果

## 输入规格

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| AI回答日志 | 日志数据 | AI客服系统 | 含回答内容、引用文章、效果标记 |
| 人工接管记录 | 事件数据 | 坐席系统 | 含接管原因和坐席修正内容 |
| 客户反馈数据 | 评价数据 | 客户端 | 对AI回答的满意度评价 |
| 知识库文章内容 | 内容数据 | 知识库系统 | 用于与AI回答对比 |
| AI模型配置 | 配置信息 | AI团队 | 当前模型版本和参数 |

## 输出规格

| 输出项 | 格式 | 接收方 | 说明 |
|--------|------|--------|------|
| AI准确率预警 | 即时通知 | 内容团队/AI团队 | 含偏差类型和紧急度 |
| 偏差归因报告 | 分析文档 | 对应处理方 | 含根因和修正建议 |
| AI效果周报/月报 | 仪表板+报告 | 管理层 | 整体趋势+TOP问题 |
| 内容修正工单 | 任务工单 | 内容生命周期Agent | 含具体修正要求 |
| AI调优工单 | 任务工单 | AI团队 | 含case和优化方向 |

## 最佳实践

1. **快速响应**：AI准确率问题影响面广（每个错误答案可能影响数十客户），发现即处理
2. **Case积累**：保存典型偏差案例库，用于AI模型训练和内容编写参考
3. **联合排查**：AI团队和内容团队应建立联合排查机制，避免互相推诿
4. **预防性验证**：重要文章更新后主动测试AI是否能正确使用新内容

## 约束条件

- AI准确率数据需排除客户问题本身模糊不清的情况（非AI/知识库问题）
- 偏差归因必须基于具体case分析，不可仅凭统计数据下结论
- 推送给AI团队的调优工单需附带足够的上下文和case
- 不可因追求准确率而限制AI回答范围（应提升质量而非减少回答量）
