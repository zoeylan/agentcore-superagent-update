# 质量评估 (Quality Assessment)

## 技能概述

质量评估技能负责对知识库内容进行系统性的质量评价，建立内容质量评分体系，识别低质量内容并推动改进。通过定量评估（指标）和定性评估（抽检）相结合的方式，持续保障知识库整体内容质量水平。

## 适用场景

- 月度知识库内容质量评估
- 新发布内容的质量验收
- 低效文章（高访问低解决率）的深度质量分析
- 坐席反馈"内容不准确/不完整"后的专项核查
- 季度审查中的质量维度评估
- 知识库质量等级认证

## 执行步骤

### 步骤1: 质量评分模型定义
- 五维度评分体系（各维度1-5分）：
  - **准确性（Accuracy）**：内容与当前业务实际是否一致，无事实错误
  - **完整性（Completeness）**：是否涵盖该主题所有必要信息，无关键遗漏
  - **可操作性（Actionability）**：读者能否依据内容独立完成操作或解决问题
  - **规范性（Compliance）**：是否符合模板要求和格式标准
  - **时效性（Timeliness）**：内容是否反映最新状态，无过时信息
- 综合质量分 = 准确性×0.3 + 完整性×0.25 + 可操作性×0.25 + 规范性×0.1 + 时效性×0.1
- 质量等级：A（>=4.5）、B（>=3.5）、C（>=2.5）、D（<2.5，需立即修正）

### 步骤2: 自动化质量检测
- 格式规范性自动检查：
  - 元数据完整性（适用产品/场景/有效期/责任人）
  - 模板结构合规（是否包含必要章节）
  - 文字规范（错别字、标点、专业术语）
- 时效性自动检测：
  - 引用的产品版本是否为当前版本
  - 提到的价格/政策是否有更新
  - 截图/界面是否与现行版本一致
- 关联性检测：
  - 是否存在死链接（引用的其他文章已删除/归档）
  - 关联推荐是否仍然相关

### 步骤3: 人工抽检与深度评审
- 月度抽检覆盖率>=10%的活跃文章
- 抽检侧重：高访问文章（TOP50）、新发布文章（本月）、被标记问题文章
- 深度评审关注：操作步骤实际可行性、边界条件是否涵盖、异议处理是否充分
- 邀请一线坐席参与评审（最了解实际使用体验）

### 步骤4: 质量问题处理与改进
- D级文章：立即下架或标记警告，48小时内修正
- C级文章：加入优化队列，2周内完成提升
- B级文章：记录优化建议，按优先级安排
- 质量趋势分析：识别质量系统性下降的领域（某产品线/某编写者）
- 输出月度质量报告，含改进措施跟踪

## 输入规格

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 待评估文章 | 知识内容 | 知识库系统 | 含完整内容和元数据 |
| 使用效果数据 | 指标数据 | 使用数据分析 | 解决率、反馈等 |
| 坐席反馈记录 | 反馈数据 | 坐席工作台 | 负面反馈详情 |
| 产品当前状态 | 参考信息 | 产品团队 | 用于核实准确性 |
| 格式规范标准 | 规则文档 | 质量标准库 | 各类型模板要求 |

## 输出规格

| 输出项 | 格式 | 接收方 | 说明 |
|--------|------|--------|------|
| 文章质量评分 | 评分卡 | 知识库系统 | 五维度+综合分 |
| 低质量文章清单 | 排序列表 | 内容生命周期Agent | 含问题描述和修改要求 |
| 月度质量报告 | 分析报告 | 管理层 | 趋势+分布+改进建议 |
| 质量改进跟踪表 | 台账 | 内部管理 | 问题→措施→效果 |

## 最佳实践

1. **评估标准公开透明**：让编写者知道评估维度和标准，从源头提升质量
2. **正向激励**：对持续A级的文章和编写者给予认可，不仅关注问题
3. **样本代表性**：抽检样本应覆盖各产品线、各类型、各编写者，避免偏差
4. **一致性校准**：多人评估同一文章时定期校准，确保标准统一

## 约束条件

- 质量评估结果不直接用于个人绩效考核（避免抵触情绪）
- D级文章处理不可超过48小时
- 自动检测结果需人工确认后才可执行下架操作
- 评估数据保留至少12个月，支持趋势分析
