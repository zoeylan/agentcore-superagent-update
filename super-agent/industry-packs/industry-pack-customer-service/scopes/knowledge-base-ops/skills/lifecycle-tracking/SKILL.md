# 生命周期跟踪 (Lifecycle Tracking)

## 技能概述

生命周期跟踪技能负责监控知识库中每篇文章从创建到归档的完整状态变迁，管理版本历史，执行有效期监控，并在季度审查中确保全量内容的时效性和准确性。

## 适用场景

- 新文章进入创建流程后的进度跟踪
- 文章有效期到期前的预警和review触发
- 季度全量审查的计划制定和执行跟踪
- 内容版本变更的记录和审计
- 过期内容的归档或更新决策

## 执行步骤

### 步骤1: 状态监控与流转
- 维护文章状态机：草稿 → 待审核 → 审核中 → 待发布 → 已发布 → 到期预警 → 待审查 → 已归档
- 每日扫描异常状态：停留在某状态超过规定时间的文章（如"待审核"超过2工作日）
- 对异常状态文章自动发送催办通知给责任人
- 记录每次状态变迁的时间戳、操作人和变更原因

### 步骤2: 有效期管理
- 文章发布时必须设定有效期（FAQ: 6个月、操作指南: 3个月、产品文档: 跟随版本、话术模板: 6个月、排查手册: 3个月）
- 有效期到期前30天触发第一次预警（通知责任人计划review）
- 有效期到期前7天触发第二次预警（如未响应则升级通知组长）
- 到期当天如仍未review，文章标记为"待确认"状态，搜索权重降低
- 超期30天未处理的文章自动归档并从AI训练集中移除

### 步骤3: 版本管理
- 每次内容修改必须创建新版本，保留完整历史
- 版本记录包含：版本号、修改时间、修改人、修改原因、变更摘要
- 支持任意版本间的差异对比（diff）
- 重大修改（内容逻辑变化）需要重新走审核流程
- 轻微修改（错别字/格式）可以快速发布但需记录

### 步骤4: 季度全面审查
- 每季度第一个月启动全量审查计划
- 按产品线/分类分配审查任务给对应SME
- 审查内容：准确性（与当前业务一致）、时效性（信息未过时）、完整性（无缺失环节）、可用性（是否仍有使用价值）
- 审查结果分为：确认有效（续期）、需要修订（进入修改流程）、建议归档（无使用价值）
- 审查覆盖率目标100%，过期内容处理率>=95%

## 输入规格

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 文章元数据 | 结构化数据 | 知识库系统 | 含状态、有效期、责任人、版本号 |
| 状态变更事件 | 系统事件 | 知识库系统 | 实时触发的状态变迁通知 |
| 审查任务分配表 | 表格 | 内容生命周期Agent | 季度审查分配计划 |
| 产品版本发布记录 | 日志 | 产品团队 | 关联文档版本管理 |

## 输出规格

| 输出项 | 格式 | 接收方 | 说明 |
|--------|------|--------|------|
| 有效期预警通知 | 消息/邮件 | 文章责任人 | 含到期时间、所需操作 |
| 异常状态报告 | 列表 | 内容生命周期Agent | 超时停留文章清单 |
| 季度审查进度报告 | 仪表板 | 管理层 | 审查覆盖率、处理结果分布 |
| 版本变更日志 | 时间线 | 审计需求方 | 完整变更历史 |

## 最佳实践

1. **自动化优先**：有效期预警、状态超时检测应全部自动化，减少人工巡检成本
2. **责任人制度**：每篇文章必须有且仅有一个主责任人，责任人离职时必须交接
3. **季度审查分批**：不要集中在一周内完成，按产品线分3-4周逐批进行，避免SME负担过重
4. **归档不删除**：归档文章移出检索但保留存储，以备追溯和参考

## 约束条件

- 版本记录不可删除或篡改
- 有效期不可无限延长，最长不超过12个月
- 季度审查不可跳过，如因资源问题需延期须管理层审批
- 自动归档前必须通知责任人，给予7天异议期
