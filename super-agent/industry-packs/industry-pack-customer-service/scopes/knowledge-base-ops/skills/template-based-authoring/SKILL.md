# 模板化编写 (Template-Based Authoring)

## 技能概述

模板化编写技能为不同类型的知识库内容提供标准化的创作框架和编写指导，确保所有知识内容遵循一致的结构、格式和质量标准。通过模板驱动的方式降低编写门槛、提升一致性、加速内容产出。

## 适用场景

- 新FAQ条目的创建
- 操作指南/分步教程的编写
- 故障排查手册/决策树的制作
- 话术模板/沟通脚本的编写
- 产品功能文档的撰写
- 将非结构化素材转化为标准知识库格式

## 执行步骤

### 步骤1: 内容类型判定与模板选择
- 根据内容需求工单中的类型标注选择对应模板：
  - **FAQ模板**：适用于有明确问答对的高频问题
  - **操作指南模板**：适用于需要分步骤完成的操作类内容
  - **故障排查模板**：适用于需要条件判断逐步排错的场景
  - **话术模板**：适用于客服沟通场景的标准用语
  - **产品文档模板**：适用于功能说明、定价、政策等产品信息

### 步骤2: 模板结构填充指导

**FAQ模板结构**：
```
标题：[客户可能的提问方式]
简答：[一句话核心答案，50字以内]
详答：[完整解答，含背景说明和注意事项]
适用条件：[什么情况下此答案有效]
例外情况：[什么情况下此答案不适用]
关联问题：[2-3个常一起问的相关问题链接]
```

**操作指南模板结构**：
```
标题：[操作目标，动宾短语]
前置条件：[执行此操作前需要满足的条件]
操作步骤：[编号列表，每步一个动作]
  步骤N：[动作描述] + [预期结果] + [截图位]
注意事项：[容易出错的点]
常见问题：[操作中可能遇到的问题及处理]
```

**故障排查模板结构**：
```
标题：[故障症状描述]
症状确认：[如何判断确实是此故障]
排查步骤：[决策树格式]
  检查点1 → [是] → 检查点2
            → [否] → 解决方案A
解决方案：[每个终端节点的解决方法]
升级条件：[什么情况下需要升级处理]
```

**话术模板结构**：
```
场景名称：[沟通场景描述]
使用时机：[什么情况下使用此话术]
开场白：[建立沟通的开头语]
核心话术：[关键信息传递的标准表达]
异议处理：[客户常见反对意见的应对]
收尾语：[结束沟通的标准方式]
禁忌：[绝对不能说的话]
```

### 步骤3: 编写规范执行
- 语言规范：使用第二人称（"您"）面向客户的内容，使用指令式面向坐席的内容
- 术语规范：严格使用标准术语表中的用语，首次出现专业术语需括号注释
- 长度规范：FAQ简答<=50字，详答<=500字；操作指南每步<=100字；话术每段<=200字
- 可读性规范：避免长句（每句<=30字）、使用列表优于大段文字、关键信息加粗

### 步骤4: 元数据补全
- 必填元数据检查清单：
  - ☐ 适用产品（产品名+版本范围）
  - ☐ 适用场景（什么情况下使用）
  - ☐ 有效期（根据内容类型设定默认值）
  - ☐ 责任人（内容准确性的第一责任人）
  - ☐ 内容类型标签
  - ☐ 关键词（3-5个，用于搜索命中）
  - ☐ 创建日期和来源说明

## 输入规格

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 内容需求工单 | 结构化表单 | 内容生命周期Agent | 含类型、主题、素材来源 |
| 原始素材 | 非结构化 | 业务专家/工单系统 | 对话记录、解决方案、产品文档 |
| 标准术语表 | 词典 | 质量标准库 | 统一用语规范 |
| 现有相关文章 | 参考内容 | 知识库系统 | 避免重复和保持一致 |

## 输出规格

| 输出项 | 格式 | 接收方 | 说明 |
|--------|------|--------|------|
| 知识库文章草稿 | 结构化文档 | 审核流程 | 符合模板的完整内容 |
| 编写建议 | 指导说明 | 业务专家 | 帮助专家补充专业信息 |
| 模板合规检查结果 | 检查报告 | 编写者 | 标注不合规的具体位置 |

## 最佳实践

1. **模板不是限制**：模板提供结构框架，具体内容仍需业务专家的专业判断
2. **先骨架后填充**：先按模板搭好结构，再逐块填充内容，比自由写作效率高50%
3. **复用优于重写**：编写前先检索是否有可以扩展/修改的现有文章
4. **一篇一问**：FAQ类内容严格每篇回答一个问题，复合问题拆分为多篇并互相关联

## 约束条件

- 所有文章必须使用指定模板，不允许自由格式
- 元数据缺失任一必填项的文章不可提交审核
- 内容不可包含个人联系方式、内部系统地址等敏感信息
- 操作指南中的步骤必须经过实际验证
