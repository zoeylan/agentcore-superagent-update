# 使用数据分析 (Usage Data Analysis)

## 技能概述

使用数据分析技能负责系统性采集、处理和分析知识库的使用行为数据，通过多维度指标揭示知识库的运行效能，识别内容质量问题和覆盖缺口，为内容优化和运营决策提供数据支撑。

## 适用场景

- 周度/月度知识库健康度评估
- 低效文章识别和诊断（高访问低解决）
- 缺失内容发现（高搜索零命中）
- AI自动回复效果归因分析
- 坐席使用行为分析和知识推荐优化
- 内容ROI评估（编写投入vs解决问题量）

## 执行步骤

### 步骤1: 数据采集与清洗
- 采集维度：
  - 搜索行为：搜索关键词、搜索频次、搜索无结果率、搜索后点击率
  - 文章使用：PV/UV、停留时间、跳出率、关联跳转、收藏/分享次数
  - 反馈信号：点赞/点踩比例、评论内容、"未解决问题"标记
  - AI调用：知识库命中率、AI回复采纳率、AI回复后人工接管率
  - 坐席行为：复制引用率、修改后使用率、反馈"内容过时"次数
- 数据清洗：去除异常值（如自动化脚本产生的批量访问）、归一化不同渠道数据
- 时间粒度：实时监控关键指标、日度汇总趋势、周度/月度分析报告

### 步骤2: 指标计算与基线建立
- 核心指标体系：
  - **搜索命中率** = (搜索有结果次数 / 总搜索次数) × 100%，目标>=90%
  - **文章解决率** = (标记"已解决" / 文章总访问) × 100%，目标>=80%
  - **AI回答准确率** = (客户确认正确 / AI基于知识库回答总数) × 100%，目标>=95%
  - **知识库活跃度** = 月内被访问文章数 / 总文章数，健康值>=60%
  - **内容新鲜度** = 3个月内更新文章数 / 总文章数，健康值>=70%
- 建立各指标的历史基线和预警阈值
- 区分自然波动和异常变化

### 步骤3: 问题诊断与归因
- **低效文章诊断**：
  - 高访问+低解决 → 内容质量问题（不准确/不完整/不清晰）
  - 高访问+高跳出 → 标题误导或内容不匹配搜索意图
  - 低访问+高解决 → 检索优化问题（好内容搜不到）
- **搜索失败归因**：
  - 内容缺失（知识库确实没有相关内容）
  - 检索失败（有内容但搜索引擎未命中，需优化同义词/标签）
  - 表达差异（用户搜索词与文章用语不同）
- **AI偏差归因**：
  - 知识内容本身有误 → 推送给内容修正流程
  - 知识正确但AI理解偏差 → 推送给AI团队调优

### 步骤4: 洞察输出与行动建议
- 生成优先级排序的优化建议清单
- 每条建议包含：问题描述、数据支撑、影响评估、建议操作、预期效果
- 区分"快速修复"（如补充同义词即可解决）和"深度优化"（需重写文章）
- 跟踪优化措施执行后的效果验证（前后对比）

## 输入规格

| 输入项 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| 搜索日志 | 日志数据 | 知识库搜索引擎 | 含关键词、结果数、点击 |
| 文章访问日志 | 日志数据 | 知识库系统 | 含PV/时长/反馈 |
| AI调用日志 | 日志数据 | AI客服系统 | 含命中文章、回复、结果 |
| 坐席反馈数据 | 结构化数据 | 坐席工作台 | 含评价、标记、建议 |
| 工单解决数据 | 关联数据 | 工单系统 | 关联知识库使用的工单结果 |

## 输出规格

| 输出项 | 格式 | 接收方 | 说明 |
|--------|------|--------|------|
| 知识库健康度报告 | 仪表板+报告 | 管理层 | 核心指标趋势+问题摘要 |
| 低效文章清单 | 排序列表 | 内容生命周期Agent | 含诊断结论和优化方向 |
| 缺失内容需求 | 需求列表 | 内容需求分析流程 | 含搜索频次和优先级 |
| AI偏差报告 | 专项报告 | AI团队/内容团队 | 含归因和建议 |

## 最佳实践

1. **数据驱动决策**：所有结论必须有数据支撑，避免基于个案推断
2. **趋势比点重要**：关注指标变化趋势，短期波动需结合业务事件解读
3. **分层分析**：按产品线、内容类型、客户群分别分析，避免平均值掩盖问题
4. **闭环验证**：优化措施实施后必须跟踪效果，验证假设是否成立

## 约束条件

- 数据分析不可涉及个人隐私信息（如具体客户身份）
- 报告中使用聚合数据，不暴露单个坐席的负面数据（绩效反馈走专门渠道）
- 数据保留周期遵守公司数据治理策略
- 指标异常时先排除数据采集故障再下结论
