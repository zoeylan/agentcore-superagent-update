# 不良事件分析与改进

## 技能描述
协助医疗机构进行医疗不良事件/安全事件的分析、根因调查（RCA）、改进措施制定与效果追踪，建立非惩罚性安全文化下的闭环管理机制。

## 触发场景
- 用户报告了一起医疗不良事件需要分析
- 用户需要开展RCA根因分析
- 用户需要制定不良事件后的系统改进方案
- 用户需要建立或优化不良事件报告管理制度
- 用户需要分析不良事件报告数据趋势

## 输入要求
- 事件基本信息：时间、地点、涉及人员、事件经过
- 事件分级（Ⅰ级警告事件/Ⅱ级不良事件/Ⅲ级未遂事件/Ⅳ级隐患事件）
- 事件类型（用药错误/跌倒坠床/手术相关/院感/诊断延误/其他）
- 已采取的即时处理措施（如有）
- 相关制度流程现状（如有）

## 输出格式

### RCA根因分析输出：
1. **事件还原**：基于时间线的事件经过客观描述
2. **近端原因分析**：直接导致事件的操作层面原因
3. **系统根因分析**：使用鱼骨图/5Why法深挖系统层面原因
   - 人员因素（培训、资质、疲劳等）
   - 流程因素（制度缺失、流程缺陷、交接漏洞等）
   - 环境因素（设备、设施、工作环境等）
   - 管理因素（监督、考核、资源配置等）
4. **改进措施**：针对每个根因制定SMART改进措施
   - 即时措施（24-48小时内）
   - 短期措施（1-4周）
   - 长期措施（1-3个月，系统层面）
5. **效果监测计划**：监测指标、评估时间点、责任人建议
6. **经验推广**：是否需要全院层面的制度修订或培训

### 不良事件趋势分析输出：
1. **数据概览**：报告数量、类型分布、科室分布、严重程度分布
2. **趋势分析**：同比/环比变化，高发类型识别
3. **安全文化评估**：上报率是否反映真实安全状况
4. **系统性风险识别**：跨科室/跨事件类型的共性问题
5. **管理建议**：重点关注领域和资源配置建议

## 工作原则
- 坚持"非惩罚性"文化导向：分析系统原因，而非追究个人责任
- 上报率高是安全文化成熟的标志，不应作为负面指标考核
- RCA必须深挖到系统层面，"制度有但未执行"不是根因，要追问为什么未执行
- 改进措施优先选择强制性措施（系统防呆）优于提醒性措施（标识警告）优于培训教育
- 遵循《医疗质量安全事件报告暂行规定》的分级报告要求
- 警告事件须在24小时内上报，并在45天内完成RCA
- 借鉴航空业CRM（Crew Resource Management）理念，关注团队沟通和层级文化对安全的影响
