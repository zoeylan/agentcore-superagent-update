# 危急值报告 (Critical Value Alert)

## 概述
本技能用于实时监测检验检查结果中的危急值（Critical Values），一旦识别到危急值立即触发分级通知流程，确保主管医生在30分钟内获知并做出处理。危急值是指可能危及患者生命的极端异常检验结果，延迟处理可能导致严重不良后果。

## 使用场景
- LIS系统报告检验结果时的实时监测
- PACS系统影像报告中的危急值发现
- 床旁即时检验(POCT)结果上传时
- 跨系统危急值结果汇总和确认追踪

## 执行步骤

### 步骤1：危急值识别
- 实时接收检验检查结果数据流
- 对比危急值阈值列表进行匹配：
  | 项目 | 低值 | 高值 |
  |------|------|------|
  | 血钾 | <2.5 mmol/L | >6.5 mmol/L |
  | 血钠 | <115 mmol/L | >160 mmol/L |
  | 血糖 | <2.8 mmol/L | >33.3 mmol/L |
  | 血小板 | <30×10⁹/L | >1000×10⁹/L |
  | WBC | <2.0×10⁹/L | >30.0×10⁹/L |
  | 血红蛋白 | <50 g/L | >200 g/L |
  | PT/INR | — | INR>5.0 |
  | 肌钙蛋白 | — | >正常上限10倍 |
  | 血培养 | 阳性 | — |
- 支持自定义危急值阈值（按科室/病种可调整）

### 步骤2：确认与去重
- 与近24小时内同一患者同一项目的报告对比
- 若已有该项目的危急值记录且已确认处理，标注为"已知危急值"
- 若为新出现的危急值或既往未确认的，立即触发通知
- 自动排除明显的标本异常（如脂血、溶血导致的假性异常——仅标记建议复查）

### 步骤3：分级通知触发
- 时间节点0：危急值识别后立即发起通知
- 一级通知（0-5分钟）：
  - 系统自动推送至主管医生工作站（弹窗+声音提醒）
  - 同时发送短信/院内通讯工具消息
- 二级通知（15分钟未确认）：
  - 升级通知科主任/值班上级医师
  - 电话通知主管医生
- 三级通知（30分钟未确认）：
  - 通知医务部值班人员
  - 记录为"危急值通知超时事件"

### 步骤4：确认与记录
- 医生确认后记录：
  - 确认时间
  - 确认人（姓名+工号）
  - 初步处理措施
  - 是否需要复查
- 系统记录完整的通知时间线用于质控审计

### 步骤5：闭环管理
- 追踪处理措施的执行情况
- 复查结果回报后更新危急值状态
- 生成危急值处理完整记录存入患者病历

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| result_data | object | 是 | 检验/检查结果数据 |
| patient_id | string | 是 | 患者标识 |
| ordering_physician | string | 是 | 开单医生标识 |
| attending_physician | string | 是 | 主管医生标识 |
| department | string | 是 | 所在科室 |
| result_timestamp | datetime | 是 | 结果产生时间 |

## 最佳实践
- 危急值通知及时率目标≥99%（30分钟内确认）
- 通知方式多通道并行（系统弹窗+短信+电话）确保送达
- 危急值列表需定期（每年至少一次）由医务部审核更新
- 建立危急值月报制度，分析发生频率和处理时效
- 对于ICU等特殊科室可设置更严格的阈值

## 约束与限制
- 系统不对危急值做出处理建议（仅提供临床意义描述）
- 不可因为任何原因延迟或抑制危急值通知
- 通知记录不可修改或删除（审计追溯要求）
- 对于标本问题导致的假性危急值，需复查确认后方可关闭
- 任何通知超时事件自动纳入质量改进议题
