# 风险标识 (Risk Identification)

## 概述
本技能用于从采集的病史信息中自动识别和标注对诊疗安全有重要影响的风险项，包括药物过敏、重大病史、传染病、特殊人群状态等，确保医生在接诊时第一时间获知关键安全信息。

## 使用场景
- 病史采集完成后的自动风险扫描
- 患者信息更新时的增量风险评估
- 诊前摘要中的风险项突出标注
- 处方开具前的过敏风险预校验

## 执行步骤

### 步骤1：过敏风险识别
- 扫描患者过敏史记录
- 匹配已知过敏原与常用药物的交叉反应关系
  - 如：青霉素过敏→头孢类交叉过敏风险评估
  - 如：磺胺过敏→含磺胺基团药物标注
- 食物过敏中可能影响用药的项目（如花生油基质药物）
- 标注过敏严重程度等级：
  - 红色（高危）：过敏性休克史、喉头水肿史
  - 黄色（中危）：皮疹、荨麻疹等非致命反应
  - 蓝色（低危）：胃肠道不适等轻微反应

### 步骤2：重大病史风险识别
- 识别影响诊疗决策的重大既往史：
  - 恶性肿瘤（需关注转移风险和治疗方案选择）
  - 器官移植（免疫抑制状态，用药禁忌多）
  - 心脑血管事件（PCI/CABG/脑梗/脑出血）
  - 肝硬化/慢性肾病（影响药物代谢）
  - 凝血功能障碍
- 标注与当次就诊可能的关联性

### 步骤3：传染病风险识别
- 标注已知传染病诊断（乙肝/丙肝/HIV/结核/梅毒等）
- 评估传播风险等级和防护要求
- 触发院感防控相关提示
- 标注为红色风险（需医护人员特殊防护）

### 步骤4：特殊人群标识
- 妊娠期/哺乳期（大量用药禁忌）
- 儿童（需按体重/年龄调整剂量）
- 老年人（多重用药风险、肝肾功能减退）
- 肝肾功能不全（需剂量调整）
- 精神疾病患者（依从性和特殊用药）

### 步骤5：风险等级综合评定
- 红色标识（高危）：严重过敏、传染病、急危重症征象
- 黄色标识（中危）：重大病史、特殊人群、中度过敏
- 信息缺失标识：关键安全信息未采集到（如过敏史空白）

### 步骤6：输出风险报告
- 生成风险标识摘要卡片
- 按优先级排序呈现
- 标注每个风险项对诊疗的具体影响建议

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_history | object | 是 | 结构化病史数据 |
| current_medications | array | 否 | 当前用药清单 |
| visit_context | object | 否 | 本次就诊相关上下文 |

## 输出规格
```json
{
  "risk_flags": [
    {
      "category": "allergy|major_disease|infectious|special_population",
      "level": "red|yellow|blue",
      "title": "风险标题",
      "detail": "详细描述",
      "clinical_impact": "对诊疗的影响说明",
      "source": "患者自述|系统记录|历史检验",
      "verified": true
    }
  ],
  "missing_critical_info": ["过敏史未确认", "肝功能未检测"],
  "overall_risk_level": "high|medium|low",
  "summary": "风险摘要文本"
}
```

## 最佳实践
- 过敏标识必须同步传递给用药审核Agent的禁忌数据库
- 对于"不确定"的过敏信息，按"存在过敏"处理（安全优先原则）
- 风险标识应在医生工作站的显著位置展示（红色置顶）
- 关键风险项识别率目标≥98%
- 信息缺失本身也是风险——未记录过敏史不等于无过敏

## 约束与限制
- 不对风险信息做临床处置建议（由医生判断如何处理）
- 传染病信息需遵循保密原则，仅对相关诊疗人员可见
- 风险等级分类标准需经医务部审核确认
- 系统无法完全替代面诊时的再次确认（尤其是过敏史）
