# 抗菌药物管理 (Antibiotic Stewardship)

## 概述
本技能依据《抗菌药物临床应用管理办法》，对抗菌药物处方执行分级管理审核，包括处方权限校验、适应症匹配、微生物送检监控、给药方案合理性评估等专项审核，支持医院抗菌药物合理使用管控目标达成。

## 使用场景
- 抗菌药物处方开具时的分级权限校验
- 抗菌药物适应症和经验性用药合理性审核
- 微生物送检率的实时监控和提醒
- 抗菌药物使用率和DDD值的监测预警
- 围术期预防用药的规范性审核

## 执行步骤

### 步骤1：抗菌药物识别与分级
- 识别处方中的抗菌药物（含抗细菌、抗真菌、抗病毒、抗寄生虫药）
- 确定药物所属管理级别：
  - **非限制使用级**：全体执业医师可处方
  - **限制使用级**：主治医师及以上可处方
  - **特殊使用级**：副主任医师及以上+会诊同意
- 核对开方医生的职称和处方权限

### 步骤2：处方权限校验
- 验证开方医生职称是否符合所开抗菌药物的权限要求
- 特殊使用级额外验证：
  - 是否有相关科室会诊记录
  - 是否有科主任或药事委员会批准
- 权限不足→**强制拦截**并提示申请会诊

### 步骤3：适应症匹配审核
- 核对患者诊断是否包含感染相关诊断
- 评估经验性用药的合理性：
  - 感染部位与所选药物的抗菌谱匹配度
  - 是否符合本院经验性治疗指南/路径
  - 社区获得性vs院内获得性感染的药物选择
- 无明确感染指征使用抗菌药物→**警示提醒**

### 步骤4：微生物送检监控
- 限制使用级和特殊使用级抗菌药物使用前：
  - 检查是否已开具微生物标本送检（血培养、痰培养、尿培养等）
  - 若未送检→**警示提醒**，建议先送检再用药
- 统计科室/全院微生物送检率（目标：限制级以上≥70%）

### 步骤5：给药方案审核
- 基于PK/PD原则评估给药方案：
  - 时间依赖型（β-内酰胺类）：评估给药频次是否足够（T>MIC时间）
  - 浓度依赖型（氨基糖苷类、氟喹诺酮类）：评估单次剂量是否足够
  - 浓度依赖+时间依赖型（万古霉素等）：评估AUC/MIC
- 疗程合理性评估（非手术预防用药通常≤24-48h）
- 肝肾功能不全时的剂量调整

### 步骤6：统计指标监控
- 实时计算并监控：
  - 门诊抗菌药物处方比例（目标≤20%）
  - 住院患者抗菌药物使用率（目标≤60%）
  - 使用强度DDD（目标≤40 DDDs/100人天）
  - I类切口预防用药率（目标≤30%）
  - 围术期预防用药时机（术前0.5-1h）
- 超标时触发科室预警

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| antibiotic_prescription | object | 是 | 抗菌药物处方详情 |
| prescriber_info | object | 是 | 开方医生信息（含职称） |
| patient_diagnosis | array | 是 | 患者诊断列表 |
| microbiology_orders | array | 否 | 微生物送检记录 |
| culture_results | array | 否 | 已有培养结果 |
| surgery_info | object | 否 | 手术信息（围术期用药时） |

## 最佳实践
- 紧急情况（脓毒症等）允许先用药后补办手续，但需限时补齐
- 结合本院细菌耐药监测数据动态调整经验用药推荐
- 对于特殊使用级抗菌药物建立每例点评制度
- 抗菌药物合规率目标≥95%
- 定期（季度）发布院内抗菌药物使用分析报告

## 约束与限制
- 急诊重症感染场景下不可因权限问题延迟抗感染治疗
- 对于确有使用指征但权限不足的情况，引导申请会诊而非简单拦截
- 微生物送检提醒为警示级别，不强制拦截处方
- 不对抗菌药物选择做优劣判断（经验用药的选择由临床医生决定）
- 需定期与感染科、药学部协同更新审核规则
