# 路径智能匹配 (Pathway Matching)

## 概述
本技能根据患者的确认诊断（主诊断ICD-10编码）、合并症、年龄、病情严重度等因素，从临床路径库中匹配最适用的路径方案。支持多路径候选排序和适用性评分，辅助医生决策是否入径及选择哪条路径。

## 使用场景
- 住院患者确认诊断后的路径推荐
- 日间手术患者的简易路径匹配
- 转科患者的路径重新评估
- 路径库更新后的存量患者重新匹配

## 执行步骤

### 步骤1：获取患者诊断信息
- 接收患者主诊断（ICD-10编码）
- 获取合并症/并发症列表
- 获取患者基本信息（年龄、性别）
- 获取病情严重度评估（如有）

### 步骤2：路径库检索
- 根据主诊断ICD-10编码在路径库中精确匹配
- 支持编码层次匹配（如4位码无匹配时尝试3位码）
- 获取所有候选路径的适用条件

### 步骤3：适用性评估
- 逐项核对路径的纳入标准：
  - 诊断匹配度
  - 年龄范围是否符合
  - 是否存在排除条件（如严重合并症、多器官功能不全等）
- 计算适用性评分（0-100分）
- 标注不入径的原因（如有排除条件）

### 步骤4：路径信息汇总
- 生成路径推荐摘要卡片：
  - 路径名称和版本
  - 预计住院天数
  - 关键诊疗节点时间线
  - 预估费用区间
  - DRG/DIP分组信息和支付标准
  - 历史入径患者的完成率和平均费用

### 步骤5：输出推荐结果
- 按适用性评分排序输出候选路径
- 适用性评分≥80%：推荐入径
- 适用性评分60-80%：可选入径（需医生确认）
- 适用性评分<60%：不建议入径，说明原因
- 无匹配路径：标注"无适用临床路径，建议常规诊疗模式"

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| primary_diagnosis | object | 是 | 主诊断（含ICD-10编码） |
| comorbidities | array | 否 | 合并症/并发症列表 |
| patient_demographics | object | 是 | 年龄、性别等 |
| severity_score | number | 否 | 病情严重度评分 |
| admission_type | enum | 否 | 入院类型（计划/急诊/转入） |

## 最佳实践
- 入径匹配准确率目标≥90%
- 定期（季度）更新路径库纳入/排除标准
- 对于适用性边界病例，提供上下文信息辅助医生决策
- 多路径候选时按住院日和费用对比辅助选择
- 结合DRG分组信息帮助医生理解入径的医保支付影响

## 约束与限制
- 入径决策权在医生——系统仅推荐，不强制入径
- 路径匹配基于确认诊断，诊断未确认时不触发匹配
- 多诊断患者可能无适用路径（复杂病例的个体化需求）
- 路径库覆盖范围有限（目前一般覆盖80-200个病种）
- 急诊入院患者可延迟24小时内完成路径匹配
