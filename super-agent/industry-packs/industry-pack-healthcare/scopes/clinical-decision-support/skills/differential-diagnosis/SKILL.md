# 鉴别诊断 (Differential Diagnosis)

## 概述
本技能基于患者临床信息（症状、体征、检查结果），利用临床知识图谱和循证医学数据库，生成按概率排序的鉴别诊断列表，并为每个诊断标注证据等级和关键支持/排除依据，为医生提供系统化的临床推理参考。

## 使用场景
- 门诊接诊时的实时鉴别诊断提示
- 疑难病例的多维度诊断分析
- 检查结果回报后的诊断概率更新
- 住院患者诊断再评估

## 执行步骤

### 步骤1：临床信息输入验证
- 检查输入的临床特征数量（≥3项方可触发）
- 验证信息质量（标注为"不确定"的信息降低权重）
- 分类信息类型：症状、体征、检查结果、病史因素
- 若信息不足，返回"信息不足，无法生成可靠鉴别诊断"提示

### 步骤2：先验概率计算
- 基于流行病学数据确定基础概率
- 考虑因素：年龄、性别、地区、季节、职业暴露
- 引入患者既往诊断作为先验调整因子

### 步骤3：似然比更新
- 对每个候选诊断，计算每项临床特征的阳性/阴性似然比
- 使用贝叶斯公式更新后验概率：
  - 后验Odds = 先验Odds × LR(+) 或 × LR(-)
- 对于缺失的特征，使用中性似然比(LR=1)处理

### 步骤4：证据等级标注
- A级：基于多个高质量随机对照试验(RCT)或Meta分析
- B级：基于单个RCT或高质量观察性研究
- C级：基于专家共识或病例系列研究
- D级：基于专家意见或临床经验推断
- 每个诊断的证据等级取其最高质量支持证据的等级

### 步骤5：生成鉴别诊断列表
- 按后验概率降序排列
- 默认输出Top5，最多不超过10个
- 每个诊断包含：
  - 诊断名称 + ICD-10编码
  - 概率区间（如25-35%）
  - 证据等级(A/B/C/D)
  - 关键支持证据（该患者哪些信息支持此诊断）
  - 关键排除证据（哪些信息不支持或需排除）
  - 确认/排除所需的关键检查

### 步骤6：置信度评估与安全提示
- 计算最高概率诊断的置信度
- 若置信度<60%：标注"证据不足，仅供参考，建议结合临床综合判断"
- 若识别到潜在急危重症（如ACS、PE、脑卒中）：立即高优先级预警
- 标注"本系统输出仅供临床参考，最终诊断由主管医生确定"

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| symptoms | array | 是 | 症状列表（含持续时间、程度） |
| signs | array | 否 | 体征列表（含数值） |
| test_results | array | 否 | 检查检验结果 |
| patient_demographics | object | 是 | 年龄、性别、基本信息 |
| past_history | array | 否 | 既往病史 |
| current_medications | array | 否 | 当前用药 |

## 最佳实践
- 诊断提示覆盖率目标：最终诊断在Top5中≥80%
- 证据等级标注准确率目标≥90%
- 对于急危重症特征（胸痛+呼吸困难+D-dimer升高等），无论概率排序如何都置顶预警
- 每季度与出院诊断/病理诊断回顾比对，优化诊断模型
- 鼓励医生反馈诊断结果用于系统持续学习

## 约束与限制
- 不可替代医生的临床判断——所有输出为参考性质
- 输入临床特征<3项时拒绝生成诊断列表
- 不对罕见病（发病率<1/100000）给出高概率排序，除非有强特征支持
- 不做确定性诊断陈述（避免"确诊为..."的表述）
- 模型局限性：对于新发疾病或知识库未收录的罕见表现可能遗漏
