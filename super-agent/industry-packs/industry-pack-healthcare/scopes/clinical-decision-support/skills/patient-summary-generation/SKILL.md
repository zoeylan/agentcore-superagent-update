# 就诊摘要生成 (Patient Summary Generation)

## 概述
本技能基于结构化病史数据和历史就诊记录，生成简洁、重点突出的就诊摘要供医生在接诊前快速浏览。摘要控制在300字以内，突出关键风险项、主要症状、近期异常结果和当前用药情况。

## 使用场景
- 诊前推送至医生工作站供快速浏览
- 会诊请求时的患者信息概要
- 转诊时的病情摘要附件
- 多学科讨论(MDT)时的患者简介

## 执行步骤

### 步骤1：信息汇总
- 整合本次预采集的病史信息
- 关联近6个月的就诊记录
- 提取近期检查检验异常结果
- 获取当前用药清单

### 步骤2：关键信息提取与优先级排序
- 第一优先级：红色风险项（过敏、传染病）
- 第二优先级：主诉和核心症状描述
- 第三优先级：近期异常检查结果（尤其是变化趋势）
- 第四优先级：当前用药（重点标注可能与本次就诊相关的药物）
- 第五优先级：重要既往史

### 步骤3：摘要文本生成
- 采用标准化模板结构：
  ```
  【风险标识】🔴 青霉素过敏（过敏性休克史）| 🟡 高血压10年
  【主诉】[主诉内容，OPQRST概要]
  【近期异常】[异常检查检验结果，标注日期和趋势]
  【当前用药】[用药清单概要]
  【既往重要信息】[与本次就诊可能相关的既往信息]
  ```
- 全文控制在300字以内
- 使用标准医学缩写以节省空间

### 步骤4：摘要推送
- 推送至医生工作站的就诊队列中对应患者卡片
- 标注预采集完成度（完整/部分/仅基本信息）
- 提供"查看完整病史"的链接入口

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| structured_history | object | 是 | 结构化病史采集结果 |
| risk_flags | array | 是 | 风险标识结果 |
| recent_results | array | 否 | 近期检查检验结果 |
| visit_records | array | 否 | 历史就诊记录 |
| summary_type | enum | 否 | 摘要类型（诊前/会诊/转诊/MDT） |

## 输出规格
```json
{
  "summary_text": "格式化摘要文本（≤300字）",
  "structured_summary": {
    "risk_alerts": ["风险标识列表"],
    "chief_complaint_brief": "主诉概要",
    "key_abnormal_results": ["近期异常结果"],
    "current_medications_brief": ["当前用药概要"],
    "relevant_history": ["相关既往信息"]
  },
  "completeness": 0.85,
  "generated_at": "2024-01-01T08:00:00Z",
  "data_freshness": "所有数据更新于24小时内"
}
```

## 最佳实践
- 摘要应能在30秒内完成阅读（医生接诊前快速浏览）
- 使用颜色编码和图标增强可视化效果
- 复诊患者突出"上次诊断"和"本次变化"
- 对于急诊患者生成超简版摘要（100字内，仅关键安全信息）
- 信息缺失时明确标注而非省略

## 约束与限制
- 摘要不包含任何诊断建议或诊疗方向提示
- 不对检查结果做临床解读（仅标注正常/异常/趋势）
- 摘要内容不可替代完整病历查阅
- 需确保摘要信息与源数据完全一致（不可推断或假设）
