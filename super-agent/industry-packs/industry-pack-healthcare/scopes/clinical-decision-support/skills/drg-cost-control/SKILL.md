# DRG费用管控 (DRG Cost Control)

## 概述
本技能结合患者DRG/DIP分组信息，实时监控住院费用累计与分组支付标准的对比关系，当费用超过阈值时分级预警，并按费用构成维度展示费用明细，辅助临床团队识别费用异常增长点，在保证医疗质量前提下控制不合理费用。

## 使用场景
- 患者入径后的费用实时监控和预警
- 住院期间费用累计进度跟踪
- 费用构成分析（药品/耗材/检查/治疗/床位）
- 出院结算前的费用合理性审查
- 科室DRG运营指标的汇总分析

## 执行步骤

### 步骤1：DRG/DIP分组确认
- 根据患者主诊断+手术操作确定DRG分组
- 获取该分组的：
  - 支付标准（均值）
  - 费用标准差和分布区间
  - 住院日标准
  - 权重(RW)和CMI值
- 标注分组置信度（入院时可能调整）

### 步骤2：费用实时归集
- 对接费用系统实时获取已发生费用
- 按费用类别归集：
  - 药品费（西药+中成药）
  - 耗材费（高值耗材+低值耗材）
  - 检查费（影像+超声+内镜等）
  - 检验费
  - 治疗费（手术+操作+治疗）
  - 护理费+床位费
  - 其他费用

### 步骤3：费用进度计算
- 计算费用消耗进度：当前累计费用/分组支付标准 × 100%
- 计算时间消耗进度：当前住院日/标准住院日 × 100%
- 计算费用消耗指数：实际费用/标准费用
- 对比两个维度进度差异（费用跑在时间前面→超支风险）

### 步骤4：分级预警
- **绿色正常**：费用进度 ≤ 时间进度 × 1.2
- **黄色预警**：累计费用超过分组均值1.2倍
  - 推送提示至经治医生
  - 标注主要超支类别
- **红色预警**：累计费用超过分组均值1.5倍
  - 推送至科主任和医保办
  - 生成费用异常分析报告
  - 建议评估可压缩的费用空间
- **严重超支**：累计费用超过分组均值2.0倍
  - 报告医务部和财务部
  - 建议启动特殊审批流程

### 步骤5：费用异常分析
- 与同DRG组近期出院病例对比：
  - 哪个费用类别偏高？偏高多少？
  - 是否有大额单项费用（高值耗材、特殊药物等）
  - 住院日是否超标？超标几天？
- 提供同组对标数据供医生参考决策

### 步骤6：费用报告输出
- 提供仪表盘数据：
  - 费用进度条（当前vs标准）
  - 费用构成饼图
  - 日均费用趋势折线
  - 与同组病例的对比排位
- 出院时生成完整费用分析报告

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_id | string | 是 | 患者标识 |
| drg_group | object | 是 | DRG分组信息和支付标准 |
| current_charges | object | 是 | 当前累计费用明细 |
| current_los | integer | 是 | 当前住院日 |
| expected_los | integer | 否 | 路径预计住院日 |

## 最佳实践
- 费用预警时机：越早发现越有调整空间
- 预警不等于费用不合理——部分超支基于患者病情是必要的
- 结合CMI值综合评估（高CMI值患者允许更高费用）
- 科室层面跟踪时间消耗指数和费用消耗指数（目标≤1.0）
- 超支病例出院后进行个案分析，区分可控与不可控因素

## 约束与限制
- 费用管控不可影响必要的医疗行为——患者安全始终优先
- 预警仅供参考，不强制限制医生的诊疗决策
- DRG分组可能在住院过程中因诊断/手术变更而调整
- 新开展技术或罕见病例可能不适用标准费用对标
- 费用数据依赖计费系统的及时性和准确性
