# 检查检验推荐 (Test Recommendation)

## 概述
本技能基于鉴别诊断假设，为医生推荐具有最高鉴别价值的检查检验项目，帮助确认或排除特定诊断。推荐时综合考虑检查的临床价值（敏感性/特异性）、费用、侵入性、等待时间等因素，支持医生做出高效的检查决策。

## 使用场景
- 鉴别诊断生成后的针对性检查推荐
- 初步检查结果回报后的进一步检查建议
- 疑难病例的系统化排查方案设计
- 常规随访的定期复查项目提醒

## 执行步骤

### 步骤1：分析鉴别诊断需求
- 接收当前鉴别诊断列表和各诊断的概率
- 识别需要确认的高概率诊断
- 识别需要排除的危险诊断（虽然概率低但后果严重）
- 评估当前已有的检查结果，避免重复推荐

### 步骤2：筛选候选检查项目
- 从知识库中提取各诊断的确认/排除检查
- 计算每项检查的信息增益：
  - 阳性似然比(LR+)：对确认诊断的价值
  - 阴性似然比(LR-)：对排除诊断的价值
- 标注检查的基本属性：
  - 类型（实验室检验/影像/功能检查/侵入性检查）
  - 费用区间
  - 结果等待时间
  - 侵入性程度

### 步骤3：优先级排序
- 综合排序依据：
  1. 临床紧急程度（急危重症排除检查优先）
  2. 信息增益（能同时鉴别多个诊断的检查优先）
  3. 可及性（门诊可完成 > 需预约 > 需住院）
  4. 费用效益比
  5. 侵入性（非侵入 > 微创 > 有创）
- 分组输出：立即执行 / 24小时内 / 择期安排

### 步骤4：生成推荐方案
- 每项推荐包含：
  - 检查名称和项目编码
  - 推荐理由（确认哪个诊断或排除哪个诊断）
  - 紧急程度
  - 预期费用
  - 注意事项（如空腹、造影剂过敏风险等）
- 组合推荐时标注检查间的逻辑关系（并行/顺序依赖）

### 步骤5：检查前须知生成
- 标注需要特殊准备的检查（空腹、停药、肠道准备等）
- 标注造影剂使用时的禁忌提醒（肾功能、过敏史）
- 提供患者告知模板（检查目的、注意事项）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| differential_list | array | 是 | 当前鉴别诊断列表 |
| existing_results | array | 否 | 已有检查结果 |
| patient_info | object | 是 | 患者基本信息和禁忌 |
| setting | enum | 否 | 就诊环境（门诊/急诊/住院） |
| cost_sensitivity | enum | 否 | 费用敏感度（高/中/低） |

```json
{
  "recommendations": [
    {
      "test_name": "检查名称",
      "test_code": "项目编码",
      "urgency": "immediate|within_24h|elective",

      "target_diagnoses": ["针对哪些鉴别诊断"],
```

## 最佳实践
- 避免"撒网式"检查——每项推荐必须有明确的鉴别诊断目的
- 优先推荐能同时鉴别多个诊断的检查（一石多鸟）
- 考虑DRG/DIP费用管控要求，避免过度检查
- 标注检查的阶梯策略（先无创后有创，先便宜后昂贵）
- 急诊场景下优先生命安全，费用为次要考虑

## 约束与限制
- 推荐不可替代医生的检查决策——医生可能基于查体发现调整方案
- 不推荐与患者已知禁忌冲突的检查（如肾功能不全的造影CT）
- 不重复推荐近期（7天内）已完成的同一检查（除非有明确复查指征）
- 对于费用较高（>1000元）或侵入性检查需特别标注供医生权衡
