# 偏差管理 (Variance Management)

## 概述
本技能管理临床路径执行过程中的偏差事件——当实际诊疗活动偏离路径预设计划时，自动识别偏差、记录偏差详情、触发审核流程，并汇总分析偏差模式用于路径优化。偏差管理是保障路径质量和持续改进的核心机制。

## 使用场景
- 路径执行中出现计划外活动时的偏差识别
- 住院日延长时的偏差记录和审核触发
- 非计划检查/治疗/手术的偏差登记
- 路径偏差的汇总统计和模式分析
- 路径优化迭代的数据支撑

## 执行步骤

### 步骤1：偏差自动识别
- 持续监控路径执行与计划的偏离：
  - **时间偏差**：住院日延长超计划天数、手术延期、检查延迟
  - **活动偏差**：增加非计划检查/治疗、取消计划项目、新增会诊
  - **药物偏差**：更换药物方案、增加/减少用药
  - **出径偏差**：提前出院（非计划）、转ICU、转院、死亡
- 自动触发偏差记录工单

### 步骤2：偏差分类与分级
- 按原因分类：
  - **患者因素**：病情变化、合并症加重、患者/家属要求、依从性差
  - **医疗因素**：诊断变更、治疗策略调整、并发症处理
  - **系统因素**：设备故障、手术排期冲突、药品短缺、床位紧张
- 按影响分级：
  - **重大偏差**：影响住院日≥3天，或费用增加≥50%，或转ICU/再手术
  - **一般偏差**：影响住院日1-2天，或费用增加<50%
  - **轻微偏差**：不影响住院日和主要费用

### 步骤3：偏差记录工单生成
- 自动预填可识别的信息：
  - 偏差发生时间
  - 偏差类型和分类
  - 涉及的路径任务节点
  - 当前路径进度
- 要求经治医生补充：
  - 偏差具体原因描述
  - 临床必要性说明
  - 后续调整计划

### 步骤4：审核流程触发
- 轻微偏差：经治医生记录即可，无需审核
- 一般偏差：需上级医师（主治以上）审核签字
- 重大偏差：需科主任审核并报医务部备案
- 审核内容：偏差原因合理性、是否需要退径

### 步骤5：偏差统计与分析
- 定期汇总统计（周/月/季度）：
  - 科室偏差率（偏差病例数/入径病例数）
  - 偏差原因分布（患者/医疗/系统因素比例）
  - 高频偏差类型排名
  - 偏差对住院日和费用的影响量化
- 识别路径设计问题：
  - 某个节点反复出现偏差→路径可能需要调整
  - 某类患者频繁偏差→纳入/排除标准可能需要修订

### 步骤6：路径优化建议
- 基于偏差分析提出路径修订建议
- 生成路径优化报告提交路径管理委员会
- 跟踪路径修订后偏差率变化

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_pathway | object | 是 | 患者路径执行记录 |
| variance_event | object | 是 | 偏差事件详情 |
| physician_input | object | 否 | 医生补充的偏差原因 |
| approval_chain | array | 否 | 审核链配置 |

## 最佳实践
- 偏差记录完整率目标≥95%（每个偏差都应有记录）
- 鼓励如实记录偏差原因，不惩罚合理的偏差
- 重大偏差24小时内完成审核
- 每季度路径管理委员会审议高频偏差并修订路径
- 路径完成率（正常出径/总入径）季度目标≥70%

## 约束与限制
- 偏差记录不等于医疗差错——多数偏差是合理的个体化调整
- 不评判偏差的临床合理性（由审核医师判断）
- 偏差统计用于路径优化和质量改进，不作为个人惩罚依据
- 病情危重需紧急处理时，偏差记录可事后补录（但不超过24h）
- 退径后不再产生偏差记录（退径本身作为一次重大偏差记录）
