# 路径执行监控 (Pathway Monitoring)

## 概述
本技能对已入径患者的临床路径执行进度进行每日监控，生成当日任务清单（检查、治疗、护理、宣教），追踪任务完成状态，识别执行延迟和遗漏，确保路径按计划推进。

## 使用场景
- 每日晨间生成当日路径任务清单
- 实时追踪各任务的执行完成状态
- 路径节点延迟或遗漏时的提醒预警
- 路径执行进度的可视化展示
- 多患者路径执行状态的科室汇总

## 执行步骤

### 步骤1：确定当前路径日
- 计算患者入径后的住院天数（路径Day N）
- 确认患者当前路径状态（执行中/暂停/偏差处理中）
- 若存在路径天数调整（如手术延期），同步更新

### 步骤2：生成当日任务清单
- 从路径模板中提取Day N的全部任务项：
  - **医嘱类**：检查、检验、治疗、手术、药物
  - **护理类**：评估（生命体征、疼痛评分、跌倒风险等）、干预、观察
  - **宣教类**：术前/术后宣教、出院指导、康复训练指导
  - **行政类**：会诊、知情同意签署、出院准备
- 每个任务包含：预计执行时间、责任角色、前置依赖

### 步骤3：状态追踪
- 对接HIS/护理系统获取任务执行状态
- 状态分类：
  - ✅ 已完成（含完成时间和执行人）
  - 🔄 进行中（已启动未完成）
  - ⏳ 待执行（未到计划时间）
  - ⚠️ 延迟（超过计划时间未执行）
  - ❌ 未执行（确认跳过/取消）

### 步骤4：延迟预警
- 任务超过计划时间2小时未执行→提醒经治医生/责任护士
- 关键节点延迟（如手术日未手术）→提醒上级医师
- 每日16:00前未完成当日非紧急任务→生成待办提醒
- 影响后续任务的前置任务延迟→标注连锁影响

### 步骤5：每日执行报告
- 每日18:00生成执行报告：
  - 当日任务完成率
  - 延迟/未执行任务列表及原因
  - 明日预计任务预告
  - 路径整体进度（已完成天数/总天数）
- 推送至经治医生和护士长工作站

### 步骤6：出径评估提醒
- 当路径进入最后1-2天时提醒评估出径条件
- 生成出径checklist（出院标准核对）
- 若满足出径条件，提醒医生及时办理出院
- 若不满足，提示哪些条件未达标

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_id | string | 是 | 患者标识 |
| pathway_id | string | 是 | 所在路径标识 |
| pathway_day | integer | 是 | 当前路径天数 |
| task_status_feed | array | 否 | 来自HIS的任务状态更新 |

## 最佳实践
- 每日任务推送时间：早8:00前完成
- 关键节点（手术日、出院日）提前24小时确认准备状态
- 科室层面汇总各患者路径进度用于晨会讨论
- 任务完成率纳入科室绩效考核参考
- 对高频延迟任务分析原因，优化路径设计

## 约束与限制
- 监控不干预临床决策——延迟原因可能是合理的医疗判断
- 任务完成状态依赖下游系统数据质量
- 手动记录的任务可能存在时间偏差
- 不监控路径外的自主临床活动
- 节假日/周末可能影响某些任务的执行时间
