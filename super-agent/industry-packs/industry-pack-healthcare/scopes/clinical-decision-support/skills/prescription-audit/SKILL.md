# 处方审核 (Prescription Audit)

## 概述
本技能对医生开具的每张处方进行实时、全面的合理性审核，覆盖过敏史核查、禁忌症排查、重复用药检测等维度，在2秒内返回审核结果。作为患者用药安全的最后自动化防线，确保不合理处方在到达药房前被拦截或提醒。

## 使用场景
- 医生工作站处方提交时的实时拦截审核
- 住院医嘱开具时的实时校验
- 处方修改后的重新审核
- 药师调配前的二次复核辅助

## 执行步骤

### 步骤1：处方信息解析
- 接收处方数据：药品名称、剂量、频次、给药途径、疗程
- 关联患者信息：诊断、过敏史、合并症、当前用药、检验结果
- 验证处方完整性（必填字段是否齐全）

### 步骤2：过敏史交叉核查
- 将处方药物及其活性成分与患者过敏记录精确匹配
- 评估交叉过敏风险：
  - 同类药物交叉过敏（如β-内酰胺类之间）
  - 共同辅料成分过敏（如聚山梨酯80）
- 匹配到已知过敏原→**强制拦截**
- 匹配到交叉过敏风险→**警示提醒**

### 步骤3：禁忌症排查
- 核对患者诊断和合并症与药物禁忌证清单
- 区分绝对禁忌和相对禁忌：
  - 绝对禁忌→**强制拦截**
  - 相对禁忌→**警示提醒**
- 特殊人群禁忌检查（妊娠FDA分级、哺乳期安全性、儿童适用性）

### 步骤4：重复用药检测
- 检查同一治疗目的的重复处方
- 检查同一药理学分类的药物叠加
- 检查不同科室/医生间的重复开方
- 重复用药→**警示提醒**（注明重复项）

### 步骤5：适应症匹配验证
- 核对处方药物的适应症是否覆盖患者当前诊断
- 识别超适应症用药（off-label use）
- 超适应症→**信息提示**（标注为超说明书用药）

### 步骤6：审核结果输出
- 分三级输出：
  - **强制拦截**：处方退回，必须修改
  - **警示提醒**：弹窗提示，医生可确认后继续
  - **信息提示**：标注展示，不打断工作流
- 拦截/警示时提供替代方案建议
- 记录审核日志（审核时间、结果、处理方式）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| prescription | object | 是 | 处方详细信息 |
| patient_profile | object | 是 | 患者完整信息档案 |
| current_medications | array | 是 | 当前用药清单 |
| diagnoses | array | 是 | 当前诊断列表 |

## 最佳实践
- 审核响应时间≤2秒，不影响医生工作效率
- 强制拦截准确率≥99.5%（极低误拦截率）
- 虚警率≤5%（避免"狼来了"效应降低医生信任）
- 替代方案建议应考虑可及性和费用因素
- 所有审核记录保留用于处方合理率统计

## 约束与限制
- 处方审核覆盖率必须100%（不可遗漏任何处方）
- 强制拦截不可被绕过（医生必须修改后重新提交）
- 警示提醒被医生确认继续时，理由记录备案供药师复核
- 系统不判断治疗方案的临床合理性（仅做安全性审核）
- 信息不完整（如过敏史缺失）时需标注风险而非默认安全
