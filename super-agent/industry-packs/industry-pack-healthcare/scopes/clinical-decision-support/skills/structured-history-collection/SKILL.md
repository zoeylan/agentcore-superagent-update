# 结构化病史采集 (Structured History Collection)

## 概述
本技能用于在患者候诊阶段通过智能问卷和自然语言交互，系统化采集患者病史信息并输出标准化结构数据。采集内容覆盖主诉、现病史、既往史、过敏史、家族史、个人史等完整病史维度。

## 使用场景
- 患者到院候诊时的诊前信息预采集
- 复诊患者的病情变化更新采集
- 急诊患者的快速关键信息采集（精简模式）
- 转诊患者的病史信息补充完善

## 执行步骤

### 步骤1：触发采集流程
- 患者签到/候诊状态变更时自动触发
- 系统检查患者是否已有近期（7天内）预采集记录
- 若有记录则进入更新模式，仅采集变化信息
- 若无记录则启动完整采集流程

### 步骤2：主诉与现病史采集
- 引导患者描述主要不适症状
- 按OPQRST框架追问：
  - Onset：什么时候开始的？突然还是逐渐？
  - Provocation：什么情况加重/缓解？
  - Quality：什么性质的不适（疼痛性质、程度等）？
  - Region：具体位置？是否放射？
  - Severity：严重程度（1-10分评分）？
  - Time：持续时间？发作频率？
- 记录伴随症状和阴性症状

### 步骤3：既往史与过敏史采集
- 既往重大疾病（高血压、糖尿病、心脏病、恶性肿瘤等）
- 手术史（手术名称、时间、医院）
- 外伤史
- 输血史
- 药物过敏史（过敏原、反应类型、严重程度）
- 食物过敏史
- 当前用药清单（药名、剂量、频次、已用时间）

### 步骤4：家族史与个人史
- 直系亲属重大疾病史
- 个人生活习惯（吸烟、饮酒、运动）
- 职业暴露史
- 女性：月经史、婚育史

### 步骤5：历史数据关联
- 自动检索患者近6个月的检查检验结果
- 关联既往就诊记录和诊断信息
- 标注异常结果和变化趋势

### 步骤6：信息结构化输出
- 将自然语言描述转换为标准医学术语
- 按照标准病史格式组织输出
- 标注信息来源（患者自述/系统导入/历史记录）
- 标注信息完整度评分

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_id | string | 是 | 患者唯一标识 |
| visit_type | enum | 是 | 就诊类型（初诊/复诊/急诊） |
| triage_info | object | 否 | 分诊时预采集的症状信息 |
| collection_mode | enum | 否 | 采集模式（完整/精简/更新） |

## 输出规格
```json
{
  "chief_complaint": "主诉文本（标准术语）",
  "present_illness": { "onset": "", "progression": "", "symptoms": [] },
  "past_history": { "diseases": [], "surgeries": [], "allergies": [] },
  "family_history": [],
  "personal_history": {},
  "current_medications": [],
  "historical_results": [],
  "completeness_score": 0.85,
  "risk_flags": [],
  "source_annotations": {}
}
```

## 最佳实践
- 采集语言应通俗易懂，避免使用专业医学术语向患者提问
- 对于老年患者或文化程度较低的患者，使用选择题而非开放式问题
- 急诊模式下仅采集关键安全信息（过敏史、当前用药、主要症状）
- 对矛盾信息标注冲突而非自动覆盖
- 预采集完成率目标≥70%

## 约束与限制
- 不对采集到的症状进行诊断性解读
- 不向患者提供任何医疗建议
- 患者拒绝回答的项目标记为"拒绝提供"而非留空
- 敏感信息（传染病史、精神病史）需获得患者明确同意后采集
- 所有数据采集符合《个人信息保护法》和《医疗机构病历管理规定》
