# 病历要素完整性审查

## 概述
本技能对电子病历进行系统性要素完整性审查，确保各类病历文书的必填项无遗漏、内容充实有效，符合《电子病历基本规范》和医院病历书写细则的要求。要素完整性是病历质量评分的基础维度，缺失关键要素可直接导致病历评为丙级。

## 适用场景
- 医生保存病历时的实时完整性校验
- 病历提交归档前的终审检查
- 科室定期病历质量抽查
- 等级评审前的全院病历集中审查
- 新入职医生病历书写辅导

## 执行步骤

### 步骤1：确定病历类型和对应检查模板
根据病历类型加载对应的必填项检查清单：
- **入院记录**：一般项目（姓名、性别、年龄、婚姻状况、职业等≥12项）、主诉、现病史、既往史、个人史、婚育史/月经史、家族史、体格检查（≥8个系统）、专科检查、辅助检查、初步诊断、医师签名
- **病程记录**：日期时间、病情变化、诊疗措施、医嘱变更原因、效果评估、下一步计划
- **手术记录**：手术日期、术前诊断、术后诊断、手术名称、手术人员、麻醉方式、手术经过、术中情况、引流物、标本
- **出院记录**：入院日期、出院日期、入院诊断、出院诊断、诊疗经过、出院情况、出院医嘱、随访计划

### 步骤2：逐项检查必填项
- 检查每个必填字段是否有内容填写
- 识别"有字段但无实质内容"的情况（如仅填"无"但缺乏临床合理性）
- 检查体格检查是否覆盖必要系统（心肺腹神经系统等）
- 验证诊断是否完整（主诊断+其他诊断）

### 步骤3：内容充实度评估
- 现病史是否包含起病时间、症状特点、演变过程、诊治经过
- 体格检查记录是否有具体数值而非笼统描述
- 辅助检查是否列出具体结果数值
- 诊疗计划是否具体可执行

### 步骤4：生成审查报告
- 列出所有缺失必填项（按严重程度分类）
- 标注内容不充分的字段和具体建议
- 计算完整性得分（缺失A类必填项直接扣5分/项，B类扣3分/项，C类扣1分/项）
- 给出总体完整性评级和修改优先级

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| record_id | string | 是 | 病历记录ID |
| record_type | enum | 是 | 病历类型 |
| record_content | object | 是 | 病历内容结构化数据 |
| check_mode | enum | 否 | 实时校验/终审检查/抽查模式 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| completeness_score | float | 完整性得分（0-100） |
| missing_items | array | 缺失项列表（含字段名、重要等级、扣分值） |
| insufficient_items | array | 内容不充分项列表（含建议） |
| overall_rating | enum | 完整/基本完整/不完整 |
| priority_fixes | array | 优先修改建议（按影响分值排序） |

## 最佳实践
- A类必填项（缺失即丙级）：主诉、入院诊断/出院诊断、手术名称、知情同意书
- 对于内容充实度的判断应结合临床合理性，避免机械化检查
- 实时校验模式应在医生书写过程中给予即时提示，不阻断书写流程
- 定期更新检查模板以适应新版规范要求
- 对于特殊病历（如急诊、抢救），可适当放宽非关键项要求

## 约束条件
- 不得修改或补充病历内容，仅提供提示和建议
- 检查结果应即时反馈，终审模式响应时间≤30秒
- 检查模板应支持按科室定制（如外科增加手术相关检查项）
- 审查记录应保存完整的检查轨迹用于质控追溯
