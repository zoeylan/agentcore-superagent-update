# 病历等级评定

## 概述
本技能根据《病历书写基本规范》和医院病历质量评分标准，对病历进行自动预评级（甲级/乙级/丙级），生成详细的评分明细和扣分依据。病历等级评定是病历质量管理的核心输出，病历甲级率≥90%是三甲医院的硬性考核指标，丙级病历实行零容忍政策。

## 适用场景
- 病历归档前的终末质控评级
- 环节质控中的实时预评级（供医生参考）
- 科室月度/季度病历质量考核
- 等级医院评审前的集中评级
- 新入职医师病历书写培训反馈

## 执行步骤

### 步骤1：加载评分标准
采用百分制评分，核心评分项目包括：
- **及时性（15分）**：各类记录是否在规定时限内完成
- **完整性（30分）**：必填项是否齐全、内容是否充实
- **一致性（25分）**：各文书间逻辑是否统一、前后是否矛盾
- **规范性（20分）**：术语使用、格式排版、签名盖章是否规范
- **科学性（10分）**：诊疗思路是否清晰、分析是否合理

### 步骤2：逐维度评分
- 调用时限检查技能获取及时性得分
- 调用完整性审查技能获取完整性得分
- 调用一致性校验技能获取一致性得分
- 独立评估规范性（ICD编码准确性、格式规范、签名完整）
- 独立评估科学性（诊断依据充分性、鉴别诊断逻辑性）

### 步骤3：判定等级
- **甲级（≥90分）**：无A类缺陷，B类缺陷≤3项
- **乙级（75-89分）**：无A类缺陷或仅1项，整体质量尚可
- **丙级（<75分 或 存在原则性错误）**：
  - 总分低于75分
  - 缺少知情同意书
  - 诊断与病情完全不符
  - 存在涂改、伪造痕迹
  - 关键签名缺失（主刀医生、上级医师等）

### 步骤4：生成评级报告
- 总分和等级判定
- 各维度得分明细
- 具体扣分点清单（含条款引用）
- 逐项修改建议（按优先级排序）
- 对丙级病历标注"需48小时内整改"

### 步骤5：归档与通知
- 甲级：直接归档入库
- 乙级：标记改进建议，归档入库
- 丙级：退回修改，通知责任医生和科室质控员，限时3个工作日修改后重新评级

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| record_id | string | 是 | 病历记录ID |
| patient_id | string | 是 | 患者ID |
| admission_id | string | 是 | 住院号 |
| assessment_type | enum | 是 | 终末评级/环节预评级/抽查评级 |
| timeliness_result | object | 否 | 时限检查结果（可自动获取） |
| completeness_result | object | 否 | 完整性检查结果（可自动获取） |
| consistency_result | object | 否 | 一致性检查结果（可自动获取） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| total_score | float | 总评分（0-100） |
| grade | enum | 甲级/乙级/丙级 |
| dimension_scores | object | 各维度得分明细 |
| deduction_items | array | 扣分项清单 |
| deduction_items[].item | string | 扣分项描述 |
| deduction_items[].points | float | 扣除分数 |
| deduction_items[].reference | string | 引用规范条款 |
| deduction_items[].suggestion | string | 修改建议 |
| is_principle_error | boolean | 是否存在原则性错误 |
| action_required | enum | 归档/退回修改/重点关注 |

## 最佳实践
- 自动预评级与人工终评的一致率目标≥85%
- 对于评级结果存疑的病历，应标记为"建议人工复核"
- 评分标准应每年修订一次，纳入最新规范要求
- 科室间评分标准应统一，避免"松紧不一"
- 新增扣分规则前应有充分的试运行和反馈收集期

## 约束条件
- 自动评级结果为"预评级"，最终等级认定需科室质控员或医务部确认
- 丙级判定应慎重，系统判定丙级时建议人工复核确认
- 不得将评级结果直接用于医生个人经济处罚（应先由质控部门审核）
- 评分数据应保密，仅对本科室和质控管理部门开放查询
