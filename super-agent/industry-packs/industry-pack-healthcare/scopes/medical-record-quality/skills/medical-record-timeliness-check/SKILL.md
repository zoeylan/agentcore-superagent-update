# 病历时限合规性检查

## 概述
本技能负责对各类电子病历的书写时限进行实时跟踪和合规性监控，确保医疗文书按照《电子病历应用管理规范》规定的时间要求完成。时限管理是病历质控的第一道防线，直接影响等级医院评审结果和医疗安全。

## 适用场景
- 患者入院后自动启动入院记录时限倒计时
- 手术完成后启动手术记录书写时限监控
- 患者出院后启动出院记录时限监控
- 病程记录定期书写时限检查（主治医师查房记录每周≥2次、副主任以上每周≥1次）
- 抢救记录6小时内补记时限监控

## 执行步骤

### 步骤1：识别病历类型与时限规则
- 入院记录：入院后8小时内完成
- 首次病程记录：入院后8小时内完成
- 手术记录：术后24小时内完成
- 出院记录：出院后24小时内完成
- 死亡记录：死亡后24小时内完成
- 死亡讨论记录：死亡后7个工作日内完成
- 转科记录：转科即时完成
- 抢救记录：抢救结束后6小时内补记
- 日常病程记录：病危患者每日至少1次，病重患者至少每2日1次，一般患者至少每3日1次

### 步骤2：启动时限倒计时
- 根据触发事件（入院、手术、出院等）自动记录起始时间
- 计算截止时间（考虑工作日规则，死亡讨论按工作日计）
- 在系统中创建时限跟踪任务

### 步骤3：分级预警通知
- **提前2小时**：首次提醒，通过系统消息通知责任医生
- **超时即时**：紧急通知主治医生，标记为超时病历
- **超时2小时**：升级通知科主任，生成超时事件记录
- **超时4小时**：升级通知医务部，纳入科室质控考核

### 步骤4：记录与统计
- 记录每次提醒的发送时间和送达确认
- 统计时限达标率（按科室、医生、病历类型维度）
- 超时病历纳入月度统计报表
- 标记反复超时的医生，建议科主任关注

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_id | string | 是 | 患者ID |
| record_type | enum | 是 | 病历类型（入院记录/首次病程/手术记录/出院记录/死亡记录等） |
| trigger_event_time | datetime | 是 | 触发事件发生时间 |
| responsible_doctor_id | string | 是 | 责任医生ID |
| department_id | string | 是 | 科室ID |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| timeliness_status | enum | 正常/预警/超时 |
| deadline | datetime | 截止时间 |
| remaining_hours | float | 剩余小时数 |
| notification_level | enum | 无/首次提醒/主治医生/科主任/医务部 |
| notification_sent | boolean | 通知是否已发送 |
| overdue_duration_hours | float | 超时时长（小时） |

## 最佳实践
- 时限计算应精确到分钟级别，避免边界情况引发争议
- 节假日和夜班期间的时限规则需根据医院具体制度调整
- 提醒消息应同时推送到医生工作站和手机端，确保送达
- 对于连续超时的科室，建议分析原因（人力不足/流程问题/系统原因）
- 时限达标率目标≥95%，低于90%时应启动专项改进

## 约束条件
- 不得因系统原因导致的延迟（如系统宕机）计入医生超时
- 急诊抢救期间的时限可根据医院规定适当延长，但需有记录
- 通知发送失败时应自动重试并记录异常
- 所有时限数据应保留至少3年用于统计分析
