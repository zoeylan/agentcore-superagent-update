# 质量报告生成

## 概述
本技能负责自动化生成各类医疗质量分析报告，包括每日指标简报、周报、月度质量分析报告、季度总结和年度质量报告。报告内容涵盖数据呈现、趋势分析、问题识别和改进建议，为医院质量管理决策提供结构化的信息输入。

## 适用场景
- 每日自动生成各科室质量指标简报
- 每周生成含趋势图和科室排名的周报
- 月度质量分析会议数据包自动准备
- 季度质量总结报告（含PDCA执行情况跟踪）
- 年度质量报告辅助撰写（等级评审材料）
- 专项分析报告（针对特定问题的深度分析）

## 执行步骤

### 步骤1：确定报告类型和模板
| 报告类型 | 频率 | 主要受众 | 核心内容 |
|----------|------|----------|----------|
| 日报 | 每日 | 科室负责人 | 关键指标数值、预警事项 |
| 周报 | 每周一 | 科主任、医务部 | 趋势图、科室排名、重点事项 |
| 月报 | 每月5日 | 院领导、医务部 | 综合分析、PDCA进展、改进建议 |
| 季报 | 季末 | 院领导、质控委员会 | 总结评估、策略调整、下季度目标 |
| 年报 | 年末 | 全院 | 年度总结、标杆对比、战略规划 |

### 步骤2：数据汇总与可视化
- 从指标监测系统提取对应周期的数据
- 计算各指标的期间值和变化率
- 生成可视化图表（折线图显示趋势、柱状图显示对比、热力图显示分布）
- 标注目标线和预警线

### 步骤3：分析与洞察提取
- 识别表现优异和需关注的科室/指标
- 提取改进明显的案例作为最佳实践分享
- 标注持续恶化的指标及已采取的措施
- 关联分析：指标间的因果关系解读

### 步骤4：建议与行动项
- 对每个问题指标提出具体改进建议
- 跟踪上期改进措施的执行情况和效果
- 提出下期重点关注事项
- 明确责任部门和时间节点

### 步骤5：报告审核与分发
- 报告生成后推送至相关负责人审阅
- 设置确认阅读机制（重要报告需签收）
- 报告存档备查

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| report_type | enum | 是 | 日报/周报/月报/季报/年报/专项 |
| report_period | object | 是 | 报告周期（start_date, end_date） |
| target_audience | array | 否 | 目标受众（影响内容详略度） |
| focus_indicators | array | 否 | 重点关注指标 |
| include_comparison | boolean | 否 | 是否包含同比/环比对比 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| report_title | string | 报告标题 |
| report_body | object | 报告正文（含各章节） |
| charts | array | 图表数据（可视化配置） |
| key_findings | array | 核心发现摘要 |
| action_items | array | 行动建议列表 |
| previous_actions_status | array | 上期行动项执行情况 |

## 最佳实践
- 报告应"结论前置"，管理者最关心的核心指标和异常事项放在最前面
- 数据可视化优于大段文字，每个核心指标配一个趋势图
- 科室排名应谨慎使用，建议采用"区间分布"而非严格排名（避免恶性竞争）
- 月报中应包含上月改进措施的"回头看"评估
- 年报应包含与同级标杆医院的对标分析

## 约束条件
- 报告数据必须经过校验，不得因计算错误影响管理决策
- 涉及医生个人的数据不得在全院级别报告中公开
- 报告生成时间：日报当日8:00前、周报周一10:00前、月报5日前
- 所有报告需保留版本记录，支持追溯和修订
