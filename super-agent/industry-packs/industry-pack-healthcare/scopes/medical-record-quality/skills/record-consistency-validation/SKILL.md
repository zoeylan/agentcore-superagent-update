# 病历内容一致性校验

## 概述
本技能对病历文书中不同部分之间的逻辑一致性进行交叉验证，识别前后矛盾、逻辑冲突和信息不匹配问题。一致性问题是病历质量的核心缺陷，反映可能的诊疗逻辑错误或记录粗疏，严重的一致性问题可能影响医疗安全和法律举证效力。

## 适用场景
- 入院记录与首次病程记录的一致性核查
- 诊断与医嘱/治疗方案的匹配性检查
- 手术同意书与手术记录的一致性验证
- 入院诊断与出院诊断变化的合理性评估
- 病程记录中病情描述与检查结果的一致性
- 护理记录与医嘱执行的一致性

## 执行步骤

### 步骤1：建立校验维度矩阵
至少覆盖以下5个核心校验维度：
1. **诊断-医嘱一致性**：确认诊断与药物治疗方案匹配（如糖尿病诊断应有降糖治疗、高血压应有降压药）
2. **主诉-现病史一致性**：主诉中的症状应在现病史中有详细展开描述
3. **入院-出院诊断一致性**：诊断变化应有病程记录说明变更原因
4. **手术同意书-手术记录一致性**：手术名称、方式、范围应一致
5. **检查结果-诊断一致性**：辅助检查结果应支持所下诊断

### 步骤2：执行交叉比对
- 提取各文书中的关键信息点（诊断名称、症状描述、检查数值等）
- 将相关信息点进行逻辑配对
- 应用临床规则库判断配对信息是否一致
- 识别矛盾点和可疑不匹配

### 步骤3：评估不一致的严重程度
- **严重不一致（A类）**：可能影响医疗安全或法律举证（如诊断与治疗完全不符、手术记录与实际操作不符）
- **中度不一致（B类）**：反映记录疏忽但不影响安全（如体温记录前后数值矛盾、时间记录不一致）
- **轻度不一致（C类）**：表述差异但不影响理解（如同一诊断使用不同表述方式）

### 步骤4：生成校验报告
- 列出所有发现的不一致项
- 标注每项的严重等级和涉及的文书位置
- 提供修改建议（明确指出应以哪份记录为准）
- 对A类不一致项标红提醒，建议优先处理

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patient_id | string | 是 | 患者ID |
| admission_id | string | 是 | 住院号/就诊号 |
| records_bundle | object | 是 | 相关联的病历文书集合 |
| validation_dimensions | array | 否 | 指定校验维度（默认全部） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| consistency_score | float | 一致性得分（0-100） |
| inconsistencies | array | 不一致项列表 |
| inconsistencies[].dimension | string | 所属校验维度 |
| inconsistencies[].severity | enum | A/B/C级 |
| inconsistencies[].location | object | 涉及的文书和具体位置 |
| inconsistencies[].description | string | 不一致描述 |
| inconsistencies[].suggestion | string | 修改建议 |

## 最佳实践
- 一致性校验应结合临床专业判断，同一疾病在不同阶段的表述变化可能是合理的
- 对于诊断变更，重点检查是否有对应的病程记录说明变更依据
- 手术相关一致性校验优先级最高，涉及法律风险
- 校验规则应定期更新，纳入新发现的常见不一致模式
- 建议在科室间共享典型不一致案例（脱敏后）用于培训

## 约束条件
- 不得对临床诊疗决策的正确性做判断，仅检查记录层面的一致性
- 对于临床合理的变化（如治疗过程中诊断修正），应有更高的容忍度
- 误报率应控制在≤10%，避免过多干扰影响医生接受度
- 涉及多学科会诊的病历，应考虑不同医生表述习惯的差异
