# 不良事件主动识别

## 概述
本技能通过预设的触发规则和多源数据监测，自动识别潜在的医疗不良事件。主动识别是不良事件管理的第一道防线，通过系统化监测弥补人工上报的滞后性和遗漏风险，确保重要安全事件不被遗漏。

## 适用场景
- 实时监测HIS/EMR系统中的异常临床事件
- 检验检查系统危急值未处理的超时监控
- 手术系统中非计划再手术的自动识别
- 护理系统中跌倒/坠床/压疮等事件捕获
- 药物系统中严重药物不良反应的自动关联

## 执行步骤

### 步骤1：配置触发规则库（≥20个规则）
核心触发规则包括：
1. 非计划再手术（同一住院期间或30天内再次手术）
2. 非计划转ICU（非术后常规入ICU）
3. 非计划再入院（出院31天内同一/相关诊断再入院）
4. 院内心跳骤停/呼吸骤停
5. 院内死亡（非预期，排除DNR患者）
6. 手术部位/手术患者/手术方式错误
7. 术中/术后大出血（需紧急输血≥4U）
8. 输血反应
9. 严重药物不良反应（ADR严重级别）
10. 致命性药物相互作用（DDI）触发
11. 危急值通知超时（>30分钟未确认处理）
12. 新生儿Apgar评分≤3分
13. 院内跌倒/坠床
14. 3期及以上压疮（院内新发）
15. 导管意外脱落（气管插管、中心静脉导管等）
16. 院内感染暴发（同一科室短期内≥3例）
17. 围手术期死亡（术后48小时内非预期死亡）
18. 产妇大出血（>1000ml）
19. 医疗器械相关不良事件
20. 异物遗留体内

### 步骤2：实时数据扫描
- 定时（每5分钟）扫描各系统的新增数据
- 将数据与触发规则进行匹配
- 对匹配结果进行初步过滤（排除已知的例外情况，如预期死亡）
- 标记触发的事件并记录触发源数据

### 步骤3：事件初步验证
- 交叉核对相关系统数据确认事件真实性
- 排除录入错误导致的误触发
- 确认事件与当前住院的关联性
- 初步评估事件严重程度

### 步骤4：事件推送与记录
- 将识别到的潜在事件推送给事件分级上报技能
- 同时通知相关科室质控员进行确认
- 记录触发规则、触发时间、数据源等信息
- 未确认的事件在24小时后自动提醒跟进

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| data_source | enum | 是 | 数据来源系统（HIS/EMR/LIS/PACS/手术/护理/药物） |
| event_data | object | 是 | 事件原始数据 |
| patient_context | object | 否 | 患者背景信息（用于排除例外） |
| trigger_rules_version | string | 否 | 规则版本号 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| event_detected | boolean | 是否检测到事件 |
| trigger_rule_id | string | 触发的规则ID |
| trigger_rule_name | string | 规则名称 |
| event_severity_preliminary | enum | 初步严重度（高/中/低） |
| patient_id | string | 涉及患者ID |
| department_id | string | 涉及科室 |
| event_description | string | 事件描述 |
| confidence | float | 检测置信度 |
| requires_confirmation | boolean | 是否需要人工确认 |

## 最佳实践
- 触发规则应定期（至少季度）回顾更新，纳入新发现的风险模式
- 设置合理的过滤阈值，平衡检出率和误报率（目标：敏感度≥95%，特异度≥80%）
- 对于高置信度事件（如手术部位错误），无需等待人工确认即可启动上报流程
- 建立"白名单"机制（如已知的计划性再手术），减少误报
- 每月统计各规则的触发次数和确认率，优化规则准确性

## 约束条件
- 系统监测不替代人工上报渠道，两个通道并行运行
- 误触发事件应在24小时内由科室质控员确认关闭
- 触发规则的增删改需经医务部和质控委员会审批
- 数据扫描不得影响临床系统的正常运行性能
