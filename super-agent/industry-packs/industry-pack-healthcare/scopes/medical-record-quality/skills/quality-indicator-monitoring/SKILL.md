# 医疗质量指标监测

## 概述
本技能负责医疗质量核心指标的实时采集、计算和阈值监控。涵盖病历质量、医疗效率、患者安全、费用控制和DRG/DIP相关等多维度指标体系，是医疗质量管理数据基础设施的核心能力。当指标偏离目标阈值时自动触发预警通知。

## 适用场景
- 每日定时采集和计算各科室质量指标
- 实时监测关键安全指标（如非计划再入院、院内感染）
- 指标越限时的即时预警触发
- 为质量分析报告提供数据支撑
- DRG/DIP运营指标的日常监控

## 执行步骤

### 步骤1：指标定义与数据采集
核心指标体系及数据源：

**病历质量指标**
- 病历甲级率 = 甲级病历数/已评级病历总数（目标≥90%）
- 病历时限达标率 = 按时完成病历数/应完成病历数（目标≥95%）

**效率指标**
- 平均住院日 = 总住院日数/出院人次（对标同级医院均值）
- 术前平均住院日 = 术前住院总日数/手术人次
- 床位使用率 = 实际占用总床日数/实际开放总床日数

**安全指标**
- 非计划再入院率 = 31天内非计划再入院人次/同期出院人次（目标≤2.4%）
- 非计划再手术率 = 非计划再手术人次/手术总人次
- 院内感染率 = 院内感染人次/同期住院患者人次
- 手术并发症率 = 手术并发症发生人次/手术总人次

**费用指标**
- 药占比 = 药品费用/医疗总费用（目标≤30%）
- 耗占比 = 卫生材料费用/医疗总费用（目标≤20%）

**DRG/DIP指标**
- CMI值 = Σ(各DRG组权重×该组病例数)/总病例数
- 时间消耗指数 = 实际平均住院日/DRG标准住院日（目标≤1.0）
- 费用消耗指数 = 实际次均费用/DRG标准费用（目标≤1.0）

### 步骤2：指标计算
- 按日/周/月周期自动计算各指标数值
- 支持按科室、病种、医生等多维度分组计算
- 计算环比和同比变化率
- 标注统计显著性（避免小样本偏差导致的虚假预警）

### 步骤3：阈值判断与预警
- 指标偏离目标值20%以上触发黄色预警
- 指标偏离目标值50%以上触发红色预警
- 安全类指标（再入院率等）采用绝对阈值判断
- 预警信息包含：指标名称、当前值、目标值、偏离幅度、涉及科室

### 步骤4：数据存储与推送
- 指标数据存入时序数据库，保留≥3年历史
- 预警信息推送相关科室负责人和医务部
- 每日生成指标简报供管理层查阅
- 数据接口供报告生成技能调用

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| calculation_period | enum | 是 | 日/周/月/季度/年度 |
| department_filter | array | 否 | 指定科室（空为全院） |
| indicator_filter | array | 否 | 指定指标（空为全部） |
| comparison_mode | enum | 否 | 环比/同比/对标 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| indicators | array | 指标结果列表 |
| indicators[].name | string | 指标名称 |
| indicators[].value | float | 当前值 |
| indicators[].target | float | 目标值 |
| indicators[].deviation_rate | float | 偏离率 |
| indicators[].trend | enum | 上升/稳定/下降 |
| indicators[].alert_level | enum | 正常/黄色预警/红色预警 |
| alerts | array | 触发的预警列表 |

## 最佳实践
- 指标计算准确率要求≥99%，上线前必须与手工计算结果比对验证
- 新开科室或病例数极少的科室，指标波动大属正常，应设置最小样本量（如≥30例才出预警）
- 节假日后的指标波动应区分常规波动和真实异常
- DRG/DIP指标需及时跟进医保政策调整更新计算规则
- 建议与同城同级医院建立对标数据交换机制

## 约束条件
- 数据采集不得影响临床业务系统性能（避开业务高峰期批量计算）
- 患者级别的指标数据必须脱敏处理后才能用于统计分析
- 指标定义变更需经医务部审批并做好历史数据衔接
- 预警发送频率应有上限控制，避免"预警疲劳"
