# 质量趋势分析

## 概述
本技能对医疗质量指标进行时间序列分析和多维度下钻，识别趋势性问题和系统性异常，为管理决策提供数据支撑。趋势分析是从"发现问题"到"理解问题"的关键环节，连续3天或连续3周的指标偏离被定义为趋势性问题，需要启动专项分析和改进。

## 适用场景
- 日常指标偏离后的根因定位
- 周度/月度质量趋势研判
- 新政策或流程变更后的效果评估
- 季节性因素对质量指标的影响分析
- 科室间对标分析识别最佳实践

## 执行步骤

### 步骤1：数据准备与趋势识别
- 提取目标指标的时间序列数据（至少6个月历史）
- 应用移动平均法平滑短期波动
- 使用统计过程控制（SPC）方法识别特殊原因变异
- 判定趋势类型：持续上升/持续下降/突变/周期性波动

### 步骤2：多维度下钻分析
当发现指标异常时，按以下维度逐层下钻：
- **科室维度**：哪些科室贡献了主要偏离？
- **病种维度**：哪些DRG组或疾病类型是主要驱动因素？
- **医生维度**：是个别医生还是普遍现象？（注意：医生层面仅供内部管理参考，不公开排名）
- **时间维度**：偏离从何时开始？是否与某事件相关？
- **患者维度**：病情复杂度（CMI）是否发生变化？

### 步骤3：关联因素分析
- 检查是否有流程变更、人员变动、政策调整等外部因素
- 分析指标间的联动关系（如平均住院日增加是否伴随CMI值上升）
- 评估季节性因素（如流感季对呼吸科指标的影响）
- 关联不良事件数据，判断是否有安全隐患

### 步骤4：趋势判定与输出
- **偶发偏离**：单次超限，无持续趋势→推送科室关注，纳入监测
- **趋势性问题**：连续3天（日指标）或3周（周指标）偏离→生成专项分析报告
- **系统性问题**：多科室/多指标同时偏离→升级至医务部层面

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| indicator_name | string | 是 | 分析的目标指标 |
| analysis_period | object | 是 | 分析时间范围（start_date, end_date） |
| drill_down_dimensions | array | 否 | 下钻维度 |
| comparison_baseline | enum | 否 | 对比基线（去年同期/上月/目标值） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| trend_type | enum | 偶发/趋势性/系统性 |
| trend_direction | enum | 上升/下降/波动 |
| start_point | date | 趋势起始时间 |
| contributing_factors | array | 主要贡献因素 |
| drill_down_results | object | 各维度下钻结果 |
| root_cause_hypothesis | array | 根因假设（按可能性排序） |
| recommended_actions | array | 建议采取的行动 |
| confidence_level | float | 分析置信度 |

## 最佳实践
- 趋势分析应区分"统计显著"和"临床显著"，小幅持续偏离可能统计显著但临床影响有限
- 下钻分析应遵循"从宏观到微观"的逻辑，避免直接跳到个人层面
- 季节性因素应建立历史基线，避免每年重复预警相同的季节性波动
- 新政策实施后应设置观察期（通常4-8周），区分适应期波动和真实问题
- 分析结论应审慎，避免在数据不充分时给出确定性结论

## 约束条件
- 医生个人维度的分析结果仅供科主任和医务部管理使用，不得公开
- 分析报告中的患者案例必须脱敏
- 根因假设应标注置信度，避免误导决策
- 趋势判定应有明确的统计标准，不得凭主观感觉
