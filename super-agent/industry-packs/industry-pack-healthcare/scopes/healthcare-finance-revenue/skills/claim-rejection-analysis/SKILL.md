# 医保拒付分析 (Claim Rejection Analysis)

## 概述
对医保拒付数据进行系统分析，分类统计拒付原因，识别高频拒付模式，生成针对性预防建议并跟踪整改效果，目标将拒付率控制在3%以下。

## 适用场景
- 医保结算后发生拒付的原因分析
- 科室拒付率超标时的专项审计
- 医保政策变化后的拒付风险预评估
- 月度/季度拒付率趋势分析

## 执行步骤

### 1. 拒付数据收集与分类
- 接收医保中心反馈的拒付明细
- 按一级分类：编码问题/费用问题/政策违规/材料缺失/其他
- 按二级分类：细化到具体规则条目（如超限价、超疗程、适应症不符等）
- 关联原始病例和费用信息

### 2. 统计分析
- 按科室维度统计拒付率和拒付金额
- 按病种维度分析高拒付率DRG组
- 时间趋势分析（月度/季度趋势线）
- 识别高频拒付原因TOP10排名

### 3. 根因分析
- 对高频拒付原因进行深度根因分析
- 区分系统性问题（流程缺陷/知识不足）和偶发性问题
- 分析是否与特定医保政策变更相关
- 评估现有防控措施的有效性

### 4. 预防策略生成
- 针对TOP5拒付原因生成预防规则
- 更新结算前预审的检查规则库
- 制定科室层面的培训和整改建议
- 设定改进目标和时间节点

### 5. 效果跟踪
- 跟踪整改措施实施后的拒付率变化
- 对比改进前后同类拒付的发生率
- 季度内同一原因≥3次视为整改未到位，升级通知
- 定期更新拒付风险预警规则库

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| rejectionRecords | array | 是 | 拒付记录列表 |
| rejectionReason | string | 是 | 拒付原因代码/描述 |
| rejectedAmount | number | 是 | 拒付金额 |
| departmentId | string | 是 | 科室代码 |
| drgCode | string | 否 | DRG分组代码 |
| timeRange | object | 是 | 分析时间范围 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| rejectionRate | number | 总体拒付率 |
| topReasons | array | TOP10拒付原因排名 |
| departmentRanking | array | 科室拒付率排名 |
| rootCauseAnalysis | object | 根因分析报告 |
| preventionStrategies | array | 预防策略建议 |
| trendAnalysis | object | 趋势分析结果 |

## 最佳实践
- 拒付分析应在收到反馈后5个工作日内完成
- 区分可申诉拒付和不可申诉拒付，可申诉的及时处理
- 高频拒付原因应纳入新入院病例的前置检查
- 医保政策变化后主动进行影响评估

## 约束条件
- 分析结果实事求是，不掩盖问题
- 拒付率目标<3%，持续超标科室需上报医务部
- 不得建议通过修改病历或伪造材料来应对拒付
- 拒付分析报告需保密，按权限查阅
