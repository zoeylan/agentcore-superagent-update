# 科室收入分析报告 (Department Revenue Reporting)

## 概述
按月自动生成各科室维度的医疗收入分析报告，包含收入总额与构成分析、DRG盈亏分布、费用消耗指数、收入漏洞统计、拒付分析等，支持科室收支平衡分析和运营决策。

## 适用场景
- 月度科室收入分析报告自动生成
- 季度经营分析会数据支撑
- 科室绩效考核的财务维度数据
- 新项目/新技术开展后的收益评估

## 执行步骤

### 1. 数据汇总
- 按科室汇总月度收入总额
- 分项收入统计：药品、耗材、检查、手术、床位、护理、其他
- 医保/自费/商保分类统计
- 门诊/住院收入分别统计

### 2. DRG盈亏分析
- 计算各DRG组实际费用vs支付标准的差额
- 识别盈余DRG组（实际<标准）和亏损DRG组（实际>标准）
- 分析亏损组的费用超支原因
- 计算科室总体DRG盈亏率

### 3. 效率指标计算
- 费用消耗指数（FCI）= 科室病例平均费用/同DRG全院平均费用
- 时间消耗指数（TCI）= 科室平均住院日/同DRG全院平均住院日
- CMI值统计
- 床日均费用和人次均费用

### 4. 收支平衡分析
- 计算科室收入-成本=贡献值
- 固定成本和变动成本分拆
- 盈亏平衡点计算
- 边际贡献率分析

### 5. 报告生成
- 组装标准化报告模板
- 生成趋势图表（12个月对比）
- 同比/环比分析
- 管理建议和风险提示
- 按权限分级推送

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| departmentId | string | 是 | 科室代码 |
| reportMonth | string | 是 | 报告月份（YYYY-MM） |
| revenueData | object | 是 | 各项收入数据 |
| costData | object | 是 | 各项成本数据 |
| drgCaseData | array | 是 | DRG病例数据 |
| historicalData | array | 否 | 历史数据（对比用） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| reportDocument | object | 完整报告文档 |
| revenueTotal | number | 收入总额 |
| revenueBreakdown | object | 收入构成明细 |
| drgProfitLoss | object | DRG盈亏分析 |
| efficiencyMetrics | object | 效率指标（FCI/TCI/CMI） |
| breakEvenAnalysis | object | 收支平衡分析 |
| recommendations | array | 管理建议 |

## 最佳实践
- 报告在次月10日前完成
- 数据异常需标注并说明原因
- 与绩效考核方案的指标口径保持一致
- 提供可视化图表便于管理层快速理解

## 约束条件
- 财务数据严格保密，按权限分级查阅
- 报告数据必须与财务系统一致
- 分析结论客观中立，不预设立场
- 不得用于不当比较导致科室间恶性竞争
