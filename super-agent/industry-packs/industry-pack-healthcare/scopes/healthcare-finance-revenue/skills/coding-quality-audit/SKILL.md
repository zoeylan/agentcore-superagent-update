# 编码质量审核 (Coding Quality Audit)

## 概述
对病案首页的诊断编码和手术操作编码进行质量审核，检查编码的准确性、完整性和规范性，确保主要诊断选择正确率≥90%、手术操作编码完整率≥95%。

## 适用场景
- 出院病案首页编码的常规质量审核
- DRG分组异常病例的编码回溯审查
- 月度编码质量统计与科室评分
- 新编码员入职后的编码质量监控

## 执行步骤

### 1. 主要诊断选择审核
- 验证主要诊断是否反映本次住院的主要原因
- 检查主诊选择是否符合"消耗最多医疗资源"原则
- 验证与出院小结中的诊断表述一致性
- 检测常见选择错误（如症状代替确诊诊断）

### 2. 手术操作编码完整性
- 比对医嘱系统中的手术/操作记录
- 检查所有有编码意义的操作是否全部编码
- 验证手术编码与手术记录的一致性
- 识别漏编的关键操作（影响DRG分组的操作优先）

### 3. 合并症/并发症审核
- 检查其他诊断编码与病程记录的一致性
- 验证CC/MCC标记的合理性
- 识别遗漏的重要合并症（影响分组权重）
- 检测虚报合并症的风险

### 4. 编码规范性检查
- ICD编码版本和位数验证
- 编码与性别/年龄的逻辑一致性
- 互斥编码检测（不应同时出现的编码组合）
- 编码精确度检查（是否使用了足够精确的编码）

### 5. 审核结果输出
- 生成编码质量评分（百分制）
- 标注具体错误和改进建议
- 统计科室/编码员维度的质量分布
- 高频错误模式识别并生成培训建议

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| caseId | string | 是 | 病例ID |
| principalDiagnosis | object | 是 | 主要诊断（编码+描述） |
| otherDiagnoses | array | 否 | 其他诊断列表 |
| procedures | array | 否 | 手术操作编码列表 |
| medicalRecordSummary | object | 是 | 病历摘要信息 |
| orderHistory | array | 否 | 医嘱执行记录 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| qualityScore | number | 编码质量评分(0-100) |
| principalDxCorrect | boolean | 主诊选择是否正确 |
| procedureCompleteness | number | 操作编码完整率 |
| errors | array | 发现的编码错误列表 |
| suggestions | array | 改进建议 |
| drgImpact | object | 编码问题对DRG的影响评估 |

## 最佳实践
- 重点审核影响DRG分组的关键编码
- 编码审核与病历内容交叉验证
- 新手编码员100%审核，资深编码员抽查20%
- 建立常见错误模式库用于快速检测

## 约束条件
- 审核结论需有客观依据（引用编码规则）
- 不得以DRG优化为由建议不真实的编码
- 审核结果用于质量改进而非惩罚
- 主要诊断正确率目标≥90%，操作完整率≥95%
