# 费用超标预警 (Cost Overrun Alerting)

## 概述
实时监控住院病例费用累积情况，当实际费用接近或超过DRG支付标准时自动触发分级预警，提供费用结构分析和控制建议，帮助临床团队在治疗过程中关注费用合理性。

## 适用场景
- 住院期间实际费用接近DRG支付标准阈值
- 长住院日病例的持续费用监控
- 高值耗材使用后的费用影响评估
- ICU转入/大型手术后的费用走势预判

## 执行步骤

### 1. DRG支付标准获取
- 确认病例的DRG分组结果和对应支付标准
- 获取该DRG组的历史费用分布（均值/P25/P50/P75）
- 确定预警阈值（默认80%支付标准，可分科室调整）
- 设置红色预警阈值（100%支付标准）

### 2. 实时费用监控
- 每日汇总已发生费用（药品+耗材+检查+治疗+床位+护理）
- 计算当前费用占DRG支付标准的比例
- 预估剩余住院日的费用走势（基于历史同类病例曲线）
- 标记异常费用增长（单日费用突增≥50%）

### 3. 预警触发与通知
- 黄色预警（80%阈值）：通知主管医生，提供费用结构分析
- 橙色预警（90%阈值）：升级通知科室负责人
- 红色预警（100%阈值）：通知科主任和医务部，建议启动费用管控措施
- 通知内容含：当前费用/支付标准/主要超支项/建议措施

### 4. 费用结构分析
- 分解费用构成（药品占比/耗材占比/检查占比/其他）
- 与同DRG组标杆病例对比各项占比
- 识别主要超支驱动因素
- 分析是否有可替代的低成本方案

### 5. 控制建议生成
- 基于费用结构分析生成针对性建议
- 药品超支：检查是否有同疗效低价替代
- 耗材超支：核实高值耗材使用必要性
- 住院日超标：评估是否可转日间手术或提前出院
- 所有建议注明不影响医疗质量前提

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| caseId | string | 是 | 住院病例ID |
| drgCode | string | 是 | DRG分组代码 |
| paymentStandard | number | 是 | DRG支付标准金额 |
| currentCosts | object | 是 | 当前各项费用明细 |
| admissionDate | date | 是 | 入院日期 |
| expectedLOS | number | 否 | 预估住院天数 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| alertLevel | enum | none/yellow/orange/red |
| costRatio | number | 当前费用/支付标准比例 |
| costBreakdown | object | 费用结构分析 |
| mainDrivers | array | 主要超支驱动因素 |
| recommendations | array | 费用控制建议 |
| projectedTotal | number | 预估出院总费用 |

## 最佳实践
- 预警阈值应根据科室特点差异化设置（如ICU可适当放宽）
- 预警通知应在工作时间内发送，避免夜间干扰
- 对已知高费用病例（大手术/重症）提前标记避免误报
- 定期回顾预警准确率调整阈值

## 约束条件
- 预警仅为提醒作用，不得干预临床必要的治疗决策
- 费用控制不能以牺牲医疗质量为代价
- 不得建议延迟必要检查或治疗以压低费用
- 预警数据和建议仅限授权管理人员查看
