# DRG病例分组 (DRG Case Grouping)

## 概述
基于病案首页信息（主要诊断、手术操作、合并症/并发症、年龄、性别等）执行DRG分组预测，验证入组准确性，识别影响分组的关键编码要素，提升DRG入组准确率至≥95%。

## 适用场景
- 住院病例出院前的DRG分组预判
- 病案首页提交后的分组结果验证
- 编码修改后的分组影响评估
- 批量病例的分组质量回顾分析

## 执行步骤

### 1. 病案首页信息提取
- 提取主要诊断（ICD-10编码）和其他诊断
- 提取手术/操作编码（ICD-9-CM-3）
- 确认患者基本信息（年龄、性别、出院方式）
- 检查信息完整性（必填项是否齐全）

### 2. MDC（主要诊断类别）判定
- 根据主要诊断确定所属MDC分类
- 验证MDC与手术操作的逻辑一致性
- 特殊MDC规则处理（如新生儿、HIV、多发伤）
- 标记MDC判定不确定的情况

### 3. DRG分组执行
- 在确定MDC内按照决策树执行分组
- 考虑手术操作类型（大/中/小手术/非手术）
- 纳入合并症/并发症的严重程度评估（CC/MCC）
- 输出DRG分组代码和权重值

### 4. 分组结果验证
- 与历史同类病例的DRG分布对比
- 检查是否存在异常高权重或低权重分组
- 验证分组与费用的合理性关系
- 标记可能的编码优化点（合规范围内）

### 5. 优化建议输出
- 识别影响分组的关键编码要素
- 提供合规范围内的编码精确化建议
- 标注不可操作的编码调整（避免upcoding）
- 生成分组分析报告

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| principalDiagnosis | string | 是 | 主要诊断ICD-10编码 |
| secondaryDiagnoses | array | 否 | 其他诊断编码列表 |
| procedures | array | 否 | 手术/操作编码列表 |
| patientAge | number | 是 | 患者年龄 |
| patientGender | string | 是 | 性别（M/F） |
| dischargeStatus | string | 是 | 出院方式 |
| lengthOfStay | number | 是 | 住院天数 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| drgCode | string | DRG分组代码 |
| drgWeight | number | DRG权重值 |
| mdc | string | 主要诊断类别 |
| paymentStandard | number | 对应支付标准金额 |
| optimizationSuggestions | array | 编码优化建议 |
| riskFlags | array | 风险标记 |

## 最佳实践
- 分组准确性以季度专家评审结果为金标准
- 关注MCC/CC的准确识别（对权重影响大）
- 对比DRG权重与实际费用，识别系统性偏差
- 新政策发布后及时更新分组规则引擎

## 约束条件
- 不得建议不合理的高编码（upcoding）
- 编码优化必须基于临床事实
- CMI提升必须来自编码准确性而非操控
- DRG入组准确率目标≥95%
