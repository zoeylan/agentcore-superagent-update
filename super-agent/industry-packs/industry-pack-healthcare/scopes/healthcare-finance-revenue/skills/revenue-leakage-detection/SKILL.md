# 收入漏洞识别 (Revenue Leakage Detection)

## 概述
通过医嘱-执行-收费三方数据比对，自动检测漏收费项目（已使用耗材未记费、已完成操作未计费、已出报告检查未收费等），在费用结清前提醒补录，堵塞收入漏洞。

## 适用场景
- 每日住院费用的漏收费批量检测
- 出院结算前的费用完整性终审
- 高值耗材使用的收费核对
- 护理操作记录与收费项的匹配验证

## 执行步骤

### 1. 数据采集与比对准备
- 提取患者医嘱数据（所有已执行的治疗/检查/用药医嘱）
- 提取执行记录（护理操作记录、手术记录、检验报告等）
- 提取收费明细（已入账的所有费用项目）
- 构建三方数据关联矩阵

### 2. 医嘱-收费比对
- 已开立且已执行的医嘱是否有对应收费项
- 识别有医嘱执行但无收费的项目（潜在漏收费）
- 识别有收费但无医嘱的项目（潜在多收费/串换）
- 批量药品/耗材的数量核对

### 3. 执行-收费比对
- 护理操作记录（换药、导管护理、特殊监护等）与护理收费对应
- 手术中使用的耗材清单与耗材收费对应
- 检验检查报告与对应检查收费对应
- 麻醉时间记录与麻醉费计算核对

### 4. 漏洞分类与优先级排定
- 按金额排序（高值优先处理）
- 按确定性分级：确定漏收费/疑似漏收费/需人工确认
- 区分可补录（未结算）和不可补录（已结算）
- 计算潜在收入损失金额

### 5. 补录提醒与统计
- 在费用结清前（出院后3个工作日内）提醒相关科室补录
- 明确标注漏收费项目、金额和补录路径
- 统计月度漏收费总量和科室分布
- 分析漏收费高频类型生成流程改进建议

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者ID |
| orders | array | 是 | 医嘱列表（含执行状态） |
| executionRecords | array | 是 | 操作执行记录 |
| chargedItems | array | 是 | 已收费项目明细 |
| admissionStatus | string | 是 | 在院/已出院状态 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| leakageItems | array | 发现的漏收费项目列表 |
| leakageAmount | number | 潜在漏收费总金额 |
| confidence | enum | confirmed/suspected/needs_review |
| actionRequired | string | 建议操作（补录/确认/忽略） |
| departmentStats | object | 科室漏收费统计 |

## 最佳实践
- 高值耗材（单价>1000元）优先检测
- 出院当日执行一次全面终审
- 建立常见漏收费模式库用于快速匹配
- 流程改进优于事后补录

## 约束条件
- 漏收费检测仅限于真实发生的医疗服务
- 不得建议对未实际发生的服务进行收费
- 疑似漏收费需经科室确认后方可补录
- 已结算病例的漏收费仅作统计不可追收
