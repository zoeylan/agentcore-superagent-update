# 医保结算智能化 (Medical Insurance Settlement)

## 概述
支持住院病例医保结算的自动化预审、提交和跟踪管理，通过智能规则引擎在结算前识别潜在合规问题，减少拒付风险，提升结算效率。

## 适用场景
- 出院病例医保结算前的合规性预审
- 批量结算提交与状态跟踪
- 结算异常处理和申诉材料准备
- 医保政策变更后的结算规则更新

## 执行步骤

### 1. 结算前数据整备
- 确认病案首页编码完成且通过质量审核
- 核实费用清单与医嘱一致性
- 检查医保患者身份和参保信息有效性
- 确认住院天数和费用在合理范围内

### 2. 合规性预审
- 诊断与治疗项目一致性验证
- 医保目录合规性检查（甲/乙/丙类分类）
- 限价药品/耗材价格核对
- 分解住院嫌疑检测（短期多次住院同一诊断）
- 超适应症用药检测
- 自费项目患者知情同意确认

### 3. 快速通道判定
- 评估病例是否满足快速通道条件
- 条件：无预审问题+编码审核通过+费用在DRG标准±20%内
- 满足条件：自动提交结算
- 不满足条件：进入人工复核队列，标注关注点

### 4. 结算提交与跟踪
- 按医保中心要求格式打包结算数据
- 提交并记录提交时间和批次号
- 跟踪结算状态（已提交/审核中/已支付/已拒付）
- 支付到账确认

### 5. 异常处理
- 拒付病例的原因解析和应对建议
- 可申诉案例的申诉材料自动准备
- 申诉跟踪直至最终裁定
- 拒付数据反馈至拒付分析流程

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| caseId | string | 是 | 住院病例ID |
| insuranceType | string | 是 | 医保类型（城镇职工/居民/新农合） |
| diagnosisCodes | array | 是 | 诊断编码列表 |
| feeDetails | array | 是 | 费用明细清单 |
| coverPageData | object | 是 | 病案首页数据 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| preAuditResult | enum | pass/issues_found/blocked |
| issues | array | 发现的合规问题列表 |
| fastTrackEligible | boolean | 是否满足快速通道 |
| settlementStatus | enum | submitted/processing/paid/rejected |
| appealMaterials | object | 申诉材料（如适用） |

## 最佳实践
- 预审覆盖率100%，不遗漏任何住院病例
- 快速通道比例目标≥70%（减轻人工审核负担）
- 拒付后7个工作日内完成申诉材料准备
- 定期与医保中心沟通了解政策动向

## 约束条件
- 不得篡改病历或费用数据以通过预审
- 发现欺诈行为必须上报
- 患者自费项目需有知情同意记录
- 结算数据需与HIS/财务系统实时同步
