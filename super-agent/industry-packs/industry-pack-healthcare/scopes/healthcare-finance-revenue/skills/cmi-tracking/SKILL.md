# CMI值追踪分析 (CMI Tracking)

## 概述
月度追踪医院和各科室的CMI值（Case Mix Index，病例组合指数）变化趋势，分析CMI变化的驱动因素，区分收治结构调整与编码质量变化的影响，为医院学科发展和运营管理提供数据支撑。

## 适用场景
- 月度CMI值变化追踪与分析
- 科室维度CMI排名与对标
- CMI下降时的根因诊断
- 学科建设规划的数据支撑

## 执行步骤

### 1. CMI数据计算
- 汇总本月所有出院病例的DRG权重值
- 计算全院CMI = Σ(DRG权重) / 总病例数
- 分科室计算各科CMI值
- 对比上月/去年同期数据

### 2. 趋势分析
- 绘制12个月CMI滚动趋势线
- 标记异常波动点（环比变化>5%）
- 季节性因素分析（是否有规律性变化）
- 与同级医院标杆CMI对比

### 3. 驱动因素分解
- 分解CMI变化为两个维度：
  - 收治结构变化（高权重病种占比变化）
  - 编码质量变化（同病种编码精确度变化）
- 识别CMI提升/下降的Top5科室贡献
- 分析是否有新开展的高难度手术/技术影响

### 4. 异常识别
- 检测CMI虚高风险（可能的编码操控信号）
- CMI持续下降趋势预警（可能反映学科能力下降）
- 单科室CMI突变分析（人员变动/收治策略调整）
- 与时间消耗指数/费用消耗指数交叉验证合理性

### 5. 管理建议输出
- 生成月度CMI分析报告
- 提出学科发展建议（基于高权重DRG组分析）
- 编码质量提升建议（CMI提升的合规路径）
- 为等级医院评审提供CMI达标路径建议

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dischargedCases | array | 是 | 本月出院病例及DRG信息 |
| historicalCMI | array | 是 | 历史CMI数据（至少12个月） |
| departmentList | array | 是 | 科室列表 |
| benchmarkCMI | number | 否 | 同级医院标杆CMI值 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| currentCMI | number | 本月全院CMI值 |
| cmiTrend | array | 12个月趋势数据 |
| departmentCMI | array | 各科室CMI排名 |
| changeDrivers | object | 变化驱动因素分解 |
| anomalies | array | 异常标记 |
| recommendations | array | 管理建议 |

## 最佳实践
- CMI提升应来自编码准确性和学科能力提升
- 关注CMI与费用消耗指数的关系（高CMI低费用消耗=高效率）
- 季度对比比月度对比更能反映真实趋势
- CMI分析结合DRG盈亏分析更有管理价值

## 约束条件
- CMI追踪是客观分析工具，不得用于施压临床科室
- 不得建议通过操控编码来虚增CMI
- CMI目标设定需结合医院实际发展阶段
- 分析报告按权限分级发布
