# 分诊评估 (Triage Assessment)

## 概述
基于采集的症状、生命体征和病史信息，执行标准化分诊评估流程，确定患者的紧急程度等级和最适合的就诊科室，确保急危重症患者得到优先救治。

## 适用场景
- 患者完成症状采集后的科室推荐
- 急诊患者的CTAS五级分诊
- 患者自述症状变化时的重新评估
- 候诊期间病情变化的再分诊

## 执行步骤

### 1. 信息汇总
- 整合症状采集结果（主诉+伴随症状+病史）
- 纳入生命体征数据（如已采集：体温、血压、心率、呼吸、血氧）
- 关联患者历史诊断和近期就诊记录
- 标注特殊人群标识（年龄、妊娠状态等）

### 2. 紧急程度评估
- 执行Red Flags危险信号筛查（一票否决制）
- 根据CTAS标准评估分级：
  - I级（复苏）：生命体征不稳、心搏骤停、严重创伤
  - II级（紧急）：急性胸痛、呼吸困难、意识改变、急性脑卒中征象
  - III级（急症）：中度疼痛、高热、中等度出血
  - IV级（亚急症）：轻度外伤、低热、轻度疼痛
  - V级（非急症）：慢性症状随访、换药、开药
- 非急诊患者标记为"普通门诊"

### 3. 科室匹配
- 基于症状-疾病-科室知识图谱进行匹配
- 输出Top3推荐科室及各自置信度
- 考虑交叉症状：如胸痛需区分心内科/呼吸科/骨科/消化科
- 纳入患者既往就诊科室作为参考权重

### 4. 置信度评估与人工介入判断
- 计算综合置信度分数（0-100%）
- 置信度≥70%：直接输出推荐结果
- 置信度50-70%：输出结果并标注"建议人工复核"
- 置信度<50%：强制触发人工分诊介入
- 多科室置信度接近（差距<10%）时展示所有选项供患者选择

### 5. 结果输出与记录
- 生成分诊评估报告
- 记录分诊决策依据（用于后续准确率统计）
- 将结果传递给预约调度Agent（普通门诊）或急诊调度（急诊）
- 症状摘要同步传递给接诊医生工作站

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者ID |
| symptomData | object | 是 | 症状采集结构化数据 |
| vitalSigns | object | 否 | 生命体征 |
| medicalHistory | object | 否 | 既往病史 |
| patientAge | number | 是 | 年龄 |
| isPregnant | boolean | 否 | 是否妊娠 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| urgencyLevel | enum | CTAS_I/II/III/IV/V/NORMAL |
| recommendedDepartments | array | Top3推荐科室（含置信度） |
| overallConfidence | number | 综合置信度（0-100%） |
| needsHumanReview | boolean | 是否需要人工复核 |
| redFlags | array | 识别到的危险信号 |
| triageRationale | string | 分诊决策依据说明 |
| estimatedWaitCategory | string | 预计等候类别 |

## 最佳实践
- Red Flags检查必须在所有其他评估之前执行
- 对儿童和老年人的症状评估应适当提高紧急等级权重
- 妊娠期患者的症状需额外考虑产科急症可能
- 分诊结果应定期与最终诊断比对，持续校准匹配算法

## 约束条件
- 分诊准确率目标≥85%
- 急诊识别敏感度≥95%（宁可过度分诊，不可漏诊）
- 不得输出诊断结论，仅限科室推荐和紧急程度评估
- 置信度<50%时必须触发人工介入，不得自行决定
- 所有分诊记录保留用于质量回溯和模型优化
