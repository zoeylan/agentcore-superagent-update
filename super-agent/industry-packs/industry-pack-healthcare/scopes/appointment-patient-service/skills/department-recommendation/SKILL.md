# 科室推荐 (Department Recommendation)

## 概述
基于症状-疾病-科室知识图谱，将患者的主诉和伴随症状映射到最合适的就诊科室，输出置信度排序的推荐列表，辅助患者选择正确的科室完成挂号。

## 适用场景
- 分诊评估环节的科室匹配
- 患者自助挂号时的科室推荐
- 多症状交叉时的多科室选择建议
- 复诊患者科室变更时的重新推荐

## 执行步骤

### 1. 症状特征提取
- 从结构化症状数据中提取核心特征：主症状、伴随症状、发病部位、病程
- 标准化为知识图谱可匹配的术语编码
- 识别症状组合模式（如"咳嗽+发热+咯痰"→呼吸系统模式）

### 2. 知识图谱匹配
- 基于症状-疾病关联权重计算各候选疾病的可能性
- 基于疾病-科室归属关系确定推荐科室
- 考虑交叉疾病的多科室归属（如糖尿病足：内分泌/血管外科/骨科）
- 纳入患者年龄、性别等流行病学因素调整权重

### 3. 科室可用性校验
- 验证推荐科室当日是否有出诊安排
- 检查号源可用性（作为参考信息，不影响推荐排序）
- 如推荐科室暂无号源，标注并提供同类替代科室

### 4. 置信度计算与排序
- 综合计算每个推荐科室的置信度分数
- 按置信度降序输出Top3
- 标注决策依据（哪些症状贡献了推荐结论）
- 如最高置信度<70%，标注"建议人工分诊确认"

### 5. 患者友好化输出
- 将专业科室名称转化为患者易理解的描述
- 附带简短说明（"您描述的症状可能与XX系统相关"）
- 对多科室推荐场景提供选择指引

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| symptoms | array | 是 | 结构化症状列表 |
| patientAge | number | 是 | 年龄 |
| patientGender | enum | 是 | 性别 |
| medicalHistory | array | 否 | 既往疾病列表 |
| previousDepartments | array | 否 | 近期就诊科室 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| recommendations | array | Top3推荐科室（含置信度和理由） |
| primaryRecommendation | object | 首选科室详情 |
| confidence | number | 最高置信度 |
| needsHumanReview | boolean | 是否需人工确认 |
| patientGuidance | string | 患者友好的选择说明 |

## 最佳实践
- 同症状在不同年龄段可能对应不同科室（如腹痛：儿童→小儿外科，青年女性→妇科/消化科）
- 保留患者近期就诊记录作为参考，复诊优先推荐原科室
- 对模糊症状（如"不舒服""全身无力"），推荐全科/内科作为首诊兜底
- 知识图谱需定期更新（至少季度更新），纳入新开科室和专病门诊

## 约束条件
- 推荐准确率目标≥85%（最终就诊科室在Top3中）
- 不得推荐已停诊或未设置的科室
- 不得给出诊断性结论，仅限科室方向推荐
- 推荐理由必须可解释、可追溯
- 特殊门诊（如发热门诊、肠道门诊）有强制分诊规则需优先执行
