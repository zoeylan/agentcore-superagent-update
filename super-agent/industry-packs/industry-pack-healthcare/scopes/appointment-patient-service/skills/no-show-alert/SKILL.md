# 爽约通知 (No-Show Alert)

## 概述
管理爽约事件发生后的患者通知流程，包括爽约告知、号源释放通知、爽约累计警告、以及限制生效通知，确保患者知情并引导行为改善。

## 适用场景
- 确认爽约后的即时通知
- 爽约累计达到警告阈值时的预警通知
- 爽约限制生效时的权限变更通知
- 爽约限制解除时的恢复通知
- 候补患者获得递补号源时的通知

## 执行步骤

### 1. 爽约确认通知
- 超约定时间15分钟确认爽约后发送通知
- 通知内容：告知本次预约已因未到诊被取消，号源已释放
- 告知当前季度爽约累计次数（如"本季度第X次爽约"）
- 说明下次如无法就诊请提前4小时取消

### 2. 累计预警通知
- 季度内第2次爽约时发送黄色预警
- 通知内容：再次爽约将触发专家号预约限制
- 明确说明限制规则和解除条件
- 提供申诉渠道信息

### 3. 限制生效通知
- 累计达到3次触发限制时立即通知
- 通知内容：专家号预约权限已限制，仅可预约普通号
- 说明限制持续时间（至当季度末）
- 提供申诉渠道和在线客服入口
- 语气坚定但不带惩罚性色彩

### 4. 号源递补通知
- 爽约释放的号源优先通知候补队列患者
- 通知内容：您候补的XX科室XX医生号源现已可用
- 提供快速预约链接（限时30分钟响应）
- 超时未响应自动通知下一位候补

### 5. 限制解除通知
- 新季度开始时自动解除限制并通知
- 通知内容：预约权限已恢复正常
- 温馨提醒按时就诊的重要性

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alertType | enum | 是 | noshow/warning/restricted/waitlist/released |
| patientId | string | 是 | 患者ID |
| noShowCount | number | 条件必填 | 当季度累计次数 |
| appointmentDetail | object | 条件必填 | 爽约的预约详情 |
| waitlistSlot | object | 条件必填 | 候补号源详情（waitlist类型时） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| alertId | string | 通知ID |
| sentStatus | enum | success/failed |
| patientAcknowledged | boolean | 患者是否已确认收到 |
| nextAction | string | 后续操作说明 |

## 最佳实践
- 爽约通知语气应客观中性，陈述事实而非指责
- 限制通知需配合解释"为何有此规则"（保障其他患者权益）
- 候补递补通知需设定合理响应时限，平衡紧迫感和便利性
- 所有通知都应提供明确的行动指引

## 约束条件
- 爽约通知必须在确认爽约后30分钟内送达
- 候补递补通知需在号源释放后5分钟内发出
- 通知中不得包含其他患者的信息
- 限制通知必须同时提供申诉渠道
- 爽约通知不得使用威胁性或惩罚性语言
