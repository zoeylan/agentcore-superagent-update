# 爽约管控 (No-Show Control)

## 概述
管理患者爽约行为的识别、记录、累计和限制执行，通过合理的奖惩机制降低爽约率（目标≤10%），提高号源利用效率，同时兼顾患者体验和公平性。

## 适用场景
- 患者未按时到诊时的爽约判定
- 爽约记录的累计和限制规则执行
- 预约时的爽约资格检查
- 爽约申诉处理
- 爽约数据统计与分析

## 执行步骤

### 1. 爽约判定
- 约定就诊时间超过15分钟，患者未签到（自助机/窗口/App）
- 系统自动标记为"疑似爽约"
- 等待至该时段结束（通常为30分钟窗口期）后确认为"爽约"
- 例外情况：急诊就诊（同日有急诊记录）、系统故障期间的预约不计入

### 2. 爽约记录与累计
- 记录爽约详情：患者ID、预约信息、爽约时间、号源类型
- 按季度滚动累计（每季度初清零）
- 区分号源类型：专家号爽约权重1.5、普通号权重1.0
- 实时更新患者爽约计数器

### 3. 限制规则执行
- 累计爽约≥3次/季度：锁定专家号预约权限，仅允许普通号
- 累计爽约≥5次/季度：限制线上预约渠道，需现场窗口预约
- 限制状态在下一季度初自动解除
- 限制期间允许通过申诉渠道申请解除

### 4. 爽约预防机制
- 预约确认环节明确告知爽约后果
- 就诊前24h和2h双重提醒
- 提供便捷的取消/改约入口（就诊前4小时前取消不计爽约）
- 对历史高爽约率患者增加确认环节

### 5. 号源释放处理
- 爽约确认后立即释放号源回池
- 通知候补队列第一位患者
- 更新当日号源可用状态

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | enum | 是 | check/record/appeal/statistics |
| patientId | string | 是 | 患者ID |
| appointmentId | string | 条件必填 | 预约ID（record时必填） |
| quarterRange | string | 否 | 查询季度范围 |
| appealReason | string | 条件必填 | 申诉原因（appeal时必填） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| noShowCount | number | 当季度爽约累计次数 |
| restrictionStatus | object | 限制状态（类型、生效期、解除条件） |
| isEligibleForExpert | boolean | 是否可预约专家号 |
| appealResult | enum | 申诉结果：approved/rejected/pending |

## 最佳实践
- 爽约判定应给予充分缓冲时间（15分钟），避免因交通延误误判
- 限制措施应通过通知明确告知患者，保障知情权
- 首次爽约给予警告但不计入（新患者引导机制）
- 特殊人群（老年人、残障人士）的爽约判定时间可适当延长至25分钟

## 约束条件
- 爽约判定必须有明确的时间证据（签到系统记录）
- 不得对因医院原因（停诊、系统故障）导致的未就诊计入爽约
- 限制解除必须有审批记录
- 爽约数据保留周期为2年，用于长期行为分析
