# 患者流量分析 (Patient Flow Analysis)

## 概述
对门诊患者流量进行历史分析、实时监控和趋势预测，为号源管理、排班优化和资源配置提供数据支撑，帮助医院实现精细化的门诊运营管理。

## 适用场景
- 每日门诊开诊前的流量预测
- 实时患者流量监控与预警
- 周/月维度的流量趋势分析
- 排班调整和资源配置的决策支持
- 节假日/季节性波动的预判

## 执行步骤

### 1. 历史数据分析
- 采集近12个月的门诊量数据（按科室、日期、时段）
- 分析周期性规律：周内分布（周一通常最高）、月内分布、季节性波动
- 识别影响因素：天气、节假日、学校开学/放假、传染病流行期
- 建立科室级别的流量基线模型

### 2. 实时流量监控
- 每15分钟统计全院及各科室的实时到诊人数
- 与同时段预测值对比，计算偏差率
- 偏差>20%时生成异常提醒（可能影响等候时间和资源调配）
- 监控各科室到诊率（实际到诊/预约数）

### 3. 短期流量预测
- T+1（明日）预测：基于预约数据+历史到诊率+天气+特殊因素
- T+7（下周）预测：基于预约趋势+周期性规律
- 输出各科室各时段的预计门诊量
- 标注高峰时段和可能的瓶颈科室

### 4. 运营决策建议
- 基于预测结果生成排班建议（是否需要增开诊室）
- 号源释放策略调整建议（高需求日提前释放更多号源）
- 人力资源调配建议（导诊、窗口、志愿者配置）
- 特殊时期预案提醒（如流感高发期呼吸科/儿科预警）

### 5. 报告输出
- 每日流量简报：实际vs预测对比、各科室TOP统计
- 每周分析报告：流量趋势、到诊率变化、异常事件回顾
- 月度运营报告：容量利用率、瓶颈分析、优化建议

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| analysisType | enum | 是 | realtime/predict/report |
| scope | enum | 是 | hospital/department/doctor |
| departmentCode | string | 条件必填 | 科室代码（department级别时） |
| dateRange | object | 条件必填 | 分析日期范围 |
| granularity | enum | 否 | hourly/daily/weekly/monthly |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| flowData | array | 流量数据序列 |
| prediction | object | 预测值（含置信区间） |
| anomalies | array | 异常标记 |
| recommendations | array | 运营建议 |
| kpiSummary | object | 关键指标摘要 |

## 最佳实践
- 预测模型需考虑"日历效应"（春节前后、暑期、开学季等）
- 流感/传染病流行期的儿科、呼吸科流量可能骤增200%以上
- 新政策实施（如新医保目录）可能影响特定科室流量
- 将天气预报纳入预测因子（恶劣天气到诊率通常下降10-20%）

## 约束条件
- T+1预测准确度目标±15%
- 实时监控延迟不超过15分钟
- 分析报告不含个人患者信息（仅聚合统计）
- 预测结果为参考建议，排班决策需人工确认
- 数据保留周期至少2年用于长期趋势分析
