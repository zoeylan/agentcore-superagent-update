# 等候时间预测 (Wait Time Prediction)

## 概述
基于实时队列数据、医生历史接诊时长和动态影响因素，计算并持续更新每位候诊患者的预计等候时间，确保预估偏差控制在±10分钟以内。

## 适用场景
- 患者签到后的初始等候时间预估
- 每5分钟的定期更新计算
- 诊室状态变化（医生暂离/加速/过号）后的即时重算
- 新增加塞患者后的全队列重新计算

## 执行步骤

### 1. 基础数据采集
- 获取当前诊室队列长度（含已签到未叫号的所有患者）
- 查询该医生近30天的平均接诊时长（按初诊/复诊分别统计）
- 获取当前时段的历史平均就诊时长（考虑时段特征）
- 确认当前患者类型分布（初诊vs复诊比例）

### 2. 基础预测计算
- 基础公式：等候时间 = 前方患者数 × 加权平均接诊时长
- 加权因素：
  - 初诊时长权重：复诊时长 × 1.4
  - 上午首诊段（8:00-9:00）：× 1.2（开机、准备时间）
  - 临近午休（11:00-11:30）：× 0.85（医生加速倾向）
  - 下午首段（14:00-14:30）：× 1.1

### 3. 动态调整因子
- 当前进度偏差：如医生当前实际进度比预期慢20%，后续预估×1.2
- 过号因素：队列中标记为"过号待回"的患者不确定性高，加5分钟缓冲
- 加塞因素：急诊转入等优先患者计入前方队列
- 医生暂离：暂离时间直接叠加到所有候诊患者预估中

### 4. 结果输出与更新策略
- 输出预估时间取整到5分钟（避免过度精确的虚假精度感）
- 增加3-5分钟正向缓冲（给患者"提前叫到"的正面体验）
- 每5分钟自动重算并与上次预估比较
- 变化>10分钟时触发主动通知

### 5. 预测质量监控
- 记录每次预测值和实际等候时间
- 每日计算预测偏差分布
- 偏差>15分钟的案例标记为异常并分析原因
- 每周更新模型参数（医生平均时长、调整因子）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roomId | string | 是 | 诊室ID |
| patientPosition | number | 是 | 患者在队列中的位置 |
| doctorId | string | 是 | 当前坐诊医生 |
| currentTime | datetime | 是 | 当前时间 |
| queueComposition | array | 否 | 队列中患者类型分布 |
| doctorStatus | enum | 否 | 医生当前状态(active/paused/delayed) |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| estimatedWaitMinutes | number | 预估等候时间（分钟，取整到5） |
| confidenceRange | object | 置信区间（最短-最长） |
| lastUpdated | datetime | 最后更新时间 |
| significantChange | boolean | 相比上次是否有显著变化 |
| changeDirection | enum | increased/decreased/stable |

## 最佳实践
- 预估值宁高勿低（等候时间"超预期短"比"超预期长"体验好）
- 不同医生的接诊节奏差异可能很大，必须个性化建模
- 高峰时段的不确定性更大，扩大置信区间提示
- 实习生跟诊日接诊时长通常延长30-50%，需纳入考虑

## 约束条件
- 预测偏差目标≤10分钟
- 更新频率：至少每5分钟一次
- 不得向患者展示过于精确的时间（如"37分钟"），统一取整到5分钟
- 预测模型参数至少每周更新一次
- 异常偏差（>20分钟）需记录并分析根因
