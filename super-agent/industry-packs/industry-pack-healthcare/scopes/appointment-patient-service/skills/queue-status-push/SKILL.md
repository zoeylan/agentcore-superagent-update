# 候诊状态推送 (Queue Status Push)

## 概述
实时向候诊患者推送排队进度信息和等候时间更新，确保患者对等候状态有清晰感知，减少焦虑感，同时在异常等候时主动提供解释和安抚。

## 适用场景
- 患者签到后的初始排队信息推送
- 排队进度发生变化时的实时更新
- 预计等候时间变更超过10分钟时的通知
- 异常等候（超预估30%）时的主动解释
- 即将叫号（前方≤3人）时的准备提醒

## 执行步骤

### 1. 签到初始推送
- 患者完成签到后立即推送当前排队位次
- 告知预计等候时间（基于队列长度和医生平均接诊时长）
- 提供诊室位置指引和候诊区设施信息

### 2. 定期状态更新
- 每当前方完成1位患者就诊时更新位次
- 预计等候时间变更>10分钟时主动推送
- 正常推送频率：位次变动时推送，无变动时每15分钟推送一次"进度确认"

### 3. 异常等候处理
- 检测到等候时间超过原始预估30%以上
- 主动推送致歉通知+原因说明（如"前方患者病情较为复杂，预计还需等候约XX分钟"）
- 超过60分钟主动推送并提供选项（继续等候/改约/转其他诊室）

### 4. 即将就诊提醒
- 前方剩余≤3人时推送"请做好就诊准备"
- 提醒患者前往诊室门口等候
- 如患者已离开候诊区（App定位判断），发送返回提醒

### 5. 叫号通知
- 轮到患者时发送叫号通知（含诊室号）
- 叫号后3分钟未应答发送二次提醒
- 5分钟未应答通知将被过号处理

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者ID |
| queuePosition | number | 是 | 当前排队位次 |
| estimatedWait | number | 是 | 预计等候分钟数 |
| previousEstimate | number | 否 | 上次预估时间（用于变化检测） |
| anomalyDetected | boolean | 否 | 是否异常等候 |
| anomalyReason | string | 否 | 异常原因 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| pushId | string | 推送ID |
| messageType | enum | status/anomaly/prepare/callNumber |
| content | string | 推送内容 |
| sentAt | datetime | 发送时间 |
| acknowledged | boolean | 是否已查看 |

## 最佳实践
- 等候时间预估应保留适当缓冲（+5分钟），给患者正面惊喜
- 异常等候解释要诚恳具体，避免"系统繁忙"等模糊说法
- 推送频率不宜过高，避免造成信息疲劳
- 提供"静默模式"选项供不想频繁收到推送的患者使用

## 约束条件
- 不得向患者透露其他患者的个人信息
- 等候时间预估偏差目标≤10分钟
- 异常等候超60分钟必须主动沟通，不可无视
- 叫号通知必须准确，不得出现叫错号情况
- 推送内容中不含诊断或病情相关敏感信息
