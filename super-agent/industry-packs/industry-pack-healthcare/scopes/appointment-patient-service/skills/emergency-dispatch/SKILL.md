# 急诊调度 (Emergency Dispatch)

## 概述
处理急诊患者的快速通道调度，确保I/II级急诊患者绕过常规预约流程直接进入急诊绿色通道，并协调急诊资源（诊室、设备、人员）的即时分配。

## 适用场景
- 智能分诊Agent识别到I/II级急诊患者
- 门诊候诊患者突发病情变化需转急诊
- 院外120急救转入的紧急接诊
- 急诊转普通门诊的后续预约安排

## 执行步骤

### 1. 急诊信号接收与确认
- 接收分诊Agent的急诊触发信号（含CTAS分级和初步症状）
- 确认急诊分级：I级（复苏）/ II级（紧急）/ III级（急症）
- I/II级立即激活绿色通道，III级进入急诊排队

### 2. 急诊资源即时分配
- 查询急诊科当前可用资源（抢救室、诊室、监护设备）
- I级：直接分配抢救室，通知急诊值班主任和护士长
- II级：分配急诊优先诊室，通知当班急诊医生
- III级：加入急诊队列，插入优先于IV/V级患者的位置

### 3. 协同通知
- 通知急诊科做好接诊准备（含患者初步信息和症状摘要）
- 如需专科会诊，同步通知相关专科值班医生
- 通知家属（如有联系方式）患者位置和状态
- 如从门诊转入，通知原候诊队列更新

### 4. 时效监控
- I级：确保收到信号后立即（≤1分钟）进入抢救
- II级：确保15分钟内开始诊疗
- III级：确保30分钟内开始诊疗
- 超时未接诊自动升级预警（通知科主任/医务部）

### 5. 后续衔接
- 急诊处理完成后，评估是否需要住院/留观/转门诊随访
- 转普通门诊：自动生成后续预约（优先号源，插入诊室队列第2位）
- 记录完整急诊就诊轨迹用于质控回顾

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者ID（急诊可临时生成） |
| ctasLevel | enum | 是 | CTAS分级：I/II/III/IV/V |
| chiefComplaint | string | 是 | 主诉 |
| vitalSigns | object | 否 | 生命体征（如已采集） |
| source | enum | 是 | 来源：triage/clinic-transfer/ambulance |
| symptomSummary | string | 否 | 症状摘要 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| emergencyId | string | 急诊就诊号 |
| assignedRoom | string | 分配诊室/抢救室 |
| assignedDoctor | string | 接诊医生 |
| estimatedWait | number | 预计等候时间（分钟） |
| greenChannelActivated | boolean | 绿色通道是否激活 |
| notifications | array | 已发送的通知列表 |

## 最佳实践
- I/II级急诊绝不等待，即使资源紧张也要启动应急预案
- 信息传递追求"够用即可"原则，不因信息采集延误救治
- 急诊转普通门诊时携带完整急诊记录避免重复检查
- 定期回顾急诊响应时效数据持续优化流程

## 约束条件
- I级急诊从识别到抢救开始不得超过1分钟
- II级急诊等候不得超过15分钟
- 急诊调度优先级高于一切常规预约操作
- 必须记录完整的时间节点用于时效审计
- 不得因患者未带证件/未缴费而拒绝急诊接诊
