# 号源管理 (Slot Management)

## 概述
管理全院所有科室和医生的号源池，包括号源的创建、释放、调配和动态优化策略执行。确保号源资源的高效利用和公平分配。

## 适用场景
- 每日排班确认后的号源初始化
- 医生停诊/替诊时的号源调配
- 基于历史数据的号源释放策略动态调整
- 实时号源状态查询和统计

## 执行步骤

### 1. 号源初始化
- 根据排班表生成每日号源池
- 按规则设定各渠道配额比例（线上60%、电话20%、窗口15%、预留5%）
- 设定号源开放时间（专家号T-7、普通号T-3）
- 标记特殊人群预留号源（≤15%）

### 2. 号源释放策略执行
- 热门专家号分批释放：T-7释放60% → T-3释放30% → 当日释放10%
- 普通号源一次性释放
- 根据历史爽约率适当超放（超放比例 = 历史爽约率 × 0.8，上限10%）

### 3. 号源动态调配
- 监控各时段号源使用率
- 低使用率时段（<50%）的号源可调配至高需求时段
- 停诊处理：冻结该医生所有未使用号源 → 触发替诊匹配 → 通知已预约患者
- 加号管理：医生审批后增加号源，标记为"加号"类型

### 4. 号源状态维护
- 实时更新号源状态（可用/已锁定/已预约/已就诊/已释放）
- 锁定超时自动释放（5分钟）
- 爽约号源自动回池（超约定时间15分钟）
- 退号号源即时释放并通知候补队列

### 5. 统计与优化
- 每日统计号源使用率、各渠道转化率、爽约率
- 按周分析号源供需匹配度
- 生成号源调配建议报告

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | enum | 是 | init/release/adjust/freeze/query |
| departmentCode | string | 条件必填 | 科室代码 |
| doctorId | string | 条件必填 | 医生ID |
| date | date | 是 | 操作日期 |
| reason | string | 条件必填 | 调配原因（adjust/freeze时必填） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| slotSummary | object | 号源汇总（总数/已用/可用/锁定） |
| adjustmentLog | array | 调配操作记录 |
| utilizationRate | number | 号源使用率 |
| recommendations | array | 优化建议 |

## 最佳实践
- 号源超放比例应基于近30天滑动窗口的爽约率动态计算
- 停诊通知应在号源冻结后30分钟内完成所有患者通知
- 替诊优先匹配同科室、同级别、相似专长的医生
- 长假前后的号源需求波动大，需提前做好弹性准备

## 约束条件
- 超放比例上限10%，避免过度预约导致现场拥挤
- 特殊人群预留号源上限15%
- 号源调配需记录明确原因和审批人
- 不得在医生未确认的情况下自动安排加号
