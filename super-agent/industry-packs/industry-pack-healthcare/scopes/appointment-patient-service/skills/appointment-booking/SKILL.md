# 预约挂号受理 (Appointment Booking)

## 概述
处理来自多渠道（App/小程序、电话、现场窗口、转诊）的患者预约挂号请求，完成身份验证、资格检查、号源锁定和预约确认的完整流程。

## 适用场景
- 患者通过任何渠道发起预约挂号请求
- 转诊患者需要预约接诊医院号源
- 候补队列中的患者获得递补机会时的自动预约

## 执行步骤

### 1. 接收预约请求（响应时间≤3秒）
- 解析请求来源渠道和患者基本信息
- 提取预约目标：科室、医生（可选）、日期、时段偏好
- 记录请求时间戳用于排队公平性保证

### 2. 患者身份验证
- 核验患者身份证号/医保卡号/就诊卡号
- 查询是否为本院已建档患者，未建档则触发建档流程
- 验证医保信息有效性（如适用）

### 3. 预约资格检查
- 查询患者爽约记录（当季度累计次数）
- 检查是否存在同日同科室重复预约
- 验证特殊号源资格（如专家号是否被爽约限制）
- 特殊人群标识确认（老年人、残障、军人、孕妇）

### 4. 号源匹配与锁定
- 查询目标号源可用性
- 号源充足：锁定号源（锁定时效5分钟）
- 号源不足：生成替代建议（同科室其他医生/相邻日期/候补队列）
- 特殊人群执行优先通道分配（不超过号源总量15%）

### 5. 预约确认
- 生成预约确认单（患者、科室、医生、时间、诊室、费用）
- 触发通知提醒Agent发送确认消息
- 更新号源池状态
- 记录预约审计日志

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者唯一标识 |
| channel | enum | 是 | 预约渠道：app/phone/window/referral |
| departmentCode | string | 是 | 目标科室代码 |
| doctorId | string | 否 | 指定医生ID |
| preferredDate | date | 是 | 期望就诊日期 |
| preferredTimeSlot | string | 否 | 期望时段（上午/下午/具体时间） |
| triageResult | object | 否 | 分诊Agent输出的推荐结果 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| bookingStatus | enum | success/waitlisted/rejected |
| confirmationId | string | 预约确认号 |
| appointmentDetail | object | 预约详情（科室、医生、时间、诊室） |
| alternativeSuggestions | array | 替代建议（号源不足时） |
| rejectionReason | string | 拒绝原因（被限制时） |

## 最佳实践
- 号源锁定后5分钟内未确认自动释放，避免资源浪费
- 高峰时段（8:00-9:00）预约请求量大，需做好并发控制
- 转诊预约给予优先级加权，但仍需遵循号源限额
- 预约失败时必须提供至少2个替代方案

## 约束条件
- 不得超额预约（加号需医生审批）
- 不得绕过爽约管控规则
- 号源冲突率必须控制在≤0.1%
- 所有预约操作必须记录完整审计日志
