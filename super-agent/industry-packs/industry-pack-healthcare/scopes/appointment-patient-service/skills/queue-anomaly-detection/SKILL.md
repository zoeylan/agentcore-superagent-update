# 候诊异常检测 (Queue Anomaly Detection)

## 概述
实时监测候诊流程中的异常情况，包括等候时间异常延长、医生进度异常、过号率升高等，及时预警并触发应对措施，保障候诊秩序和患者体验。

## 适用场景
- 实际等候时间显著超过预估的异常检测
- 医生诊间长时间无患者进出的停滞检测
- 过号率突然升高的趋势异常检测
- 候诊区人数超过承载能力的拥挤预警

## 执行步骤

### 1. 异常指标定义与监控
- 等候超时异常：实际等候超过预估30%以上
- 诊室停滞异常：诊室状态为"就诊中"超过正常时长2倍（如单诊超过30分钟）
- 过号异常：单时段过号率>20%
- 容量异常：候诊区实际人数超过座位数150%
- 队列增长异常：队列净增长速度>消化速度的持续时间>30分钟

### 2. 实时检测与预警生成
- 每分钟执行一次异常扫描
- 达到异常阈值时生成预警事件
- 预警分级：
  - 黄色预警：轻度异常，需关注（如等候超时5-15分钟）
  - 橙色预警：中度异常，需干预（如等候超时>15分钟或诊室停滞）
  - 红色预警：严重异常，需紧急处理（如医生长时间未响应、容量超限）

### 3. 根因分析
- 等候超时：分析是前方患者就诊时间长还是医生进度慢
- 诊室停滞：确认是复杂病例还是医生暂离未更新状态
- 过号升高：分析是通知不到位还是患者普遍迟到
- 容量超限：分析是到诊率异常升高还是消化速度变慢

### 4. 应对措施触发
- 黄色：通知候诊优化Agent关注，更新患者等候时间预估
- 橙色：通知诊室导诊护士确认情况，主动安抚等候患者
- 红色：通知科室负责人，考虑启动备用诊室或安排分流
- 所有异常记录并纳入日终报告

### 5. 恢复确认
- 异常消除后确认恢复正常状态
- 计算异常持续时间和影响范围（受影响患者数）
- 生成异常事件完整报告（触发-响应-处理-恢复）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roomId | string | 是 | 监控诊室ID |
| currentQueue | array | 是 | 当前队列状态 |
| doctorProgress | object | 是 | 医生当前进度信息 |
| capacityLimit | number | 否 | 候诊区容量上限 |
| historicalBaseline | object | 否 | 历史基线数据 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| anomalyDetected | boolean | 是否检测到异常 |
| anomalyType | enum | timeout/stall/noshow_spike/overcrowded |
| severity | enum | yellow/orange/red |
| rootCause | string | 初步根因分析 |
| recommendedAction | string | 建议应对措施 |
| affectedPatients | number | 受影响患者数 |

## 最佳实践
- 异常阈值应基于历史数据动态调整，避免固定阈值导致的频繁虚警
- 高峰时段的阈值适当放宽（允许更长等候）
- 异常检测应区分"偶发"和"趋势"，连续出现的轻度异常也需关注
- 应对措施应预设自动化流程，减少人工决策延迟

## 约束条件
- 异常检测延迟不超过1分钟
- 红色预警必须在5分钟内获得人工确认响应
- 异常记录保留90天用于趋势分析
- 虚警率目标<10%（避免"狼来了"效应）
- 不得因异常处理而跳过已签到患者的就诊顺序
