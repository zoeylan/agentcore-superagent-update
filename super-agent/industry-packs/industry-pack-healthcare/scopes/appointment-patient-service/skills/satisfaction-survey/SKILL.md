# 满意度回访 (Satisfaction Survey)

## 概述
在患者就诊完成后48小时内触发满意度回访调查，通过多维度评分量表收集患者体验反馈，对低满意度反馈触发人工跟进，并聚合分析数据生成服务改进报告。

## 适用场景
- 门诊就诊完成后的常规满意度回访
- 急诊就诊后的服务体验回访
- 投诉处理完成后的二次回访（确认问题已解决）
- 特定科室/医生的专项满意度调查

## 执行步骤

### 1. 回访触发
- 就诊完成后48小时自动触发（不在晚22:00-早8:00之间发送）
- 确认就诊已实际完成（有结算记录或医生完诊标记）
- 排除不适合回访的情况（住院转入、术后当日、丧亲等敏感情境）
- 选择最佳回访渠道（App内问卷优先，备选微信/短信链接）

### 2. 问卷推送
- 推送5维度评分量表（每项1-5分）：
  1. 预约便捷性：预约渠道体验、号源获取难度
  2. 等候体验：等候时间合理性、候诊环境、信息告知
  3. 医生服务：问诊充分性、解释清晰度、态度尊重度
  4. 环境设施：就诊环境、标识引导、设施便利度
  5. 总体满意度：整体就医体验、是否愿意推荐
- 附加开放性文字反馈框（选填）
- 填写时间控制在2分钟以内

### 3. 数据收集与分析
- 记录评分结果和文字反馈
- 自动分类文字反馈情感倾向（正面/中性/负面）
- 关联就诊流程数据（实际等候时间、所就诊医生等）
- 计算各维度平均分和NPS（净推荐值）

### 4. 低分预警与工单生成
- 总体满意度≤3分触发低分预警
- 自动生成投诉工单，包含：
  - 患者就诊流程回溯（等候时长、是否有异常等候等）
  - 评分明细和文字反馈
  - 患者联系方式和偏好沟通时间
- 工单24小时内必须由人工客服跟进处理

### 5. 报告生成
- 每周生成科室/医生维度的满意度报告
- 按时段分析满意度波动（高峰期vs非高峰期）
- 识别系统性问题（如某科室持续低分）
- 输出改进建议和对比趋势图

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者ID |
| appointmentId | string | 是 | 就诊预约ID |
| visitCompletedAt | datetime | 是 | 就诊完成时间 |
| doctorId | string | 是 | 接诊医生ID |
| departmentCode | string | 是 | 就诊科室 |
| actualWaitTime | number | 否 | 实际等候时间（分钟） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| surveyId | string | 回访问卷ID |
| scores | object | 5维度评分 |
| overallScore | number | 总体满意度 |
| textFeedback | string | 文字反馈 |
| sentiment | enum | positive/neutral/negative |
| ticketGenerated | boolean | 是否生成投诉工单 |
| ticketId | string | 工单ID（如有） |

## 最佳实践
- 回访问卷简短精炼，控制在2分钟内完成，提高完成率
- 低分回访的人工跟进应先倾听后解决，不急于辩解
- 高满意度患者可邀请成为"服务体验官"参与改进
- 回访数据与实际服务数据（等候时间等）交叉验证

## 约束条件
- 回访覆盖率目标≥80%
- 回访完成率目标≥40%
- 低分（≤3分）24小时内必须人工跟进
- 不在敏感时间段发送回访（丧亲、重症诊断后当日）
- 回访数据用于服务改进，不得用于对患者的差异化服务
- 每位患者每次就诊最多收到1次回访请求
