# 急诊识别 (Emergency Detection)

## 概述
在分诊过程中实时监测和识别急危重症信号（Red Flags），确保CTAS I/II级急诊患者在最短时间内被识别并触发绿色通道。急诊识别敏感度目标≥95%。

## 适用场景
- 症状采集过程中的实时危险信号监测
- 候诊患者突发病情变化的即时识别
- 门诊诊间患者病情恶化的快速响应
- 家属或护士主动报告的紧急情况处理

## 执行步骤

### 1. Red Flags实时扫描
- 在症状采集的每一步实时匹配危险信号模式
- 危险信号匹配不等待信息采集完成，即触即响
- Red Flags清单（不限于）：
  - 心血管：胸痛伴出汗/放射痛、突发心悸伴意识模糊
  - 神经系统：突发剧烈头痛("一生中最剧烈")、单侧肢体无力、言语不清、意识障碍
  - 呼吸系统：严重呼吸困难、端坐呼吸、咯血
  - 消化系统：腹痛伴板状腹/肌紧张、大量呕血/便血
  - 其他：过敏性休克征象、严重创伤、高热伴皮疹+颈项强直（儿童脑膜炎）

### 2. 生命体征异常检测
- 体温：≥40°C或≤35°C
- 血压：收缩压≥180或≤90mmHg
- 心率：≥150或≤40次/分
- 呼吸：≥30或≤8次/分
- 血氧：SpO2≤90%
- 任一指标达到阈值即触发急诊评估

### 3. 急诊等级判定
- 基于症状+体征综合判定CTAS等级
- I级判定标准：需要立即抢救干预维持生命
- II级判定标准：潜在危及生命或肢体，需紧急干预
- 判定后即时触发急诊调度流程

### 4. 绿色通道激活
- I/II级确认后≤30秒内发送急诊调度指令
- 通知内容：患者位置、初步症状、CTAS等级、可能需要的资源
- 如患者在候诊区，同时通知现场导诊护士即刻协助
- 绕过所有常规排队和预约流程

### 5. 误判兜底机制
- 对II/III级边界案例标注"建议急诊护士现场评估"
- 记录触发依据支持事后复审
- 阈值设定宁紧勿松（优先保证敏感度，可容忍一定虚警率）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| symptomStream | stream | 是 | 实时症状采集数据流 |
| vitalSigns | object | 否 | 生命体征（如已采集） |
| patientContext | object | 否 | 患者背景（年龄、基础疾病） |
| location | string | 否 | 患者当前位置（候诊区/诊室/其他） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| emergencyDetected | boolean | 是否检测到急诊信号 |
| ctasLevel | enum | CTAS等级（仅急诊时输出） |
| triggeredRedFlags | array | 触发的危险信号列表 |
| confidence | number | 判定置信度 |
| actionRequired | enum | immediate/urgent/observe/none |
| dispatchTriggered | boolean | 是否已触发急诊调度 |

## 最佳实践
- 急诊识别采用"并行扫描"机制，不阻塞正常症状采集流程
- 对老年人、免疫抑制患者的症状表现可能不典型，降低触发阈值
- 儿童vital signs阈值需按年龄调整（儿科标准）
- 定期用真实急诊案例回测，确保敏感度≥95%

## 约束条件
- 敏感度≥95%是硬性要求，特异性目标≥80%
- I级急诊从识别到触发调度≤30秒
- 宁可过度触发（虚警），不可漏诊（漏报）
- 不得因信息不完整而延迟急诊判断——信息不足时按高优先级处理
- 所有急诊识别事件必须记录完整日志用于质量审计
