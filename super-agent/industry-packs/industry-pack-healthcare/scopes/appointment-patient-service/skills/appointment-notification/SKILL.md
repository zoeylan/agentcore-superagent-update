# 预约通知 (Appointment Notification)

## 概述
管理预约全生命周期的标准化通知消息，包括预约确认、就诊提醒、停诊通知、改约确认等，确保患者在正确的时间收到准确、有用的信息引导。

## 适用场景
- 预约成功后的确认通知
- 就诊前24小时的首次提醒
- 就诊前2小时的二次提醒
- 医生停诊时的紧急通知
- 预约变更（改约/退号）的确认通知

## 执行步骤

### 1. 通知触发判定
- 接收触发事件（预约确认/定时提醒/停诊事件/变更事件）
- 确认通知类型和优先级
- 检查勿扰时段（晚22:00-早8:00不发送非紧急通知）
- 防重复检查（同一事件不重复通知）

### 2. 渠道选择
- 根据患者偏好设置选择主渠道
- 渠道优先级：App推送 > 微信服务号 > 短信 > 电话
- 紧急通知（停诊）：多渠道并发，确保触达
- 老年患者优先使用电话或短信渠道

### 3. 内容组装
- 预约确认：患者姓名、就诊时间、科室、医生、诊室位置、预计费用、注意事项
- 24h提醒：就诊时间、交通指引、携带材料清单、特殊准备要求（空腹/停药等）
- 2h提醒：实时候诊人数、预计等候时间、签到方式指引
- 停诊通知：停诊原因、一键改约链接、一键退号链接、替代方案

### 4. 消息发送与确认
- 执行消息发送
- 记录发送状态（成功/失败/未读）
- 失败后的重试机制：间隔5分钟，最多重试3次
- 多次失败后切换备用渠道

### 5. 送达跟踪
- 跟踪消息已读状态（App/微信可追踪）
- 重要通知（停诊）未确认时在30分钟后升级为电话通知
- 记录完整的消息送达轨迹用于审计

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| notificationType | enum | 是 | confirm/remind24h/remind2h/cancel/change |
| patientId | string | 是 | 患者ID |
| appointmentDetail | object | 是 | 预约详情 |
| urgency | enum | 否 | normal/urgent |
| additionalInfo | object | 否 | 附加信息（停诊原因、替代方案等） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| notificationId | string | 通知ID |
| sentChannel | string | 实际发送渠道 |
| sentStatus | enum | success/failed/pending |
| deliveryConfirmed | boolean | 是否确认送达 |
| retryCount | number | 重试次数 |

## 最佳实践
- 短信内容控制在70字以内避免分条计费
- 通知语言亲切专业，避免医学术语堆砌
- 停诊通知务必在30分钟内触达所有已预约患者
- 提醒消息中包含明确的CTA（行动指引）

## 约束条件
- 不在晚22:00至早8:00发送非紧急通知
- 同一事项最多提醒3次，避免骚扰
- 不在通知中暴露敏感诊断信息
- 遵守《个人信息保护法》相关要求
- 所有通知记录保留1年用于服务追溯
