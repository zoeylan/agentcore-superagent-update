# 症状采集 (Symptom Collection)

## 概述
通过结构化问卷和智能自然语言交互，系统化采集患者主诉、伴随症状、既往史等临床信息，为分诊决策和后续诊疗提供高质量的结构化数据输入。

## 适用场景
- 患者预约成功后的诊前信息预采集
- 患者到院候诊时的补充信息采集
- 急诊场景的快速症状摘要采集
- 分诊决策前的必要信息收集

## 执行步骤

### 1. 启动采集流程
- 根据预约科室选择对应的症状采集模板
- 确认患者身份并获取历史就诊记录
- 选择采集模式：标准模式（5-8分钟）/ 快速模式（2-3分钟，急诊用）

### 2. 主诉采集（必填）
- 引导患者用自己的语言描述主要不适
- 自然语言理解提取关键信息：症状类型、部位、性质
- 追问标准化要素：发作时间、持续时间、频率、程度（0-10评分）

### 3. 伴随症状采集
- 基于主诉关联的常见伴随症状生成针对性问卷
- 采用是/否快速筛查 + 阳性项深入追问模式
- 覆盖系统回顾中与主诉相关的重要鉴别症状

### 4. 病史信息采集
- 既往病史：慢性病、手术史、外伤史
- 过敏史：药物过敏（必须精确到药名和反应类型）、食物过敏
- 用药情况：当前使用的处方药和非处方药
- 家族史：直系亲属重大疾病
- 个人史：吸烟、饮酒、职业暴露（与主诉相关时追问）

### 5. 信息结构化输出
- 将自然语言描述转化为标准化医学术语
- 标注关键风险信息（红色标记：过敏/传染病，黄色标记：重大病史）
- 生成症状摘要供分诊Agent和接诊医生使用
- 存储原始记录（患者原话）+ 结构化数据双版本

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientId | string | 是 | 患者ID |
| appointmentId | string | 否 | 关联预约ID |
| collectionMode | enum | 是 | standard/quick |
| targetDepartment | string | 否 | 预约科室（用于模板选择） |
| existingRecords | array | 否 | 历史就诊记录引用 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| chiefComplaint | object | 结构化主诉（症状、部位、时间、程度） |
| associatedSymptoms | array | 伴随症状列表 |
| medicalHistory | object | 既往史、手术史、家族史 |
| allergies | array | 过敏信息（药物/食物，反应类型） |
| currentMedications | array | 当前用药列表 |
| riskFlags | array | 风险标识（等级+内容） |
| symptomSummary | string | 症状摘要（供医生快速浏览） |
| completeness | number | 采集完整度评分（0-100） |

## 最佳实践
- 问卷设计兼顾完整性和患者耐受度，核心问题控制在15个以内
- 对老年患者使用更大字体、更简单的语言、支持语音输入
- 过敏信息采集必须确认到具体药物名称，不接受"某种抗生素"等模糊描述
- 采集到危险信号时立即中断常规流程触发急诊评估

## 约束条件
- 主诉为必填项，不得跳过
- 过敏史确认环节不得省略（即使患者声称"无过敏"也需确认）
- 不得对症状进行诊断性解读（仅记录，不诊断）
- 患者拒绝回答的项目标记为"拒答"而非空白
- 采集数据加密存储，遵循医疗数据隐私保护要求
