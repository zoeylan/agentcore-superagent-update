# 变异报告生成 (Variant Report Generation)

## 概述
将变异解读分析结果整合为符合临床标准的基因检测报告，包含变异分类、临床意义解读、遗传咨询建议，确保报告在规定时限内生成并提交专家审核。

## 适用场景
- 临床全外显子/全基因组测序结果报告
- 单基因病Panel检测结果报告
- 产前诊断基因检测报告（紧急时限）
- 肿瘤靶向用药相关基因检测报告

## 执行步骤

### 1. 报告数据汇总
- 收集变异解读Agent的分类结果和证据链
- 整合罕见病诊断Agent的候选诊断信息（如适用）
- 收集药物基因组Agent的用药建议（如适用）
- 确认患者基本信息和送检信息

### 2. 报告结构组装
- 报告头：患者信息、送检单位、检测项目、样本信息
- 检测结果摘要：阳性/阴性结论、关键发现
- 变异详情表：每个报告变异的详细信息
- 临床意义解读：变异与疾病的关联说明
- 局限性声明：检测方法的固有限制
- 遗传咨询建议：后续建议（家系验证/定期复查等）

### 3. 质量审核检查
- 自动核查报告完整性（必填字段100%覆盖）
- 验证变异命名规范性（HGVS标准）
- 检查分类结论与证据链一致性
- 确认敏感信息脱敏处理

### 4. 时限管控
- 紧急报告（产前/肿瘤）：7个工作日时限
- 常规报告：21个工作日时限
- 超时前3个工作日自动预警
- 记录报告各阶段时间节点

### 5. 提交专家审核
- 致病/可能致病变异报告必须提交专家审核
- 生成审核摘要便于专家快速审阅
- 记录审核意见和修改痕迹
- 审核通过后生成最终版本并签发

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| variantClassifications | array | 是 | 变异分类结果列表 |
| patientInfo | object | 是 | 患者基本信息 |
| testInfo | object | 是 | 检测项目和方法信息 |
| clinicalIndication | string | 是 | 送检临床指征 |
| urgency | enum | 是 | urgent/routine |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| reportDocument | object | 完整报告文档 |
| reportStatus | enum | draft/pending_review/approved/issued |
| timelineCompliance | boolean | 是否在时限内 |
| reviewerAssignment | string | 指派的审核专家 |

## 最佳实践
- 报告语言需兼顾专业性和可读性
- 为非遗传学专科医生提供通俗化解读
- 阳性结果报告附带遗传咨询转介建议
- 保留报告版本历史便于追溯

## 约束条件
- 未经专家审核签发的报告不得发出
- 患者隐私信息严格脱敏
- 报告中不得包含确定性诊断结论（仅提供分类和解读）
- 紧急报告超时需立即升级并记录原因
