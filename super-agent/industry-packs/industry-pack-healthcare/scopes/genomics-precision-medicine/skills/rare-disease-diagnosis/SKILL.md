# 罕见病辅助诊断 (Rare Disease Diagnosis)

## 概述
整合临床表型（HPO术语）、基因组数据和家系信息，通过多维度数据分析辅助罕见病诊断，缩短平均确诊时间，输出候选诊断列表和验证建议。

## 适用场景
- 疑似遗传性疾病患者的首次诊断评估
- 多年未确诊的疑难病例重新评估
- 新生儿快速全基因组检测（rWGS）结果解读
- 产前诊断中异常表型的遗传学评估

## 执行步骤

### 1. 临床表型标准化采集
- 将临床描述转化为HPO标准术语
- 区分核心表型（主要就诊原因）和伴随表型
- 记录表型出现时间线（先天性/后天获得性）
- 标注阴性表型（排除性诊断价值）

### 2. 鉴别诊断候选生成
- 基于HPO表型组合查询OMIM/Orphanet
- 计算每个候选疾病的表型匹配度
- 考虑疾病患病率先验概率（贝叶斯框架）
- 输出Top10候选疾病列表

### 3. 基因组数据整合
- 接收变异解读Agent的分类结果
- 验证候选疾病对应基因中是否有P/LP/VUS变异
- 表型驱动（phenotype-first）与基因驱动（genotype-first）交叉验证
- 重新排序：有基因证据支持的候选优先

### 4. 家系信息整合
- 验证遗传模式与家系结构的一致性
- De novo变异在无家族史场景的权重调整
- 近亲婚配家系的纯合变异优先评估
- 不完全外显率考量

### 5. 诊断建议输出
- 输出最终候选诊断排序（标注证据强度）
- 推荐验证方案（Sanger验证/功能实验/家系扩展）
- 提供鉴别诊断至少3个
- 列出下一步建议（多学科会诊/额外检查/随访计划）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| patientPhenotypes | array | 是 | HPO标准术语列表 |
| genomicData | object | 是 | 基因组变异及ACMG分类 |
| familyPedigree | object | 否 | 家系图谱信息 |
| patientDemographics | object | 是 | 年龄、性别、种族 |
| previousTests | array | 否 | 已完成的检测结果 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| candidateDiagnoses | array | 排序候选诊断列表 |
| evidenceStrength | enum | 确定/可能/待验证 |
| differentialDiagnoses | array | 鉴别诊断≥3个 |
| validationPlan | array | 建议验证方案 |
| nextSteps | array | 下一步行动建议 |

## 最佳实践
- 始终考虑表型扩展性（患者未来可能出现新表型）
- 阴性结果也有诊断价值，应记录已排除的疾病
- 对于新生儿rWGS，优先考虑可治疗性疾病
- 定期重评估（每6-12个月，基因组数据库持续更新）

## 约束条件
- 诊断建议仅作为临床医生参考，不替代专家判断
- VUS不应作为确诊依据
- 产前诊断场景需特别谨慎，避免过度解读
- 诊断不确定时必须如实告知，不可为给出结论降低标准
