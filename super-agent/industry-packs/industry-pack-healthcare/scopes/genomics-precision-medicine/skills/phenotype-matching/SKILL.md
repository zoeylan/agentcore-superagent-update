# 表型匹配分析 (Phenotype Matching)

## 概述
将临床描述的症状和体征标准化为HPO（Human Phenotype Ontology）术语，并基于语义相似度与已知疾病表型谱进行匹配分析，为罕见病诊断提供表型层面的候选疾病排序。

## 适用场景
- 临床症状描述的HPO标准化转换
- 基于表型组合的疾病候选筛选
- 表型重叠疾病的鉴别诊断辅助
- 随访中新增表型后的诊断重评估

## 执行步骤

### 1. 临床描述HPO转换
- 接收临床医生的自然语言描述
- NLP解析提取关键症状和体征
- 映射到最精确的HPO术语（优先使用叶节点术语）
- 处理同义词和近义表达（如"智力低下"→HP:0001249）

### 2. 表型集质量评估
- 评估表型覆盖度（是否足够进行有效匹配）
- 识别高信息量表型（IC值高，诊断价值大）
- 标注不确定表型（需临床确认）
- 建议补充采集的表型信息

### 3. 语义相似度计算
- 基于HPO本体结构计算IC（Information Content）
- 使用Resnik/Lin/Jiang相似度算法
- 对患者表型集与每个候选疾病的完整表型谱进行比较
- 考虑表型频率信息（某疾病中该表型出现的频率）

### 4. 候选疾病排序
- 综合相似度得分生成候选疾病排名
- 考虑疾病流行率先验（罕见病vs极罕见病）
- 标注完美匹配和部分匹配
- 标识患者具有但疾病不典型的表型（可能提示其他共患病）

### 5. 结果输出与建议
- 输出Top10候选疾病及匹配细节
- 标注每个候选的匹配表型和未匹配表型
- 建议补充检查以区分Top候选
- 提供表型-基因组数据交叉验证的整合建议

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| clinicalDescription | string | 否 | 自然语言临床描述 |
| hpoTerms | array | 否 | 已标准化的HPO术语 |
| patientAge | number | 是 | 患者年龄 |
| onsetAge | string | 否 | 发病年龄 |
| negativeFindings | array | 否 | 阴性表型（已排除） |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| standardizedPhenotypes | array | 标准化后的HPO术语列表 |
| candidateDiseases | array | Top10候选疾病排序 |
| similarityScores | object | 详细相似度分数 |
| unmatchedPhenotypes | array | 未被任何候选解释的表型 |
| supplementaryPhenotypes | array | 建议补充采集的表型 |

## 最佳实践
- 优先使用具体HPO术语而非笼统上层术语
- 阴性表型对鉴别诊断有重要价值
- 年龄依赖性表型需动态评估（儿童可能尚未出现）
- 与基因组分析结合使用效果最佳

## 约束条件
- 表型匹配仅提供候选列表，不等同于诊断
- 数据库更新滞后可能遗漏新近描述的疾病
- NLP转换不确定时需人工确认
- 极罕见表型组合可能无法匹配任何已知疾病
