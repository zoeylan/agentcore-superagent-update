## 📋 业务范围

### 1. 预约与患者服务

> 覆盖患者从预约挂号到就诊完成后回访的全流程服务管理。

包括多渠道预约受理（线上App/小程序、电话、现场窗口、转诊）、智能预检分诊、号源调度与候诊管理、就诊提醒通知、满意度回访等环节。核心目标是缩短患者等候时间、提高分诊准确率（≥85%）、降低爽约率、提升患者满意度。

**Agent 团队：**

| Agent | 职责 |
|-------|------|
| 预约调度Agent | 管理号源池、处理多渠道预约、执行爽约管控和号源动态调配 |
| 智能分诊Agent | 基于症状采集进行科室推荐和CTAS五级急诊分诊评估 |
| 通知提醒Agent | 管理就诊提醒、候诊通知、满意度回访等全程消息服务 |
| 候诊优化Agent | 实时监控诊室候诊状态，动态优化就诊顺序和等候体验 |

**核心技能（16项）：** 预约受理、通知提醒、科室推荐、急诊识别、急诊调度、多诊室分流、爽约预警、爽约管控、患者流量分析、排队异常检测、候诊状态推送、满意度调查、号源管理、症状采集、分诊评估、等候时间预测

---

### 2. 诊疗辅助

> 为临床医生提供全流程诊疗辅助支持，AI作为医生的智能助手提供循证医学参考。

涵盖病史采集结构化、检查检验智能推荐、鉴别诊断辅助提示、用药方案审核、临床路径推荐与偏差管理。所有建议标注证据等级（A/B/C/D），最终决策权始终在医生手中。

**Agent 团队：**

| Agent | 职责 |
|-------|------|
| 病史采集Agent | 辅助医生进行结构化病史信息采集和预整理，标注关键风险信息 |
| 诊断辅助Agent | 基于临床信息生成鉴别诊断列表，推荐检查检验项目，处理危急值通知 |
| 用药审核Agent | 处方多维度实时审核（过敏、DDI、剂量、禁忌症），抗菌药物分级管理 |
| 临床路径Agent | 临床路径推荐、执行监控、偏差管理和DRG/DIP费用预警 |

**核心技能（15项）：** 抗菌药物管理、危急值通知、鉴别诊断、剂量校验、DRG费用管控、药物相互作用检测、证据等级标注、路径匹配、路径监控、患者摘要生成、处方审核、风险识别、结构化病史采集、检查推荐、偏差管理

---

### 3. 病历与医疗质控

> 覆盖电子病历全生命周期管理和医疗质量持续改进体系。

包括病历书写质量实时监控与辅助、时限合规性管理、病历等级评定、归档管理、医疗质量核心指标监测、不良事件识别与上报管理。遵循《电子病历应用管理规范》，推动PDCA持续改进。

**Agent 团队：**

| Agent | 职责 |
|-------|------|
| 病历质控Agent | 实时监控病历书写质量，时限/完整性/一致性/规范性检查，自动预评级 |
| 质量监测Agent | 医疗质量核心指标实时监测、分析预警，生成质量分析报告和PDCA建议 |
| 不良事件Agent | 不良事件识别、分级上报、RCA根因分析和整改措施跟踪 |

**核心技能（12项）：** 不良事件检测、整改措施跟踪、事件分类上报、病历时限检查、PDCA改进规划、质量指标监测、质量报告生成、质量趋势分析、病历完整性审查、病历一致性校验、病历等级评定、根因分析

---

### 4. 基因组与精准医疗

> 基因组变异解释、基因优先级排序、药物再利用分析、罕见病辅助诊断。

通过AI分析基因测序数据，识别致病变异位点，结合临床表型进行变异致病性评估（ACMG分类）。支持药物基因组学指导个体化用药，以及基于基因网络分析的药物再利用（drug repurposing）候选筛选。基因变异分类严格遵循ACMG/AMP指南，致病性变异报告必须由分子遗传学专家审核签发。

**Agent 团队：**

| Agent | 职责 |
|-------|------|
| 变异解读Agent | 基因组变异致病性评估与ACMG/AMP标准化分类，生成变异解读报告 |
| 罕见病诊断Agent | 整合临床表型（HPO术语）、基因组数据和家系信息，进行候选基因优先级排序和罕见病辅助诊断 |
| 药物基因组Agent | 基于药物基因组学证据提供个体化用药指导（PharmGKB/CPIC），支持药物再利用候选筛选 |

**核心技能（9项）：** 药物再利用分析、家系共分离分析、基因优先级排序、分子对接分析、药物基因组学指导、表型匹配、罕见病诊断、变异解读、变异报告生成

---

### 5. 医疗财务与收入优化

> DRG/DIP支付管理、医疗收入优化、费用合规审核、医保结算智能化。

在DRG/DIP支付改革背景下，帮助医院优化病案编码、控制费用结构、提升CMI值、减少医保拒付。支持费用预警、超标分析、编码质量审核和收入漏洞识别。DRG入组准确率目标≥95%，医保拒付率目标<3%。

**Agent 团队：**

| Agent | 职责 |
|-------|------|
| DRG编码专员Agent | 病案首页编码质量审核、DRG/DIP入组预测与优化、CMI值管理 |
| 收入优化Agent | 费用超标预警、收入漏洞识别（漏收费检测）、月度科室收入分析报告 |
| 合规审计Agent | 医保结算前合规性预审、拒付原因分析与预防、医保政策合规管理 |

**核心技能（9项）：** 拒付原因分析、CMI追踪、编码质量审核、费用超标预警、科室收入报告、DRG病例分组、费用结构优化、医保结算、收入漏洞检测

## ⚙️ 定制化建议

以下内容建议根据医院实际情况进行调整：

### 业务规则调整

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 专家号开放天数 | 7天 | 根据医院号源管理策略调整 |
| 普通门诊开放天数 | 3天 | 根据实际需求调整 |
| 爽约限制阈值 | 3次/季度 | 可根据医院患者群体特征调整 |
| 分诊准确率目标 | ≥85% | 初期可适当降低，逐步提升 |
| 危急值通知时限 | 30分钟 | 法规要求，不建议调整 |
| 病历时限（入院记录） | 8小时 | 法规要求，不建议调整 |
| 抗菌药物使用率（门诊） | ≤20% | 国家标准，可结合医院实际设定阶段目标 |
| 基因检测报告时限（紧急） | 7个工作日 | 产前诊断/肿瘤靶向用药紧急场景 |
| 基因检测报告时限（常规） | 21个工作日 | 常规遗传病基因检测 |
| DRG入组准确率目标 | ≥95% | 可根据医院编码团队成熟度调整阶段目标 |
| 费用超标预警阈值 | DRG标准的80% | 可分科室差异化调整 |
| 医保拒付率目标 | <3% | 超过3%的科室纳入重点监控 |

### 系统集成

- **HIS 对接**：号源管理、挂号确认、处方开具等需与 HIS 系统对接
- **EMR 对接**：病历质控需接入电子病历系统获取病历内容
- **LIS/PACS 对接**：检查检验结果回传用于诊断辅助和危急值通知
- **消息通道**：配置短信、微信、App推送等通知渠道
- **药品知识库**：对接院内药品字典和药物相互作用数据库
- **基因组分析平台**：对接NGS生信分析流水线、ClinVar/gnomAD/PharmGKB数据库
- **医保结算系统**：对接DRG/DIP分组器、医保费用清算接口
- **病案管理系统**：对接病案首页、ICD编码字典和DRG权重表

### 知识库配置

- 导入本院的临床路径方案库（建议覆盖400+病种）
- 配置本院药品目录和抗菌药物分级管理名录
- 导入分诊症状-科室映射规则（可基于本院数据优化）
- 配置医疗质量指标阈值（对标同级同类医院）
- 导入本院开展的基因检测Panel列表和变异注释数据库版本
- 配置PharmGKB/CPIC药物基因组学指南版本和本院适用的检测基因范围
- 导入本地区DRG/DIP分组方案和权重表（国家/省级版本）
- 配置医保目录（甲/乙/丙类）和限价药品耗材清单

### 权限与合规

- 配置各级医生的抗菌药物处方权限（非限制/限制/特殊使用级）
- 设置病历查阅权限（遵循最小化原则）
- 配置不良事件上报的匿名化规则
- 确保所有 AI 辅助建议界面显示"仅供参考，不构成医疗决策"声明
- 基因数据访问需双重授权认证并记录审计日志，严格遵守《人类遗传资源管理条例》
- 医保费用相关操作需配置分级审批权限，防止编码欺诈风险
