# 广告投放与智能优化 — 标准操作流程 (SOP)

## 一、流程概述

本SOP定义了跨平台广告智能投放管理的全流程标准，涵盖从投放计划制定、预算分配、受众定向、实时监控到异常处理的完整闭环。目标是通过AI Agent协同实现投放效率最大化，ROAS目标>3:1，CPM/CPA降低≥15%，异常预警响应≤5分钟。

## 三、标准流程步骤

### SOP-1: 投放计划上线前检查

**触发条件：** 新广告计划/广告组即将上线投放

**执行步骤：**
1. **素材审核确认**
   - 检查所有广告素材在目标平台的审核状态
   - 确认所有素材状态为"已通过"
   - 被拒素材需修改后重新提交，等待通过后再上线

2. **落地页可用性验证**
   - 测试落地页加载速度（桌面端<3秒，移动端<5秒）
   - 验证落地页在不同设备/浏览器的兼容性
   - 确认CTA按钮功能正常、表单可提交

3. **转化追踪代码验证**
   - 通过平台Tag Assistant验证Pixel/SDK安装正确
   - 触发测试转化事件确认数据回传正常
   - 确认归因窗口设置与业务需求一致

4. **预算设置Double-check**
   - 核对日预算金额与计划一致
   - 确认预算类型（日预算/总预算）设置正确
   - 验证各平台预算总和不超过总预算上限

**输出：** 上线检查清单（全部✓方可上线）

**异常处理：**
- 素材审核超24小时未通过 → 联系平台客服加急
- 落地页加载>5秒 → 通知技术团队优化后再上线
- 追踪代码异常 → 暂停上线，修复后重新验证

---

### SOP-2: 实时监控标准

**触发条件：** 广告计划进入投放状态（7×24小时持续执行）

**执行步骤：**
1. **数据刷新机制**
   - 每5分钟从各平台API拉取最新效果数据
   - 计算5分钟窗口、1小时窗口、当日累计三个粒度指标
   - 动态计算基线：过去7天同时段（同星期同时段）平均值

2. **多级预警规则**
   | 预警级别 | 触发条件 | 响应要求 |
   |---------|----------|----------|
   | 黄色预警 | CTR/CVR/CPA偏离基线30% | 记录+关注趋势，不强制行动 |
   | 红色预警 | CTR/CVR/CPA偏离基线50% | **5分钟内**必须响应并给出诊断 |
   | 紧急预警 | 10分钟消耗>日预算20% 或 平台API故障 | **立即**自动暂停+告警 |

3. **学习期特殊规则**
   - 新计划前3天标记为"学习期"
   - 学习期使用宽松阈值：偏离50%为黄色，偏离100%为红色
   - 学习期内不触发常规优化流程

4. **消耗进度监控**
   - 每30分钟检查消耗进度vs计划进度
   - 偏差>30%触发消耗节奏调整

**输出：** 实时健康度仪表盘 + 预警通知

**异常处理：**
- API连接中断 → 自动重试3次，失败后启用缓存数据+告警
- 数据延迟>30分钟 → 标记数据不完整时段，暂停基于该时段的预警判断

---

### SOP-3: 出价优化节奏

**触发条件：** 广告计划学习期结束（上线>3天）且已积累≥50个转化

**执行步骤：**
1. **学习期管理**
   - 新计划前3天为学习期，不做出价调整
   - 学习期结束条件：上线满3天 且 转化数≥50
   - 学习期内仅监控不干预（除非触发紧急预警）

2. **定期评估节奏**
   - 学习期结束后，每4小时评估一次出价调整机会
   - 评估维度：CPA vs 目标、ROAS vs 目标、展现份额、消耗速度

3. **调价规则**
   - 单次调价幅度：≤20%（上调或下调）
   - CPA>目标值120% → 降价10-15%
   - CPA<目标值80%且展现份额<70% → 加价10-15%
   - ROAS>目标值150%且有增量空间 → 加价5-10%
   - 两次调价间隔≥4小时

4. **调价后观察**
   - 调价后设置4小时观察窗口
   - 观察窗口内不做二次调整
   - 回滚条件：调价后CPA上升>30%持续2小时

**输出：** 出价调整记录（含调整前/后值、原因、预期效果）

**异常处理：**
- 学习期内效果极差（CPA>目标300%）→ 允许提前暂停重建计划
- 调价后效果恶化>30% → 自动回滚至调整前出价

---

### SOP-4: 预算再分配规则

**触发条件：** 每日固定时间（14:00）进行跨渠道预算评估；或渠道ROAS显著偏离目标时实时触发

**执行步骤：**
1. **渠道效果排名**
   - 计算各渠道近7天ROAS（加权近3天权重更高）
   - 标注高效渠道（ROAS>目标值150%）和低效渠道（ROAS<目标值50%）

2. **再分配触发条件**
   - 追加预算条件：ROAS>目标值150% → 该渠道可追加预算
   - 削减预算条件：ROAS<目标值50%连续2天 → 该渠道削减预算
   - 注意：仅满足1天不触发削减（可能是短期波动）

3. **执行幅度控制**
   - 追加上限：当前渠道预算的30%
   - 削减幅度：当前渠道预算的50%
   - 单次再分配不超过总预算的30%（超过需投放策略Agent审批）
   - 再分配后各渠道日预算不低于平台最低门槛

4. **效果预估与确认**
   - 追加前预估边际ROAS（通常低于当前ROAS）
   - 削减前预估对曝光量和转化量的影响
   - 重大调配（>总预算15%）需投放策略Agent确认

5. **执行时机**
   - 常规再分配在14:00执行（避免影响上午高峰竞价）
   - 紧急情况（如某渠道CPA暴涨）可实时执行

**输出：** 预算调配执行报告（各渠道调整明细+预期影响）

**异常处理：**
- 追加预算后ROAS下降>20% → 4小时后回滚
- 所有渠道ROAS均低于目标 → 暂停再分配，上报投放策略Agent评估整体策略

---

### SOP-5: 异常复盘

**触发条件：** 每次红色预警事件关闭后

**执行步骤：**
1. **事件记录**（预警关闭后24小时内完成）
   - 记录完整事件时间线（发现时间→诊断时间→处理时间→恢复时间）
   - 记录涉及的广告计划/账户
   - 记录异常指标和偏离程度

2. **根因分析**
   - 还原异常发生的完整因果链
   - 区分直接原因和根本原因
   - 标注是否为可预防的异常

3. **损失评估**
   - 计算异常期间的预算损失/效率损失
   - 对比如果立即处理vs实际处理的差异
   - 评估响应及时性是否达标

4. **改进措施**
   - 预警规则是否需要优化（阈值调整/新增规则）
   - 响应流程是否需要优化
   - 是否需要新增自动化保护机制
   - 更新预警规则库

5. **知识沉淀**
   - 将案例写入异常案例库
   - 更新诊断决策树
   - 如有通用性，形成新的SOP补充条款

**输出：** 异常复盘报告（含时间线、根因、损失、改进措施）

**异常处理：**
- 复盘超24小时未完成 → 升级提醒投放策略Agent
- 同类异常30天内重复发生 → 标记为系统性问题，需专项治理

  │     ├─→ 是否处于学习期（前3天）？

  │     ├─→ 是否触发预警？

  │     │     │     └─ 追踪故障 → 暂停+修复

  │     │
  │     ├─→ 每4小时：出价优化评估
  │     │     ├─ CPA达标 → 维持
  │     │     ├─ CPA偏高 → 降价（≤20%）

  │     │

  │           ├─ 高效渠道 → 追加（≤30%）
  │           ├─ 低效渠道（连续2天）→ 削减50%
  │           └─ 全部正常 → 维持当前分配
  │
  └─→ 异常关闭后 → 24小时内完成复盘报告

## 七、应急预案

### 场景1：平台API故障
- **检测**：API连续3次调用失败
- **响应**：降级为最后一次正常配置保持运行，不执行任何调整操作
- **通知**：即时告警人工运营团队
- **恢复**：API恢复后校验数据完整性，补拉缺失时段数据

### 场景2：预算异常超支
- **检测**：10分钟内消耗>日预算20%
- **响应**：自动暂停异常广告计划
- **通知**：紧急通知投放策略Agent和人工运营
- **恢复**：排除恶意点击/系统故障后，由投放策略Agent决策是否恢复

### 场景3：全账户效果崩塌
- **检测**：所有广告组CPA同时上升>50%
- **响应**：保持当前设置不变（可能是平台侧问题），观察30分钟
- **通知**：通知投放策略Agent + 检查平台公告 + 行业群确认
- **恢复**：如确认平台问题则等待恢复；如为自身问题则启动全面诊断

### 场景4：竞争对手恶意攻击（疑似）
- **检测**：异常高频点击+0转化+来源集中
- **响应**：启用平台无效点击防护 + 排除异常IP/设备
- **通知**：通知投放策略Agent评估是否暂停相关关键词/受众
- **恢复**：申请平台无效点击退款，调整定向排除攻击来源
