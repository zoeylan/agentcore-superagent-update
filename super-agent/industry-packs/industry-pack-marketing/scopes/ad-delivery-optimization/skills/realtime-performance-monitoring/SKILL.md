# 实时效果监控 (Real-time Performance Monitoring)

## 技能概述

该技能用于7×24小时不间断监控所有广告投放计划的核心效果指标，每5分钟刷新数据，基于多级预警规则体系在异常发生时第一时间（5分钟内）发出预警通知。覆盖全账户维度的健康度评估，确保投放效果始终处于可控状态。

## 适用场景

- 日常投放效果的持续性监控
- 新广告计划上线后的初期表现追踪
- 大促/活动期间的高频监控
- 多平台统一视图的效果汇总
- 效果趋势分析和预测

## 执行步骤

### Step 1: 监控指标体系配置
- 核心指标（必监控）：
  - CTR（点击率）：反映素材和定向匹配度
  - CVR（转化率）：反映落地页和产品吸引力
  - CPA（单次转化成本）：核心成本指标
  - ROAS（广告支出回报率）：核心收益指标
  - 消耗速度（元/小时）：预算健康度
- 辅助指标（参考）：
  - CPM（千次展现成本）：竞争强度
  - 频次（人均看到广告次数）：疲劳度
  - 展现份额：市场份额

### Step 2: 基线与阈值设定
- 基线计算：过去7天同时段平均值（区分工作日/周末）
- 预警阈值设定：
  - 黄色预警：偏离基线30%（关注级，不强制响应）
  - 红色预警：偏离基线50%（紧急级，5分钟内必须响应）
  - 紧急预警：预算异常消耗 或 全指标断崖式下跌（立即处理）
- 学习期特殊规则：新计划前3天使用宽松阈值（偏离50%为黄色，100%为红色）

### Step 3: 数据采集与刷新
- 每5分钟通过各平台API拉取最新数据
- 计算5分钟窗口、1小时窗口、当日累计三个时间粒度
- 与基线对比计算偏离度
- 标记趋势方向（上升/稳定/下降）

### Step 4: 异常检测与预警触发
- 逐广告组检查所有核心指标偏离度
- 触发预警时生成标准化预警消息
- 预警消息包含：级别、时间、受影响对象、异常指标、偏离程度
- 按预警级别分发通知给对应Agent

### Step 5: 健康度仪表盘维护
- 实时更新全账户健康度评分（0-100分）
- 各平台/各广告组的红绿灯状态
- 当日核心指标vs目标的完成进度
- 预测日终ROAS和CPA（基于当前趋势）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| platform_api_config | object | 是 | 各平台API连接配置 |
| ad_accounts | array | 是 | 监控的广告账户列表 |
| baseline_config | object | 是 | 基线计算规则 |
| alert_rules | object | 是 | 预警规则配置 |
| notification_targets | object | 是 | 通知目标Agent列表 |

## 最佳实践

1. **避免预警疲劳**：阈值设置要合理，误报率控制在<5%
2. **分级响应**：不是所有预警都需要即时行动，黄色预警可等待趋势确认
3. **趋势比快照重要**：单次偏离可能是噪声，连续偏离才是信号
4. **考虑数据延迟**：部分平台数据有15-30分钟延迟，预警要考虑这个因素
5. **新计划宽容度**：学习期效果波动大是正常的，不要过度反应

## 约束条件

- 数据刷新频率：每5分钟
- 红色预警响应时间：≤5分钟
- API调用频率需在各平台限额范围内
- 监控系统自身可用性>99.9%
- 数据延迟容忍：平台数据最多延迟30分钟
