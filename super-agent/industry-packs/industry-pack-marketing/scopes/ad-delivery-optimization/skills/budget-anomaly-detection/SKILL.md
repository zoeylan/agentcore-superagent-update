# 预算异常检测 (Budget Anomaly Detection)

## 技能概述

该技能用于实时检测广告预算消耗中的异常行为，包括突发性超支、持续零消耗、消耗速度异常跳变等场景。通过设置多级阈值和智能基线对比，在异常发生的5分钟内完成检测和初步定性，保护广告主预算安全。

## 适用场景

- 实时消耗流监控中的异常模式识别
- 预算耗尽前的提前预警
- API调用异常导致的重复扣费检测
- 恶意点击/无效流量引起的异常消耗
- 平台侧故障导致的计费异常

## 执行步骤

### Step 1: 基线建立
- 计算各广告组过去7天同时段的平均消耗速度（元/分钟）
- 建立消耗速度的正态分布模型（均值和标准差）
- 标注特殊日期（大促、周末）的基线修正系数
- 设置各级预警阈值：
  - 关注级：偏离均值1.5个标准差
  - 预警级：偏离均值2个标准差（约30%偏离）
  - 紧急级：偏离均值3个标准差（约50%偏离）或10分钟消耗>日预算20%

### Step 2: 实时监控
- 每5分钟采样一次消耗数据
- 计算当前5分钟窗口的消耗速度
- 与基线对比计算偏离度
- 同时检查累计消耗vs日预算比例

### Step 3: 异常分类
- **突发超支**：消耗速度突然升至基线3倍以上
  - 可能原因：竞品退出导致展现暴增、恶意点击攻击、API重复提交
- **持续零消耗**：连续30分钟无消耗
  - 可能原因：预算耗尽、广告被暂停/拒审、出价过低无竞价胜出
- **阶梯式跳变**：消耗速度从一个稳态突然跳到另一个稳态
  - 可能原因：竞争格局突变、平台算法调整、受众定向变更生效
- **预算即将耗尽**：按当前速度预计2小时内预算消耗完毕
  - 需要通知进行Pacing调整或预算追加

### Step 4: 响应与上报
- 紧急级异常：自动暂停相关广告计划 + 即时通知投放策略Agent
- 预警级异常：生成异常报告 + 通知预算调控Agent评估
- 关注级异常：记录日志 + 纳入下次评估窗口
- 所有异常记录写入异常日志供后续分析

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| spend_stream | array | 是 | 实时消耗流数据（5分钟粒度） |
| baseline_model | object | 是 | 7天基线模型参数 |
| daily_budget | number | 是 | 日预算总额 |
| current_cumulative_spend | number | 是 | 当日累计消耗 |
| ad_group_status | object | 否 | 广告组状态信息 |

## 输出规格

```json
{
  "anomaly_detected": true,
  "severity": "critical|warning|attention",
  "anomaly_type": "sudden_overspend",
  "details": {
    "affected_ad_groups": ["ag_001"],
    "current_rate": 120,
    "baseline_rate": 40,
    "deviation_factor": 3.0
  },
  "auto_action_taken": "已自动暂停ag_001",

}
```

## 最佳实践

1. **动态基线**：基线需每日更新，考虑星期效应和趋势变化
2. **误报控制**：紧急阈值设得足够高（3σ），避免频繁误报降低信任度
3. **快速确认**：异常检测后需在5分钟内确认是真异常还是正常波动
4. **自动保护优先**：宁可误暂停（之后恢复），不可漏检导致预算损失
5. **关联分析**：多个广告组同时异常可能指向平台侧问题而非单广告组问题

## 约束条件

- 检测频率：每5分钟一次
- 自动暂停仅限紧急级异常（10分钟消耗>日预算20%）
- 暂停操作必须即时通知投放策略Agent
- 误暂停后的恢复操作需投放策略Agent确认
- 检测逻辑不应消耗过多计算资源（单次检测<1秒）
