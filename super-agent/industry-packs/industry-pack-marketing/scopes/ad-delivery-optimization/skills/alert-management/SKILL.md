# 预警管理 (Alert Management)

## 技能概述

该技能用于管理广告投放监控系统中的预警全生命周期，包括预警规则配置、预警触发与分发、预警升级与降级、预警关闭与复盘。确保预警系统高效运转——不遗漏真异常，不因误报产生预警疲劳。

## 适用场景

- 预警规则的初始配置和动态调优
- 预警触发时的分级分发
- 预警超时未处理时的自动升级
- 预警消除后的自动关闭
- 历史预警的统计分析和规则优化

## 执行步骤

### Step 1: 预警规则管理
- 维护预警规则库，按指标分类：
  - 效果类：CTR/CVR/CPA/ROAS偏离基线阈值
  - 消耗类：消耗速度异常、预算即将耗尽
  - 技术类：追踪代码异常、API连接故障
  - 竞争类：CPM异常上涨、展现份额骤降
- 每个规则包含：触发条件、预警级别、通知目标、建议动作
- 支持规则的启用/禁用/修改/新增

### Step 2: 预警触发与分发
- 预警触发时立即生成标准化预警消息：
  ```
  【级别】【时间】【对象】【异常描述】【建议动作】
  ```
- 分发策略：
  - 黄色预警 → 效果监控Agent记录 + 汇总入日报
  - 红色预警 → 即时推送投放策略Agent + 预算调控Agent
  - 紧急预警 → 即时推送所有Agent + 自动执行保护动作
- 防重复：同一对象同一指标的预警在未关闭前不重复触发

### Step 3: 预警跟踪与升级
- 预警触发后开始计时
- 升级规则：
  - 红色预警5分钟内未响应 → 升级为紧急预警
  - 黄色预警持续2小时且趋势恶化 → 升级为红色预警
  - 紧急预警10分钟内未处理 → 触发人工告警通道
- 记录每条预警的响应时间和处理结果

### Step 4: 预警关闭
- 自动关闭条件：异常指标恢复到基线±15%范围内持续30分钟
- 手动关闭：Agent确认问题已解决或判定为误报
- 关闭时记录：处理措施、恢复时间、根因分类
- 误报标记：标记为误报的预警用于后续规则优化

### Step 5: 预警复盘与规则优化
- 每周统计预警数据：总数、各级别占比、平均响应时间、误报率
- 优化方向：
  - 误报率>10%的规则：放宽阈值
  - 漏报的异常：收紧阈值或新增规则
  - 响应时间>目标的预警：优化分发链路
- 每月更新预警规则库版本
- 维护预警案例库供团队学习

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alert_rules | array | 是 | 预警规则列表 |
| triggered_alert | object | 否 | 新触发的预警信息 |
| alert_history | array | 否 | 历史预警记录 |
| response_log | object | 否 | 响应处理记录 |

## 最佳实践

1. **宁缺毋滥**：预警规则不是越多越好，过多预警导致"狼来了"效应
2. **分级清晰**：每个级别的响应要求明确不同，避免所有预警都是"紧急"
3. **动态基线**：预警阈值应基于动态基线（7天均值），不是固定值
4. **误报治理**：每周关注误报率，持续优化规则降低噪声
5. **留痕可追溯**：每条预警的处理过程完整记录，支持复盘审计

## 约束条件

- 红色预警响应时间SLA：≤5分钟
- 预警防重复：同对象同指标未关闭时不重复触发
- 误报率目标：<10%
- 预警规则变更需记录变更原因和审批人
- 紧急预警的自动保护动作需立即通知投放策略Agent确认
