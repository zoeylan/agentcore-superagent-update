# 异常诊断 (Anomaly Diagnosis)

## 技能概述

该技能用于在效果监控触发预警后，快速定位异常根因并给出概率排序。通过系统化的排查流程，从多维度数据中抽丝剥茧找到问题本质，区分是创意疲劳、竞争加剧、受众饱和、平台波动还是技术故障，并给出对应的解决建议。

## 适用场景

- 红色预警触发后的根因分析
- 效果持续下滑的深度诊断
- 新素材/新受众上线后表现不及预期的原因分析
- 消耗异常（过快/过慢）的原因排查
- 平台侧异常的影响评估

## 执行步骤

### Step 1: 异常现象确认
- 从预警信息中提取：异常指标、偏离程度、持续时间、影响范围
- 确认是单点异常还是系统性异常
- 检查是否为数据延迟/统计口径导致的伪异常
- 如为伪异常则关闭预警，如为真异常则进入诊断流程

### Step 2: 快速排查清单（5分钟内完成）
1. **技术层面**：追踪代码是否正常？平台是否有公告/故障？API是否异常？
2. **广告层面**：广告是否被拒审？预算是否耗尽？出价是否被修改？
3. **竞争层面**：CPM是否上涨？展现份额是否下降？
4. **素材层面**：频次是否过高？素材在线多少天了？
5. **受众层面**：受众覆盖是否收缩？定向是否被修改？
6. **外部因素**：是否大促/节假日？行业是否有重大事件？

### Step 3: 深度诊断（问题定位）
- 基于快速排查结果缩小范围
- 利用对比分析定位：
  - 时间对比：异常前vs异常后的各维度指标变化
  - 横向对比：异常广告组vs正常广告组的差异
  - 平台对比：同一受众在不同平台的表现
- 检查变更日志：是否有人工操作导致的异常

### Step 4: 根因锁定
- 输出Top3最可能的根因（附概率）
- 每个根因附带支撑证据
- 标注确定性等级：确定/高度可能/待验证
- 如需进一步数据验证，明确需要什么数据

### Step 5: 解决方案与通知
- 针对每个根因输出对应的解决方案
- 按紧急程度排序行动项
- 明确责任Agent（谁来执行）
- 设定效果恢复的预期时间线
- 分发诊断结论给相关Agent

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alert_detail | object | 是 | 预警详细信息 |
| current_metrics | object | 是 | 当前各维度指标 |
| baseline_metrics | object | 是 | 基线指标 |
| change_log | array | 否 | 近期变更记录 |
| platform_status | object | 否 | 平台状态信息 |
| industry_trends | object | 否 | 行业趋势数据 |

## 最佳实践

1. **先排除技术因素**：很多"效果下降"其实是追踪代码问题
2. **看趋势不看点**：一个时间点的异常可能是噪声，趋势才是信号
3. **Occam剃刀原则**：优先考虑最简单/最常见的解释
4. **保持诊断记录**：每次诊断记录入案例库，加速未来诊断
5. **不要所有异常都归因为一个原因**：多因素叠加导致异常是常见的

## 约束条件

- 快速排查：5分钟内完成初步判断
- 深度诊断：1小时内完成完整报告（红色预警）
- 根因数量：最多输出Top3，避免信息过载
- 解决方案必须可执行（明确执行人和动作）
- 诊断不等于执行：诊断Agent只给结论和建议，不自行操作
