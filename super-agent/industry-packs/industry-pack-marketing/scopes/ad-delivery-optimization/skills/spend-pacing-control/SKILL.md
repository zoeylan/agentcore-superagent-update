# 消耗节奏管控 (Spend Pacing Control)

## 技能概述

该技能用于管控广告预算的日内消耗节奏，确保预算在全天均匀且高效地消耗，避免出现上午集中消耗下午无预算、或全天消耗不完的情况。通过分析各时段的转化效率差异，实现在高转化时段加速消耗、低效时段减缓消耗的智能Pacing策略。

## 适用场景

- 每日开始时的消耗节奏规划
- 日内消耗速度偏离计划时的实时调整
- 大促日的特殊消耗节奏管理（前置/后置集中消耗）
- 新计划上线的首日消耗观察
- 预算追加后的剩余时段消耗规划

## 执行步骤

### Step 1: 消耗计划制定
- 基于日预算计算理论每小时消耗额度
- 叠加时段效率系数：高转化时段（如12:00-14:00、20:00-23:00）分配更多预算
- 各平台差异化Pacing：
  - Google搜索：跟随搜索流量自然分布
  - Meta/TikTok信息流：集中在用户活跃时段
  - Amazon：跟随购物高峰（午间+晚间）
- 输出24小时消耗计划曲线

### Step 2: 实时偏差监控
- 每30分钟检查实际消耗vs计划消耗的偏差
- 偏差<10%：正常范围，不干预
- 偏差10-30%：黄色关注，评估是否需要微调
- 偏差>30%：需要主动调整（加速或减速）
- 计算按当前速度的日终预计总消耗

### Step 3: 调速策略执行
- **消耗过快（超速>30%）**：
  - 方案A：降低出价5-10%减缓竞价胜出率
  - 方案B：缩小受众范围减少曝光
  - 方案C：暂停低效广告组保留预算给高效组
- **消耗过慢（欠速>30%）**：
  - 方案A：提高出价5-10%增加竞价胜出率
  - 方案B：扩大受众范围增加曝光机会
  - 方案C：启用备用广告组增加消耗通道
- 选择对ROAS影响最小的调速方案

### Step 4: 异常消耗响应
- 10分钟内消耗超日预算20%：紧急暂停并告警（可能API异常或恶意点击）
- 连续3小时0消耗：检查广告状态、余额、审核状态
- 消耗速度突然跳变：排查是否有竞品退出/进入市场

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| daily_budget | number | 是 | 当日预算总额 |
| platform | string | 是 | 广告平台 |
| time_efficiency_data | object | 是 | 各时段转化效率数据 |
| current_spend | number | 是 | 当前已消耗金额 |
| current_time | string | 是 | 当前时间 |
| active_ad_groups | array | 否 | 活跃广告组列表 |

## 最佳实践

1. **考虑平台特性**：不同平台的消耗模式不同，不能用统一的线性Pacing
2. **高效时段优先**：宁可低效时段欠消耗，也不在低效时段强行加速
3. **留有余地**：计划消耗目标设为日预算的95%，留5%缓冲
4. **新计划宽容度**：新上线计划首日消耗波动大属正常，观察不干预
5. **周末/工作日差异**：周末消耗模式与工作日不同，需差异化Pacing

## 约束条件

- 调速操作每30分钟最多执行1次
- 出价调整用于调速时幅度控制在±10%以内
- 紧急暂停仅在10分钟消耗>日预算20%时触发
- 不在学习期广告组上执行调速操作
- 大促日的Pacing规则需提前1天手动确认
