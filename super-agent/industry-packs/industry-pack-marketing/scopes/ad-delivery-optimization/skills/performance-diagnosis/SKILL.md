# 投放效果诊断 (Performance Diagnosis)

## 技能概述

该技能用于对广告投放效果异常进行系统性根因分析，从数据表现逆推问题原因，输出可执行的优化方案。覆盖创意疲劳、竞争加剧、受众饱和、平台波动、落地页问题、追踪故障等常见异常场景的诊断方法论。

## 适用场景

- 效果监控Agent推送红色预警后的深度诊断
- 投放效果持续下滑（连续3天）的系统性分析
- 新素材/新受众上线后的效果评估
- 季度投放复盘中的问题归因
- 竞争对手异常行为的影响评估

## 执行步骤

### Step 1: 异常定义与量化
- 明确异常指标：CTR下降？CVR下降？CPA上升？ROAS下降？消耗异常？
- 量化偏离程度：对比基线（7天均值）的偏离百分比
- 确认异常范围：单广告组？单平台？全账户？
- 确认异常时间线：何时开始？持续多久？

### Step 2: 多维度排查矩阵

| 异常模式 | 可能原因 | 验证方法 |
|----------|----------|----------|
| CTR下降+频次上升 | 创意疲劳 | 检查频次趋势和素材在线时长 |
| CPM上升+展现份额下降 | 竞争加剧 | 检查行业CPM大盘趋势 |
| CTR正常+CVR下降 | 落地页问题 | 检查落地页加载速度和可用性 |
| 全指标异常+行业同步 | 平台波动 | 检查平台公告和行业群反馈 |
| 频次过高+CVR下降 | 受众饱和 | 检查受众池规模和频次分布 |
| 展现/点击正常+转化归零 | 追踪故障 | 检查Pixel/SDK和归因设置 |
| CVR下降+跳出率上升 | 流量质量问题 | 检查流量来源分布变化 |

### Step 3: 数据验证
- 交叉验证：同一受众在不同平台是否有类似趋势
- 时间维度：按小时/天/周拆解异常发生的精确节点
- A/B对比：异常广告组vs正常广告组的差异要素
- 外部验证：检查行业大盘数据排除系统性因素

### Step 4: 根因锁定与概率排序
- 基于排查结果给出Top3可能原因及概率
- 每个原因附带支撑证据（数据指标）
- 标注是否需要进一步数据验证

### Step 5: 优化方案输出
- 针对每个根因给出对应的优化动作
- 标注优化操作的优先级和预期恢复时间
- 明确执行Agent（通知创意Scope/调整受众/调整出价）
- 设定优化后的效果观察点

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alert_info | object | 是 | 预警信息（异常指标、偏离度、时间） |
| ad_group_data | object | 是 | 异常广告组的详细数据 |
| historical_baseline | object | 是 | 7天基线数据 |
| creative_info | object | 否 | 素材在线时长和频次数据 |
| landing_page_metrics | object | 否 | 落地页指标（加载时间、跳出率） |
| industry_benchmark | object | 否 | 行业大盘参考数据 |

## 输出规格

```json
{
  "diagnosis_summary": "诊断摘要",
  "root_causes": [
    {
      "rank": 1,
      "cause": "创意疲劳",

```

## 最佳实践

1. **先排除系统因素**：优先检查追踪代码和平台故障，避免对数据问题做业务归因
2. **多维度交叉验证**：单一指标变化不足以锁定原因，需多指标交叉印证
3. **区分信号和噪声**：短期（<2小时）波动可能是正常噪声，需观察趋势
4. **避免过度诊断**：学习期内的效果波动属于正常现象，不应触发诊断流程
5. **记录诊断案例库**：积累历史诊断案例加速未来类似问题的判断

## 约束条件

- 诊断必须基于数据证据，不做无根据的猜测
- 学习期内的效果波动不视为异常
- 诊断结论不超过Top3原因，避免信息过载
- 24小时内必须完成诊断报告（红色预警1小时内）
- 诊断结果需通知相关Agent执行，不自行操作
