# 受众测试验证 (Audience Testing & Validation)

## 技能概述

该技能用于设计和执行新受众定向的小预算测试方案，通过科学的A/B测试方法论验证新受众假设的有效性，基于统计显著性的数据分析决定是否扩大投放。确保每个新受众包都经过验证后才大规模投入。

## 适用场景

- 新发现的潜力受众群体需要验证
- Lookalike受众的效果评估
- 竞品受众定向策略的对标测试
- 受众组合优化（增加/去除特定标签的效果对比）
- 新市场/新地域的受众响应度探测

## 执行步骤

### Step 1: 测试假设定义
- 明确测试问题：这个新受众包的CPA是否能在目标范围内？
- 定义实验组：新受众定向
- 定义对照组：现有表现最好的受众定向
- 确定核心评估指标：CPA、CVR、ROAS
- 设定成功标准：如CPA≤对照组CPA×120%

### Step 2: 测试方案设计
- **预算分配**：测试预算=总预算×5-10%（确保能积累足够转化）
- **测试时长**：至少5-7天（覆盖完整周循环，消除星期效应）
- **样本量计算**：需至少30个转化事件才能得出统计显著结论
- **流量分配**：实验组和对照组保持其他条件一致（同素材、同出价策略、同投放时段）
- **排除重叠**：确保实验组和对照组受众不重叠

### Step 3: 测试执行与监控
- 按方案创建测试广告组
- 每日监控核心指标趋势
- 检查是否有干扰因素（如素材变更、外部事件）
- 记录测试日志（每日指标快照）
- 提前终止条件：CPA超目标300%连续2天可提前结束

### Step 4: 结果分析
- 汇总测试期全量数据
- 计算核心指标对比（实验组vs对照组）
- 进行统计显著性检验（p<0.05为显著）
- 分析维度下钻：不同时段/设备/创意的表现差异
- 排除异常数据点

### Step 5: 决策与后续
- **通过**（CPA≤对照组120%且统计显著）：进入扩量阶段，逐步增加预算
- **临界**（CPA在120-150%之间）：延长测试期或优化定向条件后重测
- **失败**（CPA>对照组150%或无统计显著性）：标记为无效受众，记录经验
- 输出测试报告归档

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| test_hypothesis | string | 是 | 测试假设描述 |
| experimental_audience | object | 是 | 实验组受众定向条件 |
| control_audience | object | 是 | 对照组受众定向条件 |
| test_budget | number | 是 | 测试预算 |
| success_criteria | object | 是 | 成功标准 |
| test_duration | number | 否 | 测试时长（天），默认7天 |

```json
{
  "test_id": "test_aud_20240115",
  "status": "passed|borderline|failed",
```

## 最佳实践

1. **一次一个变量**：每次测试只改变一个因素（受众），保持其他条件一致
2. **足够样本量**：至少30个转化才有统计意义，不要过早下结论
3. **完整周期**：测试至少覆盖一个完整周（7天），消除星期效应
4. **记录所有测试**：包括失败的测试也要归档，避免重复踩坑
5. **考虑外部因素**：测试期间避免大促/节假日干扰（或做标注）

## 约束条件

- 测试预算不超过总预算10%
- 最低测试时长5天
- 最低转化样本量30个
- 提前终止条件：CPA>目标值300%连续2天
- 同一时期同一平台的并行测试不超过3个（避免流量争抢）
