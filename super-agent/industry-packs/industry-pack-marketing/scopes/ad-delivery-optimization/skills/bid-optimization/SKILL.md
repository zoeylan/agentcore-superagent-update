# 智能出价优化 (Bid Optimization)

## 技能概述

该技能用于基于实时效果数据和竞争环境动态调整广告出价，在保证ROI目标的前提下最大化转化量。核心逻辑是通过分析历史转化数据的时段/受众/创意维度分布，结合实时竞价环境（CPM趋势、竞争强度），自动生成出价调整建议。

## 适用场景

- 学习期结束后的首次出价优化
- 每4小时的常规出价评估周期
- 竞争环境突变时的紧急出价响应
- CPA超目标时的出价下调决策
- ROAS表现优异时的扩量出价上调

## 执行步骤

### Step 1: 数据收集与基线计算
- 拉取目标广告组过去7天的核心指标（CPC、CPA、ROAS、CVR、展现份额）
- 计算各时段的转化率基线（区分工作日/周末、上午/下午/晚间）
- 获取竞争指标：搜索广告展现份额、信息流CPM趋势
- 确认当前出价策略类型和设定值

### Step 2: 效果评估
- 当前CPA vs 目标CPA的偏离度
- 当前ROAS vs 目标ROAS的偏离度
- 展现份额是否有提升空间（<80%说明出价可能偏低）
- 转化量趋势（上升/稳定/下降）
- 质量分/相关性分数变化

### Step 3: 竞争环境分析
- CPM/CPC趋势（上升说明竞争加剧，下降说明竞争缓和）
- 是否处于行业特殊时段（大促前、开学季等）
- 新进竞争对手的投放行为分析
- 平台算法更新对竞价的影响评估

### Step 4: 调价方案生成
- 根据评估结果确定调价方向（上调/下调/维持）
- 计算调价幅度（严格控制在±20%以内）
- 区分不同维度的差异化调价：
  - 时段调价：高转化时段上调、低效时段下调
  - 设备调价：移动端vs PC端差异化
  - 受众调价：高价值受众溢价、低价值受众折价
- 生成调价后的CPA/ROAS预测

### Step 5: 风险控制与输出
- 评估调价后的预算消耗速度变化
- 设置调价后的观察窗口（至少4小时）
- 定义回滚条件（调价后CPA上升>30%则回滚）
- 输出结构化调价指令

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ad_group_id | string | 是 | 目标广告组ID |
| platform | string | 是 | 广告平台 |
| current_bid_strategy | object | 是 | 当前出价策略和设定值 |
| performance_data_7d | object | 是 | 过去7天效果数据 |
| target_kpi | object | 是 | 目标KPI |
| competition_data | object | 否 | 竞争环境数据 |

## 输出规格

```json
{
  "recommendation": "increase|decrease|maintain",
  "adjustment_details": {
    "overall_bid_change": "+12%",
    "time_modifiers": {"morning": "+5%", "evening": "+15%", "night": "-20%"},
    "device_modifiers": {"mobile": "+10%", "desktop": "0%"},
    "audience_modifiers": {...}
  },
  "expected_impact": {
    "cpa_change": "-8%",
    "volume_change": "+15%",
    "spend_change": "+20%"
  },
  "rollback_condition": "CPA上升超过30%持续2小时",
  "next_evaluation": "4小时后"
}
```

## 最佳实践

1. **统计显著性优先**：至少积累50个转化事件再做出价调整判断
2. **小步迭代**：单次调幅控制在±20%，避免剧烈波动
3. **分维度优化**：先优化时段和设备维度，再精细到受众维度
4. **学习期保护**：平台标记为"学习中"的广告组不调价
5. **季节性校正**：大促期间的高CPM不应触发恐慌性降价

## 约束条件

- 单次调价幅度绝对值≤20%
- 两次调价间隔≥4小时（紧急情况除外）
- 学习期内（前3天）不执行出价调整
- 调价后必须设置回滚条件
- 日预算不足时不执行上调出价
