# 对话意图识别 (Conversation Intent Recognition)

## 技能概述

对话意图识别技能负责在外呼通话过程中实时分析客户的语言内容、语气特征和对话模式，准确判断客户的购买意图级别和情感状态，并据此触发相应的话术切换策略。该技能是智能外呼从"机械拨打"升级为"对话式营销"的关键能力。

## 适用场景

- 外呼通话中的实时意图判断和话术路由
- 在线客服聊天中的意向信号识别
- 客户回访中的满意度和流失倾向判断
- 对话结束后的意图标签自动归类

## 详细执行步骤

### 步骤1：对话信号采集
1. 实时接收客户语音转文字内容
2. 分析语言特征：关键词提取、句式结构、问答模式
3. 捕捉非语言信号：语速变化、停顿时长、语气词使用频率
4. 记录对话上下文：当前话题、话轮数、客户主动发言比例

### 步骤2：意图分类判断
基于多维度信号综合判断，输出四类核心意图：

**感兴趣（Interested）信号特征：**
- 主动询问产品细节（价格、功能、使用方法）
- 表达明确需求（"我正好需要"、"这个功能不错"）
- 请求更多信息（"能发个资料吗"、"有案例吗"）
- 讨论使用场景（"我们公司目前..."）

**犹豫（Hesitant）信号特征：**
- 价格敏感表达（"有点贵"、"预算有限"）
- 时间推迟（"过段时间再说"、"最近比较忙"）
- 对比竞品（"别家好像更便宜"、"我再看看"）
- 征询他人意见（"需要跟领导商量"）

**拒绝（Rejected）信号特征：**
- 明确否定（"不需要"、"没兴趣"、"别打了"）
- 快速结束对话意图（"好的知道了再说吧"）
- 否认需求存在（"我们没有这个需求"）

**投诉（Complaint）信号特征：**
- 情绪激动（提高音量、使用负面词汇）
- 威胁性表达（"我要投诉"、"骚扰电话"）
- 对频繁打扰表达强烈不满

### 步骤3：话术切换触发
1. 识别为"感兴趣" → 触发深度介绍话术（产品优势+案例+优惠）
2. 识别为"犹豫" → 触发疑虑消除话术（性价比论证+试用邀请+限时优惠）
3. 识别为"拒绝" → 触发礼貌收尾话术（感谢时间+保留联系方式+开放后续）
4. 识别为"投诉" → 立即触发安抚降级流程（道歉+标记人工跟进+加入DNC）

### 步骤4：意图置信度评估
1. 输出意图判断的置信度分数（0-100%）
2. 置信度>80%：直接执行话术切换
3. 置信度60-80%：执行试探性确认话术（"您看这样好吗..."）
4. 置信度<60%：保持当前话术继续收集信号

## 输入规格

```json
{
  "conversation_id": "string",
  "current_utterance": "string - 客户最新一句话",
  "conversation_history": [
    {
      "role": "agent|customer",
      "content": "string",
      "timestamp": "datetime"
    }
  ],
  "customer_profile": {
    "lead_grade": "string",
    "previous_intent": "string (上次通话的意图)",
    "industry": "string",
    "pain_points": ["string"]
  },
  "current_script_stage": "string - 当前话术阶段"
}
```

## 输出规格

```json
{
  "intent_classification": "interested|hesitant|rejected|complaint",
  "confidence_score": "number (0-100)",
  "sub_intent": "string - 细分意图（如 price_sensitive, need_approval）",
  "detected_signals": [
    {
      "signal_type": "keyword|tone|pattern",
      "content": "string",
      "weight": "number"
    }
  ],
  "recommended_action": "string - 建议话术切换方向",
  "script_branch_id": "string - 推荐切换到的话术分支",
  "escalation_needed": "boolean - 是否需要升级人工"
}
```

## 最佳实践

1. **上下文连贯性**：不仅看当前一句话，要结合整段对话的演变趋势判断意图（客户可能从犹豫转向感兴趣）
2. **文化适配**：中文对话中"好的"/"嗯嗯"等模糊回应不应直接判断为同意，需要结合语境
3. **过渡信号重视**：客户从拒绝到"那你说说看"是重要的转向信号，需立即抓住
4. **避免过度解读**：一个关键词不足以判断意图，需要多重信号交叉验证
5. **实时性要求**：意图判断延迟不超过500ms，确保话术切换的自然流畅

## 约束条件

- 单次意图判断响应时间≤500ms
- 意图分类准确率目标≥85%
- 投诉识别召回率必须≥95%（宁可误判也不能漏判投诉）
- 不得基于客户口音、方言等进行歧视性判断
- 对话数据处理必须符合隐私保护法规
