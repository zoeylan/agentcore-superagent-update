# 质量决策与重做策略制定

## 技能概述
本技能用于在收到质量评估Agent的评分结果后，进行综合决策——判断素材是否可交付、不合格素材应采取何种重做策略（局部修改vs完全重做vs调整Brief方向）。这是确保最终交付质量的关键决策环节。

## 适用场景
- 收到批次质量评估报告后的决策
- 合格率低于80%时的策略调整
- 重复不合格素材的根因分析和方向调整
- 品牌合规严重违规的紧急处理

## 执行步骤

### Step 1: 评估结果分析
- 汇总本批次所有素材的评估报告
- 计算合格率（pass + conditional_pass / total）
- 识别不合格素材的共性问题模式
- 评估严重程度分布（info / warning / critical）

### Step 2: 决策分支判定
```
决策树：
├── 合格率 >= 80%
│   ├── 合格素材 → 入库待交付
│   └── 不合格素材 → 执行标准重做流程
├── 合格率 60%-80%
│   ├── 分析共性问题
│   ├── 调整生成参数/prompt
│   └── 重新生成不合格批次
├── 合格率 < 60%
│   ├── 暂停该批次
│   ├── 根因分析（Brief方向问题？风格指引不清？参考图误导？）
│   └── 向上反馈建议调整Brief
└── 品牌合规Critical违规
    ├── 立即中止批次
    ├── 发出紧急告警
    └── 等待人工确认后再行动
```

### Step 3: 重做策略制定
- **局部修改**（适用于评分6-7分、问题明确）：
  - 指定具体修改项（如"背景色从#FF0000调整为品牌红#CC0033"）
  - 保留可复用部分，只重做问题部分
- **完全重做**（适用于评分<6分、问题系统性）：
  - 重新制定生成prompt
  - 更换风格方向或构图方案
  - 重做次数上限：3次
- **Brief方向调整**（适用于三次重做仍不合格）：
  - 标记该需求为"需求调整"
  - 输出问题诊断报告
  - 建议Brief修改方向

### Step 4: 重做任务下发
- 生成修改指令（具体到参数级别）
- 设置重做任务优先级为P0（快速闭环）
- 标记重做轮次（第1次/第2次/第3次）

## 输入规格
| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| batch_assessment_report | Object | ✓ | 批次评估报告 |
| individual_scores | Array | ✓ | 各素材评分详情 |
| retry_history | Array | × | 历史重做记录 |
| brand_guidelines | Object | ✓ | 品牌规范 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| decision | String | pass_batch / partial_redo / full_redo / adjust_brief / emergency_halt |
| redo_tasks | Array | 重做任务清单（含具体修改指令） |
| approved_assets | Array | 已批准素材列表 |
| escalation_alert | Object | 升级告警（如有） |
| root_cause_analysis | String | 根因分析 |

## 最佳实践
1. 第一次不合格优先采取"局部修改"，控制重做成本
2. 关注不合格模式——如果多个素材犯同一个错，说明是系统性问题
3. 三次重做规则严格执行，避免无限循环
4. Critical告警发出后不要自行恢复，必须等人工确认
5. 记录每次决策的原因，为后续优化评估标准提供数据

## 约束与限制
- 单个素材最多重做3次，超出则升级处理
- 品牌合规Critical违规不允许自动重试，必须人工介入
- 决策响应时间≤30秒，不阻塞后续任务
- 所有决策必须有可追溯的依据记录
