# 批量任务调度与并发编排

## 技能概述
本技能负责管理创意生产过程中的批量任务调度，包括多Agent并行任务分配、执行进度监控、超时降级处理和异常恢复。确保100+素材的批量生产任务能够高效、有序地完成。

## 适用场景
- 大批量素材生产任务的并行调度
- 多Agent协同工作时的依赖管理
- 任务执行超时的降级处理
- 生产进度的实时监控与瓶颈识别

## 执行步骤

### Step 1: 任务队列初始化
- 接收Brief解析后的任务清单
- 建立任务依赖关系图（DAG）
- 识别可并行执行的任务组
- 设置每个任务的超时阈值（图片30秒、视频5分钟）

### Step 2: 并发批次规划
```
批次规划规则：
- 同类型无依赖任务：最大并发数=50
- 有依赖关系的任务：前置任务完成后自动触发
- 图文合成任务：等待对应的视觉+文案两个前置任务均完成
- 预留20%并发能力应对重做任务
```

### Step 3: 任务分发与监控
- 向visual-generation Agent分发视觉生成任务
- 向content-editor Agent分发文案撰写任务
- 实时跟踪各任务状态：queued → running → completed/failed
- 每5秒刷新一次进度面板

### Step 4: 异常处理与降级
- **单任务超时**：自动重试1次（调整参数），再次超时标记为failed
- **批次超时风险**：当完成率<预期进度的70%时，启动降级策略
  - 降级策略A：减少变体数量（如从5个变体降为3个）
  - 降级策略B：降低非核心渠道优先级
  - 降级策略C：通知人工介入
- **Agent无响应**：30秒无响应视为异常，任务重新入队

### Step 5: 完成汇总
- 统计最终完成情况：成功/失败/降级处理数量
- 输出批次执行报告
- 将完成的素材提交质量评估队列

## 输入规格
| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| task_list | Array | ✓ | 待执行任务清单 |
| concurrency_limit | Number | × | 最大并发数（默认50） |
| timeout_config | Object | × | 超时配置 |
| priority_rules | Object | × | 优先级规则 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| batch_id | String | 批次唯一标识 |
| execution_report | Object | 执行情况报告 |
| progress_dashboard | Object | 进度面板数据 |
| degradation_log | Array | 降级处理记录 |

## 最佳实践
1. 任务粒度要够细——一个任务=一个素材=一次质量评估单元
2. 避免"大锁"——不要让一个慢任务阻塞整个批次
3. 监控队列深度，积压超过200个任务时预警
4. 保持重做任务的优先级高于新任务（快速闭环）
5. 记录每个任务的实际耗时，用于优化后续批次预估

## 约束与限制
- 单批次最大任务数：500
- 最大并发数不超过系统容量的80%
- 降级决策需在10秒内完成，不得阻塞主流程
- 所有任务状态变更必须有完整的日志记录
