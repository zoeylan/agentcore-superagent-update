# 多尺寸渠道适配渲染

## 技能概述
本技能负责将核心创意素材适配到不同广告平台和广告位所需的多种尺寸规格，确保素材在各种展示环境下都能保持最佳视觉效果。涵盖智能裁切、重构图布局、安全区域管理等能力。

## 适用场景
- 一套创意适配多平台多广告位（Google/Meta/TikTok/抖音/微信等）
- 横版素材转竖版、方图转宽屏等大比例差异转换
- 各平台安全区域和裁切规则适配
- 批量素材的规格化输出

## 执行步骤

### Step 1: 渠道规格确认
确认目标输出规格清单：

| 平台 | 广告位 | 尺寸 | 安全区 | 文件要求 |
|------|--------|------|--------|----------|
| Meta Feed | 信息流 | 1080×1080 | 上下各5% | JPG/PNG≤30MB |
| Meta Story | 故事 | 1080×1920 | 上下各14% | JPG/PNG≤30MB |
| Google Display | 展示 | 1200×628 | 边缘10% | JPG/PNG≤150KB |
| 抖音信息流 | 竖版 | 720×1280 | 底部20% | JPG≤2MB |
| 微信朋友圈 | 广告 | 800×800 | 无特殊 | JPG≤1MB |
| Banner通栏 | PC端 | 1920×600 | 左右各15% | PNG≤500KB |

### Step 2: 智能构图适配策略
根据原始素材构图和目标尺寸差异选择适配策略：
- **等比缩放+裁切**：适用于比例差异小的转换
- **重构图**：适用于横竖版大转换，重新安排元素位置
- **背景延展**：保持主体不变，AI扩展背景区域
- **多主体重排**：复杂构图中元素位置重新规划

### Step 3: 安全区域管理
- 标记各平台的安全区域（不被UI元素遮挡的区域）
- 确保产品主体、品牌logo、核心文案都在安全区内
- 预留文字叠加区域（通常为画面1/3面积）
- 验证各平台特殊区域（如TikTok底部互动栏、Instagram头像区域）

### Step 4: 批量渲染输出
- 按照目标规格批量输出
- 每个输出文件命名规范：`{brief_id}_{variant}_{platform}_{size}.{format}`
- 附带缩略图预览用于快速审核
- 文件大小控制在各平台限制范围内

### Step 5: 适配质量检查
- [ ] 主体完整，未被裁切关键部分
- [ ] 品牌logo在安全区内且清晰可辨
- [ ] 文字区域预留充足
- [ ] 画面平衡，无过度留白或过度拥挤
- [ ] 文件尺寸/格式符合平台要求

## 输入规格
| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| source_image | File | ✓ | 原始素材 |
| target_specs | Array | ✓ | 目标尺寸规格列表 |
| focus_point | Object | × | 主体焦点坐标 |
| text_area_requirement | Boolean | × | 是否预留文案区 |
| platform_guidelines | Object | × | 平台特殊规范 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| rendered_assets | Array | 各尺寸输出文件 |
| adaptation_method | String | 使用的适配方法 |
| safe_zone_map | Object | 安全区域标注图 |
| file_specs | Object | 文件规格信息 |

## 最佳实践
1. 核心创意设计时就考虑多尺寸——主体居中、周围留有扩展空间
2. 超宽或超窄比例（如1920×600）需要特殊构图策略，不宜强行裁切
3. 各平台的安全区数据定期更新（平台UI经常调整）
4. 文件大小优化：优先使用JPG（照片类）、PNG（需透明/文字清晰类）
5. 建议先生成高分辨率母版，再下采样到各目标尺寸

## 约束与限制
- 不得裁切产品主体的关键部分
- 输出分辨率不得低于各平台最低要求
- 文件大小必须在平台限制范围内
- 背景延展不得引入不合理元素
- AI放大不超过2x，避免质量损失
