# 多风格适配与品牌调性匹配

## 技能概述
本技能负责将统一的创意概念适配到不同的视觉风格，确保生成素材既符合品牌调性又能匹配不同受众群体和营销场景的审美偏好。涵盖风格定义、风格转换、品牌调性校准等能力。

## 适用场景
- 同一产品需要多种风格呈现（简约/奢华/活力/科技）
- 针对不同受众群体的视觉风格差异化
- 品牌调性在不同场景下的具象化表达
- 节日/季节/主题营销的风格切换
- 跨文化市场的视觉偏好适配

## 执行步骤

### Step 1: 品牌调性解析
从品牌指南中提取核心调性关键词并转化为视觉参数：
```
品牌调性 → 视觉参数映射：
- "高端奢华" → 低饱和度、深色背景、金/银点缀、极简构图
- "年轻活力" → 高饱和度、明亮色彩、动态构图、撞色搭配
- "专业可靠" → 蓝灰色系、对称构图、清晰线条、留白充裕
- "自然温暖" → 暖色调、柔光、自然材质纹理、生活化场景
- "科技未来" → 冷色调、渐变光效、几何形状、暗背景+亮点
```

### Step 2: 风格矩阵建立
为目标项目建立风格矩阵：
| 维度 | 选项A | 选项B | 选项C |
|------|-------|-------|-------|
| 色温 | 暖色(3000K) | 中性(5000K) | 冷色(7000K) |
| 饱和度 | 高(鲜艳) | 中(自然) | 低(高级灰) |
| 构图风格 | 居中对称 | 三分法 | 对角线动态 |
| 背景复杂度 | 纯色极简 | 简约场景 | 丰富场景 |
| 光影风格 | 硬光高对比 | 柔光自然 | 戏剧性光影 |

### Step 3: A/B变体风格设计
- 定义变体维度（每次只变一个维度，确保科学可比）
- 变体A：基准版本（最符合品牌调性）
- 变体B：在某个维度上的探索版本
- 变体C：在另一个维度上的探索版本
- 记录每个变体的差异点，便于效果归因

### Step 4: 跨文化风格适配
不同市场的视觉偏好差异处理：
- 东亚市场：偏好精致细节、柔和色调、留白
- 欧美市场：偏好简洁直接、高对比、大字体
- 中东市场：注意色彩禁忌（如绿色的宗教含义）
- 东南亚市场：偏好鲜艳色彩、热闹氛围

### Step 5: 风格一致性验证
确保同系列素材在风格上统一：
- 色彩一致性检查（同系列Delta E<3）
- 光影方向一致
- 构图语言统一
- 品牌元素位置固定

## 输入规格
| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| brand_tone | Object | ✓ | 品牌调性描述 |
| target_styles | Array | ✓ | 目标风格列表 |
| audience_segments | Array | × | 受众细分 |
| cultural_context | String | × | 文化市场背景 |
| variant_strategy | Object | × | 变体策略 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| style_parameters | Object | 各风格的具体视觉参数 |
| variant_plan | Array | 变体方案清单 |
| style_prompt_templates | Array | 风格化prompt模板 |
| consistency_guidelines | Object | 一致性检查标准 |

## 最佳实践
1. 品牌调性是不可逾越的底线——任何风格探索都在品牌框架内
2. A/B测试变体控制单一变量原则，否则无法归因
3. 为每种风格建立prompt模板库，避免每次从零开始
4. 参考竞品风格但不抄袭——了解行业审美趋势但保持品牌差异
5. 季节性/节日素材提前2周准备风格方案

## 约束与限制
- 风格变化不得偏离品牌色彩体系核心色
- 不得使用可能引起文化冒犯的视觉元素
- 同一批次内的风格差异应是可控的、有目的的
- 风格参数必须可量化、可复现
