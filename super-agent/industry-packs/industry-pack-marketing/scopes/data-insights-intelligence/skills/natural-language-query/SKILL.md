# 自然语言数据查询 (Natural Language Query)

## 概述

自然语言数据查询技能使营销团队能通过口语化的问题直接获取数据洞察，无需掌握SQL、Python或任何技术工具。系统将自然语言问题解析为结构化数据查询，从多个数据源获取答案，并以易于理解的方式返回结果。目标响应时间<10秒。

## 适用场景

- 日常数据查询："上周抖音渠道的ROI是多少？"
- 对比分析："本月vs上月各渠道获客成本对比"
- 排名查询："哪个受众群体转化率最高？"
- 趋势查询："近30天的CPM趋势是怎样的？"
- 归因查询："这次大促的转化主要来自哪个渠道？"
- 异常排查："为什么今天的消耗比昨天高了50%？"

## 执行步骤

### 步骤1：查询意图解析
- **实体识别**：提取查询中的关键实体
  - 时间范围：上周、本月、近30天、Q3、双11期间
  - 指标：ROI、ROAS、CTR、CVR、CPA、消耗、转化数
  - 维度：渠道、广告组、受众群体、创意、地域
  - 筛选条件：特定渠道名、特定活动名
- **意图分类**：确定查询类型
  - 单值查询（"XX是多少"）
  - 对比查询（"A和B哪个好"）
  - 排名查询（"最高/最低的是"）
  - 趋势查询（"变化趋势"）
  - 原因分析（"为什么"）
- **歧义消解**：遇到歧义时主动确认（"您说的'效果'是指CTR还是CVR？"）

### 步骤2：查询执行
- 根据解析结果确定数据源（广告平台API、数据仓库、归因系统）
- 构建结构化查询并执行
- 简单查询（单表、单指标）：目标5秒内返回
- 复杂查询（多表关联、聚合计算）：15秒内返回中间状态"正在分析..."
- 超时处理：30秒未完成的查询返回部分结果+完整结果预计时间

### 步骤3：结果组织与呈现
- **单值结果**：指标值 + 环比变化 + 同比变化 + 一句话解读
  - 示例："上周抖音渠道ROAS为3.2，环比+12%，高于目标值（3.0）"
- **对比结果**：对比表格 + 差异亮点 + 建议
- **排名结果**：TOP N列表 + 头尾差距 + 异常项标注
- **趋势结果**：趋势描述 + 拐点标注 + 驱动因素
- **原因分析**：可能原因列表（按可能性排序）+ 验证建议

### 步骤4：上下文增强
- 自动补充相关上下文信息（即使用户没有问）：
  - 行业基准对比："您的CTR为2.3%，行业均值为1.8%"
  - 历史趋势定位："当前ROAS处于近3月最高水平"
  - 关联指标联动："CVR上升的同时CPC也在上升，建议关注整体CPA"
- 如果数据存在已知质量问题，主动声明

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query_text | String | 是 | 用户的自然语言查询 |
| user_context | Object | 否 | 用户上下文（角色、关注的品牌/渠道） |
| conversation_history | Array | 否 | 对话历史（支持追问） |
| preferred_format | Enum | 否 | 偏好的返回格式（text/table/chart_description） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| answer | String | 结构化文字回答 |
| data_points | Array | 原始数据点（支持下钻） |
| data_source | String | 数据来源标注 |
| data_time_range | String | 数据覆盖时间范围 |
| confidence | Number | 回答置信度（0-1） |
| context_enrichment | Object | 上下文补充信息 |
| follow_up_suggestions | Array | 建议的追问方向 |
| response_time_ms | Number | 实际响应时间 |
| data_quality_notes | String | 数据质量说明（如有问题） |

## 最佳实践

1. **渐进式响应**：先返回核心答案，再补充上下文和建议，用户不必等待全部计算完成
2. **追问引导**：每次回答后附带2-3个相关的追问建议，引导用户深入分析
3. **指标定义透明**：首次提及某指标时附带简短定义，避免理解歧义
4. **历史对比自动化**：除非用户指定绝对值，默认同时提供环比/同比参照
5. **失败优雅降级**：某数据源不可用时，基于可用数据给出部分答案并说明局限

## 约束条件

- 响应时间SLA：简单查询<10秒，复杂查询<30秒（15秒内给出中间状态）
- 查询失败率<5%，失败时给出替代查询建议
- 不猜测数据：如果查不到数据，明确告知"该数据暂不可用"而非编造
- 敏感数据权限：遵循数据访问权限控制，不同角色看到的数据范围不同
- 不提供预测性结论：查询结果是事实数据，预测性判断应转交趋势分析技能
