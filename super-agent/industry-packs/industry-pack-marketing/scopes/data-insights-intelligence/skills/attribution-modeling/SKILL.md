# 多触点归因建模 (Attribution Modeling)

## 概述

多触点归因建模技能用于科学量化用户转化路径中各营销触点的贡献度。支持多种归因模型（首次触达、末次触达、线性、时间衰减、位置归因、数据驱动），帮助营销团队准确理解各渠道的真实价值，为预算分配提供数据支撑。

## 适用场景

- 月度归因分析：评估各渠道对转化的真实贡献
- 预算分配优化：基于归因结果调整渠道预算比例
- 新渠道价值验证：评估新增渠道在用户路径中的助攻价值
- 归因模型选择：对比不同模型结果差异，选择最适配业务的主模型
- 创意素材归因：量化不同创意对最终转化的贡献差异

## 执行步骤

### 步骤1：转化路径数据整合
- 采集各触点数据：展示广告曝光、点击、社交互动、搜索点击、直接访问、邮件打开
- 跨设备用户ID打通：通过确定性匹配（登录ID）和概率性匹配（设备指纹）关联同一用户的多设备行为
- 构建用户级转化路径序列：按时间顺序排列所有触点，标注触点类型、渠道、创意、时间戳
- 设置归因窗口：点击归因窗口（默认28天）、展示归因窗口（默认7天）

### 步骤2：归因模型计算
- **首次触达模型（First Touch）**：100%功劳归于路径中第一个触点
- **末次触达模型（Last Touch）**：100%功劳归于转化前最后一个触点
- **线性模型（Linear）**：功劳在所有触点间平均分配
- **时间衰减模型（Time Decay）**：距转化越近的触点获得越多功劳（半衰期默认7天）
- **位置归因模型（Position-Based）**：首末触点各40%，中间触点平分20%
- **数据驱动模型（Data-Driven）**：基于Shapley Value或Markov Chain计算各触点的边际贡献

### 步骤3：模型对比与选择
- 并行运行所有归因模型，输出各渠道在不同模型下的贡献度对比矩阵
- 计算模型间差异度：如果某渠道在不同模型下贡献差异>30%，标记为"归因敏感渠道"
- 基于业务特征推荐主模型：
  - 品牌广告为主 → 推荐首次触达或位置归因
  - 效果广告为主 → 推荐时间衰减或数据驱动
  - 渠道组合复杂 → 推荐数据驱动模型
- 每月执行模型校准，对比归因建议与实际预算调整后的效果

### 步骤4：归因结果输出
- 按渠道维度输出：各渠道的归因转化数、归因收入、归因ROAS
- 按创意维度输出：各创意素材的归因贡献和效率排名
- 按路径模式输出：高频转化路径TOP10及其转化率
- 输出预算优化建议：当前分配 vs 归因最优分配的差距和调整方向

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| conversion_paths | Array | 是 | 用户转化路径数据集 |
| attribution_window | Object | 否 | 归因窗口设置（click_days, view_days） |
| models_to_run | Array | 否 | 指定运行的归因模型（默认全部） |
| primary_model | String | 否 | 指定主模型（默认data-driven） |
| dimensions | Array | 否 | 分析维度（channel/campaign/creative） |
| time_range | String | 是 | 分析时间范围 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| attribution_results | Object | 各模型归因结果 |
| model_comparison | Array | 模型间对比矩阵 |
| channel_attribution | Array | 渠道级归因详情 |
| channel_attribution[].channel | String | 渠道名称 |
| channel_attribution[].attributed_conversions | Number | 归因转化数 |
| channel_attribution[].attributed_revenue | Number | 归因收入 |
| channel_attribution[].attributed_roas | Number | 归因ROAS |
| top_conversion_paths | Array | 高频转化路径 |
| budget_optimization | Object | 预算优化建议 |
| sensitive_channels | Array | 归因敏感渠道（模型间差异>30%） |

## 最佳实践

1. **多模型对比决策**：不要依赖单一归因模型，通过模型间对比理解归因的不确定性范围
2. **归因窗口适配**：高客单价/长决策周期产品延长归因窗口（如60天），低客单价/冲动消费缩短（如7天）
3. **增量归因验证**：定期通过A/B实验（如某渠道暂停投放）验证归因模型的准确性
4. **路径完整性**：关注路径覆盖率指标——如果大量转化无法追踪到完整路径（如>20%），需优先修复跟踪问题
5. **避免过度优化**：归因结果是"历史最优"而非"未来最优"，预算调整单次幅度不超过20%

## 约束条件

- 归因模型覆盖率目标：≥95%的转化路径可被归因
- 跨设备匹配率监控：确定性匹配率>60%为健康水平
- 模型校准周期：每月一次，对比归因建议执行效果
- 归因结果仅作为决策参考，不得直接自动执行大幅预算调整（>30%需人工确认）
- 数据驱动模型需要最少1000条转化路径才能产出可靠结果
