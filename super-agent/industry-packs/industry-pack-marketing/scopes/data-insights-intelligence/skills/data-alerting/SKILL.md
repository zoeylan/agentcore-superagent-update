# 数据异常告警 (Data Alerting)

## 概述

数据异常告警技能用于对营销关键指标进行持续监控，在指标出现显著异常波动时主动推送告警通知。包含异常描述、可能原因分析和建议行动，帮助营销团队及时发现并响应数据变化，抓住机会或规避风险。

## 适用场景

- 关键指标突变告警：ROAS骤降、CPA飙升、CTR异常波动
- 预算消耗异常：消耗速度远超/远低于日均节奏
- 数据源中断告警：平台API连接失败、数据延迟超阈值
- 机会发现通知：某渠道/受众表现突然显著提升
- 竞品异动告警：竞品大规模投放活动启动

## 执行步骤

### 步骤1：告警规则配置
- **指标监控规则**：
  - ROAS日环比下降>20% → 黄色告警
  - ROAS日环比下降>40% → 红色告警
  - CPA日环比上升>30% → 黄色告警
  - CTR日环比下降>30% → 黄色告警
  - 消耗速度偏离日均>50% → 红色告警
- **数据健康规则**：
  - 数据源延迟>2小时 → 黄色告警
  - 数据源中断>4小时 → 红色告警
  - 关键字段空值率>10% → 黄色告警
- **机会发现规则**：
  - 某渠道/受众ROAS日环比提升>50% → 机会通知
  - 新测试广告组表现超基线200% → 机会通知

### 步骤2：实时监控执行
- 每5分钟采集核心指标数据
- 与动态基线对比（考虑小时级别波动、星期效应、节日效应）
- 连续3个采集周期（15分钟）持续偏离基线才触发告警（避免瞬时波动误报）
- 同一告警24小时内不重复推送（除非恶化升级）

### 步骤3：告警内容生成
- **告警标题**：[严重级别] + 指标名称 + 变化方向 + 变化幅度
  - 示例："🔴 [红色告警] 信息流渠道ROAS骤降42%"
- **告警详情**：
  - 当前值 vs 基线值 vs 目标值
  - 变化开始时间和持续时长
  - 影响范围（哪些广告组/创意受影响）
- **初步原因分析**：基于关联指标变化推断可能原因（2-3个候选）
- **建议行动**：针对最可能原因给出具体操作建议

### 步骤4：告警分发与升级
- 黄色告警 → 推送至营销运营群/工作台通知
- 红色告警 → 推送至负责人+同步投放策略Agent自动响应
- 机会通知 → 推送至运营群，建议追投/扩量操作
- 告警未在1小时内确认 → 自动升级至上级负责人

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alert_rules | Array | 是 | 告警规则配置（指标、阈值、级别） |
| monitoring_targets | Array | 是 | 监控对象（渠道、广告组、整体） |
| baseline_config | Object | 否 | 基线计算方式（默认近7天动态均值） |
| notification_channels | Array | 否 | 通知渠道（IM/email/SMS） |
| suppression_rules | Object | 否 | 告警抑制规则（避免重复告警） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| alert_id | String | 告警唯一ID |
| severity | Enum | 告警级别（red/yellow/opportunity） |
| title | String | 告警标题 |
| metric_name | String | 异常指标 |
| current_value | Number | 当前值 |
| baseline_value | Number | 基线值 |
| deviation_pct | Number | 偏离百分比 |
| started_at | DateTime | 异常开始时间 |
| affected_scope | Object | 影响范围 |
| probable_causes | Array | 可能原因（按可能性排序） |
| recommended_actions | Array | 建议操作 |
| auto_response_taken | String | 已执行的自动响应（如有） |

## 最佳实践

1. **动态基线**：使用滑动窗口计算基线而非固定阈值，适应业务的自然波动节奏
2. **降噪优先**：宁可漏报低风险告警，也不要让噪音淹没真正重要的信号
3. **上下文完整**：每条告警必须包含"发生了什么+影响多大+可能为什么+建议怎么做"
4. **告警收敛**：相关联的多个指标异常合并为一条根因告警，而非分散推送
5. **反馈闭环**：跟踪告警的后续处理结果，优化告警规则的准确性

## 约束条件

- 每日告警数量控制：红色≤3条、黄色≤10条、机会通知≤5条
- 连续波动确认窗口：15分钟（3个采集周期）
- 告警响应SLA：红色告警推送后30分钟内需人工确认
- 不对已知计划内变更（如：计划暂停投放）产生误告警
- 告警内容不得包含猜测性的绝对结论，原因分析需标注为"可能原因"
