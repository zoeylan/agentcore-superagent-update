# 竞品监测 (Competitor Monitoring)

## 概述

竞品监测技能用于系统化追踪竞争对手的广告创意、投放策略、价格变动和营销活动动态。通过多数据源整合和自动化扫描，确保营销团队能及时掌握竞争格局变化并快速响应。

## 适用场景

- 每日自动扫描Top5核心竞品的广告创意库变化
- 每周全面监测Top10竞品的投放策略和渠道布局
- 竞品重大营销活动（大促、新品发布、品牌升级）的即时告警
- 竞品价格体系和促销策略的持续跟踪

## 执行步骤

### 步骤1：数据源配置与接入
- 配置竞品广告素材监测源（广告透明度库：Meta Ad Library、TikTok Creative Center、Google Ads Transparency Center）
- 接入社交媒体监听（品牌提及、话题标签、官方账号动态）
- 配置价格监测爬虫（电商平台SKU价格、促销标签）
- 设置竞品官网变更监测（产品页、活动页、价格页）

### 步骤2：日常监测执行
- 每日08:00启动Top5竞品广告素材扫描
  - 新增素材识别与分类（图片/视频/文案）
  - 素材风格变化检测（色调、排版、代言人）
  - 投放渠道变化检测（新增/撤出某平台）
- 比对昨日快照，标记显著变化项

### 步骤3：变化评级与告警
- L1级变化（常规更新）：素材常规迭代、小幅价格调整 → 记录归档
- L2级变化（值得关注）：新创意方向、新渠道试水、季节性促销 → 纳入周报
- L3级变化（重大事件）：品牌升级、大规模降价、新品类进入、核心渠道策略转变 → 24小时内即时告警

### 步骤4：情报输出
- 生成结构化竞品动态卡片：「竞品名称 | 变化类型 | 具体内容 | 发现时间 | 评级 | 建议应对」
- L3级事件附带影响评估和应对建议
- 所有数据归入竞品情报知识库供查询

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| competitor_list | Array | 是 | 监测竞品清单，含名称、品牌ID、关联平台账号 |
| monitoring_dimensions | Array | 是 | 监测维度（creative/pricing/channel/promotion） |
| alert_rules | Object | 否 | 自定义告警规则和阈值 |
| frequency | String | 是 | 监测频率（daily/weekly） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| changes_detected | Array | 检测到的变化列表 |
| change_level | Enum | 变化评级（L1/L2/L3） |
| competitor_name | String | 竞品名称 |
| change_type | String | 变化类型 |
| change_detail | String | 具体变化描述 |
| evidence_url | String | 证据链接/截图 |
| impact_assessment | String | 影响评估（L3级） |
| suggested_response | String | 建议应对措施 |
| report_timestamp | DateTime | 报告生成时间 |

## 最佳实践

1. **监测频率适配**：核心竞品（市场份额前3）每日监测，次要竞品每周监测，避免信息过载
2. **变化基线管理**：维护每个竞品的"当前状态快照"，仅报告增量变化
3. **误报控制**：设置变化显著性阈值（如价格变动>5%才触发），避免噪音信息
4. **历史趋势串联**：将单次变化放入历史时间线，识别竞品策略演变方向
5. **证据留存**：所有关键变化附带截图或原始数据快照，确保可回溯验证

## 约束条件

- 监测行为必须合规，不得违反目标平台的服务条款
- 告警推送不超过3条/天（L3级除外），避免信息疲劳
- 数据源不可用时标记"数据缺失"而非跳过，确保完整性可追溯
- 主观判断（如"创意质量下降"）必须标注为分析师观点而非事实
