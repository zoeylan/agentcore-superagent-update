# 决策级报告生成 (Decision Report Generation)

## 概述

决策级报告生成技能用于整合多维度营销数据（竞品情报、归因分析、趋势洞察、效果数据），输出面向管理层的决策级营销分析报告。报告以"可执行的建议"为导向，而非仅呈现数据，帮助CMO和营销负责人快速做出预算、策略和方向决策。

## 适用场景

- 周度市场趋势报告：行业动态、竞品变化、趋势机会总览
- 月度ROI分析报告：全渠道投资回报分析、预算优化建议
- 季度策略评审报告：策略执行回顾、目标达成分析、下季规划建议
- 专题分析报告：大促复盘、新渠道试水评估、品牌健康度分析
- 实时决策brief：需要快速决策时的精炼数据支撑

## 执行步骤

### 步骤1：明确报告目的和受众
- 确认报告类型（周报/月报/专题）和决策场景
- 确认受众层级（CMO级/营销总监级/运营经理级）→ 决定抽象程度
  - CMO级：战略方向+关键数字+决策选项
  - 总监级：策略表现+优化方向+资源需求
  - 经理级：执行细节+操作建议+排期安排

### 步骤2：数据整合与交叉分析
- 从市场情报Agent获取：竞品动态、行业趋势、爆款预测
- 从效果归因Agent获取：各渠道归因ROAS、转化路径分析、异常标记
- 从实时数据获取：最新投放效果、预算消耗情况、核心指标趋势
- 交叉关联分析：
  - 竞品策略变化 × 自身效果波动 → 识别竞争影响
  - 趋势方向 × 创意表现 → 识别内容优化机会
  - 归因结果 × 预算分配 → 量化优化空间

### 步骤3：洞察提炼与建议生成
- 从数据中提炼3-5条核心洞察（不超过5条，聚焦最高价值）
- 每条洞察转化为可执行建议：
  - 建议内容：具体做什么（渠道、幅度、方向）
  - 预期收益：执行后预计带来的改善（量化）
  - 风险评估：不执行的潜在损失 + 执行的风险点
  - 优先级和时间线：建议执行顺序和窗口期
- 当存在多个可行方案时，提供选项对比矩阵

### 步骤4：报告结构化输出
- **标准三段式结构**：
  1. 数据摘要（Executive Summary）：3-5个关键数字+一段话总结
  2. 关键洞察（Key Insights）：3-5条洞察+数据支撑+图表描述
  3. 建议行动项（Recommended Actions）：按优先级排列+预期收益+风险
- **格式要求**：
  - 关键数字加粗或高亮
  - 正面变化用绿色标注，负面变化用红色标注
  - 每个建议附带"立即行动/近期规划/长期关注"的时间标签
  - 报告总长度控制：周报2-3页，月报5-8页

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| report_type | Enum | 是 | 报告类型（weekly_trend/monthly_roi/quarterly_review/ad_hoc） |
| audience_level | Enum | 是 | 受众层级（cmo/director/manager） |
| time_range | String | 是 | 报告覆盖时间范围 |
| focus_areas | Array | 否 | 重点关注领域（如特定渠道、特定活动） |
| market_intelligence_data | Object | 是 | 市场情报数据 |
| attribution_data | Object | 是 | 归因分析数据 |
| performance_data | Object | 是 | 实时效果数据 |
| previous_report | Object | 否 | 上期报告（用于趋势对比） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| report_title | String | 报告标题 |
| executive_summary | String | 核心摘要（200字内） |
| key_metrics | Object | 关键指标仪表板 |
| insights | Array | 核心洞察列表（3-5条） |
| insights[].title | String | 洞察标题 |
| insights[].description | String | 洞察详细描述+数据支撑 |
| insights[].impact | String | 业务影响评估 |
| recommendations | Array | 建议行动项 |
| recommendations[].action | String | 具体操作建议 |
| recommendations[].expected_outcome | String | 预期收益（量化） |
| recommendations[].timeline | Enum | 时间标签（immediate/short_term/long_term） |
| recommendations[].risk | String | 风险评估 |
| recommendations[].priority | Number | 优先级排序 |
| appendix | Object | 附录（详细数据表、方法论说明） |

## 最佳实践

1. **结论前置**：最重要的洞察和建议放在报告最前面，忙碌的决策者可能只看前半页
2. **So What原则**：每个数据点都要回答"So What?"——对业务意味着什么、应该做什么
3. **对比基准明确**：所有"好"或"差"的判断必须标明对比基准（目标值/行业均值/历史同期）
4. **不确定性透明**：当建议基于不完全信息时，明确标注不确定性和假设前提
5. **行动闭环追踪**：每期报告回顾上期建议的执行情况和效果，形成PDCA循环

## 约束条件

- 周报交付时间：每周一10:00前；月报交付时间：每月5日前
- 洞察数量控制在3-5条，避免信息过载导致决策瘫痪
- 建议必须具备可执行性——包含具体的执行主体、操作内容和量化目标
- 数据标注要求：所有数据点标注来源、时间范围和口径
- 不做超出数据支撑的推断；推测性内容必须标注为"假设"或"待验证"
