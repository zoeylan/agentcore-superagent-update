# 质量指标分析 (Quality Metrics Analysis)

## 概述
对批量生成素材的质量数据进行统计分析，维护抽卡合格率看板，识别系统性质量问题趋势，并生成优化建议。该技能是质量闭环控制的数据基础，通过数据驱动的方式持续提升生产质量。

## 适用场景
- 实时监控批次抽卡合格率
- 识别一致性评分持续偏低的角色/场景
- 分析不同模型/参数配置的质量差异
- 生成质量趋势报告和优化建议

## 执行步骤

### Step 1: 数据收集与聚合
- 收集所有质检评分数据（一致性/质量/合规/情绪匹配）
- 按批次、角色、场景、模型等维度进行聚合
- 计算各维度的平均值、中位数、标准差
- 识别异常值（超出2σ的数据点）

### Step 2: 抽卡合格率计算
- 总体合格率 = 首次通过素材数 / 总生成素材数
- 分角色合格率：识别哪些角色的通过率偏低
- 分场景合格率：识别哪些场景类型容易失败
- 分模型合格率：比较不同模型的质量表现
- 时序分析：合格率是否有恶化趋势

### Step 3: 问题模式识别
通过数据分析识别系统性问题：
- **角色维度**：某角色一致性始终偏低→可能需要优化参数库
- **场景维度**：某类场景（如夜景）质量偏低→可能需要调整光照描述
- **模型维度**：某模型的人物生成质量下降→可能模型版本更新导致
- **时段维度**：特定时段质量下降→可能是API负载导致

### Step 4: 根因分析
对于识别出的系统性问题，进行根因分析：
- 一致性问题：对比高分和低分样本的Prompt差异
- 质量问题：分析低分样本的共性特征
- 合规问题：统计高频违规类型
- 生成关联分析（模型×参数×Prompt结构→质量得分）

### Step 5: 优化建议生成
基于分析结果生成可操作的优化建议：
- 针对特定角色的Prompt优化方向
- 针对特定场景类型的参数调整建议
- 模型选型的调整建议
- 流程优化建议（如增加参考图环节）

### Step 6: 报告输出与预警
- 生成批次质量报告
- 当合格率<80%时触发预警
- 当连续3批次某角色一致性<0.85时触发专项告警
- 将优化建议推送给分镜导演和素材生成引擎

## 输入规范
```json
{
  "batch_id": "批次ID",
  "quality_records": [
    {
      "asset_id": "素材ID",
      "shot_ref": "镜头引用",
      "character": "角色",
      "model_used": "使用的模型",
      "scores": {
        "consistency": 0.87,
        "quality": 0.82,

        "emotion_match": 0.90

      "verdict": "pass|fail",
      "issues": []
    }
  ],
  "historical_data": "历史批次数据（用于趋势分析）"
}
```

## 最佳实践
1. **实时性**：合格率监控应实时更新，不要等批次结束再统计
2. **趋势优于单点**：关注趋势变化比关注单个数据点更有价值
3. **可操作性**：优化建议必须具体到可执行的动作
4. **闭环验证**：优化实施后需跟踪效果，验证是否真正改善
5. **基线维护**：维护历史基线数据，用于异常检测和趋势对比

## 约束条件
- 合格率计算必须客观、不人为调整阈值
- 告警不可关闭或静默（除非问题已确认解决）
- 分析报告需附原始数据支撑，不做无数据支持的推断
- 优化建议不能直接修改参数库（需由分镜导演确认执行）
