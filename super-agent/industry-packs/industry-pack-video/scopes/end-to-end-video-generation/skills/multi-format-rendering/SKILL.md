# 多格式渲染输出 (Multi-Format Rendering)

## 概述
将组装完成的时间线和音频混缩按照目标平台的技术规格要求进行最终渲染，输出多版本可直接发布的成品视频文件。该技能需要熟悉各大短视频平台的编码参数、分辨率标准和文件限制，确保输出文件100%符合平台技术要求。

## 适用场景
- 完成合成后的多平台版本渲染
- 单一平台的规格渲染
- 渲染参数调优（画质vs文件大小平衡）
- 渲染失败后的问题诊断和重渲

## 执行步骤

### Step 1: 平台规格确认
确认目标平台的技术要求：

| 平台 | 分辨率 | 画幅 | 编码 | 最大码率 | 最大文件 | 最大时长 |
|------|--------|------|------|----------|----------|----------|
| 抖音 | 1080x1920 | 9:16 | H.264 | 15Mbps | 128MB | 15分钟 |
| 快手 | 1080x1920 | 9:16 | H.264 | 12Mbps | 100MB | 10分钟 |
| YouTube Shorts | 1080x1920 | 9:16 | H.264/H.265 | 20Mbps | 256MB | 60秒 |
| 横版通用 | 1920x1080 | 16:9 | H.264 | 12Mbps | 128MB | - |
| 720p备用 | 720x1280 | 9:16 | H.264 | 8Mbps | 50MB | - |

### Step 2: 渲染参数配置
为每个目标版本配置渲染参数：
- **编码器**: H.264（兼容性最好）/ H.265（压缩率更高）
- **码率模式**: VBR（可变码率），设置目标码率和峰值码率
- **帧率**: 24fps（电影感）/ 30fps（流畅感）
- **色彩空间**: BT.709 / sRGB
- **音频编码**: AAC-LC, 128-256kbps, 44.1kHz
- **关键帧间隔**: 2秒（用于平台快速跳转）

### Step 3: 画幅适配处理
- **竖版9:16**: 原生竖版素材直接输出
- **横版16:9→竖版**: 智能裁切（人脸居中）或添加模糊背景上下填充
- **竖版→横版**: 添加左右虚化填充或内容重排
- 确保裁切后主体仍在画面安全区内

### Step 4: 渲染执行
- 启动渲染引擎处理
- 监控渲染进度和资源占用
- 处理渲染中的异常（内存不足/磁盘空间/编码错误）
- 多版本可并行渲染提高效率

### Step 5: 质量校验
渲染完成后执行自动校验：
- **文件完整性**: 视频可正常播放到最后一帧
- **画面检查**: 无黑帧（连续>2帧）、无花屏、无绿屏
- **音频检查**: 无爆裂、无静音段（除有意设计外）、无音画偏移
- **规格检查**: 分辨率/码率/时长/文件大小均符合平台要求
- **首帧检查**: 首帧内容正常（部分编码器首帧可能异常）

### Step 6: 输出与交付
- 按标准命名规范输出文件
- 生成渲染报告（含技术参数和质量校验结果）
- 自动生成封面帧（从视频中提取或单独设计）
- 打包所有平台版本为发布包

## 输入规范
```json
{
  "timeline_file": "时间线文件路径",
  "audio_mix_file": "混缩音频文件路径",
  "target_platforms": ["douyin", "kuaishou", "youtube_shorts"],
  "additional_versions": ["720p", "horizontal"],
  "project_name": "项目名称",
  "version": "v1"
}
```

## 最佳实践
1. **先快后精**：先渲染一个低码率预览版确认内容无误，再渲染高质量最终版
2. **平台优先级**：主要发布平台优先渲染，次要平台可后续补渲
3. **文件大小控制**：通过调整码率确保文件大小在平台限制的90%以内
4. **封面帧选择**：选择视觉冲击力最强的帧作为封面（通常是情绪高潮画面）
5. **保留工程文件**：渲染后保留工程文件，便于后续修改和重渲

## 约束条件
- 输出文件必须100%符合目标平台技术规格
- 渲染后文件必须可正常播放完整时长
- 不允许输出有质量缺陷的文件（宁可报错也不交付坏文件）
- 文件命名必须遵循统一规范
- 渲染超时上限：2分钟视频渲染不超过5分钟
