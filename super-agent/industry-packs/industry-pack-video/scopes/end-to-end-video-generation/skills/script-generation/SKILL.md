# 剧本生成 (Script Generation)

## 概述
将非结构化文本素材（小说章节、IP素材、创意brief）转化为符合短视频平台标准的结构化剧本。该技能是端到端视频生成流水线的起始环节，其输出质量直接影响整条生产链路的效率和最终成片品质。

## 适用场景
- 网文/小说章节改编为短剧剧本
- 创意brief扩写为完整剧本
- 已有剧本的平台适配改写（如从横屏剧改为竖版短剧）
- IP素材的视频化脚本设计

## 执行步骤

### Step 1: 输入文本分析
- 识别文本类型（小说/剧本/brief/其他）
- 提取核心故事元素：主要人物、冲突、情节走向
- 评估内容复杂度，确定需要拆分为多少集/段
- 识别可能的合规风险点

### Step 2: 结构化拆解
- 按2分钟一集的标准时长进行内容拆分
- 为每集设计独立的叙事弧线（起承转合）
- 确保每集有独立的钩子点和悬念收尾
- 标记集与集之间的情节衔接点

### Step 3: 场次设计
- 每集拆分为4-8个场次（scene）
- 每个场次包含：场景位置、在场人物、核心事件
- 时长分配：开场钩子（3-5秒）、主体叙事（占比70%）、结尾悬念（10-15秒）
- 情绪曲线规划：确保每集有至少1个情绪高潮

### Step 4: 对白创作
- 为每个场次编写对白台词
- 对白需符合角色人设（年龄/身份/性格）
- 控制对白密度：2分钟视频8-15句有效对白
- 为每句对白标注情绪（喜/怒/哀/惧/惊/平）
- 标记旁白/独白/对话的类型

### Step 5: 节奏与钩子标注
- 标记前3秒钩子的具体内容（悬念/冲突/视觉冲击）
- 每15秒设置一个次级钩子点
- 标注情绪转折点和高潮点
- 设计片尾悬念（吸引观众看下一集）

### Step 6: 输出格式化
- 以JSON格式输出结构化剧本
- 包含所有必要字段（见输出规范）
- 附加剧本概要信息（总时长、场次数、角色数）

## 输入规范
```json
{
  "source_text": "原始文本内容",
  "source_type": "novel|screenplay|brief|ip_material",
  "target_platform": "douyin|kuaishou|youtube_shorts",
  "target_duration": "120秒（默认）",
  "episode_count": "需要拆分的集数（可选）",
  "style_preference": "风格偏好（如：甜宠/悬疑/都市/古风）",
  "special_requirements": "特殊要求（可选）"
}
```

## 最佳实践
1. **前3秒法则**：开场必须在3秒内建立冲突或悬念，切忌缓慢铺垫
2. **对白黄金比例**：对白与画面叙事的比例约为4:6，避免"话剧化"
3. **情绪过山车**：单集内情绪不应平铺，至少包含2次明显的情绪起伏
4. **角色辨识度**：每个角色的台词风格应有明显差异化
5. **平台差异化**：抖音偏好强冲突开场，快手偏好情感共鸣，YouTube Shorts偏好视觉冲击

## 约束条件
- 单集时长严格控制在目标±10%以内
- 不得包含任何平台违禁内容
- 对白中不得出现现实品牌名称（除非IP授权）
- 角色数量控制在2-4个（2分钟短剧的认知负荷上限）
- 场景数量控制在2-4个（减少AI生成的环境切换复杂度）
