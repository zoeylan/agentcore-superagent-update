# 分镜拆解 (Storyboard Decomposition)

## 概述
将结构化剧本拆解为可直接驱动AI生成模型的分镜脚本（Storyboard），为每个镜头（Shot）定义精确的视觉参数，包括景别、运镜、角度、环境描述、人物布局和情绪基调。分镜拆解的质量决定了后续AI素材生成的效率和一致性。

## 适用场景
- 新剧本的首次分镜拆解
- 质检反馈后的分镜参数调整
- 分镜脚本的精细化迭代优化
- 单场景多角度分镜方案设计

## 执行步骤

### Step 1: 剧本场次分析
- 逐场次分析剧本内容，识别每个场次的核心视觉需求
- 确定每个场次需要拆分为多少个镜头（通常3-8秒一个镜头）
- 标记场次间的转场需求（硬切/淡入/叠化等）

### Step 2: 镜头设计
为每个镜头定义以下参数：
- **景别选择**：特写（CU）→强调情绪/细节；中景（MS）→对话/互动；全景（WS）→建立空间关系；远景（LS）→环境展示
- **运镜方式**：固定→稳定叙事；推（Dolly In）→聚焦/紧张；拉（Dolly Out）→揭示/释放；摇（Pan）→跟随/展示；移（Track）→跟随动作
- **拍摄角度**：平视→平等/自然；俯拍→压迫/渺小；仰拍→崇敬/威胁
- **构图规则**：三分法/中心构图/对角线/框架式

### Step 3: 场景环境描述
- 环境类型（室内/室外/特定场所）
- 时间设定（白天/黄昏/夜晚）
- 天气/光照条件
- 环境色调基准（暖/冷/中性）
- 关键环境道具/元素

### Step 4: 人物布局设计
- 每个镜头中出现的角色
- 角色在画面中的位置（左/中/右，前/后景）
- 角色的姿态和表情需求
- 角色间的空间关系（亲密/对立/平行）

### Step 5: 时序编排
- 每个镜头的持续时长（精确到0.5秒）
- 镜头间的衔接关系
- 配音/旁白与画面的对齐时间点
- 节奏密度（快节奏场景镜头切换频率高，慢节奏场景镜头持续时间长）

### Step 6: 输出分镜脚本
- 按照标准格式输出完整分镜脚本
- 每个镜头必须有唯一ID便于追踪
- 关联剧本场次ID便于溯源

## 输入规范
```json
{
  "script": "完整的结构化剧本JSON",
  "style_reference": "视觉风格参考（如：真人实拍风/动漫风/3D渲染风）",
  "aspect_ratio": "9:16|16:9",
  "character_profiles": "已有角色参数库（如果存在）"
}
```

## 最佳实践
1. **镜头语言服务叙事**：景别和运镜选择必须与情感表达匹配，不要为了花哨而堆砌技巧
2. **AI可执行性**：分镜设计需考虑当前AI模型的实际生成能力，避免设计无法实现的镜头
3. **一致性优先**：相邻镜头的光照/色调/环境需要保持连贯
4. **节奏变化**：避免连续使用相同景别，通过景别切换创造视觉节奏感
5. **时长精确**：镜头时长需精确规划，确保总时长与剧本预设吻合

## 约束条件
- 单个镜头时长建议3-8秒（AI视频生成模型的最佳输出时长）
- 同一场景内的镜头数量不超过5个（控制生成复杂度）
- 每个镜头必须有明确的情绪基调标注（供后续Prompt生成使用）
- 镜头运动不应过于复杂（当前AI视频模型对复杂运镜支持有限）
