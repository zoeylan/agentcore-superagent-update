# 多语种翻译 (Multilingual Translation)

## 技能概述

多语种翻译技能负责将视频中提取的文本内容（ASR转写文本和OCR识别文本）翻译为目标语种，是多语种视频分发流程的核心基础能力。该技能不仅执行语言层面的转换，还需处理语种混合识别、时间轴保持、情感标注传递等视频翻译特有的技术挑战。

## 适用场景

- 视频对白/旁白的多语种翻译
- 视频画面中文字（标题/字幕/水印文字）的翻译
- 语种混合内容（如中英混杂对白）的分段翻译
- 批量视频的规模化翻译任务

## 执行步骤

### 步骤1：源文本预处理
1. 接收ASR转写结果（含时间戳、说话人标识、置信度）
2. 接收OCR识别结果（含帧位置、文字区域坐标）
3. 执行源语种确认（验证ASR识别的语种是否正确）
4. 检测语种混合情况（标记不同语种的段落边界）
5. 对低置信度段落标记为"需人工校验"

### 步骤2：翻译执行
1. 加载目标语种对应的翻译模型/API
2. 加载自定义术语表（如有）
3. 逐段执行翻译，保持原始时间戳映射
4. 对每段翻译结果进行术语表强制匹配校验
5. 记录翻译文本长度与原文的比率（用于后续语速评估）

### 步骤3：情感标注传递
1. 解析原始文本的情感标注（来自ASR情感分析）
2. 将情感标签映射到翻译后的对应文本段
3. 添加断句标记（基于目标语种的韵律规则）
4. 标注重音和停顿位置（用于配音合成参考）

### 步骤4：质量自检
1. 验证所有段落均已翻译（无遗漏）
2. 检查术语表命中率是否>=90%
3. 检查翻译文本长度比率是否在合理范围内
4. 生成翻译质量自评报告

## 输入规格

```json
{
  "source_text": [
    {
      "segment_id": "seg_001",
      "text": "原始文本内容",
      "start_time": 0.0,
      "end_time": 3.5,
      "speaker": "speaker_1",
      "source_lang": "zh-CN",
```

## 最佳实践

1. **语速预判**：翻译时主动关注文本长度比率，若日语翻译超出原文130%或英语翻译超出原文80%时主动标记语速风险
2. **上下文连贯**：同一视频内的翻译需保持人称/称谓/语气的一致性
3. **术语优先**：术语表中的条目优先级高于机器翻译模型的默认翻译
4. **时间轴完整性**：任何情况下都不能丢失时间戳映射关系

## 约束与限制

- 不支持的语种必须立即报错，不得使用近似语种替代
- 单次翻译任务的段落数上限为10000段
- 翻译结果必须为UTF-8编码
- 不得自行修改原始时间戳信息
