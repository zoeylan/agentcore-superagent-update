# 口型匹配对齐 (Lip-Sync Alignment)

## 技能概述

口型匹配对齐技能负责将AI配音的音素序列与视频画面中人物的口型动作进行对齐，使观众在观看配音视频时不会产生明显的音画不同步感。该技能通过Viseme（视觉音素）映射和时间轴微调，实现配音与口型的自然协调，目标自然度>=0.8。

## 适用场景

- 人物对话特写镜头的口型配音对齐
- 多角色同框场景的分离式口型匹配
- 不同语种间的音素-唇形映射转换
- 口型匹配质量评估与参数调优

## 执行步骤

### 步骤1：人脸关键点提取
1. 对视频帧进行人脸检测和关键点定位
2. 提取口型相关的关键点序列（嘴唇外轮廓+内轮廓，共20个关键点）
3. 按帧率生成口型关键点时间序列
4. 处理多人场景（通过说话人检测确定当前发言者）
5. 标记无人脸/人脸被遮挡/侧脸的帧（这些帧跳过口型匹配）

### 步骤2：Viseme序列生成
1. 对配音音频进行音素分解（Phoneme Segmentation）
2. 将音素序列映射为Viseme序列（基于IPA→Viseme映射表）
3. 计算每个Viseme的持续时间和时间戳
4. 不同语种使用不同的Viseme映射规则（如英语约13个基本Viseme，日语约10个）

### 步骤3：时间轴对齐
1. 将配音的Viseme时间序列与视频的口型关键点时间序列对齐
2. 使用动态时间规整（DTW）算法找到最优对齐路径
3. 对配音音频进行微时移（<200ms）以优化匹配
4. 若需要较大时移（>200ms），则需要调整音频韵律而非简单位移

### 步骤4：匹配质量评估
1. 计算口型自然度评分（配音Viseme与实际口型的重合度）
2. 评估音画延迟（目标<200ms）
3. 计算整体自然度得分（加权平均，目标>=0.8）
4. 对低分片段生成具体改进建议（如"第2.1s-2.5s处开口时机偏晚200ms"）

    "segment_id": "seg_001",
    "fps": 30,

## 最佳实践

1. **特写优先**：面部特写镜头对口型匹配要求最高（自然度需>=0.85），中远景可适当放宽
2. **无人脸兜底**：无人脸的片段直接跳过口型匹配，仅保证语速和情感匹配即可
3. **关键音素对齐**：开口（AA/AE）和闭口（M/B/P）是最容易被察觉的口型点，优先对齐
4. **避免过拟合**：不追求100%匹配（会导致音频不自然），80%以上即为可接受

## 约束与限制

- 音频时移不得超过200ms（超出会被明显感知）
- 无人脸关键点数据时跳过口型匹配（不伪造数据）
- 口型自然度评分低于0.6时标记为失败需人工干预
- 不修改视频画面（不使用deepfake技术改变口型）
