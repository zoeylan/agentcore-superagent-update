# 韵律匹配 (Prosody Matching)

## 技能概述

韵律匹配技能负责确保AI配音的韵律特征（语速、节奏、停顿、重音、语调起伏）与原始视频的节奏和情感表达保持一致。该技能解决的核心问题是：如何让跨语种配音听起来"像是为这个画面量身定做的"，而非简单的文本朗读。

## 适用场景

- 配音与原始视频的语速节奏对齐
- 情感语调的跨语种保持（原声悲伤→配音也要悲伤）
- 多角色对话场景的节奏保持（问答的停顿和节奏感）
- 语速冲突的诊断与处理方案生成

## 执行步骤

### 步骤1：原声韵律分析
1. 对原始音频提取韵律特征：pitch contour（基频曲线）、energy envelope（能量包络）、speaking rate（语速）、pause pattern（停顿模式）
2. 识别情感段落边界（情感转变点）
3. 标记关键韵律锚点（强调词、情感高峰、长停顿）
4. 生成原声韵律模板（时间轴→韵律参数的映射）

### 步骤2：目标韵律规划
1. 根据原声韵律模板，结合目标语种特征，规划配音的韵律参数
2. 处理语种间的韵律差异（如中文是声调语言、英语是重音语言、日语是音高语言）
3. 计算每个段落的韵律参数目标值
4. 对时长约束和韵律目标进行优先级排序（时长优先，韵律在时长约束内最优化）

### 步骤3：韵律参数注入
1. 将规划的韵律参数转化为SSML标记或TTS引擎的控制参数
2. 设置段落级参数（整体语速/音量/情感等级）
3. 设置词级参数（个别词的强调/弱读/延长）
4. 设置停顿参数（在特定断句标记处插入指定时长的静音）

### 步骤4：韵律质量评估
1. 合成后对比配音与原声的韵律相似度
2. 计算语速匹配度（目标偏差<15%）
3. 计算情感一致性评分（目标>=0.8）
4. 评估节奏自然度（是否有不自然的加速/减速/断续）
5. 不达标则调整参数重新合成（最多3次迭代）

## 输入规格

```json
{
  "original_audio_features": {
    "segment_id": "seg_001",
    "pitch_contour": [...],
    "energy_envelope": [...],
    "speaking_rate": 4.2,
    "pause_positions": [1.2, 2.8],
    "emotion_segments": [
      {"start": 0.0, "end": 1.8, "emotion": "neutral"},
      {"start": 1.8, "end": 3.5, "emotion": "excited"}
    ]
  },
  "target_text": "配音文本",
  "target_lang": "en",
  "duration_constraint": 3.5
}
```

## 输出规格

```json
{
  "prosody_params": {
    "segment_id": "seg_001",
    "global_rate": 1.05,
    "global_pitch_shift": -0.5,
    "emotion_sequence": ["neutral", "excited"],
    "word_level_params": [
      {"word": "amazing", "emphasis": 1.3, "duration_stretch": 1.1}
    ],
    "pause_insertions": [
      {"position_after_word": 5, "duration_ms": 300}
    ]
  },
  "matching_scores": {
    "rate_match": 0.92,
    "emotion_match": 0.85,
    "rhythm_naturalness": 0.88,
    "overall": 0.88
  }
}
```

## 最佳实践

1. **韵律优先级**：情感匹配 > 停顿位置 > 语速精确度 > 基频匹配
2. **自然度底线**：宁可语速略有偏差也不能产生不自然的语音（如机械加速）
3. **多轮优化**：韵律参数通常需要2-3次迭代才能达到最优
4. **角色区分**：不同角色的韵律特征应有明显区分（如老年人语速慢、孩子语调高）

## 约束与限制

- 语速调整范围不得超过原始语速的±30%（超出则需反馈翻译Agent）
- 情感一致性评分低于0.6时必须重新合成
- 韵律参数调整迭代次数上限为3次
- 不得为了匹配韵律而改变翻译文本内容
