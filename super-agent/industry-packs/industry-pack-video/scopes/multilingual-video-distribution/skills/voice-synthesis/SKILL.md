# 语音合成 (Voice Synthesis)

## 技能概述

语音合成技能负责将翻译后的多语种文本转化为自然流畅的AI配音音频。该技能是多语种视频分发中配音环节的核心能力，需要在保证语音自然度（MOS>=3.8）的同时，精确控制语速、情感和韵律参数，以匹配原始视频的时序约束和情感基调。

## 适用场景

- 多语种视频配音的批量生成
- 不同角色（性别/年龄/口音）的音色分配和合成
- 情感化语音合成（喜/怒/哀/乐/恐惧/惊讶等）
- 语速约束下的自适应语音生成

## 执行步骤

### 步骤1：合成参数规划
1. 接收翻译文本（含情感标注、断句标记、时长约束）
2. 为每个片段计算目标语速（目标文本字数/原始片段时长）
3. 评估目标语速是否在自然范围内（各语种有不同的自然语速区间）
4. 若目标语速超出自然范围15%以上，标记为"语速冲突"并准备反馈信息
5. 根据角色设定选择音色库（性别/年龄段/口音特征）

### 步骤2：TTS引擎调用
1. 选择目标语种对应的最优TTS引擎
2. 设置合成参数：pitch（基频）、rate（语速）、volume（音量）、emotion（情感）
3. 传入断句标记和重音标注以控制韵律
4. 执行语音合成，获取原始音频波形
5. 若TTS引擎不可用，立即告警并记录（不使用低质量替代）

### 步骤3：语速自适应调节
1. 测量合成音频的实际时长
2. 计算与目标时长的偏差百分比
3. 若偏差在±5%内，直接通过
4. 若偏差在5%-15%，通过SSML的rate参数微调语速
5. 若偏差>15%，判定为语速冲突，生成文本精简反馈

### 步骤4：质量评估
1. 执行MOS评分估算（基于模型的无参考语音质量评估）
2. 检测合成异常：爆音/杂音/断续/机械感
3. 验证情感一致性（合成语音的情感与标注是否匹配）
4. 不达标（MOS<3.5）的片段标记为失败，触发重试

## 输入规格

```json
{
  "segments": [
    {
      "segment_id": "seg_001",
      "text": "翻译后的文本",
      "target_lang": "en",
      "duration_constraint": 3.5,
      "emotion": "happy",
      "speaker_profile": {
        "gender": "female",
        "age_group": "young_adult",
        "accent": "american"
      },
      "break_marks": [1.2, 2.8],
      "emphasis_words": ["important", "amazing"]
    }
  ],
  "global_settings": {
    "audio_format": "wav",
    "sample_rate": 44100,
    "bit_depth": 16
  }
}
```

## 最佳实践

1. **语速安全区间**：各语种的自然语速范围不同——中文约4-6字/秒，英语约3-5词/秒，日语约5-8音节/秒
2. **情感渐变**：长句中情感可能变化，需按断句标记分段处理而非整句统一情感
3. **音色一致性**：同一角色在整个视频中必须使用相同的音色配置
4. **失败快速重试**：首次合成失败后，先调整参数重试一次再上报

## 约束与限制

- MOS评分低于3.5的音频绝不允许输出
- TTS引擎不可用时必须告警，不得静默跳过
- 单次合成任务超时阈值为30秒/段
- 音频输出格式必须为无损WAV（用于后续处理）或高质量MP3（用于最终分发）
