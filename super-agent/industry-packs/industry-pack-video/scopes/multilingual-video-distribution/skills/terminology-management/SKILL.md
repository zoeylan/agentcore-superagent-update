# 术语管理 (Terminology Management)

## 技能概述

术语管理技能负责维护和应用自定义术语表，确保多语种翻译过程中专业术语、品牌名称、产品名和角色名等关键词汇在所有输出版本中保持一致性。这是保障翻译专业度和品牌辨识度的关键能力，目标术语准确率>=90%。

## 适用场景

- 品牌名称和产品名的多语种统一翻译规范管理
- 行业专业术语的标准化翻译对照表维护
- 短剧角色名/地名的音译一致性保障
- 跨视频/跨批次翻译的术语一致性监控

## 执行步骤

### 步骤1：术语表加载与验证
1. 加载项目关联的自定义术语表
2. 验证术语表格式完整性（源语种→目标语种映射是否齐全）
3. 检测术语冲突（同一源词对应多个目标译法）
4. 生成术语表覆盖度报告（哪些目标语种有术语对照，哪些缺失）

### 步骤2：术语强制匹配
1. 在翻译执行阶段，对每个翻译段落执行术语表匹配
2. 匹配到的术语必须使用术语表中指定的译法（覆盖机器翻译默认结果）
3. 记录每次术语匹配的位置和应用情况
4. 对未匹配到预期术语的段落标记告警

### 步骤3：术语一致性校验
1. 翻译完成后，全局扫描同一术语在所有段落中的翻译一致性
2. 检测同一源词被翻译为不同目标词的情况（潜在不一致）
3. 计算术语准确率 = 正确应用术语数 / 应该应用术语数
4. 不一致项列入修正建议清单

### 步骤4：术语表更新建议
1. 识别翻译过程中出现的新术语（高频词汇但不在术语表中）
2. 为新术语提供建议翻译并推荐加入术语表
3. 标记可能需要更新的旧术语条目（如品牌更名等）
4. 输出术语表维护建议报告

## 输入规格

```json
{
  "terminology_table": {
    "table_id": "term_001",
    "entries": [
      {
        "source_term": "源术语",
        "translations": {
          "en": "English term",
          "ja": "日本語用語",
          "ar": "المصطلح العربي"
        },
        "context": "使用上下文说明",
        "category": "brand|product|character|technical"
      }
    ]
  },
  "translation_segments": [...],
  "target_languages": ["en", "ja", "ar"]
}
```

## 最佳实践

1. **品牌名优先级最高**：品牌名称的术语匹配优先级高于一般术语
2. **上下文消歧**：同一词在不同上下文可能对应不同术语条目，需基于上下文判断
3. **音译规范化**：角色名/地名的音译需遵循目标语种的音译规范（如中文"张伟"在日语中为"チョウ・ウェイ"而非"ちょうい"）
4. **增量学习**：每次项目完成后，将确认的新术语自动加入术语表

## 约束与限制

- 术语准确率必须>=90%，低于此阈值需触发人工审核
- 术语表条目一经确认不得由系统自动修改（只能建议更新）
- 不得跳过术语匹配步骤，即使对翻译速度有影响
- 术语表格式必须使用标准JSON Schema，不接受非结构化输入
