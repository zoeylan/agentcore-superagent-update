# 视觉内容分析 (Visual Content Analysis)

## 技能概述

视觉内容分析技能负责对视频帧进行深层次的视觉语义理解，包括场景识别、人物检测与追踪、动作识别和表情分析。该技能为下游的语义分割提供视觉维度的内容变化信号，为高光提取提供视觉冲击力的量化评估依据。

## 适用场景

- 视频场景切换检测（用于辅助语义分割）
- 人物出场统计和角色追踪（用于短剧角色分析）
- 动作场面识别（用于高光时刻判定）
- 人物表情变化检测（用于情绪高潮识别）

## 执行步骤

### 步骤1：场景识别
1. 对关键帧执行场景分类（室内/室外×具体场景类型）
2. 场景类型细分：卧室、客厅、办公室、街道、公园、餐厅等30+类别
3. 检测场景切换点（相邻帧的场景特征差异超过阈值）
4. 输出场景时间线（每个场景的起止时间和类型标签）

### 步骤2：人物检测与追踪
1. 人脸检测：定位帧中所有人脸及其边界框
2. 人脸特征提取：生成128维人脸嵌入向量
3. 跨帧关联（ReID）：通过特征相似度追踪同一人物跨帧出现
4. 人物身份建库：为视频中每个独立角色分配唯一ID
5. 生成人物出场时间线和屏占比统计

### 步骤3：动作识别
1. 对关键帧执行人体姿态估计（17关节点检测）
2. 基于时序姿态序列进行动作分类
3. 支持动作类型：站立/坐下/行走/奔跑/拥抱/打斗/跳舞/鼓掌等20+类别
4. 输出动作强度评分（0-1，用于高光识别参考）

### 步骤4：表情分析
1. 在人脸检测基础上执行表情识别
2. 7类表情分类：开心/悲伤/愤怒/惊讶/恐惧/厌恶/中性
3. 输出表情强度评分（微表情→标准表情→夸张表情）
4. 检测表情突变点（如从平静突然变为震惊）

### 步骤5：视觉特征聚合
1. 综合场景、人物、动作、表情四维信息
2. 计算每帧的视觉复杂度评分
3. 计算相邻帧的视觉变化率（用于检测画面高潮）
4. 输出完整的视觉分析时间线

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| video_file | File | 是 | 源视频文件 |
| analysis_level | Enum | 否 | full/scene_only/person_only（默认full） |
| person_db | Array | 否 | 预置人物特征库（用于已知角色匹配） |
| sampling_strategy | Enum | 否 | uniform/adaptive（默认adaptive） |

## 最佳实践

- 对于短剧内容，建议预先建立角色人脸库以提升跨分镜追踪准确率
- 自适应采样策略在动作场面密集时自动提高采样频率
- 场景切换检测的阈值应根据视频风格动态调整（快节奏剪辑vs长镜头）
- 表情分析对于侧脸和遮挡场景准确率较低，需在输出中标注置信度

## 约束与限制

- 人物追踪在频繁换装/化妆场景下可能产生ID切换
- 小于50×50像素的人脸无法进行有效的表情分析
- 动作识别要求至少5帧连续可见才能给出可靠分类
- GPU显存限制：单帧4K视频分析约需4GB显存
