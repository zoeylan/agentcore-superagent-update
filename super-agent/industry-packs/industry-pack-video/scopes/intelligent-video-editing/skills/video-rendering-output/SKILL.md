# 视频渲染输出 (Video Rendering Output)

## 技能概述

视频渲染输出技能负责将剪辑决策方案转化为最终的视频成片文件。包括视频片段拼接、转场特效渲染、音频混合、格式编码和多规格输出。该技能是整个剪辑流水线的最后环节，确保输出文件符合目标平台的技术规格要求。

## 适用场景

- 单条视频的最终渲染输出
- 批量素材的并行渲染
- 多平台多规格的矩阵式输出
- 人工微调后的重新渲染

## 执行步骤

### 步骤1：渲染任务解析
1. 解析剪辑决策文档（片段列表+转场指令+全局参数）
2. 确定输出规格矩阵：
   - 分辨率：1080p / 720p / 480p
   - 画幅比：16:9（横版）/ 9:16（竖版）/ 1:1（方形）
   - 格式：MP4(H.264) / MOV(ProRes) / WebM(VP9)
   - 码率：高质量/标准/低带宽
3. 预估渲染时间和存储空间需求
4. 分配渲染资源（GPU/CPU选择）

### 步骤2：视频轨道组装
1. 按剪辑决策的片段序列定位源视频中的对应片段
2. 精确到帧地裁切每个片段（关键帧对齐处理）
3. 按指定顺序拼接视频片段
4. 在拼接点渲染转场特效（按转场设计的参数执行）
5. 处理画幅适配（裁切/缩放/加黑边）

### 步骤3：音频轨道处理
1. 提取对应片段的音频
2. 执行音频转场处理（交叉淡入淡出）
3. 音量标准化（-14 LUFS for 流媒体 / -16 LUFS for 广播）
4. 添加背景音乐轨道（如有指定）
5. 音频与视频的精确同步对齐

### 步骤4：编码输出
1. 按目标格式执行视频编码
2. 嵌入元数据（标题/时间戳映射/创建时间）
3. 生成缩略图/封面帧
4. 执行输出文件质量校验：
   - 无黑帧检测
   - 无花屏检测
   - 音频无爆裂/无静音异常
   - 文件完整性验证
5. 生成技术质量报告

### 步骤5：时间戳映射表生成
1. 建立成片时间→原始时间的双向映射表
2. 精确到帧的映射关系
3. 标注每个片段在成片中的位置
4. 输出映射表文件（支持人工微调工具读取）

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| edit_decision_list | Object | 是 | 剪辑决策文档（片段+转场+参数） |
| source_video | File | 是 | 原始视频文件 |
| output_specs | Array | 是 | 输出规格列表（可多规格） |
| background_music | File | 否 | 背景音乐文件 |
| watermark | Object | 否 | 水印配置（位置/透明度/内容） |

## 最佳实践

- 抖音/快手推荐参数：9:16, 1080x1920, H.264, 码率8-15Mbps, AAC 128kbps
- YouTube Shorts推荐参数：9:16, 1080x1920, H.264, 码率10-20Mbps
- 渲染前应检查源视频的关键帧间距，非关键帧起切可能导致首帧花屏
- 批量渲染建议使用队列机制，避免GPU显存溢出
- 时间戳映射表对于人工微调流程至关重要，必须确保精确性

## 约束与限制

- 单次渲染的最大输出时长为2小时
- 不支持实时流式渲染（仅离线渲染）
- 复杂转场特效（3D/粒子）渲染耗时显著增加
- MOV/ProRes格式文件较大，需确保存储空间充足
- 4K输出需要专用GPU资源，普通任务不建议使用
