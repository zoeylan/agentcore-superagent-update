# 语义边界检测 (Semantic Boundary Detection)

## 技能概述

语义边界检测技能负责在视频时间轴上精确定位内容语义的自然分割点。不同于基于转场特效或静音段落的机械检测，该技能通过融合多模态信号（话题转换、场景变化、说话人切换、情绪转折等）来判定语义边界，确保每个分割点都位于内容的自然断点处。

## 适用场景

- 长视频的内容级自动分段
- 直播回放中话题切换点的定位
- 短剧中剧情场次边界的识别
- 教程/演讲中论点切换的检测

## 执行步骤

### 步骤1：多信号边界候选生成
1. 文本语义信号：基于ASR转写文本的主题一致性计算，检测话题切换点（TextTiling算法或语义嵌入余弦相似度下降点）
2. 视觉场景信号：基于场景识别结果的场景切换点
3. 说话人信号：说话人ID变化点（对于对话类内容）
4. 音频信号：背景音乐变化点、较长静音段落（>2秒）
5. 情绪信号：情感向量的突变点（情绪色调明显转换）

### 步骤2：多信号融合与评分
1. 对每个候选边界点收集所有信号的证据强度
2. 应用加权融合：文本语义(0.35) + 视觉场景(0.25) + 说话人(0.15) + 音频(0.15) + 情绪(0.10)
3. 计算综合边界置信度分数（0-1）
4. 过滤置信度<0.5的弱候选

### 步骤3：边界精修与去冗余
1. 对相邻过近的候选进行合并（间距<3秒时保留置信度更高的）
2. 将边界点精修到最近的语义自然点（句子结束、动作完成、音乐节拍）
3. 确保最小分段时长>=3秒
4. 确保最大分段时长<=原视频30%

### 步骤4：边界质量验证
1. 检查每个边界两侧的内容是否属于不同语义主题
2. 计算跨边界的语义连贯性（应低于段内连贯性）
3. 对置信度<0.8的边界标记"建议人工确认"
4. 输出边界质量报告

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| semantic_graph | Object | 是 | 多模态分析师输出的视频语义图谱 |
| min_segment_duration | Integer | 否 | 最小分段时长（ms，默认3000） |
| max_segment_duration | Integer | 否 | 最大分段时长（ms，默认原视频30%） |
| signal_weights | Object | 否 | 自定义各信号权重 |

## 最佳实践

- 对于短剧内容，话题切换信号权重可适当降低，场景切换权重提高
- 直播类内容建议提高说话人切换信号的权重
- 当视频风格为快节奏剪辑时，应提高边界置信度阈值避免过度分割
- 建议先粗筛（阈值0.5）再精修，优于一步到位的高阈值方案

## 约束与限制

- 对于纯音乐MV类内容（无语音），文本语义信号不可用，需自动降权
- 长镜头且无对白的段落可能导致边界漏检
- 非线性叙事（闪回/平行剪辑）可能产生虚假边界
