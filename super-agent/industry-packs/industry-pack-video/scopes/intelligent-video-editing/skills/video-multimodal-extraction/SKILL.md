# 视频多模态信息提取 (Video Multimodal Extraction)

## 技能概述

视频多模态信息提取是智能视频剪辑系统的基础能力，负责协调和整合ASR、OCR、视觉分析、音频分析四大管线的输出结果，构建统一的视频语义图谱。该技能的核心价值在于将离散的多模态信号在时间轴上精确对齐，形成可供下游任务消费的结构化数据。

## 适用场景

- 新视频首次入库时的全量分析
- 批量视频的自动化预处理流水线
- 剪辑编排Agent请求的补充分析（细粒度模式）
- 跨Scope数据共享时的标准化输出

## 执行步骤

### 步骤1：视频预处理
1. 解析视频容器格式，提取元数据（时长、分辨率、帧率、编码格式、音轨数量）
2. 检测视频完整性（无损坏帧、音轨连续性）
3. 根据视频时长和分辨率预估资源需求，决定是否需要分片处理
4. 对于超过2小时的视频，自动切分为30分钟片段并行处理

### 步骤2：并行管线调度
1. 同时启动ASR、OCR、视觉分析、音频分析四条管线
2. 为每条管线分配计算资源配额
3. 监控各管线进度，记录耗时和资源消耗
4. 处理管线异常（超时重试、降级策略）

### 步骤3：时间轴对齐
1. 以毫秒为精度单位建立统一时间轴
2. 将各管线输出结果映射到统一时间槽（默认1秒/槽，细粒度模式100ms/槽）
3. 处理时间偏移校正（音画不同步校正）
4. 生成时间轴完整性报告（覆盖率、空白段标记）

### 步骤4：语义图谱构建
1. 将对齐后的多模态数据组装为结构化语义图谱
2. 计算跨模态一致性评分（如ASR情感与视觉表情是否一致）
3. 标记高置信度和低置信度区域
4. 输出最终语义图谱JSON文件

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| video_file | File | 是 | 支持MP4/MOV/AVI/MKV/WebM格式 |
| analysis_mode | Enum | 否 | standard(默认)/fine_grained/quick |
| time_range | Object | 否 | 指定分析的时间范围（用于补充分析） |
| language_hint | String | 否 | 预期语种提示（提升ASR首次准确率） |

## 最佳实践

- 对于短剧内容，建议使用standard模式即可满足需求
- 对于演讲/教学类单人内容，可跳过人物追踪以节省计算资源
- 批量任务应设置合理的并发上限（建议单节点不超过5个视频并行）
- 低置信度段落应保留原始数据而非丢弃，供后续人工校验

## 约束与限制

- 单次分析的视频时长上限为4小时
- 不支持加密/DRM保护的视频文件
- 音轨数量超过4条时仅处理前4条
- 视频分辨率低于360p时OCR准确率会显著下降
