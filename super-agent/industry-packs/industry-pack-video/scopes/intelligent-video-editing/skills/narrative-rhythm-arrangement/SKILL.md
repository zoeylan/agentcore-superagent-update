# 叙事节奏编排 (Narrative Rhythm Arrangement)

## 技能概述

叙事节奏编排技能负责确定选中片段的最终排列顺序和节奏控制。不是简单地按原始时间线拼接，而是根据内容用途和受众心理，设计最优的叙事节奏——包括开头钩子、中段节奏交替、结尾收束或悬念设置等。

## 适用场景

- 投放素材的节奏设计（强钩子+快节奏+悬念结尾）
- 精华集锦的节奏安排（多样性+张弛有度）
- 预告片的悬念递进设计
- 直播切片的独立叙事重组

## 执行步骤

### 步骤1：确定编排模板
1. 根据内容用途选择节奏模板：
   - 投放素材（15-60秒）：钩子(3s) → 冲突升级(60%) → 高潮/悬念截断(最后3s)
   - 精华集锦（3-10分钟）：精彩片段A → 过渡 → 精彩片段B → 高潮 → ...递进
   - 预告片（30-90秒）：悬念开场 → 节奏快切 → 高光密集 → 情感共鸣 → 悬念收尾
   - 完整拆条（时长不限）：保持原始叙事逻辑，仅做头尾裁切
2. 加载模板的节奏曲线目标

### 步骤2：片段排序优化
1. 确定开头片段（必须是高效钩子）
2. 确定结尾片段（投放素材用悬念截断，集锦用高潮收束）
3. 中间片段排序策略：
   - 按张力递进排列（适合投放素材）
   - 按时间线顺序排列（适合完整拆条）
   - 按类型交替排列（适合精华集锦）
4. 验证排列后的节奏曲线与目标模板的匹配度

### 步骤3：节奏微调
1. 计算相邻片段的节奏对比度（快→慢为放松，慢→快为加速）
2. 检测是否存在"节奏疲劳"（连续3个以上同节奏片段）
3. 如有节奏问题，插入/替换片段进行调整
4. 确保节奏曲线的整体走势符合模板要求

### 步骤4：节奏验证
1. 模拟观众注意力曲线（基于片段吸引力+节奏变化）
2. 检测注意力低谷点（观众可能离开的时刻）
3. 如有低谷且可优化，进行片段置换或压缩
4. 输出最终编排方案和预估注意力曲线

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| selected_fragments | Array | 是 | 智能内容选择的输出片段列表 |
| purpose | String | 是 | 用途类型（investment/highlight_reel/trailer/full_clip） |
| target_duration_ms | Integer | 是 | 目标时长 |
| rhythm_template | String | 否 | 指定节奏模板（默认自动选择） |

## 最佳实践

- 投放素材的前3秒注意力预估值必须>0.85，否则应更换钩子
- 精华集锦不宜连续放置3个以上同情绪色彩的片段
- 悬念截断型结尾应在情节最高张力点切割，不给出结局
- 音乐节奏点（Beat）是优秀的节奏锚点，可据此微调片段时长

## 约束与限制

- 节奏编排是高度主观的创意决策，自动化结果仅为参考建议
- 注意力曲线预测基于统计模型，个体差异大
- 非线性编排可能影响内容的可理解性，需与连贯性评分平衡
