# OCR视频帧文字识别 (OCR Frame Recognition)

## 技能概述

OCR视频帧文字识别技能负责从视频帧中检测和识别所有文字信息，包括硬字幕、标题卡、画中画文字、弹幕叠加文字等。通过智能关键帧采样策略，在保证覆盖率的同时控制计算成本。目标识别准确率>=90%。

## 适用场景

- 视频中硬字幕的提取（无SRT文件时的字幕恢复）
- 标题卡/片头片尾文字信息提取
- 画面内嵌文字识别（如PPT内容、白板文字、产品标签）
- 弹幕/水印文字检测与分类

## 执行步骤

### 步骤1：关键帧采样策略
1. 基础采样：每秒2帧均匀采样
2. 转场帧补充：在检测到场景切换的位置额外采样
3. 文字变化检测：相邻帧文字区域变化时触发额外采样
4. 去重：对连续帧中相同文字区域进行合并去重

### 步骤2：文字检测
1. 对采样帧执行文字区域检测（Text Detection）
2. 输出文字区域的边界框（Bounding Box）坐标
3. 分类文字区域类型：字幕区（底部/顶部）、标题区、画面内嵌、弹幕
4. 过滤极小区域和噪声（面积<100px²的区域）

### 步骤3：文字识别
1. 对检测到的文字区域执行OCR识别
2. 输出识别文本和置信度分数
3. 对低置信度结果（<0.7）进行二次识别（旋转校正、对比度增强后重试）
4. 执行后处理：语言模型校正、标点补全

### 步骤4：时间轴关联
1. 将帧级OCR结果映射到时间轴
2. 合并连续帧中相同位置的重复文字（取最高置信度版本）
3. 计算每段文字的显示时间范围（起始帧→消失帧）
4. 区分持续性文字（水印/标题）和动态文字（字幕/弹幕）

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| video_file | File | 是 | 源视频文件 |
| sampling_fps | Float | 否 | 采样帧率（默认2fps） |
| roi_regions | Array | 否 | 指定关注区域（如只看底部字幕区） |
| language_hint | String | 否 | 预期文字语种 |

## 最佳实践

- 对于纯字幕提取场景，可设置ROI区域仅关注画面底部20%区域，大幅提升效率
- 短剧内容通常字幕样式统一，可通过首帧样式检测自动配置后续帧的识别参数
- 弹幕文字对于内容分析价值有限，建议根据任务需求决定是否启用弹幕识别
- 深色背景上的浅色文字和浅色背景上的深色文字需要不同的预处理策略

## 约束与限制

- 手写体/艺术字体识别准确率约70-80%，低于印刷体
- 运动模糊严重的帧中文字识别率会显著下降
- 竖排文字需要启用特殊模式处理
- 单帧处理耗时约50-100ms，超长视频需注意总耗时预算
