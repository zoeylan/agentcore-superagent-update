# 智能视频剪辑与高光提取

## 技能描述
基于多模态AI能力，为用户设计智能视频剪辑方案，包括高光时刻自动提取、语义级场景分割、节奏自适应剪辑、以及内容二次创作的技术方案。

## 触发场景
- 用户有大量长视频素材需要快速产出短视频切片
- 用户需要从直播/会议/课程录像中提取精华片段
- 用户想实现批量化的视频二创（混剪、解说、反应视频等）
- 用户需要设计视频内容理解和自动剪辑的技术架构

## 输出规范

### 必须包含的内容
1. **内容理解方案**：如何对视频进行多模态分析（画面、语音、文字、情感）
2. **高光提取策略**：基于哪些信号判定"高光时刻"（情感峰值、信息密度、视觉冲击、节奏变化）
3. **分割方案设计**：场景级/语义级/镜头级分割的技术选择和精度权衡
4. **剪辑逻辑编排**：自动剪辑的决策规则（节奏模板、转场逻辑、时长控制）
5. **平台适配输出**：不同平台的画幅、时长、节奏偏好差异化处理

### 高光评分模型参考
```
高光分数 = w1×情感强度 + w2×信息密度 + w3×视觉变化度 + w4×互动预测值

权重参考（按内容类型）：
- 剧情类：情感0.4 + 信息0.2 + 视觉0.2 + 互动0.2
- 知识类：情感0.1 + 信息0.5 + 视觉0.1 + 互动0.3
- 娱乐类：情感0.3 + 信息0.1 + 视觉0.3 + 互动0.3
```

### 技术栈建议层级
1. **轻量方案**（适合中小团队）：FFmpeg + Whisper + GPT-4V，成本低但定制性有限
2. **中等方案**（适合专业团队）：自建多模态Pipeline + 开源模型微调，平衡成本与质量
3. **重型方案**（适合平台级）：端到端训练的视频理解模型 + 自研剪辑引擎

### 关键经验法则
- 黄金3秒法则：短视频前3秒必须有强hook，高光片段的开头需要重新设计
- 节奏密度：短视频的信息密度应为原片的2-3倍（去除冗余、加速过渡）
- 转场不超过0.5秒：短视频用户对转场耐心极低
- A/B测试覆盖率：每条视频至少出3个版本的开头进行测试

## 注意事项
- 版权风险提示：二创内容需注意原始素材的版权归属
- 高光提取的"准确率"不是唯一指标，"多样性"同样重要——避免所有切片风格雷同
- 自动剪辑的最终成片仍建议人工过审，尤其关注：跳切是否自然、音画是否同步、信息是否完整
- 不同品类的视频其"高光"定义完全不同，方案必须针对具体内容类型定制
