# VOC月度报告生成技能

## 概述
本技能负责每月自动生成VOC（Voice of Customer）综合分析报告，将当月的客户反馈数据、缺陷分析结论、改进措施进展和效果验证结果整合为一份面向管理层的决策支撑报告。报告需要兼具数据深度（专业分析）和表达简洁性（高管可快速抓住要点），是制造业持续改进PDCA循环中"Check"环节的核心输出。

## 适用场景
- 每月固定时间（次月第5个工作日前）生成上月VOC报告
- 管理层临时要求的VOC专项分析报告
- 季度/年度VOC汇总报告
- 客户评审/审核前的VOC数据准备

## 执行步骤

### 步骤1：数据汇总
从各数据源汇总本月数据：
- **反馈数据**：从反馈采集Agent获取——本月反馈总量、各渠道分布、分类分布
- **分析数据**：从缺陷模式分析Agent获取——TOP缺陷模式、关联分析结论
- **改进数据**：从改进跟踪看板获取——活跃任务状态、新增/关闭数量、逾期情况
- **验证数据**：从效果验证结果获取——本月完成验证的改善、达标率
- **对比数据**：同期历史数据（同比/环比计算）

### 步骤2：趋势分析
- 反馈数量趋势（近12个月折线图数据）
- 按缺陷类型的月度分布趋势（堆叠柱状图数据）
- 改善关闭率趋势
- 客户满意度相关指标趋势（如有）
- 标注趋势拐点和关键事件（如某产品上市/某批次召回等）

### 步骤3：报告结构组织

**第一部分：执行摘要（Executive Summary）**
- 本月核心数据：反馈总量/同环比变化/关键指标达成情况
- 3个最重要的发现/风险点
- 需要管理层关注或决策的事项
- 限制在1页以内

**第二部分：反馈概况**
- 各渠道采集量统计表
- 分类分布帕累托图
- 新增vs历史趋势对比
- 客户分布分析（哪些客户投诉最多）

**第三部分：TOP缺陷深度分析**
- 本月TOP5缺陷模式卡片（每个含：描述/影响/趋势/关联分析/当前改善状态）
- 新出现的缺陷模式预警
- 已知缺陷模式的变化情况

**第四部分：改进进展汇总**
- 改善措施执行看板概览
- 本月新增8D课题
- 本月关闭的改善及效果
- 逾期任务清单及原因
- 按时关闭率指标

**第五部分：效果验证结果**
- 本月完成验证的改善列表
- 达标/未达标/部分有效的比例
- 效果验证案例分享（选1-2个代表性成功案例）

**第六部分：下月展望与建议**
- 预计下月重点改善方向
- 需要跨部门协调的事项
- 资源/预算需求预估

### 步骤4：数据可视化
为报告准备图表数据：
- 帕累托图（缺陷类型分布）
- 趋势折线图（投诉数量月度走势）
- 雷达图（各KPI指标达成情况）
- 甘特图（关键改善任务进度）
- 热力图（缺陷×产品型号分布）

### 步骤5：报告审核与发布
- 数据一致性检查（各部分引用数据是否一致）
- 结论与数据的匹配性检查
- 敏感信息检查（客户名称脱敏处理等）
- 生成最终报告（PDF格式+在线版本）
- 发布通知发送给报告接收人列表

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| report_month | String | 是 | 报告月份（YYYY-MM格式）|
| feedback_summary | Object | 是 | 当月反馈汇总数据 |
| analysis_results | Object | 是 | 缺陷分析结果汇总 |
| improvement_status | Object | 是 | 改进措施状态汇总 |
| verification_results | Array | 否 | 效果验证结果列表 |
| distribution_list | Array | 是 | 报告分发对象列表 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| report_document | File | 生成的VOC报告文件 |
| chart_data | Object | 所有图表的数据集 |
| key_findings | Array | 关键发现摘要（用于邮件正文）|
| action_items | Array | 需要管理层关注的事项 |
| kpi_scorecard | Object | KPI指标达成情况计分卡 |

## 最佳实践
- 执行摘要是最重要的部分——高管可能只看这一页，确保关键信息在此呈现
- 数据可视化遵循"less is more"原则，每个图表传递一个核心信息
- 避免堆砌数据——报告要有分析和洞察，不是数据dump
- 对比数据（同比/环比/目标对比）比绝对数据更有业务意义
- 改善成功案例的分享可以增强组织的改善信心和文化
- 报告语言要客观中立，有问题直说，不回避不美化

## 约束条件
- 月度VOC报告必须在次月第5个工作日前完成发布
- 报告数据必须与源系统数据一致（可追溯验证）
- 按时生成率目标100%
- 报告中的客户名称需要评估是否需要脱敏（内部报告通常不需要，对外分享版本需要）
- 不得在报告中做出超越数据支撑范围的结论
