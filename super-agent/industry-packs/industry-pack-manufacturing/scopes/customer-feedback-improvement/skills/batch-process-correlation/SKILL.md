# 批次与工艺关联分析技能

## 概述
本技能将客户反馈的缺陷现象追溯关联到生产过程数据，包括生产批次、工艺参数、设备状态和原材料信息。这是制造业VOC闭环中最关键也最具技术难度的环节——将"客户说产品有问题"转化为"知道是哪条线、哪台设备、哪个参数导致的问题"，从而为根因分析提供精确方向。

## 适用场景
- 缺陷模式识别后的深度追溯分析
- 单一高严重度缺陷的快速批次定位
- 批量性问题的影响范围评估（同批次产品追溯）
- 改善措施制定前的因果验证

## 执行步骤

### 步骤1：批次信息确认
- 从缺陷记录中提取产品批次号/序列号
- 在ERP/MES系统中查询该批次的生产记录：
  - 生产日期和时间段
  - 生产产线/工位
  - 操作班次/操作员
  - 使用的原材料批次
  - 经过的工序路线

### 步骤2：工艺参数关联
对该批次的生产过程获取关键工艺参数数据：
- 从MES/SCADA系统提取该批次生产期间的工艺参数记录
- 关键参数项取决于产品类型（如注塑：料温/模温/保压/冷却时间；机加工：切削速度/进给量/刀具磨损量）
- 将实际参数值与标准值对比，标注偏差（在规格内/边界/超规格）
- 检查是否有工艺参数调整记录（人为调参）

### 步骤3：设备状态关联
查询该批次生产期间的设备运行状态：
- 设备健康度评分（来自IoT监控系统）
- 是否有异常告警记录
- 设备维护记录（该批次前后是否有维护/故障维修）
- 设备运行时长（是否在长时间连续运行后生产）

### 步骤4：多因素关联矩阵构建
构建缺陷-因素关联矩阵：
- 行：缺陷反馈记录
- 列：潜在影响因素（产线/设备/班次/原材料批次/关键工艺参数/环境条件）
- 填充：每条缺陷对应的各因素值
- 分析：使用卡方检验/Fisher精确检验判断缺陷是否与特定因素值显著关联

### 步骤5：关联结论输出
- 对每个显著关联输出：
  - 关联因素及其值
  - 统计显著性（p值）
  - 关联强度（OR值/相关系数）
  - 时间关系（因素变化是否早于缺陷出现）
- 排除已知的混淆因素
- 给出根因方向建议（最可能的1-3个根因假设）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| defect_cluster | Object | 是 | 缺陷聚类结果（含批次列表）|
| batch_ids | Array | 是 | 需要分析的批次号列表 |
| production_data_source | Reference | 是 | MES/ERP数据源引用 |
| iot_data_source | Reference | 否 | IoT设备数据源引用 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| correlation_matrix | Object | 缺陷-因素关联矩阵 |
| significant_correlations | Array | 显著关联列表（含统计指标）|
| root_cause_hypotheses | Array | 根因方向假设（排序按可能性）|
| affected_scope | Object | 影响范围评估（同因素的其他批次）|
| data_gaps | Array | 数据缺失项（无法获取的信息）|

## 最佳实践
- 关联分析时不要只看单一因素，注意因素间的交互效应
- 时间序列上要注意因果方向：因素变化必须早于或同时于缺陷出现
- 对于"多个批次出现同类缺陷但生产条件各异"的情况，关注共性因素
- 原材料批次关联时需考虑混料风险（同一产品批次可能使用了多个原材料批次）
- 保持分析的客观性——数据不支持的关联不要勉强下结论

## 约束条件
- 缺陷-批次关联准确率目标>=85%
- 关联分析需在48小时内完成（从缺陷模式识别到输出关联结论）
- 统计显著性阈值p<0.05，低于此标准的关联标记为"疑似关联，需进一步验证"
- 不直接修改生产系统中的任何数据
- 涉及专利工艺参数时需注意信息保密
