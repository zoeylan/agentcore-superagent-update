# 多渠道反馈采集技能

## 概述
本技能负责从多个客户反馈渠道自动化采集数据，确保制造业B2B场景下的客户声音（VOC）不遗漏、不重复地进入分析系统。覆盖的渠道包括：客诉系统（CRM/QMS）、退货单（ERP退货模块）、售后工单（售后服务系统）、经销商报告（邮件/Portal）、社交媒体监控（行业论坛/社交平台）。

## 适用场景
- 每日定时批量采集各渠道新增反馈数据
- 新渠道接入时的配置与验证
- 数据采集异常时的诊断与恢复
- 采集完整性审计与缺口识别

## 执行步骤

### 步骤1：渠道连接状态检查
- 逐一验证各渠道API/接口的连接状态
- 检查认证token是否过期
- 确认上次采集时间戳和增量标记位置
- 输出：渠道健康状态报告（正常/异常/离线）

### 步骤2：增量数据拉取
- 基于上次采集时间戳，拉取各渠道新增数据
- 客诉系统：按创建时间>上次采集时间筛选
- 退货单：按退货申请日期筛选
- 售后工单：按工单创建时间筛选
- 经销商报告：按邮件接收时间/Portal提交时间筛选
- 社交媒体：按发布时间筛选相关关键词命中内容

### 步骤3：数据去重处理
- 基于多维度去重规则：
  - 同一客户+同一产品序列号+7天时间窗口内 → 视为同一反馈
  - 同一批次号+同一缺陷描述（语义相似度>85%）→ 合并处理
  - 跨渠道重复（同一问题在客诉系统和社交媒体都提到）→ 关联但不去除
- 输出：去重后的反馈记录列表，标注重复关联关系

### 步骤4：数据标准化
- 统一日期格式（ISO 8601）
- 统一产品型号命名（匹配产品主数据）
- 统一客户标识（匹配客户主数据）
- 补充渠道来源标签
- 生成唯一反馈ID

### 步骤5：采集结果确认与异常上报
- 统计各渠道采集数量
- 对比历史日均采集量，偏差>30%时发出预警
- 记录采集时间戳用于下次增量标记
- 生成采集日志

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| channels | Array | 是 | 需要采集的渠道列表 |
| last_collection_timestamp | DateTime | 是 | 上次采集截止时间 |
| dedup_rules | Object | 否 | 自定义去重规则覆盖默认规则 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| feedback_records | Array | 标准化后的反馈记录列表 |
| collection_summary | Object | 采集统计（各渠道数量/去重数量/异常数量）|
| channel_health | Object | 各渠道健康状态 |
| next_timestamp | DateTime | 本次采集截止时间（下次起始点）|

## 最佳实践
- 采集频率建议每日2次（上午9:00、下午15:00），确保数据时效性
- 对采集异常的渠道设置3次自动重试，间隔5分钟
- 保留原始数据的完整快照，标准化处理不修改原始记录
- 经销商报告可能为非结构化邮件/附件，需先经OCR或文本提取预处理
- 社交媒体渠道需设置关键词白名单和黑名单，避免噪音数据

## 约束条件
- 采集完整性目标>=99%，即所有渠道数据漏采率<1%
- 单次采集超时上限30分钟，超时需中断并记录未完成渠道
- 不修改源系统中的任何数据，仅执行只读拉取
- 遵循数据隐私法规，客户个人信息字段脱敏处理
