# 反馈结构化与智能分类技能

## 概述
本技能将非结构化的客户反馈文本转化为结构化数据记录，并按制造业缺陷分类体系进行智能分类。核心难点在于制造业客户反馈的多样性——从专业的B2B质量报告（包含详细技术数据）到模糊的口头描述（"产品有问题"），都需要被准确理解和分类。

## 适用场景
- 新采集的反馈数据需要结构化处理时
- 分类置信度低于阈值需要人工复核时
- 分类体系更新后的历史数据重新分类
- 反馈结构化准确率的定期抽检评估

## 执行步骤

### 步骤1：文本预处理
- 清理HTML标签、特殊字符、重复空白
- 识别并分离多语言内容（中英文混合常见于制造业B2B场景）
- 提取附件信息（照片/视频/检测报告）的文字描述
- 识别反馈中包含的数据表格并提取

### 步骤2：关键字段提取
使用NLP实体识别提取以下关键字段：
- **产品型号**：匹配产品主数据库，支持模糊匹配（客户可能使用简称/旧名称）
- **产品序列号/批次号**：识别SN/Lot No格式的编号
- **缺陷现象描述**：提取核心缺陷表述，规范化为标准术语
- **发生频率**：偶发/间歇/持续/批量性
- **严重程度**：致命（安全风险）/严重（功能丧失）/一般（功能下降）/轻微（外观瑕疵）
- **发生环境/条件**：使用温度、负载、运行时长等
- **期望处理方式**：退货退款/换货/维修/赔偿/技术支持

### 步骤3：智能分类
按四级分类体系进行分类判定：

**一级分类：**
- 产品缺陷 → 二级分类：外观缺陷/功能缺陷/可靠性缺陷/安全缺陷
- 使用问题 → 二级分类：操作不当/环境条件超标/维护不当
- 功能建议 → 二级分类：新功能需求/性能提升/易用性改善
- 其他 → 二级分类：物流损坏/包装问题/文档错误/价格争议

**分类规则：**
- 每条反馈必须给出一级分类和二级分类
- 输出分类置信度评分（0-100%）
- 置信度<80%的标记为"待人工复核"
- 一条反馈可能涉及多个问题，需拆分为多条记录分别分类

### 步骤4：批次关联（产品缺陷类）
对产品缺陷类反馈执行批次关联：
- 基于产品序列号直接关联到生产批次
- 无序列号时，基于产品型号+客户收货时间反向推算可能的生产批次范围
- 关联结果标注置信度（精确匹配/范围推算/无法关联）
- 目标：24小时内完成批次关联

### 步骤5：结果输出与质量标记
- 生成结构化反馈记录
- 标记信息完整性（哪些字段缺失需要补全）
- 标记分类质量标签（高置信/待复核/人工分类）
- 输出待人工处理队列

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| raw_feedback | Object | 是 | 原始反馈数据（含文本、附件信息）|
| product_master | Reference | 是 | 产品主数据引用（用于型号匹配）|
| classification_schema | Object | 否 | 分类体系配置（使用默认体系时可省略）|

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| structured_record | Object | 结构化反馈记录（含所有提取字段）|
| classification | Object | 分类结果（一级/二级/置信度）|
| batch_linkage | Object | 批次关联结果（批次号/置信度/方法）|
| quality_flags | Array | 质量标记（缺失字段/低置信分类等）|

## 最佳实践
- 建立制造业缺陷描述同义词库（如"异响"="噪音"="有声音"="rattling"）
- 定期（每月）用抽检结果校准分类模型，确保准确率>=90%
- 对于高价值B2B客户的反馈，即使分类为"使用问题"也建议标记关注
- 批次关联时优先使用精确的序列号匹配，减少推算误差

## 约束条件
- 结构化提取准确率目标>=90%（以月度抽检为准）
- 产品缺陷类反馈必须在24小时内完成批次关联
- 不得删除或修改原始反馈文本，结构化数据作为附加信息
- 涉及客户商业机密的内容需要脱敏标记
