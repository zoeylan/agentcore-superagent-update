# 8D改进触发管理技能

## 概述
本技能负责当缺陷模式达到预设阈值时，自动触发8D问题解决流程，并向质量管理部门推送完整的分析数据包。8D（Eight Disciplines）是制造业处理客户投诉和质量问题的标准方法论，由福特汽车公司开发，广泛应用于汽车、电子、机械等制造行业。本技能确保高频缺陷不遗漏、不延迟地进入正式改善流程。

## 适用场景
- 同类投诉数量达到高频阈值（>=5次/月）
- 新出现的缺陷类型经评估需要立即启动8D
- 客户明确要求提供8D报告
- 安全类/批量性问题的强制8D触发

## 执行步骤

### 步骤1：触发条件检查
评估以下触发条件（满足任一即触发）：
- **频率触发**：同类缺陷投诉>=5次/月（按缺陷聚类簇计数）
- **严重度触发**：单次P1/P2级缺陷（安全类/批量性）
- **客户要求触发**：客户明确要求提交8D报告
- **趋势触发**：缺陷数量连续3周上升
- **复发触发**：已关闭的8D问题3个月内复发

### 步骤2：8D数据包准备
为8D团队准备完整的分析数据包：

**D0-问题概述：**
- 缺陷现象汇总描述
- 影响产品型号/批次范围
- 影响客户列表及投诉时间线
- 当前累计投诉数量和趋势

**D1-团队建议：**
- 基于缺陷关联分析结论，建议涉及的职能部门
- 建议的团队leader（基于缺陷类型和相关产线/工序）

**D2-问题描述（5W2H）：**
- What：缺陷是什么
- Where：在哪里发现/哪个产品/哪个部位
- When：什么时候开始出现/频率如何
- Who：哪些客户受影响
- Why：为什么是问题（对客户的影响）
- How：如何表现（详细现象描述）
- How many：影响数量/比例

**附件数据：**
- 缺陷聚类分析详细数据
- 批次/工艺关联分析结果
- 原始客户反馈记录清单
- 相关检验数据/图片（如有）

### 步骤3：8D工单创建与推送
- 在质量管理系统中创建8D工单
- 设定8D关键时间节点：
  - D3临时遏制措施：48小时内
  - D4根因分析完成：7天内
  - D5永久纠正措施制定：14天内
  - D6措施实施验证：21天内
  - D7预防措施：25天内
  - D8关闭：30天内
- 推送给质量管理Scope的质量异常处理专家
- 同步通知相关产线/工序负责人

### 步骤4：触发记录与跟踪关联
- 记录触发时间、触发条件、8D工单编号
- 建立缺陷反馈记录与8D工单的关联关系
- 设置后续新增同类投诉自动关联到该8D工单
- 将8D进展信息传递给改进跟踪Agent

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| defect_cluster | Object | 是 | 触发的缺陷聚类模式 |
| trigger_type | String | 是 | 触发类型（频率/严重度/客户要求/趋势/复发）|
| correlation_results | Object | 否 | 批次工艺关联分析结果 |
| affected_customers | Array | 是 | 受影响客户列表 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| eight_d_ticket_id | String | 创建的8D工单编号 |
| data_package | Object | 推送的完整数据包 |
| notification_status | Object | 通知发送状态 |
| timeline | Object | 8D各阶段截止时间 |
| linked_feedbacks | Array | 关联的反馈记录ID列表 |

## 最佳实践
- 8D数据包的质量直接影响后续分析效率——信息越完整，根因定位越快
- 触发后48小时内必须有临时遏制措施（如加严检验、隔离在途品等），这是8D的硬性要求
- 对于客户要求的8D，需确认客户期望的回复时间节点，可能比内部标准更紧
- 同一根因可能导致多个缺陷模式，注意识别并合并为同一个8D课题
- 高频缺陷识别后48小时内必须完成8D触发，不得延迟

## 约束条件
- 高频缺陷识别后48小时内必须触发8D
- 8D工单必须在30天内关闭，超期自动升级
- 数据包中不得包含未经验证的结论，标注为"假设"即可
- 8D触发后不得自行撤销，只能由质量经理审批后取消
- 每个8D工单必须有明确的责任人和截止时间
