# 设备大修计划管理 (Overhaul Planning)

## 技能概述
管理设备大修（Overhaul）计划的全生命周期，包括大修需求评估、预算编制、提前排期（至少3个月）、资源准备和执行协调。大修是设备生命周期中最重要且资源消耗最大的维护活动，需要系统化的计划管理。

## 适用场景
- 年度设备大修计划编制（年度预算周期）
- 设备运行小时数/使用年限达到大修阈值
- 设备故障分析建议进行大修
- 设备性能持续衰退需要全面恢复

## 执行步骤

### 步骤1：大修需求评估
- 审查设备运行数据：累计运行小时数、使用年限、PM执行历史
- 分析设备性能趋势：OEE趋势、精度衰退数据、故障频率变化
- 评估关键部件寿命：参考制造商建议和实际运行数据
- 确定大修范围：全面大修（Overhaul）vs局部大修（Partial）

### 步骤2：预算编制
- 列出大修所需项目：部件更换、加工修复、外包服务、检测校验
- 估算备件成本：核心部件报价确认、消耗材料估算
- 估算人工成本：内部工时+外部服务商费用
- 预留不可预见费用（通常为预算的10-15%）
- 生成预算申请书（含投资回报分析：大修成本 vs 设备更换成本）

### 步骤3：排期规划
- 确定大修时间窗口（提前3个月锁定）：优先利用长假/年度停产期
- 评估对产能影响：大修期间替代产能方案（备机/外协/库存缓冲）
- 制定里程碑计划：备件采购→外协联系→预检→进厂施工→测试→验收
- 协调相关部门：生产（产能安排）、采购（备件到位）、安全（施工许可）

### 步骤4：执行准备跟踪
- 大修前3个月：启动长周期备件采购、联系外部服务商
- 大修前1个月：确认所有备件到位、人员排班确认、安全方案审批
- 大修前1周：预检完成、施工方案最终确认、应急预案就位
- 大修执行：日进度跟踪、质量节点验收、异常问题协调
- 大修完成：设备测试验收、性能基线重建、文档归档

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| equipment_info | object | 是 | 设备信息（型号/年限/运行小时数）|
| performance_data | object | 是 | 设备性能数据（OEE/精度/故障率）|
| manufacturer_specs | object | 否 | 制造商大修建议 |
| budget_year | string | 是 | 预算年度 |
| production_calendar | object | 否 | 年度生产日历（可用停产窗口）|

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| overhaul_assessment | object | 大修需求评估报告 |
| budget_proposal | object | 预算申请书 |
| milestone_plan | array | 里程碑计划（含责任人和日期）|
| preparation_checklist | array | 准备工作检查清单 |
| risk_mitigation | array | 风险缓解措施 |

## 最佳实践
- 大修评估应综合经济性分析：大修成本不超过设备重置价值的60%为宜
- 利用大修机会同时完成设备升级改造（如加装传感器、更新控制系统）
- 大修后建立新的性能基线，用于后续OEE和故障分析对比
- 建立大修经验库，相似设备大修可参考历史记录
- 大修验收标准应量化（精度/振动/温升等参数指标）

## 约束条件
- 大修计划必须纳入年度预算，超预算需追加审批
- 提前3个月锁定大修时间窗口，变更需总监级审批
- 大修期间设备状态标记为"停用"，不得安排生产
- 大修完成后必须进行72小时试运行验证才能恢复正式生产
