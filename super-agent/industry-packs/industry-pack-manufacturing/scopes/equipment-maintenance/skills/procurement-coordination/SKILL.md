# 采购协调 (Procurement Coordination)

## 技能概述
协调备件采购活动，包括常规采购和紧急采购的发起、审批跟进、供应商沟通和到货跟踪。该技能是备件管理与供应链管理Scope之间的桥梁，确保采购需求及时、准确地传递并执行。

## 适用场景
- 库存低于再订货点触发常规采购
- A类设备紧急维修缺件时的紧急采购
- PM计划备件预采购
- 大修专用备件的长周期采购
- 采购订单到货延迟的催货跟进

## 执行步骤

### 步骤1：采购需求确认
- 明确采购类型：
  - 常规补货：库存触发，按EOQ经济批量下单
  - PM预采购：PM计划触发，按需求清单下单
  - 紧急采购：故障维修触发，最快到货为首要目标
  - 大修采购：大修计划触发，提前3个月启动
- 核实需求信息：备件规格/数量/需求日期/质量要求
- 确认是否有替代件可用（缩短采购周期）

### 步骤2：采购请求提交
- 编制采购请求单：备件信息/数量/需求日期/预算/紧急程度
- 标注采购审批路径：
  - <10万：备件管理专员直接提交
  - 10-50万：需设备经理审批
  - >=50万：需维护总监审批
- 提交至供应链管理Scope的采购计划专员
- 紧急采购：走快速通道，同步启动审批和寻源

### 步骤3：订单执行跟踪
- 跟踪采购订单状态：已下单→供应商确认→生产中→发货→到货
- 监控关键时间节点：
  - 供应商3天内需确认交期
  - 实际交期 vs 承诺交期偏差预警
  - 到货前3天提醒仓库做好收货准备
- 延迟处理：
  - 延迟<3天：催货确认新交期
  - 延迟3-7天：评估影响并启动替代方案
  - 延迟>7天：升级到管理层并执行应急预案

### 步骤4：到货验收协调
- 通知仓库做好收货准备
- 协调质量验收（外观/规格/数量/随附证书）
- 不合格处理：退货/换货/让步接收
- 合格入库后更新库存数据

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| procurement_type | string | 是 | 采购类型（常规/紧急/PM预采购/大修）|
| parts_request | array | 是 | 备件需求清单（编号/数量/日期）|
| budget_info | object | 否 | 预算信息 |
| related_work_order | string | 否 | 关联工单号 |
| urgency_level | string | 是 | 紧急程度（正常/加急/特急）|

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| purchase_request | object | 采购请求单 |
| order_status | array | 订单跟踪状态 |
| delay_alerts | array | 延迟预警 |
| receipt_confirmation | object | 到货确认 |
| procurement_metrics | object | 采购绩效（紧急采购比例/到货准时率）|

## 最佳实践
- 紧急采购比例目标<10%——过高说明需求预测或安全库存设置有问题
- 建立关键备件的"双源供应"，避免单一供应商导致断供
- 常规采购尽量合并同一供应商的订单以降低物流成本
- 大修备件采购应列出"必须件"和"建议件"，分批采购降低预算压力
- 每次紧急采购后需复盘原因：是预测偏差还是突发故障

## 约束条件
- 紧急采购必须在需求确认后2小时内提交采购请求
- 采购请求必须附带关联工单或PM计划编号
- 超预算采购必须获得额外审批
- 供应商选择必须在合格供应商名录（AVL）内
