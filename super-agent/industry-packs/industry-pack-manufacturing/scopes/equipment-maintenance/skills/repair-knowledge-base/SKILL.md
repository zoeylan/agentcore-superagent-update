# 维修知识库管理 (Repair Knowledge Base)

## 技能概述
构建和维护设备维修知识库，将每次故障诊断和维修经验沉淀为可复用的知识条目，支持快速检索和智能推荐，使维修团队的集体经验不因人员变动而流失，新手技工也能快速获得诊断指引。

## 适用场景
- 故障维修完成后的知识沉淀
- 维修技工查询历史类似故障的处理方案
- 新设备投用后建立初始故障应对知识
- 知识库条目的定期审核和更新
- 维修培训资料的自动整理

## 执行步骤

### 步骤1：知识条目创建
- 从已关闭的故障工单中提取结构化知识：
  - 设备信息（型号/系统/子系统）
  - 故障现象（分类标签+自然语言描述）
  - 报警代码（如适用）
  - 诊断路径（实际执行的排查步骤）
  - 根本原因（5Why分析结论）
  - 维修方案（步骤/工具/备件/工时）
  - 预防建议（如何避免再次发生）
- 标注知识条目的适用范围和关键词

### 步骤2：知识检索与推荐
- 基于当前故障现象匹配历史知识条目
- 匹配维度：设备型号（精确）→ 故障现象关键词（模糊）→ 报警代码（精确）
- 返回匹配度最高的TOP5条目，附带匹配度评分
- 标注每个推荐条目的成功率（该方案历史解决此类问题的比例）

### 步骤3：知识质量管理
- 维修完成后验证知识条目准确性：本次采用该方案是否成功？
- 成功 → 提升该条目的置信度评分
- 失败 → 标记需要补充/修正，记录新的发现
- 定期审核：超过12个月未被引用的条目标记为"待审核"
- 清理：设备已淘汰的条目归档（非删除）

### 步骤4：知识统计与洞察
- 统计最常被检索和引用的知识条目（热门故障）
- 识别知识空白：某些设备/故障类型没有对应知识条目
- 分析知识条目的有效性：推荐后实际采用率和问题解决率
- 输出知识库健康度报告（覆盖率/使用率/准确率）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fault_record | object | 是（创建时）| 已关闭的故障维修记录 |
| search_query | object | 是（检索时）| 检索条件（设备/现象/代码）|
| feedback | object | 否 | 使用反馈（是否有效）|

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| knowledge_entry | object | 新建/更新的知识条目 |
| search_results | array | 检索结果（含匹配度评分）|
| knowledge_health | object | 知识库健康度报告 |
| gap_analysis | array | 知识空白识别 |

## 最佳实践
- 知识条目应以"一线维修技工可直接执行"为编写标准
- 每个条目应附操作照片/示意图的描述（如拆装顺序、测量点位置）
- 诊断步骤应按"从简单到复杂、从非侵入到侵入"排序
- 重要安全注意事项必须醒目标注
- 知识库应支持多语言（技工可能母语不同）

## 约束条件
- 每次非计划维修完成后7天内必须完成知识沉淀
- 知识条目需经维护主管审核后发布
- 安全相关操作知识必须符合最新安全规程
- 知识库条目不替代设备原厂维修手册，是补充而非替代
