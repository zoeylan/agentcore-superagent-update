# 备件需求预测 (Parts Demand Forecasting)

## 技能概述
基于设备PM计划（计划性消耗）和历史故障数据（非计划性消耗）双维度，预测未来备件需求量和需求时间，为采购决策提供前瞻性指导。该技能将被动的"缺件再买"转变为主动的"提前备料"。

## 适用场景
- 月度/季度备件需求预测编制
- PM计划发布后的备件需求匹配
- 新设备投用后的备件需求估算
- 季节性需求波动分析
- 供应商交期变化时的安全库存调整

## 执行步骤

### 步骤1：计划性需求预测
- 获取未来周期的PM计划：哪些设备/何时/做什么PM项目
- 逐项分解PM所需备件：每个PM项目消耗哪些备件、数量多少
- 汇总计划性需求：按备件编号汇总未来1-3个月的PM需求量
- 标注确定性级别：已确认PM→确定需求 / 计划中PM→预计需求

### 步骤2：非计划性需求预测
- 分析历史故障消耗数据：过去12个月各备件因故障维修消耗的数量
- 计算平均月消耗量和标准差
- 考虑趋势因素：设备老化→故障率上升→备件消耗增加
- 考虑季节因素：某些故障类型与环境条件相关（如夏季冷却系统故障）
- 预测非计划消耗 = 历史平均 × 趋势系数 × 季节系数

### 步骤3：综合需求计算
- 总需求预测 = 计划性需求 + 非计划性需求预测
- 对比当前库存：需采购量 = 总需求预测 - 当前库存 + 安全库存补充
- 标注需求时间紧迫度：需求日期 vs 采购提前期
- 识别紧急需求：采购提前期>剩余时间的备件

### 步骤4：预测准确度追踪
- 对比历史预测 vs 实际消耗，计算预测准确率
- MAPE（平均绝对百分比误差）目标 < 30%
- 分析预测偏差原因：过预测（浪费）/ 欠预测（缺件）
- 持续优化预测模型参数

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pm_plan_next_period | array | 是 | 未来PM计划及备件需求 |
| historical_consumption | array | 是 | 历史12个月备件消耗数据 |
| current_inventory | array | 是 | 当前库存水位 |
| equipment_age_data | object | 否 | 设备老化/运行数据 |
| forecast_horizon | string | 是 | 预测周期（1个月/3个月）|

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| demand_forecast | array | 需求预测（备件/数量/时间/类型）|
| procurement_suggestion | array | 采购建议（数量/建议采购时间）|
| urgent_items | array | 紧急需求项（需加快采购）|
| forecast_accuracy | object | 预测准确度回顾 |
| risk_items | array | 供应风险备件（长周期/单一来源）|

## 最佳实践
- 计划性需求预测精度应远高于非计划性需求（目标偏差<10%）
- 对于高价值备件（单价>1万元），需求预测应更保守（宁可多备）
- 设备大修期间的备件需求应单独预测（量大且集中）
- 新设备投用后前6个月参考同类设备历史数据估算
- 预测结果应与备件管理专员和维护计划调度员共同评审

## 约束条件
- 需求预测每月更新一次（PM计划发布后5天内）
- 紧急需求（提前期不足）必须当日标识并通知采购
- 预测数据必须标注确定性级别（确定/高概率/估算）
- 不对已确认淘汰设备的备件做长期需求预测
