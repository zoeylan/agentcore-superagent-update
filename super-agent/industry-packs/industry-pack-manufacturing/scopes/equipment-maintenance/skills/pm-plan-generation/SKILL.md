# PM计划生成 (PM Plan Generation)

## 技能概述
基于设备维护标准、设备关键程度分类和维护周期要求，自动生成月度/季度预防性维护（PM）计划。该技能将设备台账信息、PM标准作业清单（checklist）和维护周期规则转化为可执行的维护工单集合。

## 适用场景
- 月初生成下月PM计划（每月1-5日）
- 季度初生成季度PM计划
- 新设备纳入PM体系时初始化维护计划
- 设备健康度评分变化触发PM周期调整

## 执行步骤

### 步骤1：设备台账数据获取
- 读取当前在册设备清单，包含：设备编号、设备名称、关键程度分类（A/B类）、所属产线、上次PM完成日期
- 筛选本周期内需执行PM的设备（A类设备每月、B类设备每季度）

### 步骤2：PM标准作业清单匹配
- 为每台设备匹配对应的PM checklist（润滑/紧固/清洁/检测/校准/更换等项目）
- 确定每项PM任务的预计工时和所需备件清单
- 识别需要专业资质的检测项目（如压力容器检测、电气安规测试）

### 步骤3：PM计划初始编排
- 按设备关键程度排序：A类设备优先排入前半月
- 估算总工时需求，对比可用维修人力资源
- 生成初始PM计划表（设备/项目/计划日期/工时/备件/责任人）

### 步骤4：输出PM计划
- 输出结构化PM计划表
- 汇总备件需求清单（发送给备件管理专员）
- 标注需要协调生产停机的PM任务
- 生成计划摘要（总任务数/总工时/涉及设备数/备件需求项）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| planning_period | string | 是 | 计划周期，如"2024-03"表示3月 |
| equipment_list | array | 是 | 设备清单（含分类和上次PM日期）|
| pm_standards | object | 是 | PM标准作业清单库 |
| available_technicians | array | 否 | 可用维修技工列表 |
| health_score_overrides | array | 否 | 健康度低于70分需提前PM的设备 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| pm_plan | array | PM计划列表（设备/项目/日期/工时/备件/人员）|
| parts_requirement | array | 备件需求汇总清单 |
| production_coordination | array | 需协调停机的任务列表 |
| plan_summary | object | 计划摘要统计 |

## 最佳实践
- A类设备PM不应集中在同一天，避免维修资源瓶颈
- 计划发布前必须与备件管理专员确认备件可用性
- 保留10-15%的弹性工时应对紧急维修插单
- 设备健康度<70分的设备应提前1-2周安排PM
- 大修前2周应安排预检（pre-inspection）确认大修范围

## 约束条件
- PM计划必须在月初5个工作日内完成发布
- 同一技工单日PM工时不超过8小时
- A类设备PM时间窗口需预留至少2小时余量
- 涉及安全相关检测的PM必须由持证人员执行
