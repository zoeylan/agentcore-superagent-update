# PM执行跟踪 (PM Execution Tracking)

## 技能概述
实时跟踪预防性维护计划的执行进度，监控PM完成率和按时率，对逾期未执行的任务发出预警，分析执行偏差原因并提出改善建议。该技能确保PM计划从"纸面计划"转化为"实际执行"。

## 适用场景
- 每日PM执行进度更新和跟踪
- 逾期PM任务预警和升级
- 周度/月度PM执行率统计分析
- PM执行质量问题识别

## 执行步骤

### 步骤1：执行数据采集
- 获取PM工单状态更新：已开始/进行中/已完成/已取消/已延期
- 采集完成时间记录：实际开始时间vs计划时间、实际工时vs预计工时
- 收集执行质量数据：checklist完成率、定量检测数据（振动值/温度/间隙）
- 标记异常完工：未按checklist全部完成、跳过某些检测项

### 步骤2：进度分析计算
- 计算当期PM完成率 = 已完成PM工单数 / 计划PM工单总数 × 100%
- 计算PM按时率 = 按时完成的PM工单数 / 已完成PM工单数 × 100%
- 识别逾期任务：已超过计划日期但未完成的PM工单
- 识别风险任务：距离计划日期<3天但未开始的PM工单

### 步骤3：预警与升级
- 单任务逾期1天 → 提醒责任技工和班组长
- 单任务逾期3天 → 升级到维护主管
- 月度PM执行率<90% → 预警给维护经理并分析原因
- A类设备PM逾期 → 立即预警并评估设备风险

### 步骤4：分析与报告
- 按原因分类统计逾期任务（生产冲突/人员不足/备件缺失/其他）
- 输出PM执行周报/月报（完成率/按时率/偏差分析/改善建议）
- 趋势分析：连续3个月PM执行率变化趋势
- 识别改善机会（如某产线PM执行率持续低于平均值）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pm_plan | array | 是 | 当期PM计划（含计划日期）|
| work_order_status | array | 是 | PM工单执行状态更新 |
| completion_records | array | 否 | 完工记录（实际时间/checklist数据）|
| reporting_period | string | 是 | 报告周期 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| execution_summary | object | 执行摘要（完成率/按时率）|
| overdue_alerts | array | 逾期预警列表 |
| deviation_analysis | object | 偏差原因分析 |
| improvement_suggestions | array | 改善建议 |
| trend_data | array | 趋势数据 |

## 最佳实践
- 每日上午检查前一天PM执行情况，确保及时更新状态
- 逾期原因分析应区分"不可控因素"（紧急生产插单）和"可控因素"（人员安排不当）
- PM完成率目标>=95%，按时率目标>=90%
- 对反复逾期的设备/产线进行专项分析
- 月度报告应包含下月改善措施和跟踪计划

## 约束条件
- 预警信息必须在逾期当天发出，不得延迟
- PM执行率数据每日更新，不得使用过期数据
- 取消PM工单必须附原因说明并经主管审批
- 执行率统计口径需一致（已取消工单不计入分母）
