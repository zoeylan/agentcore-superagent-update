# 备件库存管理 (Spare Parts Inventory)

## 技能概述
实时监控和管理维修备件的库存水位，包括安全库存/最高库存/再订货点的设定与动态调整，库存状态告警，出入库管理和定期盘点。确保备件库存既能支撑设备维修需求，又不产生过多呆滞资金占用。

## 适用场景
- 实时库存水位监控和告警
- 备件出库时的工单关联登记
- 备件入库时的验收和上架
- 月度/季度循环盘点
- 呆滞备件识别和处置

## 执行步骤

### 步骤1：库存水位监控
- 实时比对每种备件的当前库存与设定参数：
  - 当前库存 > 最高库存 → 超储预警
  - 当前库存 < 再订货点 → 触发补货信号
  - 当前库存 < 安全库存 → 紧急缺件告警
  - 当前库存 = 0 → 断档告警（A类设备关键备件最高优先级）
- 标记各备件状态：正常/预警/缺货/超储

### 步骤2：出入库管理
- 出库流程：
  - 验证工单关联（每次出库必须关联PM工单号或CM工单号）
  - 更新库存数量
  - 记录领用人/时间/用途
  - 低于再订货点自动触发补货
- 入库流程：
  - 核对到货数量与采购订单
  - 质量验收（外观/规格/证书）
  - 上架登记（库位号）
  - 更新库存数量

### 步骤3：库存参数维护
- 安全库存计算 = 平均日消耗 × (采购提前期 + 安全天数)
- 再订货点 = 安全库存 + (平均日消耗 × 采购提前期)
- 最高库存 = 再订货点 + 经济订购批量
- 定期（每季度）基于最新消耗数据重新计算库存参数
- 新增备件/设备淘汰时及时调整参数

### 步骤4：呆滞备件处理
- 识别标准：超过180天无出库记录的备件
- 分析原因：设备淘汰/故障模式改变/过度采购/规格替代
- 处置方案：内部调拨→集团内调配→变卖→报废
- 计算呆滞金额并纳入月度库存报告

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inventory_data | array | 是 | 当前库存数据 |
| transaction | object | 否 | 出入库事务（出库/入库）|
| work_order_ref | string | 否 | 关联工单号（出库时必填）|
| stocktake_data | array | 否 | 盘点数据 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| inventory_status | array | 库存状态报告（含预警标识）|
| replenishment_signals | array | 补货信号列表 |
| obsolete_parts | array | 呆滞备件清单 |
| transaction_record | object | 出入库记录确认 |
| stocktake_variance | object | 盘点差异报告 |

## 最佳实践
- A类设备关键备件（不可替代/采购周期>4周）应设较高安全库存
- 低值易耗件（密封圈/螺栓等）可适当加大批量降低采购频次
- 出库必须关联工单——这是控制消耗可追溯性的关键
- 呆滞备件处置前需确认相关设备确实已无使用计划
- 年度盘点差异率目标<2%，超过5%需专项调查

## 约束条件
- 库存数据必须实时更新，不允许隔日补录
- 无工单号不允许出库（紧急情况可先出库，24小时内补工单）
- 盘点差异金额>5000元必须查明原因
- A类设备关键备件缺件时1小时内必须启动紧急采购
