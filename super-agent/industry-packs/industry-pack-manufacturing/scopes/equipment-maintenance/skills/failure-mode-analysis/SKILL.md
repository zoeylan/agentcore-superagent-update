# 故障模式分析 (Failure Mode Analysis)

## 技能概述
对设备故障数据进行统计分析，追踪MTBF（平均故障间隔）和MTTR（平均修复时间）趋势，识别设备可靠性退化模式和反复故障的设备/部件，为维护策略优化和设备更换决策提供数据支撑。

## 适用场景
- 月度/季度设备可靠性分析报告生成
- 设备故障频率突增时的趋势分析
- 同一设备30天内故障>=3次的升级评审
- 设备更换/改造投资决策的数据支持
- PM周期和策略的优化调整

## 执行步骤

### 步骤1：故障数据汇总
- 按设备/产线/故障类型汇总指定周期内的故障记录
- 计算关键可靠性指标：
  - MTBF = 总运行时间 / 故障次数
  - MTTR = 总维修时间 / 故障次数
  - 可用率 = MTBF / (MTBF + MTTR) × 100%
- 故障按类型分类统计（机械/电气/液压/气动/控制系统/其他）
- 按故障严重程度分级（停机/降速/功能丧失/性能劣化）

### 步骤2：趋势分析
- 绘制MTBF趋势（月度/季度滚动平均）：识别是否持续下降
- 绘制MTTR趋势：维修效率是否改善
- Weibull分布分析：判断设备处于"浴盆曲线"的哪个阶段（早期故障/偶发故障/耗损故障）
- 季节性分析：故障是否与温度/湿度/产能负载相关

### 步骤3：Pareto分析
- 按设备排名：故障次数TOP10设备（识别"问题设备"）
- 按部件排名：故障频率最高的部件/子系统
- 按原因排名：TOP5根本原因分类
- 计算80/20：是否存在少数设备贡献大部分停机时间

### 步骤4：改善建议输出
- 对MTBF持续下降的设备：建议调整PM频率/增加检测项/考虑大修或更换
- 对MTTR过长的故障类型：建议改进诊断方法/预备件/技能培训
- 对反复故障的设备：评估是否需要设计改造或替换
- 对30天内故障>=3次的设备：触发改造/更换评审流程
- 输出经济性分析：维修成本 vs 更换成本趋势对比

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fault_records | array | 是 | 指定周期内故障记录 |
| equipment_list | array | 是 | 分析范围设备清单 |
| analysis_period | string | 是 | 分析周期（月/季/年）|
| operating_hours | object | 否 | 设备运行小时数 |
| maintenance_costs | array | 否 | 维修成本数据 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| reliability_metrics | object | MTBF/MTTR/可用率统计 |
| trend_analysis | object | 趋势分析结论 |
| pareto_ranking | array | Pareto排名（设备/部件/原因）|
| problem_equipment | array | 问题设备清单及建议 |
| improvement_recommendations | array | 改善建议 |
| replacement_candidates | array | 建议更换/改造的设备 |

## 最佳实践
- MTBF分析需排除外部原因停机（如停电/缺料等待），仅统计设备本身故障
- 趋势分析至少需要6个月以上数据才有统计意义
- 设备更换建议应附经济性分析（LCC生命周期成本对比）
- 新设备投用后前3个月的故障应作为"早期故障"单独分析
- 分析结论应转化为具体的PM策略调整建议（频率/项目/方法）

## 约束条件
- 分析报告每月生成一次，季度报告含趋势对比
- 数据准确性依赖于工单记录完整性，需标注数据质量
- 设备更换建议需附投资回报分析，最终决策由管理层做出
- 30天内故障>=3次的升级评审必须在识别后48小时内启动
