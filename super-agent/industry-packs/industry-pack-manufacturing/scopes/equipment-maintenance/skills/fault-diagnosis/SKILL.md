# 设备故障诊断 (Fault Diagnosis)

## 技能概述
在设备故障发生时，基于故障现象描述、设备历史故障记录、传感器数据和维修知识库，快速提供系统化的故障诊断路径和维修方案建议。该技能的核心目标是缩短故障定位时间，降低MTTR。

## 适用场景
- 操作工报修时的初步故障定位指导
- IoT异常告警触发的故障预诊断
- 维修技工现场诊断辅助
- 间歇性故障/疑难故障的系统性排查

## 执行步骤

### 步骤1：故障信息采集与结构化
- 采集故障现象描述：异常表现（异音/振动/温升/精度偏差/报警代码/泄漏等）
- 确认故障上下文：发生时间、运行工况（产品型号/运行速度/负载情况）、操作动作
- 获取设备信息：设备编号、型号、投用年限、近期PM/CM记录
- 查询传感器数据：故障前后的振动/温度/电流/压力趋势

### 步骤2：故障模式匹配
- 在设备故障知识库中检索相似故障案例（按设备型号+故障现象匹配）
- 分析该设备历史故障记录：是否为反复发生的已知问题
- 参考设备维护手册的故障排除章节
- 基于故障现象推断可能的故障模式（如：异常振动→轴承磨损/不对中/松动/不平衡）

### 步骤3：诊断决策树生成
- 按可能性从高到低排列可能原因（基于先验概率和当前证据）
- 为每个可能原因设计验证步骤（观察/测量/替换/隔离）
- 标注每步诊断的预计耗时和所需工具
- 设计排除法路径：验证结果→排除某原因→进入下一个可能原因

### 步骤4：维修方案建议
- 对最可能的故障原因提供维修方案（临时修复/彻底修复）
- 列出所需备件清单和专用工具
- 评估维修难度和预计修复时间
- 标注安全注意事项（高压/高温/化学品/重物）和LOTO要求
- 判断是否需要外部支援（厂家服务/专业公司）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| equipment_id | string | 是 | 设备编号 |
| fault_symptoms | string | 是 | 故障现象描述 |
| alarm_codes | array | 否 | 设备报警代码 |
| sensor_data | object | 否 | 故障前后传感器数据 |
| operating_context | object | 否 | 故障发生时运行工况 |
| equipment_history | array | 否 | 设备历史故障记录 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| diagnosis_tree | object | 诊断决策树（按概率排序）|
| probable_causes | array | 可能原因列表（含概率评估）|
| verification_steps | array | 逐步验证方案 |
| repair_recommendation | object | 推荐维修方案 |
| parts_needed | array | 所需备件清单 |
| safety_notes | array | 安全注意事项 |
| external_support_needed | boolean | 是否需要外部支援 |

## 最佳实践
- 先执行非侵入性诊断（观察/测量），再考虑侵入性检查（拆解）
- 间歇性故障重点关注工况变化触发条件
- 对电气故障优先检查最简单的可能（松动/接触不良/保险熔断）
- 对机械故障关注近期是否有PM/CM操作（安装不当导致二次故障）
- 诊断建议应可被一线维修技工直接执行，语言简洁明确

## 约束条件
- 诊断建议必须在A类设备报修后15分钟内提供初步方向
- 不提供超出设备安全规范的操作建议
- 涉及高压电气设备必须注明需持证电工操作
- 不确定的诊断方向需明确标注置信度级别
