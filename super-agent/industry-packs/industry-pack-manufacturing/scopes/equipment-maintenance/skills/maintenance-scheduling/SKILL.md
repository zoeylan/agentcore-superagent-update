# 维护排程优化 (Maintenance Scheduling)

## 技能概述
综合考虑生产排程、维修人员可用性、备件到位状态和设备优先级等多维约束条件，对PM计划进行排程优化，找到维护效果和生产影响的最佳平衡点。该技能解决"何时维护"的调度问题。

## 适用场景
- PM计划初始生成后的排程优化
- 生产排程变更后的维护计划调整
- 紧急维修插入后的剩余PM重排
- 资源（人员/备件）变化后的计划重平衡

## 执行步骤

### 步骤1：约束条件收集
- 获取生产排程信息：计划停线时间、换线窗口、周末/节假日安排
- 获取人员可用性：维修技工排班表、技能矩阵、请假信息
- 获取备件状态：已到位备件、在途备件（预计到货日期）、缺件清单
- 获取设备优先级：A/B类分类、健康度评分、产线关键度

### 步骤2：排程约束建模
- 硬约束（不可违反）：A类设备必须在当月完成PM；安全检测必须由持证人员执行；维修不可在设备运行中进行（除非明确标注为在线维护）
- 软约束（尽量满足）：优先利用换线窗口和计划停线；避免多台A类设备同天维护；尽量连续安排同产线设备PM（减少移动时间）

### 步骤3：排程优化算法
- 优先安排：健康度最低的设备、PM已逾期/即将到期的设备
- 窗口匹配：将PM任务匹配到最近的可用停机窗口
- 资源平衡：确保每日维修工时分布均匀，避免峰值超载
- 冲突检测：标识资源冲突（同一技工/同一时段多任务）并提供替代方案

### 步骤4：输出优化排程
- 生成按日期排列的维护排程表（甘特图格式描述）
- 列出排程冲突及建议解决方案
- 标注风险项（备件未到位/窗口过短/人员紧张）
- 计算排程质量指标（资源利用率/窗口利用率/优先级满足率）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pm_plan | array | 是 | 待排程的PM任务列表 |
| production_schedule | object | 是 | 生产排程（停线窗口/换线计划）|
| technician_availability | array | 是 | 技工排班和技能矩阵 |
| parts_status | array | 是 | 备件到位状态 |
| equipment_priorities | array | 否 | 设备优先级覆盖 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| optimized_schedule | array | 优化后的排程表 |
| conflicts | array | 未解决的排程冲突 |
| risk_items | array | 排程风险标记 |
| schedule_metrics | object | 排程质量指标 |

## 最佳实践
- 排程应预留15%缓冲时间应对突发情况
- 同一设备的PM任务应尽量集中安排（一次停机完成多项维护）
- 优先利用自然停机窗口（换班交接/午休/换线）减少额外停机损失
- 关键设备PM安排在白班（人员和资源最充足的时段）
- 排程变更超过原计划20%时需重新评审

## 约束条件
- 排程优化不得违反安全规程（如带电作业限制）
- A类设备PM延期不超过3天（超过需管理层审批）
- 排程方案需在计划发布后24小时内确认
- 同一设备两次PM间隔不得超过规定周期的120%
