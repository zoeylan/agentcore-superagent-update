# 库存水位监控与优化

## 技能概述
实时监控全品类物料库存水位，通过与安全库存、最高库存、再订货点的对比分析，自动生成补货信号和超储预警，确保库存在服务水平和持有成本之间达到最优平衡。

## 适用场景
- 每日库存水位自动巡检
- 安全库存触发时的补货信号生成
- 超储物料预警与处置触发
- 库存周转率监控与异常分析
- 安全库存参数定期回顾与调整
- 生产计划变更后的库存影响评估

## 执行步骤

### 步骤1：库存数据获取与校验
- 从WMS/ERP获取实时库存数据：
  - 在手库存（On-hand）
  - 在途库存（In-transit/On-order）
  - 预留库存（Reserved for production orders）
  - 可用库存 = 在手 - 预留 + 在途
- 获取库存参数设置：安全库存/最高库存/再订货点
- 校验数据完整性（有无缺失/异常值）

### 步骤2：水位状态判定
对每个SKU进行状态判定：
- **断料风险（红色）**：可用库存 < 安全库存 × 50%，紧急补货
- **补货信号（橙色）**：可用库存 <= 再订货点，触发正常补货
- **正常水位（绿色）**：再订货点 < 可用库存 < 最高库存
- **超储预警（黄色）**：可用库存 > 最高库存
- **过量严重（紫色）**：可用库存 > 最高库存 × 150%，需评估消化方案

### 步骤3：补货量计算
- 对触发补货信号的物料：
  - 补货建议量 = 最高库存 - 可用库存（考虑在途）
  - 校对MOQ（最小起订量）和包装整数倍
  - 校对采购提前期是否能满足需求时间
  - 紧急补货标注需求优先级

### 步骤4：安全库存动态调整建议
- 基于最近6个月数据计算：
  - 平均日用量变化（需求是否稳定）
  - 采购提前期变化（供应商交期是否稳定）
  - 供应商交期达成率（交付可靠性）
- 安全库存 = 平均日用量 × (采购提前期 + 安全天数)
- 安全天数根据供应商可靠性动态调整：OTD>95%减少天数，OTD<90%增加天数

### 步骤5：库存周转率计算与分析
- 月度周转率 = 月消耗金额 / 月平均库存金额 × 12
- 对比目标值：原材料>=6次/年，成品>=12次/年
- 分析周转率偏低的物料原因（需求下降/安全库存过高/采购批量过大）
- 输出改善建议

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inventory_snapshot | Array | 是 | 实时库存快照（SKU/数量/金额） |
| inventory_parameters | Object | 是 | 库存参数（安全库存/最高/ROP） |
| consumption_history | Array | 是 | 历史消耗数据（6个月） |
| supplier_lead_times | Object | 否 | 供应商提前期数据 |
| production_forecast | Object | 否 | 未来生产计划预测 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| stock_status_dashboard | Object | 库存水位状态仪表板 |
| replenishment_signals | Array | 补货信号列表 |
| overstock_alerts | Array | 超储预警列表 |
| turnover_analysis | Object | 周转率分析报告 |
| parameter_adjustment_suggestions | Array | 安全库存调整建议 |

## 最佳实践
- 每日自动巡检确保及时发现库存异常
- A类物料（高价值）设更严格的库存上限控制
- 安全库存参数每季度回顾一次，避免固化不合时宜
- 新产品导入期给予较高安全库存，稳定后逐步优化
- 季节性物料需提前调整库存策略

## 约束与边界
- 不直接修改ERP中的库存参数（需通过审批流程变更）
- 不做入库/出库操作（仓库实际执行）
- 补货信号是建议而非自动下单（需采购计划专员确认）
- 库存金额计算基于标准成本或移动平均价
