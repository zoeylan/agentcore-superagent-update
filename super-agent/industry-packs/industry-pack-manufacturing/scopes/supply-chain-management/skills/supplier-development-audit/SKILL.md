# 供应商开发与审核管理

## 技能概述
管理供应商从发现到准入AVL的完整开发流程，包括潜在供应商寻源、初步资质筛选、现场质量体系审核（QSA）、试用考核评估和正式准入决策。确保新供应商符合企业质量和供应能力要求。

## 适用场景
- 新物料需求无现有合格供应商时启动寻源
- 降低单一来源风险需要开发替代供应商
- 现有供应商即将淘汰需储备替代
- 新项目/新产品导入时的供应商前期开发
- 年度供应商结构优化（增减供应商数量）

## 执行步骤

### 步骤1：需求分析与寻源
- 明确供应商开发需求（物料类型/技术要求/产能需求/地理位置偏好）
- 多渠道搜索潜在供应商：行业数据库、展会信息、同行推荐、自荐
- 初步匹配筛选（3-5家候选）：营业范围、资质证书、规模、区位

### 步骤2：初步资质审查
- 收集供应商基本信息：营业执照、体系认证（ISO 9001/IATF 16949）、主要客户、财务状况
- 评估技术能力：设备清单、工艺能力、研发投入、专利/技术资质
- 评估产能匹配度：现有产能利用率、为本企业预留产能比例
- 淘汰不满足基本门槛的候选（进入下一步2-3家）

### 步骤3：现场审核（QSA）
- 制定审核计划（审核日期/范围/团队组成/检查清单）
- 现场审核重点维度：
  - 质量管理体系成熟度（文档/记录/过程控制）
  - 生产设备与环境（设备精度/5S/防护措施）
  - 检验能力（检测设备/人员资质/标准品管理）
  - 物料管理（先进先出/可追溯性/仓储条件）
  - 人员能力（培训体系/关键岗位人员稳定性）
- 输出审核评分和问题清单（主要不符合项/次要不符合项/观察项）
- 审核判定：通过/有条件通过（整改后复审）/不通过

### 步骤4：试用考核
- 有条件准入（试用期通常3-6个月，3-5个批次）
- 试用期关注点：首批交付质量、交期达成、问题响应速度
- 试用期每月评估，记录表现数据
- 试用结束综合评估：是否转为正式供应商

### 步骤5：正式准入
- 汇总试用期数据和评估结论
- 提交AVL准入审批（需质量/采购/工程多方会签）
- 审批通过后更新AVL名录
- 建立供应商主数据（ERP中创建供应商编码/物料-供应商关系/价格协议）
- 开发周期管理：全流程4-8周目标

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| development_requirement | Object | 是 | 开发需求（物料/技术/产能要求） |
| candidate_suppliers | Array | 否 | 已知候选供应商（可为空启动寻源） |
| audit_criteria | Object | 是 | 审核标准和检查清单 |
| trial_period_config | Object | 否 | 试用期配置（时长/批次数/评估标准） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| sourcing_report | Object | 寻源结果报告 |
| qualification_assessment | Object | 资质审查评估 |
| audit_report | Object | 现场审核报告（含评分和问题清单） |
| trial_evaluation | Object | 试用期评估结论 |
| avl_admission_recommendation | Object | AVL准入建议 |

## 最佳实践
- 审核团队至少包含质量、采购、工程三方代表
- 审核前提供检查清单给供应商，体现透明合作理念
- 关键物料供应商必须通过现场审核，不可仅书面审查
- 试用期订单量逐步放量，不宜一次性大量导入
- 开发进度每周跟踪，避免无期限延长

## 约束与边界
- 不做最终准入决策（需多方会签审批）
- 不发放采购订单（由采购计划专员执行）
- 现场审核需实际执行（AI辅助制定计划和checklist，人工执行审核）
- 涉及保密协议（NDA）签署需法务参与
