# 采购订单执行跟踪

## 技能概述
对已下达的采购订单进行全生命周期跟踪管理，实时监控订单状态变化，预警交期风险，并在异常发生时提供应对建议。确保采购订单按时、按量、按质完成。

## 适用场景
- 每日采购订单状态巡检（自动化常规监控）
- 供应商确认交期后的计划符合性检查
- 交期风险预警与升级处理
- 订单执行异常的协调处理（延迟/短交/取消）
- 管理层采购执行情况汇报

## 执行步骤

### 步骤1：订单状态数据获取
- 从ERP采购模块获取所有活跃采购订单（PO）列表
- 获取每个PO的当前状态：已下单/已确认/生产中/已发货/在途/到货待检/已入库
- 获取供应商确认交期（Confirmed Delivery Date）
- 获取物料需求日期（Required Date from MRP/Production）

### 步骤2：交期风险识别
- 对比供应商确认交期与物料需求日期：
  - **绿色**：确认交期 <= 需求日期（正常）
  - **黄色**：确认交期 - 需求日期 <= 3天（轻微风险）
  - **红色**：确认交期 > 需求日期 + 3天（严重风险）
  - **灰色**：供应商未确认（超过3天未回复需催促）
- 检查历史交期达成率低的供应商订单（额外关注）

### 步骤3：异常分类与应对建议
- **交期延迟**：评估对生产计划的影响（是否导致停线），建议催货/调整排产/启动备选供应商
- **数量短交**：评估缺口数量能否通过安全库存覆盖，建议补货或调整生产批量
- **订单取消/变更**：评估是否需要重新寻源或调整采购策略
- **质量问题（来料不合格）**：标记联动质量Scope处理，评估补货时间

### 步骤4：跟踪报告生成
- 汇总活跃PO总数、本期关闭数、按时关闭率
- 列出TOP风险订单（按紧急程度排序）
- 计算OTD（准时交付率）趋势
- 标注需要人工干预的事项和建议动作

### 步骤5：预警通知与升级
- 黄色风险：通知采购计划专员关注
- 红色风险：立即通知采购经理并提供应对方案
- 生产停线风险：升级到供应链总监级别

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| active_po_list | Array | 是 | 活跃采购订单列表 |
| po_status_updates | Array | 是 | 订单状态更新数据 |
| production_schedule | Object | 是 | 生产排程（物料需求日期） |
| supplier_delivery_history | Object | 否 | 供应商历史交期表现 |
| inventory_buffer | Object | 否 | 当前安全库存可覆盖天数 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| order_status_summary | Object | 订单状态汇总统计 |
| risk_orders | Array | 风险订单列表（含风险等级和建议） |
| otd_metrics | Object | 准时交付率指标 |
| action_items | Array | 需人工处理的事项 |
| escalation_alerts | Array | 需升级处理的紧急事项 |

## 最佳实践
- 每日定时运行巡检（建议上午9:00），确保风险尽早暴露
- 对OTD低于90%的供应商自动增加关注频率
- 关键物料订单（A类物料）设更严格的预警阈值
- 跟踪记录保留完整时间线，便于争议时追溯

## 约束与边界
- 不直接与供应商沟通催货（需采购人员执行）
- 不修改生产排程（需由生产计划方决定调整）
- 不判定来料质量（由IQC执行检验判定）
- 预警信息需在4小时内完成确认闭环
