# 供应链压力测试

## 技能概述
通过设定各类极端情景，对供应链进行系统化压力测试，评估在不同风险场景下的供应链韧性水平，识别薄弱环节并提出改善优先级建议。帮助企业从"被动应对"转向"主动防御"。

## 适用场景
- 季度供应链韧性评估
- 年度供应链战略规划中的风险评估
- 重大供应链结构调整前的影响评估
- 外部环境重大变化时的快速压力测试
- 新产品/新市场上线前的供应链能力评估

## 执行步骤

### 步骤1：测试情景设计
设计多种压力测试情景（参考历史事件和潜在风险）：
- **单一供应商断供**：TOP供应商突然完全停止供货
- **区域性中断**：某一地区所有供应商同时受影响（地震/洪水/疫情封锁）
- **物流通道中断**：主要运输路线中断（港口罢工/运河堵塞）
- **需求突增**：客户订单突增30-50%超出预期
- **价格剧烈波动**：关键原材料价格上涨>20%
- **复合情景**：多个风险同时发生

### 步骤2：影响模拟分析
对每个情景进行量化模拟：
- **供应连续性影响**：
  - 受影响物料清单
  - 当前安全库存可覆盖天数
  - 替代方案切换时间
  - 预计停产风险天数 = max(0, 切换时间 - 库存覆盖天数)
- **成本影响**：
  - 替代供应商价差
  - 紧急空运/加急费用
  - 停产损失估算
- **客户影响**：
  - 受影响订单/客户数量
  - 预计交期延迟天数
  - 客户索赔风险

### 步骤3：韧性评分
对供应链整体和各环节进行韧性评分（0-100）：
- **替代性（30%）**：关键物料是否有替代供应商/替代物料
- **缓冲性（25%）**：安全库存是否充足覆盖中断期
- **恢复性（25%）**：中断后恢复正常供应的预计时间
- **适应性（20%）**：调整生产计划/产品组合以应对的灵活度
- 各环节评分 + 加权总分 = 供应链韧性指数

### 步骤4：薄弱环节识别
- 在压力测试中表现最差的物料/供应商/环节
- 单点故障识别（一旦失效即导致全面停顿的关键节点）
- 系统性风险集中度（如过度依赖某一地区/某一行业）
- 按影响×概率排列改善优先级

### 步骤5：改善建议输出
- 针对每个薄弱环节提出具体改善方案：
  - 开发替代供应商（附时间/成本估算）
  - 调整安全库存水位（量化调整量）
  - 签订保供协议（合约条款建议）
  - 物料设计变更以增加通用性
- 按成本效益比排列实施优先级
- 制定改善路线图（3个月/6个月/12个月里程碑）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supply_chain_structure | Object | 是 | 供应链结构数据（物料/供应商/产能/提前期） |
| inventory_data | Object | 是 | 当前库存和安全库存数据 |
| contingency_plans | Array | 否 | 现有应急预案 |
| test_scenarios | Array | 否 | 自定义测试情景（可使用默认情景） |
| historical_disruptions | Array | 否 | 历史中断事件记录 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| test_results | Array | 各情景测试结果 |
| resilience_score | Object | 供应链韧性评分 |
| weak_points | Array | 薄弱环节排名 |
| improvement_roadmap | Object | 改善路线图 |
| executive_summary | String | 管理层摘要 |

## 最佳实践
- 测试情景应基于实际风险而非臆想（参考本行业历史事件）
- 量化假设需合理：不过于乐观也不过于悲观
- 压力测试结果需行动化：识别出问题后必须有改善跟踪
- 每年至少进行2次全面压力测试，重大变化后增加临时测试
- 与管理层共享结果，争取资源投入改善

## 约束与边界
- 压力测试是模拟推演，不代表实际会发生
- 量化评估基于已知数据和合理假设，存在估算误差
- 改善建议的执行需要资源和预算支持（非零成本）
- 不做概率预测（不预测某事件何时发生），只评估发生后的影响
