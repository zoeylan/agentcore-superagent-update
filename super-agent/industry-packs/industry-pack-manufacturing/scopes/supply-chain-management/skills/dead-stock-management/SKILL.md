# 呆滞物料识别与处置管理

## 技能概述
系统化识别呆滞物料（超过180天无出库动态的库存），分析呆滞原因，制定差异化处置方案（消化/转用/降价/报废），并跟踪处置执行进度。减少库存资金占用，释放仓储空间。

## 适用场景
- 月度呆滞物料巡检与识别
- 呆滞物料处置方案制定与评审
- 呆滞物料金额控制与KPI跟踪
- 年终库存清理专项行动
- 物料变更/产品停产后的遗留库存处理

## 执行步骤

### 步骤1：呆滞物料识别
- 扫描全部库存SKU，筛选出最后出库日期距今>=180天的物料
- 按库龄分层统计：
  - 180-270天（轻度呆滞）
  - 270-365天（中度呆滞）
  - >365天（严重呆滞）
- 计算呆滞物料总金额及占总库存金额比例
- 按物料类型/供应商/产品线分类统计

### 步骤2：呆滞原因分析
对每个呆滞物料分析原因：
- **需求消失**：产品停产/客户流失/项目取消
- **过量采购**：MOQ超出实际需求/计划失误/一次性大量采购
- **工程变更**：ECN变更后旧版本物料剩余
- **质量封锁**：质量问题待处理但长期未决
- **预测偏差**：需求预测过于乐观导致库存积压
- **替代料上线**：新物料替代后旧物料未及时消化

### 步骤3：处置方案制定
根据原因和物料状态制定处置方案：
- **消化利用**：查找其他产品线/项目是否可使用，优先内部消化
- **降级使用**：降低使用标准（如正品降为样品/赠品）
- **退回供应商**：与供应商协商退货或换货（适用于采购失误/质量封锁）
- **对外销售**：通过二手市场/渠道商低价出售
- **报废处理**：无任何利用价值的物料，走报废审批流程
- 每种方案需评估预期回收金额和处置周期

### 步骤4：经济性评估
- 继续持有成本 = 库存金额 × 年持有成本率（约25%） × 预计持有时间
- 处置回收金额预估
- 处置净损失 = 账面价值 - 预期回收金额
- 对比：继续持有的沉没成本 vs 立即处置的一次性损失
- 输出经济性建议

### 步骤5：处置跟踪与闭环
- 处置方案审批后建立执行跟踪台账
- 每月更新处置进度（已消化/已退货/已报废数量）
- 处置超期（>90天未完成）预警
- 处置完成后更新库存账面、关闭呆滞物料标记
- 统计报告：月度新增呆滞 vs 处置完成 vs 在途处理

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inventory_aging_data | Array | 是 | 库存库龄数据（物料/数量/金额/最后出库日期） |
| material_usage_forecast | Object | 否 | 未来需求预测（判断是否有消化可能） |
| ecn_change_log | Array | 否 | 工程变更记录（关联呆滞原因） |
| product_lifecycle_status | Object | 否 | 产品生命周期状态（在产/停产） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| dead_stock_list | Array | 呆滞物料清单（含库龄/金额/原因） |
| disposal_plans | Array | 处置方案（含方式/预期回收/周期） |
| financial_impact | Object | 财务影响评估（损失/回收/净影响） |
| tracking_status | Object | 在途处置进度汇总 |
| prevention_suggestions | Array | 预防新增呆滞的改善建议 |

## 最佳实践
- 预防优于处置：建立物料导入时的退出机制（新物料上线时约定未消耗部分的处理方式）
- 呆滞物料月度review机制化，避免积压越来越严重
- 报废审批设置金额分级：<1万部门经理，1-5万总监，>5万总经理
- ECN变更时同步制定旧物料消化计划
- 与供应商合同中约定MOQ超量部分的退货条款

## 约束与边界
- 报废决策需财务和管理层审批，本技能仅提供方案建议
- 退供应商方案需确认合同条款支持
- 呆滞物料金额涉及资产减值，需财务部门配合处理
- 质量封锁状态物料处置需质量部门释放或判定
