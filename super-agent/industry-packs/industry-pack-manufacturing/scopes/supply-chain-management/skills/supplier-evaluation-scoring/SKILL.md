# 供应商绩效评分

## 技能概述
执行季度供应商绩效综合评分，基于质量（40%）、交期（30%）、成本（20%）、服务（10%）四维度权重体系，客观量化供应商表现，驱动供应商分级管理和改善/淘汰决策。

## 适用场景
- 季度末供应商绩效评估（常规周期性任务）
- 新供应商试用期满后的转正评估
- 供应商异常事件后的临时评估
- 年度供应商大会前的绩效汇总
- 采购策略调整前的供应商能力评估

## 执行步骤

### 步骤1：评分数据采集
- **质量维度（40%）**：
  - 从IQC系统获取来料批次合格率（权重60%）
  - 获取质量问题响应速度（SCAR回复天数，权重20%）
  - 获取改善措施有效性（同类问题复发率，权重20%）
- **交期维度（30%）**：
  - 从ERP获取订单准时交付率OTD（权重70%）
  - 获取订单确认响应时间（权重15%）
  - 获取交期承诺准确率（权重15%）
- **成本维度（20%）**：
  - 计算年度降价配合度（是否达成年度降价目标，权重50%）
  - 评估报价响应速度和竞争力（权重30%）
  - 评估隐性成本（退货/重检/加急费用，权重20%）
- **服务维度（10%）**：
  - 技术支持响应能力（权重40%）
  - 沟通配合度（权重30%）
  - 产能弹性/紧急订单配合度（权重30%）

### 步骤2：评分计算
- 各子维度按百分制评分（0-100分）
- 子维度加权得到维度得分
- 四维度按权重加权得到综合得分：总分 = 质量×0.4 + 交期×0.3 + 成本×0.2 + 服务×0.1
- 计算得分同比/环比变化趋势

### 步骤3：等级判定
- **A级（>=80分）**：优选供应商，优先分配订单，可考虑战略合作
- **B级（60-79分）**：合格供应商，正常合作，鼓励提升
- **C级（40-59分）**：整改供应商，限期改善（15天内发出整改通知）
- **D级（<40分）**：淘汰候选，评估替代来源后启动退出

### 步骤4：结果输出与行动触发
- 生成供应商评分卡（Scorecard）
- 对C级供应商：自动生成整改通知书模板（含具体问题点和改善目标）
- 对D级供应商：触发淘汰评审流程（先确认替代来源）
- 对连续两季度C级的供应商：升级为淘汰候选
- 标注评分与上季度对比的显著变化项

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplier_id | String | 是 | 供应商编码 |
| quality_data | Object | 是 | 质量维度数据（IQC合格率/SCAR响应/复发率） |
| delivery_data | Object | 是 | 交期维度数据（OTD/确认时间/承诺准确率） |
| cost_data | Object | 是 | 成本维度数据（降价/报价/隐性成本） |
| service_data | Object | 部分 | 服务维度数据（可含人工评分） |
| previous_quarter_score | Number | 否 | 上季度评分（用于趋势对比） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| overall_score | Number | 综合得分（0-100） |
| dimension_scores | Object | 四维度各项明细得分 |
| grade | String | 等级（A/B/C/D） |
| trend | String | 趋势（上升/持平/下降） |
| action_required | Array | 需触发的行动项 |
| scorecard_report | Object | 完整评分卡 |

## 最佳实践
- 数据优先使用系统自动采集数据（ERP/QMS），减少主观评分占比
- 服务维度的人工评分需多人参与（采购+质量+工程），避免个人偏见
- 评分结果需在正式沟通前先内部校验，确保数据准确
- 新品导入期供应商给予3-6个月观察期，不纳入正式评分

## 约束与边界
- 评分数据采集期为完整季度（不跨季度混合）
- 服务维度的主观评分需注明评分人和评分依据
- 淘汰决策不由本技能直接执行，仅触发评审流程
- 样本量不足（季度内交货<3次）的供应商标注"数据不足"
