# 供应中断应急预案管理

## 技能概述
为关键物料建立和维护供应中断应急预案，确保当单一供应商或物流通道中断时，有明确的替代方案和响应流程。通过预案的制定、维护和演练验证，提升供应链面对突发事件的韧性。

## 适用场景
- 新关键物料纳入应急预案覆盖
- 现有应急预案半年度review和更新
- 供应中断事件发生时的预案激活与执行
- 供应链压力测试中的预案有效性验证
- 供应商结构变更后的预案同步调整

## 执行步骤

### 步骤1：关键物料识别
- 筛选需要建立应急预案的物料：
  - Kraljic矩阵中的战略型和瓶颈型物料
  - 单一来源供应的物料
  - 采购提前期>4周的物料
  - 对生产连续性影响高的物料（缺料即停线）
- 确定预案优先级排序

### 步骤2：应急预案编制
每个关键物料的应急预案包含：
- **风险场景定义**：明确什么情况触发预案（完全断供/部分供应不足/质量问题拒收）
- **替代供应商方案**：
  - 首选替代供应商（AVL中已有/正在开发中）
  - 替代供应商切换时间（从下单到首批到货的时间）
  - 切换成本预估（价差/验证费用/模具费等）
- **替代物料方案**：
  - 功能等效替代料（需工程验证）
  - 替代料验证周期和费用
  - 客户端是否需要变更通知
- **库存覆盖评估**：
  - 当前安全库存可覆盖天数
  - 覆盖天数 vs 替代方案切换时间的gap分析
  - 是否需要调整安全库存以覆盖切换期
- **响应流程与责任**：
  - 触发条件和决策权限
  - 各阶段责任人和行动时限
  - 升级路径和沟通计划

### 步骤3：预案有效性验证
- 半年度检查每个预案的有效性：
  - 替代供应商是否仍在AVL中且状态正常
  - 替代供应商产能是否仍可支持
  - 安全库存覆盖天数是否仍然充足
  - 联系人信息是否有效
- 对关键预案进行桌面演练（tabletop exercise）
- 更新失效的预案内容

### 步骤4：预案激活执行
当实际供应中断发生时：
- 快速定位受影响物料的应急预案
- 评估当前库存覆盖剩余天数
- 启动预案中的替代方案（优先级排序执行）
- 协调相关方（采购/工程/质量/生产）快速响应
- 记录执行过程和调整偏差

### 步骤5：预案维护与改进
- 每次预案激活后进行复盘：响应时间/方案有效性/改进点
- 根据供应商变化（新增/淘汰/绩效变化）更新预案
- 纳入新识别的风险场景
- 保持预案文档版本管理

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| critical_materials | Array | 是 | 关键物料清单 |
| avl_data | Object | 是 | 合格供应商名录 |
| current_inventory | Object | 是 | 当前库存和安全库存数据 |
| supplier_capacity_info | Object | 否 | 供应商产能信息 |
| alternative_materials | Array | 否 | 可替代物料信息 |
| lead_time_data | Object | 否 | 各供应商采购提前期 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| contingency_plans | Array | 应急预案文档集合 |
| coverage_gap_analysis | Object | 库存覆盖vs切换时间gap分析 |
| validation_status | Object | 预案有效性验证状态 |
| activation_playbook | Object | 预案激活操作手册 |
| improvement_items | Array | 预案改进事项 |

## 最佳实践
- 预案必须可执行：每个步骤有明确责任人和时限
- 安全库存覆盖天数 > 替代方案切换时间，确保无缝衔接
- 替代供应商定期保持小量交易，避免"纸面供应商"
- 应急预案不只是文档，需定期演练验证
- 预案要考虑极端场景（区域性灾害导致多家供应商同时中断）

## 约束与边界
- 预案建议需管理层审批后生效
- 替代物料方案需工程和质量部门技术验证
- 预案激活决策权在供应链总监或以上
- 预案中的成本评估为预估值，实际执行可能偏差
