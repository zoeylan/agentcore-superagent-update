# MRP采购需求分析

## 技能概述
基于MRP（物料需求计划）运算结果和安全库存策略，分析物料需求并生成采购建议单。该技能是采购计划编制的核心起点，将系统化的需求数据转化为可执行的采购行动方案。

## 适用场景
- MRP运算完成后生成采购建议（常规周度/月度计划）
- 安全库存水位触发补货信号
- 紧急需求（插单、工程变更导致的临时需求）
- 生产计划变更后的采购计划联动调整

## 执行步骤

### 步骤1：需求数据采集
- 从ERP系统获取MRP运算输出的计划订单（planned orders）
- 获取当前库存水位数据（可用库存 = 在手库存 - 预留库存 + 在途库存）
- 获取安全库存参数和再订货点设置
- 确认生产计划排程（需求日期和数量）

### 步骤2：需求净计算
- 净需求 = 毛需求 - 可用库存 - 在途订单 + 安全库存
- 过滤已被覆盖的需求（在途订单可满足）
- 合并同一物料在时间窗口内的多次小需求（批量合并逻辑）
- 标注紧急需求（需求日期 - 今天 < 正常采购提前期）

### 步骤3：Kraljic物料分类与策略匹配
- 根据物料在Kraljic矩阵中的分类确定采购策略：
  - **战略型**（高利润影响+高供应风险）：匹配年度合约，按滚动预测下达Forecast
  - **杠杆型**（高利润影响+低供应风险）：启动竞价比价流程
  - **瓶颈型**（低利润影响+高供应风险）：确保安全库存充足，匹配长期保供合约
  - **常规型**（低利润影响+低供应风险）：框架协议自动补货

### 步骤4：供应商匹配与建议生成
- 从AVL中筛选该物料的合格供应商
- 按供应商绩效评分排序，优先推荐高评分供应商
- 对关键物料检查多源要求（是否有>=2家供应商）
- 生成采购建议单（含物料编码、数量、需求日期、建议供应商、预估金额）

### 步骤5：审批层级判定
- 汇总单次采购建议金额
- 匹配审批层级：<10万直接下单，10-50万采购经理，>=50万供应链总监
- 标注审批路径到采购建议单

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mrp_output | Array | 是 | MRP运算结果（物料/数量/需求日期） |
| inventory_data | Object | 是 | 当前库存水位（在手/在途/预留） |
| safety_stock_params | Object | 是 | 安全库存参数（日用量/提前期/安全天数） |
| avl_list | Array | 是 | 合格供应商名录 |
| supplier_scores | Object | 否 | 供应商绩效评分（用于优先级排序） |
| kraljic_classification | Object | 否 | 物料Kraljic分类 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| procurement_suggestions | Array | 采购建议单列表 |
| urgent_items | Array | 紧急采购项（提前期不足） |
| approval_level | String | 审批层级建议 |
| risk_alerts | Array | 风险提示（单一来源/无供应商等） |
| total_amount | Number | 预估采购总金额 |

## 最佳实践
- 净需求计算时务必扣除在途订单，避免重复采购
- 紧急需求单独标注并走快速通道，不与常规需求混合
- 批量合并时考虑MOQ（最小起订量）和包装单位
- 定期校验安全库存参数的合理性（每季度回顾）

## 约束与边界
- 不执行实际下单操作，仅生成建议供人工审批
- 不修改AVL名录，仅从中筛选供应商
- 金额预估基于最近一次采购价格，非最终成交价
- 紧急需求需标注可能的额外成本风险（加急费/空运费）
