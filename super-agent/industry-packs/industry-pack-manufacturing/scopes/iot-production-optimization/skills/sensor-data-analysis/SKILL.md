# 传感器数据分析 (Sensor Data Analysis)

## 概述
对工业IoT传感器采集的多维度实时数据流进行处理、清洗、特征提取和趋势分析，为异常检测和健康度评估提供高质量数据基础。

## 适用场景
- 接收振动、温度、电流、压力等多类型传感器的实时数据流
- 数据预处理（去噪、插值、标准化）
- 时域/频域特征提取（RMS、峰值因子、频谱分析）
- 数据质量监控（传感器离线检测、数据漂移识别）

## 执行步骤

### 步骤1：数据接入与验证
- 从OPC-UA/MQTT数据通道接收传感器原始数据
- 验证数据完整性：检查时间戳连续性、数值范围合理性
- 标记异常数据点（超出物理范围的值、NaN、重复时间戳）

### 步骤2：数据清洗与预处理
- 去除噪声：应用低通滤波器（针对振动数据）或中值滤波（针对温度数据）
- 缺失值处理：短时缺失（<5秒）线性插值，长时缺失标记为数据中断事件
- 数据标准化：将不同量纲的传感器数据归一化到可比较的尺度

### 步骤3：特征提取
- 时域特征：均值、标准差、RMS、峰值、峰值因子、峭度
- 频域特征（振动数据）：FFT频谱、主频、谐波能量占比
- 趋势特征：滑动窗口均值变化率、累积偏移量

### 步骤4：数据质量报告
- 生成数据质量评分（完整率、有效率、时效性）
- 对数据质量低于95%的传感器触发数据质量告警
- 输出清洗后的结构化数据供下游分析使用

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一编号 |
| sensor_type | enum | 传感器类型（vibration/temperature/current/pressure） |
| raw_data | array | 原始采集数据数组（含时间戳和值） |
| sampling_rate | number | 采样频率（Hz） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一编号 |
| features | object | 提取的特征集合（时域+频域） |
| data_quality_score | number | 数据质量评分（0-100） |
| quality_issues | array | 数据质量问题列表 |
| processed_at | timestamp | 处理完成时间 |

## 最佳实践
- 关键参数（振动/温度/电流）保持每秒采集频率，一般参数每分钟即可
- 频域分析窗口长度建议为采样频率的10倍以上以保证频率分辨率
- 数据清洗规则需定期校验，避免过度平滑掩盖真实异常
- 保留原始数据和清洗后数据的双份存储，支持追溯分析

## 约束条件
- 数据处理延迟不得超过5秒（从采集到特征输出）
- 不得修改原始数据存档，清洗操作只在副本上进行
- 传感器离线超过60秒必须立即触发数据质量告警
