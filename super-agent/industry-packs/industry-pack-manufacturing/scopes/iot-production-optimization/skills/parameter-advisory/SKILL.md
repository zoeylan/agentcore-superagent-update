# 参数调整建议 (Parameter Advisory)

## 概述
在用户权限验证通过后，针对设备运行参数提供查看和有限范围内的调整建议，确保参数调整在安全范围内并有明确的调整依据。

## 适用场景
- 用户询问设备参数当前设定值和允许范围
- 工程师寻求参数优化建议以提升产品质量或效率
- 设备运行异常时的参数排查和调整建议
- 新产品导入时的工艺参数推荐

## 执行步骤

### 步骤1：权限验证
- 确认用户权限等级：
  - 操作工：仅查看参数，不提供调整建议
  - 班组长：查看参数+有限调整建议（仅日常调整范围内）
  - 工程师：查看参数+全范围调整建议（含工艺优化）
  - 管理员：全部权限
- 涉及安全参数（温度上限/压力上限/速度限制），任何调整建议都需标注"需工程师确认后执行"

### 步骤2：参数信息呈现
- 呈现完整的参数信息包：
  - 参数名称和含义说明
  - 当前设定值
  - 设备规格书允许范围（绝对上下限）
  - 工艺推荐范围（基于产品工艺卡的建议范围）
  - 历史最优值（历史生产中产出最好时的参数值）
- 区分只读参数（设备固有属性）和可调参数

### 步骤3：调整建议生成
- 分析调整需求的背景（解决什么问题/达成什么目标）
- 基于历史数据和工艺知识生成调整建议：
  - 目标值：建议调整到的数值
  - 调整幅度：单次调整不超过允许范围的10%（渐进调整原则）
  - 调整依据：为什么建议这样调整
  - 预期效果：调整后预期的改善
  - 风险提示：调整可能带来的副作用
- 对于不确定的调整，建议小范围试验验证

### 步骤4：安全保障
- 所有调整建议附带安全审查：
  - 是否超出设备规格书允许范围 → 拒绝
  - 是否影响产品质量安全 → 需质量工程师确认
  - 是否影响人员安全 → 需安全主管确认
  - 是否影响设备寿命 → 需设备工程师确认
- 建议不直接执行，输出到待确认队列

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备编号 |
| parameter_name | string | 目标参数名称 |
| user_role | enum | 用户权限等级 |
| adjustment_purpose | string | 调整目的/要解决的问题 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| parameter_info | object | 参数完整信息包 |
| adjustment_suggestion | object | 调整建议（目标值/依据/预期效果） |
| safety_check_result | object | 安全审查结果 |
| approval_required | array | 需要的审批确认列表 |
| risk_warnings | array | 风险提示 |

## 最佳实践
- 遵循渐进调整原则：每次只调整一个参数，调整幅度不超过10%
- 调整后建议观察一定时间（至少一个生产批次）验证效果
- 记录每次参数调整的前后对比，建立参数优化知识库
- 对于多参数联合调整需求，建议DOE实验设计方法

## 约束条件
- 不得提供超出设备规格书允许范围的参数建议
- 安全相关参数调整建议必须标注"需人工确认后执行"
- 不直接执行参数修改，仅提供建议
- 所有调整建议必须记录审计日志（谁/何时/什么参数/调整建议/是否执行）
