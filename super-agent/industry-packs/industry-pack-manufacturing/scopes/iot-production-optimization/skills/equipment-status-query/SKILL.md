# 设备状态查询 (Equipment Status Query)

## 概述
响应操作人员和工程师的自然语言查询请求，从IoT数据平台实时获取设备运行状态信息，以清晰易懂的格式返回查询结果。

## 适用场景
- 单设备实时状态查询（"3号注塑机当前温度是多少"）
- 设备群组状态汇总（"B区有哪些设备处于告警状态"）
- 历史数据查询（"A线昨天的OEE是多少"）
- 设备对比查询（"1号和2号CNC哪台利用率更高"）
- 趋势查询（"5号设备本周温度变化趋势如何"）

## 执行步骤

### 步骤1：查询意图解析
- 解析用户自然语言查询，提取关键要素：
  - 目标设备：设备编号/名称/区域
  - 查询参数：温度/振动/电流/压力/OEE/健康度/运行状态
  - 时间范围：实时/历史时段（昨天/本周/上月）
  - 查询类型：单值/趋势/对比/汇总
- 处理模糊查询：补全缺失信息或请求用户澄清

### 步骤2：权限验证
- 验证查询用户的身份和权限等级
- 权限级别对应的查询范围：
  - 操作工：本岗位相关设备的基本运行参数
  - 班组长：本班组所有设备的运行参数和OEE
  - 工程师：全厂设备的详细参数含历史数据
  - 管理员：全部数据含敏感参数
- 越权查询拒绝并提示需要更高权限

### 步骤3：数据获取与整合
- 从IoT数据平台API获取目标设备的实时/历史数据
- 数据整合：
  - 单值查询：获取最新数据点
  - 趋势查询：获取时间序列并计算统计特征
  - 汇总查询：聚合多台设备数据
  - 对比查询：并排获取多台设备同一指标
- 附加上下文信息：参数正常范围、当前健康度评分

### 步骤4：结果格式化输出
- 以用户友好的格式返回查询结果：
  - 设备名称/编号
  - 查询参数当前值 + 正常范围
  - 状态标识：✅正常 / ⚠️注意 / 🔴异常
  - 趋势查询附带文字描述（"过去24小时温度从35°C缓慢上升至38°C"）
- 对异常数值主动提示可能原因和建议动作

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| user_query | string | 用户自然语言查询文本 |
| user_id | string | 用户身份标识 |
| user_role | enum | 用户权限等级 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| query_result | object | 查询结果数据 |
| device_info | object | 设备信息 |
| status_indicator | enum | 状态标识（normal/attention/abnormal） |
| normal_range | object | 参数正常范围 |
| context_note | string | 上下文说明/建议 |
| data_freshness | timestamp | 数据时效性（最后更新时间） |

## 最佳实践
- 回答应简洁直接，避免不必要的技术术语
- 异常值主动标注并附带简要说明，不需要用户追问
- 数据不可用时（传感器离线），明确告知并说明原因
- 支持连续对话（基于上下文的追问："那振动呢？"）

## 约束条件
- 查询响应时间<3秒
- 不得返回未经权限验证的敏感数据
- 历史数据查询范围最长90天（更早数据需走报表系统）
- 数据显示必须标注时间戳，避免用户混淆实时和历史数据
