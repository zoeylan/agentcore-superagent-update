# 排产优化 (Production Scheduling)

## 概述
综合订单优先级、设备产能、换线时间、物料齐套率和设备健康状况等多维约束，生成最优的生产排产方案，最大化产出效率同时满足客户交期要求。

## 适用场景
- 新订单/插单时的排产方案快速生成
- 设备故障或维护导致产能变化时的排产调整
- 周/日生产计划的优化编排
- 多产品混合生产线的排产决策
- 排产方案对比评估（当前方案vs优化方案）

## 执行步骤

### 步骤1：约束信息采集
- **订单信息**：订单列表、产品型号、数量、客户交期、优先级
- **设备产能**：各设备/产线可用时间、标准节拍、当前排程状态
- **换线时间**：产品切换矩阵（从产品A切换到产品B的时间）
- **物料齐套**：各订单BOM物料到位情况（齐套率%）
- **设备健康度**：来自IoT系统的设备可靠性预测（是否有维护计划影响可用时间）
- **人员排班**：各班次/区域的人员配置情况

### 步骤2：排产方案生成
- 应用约束满足算法（CSP）确保方案满足所有硬约束：
  - 硬约束：交期不能违反、设备产能不能超载、物料未齐套不排产
- 在满足硬约束基础上优化软目标：
  - 最小化总换线时间（相似产品连续排产）
  - 最大化设备利用率
  - 均衡各设备负载
  - 最小化在制品WIP
- 生成2-3个可选方案供决策者选择

### 步骤3：方案评估与对比
- 对每个方案计算关键指标：
  - 交期满足率（%的订单能准时交付）
  - 设备利用率（平均/最大/最小）
  - 总换线时间和次数
  - 预计完成时间
  - OEE预估值
- 与当前排产方案对比，量化改善幅度
- 标注方案风险点（如依赖某设备高可用率、物料齐套临界等）

### 步骤4：方案输出与确认
- 输出推荐方案及其选择理由
- 以甘特图格式描述排产时间线
- 标注关键里程碑和决策点
- 提交给生产计划员人工审批确认
- 方案确认后输出到MES系统执行

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| orders | array | 待排产订单列表（产品/数量/交期/优先级） |
| equipment_capacity | object | 设备产能和可用时间 |
| changeover_matrix | object | 换线时间矩阵 |
| material_availability | object | 物料齐套状态 |
| device_health_forecast | object | 设备健康度和维护计划 |
| current_schedule | object | 当前排程状态 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| recommended_plan | object | 推荐排产方案 |
| alternative_plans | array | 备选方案 |
| plan_comparison | object | 方案对比指标 |
| gantt_description | object | 甘特图描述 |
| risk_notes | array | 方案风险提示 |
| improvement_vs_current | object | 对比当前方案的改善点 |

## 最佳实践
- 排产方案必须考虑四要素（订单/产能/换线/物料），缺一不可
- 为设备维护预留时间窗口，不要将排产排满100%
- 插单场景下快速响应（2小时内出方案），但要评估对现有订单的影响
- 排产方案采纳率目标>=70%，低于此应分析被拒原因改善方案质量

## 约束条件
- 排产方案不得安排在物料未确认齐套的情况下开工
- 方案必须经过人工审批后方可下达MES执行
- 涉及加班排产需额外标注人工成本影响
- 排产方案变更需记录变更原因和影响评估
