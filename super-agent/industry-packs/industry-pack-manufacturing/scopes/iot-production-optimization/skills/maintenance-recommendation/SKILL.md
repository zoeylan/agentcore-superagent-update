# 维护建议生成 (Maintenance Recommendation)

## 概述
基于RUL预测结果、设备健康度趋势和生产排程信息，生成结构化的预测性维护建议，包含维护时间窗口、所需资源和预期收益评估，为维护计划调度提供决策依据。

## 适用场景
- 基于RUL预测的定期维护建议生成
- 设备健康度跌破阈值时的紧急维护建议
- 维护时机与生产排程的协调优化
- 维护资源（备件、人力、工具）的提前准备建议

## 执行步骤

### 步骤1：维护需求评估
- 综合以下信息判断维护必要性和紧迫性：
  - RUL预测结果（剩余寿命区间和置信度）
  - 当前健康度评分和退化速率
  - 设备关键程度等级（A类/B类）
  - 历史故障后果严重性（停机时间、质量影响、安全风险）
- 计算维护优先级评分 = f(紧迫性, 设备关键度, 故障后果)

### 步骤2：维护方案制定
- 确定维护类型：部件更换、润滑保养、校准调整、预防性大修
- 评估所需资源：
  - **备件清单**：从BOM和历史维修记录确定所需备件及数量
  - **人力需求**：维修工种、技能要求、预估工时
  - **工具设备**：是否需要特殊工具或外部支持
  - **停机需求**：是否需要停机、预估停机时长
- 检查备件库存：确认所需备件是否在库，不在库则需提前采购

### 步骤3：时间窗口优化
- 基于RUL建议的维护时间窗口，对照生产排程寻找最优时机：
  - 优先选择计划停机时段（周末保养、换线间隙）
  - 其次选择低负荷时段（产量较低的班次）
  - 最后考虑额外停机（需评估停机成本vs故障风险成本）
- 输出推荐维护日期/时段及其选择理由

### 步骤4：建议输出与推送
- 生成标准化维护建议文档，推送给维护计划调度员
- 内容格式符合CMMS工单创建要求
- 标注建议的紧急程度和决策截止时间（超过此日期需重新评估）
- 附带成本效益分析：预防性维护成本 vs 故障维修成本对比

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备编号 |
| rul_prediction | object | RUL预测结果 |
| health_score | object | 当前健康度及趋势 |
| production_schedule | object | 未来2周生产排程 |
| spare_parts_inventory | object | 备件库存状态 |
| maintenance_crew_availability | object | 维修人员可用性 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| recommendation_id | string | 建议编号 |
| priority_score | number | 优先级评分（1-10） |
| maintenance_type | string | 维护类型 |
| recommended_date | date | 推荐维护日期 |
| time_window | object | 维护时间窗口（earliest/latest） |
| required_parts | array | 所需备件清单 |
| estimated_duration | number | 预估工时（小时） |
| estimated_downtime | number | 预估停机时间（小时） |
| cost_benefit | object | 成本效益分析 |
| decision_deadline | date | 决策截止日期 |

## 最佳实践
- 维护建议提前期至少7天，给备件采购和人员安排留出时间
- 多台设备的维护建议尽量合并安排，减少停机次数
- 成本效益分析是说服生产部门接受停机的关键依据
- 建议采纳率目标>=70%，低于此应分析被拒原因并优化建议质量

## 约束条件
- 维护建议不得直接转化为工单执行，必须经人工审批确认
- 紧急维护建议（健康度<40）的决策窗口不超过24小时
- 备件不在库时必须标注，不得假设备件可用
- 建议必须附带不执行维护的风险说明（潜在故障后果和概率）
