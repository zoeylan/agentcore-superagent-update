# 故障排查指导 (Troubleshooting Guide)

## 概述
基于设备操作手册、故障代码库和历史维修记录，为操作人员提供智能化的故障排查步骤指导，帮助一线人员快速定位和解决常见设备问题。

## 适用场景
- 设备报错代码解读和排查步骤指导
- 设备异常现象（异响/异味/泄漏/精度偏差）的排查建议
- 日常操作中遇到问题的解决方案查询
- 维护保养操作的步骤指导

## 执行步骤

### 步骤1：问题信息采集
- 识别用户描述的问题类型：
  - 设备报错（有明确错误代码）
  - 异常现象描述（无明确代码，凭感官观察）
  - 操作困难（不知如何执行某操作）
- 提取关键信息：设备型号、错误代码/现象描述、发生时间、操作上下文
- 必要时追问确认：是否有安全风险（异味/烟雾/泄漏）

### 步骤2：安全优先判断
- **安全风险评估**：
  - 如涉及安全风险（烟雾、泄漏、异味、电气火花）→ 立即建议：停机→断电→撤离→通知安全人员
  - 如无安全风险 → 继续排查指导
- 安全相关问题不尝试远程排查，直接升级到专业人员

### 步骤3：排查方案生成
- 从故障知识库中匹配最相关的排查方案：
  - 错误代码：精确匹配代码定义和推荐排查步骤
  - 异常现象：模糊匹配历史类似案例和解决方案
- 生成结构化排查步骤：
  1. 每步操作清晰具体（做什么、在哪做、怎么做）
  2. 每步预期结果（正常应该看到什么）
  3. 如果不正常→下一步检查什么
  4. 标注哪些步骤需要停机、需要工具、需要更高权限
- 排查步骤按可能性从高到低排序（最常见原因优先检查）

### 步骤4：升级与反馈
- 如果排查步骤全部执行后问题仍未解决：
  - 建议联系维修专业人员（提供设备编号和已排查步骤摘要）
  - 自动将问题信息转发给故障诊断专家
- 记录排查结果反馈（解决/未解决/升级），用于优化知识库
- 对于首次出现的问题，标记为"新问题"待维修人员解决后补充知识库

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备编号 |
| device_model | string | 设备型号 |
| error_code | string | 错误代码（可选） |
| symptom_description | string | 异常现象描述 |
| user_role | enum | 用户角色（决定排查深度） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| safety_assessment | enum | 安全评估（safe/caution/danger） |
| troubleshooting_steps | array | 排查步骤列表 |
| probable_causes | array | 可能原因排序 |
| tools_required | array | 所需工具 |
| estimated_time | number | 预估排查时间（分钟） |
| escalation_needed | boolean | 是否需要升级到专业人员 |

## 最佳实践
- 排查步骤要考虑用户的技能水平，操作工给更详细的步骤指导
- 安全永远第一，任何有安全风险的操作都需要明确警告
- 建议从简单检查开始（视觉检查→手动操作→仪器测量），由浅入深
- 记录每次排查过程和结果，持续丰富故障知识库

## 约束条件
- 涉及安全风险的问题必须首先建议停机，不得尝试在线排查
- 不得指导用户执行超出其资质范围的操作（如电气维修需持证）
- 排查指导不替代专业维修，复杂问题必须升级
- 涉及拆卸设备的步骤必须标注"需停机并执行LOTO锁定"
