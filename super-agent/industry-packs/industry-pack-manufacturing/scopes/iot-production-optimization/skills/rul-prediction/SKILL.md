# 剩余使用寿命预测 (RUL Prediction)

## 概述
基于设备历史退化数据和实时传感器特征，运用统计模型和深度学习算法预测设备关键部件的剩余使用寿命（Remaining Useful Life），输出RUL区间估计和维护时间窗口建议。

## 适用场景
- 轴承、齿轮箱、电机等旋转部件的寿命预测
- 刀具、模具等易损件的更换时机预测
- 密封件、滤芯等耗材的更换周期优化
- 基于健康度退化趋势的剩余寿命外推

## 执行步骤

### 步骤1：退化特征构建
- 从传感器特征集中选取与退化相关的关键指标（退化指示器/Health Indicator）
- 常用退化指示器：
  - 轴承：振动RMS趋势、峭度增长、特征频率能量增长
  - 电机：绕组电阻变化率、电流不平衡增长率
  - 齿轮箱：齿轮啮合频率边带能量
- 构建退化曲线：将退化指示器随时间的变化可视化

### 步骤2：模型选择与预测
- **统计方法**：Weibull分布拟合历史寿命数据，条件概率更新
- **退化模型**：基于Wiener过程/Gamma过程的随机退化模型
- **深度学习**：LSTM/Transformer对多维时序特征进行端到端RUL预测
- 选择策略：数据充足（>100个故障样本）用深度学习，数据有限用统计方法
- 输出RUL点估计值和置信区间（如P10/P50/P90）

### 步骤3：置信度评估
- 评估预测结果的可信程度：
  - 高置信（>85%）：退化模式与历史故障高度匹配，趋势明确
  - 中置信（50-85%）：退化特征有变化但模式不完全匹配
  - 低置信（<50%）：数据不足或处于模型训练域外
- 对低置信预测标注"仅供参考"，不作为维护决策唯一依据

### 步骤4：维护窗口建议
- 基于RUL预测结果生成维护时间窗口：
  - 最早维护时间：P90（90%概率故障不会发生在此之前）
  - 推荐维护时间：P50（中位数寿命点）前2天
  - 最迟维护时间：P10（需要在此之前完成维护以避免故障）
- 结合生产排程建议最优维护时机（在计划停机或换线间隙安排）

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备编号 |
| component | string | 预测的部件名称 |
| degradation_features | array | 退化特征时序数据 |
| historical_failures | array | 该类设备/部件历史故障数据 |
| current_health_score | number | 当前健康度评分 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| rul_estimate | object | RUL估计（P10/P50/P90，单位：天） |
| confidence_level | enum | 置信度等级（high/medium/low） |
| maintenance_window | object | 建议维护时间窗口（earliest/recommended/latest） |
| failure_mode | string | 预测的故障模式 |
| key_indicators | array | 驱动预测的关键退化指标 |

## 最佳实践
- 预测准确率目标>=80%（以实际故障时间落在预测区间内为准）
- 目标提前期7天以上，给维护计划留出充分准备时间
- 对同一设备的多个部件分别预测，取最短RUL作为设备整体RUL
- 每次维护/更换后重置退化基线，开始新的RUL跟踪周期

## 约束条件
- 新安装部件（服役<30天）不进行RUL预测（退化数据不足）
- 预测结果不得作为唯一决策依据，必须结合人工经验判断
- 模型训练数据覆盖范围外的工况，预测结果需标注为"域外推断"
- 连续3次预测偏差>30%需触发模型重训练流程
