# 异常检测 (Anomaly Detection)

## 概述
应用统计规则和机器学习算法对设备传感器数据进行实时异常检测，识别设备运行中的偏离正常模式的行为，为告警生成和故障预防提供依据。

## 适用场景
- 单参数阈值突破检测（温度过高、振动过大）
- 多参数联合异常模式识别（温度+电流同时异常）
- 渐变型退化趋势检测（参数缓慢恶化）
- 新异常模式发现（从未出现过的异常特征组合）

## 执行步骤

### 步骤1：基线建立
- 基于设备正常运行的历史数据（至少30天），建立各参数的正常范围基线
- 计算统计基线：均值±3σ作为静态阈值
- 考虑设备负载状态差异，建立多工况条件下的分段基线
- 建立正常运行的特征分布模型（多元高斯/核密度估计）

### 步骤2：实时检测
- **阈值检测**：当前值与静态/动态阈值对比
- **趋势检测**：CUSUM累积和检测均值漂移，EWMA检测过程偏移
- **模式检测**：Isolation Forest对多维特征空间进行异常评分
- **上下文检测**：考虑当前工况（负载/速度/环境温度）调整判定标准

### 步骤3：异常确认与分级
- 对检测到的异常进行二次确认（排除瞬态干扰，连续3个数据点异常才确认）
- 异常分级判定：
  - 观察级：异常评分轻微，健康度>60分
  - 告警级：异常评分中等，健康度40-60分
  - 紧急级：异常评分严重，健康度<40分
- 记录异常特征签名用于后续模式匹配

### 步骤4：输出与反馈
- 生成异常检测报告（含数据证据、分级结果、建议动作）
- 对于被确认为误报的检测结果，记录反馈用于模型优化
- 更新异常模式知识库

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一编号 |
| features | object | 传感器数据特征集（来自sensor-data-analysis） |
| device_baseline | object | 设备正常运行基线参数 |
| current_workload | string | 当前设备工况/负载状态 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| is_anomaly | boolean | 是否检测到异常 |
| anomaly_type | enum | 异常类型（threshold/trend/pattern/context） |
| anomaly_score | number | 异常严重程度评分（0-1） |
| severity_level | enum | 分级（observation/warning/critical） |
| evidence | object | 异常数据证据（参数值对比） |
| recommended_action | string | 建议动作 |

## 最佳实践
- 阈值告警最简单但易误报，建议结合趋势和模式检测降低误报
- 新设备上线初期（<30天）使用保守阈值，逐步收紧至正常水平
- 设备大修/换件后需要重新建立基线（学习期7天）
- 误报率目标<15%，通过持续收集反馈迭代优化模型

## 约束条件
- 异常检测响应时间<30秒（从数据到达到检测结论输出）
- 瞬态异常（<3个连续数据点）不触发告警，仅记录日志
- 检测模型更新必须通过离线验证后方可部署到生产环境
