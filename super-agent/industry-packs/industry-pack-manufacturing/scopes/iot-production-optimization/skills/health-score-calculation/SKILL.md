# 设备健康度评分计算 (Health Score Calculation)

## 概述
综合多维度传感器数据和设备运行参数，计算设备实时健康度评分（0-100分），为设备维护决策、告警触发和排产规划提供量化依据。

## 适用场景
- 设备实时健康状态评估与展示
- 健康度低于阈值时触发维护工单
- 设备健康度趋势分析与退化预判
- 设备群组健康度对比与排名

## 执行步骤

### 步骤1：维度指标采集
采集设备健康度评估所需的各维度指标：
- **振动维度**：振动烈度（mm/s）、频谱异常度、轴承特征频率能量
- **温度维度**：温升速率、与环境温差、最高温度偏离度
- **电气维度**：电流不平衡度、谐波畸变率（THD）、功率因数偏移
- **润滑维度**：油品分析指标（如有在线传感器）
- **运行维度**：累计运行时间、负载利用率、启停次数

### 步骤2：维度子分数计算
- 每个维度独立计算子分数（0-100）
- 计算方法：基于当前值在正常范围内的位置进行线性/非线性映射
  - 值在正常范围内中心区域 → 90-100分
  - 值接近正常范围边界 → 60-90分
  - 值超出正常范围 → 根据超出程度递减至0分
- 对趋势恶化但当前值仍在范围内的情况，适当扣减（趋势惩罚因子）

### 步骤3：综合评分加权计算
- 按设备类型设定各维度权重（如旋转设备：振动40%、温度25%、电气25%、润滑10%）
- 综合评分 = Σ(维度子分数 × 维度权重)
- 应用修正因子：设备年龄修正（老旧设备适当降低基准）、季节修正（夏季温度基线上调）

### 步骤4：评分输出与触发
- 输出设备健康度综合评分和各维度子分数
- 根据评分触发相应动作：
  - 80-100分：正常，常规监控
  - 60-80分：关注，增加监控频率
  - 40-60分：告警，通知设备维护安排检查
  - <40分：紧急，建议立即停机检修
- 记录评分历史，生成退化趋势曲线

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| device_id | string | 设备唯一编号 |
| device_type | string | 设备类型（用于确定权重配置） |
| dimension_metrics | object | 各维度当前指标值 |
| historical_scores | array | 历史评分序列（用于趋势分析） |
| device_age_days | number | 设备服役天数 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| overall_score | number | 综合健康度评分（0-100） |
| dimension_scores | object | 各维度子分数明细 |
| score_change | number | 与上次评分对比变化值 |
| trend | enum | 趋势方向（improving/stable/degrading） |
| triggered_action | string | 触发的动作（none/monitor/alert/emergency） |
| degradation_rate | number | 退化速率（分/天） |

## 最佳实践
- 权重配置应根据设备类型定制化，旋转类设备振动权重高，热处理设备温度权重高
- 评分突变（单次变化>20分）需要人工复核确认，避免传感器故障导致误判
- 健康度曲线应与维护事件关联展示，验证维护效果（维护后评分应明显回升）
- 建议每季度回顾权重和阈值设定的合理性

## 约束条件
- 健康度评分必须每分钟更新一次（对A类关键设备）
- 评分变化必须可追溯到具体传感器参数变化
- 权重和阈值调整需经过工程师审批，不得自动修改
- 评分模型不适用于新安装设备的前7天（学习期）
