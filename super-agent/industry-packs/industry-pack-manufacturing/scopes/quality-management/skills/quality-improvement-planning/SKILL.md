# 质量改进规划 (Quality Improvement Planning)

## 概述
本技能负责从质量数据中识别系统性改进机会，运用六西格玛DMAIC方法论规划和管理质量改进项目。核心目标是将散点式的质量问题处理提升为有规划、有目标、有收益评估的系统性质量水平提升。

## 适用场景
- 年度/季度质量改进计划编制
- 基于Pareto分析的TOP缺陷改进项目立项
- 反复发生的慢性质量问题的专项改进
- 客户要求的PPM降低项目
- 质量成本（COQ）降低项目
- 新产品量产初期的质量稳定化项目

## 执行步骤

### 步骤1：改进机会识别
1. 数据收集与分析：
   - 近6个月缺陷Pareto分析（TOP 10缺陷类型）
   - 8D报告回顾（反复发生的问题/未彻底解决的问题）
   - 客户投诉趋势分析（高频投诉项）
   - CPK不达标的工序清单
   - 质量成本分析（内部失败/外部失败成本最高的品类）
2. 识别"关键少数"：运用80/20法则，找到贡献80%质量损失的20%因素
3. 区分急性问题（由8D处理）和慢性问题（由改进项目处理）

### 步骤2：项目优先级排序
1. 对每个改进机会进行评分：
   | 评估维度 | 权重 | 评分标准 |
   |----------|------|----------|
   | 质量影响 | 30% | 对PPM/合格率的影响程度 |
   | 成本影响 | 25% | 质量成本降低潜力 |
   | 客户影响 | 25% | 客户满意度/投诉改善 |
   | 可行性 | 20% | 技术难度/资源需求/时间约束 |
2. 综合评分排序，选择TOP 3-5作为当期重点项目
3. 评估项目间的依赖关系和资源冲突

### 步骤3：项目章程制定（DMAIC - Define）
1. 编写项目章程（Project Charter）：
   - **问题陈述**：用数据描述当前状况和差距
   - **项目目标**：量化改进目标（如将缺陷从500PPM降至100PPM）
   - **项目范围**：明确边界（包含/不包含什么）
   - **预期收益**：财务收益（COQ降低额）+ 流程收益 + 客户收益
   - **时间计划**：DMAIC各阶段里程碑
   - **团队组成**：项目负责人、核心成员、支持成员
   - **资源需求**：人力/设备/预算
2. 获得管理层批准和资源承诺

### 步骤4：项目执行跟踪（Measure → Analyze → Improve → Control）
1. **Measure阶段**：
   - 确定关键质量特性（CTQ）
   - 建立测量系统（MSA验证）
   - 收集基线数据
   - 建立过程能力基线（CPK）
2. **Analyze阶段**：
   - 数据分析识别根因（DOE/回归分析/假设检验）
   - 过程分析（流程图/价值流图）
   - 确认关键输入变量（Vital X's）
3. **Improve阶段**：
   - 设计改善方案（DOE优化）
   - 试点验证（小批量试验）
   - 全面推广实施
4. **Control阶段**：
   - 建立控制计划
   - 设置SPC监控
   - 更新标准文件
   - 移交给日常管理

### 步骤5：收益评估与结项
1. 计算实际达成效果 vs 目标：
   - PPM降低幅度
   - CPK提升幅度
   - 质量成本节降金额
   - 客户投诉减少数量
2. 计算项目ROI（投入产出比）
3. 总结经验教训和可横展的方法论
4. 正式结项报告

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| data_period | object | 是 | 分析数据的时间范围 |
| defect_pareto | array | 是 | 缺陷Pareto数据 |
| cost_of_quality | object | 否 | 质量成本数据 |
| customer_complaints | array | 否 | 客户投诉趋势数据 |
| cpk_status | array | 否 | 各工序CPK状态 |
| available_resources | object | 否 | 可用资源（人力/预算） |
| strategic_priorities | array | 否 | 公司战略优先方向 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| improvement_opportunities | array | 识别的改进机会列表 |
| priority_ranking | array | 优先级排序结果 |
| project_charters | array | 项目章程列表 |
| resource_plan | object | 资源分配计划 |
| timeline | object | 整体时间计划 |
| expected_benefits | object | 预期收益汇总 |
| risk_assessment | array | 项目风险评估 |

## 最佳实践
1. **数据说话**：改进项目立项必须有数据支撑，拒绝"拍脑袋"确定改进方向
2. **聚焦关键少数**：不贪多，每个季度3-5个重点项目比10个分散项目效果好
3. **ROI导向**：优先选择投入小、见效快、收益大的项目，建立改进信心
4. **试点先行**：改善方案先在小范围试点验证，再全面推广，降低风险
5. **标准化固化**：改善成果必须通过文件更新和SPC监控固化，否则会退化

## 约束条件
- 项目目标必须量化，不接受"提升质量水平"这类模糊目标
- 每个项目必须有明确的项目负责人（Black Belt或Green Belt）
- 项目周期一般3-6个月，超过6个月需要拆分为子项目
- 年度改进目标与公司经营目标对齐（如质量成本降低X万元）
- 改进方案不得违反法规/标准要求（如不能通过放宽规格来"改善"合格率）
