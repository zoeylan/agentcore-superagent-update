# 供应商质量体系评估 (Supplier Quality Assessment)

## 概述
本技能负责供应商质量管理体系的审核评估，涵盖新供应商准入审核（QSA - Quality System Assessment）、在供供应商定期复审、以及问题供应商专项审核。通过系统化的评估方法，判断供应商是否具备稳定交付合格产品的质量管理能力，并识别供应商质量体系中的薄弱环节和改进空间。

## 适用场景
- 新供应商开发时的质量体系准入审核
- 在供供应商的年度/双年度定期复审
- 供应商质量绩效持续低迷时的专项审核
- 新产品/新工艺导入前对供应商相关能力的专项评估
- 客户要求的二方审核（代客户对供应链进行审核）
- 供应商搬迁/重大变更后的重新评估

## 执行步骤

### 步骤1：审核策划
1. 确定审核类型和范围：
   - 初始审核（新供应商）：全范围体系评估
   - 定期复审（在供供应商）：重点关注上次审核薄弱项和近期质量表现
   - 专项审核：聚焦特定问题领域（如过程控制/测量系统/变更管理）
2. 编制审核计划：
   - 审核日期和时长（一般1-2天）
   - 审核组成员（至少1名SQE + 相关专业人员）
   - 审核范围和部门（质量/生产/仓储/检验等）
   - 审核依据标准（ISO 9001/IATF 16949/VDA 6.3/公司SQA Checklist）
3. 提前发送审核通知和资料需求清单给供应商

### 步骤2：文件审核（桌面审核）
1. 审核供应商质量管理体系文件：
   - 质量手册/质量方针/质量目标
   - 程序文件（文件控制/记录控制/不合格品控制/纠正预防措施等）
   - 过程流程图和控制计划
   - 检验规范和作业指导书
2. 评估文件体系的完整性和适宜性
3. 标注需在现场验证的关键点
4. 如适用，审核第三方认证证书有效性

### 步骤3：现场审核
1. **管理层访谈**：
   - 质量方针理解和执行
   - 资源投入（人员/设备/培训）
   - 质量目标和绩效跟踪
2. **生产过程审核**（按VDA 6.3过程审核方法）：
   - P2 项目管理：新产品开发流程、APQP执行情况
   - P3 产品和过程开发的策划：FMEA/控制计划/验证计划
   - P4 产品和过程开发的实施：样件制作/PPAP/过程确认
   - P5 供应商管理：二级供应商管控能力
   - P6 过程分析/生产：
     - P6.1 过程输入（物料管理、物流）
     - P6.2 过程流程（作业指导书执行、参数控制）
     - P6.3 人力资源（培训、资质、意识）
     - P6.4 物质资源（设备维护、测量设备、工装模具）
     - P6.5 有效性/效率/减少浪费
     - P6.6 过程输出（检验、标识、追溯）
   - P7 客户支持/满意度/服务
3. **关键检查点**：
   - 特殊过程（焊接/热处理/电镀等）是否有过程参数监控和记录
   - 测量设备是否按计划校准且在有效期内
   - 不合格品是否有效隔离和标识
   - 变更管理流程是否受控（4M变更通知）
   - 追溯性：能否从成品追溯到原材料批次
4. **记录审核发现**：分为不合格项（Major/Minor）和观察项（OFI）

### 步骤4：评分与判定
1. 按评分体系进行量化评估：
   - VDA 6.3评分：每个问题0/4/6/8/10分，计算各要素得分率和总得分率
   - 或公司自定义评分表：按检查项逐一评分
2. 综合判定标准：
   - A级（≥90分）：优秀供应商，可直接批准
   - B级（80-89分）：合格供应商，有条件批准（需整改次要问题）
   - C级（60-79分）：需改进，限期整改后复审决定
   - D级（<60分）：不合格，不批准/启动淘汰
3. 不接受Major不合格项超过2项的供应商进入合格名录
4. 安全/法规相关的Major不合格项"一票否决"

### 步骤5：审核报告与整改跟踪
1. 编制审核报告：
   - 审核概要（范围/时间/人员/结论）
   - 各评估要素得分明细
   - 不合格项和观察项清单（含客观证据描述）
   - 综合评价和等级判定
   - 改进要求和建议
2. 发送审核报告给供应商（审核后5个工作日内）
3. 要求供应商提交整改计划（15个工作日内）：
   - 每个不合格项的根因分析
   - 纠正措施和完成时限
   - 整改证据（文件/照片/记录）
4. 评审整改计划可行性
5. 必要时进行整改验证（现场复验或要求提交证据）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplier_code | string | 是 | 供应商编号 |
| audit_type | enum | 是 | INITIAL/PERIODIC/SPECIAL |
| audit_standard | enum | 是 | ISO9001/IATF16949/VDA63/CUSTOM |
| scope | string | 是 | 审核范围描述 |
| products_in_scope | array | 是 | 审核涉及的产品/工艺 |
| previous_audit_findings | array | 否 | 上次审核遗留发现项 |
| quality_performance_data | object | 否 | 近期质量绩效数据 |
| trigger_reason | string | 专项审核必填 | 触发专项审核的原因 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| audit_report_id | string | 审核报告编号 |
| audit_date | date | 审核日期 |
| overall_score | float | 总得分（百分制） |
| grade | enum | A/B/C/D |
| approval_status | enum | APPROVED/CONDITIONAL/REJECTED |
| major_findings | array | 重大不合格项 |
| minor_findings | array | 次要不合格项 |
| observations | array | 观察项 |
| strengths | array | 审核亮点 |
| improvement_requirements | array | 改进要求（含时限） |
| follow_up_plan | object | 后续跟踪计划 |

## 最佳实践
1. **基于风险的审核频次**：A类供应商每2年复审，B类每年，C类每半年，新供应商首年每季度跟踪
2. **审核前做功课**：审核前充分了解供应商近期质量表现（SCAR/来料合格率趋势），带着问题去审核
3. **看过程不看文件**：文件可以写得漂亮，关键看现场实际执行是否与文件一致
4. **追溯验证**：选择一个具体批次，从成品逆向追溯到原材料，验证全流程记录完整性
5. **关注4M变更管理**：Man/Machine/Material/Method的变更是否有通知、评审和验证机制

## 约束条件
- 审核结论必须基于客观证据，不做主观推断
- 审核组必须至少2人，避免单人审核的偏见
- 审核发现项必须与供应商现场确认，避免信息不对称
- 不以获取供应商商业机密为目的，审核范围限于质量相关
- 审核报告中不出现对供应商员工的人身评价
- 初始审核未通过的供应商，至少30天后方可申请复审

## 示例

### 示例1：新供应商初始审核评分
```
供应商: 东方精密铸造有限公司
审核类型: 初始审核（新供应商准入）
审核标准: VDA 6.3 过程审核

评分结果：
| 要素 | 得分率 |
|------|--------|
| P2 项目管理 | 78% |
| P3 产品和过程开发策划 | 82% |
| P4 产品和过程开发实施 | 75% |
| P5 供应商管理 | 68% |
| P6 过程分析/生产 | 80% |
| P7 客户支持 | 85% |
| 总得分率 | 78% |

等级判定: C级（60-79分）
审核结论: 有条件批准，需在60天内完成以下整改：
1. [Major] P5.1 二级供应商无正式评估机制 → 需建立供应商评估流程
2. [Major] P4.3 PPAP文件不完整（缺少MSA报告）→ 需补充完整PPAP
3. [Minor] P6.4 部分模具缺少预防性维护计划 → 需建立模具PM计划
4. [OFI] P6.3 新员工培训记录建议增加技能考核环节
```
