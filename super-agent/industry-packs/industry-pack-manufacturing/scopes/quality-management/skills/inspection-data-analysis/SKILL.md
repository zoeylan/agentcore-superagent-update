# 检验数据分析 (Inspection Data Analysis)

## 概述
本技能负责对来料检验（IQC）、过程检验（IPQC）和成品检验（OQC）数据进行系统性采集、分析和批次判定。根据GB/T 2828.1抽样标准和AQL（可接受质量水平）规则，对检验批次进行合格/不合格判定，并输出结构化的分析结果。

## 适用场景
- 新到货物料的来料检验批次判定
- 生产过程中的巡检数据汇总分析
- 成品出货前的OQC检验判定
- 检验方案转换（正常→加严→放宽）的决策支持
- 多批次检验数据的趋势分析

## 执行步骤

### 步骤1：确定抽样方案
1. 获取检验批次信息（物料编号、批量大小、检验水平）
2. 根据GB/T 2828.1查表确定样本量字码
3. 确定当前检验严格度（正常/加严/放宽），依据转换规则：
   - 正常→加严：连续5批中有2批不合格
   - 正常→放宽：连续10批合格且生产稳定
   - 加严→正常：连续5批合格
4. 根据AQL值和样本量字码确定Ac（接收数）和Re（拒收数）
5. 对于关键物料（安全件/功能件），执行全检方案

### 步骤2：检验数据采集与整理
1. 收集检验原始数据（计量型/计数型）
2. 验证数据完整性（检验项目是否覆盖检验规范全部要求）
3. 数据清洗：识别异常值（超出±4σ可能为记录错误）
4. 按检验项目分类汇总不合格数/不合格品数

### 步骤3：判定与处置
1. 对比实际不合格数与Ac/Re值，做出判定：
   - 不合格数 ≤ Ac → 批次合格，放行入库
   - 不合格数 ≥ Re → 批次不合格，执行不合格处置
2. 不合格批次处置选项：
   - 退货（来料）/ 返工（过程/成品）/ 报废 / 让步接收（需客户书面同意）
3. 记录判定结果和处置决定

### 步骤4：数据归档与关联分析
1. 将检验结果录入质量管理系统
2. 关联到供应商维度（供应商合格率计算）
3. 关联到工序/设备维度（过程检验）
4. 更新检验方案转换计数器

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| batch_id | string | 是 | 检验批次号 |
| material_code | string | 是 | 物料编号 |
| batch_size | integer | 是 | 批量大小 |
| inspection_type | enum | 是 | IQC/IPQC/OQC |
| inspection_level | enum | 是 | I/II/III（一般/特殊） |
| aql_value | float | 是 | AQL值（如0.65/1.0/2.5/4.0） |
| inspection_data | array | 是 | 各检验项目的测量结果 |
| supplier_code | string | 来料必填 | 供应商编号 |
| process_station | string | 过程必填 | 工序/工位编号 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| judgment | enum | PASS/FAIL/CONDITIONAL |
| sample_size | integer | 实际抽样数 |
| defect_count | integer | 不合格品数 |
| defect_rate | float | 不合格率(%) |
| accept_number | integer | 接收数Ac |
| reject_number | integer | 拒收数Re |
| severity_breakdown | object | 按严重/主要/次要分类统计 |
| disposition | enum | 放行/退货/返工/报废/让步接收 |
| trend_flag | boolean | 是否触发趋势预警 |
| related_history | array | 近期同类批次检验历史 |

## 最佳实践
1. **分级管理**：安全关键特性（SC）零缺陷要求，重要特性（CC）按AQL 0.65，一般特性按AQL 2.5
2. **数据追溯**：每条检验记录必须关联到具体的测量设备编号和检验员编号，确保可追溯
3. **转换规则严格执行**：不允许人为跳过加严检验阶段，必须通过连续5批合格才能恢复正常检验
4. **首件必检**：过程检验中首件三检（自检/互检/专检）必须在批量生产前100%完成

## 约束条件
- 不做根因分析，发现不合格仅记录现象和判定结果
- 让步接收必须有客户书面同意文件才能执行
- 测量系统GR&R必须<10%（关键特性）或<30%（一般特性），否则检验数据不可信
- 对于破坏性检验项目，抽样数量按单独标准执行，不适用AQL表

## 示例

### 示例1：来料检验判定
```
输入：
- 批次号: IQC-2024-001523
- 物料: 精密轴承 M-BRG-0012
- 批量: 500件
- 检验水平: II级
- AQL: 1.0（主要缺陷），2.5（次要缺陷）
- 当前状态: 正常检验

分析过程：
1. 批量500 + 检验水平II → 样本量字码H → 样本量50
2. AQL 1.0 → Ac=1, Re=2（主要缺陷）
3. AQL 2.5 → Ac=3, Re=4（次要缺陷）
4. 检验结果：主要缺陷0个，次要缺陷2个
5. 判定：主要缺陷0≤1(Ac) PASS，次要缺陷2≤3(Ac) PASS

输出：
- 判定: PASS
- 不合格率: 4%（次要缺陷）
- 处置: 放行入库
- 趋势标记: 该供应商近3批次要缺陷数呈上升趋势(0→1→2)，建议关注
```
