# SPC统计过程控制监控 (SPC Monitoring)

## 概述
本技能负责运用统计过程控制（SPC）方法对制造过程进行实时监控，通过控制图分析识别过程变异的特殊原因，在工序失控前发出预警。核心目标是区分过程的普通原因变异和特殊原因变异，确保关键工序始终处于统计受控状态。

## 适用场景
- 关键工序（CTQ特性）的实时SPC监控
- 过程能力指数（CPK/PPK）的计算与评估
- 控制图判异准则的自动检测与预警
- 新产品/新工序的初始过程能力研究
- SPC控制限的定期重新计算

## 执行步骤

### 步骤1：控制图类型选择
根据数据类型和子组大小选择合适的控制图：
| 数据类型 | 子组大小 | 推荐控制图 |
|----------|----------|-----------|
| 计量型 | n=2~10 | X-bar/R图 |
| 计量型 | n>10 | X-bar/S图 |
| 计量型 | n=1 | I-MR图（单值移动极差）|
| 计数型（不合格品率）| 变化 | P图 |
| 计数型（不合格品数）| 固定 | NP图 |
| 计数型（缺陷数）| 固定 | C图 |
| 计数型（单位缺陷数）| 变化 | U图 |

### 步骤2：控制限计算
1. 收集至少25个子组的历史数据（初始建立期）
2. 计算中心线（CL）和控制限（UCL/LCL）：
   - X-bar图: CL = X双杠, UCL/LCL = X双杠 ± A2*R-bar
   - R图: CL = R-bar, UCL = D4*R-bar, LCL = D3*R-bar
3. 排除初始数据中的异常点后重新计算
4. 定期重新计算控制限（建议每100个子组或工艺变更后）

### 步骤3：实时监控与判异
应用GB/T 4091-2001八大判异准则：
1. **准则1**：1个点落在A区以外（超出3σ）→ 立即告警
2. **准则2**：连续9点在中心线同侧 → 过程偏移
3. **准则3**：连续6点递增或递减 → 趋势性漂移
4. **准则4**：连续14点交替上下 → 系统性变异
5. **准则5**：连续3点中有2点在A区（2σ~3σ）→ 过程波动增大
6. **准则6**：连续5点中有4点在B区以外（1σ外）→ 偏移趋势
7. **准则7**：连续15点在C区内（1σ内）→ 控制限过宽或分层
8. **准则8**：连续8点在C区外两侧（无点在1σ内）→ 混合模式

### 步骤4：过程能力评估
1. 计算CPK = min[(USL-μ)/(3σ), (μ-LSL)/(3σ)]
2. 计算PPK（使用总体标准差替代组内标准差）
3. 能力等级判定：
   - CPK ≥ 1.67：过程能力充足（优秀）
   - 1.33 ≤ CPK < 1.67：过程能力合格（正常管控）
   - 1.0 ≤ CPK < 1.33：过程能力不足（需改善）
   - CPK < 1.0：过程能力严重不足（紧急改善）

### 步骤5：预警生成与推送
1. 根据判异结果确定预警级别：
   - 红色预警：准则1触发（超出控制限）→ 立即停机检查
   - 橙色预警：准则2/3/5/6触发 → 4小时内调查
   - 黄色预警：准则4/7/8触发 → 排入计划检查
2. 生成预警通知，包含：工序名称、特性名称、触发准则、当前数据趋势、建议措施
3. 推送给相关责任人（操作工/工艺工程师/质量工程师）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| process_id | string | 是 | 工序编号 |
| characteristic | string | 是 | 监控特性名称 |
| spec_limits | object | 是 | 规格限（USL/LSL/Target） |
| subgroup_data | array | 是 | 子组数据序列 |
| subgroup_size | integer | 是 | 子组大小 |
| chart_type | enum | 否 | 指定控制图类型（可自动选择） |
| control_limits | object | 否 | 已有控制限（首次建立时为空） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| chart_type | enum | 使用的控制图类型 |
| control_limits | object | UCL/CL/LCL数值 |
| current_status | enum | IN_CONTROL/WARNING/OUT_OF_CONTROL |
| violated_rules | array | 触发的判异准则列表 |
| cpk_value | float | 当前CPK值 |
| ppk_value | float | 当前PPK值 |
| trend_description | string | 趋势描述（自然语言） |
| alert_level | enum | RED/ORANGE/YELLOW/NONE |
| recommended_actions | array | 建议措施列表 |
| data_points_analyzed | integer | 分析的数据点数 |

## 最佳实践
1. **数据时效性**：SPC数据点录入延迟不超过15分钟，确保预警的及时性
2. **合理分组**：子组内数据应在相近时间和相同条件下采集，确保组内变异仅反映普通原因
3. **控制限冻结**：一旦建立合理的控制限，在工艺条件未变更前不随意重算（避免追踪过程）
4. **规格限vs控制限**：绝不将规格限当作控制限使用，两者概念完全不同
5. **先R图后X图**：分析时先看R图（变异性），R图受控后再看X-bar图（均值），因为组内变异不受控时X-bar图的控制限无意义

## 约束条件
- 至少需要25个子组数据才能建立可靠的控制限
- CPK计算要求过程处于统计受控状态，失控状态下仅计算PPK
- 不对非正态分布数据直接应用标准控制图，需先进行数据变换或使用非参数控制图
- 测量系统变异（GR&R）对SPC结果有影响，GR&R>30%时SPC结果不可靠

## 示例

### 示例：过程偏移预警
```
输入：
- 工序: 精密磨削 PROC-0023
- 特性: 外径尺寸 Φ25.00±0.02mm
- 子组大小: 5
- 近期15组数据均值趋势: 25.003, 25.004, 25.005, 25.004, 25.006,
  25.005, 25.007, 25.006, 25.008, 25.007, 25.008, 25.009, 25.008,
  25.009, 25.010

分析结果：
- 控制图类型: X-bar/R图
- 中心线CL: 25.000 (历史)
- UCL: 25.012, LCL: 24.988
- 当前状态: WARNING
- 触发准则:
  - 准则2 (连续9点在中心线上方) ✓ 从第1组开始
  - 准则3 (连续6点递增趋势) ✓ 第9-14组
- CPK: 1.15（低于1.33目标）
- 预警级别: 橙色
- 建议措施:
  1. 4小时内检查砂轮磨损状况和修整周期
  2. 确认工件装夹定位是否偏移
  3. 检查冷却液浓度和温度
  4. 如确认偏移，调整砂轮进给补偿
```
