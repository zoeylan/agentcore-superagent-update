# 纠正措施跟踪 (Corrective Action Tracking)

## 概述
本技能负责对质量问题纠正措施（CA）和预防措施（PA）的执行进度进行系统性跟踪管理，确保每项措施按时到位、有效验证、并持续监控防止问题复发。覆盖从措施制定、分配、执行、验证到关闭的全生命周期管理。

## 适用场景
- 8D流程中D5-D7阶段的措施跟踪
- SCAR供应商纠正措施的回复跟踪
- 内外部审核不合格项整改跟踪
- 管理评审决议事项跟踪
- 质量改进项目里程碑跟踪

## 执行步骤

### 步骤1：措施登记与分配
1. 将确定的纠正/预防措施登记入跟踪系统
2. 每项措施必须包含：
   - 措施描述（具体做什么）
   - 责任人（唯一负责人）
   - 计划完成日期
   - 量化验证标准（如何判定措施有效）
   - 优先级（紧急/高/中/低）
   - 关联问题编号（8D编号/SCAR编号/审核发现编号）
3. 通知责任人并确认接受

### 步骤2：进度监控
1. 设定检查频率：
   - 紧急措施：每天检查进度
   - 高优先级：每3天检查
   - 中/低优先级：每周检查
2. 采集进度更新（完成百分比、当前状态、遇到的阻碍）
3. 状态分类：
   - 进行中（按计划）
   - 进行中（延迟风险）
   - 已延迟
   - 等待资源/审批
   - 已完成待验证

### 步骤3：逾期预警与升级
1. 预警时间线：
   - 到期前3天：提醒责任人即将到期
   - 到期日：未完成则标记为逾期
   - 逾期7天：升级通知部门主管
   - 逾期14天：升级通知质量经理
   - 逾期30天：自动纳入管理评审议题
2. 记录逾期原因和重新计划的完成日期
3. 分析逾期模式（哪些类型的措施容易延迟）

### 步骤4：有效性验证
1. 措施完成后进入验证阶段：
   - 短期验证（完成后1-2周）：措施是否被正确执行
   - 效果验证（完成后1个月）：不良率是否下降到目标
   - 持续验证（完成后3个月）：问题是否不再复发
2. 验证方法：
   - 数据验证：对比改善前后的质量数据（PPM、CPK、合格率）
   - 过程审核：现场确认新流程/标准是否被遵循
   - 产品审核：抽检确认产品质量符合要求
3. 验证结果判定：
   - PASS：达到量化验证标准 → 措施关闭
   - FAIL：未达标 → 分析原因，制定加强措施

### 步骤5：关闭与归档
1. 所有验证通过后正式关闭措施
2. 更新相关文件：
   - 作业指导书（WI）
   - 检验标准
   - FMEA
   - 控制计划
3. 将措施和效果记录归入知识库
4. 生成关闭报告

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action_id | string | 是 | 措施唯一编号 |
| action_type | enum | 是 | CORRECTIVE/PREVENTIVE/IMPROVEMENT |
| description | string | 是 | 措施描述 |
| owner | string | 是 | 责任人 |
| due_date | date | 是 | 计划完成日期 |
| verification_criteria | string | 是 | 量化验证标准 |
| priority | enum | 是 | URGENT/HIGH/MEDIUM/LOW |
| source_reference | string | 是 | 来源编号（8D/SCAR/审核） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| status_summary | object | 当前所有措施状态汇总 |
| overdue_items | array | 逾期措施清单 |
| upcoming_due | array | 即将到期措施（3天内） |
| verification_pending | array | 待验证措施清单 |
| closure_rate | float | 按时关闭率 |
| escalation_log | array | 升级记录 |
| effectiveness_report | object | 有效性验证汇总 |

## 最佳实践
1. **SMART原则**：每项措施的验证标准必须是Specific（具体）、Measurable（可测量）、Achievable（可达成）、Relevant（相关）、Time-bound（有时限）
2. **单一责任人**：不允许措施的责任人是"XX部门"，必须到具体个人
3. **提前预警**：宁可多提醒，也不要等到逾期才发现
4. **验证独立性**：验证人不应是措施执行人自己
5. **闭环文化**：强调"有布置必有检查，有检查必有闭环"

## 约束条件
- 不接受"已完成"但无验证数据支撑的关闭请求
- 验证标准必须在措施制定时就确定，不允许事后补充
- 逾期措施的重新计划日期不允许超过原计划的2倍时长
- 同一措施最多允许2次逾期重计划，第3次自动升级到管理层
- 3个月复发验证期内同类问题复发，措施自动重新打开
