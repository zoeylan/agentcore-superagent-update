# 供应商质量绩效评分 (Supplier Performance Scoring)

## 概述
本技能负责供应商质量绩效的量化评分和动态管理，通过多维度数据采集和加权评分，建立供应商质量绩效排名和分级管理机制。评分结果直接影响供应商的份额分配、合作等级调整和淘汰决策，是供应商质量管理的数据基础和决策依据。

## 适用场景
- 月度供应商质量绩效评分（持续监控）
- 季度供应商综合绩效评审（与采购协作，质量权重40%）
- 供应商年度评优/淘汰决策数据支撑
- 新产品分配供应商选择的质量维度参考
- 供应商分级管理（优选/合格/观察/淘汰）调整依据
- SCAR关闭后的评分更新和趋势分析

## 执行步骤

### 步骤1：数据采集
1. 来料检验数据（质量维度核心数据）：
   - 来料批次总数和合格批次数 → 批次合格率
   - 来料件数和不合格件数 → PPM不良率
   - 按缺陷严重度分类统计（Critical/Major/Minor）
   - 特采/让步接收次数
2. SCAR相关数据：
   - 期间内SCAR发起数量
   - SCAR回复及时率（7天内回复比例）
   - SCAR首次评审通过率（回复质量）
   - 同类问题复发次数
3. 质量体系审核数据（如有）：
   - 最近一次审核得分和等级
   - 审核不合格项整改完成情况
4. 特殊事件记录：
   - 来料导致产线停线次数
   - 来料导致客户投诉次数
   - 未经批准的4M变更次数
   - 质量通知响应速度

### 步骤2：评分计算
1. 质量绩效评分模型（月度）：
   ```
   质量得分 = Σ(指标得分 × 权重)

   指标权重分配：
   - 来料批次合格率: 35%（月度数据）
   - PPM不良率: 25%（月度数据）
   - SCAR回复及时率: 15%
   - SCAR首次通过率: 10%
   - 问题复发率: 10%（6个月内同类问题复发次数）
   - 特殊事件扣分: 5%（停线/客诉/未授权变更）
   ```

2. 各指标得分标准（满分100分）：
   - 来料批次合格率：100%=100分，99%=90分，98%=70分，97%=50分，<96%=0分
   - PPM不良率：<100PPM=100分，100-500=80分，500-1000=60分，1000-5000=40分，>5000=0分
   - SCAR回复及时率：100%=100分，90-99%=80分，80-89%=60分，<80%=30分
   - SCAR首次通过率：100%=100分，80-99%=75分，60-79%=50分，<60%=25分
   - 问题复发率：0次=100分，1次=60分，2次=30分，≥3次=0分
   - 特殊事件：无事件=100分，每发生一次扣30分

3. 综合得分计算和等级划分：
   - A级（≥90分）：优选供应商——优先分配份额
   - B级（75-89分）：合格供应商——正常供货
   - C级（60-74分）：观察供应商——限期改善，限制新产品导入
   - D级（<60分）：整改供应商——减少份额，启动整改/淘汰评估

### 步骤3：趋势分析
1. 绘制供应商质量得分趋势图（月度）：
   - 识别持续提升/持续恶化的供应商
   - 标注得分突变点并关联原因（如产能扩张/人员变动/设备更换）
2. 供应商间横向对比：
   - 同品类供应商质量得分排名
   - 识别同品类最佳实践供应商
3. 季度评分移动平均：
   - 避免因单月异常数据导致的评分剧烈波动
   - 季度平均分作为正式评审依据

### 步骤4：分级管理决策支撑
1. 等级变动触发条件：
   - 升级条件：连续3个月评分提升1个等级 → 确认升级
   - 降级条件：连续2个月评分降至下一等级 → 触发降级预警
   - 紧急降级：发生重大质量事件（客诉/停线）→ 即时降级
2. 各等级管理措施建议：
   - A级：年度审核减免、优先试产新产品、酌情增加份额
   - B级：正常管理、年度审核
   - C级：月度绩效面谈、半年度现场审核、限制新产品、制定SQDP
   - D级：周度监控、启动替代供应商开发、逐步减少份额、限期60天整改
3. 淘汰条件（需综合评估）：
   - 连续2个季度D级
   - 发生安全/法规类重大质量事故
   - SCAR长期不关闭（超过60天）且无改善意愿

### 步骤5：评分报告生成
1. 月度供应商质量绩效简报：
   - TOP5优秀供应商和TOP5待改善供应商
   - 等级分布统计（A/B/C/D各多少家）
   - 较上月变动情况
2. 季度供应商质量绩效详报：
   - 全量供应商评分明细
   - 等级变动清单
   - 改善行动跟踪
   - 淘汰预警和替代建议
3. 输出给供应链管理Scope的综合评分接口（质量权重40%）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplier_code | string | 是 | 供应商编号 |
| evaluation_period | string | 是 | 评估期间（如2024-03） |
| iqc_data | object | 是 | 来料检验统计数据 |
| scar_data | object | 是 | SCAR统计数据 |
| audit_data | object | 否 | 审核评分数据（如有更新） |
| special_events | array | 否 | 特殊事件记录 |
| historical_scores | array | 否 | 历史评分数据（用于趋势分析） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| supplier_code | string | 供应商编号 |
| period | string | 评估期间 |
| overall_score | float | 综合质量得分（0-100） |
| grade | enum | A/B/C/D |
| dimension_scores | object | 各维度得分明细 |
| grade_change | enum | UPGRADE/STABLE/DOWNGRADE |
| trend | enum | IMPROVING/STABLE/DECLINING |
| action_required | enum | NONE/MONITOR/IMPROVE/ESCALATE |
| recommendations | array | 管理建议 |
| comparison_rank | string | 同品类排名（如"3/8"） |

## 最佳实践
1. **数据采集自动化**：评分所需数据应尽量从QMS/ERP系统自动提取，减少人工统计误差
2. **评分透明度**：评分标准和计算方法应向供应商公开，使其明确改善方向
3. **避免单一指标决策**：不因单一指标波动做重大决策，需综合多维度和趋势判断
4. **区分新旧供应商**：新供应商（导入<6个月）数据量少，评分参考性有限，单独标注
5. **评分与行动挂钩**：评分不是目的，基于评分的分级管理措施才是价值所在

## 约束条件
- 评分数据必须来自系统记录，不接受人工填写的主观评价
- 评分结果不可被单人修改，需经SQE主管确认
- 月度评分仅为监控用途，正式的等级变动决策依据季度评分
- 供应商对评分有异议时，必须提供数据证据，不接受口头申诉
- 评分系统变更（权重/标准调整）需提前1个季度通知供应商
- 不对仅有1-2批次来料的供应商进行评分（样本量不足）

## 示例

### 示例1：月度评分计算
```
供应商: 恒信电子有限公司 (SUP-0015)
评估期间: 2024年3月

原始数据：
- 来料批次: 12批，合格10批 → 批次合格率 83.3%
- 来料总数: 24000件，不合格180件 → PPM 7500
- SCAR: 本月2件，均7天内回复 → 回复及时率100%
- SCAR评审: 2件中1件首次通过 → 首次通过率50%
- 问题复发: 1次（电容焊接虚焊问题3月内再次发生）
- 特殊事件: 1次产线停线（来料引脚尺寸超差导致）

评分计算：
- 来料批次合格率(35%): 83.3% → 0分 × 35% = 0
- PPM不良率(25%): 7500 → 0分 × 25% = 0
- SCAR回复及时率(15%): 100% → 100分 × 15% = 15
- SCAR首次通过率(10%): 50% → 25分 × 10% = 2.5
- 问题复发率(10%): 1次 → 60分 × 10% = 6
- 特殊事件(5%): 1次 → 70分 × 5% = 3.5

总分: 0 + 0 + 15 + 2.5 + 6 + 3.5 = 27分
等级: D级（<60分）
趋势: 较上月(52分)大幅下降
建议: 紧急约谈，启动替代供应商开发，限期30天提交整改计划
```
