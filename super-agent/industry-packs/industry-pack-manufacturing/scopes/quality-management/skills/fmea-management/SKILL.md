# FMEA管理 (FMEA Management)

## 概述
本技能负责失效模式与效应分析（FMEA）文档的创建、维护和动态更新。FMEA是制造业中最核心的质量预防工具之一，通过系统性识别潜在失效模式、评估其影响和原因、并制定预防/检测措施，实现风险的前瞻性管控。本技能覆盖设计FMEA（DFMEA）和过程FMEA（PFMEA），遵循AIAG-VDA FMEA手册（2019版）七步法。

## 适用场景
- 新产品/新工艺开发阶段的FMEA初始编制
- 质量异常发生后的FMEA动态更新（新增失效模式或调整RPN）
- 定期FMEA评审（季度审查高RPN项目进展）
- 工艺变更/设计变更前的风险预评估
- 客户审核前的FMEA完整性检查
- IATF 16949体系要求的FMEA维护

## 执行步骤

### 步骤1：范围界定与准备
1. 明确FMEA类型（DFMEA/PFMEA）和分析对象（产品/工序）
2. 收集输入信息：
   - 产品图纸/技术规格
   - 工艺流程图（PFMEA必须基于Flow Chart）
   - 历史质量数据（过去12个月的不良率/客诉/8D记录）
   - 类似产品/工序的现有FMEA（经验借鉴）
3. 组建跨功能团队（设计/工艺/质量/生产/设备/供应商代表）
4. 确定分析深度（按工序步骤逐一展开 or 聚焦高风险工序）

### 步骤2：功能分析与失效模式识别
1. 列出分析对象的所有功能/要求
2. 针对每个功能识别潜在失效模式（功能丧失/功能退化/功能过度/间歇性功能）
3. 确定每个失效模式的潜在影响（对下一工序/最终产品/最终客户的影响）
4. 识别每个失效模式的潜在原因（人/机/料/法/环/测六方面）
5. 记录当前的预防措施（Prevention Control）和检测措施（Detection Control）

### 步骤3：风险评估（SOD评分）
1. 严重度（Severity）评分 1-10：
   - 10分：影响安全/法规合规
   - 8-9分：主要功能丧失
   - 6-7分：功能退化/客户不满
   - 4-5分：次要功能受影响
   - 1-3分：几乎无影响
2. 发生度（Occurrence）评分 1-10：
   - 基于历史数据和过程能力（CPK）
   - CPK>1.67 → O=1-2；CPK 1.33-1.67 → O=3-4；CPK<1.0 → O=7-8
3. 检测度（Detection）评分 1-10：
   - 1-2分：自动100%检测且防错
   - 3-4分：自动检测有效
   - 5-6分：人工抽检
   - 7-8分：仅目视检查
   - 9-10分：无法检测
4. 计算RPN = S × O × D，或使用AP（行动优先级）方法

### 步骤4：行动计划制定
1. 对高风险项（RPN≥120 或 AP=HIGH）制定改进措施：
   - 降低严重度：设计改进/增加冗余
   - 降低发生度：工艺优化/参数调整/防错装置
   - 降低检测度：增加检测手段/自动化检测
2. 每项措施必须有：
   - 建议措施描述
   - 责任人
   - 目标完成日期
   - 预期新SOD评分
3. 措施优先级排序：优先降低发生度（预防优于检测）

### 步骤5：跟踪验证与更新
1. 跟踪每项改进措施的执行状态
2. 措施完成后重新评估SOD评分（实际值 vs 预期值）
3. 验证RPN是否降至目标值以下
4. 触发更新的事件：
   - 新的质量异常（8D关闭后）→ 新增或修改失效模式
   - 工艺变更/设备变更 → 重新评估受影响项
   - 定期评审（每季度）→ 全面审查高RPN项

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| fmea_type | enum | 是 | DFMEA/PFMEA |
| target_item | string | 是 | 分析对象（产品型号/工序名称） |
| trigger_event | enum | 是 | NEW_DEVELOPMENT/QUALITY_ISSUE/CHANGE/PERIODIC_REVIEW |
| related_8d_ids | array | 否 | 关联的8D报告编号 |
| process_flow | array | PFMEA必填 | 工艺流程步骤列表 |
| historical_data | object | 否 | 历史不良率/PPM数据 |
| current_cpk | object | 否 | 当前关键特性CPK值 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| fmea_document_id | string | FMEA文档编号 |
| revision | string | 版本号 |
| failure_modes | array | 失效模式列表 |
| high_risk_items | array | 高RPN/高AP项清单 |
| action_plan | array | 改进行动计划（含责任人/时限） |
| rpn_summary | object | RPN分布统计和趋势 |
| change_log | array | 本次变更记录 |

## 最佳实践
1. **活文档原则**：FMEA不是一次性文档，必须随产品/工艺生命周期持续更新
2. **数据驱动评分**：SOD评分必须基于实际数据（历史不良率、CPK值），避免主观猜测
3. **关注高严重度**：即使RPN不高，严重度=9或10的项必须有专项管控措施
4. **横展思维**：一个产品/工序发现的新失效模式，应检查类似产品/工序是否存在相同风险
5. **与控制计划联动**：FMEA中的检测措施必须反映在控制计划（Control Plan）中

## 约束条件
- FMEA团队必须跨功能组建，不允许质量部门独立完成
- SOD评分必须团队共识，不接受单人评分
- 高RPN项（≥120）必须有改进措施，不允许"接受现状"
- S≥9的失效模式不论RPN高低，都必须有特殊管控
- FMEA评审至少每季度一次，有重大质量事件时即时触发更新
- 不对已淘汰产品/工序的FMEA进行更新维护

## 示例

### 示例1：质量异常触发的FMEA更新
```
触发事件：8D-2024-0089关闭，根因为注塑工序保压时间波动导致缩孔缺陷
当前FMEA状态：该失效模式原RPN = S6 × O3 × D5 = 90

更新操作：
1. 修改发生度O：基于近6个月数据，该缺陷发生率从0.1%升至0.8%，O从3调整为5
2. 更新原RPN：S6 × O5 × D5 = 150 → 超过阈值120，标记为高风险
3. 制定措施：增加保压时间SPC实时监控（目标降低D从5至2）
4. 预期新RPN：S6 × O5 × D2 = 60（目标值）
5. 责任人：工艺工程师张三，完成时限：2024-04-30
```
