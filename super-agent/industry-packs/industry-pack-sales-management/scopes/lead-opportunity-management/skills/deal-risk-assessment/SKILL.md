# 商机风险评估技能 (Deal Risk Assessment)

## 概述

对活跃商机进行多维度风险评估，识别可能导致商机丢失或延期的风险因素，为销售团队提供早期预警和风险缓解建议。风险评估是Pipeline预测准确性和赢率提升的关键支撑能力。

## 适用场景

- 每周Pipeline Review前的常规风险扫描
- 大额商机（>年度平均客单价3倍）的深度风险评估
- 商机长时间停滞时的根因分析
- 季度末重点商机的赢率校准
- 竞标场景下的竞争风险评估

## 执行步骤

### 步骤1：多维风险信号采集
扫描以下风险维度：

**时间风险**：
- 商机在当前阶段停留是否超标
- 预计关闭日期是否多次推迟
- 客户内部决策流程是否出现延迟信号

**关系风险**：
- Champion是否活跃（最近是否有互动）
- 决策链中是否有未接触的关键角色
- 客户关键联系人是否有离职/调岗迹象
- 客户对我方的响应速度是否在下降

**竞争风险**：
- 是否有竞品正在同时评估
- 客户是否提及竞品优势或要求功能对比
- 是否有价格竞争压力信号

**需求风险**：
- 客户痛点的紧迫度是否在下降
- 是否有预算被砍或冻结的信号
- 是否有项目优先级调整的迹象
- 客户对方案的engagement是否在减弱

**流程风险**：
- MEDDIC关键字段是否完整
- 阶段推进是否符合正常节奏
- 销售活动频率是否足够（外呼/会议/演示）

### 步骤2：风险量化评分
对每个维度进行0-5分评分：
- 0-1分：低风险（该维度正常）
- 2-3分：中风险（存在不确定性）
- 4-5分：高风险（明显负面信号）

综合风险分 = 各维度加权平均（时间30%+关系25%+竞争20%+需求15%+流程10%）

### 步骤3：风险分级与标注
- 综合风险分 ≤ 1.5 → 🟢 绿色（低风险，正常推进）
- 综合风险分 1.5-3.0 → 🟡 黄色（中风险，需关注）
- 综合风险分 > 3.0 → 🔴 红色（高风险，需干预）

### 步骤4：风险缓解建议
针对每个高分风险维度提供具体行动建议：
- 时间风险高 → "建议安排高层互访加速决策/创造紧迫感（如限时优惠/容量限制）"
- 关系风险高 → "建议多线联系、发展新Champion/安排客户成功案例参访"
- 竞争风险高 → "建议准备差异化Battlecard/安排在竞品弱项领域的深度Demo"
- 需求风险高 → "建议重新验证痛点紧迫度/准备ROI量化分析/对接业务决策人"
- 流程风险高 → "建议补全MEDDIC信息/增加销售活动频率/确认Decision Process"

### 步骤5：趋势追踪与预警
1. 记录每次评估的风险分数，绘制风险趋势曲线
2. 连续两周风险分上升 → 趋势恶化预警
3. 风险分从绿色跳变为红色 → 急剧恶化预警
4. 预警同步给商机Owner、销售经理和Pipeline Review会议

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| opportunity_id | String | 是 | 商机唯一标识 |
| opportunity_data | Object | 是 | 商机完整数据 |
| activity_history | Array | 是 | 历史活动记录 |
| competitive_intel | Object | 否 | 竞争情报信息 |
| historical_risk_scores | Array | 否 | 历史风险评分（用于趋势分析） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| overall_risk_score | Number | 综合风险分（0-5） |
| risk_level | String | 风险级别（green/yellow/red） |
| dimension_scores | Object | 各维度评分明细 |
| risk_factors | Array | 具体风险因素列表 |
| mitigation_actions | Array | 风险缓解建议 |
| trend_analysis | Object | 风险趋势分析 |
| win_probability | Number | 赢率估计（百分比） |

## 最佳实践

1. **客观评估**：风险评估必须基于事实和数据，不因销售的乐观表态而忽略客观风险信号
2. **及时更新**：每周至少评估一次，重大事件发生后即时更新（如Champion离职）
3. **量化表达**：用具体数据说明风险（"已停滞28天，超出平均14天的2倍"而非"停了挺久"）
4. **行动导向**：每个高风险因素都必须有对应的缓解action，评估是为了行动而非记录
5. **复盘校准**：对已关闭商机（赢/输）回溯其风险评估历史，优化评估模型的预测准确性

## 约束条件

- 风险评估不可作为唯一丢单判定依据，最终决策需人工确认
- 评估结果仅供内部使用，不可让客户知道其商机被标记为高风险
- 风险评估模型更新需经销售运营团队审批
- 数据不足时（如新建商机<1周），应标注"数据不足，待观察"而非强行评分
