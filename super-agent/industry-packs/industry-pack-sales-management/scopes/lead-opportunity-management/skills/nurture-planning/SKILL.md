# 线索培育计划技能 (Nurture Planning)

## 概述

为暂时不满足SQL条件但具有长期转化价值的线索制定系统性培育（Nurture）计划。通过持续的价值内容触达和关系维护，保持线索温度直至其满足转化条件。优秀的nurture计划可将长期线索的最终转化率提升至15-20%。

## 适用场景

- BANT评分30-59分的线索进入nurture pool时
- SQL评估暂不合格但有长期价值的线索
- 3次未接通回收到nurture pool的线索
- 客户暂无预算但表达了未来兴趣的线索
- 失去冠军（Champion离职）但公司仍有需求的线索

## 执行步骤

### 步骤1：线索分层与标签
1. 评估线索长期价值：企业规模×行业匹配度×意向强度
2. 分层：
   - A类Nurture（高价值/短周期）：预计3个月内可再次评估
   - B类Nurture（中等价值/中周期）：预计6个月内可再次评估
   - C类Nurture（低价值/长周期）：预计6-12个月，低频维护
3. 标签标注：暂缓原因（无预算/无时间线/决策人未触达/需求不明确）

### 步骤2：触达频率与渠道规划
基于分层设置触达节奏：
- A类：每2周一次主动触达（邮件/微信），每月一次电话check-in
- B类：每月一次内容推送，每季度一次check-in电话
- C类：每月一次自动化邮件，每半年一次人工review

触达渠道选择：
- 内容营销邮件（行业报告、白皮书、案例分析）
- 微信/企业微信消息（轻量级、关系维护型）
- 线上活动邀请（Webinar、产品发布会、行业沙龙）
- 线下活动邀请（客户交流会、行业峰会、私享会）

### 步骤3：内容策略制定
基于线索的暂缓原因定制内容：
- **无预算**：ROI案例、成本节约数据、分期方案介绍
- **无时间线**：行业趋势报告、竞品动态、"变化的代价"内容
- **决策人未触达**：高管视角内容、行业领袖观点、C-level dinner邀请
- **需求不明确**：痛点诊断工具、自评checklist、免费workshop

### 步骤4：激活条件设置
定义线索从nurture状态重新激活为MQL的触发条件：
- 行为触发：下载高价值内容（如定价页/产品对比/Demo请求）
- 信号触发：企业获得融资、人事变动（新CTO/VP入职）、扩张信号
- 时间触发：到达预设review时间节点
- 主动触发：线索主动回复邮件/微信表达兴趣

### 步骤5：培育效果追踪
1. 记录每次触达的互动情况（打开/点击/回复/忽略）
2. 计算engagement score的变化趋势
3. 月度review nurture pool状态变化（激活/升温/持平/降温/流失）
4. 季度评估nurture计划ROI（投入触达成本vs最终转化收入）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| lead_id | String | 是 | 线索唯一标识 |
| nurture_reason | String | 是 | 进入nurture的原因 |
| lead_profile | Object | 是 | 线索画像 |
| bant_gaps | Object | 是 | BANT各维度缺口说明 |
| available_content | Array | 否 | 可用内容资产列表 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| nurture_tier | String | 培育层级（A/B/C） |
| touch_schedule | Array | 触达时间表和渠道计划 |
| content_plan | Array | 内容推送计划（主题+内容+时间） |
| activation_triggers | Array | 激活条件列表 |
| review_date | Date | 下次人工review日期 |
| success_metrics | Object | 培育效果衡量指标 |

## 最佳实践

1. **价值优先**：每次触达必须为线索提供价值（洞察/工具/人脉），而非简单"还有兴趣吗"的ping
2. **尊重节奏**：如线索多次未互动，应降低触达频率而非增加——避免骚扰导致永久流失
3. **个性化不可少**：即使是自动化nurture序列，也要在关键节点插入个性化元素
4. **及时激活**：激活信号出现后，应在24小时内由SDR主动跟进，不可延误
5. **定期清理**：在nurture pool超过12个月且无任何互动的线索，应标记归档释放资源

## 约束条件

- Nurture邮件发送频率不超过每周1次（避免被标记为垃圾邮件）
- 所有自动化触达必须支持一键退订
- 客户明确要求停止联系时，立即从nurture序列移除
- Nurture pool总量需控制在SDR可管理范围内（人均不超过100条）
