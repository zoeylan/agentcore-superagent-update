# Pipeline Review材料生成技能 (Pipeline Review Generation)

## 概述

为每周销售Pipeline Review会议自动生成结构化的盘点材料，包括商机状态汇总、风险分析、预测变化说明和行动建议。高质量的Review材料是销售管理层有效决策的基础，目标是将Review会议从"信息收集会"转变为"策略决策会"。

## 适用场景

- 每周一次的标准Pipeline Review会议材料准备
- 月度业绩预测会议的深度Pipeline分析
- 季度末冲刺的商机紧急盘点
- 新任销售经理接手时的Pipeline状态交接
- 管理层要求的临时Pipeline报告

## 执行步骤

### 步骤1：数据汇总与快照
1. 拉取所有活跃商机的最新状态
2. 与上周数据对比，标注变化（新增/推进/停滞/丢失/金额变化）
3. 计算关键汇总指标：
   - Pipeline总额和加权Pipeline额
   - 各阶段商机数量和金额分布
   - 本周新增商机 vs 关闭商机
   - Pipeline Coverage（Pipeline额/本季度剩余配额）

### 步骤2：逐单分析（Top Deals）
对金额Top 10或全部高优先级商机生成单独分析卡片：
- 商机基本信息（客户/金额/阶段/预计日期/Owner）
- 本周进展摘要（做了什么/有什么新信息）
- 风险评估（红/黄/绿 + 风险因素说明）
- 下一步行动（具体action + 负责人 + 截止日期）
- 赢率评估及变化（相比上周上升/下降/不变）

### 步骤3：风险与异常标注
自动识别需要Review会议重点讨论的商机：
- 停滞商机（超阶段平均时长1.5倍）
- 金额变更商机（本周金额上调或下调>20%）
- 日期推迟商机（预计关闭日期延后）
- 关键信息缺失商机（MEDDIC核心字段不完整）
- 即将到期商机（预计关闭日期在本周或下周）

### 步骤4：预测影响分析
1. 本周Pipeline变化对季度预测的影响
2. Best Case / Commit / Most Likely三档预测更新
3. 与上周预测对比，标注变化幅度和原因
4. 如预测缺口明显，提出弥补建议（加速推进哪些商机/增加哪些线索投入）

### 步骤5：Review材料输出
生成结构化Review文档：
1. 一页摘要（Executive Summary）
2. 关键变化一览表
3. 逐单分析卡片
4. 风险预警列表
5. 建议讨论议题
6. 行动项跟踪表（上周action items完成情况）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pipeline_data | Object | 是 | 全量活跃商机数据 |
| last_week_snapshot | Object | 是 | 上周Pipeline快照 |
| quota_data | Object | 是 | 季度配额和已完成数据 |
| previous_actions | Array | 否 | 上周Review的action items |
| focus_deals | Array | 否 | 指定重点关注的商机列表 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| executive_summary | Object | 一页摘要 |
| pipeline_metrics | Object | Pipeline关键指标 |
| deal_cards | Array | 逐单分析卡片 |
| risk_alerts | Array | 风险预警列表 |
| forecast_update | Object | 预测更新分析 |
| discussion_topics | Array | 建议讨论议题 |
| action_tracking | Array | 行动项跟踪 |

## 最佳实践

1. **前一天生成**：Review材料必须在会议前一天（通常周四）生成并发送，给与参会者提前准备时间
2. **增量为主**：重点突出本周变化，而非重复展示不变信息——管理层时间宝贵
3. **可行动建议**：每个风险点都要附带建议行动，而非仅罗列问题
4. **一致性口径**：预测数据口径需与财务/运营对齐，避免"多个版本的真相"
5. **历史对比**：提供与去年同期/上月同期的对比，帮助管理层判断趋势

## 约束条件

- Review材料中的商机金额不可对外分享（内部机密）
- 个人业绩数据仅限该销售及其上级查看
- 材料生成不可遗漏任何活跃商机（完整性100%）
- 预测数据必须标注置信区间，不可给出无范围的点估计
