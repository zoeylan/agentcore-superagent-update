# 数据补全技能 (Data Enrichment)

## 概述

通过调用外部数据源（工商信息API、企业征信平台、社交网络等）对线索的企业和联系人信息进行自动化补全，提升数据完整度至85%以上。完整的企业画像是精准评分和智能分配的前提条件。

## 适用场景

- 新线索入库后的自动化企业信息补全
- 存量线索库的定期数据刷新（季度更新）
- 展会/合作伙伴批量导入线索的信息补齐
- 商机创建前的客户深度背调

## 执行步骤

### 步骤1：确定补全需求
1. 扫描线索记录，识别缺失字段
2. 优先补全关键字段：行业分类、企业规模（员工数）、注册资本、融资阶段、联系人职位
3. 评估已有信息可作为查询Key（公司名称、统一社会信用代码、域名等）

### 步骤2：企业基础信息补全
调用工商信息API获取：
- 企业全称和曾用名
- 注册资本和实缴资本
- 成立日期
- 行业分类（国标行业代码）
- 注册地址和经营范围
- 法人代表信息
- 员工规模区间（1-50/50-200/200-1000/1000+）

### 步骤3：融资与商业信息补全
调用企业征信/投融资数据源获取：
- 融资阶段（天使轮/A轮/.../上市/未融资）
- 最近一轮融资金额和时间
- 投资方信息
- 营收规模区间（如可获取）
- 上下游合作伙伴信息

### 步骤4：联系人信息补全
通过社交网络和企业通讯录获取：
- 联系人职位确认和层级判断
- 部门归属
- 职业经历（前公司/任职时间）
- 社交媒体主页（LinkedIn/脉脉）

### 步骤5：数据验证与写入
1. 交叉验证多源数据一致性
2. 对于冲突数据，标注数据源和获取时间，保留多版本
3. 更新线索记录并标注补全字段来源
4. 计算补全后数据完整度

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| lead_id | String | 是 | 线索唯一标识 |
| company_name | String | 是 | 企业名称（查询Key） |
| existing_data | Object | 是 | 已有线索数据 |
| enrichment_level | String | 否 | 补全深度（basic/standard/deep），默认standard |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| enriched_data | Object | 补全后的完整数据 |
| enrichment_fields | Array | 本次补全的字段列表 |
| data_sources | Object | 各字段的数据来源标注 |
| completeness_score | Number | 补全后数据完整度百分比 |
| confidence_scores | Object | 各补全字段的置信度 |
| enrichment_failed_fields | Array | 补全失败的字段及原因 |

## 最佳实践

1. **分级补全策略**：basic级补全（工商基础信息）5秒内返回；standard级（含融资信息）30秒内；deep级（含联系人深度画像）可异步返回
2. **缓存机制**：同一企业30天内重复查询使用缓存结果，减少API调用成本
3. **数据时效性标注**：所有补全数据必须标注获取时间，超过90天的数据标记为"待刷新"
4. **隐私合规**：联系人信息补全需符合个人信息保护法要求，不得采集非公开个人隐私数据
5. **成本控制**：单条线索补全API调用成本控制在1元以内，批量补全使用批量折扣接口

## 约束条件

- 不得使用非授权数据源或爬虫获取个人信息
- 补全数据仅用于内部销售活动，不得外传或转售
- 对于高敏感行业（政府/军工/金融），企业信息补全需额外审批
- API调用失败时使用优雅降级策略，不阻塞主流程
