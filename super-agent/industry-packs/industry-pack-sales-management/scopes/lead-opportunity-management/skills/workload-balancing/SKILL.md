# 工作负载均衡技能 (Workload Balancing)

## 概述

实时监控SDR团队的工作负载分布，确保线索分配不会造成个别SDR过载而影响跟进质量，同时避免部分SDR闲置浪费产能。通过动态调整分配权重实现团队效率最大化。

## 适用场景

- 线索分配前的SDR负载检查
- 团队负载周度/月度平衡分析
- SDR新入职/离职时的线索重分配
- 线索暴增（如展会后）时的容量规划
- 团队扩容/缩编决策支持

## 执行步骤

### 步骤1：负载指标采集
实时采集每位SDR的工作负载数据：
- 在手MQL线索数量（未关闭）
- 本周新增分配数量
- 本周已完成外呼数量vs计划数量
- 当前跟进中商机数量
- SDR自报工作饱和度（如有）

### 步骤2：负载评估计算
```
负载指数 = (在手MQL数/上限20) × 40% + (本周新增/日限8×剩余工作日) × 30% + (跟进中商机数/上限5) × 30%
```
- 负载指数 > 0.9 → 满载（暂停新分配）
- 负载指数 0.7-0.9 → 高负载（降低分配权重）
- 负载指数 0.4-0.7 → 正常（标准分配权重）
- 负载指数 < 0.4 → 低负载（提高分配权重）

### 步骤3：团队平衡分析
1. 计算团队负载方差和标准差
2. 识别负载严重不均的情况（个人负载与团队均值偏差>50%）
3. 对于持续低负载SDR，分析原因（技能不匹配/地域冷门/产品线窄）
4. 对于持续高负载SDR，评估是否需要支援或调整覆盖范围

### 步骤4：动态权重调整
1. 基于负载评估结果，实时更新各SDR的分配权重
2. 权重调整立即生效，影响后续分配决策
3. 对于满载SDR，将其从可分配列表中临时移除
4. 记录权重调整日志，用于后续效果分析

### 步骤5：容量预警与规划
1. 当团队整体负载指数>0.8，触发容量预警
2. 预测未来1周的线索入库量（基于历史趋势和已知营销活动）
3. 如预测负载将超出团队容量，提前建议管理层调配资源
4. 生成容量规划报告，支持团队扩编决策

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| team_members | Array | 是 | SDR团队成员列表 |
| current_assignments | Object | 是 | 各SDR当前线索和商机分配情况 |
| activity_data | Object | 是 | 各SDR近期活动数据 |
| capacity_config | Object | 否 | 容量配置参数（上限值等） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| load_index | Object | 各SDR负载指数 |
| routing_weights | Object | 各SDR当前分配权重 |
| balance_report | Object | 团队平衡分析报告 |
| capacity_alert | Object | 容量预警信息（如有） |
| recommendations | Array | 调整建议列表 |

## 最佳实践

1. **质量优先于均衡**：不为了均衡而强行将不匹配的线索分配给低负载SDR
2. **渐进式调整**：权重调整每次变化不超过20%，避免剧烈波动
3. **预见性管理**：在大型营销活动前（如展会），提前评估容量并做好分配预案
4. **新人保护**：新入职SDR前2周降低分配数量（每日上限3条），给予充分学习时间
5. **数据驱动决策**：团队扩编建议必须基于至少4周的负载数据趋势

## 约束条件

- 负载数据更新频率不低于每小时一次
- 满载状态的SDR不可强制分配新线索（VIP线索除外，需主管确认）
- 权重调整不可影响已分配线索的归属
- 容量规划报告需与管理层确认后方可执行调整
