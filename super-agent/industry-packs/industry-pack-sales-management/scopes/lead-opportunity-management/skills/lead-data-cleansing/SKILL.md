# 线索数据清洗技能 (Lead Data Cleansing)

## 概述

本技能负责对多渠道采集的原始线索数据进行标准化、去重和质量检查，确保进入销售漏斗的线索数据干净、准确、完整。数据质量直接影响后续评分准确性和SDR跟进效率，是整个线索管理流程的基石。

## 适用场景

- 官网表单、内容下载、Webinar注册等Inbound线索的实时清洗
- 展会扫码、合作伙伴推荐等Outbound线索的批量导入清洗
- 定期存量线索库的数据质量巡检和修复
- CRM系统数据迁移时的数据治理

## 执行步骤

### 步骤1：数据标准化
1. 统一字段格式：手机号标准化（+86格式）、邮箱小写化、公司名去除后缀变体（"有限公司"/"Ltd"统一）
2. 字段映射：将不同渠道的字段名映射到标准数据模型（company_name, contact_name, title, phone, email, source_channel, intent_product）
3. 空值处理：识别必填字段缺失情况，标记待补全项

### 步骤2：去重检测
1. 执行三重匹配逻辑：
   - 精确匹配：邮箱完全一致
   - 模糊匹配：公司名相似度>85% AND 手机号后8位一致
   - 跨渠道匹配：同一联系人不同渠道来源的行为合并
2. 去重决策：
   - 完全重复：合并记录，保留最新互动时间和来源信息
   - 疑似重复：标记人工复核，不自动删除
   - 新增线索：正常进入后续流程

### 步骤3：数据质量评分
1. 计算数据完整度分数（必填字段填写率）
2. 检测明显异常数据（测试邮箱如test@、无效手机号、明显虚假公司名）
3. 标记数据质量等级：A（完整度>90%）、B（70-90%）、C（<70%需补全）

### 步骤4：输出清洗报告
1. 生成批次清洗统计：总量、去重数、无效数、各质量等级分布
2. 记录清洗操作日志（可追溯）
3. 将清洗后数据传递至下一步流程

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| raw_leads | Array | 是 | 原始线索数据列表 |
| source_channel | String | 是 | 来源渠道标识 |
| batch_id | String | 是 | 批次标识符 |
| dedup_scope | String | 否 | 去重范围（全量/增量），默认增量 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| cleaned_leads | Array | 清洗后的标准化线索列表 |
| duplicates | Array | 识别的重复记录及处理结果 |
| invalid_records | Array | 无效记录及原因 |
| quality_report | Object | 清洗质量统计报告 |
| processing_time | Number | 处理耗时（秒） |

## 最佳实践

1. **宁可保留不可误删**：去重准确率目标>98%，对于相似度在85-95%之间的疑似重复，应标记人工复核而非自动合并
2. **保留历史痕迹**：合并操作必须保留原始记录的来源和时间信息，用于后续渠道归因分析
3. **批量与实时分离**：实时清洗（单条）响应时间<5秒，批量清洗（>100条）可异步处理但需在15分钟内完成
4. **定期规则更新**：每月review去重规则的误报率和漏报率，基于实际case调优匹配阈值

## 约束条件

- 不可删除任何原始数据，只能标记状态
- 去重匹配算法不可跨租户/跨组织执行
- 个人隐私信息（手机号/邮箱）在日志中必须脱敏显示
- 单批次处理上限5000条，超出需分批执行
