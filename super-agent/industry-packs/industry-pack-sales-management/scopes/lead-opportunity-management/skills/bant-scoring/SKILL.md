# BANT线索评分技能 (BANT Scoring)

## 概述

基于BANT（Budget预算、Authority决策权、Need需求、Timeline时间线）模型对线索进行自动化评分，为线索分级和优先级排序提供量化依据。评分满分100分，每个维度25分，评分结果直接决定线索的流转路径（MQL/Nurture/低质）。

## 适用场景

- 新入库线索的初始自动评分
- 线索信息更新后的评分刷新（如SDR沟通获取新信息后）
- 评分模型月度校准时的批量重新评分
- 不同渠道线索质量对比分析

## 执行步骤

### 步骤1：Budget（预算）维度评分 [0-25分]
评估信号采集：
- 25分：明确提及预算范围且与产品定价匹配
- 20分：所在公司规模/融资阶段暗示有预算能力
- 15分：同行业同规模企业历史采购数据推断
- 10分：有内容下载/Webinar参与但无预算信号
- 5分：仅有基础互动（官网浏览/简单咨询）
- 0分：明确无预算或公司规模过小（<10人且无融资）

### 步骤2：Authority（决策权）维度评分 [0-25分]
评估信号采集：
- 25分：联系人为VP/C-level/业务负责人，有直接采购决策权
- 20分：联系人为总监级别，是评估团队核心成员
- 15分：联系人为经理级别，能影响决策但非最终决策者
- 10分：联系人为执行层，但能触达决策者
- 5分：联系人职位不明或为基层员工
- 0分：明确为个人学习/调研，非企业采购行为

### 步骤3：Need（需求）维度评分 [0-25分]
评估信号采集：
- 25分：明确表达了与产品匹配的业务痛点和需求
- 20分：下载了解决方案白皮书/观看了产品Demo
- 15分：访问了定价页面/产品对比页面
- 10分：下载了行业报告/参加了通用Webinar
- 5分：仅订阅了邮件列表或关注了公众号
- 0分：无任何与产品相关的互动信号

### 步骤4：Timeline（时间线）维度评分 [0-25分]
评估信号采集：
- 25分：明确表示1个月内需要上线/采购
- 20分：表示本季度有计划评估
- 15分：表示半年内有计划
- 10分：表示有意向但无明确时间计划
- 5分：仅为长期了解/知识储备
- 0分：明确表示1年内无计划

### 步骤5：综合评分与分级
1. 计算总分 = B + A + N + T
2. 分级判定：
   - ≥60分 → MQL（Marketing Qualified Lead），进入分配队列
   - 30-59分 → Nurture Pool，进入培育序列
   - <30分 → 低质线索，标记渠道质量问题
3. 生成评分卡（含各维度得分和打分依据）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| lead_data | Object | 是 | 标准化后的线索数据 |
| behavioral_signals | Array | 否 | 行为数据（页面访问、内容下载等） |
| enrichment_data | Object | 否 | 工商补全数据（行业、规模等） |
| historical_context | Object | 否 | 历史互动记录（如存量客户新线索） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| total_score | Number | 总分（0-100） |
| budget_score | Number | Budget维度得分及依据 |
| authority_score | Number | Authority维度得分及依据 |
| need_score | Number | Need维度得分及依据 |
| timeline_score | Number | Timeline维度得分及依据 |
| grade | String | 分级结果（MQL/Nurture/Low） |
| scoring_rationale | String | 评分综合说明 |
| confidence_level | String | 评分置信度（高/中/低） |

## 最佳实践

1. **信号权重动态调整**：不同行业和产品线可能需要调整各维度权重，如高客单价产品Budget权重更高
2. **多信号叠加**：同一维度如有多个正向信号，应取最高分而非累加（避免超过25分）
3. **负面信号扣分**：如有明确负面信号（如"已有竞品且很满意"），应在对应维度大幅降分
4. **月度校准必须做**：每月对比评分与实际转化数据，识别评分偏高/偏低的维度并调整评分标准
5. **置信度标注**：基于信息完整度标注评分置信度——信息越不完整，置信度越低，应留有提升空间

## 约束条件

- 评分逻辑必须透明可解释，禁止使用不可解释的黑箱模型
- 每次评分必须记录打分依据，用于后续模型校准
- 评分模型更新需经销售运营负责人审批后方可生效
- 存量线索重新评分不可覆盖历史评分记录（需保留版本历史）
