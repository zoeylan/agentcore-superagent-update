# 首次价值实现加速 (Time-to-Value Acceleration)

## 概述
通过系统化方法加速客户首次价值实现（Time-to-Value，简称TTV），确保客户在签约后30天内体验到产品核心价值。TTV是客户早期留存的关键预测指标——TTV每延长一周，首年流失概率增加约15%。

## 适用场景
- 新客户签约后的快速价值交付
- 客户增购新模块后的快速激活
- 实施延期时的最小可行价值交付（MVP上线）
- 客户内部推广时新部门的快速见效

## 执行步骤

### 步骤1：定义价值目标
- **动作**：
  - 回顾客户购买动机和核心痛点
  - 与客户对齐"成功"的定义——具体、可衡量的业务指标
  - 确定最小可行的价值证明场景（不求全，求快速见效）
  - 设定30天TTV目标和分周里程碑
- **输出**：价值目标定义文档、Success Criteria

### 步骤2：设计快速见效路径
- **动作**：
  - 从产品功能中选择最能直接解决客户Top 1痛点的模块
  - 设计最短实施路径——跳过非必要配置，先上核心功能
  - 识别快速见效的"速赢"（Quick Win）场景
  - 制定"Day 1 / Week 1 / Week 2 / Week 4"渐进式激活计划
- **输出**：快速见效实施路径、Quick Win清单

### 步骤3：聚焦式实施
- **动作**：
  - 集中资源优先完成核心价值场景的配置
  - 为关键用户（Champion）提供1对1培训
  - 准备简化版操作指南（非完整手册，聚焦核心流程）
  - 每周与客户Champion确认使用进展
- **输出**：核心功能配置完成确认、培训记录

### 步骤4：价值验证
- **动作**：
  - 收集客户使用数据证明价值实现（如效率提升百分比、错误率下降等）
  - 与客户共同确认"首次价值实现"里程碑达成
  - 制作价值报告——用数据说话，让客户内部决策者看到ROI
  - 获取客户正面反馈（可用于后续case study）
- **输出**：价值验证报告、ROI初步计算

### 步骤5：价值扩展规划
- **动作**：
  - 基于首次价值实现的经验，规划后续功能扩展路径
  - 设定下一个价值里程碑（90天目标）
  - 将TTV达成情况同步给健康度管理师作为初始评分依据
- **输出**：价值扩展路线图

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户痛点 | array | 是 | 客户购买动机和核心痛点列表 |
| 购买模块 | array | 是 | 客户购买的产品功能模块 |
| 客户规模 | enum | 是 | 大型/中型/小型 |
| 关键Champion | object | 是 | 客户方推动者信息 |
| 签约日期 | date | 是 | 合同签署日期（TTV起始计算点） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| TTV目标定义 | document | 可衡量的价值实现标准 |
| 快速见效计划 | document | 4周渐进式激活计划 |
| 价值验证报告 | document | 量化的价值实现证据 |
| TTV达成天数 | number | 实际TTV天数（对比30天目标） |

## 最佳实践
1. **Less is More**：初始阶段不要贪多，一个核心场景跑通胜过十个功能半成品
2. **Champion优先**：先让客户方Champion感受到价值，借助其内部影响力推广
3. **数据说话**：用Before/After对比数据证明价值，而非主观感受
4. **快速迭代**：如果预设路径受阻，立即调整，不要等完美方案
5. **仪式感**：TTV达成时组织正式的"Success Milestone"庆祝，强化客户信心

## 约束条件
- TTV目标必须在签约后1周内与客户对齐确认
- 超过30天未达成TTV需启动根因分析并上报
- 快速见效不能牺牲数据安全和系统稳定性
- 价值验证数据必须真实，不可人为美化指标
