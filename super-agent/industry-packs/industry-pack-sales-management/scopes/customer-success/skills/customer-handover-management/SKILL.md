# 客户交接管理 (Customer Handover Management)

## 概述
管理从销售团队到客户成功团队（CSM）的客户交接全过程，确保客户信息、商务承诺和期望的完整传递，避免信息断层导致的客户不满和早期流失风险。

## 适用场景
- 新合同签署完成后的客户交接
- 客户CSM变更时的内部交接
- 实施期间项目负责人变更的工作交接
- 跨区域客户迁移的交接

## 执行步骤

### 步骤1：触发交接流程
- **触发条件**：合同签署完成通知到达
- **动作**：在CRM系统中创建交接任务，设置5个工作日完成期限
- **输出**：交接任务工单，含优先级和截止时间

### 步骤2：收集交接信息
- **动作**：从销售团队获取以下必填信息：
  - 客户背景：公司基本信息、行业、规模、业务痛点
  - 商务条件：合同金额、付款方式、折扣情况、合同期限
  - 期望与承诺：销售过程中的所有口头和书面承诺
  - 决策人信息：关键决策人KP、日常对接人、技术负责人
  - 技术环境：现有IT系统、集成需求、数据迁移需求
- **输出**：交接信息清单（完整度检查）

### 步骤3：信息完整性校验
- **动作**：检查五大模块（背景/商务/承诺/决策人/技术）的完整度
- **决策点**：
  - 完整度≥90%：进入下一步
  - 完整度<90%：退回销售补充，标记缺失项，设置24小时补全期限
- **输出**：完整性校验报告

### 步骤4：组织交接会议
- **动作**：安排销售AE、CSM、CSM经理三方参加的交接会议
- **议程**：客户背景介绍→商务条件确认→承诺事项逐条确认→技术需求讨论→责任交接确认
- **时长**：30-60分钟
- **输出**：交接会议纪要

### 步骤5：双方签字确认
- **动作**：销售和CSM双方在交接文档上签字确认
- **含义**：销售确认信息真实完整，CSM确认接收并理解
- **输出**：已签署的交接确认单

### 步骤6：CSM指派与通知客户
- **动作**：正式指派CSM负责人，向客户发送欢迎邮件介绍CSM
- **邮件内容**：CSM介绍、服务承诺、联系方式、下一步安排
- **输出**：客户欢迎邮件发送记录

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 合同ID | string | 是 | 关联合同唯一标识 |
| 客户名称 | string | 是 | 客户公司全称 |
| 销售负责人 | string | 是 | 原销售AE |
| 合同金额 | number | 是 | 年合同金额(元) |
| 签约日期 | date | 是 | 合同签署日期 |
| 产品模块 | array | 是 | 购买的产品/模块列表 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 交接文档 | document | 标准化交接文档(含五大模块) |
| 交接确认单 | document | 双方签字确认记录 |
| 实施启动建议 | document | 基于交接信息的实施计划初步建议 |
| CSM指派通知 | notification | 发送给客户的CSM介绍通知 |

## 最佳实践
1. **速度第一**：5个工作日是硬性SLA，优秀团队可做到3天内完成
2. **承诺对齐**：交接会议上逐条确认销售承诺，不留模糊空间
3. **温度传递**：不只传递信息，还要传递销售与客户建立的信任关系
4. **提前预热**：销售在签约前就向客户介绍CSM，降低"被抛弃"感
5. **留存备份**：所有交接材料在系统中归档，便于后续查询和争议解决

## 约束条件
- 交接完成前销售不得从CRM中释放客户所有权
- 交接信息一旦确认不可单方面修改，变更需双方同意
- 超过5个工作日未完成交接自动升级至CSM总监
- 交接文档中的客户敏感信息需遵循数据安全等级管理
