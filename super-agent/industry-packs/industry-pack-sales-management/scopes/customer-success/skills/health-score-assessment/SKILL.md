# 客户健康度评估 (Health Score Assessment)

## 概述
通过多维度数据综合评估客户健康状态，产出绿色/黄色/红色三级健康度评分，为CSM团队的客户管理优先级排序和干预决策提供数据化依据。健康度评分是客户成功管理的核心工具，是流失预测和续约判断的基础。

## 适用场景
- 月度例行健康度评分更新（覆盖100%活跃客户）
- 新客户上线后首次健康度基线建立
- 突发事件（关键联系人离职、重大投诉等）后的即时重评
- 季度评分模型校准

## 执行步骤

### 步骤1：数据采集
- **动作**：从各数据源采集以下四大维度数据：
  - **产品使用维度（权重35%）**：
    - 登录频率（日活/周活/月活对比license数）
    - 功能覆盖度（使用功能数/购买功能数）
    - 活跃用户占比（实际活跃用户/总license数）
    - 使用深度（高级功能使用率）
  - **支持体验维度（权重25%）**：
    - 工单数量趋势（环比变化）
    - 平均解决时间MTTR
    - 工单CSAT评分
    - 未关闭工单数量
  - **商务关系维度（权重25%）**：
    - 付款及时性（DSO应收账款周转天数）
    - 沟通频率（CSM互动次数/月）
    - 关键联系人变动（Champion/决策者是否变更）
    - 合同剩余时间
  - **满意度维度（权重15%）**：
    - 最近NPS分数
    - NPS变化趋势
    - 投诉频率
- **输出**：原始数据采集报告

### 步骤2：评分计算
- **动作**：
  - 每个子指标标准化为0-100分
  - 按维度权重加权计算综合分数
  - 应用行业基准修正（不同规模客户使用预期不同）
  - 产出最终健康度分数和颜色分级
- **评分标准**：
  - 绿色（健康）：综合分≥70分
  - 黄色（关注）：综合分50-69分
  - 红色（风险）：综合分<50分
- **输出**：健康度评分卡

### 步骤3：趋势分析
- **动作**：
  - 对比前3个月评分变化趋势
  - 识别快速下滑客户（即使当前仍为绿色）
  - 标注改善客户（从红/黄提升的正面案例）
  - 计算组合健康度分布（绿:黄:红比例变化）
- **输出**：趋势分析报告

### 步骤4：分级触发动作
- **动作**：
  - 绿色客户：标记增购机会，保持月度check-in节奏
  - 黄色客户：生成关注计划建议，提升touchpoint频率至双周
  - 红色客户：触发48小时挽回计划制定流程
  - 新增红色客户：立即通知CSM经理
- **输出**：分级动作触发记录

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户ID | string | 是 | 客户唯一标识 |
| 评估周期 | string | 是 | 评估的月份/周期 |
| 产品使用数据 | object | 是 | 来自产品系统的使用统计 |
| 工单数据 | object | 是 | 来自支持系统的工单统计 |
| 财务数据 | object | 是 | 付款记录和应收情况 |
| NPS数据 | object | 否 | 最近一次NPS调查结果 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 健康度分数 | number | 0-100综合评分 |
| 健康度等级 | enum | 绿色/黄色/红色 |
| 各维度明细分 | object | 四个维度各自得分 |
| 趋势标签 | enum | 上升/稳定/下滑 |
| 触发动作 | array | 需要执行的后续动作列表 |
| 风险因子 | array | 导致扣分的主要因子说明 |

## 最佳实践
1. **数据优先于直觉**：评分结果基于客观数据，避免CSM主观偏见
2. **关注变化而非绝对值**：绿色客户快速下滑比稳定黄色更值得关注
3. **权重定期校准**：每季度根据实际流失数据回溯验证模型有效性
4. **信号组合解读**：单一指标下滑不一定是风险，多维度同时恶化才是强信号
5. **人机结合**：评分系统提供基础判断，CSM补充定性洞察

## 约束条件
- 月度评分更新必须在每月前5个工作日内完成，覆盖率100%
- 评分模型参数调整需经CSM总监审批
- 客户数据访问遵循最小权限原则
- 评分结果对外（客户侧）保密，仅供内部决策使用
