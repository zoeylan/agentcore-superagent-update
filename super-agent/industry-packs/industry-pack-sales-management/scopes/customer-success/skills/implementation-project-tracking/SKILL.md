# 实施项目追踪 (Implementation Project Tracking)

## 概述
对客户实施项目进行全流程追踪和管理，从需求确认到上线验收，确保项目按时交付、风险可控、客户满意。实施期是客户感知最敏感的阶段，直接影响后续使用深度和续约意愿。

## 适用场景
- 新客户签约后的首次实施项目
- 客户增购新模块后的增量实施
- 系统升级或迁移项目
- 客户组织架构变更后的调整实施

## 执行步骤

### 步骤1：需求确认与范围定义
- **触发条件**：交接完成，CSM正式接手
- **动作**：
  - 与客户技术对接人进行需求详细确认
  - 对比合同范围与客户实际需求，识别差异
  - 产出项目范围说明书（SOW），双方确认签字
- **输出**：需求确认文档、项目范围说明书

### 步骤2：制定实施计划
- **动作**：
  - 根据需求复杂度确定实施周期（标准30天/复杂45-60天）
  - 分解实施里程碑：环境准备→配置部署→数据迁移→集成测试→用户培训→上线验收
  - 明确每个里程碑的负责人、时间节点、前置条件和验收标准
  - 识别关键依赖（如客户IT资源、第三方接口）
- **输出**：实施计划甘特图、里程碑清单

### 步骤3：资源协调与环境准备
- **动作**：
  - 协调产品团队配置环境（测试环境+生产环境）
  - 协调技术团队处理集成和定制需求
  - 确认客户方对接人到位和可用时间
  - 准备培训材料和操作手册
- **输出**：资源确认表、环境就绪报告

### 步骤4：进度追踪与风险管理
- **动作**：
  - 每周召开项目进度会议（内部+客户侧各一次）
  - 更新项目进度看板（Doing/Done/Blocked）
  - 识别和记录风险：延期风险、技术风险、人员风险
  - 对已识别风险制定应对措施（缓解/转移/接受）
- **输出**：周进度报告、风险登记册

### 步骤5：验收与上线
- **动作**：
  - 执行UAT（用户验收测试）
  - 收集测试问题并推动修复
  - 组织客户签署上线验收单
  - 制定上线后支持计划（前2周加强支持频率）
- **输出**：UAT报告、上线验收单、支持计划

### 步骤6：实施复盘
- **动作**：
  - 实施完成后5个工作日内进行项目复盘
  - 总结成功经验和改进项
  - 将客户信息和实施细节同步给健康度管理师
- **输出**：实施复盘报告

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户ID | string | 是 | 客户唯一标识 |
| 交接文档 | document | 是 | 已完成的交接文档 |
| 合同范围 | array | 是 | 购买的产品/功能模块清单 |
| 客户技术环境 | object | 是 | 现有系统、集成需求 |
| 实施优先级 | enum | 是 | 高/中/低 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 实施计划 | document | 含里程碑和甘特图的完整计划 |
| 周进度报告 | document | 每周项目状态更新 |
| 风险登记册 | document | 所有已识别风险和应对措施 |
| 上线验收单 | document | 客户签署的验收确认 |
| 复盘报告 | document | 实施总结和经验教训 |

## 最佳实践
1. **前置风险识别**：实施启动时就识别Top 3风险，不要等问题爆发
2. **客户期望管理**：定期同步进度，坏消息越早说越好
3. **里程碑庆祝**：每完成一个里程碑向客户展示进展，增强信心
4. **缓冲时间**：计划中预留20%缓冲时间应对不可预见问题
5. **文档先行**：所有需求变更必须书面确认后再执行

## 约束条件
- 实施计划变更需经客户和内部PM双方同意
- 超出合同范围的需求必须走变更流程（可能涉及补充协议）
- 上线延期超过2周需升级至CSM总监和客户VP层面沟通
- 数据迁移前必须完成数据备份和回滚方案准备
