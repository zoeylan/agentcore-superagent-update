# 增购机会识别 (Expansion Opportunity Identification)

## 概述
通过分析客户产品使用数据、组织发展动态和行业趋势，系统性识别客户增购潜力，为NDR（净收入留存率）增长提供机会管线。优秀SaaS公司的NDR>120%，增购是从现有客户获取增量收入的核心引擎。目标是年增购收入占存量收入>20%。

## 适用场景
- 月度使用数据分析中发现的增量信号
- 客户组织架构变化（扩部门、增人员）
- 产品新功能发布后的老客户激活
- QBR中客户主动提及新需求
- 客户行业出现新合规或业务要求

## 执行步骤

### 步骤1：信号监控与识别
- **动作**：
  - **使用量信号**：
    - License使用率>80%（扩容需求）
    - 频繁触达功能使用上限（升级需求）
    - 高频使用未购买模块的试用功能
    - API调用量接近阈值
  - **组织扩展信号**：
    - 客户新设部门或办公室
    - 人员规模增长（从公开信息或客户沟通中获取）
    - 客户获得新融资或业务拓展
  - **需求表达信号**：
    - QBR中提及新业务挑战
    - 提交了产品功能需求工单
    - 询问未购买产品模块的信息
  - **行业趋势信号**：
    - 新法规要求（如数据合规）可能产生新需求
    - 行业数字化趋势推动新场景
- **输出**：增购信号清单

### 步骤2：机会验证与评估
- **动作**：
  - 对识别的信号进行BANT验证：
    - Budget：客户是否有预算支持
    - Authority：对接人是否有采购决策权
    - Need：需求是否真实紧迫
    - Timeline：是否有明确的时间要求
  - 评估增购潜力金额
  - 评估成交概率
  - 确定机会类型：扩容(Seat Expansion)/升级(Upsell)/交叉销售(Cross-sell)
- **输出**：增购机会评估卡

### 步骤3：路由决策
- **决策标准**：
  - 增购金额≤年合同额50%：CSM自行推进
  - 增购金额>年合同额50%：转交销售团队作为新商机
  - 涉及全新产品线：转交对应产品销售
- **动作**：
  - 小额增购：CSM准备提案，直接与客户沟通
  - 大额增购：创建商机记录，交接给销售AE，提供客户背景和需求
- **输出**：路由决策记录

### 步骤4：机会培育
- **动作**（CSM自行推进的小额增购）：
  - 准备增购提案（价值匹配+方案+报价）
  - 安排产品演示或试用
  - 推动客户内部审批
  - 协调报价流程
- **输出**：增购提案、演示记录

### 步骤5：结果跟踪
- **动作**：
  - 记录增购结果（成功/失败/暂缓）
  - 分析增购成功模式（哪些信号转化率最高）
  - 更新增购机会pipeline
  - 月度汇总增购收入贡献
- **输出**：增购结果记录、月度增购报告

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户ID | string | 是 | 客户唯一标识 |
| 使用数据 | object | 是 | 产品使用统计和趋势 |
| 当前合同范围 | object | 是 | 已购买的模块和license数 |
| 客户互动记录 | array | 否 | 近期沟通中的需求表达 |
| 客户公开信息 | object | 否 | 融资、扩张等公开动态 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 增购机会列表 | array | 已识别的增购机会清单 |
| 机会评分 | number | 每个机会的成交概率评估 |
| 预计金额 | number | 增购预计金额 |
| 路由建议 | enum | CSM自理/转交销售 |
| 推进建议 | document | 下一步行动建议 |

## 最佳实践
1. **价值先于销售**：增购建议必须基于客户真实需求，不是为了完成指标强推
2. **数据驱动发现**：使用量数据是最客观的需求信号，优于主观判断
3. **时机很重要**：客户刚获得正面体验（TTV达成、问题解决后）是推荐增购的最佳时机
4. **低压力沟通**：以"帮助客户更好使用产品"的角度切入，而非销售姿态
5. **Champion赋能**：帮助客户方Champion准备内部汇报材料，降低其内部推动难度

## 约束条件
- 健康度为红色的客户不适合推进增购（先解决问题再说商务）
- 增购金额>年合同额50%必须转交销售，CSM不得自行处理
- 增购提案需经CSM经理审核后再发送客户
- 避免在续约谈判期间同时推增购（容易被客户作为谈判筹码）
