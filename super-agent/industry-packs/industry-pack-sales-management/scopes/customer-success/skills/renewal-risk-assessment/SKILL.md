# 续约风险评估 (Renewal Risk Assessment)

## 概述
在合同到期前90天系统性评估每个客户的续约风险等级，综合考量客户健康度、使用趋势、竞品动态和商务关系等因素，为续约策略制定提供数据化决策基础。续约风险评估是续约管理的第一步，直接决定后续投入的资源力度和策略方向。

## 适用场景
- 合同到期前90天的常规续约启动
- 客户健康度突然恶化时的提前评估
- 客户主动提出降级或不续约意向时的紧急评估
- 季度续约pipeline review的批量评估

## 执行步骤

### 步骤1：触发与信息收集
- **触发条件**：合同距到期90天（T-90）自动触发
- **动作**：
  - 获取客户当前健康度评分和趋势
  - 获取近12个月产品使用数据趋势
  - 获取客户CSM互动记录和关系温度
  - 收集竞品动态信息（是否有竞品接触信号）
  - 确认合同条款（自动续约/手动续约、涨价条款等）
- **输出**：续约评估信息包

### 步骤2：多维度风险评估
- **动作**：
  - **使用价值维度**（权重30%）：
    - 产品使用深度是否持续增长
    - 关键业务流程是否依赖产品
    - 用户覆盖率是否充分
  - **关系稳定维度**（权重25%）：
    - Champion是否仍在位且积极
    - 高层关系是否保持
    - 是否有新决策者进入需重新建立关系
  - **商务健康维度**（权重20%）：
    - 付款是否及时
    - 对价格是否敏感（历史是否多次要求折扣）
    - 预算环境是否收紧
  - **竞品威胁维度**（权重15%）：
    - 是否有竞品接触信号
    - 行业是否有新进入者
    - 客户是否提及替代方案
  - **合同条款维度**（权重10%）：
    - 是否有涨价条款触发
    - 合同是否有自动续约条款
    - 是否临近长约到期（3年约第3年比1年约风险高）
- **输出**：多维度风险评分卡

### 步骤3：风险分级与策略建议
- **动作**：
  - 综合评分产出风险等级：
    - 低风险（≥70分）：标准续约流程，CSM直接推进
    - 中风险（50-69分）：需制定针对性策略，可能需优惠激励
    - 高风险（<50分）：需高层介入，准备挽留资源包
  - 为每个风险等级输出策略建议
  - 估算续约概率和最可能的续约金额（标准/降级/升级）
- **输出**：风险等级判定、策略建议文档

### 步骤4：资源规划
- **动作**：
  - 根据风险等级确定投入资源
  - 高风险客户：分配高级CSM、预留折扣预算、安排VP互访
  - 中风险客户：增加CSM关注频率、准备增值方案
  - 低风险客户：标准流程推进，可适时推动升级续约
- **输出**：续约资源分配计划

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户ID | string | 是 | 客户唯一标识 |
| 合同到期日 | date | 是 | 当前合同到期日期 |
| 健康度评分 | object | 是 | 当前健康度及近6个月趋势 |
| 使用数据趋势 | object | 是 | 近12个月使用量变化 |
| 竞品信号 | array | 否 | 已知的竞品接触情况 |
| 合同条款 | object | 是 | 续约相关的合同条款 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 风险等级 | enum | 高/中/低 |
| 风险评分 | number | 0-100分 |
| 各维度得分 | object | 五维度明细得分 |
| 续约概率 | number | 预估续约概率百分比 |
| 策略建议 | document | 针对性续约策略建议 |
| 资源需求 | document | 续约推进所需资源 |

## 最佳实践
1. **提前启动不能等**：T-90是底线，高价值客户应T-120就开始关注
2. **数据+直觉结合**：评分系统给出基础判断，CSM的客户感知是重要补充
3. **关注变化趋势**：快速恶化的客户比持续低分客户更紧急
4. **竞品信号是最强信号**：客户一旦提及竞品，风险等级自动提升一级
5. **分清不愿续和不能续**：预算问题可以用方案灵活性解决，价值质疑需要深度沟通

## 约束条件
- T-90风险评估必须在3个工作日内完成
- 高风险客户评估结果需CSM总监确认
- 竞品信息收集不能采用不正当手段
- 风险评估结论对客户保密
