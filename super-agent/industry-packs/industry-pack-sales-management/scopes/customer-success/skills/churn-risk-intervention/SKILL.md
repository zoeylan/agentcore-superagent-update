# 流失风险干预 (Churn Risk Intervention)

## 概述
针对已识别的高风险（红色）和中风险（黄色）客户，制定并执行系统化的干预策略，将客户从流失边缘拉回健康轨道。红色客户48小时内必须产出挽回计划，黄色客户1周内制定关注计划。行业数据表明，有效的风险干预可将潜在流失客户的挽回率提升至40-60%。

## 适用场景
- 客户健康度评分降至红色（<50分）
- 客户健康度评分降至黄色（50-69分）
- 客户关键联系人（Champion/决策者）离职
- 客户明确表达不满或投诉升级
- 竞品入侵信号（客户提及竞品或要求对比）

## 执行步骤

### 步骤1：风险确认与根因分析
- **触发条件**：健康度评分降级或风险信号触发
- **动作**：
  - 确认风险信号真实性（排除数据异常）
  - 分析根因类别：产品问题/服务问题/价格问题/竞品威胁/客户业务变化
  - 评估客户价值和战略重要性（ARR、行业标杆、增长潜力）
  - 确定干预优先级和资源投入等级
- **输出**：根因分析报告

### 步骤2：制定干预计划
- **红色客户（48小时内完成）**：
  - 挽回计划必含：根因分析、行动项清单、每项负责人、执行时间线、成功标准、复评时间点
  - CSM经理必须审批并参与执行
  - 评估是否需要VP或CEO层面的高层互访
  - 准备客户挽留资源包（折扣授权、增值服务、优先支持等）
- **黄色客户（1周内完成）**：
  - 关注计划：增加touchpoint频率（月度→双周）、针对性改善措施
  - 设定4周后复评时间点
- **输出**：干预计划文档

### 步骤3：执行干预动作
- **动作**：
  - 主动联系客户关键联系人了解真实情况
  - 根据根因针对性解决：
    - 产品问题→协调产品团队优先处理/提供workaround
    - 服务问题→调整服务级别/更换CSM/增加支持频率
    - 价格问题→评估续约优惠方案（需审批授权）
    - 竞品威胁→准备竞品对比材料/强化差异化价值
    - 业务变化→重新评估产品匹配度/调整使用方案
  - 每周更新干预进展
- **输出**：干预执行记录

### 步骤4：效果评估与复评
- **动作**：
  - 红色客户2周后进行首次复评
  - 黄色客户4周后进行复评
  - 重新计算健康度评分
  - 判断干预效果：有效（颜色升级）/持平（保持观察）/无效（升级处理）
- **输出**：复评报告、颜色升降记录

### 步骤5：经验沉淀
- **动作**：
  - 记录干预策略和效果，形成案例库
  - 如果最终流失，输出流失复盘报告
  - 更新流失预测模型参数
- **输出**：干预案例记录、模型优化建议

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户ID | string | 是 | 客户唯一标识 |
| 当前健康度 | object | 是 | 当前评分及各维度明细 |
| 风险触发原因 | string | 是 | 触发干预的具体信号 |
| 客户ARR | number | 是 | 客户年度经常性收入 |
| 合同剩余月数 | number | 是 | 距离合同到期的月数 |
| 历史互动记录 | array | 是 | 近3个月的客户互动记录 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 根因分析 | document | 风险根因分类和深度分析 |
| 干预计划 | document | 含行动项、负责人、时间线的完整计划 |
| 执行进展 | document | 每周更新的执行状态 |
| 复评结果 | document | 干预效果评估和后续建议 |

## 最佳实践
1. **速度决定成败**：红色客户挽回窗口期很短，48小时内行动是底线
2. **倾听先于解决**：先了解客户真实诉求，不要急于给方案
3. **高层介入时机**：如果CSM层面无法解决，果断升级到VP互访
4. **组合拳思维**：单一措施往往不够，需要产品+服务+关系的组合干预
5. **不回避流失**：如果客户确实不适合继续使用，体面分手好过强留

## 约束条件
- 红色客户挽回计划48小时SLA是硬性要求
- 涉及折扣和优惠的挽留方案需经财务审批
- 高层互访需提前3天准备briefing材料
- 流失客户数据保留12个月，用于reactivation营销
