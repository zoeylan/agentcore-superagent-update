# 续约谈判管理 (Renewal Negotiation Management)

## 概述
管理从续约提案发送到合同签署的完整谈判过程，确保在T-0（合同到期日）前完成续约签署。续约谈判需要在维护客户关系、保护公司收入和控制折扣力度之间取得平衡。目标：续约率>90%，续约逾期率<5%。

## 适用场景
- 标准续约流程中的提案发送和签署推动
- 客户提出降级需求时的协商
- 客户表达不续约意向时的挽留谈判
- 涨价条款触发时的价格协商
- 多年期续约的长期方案谈判

## 执行步骤

### 步骤1：续约提案准备（T-60天）
- **动作**：
  - 基于风险评估结果确定续约策略：
    - 标准续约：按原条件续约，适当展示新增价值
    - 优惠续约：提供续约优惠（多年签折扣、新功能赠送等）
    - 升级续约：推荐更高版本或增加模块
  - 准备续约提案文档：
    - 合作价值回顾（过去一年的成果和ROI数据）
    - 续约方案明细（产品、服务、价格、期限）
    - 新增价值亮点（产品更新路线图、新增服务承诺）
    - 优惠条件（如有）及其条件
  - 内部审批续约条件
- **输出**：续约提案文档

### 步骤2：提案发送与初步沟通
- **动作**：
  - 通过正式渠道发送续约提案
  - 安排与客户关键决策人的续约沟通会议
  - 了解客户续约意向和关注点
  - 记录客户初步反馈和潜在异议
- **输出**：客户反馈记录

### 步骤3：异议处理与方案调整
- **常见异议及应对**：
  - 价格太高→展示ROI数据、对比竞品TCO、提供灵活付款方式
  - 使用率不高→承诺CSM加强辅导、制定使用提升计划
  - 考虑竞品→差异化价值强调、迁移成本分析、增值承诺
  - 预算收紧→方案裁剪建议、分阶段付款、短期续约过渡
  - 需要降级→分析原因、最小保留方案、未来升级路径
- **动作**：根据客户反馈调整方案（可能需要重新内部审批）
- **输出**：调整后方案、异议处理记录

### 步骤4：谈判推进（T-30天内完成）
- **动作**：
  - 推动客户进入内部审批流程
  - 协助客户方Champion准备内部汇报材料
  - 处理合同条款细节协商
  - 保持每周至少一次跟进节奏
  - 如遇僵局：升级到VP层面沟通
- **输出**：谈判进展记录

### 步骤5：签署与闭环
- **动作**：
  - 确认最终续约条件双方一致
  - 触发合同生成流程（可能需要报价与合同Scope协作）
  - 推动电子签署完成
  - 更新CRM系统续约状态
  - 将续约结果数据同步给销售数据分析Scope
- **输出**：签署完成确认、续约数据记录

### 步骤6：异常处理
- **客户确认不续约**：
  - 24小时内升级处理
  - VP层面进行最后挽留尝试
  - 如确定流失：启动优雅离场流程（数据导出、过渡期安排）
  - 输出流失原因报告
- **续约逾期**：
  - T-0仍未签署：进入逾期管理，每日催促
  - 逾期15天：暂停部分服务预警
  - 逾期30天：按合同条款处理

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 客户ID | string | 是 | 客户唯一标识 |
| 风险评估结果 | object | 是 | T-90天的续约风险评估 |
| 当前合同条款 | object | 是 | 现有合同的完整条款 |
| 续约策略建议 | string | 是 | 基于风险评估的策略类型 |
| 折扣授权范围 | object | 是 | CSM可自行决定的折扣上限 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| 续约提案 | document | 完整续约提案文档 |
| 谈判记录 | document | 全程谈判过程记录 |
| 续约结果 | enum | 成功/降级续约/流失 |
| 续约金额 | number | 最终续约合同金额 |
| 续约周期 | number | 新合同期限(月) |

## 最佳实践
1. **不打无准备之仗**：提案发送前必须有充分的价值数据支撑
2. **锚定效应**：先展示完整价值再谈价格，避免陷入纯价格博弈
3. **创造紧迫感**：适度使用优惠到期期限激励客户决策
4. **多方案策略**：准备A/B/C三个方案（理想/底线/折中），给客户选择空间
5. **关系与商务分离**：日常CSM关系维护和续约商务谈判由不同角色主导效果更好

## 约束条件
- 续约折扣超过CSM授权范围需升级审批
- 不得承诺未经产品团队确认的功能开发
- 续约条款变更需法务确认无风险
- T-30天前必须进入签署流程，否则需向CSM总监说明原因
