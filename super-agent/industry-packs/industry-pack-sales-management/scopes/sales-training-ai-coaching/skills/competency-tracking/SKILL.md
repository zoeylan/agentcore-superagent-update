# 能力追踪 (Competency Tracking)

## 技能概述

本技能负责建立和维护每位销售人员的长期能力成长档案，通过持续追踪评分趋势、分析业绩相关性、识别能力瓶颈，为个性化培训策略提供数据支撑。这是将"每次陪练"连接成"持续成长路径"的关键能力，也是证明培训ROI的核心数据基础。

## 适用场景

- 个人能力成长曲线的持续更新和可视化
- 陪练评分与实际赢率/业绩的相关性分析
- 团队能力热力图生成（识别组织能力缺口）
- 培训效果月度/季度报告输出
- 评分模型有效性验证和权重优化
- 新人ramp-up进度追踪

## 执行步骤

### Step 1: 数据采集与存储
- 每次陪练评分完成后自动入库（评分、场景、难度、时间戳）
- 定期同步CRM业绩数据（赢率、客单价、销售周期、丢单原因）
- 记录学员训练行为数据（训练频率、主动/被动训练比例、场景偏好）
- 记录难度级别变化历史

### Step 2: 个人能力画像构建
- 五维度能力雷达图（实时更新）
- 各场景类型的熟练度评估（基于通过率和平均分）
- 学习速度曲线（每提升10分所需训练次数的趋势）
- 能力稳定性评估（同一场景得分的标准差）
- 优劣势象限图：高分高相关性（核心优势）、低分高相关性（急需提升）、高分低相关性（可降低训练优先级）、低分低相关性（基础能力待建立）

### Step 3: 业绩相关性分析
- 每月计算各评分维度与以下业绩指标的皮尔逊相关系数：
  - 赢率（目标r>0.3）
  - 客单价
  - 销售周期长度（目标负相关）
  - 客户满意度
- 识别高相关性维度（对业绩影响最大的能力）
- 发现低相关或反直觉的维度（评分高但业绩无改善）
- 基于相关性分析结果建议调整训练优先级和评分权重

### Step 4: 团队能力分析
- 月度生成团队能力热力图（各维度×各人员的得分矩阵）
- 识别团队级别的能力缺口（某维度全员偏低 → 组织能力问题）
- Top Performer能力特征提取（优秀销售的共性能力特征）
- 新人vs老人能力对比分析
- 各Team/BU之间的能力对比

### Step 5: 预测与预警
- 新人ramp-up进度预测：按当前学习速度预计何时达标
- 能力退化预警：长期不训练或得分持续下降的人员标记
- 培训效果预警：团队整体某维度持续低迷，建议专项集训
- 评分模型漂移检测：当AI评分与业绩相关性下降时触发模型review

### Step 6: 报告输出
- **个人月度能力报告**：五维趋势+排名变化+下月建议
- **团队月度培训效果报告**：覆盖率、达标率、进步率
- **季度培训ROI报告**：培训投入vs业绩提升的量化对比
- **年度培训效能年报**：全面评估培训体系的有效性

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| trainee_id | string | 是 | 学员标识（个人报告）或"team"（团队报告） |
| time_range | object | 是 | 分析时间范围 |
| scoring_history | array | 是 | 历史评分记录 |
| performance_data | object | 否 | CRM业绩数据 |
| report_type | enum | 是 | personal/team/roi/correlation |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| competency_profile | object | 当前能力画像（五维雷达图数据） |
| growth_curve | array | 成长曲线时间序列数据 |
| correlation_analysis | object | 评分-业绩相关性系数 |
| team_heatmap | array | 团队能力热力图数据 |
| predictions | object | 达标预测和风险预警 |
| recommendations | array | 基于数据的训练策略建议 |
| report_document | string | 完整报告Markdown文档 |

## 最佳实践

1. **趋势重于绝对值**：60分但持续上升，比70分停滞不前更健康
2. **业绩是终极验证**：评分体系的有效性必须用业绩数据验证，不能自说自话
3. **数据量决定置信度**：至少10次陪练数据才能生成可靠的个人能力画像
4. **异常值处理**：单次异常高分或低分需标记并排除对趋势分析的干扰
5. **隐私分级**：个人详细数据仅本人和直属主管可见，团队报告使用汇总数据

## 约束条件

- 相关性分析需要至少3个月的数据积累才可输出结论
- 个人能力数据未经本人同意不得跨团队分享
- 评分权重调整需要管理层审批并公告后方可生效
- 团队排名数据需谨慎使用，避免引发内部恶性竞争
- ROI计算方法需要与财务部门对齐口径
