# 个性化辅导报告 (Personalized Coaching Report)

## 技能概述

本技能负责将多维度评分结果转化为学员可理解、可操作的个性化改进报告。报告不仅告诉学员"做得怎么样"，更重要的是告诉学员"如何做得更好"。每份报告都包含具体的对话原文分析、针对性的改进建议和可直接使用的参考话术，确保学员在阅读后能立即采取行动。

## 适用场景

- 每次陪练结束后的即时反馈报告
- 新人基线测评的能力诊断报告
- 每周训练推荐报告
- 月度能力成长报告
- 专项技能突破报告（针对反复不达标的维度）

## 执行步骤

### Step 1: 报告类型确定
- 即时反馈报告：陪练结束后立即生成，聚焦本次表现
- 周度推荐报告：汇总本周训练数据，推荐下周重点
- 月度成长报告：纵向对比一个月的进步轨迹
- 确定报告接收对象（学员本人/主管/培训管理者）

### Step 2: 核心内容组织
- **开场总评**：一句话概括本次表现和核心亮点（正向开头，建立信心）
- **得分概览**：总分+五维度分数雷达图数据+与上次对比变化（↑↓→）
- **TOP3亮点**：
  - 标注学员本次做得特别好的1-3个点
  - 引用对话原文作为证据
  - 说明为什么这是好的做法（连接到方法论）
- **TOP3待提升点**：
  - 每个点包含：问题描述→对话原文引用→扣分原因→改进方向→参考话术（至少2条）
  - 按优先级排序（影响分数最大且最易改善的排前面）
- **下一步行动**：推荐下次训练的场景类型和难度级别

### Step 3: 参考话术生成
- 从话术知识库中检索与待改进点匹配的标杆话术
- 根据学员的具体场景上下文调整话术表述（不能是通用模板）
- 每个待改进点提供2-3条不同风格的参考话术（供学员选择最适合自己的）
- 标注话术的使用场景和注意事项

### Step 4: 趋势分析（周度/月度报告）
- 绘制各维度得分的时间趋势线
- 标注进步最大的维度（正向激励）
- 标注停滞不前的维度（需要突破策略）
- 与团队平均水平对比（定位在团队中的相对位置）
- 预测按当前进步速度达到目标分数的预计时间

### Step 5: 个性化建议定制
- 基于学员性格类型调整表达方式：
  - 自信型学员：更多挑战性建议，激发竞争欲
  - 焦虑型学员：更多鼓励性表达，强调进步
  - 分析型学员：更多数据和逻辑说明
- 基于学员当前业务需求定制建议优先级：
  - 近期有重要客户拜访 → 优先提升对应场景能力
  - 近期丢单较多 → 优先分析丢单环节并强化训练

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| scoring_result | object | 是 | 多维度评分结果 |
| dialogue_log | array | 是 | 完整对话记录 |
| trainee_profile | object | 是 | 学员画像（历史数据、性格类型、当前阶段） |
| report_type | enum | 是 | instant/weekly/monthly/specialist |
| benchmark_scripts | array | 否 | 相关标杆话术 |
| team_averages | object | 否 | 团队平均分数据 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| report_id | string | 报告唯一标识 |
| summary | string | 一句话总评 |
| score_overview | object | 得分概览数据（含雷达图数据） |
| highlights | array | TOP3亮点（含原文引用） |
| improvements | array | TOP3待提升点（含建议和参考话术） |
| next_actions | object | 下一步训练建议 |
| trend_data | object | 趋势分析数据（周度/月度报告） |
| readable_report | string | 可直接阅读的Markdown格式报告 |

## 最佳实践

1. **三明治法则**：亮点→改进→鼓励——即使表现不佳也要先找到做得好的点
2. **话术不是背诵**：提供参考话术的同时说明核心逻辑，引导学员理解后灵活运用
3. **量化进步**：用具体数字展示进步（"本次说听比42:58，较上次50:50已有明显改善"）
4. **避免信息过载**：单份报告待改进点不超过3个，让学员能聚焦核心问题
5. **行动导向**：每份报告的结尾必须是清晰的下一步行动——做什么训练、什么时候、预期效果

## 约束条件

- 即时反馈报告必须在评分完成后2分钟内生成
- 报告中不得包含贬低学员的表述（如"表现糟糕""基础太差"）
- 参考话术必须来自已验证有效的知识库（不可AI即兴编造）
- 与主管共享的报告版本需隐去详细对话原文（保护学员隐私）
- 月度报告需要包含与上月的定量对比数据
