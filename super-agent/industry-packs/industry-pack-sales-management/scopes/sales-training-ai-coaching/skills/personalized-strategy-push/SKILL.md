# 个性化策略推送 (Personalized Strategy Push)

## 技能概述

本技能负责每周为全体销售人员生成和推送个性化的销售策略指导内容。推送基于三层数据驱动：个人能力短板数据（来自陪练评分）、近期业绩和商机数据（来自CRM）、以及本周业务安排（如即将拜访的客户）。核心目标是让每位销售在每周一早晨收到一条"此刻对我最有用的建议"，而非通用型的培训内容。

## 适用场景

- 每周一上午9:00前的全员定时推送
- 学员丢单后的即时策略推送
- 重要客户拜访前的专项策略推送
- 新产品/竞品变化后的紧急策略推送
- 连续表现不佳时的强化辅导推送

## 执行步骤

### Step 1: 数据收集（每周日晚自动触发）
- 拉取每位销售近7天的业绩数据：
  - 商机状态变化（新增/推进/丢单/赢单）
  - 活动量（外呼/会议/邮件数量）
  - 关键事件（大单丢失/客户流失/竞品出现）
- 拉取每位销售本周的商机和日程安排：
  - 本周计划拜访的客户列表
  - 各客户的行业、角色、决策阶段
  - 是否有报价/演示/谈判等关键节点
- 获取每位销售的最新能力画像（近30天各维度平均分和趋势）

### Step 2: 优先级判定（每位销售独立计算）
- **第一优先级——本周重要拜访**（客户拜访在72小时内）：
  - 推送该客户行业/角色的应对策略
  - 相似赢单案例的关键话术参考
  - 该客户已知痛点和决策关注点的提醒
- **第二优先级——近期丢单复盘**（7天内有丢单记录）：
  - 分析丢单阶段和可能原因
  - 推送该环节的强化话术和训练建议
  - 类似场景的赢单对比案例
- **第三优先级——能力短板强化**（无前两项触发条件时）：
  - 推送最薄弱维度的针对性训练建议
  - 推荐匹配的陪练场景和难度
  - 提供该维度的标杆话术学习材料

### Step 3: 推送内容生成
- 标题：简洁有力，点明核心建议（<20字）
- 正文结构（控制在500字以内）：
  - 引言：为什么这条建议现在对你重要（1-2句）
  - 核心建议：具体的行动指导（带编号，2-3条）
  - 参考话术：1-2条可直接使用的话术模板
  - 案例链接：相关赢单案例或训练场景的快速入口
- 语气：教练式（专业+鼓励，非居高临下）
- 格式：适配企业微信/钉钉的消息卡片格式

### Step 4: 推送执行与追踪
- 每周一上午8:30开始分批推送（避免集中发送导致延迟）
- 记录推送状态：已发送/已送达/已打开/已点击详情
- 追踪打开率：目标>60%，低于50%触发内容质量review
- 未打开处理：连续2周未打开 → 生成主管提醒通知

### Step 5: 效果评估
- 跟踪推送后7天内学员的行为变化：
  - 是否按建议进行了陪练
  - 相关维度的评分是否有提升
  - 对应商机是否有推进
- 月度分析推送内容类型与效果的关系
- 根据效果数据优化推送策略和内容模板

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sales_team_list | array | 是 | 推送对象列表 |
| performance_data | object | 是 | 各人近7天业绩数据 |
| calendar_data | object | 否 | 各人本周日程安排 |
| competency_profiles | object | 是 | 各人最新能力画像 |
| knowledge_base_access | object | 是 | 话术知识库检索接口 |
| push_channel | enum | 是 | wechat_work/dingtalk/email |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| push_batch_id | string | 推送批次标识 |
| push_items | array | 各人推送内容列表 |
| push_status | object | 推送状态追踪 |
| open_rate | float | 打开率统计 |
| effectiveness_report | object | 效果评估报告（T+7生成） |

## 最佳实践

1. **精准胜过全面**：一条切中要害的建议比十条泛泛道理更有价值
2. **时效性是关键**：周一早晨收到比周三下午收到效果差10倍
3. **可执行是底线**：每条建议必须能在当周内执行，不能是长期战略
4. **避免信息疲劳**：每周最多推送1条核心建议（宁精勿多）
5. **数据驱动迭代**：打开率<50%说明内容不够吸引人，需要优化标题和开头

## 约束条件

- 每周一上午9:00前必须完成全员推送（硬性SLA）
- 推送内容不得包含其他销售的个人数据或排名信息
- 推送频率严格控制在每周1次（特殊情况除外）
- 新人无历史数据时使用团队平均短板作为推送基准
- 推送内容需经系统合规检查后方可发送
