# 难度自适应调节 (Difficulty Adaptation)

## 技能概述

本技能负责基于学员的实时表现和历史能力数据，动态调整陪练场景的难度级别，确保训练始终处于"最近发展区"——既有挑战性又不至于打击信心。这是AI陪练系统区别于传统培训的核心优势之一，实现了真正的个性化学习路径。

## 适用场景

- 新人入职基线测评时的初始难度校准
- 陪练过程中的实时难度微调
- 连续多次通过/失败后的难度级别跳转
- 从一个场景类型切换到另一个场景类型时的难度映射
- 老销售周期性训练时的进阶挑战解锁

## 执行步骤

### Step 1: 能力基线获取
- 获取学员历史评分数据（近30天各维度平均分）
- 获取学员当前训练阶段（新人/进阶/高阶）
- 获取学员近期赢率和业绩数据（如有）
- 确定学员的薄弱维度和优势维度

### Step 2: 难度级别计算
- 基于能力基线确定起始难度：
  - 各维度平均分 < 40分 → Level 1（友善型客户，明确痛点，配合度高）
  - 40-60分 → Level 2（中性客户，需要引导，有基本异议）
  - 60-75分 → Level 3（谨慎型客户，防备较强，异议较多）
  - 75-85分 → Level 4（强势型客户，时间紧迫，挑战性强）
  - 85分以上 → Level 5（复杂场景，多角色联合，高压环境）
- 针对薄弱维度适当降低该维度的考察难度，但不降低整体场景难度

### Step 3: 实时微调机制
- 陪练过程中每5轮对话评估一次学员表现趋势
- 学员连续3轮表现优秀（信任度持续上升、关键信息获取成功）→ 客户角色适当增加挑战（如引入新异议或突发情况）
- 学员连续3轮表现不佳（信任度下降、对话僵持）→ 客户角色适当释放更多信息线索
- 微调幅度控制：单次陪练内难度波动不超过±0.5级

### Step 4: 跨场景难度映射
- 学员在某场景达到80分以上 → 该场景可升级到更高难度或解锁进阶变体
- 不同场景类型的难度独立追踪（可能开场白Level 3但异议处理Level 1）
- 新解锁场景的初始难度 = max(1, 相邻场景难度 - 1)
- 进阶场景（如多角色联合决策）需要至少3个基础场景达到Level 3

### Step 5: 难度调整反馈
- 每次难度调整后记录原因和结果
- 月度分析难度调整与评分提升的相关性
- 识别最优挑战区间（学员表现在哪个难度级别时进步最快）
- 输出难度调整建议给能力评估教练

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| trainee_id | string | 是 | 学员唯一标识 |
| historical_scores | object | 是 | 历史评分数据（各维度+各场景） |
| current_session_metrics | object | 否 | 当前会话实时表现数据 |
| scenario_type | enum | 是 | 当前场景类型 |
| current_difficulty | float | 否 | 当前难度级别 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| recommended_difficulty | float | 推荐难度级别（1.0-5.0） |
| adjustment_reason | string | 调整原因说明 |
| scenario_modifiers | object | 场景参数调整建议 |
| unlock_status | array | 新解锁/待解锁的场景或难度列表 |
| confidence | float | 推荐置信度（0-1） |

## 最佳实践

1. **循序渐进优于跳跃式升级**：即使学员单次表现出色，难度升级也应控制在0.5-1级/周，避免偶然高分导致的过快升级
2. **失败不等于需要降级**：区分"能力不足"的失败和"策略失误"的失败——后者可能只需要换个思路，不需要降低难度
3. **关注趋势而非单点**：基于近5次评分趋势而非最近一次评分做难度决策
4. **保留挑战空间**：即使学员稳定在某难度，也要定期（每2周）尝试更高难度的探索性训练
5. **个性化节奏**：不同学员的学习曲线不同——有人适合快速挑战，有人适合稳步提升，通过数据识别个人最优节奏

## 约束条件

- 新人首周强制从Level 1开始，无论其声称的经验水平如何
- 难度降级需要在报告中标注但不通知学员（避免打击信心）
- Level 5场景仅在至少3个基础场景达到Level 3后解锁
- 单日内同一学员的难度调整次数不超过2次
- 难度调整算法的参数变更需要经过至少2周的A/B测试验证
