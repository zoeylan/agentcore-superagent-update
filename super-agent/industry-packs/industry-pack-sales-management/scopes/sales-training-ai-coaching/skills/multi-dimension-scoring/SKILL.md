# 多维度评分 (Multi-Dimension Scoring)

## 技能概述

本技能是能力评估教练的核心分析能力，负责对每次陪练的完整对话数据进行系统化、多维度的定量评分。评分体系基于B2B销售方法论（MEDDIC、SPIN、Challenger Sale）和行业最佳实践设计，确保评价的客观性、一致性和业务相关性。每一个评分维度都有明确的量化标准和评判依据。

## 适用场景

- 每次陪练结束后的表现评分
- 新人基线测评的能力诊断
- 月度/季度能力趋势分析
- AI评分与人工评分的偏差校准
- 评分模型权重优化

## 执行步骤

### Step 1: 对话数据预处理
- 接收场景模拟陪练师输出的完整对话记录
- 解析对话元数据：总轮次、持续时长、说听比统计
- 将对话切分为功能段落：开场→需求挖掘→价值呈现→异议处理→推进/收尾
- 标记每轮对话的发起方和话题类型

### Step 2: 维度一——话术专业度评分（满分30分）
- **产品知识准确性（0-20分）**：
  - 检查学员提及的产品功能、参数、优势是否准确（错误一处扣3-5分）
  - 评估价值主张表述是否清晰且与客户痛点关联
  - 检查是否存在过度承诺或虚假陈述（严重扣分项）
- **行业术语使用（0-10分）**：
  - 评估是否使用客户行业的专业术语（而非内部产品术语）
  - 检查术语使用是否恰当、自然、不生硬
  - 是否展现了行业洞察力（如引用行业趋势、客户同行案例）

### Step 3: 维度二——需求挖掘深度评分（满分20分）
- 统计开放式提问占比（目标>60%）：开放式问题数 / 总提问数
- 评估痛点追问层次：
  - Level 1（表面需求）：仅了解客户要什么 → 5分
  - Level 2（业务影响）：追问这个需求对业务的影响 → 12分
  - Level 3（个人利益）：触及决策人个人的目标/压力/顾虑 → 18-20分
- 评估SPIN提问的运用：Situation → Problem → Implication → Need-payoff的递进性

### Step 4: 维度三——异议处理能力评分（满分20分）
- 识别对话中出现的异议点及其类型（价格/竞品/时机/需求质疑）
- 评估每个异议的处理完整度：
  - 认同（Acknowledge）：是否先认可客户的感受/观点 → 5分
  - 转化（Reframe）：是否成功将异议转化为讨论机会 → 8分
  - 价值重塑（Reinforce Value）：是否回到价值主张并提供证据 → 7分
- 特殊扣分项：直接否定客户观点(-5分)、情绪化回应(-3分)、回避不处理(-8分)

### Step 5: 维度四——节奏把控评分（满分15分）
- 说听比计算：学员发言字数 / 总对话字数，目标40:60（±10%为优秀10-15分，±20%为合格6-9分，超出为不合格0-5分）
- 关键节点停顿评估：在客户表达完重要信息后是否有适当停顿和回应
- 追问与总结节奏：是否在获得关键信息后及时总结确认
- 话题推进节奏：是否过快推进（客户未准备好）或过慢停滞（浪费客户时间）

### Step 6: 维度五——情绪管理评分（满分15分）
- 压力应对稳定性：面对客户拒绝/质疑/冷淡时的语言态度是否保持专业和积极
- 语气适配度：是否根据客户情绪调整自己的沟通语气（而非一成不变或过度配合）
- 挫折恢复力：在对话不利转折后能否快速调整策略而非陷入防御

### Step 7: 综合评分输出
- 计算各维度加权总分
- 与学员历史平均分对比，标注进步/退步趋势
- 确定TOP3待提升点（选择扣分最多且可改进的维度）
- 生成评分卡JSON数据

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| session_id | string | 是 | 陪练会话标识 |
| dialogue_log | array | 是 | 完整对话记录 |
| scenario_config | object | 是 | 场景配置（类型、难度、角色设定） |
| metrics_raw | object | 是 | 原始指标数据 |
| benchmark_scripts | array | 否 | 该场景的标杆话术参照 |
| historical_scores | object | 否 | 学员历史评分 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| total_score | int | 总分（0-100） |
| dimension_scores | object | 各维度得分明细 |
| score_evidences | array | 每个评分点的对话原文依据 |
| top3_improvements | array | TOP3待提升点（含建议和参考话术） |
| highlights | array | 亮点表现标注 |
| trend_comparison | object | 与历史得分对比 |
| difficulty_recommendation | object | 下次难度建议 |

## 最佳实践

1. **每个扣分必有依据**：引用对话原文说明为什么扣分，避免"感觉不好"的主观评价
2. **评分尺度校准**：每月参照人工评分结果校准AI评分，确保偏差<±10分
3. **场景难度加权**：Level 4-5的场景中即使得分较低也应考虑难度因素
4. **正向激励优先**：报告中亮点要先于待改进点展示，建立学员信心
5. **可操作的建议**：每个待改进点必须附带至少2条可直接使用的参考话术

## 约束条件

- 评分结果必须在陪练结束后5分钟内生成
- 单次评分报告中待改进点不超过3个（聚焦重点，避免信息过载）
- 评分标准的调整需要至少2周的数据验证后方可生效
- 不同难度级别使用相同评分标准（不因难度高而放宽标准）
- AI评分与人工评分月度偏差需控制在±10分以内
