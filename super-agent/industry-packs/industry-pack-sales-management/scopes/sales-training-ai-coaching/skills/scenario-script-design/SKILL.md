# 场景剧本设计 (Scenario Script Design)

## 技能概述

本技能负责基于真实丢单/赢单案例和最新市场动态，设计和构建新的AI陪练场景剧本。每个剧本是一个完整的训练单元，包含客户背景设定、对话预设走向、难度参数、关键评估点和期望产出。高质量的剧本是陪练系统持续有效的保障——必须紧贴实战、持续更新、难度可控。

## 适用场景

- 月度新场景开发（目标≥2个/月）
- 基于近期丢单案例的紧急场景构建
- 竞品重大变化后的应对训练场景
- 新产品/新功能上线后的价值呈现场景
- 进阶复合场景设计（多角色、多轮次、跨阶段）
- 行业特定场景定制（金融/医疗/制造等垂直行业）

## 执行步骤

### Step 1: 素材收集与分析
- 从CRM获取近期典型案例（优先选择有完整录音的案例）：
  - 丢单案例：分析哪个环节出了问题、客户的核心顾虑是什么
  - 赢单案例：提取成功关键因素、识别可复制的策略
  - 竞品胜出/落败案例：了解客户选择/放弃的决策逻辑
- 收集市场动态信息：
  - 竞品近期动作（新功能/降价/市场活动）
  - 行业趋势变化（政策/技术/需求变化）
  - 客户采购模式变化（决策流程/预算周期/关注重点）

### Step 2: 剧本框架设计
- **场景定位**：明确本剧本训练的核心技能点（1-2个）
- **客户画像设计**：
  - 行业与公司背景（规模、发展阶段、面临的业务挑战）
  - 决策角色（职位、性格特征、关注重点、决策权限）
  - 当前状态（是否使用竞品、预算情况、决策时间线）
  - 隐藏信息（需要销售通过提问才能发现的关键信息）
- **对话走向设计**：
  - 开场状态（客户的初始态度和防备程度）
  - 转折点设计（2-3个关键转折，如客户突然提到竞品/预算被砍）
  - 理想路径（优秀销售应该如何推进）
  - 常见陷阱（学员容易犯的错误和对应的客户反应）
  - 结束条件（成功标准/失败标准/时间限制）

### Step 3: 难度参数配置
- 客户配合度（1-5级）：决定客户给予信息的难易程度
- 异议密度（1-5级）：决定对话中出现异议的频率和强度
- 时间压力（1-5级）：决定客户表现出的紧迫程度
- 信息隐藏深度（1-5级）：决定关键信息被埋藏的层次
- 情绪挑战度（1-5级）：决定客户的态度和情绪变化幅度
- 总难度 = 各参数加权平均，映射到Level 1-5

### Step 4: 评估标准设定
- 定义本场景的评分权重分布（可能某些维度占比更高）
- 设定关键评估点（Must-have动作和Nice-to-have动作）
- 定义合格线和优秀线（基于场景特性可能与通用标准不同）
- 设计典型失败模式和对应的改进建议

### Step 5: 剧本测试与上线
- 内部测试：由至少2位资深销售进行完整测试
- 收集反馈：场景真实度、难度适当性、训练价值
- 调整优化：根据反馈修改参数和设定
- 正式上线：标注适用人群和推荐训练时机
- 效果追踪：上线后监控通过率和学员反馈评分

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source_cases | array | 是 | 参考案例列表（赢单/丢单） |
| target_skills | array | 是 | 目标训练技能点 |
| target_difficulty | int | 是 | 目标难度级别 |
| industry_focus | string | 否 | 特定行业定向 |
| competitive_context | object | 否 | 竞品相关背景信息 |
| constraints | object | 否 | 特殊约束（时长/角色限制等） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| script_id | string | 剧本唯一标识 |
| script_metadata | object | 元信息（名称/分类/难度/适用人群） |
| customer_persona | object | 完整客户画像 |
| dialogue_blueprint | object | 对话走向蓝图 |
| difficulty_params | object | 难度参数配置 |
| assessment_criteria | object | 评估标准和权重 |
| test_results | object | 测试结果和反馈 |

## 最佳实践

1. **来源真实案例**：每个剧本都应有真实案例作为基础，凭空想象的场景缺乏训练价值
2. **一个场景一个重点**：不要试图在一个剧本中训练所有技能，聚焦才有深度
3. **客户画像要立体**：不仅有职业身份，还要有性格、压力、个人目标等维度
4. **转折点是灵魂**：好的剧本一定有意想不到但合理的转折，模拟实战的不确定性
5. **定期更新淘汰**：超过6个月未更新的场景可能已与市场脱节，需要review

## 约束条件

- 每月至少新增2个合格场景剧本
- 剧本中的客户信息必须完全脱敏（不能使用真实客户名称和数据）
- 竞品相关设定必须基于公开信息
- 新剧本上线前必须经过至少2人的内部测试
- 剧本难度分布应覆盖Level 1-5各级别，不能偏向某一难度区间
