# 知识库运营管理 (Knowledge Base Curation)

## 技能概述

本技能负责销售话术向量知识库的全生命周期管理，包括内容采集、加工入库、质量审核、定期更新和过时淘汰。知识库是AI陪练系统的"弹药库"——评分基准来自这里，训练素材来自这里，个性化推送的内容也来自这里。知识库的质量和时效性直接决定了整个培训系统的有效性。

## 适用场景

- 月度赢单录音的话术提取和入库
- 竞品动态变化后的话术紧急更新
- 季度全量话术review和淘汰清理
- 新产品/新功能发布后的话术补充
- 话术使用效果数据的分析和优化
- 合规审查和风险话术剔除

## 执行步骤

### Step 1: 内容采集
- **赢单录音采集**：每月从CRM系统拉取近30天的赢单录音（优先大单和竞品胜出案例）
- **AI转录**：将语音转为文字，标注说话人、时间戳、情绪标记
- **关键片段提取**：使用NLP模型自动识别对话中的高价值片段：
  - 异议成功处理的转折点
  - 客户从犹豫变为积极的关键时刻
  - 独特的开场白或破冰方式
  - 精准的需求挖掘提问
  - 成功促单的关键话术
- **人工验证**：提取的片段需由至少1位Top Sales确认有效性

### Step 2: 内容加工
- 将原始对话片段转化为标准化话术卡片：
  - `id`: 唯一标识
  - `category`: 分类（开场白/需求挖掘/价值呈现/异议处理/促单/竞品应对）
  - `scenario_tags`: 适用场景标签（行业+客户角色+决策阶段）
  - `difficulty`: 适用难度级别
  - `script_body`: 话术正文
  - `variants`: 2-3个表述变体（不同风格）
  - `usage_context`: 使用时机和前置条件
  - `cautions`: 注意事项和禁忌
  - `source`: 来源信息（来自谁、哪个客户、何时）
  - `effectiveness_data`: 效果数据（赢率、客户反馈）
- 生成向量嵌入并存入向量知识库

### Step 3: 合规审查
- 逐条检查入库话术是否存在以下风险：
  - 过度承诺（承诺产品不具备的功能或不可实现的时间线）
  - 贬低竞品（攻击竞品而非突出自身价值）
  - 隐私泄露（引用其他客户的敏感信息作为案例）
  - 虚假宣传（夸大产品效果或市场地位）
  - 价格违规（透露不应公开的价格信息）
- 标记风险话术，修改或剔除
- 高风险案例反向生成"负面案例"——告诉销售什么不能说

### Step 4: 更新与维护
- **竞品监控**：每月check主要竞品的产品更新、定价调整、市场策略变化
- **紧急更新**：竞品重大变化后48小时内更新对应应对话术
- **产品同步**：本公司产品功能更新后同步更新相关话术（新功能话术+旧功能调整）
- **效果追踪**：每条话术附带使用数据——被引用次数、使用后赢单率、学员反馈评分

### Step 5: 季度清理
- 全量review知识库中所有话术（每季度末执行）
- 淘汰标准：使用率<5% + 无正面效果反馈 + 超过6个月未更新
- 待淘汰话术先标记为"deprecated"，观察1个月后确认无异议则正式下架
- 生成季度知识库健康度报告：总量、新增、淘汰、活跃率、效果分布

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operation_type | enum | 是 | ingest/update/review/retire/search |
| source_data | object | 否 | 原始数据（录音转录/竞品情报/产品更新） |
| script_id | string | 否 | 待操作的话术ID（更新/淘汰时需要） |
| search_query | object | 否 | 检索条件（场景标签/分类/难度） |
| review_scope | enum | 否 | 审查范围（single/category/full） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| operation_result | object | 操作结果（成功/失败/待审核） |
| new_scripts | array | 新入库的话术卡片列表 |
| updated_scripts | array | 更新的话术列表 |
| retired_scripts | array | 淘汰的话术列表 |
| compliance_flags | array | 合规风险标记 |
| health_report | object | 知识库健康度数据 |

## 最佳实践

1. **质量优于数量**：宁可少入库几条高质量话术，也不要灌入大量未验证的内容
2. **场景标签要精确**：模糊的标签会导致错误匹配——"大客户异议处理"比"异议处理"更有用
3. **保留来源可追溯**：每条话术都要能追溯到原始来源，方便质疑时验证
4. **变体覆盖风格差异**：同一个意思用不同风格表达（直接型/温和型/引导型），适配不同性格的销售
5. **负面案例同样重要**：记录什么话不能说，与正面话术互补形成完整的指导体系

## 约束条件

- 知识库中不得保留包含真实客户隐私信息的内容（需脱敏处理）
- 竞品话术必须基于公开可验证的信息，不得使用非公开渠道获取的情报
- 话术入库必须经过合规审查，无审查记录的话术不得进入正式库
- 每月新增话术目标≥10条，但不得为凑数而降低质量标准
- 淘汰话术需要保留归档（不物理删除），以备历史审计需要
