# 角色扮演对话 (Role-Play Dialogue)

## 技能概述

本技能是场景模拟陪练师的核心交互能力，负责在陪练过程中持续维持高质量的客户角色扮演对话。不同于简单的问答，角色扮演对话要求AI在每一轮对话中都保持角色一致性、情境合理性和训练价值——既要像一个真实的客户在沟通，又要确保对话具有训练效果。

## 适用场景

- 所有5大核心场景类型中的多轮对话交互
- 客户角色的情绪变化和态度转变表演
- 突发情境插入（如客户接到紧急电话、竞品销售刚拜访过）
- 学员偏离主题时的自然引导回归
- 对话陷入僵局时的角色内破冰

## 执行步骤

### Step 1: 角色初始化
- 加载客户角色设定参数（行业、职位、性格、痛点、预算、决策阶段）
- 设定初始情绪状态和信任度（0-100分制）
- 加载该角色类型的典型语言风格库
- 确定角色的隐藏议程和底线条件

### Step 2: 对话轮次管理
- 每轮接收学员输入后，进行意图分析
- 根据学员表现动态更新信任度和购买意向
- 基于角色设定生成应答（考虑：角色性格+当前情绪+对话上下文+信任度变化）
- 控制应答长度和节奏（模拟真实对话中客户的表达习惯）

### Step 3: 动态情境演化
- 监测对话进展是否符合场景预设走向
- 在关键节点触发预设事件（如第5轮后客户提到竞品）
- 根据学员表现质量决定客户的配合度变化
- 适时制造挑战（如客户突然质疑、要求降价、提出新需求）

### Step 4: 异常处理
- 学员偏离主题：角色内自然引导（如"我们先回到刚才那个话题..."）
- 学员长时间沉默：角色内施加时间压力（如"我还有5分钟就要开会了"）
- 学员使用不当话术：角色表现出相应的负面反应（而非直接纠正）
- 对话陷入死循环：引入新的情境变量打破僵局

### Step 5: 对话终结
- 判断自然结束点（达成共识/明确拒绝/约定下一步/时间到达）
- 执行角色内的自然收尾（如"好的，回头让我们团队评估一下"）
- 记录最终对话状态（信任度、购买意向、下一步约定）
- 输出对话结束标识和元数据

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| role_config | object | 是 | 客户角色完整配置 |
| trainee_message | string | 是 | 学员当前轮次输入 |
| dialogue_context | array | 是 | 历史对话记录 |
| current_trust_level | float | 是 | 当前信任度 |
| scenario_events | array | 否 | 待触发的预设事件列表 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| role_response | string | 客户角色的应答内容 |
| trust_delta | float | 信任度变化值 |
| intent_signals | array | 本轮释放的购买信号或风险信号 |
| triggered_events | array | 本轮触发的场景事件 |
| dialogue_state | enum | 对话状态（ongoing/approaching_end/terminated） |

## 最佳实践

1. **一致性是生命线**：强势CEO不会突然变成温和的倾听者，犹豫型客户不会突然爽快下单——角色转变必须有合理的触发原因
2. **信息给予有层次**：客户的痛点信息应分层释放——先给表面信息，学员追问深入后才给核心信息
3. **不完美的表达更真实**：真实客户会有停顿、重复、修正、跑题——适度的不完美让对话更真实
4. **情绪反应要自然**：如果学员说出不恰当的话，客户的负面反应应该是渐进的（先不悦→然后冷淡→最终拒绝），而非一步到位
5. **留白也是对话**：适当的短回复和沉默是真实客户的常见表现，不需要每次都给出详细回复

## 约束条件

- 角色扮演中不得使用真实人名或公司名（使用化名）
- 不得生成包含歧视、侮辱或不当内容的客户对话
- 客户角色不得主动引导学员做出不合规行为
- 单轮客户应答控制在200字以内（模拟真实对话节奏）
- 在任何情况下不得跳出角色给学员直接提示或评价
