# 数据质量治理

## 技能概述

本技能用于确保销售数据分析所依赖的底层数据达到可信赖的质量标准。数据质量是所有分析工作有效性的前提——"Garbage In, Garbage Out"在销售分析领域尤为显著。本技能覆盖数据完整性校验、一致性检查、时效性监控、异常值标记和数据质量报告生成等核心能力。

## 适用场景

- 每日自动数据质量巡检（各Scope数据同步后）
- 周报/月报生成前的数据质量预检
- 发现数据异常时的溯源和修复协调
- 新数据源接入时的质量基线评估
- 季度数据质量review和改进计划制定

## 执行步骤

### 步骤1：数据完整性校验
1. 检查关键字段缺失率：
   - 商机金额：缺失率需<2%
   - 商机阶段：缺失率需<1%
   - 预计成交日期：缺失率需<5%
   - 商机负责人：缺失率需<1%
   - 客户名称：缺失率需<1%
2. 统计各字段缺失率并与阈值（整体<5%）对比
3. 对缺失严重的字段定位到具体记录和负责人

### 步骤2：数据一致性检查
1. 跨系统字段一致性校验（CRM vs 财务系统 vs 合同系统）
2. 逻辑一致性检查：
   - 商机阶段与关联活动是否匹配（如报价阶段应有报价记录）
   - 金额变更是否有合理的关联记录
   - 时间序列合理性（创建日期≤阶段变更日期≤预计关单日期）
3. 标记不一致记录并生成修复建议

### 步骤3：数据时效性监控
1. 检查数据同步延迟：各Scope数据最后同步时间距当前<1小时
2. 识别"僵尸数据"：商机最后更新超过30天但仍处于活跃阶段
3. 监控数据源系统的可用性和同步状态
4. 对延迟超过SLA的数据源生成告警

### 步骤4：异常值检测与标记
1. 金额异常：商机金额超过该产品线历史P95值
2. 阶段异常：阶段停留时间超过平均值3倍标准差
3. 行为异常：短时间内大批量数据变更（可能是误操作或批量导入）
4. 对异常值标记但不自动修改，生成人工确认清单

### 步骤5：质量报告输出
1. 生成数据质量评分卡（完整性/一致性/时效性/准确性四个维度）
2. 输出数据质量趋势（与上期对比，是在改善还是恶化）
3. 对影响分析结论的质量问题给出风险等级
4. 生成数据质量改进建议（具体到哪个团队/系统需要改善什么）

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 数据源连接信息 | Config | 是 | 各系统数据访问配置 |
| 质量规则配置 | Rules | 是 | 阈值、检查逻辑定义 |
| 历史质量基线 | Dataset | 否 | 用于趋势对比 |
| 校验范围 | Enum | 否 | 全量/增量/指定数据集 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 质量评分卡 | Scorecard | 四维度综合评分（0-100） |
| 问题明细清单 | List | 按严重性排序的数据问题列表 |
| 修复建议 | List | 具体到记录级别的修复建议 |
| 质量趋势报告 | Chart | 数据质量随时间的变化趋势 |
| 风险评级 | Enum | 对分析结论影响的风险等级 |

## 最佳实践

- **预防优于修复**：推动数据源头（CRM录入端）的校验规则强化，比事后清洗更有效
- **分级处理**：关键字段（金额/阶段）问题优先修复，次要字段（备注/标签）可批量处理
- **透明标注**：当数据质量影响分析结论时，必须在报告中明确标注置信度降级
- **定期校准**：质量规则的阈值需要根据业务变化每季度review一次
- **激励机制建议**：可建议管理层将数据录入质量纳入销售人员KPI考核

## 约束条件

- 不直接修改源系统数据——只标记问题并通知数据负责人修复
- 数据质量不达标（完整率<90%）时必须阻断报告发布并告警
- 涉及个人绩效数据时注意数据安全和隐私保护
- 暂用历史数据填充时必须标注为"估算值"
