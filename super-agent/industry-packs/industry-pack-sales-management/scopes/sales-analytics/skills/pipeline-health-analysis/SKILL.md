# 销售漏斗健康度分析

## 技能概述

本技能用于对B2B销售漏斗各阶段进行系统性的健康度评估，通过分析各阶段转化率、停留时长、漏斗形态等关键指标，诊断销售管线的整体状态并识别薄弱环节。该技能是Pipeline Management的核心分析能力，直接支撑管理层的资源调配和策略调整决策。

## 适用场景

- 每周Pipeline Review前的漏斗状态评估
- 季度规划时的管线健康度基线建立
- 当Pipeline Coverage低于3倍时的问题诊断
- 新市场/新产品线的漏斗建设效果评估
- 销售团队扩张或缩减后的管线容量评估

## 执行步骤

### 步骤1：数据采集与验证
1. 从CRM系统拉取当前所有活跃商机数据（阶段、金额、创建日期、最后更新日期、负责人）
2. 获取历史同期数据作为对比基线（同比去年同期、环比上月/上周）
3. 校验数据完整性：关键字段（金额/阶段/预计日期/负责人）缺失率需<5%
4. 标记数据质量问题（如金额为0、阶段与日期矛盾等）

### 步骤2：分阶段转化率计算
1. 按标准销售阶段划分漏斗层级：线索→MQL→SQL→商机→报价→谈判→签约
2. 计算各阶段间的转化率（本期 vs 历史平均 vs 行业基准）
3. 计算各阶段的平均停留时长（识别瓶颈阶段）
4. 分析转化率的时间趋势（是在改善还是恶化）

### 步骤3：Pipeline Coverage计算
1. 汇总当前所有加权Pipeline金额（商机金额 × 阶段赢率权重）
2. 获取当期剩余Quota目标
3. 计算Coverage比值 = 加权Pipeline / 剩余Quota
4. 判断健康度：≥4倍（优秀）、3-4倍（健康）、2-3倍（需关注）、<2倍（危险）

### 步骤4：漏斗形态分析
1. 绘制漏斗形态图（各阶段商机数量和金额分布）
2. 识别异常形态：头重脚轻（大量早期商机但后期不足）、中间膨胀（阶段推进受阻）、底部堆积（签约阶段停滞）
3. 对比理想漏斗形态，标注偏差点

### 步骤5：输出诊断报告
1. 生成漏斗健康度评分卡（综合评分0-100）
2. 列出TOP 3风险点和改善建议
3. 按维度（区域/产品线/团队）拆解分析结果
4. 生成趋势预判（如照当前趋势，下期Coverage预计值）

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 当前活跃商机数据 | Dataset | 是 | 包含阶段、金额、日期、负责人等字段 |
| 季度/月度Quota目标 | Number | 是 | 用于计算Coverage |
| 历史同期数据 | Dataset | 是 | 用于同比/环比分析 |
| 行业基准转化率 | Config | 否 | 如无则使用内部历史基准 |
| 分析维度 | Enum | 否 | 全局/按区域/按产品线/按团队 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 漏斗健康度评分 | Score (0-100) | 综合考虑Coverage、转化率、形态的健康度分数 |
| Pipeline Coverage | Number | 加权管线覆盖率 |
| 各阶段转化率 | Table | 含当前值、历史值、基准值、偏差率 |
| 风险识别列表 | List | 按严重性排序的风险点和建议 |
| 漏斗形态诊断 | Text | 当前形态描述和偏差分析 |

## 最佳实践

- **阶段赢率权重设置**：初步接触5%、需求确认15%、方案演示30%、报价50%、谈判70%、口头承诺90%——这些权重需要根据实际历史数据每季度校准一次
- **行业基准参考**：B2B SaaS行业SQL到签约转化率约20-30%，平均销售周期90-180天
- **时间窗口选择**：周度分析看7天滑动窗口，月度分析看30天，季度分析需包含完整的季度pipeline演变历程
- **注意季节性因素**：Q4通常是签约高峰期，Q1是pipeline补充期，需要对比同期而非简单环比

## 约束条件

- 数据质量不达标（缺失率>10%）时需在报告中明确标注数据置信度降级
- 不对单个销售人员做公开排名式的漏斗健康度评判（这属于业绩洞察分析师的职责）
- 预测相关的深度分析交由收入预测技能处理，本技能聚焦于漏斗状态诊断
