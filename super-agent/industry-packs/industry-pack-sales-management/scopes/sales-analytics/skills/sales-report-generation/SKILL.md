# 销售报告生成

## 技能概述

本技能用于自动化生成高质量的销售周报和月报，将复杂的销售数据转化为结构清晰、洞察丰富、可直接用于管理层决策的报告文档。报告不是简单的数据罗列，而是包含数据解读、趋势分析、问题诊断和行动建议的完整决策支持材料。

## 适用场景

- 每周一9:00前自动生成销售周报
- 每月5号前生成月度业绩月报
- 管理层临时要求的专项分析报告
- 季度业务回顾（QBR）材料准备
- 董事会/投资人要求的业务数据更新

## 执行步骤

### 步骤1：数据汇总与计算
1. 从数据源获取报告周期内的完整数据：
   - 新增线索数、MQL/SQL转化数
   - 新增商机数和金额
   - 签约数和签约金额
   - 赢单/丢单数据
   - 活动量数据（外呼/会议/演示）
2. 计算关键指标的本期值、目标值、完成率
3. 计算同比（YoY）和环比（MoM/WoW）变化率
4. 按维度（区域/产品线/团队/个人）分别汇总

### 步骤2：指标健康度判定
1. 将各指标与目标对比，进行三色灯判定：
   - 绿灯：达成≥90%目标
   - 黄灯：达成70-90%目标（偏离10-20%）
   - 红灯：达成<70%目标（偏离>20%）
2. 对黄灯指标附加简要原因分析
3. 对红灯指标触发深度分析要求

### 步骤3：关键事件标注
1. 识别报告周期内的关键业务事件：
   - 大单签约（金额>季度Quota的10%）
   - 大单丢失
   - 团队人员变动
   - 新产品/新市场突破
   - 重要客户状态变化
2. 评估事件对业绩指标的影响
3. 在报告中以醒目方式标注

### 步骤4：洞察提炼与建议生成
1. 从数据中提炼3-5个关键洞察（非显而易见的发现）
2. 每个洞察需回答：
   - So What：这意味着什么？对业务有什么影响？
   - Now What：建议什么行动？谁负责？什么时间？
3. 建议需具体、可行动、有时间节点

### 步骤5：报告格式化输出
1. 周报格式（1页精华）：
   - Executive Summary（3句话概括本周核心）
   - 关键指标仪表盘（数字+箭头+三色灯）
   - 本周亮点和关注点（各2-3条）
   - 下周重点关注事项
2. 月报格式（完整版）：
   - Executive Summary
   - 月度业绩总览（含目标达成率）
   - 多维度拆解分析
   - 业绩排名
   - 趋势分析（含6个月趋势图描述）
   - 洞察与建议
   - 下月展望

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 报告类型 | Enum | 是 | 周报/月报/专项报告 |
| 报告周期 | DateRange | 是 | 起止日期 |
| 销售活动数据 | Dataset | 是 | 各类销售活动记录 |
| 签约/丢单数据 | Dataset | 是 | 本期成交和丢失记录 |
| 目标数据 | Dataset | 是 | 各维度的目标值 |
| 上期报告 | Document | 否 | 用于趋势对比 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 报告文档 | Document | 按标准格式输出的报告 |
| 指标摘要 | Table | 关键指标汇总表 |
| 预警事项 | List | 需要管理层关注的红灯项 |
| 行动建议 | List | 具体的改善建议 |

## 最佳实践

- **准时性优先**：报告延迟发布的价值急剧下降，宁可先发布90%准确的报告再补充修正，不要因追求完美而延迟
- **读者意识**：管理层通常只看Executive Summary和红灯项，详细数据供需要时深入查阅
- **趋势比快照更重要**：单期数据容易产生误判，始终将当期放入趋势中解读
- **避免信息过载**：周报控制在1页以内（含图表），月报控制在5页以内
- **可视化优先**：用图表和颜色传递信息比大段文字更有效

## 约束条件

- 周报必须在每周一9:00前完成发布，月报必须在每月5号前完成
- 报告中引用的数字必须可溯源，不得使用未经验证的估算值（除非明确标注）
- 涉及个人业绩排名时，报告分发范围限制为管理层（非全员可见）
- 报告模板和格式变更需经管理层审批
