# 审批决策支持技能

## 概述
审批决策支持技能为折扣审批人提供全面的决策辅助信息包，帮助管理者在有限时间内做出知情决策。该技能将分散的客户数据、历史交易数据、市场竞争数据整合为结构化的决策支持报告，降低审批决策的信息不对称。

## 适用场景
- 折扣审批请求到达审批人时，自动生成决策辅助包
- 审批人需要更多信息辅助判断时的补充分析
- 大额折扣（>20%）的深度分析支持
- 战略客户折扣的长期价值评估
- 月度折扣回顾时的数据支撑

## 执行步骤

### 步骤1：客户价值分析
- 查询客户历史购买记录和总贡献收入
- 计算客户LTV（Life Time Value）预测值
- 评估客户战略重要性（标杆客户/行业头部/高增长潜力）
- 查看客户当前合同状态和续约预期
- 评估客户钱包份额（我方收入占客户IT总预算比例）

### 步骤2：交易上下文分析
- 本次交易的利润率计算（折扣后）
- 本次交易的年化价值（多年合同折算）
- 交易紧迫性评估（客户决策时间线、竞争对手投标时间）
- 销售人员对赢率的自评和历史赢率对比
- 交易失败的机会成本评估

### 步骤3：折扣基准对比
- 同类客户（行业+规模）的历史平均折扣率
- 同产品线的历史平均折扣率
- 该销售人员的历史平均折扣率
- 本季度团队折扣预算使用进度
- 单客户累计年度折扣比率（上限30%）

### 步骤4：竞争态势评估
- 主要竞品的公开定价区间
- 该客户已知的竞品接触情况
- 竞品在同行业的近期赢单案例和报价水平
- 我方产品差异化价值点（可支撑溢价的因素）

### 步骤5：决策建议生成
- 综合以上分析给出建议结论（建议通过/建议谈判/建议驳回）
- 如建议通过，说明核心支撑理由
- 如建议谈判，提供建议折扣率区间和谈判策略
- 如建议驳回，说明风险点和替代方案
- 标注置信度水平（数据充分性评估）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| quotationId | String | 是 | 报价单编号 |
| customerId | String | 是 | 客户ID |
| discountRate | Number | 是 | 申请折扣率 |
| dealContext | Object | 是 | 交易上下文（金额、产品线、紧迫性） |
| salesRepId | String | 是 | 销售人员ID |
| competitorInfo | Object | 否 | 已知竞品信息 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| decisionPackage | Object | 完整决策辅助包 |
| customerValueAnalysis | Object | 客户价值分析 |
| benchmarkComparison | Object | 折扣基准对比 |
| competitiveLandscape | Object | 竞争态势 |
| recommendation | String | 决策建议（approve/negotiate/reject） |
| confidenceLevel | String | 建议置信度（high/medium/low） |
| keyRisks | Array | 关键风险提示 |

## 最佳实践
1. **一页纸原则**：决策辅助包的核心结论控制在一页纸，详细数据作为附件
2. **可视化呈现**：折扣基准对比使用图表展示，直观反映当前折扣在分布中的位置
3. **场景对比**：展示"批准vs驳回"两种场景下的收益/损失对比
4. **历史学习**：标注历史上类似折扣申请的审批结果和后续交易结果
5. **时间敏感性**：标注客户决策deadline，帮助审批人理解时效压力

## 约束条件
- 决策辅助包必须在审批请求发起后30分钟内生成
- 竞品价格信息标注数据时效性（超过3个月需标注"待更新"）
- 客户LTV预测模型每季度校准一次
- 建议仅供参考，不替代审批人的最终判断
- 涉及客户机密信息（如竞品报价具体数字）需标注信息来源可靠性
