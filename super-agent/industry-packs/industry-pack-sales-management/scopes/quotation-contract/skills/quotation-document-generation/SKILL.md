# 报价单文档生成技能

## 概述
报价单文档生成技能负责将产品配置和定价计算结果转化为格式规范、内容完整、可直接发送客户的专业报价单文档。报价单是企业对外的正式商务文件，其专业程度直接影响客户对企业的第一印象和信任度。

## 适用场景
- 标准产品报价单的快速生成（<2小时）
- 定制解决方案报价单的生成（附方案说明书）
- 报价修改后的版本更新和重新生成
- 多方案对比报价单的并列展示
- 报价有效期到期前的更新提醒

## 执行步骤

### 步骤1：报价单元数据组装
- 生成唯一报价编号（格式：QT-YYYYMMDD-XXX）
- 填充客户信息（公司名称、联系人、地址、统一社会信用代码）
- 填充我方信息（公司名称、销售负责人、联系方式）
- 设置报价有效期（默认30天，可自定义）
- 标注报价版本号（V1.0，修改后递增）

### 步骤2：产品明细表编制
- 按模块/分类组织产品明细
- 每行包含：序号、产品名称、规格说明、数量、单位、单价、小计
- 标注一次性费用和年度经常性费用的区分
- 如有折扣，单独列出折扣行或折扣汇总
- 计算分类小计和总计

### 步骤3：商务条款填充
- 付款条件（如：签约预付30%，上线验收后付60%，质保到期付10%）
- 交付条件（交付方式、预计上线时间、实施范围说明）
- 售后服务条款（服务期限、服务内容、响应时间SLA）
- 知识产权声明
- 争议解决方式

### 步骤4：文档格式化与输出
- 应用企业标准报价单模板（含公司Logo、页眉页脚）
- 检查排版规范（对齐、字体、表格边框）
- 生成PDF版本（不可编辑，用于正式发送）
- 生成Word版本（可编辑，用于内部流转）
- 附加方案说明书（定制报价场景）

### 步骤5：发送与追踪
- 记录报价单发送时间和接收人
- 设置有效期到期提醒（到期前7天和到期当天）
- 标记报价单状态（已发送/客户已查看/已接受/已拒绝/已过期）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerInfo | Object | 是 | 客户完整信息 |
| productDetails | Array | 是 | 产品明细（含价格计算结果） |
| commercialTerms | Object | 是 | 商务条款（付款、交付、售后） |
| validityPeriod | Number | 否 | 有效期天数（默认30） |
| solutionBrief | String | 否 | 方案说明（定制报价场景） |
| previousVersion | String | 否 | 上一版本编号（修改场景） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| quotationId | String | 报价单唯一编号 |
| quotationPDF | File | PDF格式报价单 |
| quotationWord | File | Word格式报价单 |
| solutionDoc | File | 方案说明书（如有） |
| validUntil | Date | 有效期截止日期 |
| version | String | 版本号 |

## 最佳实践
1. **客户视角排版**：将客户最关心的内容放在前面（解决方案概述→ROI预估→产品明细→价格→条款）
2. **差异化呈现**：如提供多方案，用对比表清晰展示各方案差异
3. **价值导向**：不仅列产品清单，要关联客户业务价值（"此模块帮助您实现XX目标"）
4. **留白策略**：可选项用虚线框标注，给客户升级选择的暗示
5. **一致性检查**：确保报价单中的所有数字计算正确、术语使用一致

## 约束条件
- 报价单必须包含有效期声明，过期报价自动标记作废
- 报价单中不得出现内部成本信息或毛利率数据
- 折扣信息展示方式遵循公司统一规范（百分比或金额，不混用）
- 每次报价修改必须产生新版本号，旧版本保留不删除
- 报价单PDF不可编辑，防止客户篡改条件后签约
