# 历史争议分析技能

## 概述
历史争议分析技能负责从企业历史合同执行数据中提炼风险洞察，识别高频争议条款模式、行业客户的条款偏好、以及常见的"执行时有坑"的条款陷阱。该技能为当前合同的风险评估提供数据支撑，使风险判断从经验驱动升级为数据驱动。

## 适用场景
- 新合同风险评估时查询类似合同的争议历史
- 针对特定条款类型的争议概率评估
- 特定行业/客户类型的合同风险模式分析
- 合同模板优化时的争议数据参考
- 年度合同管理复盘和经验教训总结
- 新销售人员的合同风险培训材料准备

## 执行步骤

### 步骤1：争议案例库检索
- 根据当前合同特征（行业、客户规模、合同类型、金额级别）检索相关历史争议
- 按条款类型分类检索：付款争议、交付争议、验收争议、IP争议等
- 按严重程度分类：仲裁/诉讼级别、商务协商级别、内部风险记录级别
- 筛选近3年内的有效案例（超过3年可能不反映当前市场环境）

### 步骤2：争议模式识别
- 统计各类条款争议的发生频率
- 识别高频争议条款TOP 10
- 分析争议与客户特征的关联（哪类客户更容易发生哪类争议）
- 分析争议与条款表述的关联（哪些模糊表述容易引发分歧）
- 识别季节性/周期性模式（如年底验收争议高发）

### 步骤3：争议根因分析
对高频争议条款深入分析：
- 条款本身存在歧义（表述问题）
- 执行环境变化导致条款不适用（市场变化）
- 双方对条款理解差异（沟通问题）
- 一方故意利用条款漏洞（恶意行为）
- 条款缺失导致无据可依（遗漏问题）
- 生成根因分布统计

### 步骤4：影响评估与趋势分析
- 各类争议的平均解决成本（时间+金额）
- 争议对客户关系的影响评估
- 争议数量的年度趋势（是否在改善）
- 特定条款修改后争议发生率的变化（验证模板优化效果）
- 预测当前合同发生争议的概率

### 步骤5：洞察输出与建议
- 为当前合同风险评估提供历史数据支撑
- 生成"前N名需要特别注意的条款类型"提示
- 输出类似案例的处理方式和结果参考
- 提出模板优化建议（基于争议数据）
- 生成培训案例库（脱敏后用于团队学习）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| queryContext | Object | 是 | 查询上下文（行业、客户类型、合同类型） |
| specificClauses | Array | 否 | 特定关注的条款类型 |
| timeRange | Object | 否 | 时间范围（默认近3年） |
| analysisDepth | String | 否 | 分析深度（summary/detailed） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| relevantCases | Array | 相关历史争议案例（脱敏） |
| disputePatterns | Object | 争议模式统计 |
| riskPrediction | Object | 当前合同争议概率预测 |
| lessonLearned | Array | 关键经验教训 |
| templateOptimizationSuggestions | Array | 模板优化建议 |

## 最佳实践
1. **持续积累**：每个合同争议解决后及时录入案例库，保持数据鲜活
2. **脱敏处理**：案例分享时必须脱敏处理客户名称和具体金额
3. **正反案例**：既记录争议案例，也记录成功避免争议的优秀实践
4. **量化表达**：用数据说话（"该类条款的争议发生率为12%"比"这个条款有风险"更有说服力）
5. **场景关联**：按可检索的标签组织案例，确保需要时能快速找到

## 约束条件
- 争议案例数据属于高度机密信息，不得对外分享
- 案例库引用时必须使用脱敏编号，不得暴露客户真实信息
- 统计分析结果需标注样本量和置信度
- 超过3年的案例仅作参考，不作为主要决策依据
- 涉及仲裁/诉讼的案例需标注法务部门的最终处理意见
