# 签署与归档管理技能

## 概述
签署与归档管理技能负责合同从审核通过到最终签署、归档入库的全流程管理。包括金额会签控制、签署方式选择（电子/纸质）、签署流程追踪、归档索引建立和CRM系统同步。该技能确保合同签署的合规性和归档的完整性。

## 适用场景
- 审核通过后的合同签署流程启动
- ≥100万合同的财务总监会签触发
- 电子签署平台（e签宝/法大大）的签署发起和追踪
- 纸质合同的寄送、盖章和回收管理
- 合同归档入库和CRM系统同步
- 签署后的合同变更（补充协议）管理

## 执行步骤

### 步骤1：签署前置检查
- 确认合同审核已通过（有书面审核意见）
- 确认所有协商条款已达成一致（无未决项）
- 检查合同金额：≥100万触发财务总监会签流程
- 确认合同签署人授权范围（金额是否在授权范围内）
- 确认对方签署人身份和授权

### 步骤2：会签流程管理（≥100万）
- 发起财务总监会签请求
- 会签审核要点：
  - 收入确认方式是否符合会计准则
  - 回款计划是否合理（账期、里程碑设置）
  - 是否存在收入确认风险（如过长验收期）
  - 对季度/年度收入预测的影响
- 记录会签意见并附入合同审批记录
- 会签通过后方可进入签署环节

### 步骤3：签署方式确定与执行
- 电子签署（优先推荐）：
  - 通过e签宝/法大大平台发起签署
  - 设置签署顺序（我方先签/对方先签/同时签署）
  - 发送签署链接给对方签署人
  - 追踪签署进度，3天未签署发送提醒
- 纸质签署：
  - 打印正式合同（至少两份正本）
  - 内部盖章流程
  - 寄送对方签署
  - 追踪回收进度
  - 收到对方签字盖章版后验证完整性

### 步骤4：归档入库
- 合同签署完成后24小时内完成归档
- 归档索引字段：
  - 合同编号（唯一）
  - 客户名称
  - 合同类型
  - 合同金额
  - 签约日期
  - 合同起止日期
  - 销售负责人
  - CSM负责人
  - 付款条件摘要
  - 关键里程碑日期
- 电子合同：直接入库
- 纸质合同：扫描后入库（清晰度要求：300dpi以上）

### 步骤5：系统同步与通知
- CRM系统更新：商机状态变更为"已签约"，填入合同编号和金额
- 通知客户成功团队启动交接流程
- 通知财务团队录入合同台账
- 更新销售个人和团队业绩统计
- 设置合同到期提醒（到期前90天触发续约流程）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| approvedContract | File | 是 | 审核通过的合同文本 |
| contractAmount | Number | 是 | 合同金额 |
| signingMethod | String | 是 | 签署方式（electronic/paper） |
| counterpartySigner | Object | 是 | 对方签署人信息 |
| internalApprovals | Array | 是 | 内部审批记录 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| signedContract | File | 已签署合同文件 |
| archiveRecord | Object | 归档记录（含索引信息） |
| crmUpdateStatus | String | CRM同步状态 |
| notificationsSent | Array | 已发送的通知列表 |
| renewalReminder | Object | 续约提醒设置 |

## 最佳实践
1. **电子优先**：优先推荐电子签署，缩短签署周期从平均7天到1天
2. **并行签署**：双方同时签署优于串行签署，减少等待时间
3. **催签策略**：签署发出后第1天/第3天/第5天分级催签
4. **归档即时性**：签署完成后立即归档，避免遗漏
5. **冗余备份**：电子合同多地备份，纸质合同保留扫描件和实体件双份

## 约束条件
- ≥100万合同必须完成财务总监会签方可签署，会签率目标100%
- 电子签署平台必须具有法律效力认证（CA证书）
- 纸质合同盖章必须在授权清单内的印章使用
- 归档后的合同变更只能通过补充协议实现，原合同不可修改
- 合同扫描件必须清晰可辨认，关键页（金额页、签章页）单独拍照存档
- 归档后24小时内完成CRM同步，超时需说明原因
