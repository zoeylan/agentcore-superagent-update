# 风险缓释建议技能

## 概述
风险缓释建议技能基于合同风险评估结果，为谈判团队提供具体可操作的风险对冲策略和条款修改建议。该技能的核心价值在于将风险评估结论转化为谈判桌上的具体行动方案，帮助销售团队在保护企业利益和推进成交之间找到最优平衡点。

## 适用场景
- 风险评估完成后的谈判策略制定
- 客户拒绝修改高风险条款时的替代方案设计
- 条款协商过程中的实时策略调整
- 多条款联动谈判时的交换策略规划
- 风险接受决策前的缓释措施确认

## 执行步骤

### 步骤1：风险优先级排序
- 按风险影响金额×概率进行排序
- 标记"必须解决"（不解决则建议不签）和"尽力争取"（可附条件接受）
- 识别相互关联的风险条款（解决A可能顺带解决B）
- 确定谈判的整体策略基调（强势/平等/让步换取）

### 步骤2：逐条生成修改建议
对每个中高风险条款提供：
- **理想方案**：对我方最有利的条款表述
- **折中方案**：双方平衡的条款表述
- **底线方案**：我方可接受的最低保护
- **对冲方案**：如客户坚持不改，通过其他条款进行风险对冲
  - 例：客户要求90天账期 → 要求增加逾期利息条款（日万分之五）
  - 例：客户要求无限违约赔偿 → 要求设置赔偿上限（不超过合同额）
  - 例：客户要求短验收期 → 要求明确验收测试用例清单

### 步骤3：条款交换矩阵设计
- 建立"我方可让步条款"和"我方必须坚持条款"清单
- 设计条款交换对（如果让步A则要求B）
- 评估每个交换对的净风险变化
- 准备谈判话术模板：
  - "考虑到贵方在XX方面的关切，我们愿意在付款条件上提供更多灵活性；同时希望贵方也能理解我们在YY方面的立场"
- 设置交换的优先级和底线

### 步骤4：非条款缓释手段
当条款修改空间有限时，考虑其他风险缓释手段：
- 要求提供银行履约保函/保证金
- 购买相关保险（如职业责任险）
- 分阶段执行（先小范围试点再全面部署）
- 增加里程碑付款节点（减少单次付款风险）
- 设置条款生效的触发条件/先决条件
- 缩短合同期限（降低长期风险暴露）

### 步骤5：决策建议输出
- 综合评估所有缓释措施后的残余风险
- 明确判断：缓释后风险是否可接受
- 如可接受：输出"建议签署，需落实以下缓释措施"
- 如仍不可接受：输出"建议不签署"或"需升级至VP/CEO决策"
- 附带决策所需的简要信息摘要

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| riskAssessment | Object | 是 | 风险评估结果 |
| negotiationContext | Object | 是 | 谈判上下文（客户态度、时间压力） |
| companyBottomLines | Array | 是 | 企业底线条款清单 |
| dealImportance | String | 是 | 交易战略重要性（high/medium/low） |
| previousNegotiationRounds | Array | 否 | 历史谈判轮次记录 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| negotiationStrategy | Object | 整体谈判策略 |
| clauseModifications | Array | 逐条修改建议（理想/折中/底线） |
| exchangeMatrix | Array | 条款交换矩阵 |
| nonClauseMitigation | Array | 非条款缓释手段 |
| residualRiskAssessment | Object | 缓释后残余风险评估 |
| finalRecommendation | String | 最终建议（sign/escalate/reject） |

## 最佳实践
1. **客户动机理解**：了解客户坚持某条款的真实动机（法务合规要求还是谈判策略），针对性回应
2. **价值对等**：每次让步都应获得对等回报，避免单方面退让导致对方得寸进尺
3. **创造性解决**：超越"修改条款"的思维，用商业安排解决法律风险
4. **时间压力利用**：如果我方时间充裕而客户急于签约，可以更坚持风险条款修改
5. **留有余地**：初始方案略高于理想方案，为谈判留出空间

## 约束条件
- 底线方案不得低于企业风险管理制度规定的最低保护标准
- 条款交换策略需经法务确认合规性
- 涉及知识产权核心条款的让步需CTO签字确认
- 风险接受声明金额>500万需CEO审批
- 谈判策略文档不得发送给客户方任何人
