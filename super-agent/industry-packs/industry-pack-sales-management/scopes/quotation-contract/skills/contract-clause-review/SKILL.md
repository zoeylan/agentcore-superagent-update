# 合同条款审核技能

## 概述
合同条款审核技能负责对合同条款进行系统性审核，识别风险条款、不合理条款和遗漏条款，协调法务团队完成深度审核，管理条款协商过程。该技能确保签署的每一份合同都经过充分的风险评估，在保护企业利益的同时维持合理的审核效率。

## 适用场景
- 标准合同的快速审核（内部简审流程）
- 非标合同的深度审核（法务介入流程）
- 客户甲方模板的逐条审核
- 客户修改意见的评估和回应
- 合同续签时变更条款的审核
- 补充协议的条款审核

## 执行步骤

### 步骤1：审核类型判定与分派
- 标准合同（无非标条款标记）：进入快速审核通道（1个工作日）
- 含非标条款的合同：分派法务团队深度审核（5个工作日）
- 甲方模板合同：进入逐条审核流程（时间视复杂度）
- 分配审核人（根据合同类型、行业和法务专长匹配）
- 启动审核SLA计时

### 步骤2：条款清单检查
按标准检查清单逐项审核：
- [ ] 合同主体信息准确完整
- [ ] 标的物描述与报价单一致
- [ ] 合同金额计算正确
- [ ] 付款条件明确且可执行
- [ ] 交付时间线合理
- [ ] 验收标准清晰可量化
- [ ] SLA条款匹配承诺等级
- [ ] 违约责任对等合理
- [ ] 知识产权归属明确
- [ ] 保密条款覆盖充分
- [ ] 不可抗力条款完备
- [ ] 合同终止条件合理
- [ ] 争议解决方式明确

### 步骤3：风险条款识别与标记
- 对偏离标准的条款标记风险等级（高/中/低）
- 生成风险条款清单，每条包含：
  - 原文引用和位置
  - 偏离标准的描述
  - 风险影响评估
  - 建议修改方案（A方案-理想 / B方案-底线）
- 高风险条款转交合同风险分析师深度评估

### 步骤4：法务意见管理
- 收集法务审核意见（通过/有条件通过/不通过）
- 将法务意见转化为客户可理解的修改建议
- 对"有条件通过"明确条件清单和优先级
- 对"不通过"条款区分"必须修改"和"强烈建议修改"
- 生成最终审核意见书

### 步骤5：协商过程管理
- 记录每轮条款修改的完整历史（谁改了什么、为什么、何时）
- 管理修改版本（使用修订模式/对比文档）
- 追踪未决条款和已达成共识条款
- 当协商超过3轮时评估是否需要升级决策
- 条款协商僵局时准备升级简报

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| contractDraft | File | 是 | 合同草稿/甲方模板 |
| contractType | String | 是 | 合同类型 |
| nonStandardFlags | Array | 否 | 已标记的非标条款 |
| customerModifications | Array | 否 | 客户修改意见 |
| previousVersions | Array | 否 | 历史协商版本 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| reviewResult | String | 审核结论（pass/conditional_pass/reject） |
| reviewOpinion | File | 书面审核意见书 |
| riskClauses | Array | 风险条款清单 |
| modificationSuggestions | Array | 修改建议 |
| negotiationHistory | Array | 协商历史记录 |
| pendingItems | Array | 未决条款 |

## 最佳实践
1. **底线清单前置**：在审核前就明确企业不可让步的底线条款清单，避免审核中反复内部请示
2. **客户视角翻译**：将法律术语转化为业务语言，便于销售人员向客户解释修改原因
3. **交换策略准备**：准备"如果客户在A条款不让步，我们可以在B条款要求对等保护"的预案
4. **时效意识**：审核进度实时可见，避免"法务黑洞"（提交后杳无音讯）
5. **标准化积累**：将高频出现的非标条款处理方案标准化，逐步扩充标准条款库

## 约束条件
- 所有审核意见必须留文字记录，不接受口头通过
- 审核意见须标明出具人和日期
- 非标条款修改需附风险等级说明
- 合同修改超过3处以上的需重新走完整审核流程
- 客户修改意见收到后2个工作日内需给出初步回应
