# 合同风险评估技能

## 概述
合同风险评估技能提供对合同条款的系统性风险评级能力，覆盖付款条件风险、交付承诺风险、知识产权风险、数据安全合规风险和商业条款风险五大维度。该技能将主观的"风险感知"转化为结构化的风险评分和量化影响评估，为谈判决策提供数据支撑。

## 适用场景
- 非标合同条款的深度风险评估
- 客户甲方模板的全面风险扫描
- 合同条款修改后的风险变化评估
- 大额合同（>500万）的强制风险评审
- 新行业/新客户类型的合同风险基线建立
- 合同执行中争议条款的复盘分析

## 执行步骤

### 步骤1：风险条款识别
- 逐条扫描合同，识别潜在风险条款
- 对照企业"底线条款清单"标记不可接受项
- 对照行业惯例标记偏离项
- 识别隐含风险（条款本身合理但组合后产生风险）
- 识别遗漏条款（应该有但缺失的保护性条款）

### 步骤2：五维度风险评估

**付款条件风险：**
- 账期评估（30天低风险/60天中风险/90天+高风险）
- 预付款比例（无预付款为高风险）
- 付款条件的可执行性（里程碑定义是否清晰可验证）
- 逾期付款违约金是否充分

**交付承诺风险：**
- 交付时间线与实际产能匹配度
- 验收标准的主观性（是否可能被恶意利用）
- 延迟交付惩罚条款的合理性（行业标准0.5-1%/周，上限10-20%）
- 不可抗力条款的覆盖范围

**知识产权风险：**
- IP归属条款（开发物/衍生物/背景IP）
- 竞业限制对业务的影响
- 开源组件使用的合规性
- 客户数据中的IP风险

**数据安全合规风险：**
- 数据处理权限边界
- 跨境数据传输合规性
- 等保/个人信息保护法合规
- 数据泄露责任和赔偿上限

**商业条款风险：**
- 排他性条款的范围和时限
- 最惠客户条款的触发条件
- 自动续约条款的退出机制
- 合同终止的条件和后果

### 步骤3：风险量化评估
- 对每个风险条款评估三种情景：
  - 最好情况（Best Case）：风险未触发
  - 最可能情况（Most Likely）：一般风险触发
  - 最坏情况（Worst Case）：极端风险触发
- 量化经济影响（金额或合同额百分比）
- 估算风险触发概率（基于历史数据）
- 计算风险期望值 = 概率 × 影响金额

### 步骤4：风险评级输出
- 高风险（红色）：必须修改或增加对冲条款
- 中风险（黄色）：建议协商修改，可附条件接受
- 低风险（绿色）：可以接受
- 综合风险评分（加权计算各维度得分）
- 与历史同类合同风险水平对比

### 步骤5：对冲建议生成
- 针对每个中高风险条款提供修改建议：
  - 理想修改方案（对我方最有利）
  - 底线修改方案（可接受的最低保护）
- 提供条款交换策略（此条款让步则彼条款要求对等保护）
- 识别可用的风险缓释手段（如保险、担保、分期等）
- 输出谈判优先级排序（哪些必须改、哪些可以让）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| contractDocument | File | 是 | 合同文本 |
| contractAmount | Number | 是 | 合同金额 |
| customerProfile | Object | 是 | 客户信息（行业、规模、信用评级） |
| contractType | String | 是 | 合同类型 |
| flaggedClauses | Array | 否 | 已标记的关注条款 |
| historicalDisputes | Array | 否 | 与该客户的历史争议记录 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| overallRiskScore | Number | 综合风险评分（0-100） |
| overallRiskLevel | String | 综合风险等级（high/medium/low） |
| riskDetails | Array | 各条款风险详情 |
| quantifiedImpact | Object | 量化影响评估（最好/最可能/最坏） |
| mitigationSuggestions | Array | 风险对冲建议 |
| negotiationPriority | Array | 谈判优先级排序 |
| executiveSummary | String | 管理层一页纸摘要 |

## 最佳实践
1. **历史数据驱动**：充分利用历史合同争议数据评估风险概率，避免纯主观判断
2. **行业特性考量**：不同行业的风险容忍度不同（如金融行业对数据安全零容忍）
3. **客户信用分层**：大型上市公司的付款风险通常低于中小企业，评估需差异化
4. **动态评估**：合同执行期间重大环境变化时应重新评估风险
5. **简明报告**：管理层需要一页纸结论和建议，详细分析作为附件

## 约束条件
- 高风险评估结论需经法务负责人确认后方可对外共享
- 风险评估报告保密等级为"内部机密"，不得向客户透露
- 风险评分模型每半年校准一次（对比实际争议数据）
- 合同金额>500万的高风险合同必须附管理层签字确认的风险接受声明
- 评估结论仅供决策参考，不构成法律意见（如需法律意见需外聘律师）
