# CPQ产品配置技能

## 概述
CPQ（Configure-Price-Quote）产品配置技能用于根据客户需求从产品目录中智能配置最优产品/服务组合方案。该技能是报价流程的起点，确保产品组合的可行性、兼容性和最优性。

## 适用场景
- 客户提出明确产品需求，需要快速配置标准方案
- 客户有业务痛点但不确定需要哪些产品，需要推荐方案
- 多产品捆绑销售场景，需要检查产品兼容性和依赖关系
- 续约客户升级/扩容，需要基于现有配置推荐增量方案

## 执行步骤

### 步骤1：需求信息采集与确认
- 确认客户基础信息（行业、规模、现有系统环境）
- 明确业务需求（解决什么问题、预期目标）
- 确认技术约束（部署方式、集成需求、用户规模）
- 识别必选需求和可选需求

### 步骤2：产品目录匹配
- 从产品目录中筛选符合需求的产品/模块
- 检查产品间的依赖关系（如模块A必须搭配基础平台）
- 检查产品间的兼容性（版本兼容、技术栈兼容）
- 标记需要定制开发的需求项

### 步骤3：方案组合优化
- 生成1-3个推荐方案（基础版/标准版/旗舰版）
- 每个方案包含：必选产品清单、推荐附加产品、可选增值服务
- 计算各方案的用户许可数量（基于客户规模）
- 标注各方案的差异点和适用场景

### 步骤4：可行性验证
- 标准产品组合：确认库存/License可用性
- 定制需求：提交产品/交付团队确认可行性和预估工期
- 技术集成：确认API/接口可用性和集成复杂度
- 输出可行性确认结果

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| customerProfile | Object | 是 | 客户基础信息（行业、规模、现有环境） |
| businessRequirements | Array | 是 | 业务需求列表 |
| technicalConstraints | Object | 否 | 技术约束条件 |
| budgetRange | Object | 否 | 预算区间（如客户提供） |
| existingProducts | Array | 否 | 已购产品清单（续约/增购场景） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| recommendedConfigurations | Array | 1-3个推荐方案 |
| configuration.products | Array | 产品明细（SKU、名称、规格、数量） |
| configuration.dependencies | Array | 产品依赖关系说明 |
| configuration.customItems | Array | 需定制开发的项目（如有） |
| feasibilityStatus | String | 可行性状态（confirmed/pending_review） |
| estimatedDeliveryTimeline | String | 预估交付周期 |

## 最佳实践
1. **优先标准化**：优先推荐标准产品组合，减少定制开发比例，缩短交付周期
2. **渐进式配置**：先满足核心需求，将非紧急需求规划为二期，降低首次采购门槛
3. **兼容性前置**：在方案推荐阶段就完成兼容性检查，避免签约后出现技术冲突
4. **竞品对标**：了解客户正在评估的竞品方案，确保配置的功能覆盖度不低于竞品
5. **扩展性考虑**：方案设计时预留扩展空间，为后续增购奠定基础

## 约束条件
- 产品目录更新后必须在24小时内同步最新SKU和价格
- 已下架产品不得出现在新报价配置中
- 单个方案产品数不超过20个SKU（超过需人工确认）
- 定制开发比例超过方案总额30%时需标记为"重度定制"并预警
