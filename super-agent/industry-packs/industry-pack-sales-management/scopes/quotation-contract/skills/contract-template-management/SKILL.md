# 合同模板管理技能

## 概述
合同模板管理技能负责维护企业标准合同模板库，支持基于商务条件的合同文本智能填充和自动生成。该技能的核心目标是将标准合同占比提升至80%以上，通过模板化和自动化显著缩短合同起草周期。

## 适用场景
- 基于确认报价的标准合同自动生成
- 不同业务类型的合同模板选择（产品销售/SaaS订阅/服务/框架协议）
- 合同模板版本管理和更新发布
- 条款库的维护和可选条款的智能推荐
- 甲方模板的结构化对比分析

## 执行步骤

### 步骤1：合同类型判定
- 基于交易特征判定合同类型：
  - 产品销售合同：一次性交付的软件/硬件产品
  - SaaS订阅合同：按年/月订阅的云服务
  - 实施服务合同：项目制的实施/咨询服务
  - 框架协议：多次采购的框架合同+订单模式
- 判定是否适用标准模板或需要定制

### 步骤2：模板选择与条款配置
- 从模板库中选择对应类型的最新版本模板
- 根据交易特征配置可选条款模块：
  - 付款条件条款（预付/分期/验收付/按月付）
  - SLA条款（标准/增强/企业级）
  - 数据安全条款（基础/等保二级/等保三级）
  - 知识产权条款（标准/客户专属定制）
  - 终止条款（标准30天通知/定制条件）
- 标记不适用的条款段落为删除

### 步骤3：商务信息自动填充
- 填充甲方信息（公司全称、法人、地址、统一社会信用代码）
- 填充合同标的（产品/服务清单、规格、数量、金额）
- 填充商务条件（付款条件、交付时间线、验收标准）
- 填充合同期限（起止日期、续约条件）
- 生成合同编号（格式：CT-YYYY-业务线代码-序号）

### 步骤4：非标条款标记
- 对比生成的合同与标准模板，标记偏离项
- 分类标记：轻微偏离（无需法务审核）/ 中度偏离（需法务确认）/ 重大偏离（需法务深度审核）
- 生成非标条款清单和审核要点说明
- 估算法务审核所需时间

### 步骤5：模板版本管理
- 定期审核模板合规性（法律法规更新、公司政策变更）
- 管理模板版本历史（何时修改、修改内容、审批人）
- 发布新版本时通知相关使用人员
- 旧版本模板生成的合同标记存档版本号

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dealType | String | 是 | 交易类型（product/saas/service/framework） |
| confirmedQuotation | Object | 是 | 确认的报价信息 |
| customerInfo | Object | 是 | 客户完整信息 |
| specialRequirements | Array | 否 | 特殊条款需求 |
| templateOverride | String | 否 | 指定模板版本（默认最新） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| contractDraft | File | 合同草稿文档 |
| contractId | String | 合同编号 |
| templateVersion | String | 使用的模板版本 |
| nonStandardClauses | Array | 非标条款标记列表 |
| reviewRequirement | String | 审核要求等级 |
| estimatedReviewTime | String | 预估审核时长 |

## 最佳实践
1. **模板精简**：保持模板清晰简洁，避免过多"以防万一"的冗余条款吓退客户
2. **模块化设计**：条款以模块化方式组织，支持灵活组合而非整体修改
3. **双语备案**：涉及外资客户时准备中英双语版本，以中文版为准
4. **条款互锁检查**：自动检测条款间的逻辑一致性（如付款条件与交付条件的匹配）
5. **客户适配**：大客户/上市公司通常有更严格的合规要求，预置增强型条款选项

## 约束条件
- 模板更新需经法务负责人审批后方可发布
- 已签署合同不受模板更新影响
- 使用3个月以上未更新的模板需强制复核
- 甲方模板使用时必须经过完整的逐条审核流程
- 合同编号一旦分配不可重复使用
