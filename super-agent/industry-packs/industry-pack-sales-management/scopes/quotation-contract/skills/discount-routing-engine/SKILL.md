# 折扣路由引擎技能

## 概述
折扣路由引擎技能负责根据折扣幅度自动判定审批层级并路由审批请求至对应审批人，同时管理审批时效监控、超时催办和自动升级流程。这是折扣管控的核心执行引擎，确保每笔折扣都经过恰当层级的审批且在时效要求内完成。

## 适用场景
- 新报价包含折扣时的审批流程启动
- 报价修改导致折扣率变化时的审批层级重新判定
- 审批超时时的自动催办和升级处理
- 审批驳回后的修改重提交路由
- 批量报价场景的并行审批调度

## 执行步骤

### 步骤1：折扣率计算与层级判定
- 计算综合折扣率 = (总目录价 - 总报价) / 总目录价 × 100%
- 判定审批层级：
  - 折扣率 ≤ 10%：路由至销售经理
  - 折扣率 ≤ 20%：路由至销售总监
  - 折扣率 ≤ 30%：路由至VP
  - 折扣率 > 30%：路由至CEO
- 检查特殊规则：战略客户是否有特殊折扣权限、年框协议内折扣是否免审

### 步骤2：审批人确定
- 根据销售人员的组织架构确定直属审批链路
- 检查审批人当前状态（是否在休假/出差）
- 如审批人不可用，自动路由至代理审批人
- 对于跨层级审批（如需VP审批），确定逐级还是跳级审批规则

### 步骤3：审批请求发送
- 生成审批工单（包含报价摘要、折扣详情、决策辅助信息）
- 通过企业IM/邮件/审批系统发送审批通知
- 启动SLA计时器（4小时/级）
- 记录审批发起时间和预期完成时间

### 步骤4：时效监控与升级
- 实时监控审批进度
- 4小时未审批：发送第一次催办通知
- 6小时未审批：发送第二次催办（升级提醒）
- 8小时未审批：自动升级至上级审批人代审
- 记录各环节耗时，用于SLA达成率统计

### 步骤5：结果处理与反馈
- 审批通过：通知报价配置专家更新报价单、记录审批意见
- 审批驳回：记录驳回原因、通知销售人员调整方案
- 有条件通过：记录条件要求、确认销售人员知悉并接受条件
- 更新审批记录和统计数据

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| quotationId | String | 是 | 报价单编号 |
| listPrice | Number | 是 | 目录总价 |
| quotedPrice | Number | 是 | 报价金额 |
| discountRate | Number | 是 | 折扣率(%) |
| salesRepId | String | 是 | 销售人员ID |
| discountReason | String | 是 | 折扣原因分类 |
| urgencyLevel | String | 否 | 紧急程度（normal/urgent） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| approvalLevel | String | 审批层级 |
| approverInfo | Object | 审批人信息 |
| approvalDeadline | DateTime | 审批截止时间 |
| approvalStatus | String | 审批状态 |
| approvalComments | String | 审批意见 |
| escalationHistory | Array | 升级历史记录 |

## 最佳实践
1. **预判提醒**：在销售填写折扣前预判其权限范围，避免提交后被驳回浪费时间
2. **批量合并**：同一审批人短时间内的多笔审批请求合并展示，减少打扰
3. **移动审批**：确保审批通知可在手机端完成一键审批，不受办公环境限制
4. **周末/假日规则**：非工作时间的审批请求自动顺延SLA或路由至备用审批人
5. **审批预热**：对于大单折扣，在正式提交前与审批人提前沟通，缩短正式审批时间

## 约束条件
- 折扣率为0的报价无需进入审批流程
- 年框协议内约定的折扣率以内无需逐次审批
- 同一报价的折扣只能有一个活跃审批流程
- 审批超时升级最多上升两级（如销售经理超时→总监代审，不会跳到VP）
- 紧急审批请求的SLA缩短为2小时/级
