# 折扣数据分析技能

## 概述
折扣数据分析技能负责对企业折扣数据进行多维度分析，识别折扣发放的趋势、异常和优化机会，为定价策略调整和折扣政策优化提供数据驱动的洞察。该技能是从"事后统计"升级为"主动预警和策略建议"的关键能力。

## 适用场景
- 月度折扣率报表生成和趋势分析
- 单客户累计折扣比率监控和预警
- 团队/个人折扣发放模式分析
- 折扣与赢率/利润率关联分析
- 折扣预算消耗进度追踪
- 异常折扣模式识别和预警

## 执行步骤

### 步骤1：数据采集与汇总
- 从审批记录中提取所有折扣数据（金额、比率、原因、审批人、结果）
- 按维度分类汇总：时间（周/月/季度）、团队、个人、产品线、行业、客户规模
- 计算关键指标：平均折扣率、中位数折扣率、折扣率标准差
- 标注数据完整性（覆盖率、缺失值处理）

### 步骤2：趋势分析
- 折扣率时间趋势（是否有季度末突增等规律性波动）
- 折扣原因分布变化（竞争压力占比是否在增加）
- 审批通过率趋势（是否有审批标准收紧/放松的迹象）
- 折扣与赢率的关联分析（折扣越大赢率是否越高）
- 折扣与客户续约率的关联分析（低价获客是否影响长期价值）

### 步骤3：异常检测
- 单客户累计折扣超过年度报价30%的预警
- 月度整体折扣率超过计划值15%的预警
- 某销售连续3单以上折扣率>20%的模式预警
- 某区域/行业折扣率显著偏离团队平均的预警
- 折扣率与客户规模不匹配的异常（小客户获得大折扣）

### 步骤4：优化建议生成
- 基于数据分析提出定价策略调整建议
- 识别可以缩减折扣空间的产品/场景
- 识别需要增加价格弹性的竞争激烈市场
- 建议折扣审批矩阵的阈值调整（如果数据显示当前阈值不合理）
- 输出Top 5可操作的折扣优化建议

### 步骤5：报告输出
- 生成月度折扣分析报告（管理层级别）
- 生成团队折扣对比看板（销售管理级别）
- 生成个人折扣使用明细（销售个人级别）
- 异常预警实时通知（管理层+相关负责人）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| timeRange | Object | 是 | 分析时间范围 |
| dimensions | Array | 否 | 分析维度（team/rep/product/industry） |
| benchmarks | Object | 否 | 对比基准（计划值、历史同期） |
| alertThresholds | Object | 否 | 预警阈值配置 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| summaryMetrics | Object | 汇总指标（平均折扣率、总折扣金额等） |
| trendAnalysis | Object | 趋势分析结果 |
| anomalies | Array | 检测到的异常列表 |
| recommendations | Array | 优化建议列表 |
| reportDocument | File | 格式化分析报告 |

## 最佳实践
1. **对比视角**：始终与基准对比（计划值/历史同期/行业水平），避免孤立数字无意义
2. **归因分析**：异常数据不仅要报告，更要分析原因（是市场变化还是管理松懈）
3. **可操作建议**：每个洞察必须关联一个具体可执行的建议行动
4. **分层报告**：不同层级看不同粒度——CEO看趋势，VP看团队，经理看个人
5. **前瞻预测**：基于当前趋势预测本季度末折扣率，提前预警超标风险

## 约束条件
- 月度报告在每月3号前生成（数据截止至上月最后一天）
- 异常预警在检测到后1小时内发送
- 个人折扣明细仅对其上级管理者可见，遵循最小权限原则
- 竞品价格信息在分析报告中脱敏展示
- 分析结论需标注统计显著性（样本量不足时明确说明）
