# Sensitive Content Filter（敏感内容过滤）

## 技能概述

本技能负责对输入内容进行多维度敏感信息检测和过滤，覆盖政治敏感词、色情低俗用语、暴力恐怖表述、违禁品信息、个人隐私数据（PII）等维度。通过动态维护的敏感词库和语义分析模型，在不同场景下执行精准的内容过滤策略。

## 适用场景

- 用户输入中的敏感关键词实时过滤
- PII（个人隐私信息）泄露检测和脱敏
- 社会热点事件相关敏感内容的快速识别
- 广告法违禁用语检测（绝对化用语/虚假宣传词汇）

## 执行步骤

### Step 1: 文本预处理
- 全角/半角统一、繁简转换、拼音还原
- 识别并处理常见规避手法：谐音替代、拆字、emoji替代、字母数字混合、火星文
- 处理文本中的特殊格式：HTML标签剥离、Markdown解析、URL提取

### Step 2: 多级敏感词匹配
- **一级词库**（零容忍）：涉政核心敏感词、涉恐关键词、儿童色情相关词——命中即BLOCK
- **二级词库**（高风险）：色情低俗、暴力血腥、毒品赌博——命中后结合语境判定
- **三级词库**（需语境）：可能涉及敏感的中性词——必须结合上下文语义确认
- 使用AC自动机（Aho-Corasick）实现高效多模式匹配

### Step 3: PII检测与脱敏
- 中国大陆身份证号（18位校验）
- 手机号码（11位，含虚拟运营商号段）
- 银行卡号（Luhn校验）
- 邮箱地址、物理地址、IP地址
- 检测到PII后执行脱敏处理（保留格式，替换为*号）

### Step 4: 语境敏感度评估
- 对命中二级/三级词库的内容进行上下文语义分析
- 评估敏感词在当前语境中的实际含义（如"鸡"在美食/政治/色情三种语境下含义不同）
- 输出语境调整后的最终风险等级

### Step 5: 结果汇总输出
- 汇总所有检测维度的结果
- 输出命中的敏感词列表及其位置信息
- 对需要过滤的内容给出处理建议（删除/替换/标记）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | string | 是 | 待检测文本内容 |
| scene | enum | 否 | 内容场景（chat/comment/article/product） |
| region | string | 否 | 目标发布地区（影响敏感词库选择） |
| pii_detection | boolean | 否 | 是否启用PII检测（默认true） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| is_sensitive | boolean | 是否包含敏感内容 |
| risk_level | enum | CRITICAL/HIGH/MEDIUM/LOW/SAFE |
| hits | array | 命中的敏感词详情列表 |
| hits[].word | string | 敏感词原文 |
| hits[].category | string | 敏感词类别 |
| hits[].position | int[] | 在文本中的起止位置 |
| hits[].level | int | 词库等级（1/2/3） |
| pii_detected | array | 检测到的PII信息列表 |
| filtered_content | string | 过滤/脱敏后的内容 |

## 最佳实践

1. **词库时效性**：核心敏感词库需跟踪时事热点，重大事件发生后2小时内更新
2. **误杀控制**：三级词库必须结合语境判定，避免"杀死比赛悬念"被误判为暴力内容
3. **规避手法追踪**：持续收集用户规避审核的新手法（如用emoji表达敏感含义），及时更新检测逻辑
4. **分级策略**：不同场景采用不同严格度——UGC评论区严格、企业内部知识库宽松
5. **性能与精度平衡**：一级词库走精确匹配（快），二三级词库走语义分析（准）

## 约束条件

- 一级敏感词零漏检率，误报可接受（人工复核兜底）
- 单次检测延迟<50ms（一级匹配）+ <200ms（语义分析，异步可选）
- 词库更新支持热加载，不中断服务
- PII脱敏后的数据不可逆还原
