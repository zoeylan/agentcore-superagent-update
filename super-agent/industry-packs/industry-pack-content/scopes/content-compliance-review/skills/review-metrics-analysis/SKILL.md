# Review Metrics Analysis（审核效果指标分析）

## 技能概述

本技能负责审核系统核心KPI指标的采集、计算、分析和可视化呈现。通过对FN率、FP率、自动化率、响应时延等关键指标的持续监控和趋势分析，为策略优化提供数据驱动的决策依据。支持日报/周报/月报的自动生成，以及异常指标的实时告警。

## 适用场景

- 每日审核效果日报自动生成
- 审核指标异常时的根因分析
- 策略变更前后的效果对比评估
- 灰度发布各阶段的指标监控
- 月度/季度审核效果复盘报告

## 执行步骤

### Step 1: 数据采集与清洗
- 从审核日志系统采集原始数据：审核请求量、判定结果、置信度分布、响应时间、人工复核结果
- 数据清洗：去除测试流量、排除系统异常时段数据、处理缺失值
- 数据对齐：将AI判定结果与人工终判结果进行对齐匹配
- 时间窗口划分：按小时/天/周/月聚合

### Step 2: 核心指标计算
- **漏判率（FN Rate）**：FN / (FN + TP)，目标<2%
  - FN = AI判定为安全但人工复核为违规的样本数
- **误判率（FP Rate）**：FP / (FP + TN)，目标<1.5%
  - FP = AI判定为违规但人工复核为安全的样本数
- **自动化率**：自动处理量（无需人工介入）/ 总审核量，目标>80%
- **人工复核响应时间**：从工单创建到复核完成的平均时间，目标<30分钟
- **前置拦截率**：前置拦截量 / 总请求量
- **审核服务可用性**：服务正常时间 / 总时间，目标>99.9%

### Step 3: 趋势分析与异常检测
- 绘制各指标时间序列趋势
- 使用移动平均+标准差方法检测异常点
- 连续3天超标触发策略复盘预警
- 突发异常（单时段FN/FP突增>50%）触发即时告警
- 周环比/月环比分析，识别长期趋势

### Step 4: 归因分析
- 当指标异常时，执行归因分析：
  - 按违规类型维度拆解（哪类内容贡献了最多FN/FP）
  - 按规则维度拆解（哪条规则的误判/漏判最多）
  - 按渠道/场景维度拆解（哪个来源的问题最大）
  - 按时间维度拆解（是否与某次规则变更相关）
- 输出归因结论和优化建议

### Step 5: 报告生成
- **日报**：核心指标值+环比变化+异常标注
- **周报**：指标趋势+Top问题规则+优化建议
- **月报**：全面复盘+规则库健康度+ROI分析+下月改进计划
- 报告格式：结构化数据+可视化图表+文字总结

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| report_type | enum | 是 | daily/weekly/monthly/realtime/custom |
| time_range | object | 是 | 分析时间范围（start/end） |
| dimensions | string[] | 否 | 分析维度（violation_type/rule/channel/scene） |
| compare_with | object | 否 | 对比基准（如策略变更前的时间段） |
| alert_thresholds | object | 否 | 自定义告警阈值 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| metrics | object | 核心指标集（fn_rate/fp_rate/automation_rate等） |
| trends | array | 指标时间序列数据 |
| anomalies | array | 检测到的异常点列表 |
| attribution | object | 归因分析结果 |
| recommendations | array | 策略优化建议列表 |
| report_url | string | 生成的报告链接 |
| alerts | array | 触发的告警列表 |

## 最佳实践

1. **及时性**：日报在次日10:00前自动生成，异常告警实时触发
2. **可操作性**：报告中的每条发现都应附带具体的优化建议
3. **基准对比**：始终与历史基准对比，孤立的绝对值缺乏判断意义
4. **样本验证**：指标异常时必须抽样验证原始数据，排除数据质量问题
5. **分层呈现**：管理层看摘要仪表盘，策略团队看详细归因，运维看服务健康度

## 约束条件

- 指标计算基于人工终判结果，需人工复核样本量>50才具有统计意义
- 日报自动化率统计排除降级时段（降级期间全转人工不计入）
- 实时监控粒度为5分钟，历史报告粒度为1小时
- 数据保留策略：明细数据90天，聚合数据3年
- 告警不超过每小时3条（防告警疲劳），超频时合并为摘要告警
