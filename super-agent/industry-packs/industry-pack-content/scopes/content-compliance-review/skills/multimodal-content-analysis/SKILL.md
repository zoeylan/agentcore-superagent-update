# Multimodal Content Analysis（多模态内容分析）

## 技能概述

本技能负责对文本、图片、音频、视频等多种内容形态执行合规性分析。通过调用多模态AI模型和专项检测模型，对内容的各个维度进行全面扫描，输出结构化的审核分析结果。支持单模态独立审核和跨模态关联分析，识别"单独看合规但组合后违规"的高级规避手法。

## 适用场景

- 文本内容合规审核（文章/评论/弹幕/聊天消息）
- 图片内容安全审核（用户上传图片/AI生成图片）
- 音频内容审核（语音消息/播客/音乐）
- 视频内容审核（短视频/直播回放/长视频）
- 图文混合/音视频混合的跨模态关联审核

## 执行步骤

### Step 1: 内容类型识别与预处理
- 自动识别输入内容的模态类型
- **文本预处理**：分段、语言检测、编码标准化
- **图片预处理**：格式转换、尺寸标准化、EXIF信息提取、OCR文字提取
- **音频预处理**：格式转换、采样率标准化、语音转文本（ASR）、背景音分离
- **视频预处理**：关键帧提取（自适应密度：静态场景低密度、快速切换场景高密度）、音轨分离、字幕OCR提取

### Step 2: 各维度并行审核

#### 文本维度
- 涉政敏感检测：领导人相关/政治事件/台海/涉藏/涉疆/境外势力
- 涉黄低俗检测：色情描写/软色情暗示/低俗用语
- 涉暴恐怖检测：暴力描述/恐怖宣传/自残引导/毒品信息
- 广告法违规检测：绝对化用语（最/第一/唯一）/虚假宣传/违禁商品推广
- 虚假信息检测：伪科学/未经证实的医疗声称/误导性统计
- 侵权检测：疑似抄袭/商标滥用/版权文本复制

#### 图片维度
- 色情内容检测：裸露程度分级/性暗示姿态/儿童色情（零容忍）
- 暴力血腥检测：真实暴力场景/图形化暴力/动物虐待
- 敏感符号检测：政治符号/宗教极端标识/仇恨符号/违禁旗帜
- 版权检测：水印残留/商标未授权使用/名人肖像权侵犯
- AI生成检测：深度伪造（DeepFake）/AI换脸/AI生成虚假场景

#### 音频维度
- 语音内容合规：ASR转文本后执行文本审核
- 背景音异常：隐藏语音信号/次声波嵌入/双音轨隐写
- 版权音乐检测：未授权音乐使用/音乐指纹匹配

#### 视频维度
- 关键帧审核：逐帧执行图片审核规则
- 音轨审核：执行音频审核流程
- 字幕审核：OCR提取后执行文本审核
- 时序行为分析：快速闪烁内容/渐变违规内容/关键帧插入攻击

### Step 3: 跨模态关联分析
- 文字与配图的一致性检查（如文字正经但配图低俗）
- 音频与画面的语义关联（如正常画面配违规音频）
- 字幕与实际语音的一致性（检测字幕替换规避）
- 整体内容意图推断（各单元素合规但组合后传达违规含义）

### Step 4: 综合评分输出
- 对各维度审核结果进行加权综合
- 权重由违规严重度决定：零容忍类>高风险类>中风险类
- 输出最终综合置信度分数和详细的维度评分明细
- 标注关键证据位置（文本段落/图片区域/视频时间码）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content_type | enum | 是 | text/image/audio/video/mixed |
| content | object | 是 | 内容数据（文本/URL/二进制引用） |
| review_dimensions | string[] | 否 | 指定审核维度（默认全维度） |
| context | object | 否 | 上下文信息（发布渠道/用户画像/关联内容） |
| priority | enum | 否 | 审核优先级（high/normal/low） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| overall_confidence | float | 综合违规置信度（0-1） |
| violation_labels | string[] | 违规类型标签列表 |
| dimension_scores | object | 各维度详细评分 |
| evidence | array | 关键证据列表 |
| evidence[].type | string | 证据类型（text_span/image_region/timestamp） |
| evidence[].location | object | 证据位置信息 |
| evidence[].description | string | 证据描述 |
| processing_time_ms | int | 审核处理耗时 |
| models_used | string[] | 使用的审核模型列表 |

## 最佳实践

1. **并行处理**：多维度审核并行执行，不串行等待，减少总处理延迟
2. **自适应精度**：根据内容的预估风险等级决定审核深度（低风险快速通道/高风险深度扫描）
3. **跨模态不可忽略**：即使各单模态审核通过，混合内容也必须执行跨模态关联分析
4. **零容忍即时报警**：儿童色情/恐怖主义/极端暴力等零容忍类型一旦检出立即告警，不等完整审核流程
5. **模型组合互补**：对关键维度使用2+个模型交叉验证，降低单模型盲区风险

## 约束条件

- 文本审核延迟<500ms，图片<1s，音频<3s/分钟，视频<5s/分钟
- 单次请求内容大小限制：文本<100KB，图片<20MB，音频<100MB，视频<1GB
- 视频关键帧抽检密度：普通内容1帧/秒，高风险片段5帧/秒
- 所有审核结论必须可解释（附带证据），不接受纯黑盒输出
- 模型推理结果缓存1小时（相同内容重复提交不重复计算）
