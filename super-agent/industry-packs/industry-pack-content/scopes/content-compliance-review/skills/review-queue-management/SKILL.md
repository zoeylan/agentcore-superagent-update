# Review Queue Management（人工复核队列管理）

## 技能概述

本技能负责人工复核工单的全生命周期管理，包括工单创建、优先级排序、审核员分配、SLA监控、超时升级和工作负载均衡。目标是确保人工复核环节高效运转，在SLA时限内完成所有复核任务，同时最大化审核员的生产力。

## 适用场景

- 置信度分流后创建人工复核工单
- 审核员工作队列的实时管理
- SLA违约预警和自动升级
- 审核员能力匹配和负载均衡
- 高峰时段的队列压力管理

## 执行步骤

### Step 1: 工单创建与丰富
- 接收分流决策传来的待复核内容
- 创建标准化工单，包含：
  - 工单ID（全局唯一）
  - 原始内容预览（文本摘要/图片缩略图/视频关键帧）
  - AI预判结果（置信度/违规标签/关键证据）
  - 来源信息（渠道/创作者/发布时间）
  - 建议判定和参考判例
- 标注工单属性：类型（涉政/涉黄/涉暴/广告/侵权等）、紧急程度

### Step 2: 优先级计算
- 综合以下因素计算工单优先级分数（0-100）：
  - **时效性权重（30%）**：内容已发布时长（越久越紧急）/ 即将发布的内容优先
  - **影响范围权重（25%）**：内容预估曝光量/创作者粉丝数/渠道流量
  - **违规严重度权重（30%）**：Critical>High>Medium>Low
  - **SLA剩余时间权重（15%）**：距离SLA截止时间越近优先级越高
- 特殊规则：涉政/涉恐类工单强制置顶（优先级=100）

### Step 3: 审核员匹配分配
- 根据工单类型匹配对应专项审核员：
  - 涉政类 → 资深审核员（≥2年经验/通过政治敏感性考核）
  - 涉黄类 → 专项审核员（通过心理健康评估）
  - 广告法类 → 具备法律知识的审核员
  - 多语言类 → 具备目标语言能力的审核员
- 在满足能力要求的审核员中，选择当前工作负载最低者
- 负载均衡策略：单人同时处理工单不超过5个

### Step 4: SLA监控与升级
- 实时监控各工单的处理时间：
  - 工作时间（9:00-21:00）：SLA = 30分钟
  - 非工作时间：SLA = 2小时
  - 高优先级（涉政/涉恐）：SLA = 15分钟（全天）
- 升级机制：
  - SLA 80%时间到达：发送提醒给当前审核员
  - SLA到期：自动升级给Team Lead + 重新分配给空闲审核员
  - SLA超过150%：升级给管理层 + 触发应急流程

### Step 5: 队列健康度监控
- 实时监控队列指标：
  - 队列深度（当前待处理工单数）
  - 平均等待时间
  - 审核员在线率和忙闲状态
  - SLA达标率
- 压力预警：
  - 队列深度>平时3倍：预警 + 建议增加人手
  - 平均等待时间>SLA 50%：预警 + 启动优先级重排
  - 在线审核员<最低要求：紧急呼叫备班人员

### Step 6: 复核结果收集
- 收集审核员的复核结论：通过/违规/争议
- 通过：释放内容 + 记录为AI误判样本（False Positive）
- 违规：执行处置 + 记录为AI漏判样本（False Negative）或确认样本（True Positive）
- 争议：升级给高级审核员/多人投票决策
- 所有复核结果写入训练数据集供模型迭代使用

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | enum | 是 | create/assign/escalate/complete/query |
| work_order | object | 条件 | 工单详情（create时必填） |
| reviewer_id | string | 条件 | 审核员ID（assign时必填） |
| verdict | object | 条件 | 复核结论（complete时必填） |
| queue_filter | object | 条件 | 队列查询条件（query时必填） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| order_id | string | 工单ID |
| status | enum | pending/assigned/in_review/completed/escalated |
| priority_score | int | 优先级分数（0-100） |
| assigned_reviewer | object | 分配的审核员信息 |
| sla_status | object | SLA状态（remaining_time/is_at_risk） |
| queue_metrics | object | 队列健康度指标 |
| result | object | 复核结果（complete时） |

## 最佳实践

1. **SLA分级管理**：不同类型不同SLA，不一刀切
2. **智能预排序**：工单进入队列时即完成优先级排序，审核员接单即可开始工作
3. **审核员保护**：涉及极端内容的审核员需要定期轮换和心理健康关怀
4. **数据隔离**：涉政类工单仅对通过安全审查的审核员可见
5. **高峰预案**：节假日/突发事件期间提前部署加班方案和自动化降级策略

## 约束条件

- 工单创建到审核员看到的延迟<5秒
- 单个审核员同时处理工单数≤5
- SLA达标率目标>95%
- 工单数据保留180天（含完整处理记录）
- 审核员每日工作量上限800条（防过劳导致质量下降）
