# Confidence Scoring（置信度评分）

## 技能概述

本技能负责对审核判定结果进行精准的置信度量化评分，输出0-1之间的连续置信度分数。置信度是审核系统"置信度分流"核心机制的基础——它直接决定了内容是自动放行、进入规则引擎二次判定还是转人工复核。置信度评分的准确性直接影响审核效率和安全性的平衡。

## 适用场景

- 多模态审核引擎完成各维度审核后的综合置信度计算
- 单维度审核结果的置信度校准
- 置信度阈值调优的数据支撑
- 审核模型输出不确定性的量化评估

## 执行步骤

### Step 1: 原始分数收集
- 收集各审核维度模型输出的原始概率值/分数
- 记录各模型的输出分布特征（不同模型的分数分布可能不同）
- 收集辅助信号：内容复杂度、上下文完整度、模型历史准确率

### Step 2: 分数校准（Calibration）
- 对各模型的原始输出进行校准，确保分数与实际概率对齐
  - 使用Platt Scaling或Isotonic Regression进行后处理校准
  - 校准基于历史人工复核标注数据定期更新
- 处理模型自信度过高/过低的问题：
  - 某些模型对不确定案例也输出极端分数→需要温度缩放
  - 某些模型对所有案例都输出中间分数→需要拉伸分布

### Step 3: 多维度综合加权
- 对校准后的各维度分数进行加权综合：
  - 权重基于违规严重度：零容忍类（涉政/涉恐/儿童色情）权重最高
  - 取各维度最高分作为基准（any-hit策略，有一维度高置信度即整体高风险）
  - 跨模态关联分析结果作为加权因子
- 综合公式：`final_score = max(dimension_scores) * cross_modal_factor`

### Step 4: 不确定性量化
- 计算模型预测的不确定性（aleatoric + epistemic）：
  - Aleatoric不确定性：内容本身的模糊性（如擦边球内容）
  - Epistemic不确定性：模型对该类内容经验不足
- 当不确定性高时，即使分数看似确定，也应降低最终置信度
- 对分布外（OOD）样本检测——输入与训练分布差异大时标记低置信度

### Step 5: 阈值映射与输出
- 将最终综合分数映射到三级分流区间：
  - >0.95：高置信度（自动处置）
  - 0.7-0.95：中置信度（规则引擎二次判定）
  - <0.7：低置信度（人工复核）
- 输出完整的置信度详情（综合分/各维度分/不确定性指标）
- 附带分流建议和置信度解释

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| dimension_results | array | 是 | 各审核维度的原始结果 |
| dimension_results[].dimension | string | 是 | 维度名称 |
| dimension_results[].raw_score | float | 是 | 模型原始输出分数 |
| dimension_results[].model_id | string | 是 | 使用的模型标识 |
| cross_modal_result | object | 否 | 跨模态关联分析结果 |
| content_metadata | object | 否 | 内容元数据（类型/长度/复杂度） |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| final_confidence | float | 最终综合置信度（0-1） |
| confidence_tier | enum | high/medium/low |
| routing_suggestion | enum | auto_pass/auto_block/rule_engine/human_review |
| dimension_confidences | object | 各维度校准后置信度 |
| uncertainty | object | 不确定性指标（aleatoric/epistemic） |
| calibration_version | string | 使用的校准模型版本 |
| explanation | string | 置信度评分解释说明 |

## 最佳实践

1. **定期校准**：校准模型应至少每周用最新人工标注数据更新
2. **保守策略**：不确定时给出偏低的置信度，让更多内容进入人工复核
3. **异常检测**：置信度分布出现异常偏移时告警（可能说明内容分布或模型发生漂移）
4. **阈值分离**：高置信度自动放行和自动拒绝使用不同阈值（放行要求更高置信度）
5. **可解释性**：每个置信度分数都应该能追溯到具体哪个维度/模型贡献了主要信号

## 约束条件

- 置信度计算延迟<50ms（在审核模型推理之后的轻量计算）
- 校准数据集最少需要1000条标注样本
- 阈值调整需经过A/B测试验证（不可直接在线修改）
- 置信度分布监控：每日检查分布形态，偏移>10%触发告警
- 所有置信度分数保留用于后续模型评估和阈值优化
