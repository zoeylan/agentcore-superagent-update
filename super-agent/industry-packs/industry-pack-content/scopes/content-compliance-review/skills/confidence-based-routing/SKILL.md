# Confidence-Based Routing（置信度分流决策）

## 技能概述

本技能负责基于审核引擎输出的置信度分数，执行智能分流决策——将审核结果路由到正确的后续处理通道。这是连接AI自动审核和人工复核的核心决策节点，分流策略的优劣直接影响审核自动化率和安全性的平衡。

## 适用场景

- 多模态审核引擎完成审核后的结果分流
- 中置信度内容的规则引擎二次判定
- 人工复核工单的自动创建和优先级分配
- 降级策略下的简化分流决策
- 分流阈值A/B测试

## 执行步骤

### Step 1: 置信度区间判定
- 接收审核引擎输出的置信度分数和违规分类结果
- 根据当前生效的阈值配置（默认：高>0.95，低<0.7）判定所属区间
- 注意：不同违规类型可能使用不同阈值
  - 零容忍类：自动拒绝阈值可降至0.90（宁可误判不可漏判）
  - 一般类：维持标准阈值

### Step 2: 高置信度通道（>0.95）
- **违规高置信度（>0.95 且判定为违规）**：
  - 自动执行拒绝处置
  - 写入审核拒绝日志（含完整判定依据）
  - 触发违规通知给内容创作方
  - 零容忍类同步触发监管报送流程
- **安全高置信度（>0.95 且判定为安全）**：
  - 自动放行
  - 写入审核通过日志
  - 内容释放进入正常发布流程

### Step 3: 中置信度通道（0.7-0.95）——规则引擎二次判定
- 调用规则引擎执行精细化二次判定：
  - 检查是否命中行业白名单（如医学术语/新闻引用/学术讨论）
  - 检查创作者信誉等级（高信誉用户放宽阈值）
  - 检查历史同类内容判例（相似内容历史判定结果参考）
  - 检查发布渠道的敏感度配置
- 二次判定结果：
  - **通过**：自动放行（标记为"规则引擎放行"）
  - **不通过**：创建人工复核工单
  - **不确定**：创建人工复核工单（标记为"规则引擎无法判定"）

### Step 4: 低置信度通道（<0.7）——转人工复核
- 直接创建人工复核工单
- 工单信息包含：
  - 原始内容摘要/预览
  - AI审核结果（预判结论+置信度+违规标签）
  - 关键证据高亮
  - 建议判定（供审核员参考但不替代其判断）
- 设定工单优先级（基于内容影响范围和违规严重度预估）

### Step 5: 分流结果记录与监控
- 记录每条内容的分流决策和依据
- 实时监控各通道流量分布：
  - 自动放行比例（目标>60%）
  - 规则引擎二次判定比例（目标<25%）
  - 人工复核比例（目标<15%）
- 分布异常时触发告警（如人工复核比例突增>30%）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| confidence_score | float | 是 | 综合置信度分数（0-1） |
| violation_result | object | 是 | 违规分类结果 |
| content_metadata | object | 是 | 内容元数据 |
| creator_profile | object | 否 | 创作者画像（信誉等级等） |
| channel_config | object | 否 | 发布渠道配置（敏感度等） |
| threshold_config | object | 否 | 自定义阈值配置 |

## 输出规格

| 字段 | 类型 | 说明 |
|------|------|------|
| routing_decision | enum | auto_pass/auto_block/rule_engine/human_review |
| decision_reason | string | 决策理由详细说明 |
| confidence_tier | enum | high/medium/low |
| secondary_judgment | object | 规则引擎二次判定结果（仅中置信度时有） |
| work_order | object | 人工复核工单（仅需人工时有） |
| work_order.priority | enum | critical/high/normal/low |
| work_order.suggested_verdict | string | 建议判定 |
| audit_log | object | 分流决策审计日志 |

## 最佳实践

1. **安全优先**：在不确定时，宁可多送人工复核也不轻易自动放行
2. **动态阈值**：根据当前FN/FP指标动态微调阈值，但调整幅度每次不超过±0.05
3. **渠道差异化**：高敏感渠道（如新闻资讯）使用更严格阈值，UGC社区适度宽松
4. **创作者分层**：高信誉创作者的中置信度内容可走快速通道，但零容忍类不受信誉影响
5. **降级预案**：审核服务异常时，切换到简化分流——低风险放行、高风险全转人工

## 约束条件

- 分流决策延迟<20ms（轻量级逻辑判定）
- 阈值调整须经过A/B测试验证（至少1000条样本对比）
- 零容忍类违规不适用高置信度自动放行（即使置信度>0.95判定安全也需人工确认）
- 分流决策日志保留180天
- 系统降级时的分流策略需提前定义并定期演练
