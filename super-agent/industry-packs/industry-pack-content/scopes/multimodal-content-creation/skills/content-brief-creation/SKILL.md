# 创作Brief制定 (Content Brief Creation)

## 概述

创作Brief制定技能负责将业务方的内容需求转化为结构化、可执行的创作任务描述。高质量的Brief是内容创作效率和质量的关键决定因素——一份完备的Brief可以将创作修改率降低60%以上。该技能确保每一个创作任务都有明确的方向、约束和验收标准。

## 适用场景

- 业务方提出新的内容创作需求时
- 营销Campaign需要批量内容规划时
- 需要A/B测试多版本内容时
- 紧急内容需求需要快速通道Brief时
- 跨部门协作的大型内容项目启动时

## 执行步骤

### 步骤1：需求收集与评估

1. 接收业务方原始需求描述（文字/语音/参考案例）
2. 评估需求完整度，核查五大必要要素：
   - 目标受众（Who）：人群画像/年龄/兴趣/消费能力
   - 核心信息（What）：要传达的关键卖点/情感价值
   - 调性要求（How）：情绪基调/语言风格/视觉风格
   - 渠道规格（Where）：目标发布平台及对应规格要求
   - 输出规格（Spec）：数量/格式/尺寸/时长等技术要求
3. 对不完整需求生成结构化澄清问题列表，退回业务方补充
4. 判断需求类型：常规内容/新品牌方向/紧急需求/A/B测试需求

### 步骤2：品牌规范引用

1. 确认该任务适用的品牌规范版本（主品牌/子品牌）
2. 从品牌资产库提取相关规范约束：
   - 文案调性关键词和禁用词库
   - 视觉VI参数（配色/字体/布局规则）
   - Logo使用规范
   - 竞品禁区和敏感元素清单
3. 将品牌约束嵌入Brief的约束条件部分

### 步骤3：Brief结构化输出

按照标准模板生成创作Brief：


### 步骤4：Brief审核与下发

1. 对Brief进行自检：五要素完整性/品牌规范引用正确性/规格参数可行性
2. 标记任务优先级和截止时间
3. 根据任务类型分发给对应的执行Agent
4. 建立任务追踪记录

## 输入/输出规格

### 输入
- 业务方原始需求描述
- 营销日历/Campaign规划
- 品牌资产库引用
- 历史类似任务参考

### 输出
- 结构化创作Brief（JSON格式）
- 澄清问题列表（如需求不完整时）
- 任务分发记录
- 排期时间线

## 最佳实践

1. **参考案例必须附上**：纯文字描述调性容易产生歧义，正反例（Do/Don't）最有效
2. **渠道规格要精确到数值**：不写"适合抖音"，要写"竖版1080×1920px, 15-60s, ≤50MB"
3. **核心信息控制数量**：一个Brief的核心卖点不超过3个，信息过载导致重点模糊
4. **预留修改空间**：截止时间须预留至少1轮修改缓冲（总时间的30%）
5. **紧急Brief有底线**：即使快速通道，品牌约束和核心信息仍不可省略

## 约束条件

- Brief必须包含目标受众/核心信息/调性要求/参考案例/输出规格五要素
- 缺失超过1个要素须退回业务方补充，不允许凭推测填充
- 品牌规范引用必须指向当前有效版本
- A/B测试Brief须明确变量控制（哪些维度变化/哪些保持一致）
