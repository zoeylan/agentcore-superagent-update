# 文化合规检查 (Cultural Compliance Check)

## 概述

文化合规检查技能负责对本地化后的内容进行文化层面的合规性验证，确保内容在目标市场不触犯文化禁忌、不引发文化冲突、不造成品牌声誉损害。该技能是多语言本地化的安全保障层——不仅检查"翻译对不对"，更检查"文化上能不能接受"。在品牌出海场景中，一次文化失误可能导致的PR危机远超翻译错误的影响。

## 适用场景

- 多语言内容发布前的文化合规审核
- 出海Campaign的目标市场文化风险评估
- 新进入市场的文化禁忌预研
- 节日/热点营销的文化适切性确认
- 视觉素材中的文化符号审核（手势/颜色/图腾/动物等）

## 执行步骤

### 步骤1：目标市场文化画像加载

1. 确认目标市场（国家/地区/文化圈）
2. 加载该市场的文化敏感度矩阵：

| 文化维度 | 高敏感示例 | 检查重点 |
|----------|-----------|----------|
| 宗教信仰 | 中东伊斯兰/印度多宗教/泰国佛教 | 宗教符号/饮食禁忌/斋月等 |
| 政治制度 | 各国国旗/领土争议/政治人物 | 地图标注/政治暗示/国旗使用 |
| 性别文化 | 中东保守/北欧开放 | 女性形象/性别角色/着装 |
| 颜色象征 | 白色(中日丧事)/绿色(伊斯兰) | 主色调文化含义 |
| 数字禁忌 | 4(中日韩)/13(西方)/17(意大利) | 价格/数量/日期 |
| 动物象征 | 猪(伊斯兰)/牛(印度教)/猫(埃及) | 视觉元素中的动物 |
| 手势肢体 | OK手势(巴西冒犯)/左手(中东不洁) | 视觉中的肢体语言 |
| 饮食文化 | 清真/素食/酒精禁忌 | 食品相关内容 |

### 步骤2：内容逐项文化扫描

**文本内容检查：**
1. 扫描是否包含宗教敏感词汇/引用
2. 检查比喻/修辞是否有文化歧义
3. 确认幽默/玩笑是否跨文化可接受
4. 验证称谓/敬语使用是否得体
5. 检查节日/日期引用是否适用于目标市场

**视觉内容检查：**
1. 人物形象审查（着装/肤色/性别表现/年龄）
2. 颜色使用文化含义验证
3. 手势/肢体语言文化合适性
4. 背景元素中的文化符号（建筑/文字/旗帜）
5. 食物/饮品元素的文化接受度
6. 动物元素的文化象征验证

**视频/音频内容检查：**
1. 背景音乐文化适切性（宗教音乐/民族音乐使用风险）
2. 配音语气/节奏的文化适配
3. 场景/情节是否触犯文化禁忌
4. 视觉符号的动态展示是否有文化风险

### 步骤3：风险等级评估

对发现的潜在文化问题进行风险分级：

| 风险等级 | 定义 | 处理方式 |
|----------|------|----------|
| 🔴 严重 | 触犯宗教/政治禁忌，可能引发严重公关危机 | 立即拦截，不允许发布，须重新创作 |
| 🟡 中等 | 文化不适当但不构成冒犯，影响品牌形象 | 须修改后才能发布，提供修改建议 |
| 🟢 轻微 | 非最佳实践但无风险，可优化提升 | 建议修改，不阻断发布流程 |
| ⚪ 通过 | 无文化风险 | 放行 |

### 步骤4：输出文化合规报告

```json
{
  "targetMarket": "目标市场",
  "checkDate": "检查日期",
  "overallRisk": "high|medium|low|pass",
  "culturalIssues": [
    {
      "location": "问题位置（文案第X行/图片XX区域）",
      "issue": "问题描述",
      "culturalContext": "文化背景解释（为什么这是问题）",
      "riskLevel": "severe|moderate|minor",
      "suggestion": "修改建议",
      "reference": "参考案例/文化规则来源"
    }
  ],
  "checklist": {
    "religiousSensitivity": "pass|flag",
    "politicalSensitivity": "pass|flag",
    "genderCulture": "pass|flag",
    "colorSymbolism": "pass|flag",
    "numberTaboo": "pass|flag",
    "animalSymbolism": "pass|flag",
    "gestureBodyLanguage": "pass|flag",
    "foodCulture": "pass|flag"
  },
  "verdict": "通过|需修改|拦截"
}
```

### 步骤5：案例库更新

1. 将本次发现的文化问题归档到案例库
2. 总结该市场的常见文化风险点
3. 更新文化敏感度矩阵（如有新发现）
4. 反馈给创作流程（避免同类问题重复出现）

## 输入/输出规格

### 输入
- 本地化后的内容（文案/图像/视频）
- 目标市场信息
- 内容发布渠道和受众群体
- 历史文化问题案例库（参考）

### 输出
- 文化合规检查报告（JSON格式）
- 风险标注（问题位置+严重程度+修改建议）
- 文化合规审核结论（通过/需修改/拦截）
- 文化背景说明（教育性输出，帮助团队理解禁忌原因）

## 最佳实践

1. **宁可过度谨慎**：文化冲突的公关成本远高于修改成本，有疑问就标注
2. **不依赖刻板印象**：文化理解要基于学术研究和本地反馈，不凭经验臆断
3. **年轻一代差异**：同一市场不同年龄层的文化敏感度不同，须考虑目标受众年龄
4. **时效性关注**：政治局势/社会事件可能临时改变某些元素的敏感度
5. **本地专家兜底**：AI检查发现的问题交由本地文化顾问确认，避免过度解读

## 约束条件

- 🔴严重风险必须拦截，不允许任何条件下放行
- 🟡中等风险须修改确认后才能放行
- 文化合规检查不可跳过（即使是成熟市场）
- 文化判断不确定时必须标注为中等风险，交由人工确认
- 政治相关内容（地图/国旗/领土表述）须严格遵循目标市场法律要求
