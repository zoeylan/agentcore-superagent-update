# 安全合规检查（Safety Compliance Check）

## 技能概述

本技能负责对生成素材进行内容安全和法规合规检查，确保所有交付素材不含违规、侵权或敏感内容。这是素材交付前的最后安全防线，直接关系到品牌声誉和法律风险。在中国市场环境下，内容安全合规是刚性要求，不允许任何妥协。

## 适用场景

- 每张生成素材交付前的安全扫描
- 批量素材的快速安全筛查
- 涉及人物/品牌/地理标识等敏感元素的深度审核
- 不同目标市场（国内/海外）的差异化合规检查

## 执行步骤

### Step 1: 政治敏感性检查

必查项目：
- **国家标识**：国旗、国徽、党旗的使用是否符合规定
- **地图问题**：涉及中国地图是否完整（台湾/南海诸岛/藏南）
- **领导人形象**：是否包含可识别的政治人物形象
- **政治符号**：是否包含任何政治敏感符号或标语
- **历史事件**：是否涉及敏感历史事件的不当表达
- **民族宗教**：是否存在民族歧视或宗教不当内容

### Step 2: 色情暴力检查

分级标准：
- **色情内容**：裸露程度（无/轻微/中度/重度）、性暗示程度
- **暴力内容**：血腥程度、武器展示、暴力行为描绘
- **恐怖内容**：恐怖元素、惊悚画面、令人不适的视觉
- **自残/自杀暗示**：任何引导或美化自残行为的内容

判定标准：
- 中国境内投放：按网信办标准执行，零容忍
- 海外投放：按目标市场分级制度执行

### Step 3: 知识产权检查

| 检查项 | 风险等级 | 处理方式 |
|--------|----------|----------|
| 包含可识别品牌商标 | 高 | 直接拒绝 |
| 包含知名IP形象特征 | 高 | 直接拒绝 |
| 风格高度近似某位创作者 | 中 | 标记风险，建议人工确认 |
| 使用了受版权保护的建筑外观 | 中 | 部分地区有限制，标记 |
| 包含名人肖像特征 | 高 | 直接拒绝（除非有授权） |
| 包含可识别的音乐/艺术作品 | 高 | 直接拒绝 |

### Step 4: 广告法合规检查（如素材含文字）

《广告法》常见违规：
- 绝对化用语："最"/"第一"/"唯一"/"国家级"/"全球首创"
- 虚假承诺："保证效果"/"无效退款"/"立竿见影"
- 特殊行业限制：医疗/药品/金融/教育类素材的特殊规定
- 对比广告：贬低竞品或未经许可使用竞品信息
- 使用国家机关名义：暗示政府背书

### Step 5: 个人信息保护检查

- AI生成的人脸是否可能被误认为真实特定个人
- 是否包含可识别的个人信息（姓名/电话/地址/车牌等）
- 虚拟试穿场景中使用的模特形象是否有授权
- 生成图片中是否意外出现了训练数据中的真实人物特征

### Step 6: 风险判定与输出

```json
{
  "asset_id": "ASSET-001",
  "safety_verdict": "pass",
  "risk_level": "low",
  "checks": {
    "political_sensitivity": { "pass": true, "notes": "" },
    "pornography_violence": { "pass": true, "notes": "" },
    "intellectual_property": { "pass": true, "notes": "" },
    "advertising_law": { "pass": true, "notes": "不含文字，不适用" },
    "personal_information": { "pass": true, "notes": "" }
  },
  "flags": [],
  "requires_human_review": false,
  "confidence": 0.95
}
```

风险等级定义：
- **低风险**（置信度>0.95）：自动放行
- **中风险**（置信度0.7-0.95）：标记具体风险点，建议人工复核
- **高风险**（置信度<0.7或触发红线）：直接拒绝，上报审核系统

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| asset_file | file | 是 | 待检查的素材文件 |
| target_market | string | 是 | 目标投放市场（CN/US/EU/Global） |
| content_category | string | 是 | 内容类别（电商/广告/社交/教育） |
| contains_text | boolean | 是 | 是否包含文字内容 |
| contains_person | boolean | 是 | 是否包含人物形象 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| safety_report | object | 安全检查报告 |
| verdict | enum | 判定(pass/flag_for_review/reject) |
| risk_level | enum | 风险等级(low/medium/high) |
| violation_details | array | 违规详情（如有） |
| confidence | number | 判定置信度 |

## 最佳实践

1. **宁严勿松**：安全检查有疑虑时倾向于标记人工复核，而非放行
2. **上下文考虑**：同一元素在不同上下文中风险不同（医学教育 vs 商业广告）
3. **持续更新**：关注监管政策变化和社会热点事件，及时更新检查规则
4. **全链路日志**：每次检查结果必须完整记录，支持180天内追溯
5. **批量模式提效**：大批量同类素材可采用抽检+全量快速扫描的组合策略

## 常见陷阱

- AI生成的"新面孔"可能恰好像某位名人（需要进行相似度检测）
- 忽略了图片中细微的文字内容（如街景中的招牌/T恤上的文字）
- 不同地区的合规标准差异（欧洲GDPR更严格/中东宗教禁忌不同）
- 将"风格借鉴"和"抄袭侵权"的边界判断过于简单化
- 节假日/纪念日期间的特殊敏感性未考虑（如特定日期不宜发布某类内容）
