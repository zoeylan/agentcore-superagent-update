# 质量评分评估（Quality Scoring Assessment）

## 技能概述

本技能负责对AI生成的素材进行系统化的多维度质量评估，输出标准化的质量评分和判定结论。评分结果直接驱动流程决策：通过→交付，不通过→重生成，多次不通过→人工介入。是质量闭环机制的核心执行能力。

## 适用场景

- 每张/每批生成素材的质量验收
- 重生成后的质量对比评估
- 提示词模板上线前的样本质量验证
- 周期性质量基线校准

## 执行步骤

### Step 1: 技术质量评估（满分10分）

评估维度及评分标准：

| 维度 | 10分 | 7分 | 4分 | 1分 |
|------|------|------|------|------|
| 分辨率/清晰度 | 锐利清晰，细节丰富 | 整体清晰，局部略软 | 明显模糊区域 | 整体模糊不可用 |
| AI伪影 | 无可见伪影 | 极轻微伪影不影响使用 | 明显伪影但主体完好 | 严重伪影变形 |
| 色彩准确性 | 色彩自然准确 | 轻微色偏可接受 | 明显色彩异常 | 色彩严重失真 |
| 光照一致性 | 光照自然协调 | 轻微不一致 | 光照方向矛盾 | 完全不合理的光照 |
| 构图完整性 | 主体完整无截断 | 轻微裁切可接受 | 重要元素被截断 | 主体不完整 |

常见AI伪影检查清单：
- [ ] 人物手指数量正确（5指）
- [ ] 人物面部无扭曲变形
- [ ] 文字无乱码（如有文字渲染）
- [ ] 无重复纹理/图案
- [ ] 无接缝痕迹（合成类素材）
- [ ] 无"多余肢体"问题
- [ ] 背景无逻辑错误（如悬浮物体/错误透视）

### Step 2: 创意匹配度评估（满分10分）

| 维度 | 10分 | 7分 | 4分 | 1分 |
|------|------|------|------|------|
| 需求契合 | 完全匹配需求描述 | 大方向正确细节有偏差 | 方向正确但多处偏离 | 完全不符合需求 |
| 风格一致 | 精准匹配目标风格 | 风格接近可微调 | 风格偏差较大 | 风格完全错误 |
| 美学评分 | 专业级视觉效果 | 良好的视觉质感 | 中规中矩 | 视觉效果差 |
| 品牌符合 | 完全符合品牌规范 | 基本符合小处偏差 | 多处违反品牌规范 | 与品牌调性冲突 |

### Step 3: 安全合规评估（通过/不通过）

红线检查（任一项触发直接拒绝）：
- [ ] 无涉政敏感内容（领导人形象/国旗国徽不当使用/地图问题）
- [ ] 无色情暴力内容（裸露/血腥/恐怖元素）
- [ ] 无侵权元素（可识别商标/知名IP形象/名人肖像）
- [ ] 无个人隐私泄露（可识别真人面部/个人信息）
- [ ] 无广告法违规（"最"/"第一"等极限词出现在图片文字中）
- [ ] 无其他违反法律法规的内容

### Step 4: 综合评分与判定

```
综合评分 = 技术质量分 × 0.4 + 创意匹配度分 × 0.6

判定逻辑：
- 综合评分 ≥ 7.0 且 安全合规通过 → 判定：通过
- 综合评分 < 7.0 且 重生成次数 < 3 → 判定：退回重生成
- 综合评分 < 7.0 且 重生成次数 ≥ 3 → 判定：升级人工介入
- 安全合规不通过 → 判定：安全拒绝（无论评分多高）

虚拟试穿额外判定：
- 人物一致性评分 < 8.0 → 退回重生成（即使综合分达标）
```

### Step 5: 评估报告输出

```json
{
  "asset_id": "ASSET-001",
  "scores": {
    "technical_quality": 8.2,
    "creative_match": 7.5,
    "composite_score": 7.78,
    "safety_pass": true,
    "person_consistency": null
  },
  "verdict": "pass",
  "details": {
    "technical_notes": "轻微背景噪点，不影响使用",
    "creative_notes": "色调比需求描述稍暖，可接受范围",
    "safety_notes": "无安全风险"
  },
  "improvement_suggestions": [
    "下次可在提示词中增加色温控制(cool tone)"
  ],
  "retry_count": 0
}
```

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| generated_asset | file | 是 | 待评估的生成素材 |
| original_requirement | object | 是 | 原始需求文档（用于匹配度评估） |
| brand_guidelines | object | 否 | 品牌规范（用于品牌符合度） |
| retry_count | number | 是 | 当前已重生成次数 |
| task_type | string | 是 | 任务类型（用于选择评估维度） |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| quality_report | object | 完整质量评估报告 |
| verdict | enum | 判定结论(pass/regenerate/escalate/reject) |
| score_breakdown | object | 分项评分明细 |
| improvement_suggestions | array | 改进建议 |

## 最佳实践

1. **评分一致性**：同一批次的评估标准必须一致，不能前紧后松
2. **客观依据**：每个扣分项必须有具体的可观察依据
3. **改进建议可执行**：不要只说"质量不好"，要说"手指变形→建议负向提示词增加bad hands"
4. **安全优先**：安全红线是绝对的，不因时间压力而放松
5. **记录所有评估**：即使通过的素材也保存评估记录，用于后续分析

## 常见陷阱

- 审美疲劳导致评分标准漂移（连续评估大量素材后标准会松动）
- 对"差不多"的素材过于宽容（7分线是刚性标准）
- 忽略了上下文一致性（单图OK但系列图风格不统一）
- 安全检查流于形式（不仔细看就点通过）
- 未考虑最终使用场景（放大到户外广告牌的话清晰度是否够）
