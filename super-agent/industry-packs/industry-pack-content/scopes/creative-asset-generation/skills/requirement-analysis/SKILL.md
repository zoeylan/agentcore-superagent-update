# 需求解析与结构化（Requirement Analysis）

## 技能概述

本技能负责将用户的原始创意需求（文字描述、参考图片、品牌Brief文档等）解析为结构化的需求文档，确保下游生产环节能够精准执行。这是创意素材生成流程的第一步，直接影响后续所有环节的准确性和效率。

## 适用场景

- 用户提交新的素材生成需求（文字描述/brief/参考图）
- 营销campaign的创意需求接入
- 电商大促的批量素材需求拆解
- 品牌方提供的模糊创意方向需要结构化

## 执行步骤

### Step 1: 需求完整度检查
对接收到的原始需求进行四要素验证：
- **目标受众**：是否明确了目标人群（年龄/性别/地域/消费层级）
- **使用场景**：是否说明了素材的使用渠道和场景（电商主图/社交分享/户外广告等）
- **品牌规范**：是否提供了或引用了品牌VI规范（色彩体系/字体/logo使用规则）
- **输出规格**：是否明确了交付格式（尺寸/分辨率/文件格式/数量）

缺失超过1项时生成澄清问题清单，暂停流程等待补充。

### Step 2: 意图理解与提取
从原始描述中提取核心创意要素：
- 视觉方向（写实/插画/3D/扁平/国潮等风格）
- 情感调性（高端/活力/温馨/科技感/自然等）
- 核心信息（必须传达的关键卖点/信息层级）
- 参考锚定（用户提供的参考图分析——提取风格/构图/色调特征）

### Step 3: 约束条件识别
识别并记录所有硬性约束：
- 品牌红线（禁用颜色/禁止元素/竞品规避）
- 法规约束（广告法限制词/特殊行业审批要求）
- 技术约束（平台尺寸要求/文件大小限制）
- 时间约束（交付截止时间/是否支持多轮迭代）

### Step 4: 结构化输出
生成标准化需求分析文档（JSON格式）：

```json
{
  "requirement_id": "REQ-20240101-001",
  "intent_summary": "意图摘要（一句话）",
  "target_audience": { "age": "", "gender": "", "region": "", "tier": "" },
  "usage_context": { "channel": "", "placement": "", "purpose": "" },
  "creative_direction": {
    "style": "",
    "mood": "",
    "color_palette": [],
    "reference_analysis": ""
  },
  "brand_constraints": { "vi_ref": "", "prohibitions": [] },
  "output_specs": { "sizes": [], "format": "", "quantity": 0, "resolution": "" },
  "deadline": "",
  "priority": "high/medium/low",
  "requires_human_confirmation": true/false
}
```

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| raw_description | string | 是 | 用户原始文字描述 |
| reference_images | array[url] | 否 | 参考图片链接 |
| brand_brief | document | 否 | 品牌Brief文档 |
| brand_id | string | 否 | 品牌资产库ID引用 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| structured_requirement | JSON | 结构化需求文档 |
| clarification_questions | array[string] | 澄清问题列表（如需要） |
| completeness_score | number (0-100) | 需求完整度评分 |
| risk_flags | array[string] | 风险标记（如涉及敏感题材等） |

## 最佳实践

1. **宁可多问不要假设**：当关键信息缺失时，生成具体的澄清问题比自行假设更安全
2. **参考图分析要具体**：不要只说"参考这张图的风格"，要具体拆解为色调/构图/元素/光影等维度
3. **约束条件显式记录**：隐含的品牌红线和法规限制必须被显式提取并文档化
4. **优先级明确化**：当有多个子需求时，必须明确优先级排序以指导生产顺序
5. **关键创意决策标记**：品牌主KV/新品首发等高优先级需求必须标记`requires_human_confirmation: true`

## 常见陷阱

- 用户说"好看的"——需要追问具体的美学偏好（简约/繁复/对比强烈等）
- 用户说"参考竞品"——需要确认是参考风格方向还是要差异化处理
- 用户只给了一个尺寸——需要确认是否需要多尺寸适配（通常campaign需要）
- 忽略了平台内容安全政策——某些创意方向可能在特定平台不合规
