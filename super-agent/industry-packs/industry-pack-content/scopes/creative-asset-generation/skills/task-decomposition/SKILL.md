# 任务拆解与生产计划（Task Decomposition）

## 技能概述

本技能负责将结构化的创意需求拆解为可独立执行的子任务，并制定完整的生产计划。核心挑战在于合理的粒度划分——太粗则执行困难，太细则管理成本高。同时需要考虑任务间的依赖关系、并发可能性和资源约束。

## 适用场景

- Campaign级需求的子任务拆解（主KV + 延展图 + 多尺寸适配）
- 电商大促的批量商品图生产计划制定
- 多渠道素材的并行生产编排
- 复杂合成需求的工序分解（如：先生成背景→再生成主体→最后合成）

## 执行步骤

### Step 1: 任务类型识别
根据需求分析结果判断任务类型：
- **单图任务**：单一素材生成，无需拆解
- **系列任务**：同一风格多个变体（如同一主题的5张轮播图）
- **Campaign任务**：主KV + 衍生系列 + 多尺寸适配
- **批量任务**：大量同类素材（如100个SKU的商品主图）
- **合成任务**：需要多步骤组合的复合素材

### Step 2: 子任务划分
按照以下原则拆解：
- 每个子任务有明确的输入和输出
- 每个子任务可独立分配给提示词工程师处理
- 有依赖关系的任务标注前置任务ID
- 标注哪些任务可以并行执行

### Step 3: 生产计划制定
为每个子任务制定执行计划：

```json
{
  "task_id": "TASK-001",
  "parent_requirement": "REQ-20240101-001",
  "description": "生成品牌主KV — 夏季大促主视觉",
  "task_type": "single_image",
  "priority": 1,
  "dependencies": [],
  "parallel_group": "A",
  "recommended_model": "flux-pro",
  "model_reason": "写实风格+高品质要求，Flux Pro在人物+场景合成上表现最优",
  "output_spec": { "size": "1920x1080", "format": "PNG", "dpi": 300 },
  "acceptance_criteria": [
    "质量评分≥8/10",
    "品牌色彩使用正确",
    "主体人物清晰无变形"
  ],
  "estimated_time": "5min",
  "estimated_cost": "$0.12",
  "max_retries": 3,
  "requires_human_confirm": true
}
```

### Step 4: 风险评估与预案
- 识别高风险子任务（新风格尝试/复杂合成/虚拟试穿等）
- 为高风险任务准备备选方案（备用模型/降级策略）
- 评估整体批次失败风险并设定监控阈值

### Step 5: 时间线编排
- 关键路径分析（哪些任务决定了整体交付时间）
- 并行任务分组（同一group内的任务可同时执行）
- 缓冲时间预留（重生成+人工确认的时间余量）

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| structured_requirement | JSON | 是 | 结构化需求文档 |
| available_models | array | 是 | 当前可用模型列表及配额 |
| brand_assets | object | 否 | 品牌资产库引用 |
| deadline | datetime | 否 | 交付截止时间 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| task_list | array[TaskItem] | 子任务清单 |
| production_plan | object | 生产计划（时间线+资源分配） |
| dependency_graph | object | 任务依赖关系图 |
| risk_assessment | object | 风险评估与预案 |
| total_estimated_cost | number | 总预估成本 |

## 最佳实践

1. **主KV优先原则**：Campaign任务中，主KV永远是P0优先级，延展图依赖主KV的风格锚定
2. **批量任务分批执行**：100+素材不要一次性全部投入生产，先执行10%作为样本验证质量，再批量铺开
3. **并发度控制**：考虑模型API的rate limit，合理设置并发组大小（通常5-10个并发）
4. **预留重生成预算**：时间和成本估算中预留30%余量用于重生成
5. **人工确认节点前置**：需要人工确认的子任务尽早安排，避免阻塞后续流程

## 常见陷阱

- 忽略多尺寸适配的工作量（一张主图适配到10个尺寸并非简单裁切）
- 低估虚拟试穿任务的复杂度（需要多次迭代保证一致性）
- 未考虑模型配额限制导致实际执行时被节流
- 依赖关系遗漏导致并行执行时出现数据不一致
