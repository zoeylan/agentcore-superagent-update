# SMART目标审核 (SMART Goal Review)

## 技能概述

运用SMART原则对员工设定的绩效目标进行质量审核，确保每个目标都具备具体性(Specific)、可衡量性(Measurable)、可实现性(Achievable)、相关性(Relevant)和时限性(Time-bound)五项基本要素。审核不合格的目标需退回修改并提供具体改进建议。

## 适用场景

- 考核周期初期的目标设定审核
- OKR季度回顾时的目标调整审核
- PIP改进目标的质量把关
- 试用期目标从绩效管理视角的一致性审核

## 执行步骤

### 第一步：Specific（具体性）审核
1. 检查目标描述是否清晰无歧义（能否让不了解业务的人也理解要做什么）
2. 确认目标范围边界明确（做到什么程度算完成、什么不在范围内）
3. 识别模糊用词：如"提升"、"优化"、"加强"等需要进一步量化
4. **合格标准**：一个目标只对应一件核心事项，描述中无需追问"具体指什么"

### 第二步：Measurable（可衡量性）审核
1. 检查是否有明确的量化指标（数字型：增长率、完成数量、达成金额）
2. 若为定性目标，检查是否有里程碑或验收标准（如：系统上线、通过验收评审）
3. 确认数据来源和计算方式已明确（从哪个系统取数、如何计算）
4. **合格标准**：任何人对"是否完成"不存在争议

### 第三步：Achievable（可实现性）审核
1. 参考历史数据评估目标难度（去年同期数据、行业平均水平）
2. 评估资源约束条件（人力、预算、系统支持是否足够）
3. 检查目标之间是否存在冲突（如同时要求降本和提质）
4. **合格标准**：基于历史数据判断，目标达成概率在60-80%区间（太低挫败、太高无激励）

### 第四步：Relevant（相关性）审核
1. 验证目标与直属上级/部门目标的对齐关系
2. 计算对齐度：个人目标中≥80%可追溯到上级目标
3. 识别"自嗨目标"：员工感兴趣但对组织贡献不明确的目标需标注
4. **合格标准**：每个目标都能回答"这如何帮助团队/部门实现目标"

### 第五步：Time-bound（时限性）审核
1. 检查是否有明确的截止时间或时间节点
2. 长期目标是否设有阶段性里程碑（按月或按季check-in点）
3. 时间安排是否合理（非平均分配，考虑业务节奏和依赖关系）
4. **合格标准**：能画出时间轴，标注各节点的预期交付物

### 第六步：综合评判与反馈
1. 对五项维度分别给出通过/待改进的判断
2. 整体目标集的平衡性检查（目标数量、权重分布、难度梯度）
3. 不合格项提供具体改进建议和参考示例
4. 产出审核报告并跟踪修改直至通过

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 待审核的目标文档 | 员工提交的KPI/OKR表 | 是 |
| 上级目标/部门目标 | 用于验证对齐度 | 是 |
| 历史绩效数据 | 用于校准可实现性 | 是 |
| 岗位职责说明 | 用于验证相关性 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| SMART审核评分卡 | 表格 | 每个目标的五维度通过/不通过及原因 |
| 改进建议清单 | 列表 | 针对不合格项的具体修改建议 |
| 优秀示例参考 | 文本 | 同类岗位的高质量目标示例 |
| 审核结论 | 通过/退回修改 | 整体判断及下一步行动 |

## 最佳实践

1. **先看整体再看细节**：先评估目标集的完整性和平衡性，再逐条审核SMART
2. **提供正面示例**：退回修改时附上同岗位优秀目标示例，降低员工修改成本
3. **区分硬伤和优化**：必须修改的（如完全无法衡量）vs 建议优化的（如可以更具体），分级反馈
4. **考虑岗位特殊性**：研发类目标允许更多探索性，销售类目标必须严格量化
5. **不越界设定目标**：审核者只审核质量，不替代员工和上级决定目标内容

## 约束条件

- 审核反馈需在收到目标后2个工作日内完成
- 退回修改最多允许2轮，超过2轮需上级介入协调
- 审核标准公开透明，员工可查阅审核checklist
- 不得以审核为由否定管理者已确认的目标方向（只审核表述质量，不审核业务决策）
