# 目标分解 (Goal Decomposition)

## 技能概述

将组织战略目标逐层分解为部门目标、团队目标和个人目标的专业技能。确保目标体系纵向对齐（战略→部门→团队→个人）和横向协同（跨部门依赖识别），形成完整的目标金字塔结构。

## 适用场景

- 年度/季度考核周期启动时的目标分解
- 组织战略调整后的目标重新对齐
- 新设部门或团队的首次目标建立
- 组织架构调整后的目标重映射

## 执行步骤

### 第一步：获取组织战略目标
1. 收集CEO/管理层确定的年度战略目标（通常3-5个核心方向）
2. 理解每个战略目标的业务背景、优先级和资源约束
3. 识别战略目标之间的相互依赖关系

### 第二步：部门级目标分解
1. 与各部门负责人一对一沟通，明确该部门对各战略目标的贡献点
2. 将战略目标转化为部门级可执行目标（每个部门3-7个核心目标）
3. 确认部门间的目标依赖关系（如"产品上线"依赖"技术开发"和"运营准备"）
4. 检查所有部门目标汇总后是否完整覆盖战略目标（无遗漏）

### 第三步：团队/个人级目标分解
1. 部门目标拆解为团队目标（每个团队3-5个核心目标）
2. 团队目标进一步分配至个人（每人3-5个KPI或1-3个OKR）
3. 确保个人目标的权重分配合理（核心职责目标权重≥60%）
4. 标注需要跨团队协作完成的目标并明确协作方和接口

### 第四步：对齐度验证
1. 自下而上回溯验证：每个个人目标均可追溯到部门目标和战略目标
2. 计算对齐度指标：个人目标与部门目标的对齐度≥80%
3. 识别孤立目标（无法对齐到上级目标的个人目标需要删除或归类）
4. 产出目标对齐地图（可视化展示目标层级关系）

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 组织战略目标文档 | CEO/管理层确定的年度/季度战略目标 | 是 |
| 组织架构图 | 包含部门、团队和汇报关系 | 是 |
| 历史绩效数据 | 上周期各部门/个人目标达成情况 | 是 |
| 岗位职责说明书 | 各岗位核心职责和KPI参考 | 否 |
| 业务规划文档 | 各部门年度业务计划 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 目标分解树 | 结构化文档/表格 | 战略→部门→团队→个人的完整目标树 |
| 目标对齐地图 | 可视化图表 | 展示各层级目标的对应关系 |
| 跨部门依赖矩阵 | 矩阵表格 | 标注需要跨部门协作的目标及责任方 |
| 目标质量报告 | 文本报告 | 标注未达SMART标准或对齐度不足的目标 |

## 最佳实践

1. **先纵后横**：先完成纵向分解确保战略覆盖，再做横向协同消除重叠和遗漏
2. **权重合理**：核心业务目标权重≥60%，发展类目标≤20%，协作类目标≤20%
3. **挑战度校准**：参考历史达成数据的75-85分位值设定目标值，兼顾激励性和可达性
4. **避免目标膨胀**：个人目标不超过5个KPI或3个OKR，聚焦核心价值创造
5. **双向沟通**：目标分解不是自上而下的命令，需要上下级充分讨论达成共识

## 约束条件

- 目标分解需在考核周期开始后10个工作日内完成
- 每个个人目标必须能追溯到至少一个上级目标
- 对齐度低于80%的目标方案需退回重新设计
- 不得设定与岗位职责完全无关的目标（如给技术人员设定销售指标）
