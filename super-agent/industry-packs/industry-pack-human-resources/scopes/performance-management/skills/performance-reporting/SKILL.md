# 绩效分析报告 (Performance Reporting)

## 技能概述

产出结构化、可视化的绩效分析报告，将数据洞察转化为管理层可理解和可行动的内容。报告类型包括季度绩效简报、年度绩效白皮书、专题分析报告（如离职预测、人才九宫格）和管理者仪表盘。报告的核心价值不在于呈现数据，而在于讲述"数据背后的故事"并推动行动。

## 适用场景

- 季度OKR评估后的进度报告
- 年度绩效评估完成后的综合分析报告
- 人才盘点会的绩效数据支撑报告
- 薪酬调整/晋升评审前的绩效数据包
- HR管理层月度/季度例会的绩效模块汇报

## 执行步骤

### 第一步：明确报告目标和受众
1. 确认报告类型（定期报告/专题报告/决策支撑报告）
2. 识别核心受众（CEO/HR总监/部门负责人/HR BP）
3. 明确受众关注的核心问题（"他们想知道什么"）
4. 确定报告深度和篇幅（高管看1页摘要，HR看详细分析）

### 第二步：构建报告结构
1. **执行摘要**（1页）：3-5个核心发现+关键数据+行动建议
2. **关键指标看板**：本期vs上期vs目标的对比仪表盘
3. **深度分析章节**：按维度展开的详细分析
4. **异常与风险**：需要关注的异常模式和潜在风险
5. **行动建议**：基于分析的具体改进建议和下一步计划
6. **附录**：详细数据表、方法论说明

### 第三步：数据可视化设计
1. 选择合适的图表类型：
   - 分布分析→柱状图/饼图
   - 趋势分析→折线图
   - 对比分析→分组柱状图/雷达图
   - 矩阵分析→九宫格/热力图
   - 流程/漏斗→漏斗图/桑基图
2. 确保图表的数据墨比（data-ink ratio）最大化，去除装饰性元素
3. 添加清晰的标题、坐标轴标签和数据标注
4. 使用一致的颜色编码（如红色=风险，绿色=健康）

### 第四步：洞察提炼和行动建议
1. 每个分析章节提炼1-2个核心洞察（insight）
2. 洞察遵循"So What"原则：这个数据意味着什么业务影响
3. 行动建议遵循"Now What"原则：基于洞察应该采取什么行动
4. 区分短期行动（本季度可执行）和长期建议（需要规划）

### 第五步：报告审核和分发
1. 数据准确性核验（关键数字双重检查）
2. HR总监审核报告结论和措辞（避免敏感表述）
3. 按保密级别确定分发范围（全员版/管理层版/HR内部版）
4. 选择合适的呈现方式（PPT演示/PDF发送/系统看板）

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 分析完成的数据集 | 已完成分析处理的绩效数据 | 是 |
| 报告目标和受众 | 明确为谁写、回答什么问题 | 是 |
| 报告模板 | 公司标准的报告格式模板 | 否 |
| 历史报告参考 | 过往同类报告作为参考 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 季度绩效简报 | PPT（5-10页） | 高管快速了解绩效全貌 |
| 年度绩效白皮书 | PDF（20-30页） | 全面深度的年度分析 |
| 管理者仪表盘 | 在线看板 | 实时自助查询的数据看板 |
| 决策数据包 | Excel+PPT | 为特定决策场景准备的数据 |

## 最佳实践

1. **结论前置**：最重要的信息放在最前面，忙碌的管理者只看第一页
2. **数据讲故事**：不是罗列数据，而是用数据讲述一个有逻辑的故事
3. **可比可追踪**：保持报告结构和指标的周期一致性，方便纵向对比
4. **行动导向**：每份报告必须有"下一步行动"部分，避免"看完就放下"
5. **视觉简洁**：少即是多，一页一个核心信息，避免信息过载

## 约束条件

- 季度报告在评估完成后10个工作日内产出
- 年度白皮书在年度评估完成后20个工作日内产出
- 报告中的个人数据必须脱敏处理（公开版本）
- 报告结论和对外发布内容需HR总监审批
- 报告版本需要存档管理，标注版本号和修改记录
