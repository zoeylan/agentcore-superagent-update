# 强制分布执行 (Forced Distribution Execution)

## 技能概述

执行企业年度绩效强制分布政策（S10%/A20%/B50%/C15%/D5%），包括配额计算、分配规则制定、争议处理和例外审批。强制分布是中国企业普遍采用的绩效区分机制，核心目的是避免"大锅饭"式的评估虚高，确保绩效结果具有区分度和激励性。

## 适用场景

- 年度绩效评估结果的强制分布执行
- 部门间配额分配计算
- 分布异常情况的识别和处理
- 特殊情况的例外申请审批

## 执行步骤

### 第一步：配额计算
1. 获取各部门参评人数（排除入职未满考核周期的新员工）
2. 按比例计算各部门各等级配额（四舍五入，确保总数=参评总人数）
3. 处理小团队配额不足整数的情况（<10人的团队可合并上级部门统一分配）
4. 发布配额分配表给各部门负责人

### 第二步：初步分布检查
1. 收集各部门管理者的初步评级方案
2. 对比配额计算偏差：实际分布vs目标分布
3. 标注超标部门（如S实际15%>目标10%）和不足部门（如D实际0%<目标5%）
4. 生成分布偏差报告

### 第三步：偏差处理
1. **偏差≤2%**：在允许范围内，直接通过
2. **偏差2-5%**：部门需提供书面说明，HR BP审核后可通过
3. **偏差>5%**：必须上报HR总监审批，以下场景可例外批准：
   - 部门当年业务指标显著超额完成（如营收超目标30%以上）
   - 部门经历重大组织调整（合并/拆分/大规模人员变动）
   - 部门规模过小导致统计失真（<5人）

### 第四步：争议调解
1. 管理者不愿给D评级的处理：引导其识别相对最弱的表现者，提供"发展性反馈"的正向框架
2. 部门间"抢S配额"的处理：基于量化业绩数据进行跨部门对标
3. 员工对评级结果的申诉处理：受理申诉→核实事实→校准会复议→最终裁定

### 第五步：结果锁定
1. 所有部门确认最终分布后锁定结果
2. 生成全公司绩效分布总表
3. 与薪酬模块对接（各等级对应的年终奖系数和调薪比例）
4. 存档分布决策过程文件

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 各部门参评人数 | 实际参加考核的员工人数 | 是 |
| 强制分布比例 | S/A/B/C/D的目标百分比 | 是 |
| 管理者初步评级 | 各管理者提交的员工评级建议 | 是 |
| 部门业务达成数据 | 用于例外审批的业务参考 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 配额分配表 | 表格 | 各部门各等级的配额数 |
| 分布偏差报告 | 报告 | 实际vs目标的偏差分析 |
| 例外审批记录 | 文档 | 超标部门的审批理由和结果 |
| 最终分布确认表 | 数据表 | 全公司最终绩效分布 |

## 最佳实践

1. **提前沟通规则**：考核周期开始时就明确强制分布政策，避免年末突然执行引发抵触
2. **配额弹性**：允许±2%的弹性空间，不必追求绝对精确
3. **团队规模兜底**：<10人的小团队不单独执行强制分布，合并至更大单元
4. **数据为锚**：争议时以量化业绩数据为优先参考，减少主观博弈
5. **保护管理者**：向员工传达时强调"公司政策要求"而非"管理者主动打低分"

## 约束条件

- 强制分布比例为S(10%)A(20%)B(50%)C(15%)D(5%)，全公司统一标准
- 偏差>5%必须HR总监审批，不可自行通过
- 配额计算不含入职未满6个月的新员工
- 最终分布结果确认后不可再调整（除发现重大程序违规）
- 强制分布结果为高度保密信息，仅HR和直属管理者知悉
