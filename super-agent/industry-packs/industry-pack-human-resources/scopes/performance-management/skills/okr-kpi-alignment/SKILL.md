# OKR与KPI对齐 (OKR-KPI Alignment)

## 技能概述

确保OKR（目标与关键结果）和KPI（关键绩效指标）两套目标管理体系的逻辑一致性和互补性。OKR聚焦方向引领和关键突破（What & How），KPI量化底线要求和日常产出（How Much）。两者不是替代关系，而是互补协同——OKR牵引创新和突破，KPI守护基本盘和运营底线。

## 适用场景

- 企业同时采用OKR和KPI双体系时的协调设计
- 从纯KPI体系转型为OKR+KPI混合模式
- 季度OKR制定时确保与年度KPI的一致性
- 评估周期中OKR达成与KPI考核的权重平衡设计

## 执行步骤

### 第一步：明确两套体系的定位分工
1. OKR定位：聚焦3-5个季度战略突破方向，鼓励挑战性目标（70%完成率即为优秀）
2. KPI定位：覆盖岗位基本职责和常规产出，要求100%达成为及格线
3. 权重分配建议：OKR占总绩效权重30-40%，KPI占60-70%
4. 明确OKR的"挑战性"不应影响KPI的"底线性"

### 第二步：逻辑一致性检查
1. OKR的O（目标）应能映射到KPI的某个维度或是KPI的延伸升级
2. KR（关键结果）和KPI不应存在矛盾（如OKR追求用户体验创新，KPI却只考核成本控制）
3. 识别"假OKR"：实际是KPI换了个名字，没有突破性和挑战性
4. 识别"脱节OKR"：与岗位KPI完全无关，成为"业余爱好项目"

### 第三步：冲突协调
1. 时间冲突：OKR项目占用大量时间导致KPI日常工作受影响→调整资源或降低KPI权重
2. 方向冲突：OKR鼓励探索但KPI要求稳定产出→为OKR探索设定"安全边界"
3. 评估冲突：OKR失败但KPI优秀（或反之）→明确综合评估的计算逻辑

### 第四步：评估周期对齐
1. KPI通常年度设定、年度考核；OKR季度设定、季度评估
2. 季度OKR评估为过程性参考，不直接计入年终绩效等级
3. 年度绩效综合评估时，OKR达成情况作为加分项或综合判断维度
4. 制定清晰的"OKR→年度绩效"的转化规则

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 员工OKR设定 | 季度OKR（含O和KR） | 是 |
| 员工KPI清单 | 年度KPI指标及权重 | 是 |
| 部门OKR/KPI | 上级目标参照 | 是 |
| 双体系权重规则 | 公司规定的OKR/KPI权重分配 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 对齐分析报告 | 文档 | OKR与KPI的映射关系和一致性分析 |
| 冲突标注清单 | 表格 | 识别的矛盾点及解决建议 |
| 综合评估规则 | 规则文档 | OKR与KPI结合的年度绩效计算逻辑 |
| 优化建议 | 文本 | 对OKR或KPI的调整建议 |

## 最佳实践

1. **KPI是地板，OKR是天花板**：KPI确保基本产出不掉线，OKR推动突破和创新
2. **避免双重考核**：同一件事不应同时出现在OKR和KPI中（避免重复计算）
3. **OKR失败不惩罚**：OKR鼓励挑战性，失败是学习而非过错；KPI未达标则影响绩效评级
4. **季度灵活、年度稳定**：OKR允许季度间调整方向，KPI全年保持稳定（除重大变化）
5. **岗位差异化**：创新型岗位OKR权重可适当提高，运营型岗位KPI权重为主

## 约束条件

- OKR数量控制在每人1-3个O，每个O对应2-5个KR
- KPI数量控制在每人3-5个核心指标
- OKR与KPI的总目标数不超过8个（防止注意力分散）
- 对齐检查需在目标设定审核阶段同步完成，不单独增加周期
