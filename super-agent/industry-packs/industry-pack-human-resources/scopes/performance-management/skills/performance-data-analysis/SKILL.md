# 绩效数据分析 (Performance Data Analysis)

## 技能概述

对组织绩效数据进行多维度、多层次的深度分析，挖掘绩效表现背后的规律和趋势，为管理决策提供数据支撑。分析覆盖组织、部门、管理者和个人四个层面，结合统计方法和HR业务知识产出有业务意义的洞察。

## 适用场景

- 年度/季度绩效评估完成后的数据分析
- 人才盘点的绩效维度数据支撑
- 薪酬调整和晋升决策的数据输入
- 管理者管理能力评估的数据参考
- 组织健康度诊断的绩效指标分析

## 执行步骤

### 第一步：数据准备
1. 从评估系统提取原始评估数据（评分、评级、评语关键词）
2. 关联员工主数据（岗位、职级、部门、司龄、入职渠道等）
3. 纵向关联历史绩效数据（3-5年趋势）
4. 数据清洗：处理缺失值、异常值、编码统一

### 第二步：组织层面分析
1. 全公司绩效分布统计（与目标分布的偏差分析）
2. 各周期绩效对比（是否存在通胀/紧缩趋势）
3. 绩效与业务指标的相关性（高绩效人群与业务增长的关系）
4. 绩效等级与离职率的交叉分析

### 第三步：部门层面分析
1. 部门间绩效分布对比（识别绩效虚高或虚低的部门）
2. 部门绩效与部门业务达成的一致性验证
3. 部门管理能力评估（下属绩效分布反映管理者管理水平）
4. 部门间人才流动与绩效变化的关联分析

### 第四步：管理者层面分析
1. 各管理者评分分布特征（是否存在系统性偏差）
2. 管理者评分与实际业务产出的一致性
3. 管理者团队绩效趋势（持续提升vs持续下滑）
4. 新晋管理者的评估校准建议

### 第五步：个人层面分析
1. 个人绩效历史轨迹（稳定型/上升型/下滑型/波动型）
2. 高潜人才识别（持续A+能力评估高分+发展意愿强）
3. 离职预警模型（绩效突降+司龄特征+市场热度=高离职风险）
4. 九宫格分布（绩效×潜力矩阵）

### 第六步：洞察输出
1. 提炼3-5个核心发现（有数据支撑的结论）
2. 每个发现配套行动建议（so what→now what）
3. 数据可视化呈现（选择最适合的图表类型）
4. 产出分析报告（管理层版简报+详细版报告）

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 绩效评估原始数据 | 本周期所有员工的评估数据 | 是 |
| 历史绩效数据 | 过往3-5个周期的评估数据 | 是 |
| 员工主数据 | 岗位、部门、职级、司龄等 | 是 |
| 业务指标数据 | 各部门业务完成情况 | 否 |
| 离职数据 | 历史离职人员及离职原因 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 绩效分析报告 | PPT/PDF | 管理层阅读的结构化分析报告 |
| 数据仪表盘 | 可视化看板 | 实时可查看的绩效数据看板 |
| 异常预警清单 | 列表 | 需要关注的异常模式和人员 |
| 决策支撑数据包 | 数据表 | 为薪酬/晋升/人才盘点准备的数据 |

## 最佳实践

1. **数据与业务结合**：纯数据无意义，必须结合业务背景解读
2. **趋势比快照重要**：单期数据可能有偶然性，关注多期趋势
3. **对比产生洞察**：同比（vs去年）、环比（vs上期）、横比（vs其他部门）
4. **行动导向**：每个分析发现都要回答"所以我们应该怎么做"
5. **保密分级**：组织级报告公开，部门级报告限部门负责人+HR，个人级仅HR+直属上级

## 约束条件

- 季度分析报告在评估完成后10个工作日内产出
- 年度绩效白皮书在年度评估完成后20个工作日内产出
- 个人层面分析报告为高度机密，严格限制访问权限
- 数据分析结论需经HR总监审核后方可对外发布
- 分析不做因果判断，只呈现相关性和趋势（因果判断由管理者决策）
