# 改进计划设计 (Improvement Plan Design)

## 技能概述

为不同绩效等级的员工设计差异化的后续行动方案：高绩效员工（S/A）制定发展加速计划和激励保留方案；中等绩效员工（B）制定能力提升和突破方向；低绩效员工（C/D）制定具体的改进措施和辅导方案。改进计划是绩效面谈的核心产出物，将评估结论转化为可执行的行动。

## 适用场景

- 绩效面谈后的个人发展计划（IDP）制定
- 高潜人才的加速发展方案设计
- C等级员工的关注辅导计划
- D等级员工的正式改进计划（非PIP级别，首次D的情况）

## 执行步骤

### 第一步：诊断分析
1. 回顾员工本周期绩效详情（各目标完成情况、360反馈关键词）
2. 分析绩效表现的根因：
   - 能力不足（知识/技能欠缺→培训解决）
   - 动力不足（态度/投入度问题→激励解决）
   - 资源不足（环境/工具/支持缺乏→环境调整）
   - 匹配不当（人岗不匹配→调岗评估）
3. 综合历史绩效轨迹判断是偶发问题还是持续性问题

### 第二步：S/A等级——发展加速方案
1. **职业发展路径**：明确下一步晋升方向和时间预期
2. **能力拓展计划**：识别从当前岗位到目标岗位的能力差距，安排针对性培训
3. **挑战性任务**：安排更高难度的项目或跨部门历练机会
4. **激励保留**：薪酬调整建议、股权/期权建议、特殊认可（如公开表彰）
5. **导师/mentor**：为其匹配更高层级的导师（如VP级mentor给总监级高潜）

### 第三步：B等级——稳定提升方案
1. **维持优势**：确认其强项并持续巩固
2. **突破方向**：选择1-2个最有潜力提升的维度重点发力
3. **技能提升**：安排与突破方向匹配的培训和实践机会
4. **目标挑战度调整**：下周期适当提高目标挑战度，推动其跨越B→A的门槛
5. **反馈频率**：增加管理者日常反馈频率（月度简短回顾）

### 第四步：C等级——关注辅导方案
1. **问题聚焦**：明确最关键的2-3个改进项（不贪多）
2. **短期目标**：设定30天内可见效果的改进目标
3. **辅导安排**：指定经验丰富的同事进行工作辅导（周频率）
4. **资源支持**：评估是否需要调整工作量或提供额外工具支持
5. **跟踪机制**：管理者每两周进行一次进展确认（非正式但有记录）

### 第五步：D等级（首次）——正式改进方案
1. **严肃沟通**：明确告知当前状况的严重性和后果（连续D将启动PIP）
2. **具体改进目标**：量化的60天改进目标，直接关联岗位核心职责
3. **密集支持**：培训+辅导+周度check-in的组合支持
4. **书面确认**：改进计划双方签字确认，明确评估标准和时间节点
5. **预警机制**：30天中期检查，改进不明显则提前预警

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 绩效评估结果 | 员工最终等级和各维度评分 | 是 |
| 绩效面谈记录 | 面谈中达成的共识和员工诉求 | 是 |
| 能力评估数据 | 360反馈、胜任力评估结果 | 是 |
| 培训资源清单 | 可用的培训课程和导师资源 | 否 |
| 岗位发展路径 | 职级体系和晋升要求 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 个人发展/改进计划 | 结构化文档 | 含目标、行动、时间线、资源、评估标准 |
| 培训需求清单 | 列表 | 需要安排的培训项目及优先级 |
| 跟踪日历 | 日程表 | 各check-in节点和负责人 |
| 激励方案建议 | 文档（S/A专用） | 薪酬、发展、认可等激励建议 |

## 最佳实践

1. **量身定制**：每个人的计划都应基于其具体情况，避免套用模板
2. **员工参与**：计划制定过程应让员工充分参与（非单方面强加）
3. **聚焦可控**：改进项应聚焦员工可控的因素，不设定超出权限的目标
4. **短期可见**：首个改进里程碑应在30天内可达成，增强信心
5. **资源匹配**：不能只提要求不给支持，改进计划必须配套资源承诺

## 约束条件

- 改进/发展计划需在面谈后5个工作日内完成制定
- S/A等级的发展计划需与培训发展scope协调资源
- D等级改进计划需HR BP审核后方可生效
- 改进计划的跟踪记录需归入员工绩效档案
- 改进计划不得包含歧视性或羞辱性条款
