# 异常检测与预警 (Anomaly Detection & Alerting)

## 技能概述

通过数据分析自动识别绩效管理中的异常模式和风险信号，在问题升级前及时预警相关管理者和HR。异常检测覆盖评估偏差、绩效波动、人才流失风险和流程合规性等多个维度，是绩效管理从事后反应转向事前预防的关键能力。

## 适用场景

- 评估过程中识别管理者评分偏差
- 绩效结果公布后识别高风险人员
- 季度OKR回顾中识别严重偏离目标的情况
- 日常绩效数据监控中的异常信号捕捉

## 执行步骤

### 第一步：定义异常指标体系
1. **评估偏差类**：
   - 管理者评分均值偏离全公司均值>0.5个标准差
   - 管理者评分方差异常小（趋中效应）或异常大
   - 自评与上级评差异>1.5分（满分5分制）
2. **绩效波动类**：
   - 个人绩效等级跨两级变化（如从A跌至C）
   - 部门整体绩效较上期变化>15%
   - 高潜人才绩效评级低于B
3. **风险预警类**：
   - 连续两期C评级（PIP预警边缘）
   - 连续两期D评级（需立即启动PIP）
   - 高绩效员工绩效突降+近期有面试行为（离职高风险）
4. **流程合规类**：
   - 评估超时未完成（>规定时限）
   - 面谈超时未完成（>5个工作日）
   - PIP跟踪check-in遗漏

### 第二步：数据监控与检测
1. 设定数据采集频率（实时/日/周/评估期间）
2. 运行异常检测规则，标注触发条件的数据点
3. 过滤误报（如新入职员工首次评估数据不纳入历史对比）
4. 按严重程度分级：红色（需立即处理）、黄色（需关注）、绿色（信息性）

### 第三步：预警通知
1. **红色预警**（24小时内响应）：
   - 连续2D需启动PIP → 通知HR BP + HR总监
   - 高潜人才绩效突降 → 通知直属上级 + HR BP
   - 重大评估流程超时 → 通知相关管理者上级
2. **黄色预警**（3个工作日内响应）：
   - 管理者评分偏差过大 → 通知HR BP安排校准辅导
   - 部门绩效分布异常 → 通知部门负责人+HR BP
   - 面谈超时未完成 → 通知管理者+催办
3. **绿色通知**（周报形式汇总）：
   - 评估进度通报
   - 目标完成率中期预览
   - 培训完成率跟踪

### 第四步：预警跟踪闭环
1. 记录每条预警的发出时间、接收人、响应情况
2. 跟踪预警后的处理动作是否到位
3. 对未响应的预警进行升级
4. 月度回顾预警准确率和响应率，优化检测规则

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 实时评估数据 | 评估期间的提交数据 | 是 |
| 历史绩效数据 | 用于计算趋势和基线 | 是 |
| 高潜人才名单 | 需重点监控的人员清单 | 是 |
| 预警规则配置 | 各类异常的阈值设定 | 是 |
| 员工动态数据 | 离职申请、转岗等信息 | 否 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 预警通知 | 消息/邮件 | 按严重级别的即时通知 |
| 预警看板 | 仪表盘 | 实时异常状态总览 |
| 预警周报 | 报告 | 本周所有预警汇总和处理状态 |
| 规则有效性报告 | 月报 | 预警准确率和规则优化建议 |

## 最佳实践

1. **宁可误报不可漏报**：对高风险信号降低阈值，宁可多关注也不遗漏
2. **分级通知避免疲劳**：不是所有异常都需要即时通知，避免"狼来了"
3. **提供上下文**：预警通知需附带背景信息（为什么异常、建议怎么处理）
4. **持续优化规则**：基于历史准确率定期调整阈值，减少误报率
5. **隐私保护**：个人层面预警仅通知直接相关方，不群发

## 约束条件

- 红色预警必须在检测到后4小时内发出通知
- 预警信息为敏感数据，仅向必要相关方发送
- 预警规则变更需HR总监审批
- 连续2D的PIP触发为强制规则，不可忽略
- 预警记录保存至少2年（合规审计需要）
