# 评估流程管理 (Evaluation Process Management)

## 技能概述

组织和管理绩效评估全流程的专业能力，包括评估启动、多维度数据收集（自评、上级评、360度评估）、进度监控、催办机制和结果汇总。确保评估在规定时限内高质量完成，数据完整准确。

## 适用场景

- 季度OKR评估的组织执行
- 年度绩效考核的全流程管理
- 360度评估的多方数据收集
- 试用期转正评估的流程支持

## 执行步骤

### 第一步：评估启动准备
1. 确认评估周期和时间线（自评截止日、上级评截止日、360截止日）
2. 生成评估对象清单（含在岗员工、排除长期请假/新入职未满考核周期的员工）
3. 发布评估启动通知（含评估规则、时间表、系统操作指引）
4. 确认评估关系配置正确（每位员工的上级评估人、360评估人已确认）

### 第二步：自评阶段管理（Day 1-3）
1. 开放自评入口，推送自评通知
2. 每日监控自评完成率，Day 2未提交者发送提醒
3. Day 3截止时未完成者推送至其直属上级知悉
4. 自评提交后锁定，不允许再修改

### 第三步：上级评估阶段（Day 4-8）
1. 自评完成后自动触发上级评估入口开放
2. 将自评内容同步给上级作为评估参考
3. Day 6未完成者发送首次催办，Day 7发送升级催办至HR总监
4. 检查上级评估的完整性（是否所有评估项都已填写、评语是否有实质内容）

### 第四步：360度评估阶段（Day 4-10，与上级评并行）
1. 确认360评估人选名单（通常3-5位跨部门/同级协作者）
2. 发送匿名评估邀请，明确评估维度和标准
3. 监控回收率，未回复者Day 7发送提醒
4. 确保匿名性——360评估结果只展示汇总后的平均分和关键词，不展示个人明细

### 第五步：数据汇总与完整性检查
1. 汇总所有维度评估数据：自评分、上级评分、360评分
2. 计算综合绩效得分（权重通常：上级评60%+自评10%+360评30%）
3. 检查数据异常：极端评分（满分或最低分）需标注复核
4. 生成评估完成情况统计报告（按部门/按管理者维度）

## 输入要求

| 输入项 | 说明 | 必需 |
|--------|------|------|
| 考核周期信息 | 评估的时间范围和类型（季度/年度） | 是 |
| 员工名册及评估关系 | 谁评估谁的配置 | 是 |
| 评估模板和标准 | 各维度评估量表和评分规则 | 是 |
| 目标完成度数据 | 从目标设定阶段获取的基线数据 | 是 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 评估进度报告 | 仪表盘/表格 | 实时完成率、催办记录、异常标注 |
| 综合评估得分 | 结构化数据 | 每位员工的多维度评分和加权总分 |
| 评估完成统计 | 报告 | 按时完成率、平均评估质量评分 |
| 异常标注清单 | 列表 | 需校准会重点讨论的异常案例 |

## 最佳实践

1. **提前锁定时间线**：评估日历在周期开始时就确定并通知全员，避免临时通知
2. **三级催办机制**：友善提醒→正式催办→上级升级，逐步加压
3. **评估质量检查**：不只看"是否提交"，还要检查评语质量（字数>100字、有具体事例）
4. **保护匿名性**：360评估结果绝对匿名化，评估人数<3人的不出结果
5. **异常预处理**：在校准会前标注明显异常（如自评5分上级评2分），提前准备讨论材料

## 约束条件

- 自评阶段不超过3个工作日
- 上级评估不超过5个工作日
- 360评估不超过7个工作日
- 全部评估数据必须在校准会前2天完成汇总
- 评估过程中所有数据严格保密，仅相关评估方可见
