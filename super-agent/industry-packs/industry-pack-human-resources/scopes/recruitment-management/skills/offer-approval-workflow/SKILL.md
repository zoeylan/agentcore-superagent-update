# Offer审批流转

## 技能概述
管理Offer从发起审批到最终发放的全流程，确保用人部门负责人和HR总监的双签审批高效完成，处理审批被驳回、条件修改等异常情况，终面通过后3个工作日内完成Offer审批和发放。

## 适用场景
- 标准Offer审批流程执行
- 超标薪资方案的特批流程
- 审批被驳回后的方案调整
- 批量Offer审批（校招场景）
- 加急Offer审批（候选人有其他Offer deadline）

## 执行步骤

### 第一步：审批材料准备
1. 整合Offer审批所需全部材料：
   - 候选人综合评估报告
   - 薪酬方案文档（含对标分析）
   - 用人部门负责人的录用意见
   - 入职时间建议
   - 超标说明（如有）
2. 检查材料完整性，缺失项及时补充

### 第二步：发起审批流程
1. 在审批系统中创建Offer审批单
2. 设定审批路径：
   - 标准路径：用人部门负责人→HR总监（并行或串行视公司规定）
   - 特批路径（超P75）：用人部门负责人→HR总监→CHRO/CEO
   - 校招批量路径：HR总监统一审批（已有预批的薪资标准）
3. 设定审批时效提醒（每级审批不超过1个工作日）

### 第三步：审批跟踪
1. 实时监控审批进度
2. 审批超12小时未响应→发送提醒
3. 审批超24小时未响应→电话/即时消息催促
4. 审批超48小时→升级至更高层级

### 第四步：驳回处理
| 驳回方 | 常见原因 | 处理方案 |
|--------|----------|----------|
| 用人部门 | 不认可候选人能力 | 了解具体顾虑，提供补充面试数据或安排加试 |
| 用人部门 | 薪资过高 | 重新评估薪资方案，协商可接受范围 |
| HR总监 | 薪酬体系失衡 | 调整定级或设定绩效条件式薪资 |
| HR总监 | 编制/预算问题 | 确认编制状态或申请额外预算 |

### 第五步：Offer发放
1. 双签通过后，生成正式Offer letter
2. Offer letter内容确认（法务审核条款表述）
3. 确定发放方式（邮件正式版+电话通知）
4. 设定Offer有效期（通常3-7天回复期限）
5. 发放后记录并进入候选人回复等待状态

### 第六步：Offer发放后跟踪
1. 发放次日跟进候选人收到情况
2. 候选人有疑问时及时解答
3. 临近有效期截止时主动沟通
4. 记录候选人最终回复（接受/拒绝/需考虑）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 薪酬方案文档 | 文档 | 是 | 完整的薪酬方案和对标分析 |
| 候选人评估报告 | 文档 | 是 | 综合面试评估结果 |
| 审批路径 | 枚举 | 是 | 标准/特批/校招批量 |
| 用人部门意见 | 文本 | 是 | 部门负责人的录用意见 |
| 入职日期 | 日期 | 是 | 预计入职时间 |

## 输出规范
- 审批流程状态更新：待审批/审批中/已通过/已驳回
- Offer letter正式文档
- 审批完成通知（含审批意见）
- 驳回处理方案（如被驳回）

## 最佳实践
- 在发起审批前与审批人预沟通，减少被驳回概率
- 加急Offer审批启动"电话+系统"双通道催办
- 校招Offer预审批机制：提前确认薪资标准后批量快速审批
- 审批通过后尽快发放Offer，速度是与竞对争抢人才的关键因素

## 约束条件
- Offer必须获得双签后方可正式发放，任何一方否决即不可发放
- Offer letter条款需经法务确认，避免产生法律争议
- 超标审批不可跳过流程直接发放
- Offer有效期内候选人未回复视为放弃（除非有协商延期记录）
