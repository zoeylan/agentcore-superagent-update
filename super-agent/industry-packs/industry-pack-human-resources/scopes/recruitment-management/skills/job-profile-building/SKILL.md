# 岗位画像构建

## 技能概述
将业务部门的用人需求转化为结构化的岗位画像文档，包括职位描述（JD）、能力素质模型、薪资带宽定位和候选人优先级排序，为下游简历筛选和面试评估提供标准化的参照基准。

## 适用场景
- 新岗位首次招聘，需从零构建岗位画像
- 现有岗位需求变化，需更新岗位画像
- 市场人才稀缺，需调整岗位要求以匹配现实
- 校招场景下的批量岗位画像标准化

## 执行步骤

### 第一步：需求深度访谈
1. 与用人经理进行30-60分钟结构化访谈
2. 明确岗位核心产出（这个人来了首先要解决什么问题？）
3. 了解团队现有人员配置和能力短板
4. 确认该岗位的成长路径和发展空间
5. 获取用人经理心目中的"理想候选人画像"

### 第二步：硬性条件定义
1. 学历要求：明确最低学历和优先学历（注意不得设置歧视性要求）
2. 工作年限：基于岗位复杂度设定合理年限区间
3. 技能要求：区分"必备技能"和"加分技能"
4. 行业经验：明确是否要求特定行业背景
5. 证书资质：列出岗位相关的专业认证要求

### 第三步：能力素质模型构建
1. 核心能力（2-3项）：岗位最关键的差异化能力
2. 通用能力（2-3项）：组织层面统一要求的能力
3. 专业能力（3-5项）：岗位专业领域的技术能力
4. 每项能力定义清晰的行为描述和等级标准（初级/中级/高级/专家）

### 第四步：薪资带宽定位
1. 调研市场薪酬数据（参考薪酬报告、招聘平台数据）
2. 对标内部同级别岗位薪资水平
3. 确定薪资带宽：P25（入门级）- P50（成熟级）- P75（资深级）
4. 明确薪资结构比例（固定薪资:浮动薪资）

### 第五步：JD编写与合规检查
1. 按标准模板编写职位描述（岗位概述、职责描述、任职资格、薪资福利）
2. 合规检查清单：
   - ✅ 无性别限制（除法律明确规定的岗位）
   - ✅ 无年龄歧视表述
   - ✅ 无民族/地域限制
   - ✅ 无婚育状况要求
   - ✅ 无身体条件歧视（除岗位确需）
3. 确保JD表述专业、有吸引力且真实

### 第六步：输出完整岗位画像
1. 整合所有信息形成岗位画像文档
2. 明确must-have（必须满足）vs nice-to-have（加分项）
3. 标注该岗位的市场稀缺度和招聘难度预估
4. 与用人经理确认并签字生效

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 审核通过的需求单 | 文档 | 是 | 包含基本岗位信息 |
| 用人经理访谈记录 | 文本 | 是 | 结构化访谈的要点 |
| 市场薪酬数据 | 数据 | 是 | 相关岗位的市场薪酬参考 |
| 内部薪酬对标 | 数据 | 否 | 内部同级别岗位薪资 |
| 历史招聘数据 | 数据 | 否 | 同类岗位过往招聘的经验数据 |

## 输出规范
完整的岗位画像文档，包含：
- 岗位基本信息（名称、部门、级别、汇报关系）
- 岗位职责描述（5-8条核心职责）
- 任职资格（硬性条件 + must-have/nice-to-have分级）
- 能力素质模型（含行为描述和等级标准）
- 薪资带宽建议（P25/P50/P75及结构比例）
- 招聘渠道建议
- 面试评估维度建议
- 市场供需分析和招聘难度预估

## 最佳实践
- JD长度控制在800-1200字，过长会降低候选人阅读意愿
- 能力要求不超过8项，过多会导致"完美候选人"偏差
- 每个能力维度都要有可观察、可评估的行为描述
- 定期（每季度）回顾和更新岗位画像，确保与市场和业务变化同步

## 约束条件
- JD发布前必须通过合规检查，任何歧视性表述均为硬伤
- 薪资带宽不得突破公司薪酬体系框架（除非获得特批）
- 能力模型需与公司统一的胜任力框架对齐
- 审核通过后1个工作日内完成JD编写
