# 人才库管理

## 技能概述
系统化管理企业人才资产池，对暂不匹配当前岗位但具有未来潜力的候选人进行分类标记、定期维护和适时激活，实现人才资产的长期沉淀和复用，降低未来招聘成本。

## 适用场景
- 初筛/面试未通过但有潜力的候选人入库
- 定期盘点人才库存量，匹配新增岗位
- 候选人关系维护和定期触达
- 校招批量面试后的人才池沉淀
- 离职员工回流意向跟踪

## 执行步骤

### 第一步：候选人入库分类
1. 标记入库原因：当前能力不足但有潜力/薪资不匹配/时机不对/岗位不匹配但适合其他方向
2. 分类维度：
   - 专业方向：技术/产品/运营/销售/管理/职能
   - 级别区间：初级/中级/高级/专家/管理层
   - 适用岗位标签：可能匹配的岗位类型
   - 激活时机建议：建议何时重新联系（如"半年后技术能力可能达到要求"）
3. 关键信息留存：联系方式、薪资期望、核心技能、上次沟通时间和要点

### 第二步：人才库数据维护
1. 定期清洗过期数据（超过2年未联系且无更新的记录）
2. 通过社交平台（LinkedIn、脉脉）跟踪候选人职业动态
3. 更新候选人最新工作经历和技能成长
4. 标记重要状态变化：在职/看机会/已入职竞对

### 第三步：主动匹配与激活
1. 新增招聘需求时，优先在人才库中搜索匹配候选人
2. 匹配成功时，回顾上次沟通记录和入库原因
3. 评估当前是否已解决之前的不匹配因素
4. 制定个性化的重新触达话术（体现对候选人的关注和记忆）

### 第四步：候选人关系维护
1. 对高价值候选人（高管/稀缺技术人才）保持季度性轻触达
2. 触达方式：行业报告分享、公司重大新闻通知、节日问候
3. 不以招聘为直接目的，而是维护长期关系
4. 记录每次互动情况和候选人反馈

### 第五步：人才库效果分析
1. 统计人才库激活转化率（库存候选人最终入职的比例）
2. 分析最有价值的入库来源和类别
3. 评估关系维护投入的ROI
4. 优化入库标准和激活策略

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 候选人基本信息 | 结构化数据 | 是 | 姓名、联系方式、当前公司和岗位 |
| 入库原因 | 文本 | 是 | 为什么暂不合适及未来可能匹配的原因 |
| 适用方向标签 | 标签列表 | 是 | 适合的岗位类型和领域 |
| 激活时机建议 | 文本 | 否 | 建议何时重新评估和联系 |
| 历史沟通记录 | 文本 | 否 | 过往的面试/沟通记录摘要 |

## 输出规范
- 人才库入库确认：完整的候选人档案卡
- 匹配推荐列表：新增岗位与库存候选人的匹配结果
- 人才库健康度报告：总量、分类分布、活跃度、激活率
- 关系维护计划：高价值候选人的季度触达计划

## 最佳实践
- 入库时的标签质量决定后续匹配效率，宁可多花时间做好标记
- 关系维护要真诚，不要让候选人感觉被当作"资源"
- 离职员工是最容易激活的人才库来源（已了解公司文化）
- 人才库不是"回收站"，而是"种子库"——只入库有清晰未来价值的候选人

## 约束条件
- 候选人数据存储需符合《个人信息保护法》要求
- 候选人要求删除个人信息时必须及时响应（数据删除权）
- 人才库信息不得用于招聘以外的商业目的
- 超过2年未联系的候选人记录需重新获取授权或清理
