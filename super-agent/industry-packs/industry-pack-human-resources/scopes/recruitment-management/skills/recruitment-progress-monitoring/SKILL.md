# 招聘进度监控

## 技能概述
实时跟踪所有在招岗位的招聘进度，通过关键节点里程碑监控识别超期风险，及时发出预警并提供加急策略建议，确保关键岗位招聘周期不超过30个工作日。

## 适用场景
- 日常招聘进度的周期性检视
- 招聘周期超过20天的岗位预警
- 招聘看板的周度/月度更新
- 月度招聘复盘和漏斗分析

## 执行步骤

### 第一步：建立监控基线
1. 为每个在招岗位设定预期里程碑：
   - Day 1-2：需求审核完成
   - Day 3：JD发布、渠道启动
   - Day 4-10：简历收集和初筛
   - Day 11-15：电话初面和首轮面试
   - Day 16-22：后续面试轮次
   - Day 23-27：Offer审批和发放
   - Day 28-30：Offer接受和入职确认
2. 标记每个岗位的实际进度位置

### 第二步：每日进度更新
1. 收集各环节的状态更新（简历数量、面试安排、评估完成情况）
2. 更新招聘看板的岗位状态
3. 计算每个岗位的"进度健康度"（实际进度 vs 预期进度）

### 第三步：风险预警
1. **黄色预警（Day 20+未进入面试）**：
   - 分析卡点原因（简历不足/筛选标准过高/面试官排期困难）
   - 提出调整建议
2. **橙色预警（Day 25+未发出Offer）**：
   - 升级至HR负责人关注
   - 启动加速措施（压缩面试间隔/平行推进多个候选人）
3. **红色预警（Day 30+未完成）**：
   - 启动加急流程
   - 建议组合措施：扩大渠道投入、提升薪资竞争力、适当放宽非核心要求

### 第四步：加急策略制定
当岗位进入红色预警时，提供以下策略组合建议：
1. 渠道扩展：增加猎头委托数量、启用被动候选人主动搜寻
2. 薪资调整：建议提高薪资带宽5-15%以增加竞争力
3. 条件放宽：评估可放宽的非核心要求（如年限要求降低1年、行业经验放宽）
4. 流程加速：面试轮次合并、审批快速通道

### 第五步：周度/月度报告
1. 周报内容：在招岗位数量、本周新增/关闭、各岗位进度状态、预警清单
2. 月报内容：招聘漏斗分析（需求→简历→初筛→面试→Offer→入职）、渠道效果对比、平均招聘周期趋势、超期原因分析

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 在招岗位清单 | 列表 | 是 | 所有进行中的招聘岗位 |
| 各岗位当前状态 | 枚举 | 是 | 需求审核/简历筛选/面试中/Offer阶段 |
| 岗位开始日期 | 日期 | 是 | 需求审核通过的日期 |
| 各环节数据 | 数字 | 是 | 简历数/初筛通过数/面试安排数/Offer数 |

## 输出规范
- 招聘看板更新（含Red/Yellow/Green状态标识）
- 预警通知（含具体原因分析和建议措施）
- 周度进度报告
- 月度招聘复盘分析报告（含转化漏斗图和趋势分析）
- 加急策略建议方案

## 最佳实践
- 预警不等于追责，核心目的是尽早发现问题并提供解决方案
- 区分"可控延迟"（内部流程慢）和"不可控延迟"（人才市场供给不足）
- 关注"漏斗转化率"而非单纯看"天数"，找到真正的卡点
- 月度复盘时邀请业务部门参与，共同优化流程

## 约束条件
- 预警规则统一执行，不因岗位或部门而异
- 加急策略建议需经HR负责人审批后执行
- 进度数据真实准确，不允许为规避预警而人为调整状态
- 月度报告需在次月5个工作日内完成
