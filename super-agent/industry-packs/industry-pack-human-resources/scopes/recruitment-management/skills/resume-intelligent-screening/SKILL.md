# 简历智能筛选

## 技能概述
基于岗位画像对收到的简历进行多维度匹配评估，通过结构化的筛选标准快速识别高匹配度候选人，将初筛通过率控制在15-25%的合理区间，确保简历到达后24小时内完成评估。

## 适用场景
- 单个岗位日常简历筛选
- 校招季批量简历处理（日均100+份）
- 猎头推荐简历的快速评估
- 人才库存量简历的重新匹配

## 执行步骤

### 第一步：加载筛选标准
1. 获取目标岗位的完整岗位画像
2. 明确硬性淘汰条件（一票否决项）
3. 明确加权评分维度和各维度权重
4. 确认当前岗位的通过率目标区间

### 第二步：硬性条件初筛（一票否决）
快速排除明显不匹配的简历：
1. 学历是否满足最低要求
2. 工作年限是否在要求区间内
3. 必备技能/证书是否具备
4. 薪资期望是否在预算范围内（允许±20%的弹性）
5. 地域/出差意愿是否匹配

### 第三步：多维度匹配评分
对通过硬性筛选的简历进行评分（满分10分）：
| 维度 | 权重 | 评估要素 |
|------|------|----------|
| 技能匹配度 | 30% | 核心技能覆盖度、技术栈深度 |
| 经验相关度 | 25% | 行业经验、项目经验、岗位职责相似度 |
| 教育背景 | 15% | 院校层次、专业相关度 |
| 职业稳定性 | 15% | 跳槽频率、在职时长趋势 |
| 发展潜力 | 15% | 职业上升轨迹、学习成长速度 |

### 第四步：结论判定
- 推荐（7-10分）：推送至电话初面环节
- 待定（5-6分）：标记原因，待与用人经理确认
- 不推荐（<5分）：附简要原因说明
- 人才库标记：不匹配当前岗位但可能适合其他岗位或未来岗位

### 第五步：批量数据分析
1. 每日统计筛选通过率
2. 通过率<15%：向需求分析师反馈，建议放宽条件或扩大渠道
3. 通过率>25%：评估是否标准过松，建议收紧关键维度
4. 输出各渠道简历质量对比数据

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 候选人简历 | 文档 | 是 | PDF/Word/在线简历格式 |
| 简历来源渠道 | 枚举 | 是 | Boss直聘/猎聘/内推/猎头等 |
| 目标岗位画像 | 文档 | 是 | 包含筛选标准的岗位画像 |
| 猎头推荐说明 | 文本 | 否 | 猎头对候选人的推荐理由 |

## 输出规范
- 简历评估报告：匹配度评分（1-10）、各维度得分、推荐结论、关键亮点、风险点
- 批量筛选汇总：通过/待定/不通过数量及比例
- 通过率异常报告：超出目标区间时的原因分析和建议

## 最佳实践
- 筛选时关注候选人的"成长轨迹"而非仅看当前状态
- 对于高潜力候选人（如从小公司快速成长到大厂的经历）适当放宽硬性条件
- 避免"完美候选人"偏差——满足80%核心要求即值得面试
- 猎头推荐简历应给予更高优先级处理（因为猎头已做过一轮筛选）
- 简历中的数据化成果描述（如"提升XX指标30%"）比模糊描述更有参考价值

## 约束条件
- 绝对不能因性别、年龄、婚育状况、民族等因素筛掉候选人
- 简历信息属于个人隐私，不得传播给无关人员
- 同一候选人在6个月内不应被同一岗位重复筛选
- 猎头推荐简历需与自有渠道去重后才能计入猎头业绩
