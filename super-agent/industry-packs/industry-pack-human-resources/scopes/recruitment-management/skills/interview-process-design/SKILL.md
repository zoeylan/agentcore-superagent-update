# 面试流程设计

## 技能概述
根据不同岗位类型和级别，设计标准化的面试流程方案，明确面试轮次、评估维度、面试官角色和评估工具，确保面试过程公正、高效且能有效识别目标人才。

## 适用场景
- 新岗位首次招聘需设计面试流程
- 现有面试流程效果不佳需优化
- 校招场景下的批量面试流程设计
- 高管岗位的特殊面试流程设计

## 执行步骤

### 第一步：岗位类型识别与流程模板选择
根据岗位类型选择基础面试流程模板：

| 岗位类型 | 标准流程 | 预计周期 |
|----------|----------|----------|
| 技术岗 | 在线笔试→技术面试→业务面试→HR面试 | 7个工作日 |
| 管理岗 | 案例分析→无领导小组讨论→高管面试→HR面试 | 10个工作日 |
| 销售岗 | 情景模拟→业务面试→HR面试 | 5个工作日 |
| 运营岗 | 笔试→业务面试→HR面试 | 5个工作日 |
| 校招 | 笔试→群面→业务面→HR面 | 3-5个工作日 |

### 第二步：各轮次评估维度设定
1. 每轮面试明确2-3个核心评估维度（避免重复考察）
2. 为每个维度设定评分标准（1-5分量表+行为描述锚定）
3. 设计结构化面试题目库（每个维度3-5道备选题目）
4. 确定各轮的"一票否决"条件

### 第三步：面试官角色分配
1. 技术面试官：验证专业能力深度（需同岗位或更资深的专家）
2. 业务面试官：评估业务理解和团队适配（通常为直属上级）
3. 高管面试官：评估价值观对齐和发展潜力（部门负责人+）
4. HR面试官：评估文化适配、职业动机和薪资匹配度

### 第四步：评估工具配置
1. 笔试：选择或开发与岗位相关的测试题目（技术笔试/案例分析/行测）
2. 群面：设计讨论主题、分组规则和评估表
3. 结构化面试：准备面试评估表和评分指引
4. 性格测评：选择合适的测评工具（如MBTI、DISC、大五人格——仅作参考不作决策依据）

### 第五步：流程文档化
1. 输出完整的面试流程手册
2. 包含每轮面试的SOP（时长、评估表、反馈提交要求）
3. 明确面试官培训要求（首次担任面试官需完成面试官培训）
4. 设定异常处理规则（爽约、改期、面试官临时缺席）

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 岗位画像 | 文档 | 是 | 含能力素质模型 |
| 岗位级别 | 枚举 | 是 | 影响面试轮次和面试官级别 |
| 紧急程度 | 枚举 | 是 | 决定是否需要压缩流程 |
| 可用面试官 | 列表 | 否 | 各环节可选的面试官 |

## 输出规范
- 面试流程方案：轮次设计、各轮评估重点、时间安排
- 面试评估表模板：各轮次使用的结构化评估表
- 面试官指引：面试官角色说明和评分标准
- 异常处理规则：各种异常情况的处理流程

## 最佳实践
- 面试总轮次不超过4轮，过多轮次会导致候选人流失
- 加急岗位可合并面试轮次（如技术面+业务面合并为panel面试）
- 每轮面试时长控制在45-60分钟，给面试官留出评估撰写时间
- 面试结束后即时反馈优于延迟反馈（记忆准确度随时间衰减）

## 约束条件
- 面试问题不得涉及候选人隐私（婚育、宗教、政治倾向等）
- 性格测评结果仅供参考，不得作为独立的筛选依据
- 面试官需经过基本培训后方可独立面试
- 所有面试环节需有书面评估记录留档
