# 招聘需求审核与审批

## 技能概述
对业务部门提交的招聘需求进行系统化审核，从编制合理性、预算可行性和岗位必要性三个维度评估需求的合理性，并在2个工作日内给出审核结论和改进建议。

## 适用场景
- 业务部门提交新增岗位招聘需求
- 离职补招需求的快速审批
- 组织架构调整带来的批量招聘需求
- 项目制临时用人需求评估

## 执行步骤

### 第一步：需求信息完整性检查
1. 确认需求单包含必填字段：岗位名称、部门、汇报关系、招聘人数、期望到岗时间、岗位职责概述
2. 检查需求发起人是否具有审批权限（通常需部门负责人及以上）
3. 缺失关键信息时，24小时内反馈补充清单

### 第二步：编制合理性审核
1. 查询该部门当前编制总数和在岗人数
2. 确认是否有空余编制（有→直接通过此维度；无→需确认是否已获得加编审批）
3. 评估该岗位在团队中的定位是否合理（是否与现有岗位重叠、职责边界是否清晰）

### 第三步：预算可行性审核
1. 查询该部门年度招聘预算余额
2. 评估岗位薪资预期是否在部门薪资预算范围内
3. 如涉及猎头渠道，确认猎头费用预算是否充足（通常为年薪的20-25%）
4. 预算不足时标记风险并建议解决方案（调整预算/降低薪资预期/更换渠道）

### 第四步：岗位必要性评估
1. 分析该需求的业务背景（业务增长/人员流失/新项目启动/组织变革）
2. 评估是否可通过内部调配、外包或兼职方式满足需求
3. 如为离职补招，了解离职原因，评估是否需要调整岗位定位

### 第五步：输出审核结论
1. 形成审核报告：通过/不通过/有条件通过
2. 不通过需明确说明原因和改进建议
3. 有条件通过需注明需满足的前置条件
4. 将审核通过的需求推送至岗位画像构建环节

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 岗位名称 | 文本 | 是 | 招聘的岗位标题 |
| 招聘部门 | 文本 | 是 | 发起需求的部门 |
| 招聘人数 | 数字 | 是 | 本次招聘的名额 |
| 岗位级别 | 枚举 | 是 | 初级/中级/高级/总监 |
| 期望到岗时间 | 日期 | 是 | 业务期望的最晚到岗日期 |
| 招聘原因 | 枚举 | 是 | 新增/补招/储备 |
| 预算范围 | 数值区间 | 是 | 月薪预算上下限 |
| 岗位职责 | 文本 | 是 | 主要工作内容描述 |

## 输出规范
| 字段 | 类型 | 说明 |
|------|------|------|
| 审核结论 | 枚举 | 通过/不通过/有条件通过 |
| 编制审核结果 | 文本 | 编制维度的审核详情 |
| 预算审核结果 | 文本 | 预算维度的审核详情 |
| 必要性审核结果 | 文本 | 岗位必要性的审核详情 |
| 改进建议 | 文本 | 不通过或有条件通过时的具体建议 |
| 优先级评定 | 枚举 | 紧急/高/中/低 |
| 建议招聘周期 | 数字 | 基于岗位难度预估的合理招聘天数 |

## 最佳实践
- 审核时效严格控制在2个工作日内，避免成为流程瓶颈
- 对于紧急需求（关键项目人员到位），可启动快速审核通道（4小时内完成）
- 审核不通过时，提供具体可操作的改进建议，而非简单否决
- 定期分析审核数据，识别频繁出现的问题模式（如某部门反复提交不合理需求），主动沟通解决

## 约束条件
- 不得越权审批超出自身权限范围的需求（如总监级以上岗位需CHRO审批）
- 审核标准需保持一致性，避免因人而异
- 涉及敏感岗位（财务、法务、信息安全）需增加合规审核维度
