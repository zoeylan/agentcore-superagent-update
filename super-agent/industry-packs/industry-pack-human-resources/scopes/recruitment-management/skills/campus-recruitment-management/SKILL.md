# 校园招聘管理

## 技能概述
专项管理校招季（3-4月春招、9-10月秋招）的批量招聘活动，涵盖宣讲会排期、笔试组织、群面安排、批量面试官协调等特殊场景，确保在有限时间窗口内高效完成大批量人才选拔。

## 适用场景
- 秋招/春招季的整体规划和执行
- 校园宣讲会的组织安排
- 笔试的批量组织和评阅
- 群面/无领导小组讨论的分组和执行
- 校招Offer发放的批量管理

## 执行步骤

### 第一步：校招规划（T-60天）
1. 确定校招岗位需求和HC总数
2. 选定目标院校清单（综合院校排名、专业匹配度、历史招聘效果）
3. 制定校招时间表：宣讲→网申→笔试→面试→Offer
4. 确定校招团队人员（HR+业务面试官+校园大使）
5. 准备校招物料（宣传册、展架、宣讲PPT、宣传视频）

### 第二步：宣讲会组织（T-30天至T-14天）
1. 联系目标院校就业中心预约宣讲时间和教室
2. 发布宣讲会信息（校园公众号、牛客网、学校就业网）
3. 安排宣讲主讲人（HR+业务高管/校友）
4. 准备互动环节（Q&A、现场投递、礼品发放）
5. 宣讲会后24小时内跟进参会同学的投递情况

### 第三步：笔试组织
1. 确定笔试形式：在线笔试（大规模）/ 现场笔试（小规模）
2. 协调笔试题目开发（技术笔试由技术团队出题、行测由HR统一题库）
3. 设定笔试时间和考场安排
4. 批量发送笔试通知
5. 批量阅卷和成绩统计（设定通过线：通常top 30-40%进入面试）

### 第四步：群面/批量面试安排
1. 群面分组：每组6-8人，按专业/岗位方向分组
2. 面试官安排：每组2-3位面试官（1业务+1-2 HR观察员）
3. 场地协调：确保面试室数量足够支撑并行面试
4. 时间安排：每场群面30-45分钟，日均安排10-15场
5. 评估表分发和收集流程

### 第五步：后续面试和Offer管理
1. 群面通过者进入单面（一对一业务面+HR面）
2. 压缩面试周期：单面尽量安排在群面后3天内
3. 批量Offer审批：统一审批流程，按岗位统一薪资标准
4. Offer发放策略：分批发放，预留备选名额应对拒绝
5. Offer接受截止日管理：设定合理期限（通常7-14天）

### 第六步：数据统计与复盘
1. 统计校招漏斗数据：投递→笔试→群面→单面→Offer→签约
2. 分院校效果分析：各校投递质量和转化率
3. 面试官工作量统计和反馈收集
4. 校招总成本核算（差旅+物料+平台+人力）
5. 输出校招复盘报告，为下一季优化提供依据

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 校招HC需求 | 数字 | 是 | 各岗位招聘人数 |
| 目标院校清单 | 列表 | 是 | 含院校优先级排序 |
| 校招时间窗口 | 日期范围 | 是 | 校招季起止日期 |
| 可用面试官资源 | 列表 | 是 | 面试官及可用时间 |
| 历史校招数据 | 数据 | 否 | 往年各校效果数据 |

## 输出规范
- 校招整体方案：时间表、目标院校、资源需求、预算
- 宣讲会执行清单：每场宣讲会的5W1H
- 笔试安排方案：时间、平台、通过标准
- 群面分组表：分组名单和面试官分配
- 校招数据报告：全流程漏斗和分院校分析

## 最佳实践
- 校招是雇主品牌建设的重要窗口，每个接触点都是品牌展示
- 在校园大使（已入职的往届同学）是最有说服力的宣传人
- 笔试题目需每年更新，防止历年真题泄露
- 群面话题选择要公平（避免对特定专业有利的题目）
- Offer发放不要一次性全部发完，留出buffer应对变数

## 约束条件
- 校招薪资方案统一标准，同岗位不因院校差异而薪资不同
- 不得在招聘信息中设置"仅限985/211"等院校歧视条件
- 笔试和面试评估标准对所有候选人一致
- 候选人数据收集需符合个人信息保护要求
- 未被录用的候选人简历保留期限不超过1年（除非获得授权）
