# 面试评估汇总报告

## 技能概述
汇总各轮面试官的评估反馈，形成候选人综合评估报告，通过多维度数据整合和一致性分析，为录用决策提供客观、全面的依据。

## 适用场景
- 候选人完成全部面试轮次后的综合评估
- 多位候选人的横向对比评估
- 面试意见分歧时的决策支持
- 校招批量面试后的集中评审

## 执行步骤

### 第一步：评估反馈收集与完整性检查
1. 确认所有轮次面试官已提交评估（24小时催收机制）
2. 检查评估表填写完整性（是否所有维度都有评分和文字说明）
3. 如有缺失，联系面试官补充关键信息
4. 收集面试中使用的测评工具结果（笔试成绩、案例分析评分等）

### 第二步：评估数据整合
1. 汇总各轮面试的评分数据
2. 按能力维度重新归类：
   - 专业能力维度：各轮相关评分加权平均
   - 通用能力维度：沟通、逻辑、学习能力等评分汇总
   - 文化适配维度：价值观、团队协作、职业动机等评估
3. 标记各面试官的核心观察和关键评语

### 第三步：一致性分析
1. 检查各轮面试官的评价是否一致
2. 高度一致（全部推荐或全部不推荐）→直接形成结论
3. 存在分歧（部分推荐部分保留）→分析分歧原因：
   - 是考察维度不同导致的合理差异？
   - 还是对同一维度判断不一致需要复议？
4. 重大分歧记录在案，提供给决策者参考

### 第四步：综合评估报告撰写
报告结构：
1. **候选人概览**：基本信息、岗位匹配度总评
2. **面试历程**：各轮面试时间、面试官、评分
3. **能力雷达图**：各能力维度的综合得分可视化
4. **核心优势**：3-5个突出亮点（基于面试官共识）
5. **关注风险**：需要注意的问题点或不确定性
6. **录用建议**：强烈推荐/推荐/保留/不推荐
7. **薪资建议区间**：基于评估结果建议的薪资定位

### 第五步：决策支持
1. 如有多位候选人，提供横向对比表
2. 对比维度：核心能力匹配度、薪资期望匹配度、到岗时间、稳定性风险
3. 标注推荐优先级（第一候选人/备选候选人）
4. 将报告提交给Offer管理专员和用人部门负责人

## 输入规范
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 各轮面试评估表 | 文档列表 | 是 | 每位面试官提交的结构化评估 |
| 笔试/测评成绩 | 数字 | 否 | 如有笔试或性格测评的结果 |
| 候选人简历 | 文档 | 是 | 用于核对面试反馈与简历信息一致性 |
| 岗位画像 | 文档 | 是 | 用于评估与岗位要求的匹配度 |

## 输出规范
- 候选人综合评估报告（完整版）：含全部详细评估数据和分析
- 候选人评估摘要（决策版）：1页纸的关键信息和结论
- 多候选人对比表（如适用）：横向对比的决策参考
- 面试官评价一致性分析

## 最佳实践
- 报告力求客观，区分"事实观察"和"主观判断"
- 引用面试官原话而非过度概括，保留信息的丰富度
- 风险点要诚实记录，不因倾向性而隐瞒或淡化
- 对于面试官评价矛盾的情况，建议增加加试环节而非简单取平均

## 约束条件
- 评估报告不得包含与岗位无关的个人信息评价
- 报告仅限招聘决策相关方查阅，不得随意传播
- 录用建议是"建议"而非"决定"，最终决策权在用人部门
- 评估数据需保存至少2年，用于招聘质量回溯分析
