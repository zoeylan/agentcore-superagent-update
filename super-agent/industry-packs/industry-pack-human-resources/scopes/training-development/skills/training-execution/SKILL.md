# 培训项目执行 (Training Execution)

## 技能概述

培训项目执行是将培训计划从纸面转化为实际学习体验的落地技能，涵盖培训全生命周期的运营管理——从通知发布、报名管理、现场组织到培训收尾的每一个执行环节。高质量的执行是培训效果的前提保障。

### 适用场景
- 线下面授培训的全流程组织
- 线上培训项目的运营管理
- 混合学习项目的跨形式协调
- 大型培训项目（如新员工集训、领导力项目）的项目化管理
- 学时达标的日常监控和干预

## 详细执行步骤

### Step 1: 培训前准备（T-7至T-1天）
1. T-7天：发布培训通知（含时间、地点、内容简介、准备要求）
2. T-7天：开放报名系统，设定报名截止时间
3. T-5天：监控报名人数：
   - 人数不足 → 调整时间或扩大参训范围
   - 人数超额 → 按业务优先级筛选或增加班次
4. T-3天：分发培训资料（预习材料、案例背景、软件安装指引等）
5. T-3天：确认讲师、场地、设备就绪
6. T-1天：发送培训提醒（含签到方式、注意事项）

### Step 2: 培训现场管理
1. 提前30分钟完成场地布置和设备调试
2. 学员签到管理（确保出勤记录准确）
3. 监控培训进度，处理突发情况（设备故障、讲师迟到等）
4. 管理课堂纪律和学习氛围
5. 安排茶歇、午餐等后勤保障
6. 出勤率≥90%为有效开班（低于此比例需评估是否继续）

### Step 3: 线上培训运营
1. 确保学习平台课程发布和权限配置正确
2. 监控学员登录和学习进度
3. 管理学习社区互动（答疑、讨论、作业提交）
4. 发送学习进度提醒（对落后学员推送催促）
5. 处理技术问题（平台故障、视频加载问题等）

### Step 4: 培训收尾
1. 收集Level 1满意度问卷（当天完成）
2. 组织Level 2学习测试（培训结束时）
3. 发放培训结业证书/学时认证
4. 更新学员学时记录
5. 整理培训现场资料（签到表、照片、录像）存档
6. 编写培训执行总结报告

### Step 5: 学时监控管理
1. 每季度统计全员培训学时完成进度
2. 识别学时落后人员（低于季度目标进度）
3. 对落后人员推送学习提醒和推荐课程
4. 年末仍未达标（<40学时）的人员名单报部门负责人和HR总监
5. 未达标纳入绩效考核扣分项

## 输入规格
| 输入项 | 来源 | 格式要求 |
|--------|------|----------|
| 培训日历/排期 | 年度培训计划 | 时间、课程、讲师安排 |
| 学员名单 | 报名系统/指定参训 | 姓名、部门、联系方式 |
| 培训资料 | 课程开发输出 | PPT、讲义、案例材料 |
| 场地/设备信息 | 行政部门 | 场地预订确认 |

## 输出规格
| 输出项 | 格式 | 接收方 |
|--------|------|--------|
| 培训签到记录 | 出勤表 | 学时管理/合规存档 |
| 培训执行总结 | 报告文档 | 培训管理层 |
| 学时更新数据 | 数据更新 | 学习管理系统 |
| 学时进度通报 | 季度报表 | 部门负责人/HR总监 |
| 异常情况处理记录 | 事件记录 | 质量改进 |

## 最佳实践
- 培训通知要包含"这次培训对你有什么用"的信息，提升报名意愿
- 线下培训准备Checklist标准化，避免遗漏关键环节
- 混合学习项目需要明确线上线下的衔接节奏和学员行动
- 学时管理不是目的，要引导员工主动学习而非被动凑学时
- 大型培训项目建议配备专职项目经理统筹

## 约束条件
- 培训通知必须提前7天发布
- 培训资料必须提前3天分发
- 出勤率≥90%为有效开班
- 每人每年≥40学时培训（硬性要求）
- 年度培训学时达标率≥95%
- 未达标人员名单每季度通报
