# 学习数据分析 (Learning Analytics)

## 技能概述

学习数据分析是利用多源数据对培训体系的整体效能进行量化分析和洞察挖掘的技能。通过对学习行为数据、培训效果数据、业务绩效数据的关联分析，为培训策略优化和资源配置调整提供数据驱动的决策支持。

### 适用场景
- 季度/年度培训效果汇总分析
- 培训项目对比分析（效果、成本效益）
- 学员学习行为模式分析
- 培训与业务绩效的相关性分析
- 培训体系健康度诊断
- 管理层汇报的数据支撑

## 详细执行步骤

### Step 1: 数据收集与整合
1. 学习管理系统（LMS）数据：
   - 课程完成率、学习时长、登录频率
   - 学习路径选择、内容偏好
   - 测评成绩和通过率
2. 培训评估数据：
   - Level 1-4各层级评估结果
   - 讲师评分、课程评分
3. 人力资源数据：
   - 绩效评估结果、能力评估分数
   - 晋升记录、调薪记录
   - 离职数据
4. 业务数据：
   - 部门/团队业务KPI
   - 客户满意度数据

### Step 2: 描述性分析
1. 全员学时统计（人均学时、部门分布、达标率）
2. 课程参与度分析（报名率、出勤率、完成率）
3. 培训满意度趋势分析（按季度、按课程类型）
4. 讲师授课质量排名和趋势
5. 预算执行进度分析

### Step 3: 诊断性分析
1. 学时未达标的原因分析（工作繁忙/课程不感兴趣/时间冲突）
2. 课程满意度低的根因分析（内容/讲师/形式/组织）
3. 行为转化率低的障碍分析
4. 部门间培训投入产出差异原因
5. 高离职率与培训参与度的关联分析

### Step 4: 预测性分析
1. 基于历史数据预测下季度学时达标风险人员
2. 预测哪些培训项目的行为转化效果可能不理想
3. 识别高潜力人才的发展加速机会
4. 预算消耗趋势预测和调整建议

### Step 5: 报告产出
1. 编制数据可视化仪表盘（关键指标一目了然）
2. 撰写分析报告（发现→原因→建议→行动）
3. 为不同受众定制汇报内容：
   - HR总监：宏观效能和ROI
   - 培训团队：执行质量和改进方向
   - 业务部门：人才发展进展
4. 提出数据驱动的改进建议

## 输入规格
| 输入项 | 来源 | 格式要求 |
|--------|------|----------|
| LMS数据 | 学习管理系统 | 结构化数据导出 |
| 评估结果 | 柯氏四级评估 | 各维度评分数据 |
| HR数据 | 人力资源系统 | 员工主数据+绩效数据 |
| 业务KPI | 业务系统 | 部门/团队绩效指标 |
| 预算数据 | 财务系统 | 实际支出明细 |

## 输出规格
| 输出项 | 格式 | 接收方 |
|--------|------|--------|
| 培训效果仪表盘 | 数据看板 | 全培训团队 |
| 季度分析报告 | 分析文档+图表 | HR总监/培训管理层 |
| 改进建议清单 | 行动项列表 | 培训需求分析师/课程开发 |
| 风险预警通报 | 预警通知 | 相关责任人 |

## 最佳实践
- 数据分析的价值在于"可行动的洞察"，而非堆砌数据
- 建立固定的分析模板和节奏（月度快报、季度深度、年度白皮书）
- 关注趋势变化而非绝对数值，同比和环比都很重要
- 将培训数据与业务数据关联，才能证明培训的业务价值
- 注意数据隐私合规，个人学习数据的分析和展示要脱敏

## 约束条件
- 数据统计口径需保持年度内一致，变更需说明
- 个人层面的学习数据分析遵守隐私保护原则
- 分析结论需有充分数据支撑，避免小样本推断
- 季度报告需在季度结束后10个工作日内产出
- 数据异常需在发现后24小时内验证和上报
