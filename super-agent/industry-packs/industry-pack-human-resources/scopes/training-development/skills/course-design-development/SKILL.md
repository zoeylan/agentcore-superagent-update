# 课程设计与开发 (Course Design & Development)

## 技能概述

课程设计与开发是将培训需求转化为结构化学习内容的专业技能，涵盖从课程立项、教学设计、内容开发到上线评审的全流程。核心方法论基于ADDIE模型（Analysis-Design-Development-Implementation-Evaluation），同时融合混合学习设计理念，确保课程既有理论深度又有实践应用价值。

### 适用场景
- 内部新课程的从零开发
- 现有课程的迭代升级
- 外部课程的内部化定制改造
- 管理岗必修领导力课程体系建设
- 新员工入职培训课程包设计

## 详细执行步骤

### Step 1: 课程立项评估
1. 确认课程开发需求来源和业务背景
2. 评估开发方式决策：
   - 核心能力/企业特有知识 → 内部开发（沉淀组织智慧）
   - 通用管理/技术技能 → 外部采购（效率优先）
   - 高端领导力 → 外部专家+内部案例定制
3. 评估开发资源需求（内容专家、教学设计师、开发工具）
4. 制定开发时间计划（内部课程不超过6周，外部采购不超过4周）

### Step 2: 教学设计 (Instructional Design)
1. 明确学习目标（运用Bloom's分类法：记忆→理解→应用→分析→评估→创造）
2. 分析目标学员画像（知识基础、学习偏好、时间约束）
3. 设计课程结构（模块划分、时间分配、练习比例）
4. 选择教学策略（讲授、案例分析、角色扮演、小组讨论、实操练习）
5. 设计评估方式（与Level 2学习层评估对接）

### Step 3: 内容开发
1. 与业务专家（SME）协作萃取知识和经验
2. 编写课程大纲和详细教案
3. 开发配套教学材料（PPT、案例集、练习手册、视频脚本）
4. 开发线上课程内容（如适用：微课视频、互动测验、学习社区）
5. 设计课后作业和行动学习任务

### Step 4: 内容评审
1. 准确性评审：业务专家确认内容正确性和时效性
2. 实用性评审：目标学员代表试听并反馈
3. 合规性评审：确认不含违规信息（数据安全、知识产权等）
4. 教学质量评审：教学设计规范性检查
5. 评审通过后正式上线，不通过则修改后复审

### Step 5: 课程版本管理
1. 建立课程版本号管理机制
2. 记录每次更新的变更日志
3. 设定课程有效期和定期更新周期
4. 根据培训效果评估反馈持续迭代

## 输入规格
| 输入项 | 来源 | 格式要求 |
|--------|------|----------|
| 课程开发需求单 | 培训需求分析师 | 含目标学员、学习目标、业务场景 |
| 业务知识/专家经验 | SME/业务部门 | 访谈记录、操作手册、最佳实践 |
| 课程开发预算 | 预算分配方案 | 单课程可用预算额度 |
| 已有课程资源 | 课程库 | 可复用的内容素材 |

## 输出规格
| 输出项 | 格式 | 接收方 |
|--------|------|--------|
| 课程开发计划书 | 项目计划文档 | 内部审批/团队执行 |
| 完整课程包 | 教案+PPT+材料+评估工具 | 讲师/学习平台 |
| 课程评审记录 | 评审表 | 质量存档 |
| 课程上线通知 | 通知文档 | 培训执行/学员 |

## 最佳实践
- 课程设计遵循"70-20-10"法则：70%实践应用+20%互动交流+10%正式学习
- 每门课程设计不超过3个核心学习目标，聚焦比全面更重要
- 案例必须来源于企业真实业务场景，避免通用案例的"学完就忘"
- 线上课程单个视频不超过15分钟，保持学习注意力
- 课程开发过程中持续与目标学员小范围验证，避免闭门造车

## 约束条件
- 内部课程从立项到上线不超过6周
- 外部课程采购从选型到签约不超过4周
- 所有课程上线前必须通过三维评审（准确性、实用性、合规性）
- 管理岗必修课程的开发需HR总监参与评审
- 涉及公司机密信息的课程需标注保密级别和受众限制
