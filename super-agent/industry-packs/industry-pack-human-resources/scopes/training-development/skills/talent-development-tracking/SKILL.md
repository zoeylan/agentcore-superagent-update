# 人才发展路径跟踪 (Talent Development Tracking)

## 技能概述

人才发展路径跟踪是将培训活动与员工职业发展贯通的关键技能，通过建立岗位序列能力模型、维护个人能力发展档案、跟踪发展路径上的达标进度，为人才梯队建设和晋升决策提供系统化的数据支撑。特别关键的是管理岗晋升必修课程的认证管理。

### 适用场景
- 员工职业发展路径规划与辅导
- 管理岗晋升前的必修课程完成认证
- 人才盘点中的能力发展评估
- 个人发展计划（IDP）的执行跟踪
- 岗位序列能力模型的建立和维护

## 详细执行步骤

### Step 1: 岗位序列能力模型维护
1. 为每个岗位序列建立分层能力标准：
   - 初级（P1-P3）：基础专业技能+团队协作
   - 中级（P4-P6）：深度专业能力+项目管理
   - 高级（P7-P9）：技术领导力+影响力
   - 管理序列（M1-M4）：团队管理→部门管理→事业部管理
2. 每个层级明确所需的知识、技能和素质标准
3. 标记从A层级到B层级需要完成的关键培训/认证
4. 每年评审一次能力模型时效性，根据业务变化更新

### Step 2: 员工能力发展档案管理
1. 为每位员工建立能力发展档案：
   - 当前岗位和层级
   - 已完成的培训和认证记录
   - 各维度能力评估结果（来源：绩效评估、专项认证）
   - 学习积分累计
   - 个人发展目标（IDP）
2. 持续更新档案数据（每次培训完成后自动更新）
3. 可视化展示能力雷达图（现状vs目标的差距）

### Step 3: 发展路径进度跟踪
1. 根据员工的职业发展目标，匹配对应的发展路径
2. 列出从当前层级到目标层级的所有必备条件
3. 跟踪每个条件的完成状态（已达标/进行中/未开始）
4. 对关键条件（如管理岗必修课程）设置提醒和预警
5. 定期（每季度）生成发展进度报告

### Step 4: 管理岗晋升必修课程认证
1. 维护管理岗晋升的必修课程清单：
   - M1（新任管理者）：团队管理基础、绩效管理实务、劳动法基础
   - M2（部门负责人）：战略规划、跨部门协作、人才梯队建设
   - M3及以上：变革管理、组织设计、领导力进阶
2. 跟踪每位潜在晋升候选人的课程完成状态
3. 晋升申请触发时自动检查必修课程完成情况：
   - 全部完成 → 放行进入晋升评审
   - 未完成 → 推送必修课程安排+通知延期晋升评审
4. 将认证状态同步给绩效管理模块

### Step 5: 学习积分与激励
1. 设计学习积分体系（不同培训活动赋予不同积分）
2. 积分规则：
   - 完成必修培训：基础积分
   - 完成选修培训：额外积分
   - 通过认证考试：高额积分
   - 担任内部讲师：特别积分
3. 积分与发展机会挂钩（学习资源申请、外部培训机会优先）
4. 年度学习积分排名和表彰

## 输入规格
| 输入项 | 来源 | 格式要求 |
|--------|------|----------|
| 岗位胜任力模型 | HR COE | 各序列各层级能力标准 |
| 培训完成记录 | LMS/培训执行 | 课程名称、完成时间、成绩 |
| 绩效能力评估 | 绩效管理模块 | 各维度能力评分 |
| 晋升候选人名单 | 绩效管理/HRBP | 待晋升人员信息 |
| IDP计划 | HRBP/员工 | 个人发展目标和行动计划 |

## 输出规格
| 输出项 | 格式 | 接收方 |
|--------|------|--------|
| 个人能力发展报告 | 可视化报告（雷达图） | 员工本人/直属上级 |
| 晋升课程认证状态 | 认证清单（含完成/未完成） | 绩效管理模块/晋升评审 |
| 人才发展进度仪表盘 | 数据看板 | HR管理层 |
| 学习积分排名 | 排名表 | 员工/部门 |
| 发展路径差距分析 | 分析报告 | HRBP/培训需求分析师 |

## 最佳实践
- 发展路径不应只有一条"正确道路"，要支持多通道发展（专业通道和管理通道）
- 能力模型不宜过于复杂，每个层级聚焦3-5个关键差异化能力
- 必修课程清单需与时俱进，根据业务变化和最佳实践定期更新
- 学习积分体系要避免"刷分"行为，需与实际能力提升挂钩
- 将发展路径可视化呈现给员工，增强自主发展动力

## 约束条件
- 管理岗晋升前必修课程完成率必须达到100%（硬性门槛）
- 未完成必修课程不得进入晋升评审流程
- 能力发展档案每季度至少更新一次
- 认证状态同步给绩效管理模块的时效：变更后24小时内
- 学习积分规则变更需提前1个月公示
- 员工个人发展数据的访问权限需严格管控
