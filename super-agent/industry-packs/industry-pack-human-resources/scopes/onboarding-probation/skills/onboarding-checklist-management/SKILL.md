# 入职准备清单管理 (Onboarding Checklist Management)

## 技能概述

本技能用于管理新员工入职前的标准化准备清单，确保所有准备事项按时间节点有序推进。通过结构化的清单模板和自动化的进度追踪，实现入职准备工作的零遗漏和全程可视化。

## 适用场景

- 接收到新员工Offer确认信息后启动入职准备
- 批量入职场景（如校招季）的并行准备管理
- 特殊岗位（如涉密岗位、外籍员工）的定制化准备清单
- 入职日期变更时的准备计划动态调整

## 执行步骤

### 步骤1：初始化准备清单
- 根据新员工岗位类型（技术/业务/管理）选择对应清单模板
- 填入新员工基本信息（姓名、岗位、部门、入职日期、直属上级）
- 自动计算各时间节点（T-7、T-3、T-1）的具体日期
- 标记岗位特殊要求项（如技术岗需开发环境、销售岗需CRM权限）

### 步骤2：分配责任人和截止时间
- 材料收集类事项→入职准备协调员
- IT系统开通类事项→IT对口支持人员
- 行政设备类事项→行政对口支持人员
- 部门接收准备→直属上级/团队助理
- 导师安排→导师体系协调员

### 步骤3：进度跟踪与预警
- 每日检查各事项完成状态
- T-5天未完成的事项发出黄色预警
- T-3天未完成的关键事项发出红色预警并升级
- 所有事项完成后标记清单状态为"已就绪"

### 步骤4：最终确认与交接
- T-1天进行全面检查确认
- 生成《入职准备完成确认表》
- 将准备就绪状态通知入职引导专员
- 向新员工发送入职提醒邮件/短信

## 输入规范

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 新员工姓名 | 文本 | 是 | 全名 |
| 岗位名称 | 文本 | 是 | 具体岗位 |
| 所属部门 | 文本 | 是 | 一级/二级部门 |
| 入职日期 | 日期 | 是 | 确认的报到日期 |
| 直属上级 | 文本 | 是 | 姓名和联系方式 |
| 岗位类型 | 枚举 | 是 | 技术/业务/管理/支持 |
| 特殊要求 | 文本 | 否 | 岗位特殊准备事项 |

## 输出规范

- 《入职准备进度跟踪表》：含各事项状态、负责人、截止日期
- 《入职准备完成确认表》：全部就绪后的最终确认文档
- 预警通知：逾期事项的提醒和升级通知

## 最佳实践

1. **模板化管理**：维护不同岗位类型的标准清单模板，新建时基于模板生成，避免遗漏
2. **并行推进**：IT准备、行政准备和材料收集可并行推进，不必等待串行完成
3. **buffer设计**：关键事项的实际截止日期比展示截止日期提前1天，留出缓冲
4. **批量模式**：校招季多人同时入职时，统一协调IT和行政批量处理，提高效率
5. **复盘迭代**：每季度回顾清单执行情况，将频繁出现的临时事项固化为标准项

## 约束条件

- 清单启动不得晚于T-7天，低于此时间需启动加急流程
- T-3天的系统账号开通确认是硬性截止日期，不可妥协
- 涉密岗位的安全审查事项需额外提前2周启动
- 外籍员工需增加工作签证和住宿安排事项
