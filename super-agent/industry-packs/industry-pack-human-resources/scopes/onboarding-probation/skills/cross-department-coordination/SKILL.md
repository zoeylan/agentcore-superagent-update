# 跨部门协调 (Cross-Department Coordination)

## 技能概述

本技能用于在入职准备过程中高效协调多个部门（IT、行政、业务部门、安保等）的协同配合，确保各方按时完成各自负责的准备事项。核心能力是任务分发、进度追踪、异常升级和沟通协调。

## 适用场景

- 新员工入职前的IT系统开通协调（邮箱、VPN、OA、业务系统权限）
- 行政设备准备协调（工位、电脑、门禁卡、办公用品）
- 业务部门接收准备确认（上级、导师、团队通知）
- 异常情况的跨部门升级和备选方案协调
- 紧急入职（入职日期提前）的加急协调

## 执行步骤

### 步骤1：需求拆解与任务分发
- 将入职准备需求拆解为各部门可独立执行的任务单元
- 明确每个任务的：负责部门、对接人、完成标准、截止时间
- 通过标准化的协调工单/邮件模板发送任务通知
- 确认各部门接收并安排执行

### 步骤2：进度跟踪与状态同步
- 建立统一的进度看板，各部门更新完成状态
- 每日汇总各部门进度，识别瓶颈和风险
- 主动与未按时更新状态的部门沟通确认
- 关键节点（T-3天）进行全面状态确认

### 步骤3：异常处理与升级
- 常规延迟（1天内）：直接与对接人沟通推进
- 较大延迟（1-2天）：升级至对接人直属上级
- 严重延迟（超2天或T-3天仍未完成）：启动备选方案并报HR经理
- 系统性故障（如IT系统升级导致批量无法开通）：启动应急预案

### 步骤4：确认与闭环
- 所有部门任务完成后进行验收确认（如账号测试登录、设备检查）
- 记录本次协调过程中的问题和改进点
- 向各部门发送感谢确认，维护协作关系

## 输入规范

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 新员工信息 | 对象 | 是 | 姓名、岗位、部门、入职日期 |
| 需求清单 | 列表 | 是 | 各部门需完成的具体事项 |
| 各部门对接人 | 列表 | 是 | 部门、对接人姓名、联系方式 |
| 优先级 | 枚举 | 是 | 常规/加急/批量 |

## 输出规范

- 跨部门协调任务分配表
- 进度状态汇总报告
- 异常处理记录和解决方案
- 协调完成确认通知

## 最佳实践

1. **建立固定对接人机制**：与IT、行政等部门建立固定的入职支持对接人，减少沟通成本
2. **标准化沟通模板**：使用统一的任务通知模板，含所有必要信息，避免反复确认
3. **批量合并请求**：同周入职的多位新员工，合并为一次协调请求，提高各部门处理效率
4. **预留备选方案**：对IT设备等可能缺货的事项，提前确认备选方案（如临时笔记本）
5. **感谢反馈闭环**：每次完成后给协作部门正面反馈，建立长期良好的协作关系

## 约束条件

- 协调请求必须提前至少5个工作日发出（T-5天为底线）
- 不得绕过部门对接人直接指派具体执行人
- 升级路径必须逐级进行，不可跨级升级（除紧急情况）
- 涉及安全权限类账号开通需走额外审批流程，预留更多时间
