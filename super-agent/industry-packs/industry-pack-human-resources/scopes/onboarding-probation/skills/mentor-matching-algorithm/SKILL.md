# 导师匹配算法 (Mentor Matching Algorithm)

## 技能概述

本技能用于为新员工匹配最合适的导师，通过多维度匹配评估模型，在导师资源池中找到岗位专业对口、能力互补、风格兼容且时间可用的最佳导师人选，最大化辅导效果和新员工成长速度。

## 适用场景

- 新员工入职后3个工作日内的导师分配
- 导师辅导效果不佳时的导师更换匹配
- 批量入职（校招季）的导师批量分配
- 导师资源不足时的替代方案设计
- 跨部门导师需求的特殊匹配

## 执行步骤

### 步骤1：需求画像构建
- 收集新员工信息：
  - 岗位序列和级别
  - 专业技能背景
  - 入职前的工作经验
  - 已知的待发展能力项（来自招聘评估）
  - 个人学习风格偏好（如有）
- 确定辅导重点需求：
  - 业务知识传授型（新行业新领域）
  - 技能提升型（有经验但需深化）
  - 综合融入型（综合能力和组织适应）

### 步骤2：导师池筛选
- 基础筛选条件：
  - 同岗位序列或强相关领域
  - 工作年限≥3年且绩效达B及以上
  - 当前带教人数<2人（容量限制）
  - 近半年无严重绩效/合规问题
- 排除条件：
  - 本人正在试用期
  - 近期计划离职或转岗
  - 过往辅导满意度<3.0/5.0

### 步骤3：多维匹配评分
- 专业匹配度（权重35%）：岗位技能重叠度、业务领域相关性
- 能力互补度（权重25%）：导师擅长领域与新员工待发展项的匹配
- 时间可用性（权重20%）：当前工作负荷、已有带教任务数
- 辅导经验（权重15%）：过往辅导次数、学员成长效果、满意度评分
- 物理距离（权重5%）：同楼层/同办公区为佳

### 步骤4：推荐与确认
- 产出Top 3推荐导师名单（含匹配理由和评分）
- 优先征询第1推荐人意愿
- 若拒绝则依次征询第2、第3推荐人
- 导师确认后，安排三方见面（导师+新员工+HR）
- 正式建立辅导关系，明确辅导计划和预期

### 步骤5：特殊情况处理
- 导师池无合适人选：放宽匹配条件（如跨部门导师）或安排临时导师+专项培训补充
- 批量入职：优先匹配核心岗位，一般岗位可安排1对多辅导（上限1:3）
- 导师更换：记录更换原因，分析匹配失败点，优化匹配模型

## 输入规范

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 新员工画像 | 对象 | 是 | 岗位、技能、经验、待发展项 |
| 导师资源池 | 列表 | 是 | 可用导师信息和能力标签 |
| 匹配优先级 | 枚举 | 是 | 专业深度/综合融入/技能提升 |
| 约束条件 | 对象 | 否 | 部门限制、地点限制等 |

## 输出规范

- Top 3推荐导师名单（含评分和匹配理由）
- 匹配报告（各维度得分详情）
- 导师辅导计划建议框架
- 匹配失败分析（如无合适人选时）

## 最佳实践

1. **数据驱动迭代**：基于历史辅导效果数据持续优化匹配权重
2. **双向选择**：不仅考虑匹配度，也尊重导师意愿和新员工偏好
3. **导师培训前置**：确保导师池中的成员都经过基本辅导技巧培训
4. **避免利益冲突**：导师与新员工不应有直接汇报关系
5. **定期更新资源池**：每季度刷新导师能力标签、容量和绩效状态

## 约束条件

- 导师分配必须在入职后3个工作日内完成
- 同一导师同时带教不超过2位新员工
- 导师与新员工不得有直接上下级汇报关系
- 导师自身绩效低于B时暂停其带教资格
- 匹配失败率（分配后1个月内更换）应控制在10%以内
