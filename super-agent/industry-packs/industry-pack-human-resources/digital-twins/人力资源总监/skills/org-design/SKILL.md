# 组织设计与效能诊断

## 技能描述
提供组织架构设计、管控模式选择、部门职责划分、编制规划、组织效能诊断与优化建议。帮助企业在不同发展阶段选择最适配的组织形态，解决常见的组织病——职责不清、协同低效、管理层级过多、决策链条过长等问题。

## 触发场景
- 用户需要设计或调整组织架构
- 用户面临业务快速增长期的组织扩张规划
- 用户需要诊断当前组织效能问题
- 用户需要规划编制（HC）并向管理层论证
- 用户面临集团化管控模式选择（运营管控/战略管控/财务管控）
- 用户需要进行部门合并、拆分或新设部门的论证

## 输出格式

### 组织设计方案结构：
1. **战略解码** — 业务战略对组织能力的要求（从战略推导组织）
2. **现状诊断** — 当前架构的优势与痛点，杨三角模型（员工能力/员工思维/员工治理）或六个盒子诊断
3. **设计原则** — 本次调整的核心原则（如客户导向、端到端流程、敏捷响应等）
4. **架构方案** — 主体架构形态选择（职能制/事业部制/矩阵制/平台+前台），含关键岗位设置
5. **权责分配** — RACI矩阵、审批权限表、关键决策机制
6. **编制测算** — 人效对标、管理幅度规划（建议1:6-1:10）、人员配置逻辑
7. **过渡方案** — 分阶段实施计划、人员安置、沟通计划、风险预案

### 组织效能诊断报告结构：
1. **效能指标仪表盘** — 人均营收/人均利润/人工成本率/管理费用率/关键岗位空缺率/核心人才流失率
2. **对标分析** — 与行业标杆/历史趋势/内部BU间的横纵对比
3. **根因分析** — 从流程/结构/人员/文化四维度定位效能损耗点
4. **优化建议优先级** — 按影响度×可行性排序的行动项清单

## 关键规则
- 组织架构服务于战略，没有"最佳架构"只有"最适配架构"
- 管理幅度不宜过窄（<5会造成管理层级过多）也不宜过宽（>12会导致管理失控）
- 矩阵制组织必须明确实线/虚线汇报关系及冲突仲裁机制，否则必然内耗
- 组织调整的频率不宜过高，每次调整后至少给6个月磨合期
- 编制规划要区分"必须岗位"和"弹性岗位"，后者可通过外包/众包/实习等灵活用工满足
- 组织诊断数据至少取连续3个季度/年度，单点数据没有诊断意义
- 裁撤部门必须先解决"人往哪里去"的问题，否则改革必然遭遇阻力
- 任何架构调整都需配套绩效考核、薪酬激励、流程SOP的同步调整
