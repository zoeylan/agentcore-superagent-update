# 语音技术选型决策

## 概述
为语音产品项目提供全链路技术选型建议，覆盖ASR、NLU、TTS、对话管理、端到端模型等关键技术模块，结合具体场景约束给出最优方案组合。

## 适用场景
- 新建语音系统的技术栈选型
- 现有系统的ASR/TTS引擎替换评估
- 传统级联架构vs端到端架构的决策
- 多语言语音系统的技术方案选择
- 云端vs边缘vs端侧部署方案评估
- 开源vs商业方案的对比分析

## 输入要求
用户需提供以下信息：
- **场景描述**：具体应用场景（客服/车载/硬件/电话等）
- **语言要求**：支持的语种、方言、code-switching需求
- **性能要求**：延迟、准确率、并发量、可用性SLA
- **部署约束**：云端/私有化/边缘/端侧、网络条件
- **成本预算**：API调用预算或自建算力预算
- **合规要求**：数据出境、本地化存储、行业监管
- **团队能力**：技术团队规模和AI/语音领域经验

## 输出内容
1. **技术方案对比矩阵**
   - 各候选方案在性能、成本、可控性、生态等维度的评分
   - 适用场景和不适用场景的明确说明

2. **推荐方案与理由**
   - 首选方案及核心选择理由
   - 备选方案及切换条件
   - 明确标注方案的局限性和风险

3. **全链路技术架构**
   - 前端信号处理：降噪/AEC/波束成形方案
   - ASR选型：Whisper/Paraformer/Azure/讯飞/自研的对比
   - NLU/对话管理：LLM-based vs 传统NLU vs 混合方案
   - TTS选型：CosyVoice/Azure Neural/ElevenLabs/自研的对比
   - 端到端方案：GPT-4o Realtime/Nova Sonic/Moshi的评估

4. **集成方案**
   - 各模块间的接口协议和数据流
   - 流式处理vs批量处理的权衡
   - 降级与容灾策略

5. **演进路径**
   - 从当前方案到目标方案的平滑迁移路径
   - 技术债务的识别和偿还计划
   - 新技术（端到端模型）的引入时机建议

## 核心选型维度

### ASR选型关键指标
| 维度 | 评估要点 |
|------|----------|
| 准确率 | CER/WER在目标场景的实测数据（非公开benchmark） |
| 实时率 | RTF（Real-Time Factor）<0.3为优秀 |
| 首包延迟 | 流式识别的首字输出时间 |
| 热词能力 | 领域术语/人名/产品名的定制能力 |
| 多语言 | 语种覆盖、语言切换、方言支持 |
| 私有化 | 是否支持本地部署、模型压缩能力 |

### TTS选型关键指标
| 维度 | 评估要点 |
|------|----------|
| 自然度 | MOS评分（4.0以上为优秀） |
| 情感表达 | 情绪、语速、停顿的可控性 |
| 流式合成 | 首包延迟和音频连续性 |
| 音色定制 | 少样本克隆/定制音色能力 |
| 多语言 | 跨语言合成、中英混读 |
| 并发性能 | 单GPU并发路数和资源消耗 |

### 端到端模型评估维度
| 维度 | 评估要点 |
|------|----------|
| 实时性 | 端到端延迟、打断响应速度 |
| 可控性 | 输出内容的guardrail、安全限制 |
| 成本 | Token/秒计费模型下的单通成本 |
| 可解释性 | 中间过程是否可观测和调试 |
| 稳定性 | 长对话不跑偏、不重复、不幻觉 |
| 多语言 | 实时翻译、多语言切换能力 |

## 工作方法
- 坚持"场景实测"原则：公开benchmark仅供参考，必须用真实场景数据评测
- 关注"长尾case"：主流场景各家相差不大，差异在噪声/口音/专业术语等长尾
- 评估"工程化成本"：不仅看模型效果，还看API稳定性、文档质量、技术支持
- 考虑"锁定风险"：单一供应商依赖的风险，建议关键模块保留切换能力
- 规划"AB测试"：重要选型通过线上AB实验验证，用数据说话
- 计算"全链路延迟"：单模块延迟达标不代表端到端延迟达标，注意级联延迟累积
