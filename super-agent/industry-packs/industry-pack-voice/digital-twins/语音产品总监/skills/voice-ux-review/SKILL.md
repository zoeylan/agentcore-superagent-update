# 语音交互体验评审

## 概述
对语音交互产品的用户体验进行专业评审，识别体验瓶颈，提供优化建议，确保语音交互的自然度、流畅度和任务完成效率达到行业标准。

## 适用场景
- 语音产品上线前的体验评审
- 用户反馈分析与体验优化
- VUI（Voice User Interface）对话流程设计审查
- 竞品语音体验对标分析
- 多模态交互方案的语音部分评审

## 输入要求
用户需提供以下信息（至少提供其一）：
- **对话流程图/脚本**：包含用户话术、系统回复、分支逻辑
- **体验问题描述**：用户投诉或体验数据异常
- **录音/转写文本**：真实交互的语音录音或ASR转写结果
- **产品PRD/设计文档**：语音交互相关的产品设计文档
- **关键指标数据**：识别率、任务完成率、用户打断率、平均对话轮次等

## 输出内容
1. **体验问题清单**：按严重程度P0-P3分级的体验问题列表
2. **根因分析**：每个问题的技术根因和产品根因拆解
3. **优化建议**：针对每个问题的具体解决方案
4. **对话设计最佳实践**：适用于该场景的VUI设计原则
5. **指标优化目标**：可量化的体验改进目标值

## 评审维度
### 核心体验维度
- **响应速度**：首包延迟、端到端延迟是否符合场景要求
- **打断体验**：用户能否自然打断，打断后系统的恢复策略
- **错误恢复**：ASR误识别时的纠错流程是否自然
- **确认策略**：隐式确认vs显式确认的选择是否合理
- **多轮连贯性**：上下文保持能力，指代消解是否正确
- **兜底设计**：无法识别/无法处理时的降级体验

### 进阶体验维度
- **话术自然度**：系统回复是否像"说话"而非"念稿"
- **信息密度**：语音输出的信息量是否适合听觉通道
- **主动引导**：系统是否在合适时机给出操作引导
- **情感适配**：系统语气是否匹配用户当前情绪状态
- **多模态配合**：语音与屏幕显示的信息是否互补而非冗余

## 工作方法
- 以"3秒原则"审视每个交互节点——用户等待超过3秒即为体验断裂
- 用"妈妈测试"验证话术自然度——你妈妈能否听懂并完成操作
- 关注"最后一公里"——任务完成率比识别率更重要
- 检查"边缘case覆盖"——噪声环境、方言口音、儿童/老人语音
- 对标行业标杆数据：通用ASR准确率>95%、首包延迟<500ms、任务完成率>80%
