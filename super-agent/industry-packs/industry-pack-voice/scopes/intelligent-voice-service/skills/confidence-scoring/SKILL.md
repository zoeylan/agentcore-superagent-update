# 置信度评估 (Confidence Scoring)

## 概述
对知识检索结果进行多维度置信度评估，判定答案可靠性等级，决定是直接回复、要求澄清还是标记为知识缺口。这是避免"AI幻觉"传播给客户的关键防线。

## 适用场景
- 每次知识检索后的答案质量评估
- 模糊查询场景下的答案可信度判断
- 多候选答案的可靠性对比排序
- 知识缺口发现与记录

## 执行步骤

### 1. 多维评分
对每个检索结果从多个维度计算分数：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 语义匹配度 | 0.35 | 查询与答案的语义相似度 |
| 实体覆盖率 | 0.25 | 查询中的关键实体在答案中的出现率 |
| 知识时效性 | 0.20 | 知识条目的更新时间和有效期 |
| 来源权威度 | 0.10 | 知识来源的可靠性等级 |
| 上下文一致性 | 0.10 | 答案与对话上下文的逻辑一致性 |

### 2. 综合置信度计算
```
confidence = Σ(维度分数 × 权重)
```
归一化到0-1范围。

### 3. 决策分级

| 置信度范围 | 决策 | 行为 |
|-----------|------|------|
| >= 0.85 | 高可信 | 直接回复，标记"精确匹配" |
| 0.6 - 0.85 | 中等可信 | 回复但附加确认语（"您是想问...对吗？"） |
| 0.4 - 0.6 | 低可信 | 不直接回复，推荐相似问题列表 |
| < 0.4 | 无可信答案 | 标记知识缺口，建议转人工或换种方式描述 |

### 4. 知识缺口处理
当置信度<0.4时：
- 记录原始查询到"待补充知识"队列
- 生成推荐相似问题列表（最多3个，从中等置信度结果中选取）
- 向对话编排引擎返回降级信号

### 5. 反馈循环
- 记录用户对答案的反馈（接受/拒绝/追问）
- 追问或拒绝的case自动降低对应知识条目的权威度分
- 长期低分知识条目标记为"需更新"

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| search_results | array | 检索结果列表（含答案+各维度原始分数） |
| query_text | string | 原始查询文本 |
| dialogue_context | object | 对话上下文 |
| knowledge_metadata | object | 知识条目的元数据（时效性、来源等） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| confidence_score | float | 综合置信度0-1 |
| decision | enum | HIGH_CONFIDENCE/MEDIUM/LOW/NO_ANSWER |
| dimension_scores | object | 各维度评分详情 |
| similar_questions | array | 推荐的相似问题（低置信度时） |
| knowledge_gap_flag | bool | 是否为知识缺口 |

## 最佳实践
- 宁可返回"不确定"也不给出错误答案——语音客服场景中错误回答的纠正成本极高
- 对政策条款类查询提高置信度阈值（>=0.9才给出确定答案）
- 综合多个候选结果：如Top1和Top2答案一致，置信度可适当提升
- 知识缺口积累到一定量级应触发知识库更新工单

## 约束
- 评分计算时间<50ms
- 知识缺口误标率<5%（真实有答案的情况被误标为缺口）
- 无答案场景正确降级率>=95%
