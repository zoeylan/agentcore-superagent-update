# 合规标签标注 (Compliance Tagging)

## 概述
对知识检索返回的答案进行合规性审查，标注需要附加的合规声明（风险提示、免责声明、监管要求话术），确保客服回复满足行业监管要求。

## 适用场景
- 金融产品咨询时的风险提示附加
- 医疗/健康信息回复时的免责声明
- 保险产品条款解释时的除外责任说明
- 涉及个人隐私信息处理时的告知义务
- 投资理财建议时的"不构成投资建议"声明

## 执行步骤

### 1. 行业合规规则库维护
按行业维护合规规则集：

**金融行业**：
- 涉及投资/理财产品 → 必须附加"投资有风险，过往业绩不代表未来收益"
- 涉及贷款产品 → 必须告知年化利率
- 涉及保险理赔 → 必须说明除外责任

**医疗行业**：
- 任何健康建议 → 附加"建议咨询专业医生，本信息仅供参考"
- 用药指导 → 附加"请遵医嘱用药"
- 诊断相关 → 不可给出诊断结论，仅提供就医建议

**电商行业**：
- 促销信息 → 标明活动有效期和限制条件
- 退换政策 → 完整告知退换条件和时限
- 价格承诺 → 标明"最终以实际结算为准"

### 2. 自动标签检测
对检索答案进行NLP分析：
- 识别答案涉及的行业领域
- 检测是否包含需要合规声明的关键词或主题
- 匹配对应的合规规则

### 3. 合规声明生成
根据匹配到的规则生成标准化声明：
- 声明文本从预置模板库中选取（不可AI随意生成）
- 声明必须在回复末尾或关键信息后立即附加
- 语音场景下声明需口语化但保持完整性

### 4. 合规完整性校验
- 检查生成的回复中是否已包含所有必要的合规声明
- 对缺失的声明输出告警
- 强制合规标签：标记为"mandatory"的声明不可被跳过或省略

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| answer_text | string | 待标注的答案文本 |
| business_domain | string | 业务领域标签 |
| product_category | string | 产品类别 |
| user_query_intent | string | 用户查询意图 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| compliance_tags | array | 命中的合规标签列表 |
| disclaimers | array | 需附加的声明文本列表 |
| modified_answer | string | 附加声明后的完整答案 |
| compliance_level | enum | MANDATORY/RECOMMENDED/NONE |
| missing_declarations | array | 缺失的强制声明（告警） |

## 最佳实践
- 合规声明用自然的连接词引入（"另外提醒您..."），避免生硬割裂
- 多个合规声明时按重要性排序，最重要的紧跟答案
- 政策更新时第一时间同步更新合规规则库
- 对合规声明是否实际播出进行事后校验（质检环节）

## 约束
- 强制合规标签覆盖率100%（不可遗漏）
- 合规检查增加延迟<30ms
- 合规规则库更新后30分钟内全系统生效
