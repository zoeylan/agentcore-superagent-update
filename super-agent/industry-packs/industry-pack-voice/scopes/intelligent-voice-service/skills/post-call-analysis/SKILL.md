# 通话事后分析 (Post-Call Analysis)

## 概述
在通话结束后对完整对话记录进行深度质检分析，生成多维度质检评分报告。覆盖服务态度、问题解决、合规三个核心维度，输出结构化评分和改进建议。

## 适用场景
- 每通通话结束后的自动质检
- 定期批量质检报告生成
- 低评分通话的问题诊断
- 坐席绩效评估数据支撑
- 培训需求识别

## 执行步骤

### 1. 数据聚合
通话结束后收集完整分析数据：
- 完整对话转写文本（含时间戳和说话人标注）
- 情绪轨迹数据（情绪感知引擎提供）
- 实时合规监控的事件记录
- 业务操作日志（如有）
- 用户最终满意度反馈（如有）

### 2. 服务态度评分（0-100）
评估维度：
| 子项 | 权重 | 评分标准 |
|------|------|----------|
| 语言礼貌度 | 25% | 是否使用敬语、避免负面词汇 |
| 情绪管理 | 25% | 面对用户不满时的应对表现 |
| 主动性 | 25% | 是否主动提供帮助和方案 |
| 耐心程度 | 25% | 是否耐心解释、不催促用户 |

扣分规则：
- 使用否定词开头回应：-5分/次
- 打断用户说话（非必要）：-10分/次
- 情绪失控（回怼/冷漠）：-20分/次
- 主动提供额外帮助：+5分/次

### 3. 问题解决评分（0-100）
评估维度：
| 子项 | 权重 | 评分标准 |
|------|------|----------|
| 首次解决 | 40% | 问题是否在本次通话内完全解决 |
| 方案准确性 | 30% | 提供的解决方案是否正确有效 |
| 信息完整性 | 20% | 是否提供了用户需要的所有信息 |
| 后续跟进 | 10% | 未解决问题是否有明确的跟进计划 |

### 4. 合规评分（0-100）
基于实时合规监控的累计数据：
- 必说话术完整性：每遗漏一项扣10分
- 禁止话术违规：每次扣20分
- 敏感信息泄露：每次扣30分
- 额外合规优秀表现（如主动风险提示）：+5分

### 5. 综合评级
```
综合分 = 服务态度×0.3 + 问题解决×0.4 + 合规×0.3
```

| 综合分 | 等级 | 处理 |
|--------|------|------|
| >=90 | A | 优秀，可作为培训范例 |
| 75-89 | B | 良好，正常通过 |
| 60-74 | C | 一般，标记改进点 |
| <60 | D | 不合格，需人工复核+培训 |

### 6. 改进建议生成
对B级以下通话自动生成改进建议：
- 具体指出扣分环节和原因
- 引用对话原文作为证据
- 给出参考话术或处理方式建议
- 关联推荐培训课程

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| call_transcript | array | 完整对话转写 |
| emotion_trajectory | array | 情绪轨迹数据 |
| compliance_events | array | 合规事件记录 |
| business_operations | array | 业务操作日志 |
| call_metadata | object | 通话元数据（时长、来源、业务线） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| overall_grade | enum | A/B/C/D |
| overall_score | int | 综合评分0-100 |
| dimension_scores | object | 三维度分项评分 |
| deduction_details | array | 扣分明细（含原文引用） |
| improvement_suggestions | array | 改进建议列表 |
| highlight_moments | array | 优秀/问题时刻标记 |
| need_review | bool | 是否需要人工复核 |

## 最佳实践
- 所有扣分必须有对话原文引用，不可无据扣分
- D级通话必须进入人工复核队列，不可仅凭自动评分定性
- 评分模型需定期校准（对比人工评分和自动评分的一致性）
- 优秀通话同样标记，作为正向案例库积累

## 约束
- 质检报告生成时间<通话时长的10%
- 通话质检自动化率>=80%
- 评分与人工质检评分的相关系数>=0.85
