# 风险检测 (Risk Detection)

## 概述
在通话过程中实时识别高风险事件，包括客户投诉升级、业务操作错误、法律纠纷征兆、敏感信息泄露等，并根据风险等级即时触发告警和干预措施。

## 适用场景
- 客户情绪从不满升级为投诉意图
- AI回复可能包含错误信息
- 涉及法律/监管敏感话题
- 检测到欺诈或社工攻击征兆
- 异常通话模式识别

## 执行步骤

### 1. 风险信号监测
持续监测以下风险维度：

**投诉升级风险**：
- 关键词触发："投诉"、"315"、"消费者协会"、"律师"、"起诉"、"曝光"
- 行为模式：连续多次表达不满+明确要求升级处理
- 情绪轨迹：持续愤怒（>60秒）且安抚无效

**业务操作风险**：
- 订单金额异常（超过用户历史均值的5倍）
- 短时间内多次重复操作（可能是误操作）
- 操作与用户表述不一致（说退款但执行了下单）

**信息安全风险**：
- 系统回复中意外包含其他用户信息
- 坐席/AI朗读了完整敏感信息（身份证/银行卡全号）
- 用户被引导提供不应在电话中告知的信息

**欺诈风险**：
- 来电号码与账户绑定号码不一致+要求修改关键信息
- 多次验证失败后换号再试
- 社工话术模式（"我是XX部门的..."）

### 2. 风险等级评估
| 等级 | 标准 | 响应时间 |
|------|------|----------|
| P0-紧急 | 敏感信息泄露已发生、欺诈攻击进行中 | <5秒 |
| P1-高危 | 投诉升级明确意图、大额异常操作 | <30秒 |
| P2-中危 | 持续负面情绪、操作可能有误 | <60秒 |
| P3-低危 | 潜在风险信号但未确认 | 通话结束后标记 |

### 3. 告警与干预
根据风险等级执行对应动作：

**P0处理**：
- 立即终止当前操作
- 向对话编排引擎发送强制暂停信号
- 通知人工坐席立即介入
- 记录风险事件到安全审计日志

**P1处理**：
- 向对话编排引擎发送风险提醒
- 建议切换为人工处理
- 对待执行的操作加入额外确认步骤
- 记录并通知运营负责人

**P2处理**：
- 标记为关注通话
- 对话编排引擎收到后调整策略
- 事后纳入重点质检

### 4. 事件记录与统计
- 完整记录风险事件（时间、类型、等级、处理措施、结果）
- 统计各类风险的发生频率和趋势
- 新增高频风险模式时更新检测规则

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| dialogue_stream | object | 实时对话流数据 |
| emotion_state | object | 当前情绪状态 |
| business_operations | array | 进行中的业务操作 |
| user_profile | object | 用户画像（历史行为基线） |
| call_metadata | object | 来电元数据（号码、时段等） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| risk_events | array | 检测到的风险事件列表 |
| risk_level | enum | P0/P1/P2/P3 |
| intervention_action | string | 建议干预措施 |
| evidence | object | 风险证据（原文引用+时间戳） |
| alert_targets | array | 告警通知目标 |

## 最佳实践
- P0事件宁严勿漏：敏感信息泄露和欺诈宁可误报也不可漏报
- 投诉升级检测结合语义和情绪双重信号，避免单一关键词误触发
- 风险检测规则定期review，根据新出现的风险模式更新
- 对高频误报的规则进行调优，降低告警疲劳

## 约束
- P0事件检测到告警延迟<5秒
- 敏感信息泄露检测准确率>=99%
- 投诉升级预测准确率>=90%
- 假阳性告警率<5%
