# 情绪轨迹追踪 (Emotion Trajectory Tracking)

## 概述
在通话全程维护用户情绪的时间序列变化曲线，检测关键情绪事件（突变、持续恶化、转好），为对话策略调整和质检分析提供完整的情绪演变证据链。

## 适用场景
- 通话全程的情绪时间线记录
- 情绪突变事件检测与告警
- 持续负面情绪趋势识别
- 安抚策略效果评估
- 通话结束后的情绪摘要生成

## 执行步骤

### 1. 轨迹初始化
通话开始时创建情绪轨迹对象：
```json
{
  "session_id": "xxx",
  "start_time": 1700000000,
  "trajectory": [],
  "events": [],
  "current_trend": "stable",
  "angry_count": 0
}
```

### 2. 连续记录
每200ms接收一次情绪分类结果并追加到轨迹：
- 记录时间戳、情绪标签、强度评分
- 维护滑动窗口统计（最近5秒、最近30秒的情绪分布）

### 3. 突变检测
定义突变阈值并实时监控：
- **正向突变**：强度评分在1秒内下降>40分（负面→正面）
- **负向突变**：强度评分在1秒内上升>40分（正面→负面/愤怒）
- **愤怒突变**：从任意非愤怒状态直接跳变为愤怒且强度>70

触发突变事件时立即上报：
```json
{
  "event_type": "emotion_spike",
  "direction": "negative",
  "from_state": {"label": "neutral", "intensity": 30},
  "to_state": {"label": "angry", "intensity": 85},
  "timestamp": 1700000120,
  "trigger_context": "当前对话内容摘要"
}
```

### 4. 趋势分析
基于最近30秒的情绪分布计算趋势：
- **恶化趋势**：负面/愤怒占比持续上升（连续3个10秒窗口递增）
- **改善趋势**：正面/中性占比持续上升
- **波动**：情绪标签频繁变化无明确方向
- **稳定**：情绪标签90%以上保持一致

### 5. 愤怒计数与升级
维护愤怒情绪专项计数器：
- 检测到愤怒标签（强度>60，持续>2秒）时计数+1
- 第1次：输出"高优安抚建议"信号
- 第2次：输出"主动提供解决方案"信号
- 第3次：输出"强制转人工"升级信号，附带完整轨迹摘要

### 6. 通话结束摘要
生成情绪轨迹摘要报告：
- 通话情绪总体评价（正面主导/中性平稳/负面倾向/情绪激动）
- 关键事件时间线
- 安抚策略实施点及效果评估
- 情绪评分：开始/最差/结束三点对比

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| emotion_label | enum | 当前帧情绪标签 |
| intensity_score | int | 情绪强度0-100 |
| timestamp | int64 | 时间戳 |
| dialogue_snippet | string | 当前对话片段（可选，用于事件上下文） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| trajectory_update | object | 轨迹更新状态 |
| events | array | 新检测到的事件列表 |
| current_trend | enum | IMPROVING/STABLE/WORSENING/VOLATILE |
| escalation_signal | object | 升级信号（需要转人工时） |
| summary | object | 通话结束时的轨迹摘要 |

## 最佳实践
- 突变检测要过滤掉瞬时噪声：单帧愤怒不计入愤怒计数，需持续>2秒
- 安抚策略生效后给予5-10秒观察期再判断效果
- 轨迹数据是质检的重要客观依据，需完整保留不可丢弃
- 对VIP客户的愤怒阈值可适当降低（更早介入安抚）

## 约束
- 事件检测延迟<500ms（从情绪变化发生到事件上报）
- 轨迹数据存储精度：200ms粒度保留通话全程
- 愤怒转人工触发准确率>=95%
