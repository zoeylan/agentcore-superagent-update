# 多Agent任务编排 (Multi-Agent Orchestration)

## 概述
当用户提出跨领域复杂请求时，将任务拆解为多个子任务并行或串行调度给专业Agent执行，最终将多个执行结果整合为连贯一致的语音回复。这是实现"一句话解决复杂问题"体验的核心能力。

## 适用场景
- 用户一次性提出涉及多个业务领域的问题
- 某个问题的回答需要多个知识源的综合信息
- 业务操作流程中需要跨系统协调
- 信息查询+操作执行的组合请求

## 执行步骤

### 1. 任务拆解
分析跨领域意图，拆分为原子子任务：
- 识别每个子任务的业务领域归属
- 判断子任务间的依赖关系（是否有执行顺序要求）
- 为每个子任务分配执行Agent和超时时间

### 2. 调度执行
根据依赖关系确定执行策略：
- **并行策略**：无依赖关系的子任务同时下发给对应Agent
- **串行策略**：有依赖的子任务按顺序执行（前一个结果作为后一个输入）
- **混合策略**：部分并行+部分串行的DAG执行图

### 3. 结果收集与超时处理
- 设置每个子任务的超时时间（默认3秒）
- 超时未返回的子任务标记为"部分完成"
- 收集所有已完成子任务的结果

### 4. 结果整合
将多个Agent的返回结果整合为一个连贯回复：
- 按用户原始提问的顺序组织回答
- 使用过渡词连接不同领域的答案（"关于您的订单...另外关于积分..."）
- 对冲突结果（不同Agent给出矛盾信息）进行标注并选择高置信度答案
- 控制最终回复长度（语音场景下单次回复不超过30秒朗读时长）

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| sub_intents | array | 拆解后的子意图列表 |
| dependency_graph | object | 子任务间的依赖关系 |
| context | object | 共享的对话上下文 |
| timeout_ms | int | 总超时时间 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| integrated_response | string | 整合后的完整回复文本 |
| task_results | array | 各子任务的详细执行结果 |
| completion_status | enum | FULL/PARTIAL/FAILED |
| failed_tasks | array | 失败的子任务列表及原因 |

## 最佳实践
- 子任务数量不超过5个，超过时对低优先级任务进行合并或延后
- 并行执行时为每个子任务设置独立超时，避免一个慢任务阻塞整体
- 结果整合时注意去重（多个Agent可能返回重复信息）
- 部分完成时先回复已有结果，告知用户剩余部分正在处理

## 约束
- 任务编排总耗时（含所有子任务执行）P95<3秒
- 单轮回复文本不超过200字（约30秒语音朗读时长）
- 同时并行的子任务不超过5个
