# 实时合规监控 (Realtime Compliance Monitoring)

## 概述
在通话过程中实时监控对话流，检查必说话术（开场白、风险提示、结束语等）是否在规定环节被触发，检测敏感信息泄露和合规违规行为，并在发现问题时即时发出提醒信号。

## 适用场景
- 每通通话的全程实时合规监控
- 必说话术触发检查（开场、风险提示、结束语）
- 敏感信息泄露检测
- 违规用语/承诺的实时拦截
- 行业特定合规要求的在线校验

## 执行步骤

### 1. 合规规则加载
通话开始时根据业务线加载对应合规规则集：
```json
{
  "mandatory_scripts": {
    "opening": {"trigger": "call_start", "deadline_seconds": 30, "template": "您好，这里是..."},
    "risk_warning": {"trigger": "financial_product_mentioned", "deadline_seconds": 10},
    "closing": {"trigger": "call_ending", "deadline_seconds": 15, "template": "请问还有其他..."}
  },
  "forbidden_patterns": ["绝对保证", "一定赚钱", "没有风险"],
  "sensitive_info_patterns": ["身份证号全文", "银行卡号全文", "密码"]
}
```

### 2. 流式文本匹配
对每轮对话文本进行实时匹配：
- **必说话术检查**：跟踪每个必说话术的触发条件和截止时间
  - 条件触发但到期未说 → 输出提醒信号
  - 条件触发且已说 → 标记为已完成
- **禁止话术检测**：正则匹配禁止用语
  - 匹配命中 → 立即输出告警
- **敏感信息检测**：模式匹配敏感数据格式
  - 检测到完整身份证号/银行卡号被朗读 → 高优告警

### 3. 实时提醒输出
检测到合规风险时向对话编排引擎发送实时提醒：
```json
{
  "alert_type": "compliance_reminder",
  "severity": "high",  // high/medium/low
  "message": "请说出风险提示话术",
  "rule_id": "RULE-FIN-003",
  "deadline_remaining_seconds": 5
}
```

### 4. 合规状态追踪
维护本通通话的合规状态看板：
- 各必说话术的完成状态（✓已完成 / ⚠待触发 / ✗超时未说）
- 禁止话术违规次数
- 敏感信息泄露事件数
- 实时合规评分（100分起扣）

### 5. 通话结束合规摘要
通话结束时输出合规检查结果：
- 必说话术完成清单
- 违规事件列表（含时间戳和原文）
- 本通通话合规评分
- 是否需要人工复核标记

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| dialogue_turn | object | 当前轮对话（说话人+文本+时间戳） |
| call_phase | enum | 当前通话阶段（开始/中间/结束） |
| business_events | array | 业务事件流（如提及金融产品） |
| elapsed_time | int | 通话已进行时间(秒) |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| alerts | array | 实时告警列表 |
| compliance_status | object | 各规则项完成状态 |
| realtime_score | int | 实时合规评分 |
| violations | array | 已检测到的违规列表 |

## 最佳实践
- 提醒信号要比截止时间提前5秒发出，给坐席/系统留出反应时间
- 对"近似违规"（如"基本上没有风险"）也应标记为中级告警
- 合规规则按行业和业务线精准匹配，避免无关规则的误告警
- 高频告警的通话自动升级为"重点关注"

## 约束
- 实时检测延迟<500ms（从文本输入到告警输出）
- 合规话术遗漏检出率>=98%
- 敏感信息泄露检测准确率>=99%
- 假阳性率<3%
