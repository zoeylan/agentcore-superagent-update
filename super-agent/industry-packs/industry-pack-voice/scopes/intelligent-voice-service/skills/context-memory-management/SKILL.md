# 上下文记忆管理 (Context Memory Management)

## 概述
管理智能语音客服系统的多层次记忆体系，包括通话内短期记忆（对话上下文栈）和跨通话长期记忆（24小时会话历史），实现用户再次来电时的无缝衔接体验。

## 适用场景
- 同一通话中多轮对话的上下文承接
- 用户24小时内再次来电时自动恢复上次对话上下文
- 对话中的指代消解（"刚才那个"、"上次说的"）
- 通话结束时的上下文摘要生成和持久化

## 执行步骤

### 1. 记忆加载
来电接入时执行记忆查询：
- 基于来电号码/用户ID查询24小时内的历史会话
- 若存在历史会话，加载其上下文摘要（而非完整对话记录）
- 向用户确认是否延续上次话题（"您好，上次您咨询了退款问题，请问是否需要继续处理？"）

### 2. 实时上下文维护
通话过程中持续更新上下文：
- **实体记忆**：记录所有已确认的实体信息（订单号、产品名、日期等）
- **意图记忆**：记录对话中出现的所有意图及其解决状态
- **情绪记忆**：记录情绪变化关键节点
- **操作记忆**：记录已执行的业务操作和结果

### 3. 指代消解
当用户使用代词或省略表达时：
- 在上下文栈中搜索最近的匹配实体
- 时间代词："上次"→上一通通话，"刚才"→当前通话早期
- 指示代词："那个订单"→上下文中最近提到的订单
- 省略表达："还有呢"→继续上一个问题的回答

### 4. 会话摘要生成
通话结束时自动生成结构化摘要：
```json
{
  "user_issues": ["用户提出的问题列表"],
  "resolved_issues": ["已解决的问题"],
  "pending_issues": ["未解决的问题"],
  "key_entities": {"订单号": "xxx", "产品": "yyy"},
  "last_emotion": "neutral",
  "suggested_followup": "建议的后续跟进动作"
}
```

### 5. 记忆清理与压缩
- 24小时过期的会话摘要自动清理
- 超过20轮的对话上下文执行压缩（保留关键信息，移除冗余）
- VIP用户的记忆保留期可延长至7天

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | string | 用户标识 |
| session_id | string | 当前会话ID |
| memory_operation | enum | LOAD/UPDATE/RESOLVE_REFERENCE/SUMMARIZE |
| payload | object | 操作相关数据 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| memory_content | object | 返回的记忆内容 |
| resolution_result | object | 指代消解结果 |
| summary | object | 会话摘要（结束时） |

## 最佳实践
- 记忆加载只加载摘要而非完整历史，控制Token消耗
- 指代消解时优先就近匹配，距离越远置信度越低
- 会话摘要聚焦于"未解决问题"，这是下次来电的核心上下文
- 敏感信息（身份证号、银行卡号）在摘要中做脱敏处理

## 约束
- 历史上下文加载延迟<200ms
- 摘要生成成功率>=99%
- 关键信息提取完整性>=95%
- 单会话上下文序列化大小<64KB
