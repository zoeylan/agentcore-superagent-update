# 意图路由 (Intent Routing)

## 概述
对用户语音转写文本进行快速意图分类，并将请求路由到对应的处理Agent或知识域，是对话编排引擎的核心决策能力。要求在50ms内完成路由决策，准确率>=90%。

## 适用场景
- 用户发起新的请求或提问
- 对话中途用户切换话题
- 追问后用户澄清了意图
- 多意图混合表达的拆分

## 执行步骤

### 1. 意图预分类
接收ASR转写文本和对话上下文，执行快速分类：
- **单领域意图**：明确指向单一业务领域的请求（如"查询我的账单"）
- **跨领域复杂意图**：涉及多个业务领域（如"我的订单什么时候到，顺便帮我看看积分够不够换礼品"）
- **闲聊**：非业务性对话（如"今天天气真好"）
- **无法识别**：语义不清、噪声干扰或完全不相关内容

### 2. 意图细分与实体提取
对单领域意图进一步细化：
- 查询类（账单/订单/物流/产品信息）
- 操作类（退款/改签/取消/修改信息）→ 路由至业务闭环Scope
- 投诉类（服务不满/质量问题/费用争议）
- 咨询类（产品功能/活动规则/政策说明）

### 3. 路由决策
根据意图分类确定处理路径：
| 意图类型 | 路由目标 | 优先级 |
|----------|----------|--------|
| 单领域查询/咨询 | knowledge-retrieval Agent | P2 |
| 操作类意图 | 业务闭环Scope的flow-guide Agent | P1 |
| 跨领域复杂 | 任务拆解→多Agent并行 | P1 |
| 投诉类 | knowledge-retrieval + 情绪升级策略 | P0 |
| 闲聊 | 内置闲聊回复模板 | P3 |
| 无法识别 | 追问策略 | P2 |

### 4. 多意图拆解
当检测到跨领域复杂意图时：
- 识别意图边界（如"顺便"、"另外"等连接词）
- 拆分为独立子意图
- 为每个子意图分配路由目标
- 确定执行顺序（有依赖关系的串行，无依赖的并行）

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| transcribed_text | string | ASR转写文本 |
| dialogue_context | object | 当前对话上下文（历史意图、已确认实体） |
| emotion_label | string | 当前情绪标签（影响路由优先级） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| intent_category | enum | 意图大类 |
| intent_detail | string | 意图细分标签 |
| entities | array | 提取的关键实体列表 |
| routing_target | string | 路由目标Agent |
| confidence | float | 分类置信度 |
| sub_intents | array | 拆解的子意图列表（跨领域时） |

## 最佳实践
- 结合对话上下文进行意图消歧（"取消"在订单对话中指取消订单，在闲聊后可能指取消当前话题）
- 高情绪状态下的投诉类意图自动提升优先级
- 对置信度<0.7的分类结果启动追问确认而非直接路由
- 维护热点意图缓存，对高频意图（如"查快递"）直接走快速路径

## 约束
- 路由决策总耗时<50ms
- 意图分类准确率>=90%
- 追问次数上限3次，超过必须转人工
