# 升级策略 (Escalation Strategy)

## 概述
当检测到用户情绪恶化或服务无法满足需求时，制定和执行对话升级策略。包括话术调整建议、安抚方案推荐、以及转人工决策的时机判断和交接信息准备。

## 适用场景
- 检测到负面情绪时的话术策略切换
- 愤怒情绪累计触发转人工流程
- 连续追问无法识别意图时的降级处理
- 业务操作失败后的挽回策略

## 执行步骤

### 1. 升级级别判定
根据情绪状态和对话上下文确定升级级别：

| 级别 | 触发条件 | 响应策略 |
|------|----------|----------|
| L1-话术调整 | 负面情绪(强度30-60) | 切换安抚话术模板，语速降低10% |
| L2-重点安抚 | 负面情绪(强度>60)或首次愤怒 | 高优安抚+主动提供解决方案 |
| L3-升级关注 | 第二次愤怒检测 | 表达重视+承诺解决+告知可转人工 |
| L4-强制转人工 | 第三次愤怒或极端情况 | 立即转人工，附带完整上下文 |

### 2. 安抚话术生成
根据升级级别和业务场景输出话术建议：
- **通用安抚**："非常抱歉给您带来不便，我来帮您解决这个问题"
- **等待安抚**："请您稍等片刻，我正在为您查询最新信息"
- **共情安抚**："我非常理解您的心情，如果是我遇到这种情况也会着急"
- **承诺安抚**："我一定会帮您处理好这个问题，请放心"
- **升级安抚**："为了更好地解决您的问题，我帮您转接专业人员"

### 3. TTS参数调整建议
情绪适配的语音合成参数：
- 负面情绪时：语速降低15%，基频降低5%，增加停顿
- 愤怒场景时：语速降低20%，基频降低10%，语调更加平缓沉稳
- 禁止：不得在用户愤怒时使用过于活泼的语调

### 4. 转人工交接准备
L4级别触发时，准备交接包：
```json
{
  "transfer_reason": "客户愤怒情绪第3次触发",
  "emotion_summary": "通话中情绪从中性恶化为愤怒，安抚2次未见好转",
  "dialogue_summary": "用户反映订单退款未到账，已查询3次...",
  "key_entities": {"订单号": "xxx", "问题类型": "退款"},
  "emotion_trajectory": [...],
  "suggested_handling": "建议先表达歉意，优先处理退款问题"
}
```

### 5. 效果追踪
策略执行后追踪效果：
- 话术调整后5-10秒观察情绪变化
- 情绪改善→恢复正常流程
- 情绪未改善或继续恶化→升级到下一级别

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| emotion_state | object | 当前情绪状态（标签+强度+趋势） |
| angry_count | int | 愤怒累计次数 |
| dialogue_context | object | 对话上下文摘要 |
| business_scenario | string | 当前业务场景标签 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| escalation_level | enum | L1/L2/L3/L4 |
| speech_suggestion | string | 推荐安抚话术 |
| tts_params | object | TTS参数调整建议 |
| transfer_required | bool | 是否需要转人工 |
| transfer_package | object | 转人工交接信息包 |

## 最佳实践
- 安抚话术要因场景而异：退款问题强调"已在处理"，质量问题强调"理解您的不满"
- 避免过度道歉（连续多次"对不起"会显得无能），改为行动承诺
- 转人工要提前通知用户（"为了更好地解决您的问题..."），不可突然静默转接
- L4转人工后要将通话上下文完整传递，避免用户重复描述问题

## 约束
- 第三次愤怒必须触发转人工，不可降级处理
- 转人工交接包生成时间<1秒
- 安抚策略建议输出延迟<100ms
