# 混合检索 (Hybrid Search)

## 概述
结合语义向量检索（Semantic Search）和关键词BM25匹配的双通道检索策略，通过Re-ranking融合排序实现高精度的知识库答案检索。确保Top3命中率>=85%。

## 适用场景
- 用户业务问题的答案检索
- FAQ库的变体匹配（同一问题不同表达方式）
- 产品文档中的精确信息定位
- 政策条款的精确检索（不可改写）

## 执行步骤

### 1. 查询预处理
接收对话编排引擎的查询请求：
- 提取核心查询语句（去除语气词、填充词）
- 识别查询类型：事实型/操作型/比较型/确认型
- 扩展同义词（如"退款"="退钱"="把钱退回来"）
- 提取关键实体（订单号、产品名等）用于精确过滤

### 2. 双通道并行检索

**通道A - 语义向量检索**：
- 将查询文本编码为语义向量（维度768/1024）
- 在向量索引中执行ANN最近邻搜索
- 返回Top20候选，带相似度分数

**通道B - BM25关键词匹配**：
- 对查询进行分词
- 在倒排索引中执行BM25匹配
- 返回Top20候选，带BM25分数
- 对包含精确实体匹配（如订单号）的结果加权boost

### 3. 结果融合与Re-ranking
- 合并两个通道的候选集（去重）
- 使用Cross-Encoder Re-ranker对候选进行精排
- 考虑因素：语义相关性、实体匹配度、知识时效性、来源权威度
- 输出最终排序的Top5结果

### 4. 答案适配
对Top1结果进行语音场景适配：
- 将书面语答案转换为口语化表达
- 长答案分段处理（适合TTS逐段朗读）
- 政策条款类答案保持原文精确性
- 添加必要的补充说明（如有效期、适用条件）

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| query_text | string | 用户查询文本 |
| query_type | enum | 查询类型 |
| entities | array | 已提取的实体列表 |
| knowledge_scope | string | 检索范围限定（业务线/产品线） |
| context | object | 对话上下文（辅助消歧） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| results | array | Top5检索结果（含答案文本+来源+分数） |
| best_answer | string | 口语化适配后的最佳答案 |
| search_channels | object | 各通道的匹配详情 |
| query_expansion | array | 查询扩展词列表 |

## 最佳实践
- 对精确实体查询（如订单号）优先使用BM25通道，语义检索容易丢失精确匹配
- 对模糊表达（"那个优惠活动"）依赖语义检索+对话上下文消歧
- Re-ranking阶段要考虑知识时效性：过期知识即使相似度高也应降权
- 检索失败时返回相似问题列表而非空结果

## 约束
- 检索总耗时<300ms（含两通道并行+Re-ranking）
- Top3命中率>=85%
- 支持百万级知识条目规模
