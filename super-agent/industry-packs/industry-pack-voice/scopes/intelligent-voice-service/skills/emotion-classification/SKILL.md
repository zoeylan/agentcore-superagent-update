# 情绪分类 (Emotion Classification)

## 概述
基于声学特征向量对用户当前情绪状态进行四级分类（正面/中性/负面/愤怒），并输出0-100的强度评分。要求分类准确率>=85%，延迟<200ms。

## 适用场景
- 每个分析窗口（200ms）的实时情绪判定
- 通话中用户情绪变化的连续追踪
- 为对话策略调整提供情绪依据
- 为质检报告提供情绪标注数据

## 执行步骤

### 1. 特征输入
接收声学特征分析输出的14维特征向量，以及辅助信息：
- 对话文本语义情感（如有）
- 历史3秒内的情绪标签序列
- 用户基线特征（首次来电前10秒的平均特征）

### 2. 多模态融合（可选）
当语义情感信息可用时：
- 声学情绪权重：0.6
- 语义情绪权重：0.4
- 冲突处理：当声学判定为负面但语义为正面（可能是反讽），以声学为准

### 3. 分类推理
使用预训练情绪分类模型进行推理：
- **正面（Positive）**：语速正常、语调上扬、音量适中、无异常停顿
- **中性（Neutral）**：所有特征在正常基线范围内
- **负面（Negative）**：语速偏快或偏慢、语调下降、停顿增多
- **愤怒（Angry）**：语速明显加快、音量显著升高、F0大幅波动、停顿减少

### 4. 强度评分
在分类基础上计算0-100强度评分：
- 0-30：轻微（情绪信号弱，可能误判）
- 31-60：中等（情绪信号明确）
- 61-80：强烈（情绪特征突出）
- 81-100：极端（多维特征同时达到极值）

### 5. 置信度校准
输出分类置信度：
- 置信度>0.85：高可信分类
- 置信度0.6-0.85：中等可信，可参考但需结合上下文
- 置信度<0.6：低可信，建议延迟判定（等待更多帧数据）

### 6. 时序平滑
- 对连续帧的情绪标签进行平滑处理
- 规则：情绪升级（从低到高）立即生效；情绪降级需连续3帧确认
- 防止因单帧噪声导致情绪标签频繁跳变

## 输入规格
| 字段 | 类型 | 说明 |
|------|------|------|
| feature_vector | float[] | 14维声学特征向量 |
| text_sentiment | string | ASR文本的语义情感（可选） |
| emotion_history | array | 最近3秒的情绪标签序列 |
| user_baseline | float[] | 用户基线特征向量 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| emotion_label | enum | POSITIVE/NEUTRAL/NEGATIVE/ANGRY |
| intensity_score | int | 情绪强度0-100 |
| confidence | float | 分类置信度0-1 |
| contributing_features | array | 主要贡献特征列表 |

## 最佳实践
- 为不同行业场景校准模型：客诉场景基线负面情绪高于咨询场景
- 用户首次来电的前10秒作为个体化基线，后续判断基于偏移量
- 注意区分"表达习惯性大声"和"真实愤怒"——前者F0稳定，后者F0波动大
- 老年用户的语速偏慢不应被误判为负面情绪

## 约束
- 分类延迟<200ms（从音频帧输入到标签输出）
- 四级分类准确率>=85%
- 误将中性/正面判为愤怒的假阳性率<3%
