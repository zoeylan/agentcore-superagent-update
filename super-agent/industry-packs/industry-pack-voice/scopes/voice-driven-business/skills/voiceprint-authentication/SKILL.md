# 声纹认证 (Voiceprint Authentication)

## 概述

本技能负责通过声纹生物特征进行用户身份验证，支持静默验证（后台自动比对）和显式验证（要求用户说特定内容进行比对）两种模式。声纹验证是语音驱动业务场景中最自然的身份认证方式——用户无需额外操作，在正常对话中即可完成身份确认。

## 适用场景

- 低风险操作的静默身份验证（用户无感知）
- 中风险操作的显式声纹验证（要求说特定语句）
- 声纹注册和更新流程
- 声纹异常检测（疑似代替操作或录音攻击）

## 执行步骤

### Step 1: 声纹数据采集
- **静默模式**：从用户当前对话音频中截取有效语音段（至少3秒有效语音）
- **显式模式**：播放验证引导语（"为了保护您的账户安全，请说：今天天气真好"）后录制用户响应
- 音频预处理：去噪、VAD裁剪无效段、采样率标准化（16kHz/16bit）
- 确保采集质量：信噪比>=15dB，有效语音时长>=3秒

### Step 2: 特征提取
- 从音频中提取声纹特征向量（基于深度学习的Speaker Embedding）
- 特征维度：256维/512维浮点向量
- 应用反欺骗检测（Anti-spoofing）：
  - 检测是否为录音回放（回放攻击检测）
  - 检测是否为TTS合成语音（合成攻击检测）
  - 检测是否为语音转换（变声攻击检测）
- 反欺骗检测未通过：立即拒绝并记录安全事件

### Step 3: 声纹比对
- 从安全存储中加载用户注册声纹模板（加密存储，仅在内存中解密）
- 计算当前特征与模板特征的余弦相似度
- 得分映射为置信度百分比（0-100%）
- 多模板支持：用户可能有多个注册时的声纹样本，取最高匹配分

### Step 4: 决策判定
- 置信度 >= 95%：验证通过，生成verification_token
- 置信度 90%-95%：标记为"边界通过"，建议升级验证方式
- 置信度 70%-90%：验证不通过，要求升级到更高验证方式
- 置信度 < 70%：高度怀疑非本人，直接拒绝并触发安全告警
- 反欺骗检测失败：无论置信度多高一律拒绝

### Step 5: 结果处理与日志
- 通过：生成有限时效的verification_token（5分钟）
- 未通过：返回需升级的验证方式建议
- 记录验证日志：时间、方式、置信度、结果、设备信息
- 更新用户声纹历史记录（用于检测声纹渐变趋势）

## 输入规格

```json
{
  "verification_mode": "silent | explicit",
  "user_id": "用户唯一标识",
  "audio_data": "Base64编码的音频数据 或 音频流引用",
  "audio_metadata": {
    "sample_rate": 16000,
    "channels": 1,
    "duration_seconds": 5.2,
    "snr_db": 22
  },
  "operation_risk_level": "low | medium | high",

```

## 最佳实践

1. **静默优先**：低风险场景优先使用静默验证，减少用户打断感
2. **渐进升级**：声纹置信度不足时先升级到口头密码，而非直接跳到OTP
3. **环境适应**：在噪声环境中适当降低阈值要求（从95%到93%）同时强制补充其他验证
4. **模板更新**：验证成功且置信度>98%时，用当前特征增量更新模板（适应声纹自然变化）
5. **攻击防范**：连续多次精确匹配但反欺骗得分边界时，可能是高质量深伪语音攻击

## 约束条件

- 声纹数据全程加密，不出本Agent边界
- 验证耗时不超过3秒（含特征提取和比对）
- 声纹模板存储必须符合生物信息安全法规（加密存储、访问控制、定期审计）
- 单次会话最多进行3次声纹验证尝试
- 声纹模板保留期限遵循用户协议，用户可随时要求删除
- 反欺骗检测的误拒率(FRR)必须<1%，防攻击率必须>99%
