# 自然语言解析 (Natural Language Parsing)

## 概述

自然语言解析技能将用户的口语化设备控制指令转化为结构化的控制命令。区别于通用NLU，本技能针对智能硬件控制场景深度优化，处理中文口语的省略、指代、同义表达和方言变体，支持将"太暗了"、"亮一点"、"给我开个灯"等千变万化的自然表达准确映射到明确的设备+操作+参数三元组。

## 适用场景

- 明确设备控制指令的解析（"关闭客厅灯""空调调到26度"）
- 口语化/省略式指令的补全（"亮一点""冷了""关掉"）
- 数值参数的自然语言理解（"调大一些"→+20%、"开到最大"→100%）
- 多指令连说的切分（"把灯关了空调也关了"→两条独立指令）
- 否定/取消指令的识别（"不要关灯""算了不开了"）

## 执行步骤

### Step 1: 文本预处理
1. 接收ASR转写文本（可能含识别噪声和标点缺失）
2. 去除语气词和填充词（"嗯""那个""就是"）
3. 处理ASR常见错误纠正（基于设备名称词典的后纠正，如"可听等"→"客厅灯"）
4. 识别是否包含多条指令（通过连接词"然后""也""还有"切分）

### Step 2: 意图分类
1. 判断指令大类：
   - 设备控制（开/关/调节/查询状态）
   - 场景触发（预设场景关键词匹配）
   - 设备查询（"客厅灯开着吗"）
   - 非设备指令（闲聊/音乐/天气等，标记后转发其他模块）
2. 对于设备控制类，进一步识别操作类型：
   - 绝对设置（"调到26度"→target=26）
   - 相对调节（"调高一点"→delta=+step）
   - 开关切换（"开灯""关灯"）
   - 状态查询（"现在几度"）

### Step 3: 实体抽取
1. 提取目标设备：
   - 显式指定（"客厅灯""卧室空调"）→直接匹配设备列表
   - 未指定→标记需要指代消解（交给context-management技能）
2. 提取操作动作：映射到标准操作集（power_on/power_off/set/adjust/query）
3. 提取参数值：
   - 绝对值（"26度""50%"）→直接提取
   - 相对值（"高一点""大一些"）→映射为步进值（默认20%）
   - 极值（"最大""最亮""最低"）→映射为边界值
   - 模糊值（"暗一些""温暖一点"）→映射为预设档位

### Step 4: 置信度评估
1. 对解析结果进行置信度打分（0-1）
2. 影响因素：ASR识别置信度、设备名匹配度、操作词明确度、参数值合理性
3. 置信度>=0.85→直接输出
4. 置信度0.7-0.85→输出并标记"建议确认"
5. 置信度<0.7→标记需追问

## 输入规格

```json
{
  "asr_text": "string - ASR转写文本",
  "asr_confidence": 0.92,
  "available_devices": [
    {"device_id": "light_01", "device_name": "客厅灯", "aliases": ["大灯", "客厅的灯"]}
  ],
  "user_id": "string",
  "voiceprint_id": "string - 声纹识别结果"
}
```

## 输出规格

```json
{
  "parse_result": {
    "intent_type": "device_control|scene_trigger|query|non_device",
    "commands": [
      {
        "device_id": "light_01",
        "device_name": "客厅灯",
        "action": "set_brightness",
        "parameters": {"brightness": 80},
        "parameter_type": "absolute|relative|extreme"
      }
    ],
    "confidence": 0.92,
    "requires_context": false,
    "multi_command": false
  }
}
```

## 最佳实践

- 维护一个设备别名词典（用户可能说"大灯"指"客厅主灯"），支持用户自定义别名
- 对于ASR可能的同音词错误，建立高频纠错映射表（定期从操作日志中挖掘）
- 数值步进应参考设备实际范围（亮度"高一点"在0-100范围是+20，在0-255范围是+50）
- 否定句式需要特别处理（"不要太亮"≠"不要亮"≠"关灯"）
- 口语中的程度副词需要分级映射（"一点点"→10%、"一点"→20%、"很多"→50%）

## 约束条件

- 解析总耗时<100ms（从接收文本到输出结构化结果）
- 支持的设备操作词库>=500个（覆盖各品类设备的常见表达）
- 多指令切分最多支持3条连续指令（超过3条建议用户分次说）
- 不对非设备类意图进行深度解析，识别后直接转发
