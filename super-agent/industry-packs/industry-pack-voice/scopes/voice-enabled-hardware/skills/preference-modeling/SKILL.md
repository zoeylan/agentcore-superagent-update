# 偏好建模 (Preference Modeling)

## 概述

偏好建模技能基于行为模式挖掘的结果，为每个用户构建精细化的设备使用偏好模型。模型以多维度键值结构存储，支持按时间、环境、设备类别等维度查询最匹配的偏好参数，直接服务于意图解析时的默认参数填充和模糊意图推理。

## 适用场景

- 默认参数提供："开灯"→根据当前时段返回默认亮度和色温
- 舒适度阈值个性化：每个用户对"太冷""太热"的感知不同
- 多用户偏好隔离：家庭中不同成员对同一设备的偏好差异
- 偏好冲突协商：多人共处时选择折中参数
- 偏好演化追踪：用户习惯随时间变化的自适应更新

## 执行步骤

### Step 1: 偏好维度定义
为每个设备类别定义偏好维度：
- 灯光：{亮度, 色温, 开关时机}
- 空调：{温度, 风速, 模式, 开关阈值}
- 窗帘：{开合度, 时间联动, 光照联动}
- 音箱：{音量, 内容偏好, 闹钟时间}
每个维度按上下文切分：{工作日早晨, 工作日晚间, 周末白天, 周末晚间...}

### Step 2: 偏好值计算
1. 从行为模式挖掘结果中提取每个维度的历史值序列
2. 使用指数移动平均（EMA）计算当前偏好值：
   - EMA_new = α × latest_value + (1-α) × EMA_old
   - α = 0.3（新行为权重30%，历史权重70%）
3. 计算偏好稳定度（标准差/均值），稳定度高的偏好置信度更高
4. 首次无历史数据时使用品类默认值（行业标准舒适度参数）

### Step 3: 多用户偏好管理
1. 基于声纹ID为每个用户维护独立偏好配置
2. 无法识别声纹时（如远场识别失败），使用"家庭公共偏好"（所有成员偏好的中位数）
3. 多人同时在场时的偏好协商策略：
   - 取各用户偏好的加权平均（权重可配置，默认等权）
   - 安全类参数取保守值（如家有老人时空调温度不低于24°）

### Step 4: 偏好查询服务
1. 接收查询请求：{user_id, device_category, context(time/env/room)}
2. 匹配最相关的上下文切片
3. 返回推荐参数值和置信度
4. 无匹配上下文时逐级回退：精确上下文→粗粒度上下文→设备默认值

### Step 5: 偏好更新与反馈循环
1. 每次用户操作后更新对应偏好维度的EMA值
2. 用户修正系统默认值时（如系统设80%亮度用户手动调到60%），增加修正值权重（α临时提高到0.5）
3. 用户明确否决建议时记录负反馈，对应偏好置信度下降
4. 每周生成偏好变化摘要（哪些偏好发生了显著变化）

## 输入规格

```json
{
  "query": {
    "user_id": "string",
    "voiceprint_id": "string",
    "device_category": "light|ac|curtain|speaker",
    "context": {
      "time": "ISO8601",
      "weekday": true,
      "room": "living_room",
      "environment": {"temperature": 25, "illuminance": 300}
    }
  }
}
```

## 输出规格

```json
{
  "preference": {
    "user_id": "user_001",
    "device_category": "light",
    "context_matched": "weekday_evening",
    "recommended_parameters": {
      "brightness": 70,
      "color_temp": 3000
    },
    "confidence": 0.85,
    "data_points": 23,
    "last_updated": "ISO8601",
    "fallback_parameters": {
      "brightness": 80,
      "color_temp": 4000
    },
    "multi_user_mode": false
  }
}
```

## 最佳实践

- 偏好模型的冷启动策略：前5次操作使用行业标准值，5-20次逐步过渡到个性化值，20次以上完全个性化
- EMA的衰减因子α可以按设备类别调整（灯光偏好变化快α=0.35，空调偏好稳定α=0.2）
- 偏好值必须有合理范围约束（亮度0-100%，温度16-30°），超出范围的计算结果截断到边界
- 季节变化期偏好可能大幅波动，此时应临时增大α值加速适应
- 提供"偏好重置"接口，用户可以清除学习数据重新开始

## 约束条件

- 偏好查询响应时间<20ms
- 单用户偏好模型存储空间<1MB
- 偏好更新不实时生效（批量处理，延迟<5分钟）
- 偏好数据加密存储，跨设备同步需用户授权
- 不基于偏好模型做安全相关决策（如门锁偏好不自动学习）
