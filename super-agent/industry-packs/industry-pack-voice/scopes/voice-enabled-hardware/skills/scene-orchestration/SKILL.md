# 场景编排 (Scene Orchestration)

## 概述

场景编排技能负责将用户的单一场景指令（如"我要睡觉了"）展开为多设备协同操作序列，管理设备间的依赖关系、确定最优执行顺序、并行/串行调度、以及场景执行过程中的异常处理和部分回滚。这是智能硬件语音控制从"单设备遥控器"向"全屋智能管家"跃迁的核心能力。

## 适用场景

- 预设场景一键触发（回家模式、离家模式、睡眠模式、观影模式等）
- 用户自定义场景创建和管理
- 基于时间/事件/传感器的场景自动触发
- 场景执行过程中的异常处理和降级
- 场景模板推荐和个性化定制

## 执行步骤

### Step 1: 场景模板解析
1. 接收场景触发指令（场景名称或触发条件匹配结果）
2. 加载场景模板定义（设备列表、目标状态、执行条件）
3. 检查场景中所有涉及设备的当前在线状态
4. 评估场景可执行性（所有关键设备在线→可执行；非关键设备离线→降级执行；关键设备离线→告知用户）

### Step 2: 依赖关系分析与执行顺序确定
1. 构建设备操作的依赖DAG图（如：必须先关主灯→再开夜灯；必须先关窗→再开空调）
2. 识别可并行执行的操作组（无依赖关系的操作可同时下发）
3. 生成最优执行计划：{parallel_group_1: [操作A, 操作B], then: parallel_group_2: [操作C]}
4. 预估总执行时间（基于各设备历史响应时延）

### Step 3: 编排执行
1. 按执行计划逐组下发控制指令
2. 每组操作设置独立超时（默认3秒）
3. 收集每台设备的执行结果（成功/失败/超时）
4. 当前组全部完成（或超时）后进入下一组
5. 实时更新场景执行进度

### Step 4: 异常处理与降级
1. 单设备执行失败：
   - 非关键设备→跳过，场景继续，事后通知用户
   - 关键设备→重试1次，仍失败则暂停场景，询问用户是否跳过
2. 设备响应超时：等待超时后按失败处理
3. 部分执行后用户取消：对已执行的操作执行逆向回滚（开→关，调高→恢复原值）
4. 冲突检测：场景执行中如有新的用户指令与当前场景冲突，暂停场景执行等待冲突解决

### Step 5: 执行结果汇总
1. 生成场景执行报告（成功设备/失败设备/跳过设备/总耗时）
2. 生成用户播报文本（简洁版："睡眠模式已开启"；详细版："已关闭3盏灯、拉上窗帘、空调调至26度，卧室门锁响应超时"）
3. 更新场景执行历史记录

## 输入规格

```json
{
  "scene_id": "string",
  "scene_name": "string - 如'睡眠模式'",
  "trigger_type": "voice|schedule|sensor|geo_fence",
  "device_actions": [
    {
      "device_id": "string",
      "action": "string",
      "parameters": {},
      "is_critical": true,
      "depends_on": ["other_device_id"],
      "priority": 1
    }
  ],
  "timeout_ms": 10000,
  "rollback_on_cancel": true
}
```

## 输出规格

```json
{
  "scene_id": "string",
  "execution_status": "success|partial|failed|cancelled",
  "total_devices": 5,
  "success_count": 4,
  "failed_devices": [
    {"device_id": "xxx", "reason": "timeout", "is_critical": false}
  ],
  "execution_time_ms": 2300,
  "execution_sequence": [
    {"group": 1, "devices": ["light_1", "light_2"], "status": "success"},
    {"group": 2, "devices": ["curtain_1", "ac_1"], "status": "partial"}
  ],
  "user_notification": "睡眠模式已开启，卧室窗帘响应较慢，请稍候",
  "rollback_available": true
}
```

## 最佳实践

- 场景模板设计时区分关键设备和非关键设备，关键设备失败才阻塞场景
- 执行顺序优先考虑用户体验（先执行视觉反馈明显的操作，让用户立即感知"场景开始了"）
- 夜间场景（22:00-06:00）自动降低语音反馈音量或改用静默确认（嘟声）
- 场景执行历史用于优化预估时间和发现频繁失败的设备
- 支持场景嵌套（"离家模式"可以包含"关闭所有灯光"子场景）

## 约束条件

- 单场景最大设备数：20台
- 场景总执行超时：30秒（超过30秒强制结束并报告当前状态）
- 同一时刻只能执行一个场景（新场景触发时，如有场景执行中则排队或冲突处理）
- 安全类设备（门锁、安防）在场景中的操作需要额外确认
- 场景回滚仅支持可逆操作（开关、调节），不可逆操作（如已发出的通知）无法回滚
