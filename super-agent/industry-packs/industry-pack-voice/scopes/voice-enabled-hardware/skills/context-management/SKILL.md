# 上下文管理 (Context Management)

## 概述

上下文管理技能维护语音对话过程中的上下文栈（Context Stack），用于支持连续对话中的指代消解、省略补全和增量操作。在智能硬件场景中，用户不会每次都说完整的指令，而是大量使用"这个""那个""再...一点""也关了"等依赖上下文的表达，本技能确保这些表达能正确映射到具体设备和操作。

## 适用场景

- 指代消解："把这个也关了"→确定"这个"指什么设备
- 省略补全："再亮一点"→确定操作对象和步进方向
- 连续对话追踪：跨多轮对话保持设备焦点
- 对话模式管理：连续对话模式的进入和超时退出
- 上下文恢复：用户短暂离开后返回时的上下文提示

## 执行步骤

### Step 1: 上下文栈初始化
1. 用户唤醒设备进入对话模式时，创建新的上下文栈
2. 初始化栈结构：{session_id, start_time, turns: [], focus_device: null, last_action: null}
3. 加载用户最近一次会话的焦点设备作为"弱上下文"（权重0.3）
4. 设置30秒超时计时器

### Step 2: 每轮对话更新
1. 接收当前轮次的解析结果
2. 将本轮涉及的设备、操作、参数推入栈顶
3. 更新焦点设备（focus_device = 最近一次被明确操作的设备）
4. 更新最近操作记录（last_action = {device, action, parameters, timestamp}）
5. 重置30秒超时计时器
6. 栈深度限制为10轮（超出后底部最旧记录自动弹出）

### Step 3: 指代消解查询
当意图解析发现指代词（"这个""那个""它"）时：
1. 查询栈顶最近的设备记录
2. "这个/它"→focus_device（最近被操作的设备）
3. "那个"→focus_device的前一个设备（栈中倒数第二个）
4. "都"→最近N个被提及的同类设备
5. 返回候选设备列表和置信度

### Step 4: 省略补全查询
当意图解析发现省略（只有操作没有对象，或只有方向没有具体值）时：
1. 设备省略→使用focus_device作为默认目标
2. 动作省略（"再一次"）→复用last_action
3. 参数省略（"亮一点"→对象？步进多少？）→对象用focus_device，步进用last_action中同类操作的步进值
4. 如果上下文栈为空且存在省略，返回"需要追问"标记

### Step 5: 超时与退出管理
1. 30秒计时器到期→生成退出通知（"已退出对话模式"）
2. 清空上下文栈（但保留摘要供下次弱引用）
3. 用户主动结束词检测（"没了""好了""就这样"）→立即退出
4. 异常退出（网络断开等）→保存当前栈状态，60秒内可恢复

## 输入规格

```json
{
  "session_id": "string",
  "current_turn": {
    "asr_text": "再亮一点",
    "parsed_intent": {
      "action": "adjust_brightness",
      "direction": "up",
      "device_id": null,
      "step_value": null
    }
  },
  "query_type": "resolve_reference|fill_ellipsis|get_focus"
}
```

## 最佳实践

- 焦点设备的切换应该是"强信号"驱动的——只有用户明确提到新设备名或新设备类别时才切换焦点
- "再...一点"类增量操作应保持方向一致性（用户说了3次"亮一点"，第4次省略只说"再来"也应继续调亮）
- 对话超时时间可根据用户历史行为动态调整（快速用户15秒、正常用户30秒、慢速用户45秒）
- 弱上下文引用需要降低置信度（0.3-0.5），在关键操作中不应单独依赖弱上下文做决策
- 栈记录应包含时间戳，超过10秒前的记录在指代消解中权重降低

## 约束条件

- 上下文查询响应时间<10ms
- 单会话栈最大深度：10轮
- 连续对话默认超时：30秒（可配置15-60秒）
- 上下文栈不跨会话持久化（安全考虑），仅保留焦点设备摘要作为弱引用
- 多用户场景中，上下文栈严格按声纹隔离，不同用户不共享上下文
