# 设备状态管理 (Device State Management)

## 概述

设备状态管理技能负责维护智能硬件环境中所有接入设备的实时状态图谱，包括设备发现、状态同步、健康监测和生命周期管理。这是设备编排Agent的基础能力，确保系统对物理设备世界的认知始终准确且最新。

## 适用场景

- 新设备接入时的自动发现和注册
- 设备状态实时同步（开关状态、参数值、连接质量）
- 设备离线/上线检测和通知
- 设备健康度评估和预警
- 设备固件版本管理和兼容性检查

## 执行步骤

### Step 1: 设备发现与注册
1. 监听局域网内的设备广播（mDNS/SSDP/Zigbee Announce）
2. 识别设备类型和通信协议（WiFi/Zigbee/Z-Wave/BLE/Matter）
3. 获取设备能力描述（支持的操作、参数范围、状态属性）
4. 生成设备唯一标识并注册到设备图谱中
5. 设置默认监控参数（心跳间隔、超时阈值）

### Step 2: 状态同步机制
1. 建立设备状态订阅通道（基于设备协议的事件推送）
2. 定期轮询无推送能力的设备（间隔根据设备类型配置：灯光5s、空调30s、门锁60s）
3. 接收设备主动上报的状态变更事件
4. 更新内存中的设备状态缓存
5. 状态变更事件广播给订阅方（意图解析Agent、个性化学习Agent）

### Step 3: 健康监测
1. 监控设备心跳信号，超过3个心跳周期未响应标记为"疑似离线"
2. 发送主动探测包确认设备状态
3. 确认离线后触发告警通知
4. 记录设备离线时间和频率，生成设备健康度评分
5. 健康度低于阈值（<70分）的设备加入重点监控列表

### Step 4: 状态一致性校验
1. 每5分钟执行一次全量状态快照
2. 对比缓存状态与设备实际状态（通过主动查询）
3. 发现不一致时立即更新缓存并记录异常日志
4. 连续3次不一致触发设备异常告警

## 输入规格

```json
{
  "device_id": "string - 设备唯一标识",
  "protocol": "zigbee|zwave|wifi|ble|matter",
  "capabilities": ["on_off", "brightness", "color_temp", "temperature"],
  "heartbeat_interval_ms": 5000,
  "timeout_threshold_ms": 15000
}
```

## 输出规格

```json
{
  "device_id": "string",
  "device_name": "string - 用户自定义名称",
  "device_type": "light|ac|curtain|lock|robot|sensor",
  "location": "string - 设备所在区域",
  "status": "online|offline|error|updating",
  "current_state": {
    "power": "on|off",
    "parameters": {}
  },
  "health_score": 0-100,
  "last_seen": "ISO8601 timestamp",
  "connection_quality": "excellent|good|fair|poor"
}
```

## 最佳实践

- 设备状态更新采用乐观锁机制，避免并发更新导致状态覆盖
- 对于高频状态变化的设备（如调光灯的亮度滑动），采用防抖策略（100ms内的连续变化合并为一次更新）
- 设备离线队列中的待执行指令超过5分钟自动过期，避免设备上线后执行过时操作
- 状态图谱需要持久化存储（每次变更写入日志），系统重启后能快速恢复
- 通信协议适配层应设计为插件架构，方便新增设备协议支持

## 约束条件

- 单网关支持最大设备数：Zigbee 200台、WiFi 50台、BLE 20台
- 状态同步延迟不超过100ms（从设备状态变化到系统感知）
- 内存中设备状态缓存总量不超过50MB
- 设备发现过程不得干扰已有设备的正常通信
