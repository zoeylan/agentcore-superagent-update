# 主动推荐 (Proactive Recommendation)

## 概述

主动推荐技能基于行为模式和偏好模型，在合适时机向用户发起场景建议或操作提醒。核心原则是"有用但不打扰"——只在高置信度且高价值的时机发起推荐，并严格控制推荐频率，让用户感受到系统的贴心而非烦躁。

## 适用场景

- 新场景创建建议：检测到重复操作序列，建议一键化为场景
- 场景触发建议：检测到场景触发条件满足，询问是否执行（"检测到您到家了，是否开启回家模式？"）
- 参数优化建议：发现用户频繁手动调整系统默认值，建议更新默认参数
- 时间提醒型建议：临近用户日常操作时间，提前询问（"快到您平时关灯的时间了，需要现在关灯吗？"）
- 节能建议：检测到设备长时间无意义运行，建议关闭

## 执行步骤

### Step 1: 推荐触发条件评估
1. 持续监控以下触发源：
   - 时间触发：接近用户日常模式的触发时间点（±5分钟窗口）
   - 事件触发：用户到家/离家、环境变化超阈值
   - 行为触发：用户手动执行了已识别模式的第一步操作
   - 周期触发：模式挖掘发现新模式且可靠度达标
2. 触发条件满足时进入推荐决策流程

### Step 2: 推荐价值评估
1. 计算推荐价值分数（0-1）：
   - 模式可靠度权重（0.4）：该模式的历史重复性和一致性
   - 时效性权重（0.3）：推荐时机的精准度
   - 用户接受历史（0.3）：用户对类似推荐的历史采纳率
2. 价值分数>=0.7才允许发起推荐
3. 检查频率限制：当日已发起推荐数<3

### Step 3: 推荐话术生成
1. 根据推荐类型选择话术模板：
   - 场景建议："检测到您到家了，是否开启回家模式？"
   - 新场景创建："您最近每晚都会关灯和拉窗帘，要不要创建一个'晚安场景'？"
   - 参数优化："发现您经常把灯调到60%，要不要把默认亮度改为60%？"
2. 话术要求：简短（<20字）、非侵入性、包含关键信息、以疑问句结束
3. 提供明确的肯定/否定选项（"好的"触发执行，"不用了"取消）

### Step 4: 用户反馈处理
1. 用户采纳（说"好的""开吧"等肯定词）：
   - 执行推荐操作
   - 记录正反馈，提高该模式的推荐权重
2. 用户拒绝（说"不用""算了"等否定词）：
   - 取消操作
   - 记录负反馈
   - 同类推荐在7天内不再触发
3. 用户忽略（5秒无响应）：
   - 取消推荐
   - 记录为"无响应"，连续3次无响应等同于拒绝
4. 连续2次拒绝同类推荐→该类推荐冷却7天

### Step 5: 推荐效果追踪
1. 记录每次推荐的结果（采纳/拒绝/忽略）
2. 计算周/月维度的推荐采纳率
3. 采纳率<50%的推荐类型进入优化队列
4. 采纳率>80%的高质量推荐可以升级为自动执行（需用户授权）

## 输入规格

```json
{
  "trigger": {
    "trigger_type": "time|event|behavior|periodic",
    "trigger_source": "pattern_id or event description",
    "trigger_time": "ISO8601",
    "context": {
      "user_id": "string",
      "room": "string",
      "environment": {}
    }
  },
  "candidate_recommendation": {
    "pattern_id": "string",
    "recommended_actions": [],
    "pattern_confidence": 0.85
  },
  "user_history": {
    "today_recommendation_count": 1,
    "similar_recommendation_acceptance_rate": 0.75,
    "last_rejection_date": null
  }
}
```

```json
{
  "recommendation_decision": {
    "should_recommend": true,
```

## 最佳实践

- "三不推荐"原则：用户明显忙碌时不推荐、深夜（23:00-07:00）不推荐、用户刚拒绝过时不推荐
- 推荐时机要"恰到好处"——不提前太多（用户还没准备好）也不太晚（用户已经手动做了）
- 新场景创建建议只在模式重复>=10次且近7天仍活跃时才提出
- 节能建议的措辞避免说教感（说"空调已运行2小时，要关一下吗？"而非"为了节能请关闭空调"）
- 推荐升级为自动执行需要用户在设置中显式开启，且仅限非安全类操作

## 约束条件

- 每日主动推荐上限：3次（含所有推荐类型总计）
- 推荐话术播报时长<3秒
- 等待用户响应超时：5秒
- 被拒绝的推荐冷却期：同类7天，完全相同的推荐30天
- 推荐不涉及安全设备（门锁、安防）的自动操作
- 深夜（23:00-07:00）不发起语音推荐，可改为静默记录
