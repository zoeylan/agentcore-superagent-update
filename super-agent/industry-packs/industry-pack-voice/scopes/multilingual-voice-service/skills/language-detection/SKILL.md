# 语种检测 (Language Detection)

## 概述

语种检测技能实现对输入语音流的实时语言识别，在首500毫秒音频帧内输出语种判定结果。该技能是多语言语音服务的入口环节，其准确性和实时性直接决定下游ASR模型选择的正确性。支持20+主流语言的识别，涵盖中、英、日、韩、法、德、西、阿拉伯等语言。

## 适用场景

- 多语言客服系统接入时的自动语言判定
- 国际会议中说话人语言切换检测
- 多语言内容平台的音频语种标注
- 跨境电商客服的来电语言自动识别

## 执行步骤

### 步骤1：音频特征提取
- 接收输入语音流的前500ms音频帧（至少包含8000个采样点@16kHz）
- 提取声学特征向量：Mel频谱特征（40维）、基频F0轮廓、音素分布统计
- 计算语调特征：声调变化模式（区分声调语言如中文 vs 非声调语言如英语）

### 步骤2：语种分类推理
- 将特征向量输入预训练语种识别模型（支持的模型：ECAPA-TDNN、wav2vec2-lid）
- 输出Top-3语种候选及对应置信度分数
- 若Top-1置信度>=0.85，直接确认语种判定
- 若Top-1置信度在0.7-0.85之间，标记为"中等置信度"，继续收集后续帧修正
- 若Top-1置信度<0.7，标记为"低置信度"，使用默认语言模型先行处理

### 步骤3：输出路由决策
- 生成结构化路由指令：`{language_code, confidence, model_id, fallback_flag}`
- 将路由决策发送至ASR模型调度器
- 若使用了回退策略，同步发送"待修正"信号

### 步骤4：持续修正（流式模式）
- 在后续音频帧中持续更新语种判定
- 若检测到与初始判定不一致（连续3帧新语种置信度>0.9），触发模型热切换
- 记录语种切换事件用于后续分析

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| audio_stream | bytes | 是 | PCM 16kHz 16bit单声道音频流 |
| default_language | string | 否 | 默认回退语言（ISO 639-1），默认"zh" |
| candidate_languages | string[] | 否 | 候选语言范围（缩小识别范围提升准确率） |
| session_context | object | 否 | 会话上下文（含用户历史语言偏好） |

## 输出规格

```json
{
  "detected_language": "zh",
  "confidence": 0.93,
  "dialect_hint": null,
  "top_candidates": [
    {"language": "zh", "confidence": 0.93},
    {"language": "ja", "confidence": 0.04},
    {"language": "ko", "confidence": 0.02}
  ],
  "model_recommendation": "asr-zh-v3",
  "fallback_applied": false,
  "detection_latency_ms": 320
}
```

## 最佳实践

1. **预热策略**：对高频语言（中文、英语）的模型保持常驻内存，减少首次加载延迟
2. **上下文先验**：利用用户历史语言偏好和地域信息作为先验概率，提升判定速度
3. **候选语言缩减**：在已知业务场景中（如中日双语客服），限定候选语言范围可将准确率提升至99%+
4. **静音段跳过**：检测到纯静音或纯噪声时不进行语种判定，避免误判
5. **多说话人分离**：在多人会话中，对每个说话人独立进行语种判定

## 约束与限制

- 最低音频长度：300ms（低于此长度判定准确率大幅下降）
- 不支持纯音乐或非人声内容的语种判定
- 极低信噪比（SNR<5dB）环境下准确率下降至70%以下，需标记不可靠
- 该技能仅负责语种级别判定，方言细分由dialect-recognition技能负责
