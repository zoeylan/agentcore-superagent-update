# 语音转写 (Speech Transcription)

## 概述

语音转写技能将经过语种标注和分段的语音流转化为高质量文本，支持实时流式转写和离线高精度转写两种模式。核心能力包括说话人分离（Speaker Diarization）、精确时间戳对齐、智能标点恢复、专业术语增强识别。目标转写质量：普通话WER<8%、英语WER<10%、主要方言WER<15%。

## 适用场景

- 会议录音的全文转写（1-3小时长音频）
- 客服通话的实时字幕生成
- 播客/视频内容的字幕制作
- 法律庭审、医疗问诊等专业场景的记录转写

## 执行步骤

### 步骤1：转写模式选择
- 根据输入类型判断转写模式：
  - 实时音频流 → 流式转写模式（低延迟优先，首字延迟<300ms）
  - 完整音频文件 → 离线转写模式（高精度优先，多遍解码）
- 加载语言路由Agent指定的ASR模型
- 如有领域术语表，注入到解码器的热词列表中

### 步骤2：说话人分离（离线模式）
- 使用Speaker Diarization模型对完整音频进行说话人聚类
- 输出每个语音段的说话人标签：Speaker_1, Speaker_2...
- 目标DER（Diarization Error Rate）<12%
- 支持已知说话人的声纹注册匹配（如已注册的会议参与者）

### 步骤3：ASR解码与对齐
- 流式模式：使用CTC/Attention混合解码器，每接收320ms音频输出一次增量结果
- 离线模式：使用完整注意力解码器 + LM重打分，输出最终转写结果
- 执行强制对齐（Forced Alignment）生成词级时间戳
- 时间戳精度：±200ms（词级别）、±50ms（音素级别）

### 步骤4：后处理
- 智能标点恢复：基于语义停顿和语法结构插入标点（准确率>=95%）
- 数字/日期规范化："二零二四年三月十五号" → "2024年3月15日"
- 敏感信息检测与脱敏（可选）：身份证号、银行卡号自动脱敏为"***"
- 语码混合文本格式化：保持原始混合形式，英文部分保留原始大小写

### 步骤5：质量评估
- 计算每段转写的置信度分数（基于解码器后验概率）
- 标记低置信度片段（confidence<0.7）加入人工复核队列
- 输出整体转写质量估计指标

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| audio_input | bytes/stream | 是 | 音频数据或实时音频流 |
| mode | string | 是 | "realtime" 或 "offline" |
| language | string | 是 | 语种代码（由语言路由Agent提供） |
| model_id | string | 是 | ASR模型标识 |
| hot_words | string[] | 否 | 热词列表（提升特定词识别率） |
| speaker_profiles | object[] | 否 | 已注册说话人声纹（用于说话人标注） |
| enable_diarization | bool | 否 | 是否启用说话人分离（默认true） |
| enable_punctuation | bool | 否 | 是否启用标点恢复（默认true） |

## 最佳实践

1. **热词注入**：会议场景下提前注入参会人姓名、项目代号、专业术语作为热词，可提升5-10%的命名实体识别率
2. **二次校正**：离线模式下使用LM重打分对ASR N-best候选进行重排序，优先选择语法通顺的候选
3. **说话人预注册**：对已知参与者预先注册声纹，实现"张三说：..."这样的具名标注
4. **分段策略**：长音频按自然停顿（>1秒静音）分段处理，避免超长序列导致注意力退化
5. **增量修正**：流式模式下支持对已输出文本的增量修正（如"明天"→"明年"），通过partial/final标记区分

## 约束与限制

- 单次离线处理音频时长上限：4小时（超过需分批处理）
- 说话人分离在2-8人场景表现最佳，超过8人准确率下降
- 重叠语音（多人同时说话）的转写准确率显著降低，需标记overlap
- 极低音质（电话窄带8kHz）下WER会增加3-5个百分点
- 该技能负责转写输出，不负责转写后的内容分析（由meeting-summarization等技能负责）
