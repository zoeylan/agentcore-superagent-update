# 实时翻译 (Realtime Translation)

## 概述

实时翻译技能实现语音到语音/文本的低延迟翻译管道，端到端延迟控制在2秒以内。支持20+语言间的双向翻译，适用于国际会议同声传译、跨语言客服实时交互、多语言直播字幕生成等场景。核心挑战在于在流式输入条件下保持翻译质量和上下文连贯性。

## 适用场景

- 国际会议的AI同声传译
- 跨语言客服的实时交互翻译
- 多语言视频/直播的实时字幕翻译
- 跨国企业的多方视频会议翻译

## 执行步骤

### 步骤1：翻译策略确定
- 根据场景确定翻译策略：
  - 同传模式（单向）：一方说，翻译给另一方听，延迟<2秒
  - 对话模式（双向）：双方交替说，翻译方向根据话权动态切换
  - 字幕模式（文本输出）：仅输出翻译文本，无TTS合成
- 确定源语言和目标语言对（由语言路由Agent提供源语言信息）
- 加载对应语言对的翻译模型和术语表

### 步骤2：流式翻译执行
- 接收ASR输出的增量文本片段
- 判断翻译启动时机（Segmentation Policy）：
  - 句子完整时立即翻译（优先保证质量）
  - 检测到语义单元完整（主谓宾齐全）时提前启动（平衡延迟和质量）
  - 超过3秒未成句时强制翻译当前缓冲内容（保证延迟上限）
- 执行机器翻译：源文本→预处理（术语替换标记）→翻译模型→后处理（术语还原、格式调整）
- 输出翻译结果及时间戳对齐信息

### 步骤3：上下文增强
- 维护翻译上下文窗口（最近10句对话历史）
- 指代消解："it"、"这个"等代词基于上下文确定具体指代
- 术语一致性：同一术语在整个会话中保持翻译一致
- 风格一致性：保持源语言的正式/非正式语体在目标语言中的对应

### 步骤4：TTS合成指令生成（语音输出模式）
- 为翻译后文本生成TTS合成参数：
  - 目标语言的语音模型选择
  - 语速调整（匹配源语言说话速度，避免过快或过慢）
  - 语调参数（保持与源语言相似的情感色彩）
- 将TTS合成指令发送给语音合成引擎

### 步骤5：质量监控
- 实时计算翻译延迟（source_end_time → translation_output_time）
- 对翻译结果进行质量自评估（基于翻译模型置信度）
- 低置信度翻译标记为uncertain，在字幕模式下可视化展示
- 记录翻译对用于后续质量审核和模型优化

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| source_text | string/stream | 是 | ASR输出的源语言文本（流式或完整） |
| source_language | string | 是 | 源语言代码 |
| target_language | string | 是 | 目标语言代码 |
| mode | string | 是 | "simultaneous"(同传)/"dialogue"(对话)/"subtitle"(字幕) |
| terminology_dict | object | 否 | 术语表（源语言→目标语言映射） |
| context_history | string[] | 否 | 历史上下文（前几句对话） |
| quality_priority | string | 否 | "speed"(速度优先)/"quality"(质量优先)，默认"balanced" |

## 输出规格

```json
{
  "translation": {
    "source_text": "我们这个季度的revenue增长了百分之三十",
    "translated_text": "Our revenue grew by 30% this quarter",
    "source_language": "zh",
    "target_language": "en",
    "confidence": 0.91,
    "latency_ms": 1200,
    "terminology_matched": [
      {"source": "revenue", "translation": "revenue", "from_dict": true}
    ],
    "tts_params": {
      "voice_model": "en-US-professional-male",
      "speed_factor": 1.0,
      "emotion": "neutral"
    },
    "uncertain_spans": []
  }
}
```

## 最佳实践

1. **预测性翻译**：对于结构可预测的语言对（如中→英），在句子未完成时基于语法模型预测后续结构，提前开始翻译
2. **术语优先策略**：术语表中的词汇直接查表翻译，不经过模型推理，确保100%一致性
3. **延迟自适应**：根据网络状况和模型负载动态调整翻译策略——负载高时倾向更大分段减少请求次数
4. **双向对话的话权预测**：基于对话模式预测话权切换时机，提前准备切换翻译方向
5. **文化适配**：翻译中考虑文化差异（如中文的客气表达在英文中适当简化）

## 约束与限制

- 端到端延迟严格<2秒（从源语音结束到翻译输出开始）
- 通用翻译质量BLEU>=35，领域适配后>=45
- 术语一致性>=98%
- 不支持超过2种语言的同时翻译（多语言会议需为每个语言对建立独立管道）
- 该技能不负责源语言识别（由语言路由Agent完成），仅负责翻译环节
- 对于高度口语化、含大量省略和语气词的输入，翻译质量可能下降
