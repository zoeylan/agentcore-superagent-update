# 双语对话管理 (Bilingual Dialogue Management)

## 概述

双语对话管理技能处理跨语言实时对话场景中的话权管理、方向切换、上下文记忆保持和延迟同步。典型场景：中文坐席服务英语客户时，系统需要实时管理双方的说话轮次、维护双向翻译的语境连贯性、处理插话和重叠语音。该技能是实现"无障碍跨语言沟通"的关键协调层。

## 适用场景

- 跨语言客服：中文坐席与外语客户的实时翻译对话
- 国际商务谈判：双方使用不同语言的实时翻译会话
- 远程医疗：医生与外籍患者的诊疗对话翻译
- 跨国团队协作：日常工作沟通的实时翻译辅助

## 执行步骤

### 步骤1：会话初始化
- 确定对话双方的语言配置：
  - 方A语言（如中文）和对应ASR/TTS模型
  - 方B语言（如英语）和对应ASR/TTS模型
  - 翻译方向对：A→B 和 B→A
- 初始化双向翻译管道
- 建立会话上下文记忆结构

### 步骤2：话权检测与管理
- 实时监控双方音频通道的活动状态
- 话权状态机：
  - IDLE：双方均未说话
  - A_SPEAKING：方A在说话，翻译方向A→B
  - B_SPEAKING：方B在说话，翻译方向B→A
  - TRANSITION：话权切换中（一方结束，等待另一方响应）
  - OVERLAP：双方同时说话（冲突状态）
- 话权切换触发条件：当前说话方静音>=500ms + 对方开始说话
- 重叠处理策略：优先保持当前翻译方向，对方语音暂存缓冲

### 步骤3：翻译方向动态切换
- 话权切换时，翻译管道方向同步切换
- 切换前：完成当前方向的最后一个翻译片段输出
- 切换后：立即开始新方向的翻译处理
- 切换延迟控制：<200ms（从检测到话权切换到新方向翻译开始）

### 步骤4：上下文连贯性维护
- 维护双语对话的统一上下文窗口：
  - 记录双方所有发言（原文+翻译）的时序列表
  - 追踪话题演进和关键实体
  - 维护双方的指代链（"他说的那个方案"→回溯找到具体方案）
- 将上下文信息注入翻译请求，提升翻译的连贯性

### 步骤5：延迟同步与体验优化
- 管理翻译延迟带来的对话节奏问题：
  - 当翻译输出时对方已开始新的发言→标记重叠并排队处理
  - 翻译延迟过长（>3秒）→输出中间状态提示（"正在翻译..."）
  - 双方语速差异大→通过TTS语速调整平衡节奏
- 优化体验细节：
  - 翻译完成的确认音（简短提示音表示翻译已输出）
  - 未完成翻译的占位符（"对方正在说..."）

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| channel_a_audio | stream | 是 | 方A的音频流 |
| channel_b_audio | stream | 是 | 方B的音频流 |
| language_a | string | 是 | 方A的语言代码 |
| language_b | string | 是 | 方B的语言代码 |
| session_id | string | 是 | 会话标识 |
| turn_timeout_ms | int | 否 | 话权超时时间（默认500ms） |
| overlap_strategy | string | 否 | 重叠处理策略："priority_current"/"queue"/"interrupt" |

## 最佳实践

1. **预翻译准备**：在话权切换间隙预加载新方向的翻译模型上下文，减少首句翻译延迟
2. **礼貌等待**：话权切换时插入短暂（200-300ms）的等待，避免因VAD灵敏度过高导致误切换
3. **重叠保护**：翻译TTS播放时若对方开始说话，立即降低TTS音量或暂停，优先捕获新输入
4. **上下文裁剪**：上下文窗口超过20轮时，只保留最近10轮完整对话+更早对话的摘要
5. **异常降级**：翻译管道故障时，切换到实时字幕模式（只输出文本不合成语音），保持基本沟通

## 约束与限制

- 仅支持2方对话，多方多语言会议需使用不同的管理策略
- 话权切换延迟<200ms
- 重叠语音超过3秒时可能丢失部分内容，需标记incomplete
- 该技能是协调层，不直接执行ASR/翻译/TTS，而是调度realtime-translation技能和TTS引擎
- 对于语速极快的对话（>250字/分钟），翻译延迟可能超过2秒目标
