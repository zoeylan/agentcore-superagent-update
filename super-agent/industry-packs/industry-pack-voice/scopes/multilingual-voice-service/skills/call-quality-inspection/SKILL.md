# 通话质检 (Call Quality Inspection)

## 概述

通话质检技能对客服通话进行自动化质量检测，覆盖合规话术检查、服务态度评估、情绪轨迹标注、敏感信息检测和关键信息提取。支持实时质检（通话过程中即时告警）和事后质检（通话结束后生成完整报告）两种模式。目标是将质检自动化率提升至90%以上，质检报告生成时间不超过通话时长的10%。

## 适用场景

- 金融机构客服的合规性检查（风险提示、免责声明）
- 电商客服的服务质量评估
- 保险理赔通话的信息提取与合规审核
- 坐席培训需求分析和绩效考核数据生成

## 执行步骤

### 步骤1：合规话术检查
- 加载行业合规话术模板（开场白、身份确认、风险提示、隐私声明、结束语）
- 逐项检查必说话术是否在通话中出现：
  - 完整匹配（话术原文出现）
  - 语义等价匹配（表达不同但含义相同）
  - 缺失标记（话术未出现）
- 检查话术时序：某些话术有顺序要求（如身份确认必须在业务办理之前）
- 输出合规检查清单：`[{item, status: pass/fail/partial, evidence_timestamp}]`

### 步骤2：服务态度评估
- 分析坐席语音/文本的服务态度维度：
  - 礼貌用语使用频率（"请"、"感谢"、"抱歉"）
  - 禁忌用语检测（推诿、争吵、反问句式）
  - 主动服务意识（主动提供解决方案、主动确认理解）
  - 响应及时性（用户提问后的等待时长）
- 输出服务态度评分（1-10分）及扣分原因

### 步骤3：情绪轨迹标注
- 对通话全程进行情绪时间线标注：
  - 用户情绪：每30秒标注情绪类别和强度
  - 坐席情绪：检测坐席是否保持专业态度
- 识别情绪转折点："用户在2:30从中性变为负面，原因是等待过久"
- 标注情绪管理效果：坐席安抚后用户情绪是否改善

### 步骤4：敏感信息检测
- 扫描转写文本中的敏感信息：
  - 个人身份信息（身份证号、手机号、银行卡号）
  - 业务敏感信息（密码、验证码）
  - 合规敏感信息（承诺回报率、保证性语言）
- 对检测到的敏感信息执行脱敏标注
- 检查坐席是否在不安全信道上索取敏感信息

### 步骤5：关键信息提取（按场景模板）
- 加载场景化信息提取模板，例如：
  - 保险报案：{报案时间, 事故类型, 损失描述, 涉及金额, 对方信息}
  - 投诉工单：{投诉对象, 问题描述, 期望处理方式, 紧急程度}
  - 业务办理：{办理类型, 关键参数, 确认状态}
- 从通话内容中自动填充模板字段
- 标记未获取的必填字段为"信息缺失"

### 步骤6：质检报告生成
- 汇总各维度检查结果，生成结构化质检报告
- 计算综合质检评分（合规分×40% + 服务态度分×30% + 问题解决分×30%）
- 标注高风险项（需立即关注的合规问题或服务事故）
- 生成改进建议（针对扣分项的具体指导）

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| transcription | object | 是 | 通话转写结果（含说话人标注） |
| compliance_template | object | 是 | 合规话术检查模板 |
| extraction_template | object | 否 | 信息提取模板（按业务场景） |
| emotion_data | object | 否 | 情绪感知引擎的实时情绪数据 |
| mode | string | 是 | "realtime"(实时告警) 或 "post-call"(事后报告) |

## 最佳实践

1. **行业模板预置**：为金融、保险、电商、医疗等行业预置合规话术模板，减少配置成本
2. **实时+事后双重检查**：关键合规项实时告警（如风险提示遗漏），非紧急项事后检查
3. **语义等价判断**：合规话术检查不应死板匹配原文，"你好"和"您好"应视为等价
4. **持续校准**：定期将自动质检结果与人工质检结果对比，校准评分标准
5. **正面反馈**：质检报告不仅关注问题，也要标注服务亮点，用于正向激励

## 约束与限制

- 质检报告自动生成率目标>=90%（仅复杂争议通话需人工复核）
- 合规话术检出率>=98%（漏检为严重质量事故）
- 质检报告生成时间<通话时长的10%
- 实时质检模式仅检查关键合规项，不做全面评估
- 该技能依赖高质量转写结果，转写WER>15%时质检可靠性下降
