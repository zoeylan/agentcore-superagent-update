# 会议摘要生成 (Meeting Summarization)

## 概述

会议摘要生成技能对会议转写文本进行深度分析，自动生成结构化的会议纪要，包括议题分割、逐议题摘要、决策点提取、行动项识别（含负责人和截止日期）。目标是将1-3小时的会议内容浓缩为5分钟可读的结构化文档，覆盖率>=95%确保关键决策点无遗漏。

## 适用场景

- 企业内部周会/项目进度会的自动纪要生成
- 董事会/管理层决策会议的正式记录
- 跨部门协作会议的行动项追踪
- 培训/研讨会的要点总结与知识沉淀

## 执行步骤

### 步骤1：议题自动分割
- 对转写文本进行语义分段，识别议题切换边界
- 切换信号：显式转折语（"下一个议题"、"接下来讨论"）、话题语义突变、说话人主导权切换
- 为每个议题生成标题（基于核心讨论内容的8-15字概括）
- 输出议题列表：`[{topic_id, title, start_time, end_time, key_speakers}]`

### 步骤2：逐议题摘要
- 对每个议题内容生成3-5句话的摘要，覆盖：
  - 讨论背景和提出的问题
  - 各方主要观点和争议焦点
  - 最终决策或结论（若有）
  - 遗留问题和后续方向
- 摘要中标注关键发言人（"张总提出..."、"李经理建议..."）

### 步骤3：决策点提取
- 识别会议中形成的正式决策（如"决定下周启动A方案"）
- 决策识别信号：共识性语言（"就这么定了"、"大家同意"）、条件判断语言（"如果...就..."）、否定性决策（"暂时不做"）
- 结构化输出：`{decision_id, content, rationale, participants_agreed, timestamp}`

### 步骤4：行动项提取
- 从对话中识别被分配的任务和承诺
- 提取四要素：任务描述（做什么）、负责人（谁做）、截止日期（何时完成）、优先级
- 识别信号："你负责..."、"下周一前完成..."、"记得跟进..."
- 处理模糊指派："这个谁来做？" → "我来吧" → 确认负责人
- 日期推理："下周五"→ 基于会议日期计算具体日期

### 步骤5：会议纪要组装
- 按标准模板组装完整会议纪要：
  - 会议元信息（时间、地点、参会人、时长）
  - 议程概览（议题列表+时间分配）
  - 逐议题详情（摘要+决策+行动项）
  - 全局行动项汇总表
  - 遗留问题清单
- 标注置信度，低置信度内容加[待确认]标记

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| transcription | object | 是 | speech-transcription技能的输出结果 |
| meeting_metadata | object | 否 | 会议元信息（主题、参会人名单、预设议程） |
| summary_style | string | 否 | 摘要风格："concise"(简洁)/"detailed"(详尽)，默认"concise" |
| language | string | 否 | 输出语言（默认与会议语言一致） |
| action_item_template | object | 否 | 自定义行动项字段模板 |

## 最佳实践

1. **预设议程对照**：若有预设议程，将自动识别的议题与预设议程对齐，确保不遗漏计划议题
2. **参会人名单利用**：预提供参会人名单可提升行动项中负责人识别的准确率
3. **否定决策识别**："暂时不做X"也是重要决策，不能忽略
4. **模糊日期解析**：将"下周"、"月底前"、"尽快"等模糊表达转化为具体日期或标记[日期待确认]
5. **多语言会议处理**：中英混合会议的摘要默认用中文输出，英文专有名词保留原文

## 约束与限制

- 会议摘要覆盖率目标>=95%（关键决策点无遗漏）
- 行动项提取准确率>=90%
- 输入转写文本质量直接影响摘要质量——WER>20%时摘要可靠性显著下降
- 不支持从纯音频直接生成摘要（必须先经过转写步骤）
- 对于非正式闲聊内容自动过滤，不纳入正式摘要
