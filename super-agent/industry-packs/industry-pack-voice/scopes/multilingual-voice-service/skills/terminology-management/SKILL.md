# 术语管理 (Terminology Management)

## 概述

术语管理技能负责维护、加载和应用企业专有术语表和翻译记忆库，确保在翻译和转写过程中专有名词、品牌名称、技术术语的翻译保持一致性。术语一致性是专业翻译的核心质量指标，目标>=98%。该技能为实时翻译和转写分析提供术语级别的知识支撑。

## 适用场景

- 企业专有术语在翻译中的一致使用
- 行业专业词汇的标准化翻译
- 品牌名称和产品名在多语言中的统一表达
- 翻译记忆库的积累和复用

## 执行步骤

### 步骤1：术语表加载与索引
- 根据业务场景/客户/行业加载对应术语表
- 构建术语索引结构：
  - 精确匹配索引：源语言词形 → 目标语言翻译
  - 模糊匹配索引：处理词形变化（复数、时态、缩写等）
  - 上下文匹配索引：同一术语在不同上下文中的不同翻译
- 术语表格式支持：TBX、CSV、JSON、Excel

### 步骤2：术语检测与匹配
- 在输入文本中扫描术语表中的词条
- 匹配策略（按优先级）：
  1. 精确匹配：完全一致的源语言形式
  2. 变体匹配：词形变化（capitalize/lowercase/plural等）
  3. 缩写展开：缩写形式与完整形式的关联
  4. 上下文消歧：多义术语根据上下文选择正确翻译
- 输出匹配到的术语列表及其在文本中的位置

### 步骤3：术语翻译应用
- 将翻译引擎的输出与术语表进行对照
- 若翻译引擎输出的术语翻译与术语表不一致，强制替换为术语表指定翻译
- 处理术语在目标语言中的语法适配（如性/数/格的变化）
- 标记术语表中无法覆盖的新术语（potential_new_term）

### 步骤4：翻译记忆库管理
- 将通过人工审核确认的翻译对存入翻译记忆库（TM）
- 新输入与TM进行模糊匹配（匹配率>=70%时复用）
- TM条目包含：source_text, target_text, context, domain, confidence, last_used
- 定期清理过期/低质量TM条目

### 步骤5：术语表更新与维护
- 收集翻译过程中标记的新术语候选
- 对高频出现的未覆盖术语生成添加建议
- 支持术语表版本管理（新版本生效时间、旧版本保留策略）
- 冲突处理：同一源词在不同术语表中有不同翻译时的优先级规则

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| text | string | 是 | 待检测术语的文本 |
| source_language | string | 是 | 源语言代码 |
| target_language | string | 是 | 目标语言代码 |
| terminology_sets | string[] | 是 | 要加载的术语表ID列表 |
| context | string | 否 | 上下文信息（用于消歧） |
| operation | string | 是 | "detect"(检测)/"apply"(应用)/"update"(更新) |

## 最佳实践

1. **分层术语表**：通用术语表（行业通用） < 企业术语表（公司特有） < 项目术语表（项目特有），优先级依次升高
2. **上下文消歧**：同一词在不同上下文含义不同（"苹果"可能是水果也可能是品牌），需结合上下文判断
3. **不翻译标记**：某些术语需保持源语言形式不翻译（如品牌名"华为"在英文中保持"Huawei"），需在术语表中明确标记
4. **批量预处理**：在翻译开始前批量扫描全文术语，构建替换映射表，避免逐句重复查询
5. **版本同步**：术语表更新后，对正在进行的翻译任务保持旧版本一致性，下一个新任务使用新版本

## 约束与限制

- 术语一致性目标>=98%
- 术语表单次加载大小上限：50,000条（超大术语表需分片加载）
- 术语检测不应显著增加翻译延迟（<50ms附加延迟）
- 该技能不执行翻译本身，仅负责术语的检测、匹配和一致性保障
- 新术语的正式添加需人工审核确认，自动添加仅限建议状态
