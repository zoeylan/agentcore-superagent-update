# 方言识别 (Dialect Recognition)

## 概述

方言识别技能在语种检测之后进一步细分方言变体，尤其针对中国市场200+方言的识别需求。支持粤语、闽南语、四川话、吴语（上海话）、湘语、客家话等主要方言的识别，并管理方言到ASR模型的映射和无模型时的回退策略。该技能确保方言用户也能获得高质量的语音交互体验。

## 适用场景

- 区域性客服中心接入方言来电的自动识别
- 方言内容创作平台的自动标注和分类
- 老年用户群体的方言交互（不熟悉普通话的用户）
- 方言保护和数字化项目中的方言语音采集

## 执行步骤

### 步骤1：方言可能性评估
- 接收language-detection技能的初步判定（已确认为"中文"语系）
- 分析声学特征中的方言指示信号：声调数量和模式（粤语6-9声调 vs 普通话4声调）、韵母系统特征、声母特有音素
- 计算方言存在概率：若概率>0.3，进入方言细分流程

### 步骤2：方言类型分类
- 使用方言分类模型（基于x-vector或ECAPA-TDNN的方言版本）
- 对候选方言计算后验概率，输出Top-3方言候选
- 方言分类层级：语系→方言大区→具体方言点
  - 例：汉语→粤方言→广州话/香港粤语/台山话
- 利用地域先验信息（如来电号码归属地）辅助判定

### 步骤3：模型匹配与回退
- 查询方言ASR模型可用性表：
  - 有专用模型（如粤语ASR）→路由到专用模型
  - 无专用模型但有近似方言模型→使用近似模型 + 降低置信度期望
  - 完全无匹配→回退到通用普通话模型 + 标记"方言回退"
- 对回退场景，在输出中注入`dialect_fallback: true`和`expected_wer_degradation`估计值

### 步骤4：置信度评估与反馈
- 综合声学特征匹配度、模型可用性、转写结果质量评估方言识别置信度
- 低置信度（<0.6）时建议用户确认："检测到您可能在使用[方言名称]，是否切换到对应模式？"
- 记录方言识别结果用于更新用户语言偏好Profile

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| audio_features | tensor | 是 | 语种检测阶段已提取的声学特征 |
| base_language | string | 是 | 已判定的基础语种（如"zh"） |
| geo_hint | string | 否 | 地理位置提示（省份/城市） |
| user_dialect_history | string[] | 否 | 用户历史方言使用记录 |
| available_models | string[] | 否 | 当前可用的方言ASR模型列表 |

## 输出规格

```json
{
  "is_dialect": true,
  "dialect_detected": "yue",
  "dialect_name": "粤语（广州话）",
```

## 最佳实践

1. **层级识别**：先判定方言大区（粤语区/吴语区/闽语区），再细分具体方言点，减少分类空间
2. **地域先验利用**：来电号码归属地、用户注册地等信息可提供强先验，将方言判定准确率提升5-10%
3. **用户偏好缓存**：一旦用户确认方言选择，在后续会话中直接使用缓存偏好，无需重复检测
4. **方言普通话混合**：很多用户说"带方言口音的普通话"，此时应使用普通话模型而非方言模型
5. **渐进式切换**：检测到方言后不立即切换模型，等待连续1秒以上的稳定方言音频再触发切换

## 约束与限制

- 主要方言（粤语、四川话、吴语、闽南语）识别准确率>=85%
- 次要方言（客家话、湘语、赣语等）识别为对应大区即可，不要求精确到方言点
- 普通话和"带口音的普通话"必须准确区分——后者使用普通话模型处理
- 方言ASR模型的WER目标<15%，若达不到需在输出中标记质量预警
- 该技能仅负责方言识别和模型路由建议，不执行实际转写
