# 语码混合分段 (Code-Switching Segmentation)

## 概述

语码混合分段技能处理同一语句或对话中多种语言交替使用的场景（如"这个project的deadline是下周五"），精确检测语言切换边界，将混合语音分割为同语言片段并分别路由至对应ASR引擎处理。该技能是中国商务场景中极为关键的能力——中英混合表达在职场中占比超过40%。

## 适用场景

- 中英混合的商务会议转写
- 中日/中韩混合的跨境沟通
- 科技领域的技术讨论（大量英文术语嵌入中文表达）
- 多语言教育场景的教学录音分析

## 执行步骤

### 步骤1：语码混合检测
- 对语音流进行帧级别（每10ms一帧）的语言后验概率计算
- 使用双语/多语联合声学模型同时计算多个语言的后验概率
- 当帧级语言后验概率发生突变（变化幅度>0.4）时，标记为潜在切换点

### 步骤2：切换边界精化
- 在潜在切换点前后各扩展200ms的窗口
- 使用音素级别的语言判定对窗口内每个音素进行语言归属判断
- 确定精确的切换时间戳（精度±50ms）
- 处理边界模糊情况：对于混合词（如"PPT"在中文语境中的发音）归类到主体语言

### 步骤3：片段分割与标注
- 将语音流按照切换边界分割为独立片段
- 为每个片段标注：`{segment_id, start_ms, end_ms, language, confidence}`
- 合并过短片段（<200ms的孤立片段合并到相邻片段）
- 处理嵌入词：单词级别的语言嵌入（如中文句中的单个英文词）不切割，而是标注为embedded_word

### 步骤4：路由分发
- 将各语言片段分发到对应ASR引擎
- 对嵌入词片段选择主体语言ASR + 混合语言词表
- 收集各引擎返回的转写结果
- 按时间戳顺序合并转写文本，保持原始语码混合形式

## 输入规格

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| audio_stream | bytes | 是 | PCM音频流 |
| primary_language | string | 是 | 主体语言（对话的基础语言） |
| secondary_languages | string[] | 是 | 可能嵌入的辅助语言 |
| min_segment_duration_ms | int | 否 | 最小片段时长（默认200ms） |
| merge_threshold_ms | int | 否 | 片段合并阈值（默认200ms） |

## 最佳实践

1. **双语词表融合**：为中英混合场景维护一个融合词表，包含高频嵌入英文词（meeting、project、deadline等），直接在中文ASR中识别，减少不必要的切分
2. **上下文窗口**：切换边界判定时使用前后各500ms的上下文，避免仅基于单帧做决策
3. **领域适配**：不同行业的语码混合模式不同（IT行业英文嵌入率高于金融），应加载对应领域的切换模型
4. **过切分惩罚**：设定切分数量上限（每秒不超过3个切换点），避免因噪声导致过度切分
5. **嵌入词识别**：建立常用嵌入词库，对于库中单词直接标注为embedded而不切分

## 约束与限制

- 语码混合边界检测准确率目标>=90%
- 仅支持2-3种语言混合的场景，超过3种语言混合的准确率显著下降
- 快速连续切换（<200ms内多次切换）难以精确捕捉，以主体语言处理
- 该技能依赖language-detection技能先行判定是否存在语码混合
