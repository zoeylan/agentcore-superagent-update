# 客户分层分析 (Customer Segmentation Analysis)

## 技能概述

本技能负责基于RFM模型和行为特征双维度对客户进行动态分层，输出可执行的客户分群结果。通过自动化的分层模型运算，识别高价值客户、潜力客户、流失风险客户等关键群体，为差异化运营策略提供数据决策基础。

## 适用场景

- 每周一自动运行的定期分层更新
- 新一批客户数据入库后的即时分层
- 特定营销活动前的目标人群筛选
- 客户行为异常检测（消费骤降/活跃度骤降）
- 高价值客户（TOP 20%）专属运营群体识别

## 执行步骤

### 步骤1：数据准备与清洗
- 提取近90天客户交易和行为数据作为分层基础
- 清洗异常数据（刷单/测试订单/内部员工订单过滤）
- 计算核心指标：R（最近一次消费距今天数）、F（消费频次）、M（累计消费金额）
- 补充行为特征：登录频次、浏览深度、互动率、分享率

### 步骤2：RFM模型计算
- R评分（1-5分）：按消费间隔天数五等分
- F评分（1-5分）：按消费频次五等分
- M评分（1-5分）：按消费金额五等分
- 综合RFM得分 = R×0.3 + F×0.4 + M×0.3（频次权重略高，反映复购导向）

### 步骤3：行为特征聚类
- 使用K-Means/DBSCAN对行为特征进行聚类分析
- 典型行为簇：高活跃高消费、高活跃低消费（浏览不买）、低活跃高消费（忠诚稳定）、低活跃低消费（沉默流失）
- 将行为聚类结果与RFM得分交叉分析

### 步骤4：客户分层定义与分配
- **核心高价值层（TOP 20%）**：RFM≥4.0 且 行为活跃度前20% → 1v1专属运营
- **成长潜力层（20-40%）**：RFM 3.0-4.0 且 行为趋势上升 → 价值提升策略
- **稳定大众层（40-70%）**：RFM 2.0-3.0 → 常规运营维护
- **风险预警层（70-90%）**：RFM 1.5-2.0 或 活跃度持续下降 → 激活策略
- **流失挽回层（90-100%）**：RFM<1.5 或 30天无活跃 → 挽留策略

### 步骤5：变动分析与输出
- 对比上期分层结果，识别升级/降级客户清单
- 分析层级变动原因（消费变化/活跃度变化/时间衰减）
- 输出分层结果报告和运营建议
- 将变动客户清单推送给生命周期管理师

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 分析时间窗口 | DateRange | 是 | 默认近90天，可自定义 |
| 客户画像数据 | Dataset | 是 | 含交易数据和行为数据的客户集 |
| 分层规则配置 | JSON | 否 | 自定义分层阈值，使用默认配置 |
| 上期分层结果 | Dataset | 否 | 用于变动分析对比 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 分层结果清单 | CSV/JSON | 每个客户的层级归属、RFM得分、行为标签 |
| 层级变动报告 | Report | 升降级客户清单及原因分析 |
| 群体画像摘要 | Summary | 各层级客户数量、核心特征、消费贡献占比 |
| 运营建议 | Recommendations | 各层级推荐运营策略和优先级 |

## 最佳实践

1. **动态阈值调整**：分层阈值应根据业务发展阶段和客户结构变化动态调整，避免固化
2. **业务校验**：模型输出需经业务团队抽样校验，确保分层结果符合业务直觉
3. **稳定性考量**：避免客户频繁在层级间跳动，引入"缓冲区"机制（边界值±5%时保持上期层级）
4. **多维交叉**：RFM分层不足以覆盖所有场景，需结合品类偏好、渠道偏好等维度做子分群
5. **时效衰减**：较早的消费行为应给予较低权重，引入时间衰减因子

## 约束条件

- 分层模型每周至少执行一次全量更新
- 分层结果必须覆盖100%活跃客户（无遗漏、无未分类）
- 单次分层计算超时阈值为30分钟，超时需优化算法或分批处理
- 高价值客户层占比控制在15-25%之间，避免过宽或过窄
- 分层结果变动率（单周升降级比例）超过30%时需人工审核模型
