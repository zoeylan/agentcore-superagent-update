# 生命周期策略设计 (Lifecycle Strategy Design)

## 技能概述

本技能负责基于客户所处生命周期阶段和分层数据，设计差异化的运营触达策略。涵盖策略目标设定、渠道选择、时机把控、内容匹配、激励力度规划等全要素，输出可直接执行的策略方案文档。

## 适用场景

- 每周一接收分层更新后的常规策略匹配
- 客户健康度评分预警触发的紧急挽留策略制定
- 新客欢迎流程的自动化策略设计
- 大促/活动前的客户分群策略规划
- 沉默客户批量激活方案设计

## 执行步骤

### 步骤1：策略需求分析
- 明确策略触发条件（定期更新/事件触发/预警触发）
- 确定目标客户群（接收分层数据，明确群体规模和特征）
- 分析客户当前状态和历史响应数据
- 设定策略目标（转化率/响应率/ROI/LTV提升）
- 确认可用资源预算和执行约束

### 步骤2：策略方案设计
- **渠道策略**：根据客户渠道偏好数据选择主触达渠道
  - 微信服务号（适合资讯类、活动类推送）
  - 企微1v1（适合高价值客户关怀）
  - APP Push（适合促销类、限时类通知）
  - 短信（适合重要通知、验证类场景）
  - 电话外呼（适合高价值挽留、重要回访）
- **时机策略**：基于客户活跃时段数据确定最佳触达时间
  - 工作日vs周末差异化
  - 上午/下午/晚间时段选择
  - 事件关联时机（消费后24h/生日前3天/沉默第7天）
- **内容策略**：匹配客户偏好的内容类型和利益点
  - 个性化变量插入（姓名/最近购买品/推荐品类）
  - 核心利益点突出（折扣/赠品/积分/专属服务）
  - 紧迫感营造（限时/限量/专属）
- **激励策略**：根据客户价值和目标ROI设定激励力度
  - 高价值客户：高面额专属券（成本≤预期增量利润30%）
  - 潜力客户：满减门槛券（引导消费升级）
  - 流失客户：无门槛券+赠品（降低回归门槛）

### 步骤3：策略规则配置
- 定义触发规则（IF条件 THEN动作）
- 配置频次控制（单客户单日/单周触达上限）
- 设置互斥规则（同一客户不重复参与多个活动）
- 配置优先级（挽留策略 > 提升策略 > 常规维护）
- 定义策略有效期和自动失效机制

### 步骤4：效果预估与风险评估
- 基于历史同类策略数据预估关键指标
- 识别潜在风险（过度触达/客户投诉/成本超支）
- 制定风险缓解措施和应急预案
- 设置策略熔断条件（投诉率>0.1%自动暂停）

### 步骤5：策略文档输出
- 使用标准模板生成策略方案文档
- 包含完整的执行参数和验收标准
- 输出给执行系统或人工运营团队

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 目标客户清单 | Dataset | 是 | 含客户ID、分层、健康度、标签的客户集 |
| 策略类型 | Enum | 是 | welcome/upgrade/activate/retain/maintain |
| 预算额度 | Number | 是 | 本次策略可用总预算 |
| 执行时间窗口 | DateRange | 是 | 策略执行的起止时间 |
| 历史策略效果 | Dataset | 否 | 同类策略历史执行数据，用于效果预估 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 策略方案文档 | Document | 完整的策略设计方案（含所有执行参数） |
| 执行规则配置 | JSON | 可导入执行系统的策略规则 |
| 效果预估 | Forecast | 预期转化率/ROI/成本等预估值 |
| 内容需求清单 | List | 需要知识库运营师配合的内容素材需求 |

## 最佳实践

1. **客户价值匹配**：激励力度与客户价值正相关，避免对低价值客户过度投入
2. **渠道组合**：单渠道触达率有天花板，重要策略采用多渠道组合（主渠道+备选渠道）
3. **节奏感**：避免一次性集中触达，设计有节奏的多轮触达序列（如3天/7天/14天阶梯）
4. **个性化深度**：至少做到"千人十面"级别的差异化，核心高价值客户追求"千人千面"
5. **及时止损**：策略执行24小时后查看初期数据，效果明显不达预期则及时暂停调整

## 约束条件

- 单客户每周接收的主动触达不超过3次（系统通知不计）
- 挽留策略成本不超过客户预期终身价值的10%
- 策略方案需在接收分层数据后2小时内完成
- 电话外呼仅限工作日9:00-20:00，且限VIP客户
- 所有策略需符合《个人信息保护法》和平台规则
