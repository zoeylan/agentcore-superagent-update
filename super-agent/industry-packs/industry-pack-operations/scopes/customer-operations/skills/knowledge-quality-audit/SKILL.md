# 知识质量审计 (Knowledge Quality Audit)

## 技能概述

本技能负责对知识库进行系统性的质量审计和维护，确保知识库整体准确率≥95%。包括每日增量审核（新入库内容的质量校验）和每周全量审查（全库知识的有效性验证），通过淘汰过期/不准确内容和优化低质量条目来维持知识库的健康度。

## 适用场景

- 每周五执行的全量知识有效性审查
- 新条目入库前的质量审核
- 用户反馈"答案不准确"时触发的定向审计
- 产品/政策变更后的关联知识批量审查
- 月度知识库健康度评估报告生成

## 执行步骤

### 步骤1：审计范围确定
- **增量审核**：锁定自上次审计后新增/修改的知识条目
- **全量审查**：全库知识按以下优先级排序审查：
  1. 用户负反馈标记的条目（最高优先）
  2. 使用频率高但准确度评分低的条目
  3. 超过有效期但未更新的条目
  4. 长期未被使用的条目（>90天无检索命中）
  5. 其他常规条目

### 步骤2：准确性验证
- 核对知识内容与当前业务规则是否一致
- 验证产品信息（价格/规格/功能）是否与最新版本匹配
- 检查政策类知识是否有新版本替代
- 验证流程类知识的步骤是否与当前系统操作一致
- 交叉验证：同一主题多条知识之间是否存在矛盾

### 步骤3：时效性评估
- 标注已过期内容（产品已下架/活动已结束/政策已废止）
- 标注即将过期内容（距有效期截止≤7天）
- 评估"永久有效"类知识是否仍然准确
- 对比行业/竞品动态，评估知识是否需要升级

### 步骤4：质量评分与处置
- 对每条审计知识进行质量评分（1-5分）：
  - 5分：完全准确、表述清晰、时效有效
  - 4分：准确但表述可优化
  - 3分：基本准确但有小瑕疵需修正
  - 2分：部分内容过时或不准确，需大幅修改
  - 1分：完全过时/错误，需淘汰
- 处置决策：
  - 5分：保留，标记为"已审核"
  - 4分：小幅优化后保留
  - 3分：修正后重新入库
  - 2分：标记"暂时下线"，安排重写
  - 1分：标记"淘汰"，移入归档库

### 步骤5：审计报告输出
- 统计本次审计覆盖量和各评分分布
- 输出淘汰/修改/新增的知识条目清单
- 计算知识库整体健康度指标：
  - 准确率 = 4-5分条目占比（目标≥95%）
  - 时效率 = 有效期内条目占比（目标≥90%）
  - 覆盖率 = 高频问题有对应知识的比例（目标≥85%）
- 输出优化建议（知识缺口/热点变化/趋势预判）

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 审计类型 | Enum | 是 | incremental（增量）/ full（全量）/ targeted（定向） |
| 审计范围 | Object | 否 | 定向审计时指定知识分类/ID范围 |
| 用户反馈数据 | Dataset | 否 | 用户对知识准确性的反馈记录 |
| 业务变更通知 | Array | 否 | 近期产品/政策变更清单 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 审计报告 | Report | 完整的审计结果和健康度指标 |
| 处置清单 | List | 需淘汰/修改/重写的条目清单 |
| 知识缺口报告 | Report | 发现的知识空白区域 |
| 优化建议 | Recommendations | 知识库改进方向和优先级 |

## 最佳实践

1. **风险导向**：优先审计高使用频率+低评分的知识（影响面大）
2. **变更联动**：产品/政策有变更时立即触发关联知识审计，不等周五
3. **用户反馈闭环**：用户报告不准确后24小时内完成定向审计和修正
4. **量化追踪**：每月对比知识库健康度趋势，确保持续改善
5. **预防性维护**：对即将到期的知识提前发出续期/更新提醒

## 约束条件

- 全量审查每周五必须完成，不可跳过
- 单次全量审查需在4小时工作时间内完成
- 淘汰知识不物理删除，转入归档库保留180天
- 准确率低于90%时触发紧急整改（非常规审计频率加倍）
- 审计结果需保留完整记录，支持回溯追踪
