# A/B测试管理 (A/B Testing Management)

## 技能概述

本技能负责客户运营策略的A/B测试全流程管理，包括实验设计、样本分配、效果监测、统计显著性判定和结论输出。通过科学的对照实验确保策略优化有据可依，避免经验主义决策，持续驱动运营转化率提升。

## 适用场景

- 新策略上线前的效果验证（触达方案A vs B）
- 不同渠道组合的效果对比
- 不同激励力度的ROI最优解探索
- 触达时机（上午/下午/晚间）效果对比
- 内容文案/创意素材的转化率对比

## 执行步骤

### 步骤1：实验方案设计
- 明确测试假设（H0: A/B无差异；H1: B优于A）
- 确定测试变量（单变量测试为主，避免多变量混淆）
- 定义对照组和实验组（至少2组，最多4组）
- 确定核心评估指标（主指标1个+辅助指标2-3个）
- 设定预期效果差异（MDE, Minimum Detectable Effect）

### 步骤2：样本量计算与分配
- 基于以下参数计算最小样本量：
  - 基线转化率（当前策略的平均转化率）
  - 预期提升幅度（通常设定5-20%相对提升）
  - 显著性水平α=0.05
  - 统计功效1-β=0.80
- 采用随机分流算法分配用户（hash取模保证一致性）
- 确保各组用户特征无显著差异（AA检验）

### 步骤3：实验执行与监控
- 启动实验，记录开始时间和初始状态
- 每日监控核心指标变化趋势
- 检查数据质量（分流比例偏差<1%、样本污染率<2%）
- 设置安全熔断条件（实验组显著恶化时自动停止）
- 实验周期不少于7天（覆盖完整周期波动）

### 步骤4：统计检验与结论判定
- 实验到期后进行统计检验：
  - 连续指标（如客单价）：双样本t检验
  - 比例指标（如转化率）：卡方检验或z检验
  - 计算p值和置信区间
- 判定标准：p<0.05 且 效果差异具有业务意义
- 结论分类：显著优胜→全量推广；不显著→保持现状或重新设计；显著劣势→立即终止

### 步骤5：结论输出与知识沉淀
- 输出实验报告（含实验设计/数据结果/统计检验/结论/建议）
- 显著结果：生成全量推广执行方案
- 不显著结果：分析可能原因，建议调整变量重新测试
- 将实验结论归档到策略效果知识库（供后续决策参考）

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 测试策略方案 | Object | 是 | 含对照组和实验组的完整策略描述 |
| 目标客户池 | Dataset | 是 | 可供分流的客户总池（需足够大） |
| 核心指标 | Array | 是 | 主评估指标和辅助指标定义 |
| 测试周期 | Number | 是 | 实验持续天数（最少7天） |
| 基线数据 | Object | 否 | 历史基线转化率（用于样本量计算） |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 实验报告 | Document | 完整的A/B测试结果报告 |
| 统计结论 | Enum | significant_win / significant_loss / inconclusive |
| 推广方案 | Document | 显著优胜时输出全量推广方案 |
| 经验沉淀 | Knowledge | 实验结论归档到知识库 |

## 最佳实践

1. **单变量原则**：每次只测一个变量，多变量用正交实验设计
2. **充足样本**：宁可延长测试周期也不要样本不足就下结论
3. **避免偷看**：不要在实验中途频繁检查结果并据此调整（多重比较问题）
4. **业务意义**：统计显著不等于业务显著，0.1%的转化率提升可能不值得全量投入
5. **环境控制**：实验期间避免其他重大变更（大促/系统升级等），否则结论不可信

## 约束条件

- 单次A/B测试周期不少于7天，不超过30天
- 每组最小样本量不低于1000人
- 同一客户不得同时参与超过2个A/B测试
- 涉及价格/优惠的测试需财务审批
- 测试结果3天内完成分析输出，不拖延决策
