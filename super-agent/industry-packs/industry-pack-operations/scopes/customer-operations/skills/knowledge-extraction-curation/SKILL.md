# 知识提取与整理 (Knowledge Extraction & Curation)

## 技能概述

本技能负责从海量客户交互记录中自动提取有价值的知识条目，包括FAQ、话术模板、最佳实践和运营规则。通过NLP技术实现知识的自动发现、结构化整理、语义去重和入库管理，确保知识库持续增长且无冗余。

## 适用场景

- 每日14:00定时扫描过去24小时新增交互记录
- 新产品上线后的集中知识采集
- 客户投诉集中爆发时的紧急知识补充
- 客户洞察分析师发出行为模式变化信号时的触发更新
- 季节性/活动期间的知识预备性补充

## 执行步骤

### 步骤1：数据源扫描与采集
- 扫描客服工单系统（已关闭/已解决的工单）
- 采集企微/微信社群互动记录（标记为有价值的对话）
- 获取在线客服聊天记录（高满意度评分的对话优先）
- 收集投诉反馈处理记录（含解决方案的完整case）
- 汇总内部运营人员的经验分享文档

### 步骤2：知识候选项提取
- **FAQ提取**：识别高频问题模式（同类问题出现≥5次）
  - 提取问题表述（含多种表达方式变体）
  - 提取最佳回答（参考满意度评分最高的回复）
- **话术提取**：从高转化率/高满意度对话中提取有效话术
  - 标注适用场景和客户类型
  - 标注关键话术技巧点
- **最佳实践提取**：从复杂问题的成功处理案例中提炼方法论
  - 抽象为可复用的步骤流程
  - 标注适用条件和注意事项
- **规则更新提取**：识别新的业务规则或政策变更
  - 对比现有规则库识别差异
  - 标注生效时间和影响范围

### 步骤3：语义去重检测
- 对候选知识条目进行向量化表示（BERT Sentence Embedding）
- 与现有知识库条目计算余弦相似度
- 相似度>0.85判定为语义重复：
  - 完全重复（>0.95）：丢弃新条目
  - 高度相似（0.85-0.95）：自动合并，保留最新版本内容
  - 补充关系（0.70-0.85）：作为现有条目的补充说明纳入
- 相似度<0.70：判定为新知识，进入审核队列

### 步骤4：结构化整理与标注
- 按标准模板格式化知识条目：
  - 标题（精确描述知识主题，20字以内）
  - 关键词（3-5个，用于检索匹配）
  - 分类路径（产品线>问题类型>客户类型）
  - 正文内容（问题描述+标准答案/处理步骤）
  - 适用范围（客户类型/场景条件/渠道限制）
  - 有效期（永久/截止日期/需定期复审）
  - 来源出处（源数据ID和提取时间）
- 自动标注置信度评分（基于数据源质量和出现频次）

### 步骤5：审核入库
- 高置信度条目（≥0.8）：进入快速审核通道
- 中置信度条目（0.5-0.8）：进入标准审核队列
- 低置信度条目（<0.5）：标记为"待验证"，需专人确认
- 审核确认率≥80%的条目自动入库生效
- 审核拒绝的条目标注拒绝原因供模型学习

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 数据源范围 | Array | 是 | 扫描的数据源列表和时间范围 |
| 提取类型 | Enum | 否 | faq/script/best_practice/rule/all |
| 触发信号 | Object | 否 | 来自洞察分析师的行为变化信号 |
| 现有知识库快照 | Reference | 是 | 用于去重比对的现有知识库索引 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 候选知识清单 | Array | 结构化的候选知识条目列表 |
| 去重报告 | Report | 重复/合并/新增的统计和详情 |
| 审核队列 | Queue | 按置信度排序的待审核条目 |
| 更新日志 | Log | 本次提取的数量/类型/质量统计 |

## 最佳实践

1. **质量优先于数量**：宁可漏提也不要错提，低质量知识入库会污染整个知识库
2. **多角度表述**：FAQ的问题端收集多种表述方式，提升检索命中率
3. **时效性标注**：所有知识条目必须标注有效期，过期知识比没有知识更有害
4. **来源可追溯**：每条知识都可追溯到原始数据源，便于验证和更新
5. **增量优化**：关注知识的增量价值，避免重复录入已有知识的不同表述

## 约束条件

- 每日增量扫描必须在14:00-18:00时间窗口内完成
- 单次提取的候选知识条目不超过200条（超出分批处理）
- 客户隐私信息（姓名/手机号/地址）必须脱敏后入库
- 涉及合规/法律的知识条目必须人工审核，不可自动入库
- 知识条目正文长度控制在500字以内，超长内容需拆分
