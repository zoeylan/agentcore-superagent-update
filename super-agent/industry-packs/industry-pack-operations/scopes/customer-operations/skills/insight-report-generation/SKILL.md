# 客户洞察报告生成 (Insight Report Generation)

## 技能概述

本技能负责自动化生成周度客户洞察报告，通过整合多维数据分析结果，输出包含趋势分析、异常检测和机会识别三大模块的结构化报告。报告面向运营管理层和一线运营团队，提供数据驱动的决策依据和可执行的运营建议。

## 适用场景

- 每周日自动生成周度客户洞察报告
- 月度客户运营复盘报告
- 特定事件（大促/活动/产品上线）前后的专项分析报告
- 管理层即时分析需求响应
- 客户异常预警触发的紧急分析报告

## 执行步骤

### 步骤1：数据汇总与口径对齐
- 自动采集本周期内所有客户相关核心指标数据
- 核对数据完整性（各源系统数据到达率≥99%）
- 统一指标计算口径（活跃定义/转化定义/流失定义）
- 标注数据质量状态（正常/延迟/缺失/异常）

### 步骤2：趋势分析模块
- 计算核心指标时间序列（日活/周活/交易额/客单价/复购率）
- 执行同比（YoY）和环比（WoW/MoM）分析
- 绘制趋势线并进行短期预测（ARIMA/指数平滑）
- 识别趋势拐点和加速/减速信号
- 输出趋势解读和预测展望

### 步骤3：异常检测模块
- 基于历史基线（均值±2σ）检测指标异常偏离
- 对异常客户群进行下钻分析（地域/渠道/产品维度）
- 识别异常原因假设（外部事件/竞品活动/系统故障/自然波动）
- 评估异常影响范围和严重程度
- 标注需紧急关注的异常项（红色预警）

### 步骤4：机会识别模块
- 识别高潜力客户群（高活跃未消费/高消费可升级/交叉销售机会）
- 发现运营改进点（漏斗断层/转化瓶颈/渠道低效）
- 对标行业benchmark识别差距
- 为每个机会点提供具体的运营动作建议
- 预估机会价值（潜在GMV增量/客户数增量）

### 步骤5：报告组装与质量审核
- 按标准模板组装完整报告（执行摘要→趋势→异常→机会→建议）
- 数据准确性校验（关键指标与源系统核对，差异率<0.1%）
- 分析逻辑自洽性检查（结论是否有数据支撑）
- 可读性审核（去除专业术语或补充解释、控制篇幅）
- 输出最终报告并分发给相关干系人

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 报告周期 | DateRange | 是 | 分析的时间范围（周/月/自定义） |
| 核心指标集 | Array | 是 | 需纳入分析的指标列表 |
| 对比基准 | Enum | 是 | 环比/同比/自定义基准期 |
| 受众类型 | Enum | 否 | 管理层（摘要型）/ 运营团队（详细型） |
| 专项主题 | String | 否 | 如有特定分析主题则聚焦该领域 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 完整报告 | Document | 含趋势/异常/机会三大模块的结构化报告 |
| 执行摘要 | Summary | 1页内核心发现和建议（管理层速览） |
| 数据附录 | Spreadsheet | 详细数据表格支撑 |
| 预警清单 | Alert List | 需即时关注的异常项清单 |

## 最佳实践

1. **结论先行**：报告以执行摘要开头，最重要的3个发现放在最前
2. **数据可视化**：用图表替代大段数字描述，趋势用折线图，对比用柱状图，占比用环形图
3. **可执行导向**：每个分析发现必须对应具体的运营建议（谁/做什么/何时/预期效果）
4. **分层阅读**：设计三级阅读深度——标题扫读(30秒)/摘要快读(3分钟)/全文精读(15分钟)
5. **数据溯源**：关键结论需标注数据来源和计算方法，便于验证

## 约束条件

- 周报必须在每周日24:00前完成草稿生成
- 报告中数据必须100%与源系统一致（允许四舍五入误差）
- 分析覆盖度必须含趋势、异常、建议三要素，缺一不可
- 单份报告控制在15页以内（不含数据附录）
- 预警类发现需在报告外单独即时推送，不等待报告周期
