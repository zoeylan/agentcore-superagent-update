# 客户画像构建 (Customer Profile Building)

## 技能概述

本技能负责从多渠道数据源中整合客户信息，构建并维护360度客户全景画像。涵盖跨渠道ID统一识别、多维标签体系构建、客户健康度评分计算三大核心能力，是客户运营数据基础设施的核心组件。

## 适用场景

- 新客户首次数据采集与画像初始化
- 每日T+1批量画像更新（标签刷新）
- 实时事件触发的即时标签更新（下单/退款/投诉等）
- 跨渠道客户身份合并与冲突解决
- 客户健康度评分每日刷新

## 执行步骤

### 步骤1：多源数据采集
- 连接交易系统获取订单数据（订单号、商品、金额、时间、渠道）
- 采集行为埋点数据（浏览路径、停留时长、点击热图、搜索关键词）
- 获取社交平台数据（微信互动、社群发言、分享行为）
- 拉取客服工单数据（咨询内容、投诉记录、满意度评分）
- 同步线下门店POS数据（到店频次、消费记录、会员卡使用）

### 步骤2：跨渠道ID-Mapping
- 以手机号为主键进行第一轮匹配
- 使用设备指纹(Device ID)进行第二轮补充匹配
- 通过微信UnionID/OpenID关联微信生态数据
- 使用概率匹配模型处理模糊匹配场景
- 冲突解决规则：以最近一次实名交易数据为准，标记待人工核实

### 步骤3：标签体系构建与更新
- **基础属性标签**：性别、年龄段、城市级别、职业标签
- **行为标签**：最近浏览品类、收藏偏好、购物车行为模式
- **交易标签**：RFM三维评分、客单价区间、促销敏感度
- **偏好标签**：品类偏好Top3、价格带偏好、渠道偏好
- **预测标签**：流失概率、升级概率、交叉销售概率

### 步骤4：健康度评分计算
- 消费频次变化（权重30%）：近30天vs前30天对比
- 活跃度趋势（权重25%）：登录/浏览/互动行为综合
- 投诉记录（权重20%）：近90天投诉次数和严重等级
- NPS反馈（权重15%）：最近一次NPS评分
- 权益使用率（权重10%）：优惠券/积分等权益使用情况
- 输出0-100分，<60分触发预警

### 步骤5：质量校验与输出
- 标签覆盖率检查（目标≥98%）
- 跨渠道ID匹配率检查（目标≥95%）
- 异常数据检测（极端值/逻辑矛盾/数据缺失）
- 输出画像更新日志和质量报告

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 数据源配置 | JSON | 是 | 各渠道数据源连接信息和字段映射 |
| 更新模式 | Enum | 是 | full_batch（全量）/ incremental（增量）/ realtime（实时） |
| 客户ID列表 | Array | 否 | 指定客户更新时提供，为空则全量 |
| 触发事件 | Object | 否 | 实时模式下的触发事件详情 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 客户画像数据 | JSON | 完整的客户360画像，含全维度标签 |
| 健康度评分 | Number | 0-100分，含各维度得分明细 |
| 更新日志 | Log | 本次更新的变更记录（新增/修改/删除标签） |
| 质量报告 | Report | 覆盖率、匹配率、异常数据统计 |

## 最佳实践

1. **数据新鲜度优先**：实时事件优先处理，批量更新作为补充
2. **渐进式丰富**：不要等所有数据齐全才建画像，先建基础框架再逐步补充
3. **冲突最小化**：建立明确的数据优先级规则（实名交易>行为推断>第三方数据）
4. **隐私合规**：所有数据处理遵循《个人信息保护法》，敏感信息加密存储
5. **版本管理**：画像变更保留历史版本，支持回溯分析

## 约束条件

- 跨渠道ID匹配准确率必须≥95%，低于此阈值需人工复核
- 实时标签更新延迟不超过5分钟
- 批量更新需在每日凌晨02:00-06:00窗口期内完成
- 客户隐私数据（手机号/身份证等）不得明文出现在标签中
- 单次批量处理客户数上限100万，超出需分批执行
