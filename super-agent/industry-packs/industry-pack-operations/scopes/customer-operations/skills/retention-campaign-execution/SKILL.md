# 挽留策略执行 (Retention Campaign Execution)

## 技能概述

本技能负责客户流失挽留策略的全流程执行管理，从预警触发到策略下发、从执行监控到效果回收。聚焦于将策略设计转化为实际的客户触达动作，确保挽留策略在正确的时间、通过正确的渠道、以正确的内容触达目标客户，并持续追踪挽留效果。

## 适用场景

- 客户健康度评分<60分触发的P1挽留流程
- 客户行为异常（消费骤降/活跃度骤降）的48小时干预
- 沉默客户（7/14/30天未活跃）的阶梯式激活
- 大促后沉寂客户的回访触达
- 会员降级预警客户的保级挽留

## 执行步骤

### 步骤1：挽留触发确认
- 接收预警信号（健康度<60分 / 行为异常检测 / 沉默天数达标）
- 验证客户当前状态（排除已购买/已联系/已标记勿扰的客户）
- 确认客户分层和历史价值（决定挽留投入力度）
- 检查客户近期触达记录（避免过度打扰）
- 确定挽留优先级（P1紧急/P2标准/P3观察）

### 步骤2：策略匹配与个性化
- 根据客户分层和流失原因匹配预设策略模板
- 执行个性化变量填充：
  - 客户称呼和关怀话术
  - 推荐商品（基于历史偏好）
  - 优惠券面额和使用条件
  - 触达渠道和时间
- 生成个性化触达内容（从知识库话术模板库获取）

### 步骤3：多渠道触达执行
- **第一轮（0-4小时）**：主渠道触达
  - 高价值客户：企微1v1消息 + 专属优惠券
  - 中价值客户：APP Push + 小程序弹窗
  - 普通客户：短信通知 + 优惠券自动发放
- **第二轮（24-48小时）**：未响应客户的补充触达
  - 切换备选渠道再次触达
  - 升级利益点（加大优惠力度）
- **第三轮（72小时后）**：最终挽留
  - 高价值客户：电话外呼关怀
  - 其他客户：最后一次高面额无门槛券

### 步骤4：执行监控与异常处理
- 实时监控触达状态（发送成功率/到达率/阅读率）
- 检测异常情况：
  - 渠道发送失败 → 自动切换备选渠道
  - 客户投诉/标记骚扰 → 立即停止并标记黑名单
  - 优惠券核销异常 → 通知财务核实
- 每日汇总执行进度报告

### 步骤5：效果追踪与反馈
- 挽留后7天追踪期：监控客户是否回归活跃
- 成功标准定义：7天内产生消费行为 或 恢复周活跃≥3次
- 计算挽留成功率和ROI
- 失败案例分析（挽留未成功的原因归类）
- 效果数据回传给客户洞察分析师优化预警模型
- 成功话术/方案沉淀到知识库

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 预警客户清单 | Dataset | 是 | 含客户ID、健康度、流失原因、客户价值等 |
| 挽留策略配置 | JSON | 是 | 匹配好的策略方案（渠道/内容/激励） |
| 可用优惠资源 | Object | 是 | 本批次可用的优惠券/积分/赠品库存 |
| 触达渠道状态 | Object | 是 | 各渠道可用性和当日剩余额度 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 执行进度报告 | Report | 实时触达状态和累计完成量 |
| 效果追踪报告 | Report | 7天后的挽留效果统计 |
| 成功案例库 | Knowledge | 高效挽留话术和方案沉淀 |
| 模型反馈数据 | Dataset | 回传给预警模型的效果标签 |

## 最佳实践

1. **黄金时间**：预警触发后4小时内完成首次触达，响应越快成功率越高
2. **温度递增**：先温和关怀再利益刺激，避免首次就发大额券（降低客户预期锚定）
3. **真诚为本**：挽留话术避免套路感，体现对客户的真实关注和价值认可
4. **渠道适配**：不同年龄/偏好的客户对渠道敏感度不同，年轻人偏APP，中年人偏微信
5. **止损机制**：单客户挽留成本超过其6个月预期贡献时果断放弃，资源转投高潜力客户

## 约束条件

- P1预警必须4小时内首次触达，P2在24小时内
- 单客户单次挽留流程触达总次数不超过5次
- 电话外呼限工作日9:00-20:00，单日外呼量有上限
- 优惠券发放需在预算额度内，超支需审批
- 客户标记"拒绝联系"后90天内不再主动触达
