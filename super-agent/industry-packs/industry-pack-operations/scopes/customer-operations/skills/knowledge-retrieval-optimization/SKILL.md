# 知识检索优化 (Knowledge Retrieval Optimization)

## 技能概述

本技能负责持续优化知识库的检索体验和命中率，确保一线运营人员和客服系统能快速、准确地从知识库中获取所需信息。涵盖检索算法优化、分类体系管理、同义词/近义词扩展、检索日志分析和用户体验迭代。目标是知识检索命中率≥85%。

## 适用场景

- 检索命中率低于85%时的专项优化
- 知识库结构重组（分类体系调整）
- 新业务线/新产品上线后的知识检索适配
- 季度性检索效果评估与优化
- 用户高频"未找到结果"反馈的分析处理

## 执行步骤

### 步骤1：检索效果诊断
- 分析检索日志数据：
  - 总检索量和趋势变化
  - 命中率（返回结果/总检索次数）
  - 首位命中率（第一条结果即为所需答案的比例）
  - 零结果率（完全无匹配的查询占比）
  - 平均检索耗时
- 识别高频零结果查询（知识缺口信号）
- 分析低命中查询的query特征（长句/方言/缩写/错别字）

### 步骤2：检索算法优化
- **关键词匹配优化**：
  - 扩展同义词库（如"退款"="退钱"="退货退款"="申请退"）
  - 添加行业术语映射（如"SKU"="商品编码"）
  - 配置错别字纠正规则（如"退欹"→"退款"）
- **语义检索优化**：
  - 更新语义模型（基于最新交互数据微调）
  - 调整语义相似度阈值（精准vs召回平衡）
  - 增加意图识别层（区分咨询/投诉/查询/操作指引）
- **排序策略优化**：
  - 引入知识条目质量评分作为排序因子
  - 引入使用频率和最近使用时间作为排序因子
  - 引入用户反馈正负比作为排序因子

### 步骤3：分类体系管理
- 评估当前分类体系的合理性：
  - 各分类节点下的知识条目数量分布是否均匀
  - 分类层级是否过深（建议不超过4级）
  - 是否存在分类边界模糊的情况
- 基于检索热点和知识增长趋势调整分类结构
- 维护分类标签的多级索引关系

### 步骤4：用户行为分析与优化
- 追踪用户检索行为路径（搜→点→用→反馈）
- 分析"二次搜索"行为（用户修改关键词再搜 = 首次结果不满意）
- 识别高价值检索模式并创建快速通道
- 为高频场景创建"一键直达"入口（如Top20问题）
- A/B测试不同检索结果展示方式的满意度

### 步骤5：效果验证与报告
- 对比优化前后的核心指标变化：
  - 命中率提升幅度
  - 首位准确率提升幅度
  - 零结果率下降幅度
  - 平均检索耗时变化
- 统计用户满意度反馈变化
- 输出优化效果报告和下一步计划
- 将成功优化策略沉淀为标准化操作规范

## 输入规格

| 输入项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 检索日志数据 | Dataset | 是 | 近期检索记录（含query/结果/点击/反馈） |
| 当前知识库索引 | Reference | 是 | 知识库全量索引和分类结构 |
| 用户反馈数据 | Dataset | 否 | 检索满意度评分和文字反馈 |
| 同义词库 | Dictionary | 否 | 现有同义词配置 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 优化方案 | Document | 具体的检索优化措施和实施计划 |
| 同义词扩展 | Dictionary | 新增的同义词/近义词映射 |
| 分类调整方案 | Document | 分类体系优化建议 |
| 效果报告 | Report | 优化前后指标对比 |

## 最佳实践

1. **数据驱动**：所有优化决策基于检索日志数据分析，不凭感觉调整
2. **渐进式优化**：每次只调整一个维度，观察效果后再做下一步
3. **用户视角**：站在一线运营人员角度思考检索场景，他们的表达方式≠标准术语
4. **覆盖长尾**：不仅优化高频查询，也关注长尾查询的覆盖（往往是新问题信号）
5. **平衡精准与召回**：宁可多返回几条相关结果让用户选择，不要过度精确导致漏检

## 约束条件

- 检索命中率持续低于80%时必须在72小时内启动专项优化
- 同义词库更新后需回归测试，确保不引入误匹配
- 分类体系调整需评估对现有用户使用习惯的影响
- 检索响应时间不超过500ms（含语义检索）
- 月度检索效果报告需包含趋势分析和优化计划
