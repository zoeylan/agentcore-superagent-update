# 自动化流程监控 (Automation Monitoring)

## 技能概述

自动化流程监控技能负责对所有运行中的自动化流程进行健康度监控，及时发现异常并触发修复流程，确保自动化流程可用率≥99%。该技能是自动化运维的核心保障能力。

## 适用场景

- 日常健康检查：每日自动检查所有流程运行状态
- 故障检测与响应：30分钟内发现故障并自动回退
- 性能监控：执行时间/成功率的趋势追踪
- 容量预警：数据量增长导致的性能瓶颈预测

## 执行步骤

### 步骤1：健康状态采集
- 各自动化流程的最近执行状态（成功/失败/超时/跳过）
- 执行时间趋势（是否有逐渐变慢的趋势）
- 数据处理量统计（输入/输出/异常数据条数）
- 依赖系统的可用性（数据源/目标系统/中间件）

### 步骤2：异常检测
- **执行失败**：连续失败3次触发告警（含错误信息分析）
- **执行超时**：超过正常执行时间2倍触发超时告警
- **数据异常**：输出数据量与预期偏差>30%触发检查
- **依赖异常**：上游数据源不可用或数据延迟

### 步骤3：故障响应
- 自动回退：连续失败>3次，自动暂停流程并切回人工处理模式
- 错误分类：区分暂时性错误（网络抖动/超时）和永久性错误（数据格式变更/接口失效）
- 暂时性错误：自动重试（最多3次，指数退避）
- 永久性错误：停止重试，生成维修工单，通知维护人员

### 步骤4：监控报告输出
- 日报：各流程运行状态摘要和异常事件
- 周报：成功率趋势、执行时间趋势、异常统计
- 月报：可用率统计、异常修复时长分布、改进建议
- 实时大盘：所有流程的实时状态仪表盘

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 流程执行日志 | 日志数据 | 是 | 各流程的执行记录 |
| 监控指标 | 时序数据 | 是 | 成功率/执行时间/数据量 |
| 告警规则配置 | 配置 | 是 | 各类异常的触发条件 |
| 维护人员清单 | 列表 | 否 | 各流程的负责维护人员 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 实时状态大盘 | 结构化数据 | 各流程当前状态 |
| 告警通知 | 结构化消息 | 含流程名/异常类型/影响/建议 |
| 故障分析报告 | Markdown | 根因分析和修复建议 |
| 月度运维报告 | Markdown | 可用率/异常统计/改进计划 |

## 最佳实践

1. **30分钟检出SLA**：故障从发生到被检出不超过30分钟
2. **2小时修复目标**：异常修复平均时间≤2小时
3. **告警去重**：同一故障不重复告警，首次告警后进入修复跟踪模式
4. **根因优于症状**：告警信息需包含错误上下文，帮助快速定位
5. **趋势性预警**：在故障发生前（如执行时间逐步变长）提前预警

## 约束条件

- 自动回退不需要人工审批，确保业务连续性优先
- 监控本身不能成为系统负担（轻量级探针）
- 敏感数据的监控日志需脱敏处理
- 自动化流程可用率目标≥99%（月度统计）
