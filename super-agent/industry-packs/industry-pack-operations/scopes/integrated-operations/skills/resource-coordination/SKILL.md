# 资源协调 (Resource Coordination)

## 技能概述

资源协调技能负责在多个项目/任务争夺同一资源时进行优先级排序和协调，确保关键项目的资源需求得到满足，同时最大化整体资源利用效率。该技能是跨域协同中处理冲突的核心能力。

## 适用场景

- 人力资源冲突：同一人被多个项目需要
- 预算资源竞争：多个活动争夺有限预算额度
- 系统资源争用：多个任务需要同一系统/数据支持
- 时间窗口冲突：多个活动的执行时间重叠

## 执行步骤

### 步骤1：冲突识别
- 扫描任务列表中的资源分配情况
- 识别同一资源被多个任务请求的冲突点
- 评估冲突的严重程度（是否可通过时间错开解决/是否必须二选一）
- 收集各方的需求详情（需要什么资源/什么时间段/多长时间）

### 步骤2：优先级评估
- 一级规则：按项目优先级排序 P0 > P1 > P2
- 二级规则：同级别按截止时间紧迫度排序（越早需要越优先）
- 三级规则：同等条件下，按业务影响范围排序（影响面大优先）
- 特殊规则：战略级项目/管理层指定优先的项目享有优先权

### 步骤3：协调方案制定
- **时间错开方案**：如果资源需求时段可调整，安排顺序使用
- **并行分割方案**：资源可部分共享时，按比例分配
- **替代资源方案**：寻找替代资源满足低优先级需求
- **范围调整方案**：协商低优先级任务缩小范围以减少资源需求
- 制定每个方案的影响评估

### 步骤4：沟通与确认
- 向各相关方发送协调方案（含理由/影响/替代建议）
- 收集各方反馈和异议
- 异议处理：
  - 有合理理由的：重新评估优先级
  - 无法达成一致的：升级至管理层裁决
- 确认最终方案并更新任务计划

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 冲突任务列表 | 列表 | 是 | 涉及的任务和各自需求 |
| 资源可用情况 | 结构化数据 | 是 | 被争夺资源的当前使用状态 |
| 项目优先级 | 枚举 | 是 | 各项目的优先级标注 |
| 时间线信息 | 日期 | 是 | 各任务的截止时间 |
| 替代资源清单 | 列表 | 否 | 可替代的资源选项 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 冲突分析报告 | Markdown | 冲突详情和影响评估 |
| 协调方案 | 列表 | 建议的资源分配方案 |
| 各方影响说明 | 表格 | 方案对各项目的影响 |
| 裁决请求（如需） | 结构化消息 | 需管理层决策的冲突 |

## 最佳实践

1. **客观中立**：基于优先级和数据做出排序，不偏袒任何一方
2. **透明沟通**：协调决定的理由和规则公开透明
3. **寻找双赢**：优先寻找不伤害任何一方的创造性解决方案
4. **预防优于治疗**：通过全景视图提前发现潜在冲突，在任务启动前协调
5. **记录先例**：类似冲突的处理决策记录供后续参考

## 约束条件

- 不直接裁决战略级项目间的冲突（需升级管理层）
- 协调方案需各方确认后方可执行
- 不可强制某方放弃资源，只能建议和协调
- 资源协调结果需同步更新到项目计划中
