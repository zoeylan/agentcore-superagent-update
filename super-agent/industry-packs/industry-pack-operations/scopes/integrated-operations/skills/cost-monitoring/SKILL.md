# 成本监控与预警 (Cost Monitoring)

## 技能概述

成本监控技能负责建立多级预警机制，实时跟踪各业务线的成本执行情况，在成本异常趋势形成初期即发出预警，防止成本失控。该技能是财务管控体系的防线，确保成本始终在可控范围内。

## 适用场景

- 日常成本监控：每日更新各业务线成本执行进度
- 多级预警触发：黄色10%/橙色15%/红色20%分级预警
- 成本趋势预测：基于消耗速度预测月末是否超支
- 成本结构分析：各科目/渠道的成本占比变化追踪

## 执行步骤

### 步骤1：数据采集与更新
- 每日10:00前完成前日成本数据更新
- 覆盖维度：业务线、成本科目、渠道、项目
- 累计执行额与预算对比，计算消耗进度百分比
- 同步获取业务指标（用于判断成本增加是否带来对应业务增长）

### 步骤2：预警检测
- **时间进度法**：月度已消耗预算比例 vs 时间进度比例
- **趋势外推法**：按近7日消耗速度预测月末累计支出
- **异常突变法**：单日支出超过日均值2倍触发即时检测
- 预警分级：
  - 黄色预警：偏差10-15%，建议关注
  - 橙色预警：偏差15-20%，建议干预
  - 红色预警：偏差>20%，建议冻结新增支出

### 步骤3：预警报告生成
- 预警等级和触发原因
- 当前支出明细和超支金额
- 按当前趋势的月末预测支出
- 初步归因（关联近期业务事件/策略变化）
- 建议措施和预期节省效果

### 步骤4：预警闭环跟踪
- 预警发出后跟踪是否24小时内被确认
- 记录管理层的处置决定（调整/忽略/冻结）
- 追踪措施实施后成本是否回归正常轨道
- 未响应的预警自动升级

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 实时成本数据 | 结构化数据 | 是 | 各维度当期累计支出 |
| 预算基准 | 结构化数据 | 是 | 对应周期的预算 |
| 预警阈值配置 | 数值 | 否 | 默认10%/15%/20% |
| 业务指标数据 | 结构化数据 | 否 | 用于判断成本合理性 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 成本监控日报 | 表格 | 各维度消耗进度和健康状态 |
| 预警通知 | 结构化消息 | 含等级、原因、影响、建议 |
| 月末预测报告 | 数值+文本 | 预测累计支出和偏差 |
| 预警处置跟踪 | 列表 | 各预警的响应和处理状态 |

## 最佳实践

1. **预测性预警优于事后预警**：在趋势形成初期（连续3天消耗加速）即提醒
2. **区分正常波动和异常**：大促前投放加量属于计划内增长，不应误报
3. **关联业务看成本**：成本上升但带来更高ROI时，可上调预警阈值
4. **预警疲劳防控**：同一问题不重复预警，已确认忽略的不再触发
5. **月中Check Point**：月度过半时执行一次全面成本健康检查

## 约束条件

- 预警仅为建议，冻结支出需管理层审批
- 不直接干预正在执行的投放计划
- 数据延迟情况需在报告中标注
- 预警阈值可由管理层动态调整
