# 超时升级管理 (Escalation Management)

## 技能概述

超时升级管理技能负责对逾期任务进行分级自动升级提醒，确保问题不被拖延。通过结构化的三级升级机制（负责人→上级→管理层），在适当的时间将适当的信息传递给适当的人，推动阻塞问题的解决。

## 适用场景

- 任务超时预警：任务超过约定时间50%/100%/200%的分级提醒
- 响应超时升级：分配后未确认/进度未更新的升级
- 阻塞问题升级：因外部依赖阻塞的任务升级协调
- 质量问题升级：验收不通过且修改超时的问题升级

## 执行步骤

### 步骤1：超时检测
- 实时比对任务当前状态与预定时间线
- 计算超时百分比 = (已用时间 - 计划时间) / 计划时间 × 100%
- 触发条件：
  - 一级：超时50%（已用时间达到计划时间的150%）
  - 二级：超时100%（已用时间达到计划时间的200%）
  - 三级：超时200%（已用时间达到计划时间的300%）

### 步骤2：升级通知组装
- 收集任务上下文信息：
  - 任务摘要和原始要求
  - 当前进度和已完成部分
  - 超时原因（如果已知）
  - 影响范围（下游任务/整体项目影响）
  - 建议行动（加资源/调整范围/重新排期）
- 根据升级级别确定通知对象：
  - 一级→直接负责人（催办+询问困难）
  - 二级→负责人上级（需要关注+可能需要介入）
  - 三级→管理层（需要决策+可能需要资源倾斜）

### 步骤3：升级通知发送
- 通知格式标准化（紧急程度/任务摘要/超时情况/影响/建议）
- 记录通知发送时间和接收确认
- 设置下一级升级的定时器（如果当前级别未响应）

### 步骤4：升级闭环
- 跟踪升级后的响应（是否在预期时间内有行动）
- 记录响应措施（加资源/调整时间线/变更范围）
- 更新任务状态和新的时间线
- 升级问题解决后记录处置结果供后续参考

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 任务时间线 | 日期 | 是 | 计划开始和完成时间 |
| 当前进度 | 百分比 | 是 | 任务完成百分比 |
| 升级规则配置 | 配置 | 否 | 默认50%/100%/200% |
| 通知对象层级 | 列表 | 是 | 各级别的通知目标人 |
| 任务上下文 | 文本 | 是 | 用于组装通知的信息 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 升级通知 | 结构化消息 | 含级别/摘要/影响/建议 |
| 升级记录 | 日志 | 升级历史和响应记录 |
| 超时统计 | 表格 | 各级别升级的频次和解决时长 |
| 瓶颈分析 | Markdown | 频繁超时的原因分析 |

## 最佳实践

1. **信息充分**：升级通知包含足够上下文，接收方不需要额外查询即可理解
2. **建议先行**：每条升级通知都附带建议行动，而非仅仅报告问题
3. **避免升级疲劳**：同一任务不重复升级同一级别（已发送则等待下一阈值）
4. **区分原因**：区分"执行慢"和"等待外部依赖"的不同处理方式
5. **数据驱动优化**：分析升级模式找到系统性问题（某个环节经常超时）

## 约束条件

- 升级不等于问责，目的是解决问题而非追究责任
- 升级规则可按任务优先级差异化（P0更紧、P2更宽松）
- 非工作时间的升级策略需单独配置（P0除外）
- 升级后需给予合理响应时间（一级4小时/二级2小时/三级1小时）
