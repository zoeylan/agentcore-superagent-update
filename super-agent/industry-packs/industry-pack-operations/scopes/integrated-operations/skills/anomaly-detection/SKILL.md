# 运营异常检测 (Anomaly Detection)

## 技能概述

运营异常检测技能负责对核心运营指标进行实时监控，当指标出现突变、趋势偏离或联动异常时自动触发诊断，在30分钟内输出根因分析报告。该技能是决策支持循环中的"预警雷达"，确保运营问题能被第一时间发现。

## 适用场景

- 单指标突变检测：指标偏离近期均值超过2个标准差
- 趋势偏离检测：指标长期趋势出现方向性转变
- 多指标联动异常：如UV上升但转化率下降的矛盾现象
- 目标达成风险检测：月度目标达成率<70%且剩余时间不足

## 执行步骤

### 步骤1：基线建立与更新
- 基于近30天数据计算各指标的动态基线（均值+标准差）
- 考虑周期性因素（工作日vs周末、月初vs月末、大促影响）
- 建立指标间的正常联动关系模型（如UV与GMV的比例关系）
- 每日自动更新基线参数

### 步骤2：实时异常检测
- **统计检测**：当前值 vs 动态基线，偏离>2σ标记为异常
- **趋势检测**：连续N天（N≥3）的单调递减/递增且斜率超过阈值
- **联动检测**：关联指标的变化方向或比例出现逻辑矛盾
- **目标检测**：按当前进度外推，月度目标达成概率<70%

### 步骤3：自动诊断
- 排除数据质量问题（数据源中断/采集异常/计算错误）
- 检查外部事件关联（平台政策/竞品动作/节假日/天气）
- 分析内部事件关联（策略调整/人员变动/系统变更）
- 执行相关性分析，定位最可能的根因（按概率排序）

### 步骤4：诊断报告输出
- 异常描述：哪个指标、偏离幅度、持续时间
- 影响范围：受影响的业务线/人群/渠道
- 原因分析：Top3可能原因及各自概率
- 建议验证方法：如何确认真实原因
- 应对方案：针对各种可能原因的应对预案

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 实时指标数据 | 时序数据 | 是 | 各域核心KPI实时/日频数据 |
| 历史基线数据 | 时序数据 | 是 | 近30天+去年同期数据 |
| 检测阈值配置 | 数值 | 否 | 默认2个标准差 |
| 事件日志 | 文本 | 否 | 内外部事件记录 |
| 指标联动规则 | 配置 | 否 | 指标间正常关联关系定义 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 异常预警通知 | 结构化消息 | 含指标/偏离度/紧急程度 |
| 诊断分析报告 | Markdown | 完整根因分析和建议 |
| 影响评估 | 表格 | 对业务和财务的量化影响 |
| 验证建议 | 列表 | 确认根因的验证步骤 |
| 应对预案 | 列表 | 按原因分类的应对方案 |

## 最佳实践

1. **30分钟SLA**：从检测到异常到输出诊断报告不超过30分钟
2. **避免误报**：设置最小持续时间阈值（>30分钟），过滤瞬时波动
3. **周期性排除**：大促期间/特殊事件自动调整基线，避免已知变化触发误报
4. **多层验证**：异常检测结果需通过统计+业务逻辑双重验证
5. **根因而非症状**：诊断需深挖到可行动的根因层，而非停留在表面指标

## 约束条件

- 诊断结论标注置信度（高/中/低）
- 数据源异常导致的"伪异常"需单独标注并通知数据修复
- 不自动执行应对方案，仅提供建议
- 联动异常分析需明确区分相关性和因果性
