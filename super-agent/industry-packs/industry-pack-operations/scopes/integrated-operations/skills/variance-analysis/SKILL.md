# 差异分析与归因 (Variance Analysis)

## 技能概述

差异分析技能负责监控预算执行与实际支出的偏差，当差异超过阈值（10%）时自动触发归因分析，定位根本原因并输出结构化预警报告。该技能是财务管控闭环中的核心诊断环节，确保成本失控能被及时发现和纠正。

## 适用场景

- 日常预算执行监控：每日10点更新前日数据，检测偏差
- 预算超支预警：差异>10%时自动生成归因分析报告
- 渠道效率恶化检测：单一渠道ROI连续3天下滑预警
- 预算冻结触发：超支>20%时建议冻结新增支出

## 执行步骤

### 步骤1：数据对比与偏差计算
- 获取当期实际支出数据（按业务线/科目/渠道）
- 与对应预算基准对比，计算绝对差异和百分比差异
- 标注偏差方向（超支/节余）和严重程度（黄色10%/橙色15%/红色20%）
- 检查数据完整性，缺失数据标注并预估影响

### 步骤2：归因分析
- **一级归因**：定位到业务线/科目级别（哪个部分贡献了最大偏差）
- **二级归因**：深入到驱动因子（是量变还是价变——如投放量增加还是单次点击成本上升）
- **三级归因**：区分可控因素和不可控因素
  - 可控因素：投放策略调整、活动超预算执行、人员超编等
  - 不可控因素：平台竞价环境变化、市场突发事件、政策调整等
- 计算各因子对总偏差的贡献度百分比

### 步骤3：影响评估
- 预测按当前趋势，月末/季末累计偏差将达到多少
- 评估偏差对关键业务指标（ROI/利润率/现金流）的影响
- 识别是否存在连锁反应（如投放超支是否影响其他业务线预算）

### 步骤4：建议输出
- 针对可控因素：提出具体调整方案（如缩减低效渠道投放、暂停非核心活动）
- 针对不可控因素：评估是否需要调整预算基准
- 按紧急程度排序建议行动（立即/本周/下月）
- 标注建议实施后的预期效果（节省金额/恢复达成率）

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 实际支出数据 | 结构化数据 | 是 | 当期各维度实际支出 |
| 预算基准数据 | 结构化数据 | 是 | 对应周期的预算计划 |
| 偏差阈值 | 数值 | 否 | 默认10%触发分析 |
| 历史偏差记录 | 结构化数据 | 否 | 用于趋势判断 |
| 业务事件日志 | 文本 | 否 | 活动/策略调整记录 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 差异预警报告 | Markdown | 含预警等级、偏差明细、归因分析 |
| 归因瀑布图描述 | 文本 | 各因子贡献度可视化描述 |
| 影响评估 | 表格 | 对关键指标的影响量化 |
| 建议行动清单 | 列表 | 按优先级排序的应对方案 |
| 趋势预测 | 数值 | 月末累计偏差预测值 |

## 最佳实践

1. **2小时响应SLA**：差异超过10%后，归因分析报告需在2小时内完成
2. **排除噪音**：短期波动（<3天）需与趋势性变化区分，避免频繁误报
3. **归因到人/到事**：可控因素需追溯到具体决策或事件，而非停留在数字层面
4. **历史模式匹配**：将当前偏差与历史类似场景对比，提供参考案例
5. **预警后闭环**：预警发出后需在24小时内确认管理层已知悉并决定是否行动

## 约束条件

- 归因分析基于可获取数据，明确标注数据盲区
- 归因结论标注置信度（高>85%/中60-85%/低<60%）
- 不直接执行预算调整操作，仅输出建议
- 预算冻结建议（超支>20%）需明确标注为强烈建议而非自动执行
