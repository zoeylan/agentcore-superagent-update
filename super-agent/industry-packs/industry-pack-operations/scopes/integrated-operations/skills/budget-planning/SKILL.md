# 预算编制 (Budget Planning)

## 技能概述

预算编制技能负责基于历史数据、业务目标和市场趋势，自动完成月度/季度/年度预算编制工作。该技能是财务管控循环的起点，为后续的执行监控、差异分析和资源配置提供基准框架。

## 适用场景

- 年度预算编制：基于战略目标分解为各业务线预算
- 季度预算滚动更新：结合最新执行数据修正预测
- 月度预算细化：将季度预算分解为月度可执行计划
- 活动/项目专项预算：针对特定运营活动编制独立预算

## 执行步骤

### 步骤1：数据采集与准备
- 拉取过去12个月的实际支出数据（按业务线/科目/渠道维度）
- 获取业务增长目标（GMV目标、用户增长目标、活动计划）
- 收集市场环境因子（平台政策变化、竞品动向、季节性规律）
- 确认预算编制口径和分类科目体系

### 步骤2：预算模型构建
- 基础预测：历史数据趋势外推（移动平均/指数平滑）
- 目标分解：将增长目标按渠道/时间维度拆解
- 环境调整：叠加市场因子系数（如618/双11等大促期间流量成本上浮30-50%）
- 多场景建模：输出乐观（+20%）、基准、悲观（-20%）三个版本

### 步骤3：预算分配与平衡
- 按ROI历史表现分配渠道预算权重
- 保留10-15%的弹性预算用于机动调整
- 检查各业务线预算合理性（同比增幅不超过50%需特殊说明）
- 验证总预算不超过营收预测的合理比例

### 步骤4：输出与审批
- 生成预算编制报告（含假设条件、计算逻辑、风险提示）
- 标注需要管理层确认的关键假设和决策点
- 提供预算调整建议的优先级排序

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 历史支出数据 | 结构化数据 | 是 | 至少12个月，按业务线/科目明细 |
| 业务增长目标 | 文本/数值 | 是 | 各业务线的KPI目标 |
| 预算周期 | 枚举 | 是 | 年度/季度/月度 |
| 市场环境信息 | 文本 | 否 | 平台政策、竞品动态等 |
| 上期预算执行率 | 数值 | 否 | 用于校准预测准确性 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 预算编制报告 | Markdown/表格 | 含各维度预算明细和汇总 |
| 多场景预算表 | 表格 | 乐观/基准/悲观三版 |
| 假设条件清单 | 列表 | 编制所依赖的关键假设 |
| 风险提示 | 列表 | 可能影响预算准确性的因素 |
| 审批建议 | 文本 | 需要决策层确认的事项 |

## 最佳实践

1. **滚动预测优于静态预算**：每月基于最新数据修正后续月份预测，而非死守年初预算
2. **区分固定成本和变动成本**：固定成本（人力/系统）按合同周期锁定，变动成本（投放/活动）按业务量弹性配置
3. **大促单独建模**：618/双11等大促期间的预算应独立编制，不与日常预算混合
4. **预留应急储备**：总预算的10-15%作为弹性储备，应对突发需求或机会追加
5. **同比+环比双验证**：预算结果需同时通过同比增幅合理性和环比连续性两道验证

## 约束条件

- 预算编制结果仅作为建议输出，最终审批需人工确认
- 不对具体供应商定价或合同条款做出预算承诺
- 涉及敏感财务数据时需标注数据安全等级
- 预算假设的有效期需明确标注，超过有效期需重新评估
