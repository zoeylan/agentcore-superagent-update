# 自动化方案设计 (Automation Design)

## 技能概述

自动化方案设计技能负责根据流程特点选择合适的技术方案，输出包含流程图、技术选型、实施计划和风险评估的完整自动化方案文档。该技能将"发现机会"转化为"可执行方案"。

## 适用场景

- 定时任务设计：报表自动生成、数据定时同步、定期健康检查
- 事件驱动设计：异常触发预警、状态变更通知、审批自动路由
- 数据管道设计：多源数据采集→清洗→转换→加载（ETL）
- 智能处理设计：AI辅助的内容生成、智能分类、模式识别

## 执行步骤

### 步骤1：需求细化
- 明确自动化范围（哪些环节自动化/哪些保留人工）
- 定义触发条件（定时/事件驱动/阈值触发/手动触发）
- 确认输入输出规格（数据格式/来源/目标系统）
- 明确异常处理策略（重试/跳过/回退/告警）

### 步骤2：技术方案选型
- **定时调度型**：Cron Job / 调度引擎，适合报表生成/数据同步
- **事件驱动型**：消息队列/Webhook，适合实时响应场景
- **RPA型**：UI自动化机器人，适合无API接口的系统操作
- **AI辅助型**：LLM/ML模型，适合内容生成/智能分类
- **混合型**：多种技术组合，适合复杂多步骤流程

### 步骤3：方案详细设计
- 流程图绘制（含正常路径+异常分支+人工介入节点）
- 数据流设计（各节点的输入/处理/输出规格）
- 容错机制设计：
  - 重试策略（最大重试次数/间隔时间/指数退避）
  - 超时处理（各步骤的超时阈值和超时动作）
  - 回退方案（失败时如何切回人工处理）
  - 告警机制（哪些异常需要通知谁）
- 监控点设计（关键节点的健康指标和告警规则）

### 步骤4：实施计划制定
- 开发工作量估算和人员需求
- 分阶段实施计划（开发→测试→灰度→全量）
- 测试方案：正常场景+异常场景+边界场景+压力测试
- 上线检查清单（前置条件/验收标准/回滚方案）
- 培训计划（运维人员的操作培训和异常处理指南）

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 流程评估报告 | 文档 | 是 | 流程发现阶段的评估结论 |
| 系统架构信息 | 文档 | 是 | 涉及系统的接口/能力 |
| 性能需求 | 数值 | 否 | 执行时间/并发量等要求 |
| 安全需求 | 文本 | 否 | 数据安全/访问控制要求 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 自动化方案文档 | Markdown | 完整的技术方案 |
| 流程图 | 文本描述/Mermaid | 含正常和异常路径 |
| 实施计划 | 时间线 | 分阶段的实施安排 |
| 风险评估 | 表格 | 风险/影响/缓解措施 |
| 验收标准 | 列表 | 上线前的测试和验收条件 |

## 最佳实践

1. **始终设计人工回退路径**：任何自动化都必须支持一键切回人工处理
2. **幂等性设计**：流程可安全重复执行，不会产生重复数据或副作用
3. **可观测性优先**：关键节点的日志/指标/告警比功能本身更重要
4. **渐进发布**：先在小范围验证再全量部署，降低上线风险
5. **文档即代码**：方案文档需能直接指导开发，而非泛泛而谈

## 约束条件

- 涉及资金操作的自动化必须包含审批节点
- 方案设计需考虑系统负载影响（避免大批量任务集中执行）
- 跨系统数据传输需考虑数据一致性和幂等性
- 不设计超出现有系统能力范围的方案（需评估基础设施约束）
