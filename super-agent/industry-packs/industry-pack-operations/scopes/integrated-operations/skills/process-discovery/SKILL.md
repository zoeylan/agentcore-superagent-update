# 流程发现与评估 (Process Discovery)

## 技能概述

流程发现技能负责系统化扫描各业务流程中的重复性工作，评估其自动化可行性和投入产出比，输出自动化优先级清单。该技能是流程自动化循环的起点，确保自动化投入产出最大化。

## 适用场景

- 定期巡检：每季度全面扫描各业务域流程，发现新的自动化机会
- 需求响应：接收各域提出的自动化需求，评估可行性
- 效率审计：分析运营团队时间分配，识别高耗时重复工作
- 痛点诊断：针对特定业务瓶颈的流程分析

## 执行步骤

### 步骤1：流程盘点
- 收集各业务域的标准操作流程（SOP）和日常工作清单
- 访谈或分析运营人员的时间分配（各类工作的频率和耗时）
- 识别重复性工作类型：
  - 数据录入/搬运（系统A数据搬到系统B）
  - 报表生成/格式化（定期汇总+格式处理）
  - 信息同步/通知（状态变更通知相关方）
  - 审批流转（标准化审批的路由和追踪）
  - 检查/校验（规则明确的数据质量检查）

### 步骤2：自动化评估
- **技术可行性**：数据源是否有API/导出能力？流程规则是否明确可编码？是否存在需要人工判断的节点？
- **业务价值评估**：频率×单次耗时×人力成本 = 年化节省工时
- **ROI测算**：预期节省 / 自动化实施成本（目标ROI≥2）
- **风险评估**：自动化失败的影响范围和回退方案

### 步骤3：优先级排序
- 按ROI从高到低排序
- 高频+低复杂度的流程优先（快赢项目）
- 跨多个业务域的共性流程优先（一次建设多处受益）
- 考虑依赖关系（某些流程自动化是其他流程的前提）

### 步骤4：输出评估报告
- 自动化机会清单（含描述/评估结论/优先级）
- 推荐实施路线图（分批次/分阶段）
- 预期总体效果（节省总工时/提升效率百分比）
- 暂不建议自动化的流程及原因

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 业务流程文档 | 文档/文本 | 是 | 各域的SOP和工作流描述 |
| 时间分配数据 | 结构化数据 | 否 | 运营人员的工作时间构成 |
| 系统能力清单 | 文本 | 否 | 各系统的API/导出能力 |
| 自动化需求列表 | 列表 | 否 | 各域提出的自动化需求 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 自动化机会清单 | 表格 | 流程名/评估/ROI/优先级 |
| 可行性报告 | Markdown | 各流程的详细评估 |
| 实施路线图 | 时间线 | 分批次的实施计划 |
| ROI预测 | 数值 | 各流程的预期投入产出比 |

## 最佳实践

1. **先易后难**：优先选择规则明确、数据可获取的流程，积累经验
2. **价值驱动**：不是所有能自动化的都值得自动化，ROI<2的暂缓
3. **渐进式自动化**：复杂流程可先自动化部分环节，逐步扩展
4. **用户参与**：流程发现需要一线人员配合，确认真实痛点
5. **定期更新**：业务变化可能产生新的自动化机会，需周期性扫描

## 约束条件

- 不对涉及复杂判断的流程强行自动化
- 评估需考虑变更管理成本（人员培训/流程切换/异常处理）
- 安全敏感流程（资金/数据）需额外的风控评估
- 评估结果为建议性质，实施决策需业务方确认
