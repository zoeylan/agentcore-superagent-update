# 任务全生命周期管理 (Task Lifecycle Management)

## 技能概述

任务全生命周期管理技能负责跨部门/跨域协作任务从创建到归档的完整管理，包括任务创建、分配确认、进度跟踪、依赖管理和完成验收。该技能确保每一个跨域协作任务都有清晰的责任人、时间线和交付标准。

## 适用场景

- 跨域协作任务：需要多个业务域配合完成的工作
- 项目里程碑管理：重大项目的关键节点跟踪
- 运营活动协调：大促/营销活动的跨部门准备工作
- 问题处理跟踪：舆情事件/客户投诉的跨域处理

## 执行步骤

### 步骤1：任务创建
- 接收任务需求（来源：各域发起/管理层指派/系统自动生成）
- 任务信息标准化录入：
  - 任务标题和描述（清晰、可执行）
  - 优先级：P0（紧急重要）/P1（重要不紧急）/P2（常规）
  - 截止时间和里程碑节点
  - 预期交付物和验收标准
  - 依赖关系（前置任务/关联任务）
- 自动识别任务涉及的业务域和潜在负责人

### 步骤2：任务分配
- 1小时内完成分配确认（SLA要求）
- 匹配原则：按业务域归属→按当前负载→按历史能力
- 发送分配通知：含任务详情/时间要求/验收标准
- 等待负责人确认接受（超时未确认自动升级）
- 记录分配决策依据

### 步骤3：进度跟踪
- 每日自动更新任务进度看板
- 进度更新方式：
  - 主动报告：负责人提交进度更新
  - 系统采集：关联系统状态自动同步
  - 定时巡检：未更新超过24小时的任务主动询问
- 关键路径监控：识别影响最终交付的关键任务链
- 依赖预警：上游任务延迟时提前通知下游受影响方

### 步骤4：验收与归档
- 交付物提交时触发验收流程
- 验收标准核对：逐项检查是否满足预定义标准
- 不通过：退回修改，明确修改要求和补充截止时间
- 通过：标记完成，更新项目全景视图
- 归档：完成任务存档，提取经验教训

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 任务需求描述 | 文本 | 是 | 任务内容和目标 |
| 优先级 | 枚举 | 是 | P0/P1/P2 |
| 截止时间 | 日期 | 是 | 任务完成时限 |
| 验收标准 | 列表 | 是 | 交付物的质量标准 |
| 依赖任务 | 列表 | 否 | 前置/关联任务ID |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 任务卡片 | 结构化数据 | 标准化的任务信息 |
| 进度看板 | 表格/看板 | 各任务当前状态总览 |
| 依赖关系图 | 文本描述 | 任务间的依赖和关键路径 |
| 完成通知 | 结构化消息 | 任务验收通过的通知 |

## 最佳实践

1. **1小时分配SLA**：任务创建后必须在1小时内完成分配确认
2. **SMART原则**：任务描述需具体(Specific)、可衡量(Measurable)、可实现(Achievable)、相关(Relevant)、有时限(Time-bound)
3. **进度透明**：所有相关方可实时查看任务状态，减少询问沟通成本
4. **提前预警优于事后追责**：在可能延迟时提前告警而非等到超时
5. **经验沉淀**：复杂任务完成后提取经验教训供后续参考

## 约束条件

- 不替代负责人做具体业务执行
- 任务拆分粒度需适中（不超过2周的执行周期）
- 任务状态变更需记录操作人和时间（可审计）
- P0任务的分配不受工作时间限制（7×24响应）
