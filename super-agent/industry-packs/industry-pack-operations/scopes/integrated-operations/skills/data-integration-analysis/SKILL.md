# 多源数据整合分析 (Data Integration Analysis)

## 技能概述

多源数据整合分析技能负责从客户运营、会员运营、电商运营、财务运营四大业务域采集核心指标数据，构建统一的运营数据全景视图和管理仪表盘。该技能是决策加速器的数据基础能力，确保管理层能基于完整、一致的数据做出决策。

## 适用场景

- 日常数据整合：每日自动采集各域核心KPI，更新全局仪表盘
- 管理层即时查询：响应管理层的自定义维度和指标组合分析需求
- 跨域关联分析：发现不同业务域数据之间的关联关系和联动效应
- 数据质量治理：监控数据源的完整性和一致性，发现质量问题

## 执行步骤

### 步骤1：数据源映射与采集
- 定义各业务域核心指标清单：
  - 客户域：客户总量、新增/流失、LTV、健康度分布
  - 会员域：会员活跃度、复购率、NPS、各级别分布
  - 电商域：GMV、UV、转化率、GPM、客单价
  - 财务域：营收、成本、利润率、预算执行率
- 确认数据采集频率（实时/小时/日/周）
- 执行数据抽取和标准化处理

### 步骤2：数据清洗与校验
- 缺失值检测：标注数据缺失字段并评估影响
- 一致性校验：跨系统同一指标的数值比对（如电商GMV与财务确认收入）
- 时效性检查：确认数据更新时间是否在SLA范围内
- 异常值标记：极端值、负值、逻辑矛盾等自动标记

### 步骤3：指标计算与聚合
- 基础指标：直接从数据源获取的原始指标
- 衍生指标：通过计算得出（如人效比=GMV/运营人数）
- 综合评分：多指标加权计算的运营健康度评分
- 多维度聚合：按时间/业务线/渠道等维度汇总

### 步骤4：仪表盘更新与输出
- 更新实时指标卡片（当前值/环比/同比/目标达成率）
- 生成趋势图表数据（近7天/30天/90天）
- 标注异常指标（偏离目标/偏离趋势的指标高亮）
- 输出数据质量报告（各数据源健康状态）

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 各域数据源接入 | 结构化数据 | 是 | 客户/会员/电商/财务核心指标 |
| 指标定义规范 | 文档 | 是 | 各指标的计算口径和来源 |
| 分析维度需求 | 文本 | 否 | 管理层指定的分析视角 |
| 数据更新时间戳 | 时间 | 是 | 各数据源最新更新时间 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 运营仪表盘数据 | JSON/表格 | 全局KPI卡片和趋势数据 |
| 数据质量报告 | Markdown | 各数据源健康状态和问题清单 |
| 跨域关联矩阵 | 表格 | 指标间相关性分析 |
| 即时分析报告 | Markdown | 响应管理层查询的定制分析 |

## 最佳实践

1. **单一数据真相源**：每个指标只有一个权威数据源，避免多源冲突
2. **数据血缘追踪**：所有衍生指标需记录计算公式和上游数据源
3. **标注数据时效**：所有报告和仪表盘明确标注数据截止时间
4. **降级策略**：数据源异常时标注数据可信度并提供替代方案
5. **口径一致性**：确保同一指标在不同报告中的计算口径完全一致

## 约束条件

- 不直接修改各业务域的源数据
- 数据质量异常时需及时通知数据源负责方
- 敏感业务数据（如具体客户信息）只输出聚合统计，不输出明细
- 数据整合结果的可信度需与数据源质量挂钩标注
