# 趋势预测 (Trend Forecasting)

## 技能概述

趋势预测技能负责基于历史数据和业务规律，对核心运营指标进行未来1-4周的走势预测，评估目标达成概率，为前瞻性决策提供数据支撑。该技能将"事后分析"升级为"事前预判"，让管理层能提前采取行动。

## 适用场景

- 月度目标达成预测：基于当前进度预测月末达成概率
- 成本支出预测：预测月末各科目的累计支出
- 流量和转化趋势：预测未来1-2周的流量和转化走势
- 季节性预判：大促前后的指标变化预测

## 执行步骤

### 步骤1：历史数据准备
- 获取目标指标的历史数据（至少6个月，最好12个月以上）
- 识别并标注特殊事件（大促/故障/政策变更）对应的异常数据点
- 数据预处理：去除异常值影响、缺失值插补、季节性因子提取

### 步骤2：模式识别
- 趋势成分：识别长期上升/下降/平稳趋势
- 周期成分：周内周期（工作日vs周末）、月内周期（月初/月中/月末）
- 季节成分：年度季节性（如电商年度大促节奏）
- 事件效应：量化特定事件（如大促）对指标的影响系数

### 步骤3：预测模型执行
- 基础预测：时间序列外推（加权移动平均/指数平滑）
- 周期叠加：在基础预测上叠加已识别的周期因子
- 事件调整：已知的未来事件（如已计划的大促）纳入预测
- 置信区间：输出预测值的上下界（80%置信区间）

### 步骤4：预测结果输出
- 点预测值：最可能的未来走势
- 区间预测：乐观/悲观场景
- 目标达成概率：当前趋势下达成月度目标的概率
- 关键假设：预测所依赖的前提条件
- 风险因素：可能导致预测失准的外部因素

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 历史时序数据 | 时序数据 | 是 | 至少6个月日频数据 |
| 预测时间范围 | 数值 | 是 | 预测未来N天/周 |
| 已知未来事件 | 文本 | 否 | 已计划的活动/大促等 |
| 目标值 | 数值 | 否 | 月度/季度KPI目标 |
| 外部环境信息 | 文本 | 否 | 市场变化/政策调整等 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 预测值序列 | 表格/图表描述 | 未来N天/周的预测值 |
| 置信区间 | 数值范围 | 80%置信区间的上下界 |
| 目标达成概率 | 百分比 | 按当前趋势达成目标的概率 |
| 关键假设清单 | 列表 | 预测依赖的前提条件 |
| 风险提示 | 列表 | 可能导致预测偏差的因素 |

## 最佳实践

1. **预测≠承诺**：所有预测结果需附带置信度和假设条件
2. **短期更准确**：1周内预测置信度高，2-4周递减，据此调整决策权重
3. **事件驱动修正**：当发生重大事件时立即修正预测（而非等待定期更新）
4. **回测验证**：定期对比预测值和实际值，计算预测误差率并优化模型
5. **多模型集成**：使用多个模型取平均/加权，避免单一模型偏差

## 约束条件

- 预测模型需定期回测并报告历史准确率
- 超过4周的预测不确定性大幅增加，需特别标注
- 预测不能替代业务判断，仅作为决策参考
- 当数据不足或质量差时，应明确告知预测可信度低
