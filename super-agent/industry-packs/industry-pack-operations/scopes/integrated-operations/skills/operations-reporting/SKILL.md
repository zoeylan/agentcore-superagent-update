# 运营报告生成 (Operations Reporting)

## 技能概述

运营报告生成技能负责自动化生成各类运营管理报告，重点是月度运营报告（每月3号前完成）。该技能将散落的数据和分析整合为结构化、可读性强的管理报告，服务于管理层的决策需求。

## 适用场景

- 月度运营报告：每月3号前自动生成的综合管理报告
- 周度快报：每周一输出的运营快速回顾
- 专题分析报告：针对特定问题的深度分析报告
- 管理层briefing：简洁的高管摘要型报告

## 执行步骤

### 步骤1：数据汇总（每月1-2号）
- 采集上月全量数据（覆盖客户/会员/电商/财务四大域）
- 核对数据完整性：抽样验证≥20个指标与源系统一致
- 计算同比（vs去年同月）和环比（vs上月）变化
- 汇总关键里程碑和重要事件

### 步骤2：分析生成（2号内完成）
- **绩效评估**：各KPI vs 目标达成情况（达成/未达/超额）
- **趋势分析**：关键指标的3个月/6个月趋势线和拐点识别
- **对比分析**：同比环比对比，标注显著变化的指标
- **关联分析**：指标间的联动关系和因果推断
- **问题诊断**：未达标指标的原因分析

### 步骤3：洞察提炼
- 提炼Top5关键发现（基于数据变化的重要性和可行动性排序）
- 识别风险预警（趋势性恶化的指标或即将触及红线的指标）
- 发现增长机会（表现超预期的领域和可复制的成功模式）
- 生成行动建议（每条建议附数据支撑和优先级）

### 步骤4：报告组装与输出
- Executive Summary（3-5句话概括月度运营全貌）
- KPI总览表（核心指标卡片式呈现）
- 各业务域详细分析（按域分章节）
- 关键发现与行动建议（按优先级排列）
- 下月展望与风险提示
- 附录：数据明细表和计算说明

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 全域运营数据 | 结构化数据 | 是 | 上月全量数据（四大域） |
| 目标值配置 | 表格 | 是 | 各KPI的月度/季度目标 |
| 重要事件记录 | 文本 | 否 | 上月重要业务事件 |
| 历史报告 | 文档 | 否 | 前期报告用于格式和对比 |
| 报告模板 | 文档 | 否 | 管理层偏好的报告格式 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 月度运营报告 | Markdown/PDF | 完整的管理报告文档 |
| 高管摘要 | 1页精简版 | 关键数据和结论的浓缩 |
| 数据附表 | 表格 | 详细数据明细和计算过程 |
| 行动跟踪表 | 表格 | 上月建议的执行跟踪 |

## 最佳实践

1. **3号前交付SLA**：报告生成流程严格按时间节点推进（1号采集→2号分析→3号发布）
2. **数据准确率100%**：所有关键数据需与源系统交叉验证
3. **结论先行**：报告采用金字塔原则，先结论后论据
4. **可行动导向**：每个发现都附带可执行的建议
5. **同比+环比+趋势三视角**：避免单一对比维度导致的误判
6. **上月建议闭环**：新报告中回顾上月建议的执行情况和效果

## 约束条件

- 报告中的所有数字需标注数据来源和截止时间
- 分析覆盖度需包含4大域核心指标，不遗漏
- 建议需附数据支撑，不做无依据的主观判断
- 敏感财务数据根据接收方权限决定详细程度
