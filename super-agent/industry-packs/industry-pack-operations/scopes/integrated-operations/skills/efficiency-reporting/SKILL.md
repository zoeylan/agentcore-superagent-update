# 效率提升报告 (Efficiency Reporting)

## 技能概述

效率提升报告技能负责月度量化统计所有自动化流程节省的人力工时，输出效率提升报告，为自动化投资的价值证明和持续优化提供数据支撑。该技能是流程自动化管家的价值量化工具。

## 适用场景

- 月度效率报告：每月1号输出上月自动化效率统计
- ROI追踪：自动化投入产出比的持续跟踪
- 价值证明：向管理层展示自动化投资的回报
- 优化方向识别：识别效率提升空间最大的领域

## 执行步骤

### 步骤1：工时节省计算
- 统计各自动化流程的执行次数（本月）
- 计算等效人工耗时：执行次数 × 人工单次处理时长
- 减去自动化运维投入的人力时间
- 净节省工时 = 等效人工耗时 - 自动化运维时间

### 步骤2：效率指标汇总
- 流程自动化率 = 已自动化流程数 / 总可自动化流程数
- 自动化覆盖率 = 自动处理任务量 / 总任务量
- 自动化成功率 = 成功执行次数 / 总执行次数
- 人效比提升 = 当前人均产出 / 自动化前人均产出

### 步骤3：价值量化
- 工时价值折算：节省工时 × 平均人力时薪 = 节省金额
- 错误率降低：自动化后错误率 vs 人工错误率的差异
- 响应速度提升：自动化响应时间 vs 人工响应时间
- 累计节省：自自动化上线以来的累计节省工时和金额

### 步骤4：报告生成
- 执行摘要：本月关键数字一览（节省工时/金额/自动化率）
- 各流程明细：逐个流程的执行统计和效率数据
- 趋势分析：近6个月的效率提升趋势
- 新增和下线：本月新上线/停用的流程
- 优化建议：效率还有提升空间的流程和改进方向
- 下月计划：计划新增的自动化流程和预期效果

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 流程执行统计 | 结构化数据 | 是 | 各流程的执行次数/成功率 |
| 人工基准工时 | 数值 | 是 | 各流程人工单次处理时长 |
| 人力成本标准 | 数值 | 否 | 平均时薪（用于金额折算） |
| 自动化运维工时 | 数值 | 否 | 本月运维投入的人力 |
| 历史效率数据 | 时序数据 | 否 | 用于趋势对比 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 月度效率报告 | Markdown | 完整的效率分析文档 |
| KPI仪表盘数据 | 表格 | 核心效率指标卡片 |
| 各流程明细表 | 表格 | 逐流程的执行和效率数据 |
| 趋势图表数据 | 时序数据 | 近6个月效率趋势 |
| 优化建议清单 | 列表 | 可进一步提升的方向 |

## 最佳实践

1. **保守计算**：工时节省采用保守估计，避免夸大自动化价值
2. **对照基准**：每个流程都需有明确的人工基准数据作为对比基础
3. **累计呈现**：除月度数据外，突出累计效果更有说服力
4. **关联业务价值**：不只呈现工时节省，还展示释放人力投入了什么更高价值工作
5. **持续追踪ROI**：自动化上线后持续监控实际ROI是否达到预期

## 约束条件

- 工时节省的计算方法需透明可审计
- 人工基准数据需定期校准（业务变化可能影响人工处理时长）
- 不将自动化导致的裁员作为"价值"呈现（聚焦于效率提升和人力释放）
- 报告目标是70%自动化率，需跟踪差距和达成路径
