# 协作效率报告 (Collaboration Reporting)

## 技能概述

协作效率报告技能负责输出跨域协作的效率指标分析，包括任务准时完成率、平均处理时长、瓶颈环节识别等，持续驱动协作流程的优化改进。该技能为管理层提供协作健康度的量化视图。

## 适用场景

- 周度协作效率周报：每周一输出上周协作数据
- 月度协作分析：月度深度分析协作效率趋势
- 瓶颈诊断：针对协作效率低下的环节深度分析
- 改进追踪：协作流程优化措施的效果评估

## 执行步骤

### 步骤1：数据采集
- 统计本周期内的任务数据：
  - 新建任务数/完成任务数/在途任务数/超时任务数
  - 各任务的实际处理时长 vs 计划时长
  - 各环节的等待时间和处理时间
  - 升级事件次数和响应时长
- 按业务域/优先级/任务类型分类汇总

### 步骤2：效率指标计算
- **准时完成率** = 按时完成任务数 / 总完成任务数 （目标≥80%）
- **平均处理时长** = 总处理时长 / 完成任务数
- **平均等待时间** = 任务在各环节的等待时间汇总
- **首次响应时间** = 任务创建到首次响应的时间（目标≤1小时）
- **升级率** = 触发升级的任务数 / 总任务数
- **一次通过率** = 验收一次通过的任务数 / 总验收任务数

### 步骤3：瓶颈分析
- 识别耗时最长的环节（分配/执行/审核/验收）
- 识别超时率最高的业务域或团队
- 分析超时原因分布（资源不足/依赖阻塞/需求不清/优先级冲突）
- 对比历史数据，判断是偶发还是系统性问题

### 步骤4：报告生成与建议
- 核心指标卡片（本周值/环比/趋势方向）
- 完成率和时效性的趋势图表描述
- Top3瓶颈环节和根因分析
- 改进建议（按可行性和影响度排序）
- 上期建议的执行情况和效果追踪

## 输入规格

| 输入项 | 类型 | 必需 | 说明 |
|--------|------|------|------|
| 任务执行记录 | 结构化数据 | 是 | 各任务的时间戳和状态变更 |
| 报告周期 | 枚举 | 是 | 周/月/季度 |
| 目标基准 | 数值 | 否 | 各指标的目标值 |
| 历史数据 | 时序数据 | 否 | 用于趋势对比 |

## 输出规格

| 输出项 | 格式 | 说明 |
|--------|------|------|
| 协作效率报告 | Markdown | 完整的效率分析文档 |
| KPI仪表盘 | 表格 | 核心效率指标卡片 |
| 瓶颈分析 | 图表描述 | 各环节耗时分布和问题定位 |
| 改进建议 | 列表 | 可执行的优化方案 |
| 趋势预测 | 文本 | 下周期效率走势预判 |

## 最佳实践

1. **数据说话**：所有结论基于客观数据，不做主观猜测
2. **关注趋势而非单点**：单周波动正常，需关注连续3周以上的趋势
3. **建议可操作**：每个瓶颈都附带具体、可执行的改进建议
4. **正面激励**：表现优秀的团队/域也需要在报告中被认可
5. **闭环追踪**：上期建议必须在本期报告中跟踪效果

## 约束条件

- 报告不用于个人绩效考核（只到团队/域级别）
- 数据呈现需脱敏（不暴露具体个人的响应时间）
- 效率目标需切实可行（准时完成率80%是当前目标）
- 报告需在规定时间发布（周报每周一上午/月报每月5号前）
