# 危机预警与响应 (Crisis Alert & Response)

## 技能概述

危机预警与响应技能负责在监控到负面舆情后，快速评估严重等级、分析传播态势、生成预警报告并提出应对建议方案。该技能是品牌声誉管理的核心防线，直接关系到品牌能否在舆情危机中快速反应、有效控制影响范围。核心时效要求：负面舆情检出到预警报告生成≤30分钟，重大舆情管理层通知≤5分钟。

## 适用场景

- 负面舆情达到预警阈值时的快速响应
- 传播性负面事件（转发>100）的紧急处理
- 重大危机（上热搜/媒体报道）的即时启动
- 内容引发争议评论时的处置建议
- 舆情事件的全程跟踪和阶段性评估

## 执行步骤

### 步骤1：舆情严重等级判定
1. 接收来自sentiment-monitoring的负面舆情信号
2. 快速评估指标：
   - 传播数据：当前转发量/评论量/曝光量
   - 传播速度：近1小时增速（若增速>50条/小时为快速传播）
   - 发布者影响力：KOL(粉丝>100万)/KOC(1-100万)/普通用户
   - 平台覆盖：是否多平台同时出现
   - 媒体介入：是否有主流媒体/自媒体大V转发
3. 确定严重等级：
   - **P2-普通负面**：个别投诉、低传播（转发<100）→记录并常规分配处理
   - **P1-传播性负面**：转发>100或KOL参与或多平台出现→30分钟内出预警报告
   - **P0-重大危机**：上热搜/主流媒体报道/可能引发公关危机→5分钟内通知管理层

### 步骤2：传播路径分析（P1/P0级别）
1. 追溯原始源头：首发平台/首发账号/首发时间
2. 绘制传播路径：一级扩散（直接转发）→二级扩散（二次传播）→媒体放大
3. 识别关键传播节点：哪些账号是扩散的关键推手
4. 预测扩散趋势：基于当前速度和节点影响力，预估24小时内可能的曝光量
5. 评估最坏情况：若不干预，可能的最大影响范围

### 步骤3：影响范围评估
1. 当前影响：已触达的人群规模、情感倾向分布
2. 潜在影响：可能扩散到的人群和平台
3. 业务影响评估：是否可能影响销量/会员流失/合作伙伴关系
4. 品牌形象影响：是否触及品牌核心价值观/信任基础
5. 法律风险评估：是否涉及消费者权益/虚假宣传/安全问题

### 步骤4：应对方案建议
1. 针对不同等级生成差异化应对方案：
   - **P2处置建议**：安排客服对接解决/私信沟通/记录问题反馈改善
   - **P1应对方案**（至少3套）：
     - 方案A-快速回应：官方声明+问题处理+补偿方案
     - 方案B-观察引导：监控发展+评论区引导+正面内容对冲
     - 方案C-主动沟通：联系源头沟通+定向回应+事实澄清
   - **P0危机预案**：
     - 即时动作：暂停所有品牌相关推广/通知管理层/启动危机公关团队
     - 1小时内：发布初步声明/指定发言人/建立信息同步机制
     - 24小时内：发布正式回应/公布解决方案/启动修复行动
2. 每个方案附带：预期效果/所需资源/风险评估/时间线

### 步骤5：预警报告生成与分发
1. 生成结构化预警报告：
   - 事件概述（一句话描述）
   - 严重等级和紧急程度标注
   - 传播数据和路径分析
   - 影响范围评估
   - 应对方案建议（含推荐方案标注）
   - 下一步行动项和责任人建议
2. 分发机制：
   - P2：记录入舆情事件库，分配给相关部门
   - P1：即时推送给运营负责人和相关业务部门
   - P0：5分钟内通知管理层（电话+消息双通道确保触达）
3. 同步通知社交内容策划师准备官方回应内容

### 步骤6：事件跟踪与复盘
1. 开启事件追踪模式：每30分钟更新传播数据和影响评估
2. 评估应对措施效果：舆情趋势是否缓解/稳定/恶化
3. 事件结束后归档：完整时间线/应对措施/效果评估/经验教训
4. 更新舆情事件库和应急响应模板

## 输入规格

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 负面舆情信号 | 事件触发 | 来自sentiment-monitoring的预警触发 |
| 传播数据 | 实时数据 | 转发量/评论量/曝光量/增速 |
| 发布者信息 | 结构化数据 | 账号粉丝量/影响力/历史行为 |
| 历史舆情案例 | 知识库 | 类似事件的应对经验和效果 |
| 品牌回应模板库 | 文档 | 各类场景的回应话术模板 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 舆情预警报告 | 报告文档 | 完整的事件分析和应对建议 |
| 严重等级判定 | 枚举 | P0/P1/P2及判定依据 |
| 应对方案集 | 方案列表 | 多套应对方案及推荐排序 |
| 管理层通知 | 即时消息 | P0事件的紧急通知内容 |
| 事件追踪更新 | 定期报告 | 事件发展的阶段性更新 |
| 复盘归档报告 | 文档 | 事件全流程回顾和经验总结 |

## 最佳实践

1. **宁快勿慢**：响应速度比完美方案更重要，先预警再完善
2. **不要删帖**：删帖通常会引发更大反弹，除非涉及严重不实信息
3. **真诚第一**：回应态度比内容更重要，避免官方套话和推诿
4. **分级授权**：P2可由运营团队直接处理，P1需负责人审批，P0需管理层决策
5. **预案演练**：定期进行危机模拟演练，确保流程顺畅

## 约束条件

- 负面舆情检出到预警报告生成 ≤ 30分钟
- 重大舆情管理层通知 ≤ 5分钟
- 应对方案建议在预警后1小时内输出
- 预警报告必须附带数据支撑，不可仅凭主观判断
- 对外回应内容需经过法务和管理层审批
- 响应时效达标率 ≥ 90%
