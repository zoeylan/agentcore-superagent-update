# 内容排期规划 (Content Calendar Planning)

## 技能概述

内容排期规划技能负责制定多平台社交内容的发布日历，统筹品牌营销节奏、热点预判、会员活动传播和平台流量规律，确保内容发布的节奏感和系统性。该技能是社交内容运营从"随机发布"走向"策略驱动"的关键能力。

## 适用场景

- 每周五制定下周7天的完整内容排期
- 月度内容规划（大型营销节点/品牌活动/产品上新）
- 突发热点时的排期动态调整
- 会员活动传播节奏编排
- 内容频率和平台分配的策略优化

## 执行步骤

### 步骤1：信息输入整合
1. 获取品牌营销日历（已确定的活动/促销/新品发布节点）
2. 获取会员运营日历（来自会员增长引擎的活动安排、挽留需求）
3. 获取行业热点预判（固定节日/电商大促/行业会议/竞品动态）
4. 获取各平台流量规律数据（各时段用户活跃曲线、竞争内容密度）
5. 回顾上周内容表现数据（哪类内容效果好/差，哪些时段表现优异）

### 步骤2：内容主题规划
1. 确定每日内容主题线（品牌故事/产品种草/用户互动/知识分享/热点借势）
2. 按照7:2:1原则分配内容类型：70%价值内容+20%互动内容+10%营销内容
3. 确保每周至少1条会员专属内容（强化会员身份认同）
4. 规划跨平台联动内容（同一主题的不同平台表达）
5. 预留2个"机动位"用于热点借势或紧急需求

### 步骤3：发布时间编排
1. 基于各平台用户活跃时间确定最佳发布窗口：
   - 微博：工作日12:00-13:00、18:00-20:00；周末10:00-12:00
   - 抖音：19:00-22:00（黄金时段）、12:00-13:00（午休碎片）
   - 小红书：19:00-21:00（晚间种草）、7:00-9:00（通勤浏览）
   - 知乎：20:00-23:00（深度阅读）、工作日10:00-11:00
2. 避免同一平台每日多条密集发布（小红书除外）
3. 重要内容安排在周二至周四（互动率普遍高于周一和周五）
4. 节假日内容提前1天发布预热

### 步骤4：资源匹配与排产
1. 评估每条内容的生产难度和所需资源（纯文案/需配图/需视频脚本）
2. 确认内容生产的T-1时间线：下周一发布的内容最晚周五完成生成并提交审核
3. 分配审核时间窗口（4小时内完成，超时自动升级）
4. 标记需要跨部门配合的内容（如需要产品部提供素材/需要设计制作配图）

### 步骤5：排期文档输出
1. 生成周度内容排期表（日期/平台/主题/形态/发布时间/目标KPI/负责人/状态）
2. 标注关键节点和里程碑
3. 生成审核时间倒排表
4. 设置排期变更的通知机制

## 输入规格

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 品牌营销日历 | 日历数据 | 已确定的营销活动时间线 |
| 会员活动安排 | 结构化数据 | 来自会员增长引擎的活动计划 |
| 平台流量数据 | 时序数据 | 各平台各时段的活跃度数据 |
| 上周效果数据 | 分析报告 | 上周内容表现和洞察 |
| 热点预判信息 | 列表 | 未来一周可能的热点话题 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 周度内容排期表 | 表格 | 7天×多平台的完整排期 |
| 内容生产任务清单 | 列表 | 按时间倒排的生产任务 |
| 审核时间安排 | 时间表 | 各内容的审核截止时间 |
| 资源需求清单 | 列表 | 需跨部门协调的资源 |
| 排期变更日志 | 记录 | 排期调整的原因和影响 |

## 最佳实践

1. **节奏感**：每周有主题递进感，避免内容割裂——周一品牌故事→周三产品种草→周五互动话题
2. **弹性空间**：永远保留20%的弹性空间应对突发热点和紧急需求
3. **数据复盘驱动**：每周排期迭代基于上周数据表现，不断优化时间和主题组合
4. **跨平台错峰**：避免所有平台同一时间发布，导致团队响应力分散
5. **内容储备池**：维护一个"常青内容库"，确保突发情况下有内容可发

## 约束条件

- 每周五17:00前必须完成下周排期并提交确认
- 排期一旦确认，变更需记录原因并通知所有相关方
- 每个平台每周至少发布3条内容，最多不超过每日2条（小红书可适当增加）
- 所有内容的T-1生产时间线不可压缩（确保审核质量）
- 发布准时率目标 ≥ 98%
