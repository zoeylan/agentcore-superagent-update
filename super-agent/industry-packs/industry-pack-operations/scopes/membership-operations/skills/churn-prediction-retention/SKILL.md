# 流失预警与挽留 (Churn Prediction & Retention)

## 技能概述

流失预警与挽留技能负责提前14天识别高流失风险会员，并自动触发差异化的分级挽留策略。通过多维度行为特征的实时监测和预测模型运算，在会员真正流失前进行主动干预，是会员运营域保障会员活跃度和复购率的核心防御能力。

## 适用场景

- 每日凌晨批量运行流失风险评分更新
- 会员关键行为信号触发的即时风险评估（如连续7天未登录）
- 高价值会员（铂金/钻石）的重点监控与预警
- 挽留策略执行后的效果跟踪与策略迭代
- 季节性/活动后的集中流失风险评估

## 执行步骤

### 步骤1：流失特征工程
1. 提取登录行为特征：近14/30/60天登录频率变化率、最后登录距今天数
2. 提取消费行为特征：近30天消费金额环比变化、消费频次衰减率、客单价变化
3. 提取互动行为特征：权益使用率变化、活动参与率变化、App停留时长变化
4. 提取社交行为特征：分享频率变化、社群活跃度变化、口碑传播倾向
5. 提取外部特征：竞品活动期间行为变化、行业促销季反应

### 步骤2：流失风险评分
1. 运行流失预测模型（基于历史流失样本训练的集成模型）
2. 输出流失风险评分（0-100，越低风险越高）：
   - 高风险区间：评分 < 30（预计14天内流失概率 > 70%）
   - 中风险区间：评分 30-50（预计14天内流失概率 40%-70%）
   - 低风险区间：评分 > 50（正常状态，常规运营即可）
3. 标注TOP3风险因素（如"消费金额连续3周下降"、"权益使用率为0"）
4. 与前一日评分对比，标记急剧恶化（单日下降>15分）的会员

### 步骤3：分级挽留策略触发
1. **高风险会员挽留**（评分<30，需4小时内触发）：
   - 触发1v1专属客服关怀（电话或企微消息）
   - 发放高价值定向挽留券（根据历史消费偏好匹配）
   - 提供临时升级体验（享受高一级别权益7天）
   - 创建个性化回归活动邀请
2. **中风险会员挽留**（评分30-50，当日内触发）：
   - 自动推送个性化活动邀请（基于兴趣标签匹配）
   - 启动积分加速计划（积分翻倍7天）
   - 推送会员专属内容（新品体验/限时优惠）
   - 将社交行为特征传递给社交内容策划师进行定向内容运营
3. **急剧恶化会员**（单日下降>15分）：
   - 不论当前绝对分数，立即标记为重点关注
   - 分析恶化原因（是否关联负面体验事件）
   - 24小时内完成干预

### 步骤4：挽留效果追踪
1. 挽留动作触发后，启动7天观察期
2. 追踪指标：是否回访/登录、是否产生消费、权益是否被使用、风险评分变化
3. 判定挽留结果：
   - 成功：7天内评分回升至50以上或产生消费行为
   - 部分成功：有回访但无消费，评分小幅回升
   - 失败：无任何活跃行为，评分继续下降
4. 输出挽留效果报告，反馈模型优化

### 步骤5：模型迭代与策略优化
1. 每月汇总预警准确率（实际流失率 vs 预警率）
2. 分析各挽留策略的成功率和成本效益
3. 识别高效挽留组合（如"高价值券+1v1关怀"对铂金会员效果最佳）
4. 更新挽留策略库和推荐优先级

## 输入规格

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 会员行为日志 | 时序数据 | 登录、消费、浏览、互动等行为流 |
| 会员等级信息 | 结构化数据 | 当前等级、入会时长、历史等级变动 |
| 权益使用记录 | 结构化数据 | 各权益项领取/使用/过期状态 |
| 历史挽留记录 | 结构化数据 | 过往挽留动作和结果 |
| 外部事件数据 | 事件数据 | 竞品活动、行业促销等外部信息 |

## 输出规格

| 输出项 | 类型 | 说明 |
|--------|------|------|
| 流失风险评分表 | 结构化数据 | 全量会员风险评分、风险等级、TOP3因素 |
| 高风险预警清单 | 列表 | 评分<30的会员详情和建议挽留方案 |
| 挽留任务指令 | 系统指令 | 需触发的挽留动作类型和参数 |
| 挽留效果报告 | 分析报告 | 7天追踪结果、成功率、策略效果对比 |
| 模型优化建议 | 文档 | 月度模型表现评估和迭代方向 |

## 最佳实践

1. **分层挽留预算**：钻石会员挽留预算可达其月均消费的20%，普通会员控制在5%以内
2. **避免过度打扰**：同一会员30天内最多触发2次挽留动作，避免骚扰
3. **挽留时机选择**：工作日上午10-12点触达率最高，避免深夜和午休时间
4. **情感关怀优先**：高价值会员优先使用情感关怀而非纯利益刺激
5. **流失原因闭环**：真正流失的会员尝试获取流失原因反馈，持续优化模型

## 约束条件

- 预警准确率目标 ≥ 75%（实际流失/预警人数）
- 高风险会员挽留动作必须在4小时内触发
- 挽留触达率 ≥ 95%
- 挽留成功率目标 ≥ 30%
- 同一会员30天挽留次数 ≤ 2次
- 挽留优惠券总成本不超过预期挽回LTV的15%
