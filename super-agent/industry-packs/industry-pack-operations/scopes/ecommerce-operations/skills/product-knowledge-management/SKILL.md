# 商品知识库管理 (Product Knowledge Management)

## 概述
本技能用于店铺运营管家维护结构化的商品知识库，为数字主播直播讲解、客服应答和投放素材制作提供准确、完整的产品信息支撑。知识库是直播间高质量内容输出的基础设施。

## 使用场景
- 新商品入库时创建知识卡片
- 商品信息变更后更新知识库
- 直播前确认商品知识完备性
- 客户FAQ高频问题更新
- 竞品信息更新和对比分析
- 商品表现数据回写（销量/评价/退货率）

## 执行步骤

### Step 1: 知识卡片创建/更新
为每个商品维护标准化知识卡片：

```
商品知识卡片结构：
├── 基础信息（品名/品牌/型号/SKU/价格带）
├── 规格参数（尺寸/重量/材质/成分/含量/有效期）
├── 核心卖点（Top5卖点，按重要性排序，每个含支撑证据）
├── 使用场景（2-3个典型使用场景描述）
├── 适用人群（年龄/性别/需求画像）
├── FAQ（Top10高频问题和标准答案）
├── 竞品对比（vs 主要竞品的优劣势对比表）
├── 话术建议（推荐的讲解话术关键词和禁忌词）
├── 表现数据（销量/转化率/评分/退货率/直播表现）
└── 注意事项（合规红线/易出错点/特殊限制）
```

### Step 2: 信息采集与验证
- 从供应商/品牌方获取官方产品资料
- 从电商平台获取商品详情和评价数据
- 从客服记录提取高频问题和答案
- 从直播复盘提取商品表现数据
- 所有信息需交叉验证，确保准确性

### Step 3: 知识结构化处理
- 提炼核心卖点（从海量信息中提炼Top5）
- 生成FAQ（从用户评价和咨询中总结高频问题）
- 制作竞品对比表（客观数据对比，标注优劣）
- 编写话术建议（什么该说/什么不该说）
- 标注合规红线（该品类的特殊限制）

### Step 4: 知识库质量管理
- 完备性检查：所有在售商品必须有完整知识卡片
- 准确性检查：每周抽样验证信息是否过时
- 一致性检查：与平台实际上架信息核对
- 时效性管理：促销/季节性信息及时更新/过期清理
- 版本管理：记录每次更新的变更内容和时间

### Step 5: 知识分发与应用
- 直播前：向数字主播推送当场排品的知识卡片
- 实时查询：支持弹幕问题的知识库即时检索
- 文案生成：为店铺文案撰写提供素材支撑
- 投放素材：为投放创意提供产品卖点信息
- 培训素材：新品培训和话术迭代的知识基础

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| product_id | string | 是 | 商品ID |
| source_data | object | 是 | 原始产品资料 |
| customer_feedback | array | 否 | 客户评价/咨询记录 |
| livestream_performance | object | 否 | 直播表现数据 |
| competitor_info | array | 否 | 竞品信息 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| knowledge_card | object | 完整商品知识卡片 |
| completeness_score | number | 信息完备度评分（0-100） |
| update_log | object | 本次更新变更记录 |
| quality_alerts | array | 质量问题提醒（过时/不一致/缺失） |

## 最佳实践
1. **信息分层**：区分核心信息（必须准确）和辅助信息（可参考），标注可信度
2. **用户视角**：FAQ和卖点从用户角度出发，不用行业/厂家术语
3. **动态更新**：结合销售数据和评价动态调整卖点排序
4. **竞品持续跟踪**：竞品价格/卖点/评价变化及时更新对比表
5. **直播反馈闭环**：直播中观众高频追问的问题及时补充到FAQ

## 约束条件
- 新品入库后24小时内完成知识卡片创建
- 知识卡片完备度≥90%才能进入直播排品
- 每周全量信息准确性抽查≥20%
- FAQ覆盖率≥85%（覆盖用户80%以上的常见问题）
- 知识更新延迟≤24小时（商品信息变更后）
