# 多平台上架管理 (Multi-Platform Listing)

## 概述
本技能用于将商品信息按照各电商平台的规范要求进行适配，完成多平台同步上架，并在上架后监控展示状态和合规性。确保商品在淘宝/天猫、抖音小店、快手小店、拼多多等平台正确、高效地发布。

## 使用场景
- 新商品首次多平台上架
- 商品信息变更后同步更新各平台
- 大促前价格/促销规则批量调整
- 上架后合规检查未通过的修复重新提交
- 平台规则变更后的批量适配调整

## 执行步骤

### Step 1: 平台规范确认
确认各目标平台的最新商品发布规范：

| 平台 | 标题 | 主图 | 详情 | 价格 | 特殊要求 |
|------|------|------|------|------|----------|
| 淘宝/天猫 | ≤30字 | 800×800/5张+/白底首图 | 图文详情 | 一口价+促销价 | 品牌授权/3C认证 |
| 抖音小店 | ≤20字 | 1:1/≥3张/场景图 | 图文+短视频 | 划线价+售价 | 样品审核 |
| 快手小店 | ≤30字 | 3:4竖图/≥3张 | 图文详情 | 参考价+售价 | 发货时效承诺 |
| 拼多多 | ≤60字 | 750×352轮播/≥5张 | 图文详情 | 团购价+单买价 | 坑产要求/假一赔十 |

### Step 2: 素材适配
- 标题适配：根据各平台字数和规则调整
- 主图适配：调整尺寸比例、检查文字占比（≤20%）、确认无水印/牛皮癣
- 详情页适配：调整图片尺寸、适配平台编辑器格式
- 价格适配：设置各平台价格策略（考虑平台扣点差异）
- SKU适配：确认各平台SKU设置规则

### Step 3: 上架执行
- 按平台优先级依次提交（通常先主力平台）
- 填写各平台必填字段（类目/属性/运费模板/服务承诺等）
- 设置库存同步策略（总库存分配 or 独立库存）
- 提交审核并记录提交时间

### Step 4: 审核跟踪
- 监控各平台审核状态（通常2-4小时出结果）
- 审核通过：确认前台展示正常
- 审核未通过：分析拒绝原因，修改后重新提交
  - 常见拒绝原因：类目选错/图片违规/资质缺失/标题违规
- 超时未审核：通过平台客服渠道催审

### Step 5: 上架后验证（24小时内）
- 检查各平台商品页展示是否正常
- 确认价格/库存/SKU信息无误
- 验证商品能正常加入购物车和下单
- 确认搜索可被检索（输入标题关键词）
- 检查详情页图片加载和排版

### Step 6: 异常处理
- 上架失败：排查原因（资质/类目/限制），修复后重试
- 展示异常：检查素材规格，重新上传
- 价格错误：紧急下架→修正→重新上架
- 库存不同步：检查同步策略，手动校准

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| product_info | object | 是 | 完整商品信息 |
| copywriting_package | object | 是 | 各平台适配文案包 |
| image_assets | array | 是 | 商品图片素材 |
| target_platforms | array | 是 | 目标上架平台列表 |
| pricing_strategy | object | 是 | 各平台定价策略 |
| inventory_config | object | 是 | 库存分配策略 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| listing_status | object | 各平台上架状态 |
| platform_links | object | 各平台商品链接 |
| audit_results | object | 审核结果（通过/拒绝+原因） |
| verification_report | object | 上架后验证报告 |
| issues_found | array | 发现的问题列表 |

## 最佳实践
1. **主力平台优先**：先上架GMV贡献最大的平台，确保不影响销售
2. **模板化操作**：建立各平台的标准上架模板，减少重复操作
3. **批量工具**：使用平台提供的批量上传工具提升效率
4. **库存安全**：多平台共享库存时设置安全库存（预留10%缓冲）
5. **定期巡检**：每周检查所有在售商品的展示状态和合规性

## 约束条件
- 人工确认后1小时内完成多平台上架
- 上架成功率≥98%
- 上架后24小时内完成展示验证
- 价格错误须在发现后5分钟内下架处理
- 各平台信息一致性检查（核心卖点不得冲突）
