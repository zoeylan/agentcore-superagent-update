# 人群定向管理 (Audience Targeting Management)

## 概述
本技能用于流量投放操盘手构建、管理和优化投放人群定向策略。通过精准的人群筛选和组合，确保付费流量触达高转化潜力用户，提升投放ROI。需要整合客户画像数据、平台标签和行为数据。

## 使用场景
- 新投放计划的人群定向设置
- 基于效果数据的人群优化调整
- 大促期间的人群扩展策略
- 新品推广的种子人群筛选
- 竞品粉丝/达人相似人群构建

## 执行步骤

### Step 1: 人群策略确定
根据投放目标确定人群策略方向：
- **精准策略**（高ROI目标）：窄定向+高转化人群，适合日常投放
- **扩展策略**（规模增长目标）：宽定向+相似人群扩展，适合大促
- **测试策略**（新品/新市场）：多组小预算测试，找到最优人群
- **排除策略**：排除已购/低质/竞品水军等无效人群

### Step 2: 人群包构建

**基础定向层：**
- 地域：根据物流能力和消费力选择
- 年龄：基于商品目标客群
- 性别：基于品类特性
- 消费水平：基于商品价格带匹配

**兴趣行为层：**
- 品类兴趣：近30天浏览/搜索/加购相关品类
- 行为序列：有购买行为>加购行为>浏览行为（按转化漏斗分层）
- 直播行为：看播时长>5分钟/有互动/有下单的直播活跃用户

**高级定向层：**
- DMP自定义人群：上传CRM数据构建精准种子人群
- Lookalike扩展：基于高价值客户扩展相似人群
- 达人粉丝：竞品直播间互动用户/行业KOL粉丝
- 重定向：加购未购/浏览未购/购买复购人群

### Step 3: 人群组合与排除
- 构建多层人群组合（基础∩兴趣∩行为）
- 设置排除人群：
  - 已购用户（非复购品）
  - 退货率>30%的高风险用户
  - 竞品恶意用户（频繁差评/退货）
  - 无效流量（机器人/刷单用户）
- 控制定向范围：太窄起不了量，太宽ROI低

### Step 4: A/B测试设计
- 将预算均分给2-3组不同定向的测试计划
- 测试变量单一化（每次只变一个定向维度）
- 测试周期≥48小时，确保数据显著性
- 统计显著后选择胜出方案全量投放

### Step 5: 效果追踪与优化
- 分人群包追踪核心指标（CTR/CVR/ROI/CPA）
- 每周更新人群效果排名
- 淘汰连续7天ROI<2的人群包
- 优质人群包持续扩量（Lookalike扩展比例递增）
- 定期刷新DMP人群数据（用户行为变化快）

### Step 6: 人群资产管理
- 维护人群包库（名称/描述/规模/效果/状态）
- 定期清理过期/低效人群包
- 记录人群组合的效果经验
- 跨平台人群策略复用和适配

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| campaign_objective | enum | 是 | 投放目标 |
| product_info | object | 是 | 商品信息（品类/价格/定位） |
| customer_profiles | object | 否 | 客户画像数据（CRM） |
| platform | enum | 是 | 投放平台 |
| budget_level | enum | 是 | 预算级别（影响定向宽窄） |
| historical_audience_data | object | 否 | 历史人群效果数据 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| targeting_config | object | 完整定向配置 |
| audience_packages | array | 人群包列表及描述 |
| exclusion_list | array | 排除人群配置 |
| ab_test_plan | object | A/B测试方案 |
| estimated_reach | number | 预估覆盖人数 |
| optimization_suggestions | array | 定向优化建议 |

## 最佳实践
1. **先精后宽**：新计划先用精准人群跑出模型，再逐步放宽
2. **数据驱动**：人群调整基于效果数据而非直觉
3. **定期刷新**：DMP人群和行为人群每周更新，避免时效性衰减
4. **平台特性**：抖音重兴趣标签/淘宝重搜索行为/快手重关系链
5. **预算匹配**：人群规模与预算匹配——预算小用精准，预算大可放宽

## 约束条件
- 人群包规模需在100万-5000万之间（太小起不了量，太大不精准）
- A/B测试最小样本量≥1000次曝光/组
- 人群数据使用需符合数据安全和隐私法规
- 禁止使用平台不允许的定向方式（如某些敏感人群标签）
- 每次定向调整需记录原因，支持效果追溯
