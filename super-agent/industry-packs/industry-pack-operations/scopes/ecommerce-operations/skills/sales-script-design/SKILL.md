# 话术脚本设计 (Sales Script Design)

## 概述
本技能用于直播运营策划师设计各类直播话术脚本，包括产品讲解话术、互动话术、逼单话术和应急话术。话术需兼顾转化效率和合规要求，适配不同平台调性和主播人设。

## 使用场景
- 新品上架需要设计讲解话术
- 大促活动需要设计促销话术
- 日常直播话术迭代优化
- 互动环节规则话术设计
- 应急预案话术准备

## 执行步骤

### Step 1: 话术需求分析
- 确认话术类型：产品讲解/促销逼单/互动引导/过渡衔接/应急备用
- 确认目标平台和对应合规要求
- 确认主播人设和语言风格
- 分析目标商品特性和核心卖点
- 参考历史高转化话术模式

### Step 2: 话术结构设计

**产品讲解话术结构：**
```
[Hook开场] → 痛点/场景引入（5秒）
[卖点展示] → 核心卖点1+证据（20秒）
[卖点展示] → 核心卖点2+证据（20秒）
[场景描绘] → 使用场景+效果描述（15秒）
[价格锚定] → 对比价+直播价+赠品（15秒）
[逼单收割] → 限时限量+行动指令（15秒）
```

**互动话术结构：**
```
[预告] → 告知互动内容激发期待
[规则] → 清晰说明参与方式
[引导] → 引导执行（关注/评论/点赞）
[高潮] → 倒计时/开奖/宣布
[承接] → 过渡到下一环节
```

### Step 3: 话术内容撰写
- 按结构逐段撰写具体话术
- 注入平台调性：
  - 抖音：口语化/活泼/网络用语/弹幕梗
  - 淘宝：专业/详细/数据化/对比清晰
  - 快手：亲民/真诚/家人称呼/接地气
- 嵌入SEO关键词（提升被搜索推荐概率）
- 设计多版本A/B测试话术

### Step 4: 合规审查
- 逐句检查违禁词（极限用语/虚假宣传/绝对化表达）
- 检查功效宣称是否有依据（特别是美妆/食品/保健品）
- 确认价格描述准确（不虚抬原价）
- 确认赠品/活动规则描述无歧义
- 通过平台自动审核工具预检

### Step 5: 输出与标注
- 输出完整话术脚本（含时间标注/重点标记/动作提示）
- 标注可灵活调整部分和必须严格执行部分
- 标注与视觉/操作的配合点（何时展示商品/何时上链接）
- 输出话术卡片简版（供数字主播快速参考）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| script_type | enum | 是 | 话术类型 |
| product_info | object | 是 | 商品信息（卖点/价格/竞品） |
| platform | enum | 是 | 目标平台 |
| persona_style | string | 是 | 主播人设风格 |
| promotion_rules | object | 否 | 促销活动规则 |
| historical_top_scripts | array | 否 | 历史高转化话术参考 |
| compliance_rules | object | 是 | 平台合规规则库 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| full_script | object | 完整话术脚本（含时间轴） |
| quick_reference_card | object | 话术卡片简版 |
| key_phrases | array | 核心金句列表 |
| compliance_result | object | 合规检测结果 |
| ab_variants | array | A/B测试变体 |
| estimated_duration | number | 预计讲解时长 |

## 最佳实践
1. **结果导向**：话术设计围绕转化目标，每句话都要有目的
2. **口语化**：直播话术必须读起来自然，避免书面语
3. **重复强化**：核心卖点和促销信息重复至少3次
4. **互动嵌入**：讲解中穿插小互动（"对不对"/"有没有同款的"）保持参与感
5. **持续迭代**：根据复盘数据持续优化话术，淘汰低效表达

## 约束条件
- 所有话术必须通过合规检测后才能使用
- 功效宣称必须有数据/证书/用户评价支撑
- 价格对比必须真实（不虚构原价/市场价）
- 话术时长需与排品节奏匹配（不超时/不不够用）
- 敏感品类（保健品/医疗器械/金融）需额外合规审查
