# 直播脚本执行 (Livestream Script Execution)

## 概述
本技能用于数字主播按照预设的话术脚本和场控指令，执行标准化的直播流程。包括开场、过渡、互动环节、促销环节和收场等各阶段的话术执行，确保直播内容的完整性和节奏感。

## 使用场景
- 按照直播运营策划师编排的脚本执行各环节
- 响应场控调度师的实时指令切换执行内容
- 互动环节（抽奖/秒杀/红包雨）的标准化执行
- 技术故障时执行备用话术循环
- 开场暖场和收场引导关注

## 执行步骤

### Step 1: 脚本加载与环境确认
- 加载当场直播的完整话术脚本
- 确认当前排品顺序和时间节奏规划
- 确认互动玩法规则和触发条件
- 加载当前平台的话术合规规则

### Step 2: 开场执行（前5分钟）
- 欢迎话术："欢迎来到XX直播间..."
- 今日福利预告：告知观众今天的爆款和福利
- 引导关注/加粉丝团：给出关注的即时福利
- 建立今日直播的期待感

### Step 3: 主体脚本循环执行
- 按排品顺序依次执行商品讲解脚本
- 每2-3个品之间插入互动/福利环节
- 根据场控指令实时调整执行顺序
- 过渡衔接话术保持自然流畅

### Step 4: 互动环节执行
- 抽奖环节：规则说明→参与引导→倒计时→开奖→恭喜
- 秒杀环节：预告→倒计时→上架→催拍→售罄播报
- 问答环节：提问→收集弹幕回答→公布正确答案→发奖
- 红包雨：预告→倒计时→发放→引导停留

### Step 5: 收场执行（最后5分钟）
- 今日精选回顾：快速回顾爆款商品
- 引导关注：提醒关注下场直播时间
- 感谢话术：感谢下单/互动的观众
- 预告下场：透露下场直播的福利信息

### Step 6: 应急脚本执行
- 技术故障：切入备用话术循环（聊天互动/往期好物回顾）
- 违规预警：立即停止当前内容，切换安全话题
- 流量暴增：快速切入高转化话术把握流量

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| script_document | object | 是 | 完整直播话术脚本 |
| current_phase | enum | 是 | 当前阶段（opening/main/interaction/closing/emergency） |
| director_command | object | 否 | 场控实时指令 |
| product_queue | array | 是 | 排品队列 |
| interaction_rules | object | 是 | 互动活动规则配置 |
| platform | enum | 是 | 当前直播平台 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| execution_content | string | 当前执行的话术内容 |
| phase_status | enum | 当前阶段状态 |
| next_action | string | 下一步预计执行内容 |
| command_acknowledged | boolean | 是否已确认场控指令 |
| compliance_status | string | 合规状态 |

## 最佳实践
1. **节奏感**：讲解与互动交替进行，避免观众疲劳
2. **自然过渡**：品与品之间不要硬切，用场景/话题自然衔接
3. **指令响应**：场控指令10秒内确认并开始执行切换
4. **备用方案**：始终准备30秒以上的衔接话术应对突发情况
5. **平台适配**：同一脚本在不同平台需调整语言风格和互动形式

## 约束条件
- 场控指令必须在10秒内响应执行
- 脚本执行不得偏离预设内容框架（可微调表达）
- 互动环节规则说明必须准确完整，避免争议
- 应急状态下不得自行恢复，需等待场控确认
- 所有话术实时通过合规检测
