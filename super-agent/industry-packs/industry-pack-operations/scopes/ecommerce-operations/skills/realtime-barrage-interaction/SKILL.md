# 实时弹幕互动 (Realtime Barrage Interaction)

## 概述
本技能用于在直播间实时解析观众弹幕内容，并在3秒内生成自然、得体的互动回复。核心目标是提升直播间互动率和观众停留时长，营造活跃的直播间氛围。

## 使用场景
- 观众提问商品相关问题（价格/规格/使用方法等）
- 观众发送互动内容（打招呼/表情/点赞）
- 观众表达购买意向或犹豫
- 负面评论/质疑的及时化解
- 互动活动环节（口令/问答/抽奖）

## 执行步骤

### Step 1: 弹幕分类识别（<1秒）
将弹幕实时分类为以下类型：
- **商品咨询**：关于价格、规格、使用、适用人群的问题
- **购买意向**：表达想买、已下单、求链接
- **互动社交**：打招呼、夸赞主播、日常聊天
- **质疑投诉**：对产品/价格/服务的质疑
- **口令参与**：参与抽奖/红包/互动活动
- **无效内容**：广告/刷屏/无意义内容（忽略）

### Step 2: 优先级排序
- P0（立即响应）：购买相关问题、质疑投诉
- P1（优先响应）：商品咨询、互动活动参与
- P2（择机响应）：社交互动、打招呼
- 同优先级内按时间顺序+重复频次排序

### Step 3: 回复生成（<2秒）
- 商品咨询：从知识库匹配准确答案，简洁回复
- 购买意向：鼓励+引导下单操作说明
- 社交互动：自然回应，适时融入产品信息
- 质疑化解：先共情再解释，引导正面认知
- 口令互动：确认参与，营造氛围

### Step 4: 回复输出与融入
- 回复控制在15字以内，简洁有力
- 自然融入当前讲解节奏，不打断主线
- 适时点名互动用户增加亲密感
- 高频同类问题合并回复，避免重复

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| barrage_content | string | 是 | 弹幕文本内容 |
| user_nickname | string | 是 | 发送用户昵称 |
| user_level | string | 否 | 用户等级/粉丝团标识 |
| current_product | object | 是 | 当前讲解商品信息 |
| context_state | string | 是 | 当前直播状态（讲解/互动/过渡） |
| product_faq | array | 是 | 当前商品FAQ列表 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| reply_text | string | 回复话术（≤15字） |
| reply_type | enum | 回复类型（answer/encourage/social/resolve） |
| priority | enum | 优先级（P0/P1/P2） |
| should_mention_user | boolean | 是否点名用户 |
| timestamp | number | 生成时间戳 |

## 最佳实践
1. **响应速度优先**：宁可回复简短也不能超时，3秒是硬性上限
2. **点名策略**：优先点名粉丝团成员和高频互动用户，增强归属感
3. **引导转化**：社交互动回复中巧妙植入产品信息
4. **情绪管理**：负面弹幕不对抗，用幽默或真诚化解
5. **合并回复**：同类问题每3-5条合并回复一次，节省时间

## 约束条件
- 响应延迟硬上限3秒，超时则跳过该弹幕
- 禁止回复涉及竞品对比的负面评价
- 禁止回复引导私下交易的内容
- 广告/引流弹幕识别后忽略，不做回复
- 弹幕响应率目标≥95%（P0+P1类弹幕）
