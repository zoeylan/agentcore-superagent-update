# 实时调度指挥 (Realtime Scheduling Dispatch)

## 概述
本技能用于场控调度师基于实时数据监控结果，做出调度决策并向数字主播下达精准的控制指令。核心目标是通过动态调整直播节奏、商品顺序和互动策略，最大化直播间的人气和转化效率。

## 使用场景
- 在线人数持续下降需要拉回停留
- 单品转化率低于预期需要切品
- 流量高峰到来需要安排爆款
- 互动冷场需要触发活跃事件
- 投放效果变化需要配合调整节奏

## 执行步骤

### Step 1: 决策评估
- 接收数据监控模块的异常预警或定期快照
- 评估当前状态严重程度（P0/P1/P2）
- 判断干预时机（立即/观察5分钟/下一轮次）
- 确认可用的调度手段和资源

### Step 2: 策略选择
根据问题类型选择对应策略：

**人气下降策略池：**
- 触发福利品（9.9/19.9低价引流款）
- 发起整点抽奖（关注+评论参与）
- 红包雨（停留满X分钟可领）
- 限时秒杀（制造紧迫感和话题）
- 弹幕口令互动（活跃气氛）

**转化偏低策略池：**
- 切换至高性价比款（提升转化率数据）
- 加大促销力度（追加赠品/降价）
- 调整话术强度（增加逼单频次）
- 叠加限时限量机制

**GPM偏低策略池：**
- 流量质量差→通知投放操盘手调整人群定向
- 货品问题→切换高客单利润款提升UV价值
- 综合问题→组合调整（换品+优化投放）

### Step 3: 指令生成与下达
- 格式：[指令类型] + [具体内容] + [数据依据] + [预期效果]
- 明确执行时间要求（立即/XX秒后/等当前品讲完）
- 标注优先级和是否可延迟

### Step 4: 效果跟踪
- 指令下达后持续监控5-10分钟
- 评估干预效果是否达预期
- 未达预期则启动升级策略
- 记录干预效果数据用于策略优化

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| anomaly_alert | object | 是 | 触发调度的数据异常信息 |
| current_state | object | 是 | 直播间当前全量状态 |
| available_products | array | 是 | 可切换的商品列表 |
| interaction_tools | array | 是 | 可用的互动工具 |
| remaining_budget | object | 否 | 互动预算剩余情况 |
| historical_effectiveness | object | 否 | 历史策略效果数据 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| command | object | 调度指令（类型+内容+时机） |
| data_basis | string | 数据依据说明 |
| expected_outcome | string | 预期效果描述 |
| priority | enum | 指令优先级 |
| follow_up_plan | string | 若无效的后续方案 |

## 最佳实践
1. **数据驱动**：每个决策必须有明确的数据依据，拒绝拍脑袋
2. **节奏感**：调度间隔≥5分钟，避免频繁切换造成观众混乱
3. **递进策略**：先用轻量手段（话术调整）再用重手段（切品/抽奖）
4. **成本意识**：福利品/抽奖有成本，需在GMV目标内合理使用
5. **效果验证**：每次干预后必须跟踪效果，积累决策经验库

## 约束条件
- 最短调度间隔5分钟（紧急情况除外）
- 每场直播福利品使用不超过总排品数的20%
- 指令下达后10秒内数字主播必须开始响应
- 涉及追加预算的决策需上报确认
- 所有调度决策需记录日志用于复盘
