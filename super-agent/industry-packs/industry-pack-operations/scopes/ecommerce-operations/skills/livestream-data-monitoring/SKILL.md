# 直播数据监控 (Livestream Data Monitoring)

## 概述
本技能用于场控调度师实时采集、分析和监控直播间全量核心数据指标，识别数据异常和趋势变化，为调度决策提供数据基础。监控粒度为分钟级，异常识别响应在30秒内。

## 使用场景
- 直播全程实时数据跟踪和可视化
- 核心指标异常检测和预警触发
- 每15分钟数据快照输出
- 与历史同期数据对比分析
- 流量波动原因初步诊断

## 执行步骤

### Step 1: 数据源接入与采集
- 接入直播平台实时数据流（API/数据推送）
- 采集核心指标：在线人数、累计UV、停留时长、互动率（点赞+评论+分享）、商品点击率、转化率、GMV实时累计、GPM
- 采集辅助指标：新关注数、粉丝团加入数、礼物收入、分享次数
- 数据刷新频率：核心指标每10秒/辅助指标每60秒

### Step 2: 趋势计算与基线对比
- 计算各指标的5分钟滑动平均值
- 与历史同期（同时段/同品类/同活动类型）基线对比
- 计算偏离度 = (当前值 - 基线值) / 基线值 × 100%
- 标注趋势方向：上升↑/平稳→/下降↓

### Step 3: 异常检测与预警
- **人气异常**：在线人数5分钟内下降>30%，或连续5分钟趋势性下滑
- **转化异常**：单品讲解转化率<预期50%（基于历史该品表现）
- **GPM异常**：GPM低于平台同时段均值的70%
- **互动异常**：互动率连续10分钟低于2%
- **流量异常**：UV突增（可能是投放起量）或骤降（可能是限流）

### Step 4: 数据快照输出（每15分钟）
- 输出结构化数据报表，包含所有核心指标当前值、趋势和基线对比
- 标注本时段的关键事件（切品/互动/投放变化）
- 给出本时段综合评分（S/A/B/C/D五级）

### Step 5: 诊断建议
- 对于检测到的异常，初步诊断可能原因：
  - 人气下降：品吸引力不足/时段流量自然衰减/投放暂停
  - 转化偏低：价格竞争力不足/话术力度不够/人群不匹配
  - GPM偏低：UV价值低（流量质量差）或转化低（货品问题）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| platform_data_stream | stream | 是 | 平台实时数据流 |
| historical_baseline | object | 是 | 历史同期基线数据 |
| current_product_info | object | 是 | 当前讲解商品信息 |
| traffic_source | object | 否 | 流量来源构成 |
| livestream_plan | object | 是 | 本场直播计划（含目标） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| realtime_metrics | object | 全量实时指标 |
| trend_analysis | object | 趋势分析（方向+幅度） |
| anomaly_alerts | array | 异常预警列表（含等级+原因） |
| snapshot_report | object | 15分钟数据快照 |
| diagnosis | object | 异常初步诊断和建议 |

## 最佳实践
1. **分层监控**：核心指标（在线/转化/GPM）实时紧盯，辅助指标定期检查
2. **消除噪声**：使用滑动平均而非瞬时值判断趋势，避免毛刺误报
3. **关联分析**：指标联动分析（如在线数升+转化降=流量质量下降）
4. **上下文感知**：考虑当前时段、品类、活动类型对基线的影响
5. **预警分级**：避免过度预警造成"狼来了"效应

## 约束条件
- 数据延迟不超过10秒
- 预警误报率控制在10%以内
- 快照输出间隔严格15分钟
- 异常检测从发生到预警≤30秒
- 不直接做调度决策，仅提供数据支撑
