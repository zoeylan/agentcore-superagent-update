# ROI实时优化 (ROI Realtime Optimization)

## 概述
本技能用于流量投放操盘手对运行中的投放计划进行实时监控和自动优化操作。通过持续监控ROI、CPA、CVR等核心指标，执行加码/调整/暂停等优化动作，确保投放效率始终在最优状态。

## 使用场景
- 投放计划运行中的持续ROI监控
- 低效计划的自动识别和暂停
- 高效计划的追加预算操作
- 出价/定向/创意的实时调整
- 日预算消耗节奏的动态控制

## 执行步骤

### Step 1: 数据监控（每小时）
实时采集各投放计划的核心指标：
- 消耗金额/消耗速度（元/小时）
- 曝光量/CPM（千次曝光成本）
- 点击量/CPC（单次点击成本）/CTR（点击率）
- 转化量/CPA（单次转化成本）/CVR（转化率）
- GMV产出/ROI（投入产出比）
- 人群画像匹配度

### Step 2: 自动规则执行
根据预设规则自动执行优化：

**加码规则（ROI≥3）：**
- 计划ROI≥3且消耗稳定 → 提升预算20%
- 计划ROI≥5 → 提升预算50%并复制创建相似计划
- 加码后持续监控，确保ROI不因放量而大幅下降

**调整规则（1≤ROI<3）：**
- ROI在2-3之间 → 优化定向（收窄到高转化人群）
- ROI在1-2之间 → 更换创意素材+调整出价-10%
- 调整后观察2小时，仍无改善则降级为暂停候选

**暂停规则（ROI<1）：**
- 单计划ROI<1持续30分钟 → 自动暂停
- 整体ROI<1.5持续1小时 → 全量暂停+预警通知
- 暂停后记录原因，用于后续策略优化

### Step 3: 素材效果追踪
- 追踪各创意素材的CTR和CVR
- CTR<行业均值50% → 标记为低效素材，停止使用
- CTR>行业均值200% → 标记为优质素材，优先使用
- 定期淘汰老素材，引入新素材保持新鲜度

### Step 4: 人群效果分析
- 分人群包统计ROI表现
- 高ROI人群包：增加预算分配
- 低ROI人群包：缩减预算或排除
- 发现新高效人群：创建相似人群扩展

### Step 5: 异常处理
- 消耗暴增（速度>平均2倍）→ 检查是否有异常点击/竞争加剧
- 消耗停滞（连续1小时无消耗）→ 检查余额/审核状态/出价竞争力
- ROI骤降（较前1小时下降>50%）→ 检查转化链路/页面/库存
- 数据异常（指标不合理波动）→ 标记并人工复核

### Step 6: 优化日志记录
- 记录每次优化操作：时间+计划+动作+原因+效果
- 日终汇总输出优化效果报告
- 积累优化经验数据用于策略迭代

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| campaign_metrics | object | 是 | 各计划实时指标数据 |
| optimization_rules | object | 是 | 预设优化规则配置 |
| budget_remaining | object | 是 | 剩余预算情况 |
| creative_performance | array | 是 | 素材效果数据 |
| audience_performance | array | 是 | 人群包效果数据 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| optimization_actions | array | 本轮执行的优化动作 |
| campaign_status_update | object | 各计划状态更新 |
| alerts | array | 异常预警信息 |
| optimization_log | array | 优化操作日志 |
| hourly_summary | object | 小时效果汇总 |

## 最佳实践
1. **数据冷静期**：新计划前2小时数据不做判断（冷启动期数据不稳定）
2. **小步快跑**：每次调整幅度≤20%，避免大幅波动
3. **多维判断**：不只看ROI，综合考虑消耗速度、转化量、人群质量
4. **对照实验**：调整时保留原版本对比，确认优化有效
5. **全局视角**：单计划优化需考虑对整体预算分配的影响

## 约束条件
- ROI<1超过30分钟必须暂停，无例外
- 单次加码幅度≤50%（避免预算失控）
- 优化频率≤每小时1次/计划（避免过度优化）
- 每日暂停计划数不超过总计划50%（否则需检查整体策略）
- 所有操作保留完整日志，支持复盘审计
