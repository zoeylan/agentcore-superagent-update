# 电商运营标准操作流程 (SOP)

## 一、概述

本SOP定义电商运营域的标准操作流程，覆盖直播执行、商品上架、投放管理和复盘迭代四大核心流程。所有流程以数据驱动为核心原则，以GMV最大化和ROI≥3为运营红线，确保AI电商运营体系7×24小时高效、合规运转。

## 三、SOP-1 直播执行流程

### 3.1 触发条件
- 定时触发：按照直播排期表，开播前T-2小时自动启动
- 手动触发：临时加场直播时由直播运营策划师手动启动

### 3.2 流程步骤

#### Step 1: 开播准备（T-2小时 → T-0）

| 时间节点 | 执行动作 | 负责角色 | 输出物 |
|----------|----------|----------|--------|
| T-2h | 确认直播计划和排品终版 | 直播运营策划师 | 终版直播计划书 |
| T-2h | 设备状态检查（主播实例/画面/声音） | 场控调度师 | 设备检查清单✓ |
| T-2h | 商品知识库完备性确认 | 店铺运营管家 | 知识卡片就绪确认 |
| T-2h | 预热投放计划启动 | 流量投放操盘手 | 投放计划上线确认 |
| T-1h | 话术脚本加载到数字主播 | 直播运营策划师 | 脚本加载确认 |
| T-30min | 排品顺序和互动规则确认 | 场控调度师 | 排品/互动配置确认 |
| T-10min | 最终全量检查（Checklist） | 场控调度师 | 开播许可 |

#### Step 2: 直播执行（T-0 → 收播）

| 执行频率 | 执行动作 | 负责角色 | 输出物 |
|----------|----------|----------|--------|
| 持续 | 产品讲解与弹幕互动 | 数字主播 | 直播内容流 |
| 每10秒 | 核心数据实时采集 | 场控调度师 | 实时数据流 |
| 每15分钟 | 数据快照输出与趋势判断 | 场控调度师 | 数据快照报告 |
| 每30分钟 | 发起一次大互动事件 | 场控调度师→数字主播 | 互动执行记录 |
| 按需 | 排品切换/节奏调整指令 | 场控调度师→数字主播 | 调度指令日志 |
| 每小时 | 投放效果检查与优化 | 流量投放操盘手 | ROI检查报告 |
| 按需 | 流量异常时加投/减投 | 流量投放操盘手 | 投放调整记录 |

#### Step 3: 异常处理

```
[异常检测] → [严重等级判定]
    ├── P0（违规/故障）→ 10秒内静音/切画面 → 通知人工
    ├── P1（限流/攻击）→ 5分钟内调整策略 → 通知相关方
    └── P2（轻微问题）→ 15分钟内处理 → 记录日志
```

#### Step 4: 收播与数据保存

| 时间节点 | 执行动作 | 负责角色 |
|----------|----------|----------|
| 收播时 | 执行收场话术（回顾/感谢/预告） | 数字主播 |
| 收播后立即 | 保存完整直播数据 | 场控调度师 |
| 收播后立即 | 关闭直播引流投放计划 | 流量投放操盘手 |
| 收播后+60min | 完成复盘报告 | 直播运营策划师 |

### 3.3 质量检查点（KPI）
- ✅ 弹幕响应率 ≥ 95%
- ✅ 弹幕响应延迟 < 3秒
- ✅ 排品完成率 = 100%（计划中的品全部讲解完毕）
- ✅ 数据快照准时输出率 = 100%
- ✅ 互动事件执行率 ≥ 90%（每30分钟一次）
- ✅ 零违规事件
- ✅ 复盘报告60分钟内完成

### 3.4 异常处理预案
| 异常场景 | 处理方案 | 响应时限 | 升级条件 |
|----------|----------|----------|----------|
| 数字主播断流 | 切备用实例/播放预录内容 | 10秒 | 30秒未恢复→通知人工 |
| 平台违规预警 | 静音+切安全画面+人工复核 | 5秒 | 确认违规→暂停直播 |
| 在线人数暴跌 | 福利品+抽奖+通知加投 | 5分钟 | 15分钟无效→人工决策 |
| 商品链接异常 | 跳过该品+通知店铺管家修复 | 1分钟 | 多品异常→暂停检查 |
| 投放消耗异常 | 暂停异常计划+排查原因 | 30分钟 | 损失>预算10%→升级 |

## 四、SOP-2 商品上架流程

### 4.1 触发条件
- 新品入库：供应商/品牌方提供新商品信息
- 优化触发：商品转化率低于品类均值50%
- 定期触发：每周商品巡检发现需要优化的商品

### 4.2 流程步骤

#### Step 1: 商品信息采集（T+0 → T+2h）
- 采集商品基础信息（品名/规格/成分/功效/价格）
- 采集竞品信息（TOP5同类商品表现）
- 采集平台搜索热词数据
- 确认商品资质证书（3C/食品许可/品牌授权等）
- **输出**：商品信息采集表（完备度≥90%）

#### Step 2: AI文案生成（T+2h → T+4h）
- 基于采集信息生成多平台文案包
- 标题生成（3-5个版本/平台）
- 五点卖点描述撰写
- 详情页文案框架生成
- SEO关键词策略确定
- **输出**：文案初稿包

#### Step 3: 合规审查（T+4h → T+5h）
- 广告法合规检测（自动）
- 平台违禁词检测（自动）
- 功效宣称证据核实（人工确认）
- **输出**：合规检查报告（通过/修改建议）

#### Step 4: 人工确认（T+5h → T+6h）
- 运营人员审核文案质量和准确性
- 确认/修改文案内容
- 审批上架（通过/退回/修改）
- **输出**：审批通过的终版文案

#### Step 5: 多平台上架（T+6h → T+7h）
- 按平台规范适配素材
- 依次提交各平台上架
- 记录提交时间和状态
- **输出**：各平台上架状态表

#### Step 6: 上架验证（T+7h → T+31h）
- 24小时内完成各平台展示验证
- 确认搜索可检索
- 确认价格/库存/SKU无误
- 确认详情页展示正常
- **输出**：上架验证报告

#### Step 7: 知识库同步
- 创建/更新商品知识卡片
- 同步到数字主播知识库
- 准备直播话术建议
- **输出**：商品知识卡片

### 4.3 质量检查点（KPI）
- ✅ 文案SEO关键词命中率 ≥ 90%
- ✅ 主图合规率 = 100%（无违规元素）
- ✅ 上架成功率 ≥ 98%
- ✅ 文案生成时效 ≤ 2小时（信息采集后）
- ✅ 全流程时效 ≤ 31小时（从信息到验证完成）
- ✅ 文案合规通过率 ≥ 95%（首次提交）

### 4.4 决策树

```
商品上架后监控：
├── 上架7天销量=0？
│   ├── 是 → 标记滞销 → 分析原因
│   │   ├── 流量不足 → 建议直播重点推荐 / 付费推广
│   │   ├── 转化率低 → 优化文案/主图/价格
│   │   └── 品类问题 → 评估是否下架
│   └── 否 → 继续正常监控
├── 差评率>5%？
│   ├── 是 → 暂停推广 → 分析差评原因
│   │   ├── 品质问题 → 通知供应链 → 考虑下架
│   │   ├── 描述不符 → 修改文案/主图
│   │   └── 物流问题 → 优化发货方案
│   └── 否 → 继续正常推广
└── 转化率<品类均值50%？
    ├── 是 → 触发文案优化流程
    └── 否 → 维持现状
```

## 五、SOP-3 投放管理流程

### 5.1 触发条件
- 每日触发：每日9:00启动当日投放计划调整
- 实时触发：ROI异常时自动触发优化流程
- 事件触发：新直播场次开播前T-2小时启动预热投放

### 5.2 流程步骤

#### Step 1: 每日投放计划调整（9:00前）
- 复盘前日投放数据（分平台/分计划/分人群）
- 识别高效计划（ROI≥3）和低效计划（ROI<2）
- 调整今日预算分配（增投高效/减投低效/暂停失效）
- 更新出价策略和人群定向
- 确认今日创意素材库（补充/更换/淘汰）
- **输出**：当日投放计划调整方案

#### Step 2: 实时监控与优化（全天）
- 每小时自动检查各计划ROI
- 执行自动优化规则：

| ROI区间 | 动作 | 响应时间 |
|---------|------|----------|
| ROI ≥ 5 | 加码50%预算+复制新计划 | 确认后立即 |
| 3 ≤ ROI < 5 | 加码20%预算 | 确认后立即 |
| 2 ≤ ROI < 3 | 优化定向/出价 | 2小时内 |
| 1 ≤ ROI < 2 | 更换素材+调整出价-10% | 1小时内 |
| ROI < 1（30分钟） | 自动暂停 | 立即 |

#### Step 3: 直播协同投放
- 开播前2小时：启动预热计划
- 开播后：切换为转化目标计划
- 接收场控加投需求：5分钟内响应
- 接收场控减投需求：立即执行
- 收播后：关闭直播引流计划

#### Step 4: 日投放复盘（22:00前）
- 汇总当日全量投放数据
- 分平台/分计划/分人群效果分析
- 输出TOP5高效计划经验和BOTTOM5低效计划教训
- 次日优化方向和具体行动计划
- **输出**：日投放复盘报告

### 5.3 质量检查点（KPI）
- ✅ ROI异常计划暂停响应时间 ≤ 30分钟
- ✅ 日预算消耗偏差 ≤ 10%
- ✅ 整体投放ROI ≥ 3
- ✅ 场控加投需求响应时间 ≤ 5分钟
- ✅ 日复盘报告22:00前完成
- ✅ 每日计划调整9:00前完成

### 5.4 决策树

```
投放异常决策：
├── 单计划ROI<1持续30分钟？
│   └── 是 → 自动暂停 → 记录原因 → 分析是素材/定向/出价问题
├── 整体ROI<目标值？
│   ├── 检查各计划ROI分布
│   │   ├── 少数计划拉低 → 暂停低效计划
│   │   ├── 整体偏低 → 重新评估定向策略
│   │   └── 素材疲劳（CTR下降）→ 更换创意素材
│   └── 通知直播运营策划师评估商品问题
├── 日预算即将耗尽但GMV目标未达？
│   ├── 评估当前ROI
│   │   ├── ROI≥3 → 建议追加预算（预估增量GMV）
│   │   ├── ROI 2-3 → 谨慎追加，仅加码高效计划
│   │   └── ROI<2 → 不追加，优化效率优先
│   └── 输出追加预算申请（含边际收益评估）
└── 消耗过快（前半天消耗>60%）？
    └── 是 → 降低出价5-10% → 控制消耗节奏 → 确保全天覆盖
```

## 六、SOP-4 复盘迭代流程

### 6.1 触发条件
- 每场直播结束：T+60分钟内启动场次复盘
- 每周一：启动周度趋势复盘
- 异常事件后：24小时内启动专项复盘

### 6.2 流程步骤

#### Step 1: 场次复盘（收播后60分钟内）
- 拉取平台后台全量数据并核对
- 生成标准复盘报告（执行摘要→数据总览→分品分析→话术效果→流量分析→优化建议）
- 对比计划目标达成情况
- 提炼本场关键发现（3-5条）
- 输出下场直播具体优化建议（5-10条）
- **输出**：场次复盘报告

#### Step 2: 商品表现反馈
- 将各品直播表现数据同步给店铺运营管家
- 标注爆款商品（转化率TOP3）→ 加大备货/优化排品位置
- 标注滞销商品（转化率BOTTOM3）→ 优化话术/调整价格/下场少排
- **输出**：商品表现评估表

#### Step 3: 投放效果反馈
- 将各渠道流量质量数据同步给流量投放操盘手
- 高质量流量来源：建议加大投放
- 低质量流量来源：建议调整定向或暂停
- **输出**：流量质量评估报告

#### Step 4: 爆款规律提炼
- 从近30天数据中提取爆款共性特征
  - 品类/价格带/讲解时长/话术风格/排品位置
- 更新爆款规律库
- 输出选品建议（基于规律推荐下一批重点推品）
- **输出**：爆款规律分析报告

#### Step 5: 周度趋势报告（每周一）
- 汇总本周7场直播数据
- 周度GMV/UV/转化率趋势
- 周度投放ROI趋势
- 周度互动率/停留时长趋势
- 与上周同比和月度目标对比
- 本周关键改进点和下周重点方向
- **输出**：周度运营趋势报告

### 6.3 质量检查点（KPI）
- ✅ 复盘数据准确率 = 100%（与平台后台核对）
- ✅ 场次复盘报告60分钟内完成
- ✅ 周度趋势报告每周一12:00前输出
- ✅ 优化建议可执行率 ≥ 80%
- ✅ 爆款规律库每周更新
- ✅ 优化建议下场执行率 ≥ 90%

### 6.4 知识沉淀闭环

```
直播执行 → 数据采集 → 复盘分析 → 规律提炼 → 策略优化 → 下场执行
    ↑                                                           |
    └───────────────── 持续迭代循环 ──────────────────────────────┘

沉淀内容：
├── 爆款规律库（品类×价格×话术×时段 的最优组合）
├── 话术效果库（高转化话术模板 + 低效话术黑名单）
├── 投放经验库（人群×素材×出价 的最优配置）
├── 异常处理库（问题×方案×效果 的经验积累）
└── 平台算法洞察（各平台流量分配规律）
```

## 七、跨流程协作协议

### 7.1 信息流转规则
| 源角色 | 目标角色 | 信息内容 | 频率 | SLA |
|--------|----------|----------|------|-----|
| 直播运营策划师 | 场控调度师 | 直播计划/排品策略 | 每场T-1天 | 开播前24h交付 |
| 直播运营策划师 | 店铺运营管家 | 商品表现反馈 | 每场复盘后 | 复盘后2h内 |
| 直播运营策划师 | 流量投放操盘手 | 投放效果反馈 | 每场复盘后 | 复盘后2h内 |
| 场控调度师 | 数字主播 | 实时调度指令 | 实时 | 下达后10s响应 |
| 场控调度师 | 流量投放操盘手 | 加投/减投需求 | 实时 | 需求后5min响应 |
| 店铺运营管家 | 数字主播 | 商品知识卡片 | 每场开播前 | T-2h就绪 |
| 店铺运营管家 | 场控调度师 | 库存/价格信息 | 实时 | 变更后5min同步 |
| 流量投放操盘手 | 场控调度师 | 投放状态变化 | 实时 | 变更后5min通知 |

### 7.2 升级通知机制
- 一级升级：角色间直接协调（5分钟内）
- 二级升级：通知运营主管（15分钟内）
- 三级升级：通知管理层（30分钟内）

升级触发条件：
- 角色间协调超时（超过SLA 2倍）→ 二级
- 影响GMV超过日目标10% → 二级
- 平台处罚风险/重大故障 → 三级
- 预算超支>20% → 三级
