# 峰谷电价调度 (Peak-Valley Scheduling)

## 技能概述

峰谷电价调度技能负责根据分时电价策略智能排程可调度家电设备的运行时间，将高耗电负载从峰电时段转移至谷电时段运行，在不影响用户使用体验的前提下最大化电费节省。这是家庭能耗优化的核心能力，直接影响月度电费支出。

### 适用场景
- EV充电桩的充电时段智能规划（通常23:00后充电最经济）
- 热水器预加热时段优化（凌晨谷电加热，白天保温使用）
- 洗碗机/洗衣机/烘干机的延迟启动调度
- 储能电池的充放电时段管理
- 用户设定月度电费预算后的自动调度优化

## 执行步骤

### 步骤1：电价时段信息获取
1. 查询当地电网分时电价政策：
   - 峰时段：时间范围 + 单价（元/kWh）
   - 平时段：时间范围 + 单价
   - 谷时段：时间范围 + 单价
   - 尖峰时段（部分地区夏季）：时间范围 + 单价
2. 检查是否有阶梯电价叠加（月度用电超过档位后单价上浮）
3. 获取当月已用电量和剩余预算
4. 判断当前所处电价时段

### 步骤2：可调度负载识别
1. 扫描家庭设备列表，标记可调度设备：
   - EV充电桩：可调度（用户指定充满时间即可）
   - 电热水器：可调度（提前加热+保温模式）
   - 洗碗机/洗衣机/烘干机：可调度（延迟启动）
   - 储能电池充电：可调度
2. 标记不可调度设备：
   - 空调/暖气：实时使用，不可延迟
   - 照明：实时使用
   - 冰箱：持续运行，负载恒定
3. 获取可调度设备的功率参数和运行时长需求

### 步骤3：约束条件收集
1. 用户使用约束：
   - EV充电：明天几点出门？（需要在出门前充满）
   - 热水器：早上几点需要热水？（需提前加热完成）
   - 洗衣机：衣服能否过夜？（部分用户不接受湿衣过夜）
2. 设备运行约束：
   - EV充电时长（根据当前电量和目标电量计算）
   - 热水器加热时长（根据水温差和容量计算）
   - 洗碗机/洗衣机运行时长（程序设定）
3. 电力约束：
   - 家庭总进线容量（避免同时启动导致跳闸）
   - 储能电池SOC上下限

### 步骤4：最优调度方案生成
1. 目标函数：最小化电费 = Σ(设备功率 × 运行时长 × 所在时段电价)
2. 约束满足：所有使用需求在截止时间前完成
3. 生成调度方案：
   - 优先将所有可调度负载排入谷电时段(23:00-7:00)
   - 如果谷电时段容量不足（进线容量限制），按优先级排队
   - 优先级：EV充电 > 热水器 > 储能充电 > 洗衣机 > 洗碗机
4. 计算节省金额：对比"全峰电执行"和"调度后执行"的费用差
5. 评估舒适度影响分数（0=无影响，10=严重影响）

### 步骤5：调度方案执行与监控
1. 在计划时间点发送启动指令给对应设备
2. 监控设备实际运行状态（是否正常启动/是否提前结束）
3. 动态调整：如果用户临时改变计划（如提前出门），重新计算方案
4. 记录实际执行数据（用于后续优化和报告生成）
5. 异常处理：设备未按计划启动→告警+尝试重新调度

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| tariff_schedule | object | 是 | 分时电价配置{peak, flat, valley时段及单价} |
| schedulable_devices | array | 是 | 可调度设备列表（含功率、运行时长） |
| user_constraints | object | 是 | 用户使用截止时间约束 |
| power_capacity_kw | float | 是 | 家庭总进线容量(kW) |
| battery_soc | float | 否 | 储能电池当前SOC(%) |
| monthly_budget | float | 否 | 月度电费预算(元) |
| monthly_used_kwh | float | 否 | 当月已用电量(kWh) |

## 最佳实践

1. **预留安全余量**：EV充电完成时间比用户出门时间早30分钟，避免意外延迟
2. **错开大功率设备启动**：避免同时启动多个大功率设备导致总负载超过进线容量
3. **天气关联优化**：明天降温→热水器多加热一些；明天高温→EV空调预冷提前
4. **学习用户实际使用模式**：记录用户几点真正用热水/几点出门，比用户手动设置更准
5. **不牺牲核心体验**：如果用户早上6点需要热水但谷电5点才结束，宁可4:30开始加热也要保证有热水

## 约束条件

- 调度方案必须在用户约束截止时间前完成所有任务
- 家庭总负载任何时刻不超过进线容量的90%
- 谷电利用率目标：≥70%（针对可调度负载）
- 调度方案生成时间 < 5秒
- 舒适度影响评分 > 5分的方案需用户确认才执行
- 月度电费预算偏差控制在10%以内
