# 能耗报告生成 (Energy Report Generation)

## 技能概述

能耗报告生成技能负责定期（日/周/月）自动生成家庭能耗分析报告，为用户提供直观的用电数据可视化、设备能耗排行、峰谷电量分布、节能金额统计、碳减排数据以及个性化节能建议。报告是用户理解自家用电模式和优化空间的核心信息载体。

### 适用场景
- 每日能耗快报（早晨推送前一天用电概况）
- 每周能耗周报（对比上周，发现异常用电模式）
- 每月能耗月报（含费用对比、预算执行情况、节能成效）
- 用户主动查询特定设备/时段的用电详情
- 能耗异常告警后的详细分析报告

## 执行步骤

### 步骤1：数据采集与聚合
1. 从设备功耗监测系统获取原始用电数据：
   - 各设备每小时用电量(kWh)
   - 分时段（峰/平/谷）累计电量
   - 实时功率峰值记录
2. 从电价系统获取费用数据：
   - 各时段实际电费
   - 阶梯电价档位状态
3. 从调度系统获取优化数据：
   - 调度执行记录（哪些负载被移至谷电）
   - 实际节省金额

### 步骤2：统计指标计算
1. **总量指标**：
   - 报告期总用电量(kWh)和总电费(元)
   - 同比变化（vs 去年同期）和环比变化（vs 上期）
2. **分布指标**：
   - 峰/平/谷电量占比
   - 各设备用电量占比（Top 10排行）
   - 按时段的用电曲线（24小时功率曲线）
3. **效率指标**：
   - 谷电利用率（可调度负载中实际在谷电运行的比例）
   - 光伏自用率（发电量中直接消耗的比例）
   - 储能循环效率
4. **经济指标**：
   - 通过调度节省的电费金额
   - 光伏发电等效节省金额
   - 月度预算执行率

### 步骤3：碳减排数据计算
1. 获取当地电网排放因子（中国各省不同，约0.5-0.9 kgCO₂/kWh）
2. 计算光伏发电减排量 = 光伏发电量 × 电网排放因子
3. 计算调度减排量（谷电通常核电/风电占比高，碳强度更低）
4. 折算为等效数据（相当于种了N棵树 / 少开了N公里车）

### 步骤4：个性化节能建议生成
1. 分析用电模式异常：
   - 某设备用电量突增>30%→ 可能是效率下降，建议检修
   - 峰电时段高耗设备运行→ 建议调整至谷电
   - 空调设定温度与外温差>10度→ 建议适当调整
2. 根据用户配置生成针对性建议（最多3条）：
   - 优先级：高节省潜力 > 易执行 > 习惯改变
3. 与上期建议对比：用户采纳了哪些？效果如何？

### 步骤5：报告格式化与推送
1. 按报告类型选择模板：
   - 日报：简洁版（关键数字 + 1条建议），适合推送通知
   - 周报：标准版（图表 + 排行 + 对比 + 建议）
   - 月报：完整版（全部指标 + 趋势 + 预算 + 碳数据）
2. 渲染可视化图表（柱状图/折线图/饼图）
3. 通过用户偏好渠道推送（APP消息/邮件/微信服务号）
4. 存档报告供历史查询

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| report_type | enum | 是 | daily/weekly/monthly |
| period_start | datetime | 是 | 报告起始时间 |
| period_end | datetime | 是 | 报告结束时间 |
| residence_id | string | 是 | 住所ID |
| device_energy_data | array | 是 | 各设备能耗原始数据 |
| tariff_config | object | 是 | 电价配置 |
| solar_data | object | 否 | 光伏发电数据 |
| battery_data | object | 否 | 储能电池充放电数据 |
| budget_config | object | 否 | 月度预算配置 |

## 最佳实践

1. **日报简洁为主**：用户不需要每天看详细报告，突出异常和关键变化即可
2. **月报展示成就感**：突出节省了多少钱、减排了多少碳，正向激励用户
3. **建议要可执行**：不说"减少用电"，而说"将热水器从18:00加热改为23:00加热，预计省15元/月"
4. **异常及时告知**：某设备用电量突增30%以上不等月报，当日即推送告警
5. **数据可溯源**：报告中的每个数字都要能追溯到原始设备数据，支持用户质疑时的详细查询

## 约束条件

- 日报在每天8:00前自动生成并推送
- 周报在每周一9:00前生成
- 月报在每月1日10:00前生成
- 报告生成耗时 < 30秒
- 历史报告保留期限：24个月
- 能耗数据精度：精确到0.1kWh
- 费用数据精度：精确到0.01元
