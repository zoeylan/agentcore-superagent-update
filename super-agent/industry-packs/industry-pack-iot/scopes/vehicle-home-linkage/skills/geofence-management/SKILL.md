# 地理围栏管理 (Geofence Management)

## 技能概述

地理围栏管理技能负责创建、配置、维护和评估地理围栏规则，是车家联动的核心触发基础设施。该技能覆盖围栏的全生命周期管理，从创建配置到运行时的进入/离开事件检测，以及围栏抖动抑制等关键可靠性保障。

### 适用场景
- 用户设置"到家自动开灯"、"离家布防"等地理联动规则
- 系统检测到用户穿越围栏边界时的事件触发判定
- 围栏参数的动态调整（半径、灵敏度、生效时段）
- 多围栏嵌套场景的优先级管理（小区围栏 vs 家门口围栏）

## 执行步骤

### 步骤1：围栏创建与配置
1. 接收围栏定义请求（中心点坐标、半径、名称、关联场景）
2. 验证坐标有效性（经纬度范围校验、是否在中国境内）
3. 设置围栏参数：
   - 半径：默认500米，最小100米，最大5000米
   - 触发方向：进入(enter)/离开(exit)/双向(both)
   - 生效时段：全天/仅工作日/仅周末/自定义时间段
   - 防抖窗口：默认5分钟，可配置1-15分钟
4. 持久化围栏配置并激活监听

### 步骤2：围栏事件检测
1. 接收实时位置更新数据（经纬度、精度、时间戳、数据源）
2. 计算位置点与围栏中心的Haversine距离
3. 判断是否穿越围栏边界（从内到外=exit，从外到内=enter）
4. 记录穿越事件候选（尚未确认）

### 步骤3：防抖验证
1. 检查最近5分钟（防抖窗口）内是否有反向穿越事件
2. 如果存在反向事件→判定为围栏边缘抖动→丢弃本次事件
3. 如果无反向事件→确认触发→生成围栏事件
4. 输出确认后的触发事件（含完整上下文信息）

### 步骤4：围栏健康监测
1. 定期统计各围栏的触发频率和抖动率
2. 抖动率超过20%的围栏标记为"需优化"（建议增大半径或调整中心点）
3. 长期未触发的围栏提醒用户是否需要保留

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| latitude | float | 是 | 围栏中心纬度 (-90 ~ 90) |
| longitude | float | 是 | 围栏中心经度 (-180 ~ 180) |
| radius_meters | int | 是 | 围栏半径（100-5000米） |
| fence_name | string | 是 | 围栏名称 |
| trigger_direction | enum | 是 | enter/exit/both |
| linked_scene_id | string | 否 | 关联的联动场景ID |
| active_schedule | object | 否 | 生效时间段配置 |
| debounce_minutes | int | 否 | 防抖窗口（默认5分钟） |

## 最佳实践

1. **城区围栏半径建议200-500米**：城市高楼密集区GPS漂移较大，围栏过小会频繁误触发
2. **高速公路场景扩大围栏**：车辆高速通过时定位更新间隔可能导致"跳过"围栏，建议扩大至1000米以上
3. **嵌套围栏从外到内触发**：大围栏（小区入口500米）触发"预开空调"，小围栏（楼下100米）触发"开灯迎宾"
4. **夜间降低定位频率**：23:00-6:00用户通常在家，可降低定位采样至每5分钟一次以省电
5. **WiFi辅助判定在家状态**：连接到家庭WiFi即可确认"在家"，无需持续GPS定位

## 约束条件

- 围栏数量上限：单用户不超过20个活跃围栏
- 位置数据保留策略：原始轨迹不持久化，仅保留事件日志30天
- 围栏触发到事件输出延迟 < 2秒
- 防抖窗口内的事件不计入触发统计
