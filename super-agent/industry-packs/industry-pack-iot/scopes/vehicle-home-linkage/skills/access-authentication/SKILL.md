# 访问认证与权限管理 (Access Authentication)

## 技能概述

访问认证与权限管理技能负责远程控制操作的全链路安全保障，包括身份验证、设备绑定关系验证、权限检查、敏感操作二次确认，以及异常访问检测。该技能是防止未授权访问家居设备的核心安全屏障。

### 适用场景
- 用户通过APP发起远程控制时的身份验证
- 敏感设备操作（门锁/摄像头/安防）的二次认证
- 家庭成员权限分级管理（管理员/成员/访客）
- 访客临时授权与自动过期管理
- 异常登录检测（异地/新设备/非常规时间）
- 操作日志审计与安全报告

## 执行步骤

### 步骤1：身份验证（第一因素）
1. 验证用户Token有效性（JWT解码+签名验证+过期检查）
2. 检查Token关联的用户ID和设备指纹
3. 验证请求来源设备是否为已注册设备（设备指纹匹配）
4. 如Token过期→引导重新登录（不允许自动续期敏感操作Token）
5. 记录认证尝试日志（成功/失败/来源IP/设备信息）

### 步骤2：设备绑定关系验证（第二因素）
1. 查询用户与目标设备的绑定关系（用户是否是该设备所属住所的成员）
2. 验证操作权限级别：
   - 管理员：全部设备全部操作
   - 成员：已授权设备的常规操作（不含安防配置修改）
   - 访客：仅限指定设备的基础操作（开关/调节），且在授权有效期内
3. 访客权限有效期检查（默认24小时，超时自动撤销）
4. 绑定关系不存在→拒绝操作并告警

### 步骤3：敏感操作二次确认
1. 判断操作是否属于敏感类别：
   - 门锁操作（开锁/修改密码）→ 必须二次确认
   - 摄像头操作（开启/查看画面/下载录像）→ 必须二次确认
   - 安防系统（撤防/修改配置）→ 必须二次确认
   - 全屋断电/重置操作 → 必须二次确认
2. 触发二次确认通道（按优先级选择）：
   - 生物特征认证（指纹/面容ID）→ 优先
   - 短信验证码 → 备选
   - 邮箱验证码 → 最后手段
3. 等待确认结果（超时5分钟自动取消）
4. 确认通过后生成临时操作令牌（有效期10分钟，单次使用）

### 步骤4：异常访问检测
1. 检测规则（任一触发即标记异常）：
   - 登录IP地理位置与常用位置偏差>500km
   - 首次使用的设备发起敏感操作
   - 非常规时间段的密集操作（凌晨2-5点连续操作）
   - 短时间内多次认证失败（5分钟内≥3次）
   - 同一账号在不同地理位置同时活跃
2. 异常标记后处理：
   - 轻度异常：允许操作但通知管理员
   - 中度异常：要求额外验证后才允许操作
   - 重度异常：立即锁定账号并通知所有管理员

### 步骤5：操作审计日志
1. 记录完整的操作链路信息：
   - 操作者身份（用户ID/角色/登录设备）
   - 操作目标（设备ID/设备名/所属住所）
   - 操作内容（action/parameters）
   - 认证方式（Token/二次确认方式）
   - 操作结果（成功/失败/拒绝/原因）
   - 时间戳和耗时
2. 日志不可篡改（追加写入+哈希链验证）
3. 支持按时间/用户/设备/操作类型的多维度查询

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_token | string | 是 | JWT认证令牌 |
| device_fingerprint | string | 是 | 请求来源设备指纹 |
| target_device_id | string | 是 | 目标控制设备ID |
| action | string | 是 | 请求执行的操作 |
| request_ip | string | 是 | 请求来源IP |
| request_timestamp | datetime | 是 | 请求时间 |
| second_factor | object | 否 | 二次认证信息（如果已提供） |

## 输出规格

{

## 最佳实践

1. **零信任原则**：每次操作独立验证，不因"刚验证过"而跳过检查
2. **最小权限原则**：访客权限尽可能小，仅授权明确需要的设备和操作
3. **生物特征优先**：二次确认优先使用设备端生物特征（无网络延迟且更安全）
4. **防暴力破解**：连续5次认证失败后锁定账号30分钟
5. **管理员操作通知**：管理员的任何权限变更操作自动通知所有成员

## 约束条件

- 身份验证耗时 < 200ms
- 二次确认超时时间：5分钟
- 临时操作令牌有效期：10分钟（单次使用后立即失效）
- 访客权限最长有效期：7天（默认24小时）
- 审计日志保留期限：365天
- 认证失败锁定阈值：5次/5分钟
