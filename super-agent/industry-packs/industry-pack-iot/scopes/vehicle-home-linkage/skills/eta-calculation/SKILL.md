# ETA动态计算 (ETA Calculation)

## 技能概述

ETA（Estimated Time of Arrival）动态计算技能负责根据用户当前位置、移动速度、历史通勤数据和实时路况，精确预估用户到达目标地点（通常是家）的时间。该预估结果直接决定家居设备的预启动时机——例如，空调需要提前15分钟开启才能在用户到家时达到舒适温度。

### 适用场景
- 下班回家途中，计算何时预开空调/热水器
- 判断是否需要提前启动"回家迎宾"场景
- 多路径通勤的智能识别（用户今天走了不同的路）
- 动态调整预启动时机（堵车时延后、畅通时提前）

## 执行步骤

### 步骤1：数据采集与预处理
1. 获取用户当前位置（经纬度、精度）
2. 获取最近N个位置点（用于计算瞬时速度和移动方向）
3. 计算当前车速（基于最近3个GPS点的平均速度）
4. 确定移动方向（是否朝向目标围栏方向移动）
5. 查询目标点坐标（家/办公室等已配置地址）

### 步骤2：路径距离估算
1. 计算直线距离（Haversine公式）
2. 应用道路绕行系数（城市道路通常为直线距离的1.3-1.5倍）
3. 如有历史通勤路径数据，使用实际路径距离
4. 考虑当前位置是否在已知通勤路线上

### 步骤3：速度模型选择
1. **实时速度模型**：使用当前GPS测速（适用于高速/快速路段）
2. **历史统计模型**：使用同时段同路段的历史平均速度（适用于拥堵路段）
3. **混合模型**：根据路段类型加权融合实时速度和历史速度
4. 拥堵路段权重：历史速度 × 0.7 + 实时速度 × 0.3
5. 畅通路段权重：实时速度 × 0.8 + 历史速度 × 0.2

### 步骤4：ETA计算与置信度评估
1. ETA = 剩余距离 / 预估平均速度
2. 加入停车/上楼时间余量（默认+5分钟）
3. 计算置信度：
   - 高置信度(>0.8)：在已知通勤路线上、速度稳定、距离<10km
   - 中置信度(0.5-0.8)：未知路线但方向正确、速度波动中等
   - 低置信度(<0.5)：方向不确定、速度异常、距离>30km
4. 每2分钟重新计算一次ETA并更新

### 步骤5：预启动时机决策
1. 查询设备预热需求时间（空调15-20分钟、热水器30-45分钟、地暖60分钟）
2. 当ETA ≤ 设备预热时间 + 安全余量(3分钟) 时，触发预启动
3. 如果ETA持续波动（堵车/绕路），动态调整预启动时机
4. 避免过早启动造成能源浪费（宁可晚1分钟，不早10分钟）

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current_location | object | 是 | {latitude, longitude, accuracy, timestamp} |
| target_location | object | 是 | {latitude, longitude, name} |
| recent_positions | array | 是 | 最近5-10个位置点（含时间戳） |
| historical_commute | object | 否 | 历史通勤统计数据 |
| traffic_condition | enum | 否 | smooth/moderate/congested |
| vehicle_speed_kmh | float | 否 | 车辆OBD上报的速度 |

## 最佳实践

1. **避免频繁变更ETA导致设备反复启停**：ETA变化<3分钟时不更新预启动决策
2. **高峰时段使用保守估计**：17:00-19:00通勤高峰期，ETA自动上浮20%
3. **学习个人通勤模式**：记录每周同一时段的实际通勤时间，持续优化模型
4. **设备预热时间应可配置**：不同品牌空调制冷/制热速度不同，应允许用户校准
5. **极端天气修正**：暴雨/暴雪天气自动将ETA上浮30-50%

## 约束条件

- ETA预测误差目标：< 5分钟（80%情况下）
- 最短触发距离：2km以内才启动ETA计算（太远则不可靠）
- 最长预测范围：60分钟（超过60分钟的ETA不触发设备预启动）
- 计算频率：接近围栏（<5km）时每1分钟计算一次，较远时每5分钟
