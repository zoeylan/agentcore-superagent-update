# 指令队列与重发管理 (Command Queue Management)

## 技能概述

指令队列与重发管理技能负责在网络不稳定场景下保障远程控制指令的可靠送达。当云端与家庭网关之间的网络连接中断时，待执行的控制指令进入持久化队列等待；网络恢复后按优先级重新发送，确保用户的远程控制意图最终得到执行（或在超时后明确告知失败）。

### 适用场景
- 家庭宽带网络临时中断（路由器重启/运营商故障）
- 4G/5G网络到家庭WiFi的切换间隙
- 大批量联动指令导致网关暂时过载
- 设备端暂时离线后恢复
- 用户在弱网环境（电梯/地下室）发出控制指令

## 执行步骤

### 步骤1：指令入队判定
1. 接收需要下发的控制指令
2. 尝试立即发送至目标网关/设备
3. 检测发送结果：
   - 成功：正常流程，不入队
   - 超时（3秒无响应）：判定网络可能中断→执行步骤2
   - 连接拒绝/DNS失败：确定网络中断→执行步骤2
   - 网关返回过载(429)：暂时过载→延迟2秒后重试1次→仍失败则入队

### 步骤2：指令持久化入队
1. 将指令写入持久化队列（避免进程重启丢失），包含：
   - 指令完整内容（目标/操作/参数）
   - 入队时间戳
   - 优先级（critical/high/normal/low）
   - 过期时间（入队时间 + TTL）
   - 重试计数器（初始为0）
   - 来源追踪ID
2. 优先级规则：
   - Critical：安全相关（报警/门锁）→ TTL 60分钟
   - High：联动触发（回家场景）→ TTL 30分钟
   - Normal：用户手动操作 → TTL 30分钟
   - Low：定时任务/能耗调度 → TTL 15分钟
3. 通知用户："指令已排队，将在网络恢复后自动执行"

### 步骤3：网络状态监测
1. 启动网络恢复探测器（每30秒ping网关一次）
2. 监听网络状态变更事件（WiFi连接/断开、蜂窝切换）
3. 收到网关心跳恢复信号 → 触发队列重发
4. 连续3次ping成功 → 确认网络恢复

### 步骤4：队列重发执行
1. 网络确认恢复后，按优先级从高到低取出队列指令
2. 检查每条指令是否已过期（当前时间 > 入队时间 + TTL）
   - 已过期：丢弃并通知用户"指令已超时未执行"
   - 未过期：继续执行
3. 逐条发送（间隔500ms，避免网关过载）
4. 每条指令等待ACK确认
5. 发送成功：从队列移除，更新用户通知
6. 发送仍失败：重试计数+1，如果<3次则保留队列等下一轮

### 步骤5：超时处理与用户通知
1. 队列中指令超过TTL → 标记为"已放弃"
2. 生成放弃原因说明：
   - "网络中断持续超过30分钟，空调开启指令已取消"
   - "回家联动因网络超时未执行，请到家后手动操作"
3. 通过推送通道主动通知用户（不能静默丢弃）
4. 记录失败日志供分析网络可靠性趋势

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| command | object | 是 | 完整的控制指令内容 |
| priority | enum | 是 | critical/high/normal/low |
| ttl_minutes | int | 否 | 自定义TTL（默认按优先级） |
| source_id | string | 是 | 来源追踪ID |
| retry_policy | object | 否 | 自定义重试策略 |
| notify_on_queue | bool | 否 | 入队时是否通知用户（默认true） |

## 输出规格

```json
{
  "queue_status": "delivered",
  "command_id": "cmd_20240115_001",
  "queued_at": "2024-01-15T18:30:00+08:00",
  "delivered_at": "2024-01-15T18:35:22+08:00",
  "retry_count": 1,
  "queue_duration_seconds": 322,
  "delivery_result": "success",
  "device_ack": true
}
```

## 最佳实践

1. **不要在队列中堆积冲突指令**：如果队列中已有"开空调26度"，新来"开空调24度"应覆盖前者
2. **安全指令永远最高优先级**：布防/报警类指令即使队列满也要挤入
3. **过期通知比静默丢弃更重要**：用户宁可收到"失败"通知也不愿不知道结果
4. **网络恢复后渐进发送**：避免100条积压指令瞬间涌入导致网关二次过载
5. **指令合并优化**：队列中对同一设备的多次参数调整可合并为最终状态一次下发

## 约束条件

- 队列最大容量：200条指令
- 指令最大TTL：60分钟（安全类）/ 30分钟（其他）
- 最大重试次数：3次
- 重发间隔：首次立即→ 2秒 → 5秒 → 放弃
- 网络探测频率：断网后每30秒探测一次
- 队列持久化延迟：< 100ms（确保进程崩溃不丢失）
