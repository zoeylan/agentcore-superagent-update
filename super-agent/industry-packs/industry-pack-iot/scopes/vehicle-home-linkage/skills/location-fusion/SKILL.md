# 多源定位数据融合 (Location Fusion)

## 技能概述

多源定位数据融合技能负责整合GPS卫星定位、WiFi指纹定位、蜂窝基站定位、车辆OBD数据、以及蓝牙Beacon信号等多种位置数据源，输出统一的、高置信度的用户位置估计。不同定位源在精度、功耗、可用性方面各有优劣，融合算法需要根据场景动态选择最优组合。

### 适用场景
- GPS信号弱（室内/地下车库）时的备用定位
- 通过WiFi连接快速判断"在家/在公司"状态
- 车辆点火/熄火事件辅助判断出发/到达
- 多数据源相互验证提升定位置信度
- 根据定位精度需求动态调节功耗

## 执行步骤

### 步骤1：数据源状态评估
1. 检查各定位源当前可用性：
   - GPS：是否有足够卫星（≥4颗）、信号强度、最近更新时间
   - WiFi：是否扫描到已知AP、信号强度、AP数据库匹配情况
   - 蜂窝：当前基站ID、信号强度、基站位置库是否有数据
   - 车辆OBD：是否通过蓝牙连接、车速/里程数据是否可用
   - BLE Beacon：是否检测到已注册的Beacon设备
2. 标记不可用或数据过期（>30秒未更新）的数据源

### 步骤2：单源定位结果获取
1. GPS定位：直接获取经纬度+精度（通常5-15米）
2. WiFi定位：基于扫描到的AP MAC地址查询位置库（精度30-50米）
3. 蜂窝定位：基于服务基站和邻区基站三角定位（精度100-500米）
4. 车辆状态推断：点火=出发、熄火=到达、速度>0=途中
5. WiFi SSID匹配：连接到"Home_WiFi"→确认在家（精度=围栏内确认）

### 步骤3：加权融合计算
1. 为每个数据源分配权重（基于当前精度和可靠性）：
   - GPS精度<20米：权重0.7
   - WiFi已知AP匹配：权重0.5（与GPS互补）
   - 蜂窝定位：权重0.2（作为兜底验证）
   - 车辆OBD状态：权重0.8（高可靠性布尔判断）
   - 家庭WiFi连接：权重1.0（确定性判断"在家"）
2. 使用加权平均计算融合位置坐标
3. 融合精度 = min(各源精度) × 融合修正系数(0.7-0.9)

### 步骤4：异常检测与过滤
1. 检测位置跳变：两次位置更新间的速度是否超过物理上限（300km/h）
2. 检测数据源冲突：GPS显示在A位置但WiFi显示连接到B位置的AP→标记异常
3. 过滤GPS多径效应导致的瞬时跳变（高楼区域常见）
4. 对异常数据降权处理而非直接丢弃

### 步骤5：功耗策略动态调整
1. **省电模式**（在家/深夜）：仅依赖WiFi连接状态，关闭GPS
2. **平衡模式**（日常）：WiFi + 蜂窝，每5分钟一次GPS校准
3. **精确模式**（接近围栏）：GPS持续开启 + WiFi辅助，1秒更新
4. 根据与最近围栏的距离自动切换模式

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| gps_data | object | 否 | {lat, lng, accuracy_m, satellites, timestamp} |
| wifi_scan | array | 否 | [{bssid, ssid, rssi, known_location}] |
| cell_info | object | 否 | {cell_id, lac, mcc, mnc, rssi} |
| vehicle_obd | object | 否 | {ignition: bool, speed_kmh, odometer_km} |
| ble_beacons | array | 否 | [{uuid, major, minor, rssi, registered_location}] |
| power_mode | enum | 否 | saving/balanced/precise |

## 最佳实践

1. **家庭WiFi作为确定性锚点**：连接到已注册的家庭WiFi即100%确认在家，无需其他验证
2. **车辆OBD作为状态转换触发器**：点火事件触发定位精度提升，熄火事件触发围栏到达判定
3. **地下车库场景**：GPS失效时，依赖车辆熄火+WiFi扫描来判断到达
4. **避免在室内持续请求GPS**：检测到WiFi已确认位置后立即关闭GPS以省电
5. **数据源一致性校验**：至少2个独立数据源一致时才输出高置信度结果

## 约束条件

- 至少1个定位源可用才输出位置结果
- 位置更新频率：精确模式≤1秒，平衡模式≤60秒，省电模式≤300秒
- 所有原始定位数据仅在内存中缓存（最近100个点），不持久化
- 融合算法延迟 < 100ms
- 电池消耗：省电模式日均<2%，平衡模式<5%，精确模式<15%（仅短时使用）
