# 远程设备控制 (Remote Device Control)

## 技能概述

远程设备控制技能提供用户不在家时对家居设备进行远程操控的核心能力，包括单设备控制、设备分组批量控制、定时任务远程下发，以及多住所间的设备统一管理。该技能确保远程指令能够可靠地到达目标设备并返回执行结果确认。

### 适用场景
- 用户出差期间远程查看和控制家中设备
- 接收地理围栏联动触发后的设备批量控制指令
- 远程下发定时任务（如下周三预开热水器）
- 多住所（主宅/度假屋/父母家）设备统一管理界面
- 设备分组操作（"关闭所有灯"、"客厅设备全部待机"）

## 执行步骤

### 步骤1：指令接收与解析
1. 接收控制请求（来源：用户APP手动操作 / 地理围栏自动触发 / 定时任务引擎）
2. 解析指令内容：目标设备ID/分组ID、操作类型(on/off/set/query)、参数值
3. 验证指令格式完整性（必填字段是否齐全、参数值是否在设备支持范围内）
4. 标记指令来源和优先级（安全类 > 联动触发 > 用户手动 > 定时任务）

### 步骤2：目标设备定位
1. 查询设备注册表获取目标设备信息（所属住所、通信协议、网关地址）
2. 确认设备当前在线状态（从状态同步监控器获取心跳数据）
3. 如果是分组操作，展开分组为具体设备列表
4. 检查设备是否处于"本地占用"状态（有人正在本地操作）

### 步骤3：指令下发与确认
1. 将标准化指令通过对应网关/协议通道发送至设备
2. 启动ACK等待计时器（超时：WiFi设备3秒，Zigbee设备5秒，蓝牙设备8秒）
3. 接收设备ACK确认执行
4. 如未收到ACK→执行重试策略（最多重试2次，间隔递增1s/3s）
5. 批量控制时并行下发，逐个收集ACK结果

### 步骤4：结果汇总与反馈
1. 汇总所有设备的执行结果（成功/失败/超时/被拒绝）
2. 生成执行结果摘要：成功数/失败数/失败原因
3. 通过推送通道反馈给用户（成功静默通知/失败主动弹窗告知）
4. 记录完整操作日志（审计用途）

### 步骤5：定时任务管理
1. 接收用户远程下发的定时任务（执行时间+控制指令）
2. 将任务存储至目标住所的本地网关任务队列
3. 到达执行时间后由本地网关独立执行（不依赖云端连接）
4. 执行结果异步上报至云端同步给用户

## 输入规格

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| request_source | enum | 是 | user_app/geofence_trigger/timer_task |
| target_type | enum | 是 | single_device/device_group/residence |
| target_id | string | 是 | 设备ID或分组ID |
| action | enum | 是 | power_on/power_off/set_param/query_state |
| parameters | object | 否 | 设备控制参数（如 {temperature: 26, mode: "cool"}） |
| residence_id | string | 是 | 所属住所ID |
| scheduled_time | datetime | 否 | 定时执行时间（立即执行时为空） |
| priority | enum | 否 | critical/high/normal/low |

## 最佳实践

1. **本地优先原则**：检测到设备正在本地操作时，延迟远程指令并通知用户
2. **批量操作渐进执行**：避免同时给10+设备发送指令导致网关过载，分批次（每批5个）执行
3. **状态验证**：控制后查询一次设备实际状态，确认执行结果与预期一致
4. **离线设备友好提示**：设备离线时不是简单报错，而是提供建议（检查WiFi/重启网关）
5. **住所切换响应**：多住所管理时，切换住所后2秒内完成设备列表加载

## 约束条件

- 单次批量控制上限：50个设备
- 指令队列最大深度：100条（超出时拒绝新指令并告警）
- 定时任务最大数量：每住所50条活跃定时任务
- 远程控制端到端延迟目标：<3秒（不含网络传输时间）
- 操作日志保留期限：180天
