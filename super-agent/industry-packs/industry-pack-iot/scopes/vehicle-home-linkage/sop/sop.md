# 车家互联与远程管理 — 标准操作规程 (SOP)

## 1. 概述

本SOP定义了车家互联与远程管理业务域的标准操作流程、质量检查点、职责分配和异常处理机制。涵盖三大核心业务流程：车家联动响应流程、远程设备控制流程、能耗智能调度流程。

### 适用范围
- 基于地理围栏的家居自动化联动
- 用户远程设备监控与控制
- 多住所统一管理
- 峰谷电价调度与能耗优化
- 光伏储能协同管理

### 核心目标
- 车家联动端到端响应 < 30秒
- 远程控制指令送达率 ≥ 99%
- 谷电利用率 ≥ 70%（可调度负载）
- 安全认证100%执行，零未授权访问

## 3. 车家联动响应流程

### 3.1 触发条件
- 用户位置穿越已配置的地理围栏边界
- 车辆OBD数据显示点火/熄火状态变更
- WiFi连接状态变更（连接到家庭WiFi/断开家庭WiFi）

### 3.2 流程步骤

```
┌─────────────┐    ┌──────────────┐    ┌─────────────┐    ┌──────────────┐
│ 位置数据    │───▶│ 围栏穿越    │───▶│ 防抖验证    │───▶│ 事件确认    │
│ 持续监听    │    │ 检测        │    │ (5分钟窗口) │    │ 输出        │
└─────────────┘    └──────────────┘    └─────────────┘    └──────────────┘
                                              │                    │
                                              │ 有反向事件          │
                                              ▼                    ▼
                                        ┌──────────┐     ┌──────────────┐
                                        │ 丢弃事件 │     │ 查询联动规则 │
                                        │ (抖动)   │     │              │
                                        └──────────┘     └──────┬───────┘
                                                                │
                                                   ┌────────────┼───────────┐
                                                   │            │           │
                                                   ▼            ▼           ▼
                                             ┌──────────┐ ┌──────────┐ ┌──────────┐
                                             │ 立即执行 │ │ ETA预判  │ │ 无匹配   │
                                             │ (离家)   │ │ (回家)   │ │ 规则     │
                                             └────┬─────┘ └────┬─────┘ └──────────┘
                                                  │            │
                                                  ▼            ▼
                                            ┌───────────────────────┐
                                            │ 发送联动指令至        │
                                            │ 远程控制管理器        │
                                            └───────────┬───────────┘
                                                        │
                                                        ▼
                                            ┌───────────────────────┐
                                            │ 鉴权 → 下发 → 确认   │
                                            └───────────────────────┘
```

### 3.3 操作规范

| 步骤 | 操作 | 输出 | 时限 |
|------|------|------|------|
| 位置监听 | 多源定位融合（GPS+WiFi+OBD） | 融合位置坐标+置信度 | 实时 |
| 围栏检测 | Haversine距离计算+边界判定 | 穿越事件候选 | < 2秒 |
| 防抖验证 | 5分钟窗口内反向事件检查 | 确认/丢弃决定 | 即时 |
| ETA计算 | 距离+速度+路况→到达时间预估 | ETA秒数+置信度 | < 1秒 |
| 联动触发 | 生成完整触发事件+上下文 | 结构化事件报文 | < 1秒 |
| 指令下发 | 通过远程控制管理器下发 | 设备ACK确认 | < 30秒(端到端) |

### 3.4 异常处理

| 异常场景 | 处理策略 | 升级条件 |
|---------|---------|---------|
| GPS信号丢失 | 切换至WiFi/蜂窝定位，降低围栏精度要求 | 所有定位源失效>10分钟→通知用户 |
| 围栏持续抖动 | 建议用户增大围栏半径或调整中心点 | 抖动率>20%持续3天→自动增大半径10% |
| ETA预测大幅偏差 | 每2分钟重新计算，动态调整预启动 | 设备已启动但用户掉头→立即关闭并通知 |
| 联动指令执行失败 | 重试1次→失败则告知用户手动操作 | 连续3次联动失败→检查网络/设备状态 |

### 3.5 质量检查点

- ✅ 围栏触发到设备响应延迟 < 30秒（KPI，每日监控）
- ✅ ETA预测误差 < 5分钟（80%情况下）
- ✅ 误触发率 < 3%（按月统计）
- ✅ 防抖算法有效率 > 95%（真实抖动被过滤的比例）

## 4. 远程设备控制流程

### 4.1 触发条件
- 用户通过APP主动发起远程控制操作
- 地理围栏联动触发自动控制
- 定时任务到达执行时间
- 家属远程关怀操作

### 4.2 流程决策树

```
用户发起远程控制请求
    │
    ▼
┌──────────────────┐
│ 第一步：身份验证 │

└────────┬─────────┘
         │
    验证通过？─── 否 ──▶ 拒绝 + 记录 + 异常检测
         │
         是
         ▼
┌──────────────────────┐
│ 第二步：权限检查     │
│ 用户角色 vs 目标设备 │
└────────┬─────────────┘
         │
    权限充足？─── 否 ──▶ 拒绝 + 告知原因("您无权操作此设备")
         │
         是
         ▼
┌──────────────────────┐
│ 第三步：是否敏感操作？│
└────────┬─────────────┘
         │
    ┌────┴────┐
    │         │
   是         否
    │         │
    ▼         │
┌────────────┐│

└─────┬──────┘│
      │       │

      │       │

      │       │
      ▼       ▼
┌──────────────────────┐
│ 第四步：本地冲突检查 │
└────────┬─────────────┘
         │
    有本地冲突？─── 是 ──▶ 本地优先 + 通知用户"设备被本地占用"
```

### 4.3 敏感操作清单

| 设备类型 | 敏感操作 | 二次确认方式 | 超时时间 |
|---------|---------|-------------|---------|
| 智能门锁 | 开锁/改密码/添加指纹 | 生物特征 + 短信 | 3分钟 |
| 摄像头 | 开启/查看实时画面/下载录像 | 生物特征 | 5分钟 |
| 安防系统 | 撤防/修改布防区域 | 短信验证码 | 3分钟 |
| 全屋控制 | 全屋断电/恢复出厂 | 生物特征 + 短信 | 3分钟 |
| 车库门 | 远程开门 | 生物特征 | 3分钟 |

### 4.4 网络中断处理规范

1. **检测**：指令发送后3秒无ACK → 判定可能中断
2. **重试**：立即重试1次 → 仍失败确认中断
3. **入队**：指令写入持久化队列，标记优先级和TTL
4. **探测**：每30秒ping网关，检测网络恢复
5. **重发**：网络恢复后按优先级逐条重发（间隔500ms）
6. **超时**：队列指令超过TTL → 丢弃 + 主动通知用户
7. **统计**：记录中断时长/影响指令数/恢复率

### 4.5 质量检查点

- ✅ 身份验证100%执行（零跳过）
- ✅ 敏感操作二次确认100%触发
- ✅ 异常登录检测（异地/新设备）即时告警 < 10秒
- ✅ 远程指令送达率 ≥ 99%（含重试）
- ✅ 断网恢复后指令重发成功率 ≥ 95%
- ✅ 操作日志完整性 ≥ 99.9%
- ✅ 告警推送延迟 < 10秒

## 5. 能耗智能调度流程

### 5.1 触发条件
- 每日22:30自动生成次日调度方案（为23:00谷电开始做准备）
- 电价时段切换时重新评估当前负载
- 用户修改使用计划（如明天提前出门）
- 光伏发电量显著偏离预测（天气突变）
- 储能电池SOC接近上下限

### 5.2 流程步骤

| 步骤 | 操作 | 输入 | 输出 | 决策点 |
|------|------|------|------|--------|
| 1 | 获取电价时段配置 | 电网电价政策 | 峰/平/谷时段表+单价 | - |
| 2 | 扫描可调度设备 | 设备注册表 | 可调度设备列表+功率+时长 | - |
| 3 | 收集用户约束 | 用户偏好/历史习惯 | 使用截止时间/不可延迟的负载 | - |
| 4 | 预测光伏发电 | 天气预报+光伏参数 | 24h发电功率曲线 | - |
| 5 | 生成调度方案 | 步骤1-4全部数据 | 设备启停时间表+节省金额 | 是否影响舒适度？ |
| 6 | 舒适度评估 | 调度方案+用户偏好 | 舒适度影响评分(0-10) | 评分>5→需用户确认 |
| 7 | 执行调度 | 确认后的方案 | 设备启停指令 | - |
| 8 | 实时监控调整 | 实际运行数据 | 动态修正指令 | 偏差>20%→重新调度 |
| 9 | 记录节能数据 | 执行结果 | 日度节能统计 | - |
| 10 | 生成报告 | 周期累计数据 | 日/周/月能耗报告 | - |

### 5.3 调度优先级矩阵

| 设备类型 | 可延迟性 | 典型功率 | 调度优先级 | 备注 |
|---------|---------|---------|-----------|------|
| EV充电桩 | 高（隔夜充电） | 7-11kW | 最高 | 节省金额最大 |
| 电热水器 | 中（需提前预热） | 2-3kW | 高 | 需匹配用水时间 |
| 储能电池充电 | 高（灵活） | 3-5kW | 高 | 峰电放电收益 |
| 洗碗机 | 高（餐后延迟启动） | 1.5-2kW | 中 | 噪音约束（不宜凌晨） |
| 洗衣机 | 中（衣物不宜久放） | 0.5-1kW | 中 | 用户可能不接受过夜 |
| 烘干机 | 中 | 2-3kW | 中 | 可跟随洗衣机 |
| 地暖 | 低（预热时间长） | 3-8kW | 低 | 舒适度影响大 |

### 5.4 异常处理

| 异常场景 | 处理策略 | 影响评估 |
|---------|---------|---------|
| 电价政策临时调整 | 重新获取电价→重新生成方案 | 可能影响当日节省金额 |
| 设备未按计划启动 | 告警+尝试重新启动→失败则通知用户 | 该设备节能收益归零 |
| 进线容量即将超限 | 立即延迟低优先级设备启动 | 调度计划临时调整 |
| 光伏发电远低于预测 | 增加谷电充电量补偿 | 储能放电时间缩短 |
| 用户临时改变计划 | 重新计算方案，保证用户需求优先 | 谷电利用率可能下降 |

### 5.5 质量检查点

- ✅ 谷电利用率 ≥ 70%（可调度负载，按月统计）
- ✅ 月度能耗预算偏差 < 10%
- ✅ 能耗报告按时生成率 100%
- ✅ 调度方案生成时间 < 5秒
- ✅ 光伏自用率 ≥ 60%
- ✅ 储能充放电效率 ≥ 90%

## 6. 多住所管理规范

### 6.1 权限隔离要求
- 不同住所的设备列表完全隔离
- 用户在住所A的管理员身份不自动继承到住所B
- 访客权限仅对指定住所有效
- 季度审计：验证权限隔离无串扰

### 6.2 性能要求
- 住所切换响应 < 2秒
- 设备列表同步延迟 < 5秒
- 单用户最多管理5个住所

### 6.3 数据隔离
- 能耗报告按住所独立生成
- 操作日志按住所分区存储
- 联动规则按住所独立配置

## 8. 安全合规要求

### 8.1 数据安全
- 位置数据：不持久化原始轨迹，仅保留脱敏事件日志（保留30天）
- 操作日志：不可篡改，保留365天
- 能耗数据：按住所隔离存储，保留24个月
- 视频/图片：端到端加密，用户可随时删除

### 8.2 访问安全
- 所有远程操作必须经过双重认证
- 敏感操作必须二次确认（零例外）
- 认证失败5次锁定30分钟
- 异地登录强制额外验证

### 8.3 合规审计
- 季度：权限配置全量审计
- 月度：操作日志完整性验证
- 周度：异常访问检测规则有效性验证
- 每日：安全事件汇总报告
