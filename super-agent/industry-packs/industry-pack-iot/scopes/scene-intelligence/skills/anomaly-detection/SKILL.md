# 传感器异常检测 (Anomaly Detection)

## 概述

本技能负责实时监测传感器数据流中的异常模式，区分传感器硬件故障、环境真实突变和数据通信错误，并执行相应的处理策略。异常检测的准确性直接影响场景推理的可靠性——漏检可能导致错误决策，误检则浪费系统资源和用户注意力。

## 适用场景

- 传感器数据流实时异常监测
- 传感器硬件故障的早期预警
- 数据通信链路中断检测
- 环境真实突变与设备故障的鉴别
- 传感器数据可信度评估与降级标记

## 执行步骤

### 1. 异常模式定义与分类

| 异常类型 | 特征 | 示例 | 严重等级 |
|---------|------|------|---------|
| 全零值 | 连续N个采样点为0 | 温度持续报0°C | 高 |
| 固定值 | 连续N个采样点完全相同 | 湿度24小时不变 | 中 |
| 物理越界 | 超出传感器物理可能范围 | 室内温度报80°C | 高 |
| 突变跳跃 | 相邻采样点变化超过物理可能 | 温度5分钟涨10°C | 高 |
| 趋势异常 | 与同时段历史趋势严重偏离 | 夏天午后温度持续下降 | 低 |
| 通信超时 | 超过3个采集周期未收到数据 | 传感器15分钟无响应 | 中 |
| 数据漂移 | 长期缓慢偏离校准基线 | 温度读数逐月偏高 | 低 |

### 2. 检测算法执行

- **实时规则检测**（每条数据到达时执行）：
  - 物理范围检查：温度[-40, 60]、湿度[0, 100]、PM2.5[0, 1000]
  - 变化速率检查：|当前值 - 上一个值| / 时间间隔 < 最大合理变化速率
  - 全零/固定值检查：连续3个以上采样点相同触发告警

- **统计异常检测**（滑动窗口，每小时评估）：
  - 计算最近7天同时段的均值μ和标准差σ
  - |当前值 - μ| > 3σ 标记为统计异常
  - 结合多传感器交叉验证（如相邻房间温差验证）

- **趋势异常检测**（每日评估）：
  - 将当日数据趋势与历史同类日（同星期/同天气）对比
  - DTW（动态时间规整）距离超过阈值标记为趋势异常
  - 传感器数据漂移检测：月度均值与初始校准值的偏差监测

### 3. 异常确认与鉴别

- 单传感器异常 + 相邻传感器正常 → 判定为硬件故障
- 多传感器同时异常 + 物理可解释（如开窗导致温度突变） → 判定为环境真实变化
- 多传感器同时无数据 → 判定为网关/通信故障
- 单传感器周期性异常（如每天同一时刻） → 判定为环境干扰（如阳光直射传感器）

### 4. 处理策略执行

- **硬件故障**：标记该传感器数据不可信→使用历史同时段均值替代→生成维修工单→通知用户
- **通信故障**：启动重连机制→使用最后有效值（衰减置信度）→超时30分钟报警
- **环境突变（确认为真实变化）**：保留数据→更新统计基线→触发场景推理事件
- **数据漂移**：建议用户重新校准传感器→调整补偿系数

## 输入规格

```json
{
  "sensor_id": "temp_humidity_living_01",
  "current_reading": {"temperature": 45.0, "humidity": 30},
  "previous_readings": [
    {"temperature": 26.5, "humidity": 55, "timestamp": "..."},
    {"temperature": 26.3, "humidity": 54, "timestamp": "..."}
  ],
  "historical_baseline": {"mean": 26.0, "std": 2.5, "period": "same_hour_7days"},
  "adjacent_sensors": [
    {"sensor_id": "temp_humidity_bedroom_01", "temperature": 25.8}
  ]
}
```

## 输出规格

```json
{
  "anomaly_detected": true,
  "anomaly_type": "sudden_jump",
  "severity": "high",
  "confidence": 0.95,
  "evidence": "温度从26.5°C突变至45.0°C（5分钟内变化18.5°C），超过最大合理变化速率(5°C/5min)。相邻传感器读数正常(25.8°C)，判定为传感器硬件故障。",
  "action": {
    "mark_untrusted": true,
    "substitute_value": 26.4,
    "substitute_method": "7day_same_hour_mean",
    "alert": {"type": "maintenance_required", "target": "user"},
    "notify_downstream": true
  }
}
```

## 最佳实践

- 新安装的传感器需要至少48小时的「学习期」建立统计基线，期间放宽异常阈值
- 季节交替时主动调整统计基线的计算窗口（避免将正常的季节变化误判为异常）
- 异常检测结果应记录完整日志，支持事后分析和算法调优
- 对于低严重度异常（如轻微漂移），采用累积计分机制——连续多日轻微异常升级为中度告警

## 约束条件

- 实时规则检测延迟<50ms（不阻塞数据流处理管线）
- 异常检测不修改原始数据，仅附加标注
- 误报率应控制在<10%，漏报率<5%（优先避免漏报）
- 告警去重：同一传感器同一类型异常30分钟内不重复告警
