# 传感器数据融合 (Sensor Data Fusion)

## 概述

本技能负责将全屋多种异构传感器的原始数据进行清洗、校准和融合，输出统一格式的环境状态向量。这是场景智能系统的数据基础层，数据质量直接决定下游推理的准确性。

## 适用场景

- 多传感器数据实时汇聚与统一处理
- 跨传感器类型的衍生指标计算（如体感温度PMV）
- 传感器数据采集频率管理与同步对齐
- 环境状态变化检测与阈值判断

## 执行步骤

### 1. 数据采集与频率管理
- 按预设频率从各传感器拉取/接收数据：
  - 温湿度：每5分钟
  - 光照：每1分钟
  - 人体感应：实时事件驱动
  - 空气质量（CO2/PM2.5/TVOC）：每10分钟
  - 噪音：每5分钟
  - 门窗磁：事件驱动
- 对不同频率的数据进行时间戳对齐（以最近有效值为准）

### 2. 数据清洗
- **去噪处理**：应用卡尔曼滤波（温湿度）或移动平均（光照/噪音）消除传感器噪声
- **缺失值插值**：
  - 缺失率<=2%：线性插值补充
  - 缺失率>2%且<=5%：样条插值 + 标记数据质量降级
  - 缺失率>5%：该传感器数据标记为不可信
- **异常值过滤**：
  - 物理范围硬约束（温度-40~60°C，湿度0~100%）
  - 3σ统计约束（偏离滑动窗口均值超过3个标准差）
  - 突变速率约束（温度5分钟内变化>5°C标记异常）

### 3. 多传感器融合计算
- **体感温度**：综合室内温度、湿度、风速（空调出风）计算PMV指数
- **综合舒适度指数**：加权融合温度舒适度(40%) + 光照舒适度(25%) + 空气质量(25%) + 噪音(10%)
- **空间占用状态**：融合人体感应 + 门窗磁 + 用电模式判断各房间是否有人

### 4. 环境状态向量输出
- 生成标准化JSON格式的环境状态向量
- 计算与上一次输出的delta变化值
- 计算数据质量评分（0-1）
- 当delta超过阈值时触发「环境变化」事件

## 输入规格

```json
{
  "sensor_readings": [
    {
      "sensor_id": "temp_humidity_01",
      "type": "temperature_humidity",
      "room": "living_room",
      "values": {"temperature": 26.5, "humidity": 55},
      "timestamp": "2024-01-15T10:00:00Z"
    }
  ],
  "external_data": {
    "weather": {"outdoor_temp": 32, "humidity": 70, "aqi": 85},
    "solar": {"sunrise": "06:30", "sunset": "18:45"}
  }
}
```

## 输出规格

```json
{
  "environment_state": {
    "rooms": {
      "living_room": {
        "temperature": 26.5,
        "humidity": 55,
        "perceived_temp": 27.8,
        "light_level": 450,
        "air_quality_score": 0.82,
        "noise_level": 35,
        "occupied": true
      }
    },
    "comfort_index": 0.78,
    "data_quality_score": 0.95,
    "delta_from_previous": {
      "max_change_param": "temperature",
      "max_change_value": 1.5,
      "threshold_exceeded": false
    }
  },
  "timestamp": "2024-01-15T10:00:00Z",
  "data_reliability": "trusted"
}
```

## 最佳实践

- 始终维护至少7天的历史数据滑动窗口，用于异常检测基线和缺失值替代
- 传感器更换或重新校准后，主动重置该传感器的统计基线
- 融合计算使用确定性算法，确保相同输入产生相同输出（可审计）
- 当多个房间传感器数据互相矛盾时（如隔壁房温差>10°C），优先怀疑硬件故障

## 约束条件

- 所有数据处理必须在本地边缘设备完成，不上传云端
- 单次融合计算延迟不超过200ms
- 内存占用不超过256MB（含7天历史数据缓存）
- 不对原始数据做不可逆修改，原始读数始终保留可追溯
