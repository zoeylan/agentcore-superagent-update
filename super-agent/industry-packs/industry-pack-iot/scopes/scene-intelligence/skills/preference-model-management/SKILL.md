# 偏好模型管理 (Preference Model Management)

## 概述

本技能负责维护每位家庭成员的个性化偏好模型的全生命周期：创建、训练、更新、评估、版本管理和退化检测。偏好模型是场景智能系统实现"个性化"的核心数据资产，其准确性直接影响用户对系统推荐的接受率。

## 适用场景

- 新用户偏好模型初始化与冷启动管理
- 基于新数据的模型增量更新
- 用户负反馈（手动覆盖）的即时模型修正
- 模型质量评估与置信度计算
- 偏好漂移检测与模型重建决策
- 多用户模型隔离与切换

## 执行步骤

### 1. 模型结构定义
每位用户的偏好模型包含以下维度：
- **温度偏好**：不同场景(睡眠/工作/休闲)×不同季节×不同时段的期望温度分布
- **光照偏好**：各房间×各时段的照明亮度和色温偏好
- **空气质量偏好**：通风策略（开窗vs空气净化器）×外部AQI阈值
- **场景组合偏好**：预定义场景（电影模式/阅读模式/聚会模式）的设备参数组合
- **能耗敏感度**：用户对能耗成本的敏感程度（影响峰电时段决策权重）

### 2. 模型更新策略

#### 正常增量更新（用户接受推荐时）
- 使用指数移动平均（EMA）更新偏好参数：新值 = α×当前操作值 + (1-α)×历史均值
- α（学习率）= 0.2（稳定期）/ 0.5（学习期）/ 0.4（检测到漂移时）
- 更新后重新计算置信度评分

#### 负反馈快速修正（用户手动覆盖时）
- 将本次推荐值标记为"不合适"
- 用户实际调整值以高权重（α=0.6）立即融入模型
- 检查是否连续3次被覆盖同一参数→如果是，触发该维度的深度重学习

#### 定期全量评估（每周）
- 计算模型预测值与用户实际操作值的MAE（平均绝对误差）
- MAE < 阈值：模型健康
- MAE > 阈值且上升趋势：触发偏好漂移告警

### 3. 置信度计算

置信度 = min(数据充分度, 近期一致性, 模型年龄因子)

- **数据充分度** = min(1, 有效样本数 / 最低样本要求)
  - 温度偏好最低样本：14天 × 3时段 = 42个数据点
  - 光照偏好最低样本：7天 × 各房间 = 按房间数×7
- **近期一致性** = 1 - (最近7天预测MAE / 容忍范围)
- **模型年龄因子** = 最后一次更新距今天数映射到衰减函数（30天以上开始衰减）

### 4. 偏好漂移检测
- 每月执行一次漂移评估：比较本月偏好参数与上月的差异
- 单一维度变化>30%：该维度标记为"漂移中"，增加学习率
- 多维度同时变化>20%：可能是生活方式改变，触发全面重学习评估
- 漂移确认后通知用户："我们注意到您的xx偏好似乎有变化，正在重新学习适应中"

### 5. 模型版本管理
- 每次显著更新保存模型快照（支持回滚）
- 保留最近5个版本
- A/B测试能力：新模型上线后对比旧模型，若新模型接受率下降>10%，自动回滚

## 输入规格

```json
{
  "user_id": "user_001",
  "update_type": "negative_feedback",
  "context": {
    "scene": "sleep",
    "time": "23:00",
    "season": "summer",
    "outdoor_temp": 28
  },
  "system_recommendation": {"ac_temp": 26, "ac_mode": "cool"},
  "user_actual_action": {"ac_temp": 24, "ac_mode": "cool"},
  "current_model_version": "v3.2"
}
```

## 最佳实践

- 负反馈的权重应显著高于正反馈（用户主动操作的信号强度远大于被动接受）
- 节假日期间的偏好数据应单独处理，不污染工作日模型
- 模型回滚时向用户透明说明："暂时回到之前的偏好设置，将继续学习您的新习惯"
- 家庭成员增减时触发全面模型重评估（新成员进入可能改变所有人的空间使用模式）

## 约束条件

- 每个用户的偏好模型存储不超过5MB
- 增量更新延迟<500ms（负反馈需要即时响应）
- 模型数据加密存储，不同用户间严格隔离
- 用户请求删除数据时，24小时内完全清除偏好模型和原始训练数据
