# 场景触发推理 (Scene Trigger Reasoning)

## 概述

本技能是场景智能域的核心推理能力，负责基于多维输入（环境变化、时间上下文、用户行为信号）判断是否应触发场景切换，并通过LLM推理生成最优的设备参数组合方案。场景触发的准确性是用户信任的基础——误触发会快速消耗用户耐心，漏触发则无法体现系统价值。

## 适用场景

- 环境状态变化超阈值时的场景切换判断
- 基于时间规律的主动场景预触发
- 用户行为信号（回家/离家/起床/睡觉）触发的场景联动
- 外部事件（天气突变/日历事件）驱动的场景调整
- 多条件组合触发的复杂场景推理

## 执行步骤

### 1. 触发条件评估

#### 触发信号来源：
| 信号类型 | 触发条件 | 阈值示例 |
|---------|---------|---------|
| 环境变化 | delta超过阈值 | 温度变化>2°C、光照变化>30% |
| 时间匹配 | 接近用户作息时间 | 距离起床时间<5分钟 |
| 人员变化 | 检测到入离 | 人体感应+门磁联动确认 |
| 天气突变 | 预报变化超阈值 | 2小时内降温>5°C |
| 日历事件 | 事件即将开始 | 距离事件开始<15分钟 |

#### 触发防抖机制：
- 环境变化触发需持续≥3分钟（避免瞬时波动）
- 人员进出触发需5分钟冷却期（避免门口徘徊误判）
- 同一场景30分钟内不重复触发（除非有明确的反向信号）

### 2. 场景识别推理

基于当前所有输入信号，LLM推理当前应处于哪个场景：

- **标准场景库**：起床、离家、回家、工作/学习、用餐、休闲娱乐、睡前放松、睡眠
- **组合场景**：回家+天热=回家制冷场景、起床+工作日=快速出门场景
- **自定义场景**：用户主动创建的个性化场景

推理决策树：
```
时间信号 + 人员信号 → 初步场景判定
→ + 环境信号 → 参数需求确认
→ + 偏好数据 → 个性化参数修正
→ + 能耗约束 → 最终方案生成
```

### 3. 设备参数优化计算

对于已识别的目标场景，计算各设备最优参数：

- **空调参数**：目标温度 = 用户偏好温度 ± 能耗修正 ± 人数修正
- **灯光参数**：亮度 = f(自然光, 时段, 活动类型) × 用户偏好系数
- **窗帘参数**：开合度 = f(日照角度, 室内温度, 隐私需求)
- **空气净化**：风速 = f(室内AQI, 室外AQI, 是否适合开窗)

多设备联动顺序优化：
1. 先调空调/暖气（影响体感的主设备）
2. 再调灯光/窗帘（即时感知的辅助设备）
3. 最后调空气净化/加湿（缓慢生效的环境设备）

### 4. 置信度评分计算

```
总置信度 = 数据质量权重(0.3) × 数据质量评分
          + 偏好匹配权重(0.4) × 偏好模型置信度
          + 场景确定性权重(0.3) × 场景识别确定性

场景识别确定性 = 触发信号数量和强度的函数
  - 单信号触发：0.6-0.7
  - 双信号互证：0.7-0.85
  - 三信号及以上互证：0.85-0.95
```

### 5. 推荐理由生成

为每个推荐方案生成人类可读的理由文本，用于：
- 置信度不足时向用户展示（辅助用户决策）
- 记录日志（支持后续分析和模型改进）
- 用户主动查询"为什么这样设置"时的解释

理由模板：
"检测到[触发信号]，根据您[偏好来源]的习惯，建议将[设备]调整为[参数]。预计[效果描述]。"

## 最佳实践

- 场景触发采用"多信号互证"原则——单一信号可提高警觉但不宜直接触发
- 新场景第一次触发时置信度主动降低0.1（保守策略），成功执行后恢复
- 推荐理由文本应简洁(不超过50字)、具体(包含关键数字)、有温度(用"您"而非"用户")
- 记录每次触发/未触发的决策过程，用于持续优化阈值参数

## 约束条件

- 单次推理延迟<2秒（用户感知的场景响应时间<3秒含执行）
- 置信度<0.6的方案绝不自动执行
- 安全相关设备（燃气阀/门锁）不纳入场景自动控制范围
- 用户明确处于"勿扰模式"时暂停所有非紧急场景触发
