# 冷启动策略 (Cold Start Strategy)

## 概述

本技能专门处理新用户/新设备/新场景加入时数据不足的问题，通过群体智慧默认值、渐进式学习和主动探索策略，在缺乏个性化数据时仍能提供合理的场景推荐，同时高效收集用户偏好数据以尽快完成冷启动过渡。

## 适用场景

- 新用户注册后的初始偏好配置
- 新设备接入后的默认参数设定
- 用户进入从未使用过的场景类型
- 系统重置或数据丢失后的恢复策略
- 新家庭成员加入时的快速偏好初始化

## 执行步骤

### 1. 冷启动状态判定
- **完全冷启动**：用户操作记录<3天，无任何偏好数据
- **部分冷启动**：用户操作记录3-7天，部分维度有数据但不充分
- **场景冷启动**：用户整体数据充足，但首次进入某个特定场景
- **设备冷启动**：新设备接入，无该设备的使用偏好历史

### 2. 默认值策略

#### 完全冷启动默认值来源（按优先级）：
1. **用户主动设置**：入住引导问卷（"您通常几点起床？""喜欢多少度的空调？"）
2. **人口统计默认值**：基于地区+季节+房屋类型的群体统计值
   - 夏季空调默认：南方26°C / 北方25°C
   - 冬季暖气默认：北方22°C / 南方20°C
   - 照明默认：白天4500K暖白 / 晚间3000K暖黄
3. **行业标准值**：GB/T 18049人体热舒适标准（PMV在[-0.5, 0.5]范围）
4. **保守安全值**：当以上均不可用时，使用最保守的节能设置

#### 渐进过渡策略：
- 第1-3天：100%使用默认值，收集用户反馈
- 第4-7天：70%默认值 + 30%已学习偏好（权重渐变）
- 第8-14天：30%默认值 + 70%已学习偏好
- 第15天起：完全切换为个性化模型（如置信度达标）

### 3. 主动探索策略
- **微调探索**：在默认值基础上做±1°C的微小变化，观察用户是否调整
- **时段探索**：不同时段尝试略有差异的参数，通过接受率学习时段偏好
- **场景提问**：在合适时机（如用户主动交互时）询问偏好问题
  - "现在温度舒服吗？"
  - "您希望睡觉时灯光怎样调整？"
- **探索频率限制**：每天最多2次主动提问，避免打扰

### 4. 快速学习加速
- 用户每次手动操作给予高权重（冷启动期α=0.5，高于正常期的0.2）
- 首次拒绝推荐的信息量大于首次接受（拒绝明确表达了"不喜欢"）
- 多维度联合学习：如果用户调高温度同时调暗灯光→推断"怕热+喜欢柔和光线"
- 相似用户迁移学习：如果同一家庭的其他成员已有成熟模型，新成员可参考（但不直接复制）

### 5. 冷启动退出判定
- 退出条件（需全部满足）：
  - 累计有效操作数据>=7天
  - 核心维度（温度+光照）置信度>=0.7
  - 最近3天推荐接受率>=70%
- 退出后动作：
  - 移除"学习中"标签
  - 将学习率从冷启动值(0.5)调整为正常值(0.2)
  - 生成"学习完成"通知告知用户

## 最佳实践

- 冷启动期间系统行为应保守——宁可让用户手动操作也不要做出让用户反感的自动推荐
- 入住引导问卷应简短（3-5个问题），不要让用户觉得系统"很麻烦"
- 冷启动状态对用户透明——让用户知道系统正在学习，而不是"不够智能"
- 家庭共用设备（如客厅空调）的冷启动优先参考家庭中最早注册成员的偏好

## 约束条件

- 冷启动期的自动执行置信度阈值提高至0.95（正常期0.85），减少误触发风险
- 探索性微调不超过标准偏差的1个单位（如温度±1°C，亮度±10%）
- 主动提问必须在用户已有交互上下文中进行，不在无交互时突然打断
- 完全冷启动期不超过14天——如14天后仍未达到退出条件，强制切换为最佳可用模型
