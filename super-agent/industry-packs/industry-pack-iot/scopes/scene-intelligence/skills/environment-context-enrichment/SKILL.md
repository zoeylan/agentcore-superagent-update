# 环境上下文增强 (Environment Context Enrichment)

## 概述

本技能负责集成外部数据源（天气预报、日出日落、空气质量指数、日历事件等），与室内传感器数据融合，构建完整的环境上下文信息。外部数据为场景推理提供了超越室内传感器能力的预见性信息，如即将到来的天气变化、日照角度预测等。

## 适用场景

- 集成气象API获取未来24小时天气预测
- 获取日出日落时间计算自然光变化趋势
- 拉取室外AQI指数辅助开窗/空气净化决策
- 融合日历事件上下文（如"即将有会议"影响场景切换）
- 获取峰谷电价时段信息供能耗决策参考

## 执行步骤

### 1. 外部数据源管理
- 维护外部数据源注册表（API端点/刷新频率/降级策略）
- 气象数据：每小时从天气API拉取一次，包含温度/湿度/风速/降水概率/AQI
- 天文数据：每日计算日出日落/民用曙暮光时间
- 电价数据：从电网公司API获取当日峰平谷时段（或使用本地规则表）
- 日历事件：从用户授权的日历服务获取当日事件列表

### 2. 数据可靠性评估
- 检查API响应时效性（超过2小时未更新的气象数据标记为stale）
- 对预测数据附带置信区间（如温度预测±2°C）
- API不可用时执行降级策略：
  - 气象数据：使用昨日同时段数据 + 季节性趋势修正
  - AQI数据：使用最后有效值 + "数据可能过时"标注
  - 日历数据：标记为不可用，不参与决策

### 3. 上下文关联构建
- 计算室内外温差（影响是否建议开窗通风）
- 计算当前时刻的自然光照射角度和强度预估
- 标注当前电价时段（峰/平/谷）和下一次切换时间
- 解析日历事件为场景标签（如"起床闹钟→起床场景即将触发"）
- 标注天气突变预警（如2小时内降温>5°C）

### 4. 上下文向量输出
- 将所有外部上下文打包为标准JSON附加到环境状态向量
- 标注每个数据项的来源、更新时间和可靠性等级

## 最佳实践

- 外部API调用做好限频和缓存，避免超额调用产生费用
- 天气预测数据的置信度随时间衰减——3小时后的预测仅供参考
- 日历事件解析应保守处理，无法确定的事件不强制映射场景标签
- 电价时段规则应支持用户手动配置（不同城市/不同电力公司规则不同）

## 约束条件

- 外部API调用必须使用HTTPS加密通道
- 用户日历数据仅提取时间和地点，不读取事件详细内容（隐私最小化）
- API密钥存储于安全区域，不在数据流中暴露
- 外部数据获取失败不得阻塞本地传感器数据的正常处理
