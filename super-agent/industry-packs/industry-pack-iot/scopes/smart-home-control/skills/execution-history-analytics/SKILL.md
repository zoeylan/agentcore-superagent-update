# 执行历史分析 (Execution History Analytics)

## 概述

本技能负责记录、存储和分析自动化规则的执行历史数据，为用户提供执行日志查询、统计报表、效果评估和优化建议。通过对历史数据的分析，帮助用户了解自动化规则的实际运行效果，发现低效规则并提出优化建议。

### 适用场景
- 用户查询某条规则的执行记录
- 规则执行效果的统计分析
- 低效/失败率高的规则识别和优化建议
- 规则使用率分析（发现从未触发的规则）
- 能耗影响评估（某规则带来的能耗变化）

## 执行步骤

### Step 1: 执行记录采集
1. 每次规则触发时记录完整的执行上下文：
```json
{
  "execution_id": "uuid",
  "rule_id": "规则ID",
  "trigger_time": "2024-01-15T18:00:00Z",
  "trigger_reason": "条件满足/定时触发/手动触发",
  "trigger_context": {
    "sensor_values": {"temperature": 28.5, "humidity": 65},
    "time_of_day": "evening",
    "user_presence": true
  },
  "actions_executed": [
    {"device": "air_conditioner", "action": "power_on", "result": "success", "latency_ms": 320},
    {"device": "curtain", "action": "close", "result": "timeout", "latency_ms": 500}
  ],
  "overall_result": "partial_success",
  "total_latency_ms": 520,
  "user_override_within_5min": false
}
```
2. 记录执行后5分钟内的用户手动操作（作为规则效果的间接反馈）
3. 数据持久化存储（保留90天详情，1年汇总统计）

### Step 2: 统计分析
1. **单规则统计**：
   - 触发频率（日/周/月）
   - 执行成功率（全部成功/部分成功/全部失败）
   - 平均执行延迟
   - 用户覆盖率（执行后被手动修改的比例）
   - 活跃时段分布（主要在哪些时间段触发）
2. **跨规则统计**：
   - 规则触发频率排行（TOP 10最活跃规则）
   - 失败率排行（TOP 10最不稳定规则）
   - 从未触发规则列表（创建超过30天未触发）
   - 规则间的关联触发模式
3. **系统级统计**：
   - 每日规则执行总量
   - 系统整体执行成功率
   - 设备负载分布（哪些设备被自动化控制最频繁）

### Step 3: 效果评估
1. 规则效果指标计算：
   - **有效率** = 1 - (用户5分钟内手动覆盖次数 / 总触发次数)
   - **节能效果** = 规则启用前后同期能耗对比
   - **便利性评分** = 规则替代的等效手动操作次数/月
2. 低效规则识别条件：
   - 有效率 < 60%（用户经常修改其结果）
   - 连续30天未触发（条件可能已过时）
   - 失败率 > 20%（设备可能有问题）
   - 触发后平均延迟 > 2秒（性能异常）

### Step 4: 优化建议生成
1. 基于分析结果生成规则优化建议：
   - 有效率低→建议调整操作参数（如温度设高了导致用户总是手动调低）
   - 从未触发→建议检查触发条件是否合理或删除规则
   - 高失败率→建议检查相关设备状态或更换为备选设备
   - 同类规则多→建议合并为一条复合规则
2. 推荐新规则：
   - 分析用户重复性手动操作（每天相同时间做相同操作3次以上）
   - 生成对应的自动化规则推荐
3. 定期报告（每周）发送给用户：规则运行概要+优化建议

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| query_type | enum | 是 | log_query/statistics/effectiveness/recommendation |
| rule_id | string | 否 | 指定规则ID（查询单规则时） |
| time_range | object | 否 | 时间范围（默认最近7天） |
| aggregation | enum | 否 | daily/weekly/monthly |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| execution_logs | array | 执行记录列表 |
| statistics | object | 统计结果 |
| effectiveness_score | float | 规则有效性评分(0-1) |
| recommendations | array | 优化建议列表 |
| weekly_report | object | 周报内容 |

## 最佳实践

1. **隐私保护**：分析数据脱敏处理，不暴露具体环境传感器原始值
2. **趋势检测**：关注指标趋势而非绝对值（效果率持续下降比当前略低更值得关注）
3. **上下文关联**：建议修改规则时附带触发失败的上下文案例帮助用户理解
4. **非打扰推送**：优化建议通过周报聚合推送，不做实时打扰
5. **A/B对比**：规则修改后与修改前同期数据对比，量化优化效果

## 约束条件

- 执行记录写入延迟 < 200ms
- 统计查询响应 < 2秒
- 详细记录保留90天，汇总统计保留1年
- 每日最大记录条数 10000条
- 优化建议每周最多生成5条（避免信息过载）
