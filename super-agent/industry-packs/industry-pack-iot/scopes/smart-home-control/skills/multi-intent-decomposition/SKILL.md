# 多意图拆分 (Multi-Intent Decomposition)

## 概述

本技能专门处理用户单条指令中包含多个操作目标的复杂场景。在实际使用中，用户经常会发出组合指令如"把客厅灯关了，空调调到26度，然后把窗帘拉上"，需要将其拆分为独立的可执行意图单元，并正确标注意图间的依赖关系。

### 适用场景
- 用户一次性发出多个设备控制指令
- 场景触发需要生成多个关联操作
- 复合条件指令（"如果...就...同时..."）
- 包含时序要求的连续操作

## 执行步骤

### Step 1: 多意图检测
1. 分析输入文本的句法结构，识别并列连词（和、还有、同时、然后、接着）
2. 检测多个动词短语（开、关、调、设）对应不同宾语（设备）
3. 识别隐含的多意图模式：
   - "回家模式"→展开为多个预定义操作
   - "关掉所有灯"→按设备清单展开
   - "跟昨天一样"→查询历史操作序列并复制

### Step 2: 意图边界切分
1. 使用句法分析确定每个独立意图的文本边界
2. 处理省略还原：「把空调和灯都关了」→空调关+灯关（操作共享）
3. 处理参数分配：「空调26度灯调暗」→空调set_param:26°C + 灯set_param:brightness_down
4. 确保每个拆分后的意图是自洽的独立操作单元

### Step 3: 依赖关系标注
1. 分析意图间的执行关系：
   - **并行(parallel)**：无依赖，可同时执行。如"开灯开空调"
   - **顺序(sequential)**：有时序要求。如"先关窗再开空调"
   - **条件(conditional)**：前置条件。如"如果温度超过28度就开空调"
2. 标注依赖关系图（DAG结构）
3. 检测循环依赖（不应存在），发现则报错

### Step 4: 拆分结果验证
1. 验证每个子意图的完整性（必须包含设备+操作）
2. 验证参数归属正确性（参数不会被错误分配到其他意图）
3. 验证拆分后意图数量合理（1-5个，超过5个需要确认）
4. 输出拆分后的意图数组及依赖关系图

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| parsed_text | string | 是 | 预处理后的文本 |
| detected_entities | array | 是 | 已识别的设备实体列表 |
| detected_actions | array | 是 | 已识别的操作动词列表 |
| scene_templates | object | 否 | 可用的场景模板映射表 |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| intent_count | integer | 拆分后的意图数量 |
| intents | array | 独立意图结构体列表 |
| dependency_graph | object | 意图间依赖关系（邻接表形式） |
| execution_order | array | 建议的执行顺序（拓扑排序结果） |

## 最佳实践

1. **保守拆分**：宁可少拆（保持一个较大意图）也不要错误拆分导致意图语义破坏
2. **语境一致性**：拆分后每个意图的上下文信息应继承原始输入的全局上下文
3. **操作共享识别**：「都关了」「全部打开」类表达中操作符应正确复制到每个目标设备
4. **时序词敏感**：「先...再...然后...」明确表示顺序依赖，「同时」「一起」表示并行
5. **数量控制**：单条用户指令拆分超过5个意图时很可能是场景触发，应查询场景模板

## 约束条件

- 拆分处理时间 < 100ms
- 多意图拆分完整率目标 ≥ 90%（所有该拆分的意图都被正确拆分）
- 误拆分率 < 2%（不该拆的被错误拆分）
- 最大支持拆分为5个独立意图
- 依赖关系图必须是有向无环图（DAG）
