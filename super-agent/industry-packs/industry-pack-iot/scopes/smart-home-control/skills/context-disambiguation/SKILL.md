# 上下文消歧 (Context Disambiguation)

## 概述

本技能处理用户指令中的模糊表达、指代引用和隐含信息，利用对话上下文、设备状态上下文、用户偏好和空间位置等多维信息源进行消歧。智能家居场景中用户表达极度口语化，如"把那个关了"、"有点热"、"调一下"等需要结合上下文才能准确理解的表达占比超过40%。

### 适用场景
- 代词指代消解（"那个"、"这个"、"它"指代哪个设备）
- 模糊参数理解（"高一点"、"暗一些"的具体数值映射）
- 省略主语补全（"关了"→关什么？）
- 隐含意图推断（"有点热"→降温操作）

## 执行步骤

### Step 1: 歧义点识别
1. 扫描解析后文本中的歧义标记：
   - 代词无明确先行词：「它」「那个」「这个」
   - 操作无明确对象：「关了」「开一下」
   - 参数无精确值：「高一点」「调低」「合适的温度」
   - 隐含意图表达：「好热」「太吵了」「看不清」
2. 为每个歧义点标记类型和所需的消歧信息类别

### Step 2: 多源上下文检索
1. **对话上下文**：检索最近3轮对话，查找可能的指代先行词
2. **操作上下文**：检索最近5分钟的设备操作记录，最近操作的设备最可能是指代目标
3. **空间上下文**：根据用户当前所在房间，缩小设备候选范围
4. **时间上下文**：根据当前时间段（早晨/白天/晚上/深夜）推断合理操作
5. **偏好上下文**：查询用户历史偏好（该用户说"合适温度"通常是几度）

### Step 3: 候选排序与选择
1. 对每个歧义点生成候选解释列表（最多5个）
2. 基于多源上下文为候选打分：
   - 对话近因性（最近提到的设备得分高）：权重 0.3
   - 空间相关性（同一房间的设备得分高）：权重 0.25
   - 操作频率（用户常操作的设备得分高）：权重 0.2
   - 时间合理性（当前时段合理的操作得分高）：权重 0.15
   - 语义匹配度（表达与设备类型的语义关联）：权重 0.1
3. 取最高分候选作为消歧结果

### Step 4: 置信度评估与决策
1. 计算消歧置信度：最高分与次高分的分差映射为置信度
2. 置信度 ≥ 0.8：直接采用消歧结果
3. 置信度 0.5-0.8：采用但标记为"低置信度"供下游审核
4. 置信度 < 0.5：触发澄清对话，提供top-2或top-3选项供用户选择
5. 澄清对话模板：「您是想{操作}{设备A}，还是{设备B}？」

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ambiguous_elements | array | 是 | 待消歧的元素列表 |
| dialog_history | array | 否 | 最近3轮对话记录 |
| recent_operations | array | 否 | 最近5分钟操作记录 |
| user_location | string | 否 | 用户当前房间 |
| user_preferences | object | 否 | 用户偏好模型 |
| device_states | object | 否 | 当前设备状态快照 |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| disambiguated_elements | array | 消歧后的元素列表 |
| disambiguation_method | string | 使用的消歧方法 |
| confidence | float | 消歧置信度 |
| needs_user_confirmation | boolean | 是否需要用户确认 |
| confirmation_prompt | string | 确认提示文本（需要确认时） |

## 最佳实践

1. **最近优先原则**：无其他强信号时，最近操作/提到的设备是最可能的指代目标
2. **空间约束优先**：用户说"开灯"时，先考虑当前房间的灯，而非全屋所有灯
3. **安全保守原则**：涉及安全设备的消歧必须走用户确认，即使置信度很高
4. **学习积累**：每次用户确认后记录消歧案例，逐步提高同类场景的准确率
5. **避免循环澄清**：如果两轮澄清后仍无法确定，提供最可能选项并告知不确定性

## 约束条件

- 消歧处理时间 < 150ms
- 消歧准确率目标 ≥ 88%（包含自动消歧和用户确认后）
- 澄清对话最多2轮
- 每次澄清提供2-3个选项（不超过3个避免选择困难）
- 不得在消歧过程中触发任何设备操作
