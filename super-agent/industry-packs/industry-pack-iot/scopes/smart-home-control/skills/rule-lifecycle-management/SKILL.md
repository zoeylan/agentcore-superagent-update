# 规则生命周期管理 (Rule Lifecycle Management)

## 概述

本技能负责自动化规则的完整生命周期管理，包括规则创建、验证、启用、执行、暂停、修改和删除。规则是用户将重复性操作固化为自动执行的核心手段，良好的规则管理直接决定了用户的智能家居自动化体验质量。

### 适用场景
- 用户创建新的定时任务或条件触发规则
- 场景推荐转化为永久自动化规则
- 规则冲突检测和优先级调整
- 规则启用/禁用状态切换
- 规则模板的应用和自定义

## 执行步骤

### Step 1: 规则创建与验证
1. 接收规则定义（用户配置/场景推荐/自然语言转规则）：
```json
{
  "rule_id": "uuid",
  "name": "下班回家模式",
  "trigger": {
    "type": "time|condition|event|compound",
    "expression": "cron(0 18 * * 1-5) AND sensor.presence == true"
  },
  "actions": [
    {"device": "living_room_light", "action": "power_on", "params": {"brightness": 80}},
    {"device": "air_conditioner", "action": "set_param", "params": {"temp": 25, "mode": "auto"}}
  ],
  "priority": 5,
  "enabled": true,
  "cooldown_minutes": 30
}
```
2. 规则验证清单：
   - 触发条件语法正确性（cron表达式/条件表达式合法）
   - 目标设备存在且在注册表中为active状态
   - 操作参数在设备能力范围内
   - 规则名称不重复
   - 活跃规则数未超限（200条上限）

### Step 2: 冲突检测
1. 将新规则与现有活跃规则进行冲突分析：
   - **时间重叠冲突**：两条规则可能在同一时刻触发且操作同一设备
   - **逻辑矛盾冲突**：两条规则对同一设备产生相反操作
   - **资源竞争冲突**：多条规则同时触发超过电力负载限制
2. 冲突处理策略：
   - 无冲突：直接通过
   - 有冲突但优先级明确：提示用户冲突信息和自动解决方案
   - 有冲突且无法自动解决：要求用户选择处理方式
3. 冲突解决选项：调整优先级/修改触发条件/设为互斥组/删除旧规则

### Step 3: 规则启用与执行
1. 通过验证的规则进入"活跃"状态
2. 时间触发规则注册到定时调度器
3. 条件触发规则注册到条件评估引擎
4. 事件触发规则注册到事件监听器
5. 规则首次执行记录（确认实际可执行）

### Step 4: 规则维护操作
1. **暂停/恢复**：临时禁用不删除（如旅行期间暂停家中规则）
2. **修改**：更新触发条件/操作/参数/优先级（需重新验证和冲突检测）
3. **删除**：永久移除（同时清理关联的执行历史和统计数据）
4. **批量操作**：按分组/场景批量启用/禁用
5. **规则导出/导入**：支持规则配置的备份和迁移

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operation | enum | 是 | create/update/delete/enable/disable/export |
| rule_definition | object | 否 | 规则完整定义（create/update时必填） |
| rule_id | string | 否 | 规则ID（update/delete/enable/disable时必填） |
| conflict_resolution | enum | 否 | 冲突解决策略选择 |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| operation_result | enum | success/validation_failed/conflict_detected |
| rule_id | string | 规则ID |
| validation_errors | array | 验证错误列表（如有） |
| conflicts | array | 冲突规则列表（如有） |
| suggested_resolution | string | 建议的冲突解决方案 |

## 最佳实践

1. **验证前置**：规则保存前完成所有验证，避免无效规则进入活跃库
2. **版本追踪**：规则每次修改保留历史版本，支持回滚到任意历史版本
3. **试运行模式**：新规则可设为"仅通知不执行"模式运行1周，确认无误后切换为自动执行
4. **关联分析**：删除规则前检查是否被其他规则引用（如规则A触发后禁用规则B）
5. **健康检查**：定期检查活跃规则中引用的设备是否仍然在线/存在

## 约束条件

- 每户活跃规则上限 200条
- 规则验证响应时间 < 500ms
- 冲突检测时间 < 1秒（与全量活跃规则对比）
- 规则状态切换即时生效（< 100ms）
- 规则执行历史保留90天
