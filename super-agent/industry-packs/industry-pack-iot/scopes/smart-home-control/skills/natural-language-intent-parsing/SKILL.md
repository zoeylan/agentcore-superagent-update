# 自然语言意图解析 (Natural Language Intent Parsing)

## 概述

本技能负责将用户的自然语言输入（语音ASR文本、键入文字、图片OCR结果）转换为标准化的设备控制意图结构。这是智能家居控制链路的第一环节，直接决定了整个系统对用户需求的理解准确度。

### 适用场景
- 用户通过语音助手发出设备控制指令
- 用户通过App文字输入框输入控制命令
- 用户通过图片（如设备面板截图）触发控制需求
- 系统接收到ASR引擎转写的语音文本

## 执行步骤

### Step 1: 输入预处理
1. 接收原始输入文本（含来源标记：voice/text/ocr）
2. 执行文本规范化：去除语气词（嗯、啊、那个）、纠正ASR常见错误（同音字替换）、统一标点符号
3. 识别输入语言/方言标记（普通话/粤语/四川话等），必要时进行方言转标准普通话
4. 提取上下文信息：当前时间、用户位置（哪个房间）、最近操作历史（最近3条）

### Step 2: 意图分类
1. 对预处理后的文本进行一级意图分类：
   - `control`: 设备控制操作（开/关/调节）
   - `query`: 状态查询（某设备当前是什么状态）
   - `setting`: 系统设置（修改设备别名/设置定时）
   - `cancel`: 取消操作（撤销上一步/停止执行）
2. 输出分类结果及置信度分数（0-1）
3. 置信度 < 0.7 时标记为需要澄清

### Step 3: 实体抽取
1. 识别目标设备实体：
   - 直接指称：「空调」「客厅灯」「卧室窗帘」
   - 别名指称：「小爱」（指空气净化器，用户自定义）
   - 位置推断：根据用户当前位置推断「灯」指哪个灯
2. 识别操作参数：
   - 精确值：「26度」「50%亮度」「制冷模式」
   - 相对值：「高一点」「调暗」「大一些」→映射为±步进值
   - 模糊值：「有点热」「太暗了」→根据偏好模型映射为具体参数
3. 识别时间限定：「5分钟后」「明天早上」「睡觉前」

### Step 4: 意图结构化输出
1. 组装标准化意图结构体：
```json
{
  "intent_id": "uuid",
  "action_type": "power_on|power_off|set_param|query_status",
  "target_device": "device_id",
  "parameters": {"key": "value"},
  "confidence": 0.95,
  "security_sensitive": false,
  "source_text": "原始文本片段",
  "context": {"room": "客厅", "time": "2024-01-01T20:00:00"}
}
```
2. 对安全敏感设备（门锁/摄像头/燃气阀门）标记 `security_sensitive: true`
3. 输出最终意图结构或澄清请求

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| raw_text | string | 是 | 用户原始输入文本 |
| input_source | enum | 是 | voice/text/ocr |
| user_id | string | 是 | 用户标识 |
| session_context | object | 否 | 对话上下文（最近3轮） |
| current_room | string | 否 | 用户当前所在房间 |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| intents | array | 结构化意图列表 |
| needs_clarification | boolean | 是否需要澄清 |
| clarification_options | array | 澄清选项（需要澄清时提供） |
| parse_confidence | float | 整体解析置信度 |

## 最佳实践

1. **上下文优先**：相同表达在不同上下文下含义不同，「关了」可能指最近操作的设备
2. **渐进式消歧**：先用高置信度规则过滤，再用LLM处理模糊案例，避免所有请求都走大模型推理
3. **用户习惯记忆**：记录用户的表达习惯（如某用户说「灯」永远指卧室灯），提高后续解析准确率
4. **错误容错**：ASR转写错误率约5-10%，需要建立同音字/近音字纠错词典
5. **安全兜底**：任何涉及安全设备的操作，即使置信度高也要标记敏感标志

## 约束条件

- 单次解析响应时间 < 200ms（P99 < 500ms）
- 意图分类准确率目标 ≥ 95%
- 实体抽取F1分数目标 ≥ 92%
- 最多支持单条输入拆分为5个独立意图
- 不得自行执行任何设备控制操作
