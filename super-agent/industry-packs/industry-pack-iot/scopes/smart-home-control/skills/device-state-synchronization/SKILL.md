# 设备状态同步 (Device State Synchronization)

## 概述

本技能负责维护全屋设备的实时状态镜像（Device Shadow），通过设备心跳监听和状态变更事件处理，确保系统内的设备状态数据与物理设备的实际状态保持一致。状态同步延迟控制在1秒以内，是整个智能家居系统做出正确决策的数据基础。

### 适用场景
- 设备主动上报状态变更（如用户通过物理按键操作设备）
- 设备心跳超时判定设备离线
- 系统启动时的全量状态同步
- 控制指令执行后的状态确认更新
- 外部系统查询设备当前状态

## 执行步骤

### Step 1: 状态数据采集
1. **心跳监听**：
   - WiFi设备：每30秒一次心跳包，含设备基本状态
   - Zigbee设备：每60秒一次心跳（通过网关汇报）
   - BLE设备：按需连接时同步状态
   - Matter设备：通过Subscription机制实时接收
2. **事件监听**：
   - 设备状态变更事件（用户物理操作/自身状态切换）
   - 告警事件（故障码/异常状态）
   - 设备上线/下线事件
3. **主动查询**：
   - 心跳间隔内需要最新状态时主动Query
   - 控制指令下发后的确认查询

### Step 2: 状态镜像更新
1. 接收到状态数据后进行校验：
   - 时间戳有效性（不接受过时数据覆盖新数据）
   - 数据格式校验（类型/范围/完整性）
   - 序列号连续性（检测是否有丢失事件）
2. 更新Device Shadow中对应设备的状态字段：
   - `online_status`：在线/离线/休眠
   - `running_params`：当前运行参数键值对
   - `last_heartbeat`：最后心跳时间
   - `last_updated`：最后状态更新时间
   - `health_score`：设备健康度评分（0-100）
3. 状态变更时生成变更事件（含before/after对比）
4. 变更事件发布到事件总线供订阅方消费

### Step 3: 在线状态判定
1. 维护每台设备的心跳超时计时器
2. 超时判定规则：
   - WiFi设备：90秒（3个心跳周期）未收到心跳 → 标记离线
   - Zigbee设备：180秒（3个心跳周期）未收到心跳 → 标记离线
   - BLE设备：最后连接断开后 → 标记为待连接状态
3. 设备重新上线时：
   - 更新online_status为在线
   - 执行全量状态同步（设备可能在离线期间被物理操作过）
   - 生成设备重新上线事件

### Step 4: 状态查询服务
1. 提供实时状态查询API（响应<100ms）：
   - 单设备状态查询
   - 房间设备批量状态查询
   - 全屋设备状态快照
   - 条件过滤查询（如所有离线设备）
2. 提供状态变更历史查询（最近30天）：
   - 按设备查询
   - 按时间范围查询
   - 按变更类型查询

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| event_type | enum | 是 | heartbeat/state_change/alert/query |
| device_id | string | 是 | 设备唯一标识 |
| payload | object | 是 | 状态数据或查询条件 |
| timestamp | long | 是 | 事件时间戳(ms) |
| sequence_no | long | 否 | 事件序列号 |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| device_shadow | object | 设备当前完整状态镜像 |
| state_changed | boolean | 状态是否有变更 |
| change_detail | object | 变更详情（before/after） |
| published_events | array | 已发布的变更事件列表 |

## 最佳实践

1. **最终一致性**：短暂不一致可接受，但必须在1秒内收敛到一致状态
2. **版本号机制**：每次状态更新递增版本号，防止乱序更新
3. **增量同步**：正常情况下只同步变更字段，避免全量传输的带宽浪费
4. **离线缓冲**：设备重新上线时批量同步离线期间积压的状态变更
5. **读写分离**：高频读取走缓存，写入触发缓存更新，避免读写竞争

## 约束条件

- 状态同步延迟 < 1秒（从设备变更到镜像更新）
- 状态查询响应 < 100ms
- Device Shadow存储保留最近30天历史
- 单个Device Shadow大小 < 10KB
- 事件总线吞吐量 ≥ 1000事件/秒
