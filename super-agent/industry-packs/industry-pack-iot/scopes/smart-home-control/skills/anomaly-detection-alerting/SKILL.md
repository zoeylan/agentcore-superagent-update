# 异常检测与告警 (Anomaly Detection & Alerting)

## 概述

本技能负责实时监控设备运行数据，通过规则引擎和异常模式检测识别设备故障、安全风险和异常行为，并按分级策略触发告警通知。这是保障智能家居安全运行的关键防线，需要在灵敏度（不漏报）和特异性（不误报）之间取得平衡。

### 适用场景
- 设备离线超时检测
- 设备异常功耗监测
- 传感器数据越界告警
- 设备故障码解析与处理
- 安全相关传感器触发（烟雾/燃气/水浸）
- 异常用电模式识别

## 执行步骤

### Step 1: 数据流监控
1. 持续接收设备状态变更事件流
2. 维护每台设备的关键指标时间序列：
   - 功率消耗（实时瓦数）
   - 运行温度（设备自身温度）
   - 通信质量（心跳丢包率/响应延迟）
   - 传感器读数（环境数据类设备）
3. 对每个指标维护滑动窗口统计（均值/标准差/趋势）

### Step 2: 规则引擎评估
1. **阈值规则**（实时评估）：
   - 功率超过额定值120%持续60秒 → 过载告警
   - 设备表面温度超过安全阈值 → 过热告警
   - 传感器读数超出物理合理范围 → 传感器故障
   - 烟雾/燃气/水浸传感器触发 → 安全紧急告警
2. **模式规则**（滑动窗口评估）：
   - 设备功耗持续上升且偏离历史均值2σ → 疑似故障
   - 心跳丢包率超过20% → 通信质量下降告警
   - 设备频繁重启（1小时内>3次）→ 硬件异常
3. **关联规则**（跨设备评估）：
   - 同一回路多设备异常 → 疑似电路问题
   - 多个温度传感器同时急剧变化 → 环境异常事件

### Step 3: 告警分级与通知
1. 告警分级定义：
   - **紧急(P0)**：人身/财产安全风险。如燃气泄漏、电器过热冒烟、水浸漫延
     - 处置：立即推送（电话+短信+APP）、自动执行安全预案（关闭燃气阀/断电）
   - **重要(P1)**：影响正常使用。如核心设备离线、空调压缩机故障
     - 处置：即时APP推送、建议检修措施
   - **提示(P2)**：需要关注。如滤网需更换、固件可升级、设备运行时间过长
     - 处置：汇入每日摘要、不打扰式提醒
2. 告警去重：相同设备相同告警10分钟内不重复推送
3. 告警升级：P1告警30分钟未处理自动升级为P0

### Step 4: 告警记录与闭环
1. 记录每条告警的完整生命周期：
   - 触发时间 → 通知时间 → 确认时间 → 解决时间
2. 支持告警确认（acknowledge）和解决（resolve）操作
3. 未解决告警的定期提醒（P1每2小时、P2每天）
4. 告警统计报表（周度/月度）：各级别数量/平均响应时间/TOP故障设备

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| device_id | string | 是 | 设备标识 |
| metric_name | string | 是 | 指标名称 |
| metric_value | float | 是 | 指标当前值 |
| timestamp | long | 是 | 数据时间戳 |
| device_spec | object | 否 | 设备规格参数（额定功率等） |

## 输出规范

| 字段 | 类型 | 说明 |
|------|------|------|
| alert_triggered | boolean | 是否触发告警 |
| alert_level | enum | P0/P1/P2 |
| alert_type | string | 告警类型编码 |
| description | string | 人类可读的告警描述 |
| suggested_action | string | 建议处置措施 |
| auto_action_taken | object | 已自动执行的安全预案（如有） |

## 最佳实践

1. **安全优先**：安全类告警（P0）永远不做延迟、不做去重、不做抑制
2. **避免告警疲劳**：P2级别告警合并推送，避免高频提醒导致用户忽视重要告警
3. **基线自适应**：设备功耗基线应随季节（空调夏季功耗高于春秋）动态调整
4. **关联分析**：多设备同时异常时优先排查公共原因（电路/网络）而非逐个告警
5. **历史对比**：新告警触发时展示该设备历史同类告警的频率和解决方案

## 约束条件

- P0告警检测到推送延迟 < 5秒
- P1告警检测到推送延迟 < 10秒
- 异常检测规则评估周期 ≤ 5秒
- 告警误报率 < 5%（P0级别 < 1%）
- 告警漏报率 < 1%（P0级别 0%容忍）
- 安全预案自动执行延迟 < 3秒
