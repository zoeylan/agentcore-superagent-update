# 智能家居控制 — 标准操作流程 (SOP)

## 1. 概述

本SOP定义了智能家居控制域从用户指令接收到设备执行反馈的全流程标准操作规范。涵盖意图解析、设备调度、状态同步、异常处理和自动化规则执行五大核心流程，确保设备控制端到端响应时间<500ms（P99<800ms），意图识别准确率≥95%，多设备并发控制成功率≥99%。

---

## 3. 核心流程 — 用户指令控制流

### 3.1 意图解析阶段

#### 触发条件
- 用户通过语音/文字/图片发出设备控制相关的输入
- 其他业务域（场景智能/车家互联）转发的控制请求

#### 操作步骤

**Step 1: 输入接收与预处理**
- 接收原始输入，标记来源类型（voice/text/ocr/api）
- ASR文本执行后处理：同音字纠错、语气词去除、标点规范化
- 记录请求时间戳（用于端到端延迟统计的起点）

**Step 2: 意图分类**
- 执行一级分类：control / query / setting / cancel
- 分类置信度 ≥ 0.7 → 继续
- 分类置信度 < 0.7 → 进入澄清流程

**Step 3: 实体抽取与消歧**
- 抽取设备实体、操作类型、参数值
- 存在歧义时按消歧优先级（对话上下文 > 空间位置 > 操作频率 > 时间合理性）排序候选
- 消歧置信度 < 0.5 → 触发用户确认（最多2轮）

**Step 4: 多意图拆分**
- 检测输入中是否包含多个操作目标
- 拆分为独立意图单元，标注依赖关系
- 拆分后每个意图独立验证完整性

#### 输出
- 结构化意图列表（含设备ID、操作类型、参数、置信度、安全标记）
- 或：澄清请求（含选项列表）

#### 异常处理
- ASR转写质量极差（识别率<50%）：请求用户重新说一遍，建议靠近麦克风
- 无法识别任何设备实体：提示"我没有找到您要控制的设备，请说明具体设备名称"
- 2轮澄清仍无法确定：结束对话，提示更明确的表达方式示例

#### 质量检查点
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 意图分类准确率 | ≥ 95% | 每周采样100条人工标注对比 |
| 实体抽取F1 | ≥ 92% | 每周采样评估 |
| 多意图拆分完整率 | ≥ 90% | 含多意图样本的召回率 |
| 解析响应时间 | P50<100ms, P99<300ms | 系统监控 |

### 3.2 设备调度执行阶段

#### 触发条件
- 意图解析引擎输出结构化意图列表
- 自动化任务引擎触发规则执行
- 场景智能域/车家互联域发送的控制请求

#### 操作步骤

**Step 1: 接入鉴权**
- 验证请求来源合法性
- 检查安全敏感标记：`security_sensitive: true` → 触发身份验证
- 确认操作权限（该来源是否有权控制目标设备）

**Step 2: 设备可达性检查**
- 查询设备注册表获取设备元信息
- 通过状态同步监控器确认设备在线状态
- 设备离线处理：
  - 返回设备不可用信息（含最后在线时间）
  - 检查同类备选设备是否可用
  - 建议用户检查设备网络/电源

**Step 3: 指令冲突检测**
- 检测当前意图列表中的内部冲突（同一设备矛盾操作）
- 检测与正在执行中指令的冲突
- 冲突优先级规则：安全 > 用户实时 > 场景推荐 > 自动化 > 远程
- 无法自动解决的冲突 → 请求用户确认

**Step 4: 参数映射与协议转换**
- 查询设备能力模型，验证参数范围
- 参数越界 → 自动clip到最近合法值 + 通知用户
- 通用参数 → 设备特定编码格式
- 选择通信协议和通道

**Step 5: 指令下发与确认**
- 并行/串行策略执行（按依赖图）
- 每条指令等待ACK（超时500ms）
- 超时 → 重试1次（相同序列号）
- 重试仍失败 → 标记失败，继续其他不受影响的指令

**Step 6: 结果汇总与反馈**
- 汇总所有设备的执行结果
- 生成用户可理解的执行摘要
- 全部成功："已帮您完成XX操作"
- 部分失败："XX已完成，但YY设备暂时无法响应"

#### 输出
- 执行结果摘要（成功/部分成功/失败）
- 状态更新事件（发送给状态同步监控器）
- 执行日志（记录完整执行轨迹）

#### 异常处理
- 全部设备离线：通知用户检查家庭网络/网关状态
- 批量执行部分失败：保留成功操作，汇报失败设备列表和原因
- 指令冲突用户未响应（30秒超时）：取消本次操作，保持现状
- 设备ACK后实际未执行（状态未变更）：标记为设备故障，通知维护

#### 质量检查点
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 端到端响应时间 | P50<300ms, P99<800ms | 从收到意图到返回结果 |
| 指令格式校验通过率 | 100% | 自动校验 |
| 设备ACK接收率 | ≥ 99.5% | 系统监控 |
| 首次执行成功率 | ≥ 98% | (成功+部分成功)/总请求 |

### 3.3 状态同步与异常检测阶段

#### 触发条件
- 设备心跳事件（周期性）
- 设备状态变更事件（事件驱动）
- 控制指令执行后的状态确认
- 异常检测规则的定时评估

#### 操作步骤

**Step 1: 状态数据接收与更新**
- 接收设备心跳/状态变更事件
- 时间戳验证（拒绝过时数据）
- 更新Device Shadow对应字段
- 发布状态变更事件到事件总线

**Step 2: 在线状态监控**
- 维护每设备心跳超时计时器
- WiFi设备90秒 / Zigbee设备180秒 未心跳 → 标记离线
- 设备重新上线 → 全量状态同步 + 上线事件通知

**Step 3: 异常检测**
- 实时阈值规则评估（功耗/温度/数据范围）
- 滑动窗口模式检测（趋势异常/周期异常）
- 跨设备关联分析（公共原因排查）

**Step 4: 告警分级与推送**
- P0（紧急）：即时推送 + 自动安全预案
- P1（重要）：即时APP推送
- P2（提示）：汇入每日摘要

#### 输出
- 更新后的Device Shadow
- 状态变更事件流
- 分级告警通知

#### 异常处理
- 大面积设备离线（>30%）：判断为网关/网络故障，发送网络级告警而非逐设备告警
- 传感器数据全零：标记传感器故障，使用最后有效值+过时标记
- 告警风暴（1分钟内>20条同类告警）：触发告警聚合，合并推送

#### 质量检查点
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 设备离线检测时间 | < 30秒 | 从真实离线到系统判定 |
| 状态同步延迟 | < 1秒 | 设备变更到Shadow更新 |
| 告警推送延迟 | P0<5秒, P1<10秒 | 检测到异常到通知送达 |
| 异常检测准确率 | ≥ 95% | 采样人工复核 |
| 故障自动恢复率 | ≥ 80% | 自动恢复成功/所有故障 |

### 3.4 自动化规则执行阶段

#### 触发条件
- 定时任务到达执行时间
- 条件表达式评估结果从false变为true且通过防抖验证
- 事件触发条件满足

#### 操作步骤

**Step 1: 触发确认**
- 验证规则仍处于启用状态（可能在防抖期间被用户禁用）
- 验证未处于冷却期（距上次触发>冷却时间）
- 创建执行上下文快照（触发时的环境数据）

**Step 2: 操作指令生成**
- 读取规则定义的action列表
- 将action转换为标准化意图格式
- 标记意图来源为 `automation`
- 检查规则优先级（与可能的并发请求比较）

**Step 3: 提交执行**
- 将意图列表发送给设备调度控制器
- 等待执行结果返回
- 记录执行详情到执行历史

**Step 4: 结果评估与记录**
- 记录执行成功/失败状态
- 执行失败时：不自动重试（避免规则循环），记录失败原因
- 监测5分钟内用户是否手动覆盖（作为规则效果反馈信号）
- 更新规则统计数据（累计触发次数/成功率）

#### 输出
- 执行结果记录
- 规则统计更新
- 用户通知（可配置）

#### 异常处理
- 规则引用的设备已被删除：自动禁用规则 + 通知用户
- 规则执行造成其他规则连锁触发（循环检测）：中断执行链 + 标记规则冲突
- 大量规则同时触发（>10条/秒）：排队执行，按优先级排序

#### 质量检查点
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 定时任务准时率 | 误差<1秒 | 系统时钟对比 |
| 条件评估延迟 | <50ms/规则 | 性能监控 |
| 规则执行成功率 | ≥ 98% | 执行历史统计 |
| 误触发率 | < 2% | 用户报告+日志审计 |

## 4. 决策树

```
用户输入到达
│
├─ 意图解析
│   ├─ 置信度 ≥ 0.7?
│   │   ├─ YES → 输出结构化意图
│   │   └─ NO → 发起澄清（最多2轮）
│   │       ├─ 澄清成功 → 输出结构化意图
│   │       └─ 澄清失败 → 告知用户无法理解，终止
│   │
│   └─ 安全敏感操作?
│       ├─ YES → 标记security_sensitive
│       └─ NO → 正常流转
│
├─ 设备调度
│   ├─ 设备在线?
│   │   ├─ YES → 继续
│   │   └─ NO → 通知用户+建议备选
│   │
│   ├─ 存在指令冲突?
│   │   ├─ YES → 安全冲突?
│   │   │   ├─ YES → 安全操作优先执行
│   │   │   └─ NO → 请求用户确认
│   │   └─ NO → 继续
│   │
│   ├─ 参数在范围内?
│   │   ├─ YES → 继续
│   │   └─ NO → Clip到边界值+通知
│   │
│   └─ 设备ACK?
│       ├─ YES → 执行成功
│       └─ NO → 重试1次
│           ├─ 重试成功 → 执行成功
│           └─ 重试失败 → 标记失败+通知用户
│
├─ 状态更新
│   ├─ 异常检测触发?
│   │   ├─ P0紧急 → 立即推送+安全预案
│   │   ├─ P1重要 → 即时推送
│   │   └─ P2提示 → 汇入摘要
│   └─ 正常 → 更新Shadow
│
└─ 自动化（并行路径）
    ├─ 条件满足?
    │   ├─ YES → 防抖验证
    │   │   ├─ 持续满足 → 触发规则执行
    │   │   └─ 中途不满足 → 重置，不触发
    │   └─ NO → 继续监听
    └─ 冷却期内?
        ├─ YES → 跳过本次触发
        └─ NO → 正常触发
```

## 7. 安全操作规范

### 7.1 敏感设备操作流程
1. 门锁/摄像头/燃气阀门等操作 **必须** 经过身份验证
2. 身份验证方式：生物识别（指纹/人脸）> PIN码 > APP确认
3. 验证超时30秒，超时取消操作
4. 连续3次验证失败锁定该设备操作15分钟

### 7.2 安全预案自动执行规则
- 烟雾报警器触发 → 关闭燃气阀门 + 打开所有通风设备 + 通知所有家庭成员
- 燃气泄漏检测 → 关闭燃气阀门 + 禁止所有可能产生火花的设备操作 + 紧急通知
- 水浸传感器触发 → 关闭进水阀门 + 通知用户
- 设备过热告警 → 切断该设备电源 + 通知用户

### 7.3 异常用电处理
- 单设备功率超额定120%持续60秒 → P1告警
- 单设备功率超额定150% → P0告警 + 自动断电
- 全屋功率接近总开关限额80% → P2提示
- 全屋功率超过总开关限额90% → P1告警 + 建议关闭非必要设备
