# 隐私权限管理

## 概述
本技能管理老人与家属之间的数据访问权限和隐私边界，确保老人的尊严和自主权不被过度监控侵犯，同时满足家属的合理关心需求。核心理念是「保护而非监控」——让老人感到被关心而非被监视。

## 适用场景
- 老人配置/修改自己的隐私偏好
- 家属请求查看老人数据时的权限校验
- 家属发起远程设备控制时的权限检查
- 新家属加入系统时的权限初始化
- 定期权限审计和异常访问检测

## 执行步骤

### 步骤1：权限模型维护
- 管理三级权限角色：管理员（通常为主要子女）/成员/访客
- 维护数据类型分类：
  - 紧急安全数据（跌倒/心率骤变）→ 所有角色可接收
  - 健康趋势数据（血压/睡眠）→ 管理员+成员
  - 生活行为数据（作息/活动量）→ 仅管理员
  - 实时位置/视频 → 需老人单独授权
- 维护设备控制权限：哪些设备允许远程控制/查看

### 步骤2：老人自主配置
- 提供简化的语音配置界面（"我不想让他们看到我几点睡觉"）
- 支持一键模板选择：
  - 宽松模式：仅紧急异常通知
  - 标准模式：紧急+重要异常通知
  - 详细模式：所有异常+定期报告
- 随时可通过语音/大字体界面修改设置
- 配置变更立即生效，无需审批

### 步骤3：实时权限校验
- 每次数据访问/推送前进行权限检查
- 校验流程：请求者身份 → 角色确认 → 数据类型匹配 → 老人配置检查 → 放行/拒绝
- 被拒绝的请求记录日志但不通知老人（避免造成家庭矛盾）
- 敏感设备操作（摄像头/门锁）额外验证

### 步骤4：透明机制
- 老人可查看所有家属的访问记录（"谁看过我的什么信息"）
- 远程控制操作记录对老人可见（"您的女儿刚才帮您调了空调温度"）
- 权限变更通知老人确认（如家属请求提升权限级别）

### 步骤5：定期审计
- 每月自动生成权限审计报告
- 检测异常访问模式（如频繁查看老人位置）
- 访客权限自动过期（默认24小时）
- 长期未使用的家属账号建议降权

## 输入规格
```json
{
  "request_type": "access_check|config_update|audit",
  "requester": {
    "id": "family_member_001",
    "name": "李女士",
    "role": "admin",
    "relation": "女儿"
  },
  "target_data": {
    "type": "health_trend|behavior|realtime_location|device_status",
    "specific_fields": ["sleep_schedule", "daily_steps"]
  },
  "elder_privacy_config": {
    "mode": "standard",
    "custom_blocks": ["detailed_activity_timeline"],
    "last_updated": "2024-01-10T10:00:00"
  }
}
```

## 输出规格
```json
{
  "access_granted": true,
  "filtered_fields": ["detailed_activity_timeline"],
  "accessible_data": ["sleep_quality_score", "daily_steps_summary"],
  "audit_log": {
    "requester": "李女士",
    "action": "view_health_trend",
    "result": "partial_access",
    "filtered_reason": "elder_privacy_config",
    "timestamp": "2024-01-15T15:00:00"
  }
}
```

## 最佳实践
- 默认采用「最小必要」原则——不确定时倾向于保护隐私
- 对老人的配置操作提供耐心引导，不因操作慢而催促
- 权限冲突时老人意愿优先于家属请求（紧急安全类除外）
- 新用户首次配置时使用「标准模式」作为默认
- 避免让老人感到「被管控」——界面措辞使用「分享」而非「允许查看」

## 约束条件
- 数据访问权限100%按配置执行，不可绕过
- 权限校验延迟 < 100ms（不影响用户体验）
- 老人配置变更实时生效（< 1秒）
- 紧急安全数据不受隐私配置限制（生命安全优先）
- 所有访问日志保留至少90天
- 不可向家属暴露老人的权限配置详情（如"他屏蔽了你"）
