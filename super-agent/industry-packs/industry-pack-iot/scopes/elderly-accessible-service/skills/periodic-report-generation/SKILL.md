# 周期性报告生成

## 概述
本技能负责自动生成面向家属的日报/周报/月报，汇总老人的健康数据、生活规律、设备使用情况、异常事件等信息，以结构化且易读的方式呈现，让家属全面了解老人的近期状况而无需实时监控。

## 适用场景
- 每日晨间推送前一天的简要日报
- 每周日推送本周综合周报
- 每月1日推送上月健康月报
- 家属主动请求查看某时段报告
- 异常事件后的专项事件报告

## 执行步骤

### 步骤1：数据汇总采集
- 健康数据：心率均值/血压趋势/血氧/睡眠时长与质量/步数
- 行为数据：作息规律（起床/就寝时间）/外出次数/用水用电
- 用药数据：服药依从率/漏服次数/确认延迟统计
- 设备使用：常用设备/交互次数/交互成功率
- 异常事件：本期发生的各级异常及处理结果

### 步骤2：趋势分析
- 与上一周期对比（环比变化）
- 与历史基线对比（是否在正常范围内）
- 标注显著变化（正面用绿色标记，负面用橙色标记）
- 生成简洁的趋势描述（如"本周步数较上周增加15%"）

### 步骤3：报告结构化生成
- 日报结构：整体状态评分(emoji) + 关键指标 + 异常标注（如有）
- 周报结构：
  - 本周健康评分（1-100分）
  - 关键指标趋势图描述
  - 亮点（积极变化）
  - 关注点（需要注意的指标）
  - 用药依从性统计
  - 异常事件回顾（如有）
- 月报结构：在周报基础上增加长期趋势分析和医疗建议提示

### 步骤4：隐私过滤
- 根据老人隐私配置过滤不允许推送的数据类型
- 根据接收者权限级别调整报告详细程度
- 管理员可见完整报告，普通成员仅见摘要

### 步骤5：报告发送
- 按配置的推送时间发送（如每日8:00、每周一9:00）
- 通过APP推送发送（含可视化卡片）
- 支持导出PDF（供就医时展示给医生）
- 记录发送和已读状态

```json
{
  "report_type": "daily|weekly|monthly|event",
  "period": {
    "start": "2024-01-08",

  },
  "data_sources": {
    "health_metrics": {"心率/血压/血氧等汇总数据"},
```

## 最佳实践
- 报告中必须包含积极信息（不能只报告问题，要让家属安心）
- 使用直观的表述而非纯数字（"血压略偏高"优于"150/95mmHg"）
- 异常事件在报告中仅简述，详情引导查看专项报告
- 对无异常的日子简化日报（"今天一切正常😊"即可）
- 月报中提供可打印的健康数据汇总页，便于就医携带

## 约束条件
- 日报/周报/月报自动生成率100%
- 报告推送延迟不超过配置时间的5分钟
- 不可在报告中包含医疗诊断或治疗建议
- 隐私过滤100%执行
- 报告数据必须准确（与原始数据一致，不可有统计错误）
- PDF导出包含完整周期数据（不因隐私过滤而截断自留存档）
