# 分级通知策略

## 概述
本技能实现智能化的异常事件分级通知机制，根据事件严重程度（紧急/重要/轻度）选择不同的通知渠道、通知对象和通知频率，避免过度通知导致的「狼来了」效应，确保家属对告警保持足够的重视度。

## 适用场景
- 紧急事件通知（跌倒/心率骤变/紧急呼叫）
- 重要事件通知（作息持续异常/用药未确认/生活规律偏移）
- 轻度事件汇总（日常观察/活动量变化/睡眠质量波动）
- 定期报告推送（日报/周报/月报）
- 家属主动查询老人状态的响应

## 执行步骤

### 步骤1：事件分级判定
- 接收异常事件输入，确认事件级别
- 紧急：直接生命安全威胁 → 全通道立即通知
- 重要：需要关注但非即刻危险 → APP推送+可选短信
- 轻度：供参考的观察信息 → 汇总至日报/周报

### 步骤2：通知对象确定
- 查询家属通知配置表（谁接收什么级别的通知）
- 紧急事件：通知所有已配置的紧急联系人
- 重要事件：仅通知主要家属（配置为"重要"接收人的成员）
- 轻度事件：仅对订阅了日报/周报的家属推送
- 尊重老人隐私配置（过滤老人不允许推送的数据类型）

### 步骤3：通知内容生成
- 紧急通知：简洁明确+行动指引
  - 示例："【紧急】您的父亲张XX于14:30在客厅跌倒，已拨打120，请立即回电。"
- 重要通知：现象描述+趋势+建议
  - 示例："张爷爷连续2天起床时间比平时晚2小时以上，可能需要关注。"
- 周报：数据汇总+趋势图+异常标注+积极信息
  - 包含：本周健康评分/活动量趋势/睡眠质量/用药依从性

### 步骤4：通知发送与回执
- 紧急通知：电话(优先)+短信(同步)+APP推送(同步)
- 重要通知：APP推送(主)+短信(备，若24小时APP未读)
- 轻度汇总：仅APP推送
- 记录通知发送状态和已读/回执状态

### 步骤5：通知效果评估
- 追踪家属响应时间（紧急通知目标<5分钟回应）
- 监控通知已读率（重要通知24小时已读率应>=90%）
- 统计家属对通知的操作响应率（是否采取行动）
- 动态调整策略（如某家属对APP推送响应差，升级为短信）

## 输入规格
```json
{
  "event": {
    "level": "emergency|important|mild",
    "category": "fall|vital_signs|routine_change|medication|activity",
    "description": "事件描述",
    "data": {"详细数据"},
    "timestamp": "2024-01-15T14:30:00"
  },
  "notification_config": {
    "family_members": [
      {
        "name": "李女士",
        "relation": "女儿",
        "receive_levels": ["emergency", "important"],
        "channels": ["phone", "sms", "app"],
        "is_primary": true
      }
    ],
    "elder_privacy_settings": {
      "allowed_data_types": ["emergency", "medication", "routine"],
      "blocked_data_types": ["detailed_activity"]
    }
  }
}
```

## 最佳实践
- 紧急通知宁多勿少，重要通知适度精准，轻度通知宁少勿多
- 同一类型重要事件24小时内不重复推送（除非升级为紧急）
- 周报中包含积极信息（如"本周步数比上周增加了10%"），而非只报告问题
- 节假日/深夜时段非紧急通知延迟至合适时间推送
- 新家属加入时发送一次通知分级说明，帮助理解不同级别的含义

## 约束条件
- 紧急通知延迟 < 10秒
- 重要通知延迟 < 5分钟
- 日常无异常时家属推送为零（严格执行）
- 通知分级准确率 >= 90%
- 隐私过滤100%执行（不可泄露老人不愿共享的数据）
- 每月误报导致的紧急通知不超过2次
