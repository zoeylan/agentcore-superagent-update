# 老年友好语音播报反馈

## 概述
本技能负责将系统操作结果、提醒通知、引导确认等信息转换为适合老年人接收的语音播报内容，确保语速适中、用词简单、表达温暖、信息明确，让老人清楚知道系统做了什么或需要他做什么。

## 适用场景
- 设备控制执行完成后的结果确认播报
- 意图不明确时的引导确认对话
- 用药提醒的语音通知
- 环境异常的温馨提示（如"窗户没关，外面要下雨了"）
- 系统状态变化的主动告知

## 执行步骤

### 步骤1：确定播报类型和紧急度
- 分类：执行确认/引导询问/定时提醒/异常告知/温馨提示
- 紧急度：紧急（立即播报+高音量）/普通（正常播报）/低优先级（等待空闲播报）

### 步骤2：生成播报文本
- 使用简单词汇（避免技术术语，如不说"设备离线"而说"那个电器好像没反应了"）
- 句式简短（每句不超过15个字，复杂信息分句播报）
- 使用亲切称谓（根据配置使用"张爷爷"/"王奶奶"等）
- 关键信息前置（先说结果，再说细节）

### 步骤3：语音参数配置
- 语速：正常语速的70-80%（可按用户配置调整）
- 音量：根据环境噪音自适应（白天略高/夜间降低）
- 重复：重要信息播报2次（如用药提醒）
- 停顿：句间停顿0.5-1秒，让老人有处理时间

### 步骤4：等待响应（如需要）
- 引导确认类播报后等待用户响应
- 等待时长配置：普通确认30秒/紧急确认10秒
- 超时未响应：重复播报一次后标记为未确认

## 输入规格
```json
{
  "feedback_type": "execution_confirm|guidance_ask|reminder|alert|tip",
  "content": {
    "action_performed": "关闭卧室灯",
    "device_name": "卧室灯",
    "success": true
  },
  "user_profile": {
    "name": "张爷爷",
    "preferred_speed": 0.75,
    "hearing_level": "normal|slight_loss|moderate_loss"
  },
  "environment": {
    "noise_level": "quiet|moderate|noisy",
    "time_period": "morning|afternoon|evening|night"
  }
}
```

## 输出规格
```json
{
  "broadcast_text": "张爷爷，已经帮您关掉卧室的灯了。",
  "speech_params": {
    "speed_ratio": 0.75,
    "volume_level": 0.6,
    "repeat_count": 1,
    "pause_after_ms": 500
  },
  "requires_response": false,
  "response_timeout_ms": null,
  "fallback_text": null
}
```

## 最佳实践
- 执行确认必须明确说出设备名称和操作结果，不可含糊
- 避免使用否定句式（不说"没有失败"，说"成功了"）
- 夜间时段（22:00-7:00）降低音量并简化播报
- 对听力下降的用户自动提高音量和放慢语速
- 连续多个播报时，合并为一句（如"帮您关了灯，也调低了空调"）

## 约束条件
- 播报延迟：操作完成后 < 1秒 内开始播报
- 单次播报时长不超过10秒（避免老人注意力分散）
- 必须使用配置的称谓，不可使用"用户"等冷冰冰的称呼
- 不可播报技术错误详情（如"HTTP 503"），转换为通俗描述
