# 模糊表达意图理解

## 概述
本技能专门处理老年用户高度口语化、省略化、指代不明的模糊表达，通过融合上下文历史、用户习惯、时间场景和设备状态进行高容错推断，将模糊表达转化为明确的设备控制意图。

## 适用场景
- 指代不明的表达："把那个关了"、"开一下那个"
- 模糊感受描述："太热了"、"有点暗"、"吵得很"
- 省略主语/宾语的指令："关了吧"、"调高点"
- 含糊的场景描述："弄得舒服点"、"睡觉了"
- 带有情绪的隐含需求："头疼"（可能暗示需要调暗灯光/降噪）

## 执行步骤

### 步骤1：表达类型分类
- 判断模糊表达类型：指代型/感受型/省略型/场景型/隐含型
- 评估模糊程度等级（1-5级，5为最模糊）

### 步骤2：上下文融合推断
- 查询最近10次交互历史，寻找相关设备/操作引用
- 获取当前时间段常用设备列表（如晚上10点→卧室灯光相关概率高）
- 查询用户历史习惯模式（该用户说"那个"通常指什么设备）
- 获取当前所有设备状态（只有开着的设备才可能被"关"）

### 步骤3：候选意图生成
- 基于多维度信息生成Top-3候选意图列表
- 每个候选附带置信度评分和推断依据
- 若最高候选置信度 >= 80%，视为可确定意图

### 步骤4：确定或引导
- 置信度 >= 80%：直接输出标准化意图，执行控制
- 置信度 60%-80%：生成引导确认语句（如"您是想关卧室的灯吗？"）
- 置信度 < 60%：提供Top-2选项供用户选择

### 步骤5：反馈学习
- 用户确认/否定后更新该用户的表达习惯模型
- 记录成功匹配的模糊表达→意图映射对
- 积累数据用于提升理解准确率

## 输入规格
```json
{
  "standardized_text": "标准化后的用户表达",
  "context": {
    "recent_interactions": ["最近交互记录"],
    "current_time": "2024-01-15T22:30:00",
    "active_devices": ["当前开启的设备列表"],
    "user_location": "bedroom",
    "user_habits": {"常用表达映射历史"}
  }
}
```

## 输出规格
```json
{
  "intent_resolved": true,
  "primary_intent": {
    "target_device": "bedroom_light",
    "action": "turn_off",
    "parameters": {},
    "confidence": 0.85
  },
  "alternative_intents": [],
  "guidance_needed": false,
  "guidance_text": null,
  "reasoning": "用户说'关了吧'，当前时间22:30，卧室灯开启中，历史习惯显示该时段该表达90%指向卧室灯"
}
```

## 最佳实践
- 始终优先使用用户个人历史习惯数据，其次使用群体统计规律
- 对可能造成不适的操作（关暖气、关灯）提高确认阈值至90%
- 「安全优先」原则：涉及安全设备（门锁/摄像头/燃气阀）时必须明确确认
- 模糊表达理解能力随使用时间增长持续提升（个性化学习）

## 约束条件
- 模糊表达理解准确率目标 >= 80%
- 引导对话最多3轮，超过则直接提供选项
- 不可在用户未确认的情况下执行不可逆操作
- 不可基于单一维度做推断（必须多维度交叉验证）
