# 用药提醒与确认闭环

## 概述
本技能管理老年用户的完整用药提醒流程，包括服药计划维护、准时语音提醒、服药确认跟踪、未确认升级通知，确保老人按时按量服药，形成可靠的服药管理闭环。

## 适用场景
- 每日定时服药提醒（早/中/晚/睡前等多时段）
- 特殊用药提醒（需空腹/饭后/间隔时间等条件约束）
- 服药确认跟踪与未确认告警
- 药物冲突检查（多种药物之间的时间间隔要求）
- 药品库存预估与补药提醒

## 执行步骤

### 步骤1：服药计划管理
- 维护药品清单：药品名称/剂量/服用方式/存储要求/外观描述
- 配置服用时间表：精确到分钟的每日服药时间
- 设置药物约束条件：空腹要求/药物间隔/饮食禁忌
- 支持计划CRUD操作（新增/修改/暂停/恢复）

### 步骤2：准时提醒触发
- 到达服药时间时立即发起语音提醒
- 提醒内容包含：称谓+药品名+剂量+外观描述+服用方式
- 示例："张爷爷，现在该吃降压药了。白色圆形小药片，一片，用温水送服。"
- 提醒时间误差 < 1分钟

### 步骤3：确认等待与跟踪
- 等待用户语音确认（"吃了"/"好了"/"知道了"等）
- 确认窗口：30分钟
- 可接受的确认方式：语音回复/按下确认按钮/药盒开启传感器触发
- 确认后记录服药时间戳和确认方式

### 步骤4：未确认升级处理
- 30分钟未确认→发起二次提醒（语调略提升紧迫感但仍温和）
- 二次提醒内容："张爷爷，您的降压药还没吃呢，现在方便吃一下吗？"
- 二次提醒后15分钟仍未确认→通知主要家属（APP推送）
- 通知内容包含：哪种药/应服时间/已过时长

### 步骤5：服药记录与分析
- 记录每次服药的：计划时间/实际确认时间/是否准时/确认方式
- 计算服药依从性指标（周/月维度）
- 连续多次未确认→标记为重要异常→纳入健康异常检测

## 最佳实践
- 使用药品通俗别名而非专业药名（"降压药"而非"氨氯地平"）
- 描述药品外观帮助老人识别（颜色/形状/大小）
- 考虑老人作息节奏，若检测到老人尚在睡眠中，适当延迟非紧急用药提醒
- 对容易忘药的时段（如午间）适当提前1-2分钟提醒
- 药品更换时需要额外详细说明新药特征

## 约束条件
- 提醒准时率 100%（误差 < 1分钟）
- 服药确认闭环率 >= 90%
- 未确认告警发送及时率 100%
- 不可自行调整药物剂量或时间（必须医嘱为准）
- 不可对药物疗效做任何评价或建议
