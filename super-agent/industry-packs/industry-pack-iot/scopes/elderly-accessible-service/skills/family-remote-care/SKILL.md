# 家属远程关怀

## 概述
本技能支持家属通过远程方式对老人进行关怀操作，包括发起视频通话、查看老人居住环境状态、下发温馨提醒、远程调整设备参数等，在满足家属关心需求的同时严格遵循权限边界和老人自主权。

## 适用场景
- 家属远程发起视频/语音通话
- 家属查看老人家环境状态（温度/湿度/空气质量）
- 家属远程调整设备参数（调高空调温度/开灯等）
- 家属下发温馨提醒消息（"爸，今天降温记得穿厚一点"）
- 家属查看设备运行状态和能耗

## 执行步骤

### 步骤1：家属身份验证
- 验证请求来源的登录状态和设备绑定
- 确认请求者的家属角色和权限级别
- 敏感操作（摄像头查看/门锁控制）触发二次验证

### 步骤2：权限校验
- 调用隐私权限管理技能进行权限检查
- 确认该家属是否有权执行请求的操作
- 确认老人的隐私配置是否允许该类操作
- 权限不足时返回友好提示（不暴露配置细节）

### 步骤3：操作执行
- 视频通话：向老人端发起通话请求（需老人接听或超时）
- 环境查看：返回当前温湿度/空气质量等非隐私数据
- 设备控制：验证设备在允许列表中→下发控制指令
- 温馨提醒：转化为语音播报推送至老人端
- 所有操作记录日志（对老人可见）

### 步骤4：老人端通知
- 远程设备控制执行后告知老人："您的女儿帮您把空调调到了26度"
- 视频通话请求以友好方式展示："您的女儿想和您视频通话，接吗？"
- 温馨提醒以温和方式播报，不打断正在进行的操作

### 步骤5：操作反馈
- 向家属返回操作执行结果
- 设备控制类：返回当前设备状态确认
- 通话类：返回通话状态（接通/未接/挂断）
- 提醒类：返回已播报确认

## 输入规格
```json
{
  "requester": {
    "id": "family_001",
    "name": "李女士",
    "role": "admin",
    "verified": true
  },
  "action": {
    "type": "video_call|view_environment|control_device|send_reminder",
    "target": {
      "device_id": "living_room_ac",
      "operation": "set_temperature",
      "parameters": {"temperature": 26}
    }
  },
  "elder_status": {
    "is_home": true,
    "current_activity": "resting",
    "last_interaction": "2024-01-15T14:00:00"
  }
}
```

## 最佳实践
- 远程操作不应打断老人正在进行的活动（如正在通话时不调整电视音量）
- 视频通话请求应给老人充足的响应时间（30秒以上）
- 设备控制操作附带简短说明（"天冷了帮您调高一点暖气"），让老人理解意图
- 避免深夜时段的非紧急远程操作（22:00-7:00静默期）
- 鼓励正向互动（视频通话/温馨提醒）而非单纯监控型操作

## 约束条件
- 所有远程操作必须通过权限校验（100%执行）
- 操作结果必须告知老人（透明原则）
- 静默期（22:00-7:00）非紧急操作自动延迟
- 本地操作优先——老人正在操作的设备不可被远程覆盖
- 摄像头查看必须经老人实时授权（不可静默开启）
- 操作日志完整保留，老人随时可查
