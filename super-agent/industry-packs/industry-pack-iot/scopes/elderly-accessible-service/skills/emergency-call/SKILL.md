# 紧急呼叫与应急响应

## 概述
本技能在检测到紧急事件（跌倒确认、心率骤变、用户主动求助等）时，立即启动多通道紧急呼叫流程，确保在最短时间内联系到救援力量和家属，同时对老人进行安抚和现场信息记录。这是适老化服务中「生命线」级别的技能，要求100%可靠。

## 适用场景
- 跌倒检测确认后的紧急呼叫
- 生理指标突发异常（心率骤变/血氧骤降）
- 老人主动语音求助（"救命"/"帮帮我"/"打电话"）
- 老人按下物理紧急按钮
- 长时间无活动信号（48小时+）的最终升级

## 执行步骤

### 步骤1：紧急事件确认
- 接收跌倒检测/健康异常/用户求助触发信号
- 对非显式求助类事件，先执行10秒语音确认（"您还好吗？"）
- 主动求助/物理按钮/生理指标极值→跳过确认直接响应

### 步骤2：多通道呼叫启动（5秒内）
- 通道1：拨打第一紧急联系人（通常为子女）
- 通道2：发送短信至所有紧急联系人（含事件概述和位置）
- 通道3（如配置启用）：拨打120急救电话
- 通道4：APP推送紧急告警至所有家属

### 步骤3：呼叫跟踪与重试
- 第一联系人15秒未接听→拨打第二联系人
- 所有联系人未接听→2分钟后重试第一轮
- 持续重试直到有人接听或达到最大重试次数（10次）
- 每次重试同时更新短信/推送通知（"仍未接通，请尽快回复"）

### 步骤4：现场信息记录
- 开启摄像头记录现场画面（如隐私配置允许）
- 记录事件触发时间/触发源/传感器数据快照
- 记录老人语音（如有）/环境声音
- 生成事件时间线供救援人员参考

### 步骤5：老人安抚
- 持续语音安抚："别着急，已经帮您联系家人了"
- 每30秒播报一次当前状态："正在呼叫您的女儿李女士"
- 联系人接通后告知："您的女儿已经接到电话了，马上就来"
- 引导老人保持镇定、不要移动（如跌倒场景）

### 步骤6：事后报告
- 生成完整事件报告（时间线/响应过程/联系结果）
- 记录从检测到首次通话接通的总耗时
- 供家属和医疗人员事后参考

## 最佳实践
- 紧急呼叫通道必须独立于智能家居网络（使用4G/5G蜂窝网络备份）
- 定期进行呼叫测试（每月1次静默测试确保通道畅通）
- 紧急联系人列表至少配置3人，确保有人可达
- 120呼叫时自动播报地址信息和老人基本健康信息
- 事件全过程留痕，支持事后复盘和系统优化

## 约束条件
- 呼叫发起成功率 100%（不可有任何技术原因导致失败）
- 从检测确认到呼叫发起 < 5秒
- 首次接通率 >= 90%（含多联系人轮拨）
- 从检测到首个联系人接通 < 60秒
- 不可因任何非紧急事件误触发紧急呼叫流程
- 网络中断时必须有蜂窝备份通道
