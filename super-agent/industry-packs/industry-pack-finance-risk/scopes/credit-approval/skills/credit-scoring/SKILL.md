# 信用评分计算

## 技能概述
本技能运用内部信用评分模型（A卡-申请评分卡），综合征信数据、申请信息、反欺诈标签和外部数据，计算申请人的综合信用评分，并根据评分进行风险等级划分。信用评分是信贷审批决策的核心量化依据，模型的区分度（KS值）直接决定了资产质量。

## 适用场景
- 个人信贷申请的信用评分计算
- 企业信贷申请的企业信用评分计算
- 白户客户的替代数据评分
- 评分模型效果监控和校准

## 执行步骤

### 1. 特征工程
- 从征信报告分析结果提取评分特征：逾期指标、负债指标、信贷历史指标、查询指标
- 从申请信息提取特征：年龄、学历、职业、工作年限、收入、资产状况
- 从反欺诈结果提取特征：欺诈评分、命中规则数量、风险标签
- 对缺失特征执行标准化的缺失值处理策略（按模型文档规定）

### 2. 模型评分计算
- 调用A卡申请评分模型，输入标准化特征向量
- 获取模型输出的原始分值，映射至标准评分区间（300-900分）
- 输出评分分解：各特征对评分的贡献度（正向/负向及分值）
- 记录模型版本号和评分时间戳

### 3. 秒拒判定
- 征信评分（央行征信中心评分）低于600分的申请触发"秒拒"
- 内部综合评分低于机构设定的拒贷线的申请进入拒绝通道
- 秒拒申请生成标准化拒贷理由（不透露具体评分和模型细节）

### 4. 风险等级划分
- **A级（优质）**：综合评分≥750，历史无逾期，负债率低
- **B级（良好）**：综合评分700-749，偶发轻微逾期，负债率适中
- **C级（一般）**：综合评分650-699，有一定逾期历史，负债率较高
- **D级（高风险）**：综合评分600-649，逾期较频繁，负债率高
- D级以下直接拒绝

### 5. 白户评分（替代数据方案）
- 当征信数据不足（无信贷记录或信贷记录少于6个月）时启用
- 整合替代数据源：运营商数据、社保/公积金数据、电商消费数据、税务数据
- 调用白户专用评分模型
- 白户评分结果标注"替代数据评分"标签，提示评分置信度低于常规评分

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| creditReportData | object | 是 | 征信报告解析结果 |
| applicationData | object | 是 | 申请人信息 |
| fraudCheckResult | object | 是 | 反欺诈检测结果 |
| alternativeData | object | 否 | 替代数据（白户时使用） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| creditScore | number | 综合信用评分（300-900） |
| riskGrade | string | 风险等级：A/B/C/D/REJECT |
| scoreBreakdown | array | 评分分解（各特征贡献度） |
| isInstantReject | boolean | 是否触发秒拒 |
| rejectReason | string | 拒贷理由代码（如适用） |
| scoringMethod | string | 评分方式：standard/alternative |
| modelVersion | string | 使用的模型版本号 |

## 最佳实践
- 模型KS值须维持在0.30以上，低于0.25时触发模型重训
- 定期（月度）执行PSI（Population Stability Index）监控，评估入模特征的稳定性
- 评分结果应与实际违约表现定期比对，验证模型的校准度
- 评分分解须保留，满足监管对"可解释性"的要求

## 约束条件
- 评分模型不得使用种族、性别、宗教等法律禁止的歧视性变量
- 评分结果不得直接展示给客户（仅展示评级结果和大类理由）
- 模型更新须经模型验证委员会审批后方可上线
- 白户替代数据的使用须获得客户单独授权
