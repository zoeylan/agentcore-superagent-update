# 合同要素校验

## 技能概述
本技能对贷款合同的核心要素、签署有效性和附件完整性进行系统化校验，确保合同法律效力完备、要素与审批决策一致，是放款前的关键控制环节。

## 适用场景
- 放款前的合同完整性校验
- 合同变更（如展期、降额）后的重新校验
- 批量合同的合规性抽检
- 审计检查中的合同要素核查

## 执行步骤

### 1. 合同核心要素校验
- 逐项比对合同要素与审批决策书：
  - 借款人名称/姓名、身份证号/统一社会信用代码
  - 贷款金额（大小写一致且与审批金额一致）
  - 贷款年化利率/月利率（与审批利率一致）
  - 贷款期限和到期日
  - 还款方式（等额本息/等额本金/先息后本等）
  - 还款日（每月固定日期）
  - 贷款用途
  - 提前还款条款
  - 逾期罚息条款

### 2. 电子签章有效性验证
- 验证CA证书的有效性（颁发机构为持牌CA、证书在有效期内、未被吊销）
- 验证签署人身份与借款人一致
- 验证签署时间戳的可信性（来自可信时间戳服务）
- 验证文档在签署后未被篡改（哈希值校验）

### 3. 附件完整性检查
- 根据产品类型和审批条件确定必须的附件清单：
  - 通用：征信授权书、身份证明复印件
  - 抵押贷：抵押合同、抵押物评估报告、抵押登记证明
  - 担保贷：担保合同、担保人征信授权书和身份证明
  - 企业贷：公司章程、董事会/股东会决议、财务报表
- 逐项确认附件是否齐全且有效

### 4. 合规条款检查
- 确认合同中包含监管要求的必备条款（如提前还款权利、个人信息保护条款、投诉渠道等）
- 确认APR等综合融资成本已在合同中明确披露
- 检查是否存在不当条款（如不合理的违约金比例、单方面变更利率的条款等）

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| contractDocument | object | 是 | 合同文档信息 |
| approvalDecision | object | 是 | 审批决策书 |
| requiredAttachments | array | 是 | 必须的附件清单 |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| verificationResult | string | 校验结果：PASS/FAIL |
| elementCheckList | array | 各要素校验明细 |
| signatureVerification | object | 签章验证结果 |
| attachmentCheckList | array | 附件完整性检查明细 |
| discrepancies | array | 发现的不一致项列表 |

## 最佳实践
- 合同要素校验应在签约完成后、放款前执行
- 发现任何不一致都应中断放款并反馈修正
- 合同模板更新后应同步更新校验规则

## 约束条件
- 合同要素与审批决策的一致性必须100%
- 电子签章校验不通过的合同不得放款
- 校验过程和结果须完整留痕
