# 合规性审查

## 技能概述
本技能对信用评估通过的贷款申请执行全面的合规性审查，确保每笔贷款在用途、利率、客户适当性、集中度和关联交易等维度均符合监管要求和机构内部政策。合规审查是审批流程中的"守门人"环节，一旦放过合规瑕疵将直接面临监管处罚和法律风险。

## 适用场景
- 所有通过信用评估的贷款申请的合规终审
- 有条件批准后补充材料的合规复查
- 监管政策更新后的存量贷款合规重检
- 审计抽查中发现的合规疑点复核

## 执行步骤

### 1. 贷款用途合规审查
- **消费贷款**：确认贷款用途为合法消费（家装、教育、医疗、旅游等），明确排除以下禁止用途：
  - 购买住房（含首付款）
  - 投资股票、基金、期货等金融资产
  - 民间借贷（转贷行为）
  - 赌博、违法活动
- **经营贷款**：核实是否有真实经营场景，检查贸易合同/发票/采购订单等交易背景材料
- **企业贷款**：核查贷款用途是否与企业经营范围匹配，资金流向是否有受托支付安排

### 2. 利率与费用合规审查
- 验证贷款年化利率（APR）不超过同期LPR的4倍
- 验证综合融资成本（含利息、服务费、保险费、担保费等IRR口径）在合规范围内
- 检查是否存在"砍头息"（预先扣除利息/费用导致实际到手金额低于合同金额）
- 确认费率计算方式已按监管要求向客户充分披露

### 3. 客户适当性审查
- 评估贷款产品是否与客户的风险承受能力匹配
- 检查是否存在向低收入群体推荐高额高息产品的不当行为
- 核实特殊群体限制：在校学生、现役军人、公务员等的借贷限制
- 高龄借款人（>60岁）须核实是否有明确的还款来源

### 4. 集中度与关联交易审查
- **单一客户集中度**：检查该客户在本机构的贷款余额（含本笔）不超过资本净额的10%
- **集团客户集中度**：同一集团关联客户的授信总额不超过资本净额的15%
- **行业集中度**：检查该行业贷款余额占比是否接近或超过内部限额
- **关联交易**：识别借款人与机构股东、高管、员工之间的关联关系，关联贷款须经董事会审批

### 5. 反洗钱合规审查
- 核查客户是否在反洗钱制裁名单中（联合国/OFAC/中国反洗钱名单）
- 对大额贷款（个人>20万/企业>100万）执行资金来源/用途的增强尽职调查
- 检查是否存在可疑交易特征（如短期内频繁借贷-还款循环）

### 6. 合规审查结论
- 逐项出具审查结论：通过/不通过/豁免（附豁免理由）
- 不通过项必须说明违反的具体法规条款和内部政策条目
- 生成合规审查报告，附完整的审查清单和签核记录

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| creditAssessmentReport | object | 是 | 信用评估报告全文 |
| loanTerms | object | 是 | 贷款要素（金额、利率、期限、用途） |
| applicantProfile | object | 是 | 客户画像信息 |
| existingExposure | object | 是 | 客户在本机构的存量授信信息 |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| complianceResult | string | 合规审查结论：PASS/FAIL/CONDITIONAL |
| checklistItems | array | 各维度审查结果明细 |
| failedItems | array | 未通过项列表（含违反的法规/政策引用） |
| complianceReportId | string | 合规审查报告编号 |

## 最佳实践
- 合规审查清单应随监管政策更新及时调整
- 对于监管灰色地带的判断应采取保守策略，必要时升级至合规部门判定
- 合规审查结果应独立于信用评估结论——即使信用评估为A级，合规不通过也必须拒绝

## 约束条件
- 合规审查不得被跳过或简化执行
- 合规标准不得因业绩压力而降低
- 所有审查记录须保留至少贷款结清后5年
- 合规审查人员/系统须与信贷营销/信用评估独立
