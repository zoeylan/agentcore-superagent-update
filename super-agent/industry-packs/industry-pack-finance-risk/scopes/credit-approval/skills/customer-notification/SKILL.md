# 客户通知

## 技能概述
本技能负责在信贷审批各关键节点向客户发送多渠道通知，包括审批结果通知、放款到账通知、还款计划通知和拒贷通知。通知内容须合规、准确、及时，既保障客户知情权又不泄露内部风控细节。

## 适用场景
- 审批通过后的结果通知
- 放款到账的实时通知
- 还款计划和首期还款日提醒
- 申请被拒绝的拒贷通知
- 有条件批准的补件通知
- 审批进度查询的状态回复

## 执行步骤

### 1. 通知内容生成
- 根据通知类型选择标准模板：
  - **审批通过通知**：恭喜语 + 批准金额 + 利率 + 期限 + 后续签约指引
  - **放款到账通知**：到账金额 + 到账时间 + 还款起始日 + 月供金额 + 还款方式说明
  - **还款提醒**：还款日期 + 应还金额（本金+利息明细）+ 还款账户 + 还款方式
  - **拒贷通知**：感谢申请 + 拒贷原因大类（如"综合信用评估未通过"） + 申诉渠道 + 客服联系方式
  - **补件通知**：缺失材料清单 + 具体格式要求 + 提交截止时间 + 提交方式
- 填充个性化字段（客户姓名、具体金额等）
- 敏感信息脱敏处理（如银行卡号仅显示后4位）

### 2. 多渠道发送
- **短信**：通过短信网关发送，控制单条短信在70字以内（含签名），超长分段
- **APP推送**：通过推送服务发送富文本通知，支持点击跳转至详情页
- **邮件**：发送HTML格式邮件，包含完整的通知内容和附件（如还款计划表PDF）
- 根据客户偏好设置和通知重要程度选择发送渠道（重要通知如放款到账须全渠道触达）

### 3. 送达确认与重试
- 监控各渠道的送达状态：
  - 短信：获取运营商回执（成功/失败/未知）
  - APP推送：获取客户端接收确认
  - 邮件：获取邮件服务器投递状态
- 首次发送失败时自动重试（最多2次，间隔10分钟）
- 全渠道均送达失败时触发人工跟进告警

### 4. 通知记录归档
- 记录每条通知的发送时间、渠道、内容摘要、送达状态
- 支持客户查询历史通知记录
- 通知记录纳入贷款档案，满足审计要求

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| notificationType | string | 是 | 通知类型 |
| recipientInfo | object | 是 | 收件人信息（手机号、邮箱、APP用户ID） |
| notificationData | object | 是 | 通知内容数据（具体金额、日期等） |
| channels | array | 否 | 指定发送渠道（默认全渠道） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| notificationId | string | 通知编号 |
| channelResults | array | 各渠道发送结果 |
| overallDeliveryStatus | string | 综合送达状态 |
| sentTimestamp | string | 发送时间 |

## 最佳实践
- 通知措辞应简洁友好，避免使用专业术语
- 拒贷通知应谨慎措辞，不得包含歧视性内容或泄露模型细节
- 通知发送应尊重客户的免打扰时段设置（通常21:00-08:00不发送非紧急通知）
- 送达率目标为99%以上

## 约束条件
- 不得在通知中泄露内部评分、模型阈值和具体规则
- 不得发送与贷款业务无关的营销内容
- 通知频次须符合监管要求和客户偏好设置
- 所有通知内容须经过合规审核的模板库管理
