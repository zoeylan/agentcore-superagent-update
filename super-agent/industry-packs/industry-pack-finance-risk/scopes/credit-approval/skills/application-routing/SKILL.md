# 申请路由分发

## 技能概述
本技能负责根据贷款申请的类型、金额、客户类别等维度，将申请智能路由至对应的审批通道和处理流程。不同类型的申请有不同的SLA要求、审批权限和所需材料，精准的路由是保证审批时效和流程正确性的前提。

## 适用场景
- 新提交的贷款申请的首次分类与路由
- 有条件批准后补充材料的回流路由
- 审批策略调整后的存量申请重路由
- 异常申请的特殊通道分配

## 执行步骤

### 1. 申请类型识别
- 根据申请人性质判断大类：个人客户 → 个人信贷通道；企业客户 → 企业信贷通道
- 个人信贷细分：消费贷（无指定用途/消费用途）、经营贷（有营业执照/经营流水）
- 企业信贷细分：流动资金贷款（短期周转）、项目贷款（固定资产投资/项目建设）

### 2. 审批通道分配
- **个人信贷自动审批通道**：申请金额≤30万元，产品为标准消费贷/经营贷，适用48小时SLA
- **个人信贷人工审批通道**：申请金额>30万元或非标产品，适用48小时SLA但需人工节点
- **企业信贷标准通道**：流动资金贷款≤500万元，适用5个工作日SLA
- **企业信贷审委会通道**：金额>500万元或项目贷款，需提交审批委员会，适用5个工作日SLA
- **秒拒通道**：征信评分<600分或命中硬性拒绝规则的申请，直接进入快速拒绝流程

### 3. 优先级标记
- VIP客户/白名单客户：标记为高优先级，优先处理
- 大额申请（个人>30万/企业>500万）：标记为重点审批件
- 补件回流：标记为续审件，插入原流程位置继续审批

### 4. SLA计时启动
- 为每笔申请设置SLA计时器，记录进入时间和目标完成时间
- 到达SLA的70%时触发黄色预警，90%时触发红色预警
- SLA超时自动升级至管理层关注

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| applicantType | string | 是 | 申请人类型：personal/corporate |
| productCode | string | 是 | 申请产品代码 |
| applicationAmount | number | 是 | 申请金额 |
| admissionResult | string | 是 | 准入校验结果 |
| customerLevel | string | 否 | 客户等级（VIP/普通） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| routingChannel | string | 路由通道标识 |
| approvalMode | string | 审批模式：auto/manual/committee |
| slaDeadline | string | SLA截止时间 |
| priority | string | 优先级：high/normal/low |
| nextAgent | string | 下一个处理Agent标识 |

## 最佳实践
- 路由规则应与产品管理部门保持同步更新
- 路由决策日志完整保留，支持审计和流程优化分析
- 路由通道的负载应均衡，避免某一通道积压

## 约束条件
- 不得将企业信贷路由至个人信贷自动审批通道
- 不得修改或跳过SLA计时机制
- 路由决策需在接收申请后5秒内完成
