# 准入规则校验

## 技能概述
本技能执行信贷申请的基础准入规则校验，作为贷款申请的第一道"门槛"，通过预设的硬性规则筛选出不符合基本贷款条件的申请，避免不合格申请进入后续的反欺诈和信用评估环节，节约系统资源和审批时间。

## 适用场景
- 新贷款申请提交后的首次准入校验
- 有条件批准后客户补充材料的重新校验
- 准入规则更新后的批量重检
- 白名单/预审批客户的快速准入确认

## 执行步骤

### 1. 申请人基本资格校验
- **年龄校验**：个人客户年龄需在18-65周岁之间（以身份证出生日期计算至申请日）
- **国籍/身份校验**：确认为中国公民或符合条件的外籍人士（需持有有效居留许可）
- **行为能力校验**：核实申请人具有完全民事行为能力（排除被宣告为限制/无民事行为能力人）

### 2. 地域与行业准入校验
- **地域准入**：核对申请人户籍地/常住地/企业注册地是否在机构服务覆盖区域内
- **行业黑名单**：企业客户的经营行业是否属于禁入行业（房地产开发、矿业开采、钢铁产能过剩行业、博彩娱乐等），依据内部行业政策名单执行
- **行业限制**：半限制行业（如物流、餐饮等波动性行业）标记为"需加强审查"而非直接拒绝

### 3. 黑名单与负面信息比对
- **内部黑名单**：比对机构内部的历史欺诈客户名单、历史坏账客户名单、合规违规名单
- **外部黑名单**：比对法院被执行人名单、反洗钱制裁名单（联合国/OFAC）、行业共享黑名单
- **关联黑名单**：检查申请人的直系亲属、企业实际控制人是否在黑名单中

### 4. 征信授权合规性检查
- 确认客户已签署征信查询授权书
- 校验授权书签名的真实性（与申请人签名比对）
- 核实授权范围覆盖本次查询需求（个人基本信息、信贷记录、公共信息等）
- 检查授权书的签署日期和有效期（通常有效期为30天）

### 5. 产品准入适配
- 核对申请金额是否在产品额度范围内（个人消费贷单笔上限50万元）
- 核对申请期限是否在产品期限范围内
- 核对客户类型与产品类型的匹配性（如个人客户不可申请企业流动资金贷款）

### 6. 结果输出与处理
- **全部通过**：生成"准入通过"标签，申请流转至反欺诈核查环节
- **硬性规则命中**：生成标准化拒绝通知（含拒绝原因代码和大类描述），申请终止并归档
- **软性规则命中**：标记风险提示标签，申请继续流转但附带风险说明

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| applicantInfo | object | 是 | 申请人基本信息（OCR提取+表单填写） |
| productCode | string | 是 | 申请产品代码 |
| applicationAmount | number | 是 | 申请金额（元） |
| applicationTerm | number | 是 | 申请期限（月） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| admissionResult | string | 准入结果：PASS / REJECT / REVIEW |
| hitRules | array | 命中规则列表（规则编号、描述、严重级别） |
| rejectionCode | string | 拒绝原因代码（如适用） |
| rejectionMessage | string | 客户可见的拒绝原因描述 |
| riskTags | array | 风险标签列表 |
| checkTimestamp | string | 校验完成时间戳 |

## 最佳实践
- 准入规则应按"成本从低到高"的顺序执行（先本地校验再外部查询），减少不必要的外部调用费用
- 规则命中率应月度监控，命中率异常波动超过5%时需触发规则审查
- 拒绝通知的措辞应合规且友好，不得泄露具体规则阈值
- 准入规则更新需经过合规审核后方可上线

## 约束条件
- 不得绕过任何硬性准入规则
- 不得向客户透露黑名单比对的具体数据来源
- 准入校验需在收到申请后30秒内完成
- 所有规则命中记录须完整保留供审计
