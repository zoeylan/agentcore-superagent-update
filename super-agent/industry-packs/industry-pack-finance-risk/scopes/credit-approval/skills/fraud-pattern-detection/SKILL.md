# 欺诈模式检测

## 技能概述
本技能运用规则引擎和机器学习模型的双层架构，对贷款申请进行多维度欺诈模式检测，识别第一方欺诈（申请人蓄意造假）、第三方欺诈（身份盗用）和团伙欺诈（有组织骗贷）等各类欺诈行为，输出反欺诈综合评分和命中规则明细。

## 适用场景
- 所有通过准入校验的贷款申请的常规欺诈检测
- 疑似材料伪造案件的深度欺诈排查
- 批量申请的团伙欺诈模式扫描
- 反欺诈规则迭代后的回溯检测

## 执行步骤

### 1. 规则引擎检测
- 加载当前有效的反欺诈规则集（含阈值参数）
- 逐条执行规则匹配：
  - **申请频率规则**：同一身份证/手机号/设备在N天内申请次数超阈值
  - **信息一致性规则**：申请信息与历史申请/征信报告存在矛盾
  - **高风险特征规则**：新注册手机号、虚拟运营商号码、凌晨时段申请等
  - **金额异常规则**：申请金额与收入水平严重不匹配
  - **地域异常规则**：申请地、户籍地、设备定位地三者严重不一致
- 记录每条命中规则的编号、描述和权重分值

### 2. 设备指纹分析
- 采集并分析申请设备的指纹信息（设备型号、操作系统、屏幕分辨率、已安装APP列表哈希等）
- 检测模拟器/虚拟机环境（如Root/越狱、Xposed框架、模拟器特征）
- 查询设备指纹库：该设备是否关联过其他申请账号、是否在设备黑名单中
- 分析设备行为特征：打字速度、页面停留时间、填写模式是否符合正常人类行为

### 3. ML模型评分
- 构建实时特征向量：将申请人信息、行为数据、设备数据、外部数据转化为模型输入特征
- 调用反欺诈ML模型（如XGBoost/LightGBM集成模型）计算欺诈概率
- 将模型输出的概率值映射为0-100分的反欺诈评分
- 输出模型评分的Top5特征重要性（支持可解释性要求）

### 4. 综合评分与分级
- 融合规则引擎评分和ML模型评分，计算反欺诈综合评分
- 分级决策：<50分（低风险）放行，50-80分（中风险）标注风险后流转，>80分（高风险）拦截
- 对高风险申请自动生成欺诈调查工单

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| structuredData | object | 是 | 进件预处理输出的结构化数据 |
| deviceFingerprint | object | 是 | 设备指纹数据 |
| behaviorData | object | 否 | 申请行为数据 |
| identityVerificationResult | object | 是 | 身份核验结果 |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| fraudScore | number | 反欺诈综合评分（0-100） |
| riskLevel | string | 风险等级：LOW/MEDIUM/HIGH |
| hitRules | array | 命中规则列表（含编号、描述、权重） |
| modelScore | number | ML模型原始评分 |
| topFeatures | array | 模型评分Top5特征及贡献度 |
| decision | string | 处理决策：PASS/FLAG/BLOCK |
| investigationOrder | object | 欺诈调查工单（仅高风险时生成） |

## 最佳实践
- 规则引擎和ML模型应并行执行以缩短检测延迟
- 定期（至少每月）回检规则命中率和模型AUC，识别性能退化
- 新规则上线前须在历史数据上回测，评估误拦截影响
- 保持规则库的可解释性——每条规则须有明确的业务逻辑说明

## 约束条件
- 检测覆盖率必须为100%，不得有任何申请绕过检测
- 误拦截率（FPR）须控制在3%以内
- 单笔检测延迟不超过3秒
- 检测结果和证据链须完整保留至少5年
