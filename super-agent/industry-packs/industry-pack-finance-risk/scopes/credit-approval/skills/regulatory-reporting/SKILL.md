# 监管报送

## 技能概述
本技能负责信贷审批过程中涉及的各类监管报送材料的生成和提交，确保审批数据和决策信息按照监管要求的格式、时效和内容完整性进行报送。包括征信信息报送、大额授信报送、可疑交易报告以及日常监管报表。

## 适用场景
- 贷款审批完成后的征信信息报送
- 大额授信的监管备案
- 审批过程中发现可疑交易的报送
- 定期（月度/季度/年度）监管报表编制
- 监管现场检查的数据准备

## 执行步骤

### 1. 征信信息报送
- 贷款审批完成（批准并放款/拒绝）后，按照央行征信中心要求的数据格式更新征信系统
- 报送内容：贷款基本信息（金额、利率、期限）、审批结果、放款信息
- 报送时效：放款后T+1个工作日内完成报送
- 数据质量校验：报送前自动核对数据完整性和格式合规性

### 2. 大额授信信息报送
- 单一客户授信余额达到报送阈值时（按监管要求，通常为机构资本净额的一定比例），向监管部门报送
- 报送内容：客户基本信息、授信总额、各笔贷款明细、风险分类
- 集团客户关联授信的汇总报送

### 3. 可疑交易/欺诈报告
- 审批过程中发现的欺诈线索（反欺诈Agent拦截的高风险件中确认的欺诈案件），按《反洗钱法》要求报送可疑交易报告
- 重大欺诈案件（金额>50万或涉及团伙）24小时内完成报送材料准备
- 报送至中国反洗钱监测分析中心

### 4. 定期监管报表
- 月度：信贷投放统计（笔数、金额、行业分布、区域分布）、审批通过率、不良预警
- 季度：资产质量分析、拨备计提测算、集中度分析
- 年度：信贷政策执行情况评估、模型验证报告

### 5. 审计数据准备
- 维护审批决策的完整审计轨迹
- 确保每笔贷款的审批流程文档完整可查
- 支持监管现场检查的数据快速抽取和报告生成

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| reportType | string | 是 | 报送类型 |
| applicationData | object | 是 | 贷款申请和审批数据 |
| reportingPeriod | string | 否 | 报送期间（定期报表时使用） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| reportId | string | 报送编号 |
| reportContent | object | 报送内容 |
| submissionStatus | string | 报送状态 |
| submissionTimestamp | string | 报送时间 |

## 最佳实践
- 报送数据与内部系统数据保持一致性，避免"两套账"
- 建立报送数据的自动核对机制，发现偏差及时修正
- 保留报送记录和回执，满足监管检查需要

## 约束条件
- 报送时效必须满足监管要求（逾期将面临行政处罚）
- 报送内容的准确性由合规部门最终审核确认
- 涉及客户信息的报送须在授权范围内
- 报送数据须加密传输
