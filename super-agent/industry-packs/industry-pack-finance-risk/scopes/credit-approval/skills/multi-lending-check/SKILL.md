# 多头借贷检测

## 技能概述
本技能查询申请人在多个金融机构和网贷平台的借贷情况，识别过度负债、以贷养贷、短期集中申请等多头借贷风险信号。多头借贷是信贷风控中的重要风险指标，过度负债的借款人违约概率显著高于正常客户。

## 适用场景
- 所有贷款申请的常规多头借贷排查
- 信用评估中的负债验证辅助
- 存量客户的多头风险定期监控
- 贷后预警中的多头变化追踪

## 执行步骤

### 1. 征信查询记录分析
- 调取申请人央行征信报告中的"查询记录"段
- 统计近1个月、近3个月、近6个月的贷款审批类查询次数
- 风险阈值判定：近1个月查询≥4次为"高多头风险"，2-3次为"中多头风险"
- 识别查询机构类型分布（银行、消金、小贷、网贷），网贷占比高者风险更大

### 2. 第三方多头数据查询
- 通过持牌征信机构（如百行征信、朴道征信）查询申请人的网贷借贷数据
- 获取当前在贷平台数、在贷笔数、在贷总余额
- 获取近期申请记录（各平台的申请时间和申请金额）
- 计算多头借贷指数（综合考虑在贷笔数、申请频率、余额增长趋势）

### 3. 多头风险评估
- **低风险**：在贷平台数≤2，近3个月查询≤3次，在贷余额占收入比<30%
- **中风险**：在贷平台数3-5，近3个月查询4-6次，在贷余额占收入比30-50%
- **高风险**：在贷平台数>5，近3个月查询>6次，在贷余额占收入比>50%
- **极高风险**：近1个月内在贷余额显著增加（环比增长>30%），疑似以贷养贷

### 4. 多头趋势分析
- 对比近3个月和近6个月的多头数据变化趋势
- 上升趋势（在贷笔数/余额持续增加）标记为负面信号
- 下降趋势（在贷笔数/余额持续减少）标记为正面信号
- 突变模式（短期内急剧增加）标记为高风险预警

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicantName | string | 是 | 申请人姓名 |
| idNumber | string | 是 | 身份证号码 |
| phoneNumber | string | 是 | 手机号码 |
| creditReportQuerySection | object | 否 | 征信报告查询记录段（如已获取） |
| monthlyIncome | number | 否 | 月收入（用于负债收入比计算） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| multiLendingScore | number | 多头借贷风险评分（0-100） |
| riskLevel | string | 风险等级：LOW/MEDIUM/HIGH/EXTREME |
| queryCount | object | 各时间窗口的查询次数统计 |
| activeLoanSummary | object | 当前在贷汇总（平台数、笔数、余额） |
| trendAnalysis | object | 多头趋势分析结果 |
| riskSignals | array | 风险信号列表 |

## 最佳实践
- 多头数据应与收入数据结合分析，高收入客户的合理负债容忍度更高
- 不同类型的借贷机构应区分权重——银行借贷vs网贷的风险含义不同
- 多头检测结果应纳入信用评估的DTI计算，避免低估实际负债水平
- 注意区分"查询次数多"和"实际借贷多"——部分查询可能仅为预审批

## 约束条件
- 多头查询必须获得客户征信授权
- 第三方数据查询接口须为持牌征信机构提供
- 查询记录本身也会被记录在征信报告中，应避免不必要的重复查询
- 多头数据使用仅限于信贷风控用途
