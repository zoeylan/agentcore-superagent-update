# 风险定价

## 技能概述
本技能基于信用评分和风险等级，确定贷款的建议额度区间和利率区间，实现"风险与收益匹配"的定价原则。风险定价是信贷业务盈利模型的核心——过高定价会流失优质客户，过低定价则无法覆盖风险成本。

## 适用场景
- 新贷款申请的额度和利率建议
- 额度提升/降低的重新定价
- 利率调整周期的批量重定价
- 产品定价策略调整后的测算

## 执行步骤

### 1. 额度核定
- 基于还款能力分析确定最大可承受额度
- 基于信用评分和风险等级确定评分维度额度上限
- 取两者较低值作为额度上限
- 参照产品额度限制（如个人消费贷单笔上限50万元）进行封顶
- 结合抵押物评估价值确定抵押贷款额度（LTV上限：住宅70%、商业60%、工业50%）
- 额度建议输出为区间形式（最低建议额度~最高建议额度）

### 2. 利率定价
- 确定基准利率：以同期LPR（贷款市场报价利率）为基准
- 叠加风险溢价：根据风险等级确定加点幅度
  - A级：LPR + 0~50BP
  - B级：LPR + 50~150BP
  - C级：LPR + 150~300BP
  - D级：LPR + 300~500BP
- 校验利率上限：确保年化利率不超过同期LPR的4倍（司法保护上限）
- 校验IRR口径的综合融资成本（含服务费、保险费等）是否在合规范围内

### 3. 期限建议
- 根据贷款用途和客户类型建议期限范围
- 消费贷：3-36个月为主
- 经营贷：6-60个月为主
- 企业流动资金贷款：6-12个月为主
- 项目贷款：根据项目回收期确定，通常1-5年

### 4. 定价方案输出
- 生成定价方案（含推荐方案和备选方案）
- 推荐方案：最优额度+对应利率+建议期限+还款方式
- 备选方案：降额提价/提额降额等替代选项
- 计算贷款的预期损失率（EL = PD × LGD × EAD）和风险资本占用

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| creditScore | number | 是 | 综合信用评分 |
| riskGrade | string | 是 | 风险等级 |
| maxAffordableAmount | number | 是 | 最大可承受额度 |
| collateralValue | number | 否 | 抵押物评估价值（如有） |
| productCode | string | 是 | 产品代码 |
| requestedAmount | number | 是 | 客户申请金额 |
| requestedTerm | number | 是 | 客户申请期限 |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| recommendedAmount | object | 建议额度（min/max/recommended） |
| recommendedRate | object | 建议利率（annualRate/monthlyRate/apr） |
| recommendedTerm | number | 建议期限（月） |
| repaymentMethod | string | 建议还款方式 |
| expectedLossRate | number | 预期损失率 |
| pricingRationale | string | 定价理由说明 |
| alternativeOptions | array | 备选方案列表 |

## 最佳实践
- 定价应综合考虑"风险成本+资金成本+运营成本+利润目标"四个维度
- 利率定价结果应与同业竞品水平对标，避免极端定价
- 预期损失率计算应使用最新的PD/LGD模型参数
- 定价策略调整应做压力测试，评估对通过率和预期收益的影响

## 约束条件
- 利率绝对不得超过同期LPR的4倍
- 额度不得超过产品规定的上限
- 定价模型参数调整须经风险管理委员会审批
- 定价结果须向客户充分披露（APR、总还款额等）
