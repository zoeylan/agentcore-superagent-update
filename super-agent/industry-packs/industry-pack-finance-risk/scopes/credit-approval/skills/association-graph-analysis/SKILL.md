# 关联图谱分析

## 技能概述
本技能基于图计算技术，通过多维关联因子构建申请人的社会关系网络和行为关联网络，识别团伙欺诈、中介代办、关联人互保等隐蔽的欺诈模式和风险传导路径。关联图谱分析是发现组织化、隐蔽性强的欺诈行为的关键能力。

## 适用场景
- 新申请的常规关联风险扫描
- 疑似团伙欺诈的深度关联扩展分析
- 企业信贷的实际控制人穿透与关联交易识别
- 担保链/互保圈风险排查

## 执行步骤

### 1. 关联因子采集
- 采集以下维度的关联因子：
  - **通讯关联**：手机号、紧急联系人号码、通讯录共现关系
  - **设备关联**：设备ID、MAC地址、Wi-Fi热点BSSID
  - **网络关联**：IP地址、GPS经纬度（模糊化至小区级别）
  - **地址关联**：户籍地址、居住地址、工作地址、收货地址
  - **资金关联**：银行卡号、收款账户、历史转账对手方
  - **企业关联**：股权结构、法人代表、监事、高管交叉任职

### 2. 图谱构建与扩展
- 以当前申请人为中心节点，基于关联因子向外扩展1-3度关系
- 将关联实体（人、设备、地址、企业等）和关联关系存入图数据库
- 对每条关联边标注关联类型、关联强度和关联时间

### 3. 异常模式识别
- **团伙申请检测**：短时间窗口内（如7天），多个申请人共享≥2个关联因子（如同一设备+同一Wi-Fi），触发团伙告警
- **中介代办检测**：同一设备或同一IP在短期内提交多笔不同人的申请
- **互保圈检测**：多个借款人相互担保形成闭环（A担保B，B担保C，C担保A）
- **壳公司检测**：企业注册地址关联大量企业（同一地址注册>10家公司），企业成立时间短（<1年）且快速申请大额贷款
- **关联人风险传导**：申请人的1度关联人中存在逾期、欺诈、黑名单记录

### 4. 风险评估输出
- 输出关联图谱可视化数据（节点、边、标签）
- 标注发现的异常关联模式和具体证据
- 计算关联风险评分，纳入反欺诈综合评分

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| applicantIdentifiers | object | 是 | 申请人各维度标识信息 |
| deviceInfo | object | 是 | 设备和网络信息 |
| corporateInfo | object | 否 | 企业信息（企业信贷时必填） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| graphData | object | 关联图谱数据（节点和边的列表） |
| anomalyPatterns | array | 发现的异常关联模式列表 |
| associationRiskScore | number | 关联风险评分（0-100） |
| relatedRiskEntities | array | 关联的风险实体（黑名单人/逾期客户等） |

## 最佳实践
- 关联扩展深度需根据场景控制：常规扫描1-2度，深度调查可扩展至3度
- 图谱数据应定期清理过期关联（如超过2年未活跃的关联边）
- 关联分析结果应结合业务场景解读，避免过度依赖纯数据关联产生误判

## 约束条件
- 关联分析中涉及的个人信息须遵循《个人信息保护法》最小必要原则
- 通讯录数据使用须获得客户明确授权
- 图谱数据访问须有权限控制，避免未授权的数据查阅
- 分析结果不得用于征信查询以外的目的
