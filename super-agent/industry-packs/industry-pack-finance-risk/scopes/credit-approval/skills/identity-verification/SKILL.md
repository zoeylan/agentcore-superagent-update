# 身份核验

## 技能概述
本技能对贷款申请人执行多维度身份真实性核验，通过活体检测、权威数据源比对和证件防伪检测，确认申请人身份的真实性和一致性，是反欺诈检测的第一道防线，主要防范身份冒用（第三方欺诈）和虚假身份（第一方欺诈）两类风险。

## 适用场景
- 个人信贷申请人的身份真实性验证
- 企业信贷的法定代表人/实际控制人身份核验
- 担保人的身份核验
- 贷款签约环节的身份再次确认

## 执行步骤

### 1. 活体检测结果校验
- 获取客户在申请环节完成的活体检测结果（人脸识别+动作指令）
- 校验活体检测的真实性：检测是否存在照片翻拍攻击、视频回放攻击、3D面具攻击、深度伪造（Deepfake）攻击的痕迹
- 验证活体检测照片与身份证照片的人脸比对相似度（阈值≥85%）
- 记录活体检测的设备信息、地理位置和时间戳

### 2. 权威数据源比对
- **公安二要素核验**：将姓名+身份证号码发送至公安部公民身份信息核查系统，确认一致性
- **公安三要素核验**（如需要）：增加人脸照片比对，适用于高额贷款或高风险标记的申请
- **手机号三要素核验**：姓名+身份证号+手机号的运营商实名制信息一致性比对
- **银行卡四要素核验**：姓名+身份证号+银行卡号+预留手机号的一致性校验

### 3. 证件防伪检测
- 身份证有效期校验（不接受已过期或即将在贷款期限内过期的身份证）
- 身份证照片质量和防伪特征检测（国徽、长城水印、荧光标记等）
- 检测证件图片是否经过PS修改（像素一致性分析、EXIF信息检测）
- 营业执照真伪验证（通过国家企业信用信息公示系统交叉核实）

### 4. 核验结果汇总
- 生成身份核验综合评分（基于各维度核验结果加权计算）
- 标记未通过的核验维度及具体原因
- 对核验异常的情况生成风险标签（如"人脸比对低于阈值"、"证件疑似伪造"等）

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicantName | string | 是 | 申请人姓名 |
| idNumber | string | 是 | 身份证号码 |
| idCardImages | object | 是 | 身份证正反面图片URL |
| livenessResult | object | 是 | 活体检测结果数据 |
| phoneNumber | string | 是 | 手机号码 |
| bankCardNumber | string | 否 | 银行卡号（如有） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| verificationScore | number | 身份核验综合评分（0-100） |
| livenessCheck | object | 活体检测结果（通过/未通过及原因） |
| authorityCheck | object | 权威数据源比对结果（各维度明细） |
| forgeryCheck | object | 证件防伪检测结果 |
| riskFlags | array | 风险标记列表 |
| overallResult | string | 综合结论：PASS/FAIL/REVIEW |

## 最佳实践
- 活体检测与证件核验应在同一会话中完成，避免时间差被利用
- 核验失败的具体原因不应直接展示给客户（防止欺诈者针对性优化）
- 对于权威数据源响应超时的情况，应设置合理的重试机制（最多3次，间隔10秒）
- 所有核验请求和响应数据须加密传输和存储

## 约束条件
- 权威核验接口调用需遵循数据最小化原则，仅查询必要字段
- 人脸照片数据在核验完成后不得长期存储原始图片（存储特征值即可）
- 核验结果须保留至少5年供审计追溯
- 每次征信查询前必须确认已获得有效授权
