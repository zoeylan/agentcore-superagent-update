# 审批决策

## 技能概述
本技能综合前序环节（进件预处理、反欺诈核查、信用评估、合规审查）的全部评估结果，做出最终的贷款审批决策，确定贷款的最终要素（额度、利率、期限、还款方式），并根据金额和风险等级判定是否需要上报审批委员会。

## 适用场景
- 个人信贷在授权额度内的自动审批决策
- 超授权额度的贷款申请的审批建议书生成
- 企业信贷的审批委员会提案准备
- 有条件批准后补件完成的续审决策

## 执行步骤

### 1. 审批要素汇总
- 汇集全部前序评估结果：
  - 进件预处理：资料完整性、准入结果、风险标签
  - 反欺诈核查：欺诈评分、命中规则、风险等级
  - 信用评估：信用评分、风险等级、建议额度/利率、还款能力评级
  - 合规审查：合规检查结果、不通过项（如有）
- 检查各环节数据的一致性和完整性

### 2. 决策规则执行
- **直接批准条件**：合规审查通过 + 风险等级A/B + 个人信贷金额≤30万 → 自动批准
- **有条件批准**：合规审查通过但风险缓释不足 → 生成补充条件（增加担保人/提供抵押物/降额等）
- **直接拒绝条件**：合规审查不通过 / 秒拒标记 / 反欺诈高风险拦截 → 拒绝
- **上报审批委员会**：个人信贷>30万 / 风险等级C/D且金额较大 / 全部企业信贷 → 生成审批委员会提案

### 3. 最终贷款要素确定
- 确认最终批准额度（不超过信用评估建议的上限且不超过客户申请额度）
- 确认最终利率（在风险定价建议区间内取值）
- 确认还款方式和期限
- 生成还款计划模拟（月供金额、总利息、总还款额）

### 4. 审批决策书生成
- 生成标准化审批决策书，包含：
  - 决策结论（批准/有条件批准/拒绝/上报委员会）
  - 最终贷款要素（如批准）
  - 决策理由摘要（综合各环节的关键结论）
  - 风险提示（需关注的风险点）
  - 附加条件（如有条件批准）
- 记录审批决策的审计轨迹（时间、决策者/系统、依据）

### 5. 审批一致性保障
- 执行审批一致性规则：同一申请在不同时间点审批应得到一致结论
- 定期抽取历史案例进行一致性测试（通过率目标≥95%）
- 对边界案例（评分接近分界线的申请）标记"边界案例"标签，建议人工复核

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| intakeResult | object | 是 | 进件预处理结果 |
| fraudCheckResult | object | 是 | 反欺诈核查结果 |
| creditAssessmentReport | object | 是 | 信用评估报告 |
| complianceReviewResult | object | 是 | 合规审查结果 |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| decision | string | 审批决策：APPROVED/CONDITIONAL/REJECTED/COMMITTEE |
| approvedTerms | object | 批准的贷款要素（如适用） |
| conditions | array | 附加条件列表（如适用） |
| rejectionReason | object | 拒贷理由（如适用） |
| decisionRationale | string | 决策理由摘要 |
| auditTrail | object | 审计轨迹信息 |
| committeeProposal | object | 审批委员会提案（如需上报） |

## 最佳实践
- 审批决策应充分体现"审贷分离"原则
- 拒贷理由应合规且友好，既满足客户知情权又不泄露模型细节
- 审批决策书应作为法律文件级别管理，确保100%可追溯
- 自动审批占比目标为个人信贷的70%以上

## 约束条件
- 不得超越授权权限进行审批
- 不得修改前序环节的评估结论
- 审批决策100%留痕，禁止事后篡改
- 拒贷通知不得包含歧视性内容
