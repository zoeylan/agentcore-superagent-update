# 放款执行

## 技能概述
本技能负责将审批通过的贷款指令安全、准确地推送至核心银行系统完成资金划付，并全程监控放款状态直至资金到账。放款执行是资金安全的最后一道关卡，任何差错都将直接导致资金损失。

## 适用场景
- 审批通过后的常规放款执行
- 有条件批准的条件满足后的放款
- 放款失败后的重试处理
- 批量放款（如白名单预审批客户）

## 执行步骤

### 1. 放款前置检查
- **账户验证**：
  - 核实收款账户与签约预留账户一致
  - 查询账户状态（正常/冻结/销户）
  - 对公贷款确认账户户名与借款企业名称一致
  - 受托支付确认交易对手账户信息
- **合同校验**：
  - 电子签章有效性验证（CA证书有效期、签署人身份）
  - 贷款合同核心要素（金额、利率、期限）与审批决策一致
  - 附件完整性（担保合同、抵押合同、保险保单等）
- **抵押登记确认**（抵押贷款）：
  - 确认抵押登记已完成
  - 登记信息与合同一致（抵押物、金额、权利人）
- **额度校验**：
  - 本笔放款未超过审批额度
  - 未触及机构日放款总限额

### 2. 放款指令生成与提交
- 按照核心银行系统接口规范生成放款指令报文
- 指令要素：借据号、放款金额（精确到分）、放款账户、收款账户、利率、起息日、到期日、还款方式
- 指令提交前执行金额、账号的二次校验
- 提交指令至核心银行系统并获取受理回执

### 3. 放款状态监控
- 实时轮询放款状态（提交→受理→处理中→成功/失败）
- 放款成功：记录资金到账时间和交易流水号
- 放款失败：记录失败原因代码和描述
- 监控超时：个人信贷放款超过30分钟未返回结果触发告警

### 4. 异常处理
- 放款失败自动重试：最多3次，间隔5分钟
- 3次重试均失败：触发人工介入告警，状态更新为"放款异常"
- 常见失败原因处理：
  - 账户异常 → 通知客户更换账户
  - 额度不足 → 等待额度释放后重试
  - 系统故障 → 等待系统恢复后重试
  - 合规拦截 → 反馈合规审批Agent核查

### 5. 放款后处理
- 生成借据和还款计划
- 更新贷款台账（在贷余额、户数等）
- 触发征信信息报送流程
- 同步贷款信息至贷后监控系统

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| approvalDecision | object | 是 | 审批决策书（含最终贷款要素） |
| contractInfo | object | 是 | 合同信息（含签约状态） |
| disbursementAccount | object | 是 | 放款/收款账户信息 |
| collateralRegistration | object | 否 | 抵押登记信息（抵押贷款时必填） |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| disbursementStatus | string | 放款状态：SUCCESS/FAILED/PENDING/EXCEPTION |
| transactionId | string | 交易流水号（成功时） |
| disbursementTime | string | 放款到账时间 |
| loanNoteId | string | 借据编号 |
| repaymentSchedule | array | 还款计划 |
| failureReason | string | 失败原因（失败时） |
| retryCount | number | 重试次数 |

## 最佳实践
- 放款指令提交应采用幂等设计，避免重复放款
- 大额放款（>100万）建议增加人工复核确认环节
- 放款高峰期（如月末）应提前确认系统容量
- 放款成功后立即冻结审批额度，防止重复使用

## 约束条件
- 放款金额与审批金额必须完全一致（精确到分）
- 非工作时间（系统维护窗口）不得执行放款
- 放款指令和执行结果须完整保留
- 单笔放款超时上限为4小时（个人信贷）
