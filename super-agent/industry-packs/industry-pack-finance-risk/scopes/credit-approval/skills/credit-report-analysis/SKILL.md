# 征信报告分析

## 技能概述
本技能负责调取并深度解析央行征信中心的个人/企业征信报告，从信贷记录、逾期历史、担保信息、公共信息等多个维度提取关键风险指标，为信用评分和还款能力分析提供核心数据输入。征信报告是信用评估的基石数据源，解析准确率直接影响评分质量和审批决策的可靠性。

## 适用场景
- 个人信贷申请的征信报告解析
- 企业信贷申请的企业征信报告解析
- 担保人的征信资质评估
- 信用评估中的征信数据补充查询

## 执行步骤

### 1. 征信报告调取
- 确认征信授权书有效（签署日期在30天内、授权范围覆盖本次查询）
- 通过征信系统接口提交查询请求（个人征信/企业征信）
- 验证返回报告的完整性（各段落齐全、时间戳正确）
- 如征信系统不可用，将申请放入等待队列（个人信贷超时2小时、企业信贷超时8小时后告警）

### 2. 信贷记录解析
- 提取所有信贷账户信息：贷款类型、放贷机构、授信额度、已用额度、贷款余额、贷款状态
- 计算信贷汇总指标：总授信额度、总已用额度、授信使用率、在贷笔数、信用卡张数
- 区分正常账户、关注账户、非正常账户（逾期/呆账/核销等）
- 提取贷款发放日期和到期日期，分析存量贷款的期限结构

### 3. 逾期历史分析
- 提取近24个月的月度还款状态（"1"到"7"代表不同逾期天数，"N"正常，"C"结清等）
- 计算关键逾期指标：
  - 最近24个月最长连续逾期次数
  - 最近24个月累计逾期次数
  - 历史最高逾期金额
  - 最近一次逾期的时间距今天数
- 逾期严重程度分级：无逾期 > 偶发逾期（24个月内≤2次且无连续逾期） > 频繁逾期（>2次或有连续逾期） > 严重逾期（有"4"以上或呆账记录）

### 4. 担保信息分析
- 提取对外担保记录：被担保人、担保金额、担保类型、担保状态
- 计算对外担保总余额和担保代偿风险
- 标记担保链风险（多人互保、担保金额过大等）

### 5. 公共信息与特殊记录
- 提取公共信息：欠税记录、民事判决记录、强制执行记录、行政处罚记录
- 提取特殊记录：关注信息、异议标注、本人声明
- 对存在公共负面信息的申请标记高风险标签

### 6. 结构化输出
- 将所有解析结果结构化为标准数据格式
- 标注征信评分（如央行提供的评分）
- 生成征信风险摘要（关键风险点的简洁描述）

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| applicantName | string | 是 | 申请人姓名 |
| idNumber | string | 是 | 身份证号码/统一社会信用代码 |
| queryType | string | 是 | 查询类型：personal/corporate |
| authorizationId | string | 是 | 征信授权书编号 |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| creditScore | number | 央行征信评分（如有） |
| loanSummary | object | 信贷记录汇总 |
| delinquencyAnalysis | object | 逾期历史分析结果 |
| guaranteeSummary | object | 担保信息汇总 |
| publicRecords | array | 公共信息记录 |
| riskSummary | string | 征信风险摘要 |
| rawReportId | string | 原始报告存档编号 |

## 最佳实践
- 征信报告解析准确率必须达到99.5%以上，关键字段（逾期记录、余额）需双重校验
- 同一客户的多次查询应合并处理，避免产生不必要的征信查询记录
- 征信报告数据不得修改、不得外传，仅用于本次信贷审批决策
- 解析结果应包含原始数据引用，方便人工复核时溯源

## 约束条件
- 每次征信查询必须有对应的有效授权书
- 征信数据的使用严格限于信贷审批用途
- 征信报告原始数据须加密存储，访问须有审计日志
- 征信查询次数本身会影响客户信用，应避免不必要的重复查询
