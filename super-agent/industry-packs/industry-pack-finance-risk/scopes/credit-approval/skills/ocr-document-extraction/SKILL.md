# OCR文档信息提取

## 技能概述
本技能负责对信贷申请中提交的各类纸质/电子材料进行OCR（光学字符识别）和NLP（自然语言处理）智能提取，将非结构化的证照、流水、证明文件转化为结构化的数据字段，为后续审批环节提供可靠的数据基础。

## 适用场景
- 个人信贷申请中的身份证、户口本、结婚证等证照识别
- 银行流水（纸质打印件或PDF电子版）的交易明细提取与收入汇总
- 房产证、车辆登记证的产权信息提取
- 企业营业执照、财务报表、纳税申报表的关键信息提取
- 征信授权书的签署信息和授权范围提取

## 执行步骤

### 1. 材料分类与预处理
- 接收上传的材料文件（支持JPG/PNG/PDF/TIFF格式）
- 通过文档分类模型自动识别材料类型（身份证/营业执照/银行流水/房产证/收入证明等）
- 执行图片预处理：倾斜校正、去噪、对比度增强、分辨率归一化

### 2. OCR识别与字段提取
- 调用对应材料类型的专用OCR模板进行识别
- 身份证：提取姓名、性别、民族、出生日期、住址、身份证号码、签发机关、有效期
- 营业执照：提取企业名称、统一社会信用代码、法定代表人、注册资本、成立日期、经营范围、营业期限
- 银行流水：逐行提取交易日期、摘要、交易金额（收入/支出）、余额，并自动计算月均收入、月均支出
- 房产证：提取权利人、坐落地址、面积、用途、土地使用权取得方式

### 3. 置信度评估与人工复核标记
- 对每个提取字段标注OCR置信度分数（0-100）
- 置信度≥90%的字段标记为"高置信"，可直接使用
- 置信度在70%-90%的字段标记为"中置信"，建议交叉核验
- 置信度<70%的字段标记为"低置信"，必须人工复核
- 整体文档中有任何字段低于90%置信度时，在申请记录中标注"OCR需复核"标签

### 4. 交叉核验
- 将OCR提取的关键字段（姓名、身份证号、企业名称等）与申请表填报数据进行自动比对
- 标记不一致的字段及差异描述（如"身份证地址与申请表填写地址不一致"）
- 生成交叉核验报告，附带一致/不一致/无法比对的明细

### 5. 防伪初筛
- 对证照类材料执行基础防伪检测：PS痕迹检测、翻拍检测、有效期校验
- 对营业执照的统一社会信用代码执行校验位验证
- 标记可疑材料并附上检测证据（如PS区域标注）

## 输入规格
| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| applicationId | string | 是 | 贷款申请编号 |
| documents | array | 是 | 材料文件列表，每项包含fileUrl、fileName、uploadTime |
| applicationType | string | 是 | 申请类型：personal/corporate |

## 输出规格
| 字段 | 类型 | 描述 |
|------|------|------|
| extractedData | object | 各材料的结构化提取结果 |
| confidenceReport | array | 每个字段的置信度评分 |
| crossCheckResult | object | 交叉核验结果（一致/不一致/无法比对） |
| forgeryFlags | array | 防伪检测标记（如有异常） |
| needsManualReview | boolean | 是否需要人工复核 |
| reviewFields | array | 需人工复核的具体字段列表 |

## 最佳实践
- 对模糊不清的材料，优先要求客户重新上传而非强行识别
- 银行流水建议要求提供近6个月完整流水，仅有部分月份时标记数据不完整
- 企业财务报表应同时提取资产负债表、利润表、现金流量表三表数据
- 所有OCR原始识别结果和材料原件必须留存，满足审计追溯要求

## 约束条件
- 不得修改OCR识别结果以"优化"数据质量
- 不得在置信度低于70%时直接使用提取结果
- 单笔申请材料处理总耗时不超过5分钟（个人信贷）/ 15分钟（企业信贷）
- 材料原件和OCR结果须保存至少5年（满足监管审计要求）
