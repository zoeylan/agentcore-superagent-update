# 催收合规监控技能 (Collection Compliance Monitoring)

## 技能概述

催收合规监控技能是催收执行Agent (collection-executor) 的合规守护核心，负责对所有催收活动进行实时、全量的合规性检测与监控。本技能依据《民法典》合同编、《个人信息保护法》、银保监会催收行为规范及机构内部催收合规制度，对催收时间窗口、联系频次、话术内容、第三方联系禁令、暂停催收保护等五大合规维度进行全方位监控，确保每一个催收动作均在合规框架内执行。

催收合规是不可逾越的底线。任何合规违规不仅面临监管处罚风险（罚款、业务限制），更可能引发客户投诉、媒体曝光和声誉损失。本技能通过"事前预检 + 事中拦截 + 事后审计"三层防线，将合规违规率控制在最低水平。

### 业务价值

- 实时合规拦截可将催收投诉率降低60%以上
- 合规自动化监控替代人工抽检，检查覆盖率从20%提升至100%
- 合规违规事件的平均发现时间从T+1（次日抽检）缩短至实时（秒级）

### 适用场景

| 场景 | 说明 |
|------|------|
| 催收动作事前预检 | 在每个催收动作执行前进行合规性校验，不通过则拦截 |
| 催收过程实时监控 | AI外呼和人工通话过程中实时检测话术合规性 |
| 每日合规巡检 | 每日定时扫描全部催收记录，发现事前/事中未捕获的违规 |
| 合规预警生成 | 发现违规行为时实时生成预警通知相关人员 |
| 合规报告产出 | 按日/周/月生成合规监控报告，支撑管理层决策 |
| 催收员合规评分 | 对催收员的合规表现进行量化评分，纳入绩效考核 |

## 详细执行步骤

### 第一层防线：事前预检 (Pre-Execution Check)

每个催收动作在执行前必须通过以下合规预检，任一项不通过则拦截该动作：

#### Check 1: 催收时间窗口校验

```
规则: 所有催收动作（短信、电话、APP推送）仅允许在 08:00-21:00 之间执行
检测逻辑:
  current_time = 获取当前系统时间（精确到分钟）
  IF current_time < 08:00 OR current_time >= 21:00:
    BLOCK 催收动作
    RETURN {compliant: false, violation: "COMP-TIME-001", message: "当前时间不在催收允许时间窗口(08:00-21:00)内"}
  ELSE:
    PASS
```

**特殊说明**：
- 定时任务（如短信批量发送）必须将执行时间设定在 08:00-20:30 之间，预留30分钟缓冲
- 时区统一使用 UTC+8（北京时间），跨时区客户按客户所在地时间判断
- 节假日无特殊规则，与工作日相同（08:00-21:00）

#### Check 2: 每日联系频次校验

```
规则: 单个案件单日催收联系总次数（含所有渠道）不得超过 3次
检测逻辑:
  today_contacts = 查询该案件今日已执行的催收联系次数
  统计范围: SMS发送(成功) + 电话拨打(含未接通) + APP推送(已送达)
  IF today_contacts >= 3:
    BLOCK 催收动作
    RETURN {compliant: false, violation: "COMP-FREQ-001", message: "该案件今日催收联系已达3次上限"}
  ELSE:
    remaining = 3 - today_contacts
    PASS with WARNING if remaining == 1: "该案件今日仅剩1次联系机会，请合理安排"
```

**频次计算规则**：
| 渠道 | 计次标准 | 说明 |
|------|---------|------|
| SMS | 每条发送成功计1次 | 发送失败不计次 |
| AI外呼 | 每次拨打计1次 | 含未接通，因为对客户产生了打扰 |
| 人工电话 | 每次拨打计1次 | 含未接通 |
| APP推送 | 每条送达计1次 | 未送达不计次 |

> **注意**：承诺还款提醒短信不纳入频次计算（属于客户确认的服务性通知），但仍需遵守时间窗口规则。

#### Check 3: 案件状态校验

```
规则: 以下状态的案件禁止执行任何催收动作
检测逻辑:
  case_status = 查询案件当前状态
  blocked_statuses = ["投诉中", "协商中", "法务冻结", "暂停催收", "已结清"]
  IF case_status IN blocked_statuses:
    BLOCK 催收动作
    RETURN {
      compliant: false,
      violation: "COMP-STATUS-001",
      message: "案件当前状态为'{case_status}'，禁止催收",
      resume_condition: "等待状态变更为'正常催收'"
    }
  ELSE:
    PASS
```

#### Check 4: 客户联系许可校验

```
规则: 不得联系借款人以外的无关第三方
检测逻辑:
  contact_target = 获取本次催收联系对象
  IF contact_target == "借款人本人":
    PASS
  ELIF contact_target == "紧急联系人":
    IF 联系目的 == "转达还款通知":
      PASS with RESTRICTION: "仅可通知转达还款信息，不得施加任何形式的压力"
    ELSE:
      BLOCK
  ELSE:
    BLOCK
    RETURN {compliant: false, violation: "COMP-CONTACT-001", message: "禁止联系借款人以外的无关第三方"}
```

#### Check 5: 话术模板校验

```
规则: 催收话术必须使用经合规审核通过的标准模板
检测逻辑:
  template_id = 获取本次使用的话术模板ID
  IF template_id NOT IN approved_template_library:
    BLOCK
    RETURN {compliant: false, violation: "COMP-SCRIPT-001", message: "话术模板未经合规审核通过"}
  ELIF template_is_expired(template_id):
    BLOCK
    RETURN {compliant: false, violation: "COMP-SCRIPT-002", message: "话术模板已过期，请使用最新版本"}
  ELSE:
    PASS
```

### 第二层防线：事中实时监控 (Real-Time Monitoring)

针对AI外呼和人工通话的全程实时监控：

#### Monitor 1: AI外呼话术实时检测

1. AI外呼过程中实时分析对话内容，检测以下违规关键词/语义：

| 违规类型 | 检测规则 | 严重等级 | 处置方式 |
|----------|---------|---------|---------|
| 威胁恐吓 | 包含"起诉你"、"上门催收"、"通知你单位"等威胁性表述 | **严重** | 立即终止通话 |
| 侮辱性语言 | 包含侮辱、歧视、人身攻击类词汇 | **严重** | 立即终止通话 |
| 虚假信息 | 虚构法律后果、伪称公检法、虚报逾期金额 | **严重** | 立即终止通话 |
| 隐私泄露 | 向非本人透露借款人逾期信息 | **严重** | 立即终止通话 |
| 过度施压 | 反复强调后果、不给客户回应时间 | 中等 | 实时调整对话策略 |
| 偏离模板 | 对话内容偏离审批话术框架 | 低 | 记录并纳入事后审计 |

2. AI外呼的情绪识别联动：
   - 当客户情绪识别为"愤怒"且持续超过15秒，自动触发通话终止
   - 通话终止后自动标记案件为"敏感案件"，暂停24小时后续催收

#### Monitor 2: 人工通话合规监控

1. 人工通话启用实时语音转文本 (ASR)，对转写文本进行合规关键词检测
2. 检测到潜在违规时：
   - 低风险：在催收员界面弹出合规提示（黄色预警）
   - 中风险：弹出强制确认提示，催收员需点击确认后继续（橙色预警）
   - 高风险：向组长实时推送预警，必要时系统强制插入"合规话术引导"语音
3. 通话录音全量保存，保留期限不少于2年

#### Monitor 3: 通话时长监控

```
规则:
  - AI外呼单次通话不超过5分钟
  - 人工电话单次通话建议不超过15分钟
  - 超时后系统发出提醒，但不强制终止（人工电话）
```

### 第三层防线：事后审计 (Post-Execution Audit)

#### Audit 1: 每日全量合规扫描

每日 22:00 自动执行全量合规扫描：

1. **时间合规回查**：扫描全部催收记录，确认无 21:00 后执行的催收动作
2. **频次合规回查**：按案件维度统计每日联系次数，标记超限案件
3. **话术合规回查**：对当日所有AI外呼和人工通话录音进行离线NLP分析，检测遗漏的违规话术
4. **状态合规回查**：检查是否有"投诉中"/"协商中"案件被催收
5. **第三方联系回查**：检查是否有联系非借款人本人的记录

#### Audit 2: 催收员合规评分

按月对每位催收员生成合规评分卡：

```
催收员合规评分卡
================
催收员ID: COL-2025-0088
评分周期: 2025年4月
总催收量: 856次

合规评分维度:
  时间合规: 100分 (0次违规)
  频次合规: 98分 (2次当日超频, 均为系统延迟导致)
  话术合规: 95分 (4次轻微偏离模板, 无严重违规)
  状态合规: 100分 (0次违规)
  联系对象合规: 100分 (0次违规)

综合合规评分: 98.6分 (优秀)
合规等级: A

改进建议:
  - 注意在高峰时段的催收频次管控, 建议间隔操作
  - 话术偏离模板的4次情况已记录, 建议加强标准话术培训
```

## 最佳实践与约束

### 最佳实践

1. **预检优先原则**：所有催收动作必须先通过事前预检再执行，严禁绕过合规检查直接执行催收
2. **宁严勿松**：合规判断的边界情况（如20:58的催收电话）应从严处理，预留安全余量
3. **违规零容忍**：严重违规（威胁恐吓、隐私泄露、时间窗口外催收）实行零容忍，触发即处理
4. **录音全量保存**：所有通话录音保留不少于2年，作为合规审计和争议处理的证据
5. **合规培训联动**：将合规监控发现的共性问题纳入催收员培训计划，从源头降低违规风险
6. **技术容错设计**：对系统层面可能导致的合规风险（如定时任务延迟）设置充足的安全缓冲
7. **投诉关联分析**：将合规监控数据与客户投诉数据关联分析，识别高投诉风险的催收模式

### 约束条件

| 约束项 | 要求 | 说明 |
|--------|------|------|
| 预检响应时间 | < 200ms | 不可因合规检查显著延迟催收动作执行 |
| 实时监控延迟 | < 2秒 | 话术违规检测到告警的端到端延迟 |
| 每日审计时间 | 22:00-次日06:00 | 不占用业务高峰期系统资源 |
| 录音保留期 | >= 2年 | 监管要求最低保留期限 |
| 合规率目标 | >= 99.5% | 低于此值触发管理层专项关注 |
| 严重违规上报 | 实时 | 严重违规事件必须实时通知合规部门和业务负责人 |
| 合规报告产出 | 日报T+0，月报T+3工作日 | 确保管理层及时获取合规信息 |

### 合规法规依据

| 法规 | 相关条款 | 对应合规检查维度 |
|------|---------|------------------|
| 《民法典》合同编 | 债务追索权利与限制 | 联系对象、催收方式 |
| 《个人信息保护法》 | 个人信息使用限制 | 隐私保护、第三方联系禁令 |
| 银保监会催收行为规范 | 催收时间、频次、话术 | 时间窗口、频次、话术合规 |
| 《互联网金融个人网络消费信贷贷后催收风控指引》 | 全面催收行为规范 | 全部五大维度 |

## 示例

### 示例1：事前预检通过

**场景**：系统准备在14:30对案件CASE-001发起AI外呼，今日已发送1条短信。

```
预检请求:
  案件: CASE-001
  动作: AI_CALL
  时间: 14:30
  模板: TMPL-AI-M1-003
  联系对象: 借款人本人

预检结果:
  [PASS] 时间窗口: 14:30在08:00-21:00范围内
  [PASS] 联系频次: 今日已联系1次, 未超过3次上限
  [PASS] 案件状态: 正常催收
  [PASS] 联系对象: 借款人本人
  [PASS] 话术模板: TMPL-AI-M1-003审核通过, 有效期至2025-06-30

  总体结果: 合规通过, 允许执行
  提示: 今日剩余2次联系机会
```

### 示例2：事前预检拦截

**场景**：催收员在20:55准备对案件CASE-002拨打催收电话，该案件今日已联系3次。

```
预检请求:
  案件: CASE-002
  动作: MANUAL_CALL
  时间: 20:55
  联系对象: 借款人本人

预检结果:
  [PASS] 时间窗口: 20:55在08:00-21:00范围内 (但触发预警: 距窗口关闭仅5分钟)
  [FAIL] 联系频次: 今日已联系3次, 已达上限!
    - 09:15 SMS发送成功
    - 11:30 AI外呼(未接通)
    - 15:40 人工电话(已接通, 客户承诺还款)
  [PASS] 案件状态: 正常催收
  [PASS] 联系对象: 借款人本人

  总体结果: 合规不通过, 拦截执行
  拦截原因: 该案件今日催收联系已达3次上限(COMP-FREQ-001)
  建议: 明日08:00后再安排联系
```

### 示例3：实时监控触发预警

**场景**：人工催收通话过程中，催收员话术偏离合规范围。

```

  会话ID: CALL-2025-0413-01234
  催收员: COL-0088
  案件: CASE-003
  通话时间: 10:15-10:22

  [10:15:30] 正常: "您好，这里是XX金融客户服务中心..."
  [10:16:45] 正常: "您的贷款有一笔逾期款项需要处理..."
```
