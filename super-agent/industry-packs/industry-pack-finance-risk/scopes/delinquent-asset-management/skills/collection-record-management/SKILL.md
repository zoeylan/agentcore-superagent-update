# 催收记录管理技能 (Collection Record Management)

## 技能概述

催收记录管理技能负责对催收执行Agent (collection-executor) 所有催收交互进行结构化记录、标准化存储和智能化分析。每一次催收触达——无论是短信发送、AI外呼、APP推送还是人工电话——都必须形成完整、规范的催收记录，涵盖联系时间、渠道、沟通摘要、客户态度评估、承诺详情和下次跟进计划。本技能是催收数据闭环的基石，为下游的智能分案优化、催收数据分析和合规审计提供高质量的数据支撑。

### 业务价值

- 标准化的催收记录使智能分案Agent的策略匹配准确率提升15-20%
- 结构化的承诺跟踪将承诺还款履约率提升10-15%（通过及时提醒机制）
- 完整的催收记录链为合规审计和争议处理提供不可篡改的证据链

### 适用场景

| 场景 | 说明 |
|------|------|
| 自动化催收记录生成 | SMS、AI外呼、APP推送等自动化渠道的催收结果自动记录 |
| 人工催收记录辅助 | 辅助催收员快速录入结构化通话记录，减少手工录入负担 |
| 承诺还款跟踪 | 记录客户承诺还款详情，管理到期提醒和履约核查 |
| 催收时间线展示 | 按时间轴展示案件完整催收历程，支撑催收员快速了解案件全貌 |
| 数据标准化输出 | 将催收记录按标准格式输出至智能分案Agent和催收数据分析Agent |
| 催收摘要自动生成 | 基于历史催收记录自动生成案件催收摘要，辅助M2人工催收 |

## 详细执行步骤

### Step 1: 催收记录数据采集

#### 1.1 自动化渠道记录采集

对于SMS、AI外呼和APP推送等自动化催收渠道，系统自动采集催收执行数据并生成记录：

**SMS记录自动采集字段：**
| 字段 | 来源 | 说明 |
|------|------|------|
| 发送时间 | 短信网关 | 精确到秒 |
| 模板ID | 催收执行技能 | 使用的短信模板编号 |
| 送达状态 | 短信网关回执 | 成功/失败/号码无效/黑名单拦截 |
| 失败原因 | 短信网关 | 如发送失败则记录原因代码 |

**AI外呼记录自动采集字段：**
| 字段 | 来源 | 说明 |
|------|------|------|
| 拨打时间 | 外呼系统 | 精确到秒 |
| 接通状态 | 外呼系统 | 已接通/未接通/占线/关机/空号 |
| 通话时长 | 外呼系统 | 秒数 |
| 对话转写文本 | ASR引擎 | 全量通话转写 |
| 情绪识别结果 | 情绪分析引擎 | 客户情绪标签及置信度 |
| 意图识别结果 | NLU引擎 | 客户意图分类（愿意还款/需要协商/拒绝还款/争议/其他） |
| 承诺信息提取 | NLU引擎 | 从对话中自动提取的承诺还款金额和日期 |

**APP推送记录自动采集字段：**
| 字段 | 来源 | 说明 |
|------|------|------|
| 推送时间 | 推送服务 | 精确到秒 |
| 送达状态 | 推送回执 | 已送达/未送达 |
| 打开状态 | APP埋点 | 是否点击打开 |
| 后续行为 | APP埋点 | 是否点击还款入口/查看账单详情 |

#### 1.2 人工催收记录辅助录入

人工催收电话结束后，系统提供结构化录入界面，辅助催收员快速完成记录：

1. **通话基础信息**（系统自动填充）：
   - 催收员ID、案件ID、拨打时间、通话时长
   - 通话录音文件关联

2. **沟通内容录入**（催收员填写，系统辅助）：
   - 沟通摘要：系统基于通话录音ASR转写自动生成摘要草稿，催收员确认或修改
   - 客户态度评估：下拉选择（配合/一般配合/消极/抵触/愤怒/其他）
   - 客户陈述要点：自由文本（如客户提到的困难原因、还款计划等）
   - 承诺还款信息：结构化录入（金额、日期、方式）
   - 是否需要转交处理：是否转交客户沟通Agent或升级处理

3. **下次跟进计划**（催收员制定）：
   - 下次跟进日期和时间
   - 跟进方式（电话/短信/其他）
   - 跟进重点说明

### Step 3: 客户态度评估标准

对客户态度进行标准化评估，确保不同催收员和渠道的评估结果具有可比性：

| 态度分类 | 评分范围 | 判定标准 | 后续策略建议 |
|----------|---------|---------|-------------|
| 配合 | 80-100 | 客户知晓逾期、态度友好、主动提出还款计划 | 正常跟踪承诺还款 |
| 一般配合 | 60-79 | 客户知晓逾期、愿意沟通但未给出明确还款计划 | 强化引导，提供还款方案选项 |
| 消极 | 40-59 | 客户沉默或敷衍回应、不主动配合但未明确拒绝 | 调整沟通策略，尝试找到客户痛点 |
| 抵触 | 20-39 | 客户明确表示不愿还款或对催收行为不满 | 降低催收频率，评估是否转交客户沟通Agent |
| 愤怒 | 0-19 | 客户情绪激动、言语激烈、可能威胁投诉 | 立即停止催收，标记敏感案件，转交客户沟通Agent |

**AI外呼自动评估**：基于情绪识别引擎和NLU意图分析自动生成态度评分，置信度 < 70% 时标记为"需人工复核"。

**人工催收辅助评估**：催收员选择态度分类后，系统根据通话录音ASR分析进行交叉验证，偏差过大时提示催收员复核。

### Step 4: 承诺还款跟踪管理

#### 4.1 承诺信息录入

每当客户做出还款承诺时，必须录入以下结构化信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 承诺类型 | 是 | 全额还款 / 部分还款 / 分期首期 / 最低还款额 |
| 承诺金额 | 是 | 精确到分 |
| 承诺日期 | 是 | 客户承诺的还款截止日期 |
| 承诺还款方式 | 否 | APP/银行转账/线下还款等 |
| 承诺置信度 | 是 | 高/中/低（由催收员或AI综合判断） |

#### 4.2 承诺到期提醒触发

```
提醒规则:
  承诺日期 - 3天: 无操作 (仅内部监控)
  承诺日期 - 1天: 发送提醒短信 (使用承诺提醒专用模板)
    模板: "尊敬的客户，温馨提醒您于{promise_date}有一笔{amount}元还款到期，
           请提前做好资金安排。还款请登录APP操作。"
  承诺日期当天 20:00: 系统自动检查还款入账
    IF 已入账:
      更新承诺状态为"已履约"
      更新案件状态
    ELSE:
      更新承诺状态为"待确认"
      继续监控至承诺日期+1天 12:00
  承诺日期 + 1天 12:00:
    IF 仍未入账:
      更新承诺状态为"未履约"
      触发下一轮催收动作
      记录违约历史，影响客户承诺置信度评估
```

#### 5.2 案件催收摘要自动生成

当案件从M1升级至M2、或催收员首次接手案件时，系统基于催收时间线自动生成催收摘要：

```
自动生成逻辑:
1. 统计M1阶段催收数据:
   - 总触达次数、成功联系次数、接通率
   - 各渠道触达分布
   - 客户态度变化趋势
2. 提取关键信息:
   - 客户陈述的逾期原因
   - 承诺还款历史及履约情况
   - 特殊标注（失联/拒绝/争议等）
3. 生成策略建议:
   - 基于客户态度和历史响应，推荐M2阶段沟通策略
   - 基于承诺履约记录，评估客户可信度
   - 建议话术要点和注意事项
```

### Step 6: 数据质量管控

#### 6.1 数据完整性校验

每条催收记录入库前进行数据质量校验：

| 校验项 | 规则 | 处理方式 |
|--------|------|---------|
| 必填字段完整性 | 所有必填字段不可为空 | 拒绝入库，提示补充 |
| 时间逻辑性 | 结束时间 > 开始时间 | 自动修正或标记异常 |
| 金额合理性 | 承诺金额 <= 逾期总额的120% | 超出则标记复核 |
| 态度评分一致性 | AI评分与催收员评分偏差 < 30分 | 偏差过大则提示复核 |
| 去重校验 | 同一案件同一时间不可有重复记录 | 拒绝重复入库 |

#### 6.2 数据标准化输出

催收记录按以下标准接口输出至下游系统：

| 下游系统 | 输出内容 | 输出频率 | 格式 |
|----------|---------|---------|------|
| 智能分案Agent | 联系结果、客户态度、承诺信息 | 实时 | JSON API |
| 催收数据分析Agent | 全量催收记录 | 实时 + 日批量 | JSON API / CSV |
| 合规审计系统 | 带录音关联的完整记录 | 日批量 | JSON + Audio Link |
| 核心信贷系统 | 还款承诺和履约状态 | 实时 | JSON API |

## 最佳实践与约束

### 最佳实践

1. **即时记录原则**：催收记录应在催收动作完成后立即生成（自动化渠道）或5分钟内录入（人工渠道），避免事后补录导致信息遗漏
2. **摘要精准简洁**：沟通摘要应包含关键信息（客户态度、原因陈述、承诺详情），控制在200字以内，避免冗余描述
3. **态度评估客观化**：客户态度评估应基于客户实际言行，避免催收员主观偏见。AI辅助评估与人工评估交叉验证机制可提升评估准确性
4. **承诺信息完整性**：客户做出的每一个还款承诺都必须记录金额、日期和方式，模糊承诺（如"过几天还"）应引导客户明确具体日期
5. **跟进计划必填**：每条催收记录都必须包含明确的下次跟进计划，确保催收流程不出现"断档"
6. **数据不可篡改**：催收记录一旦生成不可删除，修改操作需留存版本记录（audit trail）
7. **隐私保护**：催收记录中的客户个人信息（手机号、身份证号等）必须脱敏存储，仅在必要时按权限展示

### 约束条件

| 约束项 | 要求 | 说明 |
|--------|------|------|
| 记录生成时效 | 自动化渠道：实时；人工渠道：<= 5分钟 | 超时提醒催收员尽快录入 |
| 必填字段 | 不可为空 | 系统强制校验 |
| 数据保留期 | >= 5年 | 监管要求及争议处理需要 |
| 录音保留期 | >= 2年 | 合规审计要求 |
| 数据质量评分 | >= 90分 | 低于90分的记录标记为"需补充" |
| 承诺跟踪覆盖 | 100% | 每个承诺必须设置到期提醒 |
| 下游同步时效 | 实时接口 < 5秒，批量接口 < T+1 | 确保下游系统数据及时性 |
| 并发写入 | 同一案件同一时刻仅允许一条记录写入 | 乐观锁机制防止数据冲突 |

### 示例4：数据质量问题处理

**场景**：催收员录入记录时数据质量校验发现问题。

```

  案件: CASE-2025-0413-00567

  沟通摘要: "客户同意还款"  ← 过于简短
  态度评估: 配合(90分)  ← AI评估为55分, 偏差35分
  承诺金额: ¥150,000  ← 逾期总额仅¥8,000
  下次跟进: 未填写  ← 必填字段

数据质量校验结果 (总分: 52分, 不合格):

  [问题1] 沟通摘要过于简短 (最低50字, 当前5字)
    提示: 请补充客户态度、逾期原因、具体还款安排等关键信息
    建议: 点击"AI辅助生成"按钮, 系统将根据通话录音自动生成摘要草稿

  [问题2] 态度评分偏差过大
    催收员评分: 90分(配合)
    AI分析评分: 55分(消极)
    偏差: 35分 > 阈值30分
    提示: 请复核客户态度评估, AI分析显示客户通话中有多次沉默和回避性回答

    逾期总额: ¥8,000
    比值: 1875% > 阈值120%
    提示: 承诺金额远超逾期总额, 请确认是否录入错误

  [问题4] 下次跟进计划缺失

  操作: 拒绝入库, 要求催收员修正后重新提交
```
