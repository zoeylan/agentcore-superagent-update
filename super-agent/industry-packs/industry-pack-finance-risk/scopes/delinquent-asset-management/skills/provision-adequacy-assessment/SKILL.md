# 拨备充足性评估技能 (Provision Adequacy Assessment)

## 一、技能概述

### 1.1 技能定位

拨备充足性评估是催收数据分析Agent（collection-analytics）的监管合规与财务决策支持技能，负责基于逾期资产的阶段分布和预期损失模型，评估金融机构当前贷款损失准备金（Loan Loss Provision）是否充足。这是连接逾期资产治理运营数据与财务报表、监管合规之间的关键桥梁。

拨备不足意味着机构低估了信用风险敞口，可能导致利润虚增、资本充足率不达标，严重时触发监管处罚甚至系统性风险；拨备过多则意味着过度谨慎，压缩了机构的盈利空间和资本使用效率。本技能的核心目标是帮助机构在**合规底线**和**经营效率**之间找到最优平衡点。

### 1.2 核心能力

- **预期信用损失（ECL）计算**：基于 PD（违约概率）x LGD（违约损失率）x EAD（违约风险敞口）的标准框架计算各M-stage的预期损失
- **拨备覆盖率评估**：计算并监控拨备覆盖率（Provision Coverage Ratio），确保满足监管底线 >= 150%
- **拨备缺口/盈余分析**：量化当前拨备与应提拨备之间的差额，提出调整建议
- **拨备敏感性分析**：评估关键参数（PD/LGD）变动对拨备需求的影响
- **监管报表支撑**：为银保监会相关监管报表提供拨备充足性的数据基础

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 月度/季度拨备评估 | 定期评估拨备充足性，提出拨备调整建议 |
| 监管报表编制 | 为银保监会1104报表体系中的资产质量和拨备相关报表提供数据 |
| 年度财务审计 | 支持外部审计师对预期信用损失和拨备计提合理性的审查 |
| IFRS 9 / 新金融工具准则合规 | 按三阶段模型（Stage 1/2/3）计算预期信用损失 |
| 资本充足率管理 | 拨备水平直接影响核心一级资本充足率（CET1 Ratio）的计算 |
| 经营层决策 | 帮助管理层理解拨备变动对利润和资本的影响 |

## 二、执行步骤

### 步骤1：逾期资产阶段分布映射

**目标**：将逾期资产台账数据按拨备计提所需的分类体系进行映射。

**分类体系**：

1. **五级分类映射**（监管报表口径）：

| 五级分类 | M-stage 映射 | 说明 |
|----------|-------------|------|
| 正常（Normal） | M0 | 未逾期，按时还款 |
| 关注（Special Mention） | M1（DPD 1-30天） | 存在可能影响偿还的不利因素 |
| 次级（Substandard） | M2-M3（DPD 31-90天） | 还款能力出现明显问题 |
| 可疑（Doubtful） | M3+（DPD 91-360天） | 即使采取措施，也可能损失部分本金 |
| 损失（Loss） | DPD 360+或已核销 | 在合理手段后仍无法收回，或收回极少 |

2. **IFRS 9 三阶段映射**（财务报表口径）：

| IFRS 9 阶段 | 信用风险变化 | ECL 计算范围 | 典型映射 |
|-------------|------------|-------------|---------|
| Stage 1 | 初始确认后信用风险未显著增加 | 12个月ECL | M0 + 部分M1 |
| Stage 2 | 信用风险显著增加但未减值 | 整个存续期ECL | 部分M1 + M2 |
| Stage 3 | 已发生信用减值 | 整个存续期ECL | M3 + M3+ + 核销 |

### 步骤2：PD/LGD/EAD 参数估计

**目标**：估计预期信用损失（ECL）计算所需的三大核心参数。

**参数定义与估计方法**：

#### PD（Probability of Default，违约概率）

定义：在给定时间范围内，借款人从当前状态转为违约状态的概率。

```
各M-stage的PD估计（基于历史12个月迁徙率数据）：

M-stage    12个月PD    计算依据
M0         3.2%       M0→M1→...→Default 的累计迁徙概率
M1         42.5%      M1→M2→...→Default 的累计迁徙概率
M2         65.8%      M2→M3→...→Default 的累计迁徙概率
M3         82.3%      M3→M3+→Default 的累计迁徙概率
M3+        92.5%      M3+→Default 的累计迁徙概率
核销        100%       已确认违约
```

**PD估计方法**：
- **历史迁徙率法**：基于历史迁徙率矩阵，通过矩阵乘法计算多步累计违约概率
  - PD_12m = 1 - (1 - Roll_Rate_1) × (1 - Roll_Rate_2) × ... × (1 - Roll_Rate_n)
- **前瞻性调整（Forward-Looking Adjustment）**：
  - IFRS 9 要求 PD 须反映前瞻性信息（宏观经济预期）
  - 调整方法：PD_adjusted = PD_historical × (1 + β × Δ宏观指标)
  - 其中 β 为宏观敏感系数，通过历史回归估计

#### LGD（Loss Given Default，违约损失率）

定义：违约发生后，实际损失金额占违约时风险敞口的比例。

```
LGD = 1 - Recovery Rate (违约后回收率)

各类贷款的LGD估计：

贷款类型        有无抵押   LGD估计    置信区间(95%)    数据来源
信用卡          无抵押    75-85%     [72%, 88%]      近3年核销回收数据
个人消费贷      无抵押    70-80%     [67%, 83%]      近3年核销回收数据
住房按揭贷款    有抵押    25-40%     [22%, 43%]      近3年核销回收+抵押物处置数据
小微经营贷      部分抵押  50-65%     [47%, 68%]      近3年核销回收数据
汽车分期        有抵押    35-50%     [32%, 53%]      近3年核销回收+车辆处置数据
```

**LGD估计方法**：
- **实际回收法（Workout LGD）**：基于已核销/已违约贷款的实际回收数据
  - LGD = 1 - Σ(回收现金流的现值) / EAD_at_default
  - 回收现金流须折现至违约时点（使用原始有效利率或合同利率）
- **抵押物估值调整**：有抵押贷款须考虑抵押物当前市值、处置折扣率和处置周期
  - LGD = 1 - min(抵押物市值 × 处置折扣率 / EAD, 1)

#### EAD（Exposure at Default，违约风险敞口）

定义：违约时点的预期风险敞口金额。

```
各类贷款的EAD计算：

贷款类型        EAD计算方法
定期贷款        EAD = 当前未偿还本金余额 + 应计利息
信用卡          EAD = 已使用额度 + CCF × 未使用额度
                (CCF = Credit Conversion Factor，信用转换系数，通常50-75%)
循环贷款        EAD = 当前余额 + CCF × (授信额度 - 当前余额)
```

### 步骤3：预期信用损失（ECL）计算

**目标**：基于PD × LGD × EAD计算各阶段和总体的预期信用损失。

**计算公式**：

```
ECL = PD × LGD × EAD

总ECL = Σ(各贷款的ECL) = Σ(PD_i × LGD_i × EAD_i)
```

**按M-stage汇总的ECL计算**：

```

M-stage   EAD(亿)   PD       LGD      ECL(亿)    ECL率
M0        430.50    3.2%     78.0%     10.75      2.5%
M1        6.80      42.5%    78.0%     2.25       33.1%
M2        2.85      65.8%    78.0%     1.46       51.3%
M3        1.43      82.3%    78.0%     0.92       64.2%
M3+       3.15      92.5%    78.0%     2.27       72.1%
核销       0.99     100.0%    85.0%     0.84       85.0%
──────────────────────────────────────────────────────
合计      445.72    --        --       18.49      4.15%
```

**IFRS 9分阶段ECL**：

```
IFRS 9 ECL计算：

阶段         EAD(亿)    ECL(亿)    ECL覆盖率   说明
Stage 1      435.20     11.08      2.5%        12个月ECL
Stage 2      5.95       2.58       43.4%       存续期ECL
Stage 3      4.57       4.83       72.3%*      存续期ECL（含核销在催）
──────────────────────────────────────────────────
合计         445.72     18.49      4.15%

*Stage 3的ECL覆盖率高于PD×LGD的简单计算，因包含了个案评估调整
```

### 步骤4：拨备覆盖率计算与监管合规检查

**目标**：计算拨备覆盖率，检查是否满足监管要求。

**核心监管指标**：

1. **拨备覆盖率（Provision Coverage Ratio）**：

```
拨备覆盖率 = 贷款损失准备金余额 / 不良贷款余额 × 100%

监管红线：≥ 150%

当前状况：
  贷款损失准备金余额：15.80亿元
  不良贷款余额（次级+可疑+损失）：8.42亿元
  拨备覆盖率 = 15.80 / 8.42 × 100% = 187.6%

  ✅ 满足监管要求（≥ 150%）
  安全边际：187.6% - 150% = 37.6个百分点
```

2. **贷款拨备率（Loan Provision Ratio）**：

```
贷款拨备率 = 贷款损失准备金余额 / 贷款总余额 × 100%

监管参考值：≥ 2.5%（部分监管机构要求）

当前状况：
  贷款损失准备金余额：15.80亿元
  贷款总余额：445.72亿元
  贷款拨备率 = 15.80 / 445.72 × 100% = 3.54%

  ✅ 满足要求（≥ 2.5%）
```

3. **拨备缺口/盈余分析**：

```
拨备充足性评估：

评估维度              应提拨备(亿)   已提拨备(亿)   缺口/盈余(亿)   状态
ECL模型计算           18.49         15.80         -2.69          ⚠️ 略有不足
监管底线(150%)        12.63         15.80         +3.17          ✅ 充足
五级分类比例法         14.25         15.80         +1.55          ✅ 充足
```

### 步骤5：前瞻性拨备需求预测

**目标**：基于回收预测和迁徙率预测，预估未来3个月的拨备需求变动。

**预测方法**：

1. **基于迁徙率的不良贷款预测**：
   - 使用最新迁徙率矩阵，预测未来各月的不良贷款余额
   - 不良贷款余额变动 = 新增不良 - 回收金额 - 核销金额

2. **基于ECL的拨备需求预测**：
   - 使用预测的资产阶段分布，重新计算ECL
   - 拨备需求变动 = 新的ECL - 当前ECL

3. **前瞻性调整**：
   - 纳入宏观经济预期（GDP增速预测、失业率预测等）
   - 当宏观环境预期恶化时，上调PD参数

```
未来3个月拨备需求预测：

月份        预计不良余额(亿)  预计ECL(亿)  拨备缺口变动(亿)  拨备覆盖率预估
2026-04     8.65(+2.7%)      18.95         -3.15           182.7%
2026-05     8.82(+2.0%)      19.35         -3.55           179.1%
2026-06     8.95(+1.5%)      19.68         -3.88           176.5%

趋势判断：
  不良贷款余额预计持续小幅上升，拨备覆盖率从187.6%逐步下降至176.5%，
  仍在150%监管红线之上，但安全边际从37.6pp收窄至26.5pp。
  建议在Q2末评估是否需要增提拨备。
```

### 步骤6：拨备调整建议与报告

**目标**：综合评估结果，输出拨备调整建议和管理报告。

**调整建议框架**：

1. **合规达标建议**：确保拨备覆盖率持续满足 >= 150% 的监管底线
2. **风险审慎建议**：基于ECL模型计算的应提拨备与实际拨备的差异
3. **前瞻预防建议**：考虑未来资产质量变化趋势的预防性增提建议
4. **利润影响测算**：拨备调整对当期利润表的影响

**报告结构**：

```

一、评估结论（交通灯评级）
  拨备覆盖率：187.6% → 🟢 绿灯（≥180%: 充足）
  ECL覆盖评估：85.4% → 🟡 黄灯（<100%: 略有缺口）
  前瞻趋势：下行 → 🟡 黄灯（安全边际收窄中）
  综合评级：🟡 黄灯——当前达标但存在未来压力

二、调整建议
  方案A（保守方案）：增提拨备2.69亿元，使拨备完全覆盖ECL
  方案B（均衡方案）：增提拨备1.50亿元，维持拨备覆盖率在190%以上
  方案C（审慎方案）：暂不调整，但在Q2末（6月）重新评估

  理由：兼顾风险审慎和利润影响，增提1.50亿元后拨备覆盖率约为205%，
        安全边际充足，同时对当期利润影响可控（税前利润影响约-1.50亿元）。
```

## 四、最佳实践与约束

### 4.1 监管合规硬性要求

- **拨备覆盖率 >= 150%**：这是银保监会的监管红线，低于此标准将面临监管约谈或处罚，**任何拨备调整建议不得导致拨备覆盖率低于此底线**
- **贷款拨备率**：部分银保监局要求 >= 2.5%，需根据所在辖区的具体要求执行
- **前瞻性要求**：根据IFRS 9 / 新金融工具准则，ECL计算必须纳入前瞻性信息（Forward-Looking Information），不得仅依赖历史数据
- **五级分类准确性**：拨备计提的基础是贷款分类，分类错误将直接导致拨备计提不准确；须与信贷审查部门定期核对分类结果

### 4.2 参数估计要求

- **PD参数**：
  - 须基于至少 **36个月** 的历史数据估计
  - 须进行前瞻性调整，反映当前宏观经济环境
  - 须每季度重新估计一次
  - 不同产品类型须分别估计PD（不得使用统一PD）

- **LGD参数**：
  - 须基于实际回收数据（Workout LGD），不得仅使用监管规定的标准值
  - 回收现金流须折现至违约时点
  - 有抵押贷款须考虑抵押物当前市值，不得使用初始评估值
  - 须考虑经济周期效应（Downturn LGD）

- **EAD参数**：
  - 循环信贷（信用卡、循环贷款）须包含未使用额度的信用转换系数（CCF）
  - CCF须基于历史数据估计，通常在50%-75%之间

### 4.3 数据质量要求

- **数据准确率 >= 99.5%**：拨备评估直接影响财务报表和监管报表，对数据准确性要求极高
- **双重校验**：所有对外报送的拨备数据须经过双重校验（模型计算 + 人工审核）
- **审计可追溯**：ECL计算的每一步须有完整的审计轨迹（Audit Trail），包括参数来源、计算过程、调整依据
- **数据一致性**：拨备评估使用的贷款台账数据须与看板监控（collection-dashboard-monitoring）技能使用的数据完全一致

### 4.4 约束与限制

- 本技能提供拨备充足性的**评估和建议**，实际拨备的计提和调整须经过财务部门和风控委员会的正式审批流程
- 拨备调整建议须同时考虑利润影响，不得仅从风险角度单方面建议大幅增提而不评估对利润和资本的影响
- 参数估计中的"管理层覆盖"（Management Overlay）——即基于专家判断的主观调整——须有明确的依据说明和审批记录，不得滥用
- 本技能不替代外部审计师对ECL计算合理性的独立判断
- 与回收预测技能（recovery-forecasting）存在紧密的数据依赖：回收预测直接影响LGD参数和前瞻性拨备需求
