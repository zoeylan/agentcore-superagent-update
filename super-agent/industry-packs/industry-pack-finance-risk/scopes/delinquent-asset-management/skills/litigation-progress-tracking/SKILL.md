# 诉讼进度跟踪技能 (Litigation Progress Tracking)

## 技能概述

本技能用于对已进入诉讼程序的M3+逾期案件进行全生命周期的进度跟踪与管理，覆盖从立案到最终执行完毕的每一个关键节点。通过系统化的进度监控、截止日期管理和异常预警机制，确保诉讼案件在每个阶段按时推进，及时发现和处理停滞案件及执行困难。

诉讼进度跟踪是法务处置Agent的"过程管理"核心技能，与前端的法务可行性评估、诉讼材料准备形成完整的闭环——评估决定是否诉讼，材料准备启动诉讼，进度跟踪确保诉讼执行到位。

### 适用场景

| 场景 | 说明 |
|------|------|
| 诉讼案件日常管理 | 对在诉案件进行日常进度跟踪，确保每个节点按时推进 |
| 月度进度汇报 | 每月生成全量在诉案件的进度报告，向管理层汇报诉讼回收进展 |
| 停滞案件预警 | 自动识别超出正常周期的案件，触发预警并推动问题解决 |
| 执行困难案件管理 | 对进入执行阶段但遭遇执行障碍的案件进行专项跟踪 |
| 律师/律所绩效评估 | 基于案件进度数据评估委托律师/律所的办案效率 |
| 诉讼时效与截止日期管理 | 管理上诉期限、执行申请期限等关键法律时效 |

## 执行步骤

### 步骤一：诉讼全生命周期阶段定义

每个诉讼案件按以下标准阶段进行跟踪：

```
立案(Filing) → 送达(Service) → 举证(Evidence Exchange) → 调解(Mediation)
→ 开庭(Hearing) → 判决(Judgment) → [上诉(Appeal)] → 执行申请(Enforcement Application)
→ 强制执行(Compulsory Enforcement) → 执行完毕/终本(Completion/Suspension)
```

各阶段标准周期与关键动作：

| 阶段 | 标准周期 | 关键动作 | 预警阈值 |
|------|----------|----------|----------|
| 立案 | 7个工作日 | 提交材料、缴纳诉讼费、获取案号 | 超过10个工作日未获案号 |
| 送达 | 15-60天 | 法院向被告送达起诉状副本（直接送达/邮寄送达/公告送达） | 公告送达启动（周期延长60天） |
| 举证 | 15-30天 | 举证期限届满、证据交换 | 对方提出新证据需要补充回应 |
| 调解 | 视情况 | 法院组织调解，记录调解结果 | 调解超过30天无结果 |
| 开庭 | 排期后30天内 | 出庭应诉、法庭辩论 | 排期超过60天 |
| 判决 | 简易程序3个月/普通程序6个月 | 收到判决书、核对判决内容 | 超出审限未判决 |
| 上诉期 | 判决送达后15天 | 评估是否上诉、对方是否上诉 | 上诉期即将届满（剩余3天） |
| 执行申请 | 判决生效后2年内 | 向执行法院提交执行申请书 | 距执行申请时效届满不足3个月 |
| 强制执行 | 6-12个月 | 法院执行查控、拍卖、划扣 | 执行超过6个月无回款 |
| 终本/完结 | - | 执行回款到账或裁定终结本次执行 | - |

### 步骤二：案件进度录入与更新

1. **进度信息采集来源**：
   - 委托律师定期反馈（每周/每两周）
   - 法院公告系统查询（中国裁判文书网、审判流程信息公开网）
   - 法院电话/现场查询
   - 执行信息公开平台查询

2. **进度更新记录要素**：

```json
{
  "case_id": "案件编号",
  "court_case_number": "法院案号（如：(2026)X民初XXXX号）",
  "update_date": "更新日期",
  "current_stage": "当前阶段",
  "stage_details": "阶段详情描述",
  "next_action": "下一步待办事项",
  "next_deadline": "下一个截止日期",
  "handler": "经办人/律师",
  "remarks": "备注说明"
}
```

3. **更新频率要求**：
   - 关键节点（立案、开庭、判决、执行回款）：**实时更新**（当日内）
   - 常规进度：**每月至少更新一次**
   - 停滞案件：**每两周更新一次**，直至恢复正常推进

### 步骤三：关键里程碑监控

为每个案件设置以下关键里程碑并监控达成情况：

| 里程碑 | 监控指标 | 达标标准 |
|--------|----------|----------|
| 成功立案 | 立案时效 | 材料提交后7个工作日内获取案号 |
| 成功送达 | 送达方式与时效 | 直接送达/邮寄送达15天内完成 |
| 完成开庭 | 开庭排期效率 | 立案后90天内完成首次开庭（简易程序） |
| 获得胜诉判决 | 判决结果 | 全部或主要诉讼请求获法院支持 |
| 判决生效 | 是否有上诉 | 上诉期满无上诉或二审维持原判 |
| 申请执行 | 执行申请时效 | 判决生效后30天内提交执行申请 |
| 首笔执行回款 | 执行到账时效 | 执行申请后3个月内有首笔回款 |
| 执行完毕 | 执行到位率 | 判决金额全额执行到位 |

### 步骤四：停滞案件预警与处理

**自动预警规则：**

| 预警级别 | 触发条件 | 处理要求 |
|----------|----------|----------|
| 黄色预警 | 当前阶段停留时间超过标准周期的1.5倍 | 联系律师/法院了解原因，制定推进计划 |
| 橙色预警 | 当前阶段停留时间超过标准周期的2倍 | 升级至法务主管关注，评估是否更换律师/策略调整 |
| 红色预警 | 关键时效即将届满（上诉期、执行申请期）或连续3个月无任何进展 | 紧急处理，法务主管直接介入，必要时启动替代方案 |

**常见停滞原因及对策：**

| 停滞原因 | 对策 |
|----------|------|
| 送达困难（被告地址不明） | 申请公告送达；通过工商、社保等渠道补充地址信息 |
| 法院排期积压 | 与法院沟通争取提前排期；对紧急案件申请优先审理 |
| 被告提出管辖权异议 | 配合律师及时应对，准备管辖权异议答辩材料 |
| 执行无可供执行财产 | 申请终结本次执行，纳入定期恢复执行监控名单 |
| 被执行人申请破产 | 及时申报债权，参与破产程序 |
| 被执行人提出执行异议 | 配合律师准备异议答辩，必要时申请执行异议之诉 |

### 步骤五：执行阶段专项跟踪

执行阶段是诉讼回款的关键环节，需要进行更精细化的跟踪：

1. **执行措施跟踪**：
   - 网络查控结果（冻结/划扣了哪些账户、金额多少）
   - 不动产查封情况（是否成功查封、评估拍卖进展）
   - 车辆查封扣押情况
   - 限制高消费令、失信被执行人名单公布情况
   - 司法拘留申请情况

2. **执行回款记录**：
   - 每笔回款的时间、金额、来源
   - 累计回款金额与判决金额的对比
   - 剩余未执行金额

3. **终结本次执行(终本)管理**：
   - 终本案件纳入定期恢复执行监控
   - 每季度查询被执行人财产变动情况
   - 发现新财产线索时及时申请恢复执行

### 步骤六：进度报告生成

**月度诉讼进度报告**包含以下内容：

```
一、案件总览
  - 在诉案件总数及各阶段分布
  - 本月新增立案数
  - 本月结案数（判决生效/执行完毕/调解结案）
  - 在诉案件总标的额

二、关键指标
  - 平均诉讼周期（立案至判决）
  - 胜诉率（本月判决案件）
  - 执行到位率（本月执行案件）
  - 诉讼回款金额（本月实际到账）

三、停滞案件清单
  - 黄色预警案件列表及原因
  - 橙色预警案件列表及处理进展
  - 红色预警案件列表及应急措施

四、重点案件进展
  - 大标的案件（>100万元）单独列示
  - 有重大进展的案件（判决、回款等）

五、下月工作计划
  - 预计开庭案件清单
  - 预计判决案件清单
  - 需要关注的时效截止日期

六、律师/律所绩效摘要
  - 各委托律师/律所的案件数及推进效率
  - 绩效排名与改进建议
```

## 最佳实践与约束

### 最佳实践

1. **进度更新不过夜**：关键节点（立案确认、开庭结果、判决送达、执行回款）的进度信息应在获知当日完成系统更新，避免信息滞后影响决策。

2. **律师沟通机制标准化**：与委托律师建立固定的进度反馈机制——每周五提交案件周报（模板化），关键节点当日电话/邮件通知。对反馈不及时的律师进行提醒和记录。

3. **终本案件不放弃**：终结本次执行（终本）不等于案件终结。应将终本案件纳入定期监控名单，每季度通过执行查控系统查询被执行人财产变动，一旦发现新增财产及时申请恢复执行。

4. **数据驱动的律师评估**：基于进度跟踪数据建立律师绩效评估模型，主要指标包括：案件推进效率（各阶段实际周期vs标准周期）、胜诉率、执行到位率、进度反馈及时性。绩效数据作为律师续聘和案件分配的依据。

5. **时效管理前置预警**：对所有法律时效（上诉期15天、执行申请期2年、恢复执行期限等）设置多级提前提醒：到期前30天、15天、7天、3天分别触发提醒。

### 约束条件

| 约束项 | 要求 |
|--------|------|
| 更新频率 | 诉讼进度须 **月度更新** 至案件管理系统，关键节点实时更新 |
| 报告时效 | 月度进度报告须在次月 **第5个工作日** 前完成 |
| 预警响应 | 红色预警案件须在 **24小时** 内启动应急处理 |
| 时效管理 | 所有法律时效截止日期必须录入系统并设置提前提醒，**零遗漏** |
| 回款核验 | 执行回款到账后须在 **1个工作日** 内与财务确认并更新系统 |
| 数据准确性 | 进度信息须与法院系统/律师反馈交叉验证，准确率 >= 99% |
| 保密要求 | 诉讼策略信息、调解方案等属于机密信息，仅限授权人员查阅 |
