# 催收数据看板监控技能 (Collection Dashboard Monitoring)

## 一、技能概述

### 1.1 技能定位

催收数据看板监控是催收数据分析Agent（collection-analytics）的核心基础技能，负责构建和维护逾期资产治理的全景数据看板。该技能实现对各逾期阶段（M1/M2/M3+/核销）在管资产的实时追踪，为管理层提供一目了然的运营全貌，并通过异常检测机制实现关键指标变动的自动预警。

看板监控是所有其他分析技能的数据基座——迁徙率分析、催收效能评估、回收预测和拨备评估均依赖本技能产出的基础指标数据。

### 1.2 核心能力

- **多维度指标追踪**：按M-stage（M1/M2/M3/M3+/核销）追踪在管余额（Outstanding Balance）、在管户数（Case Count）、新增逾期（New Delinquency Inflow）、回收金额（Recovery Amount）、催收成本（Collection Cost）
- **多维度下钻分析**：支持按产品类型（Product Type）、客户分群（Customer Segment）、地区（Region）、放款年份/批次（Origination Vintage）进行交叉分析
- **日度监控简报**：每日自动生成关键指标变动摘要，标记异常事件
- **趋势可视化**：提供时间序列趋势图、环比/同比对比图、占比分布图等可视化输出

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 日常运营监控 | 每日早会前产出前一日关键指标变动简报，辅助管理层快速掌握逾期资产全貌 |
| 新增逾期归因分析 | 当新增逾期金额/户数出现异常波动时，通过多维下钻定位具体产品、客群或地区的问题来源 |
| 管理层汇报 | 为月度经营分析会、风控委员会提供标准化的逾期资产治理看板 |
| 跨部门数据对齐 | 作为逾期资产治理的"单一事实源"（Single Source of Truth），确保风控、催收、财务等部门的数据口径一致 |
| 监管报送准备 | 为银保监会/央行监管报表中的不良资产相关指标提供基础数据支撑 |

## 二、执行步骤

### 步骤1：数据采集与预处理

**目标**：从核心系统采集逾期资产原始数据，完成清洗和标准化处理。

**操作细节**：

1. **数据源接入**：
   - 核心信贷系统（Loan Management System）：获取贷款台账、还款记录、逾期状态
   - 催收管理系统（Collection System）：获取催收动作记录、回收入账明细
   - 财务系统：获取催收成本数据（人力成本、外包费用、诉讼费用等）
   - 客户信息系统：获取客群标签、地区信息

2. **数据清洗规则**：
   - 剔除测试账户和内部员工账户
   - 处理重复记录（以最新状态为准）
   - 统一逾期天数（DPD, Days Past Due）计算口径：以应还日次日开始计算
   - 金额字段精度统一为分（避免浮点数精度问题），展示时转换为元/万元

3. **M-stage 分级映射**：
   - M0：未逾期（Current）
   - M1：逾期1-30天（DPD 1-30）
   - M2：逾期31-60天（DPD 31-60）
   - M3：逾期61-90天（DPD 61-90）
   - M3+：逾期90天以上（DPD 90+），含M4（91-120天）、M5（121-150天）、M6（151-180天）
   - 核销（Write-off）：根据机构政策通常为DPD 180+或经审批核销

4. **数据校验**：
   - 总余额交叉验证：各M-stage在管余额之和 = 总逾期余额
   - 与财务总账数据核对，偏差须 < 0.01%
   - 数据时效性检查：确保数据截至T-1日（前一自然日）

### 步骤2：核心指标计算

**目标**：计算看板所需的全量核心指标。

**指标体系**：

```
一、规模指标
├── 在管余额（Outstanding Balance）：各M-stage的当前在管本金余额
├── 在管户数（Case Count）：各M-stage的当前在管账户数
├── 在管金额均值（Average Balance per Case）：在管余额 / 在管户数
├── 新增逾期金额（New Delinquency Inflow Amount）：当期新进入M1的本金余额
├── 新增逾期户数（New Delinquency Inflow Count）：当期新进入M1的账户数
└── 逾期率（Delinquency Rate）：M1+在管余额 / 总贷款余额

二、回收指标
├── 回收金额（Recovery Amount）：当期实际回收入账金额
├── 回收率（Recovery Rate）：回收金额 / 期初在管余额
├── 回收户数（Recovered Case Count）：当期完全结清的账户数
└── 净回收金额（Net Recovery）：回收金额 - 催收成本

三、成本指标
├── 催收总成本（Total Collection Cost）：包含人力、外包、系统、诉讼等全部成本
├── 单位催收成本（Cost per Dollar Collected）：催收总成本 / 回收金额
├── 成本收入比（Cost to Collect Ratio）：催收总成本 / 回收金额 × 100%
└── 分渠道成本（Cost by Channel）：各催收渠道的成本拆分
```

### 步骤3：多维度下钻分析

**目标**：对核心指标进行多维度交叉分析，揭示结构性特征。

**分析维度**：

1. **产品类型维度（Product Type）**：
   - 信用卡（Credit Card）
   - 个人消费贷（Personal Loan）
   - 住房按揭贷款（Mortgage）
   - 小微经营贷（SME Loan）
   - 汽车分期（Auto Loan）
   - 各产品的逾期率、回收率对比分析

2. **客户分群维度（Customer Segment）**：
   - 按风险评分（Risk Score）分群：高/中/低风险
   - 按授信额度区间分群
   - 按客户生命周期阶段分群：新客/存量/高价值客户
   - 首逾客户 vs 复逾客户分析

3. **地区维度（Region）**：
   - 省级/城市级逾期分布热力图
   - 区域逾期率排名及环比变动
   - 重点关注经济下行区域

4. **放款批次维度（Origination Vintage）**：
   - 按放款月份/季度进行Vintage分析
   - 同一放款批次在不同账龄（MOB, Month on Book）的逾期率走势
   - 识别资产质量较差的放款批次，反馈至审批策略

5. **时间维度**：
   - 日度/周度/月度数据粒度切换
   - 同比（YoY）/环比（MoM）/定基比对比
   - 滚动12个月趋势分析

### 步骤4：异常检测与预警

**目标**：自动识别关键指标的异常变动，及时触发预警。

**异常检测规则**：

| 指标 | 预警阈值 | 预警级别 |
|------|----------|----------|
| 新增逾期金额 | 日环比增长 > 20% | 黄色预警 |
| 新增逾期金额 | 日环比增长 > 50% | 红色预警 |
| M1在管余额 | 月环比增长 > 10% | 黄色预警 |
| M3+在管余额 | 月环比增长 > 5% | 红色预警 |
| 日回收金额 | 较近7日均值下降 > 30% | 黄色预警 |
| 逾期率（Delinquency Rate） | 月环比上升 > 0.5个百分点 | 红色预警 |
| 成本收入比 | 超过15% | 黄色预警 |
| 成本收入比 | 超过25% | 红色预警 |

**异常归因**：当触发预警时，自动进行多维度下钻定位异常来源：
- 是否集中在特定产品？
- 是否集中在特定地区？
- 是否与特定放款批次相关？
- 是否存在系统性因素（如宏观经济事件、政策变化）？

### 步骤5：看板渲染与简报生成

**目标**：将分析结果输出为可视化看板和文字简报。

**日度监控简报模板**：

```
【逾期资产日度监控简报】
报告日期：YYYY-MM-DD（数据截至T-1日）

一、全局概览
  - 总在管余额：XX亿元（环比 +/-X.X%）
  - 总在管户数：XX万户（环比 +/-X.X%）
  - 当日回收金额：XX万元（较7日均值 +/-X.X%）

二、各阶段分布
  | 阶段 | 在管余额 | 环比变动 | 在管户数 | 环比变动 |
  | M1   | XX亿    | +X.X%   | XX万户  | +X.X%   |
  | M2   | XX亿    | +X.X%   | XX万户  | +X.X%   |
  | M3   | XX亿    | +X.X%   | XX万户  | +X.X%   |
  | M3+  | XX亿    | +X.X%   | XX万户  | +X.X%   |

三、新增逾期
  - 当日新增M1：XX万元 / XX户（环比 +/-X.X%）
  - 近7日趋势：[趋势描述]

四、异常预警（如有）
  - [预警事项1]：[详细描述及初步归因]
  - [预警事项2]：[详细描述及初步归因]

  - [事项1]
  - [事项2]
```

## 三、输入/输出规格

### 3.1 输入

| 输入项 | 数据类型 | 是否必需 | 说明 |
|--------|----------|----------|------|
| loan_portfolio_data | DataFrame | 是 | 贷款台账数据，包含贷款编号、产品类型、放款日期、本金余额、逾期天数（DPD）、M-stage、客群标签、地区等字段 |
| repayment_records | DataFrame | 是 | 还款记录数据，包含还款日期、还款金额、还款类型（正常还款/催收回收/提前结清）等 |

| drill_down_results | JSON | 多维度下钻分析结果 |
| visualizations | List[Chart] | 可视化图表列表，包含趋势图、分布图、热力图等 |

## 四、最佳实践与约束

### 4.1 数据质量要求

- **准确率要求 >= 99.5%**：所有对外发布的指标数据必须经过双重校验（系统自动校验 + 人工抽查）
- **数据溯源**：每个指标必须可追溯至原始数据源，记录计算公式和中间过程
- **口径一致性**：与财务总账、监管报表的数据口径必须对齐，存在差异时必须注明调整说明
- **时效性**：日度简报须在每日 T+1 日上午 8:00 前产出；月度报告须在次月 T+3 个工作日内产出

### 4.2 看板设计原则

- **信息分层**：遵循"总览-分类-明细"三层结构，避免信息过载
- **变动优先**：优先展示变动幅度大的指标，静态数据弱化展示
- **对比清晰**：所有指标必须伴随环比/同比对比值，使用颜色标记方向（红色上升/绿色下降，逾期类指标上升为负面）
- **可操作性**：异常预警必须伴随初步归因和建议动作，而非仅展示数字

### 4.3 约束与限制

- 本技能仅负责数据监控和看板展示，**不直接参与催收策略调整或执行**，策略建议需流转至对应的执行Agent
- 涉及客户维度的明细数据展示须遵守《个人信息保护法》，看板中不展示客户个人敏感信息（姓名、身份证号、手机号等），仅展示脱敏后的聚合统计数据
- 对外报送数据（监管报表、管理层报告）须经过 **审批流程**，不得直接推送未经审核的数据
- 预警触发后的跟进处理不在本技能范围内，需由对应的处理流程接管

### 4.4 性能要求

- 日度看板数据刷新：全量指标计算应在 30 分钟内完成
- 多维度下钻查询响应时间 < 5 秒
- 看板并发访问支持 >= 50 用户

## 五、示例

### 示例1：日常运营监控

**场景**：每日早晨自动生成前一日的逾期资产监控简报。

**输入**：
```json
{
  "report_date": "2026-04-12",
  "comparison_period": "DoD"
}
```

**执行流程**：
1. 采集2026-04-12截止的贷款台账快照数据
2. 计算各M-stage在管余额和户数
3. 与2026-04-11数据对比计算环比变动
4. 统计当日新增M1逾期金额/户数
5. 汇总当日回收金额
6. 运行异常检测规则，检查是否触发预警
7. 生成日度监控简报

**输出（简报节选）**：
```
【逾期资产日度监控简报】2026-04-12

一、全局概览
  总在管余额：15.23亿元（环比 +0.8%）
  总在管户数：8.77万户（环比 +0.5%）
  当日回收金额：1,250万元（较7日均值 -3.2%）

二、异常预警
  [黄色预警] 新增M1逾期金额 3,500万元，日环比增长25.0%（阈值20%）
  初步归因：华东地区个人消费贷新增逾期激增，占增量的62%，
  建议关注该区域近期放款批次的资产质量。
```

### 示例2：管理层月度汇报

**场景**：月度风控委员会需要逾期资产治理全景报告。

**输入**：
```json
{
  "report_date": "2026-03-31",
  "comparison_period": "MoM",
  "dimension_filters": {},
  "include_vintage_analysis": true
}
```

**执行流程**：
1. 采集2026年3月全月数据
2. 计算月度全量指标（规模/回收/成本三大类）
3. 与2026年2月数据对比计算月环比
4. 与2025年3月数据对比计算同比
5. 按产品类型、客群、地区分别生成多维度分析
6. 生成Vintage分析图表（各放款批次的逾期率走势）
7. 输出结构化月度报告及配套图表

**输出（核心指标摘要）**：
```
2026年3月逾期资产治理月报

核心指标：
  逾期率（M1+）：3.42%（MoM +0.15pp, YoY -0.28pp）
  M1在管余额：6.80亿元（MoM +2.3%）
  M3+在管余额：3.15亿元（MoM -1.2%）[积极信号]
  月度回收金额：3.75亿元（MoM +5.1%）
  回收率：24.6%（MoM +0.8pp）
  成本收入比：8.7%（MoM -0.3pp）

关键发现：
  1. M1新增逾期持续走高，需关注近3个月放款批次的首逾表现
  2. M3+在管余额持续下降，法务催收渠道回收效果显著
  3. 华东地区逾期率上升至4.1%，高于全国均值0.7pp，建议专项关注
```

### 示例3：特定维度下钻分析

**场景**：管理层发现某月新增逾期异常上升，要求深入分析原因。

**输入**：
```json
{
  "report_date": "2026-03-31",
  "dimension_filters": {
    "m_stage": "M1",
    "metric": "new_inflow"
  },
  "drill_down_dimensions": ["product_type", "region", "origination_vintage", "customer_segment"]
}
```

**执行流程**：
1. 提取3月新增M1逾期的全量明细数据
2. 按产品类型分解：发现个人消费贷占新增M1的58%（较上月提升12pp）
3. 进一步按地区下钻：华东地区占个人消费贷新增M1的45%
4. 按放款批次分析：2025年Q4放款批次的MOB3逾期率达2.8%，显著高于历史同期的1.5%
5. 按客群分析：低风险评分（Score < 600）客群占比从22%上升至31%
6. 综合归因，输出分析结论和建议

**输出（归因分析结论）**：
```
新增M1逾期异常上升归因分析：

主因：2025年Q4放款批次资产质量偏弱
  - 该批次MOB3逾期率2.8%，为近12个月放款批次中最高
  - 主要集中在华东地区个人消费贷产品
  - 该批次低分客群（Score<600）占比31%，高于历史均值22%

建议：
  1. 联动审批策略团队，审查2025年Q4的审批政策是否存在准入标准放松
  2. 对该批次在管客户实施提前催收干预（Pre-delinquency Outreach）
  3. 后续放款批次恢复低分客群占比至25%以内的管控目标
```
