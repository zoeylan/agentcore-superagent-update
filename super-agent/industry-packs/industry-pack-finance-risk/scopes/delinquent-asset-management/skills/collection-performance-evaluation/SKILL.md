# 催收效能评估技能 (Collection Performance Evaluation)

## 一、技能概述

### 1.1 技能定位

催收效能评估是催收数据分析Agent（collection-analytics）的核心分析技能，负责从**渠道（Channel）、策略（Strategy）、人员（Collector）**三大维度系统化评估催收运营效果。该技能通过量化各环节的投入产出比，帮助管理层识别高效催收路径、淘汰低效策略，并为催收资源的精准配置提供数据依据。

本技能的独特价值在于：不仅展示"结果如何"，更揭示"为什么这样"以及"如何改进"——通过统计严谨的A/B测试评估和多维度绩效对比，将催收运营从"经验驱动"推向"数据驱动"。

### 1.2 核心能力

- **渠道效能分析**：评估各催收渠道（SMS短信/AI外呼/人工电话/上门催收/法务催收）的全链路转化效率和单位成本
- **策略效能分析**：评估不同催收策略组合（如渠道组合、触达时机、话术版本、催收力度分级）的ROI
- **人员绩效评估**：对催收人员进行多维度绩效评价，生成排名和绩效分布报告
- **A/B测试效果评估**：基于统计显著性检验（P < 0.05）的严格A/B测试评估，支持策略决策
- **成本效率优化建议**：基于分析结果提出催收资源配置优化建议

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 月度催收效能报告 | 全面评估各渠道、策略、人员的月度催收绩效，输出绩效排名和改进建议 |
| A/B测试结果评估 | 当催收策略团队实施A/B测试后，提供统计严谨的效果评估结论 |
| 催收资源配置优化 | 基于各渠道/策略的ROI分析，建议最优的资源分配方案 |
| 外包催收机构评价 | 评估外包催收机构的绩效表现，支持续约/淘汰决策 |
| 催收人员考核 | 为催收团队的绩效考核提供客观的数据支撑 |
| 新策略上线评估 | 新催收策略上线后的效果跟踪和快速评估 |

## 二、执行步骤

### 步骤1：催收漏斗指标计算

**目标**：构建催收全链路转化漏斗，计算各环节的核心效能指标。

**催收漏斗模型**：

```
案件分配(Assignment)
    ↓
触达尝试(Attempt)
    ↓                    ← 触达率(Contact Rate)
有效触达(Contact)
    ↓                    ← 承诺率(Promise-to-Pay Rate, PTP Rate)
客户承诺还款(Promise)
    ↓                    ← 履约率(Kept-Promise Rate, KP Rate)
实际回款(Recovery)       ← 回收率(Recovery Rate)
```

**核心指标定义**：

| 指标 | 英文名 | 计算公式 | 说明 |
|------|--------|----------|------|
| 触达率 | Contact Rate | 有效触达次数 / 触达尝试次数 × 100% | 反映渠道的可达性 |
| 右方联系率 | Right Party Contact (RPC) Rate | 联系到借款人本人次数 / 触达尝试次数 × 100% | 更严格的触达指标 |
| 承诺率 | Promise-to-Pay (PTP) Rate | 作出还款承诺的案件数 / 有效触达案件数 × 100% | 反映催收话术/沟通效果 |
| 履约率 | Kept-Promise (KP) Rate | 履行承诺还款的案件数 / 作出还款承诺的案件数 × 100% | 反映承诺的真实性和跟进效果 |
| 回收率 | Recovery Rate | 实际回收金额 / 期初在管余额 × 100% | 核心结果指标 |
| 结案率 | Resolution Rate | 结清案件数 / 期初在管案件数 × 100% | 反映案件处置完成度 |
| 单位催收成本 | Cost per Dollar Collected | 催收总成本 / 回收金额 | 每回收1元所花费的成本 |
| 渠道ROI | Channel ROI | (回收金额 - 渠道成本) / 渠道成本 × 100% | 渠道投入产出比 |

### 步骤2：渠道效能分析

**目标**：评估各催收渠道的效能差异，识别最优渠道组合。

**分析渠道**：

1. **SMS短信催收**：
   - 指标：送达率、阅读率（如可追踪）、触发还款率
   - 成本结构：短信平台费 + 系统开发维护成本
   - 适用阶段：M1早期提醒，成本最低但转化率较低

2. **AI外呼催收**：
   - 指标：接通率、完成对话率、PTP Rate、KP Rate
   - 成本结构：语音平台费 + AI模型运维成本
   - 适用阶段：M1-M2大批量触达，效率高但对复杂案件效果有限

3. **人工电话催收**：
   - 指标：接通率、RPC Rate、PTP Rate、KP Rate、单通时长
   - 成本结构：催收人员薪酬 + 话费 + 培训成本 + 管理成本
   - 适用阶段：M1-M3全阶段核心催收手段

4. **上门催收**：
   - 指标：上门成功率、面谈完成率、PTP Rate、KP Rate
   - 成本结构：人工差旅费 + 人员时间成本
   - 适用阶段：M2-M3高风险或高金额案件

5. **法务催收/诉讼**：
   - 指标：发函响应率、立案成功率、判决执行率、法务回收率
   - 成本结构：律师费 + 诉讼费 + 执行费
   - 适用阶段：M3+或恶意拖欠案件

**分析输出**：

```
渠道效能对比矩阵（2026年3月）：

渠道         触达率    PTP率    KP率     回收率   单位成本   ROI
SMS短信      92.3%    3.2%     45.8%    1.5%    ¥0.02    75.0x
AI外呼       68.5%    12.8%    52.3%    6.7%    ¥0.08    12.5x
人工电话     55.2%    28.5%    65.2%    18.6%   ¥0.15    6.7x
上门催收     78.0%    45.2%    72.0%    32.5%   ¥0.35    2.9x
法务催收     --       --       --       15.8%   ¥0.42    2.4x
```

### 步骤3：策略效能分析

**目标**：评估不同催收策略组合的有效性和ROI。

**策略维度**：

1. **渠道组合策略**：
   - 单一渠道 vs 多渠道组合
   - 渠道优先级排序（先SMS后AI外呼 vs 先AI外呼后人工）
   - 评估不同组合的整体回收率和成本

2. **触达时机策略**：
   - DPD1 vs DPD3 vs DPD7 首次触达
   - 工作日 vs 周末触达效果对比
   - 不同时段（上午/下午/傍晚）的触达率对比
   - 注意合规约束：禁止21:00-08:00催收、单日联系不超过3次

3. **催收力度分级**：
   - 温和提醒 vs 标准催收 vs 强硬催收的效果对比
   - 不同M-stage的最优力度等级

4. **话术版本**：
   - A/B测试不同话术版本的PTP Rate和KP Rate
   - 个性化话术 vs 标准话术的效果对比

### 步骤4：A/B测试效果评估

**目标**：为催收策略A/B测试提供统计严谨的效果评估。

**评估方法论**：

1. **实验设计验证**：
   - 检查实验组和对照组的样本量是否满足最小样本量要求
   - 验证分组的随机性（检查组间基线特征是否均衡）
   - 确认实验期间无交叉污染（同一客户不能同时被分入不同组）

2. **最小样本量计算**：

```
最小样本量 = (Z_α/2 + Z_β)² × [P₁(1-P₁) + P₂(1-P₂)] / (P₁ - P₂)²

其中：
  Z_α/2 = 1.96（显著性水平 α = 0.05 时）
  Z_β = 0.84（统计检验力 1-β = 0.80 时）
  P₁ = 对照组预期转化率
  P₂ = 实验组预期转化率
```

3. **统计检验方法**：

| 比较类型 | 检验方法 | 适用条件 |
|----------|----------|----------|
| 两组比例比较（如PTP Rate） | 卡方检验（Chi-Square Test）或Z检验 | 样本量 >= 30 |
| 两组均值比较（如回收金额） | 独立样本t检验（两样本方差齐性时）或Welch t检验 | 数据近似正态分布 |
| 多组比较 | 方差分析（ANOVA）+ 事后多重比较（Tukey HSD） | 三组及以上的策略对比 |
| 非正态分布数据 | Mann-Whitney U检验 / Kruskal-Wallis检验 | 金额数据的偏态分布 |

4. **显著性判定标准**：
   - **P值 < 0.05 为显著**，这是硬性要求
   - 同时报告效应量（Effect Size）：Cohen's d 或相对提升比例
   - 计算95%置信区间（Confidence Interval）
   - 对于多重比较，使用Bonferroni校正或FDR校正控制总体错误率

5. **评估报告模板**：

```
A/B测试评估报告
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

测试名称：[测试名称]
测试周期：YYYY-MM-DD 至 YYYY-MM-DD
测试目标指标：[主要评估指标]

实验设计：
  对照组(A)：[策略描述]，样本量 N_A = X
  实验组(B)：[策略描述]，样本量 N_B = X

基线均衡性检验：
  [各基线特征的组间差异检验结果]

核心结果：
  指标          对照组(A)    实验组(B)    差异      P值      结论
  PTP Rate      25.3%       28.7%       +3.4pp    0.012    显著 ✓
  KP Rate       62.1%       64.5%       +2.4pp    0.087    不显著 ✗
  Recovery Rate 15.2%       17.8%       +2.6pp    0.003    显著 ✓
  Cost/Dollar   ¥0.18       ¥0.16       -¥0.02    0.041    显著 ✓

结论与建议：
  实验组策略在PTP Rate和Recovery Rate上显著优于对照组，
  且单位催收成本更低。建议全量推广实验组策略。

注意事项：
  [可能的混杂因素和局限性说明]
```

### 步骤5：催收人员绩效评估

**目标**：对催收人员进行公平、客观、多维度的绩效评估。

**评估体系**：

1. **绩效指标权重**：

| 指标 | 权重 | 说明 |
|------|------|------|
| 回收金额完成率 | 30% | 实际回收金额 / 目标回收金额 |
| 回收率 | 25% | 回收金额 / 分配在管余额 |
| PTP Rate | 15% | 承诺还款率 |
| KP Rate | 15% | 承诺履约率 |
| 合规评分 | 15% | 合规操作的达标率（无违规扣分） |

2. **公平性调整**：
   - 考虑案件难度差异：按M-stage、余额区间、历史催收次数等因素对案件进行难度分级
   - 调整后对比：将催收人员的实际绩效与"难度调整后的预期绩效"对比
   - 避免"挑肥拣瘦"：剔除案件分配不均对绩效对比的影响

3. **排名方法**：
   - 综合评分 = Σ(各指标标准化得分 × 权重)
   - 标准化方法：Z-Score标准化（均值0，标准差1）
   - 按综合评分排名，生成Top/Bottom 10%清单

4. **输出**：
   - 催收人员绩效分布图（直方图 + 箱线图）
   - Top 10%高绩效催收员名单及其最佳实践特征
   - Bottom 10%低绩效催收员名单及改进建议
   - 团队/组长级别的汇总绩效对比

### 步骤6：效能分析报告与优化建议

**目标**：综合各维度分析结果，输出可执行的优化建议。

**报告结构**：

1. **核心发现摘要**（Executive Summary）
2. **渠道效能对比**：各渠道的漏斗指标和ROI排名
3. **策略效能对比**：各策略组合的回收率和成本效率
4. **A/B测试结论**（如有）：哪些策略已通过统计检验验证有效
5. **人员绩效分布**：排名和关键观察
6. **优化建议**：
   - 资源配置调整建议（如增加/减少某渠道的投入）
   - 策略调整建议（如推广A/B测试中胜出的策略）
   - 人员管理建议（如高绩效催收员的经验推广、低绩效人员的培训计划）

## 四、最佳实践与约束

### 4.1 A/B测试评估的严格要求

- **P值 < 0.05 是硬性标准**：不得仅凭趋势或直觉判断策略优劣，必须通过统计显著性检验
- **最小样本量保障**：在测试启动前即需计算所需最小样本量，样本不足时须延长测试周期，不得提前终止并下结论
- **避免"偷看"（Peeking Problem）**：不得在实验期间多次查看P值并在首次达到0.05时终止实验，应严格按预设的测试周期执行
- **多重比较校正**：当同时评估多个指标或多个实验组时，须使用Bonferroni校正或FDR校正
- **效应量报告**：P值仅反映统计显著性，不反映实际业务意义，须同时报告效应量和置信区间

### 4.2 人员绩效评估的公平性要求

- **案件难度调整**：必须考虑不同催收人员分配到的案件难度差异（M-stage分布、余额分布、历史催收次数），不得直接比较原始绩效指标
- **样本量充足**：单个催收人员的月度分配案件数须 >= 100件，样本过少时不纳入排名
- **排除异常值**：单笔大额回收可能导致个别催收员绩效虚高，需识别并标注此类outlier
- **合规一票否决**：存在严重合规违规（如暴力催收、违规时间催收）的催收人员，绩效评分直接降为最低档

### 4.3 数据准确性要求

- **回收归因准确性**：多渠道催收的回收金额归因须有明确规则（如"最后触达渠道归因"或"加权归因"），规则需预先设定并保持一致
- **成本分摊规则**：共享成本（如系统成本、管理成本）须有合理的分摊规则，分摊方法需在报告中注明
- **数据时效**：催收动作数据须实时同步，回收金额数据T+1日可用

### 4.4 合规约束

- 催收效能评估中涉及的客户联系记录须遵守《个人信息保护法》，评估报告中不展示客户个人信息
- 评估不得鼓励催收人员采取违规手段提升绩效，合规指标必须纳入考核体系
- 催收时间合规红线：禁止21:00-08:00催收，单日联系不超过3次，禁止骚扰联系人
- 所有绩效数据仅用于内部管理评估，不得泄露给外部

### 4.5 约束与限制

- 本技能仅负责评估分析和建议输出，**不直接调整催收策略或人员分配**
- 策略调整建议需流转至催收策略匹配技能（collection-strategy-matching）和资源优化配置技能（resource-optimization-allocation）执行
- 外包催收机构评价结果需流转至外包管理技能（outsource-management）执行后续管理动作

### 示例2：A/B测试评估

**场景**：评估M1阶段"首次触达时机"的A/B测试——DPD1首次触达 vs DPD3首次触达。

**输入**：
```json
{
  "analysis_period": { "start": "2026-02-15", "end": "2026-03-15" },
  "ab_test_config": {
    "test_name": "M1首次触达时机测试",
    "control": { "name": "DPD3触达（现行策略）", "group_id": "group_A" },
    "treatment": { "name": "DPD1触达（提前触达）", "group_id": "group_B" },
    "primary_metric": "recovery_rate",
    "secondary_metrics": ["ptp_rate", "kp_rate", "cost_per_dollar"],
    "significance_level": 0.05
  }
}
```

**执行流程**：
1. 验证实验设计：检查样本量是否充足、分组是否随机
2. 检验基线均衡性：对照组和实验组的案件特征是否均衡
3. 计算各组的核心指标
4. 执行统计显著性检验
5. 计算效应量和置信区间
6. 输出评估结论

  建议：全量推广DPD1首次触达策略至所有M1案件。
