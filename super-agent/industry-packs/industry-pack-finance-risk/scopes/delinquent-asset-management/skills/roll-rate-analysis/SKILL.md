# 迁徙率分析技能 (Roll Rate Analysis)

## 一、技能概述

### 1.1 技能定位

迁徙率分析是催收数据分析Agent（collection-analytics）**最关键的早期预警技能**。迁徙率（Roll Rate）反映逾期资产在不同逾期阶段之间的转化概率，是衡量资产质量恶化速度的核心前瞻指标。当迁徙率出现异常上升时，往往意味着未来数月的不良贷款率（NPL Ratio）将显著攀升，因此必须作为最高优先级的监控对象。

本技能的核心价值在于：在不良资产实际形成**之前**提供预警信号，为管理层赢得2-3个月的策略调整窗口期。

### 1.2 核心能力

- **迁徙率矩阵计算**：构建完整的月度迁徙率矩阵（Transition Matrix），覆盖M0→M1、M1→M2、M2→M3、M3→M3+、M3+→Write-off 全链路
- **异常预警触发**：月度迁徙率环比上升 > 2个百分点时**立即触发预警**，这是本技能的硬性约束
- **Vintage 分析（账龄分析）**：按放款批次追踪不同账龄（MOB）下的迁徙率演变
- **Cohort 跟踪（队列追踪）**：按首次逾期月份组建队列，追踪同一队列在后续月份的迁徙路径
- **趋势分解（Trend Decomposition）**：将迁徙率变动分解为结构效应（Portfolio Mix Effect）和行为效应（Behavioral Effect）

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 月度资产质量预警 | 每月计算迁徙率矩阵，识别恶化趋势，触发早期预警 |
| 审批策略评估反馈 | 通过Vintage迁徙率评估不同放款批次的资产质量，反馈至审批策略优化 |
| 催收策略效果验证 | 通过迁徙率的前后对比验证催收策略调整是否有效降低了恶化率 |
| 压力测试情景分析 | 基于迁徙率矩阵构建资产质量恶化的压力测试情景 |
| 同业对标分析 | 将本机构迁徙率与行业基准（Benchmark）对比，评估相对表现 |
| 宏观经济影响评估 | 分析宏观经济指标变动（如失业率、PMI）与迁徙率的相关性 |

## 二、执行步骤

### 步骤1：迁徙率矩阵构建

**目标**：计算当月完整的逾期阶段迁徙率矩阵。

**计算方法**：

迁徙率（Roll Rate）定义为：在期初处于某一逾期阶段的资产，到期末迁移至下一个更深逾期阶段的比例。

**核心公式**：

```
Roll Rate (Mx → Mx+1) = 期末处于Mx+1阶段的资产余额（仅限期初在Mx阶段的部分）
                        ────────────────────────────────────────────────────────
                        期初处于Mx阶段的总资产余额
```

**完整迁徙率矩阵**：

```
                 期末状态
期初状态    Current(M0)   M1      M2      M3      M3+     Write-off   结清
Current     ██████████   →R01    -       -       -       -           →Cure
M1          ←Cure_1      ██     →R12    -       -       -           →Cure
M2          ←Cure_2      -      ██      →R23    -       -           →Cure
M3          ←Cure_3      -      -       ██      →R3+    -           →Cure
M3+         ←Cure_3+     -      -       -       ██      →Rwo        →Cure
Write-off   -            -      -       -       -       ██          →Recovery
```

其中：
- **R01**（M0→M1 Roll Rate）：正常资产恶化率，反映新增逾期的速度
- **R12**（M1→M2 Roll Rate）：M1催收失败率，反映早期催收效果
- **R23**（M2→M3 Roll Rate）：M2催收失败率，反映中期催收效果
- **R3+**（M3→M3+ Roll Rate）：深度逾期恶化率
- **Rwo**（M3+→Write-off Roll Rate）：核销转化率
- **Cure Rate**（治愈率）：从逾期状态恢复至正常还款的比例

**计算步骤**：

1. 提取期初（月初第1天）各M-stage的资产台账快照
2. 提取期末（月末最后一天）各M-stage的资产台账快照
3. 逐笔匹配同一贷款在期初和期末的M-stage状态
4. 按期初M-stage分组，统计期末流向各M-stage的金额占比
5. 生成完整迁徙率矩阵

**重要注意事项**：
- 计算须同时基于**金额口径**和**户数口径**，大额案件可能导致金额口径波动较大
- 期间内新发放贷款不纳入当月迁徙率计算（避免分母膨胀稀释）
- 期间内结清/核销的贷款需单独统计，不纳入"留存"计算

### 步骤2：异常预警判定

**目标**：对计算所得迁徙率进行异常检测，触发必要预警。

**硬性预警规则（不可配置）**：

> **当任一关键迁徙率的月环比（MoM）上升超过 2 个百分点时，必须立即触发红色预警。**

| 迁徙率指标 | 预警阈值（月环比变动） | 预警级别 | 响应要求 |
|------------|----------------------|----------|----------|
| M0→M1 | 上升 > 1pp | 黄色预警 | 24小时内出具分析报告 |
| M0→M1 | 上升 > 2pp | **红色预警** | **立即触发，2小时内通知管理层** |
| M1→M2 | 上升 > 2pp | **红色预警** | **立即触发，2小时内通知管理层** |
| M2→M3 | 上升 > 2pp | **红色预警** | **立即触发，2小时内通知管理层** |
| M3→Write-off | 上升 > 2pp | **红色预警** | **立即触发，2小时内通知管理层** |
| Cure Rate（M1） | 下降 > 3pp | 黄色预警 | 24小时内出具分析报告 |
| Cure Rate（M2） | 下降 > 2pp | 黄色预警 | 24小时内出具分析报告 |

**预警信息须包含**：
- 触发的具体指标及当前值/历史值对比
- 环比变动的绝对值和相对幅度
- 初步归因分析（至少覆盖产品、地区、Vintage三个维度的下钻）
- 影响预估：按当前迁徙率外推，预估未来2-3个月的NPL增量

### 步骤3：Vintage 分析（账龄分析）

**目标**：按放款批次追踪资产质量的全生命周期演变。

**分析方法**：

1. **Vintage 定义**：按放款月份（Origination Month）对贷款进行分组，形成不同的Vintage队列
2. **账龄（MOB, Month on Book）**：贷款从发放到观察时点经过的月份数
3. **核心输出**：

```
Vintage迁徙率矩阵示例（M0→M1 Roll Rate）：

Vintage      MOB3    MOB6    MOB9    MOB12   MOB18   MOB24
2025-01      1.2%    1.8%    2.1%    2.3%    2.5%    2.6%
2025-04      1.5%    2.2%    2.8%    3.1%    --      --
2025-07      1.8%    2.5%    3.2%    --      --      --
2025-10      2.1%    3.0%    --      --      --      --
2026-01      2.4%    --      --      --      --      --
```

4. **异常识别**：
   - 同一MOB下，新Vintage的迁徙率是否显著高于历史Vintage？
   - 是否存在"加速恶化"现象——某Vintage在早期MOB的迁徙率就明显偏高？
   - 将Vintage迁徙率曲线与"基准曲线"（Benchmark Curve，取历史3年均值）对比

5. **输出格式**：Vintage热力图（颜色深浅表示迁徙率高低）+ 折线图对比

### 步骤4：Cohort 跟踪（队列追踪）

**目标**：追踪同一批首次逾期客户的后续迁徙路径。

**分析方法**：

1. **Cohort 定义**：按首次进入M1的月份对客户进行分组
2. **追踪内容**：
   - 该Cohort在后续1/3/6/12个月内的迁徙分布
   - 治愈率（Cure Rate）随时间的衰减曲线
   - 最终处置分布：治愈占比、核销占比、仍在管占比

3. **核心输出**：

```
Cohort追踪示例（2025年10月首逾队列，共5,000户）：

首逾后月数    治愈(Cured)   M1停留   M2迁徙   M3迁徙   M3+迁徙   核销
Month+1       35.0%        20.0%    45.0%    -        -         -
Month+3       48.0%        5.0%     12.0%    25.0%    10.0%     -
Month+6       55.0%        2.0%     3.0%     8.0%     27.0%     5.0%
Month+12      60.0%        0.5%     0.5%     2.0%     22.0%     15.0%
```

4. **关键洞察**：
   - 不同Cohort的治愈率是否在下降？（催收效果恶化信号）
   - 从首逾到核销的平均周期是否在缩短？（恶化加速信号）
   - 哪些客户特征与高治愈率相关？（用于催收策略优化）

### 步骤5：趋势分解

**目标**：将迁徙率的总体变动分解为可解释的驱动因素。

**分解方法**：

1. **结构效应（Portfolio Mix Effect）**：
   - 由于组合结构变化导致的迁徙率变动
   - 例如：高风险产品占比上升，即使各产品自身迁徙率不变，总体迁徙率也会上升
   - 计算方法：固定各产品迁徙率不变，仅改变占比权重

2. **行为效应（Behavioral Effect）**：
   - 在组合结构不变的前提下，由于客户还款行为变化导致的迁徙率变动
   - 反映真实的资产质量或催收效果变化
   - 计算方法：固定各产品占比权重不变，仅改变各产品迁徙率

3. **分解公式**：

```
总变动 = 结构效应 + 行为效应 + 交互效应

ΔR_total = Σ(ΔW_i × R_i_base) + Σ(W_i_base × ΔR_i) + Σ(ΔW_i × ΔR_i)

其中：
  ΔR_total = 总迁徙率变动
  ΔW_i = 第i个产品/客群的权重变动
  R_i_base = 第i个产品/客群的基期迁徙率
  W_i_base = 第i个产品/客群的基期权重
  ΔR_i = 第i个产品/客群的迁徙率变动
```

4. **宏观经济关联分析**：
   - 计算迁徙率与宏观指标的相关系数：GDP增速、失业率、PMI、CPI、社融增量
   - 建立领先/滞后关系：宏观指标通常领先迁徙率1-3个月
   - 当宏观领先指标恶化时，提前预警迁徙率可能上升

## 四、最佳实践与约束

### 4.1 计算准确性要求

- **数据一致性**：期初和期末的贷款台账快照必须基于相同的统计口径，不得混用不同系统的数据源
- **贷款匹配率**：期初和期末台账中同一贷款的匹配率须 >= 99.9%，未匹配的贷款需单独排查（可能为期间新发放/结清/转让/核销）
- **双口径校验**：金额口径和户数口径的迁徙率应同时计算，当两者出现显著背离时（如金额口径迁徙率上升而户数口径未变），说明大额案件驱动了变动，需重点关注
- **数据准确率要求 >= 99.5%**

### 4.2 预警时效性要求

- **月度迁徙率预警**：必须在每月T+2个工作日内完成计算并触发预警
- **红色预警响应**：触发后 **2小时内** 通知风控委员会成员和相关管理层
- **预警分析报告**：红色预警触发后 **24小时内** 出具完整的归因分析报告
- **预警不得延迟**：这是硬性约束，不得因数据未全部到齐而延迟预警触发，可先基于已有数据触发初步预警，后续补充完整分析

### 4.3 分析方法论约束

- **Vintage 分析**须覆盖至少 **近24个月** 的放款批次，以确保有足够的MOB深度
- **Cohort 追踪**须至少追踪 **12个月**，以覆盖从首逾到最终处置的完整生命周期
- **趋势分解**中，结构效应 + 行为效应 + 交互效应之和必须等于总变动量（验证分解的完整性）
- **宏观关联分析**需注意伪相关性，相关系数须通过统计显著性检验（P < 0.05）

### 4.4 约束与限制

- 迁徙率分析仅提供预警和洞察，**不直接调整催收策略**——策略调整建议需提交至催收策略匹配（collection-strategy-matching）技能处理
- 单一月份的迁徙率波动可能受季节性因素影响（如春节前后），需结合至少3个月的趋势判断，**避免对单月数据过度反应**
- 行业基准数据（Benchmark）可能存在滞后性和统计口径差异，对标分析时须注明数据来源和口径差异
- 宏观经济关联分析为辅助参考，**不可作为预警的唯一依据**

## 五、示例

### 示例1：月度迁徙率常规监控

**场景**：每月T+2日计算上月迁徙率矩阵，进行常规监控。

**输入**：
```json
{
  "analysis_month": "2026-03",
  "lookback_months": 12,
  "calculation_basis": "both"
}
```

**执行流程**：
1. 提取2026年3月1日和3月31日的贷款台账快照
2. 逐笔匹配贷款的期初/期末M-stage状态
3. 计算完整迁徙率矩阵（金额+户数双口径）
4. 与2026年2月迁徙率对比，计算月环比变动
5. 检查是否触发预警规则
6. 输出迁徙率矩阵和趋势图

**输出（摘要）**：
```
2026年3月迁徙率分析报告

迁徙率矩阵（金额口径）：
  M0→M1:  3.2%  (MoM +0.3pp) ✅ 正常范围
  M1→M2: 38.5%  (MoM +2.5pp) 🔴 红色预警！超过2pp阈值
  M2→M3: 45.2%  (MoM +1.2pp) ✅ 正常范围
  M3→M3+: 62.8% (MoM -0.5pp) ✅ 正常范围
  M1治愈率: 35.5% (MoM -1.8pp) ⚠️ 需关注

红色预警：
  M1→M2迁徙率38.5%，环比上升2.5pp，为近12个月最高值。

  归因分析：
  - 结构效应贡献: 0.3pp（个人消费贷占M1余额比重上升2.1pp）
  - 行为效应贡献: 2.0pp（主因）
    - 个人消费贷M1→M2迁徙率从34%→41%（+7pp）
    - 华东地区M1催收治愈率下降3.2pp
  - 交互效应: 0.2pp

  影响预估：
  若M1→M2迁徙率维持当前水平，预计5月M3+余额将增加约1.2亿元。

  建议行动：
```
