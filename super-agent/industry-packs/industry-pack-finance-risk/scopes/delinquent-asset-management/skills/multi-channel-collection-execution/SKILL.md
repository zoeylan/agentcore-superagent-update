# 多渠道催收执行技能 (Multi-Channel Collection Execution)

## 技能概述

多渠道催收执行技能是催收执行Agent (collection-executor) 的核心技能，负责M1（1-30天逾期）和M2（31-60天逾期）阶段的全渠道客户触达与催收动作编排。本技能按照逾期时间节点自动触发分层触达策略，覆盖短信提醒、AI智能外呼、APP推送、人工软催收电话四大渠道，形成"由轻到重、由自动到人工"的渐进式催收压力梯度。

### 业务价值

- M1阶段回收率每提升1个百分点，可减少后续M2/M3阶段约0.3-0.5个百分点的处置压力
- 自动化催收覆盖M1阶段70%以上的案件处理量，显著降低人力成本
- 多渠道协同触达将客户联系成功率提升至行业平均水平的1.3-1.5倍

### 适用场景

| 场景 | 说明 |
|------|------|
| M1自动化催收流程执行 | 针对新增M1逾期案件，按T+1/T+3/T+5/T+7节点自动触发多渠道催收动作 |
| M2人工催收辅助 | 为催收员生成案件摘要、建议话术要点、历史催收记录汇总 |
| 催收任务队列管理 | 根据分案策略生成每日催收任务队列，按优先级排序执行 |
| 承诺还款跟踪 | 跟踪客户承诺还款日期，到期前24小时自动触发提醒 |
| 失联客户升级 | 多渠道触达均失败的客户标记为失联，触发策略升级流程 |

## 详细执行步骤

### 阶段一：M1自动化催收流程

#### Step 1: 催收任务队列生成（每日T+0）

1. 从智能分案Agent (case-allocator) 接收当日催收任务分配清单
2. 校验每条案件的基本信息完整性（客户ID、联系方式、逾期金额、逾期天数）
3. 检查案件状态——排除已标记为"投诉中"、"协商中"、"暂停催收"的案件
4. 按催收优先级对任务排序：
   - Priority 1: 逾期金额 > 50,000元 且 客户历史还款良好
   - Priority 2: 逾期金额 10,000-50,000元
   - Priority 3: 逾期金额 < 10,000元
5. 生成当日催收任务队列，分配至对应渠道执行器

#### Step 2: T+1 短信提醒（首逾次日）

1. 从合规模板库中选取适配的短信模板：
   - 模板类型：柔性提醒（首逾专用）
   - 内容要素：客户姓名（脱敏）、逾期金额、还款截止建议日期、还款渠道指引
2. 执行合规预检：
   - 确认当前时间在 08:00-21:00 之间
   - 确认该案件当日催收联系次数 < 3
   - 确认客户未在"禁止联系"名单中
3. 调用短信网关发送催收提醒
4. 记录发送状态（成功/失败/号码无效）
5. 短信送达率目标：>= 95%

**短信模板示例（须经合规审批）：**
```
【XX金融】尊敬的客户，您名下贷款（尾号XXXX）有一笔{amount}元款项已于{date}到期，
请及时处理以免影响您的个人信用记录。还款请登录APP或拨打客服热线400-XXX-XXXX。
```

#### Step 3: T+3 AI智能外呼

1. 筛选T+1短信发送后仍未还款的案件
2. 启动AI智能外呼系统，配置以下参数：
   - 外呼时段：09:00-12:00, 14:00-18:00（黄金时段优先）
   - 最大振铃时长：30秒
   - 单次通话最大时长：5分钟
3. AI外呼多轮对话流程：
   ```
   开场白 → 身份确认 → 逾期告知 → 情绪识别与应对 → 还款意愿探测
   → 还款金额确认 → 还款日期约定 → 还款渠道引导 → 结束语
   ```
4. **情绪识别模块**：
   - 实时分析客户语音语调，识别以下情绪状态：
     - 配合型：正常引导还款
     - 焦虑型：安抚 + 提供还款方案选项
     - 抵触型：降低催收强度，记录原因，标记人工跟进
     - 愤怒型：立即礼貌结束通话，标记为敏感案件
5. **还款引导策略**：
   - 全额还款引导（优先）
   - 最低还款额引导（客户明确暂时困难时）
   - 还款方案咨询引导（客户需协商时，转入还款方案管理技能）
6. 记录通话结果：接通/未接通、通话时长、客户意向分类、承诺还款金额与日期
7. AI外呼接通率目标：>= 45%

#### Step 4: T+5 APP推送通知

1. 筛选T+3 AI外呼后仍未还款的案件
2. 检查客户APP登录活跃状态（近30天是否有登录记录）
3. 对活跃用户发送APP站内推送：
   - 推送类型：还款提醒（带快捷还款入口）
   - 内容要素：逾期金额、逾期天数、一键还款按钮
4. 对非活跃用户同步发送短信补充提醒（使用中度提醒模板）
5. 记录推送送达与点击情况

#### Step 5: T+7 人工软催收电话

1. 筛选前序自动化渠道均未促成还款的案件
2. 生成人工催收任务单，包含：
   - 客户基本信息摘要
   - 逾期详情（金额、天数、产品类型）
   - 前序催收记录汇总（短信送达情况、AI外呼结果、APP推送反馈）
   - 建议沟通策略与话术要点
3. 分配至催收员工作队列
4. 催收员完成外呼后录入通话记录（转交催收记录管理技能处理）

#### Step 7: 催收员实时辅助

1. 催收员发起外呼时，实时展示案件摘要与建议话术
2. 通话过程中支持：
   - 关键信息快速查询（还款历史、合同条款）
   - 还款方案实时计算（调用还款方案管理技能）
   - 合规话术提示（避免触碰红线用语）
3. 通话结束后引导催收员录入结构化通话记录

#### Step 8: 策略升级与异常处理

1. **失联客户处理**（连续3次多渠道触达失败）：
   - 标记为"疑似失联"
   - 尝试备用联系方式（预留的其他手机号）
   - T+15仍无法联系则上报智能分案Agent，建议升级处理
2. **拒绝还款客户处理**：
   - 记录拒绝原因
   - 如客户提出争议，转交客户沟通Agent (customer-negotiator)
   - 如客户明确表示无还款意愿，标记并上报
3. **承诺还款跟踪**：
   - 记录客户承诺还款日期和金额
   - 承诺日期前24小时发送提醒短信
   - 承诺日期当天检查还款入账情况
   - 未履约则触发下一轮催收动作

## 最佳实践与约束

### 合规红线（不可逾越）

| 规则编号 | 合规要求 | 说明 |
|----------|----------|------|
| COMP-01 | 催收时间窗口 | 严禁在 21:00 至次日 08:00 之间发送催收短信或拨打催收电话 |
| COMP-02 | 每日联系频次 | 单个案件单日催收联系总次数（含所有渠道）不得超过 **3次** |
| COMP-03 | 话术合规 | 100%使用经合规审核通过的标准模板，禁止威胁、恐吓、侮辱性语言 |
| COMP-04 | 第三方联系禁令 | 不得联系借款人以外的无关第三方（紧急联系人除外，且仅可通知转达） |
| COMP-05 | 暂停催收保护 | 对"投诉中"或"协商中"案件，暂停一切催收动作直至状态变更 |

### 最佳实践

1. **渠道协同而非堆叠**：多渠道触达应按时间节点分层递进，避免同一时间多渠道同时触达造成客户反感
2. **黄金时段优先**：AI外呼和人工电话优先安排在 09:00-11:30 和 14:00-17:30 时段，接通率可提升15-20%
3. **情绪识别及时响应**：AI外呼检测到客户愤怒情绪时应立即礼貌结束通话，避免升级为投诉
4. **承诺管理闭环**：每一个客户承诺必须录入系统并设置到期提醒，承诺履约率是衡量催收质量的核心指标
5. **数据标准化**：所有催收执行结果必须按标准格式输出，确保与智能分案Agent和催收数据分析Agent的数据接口一致
6. **渐进式压力策略**：M1阶段以提醒和引导为主（柔性话术），M2阶段逐步增加催收力度但始终不突破合规底线
7. **个性化触达**：根据客户画像（年龄、职业、历史行为）选择最优触达时间和渠道组合

### 约束条件

- 本技能仅处理M1和M2阶段案件，M3+案件应转交法务处置Agent (legal-disposition)
- 催收执行前必须通过合规监控技能 (compliance-monitoring) 的预检
- 所有催收记录必须通过催收记录管理技能 (collection-record-management) 进行标准化存储
- 还款方案相关操作需调用还款方案管理技能 (repayment-plan-management)
- 客户争议和投诉案件需转交客户沟通Agent (customer-negotiator)，不在本技能处理范围内

## 示例

### 示例1：M1标准自动化催收流程

**场景**：客户张先生个人消费贷逾期3天，逾期金额5,200元，首次逾期。

```
Day 1 (T+1) - 短信提醒:
  渠道: SMS
  模板: TMPL-M1-SOFT-REMIND-01
  状态: 送达成功
  客户响应: 无

Day 3 (T+3) - AI智能外呼:
  渠道: AI_CALL
  时间: 10:30
  接通: 是
  时长: 3分05秒
  情绪识别: 轻微焦虑
  对话摘要: 客户表示知晓逾期，近期资金紧张，承诺4月18日还款
  承诺金额: 5,200元（全额）
  承诺日期: 2025-04-18

Day 5 (T+5) - 跳过APP推送（客户已承诺还款）

Day 7 (T+7) - 承诺还款前一天:
  渠道: SMS
  模板: TMPL-PROMISE-REMIND-01
  内容: 提醒客户明日承诺还款到期

Day 8 (T+8) - 承诺还款日:
  系统检查: 还款到账 5,200元
  结果: 案件结清，标记为"已回收"
```

### 示例2：M1催收升级至M2

**场景**：客户李女士信用贷逾期，M1阶段多渠道催收未成功促成还款。

```
Day 1 (T+1) - 短信提醒: 送达成功，无响应
Day 3 (T+3) - AI外呼: 未接通（振铃无应答）
Day 5 (T+5) - APP推送: 送达成功，客户点击查看但未还款
Day 5 (T+5) - 补充短信: 中度提醒模板，送达成功
Day 7 (T+7) - 人工软催电话:
  接通，客户表示工作变动导致收入中断
  承诺两周内还款，未给出具体日期
Day 15 (T+15) - 二次AI外呼: 未接通
Day 20 (T+20) - 人工电话: 未接通
Day 25 (T+25) - 标记"疑似失联"
Day 30 (T+30) - 案件升级至M2:
  生成M2案件摘要
  汇总M1阶段全部催收记录
  标注关键信息: "客户称工作变动，承诺未明确日期，后续疑似失联"
  上报智能分案Agent，建议分配至经验丰富催收员
```
