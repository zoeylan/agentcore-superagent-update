# 还款方案管理技能 (Repayment Plan Management)

## 技能概述

还款方案管理技能负责维护和执行催收执行Agent (collection-executor) 的减免与还款重组规则库，为M1/M2阶段逾期客户提供合理的还款方案选项，在最大化回收率的同时兼顾客户还款能力和意愿。本技能覆盖滞纳金减免审批、分期还款方案设计、NPV（Net Present Value，净现值）影响测算、方案执行跟踪和违约预警全流程。

### 业务价值

- 合理的减免和分期方案可将M1回收率提升5-10个百分点，M2回收率提升10-15个百分点
- 通过NPV测算确保每一个减免方案在财务上可接受，避免过度让利
- 标准化的授权矩阵和自动审批机制将方案审批时效从平均4小时缩短至实时响应

### 适用场景

| 场景 | 说明 |
|------|------|
| M1滞纳金减免 | 客户逾期1-30天，申请减免部分滞纳金以促成一次性结清 |
| M2分期还款方案 | 客户逾期31-60天，无法一次性还清，需要分期还款安排 |
| 授权矩阵内自动审批 | 减免金额或分期方案在预设授权范围内，系统自动审批 |
| 超限方案上报审批 | 减免金额或分期条件超出授权矩阵范围，自动升级至人工审批 |
| 方案执行跟踪 | 跟踪已生效方案的还款执行情况，发现违约及时预警 |
| NPV影响分析 | 评估不同还款方案对资产净现值的影响，支撑方案决策 |

## 详细执行步骤

### Step 1: 还款方案规则库管理

维护结构化的减免与分期规则库，规则按逾期阶段、客户风险等级、逾期金额分层配置：

#### M1阶段减免规则

| 规则ID | 客户风险等级 | 逾期金额范围 | 最大减免比例 | 减免对象 | 审批方式 |
|--------|-------------|-------------|-------------|---------|---------|
| M1-R01 | A/A+ | 不限 | 滞纳金30% | 滞纳金 | 自动审批 |
| M1-R02 | B/B+ | <= 50,000元 | 滞纳金25% | 滞纳金 | 自动审批 |
| M1-R03 | B/B+ | > 50,000元 | 滞纳金20% | 滞纳金 | 自动审批 |
| M1-R04 | C及以下 | <= 30,000元 | 滞纳金15% | 滞纳金 | 自动审批 |
| M1-R05 | C及以下 | > 30,000元 | 滞纳金10% | 滞纳金 | 自动审批 |
| M1-R06 | 任意 | 任意 | 滞纳金 > 30% | 滞纳金 | **人工审批** |

#### M2阶段分期规则

| 规则ID | 客户风险等级 | 逾期金额范围 | 最大分期数 | 首期比例 | 审批方式 |
|--------|-------------|-------------|-----------|---------|---------|
| M2-R01 | A/A+ | 不限 | 6期 | >= 20% | 自动审批 |
| M2-R02 | B/B+ | <= 50,000元 | 6期 | >= 25% | 自动审批 |
| M2-R03 | B/B+ | > 50,000元 | 4期 | >= 30% | 自动审批 |
| M2-R04 | C及以下 | <= 30,000元 | 4期 | >= 30% | 自动审批 |
| M2-R05 | C及以下 | > 30,000元 | 3期 | >= 35% | 自动审批 |
| M2-R06 | 任意 | 任意 | > 6期 | 任意 | **人工审批** |

> **注意**：M2分期方案中，减免部分同样适用——除分期安排外，可叠加滞纳金减免（M2阶段减免上限参照M1规则上浮5个百分点）。本金原则上不予减免。

### Step 2: 方案匹配与生成

当催收过程中客户表达还款困难或协商意愿时，系统自动匹配适用方案：

1. **信息采集**：
   - 获取客户当前逾期阶段、风险等级、逾期金额
   - 查询客户历史减免记录（同一客户12个月内减免次数限制：M1阶段最多2次，M2阶段最多1次）
   - 确认客户不在减免黑名单中（历史恶意利用减免政策的客户）

2. **规则匹配**：
   - 根据逾期阶段、风险等级、金额范围匹配适用规则
   - 计算可用的减免额度上限和分期方案选项

3. **方案生成**：
   - 生成2-3个梯度方案供客户选择
   - 每个方案包含：还款总额、减免金额、分期安排（如适用）、每期金额、还款截止日期

### Step 3: NPV影响测算

对每个候选方案进行Net Present Value影响分析，确保减免决策在财务上合理：

1. **基准场景**（无减免/分期）：
   - 按当前回收概率预估未来现金流
   - 折现率采用机构资金成本 + 风险溢价（通常为年化8-12%）

2. **方案场景**：
   - 减免金额视为即时损失
   - 分期方案按各期预期还款金额和违约概率计算预期现金流
   - 考虑时间价值折现

3. **NPV差异计算**：
   ```
   NPV_Impact = NPV(方案场景) - NPV(基准场景)
   ```
   - NPV_Impact > 0：方案优于不减免（客户在减免激励下还款概率显著提升）
   - NPV_Impact < 0 但在可接受范围内（通常 < 逾期金额的10%）：方案可接受
   - NPV_Impact < -10% * 逾期金额：方案让利过大，建议调整或上报审批

4. **参数配置**：
   ```
   discount_rate: 10% (年化)
   m1_repayment_probability_no_waiver: 75%
   m1_repayment_probability_with_waiver: 90%
   m2_repayment_probability_no_plan: 45%
   m2_repayment_probability_with_plan: 70%
   installment_default_rate_per_period: 5%
   ```

### Step 4: 授权审批流程

#### 自动审批

满足以下全部条件时自动审批通过：
1. 方案匹配的规则审批方式为"自动审批"
2. 客户12个月内同阶段减免次数未超限
3. NPV影响在可接受范围内
4. 客户不在减免黑名单中

审批通过后立即生成还款方案协议，发送至客户确认。

#### 人工审批上报

以下情况需上报人工审批：
1. 减免比例超出授权矩阵上限
2. 分期期数超过6期
3. 客户12个月内减免次数已达上限但仍需减免
4. NPV影响超出可接受范围
5. 单案减免金额 > 5,000元

上报信息包：
```json
{
  "approval_request_id": "APR-2025-0413-00012",
  "case_id": "CASE-2025-0413-00456",
  "requested_plan": { "...方案详情..." },

  "npv_analysis": { "...NPV测算结果..." },

  "sla_hours": 4

```

### Step 5: 方案执行跟踪

方案生效后进入执行跟踪阶段：

1. **还款日历生成**：
   - 为每个已生效的分期方案生成还款日历
   - 每期还款日前3天发送预提醒短信
   - 还款日当天检查入账情况

2. **正常履约处理**：
   - 确认还款到账后更新方案执行状态
   - 全部期数还清后标记方案"已完成"，案件结清

3. **违约预警与处理**：
   - 还款日当天20:00前未入账：发送当日提醒
   - 还款日T+1未入账：标记"逾期预警"，自动发送催缴通知
   - 还款日T+3未入账：标记"方案违约"，触发以下处理：
     - 方案自动终止（剩余未还金额恢复为逾期状态）
     - 已减免金额是否追回根据方案条款确定
     - 上报智能分案Agent重新评估催收策略
     - 记录违约历史，影响客户未来方案申请的审批

4. **方案执行看板**：

| 指标 | 说明 | 目标值 |
|------|------|--------|
| 方案达成率 | 协商成功的方案数 / 总协商次数 | >= 60% |
| 首期履约率 | 首期按时还款的方案数 / 已生效方案数 | >= 85% |
| 全程履约率 | 全部期数按时完成的方案数 / 已生效方案数 | >= 70% |
| 平均减免比例 | 实际减免金额 / 应收滞纳金总额 | <= 20% |
| 方案审批时效 | 从申请到审批完成的平均时间 | <= 4小时 |

## 最佳实践与约束

### 最佳实践

1. **先促全额后谈方案**：催收沟通中应优先引导客户全额还款，客户明确表示困难后再提供减免/分期方案
2. **梯度方案策略**：每次为客户提供2-3个梯度方案（从机构最优到客户最优），引导客户选择折中方案
3. **NPV驱动决策**：所有减免决策必须经过NPV测算，避免"拍脑袋"式减免
4. **首期金额锚定**：分期方案的首期金额应占总额的20-35%，较高的首期金额能有效筛选出有真实还款意愿的客户
5. **限时方案策略**：减免方案设置有效期（通常3-7天），制造适度紧迫感促进客户决策
6. **违约历史权重**：对历史方案违约客户，收紧减免力度和分期期数，降低机构损失风险
7. **减免预算管控**：月度/季度减免总额需在预算范围内，临近预算上限时优先保障高金额案件的减免额度

### 约束条件

| 约束项 | 限制 | 说明 |
|--------|------|------|
| 本金减免 | **禁止** | 任何情况下不得减免贷款本金，仅可减免滞纳金/罚息 |
| 单客户年度减免次数 | M1 <= 2次，M2 <= 1次 | 12个月滚动窗口内 |
| 最大分期期数 | 6期（自动审批），超出需人工 | 特殊情况下人工审批最多可批准12期 |
| M1最大减免比例 | 滞纳金30%（自动审批上限） | 超出需上报 |
| 方案有效期 | 生成后7天 | 超期未确认自动失效 |
| 审批时效SLA | 自动审批：实时；人工审批：<= 4小时 | 超时自动升级 |
| 方案叠加限制 | 不可与困难客户援助方案同时生效 | 二选一 |
| 减免黑名单 | 历史恶意利用减免政策的客户禁止减免 | 由风控部门维护名单 |

## 示例

### 示例1：M1滞纳金减免（自动审批）

**场景**：客户王先生，风险等级B+，个人消费贷逾期22天，逾期金额8,000元，累计滞纳金320元。客户在AI外呼中表示愿意全额还款但希望减免部分滞纳金。

```
请求:
  类型: FEE_WAIVER
  客户风险等级: B+
  逾期金额: ¥8,000
  累计滞纳金: ¥320
  客户诉求: 减免部分滞纳金后一次性结清
  历史减免: 无

规则匹配: M1-R02 (B+等级, <=50,000元, 滞纳金最高减免25%)

生成方案:
  方案A (推荐): 减免滞纳金25% (¥80), 支付 ¥8,240, 限期3天内还款
  方案B: 减免滞纳金15% (¥48), 支付 ¥8,272, 限期5天内还款
  方案C: 全额支付 ¥8,320, 无减免

NPV分析:
  方案A: NPV_Impact = +¥120 (减免¥80但回收概率从75%提升至92%)
  推荐: 方案A (NPV正向, 促成快速回收)

审批: 自动通过 (规则M1-R02, 减免25%在授权范围内)
```

### 示例2：M2分期方案（自动审批）

**场景**：客户赵女士，风险等级B，逾期45天，逾期金额25,000元，累计滞纳金1,250元。客户表示因家庭变故短期内无法一次性还清，希望分期还款。

```
请求:
  类型: BOTH (减免 + 分期)
  客户风险等级: B
  逾期金额: ¥25,000
  累计滞纳金: ¥1,250
  客户诉求: 分4-6期还款, 希望减免部分滞纳金
  历史减免: M1阶段曾减免1次

规则匹配: M2-R02 (B+等级, <=50,000元, 最多6期, 首期>=25%)

生成方案:
  方案A: 滞纳金减免25%(¥312.5), 分3期
    第1期: ¥8,750 (2025-04-25)
    第2期: ¥8,594 (2025-05-25)
    第3期: ¥8,593 (2025-06-25)
    总支付: ¥25,937.5
    NPV_Impact: -¥380

  方案B (推荐): 滞纳金减免15%(¥187.5), 分4期
    第1期: ¥6,766 (2025-04-25)
    第2期: ¥6,099 (2025-05-25)
    第3期: ¥6,099 (2025-06-25)
    第4期: ¥6,098 (2025-07-25)
    总支付: ¥25,062.5 (含首期比例约26%)
    NPV_Impact: -¥620

  方案C: 无减免, 分6期
    第1期: ¥6,563 (2025-04-25)
    第2-6期: 各¥3,937 (按月)
    总支付: ¥26,250
    NPV_Impact: -¥980

审批: 方案A/B/C均在授权范围内, 自动通过
推荐: 方案B (平衡客户还款能力与NPV影响)
```

### 示例3：超限方案上报人工审批

**场景**：客户陈先生，风险等级C，逾期55天，逾期金额80,000元，累计滞纳金6,400元。客户提供医院诊断证明，因重大疾病请求大幅减免并分12期还款。

```
请求:
  类型: BOTH
  客户风险等级: C
  逾期金额: ¥80,000
  累计滞纳金: ¥6,400
  客户诉求: 减免全部滞纳金, 分12期还款
  特殊情况: 提供三甲医院重大疾病诊断证明

规则匹配: M2-R05 (C等级, >30,000元, 最多3期, 首期>=35%)
客户诉求超出授权范围:
  - 减免100%滞纳金 > 规则上限15%
  - 分12期 > 规则上限3期

自动审批: 不通过 (超出授权矩阵)

上报人工审批:
  审批请求编号: APR-2025-0413-00034
```
