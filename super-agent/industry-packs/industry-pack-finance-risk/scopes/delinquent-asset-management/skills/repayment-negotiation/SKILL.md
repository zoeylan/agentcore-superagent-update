# 还款方案协商技能 (Repayment Negotiation)

> **所属Agent**: 客户沟通Agent (customer-negotiator)
> **所属Scope**: 逾期资产治理 (Delinquent Asset Management)
> **所属Industry Pack**: 金融风控 (Financial Risk Control)
> **技能标识**: `repayment-negotiation`
> **SOP编号**: SOP-DA-04

---

## 一、技能概述

还款方案协商技能负责为逾期客户设计个性化的还款方案，以推动债务化解。本技能的核心在于综合评估客户的逾期原因、还款能力、还款意愿和贷款特征，在机构授权矩阵范围内设计出客户能接受、机构可承受的最优还款方案，实现 **"债务回收最大化"与"客户关系维护"之间的最优平衡**。

每个方案必须基于严格的 **NPV (Net Present Value, 净现值) 分析**，量化评估不同方案对机构资产回收的财务影响，确保方案决策有据可依。所有方案必须在授权矩阵范围内执行，超限方案必须上报人工审批。

### 适用场景

| 场景编号 | 场景描述 | 触发来源 |
|---------|---------|---------|
| S1 | 客户主动申请延期还款 | 客户通过客服/APP主动提出 |
| S2 | 客户主动申请分期重组 | 客户希望将剩余欠款重新分期 |
| S3 | 客户申请部分减免结清 | 客户希望一次性结清但要求减免部分息费 |
| S4 | 催收过程中客户表达还款困难 | 催收执行Agent转介 |
| S5 | M2阶段案件的还款方案主动设计 | 智能分案Agent分配至协商通道 |
| S6 | 客户通过投诉渠道间接表达还款方案需求 | 客户投诉处理技能转介 |
| S7 | 法务催收前的最后协商机会 | 法务处置Agent发起的M3+案件协商 |

### 方案类型总览

| 方案类型 | 英文名称 | 适用客群 | 核心特征 |
|---------|---------|---------|---------|
| 延期还款 | Deferred Payment | 短期资金周转困难的客户 | 延后还款日期，给予缓冲期 |
| 分期重组 | Installment Restructuring | 有还款意愿但当前还款能力不足的客户 | 将剩余欠款重新拆分为多期 |
| 部分减免结清 | Partial Waiver Settlement | 有一次性结清能力但金额不足全额的客户 | 减免部分息费/罚息，一次性结清 |
| 组合方案 | Hybrid Plan | 情况复杂的客户 | 以上方案类型的组合 |

## 二、详细执行步骤

### 步骤 1：客户情况评估 (Customer Assessment)

**目标**: 全面了解客户的逾期原因、还款能力和还款意愿，为方案设计提供依据。

**执行动作**:

1.1 **贷款信息提取**
- 从核心银行系统获取贷款基础信息：
  - 贷款合同号、产品类型、原始贷款金额、放款日期、贷款期限
  - 当前剩余本金 (Outstanding Principal)
  - 应还未还利息 (Accrued Interest)
  - 罚息 (Penalty Interest)
  - 滞纳金 (Late Fee)
  - 其他费用
  - 已还款期数、剩余期数
  - 逾期天数、逾期阶段（M1/M2/M3+）

1.2 **客户画像分析**
- 从CRM系统和催收系统获取客户画像数据：
  - **还款历史**: 历史还款记录（是否有过逾期、逾期频率和持续时间）
  - **催收响应**: 历史催收中的沟通态度、承诺还款履约率
  - **逾期原因**: 客户自述或催收沟通中获取的逾期原因分类
    - 短期周转困难（发工资时间差、临时大额支出）
    - 收入下降（失业、降薪、生意亏损）
    - 重大变故（疾病、事故、家庭变故）
    - 多头负债压力
    - 还款意愿缺失
  - **当前经济状况**: 客户自述的收入水平、负债情况、家庭状况

1.3 **还款意愿与还款能力评分**

| 评估维度 | 评分因子 | 权重 |
|---------|---------|------|
| 还款意愿 | 催收沟通配合度、主动沟通频率、历史承诺履约率、是否主动提出方案 | 40% |
| 还款能力 | 收入稳定性、负债收入比(DTI)、逾期金额相对收入的比例、资产状况 | 40% |
| 贷款特征 | 逾期阶段、逾期金额、产品类型、担保方式 | 20% |

评分结果分级：
- **A级**（高意愿+高能力）: 优先推荐一次性结清方案，给予适度减免激励
- **B级**（高意愿+低能力）: 推荐分期重组方案，降低每期还款压力
- **C级**（低意愿+高能力）: 强调逾期后果（征信影响、法务风险），设计限时优惠方案
- **D级**（低意愿+低能力）: 评估是否转入困难客户援助流程或法务处置通道

### 步骤 2：方案设计 (Plan Design)

**目标**: 基于客户评估结果，设计1-3个差异化的还款方案供客户选择。

**执行动作**:

2.1 **延期还款方案设计 (Deferred Payment Plan)**

适用条件：客户短期内有明确的收入来源或资金回笼预期。

| 参数 | 规则 |
|------|------|
| 延期期限 | M1: 最长延期15天; M2: 最长延期30天 |
| 延期期间利息 | 按合同利率正常计息，延期期间不产生额外罚息 |
| 延期次数限制 | 同一合同12个月内最多申请2次延期 |
| 授权级别 | 系统自动审批（在参数范围内） |

2.2 **分期重组方案设计 (Installment Restructuring Plan)**

适用条件：客户有持续还款意愿但当前每期还款金额超出还款能力。

| 参数 | 规则 |
|------|------|
| 重组本金 | 当前剩余本金 + 应还未还利息（罚息是否纳入根据方案类型决定） |
| 重组期数 | M1: 最长3期; M2: 最长6期; M3+: 最长12期（需人工审批） |
| 重组利率 | 不低于原合同利率，可根据客户评级给予利率优惠（最高下浮10%） |
| 每期最低还款额 | 不低于重组本金的 1/重组期数（等额本金方式） |
| 授权级别 | M1-M2期数范围内系统自动审批; M3+需人工审批 |

2.3 **部分减免结清方案设计 (Partial Waiver Settlement Plan)**

适用条件：客户能够一次性支付一定金额结清债务，但金额不足以覆盖全部欠款。

减免授权矩阵 (Waiver Authorization Matrix):

| 减免项目 | 自动审批权限 | 需人工审批 |
|---------|-------------|-----------|
| M1阶段滞纳金减免 | 减免比例 **≤30%** | 减免比例 >30% |
| M2阶段滞纳金减免 | 减免比例 **≤50%** | 减免比例 >50% |
| 罚息减免 | 减免金额 **≤5,000 CNY** 且减免比例 **≤50%** | 减免金额 >5,000 CNY 或减免比例 >50% |
| 利息减免 | 不支持自动审批 | 所有利息减免均需人工审批 |
| 本金减免 | 不支持自动审批 | 所有本金减免均需人工审批（极端情况适用） |

**关键约束**: 减免金额 >5,000 CNY **或** 减免比例 >50% 的任何方案，**必须上报人工审批**。

2.4 **NPV分析 (Net Present Value Analysis)**

每个方案必须计算NPV，量化机构的财务影响：

```
NPV方案 = Σ [预期回收现金流t / (1 + 折现率)^t] - 催收成本

其中：
- 预期回收现金流t = 方案约定的第t期还款金额 × 预期履约概率
- 折现率 = 机构资金成本率（通常取4%-6%）
- 催收成本 = 后续催收该案件的预期成本
- t = 还款期数（按月）

关键比较指标：
- NPV方案 vs NPV继续催收（不协商方案，按当前催收策略继续催收的预期回收）
- NPV方案 vs NPV法务处置（转入法务催收/诉讼的预期回收）
- NPV回收率 = NPV方案 / 欠款总额
```

方案可行性判定：
- **NPV方案 > NPV继续催收**: 方案对机构有利，推荐执行
- **NPV方案 ≤ NPV继续催收 但 NPV方案 > NPV法务处置**: 方案可接受，视客户情况决定
- **NPV方案 ≤ NPV法务处置**: 方案不可接受，建议转入法务处置通道

### 步骤 3：方案协商与确认 (Negotiation & Confirmation)

**目标**: 与客户沟通方案内容，达成一致并完成方案确认。

**执行动作**:

3.1 **方案呈现**
- 向客户清晰呈现1-3个可选方案，说明每个方案的：
  - 还款金额和期限
  - 减免内容（如适用）
  - 逾期记录处理（结清后征信更新说明）
  - 方案有效期（通常为方案确认后7天内需首笔还款/结清）
- 客观说明不同方案的利弊，不施加不当压力

3.2 **客户反馈处理**
- **客户接受方案**: 进入方案确认和签约流程
- **客户提出修改意见**: 在授权范围内调整方案参数，重新计算NPV
- **客户拒绝所有方案**: 记录客户反馈原因，评估是否需要上报更高权限审批或转入其他处理通道
- **客户需要考虑时间**: 给予合理考虑时间（通常3-5天），到期前主动跟进

3.3 **方案签约确认**
- 生成电子协议文件，明确：方案内容、还款计划表、违约条款（如未按方案还款则方案失效）
- 通过电子签名/短信验证码确认方式完成签约
- 方案生效后在核心银行系统中更新还款计划

### 步骤 4：方案执行跟踪 (Execution Tracking)

**目标**: 确保客户按方案约定执行还款，及时处理异常。

**执行动作**:

4.1 **还款提醒**
- 每期还款日前 **3天** 发送还款提醒（短信/APP推送）
- 还款日当天未还款的，当日晚间再次提醒

4.2 **履约监控**
- 实时监控方案下每期还款的到账情况
- 还款到账后确认金额是否与方案约定一致
- 全部还款完成后触发结案流程

4.3 **违约处理**
- 方案约定的还款日逾期 **3天** 仍未还款：电话联系客户了解原因
- 逾期 **7天**: 方案自动失效预警，通知客户方案面临取消风险
- 逾期 **15天**: 方案正式失效，案件退回原催收流程，由催收执行Agent继续跟进
- 方案失效后，已减免的金额恢复计入欠款余额

4.4 **执行结果记录**
- 方案执行结果反馈给催收执行Agent和催收数据分析Agent
- 记录方案达成率、履约率等关键指标
- 用于优化后续方案设计的参数和策略

## 四、最佳实践与约束

### 最佳实践

1. **"多方案对比"策略**: 始终设计2-3个差异化方案供客户选择（如"一次性结清享最大减免"与"分期还款零减免"），让客户有选择权的同时引导向机构最优方案
2. **NPV驱动决策**: 所有方案设计以NPV分析为核心依据。不要仅凭经验或直觉判断方案优劣，数据化比较才能确保方案对机构有利
3. **预期履约概率校准**: NPV计算中的"预期履约概率"应基于历史数据校准。相同逾期阶段、相同客户评级的历史方案履约率是最可靠的参考
4. **限时优惠机制**: 对于一次性结清方案，设置限时优惠（如"7天内结清额外减免5%"），创造紧迫感促进决策
5. **方案阶梯设计**: 在方案有效期内设置阶梯减免（如前3天结清减免最多，之后递减），激励客户尽早行动
6. **关注DTI合理性**: 分期重组方案中，每期还款金额不应超过客户月收入的 **50%**（含其他负债的月还款额），否则方案不可持续
7. **方案失败复盘**: 对于客户拒绝或未履约的方案，记录失败原因并反馈至方案参数优化。高频出现的失败模式应触发策略调整

### 约束条件

1. **授权矩阵硬限制**:
   - M1阶段滞纳金减免上限 **30%**（自动审批）
   - M2阶段可协商最长 **6期** 分期方案（自动审批）
   - 减免金额 >**5,000 CNY** 的方案必须上报人工审批
   - 减免比例 >**50%** 的方案必须上报人工审批
   - 任何涉及利息或本金减免的方案必须上报人工审批
2. **禁止虚假承诺**: 严禁承诺超出授权范围的减免方案，或做出"一定能批准"等不确定性承诺
3. **禁止利益输送**: 不得因客户的投诉威胁、媒体曝光威胁等因素而给予不合理的超规格减免
4. **方案有效期**: 所有方案必须设置有效期（通常为确认后7天），逾期未执行的方案自动失效
5. **方案排他性**: 同一案件同一时间只能有一个生效的还款方案，新方案生效前须终止旧方案
6. **NPV底线**: NPV回收率低于法务处置预期回收率的方案不得批准（除特殊审批外）
7. **征信记录**: 方案设计不得承诺修改客户征信记录。结清后征信记录更新按央行征信中心规则执行
8. **审计留痕**: 方案设计的全过程（评估、NPV计算、审批、客户确认、执行）必须完整记录，满足内部审计和监管检查要求

## 五、示例

### 示例 1：M2阶段分期重组方案

**客户情况**:
- 客户王某，消费贷产品，原贷款金额80,000 CNY，已还12期
- 当前逾期阶段：M2（逾期45天）
- 欠款明细：剩余本金 35,200 CNY，应还利息 1,760 CNY，罚息 880 CNY，滞纳金 420 CNY
- 欠款总额：38,260 CNY
- 逾期原因：因公司裁员失业，目前已找到新工作但薪资下降约30%
- 客户月收入：8,500 CNY（原12,000 CNY），其他负债月还 2,800 CNY
- 还款意愿评分：B级（高意愿 + 较低能力）

**方案设计**:

```
方案A: 6期分期重组（推荐方案）
├── 重组本金: 37,840 CNY（本金35,200 + 利息1,760 + 罚息880）
├── 滞纳金处理: 减免50%（减免210 CNY，需人工审批因M2减免但金额<5000且比例=50%）
├── 实际应还总额: 37,840 + 210 = 38,050 CNY
├── 重组期数: 6期
├── 重组利率: 原合同利率（年化12%）
├── 每期还款额: 约6,560 CNY
├── 客户DTI: (6,560 + 2,800) / 8,500 = 110% ← 超出50%限制！
└── 判定: DTI过高，需调整方案

方案A（修订）: 6期分期重组 + 前2期减半
├── 重组方式: 前2期每期还款3,280 CNY，后4期每期还款7,873 CNY
```
