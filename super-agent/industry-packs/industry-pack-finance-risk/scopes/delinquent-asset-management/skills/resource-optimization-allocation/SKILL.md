# 资源优化配置技能 (Resource Optimization Allocation)

## 技能概述

资源优化配置是智能分案Agent（case-allocator）的资源调度技能，负责将催收案件与催收资源（人工催收员、AI催收系统、法务团队、委外机构）进行最优匹配和分配。该技能在催收策略匹配完成后执行，解决的核心问题是：在有限的催收资源约束下，如何将正确的案件分配给最合适的资源，实现整体回收效率最大化。

本技能综合考虑以下关键因素：
- **资源产能约束**：每个催收员/系统的处理能力上限（案件数、通话时长、工作时间）
- **专业能力匹配**：资源的专长领域与案件特征的契合度
- **历史绩效数据**：资源的历史回收率、客户满意度、合规表现
- **工作负荷均衡**：避免资源间负荷差异过大，确保公平性和可持续性
- **时效性要求**：高优先级案件须优先获得优质资源

该技能的目标函数是：**在满足所有约束条件的前提下，最大化预期总回收金额**。

## 适用场景

| 场景编号 | 场景名称 | 触发条件 | 说明 |
|---------|---------|---------|------|
| UC-01 | 每日批量案件分配 | 策略匹配完成后自动触发 | 核心场景，将当日完成策略匹配的全量案件分配至具体资源 |
| UC-02 | 新案件实时分配 | 盘中新增逾期案件（非批量） | 对日间新增的逾期案件进行实时资源分配 |
| UC-03 | 资源变动后重平衡 | 催收员请假/离职/新入职 | 当资源池发生变化时，对受影响案件进行重新分配 |
| UC-04 | 绩效驱动的重分配 | 月度/周度绩效评估后 | 根据催收员绩效表现调整案件分配权重和案件类型 |
| UC-05 | 策略升级后资源调整 | 案件从M1升级至M2/M3+ | 策略升级时，将案件从自动化系统迁移至人工/法务资源 |
| UC-06 | 季末/年末冲刺调配 | 考核周期节点 | 在考核节点集中优质资源攻坚高价值案件 |

## 详细执行步骤

### 步骤一：资源池盘点与状态更新

**目标**：获取当前所有可用催收资源的实时状态和产能信息。

#### 1.1 人工催收员资源池

**采集信息**：

| 信息类别 | 具体字段 | 数据来源 |
|---------|---------|---------|
| 基础信息 | 员工ID、姓名、团队、职级、入职时间 | HR系统 |
| 排班状态 | 当日是否在岗、排班时段、是否请假 | 排班系统 |
| 产能参数 | 日最大处理案件数、日最大通话时长 | 产能配置表 |
| 当前负荷 | 在手案件数、今日已处理案件数、当前通话状态 | 催收系统实时数据 |
| 专业能力 | 擅长产品类型、擅长客户类型、语言能力、M阶段经验 | 能力标签系统 |
| 绩效数据 | 近30天回收率、近30天承诺履约率、近30天投诉率 | 绩效系统 |
| 授权额度 | 可处理的最大单案金额、减免审批权限 | 授权管理系统 |

**产能上限标准**：

| 催收员级别 | M1案件日上限 | M2案件日上限 | M3+案件日上限 | 日通话时长上限 |
|-----------|-------------|-------------|--------------|-------------|
| 初级催收员（<6个月） | 80件 | 40件 | N/A | 5小时 |
| 中级催收员（6-24个月） | 100件 | 60件 | 10件 | 6小时 |
| 资深催收员（>24个月） | 120件 | 80件 | 20件 | 7小时 |
| 组长/主管 | 60件 | 40件 | 30件 | 5小时 |

**负荷计算公式**：
```
当前负荷率 = 当前在手案件数 / 日最大处理案件数 × 100%

负荷状态判定：
├── 负荷率 < 70%  → GREEN（可接收新案件）
├── 负荷率 70%-90% → YELLOW（接近饱和，仅接收高优先级案件）
├── 负荷率 90%-100% → RED（饱和，不再分配新案件）
└── 负荷率 > 100%  → OVERLOAD（超载，需转移部分案件）
```

#### 1.2 AI催收系统资源池

| 资源类型 | 产能参数 | 当前状态 |
|---------|---------|---------|
| AI外呼系统 | 并发线路数（如500路）、日最大外呼量（如20,000通） | 当前并发数、今日已拨打数、成功接通数 |
| 短信发送通道 | 日最大发送量（如100,000条）、峰值发送速率 | 今日已发送数、通道拥堵状态 |
| APP推送系统 | 日最大推送量（无硬上限）、推送到达率 | 今日已推送数、推送队列长度 |

#### 1.3 法务团队资源池

| 资源类型 | 产能参数 | 说明 |
|---------|---------|------|
| 内部法务专员 | 同时处理案件数上限：30-50件/人 | 按案件金额分级：大额（>50万）、中额（10-50万）、小额（<10万） |
| 合作律师事务所 | 按合同约定的月度案件处理量 | 按地域和专长分配 |
| 委外催收机构 | 按合同约定的在管案件量上限 | 按地域覆盖范围和资质分配 |

### 步骤二：案件优先级排序

**目标**：对待分配案件按优先级排序，确保高价值、高紧迫度的案件优先获得优质资源。

**优先级评分模型**：

```
案件优先级分 = 金额权重(30%) + 时效权重(25%) + 回收概率权重(25%) + 策略需求权重(20%)

金额权重评分：
├── 逾期金额 > 100,000 → 90-100分
├── 逾期金额 50,000-100,000 → 70-90分
├── 逾期金额 20,000-50,000 → 50-70分
├── 逾期金额 5,000-20,000 → 30-50分
└── 逾期金额 < 5,000 → 10-30分

时效权重评分：
├── 临近阶段迁徙（DPD距阈值≤5天） → 90-100分
├── 阶段中段 → 40-60分
└── 刚进入新阶段 → 20-40分

回收概率权重：
├── 预期回收率 > 70% → 80-100分（优先投入，高产出）
├── 预期回收率 50%-70% → 60-80分
├── 预期回收率 30%-50% → 40-60分
└── 预期回收率 < 30% → 20-40分

策略需求权重：
├── 需要资深催收员/法务专家 → 80-100分
├── 需要普通人工催收员 → 50-70分
└── 自动化系统可处理 → 20-40分
```

**优先级分档**：

| 优先级 | 分值范围 | 分配规则 |
|--------|---------|---------|
| P0 - 紧急 | 85-100 | 优先分配最优资源，确保2小时内启动催收 |
| P1 - 高 | 70-85 | 优先分配，确保当日启动催收 |
| P2 - 中 | 50-70 | 正常分配，确保24小时内启动催收 |
| P3 - 低 | <50 | 按资源空余情况分配，48小时内启动 |

### 步骤三：案件-资源最优匹配

**目标**：执行案件与资源的最优化匹配，生成分配方案。

#### 3.1 M1案件自动化系统分配

M1案件主要分配至AI催收系统，分配逻辑相对简单：

```
M1自动化分配逻辑：

1. 检查AI外呼系统当日剩余容量
   ├── 剩余容量 > 待分配M1案件数 → 全量分配
   └── 剩余容量 < 待分配M1案件数 → 按优先级排序，优先分配高优先级案件

2. 渠道容量分配：
   ├── APP Push：无容量限制，全量推送
   ├── SMS短信：按通道剩余额度分配，优先高优先级
   └── AI外呼：按线路并发和时段分布进行排期
       ├── 上午时段（09:00-12:00）：30%案件量
       ├── 下午时段（14:00-18:00）：40%案件量
       └── 晚间时段（18:00-21:00）：30%案件量

3. M1-High案件人工催收兜底：
   └── AI外呼3次未接通的M1-High案件 → 自动转入人工催收队列
```

#### 3.2 M2案件催收员匹配算法

M2案件需要精细化的人工催收员匹配，这是本技能的核心难点。

**匹配模型 — 综合适配度评分**：

```
适配度(案件i, 催收员j) = Σ (维度评分 × 权重)

维度构成：
├── 能力匹配度 (Skill Match)          权重: 30%
│   ├── 产品类型匹配：催收员是否擅长该贷款产品
│   ├── 客户类型匹配：催收员是否擅长该客户画像
│   └── M阶段经验：催收员在该M阶段的处理经验
│
├── 绩效表现 (Performance)             权重: 25%
│   ├── 近30天同类案件回收率
│   ├── 近30天承诺履约率
│   └── 案件处理效率（平均处理天数）
│
├── 负荷均衡 (Workload Balance)        权重: 20%
│   ├── 当前负荷率（越低分越高）
│   ├── 当日已处理案件数
│   └── 在手案件金额总量均衡性
│
├── 地域/语言匹配 (Locale Match)       权重: 15%
│   ├── 催收员是否熟悉客户所在地区
│   ├── 方言能力匹配
│   └── 时区匹配（针对跨区域催收中心）
│
└── 合规表现 (Compliance)              权重: 10%
    ├── 近90天投诉率（越低分越高）
    ├── 合规抽检通过率
    └── 有无合规违规记录
```

**匹配约束（硬性条件，不满足即排除）**：

| 约束类型 | 规则 | 说明 |
|---------|------|------|
| 产能约束 | 催收员当前负荷率 < 100% | 已满载的催收员不分配新案件 |
| 金额授权 | 案件金额 ≤ 催收员授权上限 | 超出授权额度的案件不可分配 |
| 利益回避 | 催收员与客户无利益关联 | 同名/同地址/亲属关系排除 |
| 合规限制 | 催收员近30天无重大合规违规 | 有违规记录的催收员暂停分案 |
| 在岗状态 | 催收员当日在岗且处于可用状态 | 请假/培训/会议中排除 |
| 连续案件 | 同一客户多笔案件分配同一催收员 | 避免多头催收 |

**匹配求解算法**：

```
采用贪心算法 + 局部优化的两阶段方法：

阶段一：贪心初始分配
  1. 将案件按优先级从高到低排序
  2. 对每个案件，计算其与所有可用催收员的适配度分数
  3. 将案件分配给适配度最高且未满载的催收员
  4. 更新该催收员的负荷状态

阶段二：负荷均衡优化
  1. 检查所有催收员的负荷分布
  2. 如果最高负荷率与最低负荷率差距 > 20%，触发再平衡
  3. 从高负荷催收员中取出低优先级案件
  4. 重新分配至低负荷催收员（需满足最低适配度阈值 ≥ 60分）
  5. 迭代直到负荷差距 ≤ 20% 或无法进一步优化
```

#### 3.3 M3+案件法务资源分配

```
M3+资源分配逻辑：

法务催收案件（M3+-Recoverable）：
├── 大额案件（>500,000元）→ 分配高级法务专员 + 外部律所
├── 中额案件（100,000-500,000元）→ 分配中级法务专员
├── 小额案件（<100,000元）→ 批量处理（支付令/小额诉讼流程）
└── 匹配规则：
    ├── 按法务专员的地域管辖匹配（被告所在地法院）
    ├── 按案件类型专长匹配（信用贷vs抵押贷）
    └── 按法务专员当前在手案件数量平衡

委外催收案件（M3+-Marginal）：
├── 按客户地域匹配委外机构
├── 按委外机构历史绩效排名优先分配
├── 单个委外机构月度新增案件不超过合同约定上限
└── 同一案件不得同时委托多家机构

核销/转让案件（M3+-Write-off）：
├── 进入核销审批队列（按金额分级审批）
├── 或进入资产包打包队列（按打包规则汇集）
└── 不占用催收资源产能
```

### 步骤四：分配结果校验与冲突处理

**目标**：对分配结果进行完整性和合理性校验，处理分配冲突。

**校验规则**：

1. **完整性校验**
   - 所有待分配案件均已分配至具体资源（分配覆盖率 = 100%）
   - 若因资源不足导致部分案件无法分配，标记为`PENDING_ALLOCATION`并触发资源扩容告警

2. **合理性校验**
   - 单个催收员的分配案件数未超过日上限
   - 单个催收员的分配案件金额总量在合理范围内（避免集中风险）
   - 高优先级案件确实分配至了较优质的资源

3. **冲突处理**
   - 多个高优先级案件争抢同一催收员 → 按适配度分数决定，次优案件分配至次优催收员
   - 催收员产能溢出 → 将低优先级案件回退至待分配队列
   - 客户投诉后案件需更换催收员 → 从原催收员释放，重新匹配（排除原催收员）

### 步骤五：分配结果下发与执行

**目标**：将最终分配方案下发至各执行系统和人员。

**下发动作**：

| 资源类型 | 下发方式 | 内容 |
|---------|---------|------|
| AI催收系统 | API接口推送 | 案件列表、渠道编排、话术模板ID、时间排期 |
| 人工催收员 | 催收工作台任务推送 | 案件摘要、建议话术、客户信息、策略要求、优先级标记 |
| 法务专员 | 法务工单系统推送 | 案件详情、资产线索、法务评估报告、时限要求 |
| 委外机构 | 数据文件传输（加密） | 脱敏后的案件信息包（按数据安全规范传输） |

**催收员工作台信息包**：
```
每个M2案件分配至催收员时，生成如下信息包：

1. 案件摘要卡片
   ├── 客户姓名（脱敏）、年龄、职业
   ├── 贷款产品、逾期金额、逾期天数
   ├── 风险等级标签（如 M2-Mid）
   └── 优先级标签（如 P1-高优先级）

2. 催收历史时间线
   ├── M1阶段催收记录摘要
   ├── 各渠道触达记录和客户响应
   ├── 历史承诺还款记录和履约情况
   └── 客户情绪/态度标签

3. 建议沟通话术
   ├── 开场白模板
   ├── 关键沟通要点
   ├── 预设异议处理方案
   └── 还款方案选项和权限范围

4. 合规提醒
   ├── 客户特殊标记（如有）
   ├── 今日剩余可联系次数
   ├── 话术禁用词清单
   └── 录音质检要求
```

### 步骤六：分配效果监控与动态调整

**目标**：持续监控资源分配效果，在执行过程中进行动态调整。

**监控指标**：

| 指标名称 | 监控频率 | 告警阈值 | 响应动作 |
|---------|---------|---------|---------|
| 催收员负荷率分布 | 实时 | 任一催收员负荷 > 95% | 触发案件转移 |
| 案件首次触达时效 | 每小时 | P0案件2小时未触达 / P1案件当日未触达 | 告警+自动重分配 |
| AI外呼系统队列积压 | 每15分钟 | 队列长度 > 2倍日均 | 扩容线路/延后低优先级 |
| 催收员在线状态 | 实时 | 排班催收员离线 > 30分钟 | 将其案件临时托管给同组成员 |
| 分配均衡指数 | 每日 | 团队内负荷率标准差 > 15% | 次日分配时增加均衡权重 |

**动态调整机制**：

1. **日内微调**
   - 催收员临时请假：自动将其案件标记为`PENDING_REASSIGN`，在同组内重新分配
   - AI系统异常：降级处理，将AI外呼案件回退至短信渠道

2. **周度调整**
   - 根据本周催收效果数据，微调催收员的适配度评分参数
   - 对连续2周回收率低于团队均值的催收员，减少分配量并安排辅导

3. **月度优化**
   - 全面评估催收员绩效，更新能力标签
   - 调整产能上限参数（表现优秀者可适度提高上限）
   - 优化匹配算法权重参数

## 最佳实践与约束条件

### 最佳实践

1. **价值导向的资源配置**
   - 遵循"二八原则"：将80%的优质资源集中在贡献80%回收金额的20%高价值案件上
   - 高金额 + 高回收概率的案件优先获得最优资源
   - 对于回收概率极低的案件（<10%），应降低资源投入，避免无效消耗

2. **催收员成长路径设计**
   - 初级催收员先分配M1-Mid/Low案件积累经验，逐步过渡到M2案件
   - 为催收员设置能力成长目标，通过案件分配促进能力提升
   - 不应永远只将困难案件分配给最优秀的催收员（会导致人才流失）

3. **负荷弹性管理**
   - 日常分配按产能上限的80%控制，预留20%弹性应对盘中突发情况
   - 周末和节假日降低分配量至工作日的60%
   - 季末冲刺期可临时提高产能上限至110%（不超过2周）

4. **委外机构管理**
   - 建立委外机构的"赛马机制"：同类案件分配给多家机构，按季度绩效排名调整分配比例
   - 新合作的委外机构设置"试用期"（3个月），期间分配量不超过总量的10%
   - 对委外机构实施月度合规审计和季度绩效考核

5. **避免"樱桃采摘"（Cherry Picking）**
   - 案件分配须由系统自动完成，不允许催收员自行选择案件
   - 确保困难案件也被公平分配，而非全部集中在少数催收员身上
   - 监控催收员的案件处理顺序，防止只处理简单案件而搁置困难案件

### 约束条件

1. **产能硬约束**
   - 催收员日处理案件数严格不超过其级别对应的上限
   - 催收员日通话时长不超过规定上限（防止过劳）
   - AI外呼系统并发量不超过线路资源上限
   - 委外机构月度案件量不超过合同约定上限

2. **合规约束**
   - 同一案件不得同时分配给内部催收和委外机构（避免双重催收）
   - 催收员与客户不得存在利益关联
   - 委外案件传输须使用加密通道，客户信息脱敏处理
   - 催收员合规违规处于整改期的，不分配新案件

3. **公平性约束**
   - 团队内催收员负荷率标准差不超过15%
   - 不同催收员分配的案件难度应大致均衡（通过案件加权难度系数衡量）
   - 奖金挂钩的考核指标应公平对待案件难度差异

4. **时效约束**
   - 批量分配（1万件）须在30分钟内完成
   - 单案件实时分配须在30秒内完成
   - P0案件分配后2小时内须完成首次触达
   - 催收员请假导致的案件重分配须在1小时内完成

### 示例一：M2案件催收员最优匹配

**场景**：一笔个体工商户小微贷款M2案件，需要分配经验丰富的催收员。

**待分配案件**：
- 案件：M2-Mid，逾期金额18,500元，小微经营贷
- 客户：浙江杭州个体工商户，普通话
- 优先级：P1

**候选催收员对比**：

| 维度 | 张华 (COL-001) | 王芳 (COL-002) | 刘明 (COL-003) |
|------|----------------|----------------|----------------|
| 级别 | 资深 | 中级 | 资深 |
| 产品专长 | 小微贷 + 消费贷 | 信用卡 + 消费贷 | 小微贷 + 抵押贷 |
| 当前负荷 | 65%（52/80件） | 72%（43/60件） | 88%（70/80件） |
| 近30天回收率 | 48% | 42% | 52% |
| 方言能力 | 浙江话 | 无 | 四川话 |
| 投诉率 | 1.0‰ | 1.5‰ | 0.5‰ |

**匹配计算**：

```
张华适配度：
  能力匹配：92分 × 30% = 27.6（小微贷专长 + 浙江话能力）
  绩效表现：85分 × 25% = 21.25（回收率48%，处于中上水平）
  负荷均衡：78分 × 20% = 15.6（负荷65%，空间充裕）
  地域匹配：95分 × 15% = 14.25（浙江话匹配度高）
  合规表现：88分 × 10% = 8.8（投诉率低）
  总分：87.5

王芳适配度：
  能力匹配：45分 × 30% = 13.5（无小微贷经验）
  绩效表现：72分 × 25% = 18.0
  负荷均衡：62分 × 20% = 12.4
  地域匹配：50分 × 15% = 7.5
  合规表现：78分 × 10% = 7.8
  总分：59.2

刘明适配度：
  能力匹配：88分 × 30% = 26.4（小微贷专长）
  绩效表现：90分 × 25% = 22.5（回收率最高）
  负荷均衡：35分 × 20% = 7.0（负荷88%，接近饱和）
  地域匹配：55分 × 15% = 8.25（无浙江话能力）
  合规表现：95分 × 10% = 9.5
  总分：73.65
```

**分配结果**：张华（COL-001），适配度87.5分为最高。虽然刘明回收率更高，但负荷接近饱和且缺乏地域匹配优势。

### 示例二：资源不足时的优先级调度

**场景**：某日M2新增案件量激增30%，催收员资源不足以覆盖全量案件。

**当前资源状态**：
- M2催收团队：48人在岗
- 团队总产能：48 × 60（平均） = 2,880件/日
- 当日待分配：3,750件（超出产能30%）

**调度方案**：

```
阶段一：按优先级分配（覆盖核心案件）
  P0案件（85件）→ 全量分配至Top催收员，确保2小时触达
  P1案件（420件）→ 分配至资深/中级催收员
  P2案件（2,100件）→ 正常分配至可用催收员

  小计：已分配2,605件，剩余1,145件

阶段二：产能弹性调整
  1. 启动加班机制：自愿加班催收员产能提升20% → 释放约480件产能
  2. M1-High案件中原计划人工催收的部分降级为AI催收 → 释放约150件产能
  3. 小计：新增释放630件产能

  累计分配：3,235件，剩余515件

阶段三：溢出处理
  P3案件（515件）→ 标记DEFERRED，延后至T+1分配
  └── 对其中DPD接近60天的紧急案件优先处理
  └── 其余案件按优先级排入次日分配队列

告警输出：
  [CAPACITY_ALERT] M2催收资源缺口515件/日，建议：
  1. 短期：协调M1团队抽调5人支援M2（预计释放300件产能）
  2. 中期：评估招聘2-3名中级催收员
  3. 长期：优化M1自动化策略效果，减少M1→M2迁徙量
```

### 示例三：催收员请假后的案件重分配

**场景**：资深催收员刘明（COL-003）因病请假3天，其在手70件M2案件需要重新分配。

**重分配方案**：

```
步骤一：案件分类
  刘明在手70件案件按优先级分类：
  ├── P0/P1高优先级案件：15件（含3件近期承诺还款跟进）
  ├── P2普通案件：40件
  └── P3低优先级案件：15件

步骤二：高优先级案件紧急转移
  15件P0/P1案件 → 分配至同组资深催收员赵强（当前负荷70%）
  转移信息包含：
  ├── 刘明的催收记录和沟通备注
  ├── 客户当前沟通状态和待办事项
  └── 3件承诺还款案件的到期日期（需在到期前1天提醒）

步骤三：普通案件团队内分散
  40件P2案件 → 在M2-TEAM-A的6名可用催收员间均匀分配
  每人额外承接约7件，负荷增加约10%（在可接受范围内）

步骤四：低优先级案件延后
  15件P3案件 → 暂不分配，等刘明返岗后继续跟进
  期间设置自动提醒（短信），防止客户完全失联

步骤五：刘明返岗后的案件回收
  刘明返岗后：
  ├── P3案件自动回归刘明名下
  ├── P2案件中未完成的案件根据当前状态决定是否回归
  └── P0/P1案件中赵强已建立沟通关系的案件留在赵强名下
```
