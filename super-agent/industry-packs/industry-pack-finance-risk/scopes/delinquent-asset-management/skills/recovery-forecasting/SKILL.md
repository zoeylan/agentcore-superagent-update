# 回收预测技能 (Recovery Forecasting)

## 一、技能概述

### 1.1 技能定位

回收预测是催收数据分析Agent（collection-analytics）的前瞻性分析技能，负责基于多因素模型预测未来3个月的逾期资产回收金额。准确的回收预测是金融机构财务规划和拨备计提决策的关键输入——它直接影响损益表（P&L）中的减值准备（Impairment Provision）和利润预期，以及资本充足率的计算。

本技能的核心价值在于：将催收运营的"不确定性"转化为"可管理的风险区间"，使财务和管理层能够基于量化的预期做出前瞻性决策，而非被动应对。

### 1.2 核心能力

- **月度回收金额预测**：构建多因素预测模型，提供未来3个月的逐月回收金额预测
- **置信区间估计**：为每个预测值提供置信区间（通常为80%和95%），量化预测的不确定性
- **多情景预测**：提供基准情景、乐观情景、悲观情景三套预测，支持管理层的情景规划
- **预测偏差控制**：预测偏差须控制在 +/-10% 以内，超出偏差须分析原因并调整模型
- **预测因子分解**：将预测结果分解为各驱动因子的贡献，提升预测的可解释性

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 月度财务规划 | 为财务部门提供未来3个月的回收金额预测，支撑营收预测和现金流规划 |
| 拨备计提决策 | 回收预测直接影响预期信用损失（ECL, Expected Credit Loss）的计算，是拨备计提的核心输入 |
| 催收预算编制 | 基于回收预测确定催收投入预算，优化成本收入比 |
| 年度经营计划 | 为年度不良资产处置目标和回收目标提供量化依据 |
| 压力测试 | 结合宏观经济压力情景，评估极端条件下的回收金额变动 |
| 资产转让定价 | 预测剩余可回收金额，为不良资产包转让提供定价参考 |

## 二、执行步骤

### 步骤1：存量案件结构分析

**目标**：对当前在管的逾期资产进行结构化分析，作为预测的基础。

**分析内容**：

1. **M-stage 分布分析**：

```
在管案件结构快照（截至预测基准日）：

M-stage    在管余额(亿)    占比     户数(万)    占比    平均余额(万)
M1         6.80           44.7%    4.23       48.3%   1.61
M2         2.85           18.7%    1.52       17.4%   1.88
M3         1.43           9.4%     0.72       8.2%    1.99
M3+(非核销) 3.15          20.7%    1.78       20.3%   1.77
已核销在催   0.99          6.5%     0.52       5.9%    1.90
合计        15.22         100%     8.77       100%    1.74
```

2. **案件账龄分布**：各M-stage内的DPD（逾期天数）分布
3. **案件金额分布**：大/中/小额案件的占比及历史回收率差异
4. **催收阶段分布**：各案件当前所处的催收阶段（首次催收/跟进催收/法务阶段等）
5. **客群画像分布**：风险评分、产品类型、地区等维度的结构

### 步骤2：历史回收模式分析

**目标**：从历史数据中提取各M-stage的回收率模式和时间衰减特征。

**分析方法**：

1. **分M-stage的月度回收率曲线**：

```
各M-stage的典型月度回收率（占期初余额比例）：

进入该M-stage后的月数:  Month1   Month2   Month3   Month6   Month12
M1回收率:               12.5%    6.8%     4.2%     1.5%     0.3%
M2回收率:               8.2%     5.1%     3.5%     1.2%     0.2%
M3回收率:               5.5%     3.8%     2.8%     1.0%     0.2%
M3+回收率:              2.8%     2.2%     1.8%     0.8%     0.1%
核销回收率:             1.2%     0.9%     0.7%     0.3%     0.05%
```

2. **回收率的时间衰减模型**：
   - 回收率通常遵循指数衰减模式：R(t) = R₀ × e^(-λt)
   - 其中 R₀ 为初始回收率，λ 为衰减系数，t 为进入该M-stage后的月数
   - 不同M-stage的 R₀ 和 λ 参数不同

3. **回收率的条件分布**：
   - 按产品类型细分的回收率差异
   - 按余额区间细分的回收率差异（大额案件通常回收率较低但绝对金额更高）
   - 按催收渠道/策略细分的回收率差异

### 步骤3：季节性因素建模

**目标**：识别和量化影响回收金额的季节性波动模式。

**季节性因素**：

1. **月度季节性（Monthly Seasonality）**：
   - 春节月份（1-2月）：回收率通常下降15-25%（催收暂停 + 客户消费高峰）
   - 年末月份（11-12月）：回收率通常上升10-15%（年终奖发放 + 催收年末冲量）
   - 季末月份（3/6/9/12月）：可能因考核压力略有上升

2. **周内效应（Day-of-Week Effect）**：
   - 周一/周五回款金额通常高于周中
   - 月初（发薪日后）回款集中度较高

3. **节假日效应**：
   - 国庆、春节等长假前后的回收率变化
   - 双十一等消费高峰对客户还款能力的影响

**季节性建模方法**：
- 使用加法/乘法季节性分解：Y(t) = Trend(t) × Seasonal(t) × Residual(t)
- 或使用STL分解（Seasonal and Trend decomposition using Loess）
- 计算季节性系数（Seasonal Index）：各月回收金额 / 12个月移动平均

### 步骤4：宏观经济指标纳入

**目标**：将宏观经济环境变化纳入预测模型，捕捉系统性风险因素。

**关键宏观指标**：

| 宏观指标 | 影响方向 | 领先期 | 说明 |
|----------|----------|--------|------|
| GDP增速 | 正相关 | 1-2季度 | 经济增长改善客户还款能力 |
| 城镇调查失业率 | 负相关 | 1-3个月 | 失业率上升直接影响还款能力 |
| PMI（制造业采购经理指数） | 正相关 | 1-2个月 | 经济景气度的领先指标 |
| CPI（居民消费价格指数） | 负相关 | 即时 | 通胀侵蚀可支配收入 |
| 社会融资规模增量 | 正相关 | 1-3个月 | 信用环境宽松有助于客户"借新还旧" |
| 居民可支配收入增速 | 正相关 | 即时 | 直接影响还款能力 |
| 房价指数 | 正相关 | 3-6个月 | 影响有抵押贷款的回收预期 |

**纳入方法**：
- 计算各宏观指标与历史回收率的交叉相关系数（Cross-Correlation），确定最优领先期
- 在回归模型中作为外生变量（Exogenous Variables）纳入
- 使用最新发布的宏观数据和市场一致预期（Consensus Forecast）

### 步骤5：预测模型构建与运算

**目标**：综合案件结构、历史模式、季节性和宏观因素，构建月度回收金额预测模型。

**模型架构**：

```
月度回收预测 = Σ(各M-stage的预测回收金额)

各M-stage预测回收金额 = 期初在管余额
                      × 基础回收率（基于历史模式和账龄衰减）
                      × 季节性调整系数
                      × 宏观经济调整系数
                      × 催收策略调整系数（如有策略变更）
```

**具体模型选择**：

1. **基础模型——分层回收率模型（Segmented Recovery Rate Model）**：
   - 将案件按M-stage × 产品类型 × 账龄分层
   - 每层使用历史平均回收率作为基准预测
   - 优点：简单透明，易于解释
   - 适用：数据量有限或模型初建阶段

2. **进阶模型——时间序列模型**：
   - ARIMA / SARIMA：捕捉趋势和季节性
   - Prophet（Facebook开源）：适合有节假日效应的时间序列
   - 优点：自动捕捉趋势和季节性
   - 适用：有24个月以上的历史回收数据

3. **高级模型——多因素回归模型**：
   - 因变量：月度回收金额
   - 自变量：期初在管余额（各M-stage）、迁徙率、季节性哑变量、宏观指标
   - 方法：多元线性回归 / 弹性网回归（Elastic Net）/ 梯度提升树（Gradient Boosting）
   - 优点：多因素综合考量，预测精度更高
   - 适用：有丰富的历史数据和宏观数据

**置信区间估计**：
- 基于历史预测误差的分布计算置信区间
- 80%置信区间：预测值 +/- 1.28 × 历史预测标准误差
- 95%置信区间：预测值 +/- 1.96 × 历史预测标准误差
- 或使用Bootstrap方法进行非参数置信区间估计

### 步骤6：多情景预测与敏感性分析

**目标**：提供多套情景预测，覆盖不同的假设条件。

**情景设定**：

| 情景 | 假设条件 | 说明 |
|------|----------|------|
| 基准情景（Base Case） | 当前趋势延续，宏观环境稳定 | 最可能发生的情景，作为主预测值 |
| 乐观情景（Upside Case） | 催收策略优化见效 + 宏观环境改善 | 回收率在基准基础上提升10-15% |
| 悲观情景（Downside Case） | 催收效果下滑 + 宏观环境恶化 | 回收率在基准基础上下降15-25% |
| 极端压力情景（Stress Case） | 经济衰退，失业率显著上升 | 回收率在基准基础上下降30-50%，仅用于压力测试 |

**敏感性分析**：
- 单因素敏感性：分别调整关键假设（回收率 +/-5pp、迁徙率 +/-2pp、新增逾期 +/-20%），观察预测结果变动
- 输出敏感性瀑布图（Waterfall Chart），展示各因素对预测结果的边际影响

### 步骤7：预测偏差监控与模型校准

**目标**：持续监控预测准确性，确保偏差在 +/-10% 以内。

**偏差计算**：

```
预测偏差 = (预测值 - 实际值) / 实际值 × 100%

可接受范围：-10% ≤ 偏差 ≤ +10%
```

**偏差监控机制**：

1. **月度偏差回顾**：每月实际回收金额出来后，与3个月前的预测值对比
2. **偏差归因分析**：当偏差超出 +/-10% 时，分析驱动因素：
   - 案件结构变化是否超出预期？
   - 回收率是否偏离历史模式？
   - 是否有未预见的宏观事件？
   - 是否有催收策略的重大调整？
3. **模型校准**：根据偏差分析结果调整模型参数
   - 定期（至少每季度一次）使用最新数据重新拟合模型
   - 滚动窗口更新：使用最近12-24个月数据重新估计参数

**偏差追踪表**：

```
预测偏差追踪：

预测月   目标月   预测值(亿)  实际值(亿)  偏差      状态
2025-12  2026-01  3.50       3.38       +3.5%    ✅ 达标
2025-12  2026-02  3.20       2.85       +12.3%   ❌ 超出（春节效应低估）
2025-12  2026-03  3.65       3.75       -2.7%    ✅ 达标
2026-01  2026-02  2.95       2.85       +3.5%    ✅ 达标（校准后改善）
2026-01  2026-03  3.60       3.75       -4.0%    ✅ 达标
2026-01  2026-04  3.80       --         --       待验证
```

## 四、最佳实践与约束

### 4.1 预测准确性要求

- **核心硬性指标**：月度回收金额的预测偏差须控制在 **+/-10%** 以内
- **偏差超限处理**：连续2个月偏差超过 +/-10%，须在T+5日内完成偏差归因分析并提交模型校准方案
- **模型评估指标**：
  - MAPE（Mean Absolute Percentage Error）< 10%
  - 方向准确率（预测增减方向与实际一致）> 80%
  - 80%置信区间覆盖率 > 75%（即实际值落在80%CI内的比例）

### 4.2 模型管理要求

- **模型文档化**：每个预测模型须有完整的模型文档，包含：模型假设、变量选择、参数估计方法、验证结果
- **模型版本管理**：每次模型更新须记录版本号、更新内容、更新原因、前后预测精度对比
- **模型审查**：预测模型须每半年接受一次独立审查（Model Validation），确认模型仍适用于当前环境
- **避免过拟合**：模型的训练集和验证集须分开（至少保留最近3个月作为样本外验证集），不得使用全量数据拟合后直接用于预测

### 4.3 假设管理

- **假设透明化**：所有预测假设必须明确记录，包括：
  - 回收率假设（各M-stage）
  - 迁徙率假设（案件结构演变）
  - 新增逾期假设
  - 宏观经济假设
  - 催收策略假设（是否有变更）
- **假设审批**：关键假设变更须经风控/财务负责人审批
- **假设敏感性**：须展示各假设的敏感性分析，使决策者理解假设变化对预测的影响

### 4.4 约束与限制

- 回收预测的输出是**决策支持信息**，不直接触发催收执行动作
- 预测结果用于拨备计提时，须与拨备充足性评估技能（provision-adequacy-assessment）协同工作
- 预测模型不包括以下极端场景的自动处理：重大政策变化（如大规模债务减免）、系统性金融危机、自然灾害等，这些需通过专项压力测试评估
- 预测期限超过3个月时，置信区间将显著扩大，**6个月以上的预测仅作参考**，不建议作为精确计划依据
- 新产品上线不足12个月的，由于历史数据不足，须使用类似产品的经验数据作为先验假设，并标注"数据受限预测"

## 五、示例

### 示例1：标准月度回收预测

**场景**：2026年4月初生成未来3个月（4月/5月/6月）的回收金额预测，支撑Q2财务规划。

**输入**：
```json
{
  "forecast_date": "2026-04-01",
  "forecast_horizon": 3,
  "confidence_levels": [0.80, 0.95]
}
```

**执行流程**：
1. 分析截至2026-03-31的在管案件结构
2. 提取近24个月的历史月度回收数据
3. 获取最新迁徙率矩阵（来自roll-rate-analysis技能）
4. 纳入季节性因素：4月无特殊节假日效应，5月有五一假期微弱影响，6月为季末

### 示例2：催收策略变更的回收影响预测

**场景**：催收团队计划从4月起在M1阶段推广DPD1首次触达策略（A/B测试已验证有效），需评估该策略变更对未来回收金额的影响。

### 示例3：预测偏差归因分析

**场景**：2026年2月的实际回收金额为2.85亿元，而3个月前（2025年12月）的预测值为3.20亿元，偏差+12.3%，超出 +/-10% 阈值。

**输入**：
```json
{
  "target_month": "2026-02",
  "forecast_value": 320000000,
  "actual_value": 285000000,
  "deviation_pct": 0.123
}
```

**执行流程**：
1. 对比预测假设与实际情况
2. 分解偏差的来源
3. 提出模型校准建议

**输出（偏差归因报告）**：
```
2026年2月回收预测偏差归因分析

预测偏差：+12.3%（预测3.20亿 vs 实际2.85亿），超出+/-10%阈值

偏差分解：
  偏差来源               贡献金额    贡献占比
  ─────────────────────────────────────────
  春节效应低估            -2,100万   60.0%    ← 主因
  M1回收率偏离            -850万     24.3%
  新增逾期超出预期         -350万     10.0%
  其他（宏观+策略因素）     -200万     5.7%
  ─────────────────────────────────────────
  总偏差                  -3,500万   100%

归因分析：
  1. 春节效应低估（60%）：模型使用的春节月季节性系数为0.88，
     但2026年春节在1月底-2月初，影响了2月前两周的催收作业，
     实际季节性系数应为0.82。历史数据中春节跨月的样本较少，
     导致系数估计偏高。

  2. M1回收率偏离（24.3%）：2月实际M1回收率为10.8%，
     低于预测假设的12.5%。主要原因是华东地区个人消费贷回收率
     下降至8.2%（预测12.0%），与迁徙率恶化趋势一致。

模型校准建议：
  1. 更新春节月季节性系数：区分春节在1月/2月/跨月三种场景，
     分别设定不同的季节性系数
  2. 纳入迁徙率变动对回收率预测的影响：当M1→M2迁徙率上升时，
     相应下调M1回收率预测
  3. 校准后，对2月的事后预测值为2.92亿（偏差+2.5%），达标
```
