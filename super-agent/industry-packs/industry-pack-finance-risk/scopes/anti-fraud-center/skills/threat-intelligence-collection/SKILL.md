# 威胁情报收集 (Threat Intelligence Collection)

## 技能概述

威胁情报收集是欺诈情报分析Agent的核心能力，通过多渠道、结构化的情报收集和评估，为机构提供欺诈威胁的早期预警。金融欺诈领域的黑产技术升级周期约3-6个月，及时的情报收集和评估使机构能够在威胁转化为实际损失之前部署防御。

### 适用场景
- 监管风险提示和行业协会通报的接收与评估
- 黑产论坛技术动态的跟踪监控
- 暗网数据泄露事件的影响评估（涉及本机构客户数据）
- 同业欺诈案例的收集与分析
- 恶意软件/钓鱼网站的技术情报分析

## 执行步骤

### 第一步：情报源管理
1. 维护多渠道情报源清单（至少5个独立来源）：
   - **监管渠道**：央行反洗钱局风险提示、金融监管总局通报、公安部经侦局预警
   - **行业协会**：中国支付清算协会通报、中国互联网金融协会简报
   - **同业网络**：与其他金融机构反欺诈团队的定期信息交换
   - **技术监控**：黑产技术论坛爬虫、暗网监控服务（合规供应商提供）
   - **第三方情报**：安全厂商威胁情报订阅（如金融ISAC成员共享）
2. 每月评估各情报源的价值（情报质量、时效性、独特性），淘汰低效来源
3. 确保所有情报收集手段在法律合规框架内

### 第二步：情报接收与初筛
1. 建立自动化情报接收管道：
   - 监管/协会通报：邮件/系统自动接收，关键词触发提醒
   - 技术监控：爬虫/API自动采集，增量更新
   - 同业共享：按协议定期交换
2. 初筛过滤：
   - 去重（与已有情报库比对）
   - 相关性初判（与本机构业务场景的关联度）
   - 时效性评估（情报的新鲜度，超过30天的降级处理）

### 第三步：结构化处理
1. 将原始情报标准化为结构化格式：
   - 情报ID、来源、接收时间、原始内容
   - 欺诈类型分类（按统一分类体系）
   - 攻击向量（技术手段描述）
   - 目标场景（适用的业务渠道/产品类型）
   - 涉及的IOC（Indicators of Compromise）：IP段、设备特征、账户模式等
2. 标注情报可信度（基于来源可靠性和佐证信息）：
   - A级（高可信）：监管正式通报、多源交叉验证
   - B级（中可信）：单一可靠来源、有部分佐证
   - C级（低可信）：单一非官方来源、未验证

### 第四步：威胁评估
1. 对每条结构化情报评估对本机构的威胁等级：
   - **高威胁**：直接威胁本机构（如针对本机构的定向攻击、大规模客户数据泄露、已在同类机构造成重大损失的新型手法）
   - **中威胁**：间接相关（如同行业其他机构发生的新型欺诈、黑产技术更新但尚未针对本机构）
   - **低威胁**：参考性（如其他行业的欺诈趋势、通用安全漏洞通报）
2. 威胁评估维度：
   - 攻击可行性（本机构是否存在相同脆弱点）
   - 潜在影响范围（涉及的客户群/交易量/金额规模）
   - 时间紧迫性（攻击者是否已具备攻击能力）

### 第五步：分发与响应
1. **高威胁情报**：
   - 2小时内完成评估并生成紧急预警
   - 直接推送至规则策略Agent要求立即响应
   - 同步通知案件调查Agent排查是否已有关联事件
   - 抄送反欺诈负责人和安全团队
2. **中威胁情报**：
   - 按常规周期纳入规则策略Agent的迭代排期
   - 在周度情报简报中重点呈现
3. **低威胁情报**：
   - 归档入欺诈知识库
   - 在月度情报报告中概述

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| intelligenceSource | enum | 是 | 情报来源类型 |
| rawContent | string | 是 | 原始情报内容 |
| sourceCredibility | enum | 否 | 来源可信度 A/B/C |
| receivedTimestamp | datetime | 是 | 接收时间 |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| intelligenceId | string | 情报唯一标识 |
| structuredIntel | object | 结构化情报（标准格式） |
| threatLevel | enum | HIGH/MEDIUM/LOW |
| credibilityRating | enum | A/B/C |
| impactAssessment | object | 影响评估 |
| iocList | array | 相关IOC列表 |
| distributionActions | array | 分发动作记录 |
| responseDeadline | datetime | 响应截止时间（高威胁情报） |

## 最佳实践
- 建立情报自动化处理管道（接收→初筛→结构化→评估→分发），减少人工环节
- 高威胁情报评估采用"双人评审"机制，降低误判风险
- 维护IOC库并与实时监控系统对接，实现情报到规则的快速转化
- 与同业建立互信的情报共享机制，遵循TLP（Traffic Light Protocol）信息共享标准

## 约束条件
- 情报收集必须在法律合规框架内，禁止使用非法手段
- 涉及客户隐私数据的情报处理须符合《个人信息保护法》
- 高威胁情报必须在2小时内完成评估和响应建议输出
- 情报来源不少于5个独立渠道，避免信息来源单一
