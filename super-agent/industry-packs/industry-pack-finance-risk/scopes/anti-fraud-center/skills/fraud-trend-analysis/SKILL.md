# 欺诈趋势分析 (Fraud Trend Analysis)

## 技能概述

欺诈趋势分析是规则策略Agent的战略分析能力，通过对本机构历史欺诈数据和行业趋势的系统化分析，识别欺诈态势的变化趋势，为检测策略的前瞻性调整提供数据支撑。该技能连接了"事后分析"与"事前防御"。

### 适用场景
- 月度/季度反欺诈态势报告编制
- 欺诈类型分布变化趋势追踪
- 新型欺诈手法的统计识别（规则未覆盖的聚类异常）
- 规则有效性的周期性评估
- 跨渠道、跨产品的欺诈风险对比分析

## 执行步骤

### 第一步：数据准备
1. 汇集分析所需的多源数据：
   - 案件调查Agent产出的已定性案件库（至少近12个月）
   - 实时监控Agent的规则命中日志和模型评分日志
   - 交易总量数据（分渠道、分产品、分客户群体）
   - 行业通报和同业共享的欺诈趋势数据
2. 数据清洗与标准化：统一时间粒度、金额单位、分类编码

### 第二步：多维度趋势分析
1. **欺诈总量趋势**：
   - 月度欺诈案件数、涉案总金额、欺诈损失率（欺诈损失/交易总额）
   - 同比、环比变化，识别季节性规律和长期趋势
2. **欺诈类型分布**：
   - 各欺诈类型（第一方/第三方/团伙/内部/网络）的占比及变化
   - 新兴欺诈类型的增长率监控
3. **渠道分布分析**：
   - 各交易渠道（手机银行/网银/POS/API等）的欺诈发生率对比
   - 识别高风险渠道和渠道间的风险迁移趋势
4. **客户群体分析**：
   - 按年龄段、客户等级、产品类型等维度分析欺诈分布
   - 识别高风险客户群体特征

### 第三步：检测效果评估
1. **整体效果指标**：
   - 欺诈检出率（已拦截欺诈金额/总欺诈金额）趋势
   - 误报率趋势
   - 平均检测响应时间趋势
2. **规则有效性排名**：
   - 各规则的检出贡献度排名（按拦截金额）
   - 各规则的误报率排名
   - 识别高效规则（高检出低误报）和低效规则（低检出高误报）
   - 输出Top10高效规则和Bottom10低效规则

### 第四步：异常模式发现
1. 对未被规则命中但事后被确认为欺诈的案件进行聚类分析
2. 识别规则引擎和模型共同未覆盖的"盲区"
3. 提炼盲区案件的共同特征，形成新规则的候选模式
4. 对规则命中率的异常波动（突然增高或降低）进行根因分析

### 第五步：态势报告输出
1. 编制结构化态势报告：
   - 执行摘要（关键发现和行动建议）
   - 欺诈总量与趋势分析
   - 欺诈类型与渠道分布
   - 检测效果评估
   - 新发现的风险点和盲区
   - 下一周期策略重点建议
2. 报告受众：反欺诈团队（详细版）、管理层（摘要版）、监管报送（合规版）

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| analysisPeriod | object | 是 | 分析时间窗口 |
| reportType | enum | 是 | MONTHLY/QUARTERLY/AD_HOC |
| dimensions | array | 否 | 分析维度（默认全维度） |
| benchmarkData | object | 否 | 行业对标数据 |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| trendReport | object | 完整趋势分析报告 |
| keyFindings | array | 关键发现列表 |
| riskAlerts | array | 风险预警项 |
| ruleEffectiveness | object | 规则有效性评估 |
| blindSpotAnalysis | object | 检测盲区分析 |
| actionRecommendations | array | 策略调整建议 |

## 最佳实践
- 趋势分析需要足够长的时间跨度（建议至少12个月），才能识别季节性规律
- 与行业对标数据结合分析，避免"井底之蛙"——本机构指标改善可能只是行业整体好转
- 将趋势分析结果自动关联到规则优化工作流，实现"数据驱动策略迭代"的闭环
- 关注"未遂欺诈"（被拦截的欺诈尝试）的趋势，它反映了攻击者的尝试强度和策略变化

## 约束条件
- 月度报告在次月T+3个工作日内产出
- 数据分析必须基于已标注数据，未定性案件不纳入统计
- 报告中的对标数据引用需标注来源
- 管理层摘要版报告不超过2页，聚焦关键数据和行动建议
