# 监管报送材料生成 (Regulatory Report Generation)

## 技能概述

监管报送材料生成是案件调查Agent满足合规义务的关键能力。根据《反洗钱法》和人民银行相关规定，金融机构必须对符合条件的可疑交易在规定时限内向中国反洗钱监测分析中心报送可疑交易报告。重大欺诈案件要求24小时内完成报送材料准备。

### 适用场景
- 确认欺诈案件的可疑交易报告（STR）生成
- 重大欺诈案件（涉案金额>50万元或涉及团伙）的紧急报送
- 大额交易报告（CTR）的自动生成
- 公安机关协查函的回复材料准备
- 监管专项检查的历史案件材料汇编

## 执行步骤

### 第一步：报送条件判定
1. 检查案件是否满足可疑交易报告的触发条件：
   - 确认欺诈案件（定性结论=CONFIRMED_FRAUD）→ 必须报送
   - 涉案金额超过50万元人民币 → 重大可疑交易，24小时内报送
   - 涉及团伙（3人以上关联）→ 重大可疑交易
   - 涉及跨境资金流动 → 可疑交易+大额交易双重报送
   - 涉及特定高风险国家/地区 → 紧急报送
2. 确定报告类型：可疑交易报告（STR）、大额交易报告（CTR）、紧急报告

### 第二步：报告要素提取
1. 从案件调查报告中提取标准报告要素：
   - **主体信息**：涉案自然人/法人的身份信息（姓名/名称、证件号码、地址、联系方式）
   - **账户信息**：涉案账户的完整开户信息、账户类型、开户日期
   - **交易信息**：可疑交易明细（日期、金额、币种、对手方、交易渠道、交易摘要）
   - **可疑特征描述**：触发报送的可疑特征、欺诈手法分析、交易异常点
   - **机构已采取措施**：已执行的拦截、冻结、黑名单等处置措施
2. 数据脱敏检查：确认报告中不包含系统内部ID、模型评分等技术细节

### 第三步：报告编制
1. 按照人民银行《金融机构大额交易和可疑交易报告管理办法》规定的格式编制报告
2. 可疑特征描述采用标准化的可疑交易特征代码（参照反洗钱监测分析中心的特征分类编码）
3. 补充机构分析意见：基于调查结论给出机构对可疑程度的判断
4. 附件准备：交易流水、关联图谱、时间线等支撑材料

### 第四步：合规审查
1. 报告内容的完整性检查：所有必填字段已填写，数据准确无矛盾
2. 格式合规性检查：符合电子化报送的XML/JSON格式要求
3. 时效性检查：确认在规定时限内完成（重大案件24小时、一般案件5个工作日）
4. 提交至合规部门进行最终审签

### 第五步：报送与归档
1. 通过人民银行反洗钱数据报送系统（AMLD系统）进行电子化报送
2. 接收报送回执，确认报送成功
3. 完整报送材料归档，保留期≥5年
4. 后续如监管机构要求补充材料，在收到要求后3个工作日内响应

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| caseId | string | 是 | 案件编号 |
| investigationReport | object | 是 | 完整调查报告 |
| reportType | enum | 是 | STR(可疑交易报告)/CTR(大额交易报告)/URGENT(紧急报告) |
| deadline | datetime | 是 | 报送截止时间 |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| reportId | string | 报告编号 |
| reportType | enum | 报告类型 |
| reportContent | object | 报告完整内容（结构化） |
| complianceCheckResult | object | 合规检查结果 |
| submissionStatus | enum | DRAFT/PENDING_REVIEW/SUBMITTED/ACKNOWLEDGED |
| submissionTimestamp | datetime | 报送时间 |

## 最佳实践
- 建立报送模板库，覆盖常见欺诈类型的标准化描述
- 可疑特征描述要做到"事实清晰、逻辑完整、语言规范"，避免主观臆断
- 维护与反洗钱监测分析中心的沟通渠道，及时了解报送格式更新
- 定期回顾历史报送的监管反馈，优化报告质量

## 约束条件
- 重大可疑交易报告必须在24小时内完成报送
- 报告内容不得包含未经验证的推测信息
- 报告中的客户信息使用限制：仅用于反洗钱/反欺诈目的，不得泄露
- 报送系统登录需双因素认证，操作日志完整留存
