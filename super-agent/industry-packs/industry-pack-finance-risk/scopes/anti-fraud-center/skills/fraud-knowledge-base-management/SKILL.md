# 欺诈知识库管理 (Fraud Knowledge Base Management)

## 技能概述

欺诈知识库管理是欺诈情报分析Agent维护机构反欺诈知识资产的核心能力。知识库是反欺诈团队的"组织记忆"，系统化地存储和管理欺诈手法分类、典型案例、黑产工具链图谱和防御方案，为规则制定、案件调查和人员培训提供知识支撑。

### 适用场景
- 新型欺诈手法的知识条目创建和分类
- 典型欺诈案例的结构化入库（含作案手法、检测要点、防御方案）
- 黑产工具链图谱的更新维护
- 知识库的季度审查和内容更新
- 面向新员工培训的案例库输出

## 执行步骤

### 第一步：知识分类体系维护
1. 维护统一的欺诈手法分类体系（三级分类）：
   - **一级分类**：第一方欺诈、第三方欺诈、团伙欺诈、内部欺诈、网络欺诈
   - **二级分类**（示例）：
     - 第三方欺诈 → 身份盗用、账户盗用、卡片盗刷、SIM卡劫持
     - 团伙欺诈 → 骗贷团伙、跑分团伙、养卡套现团伙、电信诈骗团伙
     - 网络欺诈 → 钓鱼攻击、社会工程、木马/远控、中间人攻击
   - **三级分类**：具体手法变种
2. 每季度审查分类体系，根据新发现的欺诈手法增加/调整分类

### 第二步：知识条目创建
1. 新增知识条目时，按标准模板填写：
   - **基本信息**：条目ID、标题、分类（三级）、创建日期、更新日期、作者
   - **手法描述**：攻击流程（步骤化描述）、使用的工具/技术、典型特征
   - **检测要点**：关键检测信号、可用于识别的特征、相关规则ID
   - **防御方案**：推荐的防御措施、已部署的规则/模型覆盖情况
   - **典型案例**：关联的实际案件编号（脱敏处理）
   - **参考来源**：情报来源、行业报告链接、学术论文引用
2. 对敏感信息进行脱敏处理（客户姓名、账号等替换为代码）

### 第三步：案例库管理
1. 典型案例入库标准：
   - 代表性新型欺诈手法（首次发现）
   - 高金额案件（涉案>100万元）
   - 团伙案件（涉及3人以上关联网络）
   - 具有培训价值的经典案例
2. 案例结构化格式：
   - 案例概述（一句话摘要）
   - 作案手法详解（含时间线和资金流转图）
   - 检测经过（如何被发现、哪些规则/模型命中）
   - 调查过程（关键调查步骤和发现）
   - 处置结果（追回金额、账户处置、是否报送监管）
   - 经验教训（检测改进建议）

### 第四步：黑产图谱维护
1. 维护黑产工具链图谱，覆盖欺诈全链条：
   - **信息获取环节**：社工库、数据泄露、钓鱼获取
   - **身份伪造环节**：虚假证件制作、活体检测绕过工具、代理IP/VPN
   - **欺诈执行环节**：批量注册工具、自动化脚本、远程控制软件
   - **资金转移环节**：跑分平台、虚拟货币洗钱、多层分散转账
2. 记录各环节的技术细节、成本和可获取性，评估威胁水平

### 第五步：知识库质量管理
1. 每季度执行知识库审查：
   - 过期内容标识（超过12个月未更新且相关手法已演变的条目）
   - 引用频次统计（识别高价值和低利用率条目）
   - 分类覆盖度评估（是否存在分类盲区）
2. 更新指标：季度更新率≥20%（知识条目数）
3. 生成知识库健康度报告

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| operationType | enum | 是 | CREATE/UPDATE/QUERY/REVIEW |
| knowledgeType | enum | 是 | FRAUD_METHOD/CASE_STUDY/TOOL_CHAIN/DEFENSE_GUIDE |
| content | object | 条件必填 | 知识条目内容（创建/更新时） |
| queryParams | object | 条件必填 | 查询参数（查询时） |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| entryId | string | 知识条目ID |
| operationResult | enum | SUCCESS/FAILED/NEEDS_REVIEW |
| knowledgeEntry | object | 知识条目完整内容 |
| relatedEntries | array | 关联知识条目推荐 |
| kbHealthReport | object | 知识库健康度报告（审查时） |

## 最佳实践
- 知识库与规则库建立双向链接——每条规则关联其检测的欺诈手法知识条目，每个知识条目关联已部署的防御规则
- 建立"知识贡献激励机制"，鼓励调查员提交新发现的欺诈手法
- 知识库支持全文搜索和标签检索，降低使用门槛
- 定期输出"高频查阅知识条目排行"，了解团队关注焦点

## 约束条件
- 知识库中不得存储未脱敏的客户个人信息
- 知识条目的创建/修改必须记录变更历史（版本管理）
- 涉及黑产技术细节的条目需限制访问权限（仅反欺诈团队成员）
- 知识库季度更新维护，过期条目不超过总量的30%
