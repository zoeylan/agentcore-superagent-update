# 团伙欺诈网络分析 (Gang Fraud Network Analysis)

## 技能概述

团伙欺诈网络分析是案件调查Agent的高阶分析能力，通过关联图谱技术从单个可疑账户出发，逐步扩展发现整个欺诈团伙的网络结构、资金通道和核心节点。团伙欺诈是金融机构面临的最大规模欺诈威胁，单个团伙的涉案金额通常是个体欺诈的10-100倍。

### 适用场景
- 从单个高风险告警扩展发现关联欺诈团伙
- 骗贷团伙的网络结构和分工识别
- 跑分/洗钱网络的资金通道映射
- 内外勾结欺诈的关联关系发现
- 跨机构欺诈团伙的协同排查

## 执行步骤

### 第一步：种子节点确定
1. 以确认或高度疑似欺诈的账户/设备/手机号作为种子节点
2. 提取种子节点的全维度属性：开户信息、交易历史、设备信息、联系方式、地理位置
3. 确定分析目标：识别团伙成员？追踪资金去向？发现资金来源？

### 第二步：一度关联扩展
1. 基于多维度关联关系向外扩展：
   - **设备关联**：共享同一设备指纹的其他账户
   - **IP关联**：共享同一IP地址（尤其是非公共WiFi的IP）的其他账户
   - **手机号关联**：共享同一手机号或手机号段的账户
   - **资金关联**：与种子节点有直接资金往来的对手方账户
   - **地址关联**：注册地址/联系地址高度相似的账户
   - **行为关联**：交易模式高度相似（时间、频率、金额模式）的账户
2. 对每个一度关联节点计算关联强度得分（多维度关联越多分越高）
3. 过滤低强度关联（得分<阈值），避免噪声

### 第三步：多跳扩展与网络构建
1. 对关联强度得分超过阈值的一度节点，重复执行关联扩展（最多3-4跳）
2. 在扩展过程中构建关联图谱：
   - 节点：账户/设备/IP/手机号/地址
   - 边：关联类型及强度
3. 应用社区发现算法（如Louvain、Label Propagation）识别紧密连接的子图（疑似团伙）
4. 设置扩展边界条件：单次分析涉及节点不超过500个，避免无限扩展

### 第四步：网络角色识别
1. 基于图分析指标识别关键角色：
   - **核心组织者**：高中心度（Betweenness Centrality）节点，控制资金流向
   - **资金通道**：高度数（Degree）节点，连接多个团伙成员
   - **末端执行者**：低度数边缘节点，通常是被招募的"人头户"
   - **资金出口**：资金最终汇集的账户，通常关联提现/跨境转账
2. 绘制团伙组织结构和资金流转路径图
3. 评估团伙规模、活跃时间窗口、涉及总金额

### 第五步：分析报告输出
1. 输出团伙欺诈分析报告：团伙结构图、核心成员列表、资金链路图、时间线、涉案金额汇总
2. 标注尚未确认但高度可疑的节点，建议后续监控
3. 提炼团伙欺诈模式特征，输出给规则策略Agent生成检测规则

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| seedNodes | array | 是 | 种子节点列表（账户ID/设备ID） |
| analysisDepth | integer | 否 | 扩展跳数（默认3） |
| associationThreshold | float | 否 | 关联强度阈值（默认0.6） |
| maxNodes | integer | 否 | 最大分析节点数（默认500） |
| timeWindow | object | 否 | 分析时间窗口 |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| networkId | string | 网络分析编号 |
| totalNodes | integer | 发现的总节点数 |
| communities | array | 识别的社区/团伙列表 |
| coreMembers | array | 核心成员列表及角色标注 |
| fundFlowMap | object | 资金流转路径 |
| totalInvolvedAmount | decimal | 涉案总金额 |
| patternSignature | object | 团伙欺诈模式特征签名 |
| recommendedActions | array | 建议处置动作 |

## 最佳实践
- 使用图数据库（如Neo4j、TigerGraph）存储和查询关联关系，支持高效的多跳查询
- 关联扩展时优先使用强关联维度（设备指纹>资金往来>IP地址>手机号>地址相似度）
- 对大型网络分析（节点>200），采用分批分析策略，先找核心结构再扩展边缘
- 分析结果可视化输出（力导向图），方便人工复核和汇报

## 约束条件
- 单次分析的节点上限为500，超过需分批或人工确认
- 关联扩展涉及的数据查询必须在授权范围内
- 团伙欺诈的定性结论需要至少3个独立关联维度的证据支撑
- 涉及跨机构数据的关联分析需通过合规审批
