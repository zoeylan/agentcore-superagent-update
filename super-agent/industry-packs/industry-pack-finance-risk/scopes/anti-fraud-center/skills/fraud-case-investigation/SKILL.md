# 欺诈案件调查 (Fraud Case Investigation)

## 技能概述

欺诈案件调查是案件调查Agent的核心能力，涵盖从接收告警到出具定性结论的完整调查流程。通过系统化的信息汇集、多维度交叉分析和证据链构建，实现对可疑交易的深度调查和欺诈定性。

### 适用场景
- 实时监控系统推送的高风险拦截告警调查
- 中风险交易的待复核案件批量处理
- 客户报案的盗刷/账户盗用事件调查
- 同业通报的跨机构欺诈线索排查
- 公安协查函涉及的账户和交易调查

## 执行步骤

### 第一步：案件受理与分级
1. 接收告警/线索，自动解析来源类型和优先级：
   - **紧急案件**（P0）：高风险拦截告警（评分>80）、客户实时报案（盗刷进行中）、公安紧急协查——30分钟内启动调查
   - **高优先级**（P1）：中风险复核案件、同业欺诈通报关联排查——4小时内启动
   - **常规案件**（P2）：历史可疑交易回溯、内部审计线索——24小时内启动
2. 创建案件工单，分配案件编号，记录受理时间

### 第二步：信息汇集
1. **交易数据拉取**：涉案交易的完整流水（含前后72小时的关联交易）、资金流向追踪（最多追溯3层转账链路）
2. **账户信息查询**：涉案账户的开户信息、KYC资料、历史交易模式基线、账户状态变更记录
3. **设备与网络数据**：涉案交易的设备指纹、IP地址及归属地、User-Agent信息、登录/操作时间线
4. **外部数据补充**：征信记录异常检查、司法诉讼信息、工商信息（企业账户）、黑名单/灰名单命中状态
5. 信息汇集超时控制：P0案件30分钟内完成，P1案件2小时内完成

### 第三步：多维度分析
1. **交易行为分析**：
   - 与账户历史交易模式的偏离度分析（金额、频率、时间段、对手方）
   - 资金流向分析——识别分层转账（Layering）、资金归集（Collection）等典型洗钱/欺诈模式
2. **时间线重建**：
   - 构建涉案事件的完整时间线：设备登录→密码修改→交易发起→资金到账→资金转出
   - 识别时间线中的异常节点（如凌晨操作、跨时区操作、极短时间内完成多步操作）
3. **身份核验**：
   - 比对交易发起人的设备/IP/地理位置与账户持有人的常用模式
   - 检查是否存在身份冒用迹象（新设备+新IP+大额操作的组合）
4. **模式匹配**：
   - 将案件特征与已知欺诈模式库进行匹配（如电信诈骗模式、养卡套现模式、跑分模式等）
   - 识别当前案件的欺诈手法分类

### 第四步：定性结论
1. 基于分析证据，给出定性结论：
   - **确认欺诈**（置信度>90%）：证据链完整，欺诈手法明确
   - **疑似欺诈**（置信度60-90%）：存在较强欺诈特征，但证据链不完整
   - **误报**（置信度<60%）：可合理解释的正常交易行为
2. 撰写调查结论摘要，包含欺诈定性、涉及金额、受害方、作案手法简述

### 第五步：处置建议与执行
1. **确认欺诈**：触发账户冻结、黑名单更新（1小时内生效）、资金拦截（如资金尚未转出）、通知受害客户
2. **疑似欺诈**：将账户加入灰名单（增强监控）、标记持续关注、扩展排查关联账户
3. **误报**：解除拦截/监控标记、更新告警至已关闭、记录误报原因供规则优化

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alertId | string | 是 | 告警/线索唯一标识 |
| alertSource | enum | 是 | REALTIME_MONITOR/CUSTOMER_REPORT/PEER_NOTIFICATION/POLICE_REQUEST |
| transactionIds | array | 是 | 涉案交易ID列表 |
| accountIds | array | 是 | 涉案账户ID列表 |
| priority | enum | 否 | P0/P1/P2（可根据规则自动判定） |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| caseId | string | 案件编号 |
| conclusion | enum | CONFIRMED_FRAUD / SUSPECTED_FRAUD / FALSE_POSITIVE |
| confidenceScore | integer (0-100) | 定性置信度 |
| fraudType | enum | 欺诈类型分类 |
| involvedAmount | decimal | 涉案总金额 |
| investigationReport | object | 完整调查报告 |
| dispositionActions | array | 已执行的处置动作列表 |
| ruleOptimizationNotes | string | 规则优化建议（误报案件必填） |

## 最佳实践
- 建立标准化的调查检查清单（Checklist），确保每个维度不遗漏
- 对于团伙欺诈线索，优先固定已发现的证据，再逐步扩展关联分析
- 调查过程中的所有数据查询和分析操作必须留痕，形成完整的调查轨迹
- 高价值案件（涉案金额>100万元）建议双人复核

## 约束条件
- P0案件必须30分钟内启动，48小时内出具调查报告
- 调查过程中不得直接联系客户，需通过业务部门客服渠道
- 定性结论必须基于多维度证据，不得仅凭单一数据源定性
- 所有调查数据的访问必须在权限控制范围内，符合数据最小化原则
