# 模型性能监控 (Model Performance Monitoring)

## 技能概述

模型性能监控是规则策略Agent管理ML模型生命周期的关键能力。反欺诈ML模型在生产环境中会因数据分布漂移（Data Drift）、概念漂移（Concept Drift）和黑产手法迭代而逐步退化，持续监控和及时干预是保持模型有效性的核心手段。

### 适用场景
- 周度模型性能指标监控与趋势分析
- 模型特征漂移检测（PSI监控）
- 模型退化告警与重训触发评估
- 新模型上线前的回测和A/B测试评估
- 模型版本管理和切换决策

## 执行步骤

### 第一步：性能指标计算
1. 以周为周期，基于已标注数据（调查定性结果回标）计算模型核心指标：
   - **AUC**（Area Under ROC Curve）：模型区分欺诈/非欺诈的整体能力，行业基准>0.90
   - **KS值**（Kolmogorov-Smirnov）：模型区分度，行业基准>0.35
   - **Precision@k**：在模型评分Top-k%的交易中，实际欺诈的占比
   - **Recall@FPR=5%**：在误报率控制在5%时的欺诈检出率
   - **F1-Score**：Precision和Recall的调和平均
2. 按交易渠道、交易类型、客户群体等维度分别计算，识别模型在子群体上的性能差异

### 第二步：漂移检测
1. **特征漂移检测**：
   - 计算每个模型输入特征的PSI（Population Stability Index）
   - PSI < 0.1 → 稳定，0.1-0.25 → 轻微漂移需关注，> 0.25 → 显著漂移需干预
   - 识别漂移最严重的Top5特征，分析漂移原因
2. **预测分布漂移检测**：
   - 对比近期模型评分分布与训练期评分分布
   - 检查评分分布的均值、方差、分位数变化
3. **标签漂移检测**：
   - 对比近期欺诈率与训练期欺诈率
   - 识别是否出现新的欺诈模式导致标签分布变化

### 第三步：告警与评估
1. 触发告警条件：
   - AUC周度下降 >0.02 → 橙色告警（性能退化）
   - AUC下降 >0.05 或 AUC绝对值 <0.85 → 红色告警（模型失效风险）
   - 多个特征PSI >0.25 → 橙色告警（数据环境变化）
   - Recall@FPR=5% 下降 >5个百分点 → 红色告警
2. 告警触发后，评估是否需要模型重训：
   - 短期波动（仅最近1周异常）→ 继续观察
   - 持续退化（连续2周以上）→ 启动重训评估
   - 突然崩塌（单周剧烈下降）→ 紧急排查（可能是数据质量问题或重大欺诈模式变化）

### 第四步：重训触发与管理
1. 确认需要重训后，编制模型重训需求：
   - 重训原因分析、建议的训练数据时间窗口、特征工程调整建议
   - 新增的已标注样本（近期案件调查的定性结果）
2. 提交至数据科学团队执行模型训练
3. 新模型就绪后，执行上线前验证：
   - 离线回测：在保留测试集上的指标必须优于当前在线模型
   - A/B测试：5%流量灰度，比较新旧模型的实际检出效果
   - 灰度期≥1周，确认无性能退化后全量切换

### 第五步：报告输出
1. 生成周度模型监控报告：
   - 各项性能指标及周环比变化
   - 特征漂移热力图（各特征PSI值）
   - 模型评分分布变化趋势
   - 活跃告警及处理状态
   - 建议行动项
2. 生成月度/季度模型回顾报告：
   - 模型性能长期趋势
   - 本周期内的重训和切换记录
   - 与业务指标（欺诈损失率、误报率）的关联分析

## 输入/输出规范

### 输入
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| modelId | string | 是 | 模型唯一标识 |
| evaluationPeriod | object | 是 | 评估时间窗口（起止日期） |
| labeledData | object | 是 | 已标注样本数据源 |
| baselineMetrics | object | 否 | 基线指标（默认使用模型上线时的指标） |

### 输出
| 字段 | 类型 | 说明 |
|------|------|------|
| modelId | string | 模型标识 |
| currentMetrics | object | 当前周期各项性能指标 |
| metricsTrend | array | 历史指标趋势（近12周） |
| driftAnalysis | object | 漂移检测结果 |
| alerts | array | 触发的告警列表 |
| retrainRecommendation | object | 重训建议（如有） |
| monitoringReport | object | 完整监控报告 |

## 最佳实践
- 标注数据的回标时效直接影响监控的时效性，建议案件调查结果在定性后24小时内完成回标
- 维护"Shadow Model"（影子模型）机制，新训练的模型在正式切换前以shadow模式运行1周
- 关注模型在不同子群体上的公平性指标，避免对特定群体的系统性偏差
- AUC并非唯一指标——业务场景更关注在低FPR下的Recall（即在可接受的误报水平下能抓住多少欺诈）

## 约束条件
- 模型重训周期不超过季度（即使指标稳定也需要定期更新，吸收新样本）
- 新模型上线必须通过回测+A/B测试双重验证
- 模型切换必须支持秒级回滚（保留上一版本模型的在线服务）
- 所有模型版本及其评估数据必须归档，满足监管的模型治理要求
