# 投后风险监控 (Post-Investment Monitoring)

## 概述

本技能负责对客户已持有的投资组合进行持续的风险监控和绩效跟踪，通过定期报告和事件驱动预警相结合的方式，及时发现组合风险暴露变化并触发相应的调整建议。覆盖市场风险、流动性风险、集中度风险和偏离度风险四大监控维度。

## 适用场景

- 月度/季度投后风险报告定期生成
- 市场大幅波动时的紧急风险评估
- 组合偏离度超阈值的再平衡触发
- 产品到期/开放日的流动性时间线管理
- 基金净值预警（清盘线、止损线监控）

## 执行步骤

### 步骤1：持仓数据同步
1. 获取客户最新持仓数据（产品、份额、成本、市值）
2. 获取各产品最新净值/价格
3. 计算组合总市值和各产品占比
4. 对比目标配置比例，计算偏离度

### 步骤2：绩效分析
1. 计算组合绝对收益（本期收益率、累计收益率、年化收益率）
2. 计算相对收益（vs基准指数的超额收益/Alpha）
3. 逐产品贡献度分析（每只产品对组合收益的贡献）
4. 与同类组合的排名对比

### 步骤3：风险指标计算
1. **市场风险**：
   - 组合VaR（95%/99%，1日/10日）
   - 组合CVaR（条件在险价值）
   - 压力测试（历史情景：股灾/债灾/流动性危机）
   - Beta敞口分析
2. **流动性风险**：
   - 各产品到期/开放时间线
   - T+N变现能力分析（1天/7天/30天内可变现比例）
   - 赎回压力监控（关注产品近期赎回量是否异常放大）
3. **集中度风险**：
   - 单一产品集中度（占比>30%触发预警）
   - 单一管理人集中度
   - 单一行业/板块集中度
   - 单一资产类别偏离度
4. **偏离度风险**：
   - 各资产类别实际占比vs目标占比的偏离
   - 综合偏离度计算（加权平方根）

### 步骤4：预警判定与触发
1. 检查是否触发预警阈值：
   - 组合偏离度>10% → 触发再平衡建议
   - 单一资产集中度>30% → 触发集中度预警
   - 产品净值跌破预警线 → 触发止损预警
   - 产品净值触及清盘线 → 触发清盘预警
   - 流动性覆盖率<100% → 触发流动性预警
2. 对触发的预警确定优先级和处置建议

### 步骤5：报告生成与通知
1. 生成标准化月度风险报告
2. 对触发预警的情况生成专项告警通知
3. 再平衡建议须在触发后48小时内通知客户
4. 重大风险预警实时通知（不等月报）

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| customer_id | String | 是 | 客户标识 |
| holdings | Array | 是 | 当前持仓明细 |
| target_allocation | Object | 是 | 目标配置比例 |
| market_data | Object | 是 | 最新市场数据 |
| alert_thresholds | Object | 否 | 自定义预警阈值 |
| report_type | Enum | 否 | monthly/quarterly/adhoc，默认monthly |

## 最佳实践

1. **分层监控频率**：高风险组合日度监控，中风险周度，低风险月度
2. **预警分级**：区分信息性提示、注意事项、需要行动三个级别，避免预警疲劳
3. **市场联动**：在重大市场事件时主动进行临时评估，不等定期报告
4. **可视化呈现**：对客户端的报告应图文并茂，降低理解门槛

## 约束条件

- 月度风险报告必须在T+5个工作日内产出
- 再平衡建议触发后48小时内通知客户
- 清盘预警必须实时通知（不超过30分钟）
- 报告数据准确性≥99.5%
- 所有风险指标计算方法论必须符合行业标准（如VaR采用历史模拟法或蒙特卡洛法）
