# 理赔反欺诈检测 (Claims Fraud Detection)

## 概述

本技能负责在保险理赔审核流程中识别潜在的欺诈行为，通过规则引擎和机器学习模型的协同检测，覆盖虚假事故、夸大损失、带病投保、虚假住院、重复理赔等常见保险欺诈模式。行业数据显示理赔欺诈占比约5-10%，有效的欺诈检测可显著降低赔付损失。

## 适用场景

- 所有理赔案件的常规反欺诈筛查
- 高金额理赔的强化欺诈检测
- 短期内多次理赔的关联分析
- 带病投保（等待期内理赔）的核查
- 团伙欺诈的模式识别和关联扩展

## 执行步骤

### 步骤1：规则引擎筛查
1. 执行基础欺诈规则检测：
   - **时间异常**：投保后短期内（30天内）即理赔、等待期结束立即理赔
   - **频率异常**：同一客户12个月内理赔次数超过3次
   - **金额异常**：单次理赔金额接近保额上限（>80%）
   - **关联异常**：同一事故关联多份保单、同一医院/修理厂高频出现
   - **信息不一致**：理赔描述与医疗记录不符、事故描述与定损不符
2. 对每条命中规则记录规则ID和命中详情

### 步骤2：模型评分
1. 输入案件特征向量至欺诈检测模型
2. 特征包括：客户画像特征（投保历史、理赔历史、信用评分）、案件特征（金额、时间距投保间隔、理赔类型）、行为特征（报案方式、材料提交速度、沟通配合度）
3. 输出欺诈概率评分（0-100）

### 步骤3：关联图谱分析
1. 对嫌疑案件进行关联扩展：
   - 人员关联：同一投保人/被保人/受益人的其他保单和理赔记录
   - 事件关联：同一事故/同一医疗机构/同一代理人的案件聚集
   - 设备关联：同一手机/IP提交的多份理赔申请
2. 识别团伙欺诈模式（多人串通、医患勾结等）

### 步骤4：欺诈分级与处置
1. **低风险（评分<30，无规则命中）**：放行，正常理赔流程
2. **中风险（评分30-60，或命中非核心规则）**：标注风险点，在理赔报告中附注，流程正常进行但增加人工关注
3. **高风险（评分>60，或命中核心欺诈规则）**：暂停理赔流程，生成欺诈线索报告，转人工专项调查
4. **确认欺诈（评分>85+多规则命中+关联证据充分）**：暂停理赔+冻结关联保单+推送反欺诈中心

### 步骤5：报告输出
1. 生成反欺诈检测报告，包含：命中规则明细、模型评分及分值解释、关联分析结果、风险等级判定、处置建议
2. 对高风险案件输出详细的欺诈线索描述，供人工调查使用

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| claim_id | String | 是 | 理赔案件编号 |
| claim_details | Object | 是 | 理赔案件详情（类型、金额、时间、描述） |
| policy_history | Object | 是 | 客户投保和理赔历史 |
| material_verification | Object | 是 | 材料核验结果 |
| external_data | Object | 否 | 外部数据（医院记录、征信等） |

## 最佳实践

1. **Precision优先**：欺诈检测模型应优先保证精确率（Precision≥85%），避免对正常客户造成不良体验
2. **动态阈值**：不同险种、不同金额段的欺诈检测阈值应有所区别
3. **反馈闭环**：每个调查结论（确认欺诈/误报）必须回馈模型训练，持续提升检测能力
4. **合规调查**：欺诈调查过程必须合法合规，不得侵犯客户隐私权

## 约束条件

- 反欺诈模型Precision≥85%（月度评估）
- 检测结果不得作为唯一拒赔依据，高风险标记需人工确认后方可拒赔
- 所有欺诈标记必须附具体证据和理由，不得仅凭模型评分定性
- 欺诈检测不得对特定人群（年龄、性别、地域）产生歧视性判定
