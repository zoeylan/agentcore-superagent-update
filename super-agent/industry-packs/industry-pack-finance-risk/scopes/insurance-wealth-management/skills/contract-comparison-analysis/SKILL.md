# 合同条款对比分析 (Contract Comparison Analysis)

## 概述

本技能负责对同类保险产品或财富管理产品的合同条款进行系统化对比分析，帮助识别不同产品间的条款差异、优劣势和潜在风险点。支持同公司不同版本的条款变更追踪，以及不同公司同类产品的竞品条款对比。

## 适用场景

- 同类保险产品不同公司的条款竞品分析
- 产品条款升级前后的变更影响评估
- 理赔争议时对条款歧义的多角度分析
- 监管政策变化后的条款合规性批量审查
- 客户选品时的条款差异说明

## 执行步骤

### 步骤1：对比维度确定
1. 根据产品类型确定核心对比维度
   - 保险产品：保险责任范围、免责条款数量与范围、等待期、赔付比例/限额、免赔额、续保条件
   - 基金产品：投资范围、费率结构、流动性安排、风险等级、清盘条件
2. 确定对比基准（行业标准/监管最低要求/特定竞品）

### 步骤2：条款对齐
1. 将待对比产品的条款按统一分类体系进行对齐
2. 识别各产品独有的条款（A有B无的情况）
3. 对相同类别下的条款进行语义匹配（即使表述不同但含义相同的条款）

### 步骤3：差异分析
1. 对每个对比维度生成差异矩阵
2. 对差异进行影响评级：重大差异（影响赔付/收益）/ 一般差异（影响体验）/ 微小差异（仅表述不同）
3. 对重大差异点进行详细解读说明

### 步骤4：输出对比报告
1. 生成结构化的对比矩阵表格
2. 对各产品的优劣势进行客观总结
3. 标注需要关注的风险点（如某产品免责条款明显多于同类）
4. 如发现与监管要求不符的条款，生成合规风险提示

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| products | Array | 是 | 待对比产品列表（2-5个产品的结构化条款数据） |
| comparison_type | Enum | 是 | cross_company（跨公司）/ version_change（版本变更）/ compliance_check（合规审查） |
| focus_dimensions | Array | 否 | 重点关注的对比维度 |
| benchmark | String | 否 | 对比基准标识 |

## 最佳实践

1. **语义对比优于字面对比**：不同公司的条款用语可能不同但含义相同，需做语义层面的对齐
2. **关注隐藏条款**：特别注意附件、特别约定、批注中的限制性条款，这些往往被忽视
3. **动态对比基准**：对比基准应随监管政策更新而调整
4. **量化差异**：尽可能将定性差异量化（如理赔成功率估算、费用差异金额化）

## 约束条件

- 对比分析必须客观中立，不得带有产品推荐倾向
- 分析结论必须有条款原文支撑，不得主观推断
- 涉及监管合规判断的结论需标注不确定性
- 对比报告不构成投资建议或产品推荐
