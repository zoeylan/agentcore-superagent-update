# 客户画像构建 (Customer Profiling)

## 概述

本技能负责构建财富管理客户的360度画像，整合多维度数据源生成包含风险偏好、投资能力、流动性需求和适当性分类的综合画像，为个性化资产配置建议和合规适当性匹配提供数据基础。

## 适用场景

- 新客户开户时的初始画像建立
- 客户定期风险测评更新
- 投资行为变化后的画像动态调整
- 适当性合规检查时的客户分类确认
- 大额交易前的客户资格审核

## 执行步骤

### 步骤1：基础信息整合
1. 获取客户身份信息：年龄、职业、学历、家庭状况
2. 获取财务数据：年收入（近三年平均）、总资产规模、负债情况、房产/车辆等固定资产
3. 获取金融资产明细：银行存款、理财产品、基金持仓、股票持仓、保险保单
4. 验证合格投资者资格（私募产品场景）：个人金融资产≥300万或年均收入≥50万

### 步骤2：风险偏好评估
1. 整合风险测评问卷结果（KYC问卷）
2. 分析历史投资行为偏好：
   - 持仓品种的风险分布（低风险/中风险/高风险占比）
   - 持仓周期分析（平均持有时间）
   - 止损行为分析（亏损容忍度）
   - 交易频率和集中度
3. 综合评定客户风险等级：C1（保守型）/C2（稳健型）/C3（平衡型）/C4（积极型）/C5（激进型）
4. 评估客户声明风险偏好与实际行为的一致性，标注矛盾点

### 步骤3：投资能力分析
1. 计算可投资资产规模（总资产-负债-生活必需资金-应急储备）
2. 分析收入稳定性（收入来源多元化程度、职业稳定性）
3. 评估流动性需求（近期大额支出计划、家庭生命周期阶段）
4. 确定适合的投资期限（短期<1年/中期1-3年/长期>3年）

### 步骤4：适当性分类
1. 根据《证券期货投资者适当性管理办法》执行适当性分类
2. 确定客户可匹配的产品风险等级上限：
   - C1：仅匹配R1产品
   - C2：匹配R1-R2产品
   - C3：匹配R1-R3产品
   - C4：匹配R1-R4产品
   - C5：匹配R1-R5产品
3. 确认特殊资格：合格投资者认定（私募）、专业投资者认定
4. 记录适当性分类结果和依据，备查

### 步骤5：画像输出与更新策略
1. 生成完整的客户画像报告
2. 设定画像更新触发条件：
   - 定期更新：风险测评有效期满（通常1年）
   - 事件触发：大额资产变动、职业变化、家庭变故
   - 行为触发：投资行为明显偏离画像预期
3. 标注画像中的不确定性和待补充信息

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| customer_id | String | 是 | 客户唯一标识 |
| basic_info | Object | 是 | 基本身份信息 |
| financial_data | Object | 是 | 财务和资产数据 |
| risk_questionnaire | Object | 是 | 风险测评问卷结果 |
| investment_history | Array | 否 | 历史投资记录 |
| current_holdings | Array | 否 | 当前持仓数据 |

## 最佳实践

1. **行为与声明对照**：客户声称的风险偏好可能与实际行为不一致，应以更保守的判定为准
2. **动态画像**：客户画像不是静态标签，应随客户生命周期和市场环境动态调整
3. **数据时效性**：超过1年未更新的财务数据应标记为"待更新"，降低画像置信度
4. **隐私保护**：客户数据的使用必须在授权范围内，不得超范围使用

## 约束条件

- 风险测评覆盖率100%（无测评客户不可购买R3以上产品）
- 画像数据每年至少更新一次（风险测评有效期1年）
- 适当性分类必须严格按照监管规定执行，不得人为降低客户风险等级
- 客户画像数据不得用于未经授权的营销或共享
