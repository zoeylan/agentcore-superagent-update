# 赎回份额与金额计算 (Share Calculation)

## 概述

本技能负责在赎回合规校验通过后，根据基金最新确认净值、适用的赎回费率和持有期限，精确计算客户应收的赎回金额。计算必须在T+3个工作日内完成，准确率要求100%（精确到分），是赎回流程中的核心业务计算环节。

## 适用场景

- 正常赎回申请的金额计算
- 巨额赎回中按比例确认部分的金额计算
- 基金清盘时的份额清算计算
- 部分赎回后剩余份额和金额的确认
- 赎回费率的阶梯计算（按持有期限）

## 执行步骤

### 步骤1：确认计算基准
1. 获取适用的基金净值（通常为赎回申请确认日的净值）
2. 确认净值来源和发布时间（须为托管人确认的官方净值）
3. 获取客户申请赎回的份额数量
4. 对巨额赎回场景，获取确认比例后计算实际确认份额

### 步骤2：赎回费率确定
1. 获取基金合同约定的赎回费率结构（通常按持有期限阶梯递减）：
   - 持有<7天：1.5%（惩罚性费率）
   - 持有7天-1年：0.5%
   - 持有1年-2年：0.25%
   - 持有≥2年：0%
2. 计算客户的实际持有天数（从认购/申购确认日至赎回确认日）
3. 确定适用的费率档位
4. 处理多次申购份额的费率计算（先进先出法FIFO）

### 步骤3：赎回金额计算
1. 基本公式：
   ```
   赎回总金额 = 赎回份额 × 赎回确认日基金净值
   赎回费用 = 赎回总金额 × 适用费率
   客户应收金额 = 赎回总金额 - 赎回费用
   ```
2. 对多笔申购份额分别计算费率后汇总：
   ```
   应收金额 = Σ(各笔份额 × 净值 × (1 - 各笔适用费率))
   ```
3. 精确到分（保留两位小数），采用银行家舍入法

### 步骤4：特殊场景处理
1. **巨额赎回比例确认**：
   ```
   确认份额 = 申请份额 × 确认比例
   延期份额 = 申请份额 - 确认份额
   ```
2. **基金清盘清算**：
   ```
   清算金额 = 持有份额 / 基金总份额 × 基金清算净资产
   ```
3. **强制赎回（触发止损/爆仓）**：按触发时点净值计算
4. **红利再投资份额**：需区分本金份额和红利份额分别计算费率

### 步骤5：计算结果验证
1. 校验计算结果的合理性（应收金额>0、费用≤总金额、金额不超过基金可用现金）
2. 与客户认购成本进行盈亏对比（信息性，非决策性）
3. 生成详细的计算明细单（含每步计算过程）
4. 对大额赎回（>500万）计算结果标记为需人工复核

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| redemption_id | String | 是 | 赎回申请编号 |
| confirmed_shares | Number | 是 | 确认赎回份额 |
| nav_date | Date | 是 | 适用净值日期 |
| nav_value | Number | 是 | 确认日基金净值（精确到4位小数） |
| fee_schedule | Object | 是 | 赎回费率结构 |
| purchase_records | Array | 是 | 客户历次认购/申购记录（用于FIFO费率计算） |
| special_scenario | Enum | 否 | normal/massive/liquidation/forced |

## 最佳实践

1. **净值确认时效**：确保使用的净值为托管人正式确认版本，避免使用估算净值
2. **FIFO严格执行**：多次申购份额的费率计算必须严格按先进先出原则
3. **异常净值处理**：如遇净值估值调整（如估值错误更正），需重新计算并通知客户
4. **并发控制**：同一账户同时有多笔赎回时，注意份额扣减的并发一致性

## 约束条件

- 份额计算必须T+3完成，准确率100%
- 金额精确到分（0.01元），采用银行家舍入法
- 大额赎回（>500万）计算结果须人工复核确认
- 计算明细必须完整留存供审计和客户查询
- 净值必须为托管人确认的正式净值
