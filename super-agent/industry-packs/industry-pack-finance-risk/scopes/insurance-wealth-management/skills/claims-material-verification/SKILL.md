# 理赔材料核验 (Claims Material Verification)

## 概述

本技能负责对保险理赔申请中提交的各类证明材料进行真实性、完整性和一致性核验。通过OCR识别、交叉验证、合理性校验等技术手段，高效识别材料造假、信息不一致和材料缺失等问题，是理赔审核流程的关键质量控制环节。

## 适用场景

- 医疗险理赔的医疗发票、诊断证明、出院小结核验
- 车险理赔的事故认定书、维修发票、现场照片核验
- 意外险理赔的事故证明、伤残鉴定材料核验
- 重疾险理赔的病理报告、确诊证明核验
- 财产险理赔的损失评估报告、维修清单核验

## 执行步骤

### 步骤1：材料完整性检查
1. 根据理赔类型（医疗/意外/重疾/车险/财产）获取必须材料清单
2. 逐项核对客户提交材料的完整性
3. 对缺失材料生成精确的补件通知（明确缺失项目和具体要求）
4. 材料不完整时暂停审核时钟，避免影响法定理赔时限

### 步骤2：OCR信息提取
1. 对纸质材料的扫描件/照片执行OCR识别
2. 提取关键信息字段：
   - 医疗发票：医院名称、发票号码、金额、项目明细、就诊日期
   - 事故证明：事故时间、地点、当事人、责任认定
   - 诊断证明：诊断结论、医生签名、医院公章、日期
3. OCR识别置信度<90%的字段标注为待人工确认

### 步骤3：真实性验证
1. **医疗发票验真**：与医院HIS系统或医保系统进行交叉验证（发票号、金额、就诊记录匹配）
2. **事故证明验真**：校验出具机关的公章真实性、证明文号格式合规性
3. **PS检测**：对图片类材料进行图像篡改检测（元数据分析、像素异常检测）
4. **时间线逻辑验证**：校验各材料的时间逻辑一致性（如出院日期不应早于入院日期）

### 步骤4：合理性校验
1. **医疗费用合理性**：对比同地区同级别医院的平均费用水平，标注异常高额项目
2. **维修费用合理性**：对比市场价格库，标注超出合理范围的维修项目
3. **治疗方案合理性**：校验诊断与治疗方案的匹配性（如轻微外伤却住院数周）
4. **多次理赔关联分析**：同一客户短期内多次理赔的材料关联性检查

### 步骤5：核验结论输出
1. 对每份材料输出核验状态：通过/存疑/伪造/缺失
2. 对存疑材料标注具体疑点和建议的确认方式
3. 对材料整体完整性和一致性给出综合评价
4. 生成材料核验报告供后续审核决策参考

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| claim_id | String | 是 | 理赔案件编号 |
| claim_type | Enum | 是 | medical/accident/critical_illness/auto/property |
| submitted_materials | Array[File] | 是 | 客户提交的材料文件列表 |
| policy_info | Object | 是 | 关联保单信息（险种、保额、投保日期等） |

## 最佳实践

1. **多源交叉验证**：单一来源验证不足以确定真实性，应尽可能从多个独立来源交叉确认
2. **保护客户隐私**：材料核验过程中接触的医疗信息等敏感数据须严格按照《个人信息保护法》处理
3. **避免过度延迟**：材料核验不应成为理赔流程的瓶颈，标准材料核验应在4小时内完成
4. **存疑不等于否定**：材料存疑时应给客户补充说明的机会，而非直接拒赔

## 约束条件

- 材料核验必须在收到申请后24小时内完成初审
- 材料缺失时必须一次性告知全部缺失项，避免多次补件
- 不得仅因材料形式瑕疵（如照片不清晰）拒赔，应给予补提机会
- 所有核验记录必须完整留存，支持后续审计和客户申诉查证
