# 赎回合规校验 (Redemption Compliance Check)

## 概述

本技能负责对私募基金和非标资管产品的赎回申请进行全面的合规性校验，确保每笔赎回符合基金合同约定的开放日、锁定期、最低持有期限、赎回限额等规则。作为赎回流程的第一道关卡，合规校验结果直接决定赎回申请是被受理还是被拒绝。

## 适用场景

- 客户提交常规赎回申请时的合规预检
- 巨额赎回场景下的额度校验
- 基金暂停赎回期间的申请处理
- 锁定期内客户紧急赎回请求的评估
- 产品清盘时的赎回规则变更处理

## 执行步骤

### 步骤1：基金状态核查
1. 确认基金当前运营状态：正常运作/暂停赎回/清盘中/已终止
2. 如基金处于暂停赎回状态，直接拒绝并告知原因和预计恢复时间
3. 如基金处于清盘中，引导客户进入清算分配流程

### 步骤2：开放日校验
1. 从基金合同获取开放日规则（每月开放/每季开放/半年开放等）
2. 确认申请日是否为有效开放日或开放日前的有效申请窗口期
3. 如不在开放日，告知客户最近的下一个开放日和申请截止时间
4. 处理特殊约定（如首次开放日、追加开放日）

### 步骤3：锁定期与最低持有期限校验
1. 获取客户该产品的认购/申购日期
2. 计算客户实际持有天数
3. 对比基金合同约定的锁定期和最低持有期限
4. 锁定期内的赎回申请原则上拒绝，但需告知客户剩余锁定天数
5. 处理特殊情况（如因不可抗力可豁免锁定期的条款）

### 步骤4：份额与限额校验
1. 确认客户账户可赎回份额（总持有份额-已冻结份额-质押份额）
2. 校验赎回份额是否超出客户可赎回余额
3. 校验赎回后剩余份额是否低于最低持有份额要求
4. 计算本次赎回占基金总份额的比例
5. 判断是否触发巨额赎回（通常为单日赎回>基金总份额10%）

### 步骤5：巨额赎回处理判定
1. 如触发巨额赎回：
   - 计算各申请人的确认比例
   - 确定延期赎回部分的处理方式（按比例延期/全额延期）
   - 生成巨额赎回确认通知，给投资者选择权
2. 如未触发巨额赎回但接近阈值：标记为关注，预警后续申请

### 步骤6：校验结果输出
1. 汇总所有校验项目的结果
2. 对通过的申请：生成受理确认，标注预计到账日期
3. 对拒绝的申请：明确告知每一条不通过的原因和解决建议
4. 对巨额赎回：生成投资者确认函待签署

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| redemption_request | Object | 是 | 赎回申请（客户ID、产品ID、赎回份额/金额） |
| fund_contract_terms | Object | 是 | 基金合同赎回条款（来自合同解析Agent） |
| customer_holding | Object | 是 | 客户持仓信息（份额、认购日期、冻结状态） |
| fund_status | Object | 是 | 基金当前状态（净值、总份额、运营状态） |
| daily_redemption_total | Number | 否 | 当日已受理的赎回总份额 |

## 最佳实践

1. **一次性告知**：如有多项不通过，应一次性告知所有问题，避免客户反复提交
2. **前置提醒**：在开放日前主动提醒有赎回意向的客户做好准备
3. **边界处理**：对锁定期即将结束的客户，明确告知最早可赎回日期
4. **应急预案**：对基金暂停赎回的情况，应有标准化的投资者沟通方案

## 约束条件

- 赎回合规校验必须T+0完成（当日受理当日校验）
- 校验结果必须有明确的合同条款依据
- 巨额赎回触发后必须2小时内启动应急流程
- 拒绝赎回必须告知客户具体原因和最近可赎回时间
