# 流动性监控 (Liquidity Monitoring)

## 概述

本技能负责对私募基金和非标资管产品的流动性状况进行持续监控，通过现金储备比例、赎回压力指标、净值预警等多维度指标，实时评估各产品的流动性风险水平，在流动性危机发生前提前预警并触发应急处置机制。

## 适用场景

- 每工作日流动性日报生成
- 市场剧烈波动时的实时流动性评估
- 基金净值逼近清盘线时的持续监控
- 大额赎回申请后的流动性压力测试
- 年末/季末等赎回高峰期的前瞻性评估

## 执行步骤

### 步骤1：数据采集
1. 获取各产品的最新净值和日变动
2. 获取各产品的现金及现金等价物余额
3. 获取各产品的总资产规模和份额
4. 获取各产品近期赎回申请量（已确认+排队中）
5. 获取市场数据（利率、信用利差、市场波动率）

### 步骤2：流动性指标计算
1. **现金储备比例**：
   ```
   现金储备比例 = (现金 + 现金等价物) / 基金净资产
   ```
   预警线：<10%（黄色预警）、<5%（红色预警）

2. **赎回压力指标**：
   ```
   赎回压力 = (待处理赎回量 + 预估未来7天赎回量) / 可用现金
   ```
   预警线：>0.8（黄色）、>1.0（红色，现金不足以覆盖赎回）

3. **净值预警**：
   ```
   距清盘线距离 = (当前净值 - 清盘线净值) / 清盘线净值 × 100%
   ```
   预警线：<20%（关注）、<10%（黄色预警）、<5%（红色预警）

4. **资产变现能力评估**：
   - 1日可变现比例（T+0/T+1流动性资产占比）
   - 7日可变现比例
   - 30日可变现比例
   - 非流动性资产（锁定期资产、不可转让资产）占比

### 步骤3：压力测试
1. **赎回压力测试**：模拟极端赎回场景
   - 场景A：20%份额同时赎回
   - 场景B：连续5个开放日每日10%赎回
   - 场景C：最大单一投资者全部赎回
2. **市场压力测试**：模拟市场极端情景
   - 利率上行200bp对债券资产的影响
   - 股市下跌30%对权益资产的影响
   - 信用事件（持仓债券违约）的影响
3. 评估各压力场景下是否触发清盘线或暂停赎回

### 步骤4：预警分级与处置
1. **绿色（正常）**：所有指标在健康范围内，常规监控
2. **黄色（关注）**：部分指标接近预警线
   - 通知基金管理人关注
   - 增加监控频率（日度→盘中）
   - 通知资产配置Agent降低该产品推荐权重
3. **红色（预警）**：指标突破预警线
   - 立即通知相关各方（管理人、托管人、资产配置Agent）
   - 评估是否需要暂停赎回
   - 准备投资者沟通方案
4. **黑色（危机）**：触发清盘线或暂停赎回
   - 30分钟内启动清算/暂停程序
   - 全量投资者即时通知
   - 冻结新的配置建议

### 步骤5：日报生成与分发
1. 汇总所有监控产品的流动性状态
2. 标注预警产品和预警原因
3. 输出趋势分析（近7日/30日流动性变化趋势）
4. 每工作日9:00前完成产出并分发给相关方

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| product_list | Array | 是 | 监控产品列表 |
| fund_data | Object | 是 | 各产品的净值、规模、现金数据 |
| redemption_queue | Array | 是 | 各产品待处理赎回队列 |
| market_data | Object | 是 | 市场环境数据 |
| liquidation_thresholds | Object | 是 | 各产品清盘线设置 |

## 最佳实践

1. **前瞻性监控**：不要等到指标突破才预警，关注趋势恶化的速度
2. **联动通知**：流动性预警必须同步通知资产配置Agent，避免继续向客户推荐该产品
3. **季节性考虑**：年末、季末通常是赎回高峰，应提前增加监控力度
4. **管理人沟通**：对黄色预警产品主动与基金管理人沟通其流动性管理计划

## 约束条件

- 流动性日报必须每工作日9:00前产出
- 红色预警必须实时触发（不超过30分钟延迟）
- 清盘触发后30分钟内启动清算程序
- 压力测试至少每周执行一次（市场波动期间每日）
- 预警信号必须同步给资产配置Agent和相关投资者
