# 置信度评估 (Confidence Assessment)

## 概述

本技能负责对AI自动提取的合同条款信息进行可靠性评估，通过多维度的置信度评分机制，精确识别需要人工确认的低置信度字段，确保AI决策的可靠性并合理控制人工介入范围。

## 适用场景

- 合同条款自动提取后的质量评估
- 理赔审核中自动决策的可信度评估
- 赔付金额计算结果的准确性评估
- OCR识别结果的可靠性判定
- 任何AI输出需要决定是否需人工复核的场景

## 执行步骤

### 步骤1：多维度置信度计算
1. **文本清晰度维度**（权重20%）：OCR识别质量、文档扫描清晰度、文字完整性
2. **语义确定性维度**（权重30%）：条款表述是否明确无歧义、是否存在多种合理解读
3. **模板匹配维度**（权重25%）：提取结果与已有模板的匹配程度、是否为标准化条款
4. **逻辑一致性维度**（权重15%）：提取结果内部逻辑是否自洽、与其他已知信息是否一致
5. **数值合理性维度**（权重10%）：数值类字段是否在行业合理范围内

### 步骤2：阈值判定
1. 综合置信度≥95%：标记为"已确认"，可直接用于下游决策
2. 综合置信度85%-95%：标记为"高置信-建议确认"，非关键字段可放行，关键字段需确认
3. 综合置信度70%-85%：标记为"中置信-需要确认"，必须人工确认后方可使用
4. 综合置信度<70%：标记为"低置信-强制复核"，可能存在识别错误

### 步骤3：低置信原因分析
1. 对每个低置信度字段输出具体原因标签
   - `OCR_UNCLEAR`：原文图像模糊导致识别不确定
   - `AMBIGUOUS_WORDING`：条款表述存在歧义
   - `NON_STANDARD_CLAUSE`：非标准条款，无模板参考
   - `LOGICAL_CONFLICT`：与其他条款存在逻辑冲突
   - `VALUE_OUTLIER`：数值超出行业正常范围
   - `CROSS_REFERENCE_MISSING`：引用条款未找到
2. 对每个原因提供具体说明和建议的确认方式

### 步骤4：人工确认任务生成
1. 将需要人工确认的字段汇总为确认任务清单
2. 按优先级排序（影响赔付决策的字段优先）
3. 提供原文截图/定位和AI提取结果供对照
4. 设定确认时效（紧急案件2小时/常规4小时/批量24小时）

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| extracted_data | Object | 是 | AI提取的结构化数据 |
| source_document | File | 是 | 原始文档用于对照 |
| context_type | Enum | 是 | contract_extraction/claims_review/payout_calc |
| urgency_level | Enum | 否 | urgent/normal/batch，默认normal |

## 输出规格

```json
{
  "assessment_id": "CA-2024-001",
  "overall_confidence": 0.93,
  "field_assessments": [
    {
      "field_path": "clauses.coverage.trigger_condition",
      "confidence": 0.72,
      "status": "needs_review",
      "reasons": ["AMBIGUOUS_WORDING"],

      "review_priority": "high"
    }

  "review_tasks": [],

  "needs_review_count": 3

```

## 最佳实践

1. **宁高勿低**：在不确定时倾向于降低置信度，宁可多一次人工确认也不允许错误信息流入下游
2. **上下文敏感**：同一置信度在不同场景下的处理策略应不同（如涉及赔付金额的字段应更保守）
3. **持续校准**：定期用人工确认结果反馈校准置信度模型，确保评分准确性
4. **批量效率**：对同类型低置信字段可批量提交确认，减少人工操作次数

## 约束条件

- 置信度<95%的字段必须100%标记为待人工确认
- 涉及赔付金额计算的字段，置信度阈值提高至98%
- 人工确认结果必须回写更新置信度模型
- 置信度评估本身不得修改原始提取数据
