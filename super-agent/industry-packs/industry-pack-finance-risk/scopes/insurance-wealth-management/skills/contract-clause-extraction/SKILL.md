# 合同条款智能提取 (Contract Clause Extraction)

## 概述

本技能负责从非结构化的保险合同和财富管理协议文本中，运用NLP技术精准提取关键条款信息并转化为标准化的结构化数据。适用于保险合同（投保单、保险条款、批单、特别约定）和财富管理协议（基金合同、资管计划说明书、私募基金协议）的自动化解析。

## 适用场景

- 新保险产品上市时的条款入库
- 客户投保后保单条款的结构化处理
- 基金合同签署后的关键参数提取
- 批单/补充协议的增量变更解析
- 条款模板库的批量维护和更新

## 执行步骤

### 步骤1：文档预处理
1. 识别文档类型（保险合同/基金合同/资管计划说明书/私募协议）
2. 执行OCR处理（如为扫描件），确保文字识别准确率≥99%
3. 进行文档结构分析，识别章节、条款编号、表格等结构要素
4. 对合同进行分页标注，建立原文定位索引

### 步骤2：条款分类识别
1. **保险合同**识别以下条款类别：
   - 保险责任条款（承保范围、触发条件）
   - 免责条款（除外责任、免赔事项）
   - 赔付规则（赔付比例、免赔额、最高限额、赔付方式）
   - 时效条款（等待期、犹豫期、保险期间）
   - 退保/续保条款（现金价值、自动续保规则）
   - 特别约定（加费、除外、限制条件）
2. **财富管理协议**识别以下条款类别：
   - 投资范围与限制
   - 风险等级与评估
   - 费率结构（认购费、管理费、托管费、赎回费）
   - 流动性安排（开放日、锁定期、最低持有期限）
   - 巨额赎回条款（触发阈值、处理机制）
   - 清盘条件（净值触发线、份额下限）

### 步骤3：要素提取与结构化
1. 对每个识别的条款进行精细化要素提取
2. 将自然语言描述转化为结构化字段（JSON格式）
3. 提取数值参数（金额、比例、期限、日期等）
4. 识别条件逻辑关系（if-then规则、例外情形、并列条件）
5. 处理交叉引用（条款间的引用关系、附件引用）

### 步骤4：置信度评估与标注
1. 对每个提取字段计算置信度分数（0-100%）
2. 置信度≥95%的字段标记为"已确认"
3. 置信度<95%的字段标记为"待人工确认"，并附注低置信原因
4. 对存在歧义的条款提供多种可能的解释说明

### 步骤5：质量校验与入库
1. 执行逻辑一致性校验（如免赔额不应大于保额）
2. 与条款模板库进行偏差比对，标注非标准条款
3. 生成结构化数据并写入条款知识库
4. 对增量变更（批单）执行diff比对并更新已有数据

## 输入规格

| 输入字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| document_file | File | 是 | 合同文档（PDF/Word/图片） |
| document_type | Enum | 是 | insurance_policy/fund_contract/asset_plan/private_fund |
| policy_id | String | 否 | 关联保单/产品编号（增量解析时必填） |
| extraction_mode | Enum | 否 | full（全量）/ incremental（增量），默认full |

## 最佳实践

1. **模板匹配优先**：先尝试与已有模板库匹配，匹配成功的条款可大幅提升准确率
2. **上下文关联**：条款提取必须考虑上下文，避免孤立提取导致语义丢失
3. **版本管理**：保留每次解析的历史版本，支持条款变更的追溯分析
4. **术语标准化**：使用行业标准术语词典统一表述，避免同一概念的不同表达造成混淆
5. **异常条款标注**：对于明显偏离行业惯例的非标准条款，特别标注以引起下游Agent注意

## 约束条件

- 新合同必须在24小时内完成全量解析入库
- 批单/变更必须在4小时内完成增量更新
- 关键条款提取准确率≥95%
- 置信度<95%的字段必须100%标记为待人工确认
- 不得对条款进行主观解读或法律判断，仅提取客观内容
