# 策略参数调优 (Strategy Parameter Tuning)

## 概述
策略参数调优是将数据分析结论转化为实际风控策略变更的执行技能。涵盖风险阈值调整、验证策略优化、动态策略规则配置等，确保参数变更经过完整的提案→审批→灰度→全量生效流程，实现风控效果的持续优化。

## 适用场景
- 风险评分阈值的优化调整（如调整各级别分界点）
- 验证策略的变更（如特定场景下的验证方式调整）
- 动态风控策略的配置（如大促期间自动收紧策略）
- A/B测试通过后的全量推广执行
- 异常事件后的紧急策略调整

## 执行步骤

### 步骤1：调优提案生成
- 基于指标监控和A/B测试结果，生成标准化调优提案：
  - **变更描述**：明确变更内容（参数名、当前值、目标值）
  - **变更依据**：数据支撑（指标分析、A/B测试结果、异常分析结论）
  - **预期效果**：量化预期影响（通过率变化、欺诈率变化等，含置信区间）
  - **影响范围**：受影响的交易量/客群/渠道估算
  - **风险评估**：潜在风险和最坏情况分析
  - **回滚方案**：如何快速回退到变更前状态

### 步骤2：审批流程
- 将提案提交至审批流程：
  - 常规调优：风控经理审批 → 1个工作日内
  - 重大变更（影响>10%交易量）：风控总监审批 → 需审批委员会讨论
  - 紧急调优（安全事件响应）：快速审批通道 → 2小时内
- 审批通过后生成变更工单

### 步骤3：灰度发布与观测
- 灰度策略执行：
  - 第一阶段：5%流量验证（≥24小时）
  - 第二阶段：20%流量验证（≥24小时）
  - 第三阶段：50%流量验证（≥24小时）
  - 全量发布
- 每阶段的观测检查点：
  - 主要指标是否符合预期
  - 护栏指标是否有恶化
  - 系统性能是否受影响
  - 客户投诉是否有异常
- 任一检查点不通过则暂停扩量

### 步骤4：全量生效与验证
- 全量切换后持续观测72小时
- 确认效果稳定后关闭变更工单
- 更新策略配置文档和版本记录
- 策略变更审批后1小时内完成全量部署

### 步骤5：动态策略管理
- 维护时间触发策略：
  - 高风险时段自动收紧（如凌晨2-5点）
  - 大促期间预设策略（提前配置、定时生效）
  - 节假日前后特殊策略
- 维护事件触发策略：
  - 欺诈事件爆发 → 自动收紧至最保守策略
  - 系统负载过高 → 自动切换轻量级规则
  - 误拦截率飙升 → 自动放宽至备用阈值

## 输入规格
```json
{
  "operation_type": "string - PROPOSE/APPROVE/DEPLOY/ROLLBACK",
  "change_type": "string - THRESHOLD/VERIFICATION/DYNAMIC_RULE/EMERGENCY",
  "parameters": [
    {
      "param_name": "string - 参数标识",
      "current_value": "any - 当前值",
      "proposed_value": "any - 建议值",
      "scope": "string - 影响范围"
    }
  ],
  "justification": "string - 变更依据",
  "ab_test_id": "string - 关联的A/B测试ID（如有）"
}
```

## 输出规格
```json
{
  "change_id": "string - 变更工单编号",
  "status": "string - PROPOSED/APPROVED/DEPLOYING/ACTIVE/ROLLED_BACK",
  "deployment_phase": "string - CANARY_5/CANARY_20/CANARY_50/FULL",
  "effective_time": "ISO8601 - 生效时间",
  "monitoring_results": {
    "phase": "string",
    "metrics_check": "string - PASS/FAIL",
    "observations": "array"
  },
  "rollback_ready": "boolean",
  "next_action": "string - 下一步建议"
}
```

## 最佳实践
- 每次变更只调整一个参数，便于归因分析
- 紧急策略调整后也要补充正式提案和审批流程
- 保持完整的参数版本历史，支持任意版本的回滚
- 建立策略变更的通知机制，确保所有相关方知悉变更

## 约束条件
- 策略变更审批后1小时内完成全量部署
- 非紧急变更必须经过灰度验证
- 所有变更必须有回滚预案
- 参数变更日志保留≥3年
- 不得擅自绕过审批流程直接修改生产参数
