# 外汇额度监控 (Forex Quota Monitoring)

## 概述
外汇额度监控技能负责实时追踪个人客户年度便利化购汇/结汇额度（每人每年等值5万美元），确保客户外汇交易在法定额度范围内合规进行。同时识别和防范分拆购汇等规避额度管制的违规行为。

## 适用场景
- 个人客户购汇/结汇申请的额度校验
- 年度额度使用进度实时查询
- 分拆购汇行为的实时监测和阻断
- 超额度购汇申请的合规审批流程
- 年初额度重置和历史额度统计

## 执行步骤

### 步骤1：额度状态实时查询
- 查询客户当年度已使用额度：
  - 购汇已用额度（各币种折合美元累计）
  - 结汇已用额度（各币种折合美元累计）
- 按当日汇率计算本次交易的等值美元金额
- 计算交易后的累计使用额度

### 步骤2：额度合规判定
- **额度充足（交易后累计 ≤ 5万美元）**：
  - 放行交易
  - 更新已用额度
  - 如达到80%使用率，生成客户提醒
- **额度不足（交易后累计 > 5万美元）**：
  - 拦截交易
  - 告知客户已用额度和剩余额度
  - 引导客户走超额度购汇审批流程（需提供真实用途证明）

### 步骤3：分拆购汇监测
- 检测以下分拆规避模式：
  - 同一客户通过多个银行分散购汇（通过征信/外管局接口联查）
  - 多个关联自然人（家庭成员、代持人）集中购汇
  - 短时间内频繁小额购汇（累计接近5万美元上限）
  - 先购汇后通过地下钱庄转移
- 命中分拆特征时：
  - 拦截当前交易
  - 标记客户为疑似分拆购汇
  - 上报外汇管理部门

### 步骤4：额度管理与报告
- 年度额度在每年1月1日自动重置
- 生成客户额度使用明细（支持客户查询）
- 统计机构整体额度使用情况
- 配合外汇管理局的额度核查

## 输入规格
```json
{
  "customer_id": "string",
  "operation_type": "string - BUY_FX/SELL_FX",
  "amount": "number - 交易金额",
  "currency": "string - 交易币种",
  "purpose": "string - 购汇/结汇用途",
  "operation": "string - CHECK/DEDUCT/QUERY"
}
```

## 输出规格
```json
{
  "quota_check_result": "string - SUFFICIENT/INSUFFICIENT/SPLIT_SUSPECTED",
  "annual_limit_usd": 50000,
  "used_amount_usd": "number - 已用额度(美元)",
  "remaining_amount_usd": "number - 剩余额度(美元)",
  "this_transaction_usd": "number - 本次交易折合美元",
  "utilization_rate": "number - 使用率百分比",
  "split_risk_indicators": "array - 分拆风险指标（如有）",
  "warning_message": "string - 预警信息"
}
```

## 最佳实践
- 额度数据应与外管局系统实时同步，防止跨行重复购汇
- 汇率折算应使用统一的即时汇率源，避免因汇率差异导致额度误差
- 对即将用完额度的客户提前提醒，提升服务体验
- 分拆购汇监测规则应定期更新以覆盖新型规避手法

## 约束条件
- 额度数据实时准确性要求100%
- 额度扣减必须与交易执行原子绑定（避免扣了额度但交易未成功）
- 不得协助或容忍分拆购汇行为
- 超额度交易的审批必须有真实用途证明
