# 支付授权决策 (Payment Authorization Decision)

## 概述
支付授权决策是基于实时风险评分做出交易授权判定的核心技能。根据风险评分分级，动态选择验证强度（免验/短信/人脸/拦截），并处理大额交易增强验证逻辑，确保在风险可控的前提下最大化客户支付体验。

## 适用场景
- 所有通过风险评分后需要做出授权决策的支付交易
- 触发大额交易增强验证规则的交易
- 需要人工确认的高风险拦截交易
- 验证通过/失败后的最终授权/拒绝

## 执行步骤

### 步骤1：风险等级判定与策略路由
- 接收实时风险评分结果
- 根据当前生效的阈值配置执行分级：
  - 低风险（< 30分）：路由至免验授权通道
  - 中低风险（30-50分）：路由至短信验证码验证
  - 中高风险（50-75分）：路由至人脸识别验证
  - 高风险（> 75分）：路由至拦截+人工确认通道

### 步骤2：大额交易增强验证叠加
- 判断交易是否达到大额报告标准：
  - 个人现金交易 ≥ 5万元
  - 企业现金交易 ≥ 20万元
  - 个人转账 ≥ 50万元
  - 企业转账 ≥ 200万元
- 达标交易在原有验证要求基础上叠加增强核验（至少提升一级验证强度）
- 标记为大额交易，同步推送给AML监测Agent

### 步骤3：执行验证与结果处理
- 免验通道：直接生成APPROVE决策
- 验证通道：下发验证请求，等待验证结果
  - 验证通过：生成APPROVE决策
  - 验证失败/超时：生成DENY决策
  - 客户放弃验证：记录Drop-off事件，生成ABANDON状态
- 拦截通道：生成BLOCK决策，推送人工确认工单

### 步骤4：决策输出与日志记录
- 生成标准格式的授权决策响应
- 写入完整决策日志（含所有评分因子、决策路径、时间戳）
- 高风险拦截交易同步推送AML监测Agent
- 更新客户当日/当月累计授权金额

## 输入规格
```json
{
  "transaction_id": "string",
  "risk_score": "integer - 风险评分",
  "risk_level": "string - 风险等级",
  "amount": "number - 交易金额",
  "customer_type": "string - PERSONAL/ENTERPRISE",
  "transaction_type": "string - CASH/TRANSFER",
  "top_factors": "array - 主要风险因子"
}
```

## 输出规格
```json
{
  "decision": "string - APPROVE/CHALLENGE/DENY/BLOCK",
  "challenge_method": "string - SMS/FACE_ID/MANUAL (仅CHALLENGE时)",
  "large_transaction_flag": "boolean - 是否大额交易",
  "enhanced_verification": "boolean - 是否触发增强验证",
  "decision_reason": "string - 决策原因摘要",
  "aml_push_required": "boolean - 是否需推送AML",
  "decision_timestamp": "ISO8601"
}
```

## 最佳实践
- 决策逻辑应可配置化（阈值通过配置中心管理），避免硬编码
- 验证放弃率需实时监控，超过15%时立即告警并触发策略审查
- 人工确认工单应包含足够上下文信息（交易详情、风险因子、客户历史），减少确认处理时间
- 拦截决策必须附带对客户友好的通知文案（不泄露模型细节）

## 约束条件
- 整体决策链路（评分+决策）总耗时P99≤100ms
- 决策日志保留≥5年，满足监管审计要求
- 拦截率异常升高（日环比>5%）需自动触发告警
- 高风险拦截交易100%推送AML监测，不得遗漏
