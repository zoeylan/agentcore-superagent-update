# 客户洗钱风险评级 (Customer Risk Rating)

## 概述
客户洗钱风险评级是AML合规体系的基础能力，基于客户身份特征、业务特征和交易行为模式对客户进行洗钱风险分级（低/中/高），高风险客户将被纳入增强监控机制。评级结果直接影响监控力度、尽职调查深度和资源配置优先级。

## 适用场景
- 新客户开户时的初始风险评级
- 季度定期重新评估（覆盖全量客户）
- 重大事件触发的即时重评（如STR提交、制裁命中、异常交易模式发现）
- 客户信息/业务性质发生重大变更时
- 跨域数据输入触发的评级调整（如信贷逾期、司法涉诉）

## 执行步骤

### 步骤1：风险因子采集
- **客户身份因子**：
  - 国籍/注册地（高风险国家/地区加权）
  - 政治公众人物（PEP）身份
  - 职业/行业（现金密集型、高风险行业加权）
  - 证件有效性和KYC完整度
- **业务关系因子**：
  - 开户渠道（远程开户风险高于柜面）
  - 账户用途与客户申报业务的匹配度
  - 产品/服务复杂度
  - 客户关系存续时间
- **交易行为因子**：
  - 历史可疑交易告警次数
  - 大额/异常交易频率
  - 交易对手风险分布
  - 跨境交易占比和方向

### 步骤2：风险评分计算
- 各因子按权重加权求和，生成综合风险评分（0-100）
- 评分阈值切分：
  - 低风险（0-39分）：标准监控
  - 中风险（40-69分）：增强监控
  - 高风险（70-100分）：最严格监控
- 存在一票否决条件时直接定为高风险：
  - PEP身份
  - 曾有STR提交记录
  - 制裁名单模糊匹配历史
  - 司法机关协查记录

### 步骤3：评级结果生效与执行
- 新评级结果与上次评级对比，记录变动原因
- 评级上调（风险升高）立即生效
- 评级下调（风险降低）需合规人员审批后生效
- 通知相关系统执行差异化监控策略：
  - 高风险：全量交易实时告警、季度EDD、限制高风险产品
  - 中风险：重点交易监控、年度审查
  - 低风险：常规批量监控

### 步骤4：评级审计与报告
- 生成评级变动报告（含变动原因、支撑数据）
- 评级覆盖率统计（目标100%）
- 各级别客户分布统计
- 未及时重评客户的超期预警

## 输入规格
```json
{
  "customer_id": "string",
  "evaluation_trigger": "string - ONBOARDING/PERIODIC/EVENT/CHANGE",
  "trigger_event": "string - 触发事件描述（如有）",
  "override_factors": "array - 外部输入的额外风险因子"
}
```

## 输出规格
```json
{
  "customer_id": "string",
  "risk_rating": "string - LOW/MEDIUM/HIGH",
  "risk_score": "integer - 0-100",
  "previous_rating": "string - 上次评级",
  "rating_change": "string - UPGRADE/DOWNGRADE/UNCHANGED",
  "key_factors": "array - 主要风险因子及贡献度",
  "veto_conditions": "array - 命中的一票否决条件（如有）",
  "next_review_date": "date - 下次评估日期",
  "monitoring_level": "string - 对应的监控级别"
}
```

## 最佳实践
- 评级模型的因子权重应基于本机构历史STR数据和风险案例校准
- 评级结果应在各业务系统间实时同步（支付授权、信贷审批等）
- 高风险客户评级变动应通知合规负责人和业务部门
- 建立评级模型的回溯验证机制（实际发生的STR/案件是否集中在高风险客户）

## 约束条件
- 季度重评覆盖率100%
- 评级上调立即生效，不得有延迟
- 评级数据属于敏感合规数据，访问需权限控制
- 评级变动审计日志保留≥5年
