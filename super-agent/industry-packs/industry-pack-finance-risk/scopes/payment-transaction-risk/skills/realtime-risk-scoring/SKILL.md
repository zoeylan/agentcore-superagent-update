# 实时风险评分 (Realtime Risk Scoring)

## 概述
实时风险评分是支付授权Agent的核心能力，在100ms时间窗口内综合多维度交易特征计算风险评分（0-100分），为授权决策提供量化依据。该技能需要在极致性能约束下完成特征提取、模型推理和评分输出的全链路处理。

## 适用场景
- 银行卡支付交易的实时风险评估
- 网银转账请求的风险评分
- 快捷支付和二维码支付的风险量化
- 任何需要实时风险量化判断的支付场景

## 执行步骤

### 步骤1：交易特征提取
- 从支付请求中解析基础字段：交易金额、交易时间、支付渠道、商户信息（MCC码）、设备标识、IP地址、地理位置
- 从实时特征存储中提取时序特征：近1小时/24小时/7天的交易次数、累计金额、最大单笔金额
- 查询客户画像数据：历史交易模式（均值、标准差）、常用设备列表、常用地理位置、交易时间分布

### 步骤2：多维度风险因子计算
- **金额异常度**：当前金额与客户历史均值的偏离程度（Z-Score）
- **时间异常度**：交易时间是否落在客户非典型活跃时段
- **设备一致性**：当前设备是否为已知可信设备
- **地理位置一致性**：是否与历史常驻地匹配、是否存在不可能的地理位移（velocity check）
- **收款方风险**：商户MCC码风险分类、收款方黑名单比对
- **行为频次异常**：短时间窗口内的交易密度是否超出正常范围

### 步骤3：模型推理与评分输出
- 将特征向量输入实时评分模型（XGBoost/LightGBM ensemble）
- 模型输出原始概率值，通过Score Mapping函数转换为0-100整数评分
- 输出Top 3贡献因子及其SHAP值，支持决策可解释性

### 步骤4：评分质量校验
- 检查评分是否落在合理区间（异常高分/低分需要二次确认）
- 记录评分延迟，超过80ms时触发告警
- 将评分结果写入决策日志

## 输入规格
```json
{
  "transaction_id": "string - 交易唯一标识",
  "amount": "number - 交易金额（元）",
  "currency": "string - 币种",
  "timestamp": "ISO8601 - 交易时间",
  "channel": "string - 支付渠道(CARD/TRANSFER/QUICK/QR)",
  "merchant": {
    "id": "string",
    "mcc": "string - 商户类别码",
    "name": "string"
  },
  "device": {
    "fingerprint": "string - 设备指纹",
    "ip": "string",
    "geo": {"lat": "number", "lng": "number"}
  },
  "customer_id": "string - 客户标识"
}
```

## 输出规格
```json
{
  "risk_score": "integer - 0-100风险评分",
  "risk_level": "string - LOW/MEDIUM_LOW/MEDIUM_HIGH/HIGH",
  "top_factors": [
    {"factor": "string - 因子名称", "contribution": "number - 贡献度", "detail": "string - 说明"}
  ],
  "model_version": "string - 使用的模型版本",
  "latency_ms": "number - 评分耗时",
  "timestamp": "ISO8601 - 评分时间"
}
```

## 最佳实践
- 特征提取和模型推理应并行执行以压缩延迟
- 使用本地缓存（Redis/内存）存储高频访问的客户画像数据，避免远程调用
- 评分模型应部署多副本，支持负载均衡和故障隔离
- 定期（每周）监控评分分布的稳定性（PSI），及时发现数据漂移

## 约束条件
- 端到端评分延迟P99≤50ms（为下游决策预留时间）
- 模型推理失败时降级为规则评分，不得返回空结果
- 评分日志100%落盘，不可丢失
- 不得在评分过程中修改客户数据或交易状态
