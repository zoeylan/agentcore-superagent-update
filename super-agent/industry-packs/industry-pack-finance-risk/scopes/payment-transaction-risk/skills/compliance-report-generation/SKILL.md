# 合规报告生成 (Compliance Report Generation)

## 概述
合规报告生成技能负责定期产出符合监管要求的风控运营报告和反洗钱合规报告，为管理层决策和监管报送提供标准化、准确的数据支撑。报告涵盖支付风控效能、AML监测成效、跨境合规执行等多个维度。

## 适用场景
- 月度反洗钱合规报告（内部管理用）
- 月度支付风控运营报告
- 季度合规回顾报告（监管报送用）
- 年度反洗钱工作报告
- 专项合规分析报告（应监管要求或特定事件）

## 执行步骤

### 步骤1：数据汇集与清洗
- 从各Agent和业务系统汇集报告所需数据：
  - **支付授权数据**：交易量、通过率、拦截率、验证类型分布、延迟数据
  - **AML监测数据**：告警数、STR提交数、大额报告数、制裁筛查命中数
  - **跨境合规数据**：跨境交易量、合规通过率、EDD执行数、额度使用统计
  - **客户风险数据**：各级别客户分布、评级变动统计
- 数据清洗与一致性校验：
  - 交叉核对各源数据的汇总数（确保口径一致）
  - 标记数据缺失或异常值
  - 数据准确率要求≥99.5%

### 步骤2：报告内容编制
- **月度AML合规报告**内容模板：
  1. 执行摘要（关键数据一览）
  2. 可疑交易监测统计（告警数/确认率/STR提交数/类型分布）
  3. 大额交易报告统计（报告数/及时率/金额分布）
  4. 制裁名单筛查统计（筛查量/命中数/处置结果）
  5. 客户风险评级分布与变动
  6. 规则有效性分析（各规则命中率/确认率）
  7. 合规指标达标情况
  8. 问题与改进建议

- **月度风控运营报告**内容模板：
  1. 交易风控全景数据
  2. 风险拦截效能分析
  3. 客户体验指标（通过率/放弃率/投诉率）
  4. 模型性能健康度
  5. 策略变更与效果跟踪
  6. 异常事件回顾
  7. 下月优化计划

### 步骤3：可视化与格式化
- 关键指标配图表（趋势图、分布图、对比图）
- 红黄绿状态标记（达标/关注/不达标）
- 同比/环比标注
- 报告版本号和生成时间戳

### 步骤4：审核与分发
- 报告初稿生成后提交合规负责人审核
- 审核通过后按权限分发：
  - 内部管理报告 → 风控部门、管理层
  - 监管报送报告 → 合规部门 → 监管机构
- 归档留存（电子版+必要时纸质版）

## 输入规格
```json
{
  "report_type": "string - AML_MONTHLY/RISK_OPS_MONTHLY/QUARTERLY_REVIEW/ANNUAL/SPECIAL",
  "reporting_period": {"start": "date", "end": "date"},
  "target_audience": "string - INTERNAL/REGULATORY/MANAGEMENT",
  "special_focus": "array - 特别关注主题（如有）"
}
```

## 输出规格
```json
{
  "report_id": "string",
  "report_type": "string",
  "report_period": "string",
  "generation_date": "date",
  "data_completeness": "number - 数据完整度百分比",
  "key_highlights": "array - 报告要点摘要",
  "compliance_status": {
    "str_timeliness": "string - 达标/不达标",
    "ltr_timeliness": "string",
    "sanctions_coverage": "string",
    "overall_assessment": "string"
  },
  "report_content": "object - 完整报告内容",
  "review_status": "string - DRAFT/REVIEWED/APPROVED/DISTRIBUTED"
}
```

## 最佳实践
- 建立报告模板库，确保各期报告格式一致可比
- 自动化数据采集和基础报表生成，人工聚焦分析和建议
- 报告应有明确的"So What"——不仅展示数据，更要解读含义
- 对监管报送报告的格式和内容需与合规部门提前确认

## 约束条件
- 月度报告T+5个工作日内产出
- 数据准确率≥99.5%
- 监管报送报告必须经合规负责人签批
- 报告保留≥5年
- 涉及客户敏感数据的报告需脱敏处理后方可分发
