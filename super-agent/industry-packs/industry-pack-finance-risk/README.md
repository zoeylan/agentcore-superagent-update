## 行业概述

金融风控是银行、消费金融公司、小贷机构等金融组织的核心能力支柱，贯穿信贷业务的全生命周期——从**贷前**的客户准入与风险定价，到**贷中**的存量监控与预警响应，再到**贷后**的逾期资产处置与损失缓释。

中国金融风控行业受国家金融监督管理总局（原银保监会）严格监管，核心法规包括《商业银行法》《贷款通则》《反洗钱法》及《个人信息保护法》等。行业正经历从"规则驱动"到"数据+AI 驱动"的深刻变革，大数据征信、机器学习评分模型、实时流计算反欺诈和智能催收等技术正在全面重塑风控体系。

## 业务域详情

### 📋 信贷审批

**目录：** `scopes/credit-approval`

信贷审批是金融风控的入口环节，决定了资产质量的起点。本业务域覆盖从客户提交贷款申请到最终放款（或拒绝）的完整审批流程，支持个人消费贷/经营贷与企业流动资金贷款/项目贷款两大类场景。

**核心业务规则：**
- 个人信贷单笔上限 50 万，企业信贷需额外尽调流程
- 征信评分低于 600 分直接拒绝（"秒拒"通道）
- 反欺诈命中规则需人工复核
- 审批时效：个人信贷 48 小时内，企业信贷 5 个工作日内

**Agent 团队（5 位）：**

| Agent | 角色 | 职责概述 |
|-------|------|----------|
| 进件预处理 Agent | 第一道关卡 | 资料完整性校验、OCR 信息提取、基础准入筛查 |
| 反欺诈核查 Agent | 贷前反欺诈专家 | 身份核验、设备指纹分析、关联图谱检测、多头借贷查询 |
| 信用评估 Agent | 信用风险量化专家 | 征信报告解析、信用评分、还款能力分析、风险定价 |
| 合规审批 Agent | 合规终审与决策 | 合规性审查、最终额度审批、审批决策留痕 |
| 放款通知 Agent | 放款执行 | 放款前置检查、合同校验、资金发放、多渠道通知 |

**配备技能（17 项）：** 准入规则校验、申请路由、审批决策、关联图谱分析、合规审查、合同验证、征信报告分析、信用评分、客户通知、放款执行、欺诈模式检测、身份验证、多头借贷查询、OCR 文档提取、监管报送、还款能力分析、风险定价

### ⚖️ 逾期资产治理

**目录：** `scopes/delinquent-asset-management`

逾期资产治理是金融风控的"止血"环节，直接影响不良贷款率（NPL）和资产质量。本业务域按行业通行的逾期分级体系运作：
- **M1（1-30 天）**：早期逾期，以自动化催收为主（AI 外呼/短信）
- **M2（31-60 天）**：中期逾期，升级为人工催收，注重沟通策略
- **M3+（60 天以上）**：晚期逾期，法务催收或不良资产处置

**核心业务规则：**
- M1 阶段以短信和 AI 外呼为主，M2 阶段升级为人工催收
- M3+ 阶段评估是否转入法务催收或核销
- 催收过程必须合规，不得使用暴力催收手段
- 每月生成逾期资产治理报告（回收率、催收成本等指标）

**Agent 团队（5 位）：**

| Agent | 角色 | 职责概述 |
|-------|------|----------|
| 智能分案 Agent | 调度中枢 | 案件分级、策略匹配、催收资源最优分配 |
| 催收执行 Agent | M1/M2 执行者 | 多渠道客户触达、AI 外呼、承诺跟进、合规监控 |
| 客户沟通 Agent | 诉求与协商 | 账单异议处理、减免评估、个性化还款方案设计、投诉处理 |
| 法务处置 Agent | M3+ 法律处置 | 诉讼可行性评估、起诉材料准备、委外催收管理、核销申报 |
| 催收数据分析 Agent | 数据分析中枢 | 回收率监控、迁徙率分析、回收预测、A/B 测试评估 |

**配备技能（22 项）：** 资产处置评估、案件分级评估、催收看板监控、催收绩效评估、催收记录管理、催收策略匹配、合规监控、客户投诉处理、争议调查、动态升级监控、困难援助处理、法务可行性评估、诉讼文书准备、诉讼进度跟踪、多渠道催收执行、委外管理、拨备充足性评估、回收预测、还款协商、还款计划管理、资源优化分配、迁徙率分析

### 🛡️ 反欺诈中心

**目录：** `scopes/anti-fraud-center`

反欺诈中心是金融风控的"安全盾牌"，采用"规则引擎 + 机器学习模型"双层检测架构，覆盖实时交易监控、欺诈模式识别、可疑交易调查、规则引擎管理和前瞻性欺诈趋势预警。

**核心业务规则：**
- 实时交易监控延迟不超过 200ms
- 欺诈概率 >80% 自动拦截，50-80% 转人工复核
- 每周更新反欺诈规则库
- 重大欺诈案件 24 小时内上报监管

| Agent | 角色 | 职责概述 |
|-------|------|----------|

| 案件调查 Agent | 深度调查专家 | 可疑交易告警调查、团伙欺诈关联分析、监管报送 |
| 规则策略 Agent | 策略管理者 | 规则库全生命周期管理、模型性能监控、阈值调优 |
| 欺诈情报分析 Agent | 情报侦察兵 | 多渠道欺诈情报收集、威胁评估、知识库维护 |

**配备技能（12 项）：** 跨机构协作、欺诈案件调查、欺诈知识库管理、欺诈趋势分析、团伙欺诈网络分析、模型性能监控、实时交易评分、监管报告生成、风险决策路由、规则生命周期管理、系统健康监控、威胁情报收集

### 🏦 保险与财富管理

**目录：** `scopes/insurance-wealth-management`

保险与财富管理业务域覆盖保险理赔全流程（合同解析→材料核验→赔付决策）、财富管理配置流程（客户画像→适当性匹配→配置建议→投后监控）和私募赎回流程（合规校验→份额计算→资金划付）三条主业务线。

**核心业务规则：**
- 保险合同条款解析置信度 <95% 的字段须人工确认
- 标准理赔 5 万元以下自动审批，5-50 万元人工复核，50 万元以上终审
- 理赔反欺诈检测覆盖所有案件，命中规则强制人工审查
- 资产配置建议须通过适当性匹配合规校验
- 私募基金赎回须在开放日窗口期内受理

### 💳 支付与交易风控

**目录：** `scopes/payment-transaction-risk`

支付与交易风控业务域覆盖支付授权、反洗钱（AML）监测、跨境合规和策略调优的全业务流程，实现从支付请求接收到风控决策、合规报送和策略优化的端到端自动化处理。采用"实时评分 + 规则引擎 + 人工复核"三层架构，在保障资金安全的同时优化客户支付体验。

**核心业务规则：**
- 实时风险评分 100ms 内完成，主模型超时降级至备用规则引擎
- 评分分级授权：<30 免验、30-50 短信验证、50-75 人脸识别、>75 拦截
- 大额交易标准：个人现金≥5万/企业现金≥20万/个人转账≥50万/企业转账≥200万
- 可疑交易报告（STR）需在发现后 24 小时内提交人民银行反洗钱中心
- 跨境支付须验证贸易背景真实性和外汇管制合规

| 支付授权 Agent | 实时风控引擎 | 100ms 内完成交易风险评估并决定授权策略（免验/验证/拦截） |
| AML 监测 Agent | 反洗钱专家 | 可疑交易识别、大额报告生成、制裁名单筛查 |
| 跨境合规 Agent | 跨境审核专家 | 跨境支付交易背景真实性验证和外汇管制合规审核 |
| 策略调优 Agent | 策略优化者 | 风控参数调优、A/B 测试评估、效果监控和安全-体验平衡管理 |

**配备技能（17 项）：** A/B 测试设计评估、合规报告生成、跨境身份验证、客户风险评级、增强尽职调查、外汇额度监控、大额交易报告、模型健康评估、支付授权决策、实时风险评分、风控指标监控、制裁名单筛查、策略参数调优、可疑交易检测、超时降级处理、贸易背景验证、交易限额管理

## 跨域协同

五大业务域形成金融风控的完整闭环：

```
信贷审批 ──黑名单/规则共享──→ 反欺诈中心
    │                              │
    │ 贷款信息                      │ 欺诈情报预警
    ↓                              ↓
逾期资产治理 ←──资产质量反馈──→ 信贷审批
    │
    │ 欺诈线索                支付与交易风控
    ↓                         │        │
反欺诈中心 ←──可疑交易──┘        │
                                       │ AML情报
保险与财富管理 ←──跨域风控联动──┘
```

- **信贷审批 → 反欺诈中心**：共享黑名单库、规则引擎和设备指纹库；贷前欺诈线索传递深度调查
- **信贷审批 ↔ 逾期资产治理**：审批通过的贷款信息流转支撑智能分案；逾期趋势信号驱动审批策略动态调整
- **逾期资产治理 → 反欺诈中心**：催收中发现的欺诈线索传递调查；欺诈案件直接转法务处置通道
- **反欺诈中心 → 全域**：行业欺诈趋势同时预警信贷审批和逾期资产治理
- **支付与交易风控 → 反欺诈中心**：实时交易中发现的可疑交易线索传递深度调查；AML 监测共享制裁名单和情报
- **支付与交易风控 ↔ 保险与财富管理**：大额资金赎回的交易风控联动；投保/理赔资金流的反洗钱监测
- **保险与财富管理 → 信贷审批**：客户资产配置信息辅助信用评估和风险定价

## 定制建议

以下是根据您具体业务场景建议调整的内容：

### 业务参数调整

| 调整项 | 默认值 | 说明 |
|--------|--------|------|
| 个人信贷单笔上限 | 50 万元 | 根据机构资质和监管要求调整 |
| 征信评分拒绝阈值 | 600 分 | 根据目标客群和风险偏好调整，可按产品线差异化设置 |
| 反欺诈拦截阈值 | 80 分 | 根据误报率容忍度调整，降低阈值提高拦截率但增加误报 |
| 反欺诈复核区间 | 50-80 分 | 根据人工复核产能调整区间宽度 |
| 实时监控延迟 SLA | 200ms | 根据基础设施能力和业务要求调整 |
| 审批时效（个人） | 48 小时 | 头部机构可压缩至分钟级 |
| 审批时效（企业） | 5 工作日 | 涉及复杂尽调的项目贷款可适当放宽 |

### 催收策略调整

- **逾期分级阈值**：M1/M2/M3 的天数划分可根据产品特性调整（如信用卡通常以账单周期为单位）
- **催收渠道偏好**：根据客群特征调整 AI 外呼、短信、APP 推送的优先级和频次
- **减免授权矩阵**：根据机构内部权限体系配置不同级别的减免审批权限
- **法务催收阈值**：根据地区司法环境和诉讼成本调整法务处置的金额门槛
