# 安全策略与制度建设

## 描述
制定和优化企业信息安全管理制度体系，包括安全策略、安全标准、安全流程和操作规程，确保制度体系符合合规要求并具备可执行性。

## 触发场景
- 构建或完善企业安全制度体系
- 制定具体安全管理制度（如访问控制策略、数据分类分级制度）
- 安全流程设计（如变更管理流程、漏洞管理流程）
- 安全制度年度修订和评审
- 安全意识培训内容设计
- 安全考核指标(KPI/KRI)设计

## 输入格式
请提供以下信息：
- **制度类型**：策略/标准/流程/规程/指南
- **适用范围**：全公司/特定部门/特定系统
- **合规要求**：需要满足的法规或标准要求
- **组织现状**：当前安全管理成熟度和执行情况
- **行业特性**：所属行业的监管特点
- **受众**：制度的执行对象

## 输出内容
1. **制度框架**：制度体系的层级结构和覆盖范围
2. **制度正文**：包含目的、范围、职责、具体要求、例外处理
3. **配套流程图**：关键流程的可视化描述
4. **RACI矩阵**：各环节的责任分配
5. **考核指标**：制度执行效果的衡量指标
6. **实施计划**：制度发布、培训、试运行的时间安排
7. **常见问题解答**：预判执行中可能遇到的问题

## 专业准则
- 制度体系四层架构：策略(Why) → 标准(What) → 流程(How) → 规程(Detail)
- 制度不是越多越好，关键是可执行、可检查、可度量
- 等保2.0要求的制度文档：安全管理制度、安全管理机构、安全管理人员、安全建设管理、安全运维管理五大类
- 制度修订周期：策略类年度修订、标准类半年度、流程类按需更新
- 安全KPI设计原则：SMART原则，且必须包含滞后指标和领先指标
  - 滞后指标示例：安全事件数量、数据泄露次数
  - 领先指标示例：漏洞修复及时率、安全培训覆盖率、钓鱼演练点击率
- 制度落地三板斧：培训宣贯、技术固化、检查考核
- 安全意识培训频率：全员年度培训 + 新员工入职培训 + 关键岗位季度专项培训
- 制度需要考虑例外处理机制，避免因制度僵化影响业务敏捷性
