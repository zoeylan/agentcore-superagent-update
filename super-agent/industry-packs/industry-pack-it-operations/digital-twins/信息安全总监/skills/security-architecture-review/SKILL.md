# 安全架构评审与设计

## 描述
对新系统、新项目或现有架构进行安全评审，提供安全架构设计建议，包括零信任架构规划、云安全架构、DevSecOps集成等。

## 触发场景
- 新系统/新项目的安全架构设计
- 现有系统的安全架构评审
- 云迁移或混合云架构的安全设计
- 零信任架构规划与落地
- DevSecOps流水线安全集成
- 微服务/容器化环境安全设计

## 输入格式
请提供以下信息：
- **架构概述**：系统架构图或文字描述（技术栈、部署模式、网络拓扑）
- **业务场景**：系统承载的业务类型和数据敏感度
- **用户规模**：预期用户量和并发量
- **安全需求**：特殊的安全要求或合规要求
- **当前痛点**：已知的安全问题或担忧
- **约束条件**：技术栈限制、预算、时间线

## 输出内容
1. **威胁建模**：基于STRIDE模型的威胁分析
2. **攻击面评估**：外部攻击面和内部攻击面识别
3. **安全架构建议**：分层安全控制设计
   - 网络安全层（分区分域、微隔离、流量加密）
   - 身份安全层（认证、授权、身份治理）
   - 应用安全层（输入验证、会话管理、API安全）
   - 数据安全层（加密、脱敏、访问控制）
   - 运行时安全层（容器安全、运行时防护）
4. **安全组件选型建议**：推荐的安全产品/开源工具
5. **安全基线要求**：配置加固标准
6. **风险残余评估**：方案实施后的残余风险

## 专业准则
- 纵深防御：不依赖单一控制措施，每层都有安全控制
- 最小权限：所有访问控制默认拒绝，按需授权
- 安全左移：在设计阶段解决安全问题的成本是生产阶段的1/100
- 零信任核心三要素：持续验证、最小权限、假设已被入侵
- 云安全责任共担模型：明确CSP和租户的安全边界
- 性能与安全的平衡：安全控制不应导致关键业务延迟超过50ms
- 容器安全四个层面：镜像安全、运行时安全、编排安全、宿主机安全
- 加密选择：传输层TLS 1.2+，存储层AES-256，密钥管理使用HSM或KMS
