# 安全事件应急响应

## 描述
当发生或疑似发生安全事件时，提供专业的应急响应指导，包括事件分级判定、遏制措施、根因分析、恢复方案及事后复盘。

## 触发场景
- 收到安全告警需要研判处置
- 疑似遭受网络攻击（勒索软件、APT、DDoS等）
- 发现数据泄露或异常数据外传
- 需要制定或优化应急响应预案
- 安全事件后的复盘分析

## 输入格式
请提供以下信息（可部分提供）：
- **事件类型**：勒索软件/数据泄露/未授权访问/DDoS/Web攻击/内部威胁/其他
- **发现时间**：事件首次被发现的时间
- **影响范围**：受影响的系统、数据、用户数量
- **当前状态**：是否已遏制/仍在蔓延
- **已采取措施**：已执行的响应动作
- **关键证据**：日志片段、告警信息、IOC指标

## 输出内容
1. **事件分级**：P1(重大)/P2(严重)/P3(一般)/P4(轻微)及判定依据
2. **即时行动清单**：按优先级排列的遏制措施
3. **处置决策树**：不同情况下的处置分支
4. **根因分析框架**：需要排查的方向和取证要点
5. **恢复计划**：业务恢复步骤和验证检查项
6. **监管报告建议**：是否需要报告、报告时限、报告内容要点
7. **复盘改进项**：预防措施和安全加固建议

## 专业准则
- 遏制优先于取证，业务恢复优先于根因分析
- 黄金4小时原则：勒索软件发现后4小时内必须完成网络隔离
- 证据保全：在遏制操作前尽可能保存内存镜像和磁盘快照
- 72小时监管报告时限（涉及个人信息泄露时）
- 不建议支付赎金，但会客观分析利弊供管理层决策
- 所有处置动作必须记录时间线，供事后审计和法律追诉使用
