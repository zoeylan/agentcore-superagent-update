# 安全合规评估与整改

## 描述
针对等保2.0、ISO 27001、数据安全法、个人信息保护法等合规要求，提供差距分析、整改方案制定、测评准备等专业指导。

## 触发场景
- 准备等保测评（定级、备案、整改、测评）
- ISO 27001认证或监督审核准备
- 数据安全合规评估（数据出境、个人信息保护影响评估）
- 监管检查或行业专项检查应对
- 新系统上线前的合规性审查

## 输入格式
请提供以下信息：
- **合规框架**：等保2.0/ISO 27001/个保法/数据安全法/行业规范
- **系统/业务描述**：需要评估的系统或业务场景
- **当前状态**：已有的安全措施和控制手段
- **目标等级**：等保几级/认证范围
- **时间要求**：预期完成时限
- **特殊约束**：预算限制、技术限制等

## 输出内容
1. **合规差距分析**：逐项对照标准要求的差距清单
2. **风险评级**：每项差距的风险等级（高/中/低）
3. **整改方案**：具体技术措施和管理措施
4. **优先级排序**：基于风险等级和实施难度的优先级矩阵
5. **预算估算**：整改所需的大致投入（设备、服务、人力）
6. **时间计划**：分阶段整改路线图
7. **证据准备清单**：测评/审核时需要准备的文档和记录

## 专业准则
- 等保2.0三级系统必须满足的「硬性要求」：双因素认证、安全审计、入侵检测、数据备份
- 高危漏洞在测评前必须修复，中危需有整改计划和时间表
- ISO 27001重点关注PDCA循环和持续改进的证据
- 个保法合规核心：合法性基础、最小必要、告知同意、数据主体权利响应
- 等保测评整改期通常为3-6个月，建议预留缓冲期
- 合规不等于安全，但合规是安全的底线和最低要求
- 引用标准时精确到具体条款编号（如GB/T 22239-2019第8.1.4.1条）
