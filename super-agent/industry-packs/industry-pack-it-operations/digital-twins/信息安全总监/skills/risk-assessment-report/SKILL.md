# 安全风险评估与汇报

## 描述
进行系统化的安全风险评估，生成面向不同受众（管理层/技术团队/监管部门）的风险报告，提供风险量化分析和投资优先级建议。

## 触发场景
- 年度/季度安全风险评估
- 向董事会或管理层汇报安全态势
- 新威胁出现后的影响评估
- 安全预算申请和投资论证
- 第三方/供应链风险评估
- 并购尽职调查中的安全评估

## 输入格式
请提供以下信息：
- **评估范围**：需要评估的系统、业务或组织范围
- **资产清单**：关键资产列表及业务价值
- **已知威胁**：当前面临的主要威胁
- **现有控制**：已部署的安全措施
- **汇报对象**：报告面向的受众（董事会/技术团队/监管）
- **特殊关注点**：管理层特别关心的领域

## 输出内容
1. **风险全景图**：按业务域/技术域的风险热力图
2. **Top N风险清单**：按严重程度排序的关键风险
3. **风险量化分析**：
   - 发生可能性（1-5级）
   - 业务影响（财务/声誉/合规/运营）
   - 风险值 = 可能性 × 影响
   - 年化预期损失(ALE)估算
4. **风险处置建议**：规避/转移/缓解/接受
5. **投资优先级**：安全投入的ROI分析和优先级排序
6. **对标分析**：与行业安全成熟度基准的差距
7. **趋势分析**：风险变化趋势和预测

## 专业准则
- 用业务语言而非技术术语向管理层汇报
- 风险量化优先使用FAIR模型，无法量化时使用5×5风险矩阵
- 安全投资建议的ROI计算：ROSI = (ALE × 缓解率) - 安全投入成本
- 风险接受必须由对应业务owner签字确认，CISO不应独自承担风险接受决策
- Top风险不超过10个，避免信息过载
- 每个风险必须有明确的风险owner和处置时间表
- 供应链风险评估关注：财务稳定性、安全认证、历史事件、技术依赖度
- 行业安全成熟度参考：CMM 1-5级，多数企业在2-3级之间
