# 法规追踪 (Regulation Tracking)

## 概述
法规追踪技能用于持续监控中国网络安全法规体系的变化和更新，评估新法规/新要求对组织当前合规状态的影响，及时触发合规差距评估和整改。确保组织在法规变化后能够快速适应并持续保持合规。

## 使用场景
- 国家网信办/公安部/工信部发布新的安全法规或标准
- 现有法规修订或发布实施细则
- 行业监管部门发布特定行业安全要求
- 需要评估法规变化对组织的合规影响
- 等保标准版本更新（如从2.0到未来的3.0）

## 执行步骤

### 第一步：法规信息监控
1. 监控信息来源：
   - 国家网信办官网（政策法规/标准文件）
   - 公安部网络安全保卫局（等级保护/关键信息基础设施保护）
   - 工业和信息化部（数据安全/个人信息保护行业规范）
   - 国家标准化委员会（GB/T标准发布和修订）
   - 行业协会和安全社区（解读和实施指南）
2. 建立法规库并分类管理：
   - 法律层面：《网络安全法》《数据安全法》《个人信息保护法》
   - 行政法规层面：《关键信息基础设施安全保护条例》等
   - 部门规章和标准：等保系列标准/数据安全标准等
3. 标注每个法规的适用范围、生效日期、处罚条款

### 第二步：变更影响评估
1. 识别法规变更的具体内容（新增要求/修改要求/废止要求）
2. 映射到组织当前的合规基线（哪些控制项受影响）
3. 评估影响等级：
   - 高影响：需要新增重大安全投入/可能导致当前不合规
   - 中影响：需要调整现有控制措施/补充管理制度
   - 低影响：仅需文档更新/微调配置
4. 确定合规过渡期（法规生效日期前需要完成整改）

### 第三步：合规差距更新
1. 将法规变更转化为合规基线的更新（新增/修改控制项）
2. 对新增控制项执行现状评估
3. 识别新产生的差距项
4. 纳入整改计划管理

### 第四步：信息传达与行动驱动
1. 生成法规变更影响评估报告
2. 通知相关利益方（CISO/CTO/法务/业务负责人）
3. 高影响变更启动专项整改项目
4. 更新合规基线版本号和变更日志
5. 同步到合规状态仪表盘

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| regulation_update | object | 是 | 法规变更信息 |
| current_baseline | object | 是 | 当前合规基线 |
| organization_profile | object | 是 | 组织信息（行业/规模/系统等级） |
| existing_controls | array | 否 | 现有安全控制措施 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| impact_assessment | object | 影响评估结果 |
| affected_controls | array | 受影响的控制项 |
| new_gaps | array | 新产生的差距 |
| baseline_updates | object | 基线更新内容 |
| action_required | array | 需要执行的动作 |
| transition_deadline | date | 合规过渡期截止日 |

## 最佳实践
- 法规追踪需要法律和技术双重视角——既要理解法律条文也要能翻译为技术要求
- 关注法规的实施细则和执法案例——它们比法律条文本身更能指导实践
- 行业对标——关注同行业其他组织的合规实践和处罚案例
- 法规解读要保守——宁可过度合规也不要冒不合规的风险
- 建立与监管机构的沟通渠道——在理解模糊条款时获取官方指导

## 约束条件
- 法规追踪信息来源必须为官方渠道（避免不准确的二手解读）
- 影响评估必须在法规发布后7天内完成初步评估
- 高影响变更必须在24小时内上报CISO
- 合规建议不构成法律意见——重大合规决策需要法务团队确认
- 法规库需每周更新一次，确保不遗漏
