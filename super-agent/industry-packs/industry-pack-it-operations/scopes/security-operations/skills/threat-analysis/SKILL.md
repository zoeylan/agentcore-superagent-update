# 威胁分析 (Threat Analysis)

## 概述
威胁分析技能用于对安全告警进行深度研判，确定告警是否代表真实威胁、攻击类型、攻击阶段和潜在影响。基于MITRE ATT&CK框架和Cyber Kill Chain模型进行结构化分析。

## 使用场景
- 收到SIEM/SOC平台推送的安全告警需要研判真伪
- 需要判断告警对应的攻击战术、技术和过程（TTPs）
- 需要评估威胁的严重程度和事件分级（S1-S4）
- 需要关联多条告警判断是否为同一攻击活动的不同阶段

## 执行步骤

### 第一步：告警信息收集
1. 获取告警原始数据（源IP、目标IP、时间戳、告警规则ID、告警详情）
2. 查询源IP的威胁情报信息（是否为已知恶意IP/Tor出口/代理节点）
3. 查询目标资产信息（资产重要性、所属业务、暴露面）
4. 获取告警时间窗口内的关联日志（±30分钟）

### 第二步：攻击模式识别
1. 将告警特征映射到MITRE ATT&CK矩阵的具体技术编号
2. 判断攻击所处Kill Chain阶段（侦察/武器化/投递/利用/安装/命令控制/行动）
3. 检查是否存在关联的前序/后续攻击活动
4. 评估攻击者的技术水平（脚本小子/有组织犯罪/APT）

### 第三步：威胁研判与定级
1. 综合分析结果给出研判结论：确认攻击/可疑活动/误报/合法行为
2. 对确认的攻击进行事件分级（S1-S4）
3. 评估潜在影响范围（受影响系统数量、数据敏感性、业务影响）
4. 输出置信度评分（0-100%）

### 第四步：输出研判报告
1. 编写结构化研判报告（含证据链和分析推理过程）
2. 给出处置建议（是否需要启动应急响应/加入观察清单/关闭告警）
3. 标注需要进一步调查的方向

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alert_id | string | 是 | 告警唯一标识 |
| alert_source | string | 是 | 告警来源设备（WAF/IDS/HIDS/EDR/SIEM） |
| source_ip | string | 是 | 攻击来源IP |
| dest_ip | string | 是 | 目标IP |
| alert_type | string | 是 | 告警类型 |
| raw_log | string | 是 | 原始日志内容 |
| timestamp | datetime | 是 | 告警时间 |
| context_alerts | array | 否 | 时间窗口内的关联告警 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| verdict | enum | 研判结论：confirmed_attack/suspicious/false_positive/legitimate |
| severity_level | enum | 事件等级：S1/S2/S3/S4 |
| attack_type | string | 攻击类型描述 |
| mitre_techniques | array | 对应的ATT&CK技术编号 |
| kill_chain_phase | string | Kill Chain阶段 |
| confidence_score | number | 置信度（0-100） |
| affected_assets | array | 受影响资产列表 |
| recommended_action | string | 建议处置动作 |
| analysis_report | string | 详细分析报告 |

## 最佳实践
- 单条告警不做结论，至少关联15分钟时间窗口的相关事件
- 误报也有价值——记录误报原因用于优化告警规则
- 高危研判（S1/S2）必须在15分钟内完成初步研判
- 遇到不确定的情况，宁可升级（标记为可疑加入观察）也不要轻易标记误报
- APT攻击通常多阶段展开，单看某个阶段可能像正常行为

## 约束条件
- 研判时间SLA：S1级别告警≤15分钟完成研判
- 不得在未保留取证现场的情况下建议清除操作
- 涉及数据泄露的判断需要额外证据支撑，不能仅凭流量异常
- 威胁情报数据需注明来源和时效性
