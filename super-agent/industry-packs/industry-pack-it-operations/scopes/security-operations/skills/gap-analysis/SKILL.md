# 差距分析 (Gap Analysis)

## 概述
差距分析技能用于将合规基线检查的结果转化为可行动的整改计划，分析当前状态与目标要求之间的差距，评估差距的风险等级，并制定优先级排序的整改路线图。

## 使用场景
- 合规基线检查后对不达标项进行差距分析
- 新法规发布后评估现状与新要求的差距
- 等保复评预审中发现的不合规项分析
- 安全事件后暴露的控制缺失分析

## 执行步骤

### 第一步：差距识别与分类
1. 从基线检查结果中提取所有"部分达标"和"不达标"项
2. 按控制域分类（网络安全/主机安全/应用安全/数据安全/管理制度等）
3. 区分技术差距（需要技术措施）和管理差距（需要制度/流程）
4. 标注差距的严重程度（按等保评测扣分权重）
5. 识别新增差距（本次新发现）和遗留差距（历史未关闭）

### 第二步：风险评估
1. 评估每个差距项被利用的可能性
2. 评估被利用后的潜在影响
3. 考虑是否有补偿控制措施降低实际风险
4. 综合计算风险等级（高/中/低）
5. 关联已知的安全事件（该差距是否曾导致安全问题）

### 第三步：整改方案制定
1. 为每个差距项制定具体的整改方案：
   - 技术措施：需要部署/配置/升级什么
   - 管理措施：需要制定/修订什么制度和流程
   - 人员措施：需要培训/配备什么能力
2. 评估整改成本（人力/资金/时间）
3. 评估整改对业务的影响
4. 确定整改Owner（责任部门/个人）
5. 设定整改Deadline（基于风险等级和等保复评时间倒推）

### 第四步：整改路线图
1. 按优先级排列整改项（高风险+低成本优先）
2. 考虑整改项之间的依赖关系
3. 分阶段规划（短期应急<30天/中期整改<90天/长期建设<180天）
4. 设定里程碑和验收标准
5. 预留等保复评前的缓冲时间（≥30天）

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| baseline_results | object | 是 | 合规基线检查结果 |
| target_standard | string | 是 | 目标合规标准（如：等保2.0三级） |
| current_controls | array | 否 | 现有安全控制措施清单 |
| budget_constraints | object | 否 | 预算约束 |
| timeline_constraints | object | 否 | 时间约束（如等保复评时间） |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| gaps | array | 差距清单（含分类/严重度/风险等级） |
| remediation_plans | array | 整改方案（含Owner/Deadline/成本） |
| roadmap | object | 分阶段整改路线图 |
| risk_summary | object | 风险汇总（高/中/低数量分布） |
| effort_estimation | object | 整改工作量估算 |

## 最佳实践
- 差距分析不是找茬——重点是帮助组织了解风险并制定可行的改进路径
- 整改方案要务实——考虑组织的实际资源和能力
- 高风险差距即使修复成本高也必须优先处置
- 补偿控制措施可以临时降低风险，但不能替代最终整改
- 整改路线图需要定期回顾和调整（每月Review）

## 约束条件
- 差距分析报告需要安全负责人和业务负责人共同签字确认
- 高风险差距的整改Deadline不得超过30天
- 等保复评前所有高风险和中风险差距必须关闭
- 整改方案的实施需要通过变更管理流程
- 遗留超过6个月的差距项需要CISO特批延期或接受风险
