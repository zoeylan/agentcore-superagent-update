# 告警关联分析 (Alert Correlation)

## 概述
告警关联分析技能用于从海量安全告警中识别有意义的模式和关联关系，将离散的告警事件聚合为完整的攻击场景，大幅降低误报率并提升威胁发现能力。面对WAF/IDS日均数万条告警，有效告警可能不足1%的现实，本技能是安全运营效率的关键保障。

## 使用场景
- SIEM平台产生大量告警需要降噪和聚合
- 需要从多源告警中识别同一攻击活动的不同表现
- 需要区分告警风暴（级联告警）和多次独立攻击
- 需要评估告警规则的有效性并优化规则库

## 执行步骤

### 第一步：告警预处理
1. 告警去重——相同来源、相同规则、相同目标在5分钟内的重复告警合并
2. 告警富化——补充资产信息（所属业务/重要性级别）、威胁情报（IP信誉）
3. 告警标准化——统一不同安全设备的告警格式到CEF/LEEF/Syslog标准
4. 过滤已知白名单（已确认的合法扫描/运维操作/业务行为）

### 第二步：时间维度关联
1. 基于时间窗口（5/15/60分钟）聚合同一来源的告警
2. 识别告警爆发模式（短时间内告警数量突增）
3. 检测慢速攻击模式（分散在长时间周期内的低频告警）
4. 关联基础设施事件时间线（网络抖动/服务重启等与告警的时间吻合）

### 第三步：拓扑维度关联
1. 基于网络拓扑关联同一攻击路径上的告警（外网→DMZ→内网）
2. 识别横向移动模式（同一源IP触发多个内网系统的告警）
3. 基于服务调用链关联上下游系统的告警
4. 关联同一攻击者的多个跳板IP/C2域名

### 第四步：模式匹配与场景重建
1. 匹配已知攻击模式（暴力破解→成功登录→提权→数据外传）
2. 与ATT&CK攻击链进行对齐，识别攻击进展阶段
3. 计算关联告警的整体威胁评分
4. 输出攻击场景描述和关联告警列表

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| alerts | array | 是 | 待分析的告警列表 |
| time_window | string | 否 | 分析时间窗口（默认1小时） |
| correlation_rules | array | 否 | 自定义关联规则 |
| topology_data | object | 否 | 网络拓扑信息 |
| historical_patterns | array | 否 | 历史攻击模式库 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| correlated_groups | array | 关联告警组 |
| attack_scenarios | array | 识别的攻击场景 |
| noise_alerts | array | 识别的噪声告警 |
| correlation_confidence | number | 关联置信度 |
| reduction_rate | number | 告警降噪率 |
| rule_effectiveness | object | 规则有效性评估 |

## 最佳实践
- 关联分析不是简单的告警合并，需要理解攻击逻辑
- 保持对低频但高价值告警的敏感度（APT特征）
- 定期回顾误报和漏报案例，持续优化关联规则
- 告警降噪目标：研判后误报率≤10%
- 保留原始告警数据，即使被标记为噪声（可能后续关联分析有用）

## 约束条件
- 降噪不等于删除——所有原始告警必须保留供审计
- 关联规则更新必须经过安全团队评审
- 白名单维护需要定期复查（避免攻击者利用白名单绕过检测）
- 关联分析超时时间≤5分钟（确保实时性）
