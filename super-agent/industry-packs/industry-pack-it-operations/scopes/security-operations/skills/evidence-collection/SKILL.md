# 合规证据收集 (Evidence Collection)

## 概述
合规证据收集技能用于系统性地采集和组织等保合规评测所需的证据材料，确保在年度复评中能够提供完整、有效、时效性合格的合规证据，支撑各控制项的达标结论。

## 使用场景
- 年度等保复评前的证据材料准备
- 监管检查要求提供合规证据
- 合规自查中需要证据支撑检查结论
- 新系统纳入等保范围需要建立证据基线

## 执行步骤

### 第一步：证据需求梳理
1. 对照等保评测项清单，确定每个控制项需要的证据类型：
   - 文档类：安全策略/管理制度/操作规程/应急预案
   - 配置类：系统配置截图/安全设备策略导出/网络拓扑图
   - 记录类：审计日志样本/操作记录/变更记录/巡检报告
   - 测试类：渗透测试报告/漏洞扫描报告/备份恢复测试记录
   - 培训类：安全培训签到表/考试成绩/培训课件
2. 确定证据的时效性要求（多数需要评测年度内的）
3. 标注可复用的历史证据和需要重新采集的证据

### 第二步：证据采集执行
1. **自动化采集**（技术类证据）：
   - 系统配置导出（密码策略/审计策略/访问控制配置）
   - 安全设备策略快照（防火墙/IDS/WAF/杀毒软件）
   - 日志样本提取（时间范围内的审计日志/操作日志）
   - 监控大盘截图（证明监控覆盖和告警机制有效）
2. **人工收集**（管理类证据）：
   - 制度文档版本确认和有效性验证
   - 审批记录和签字确认
   - 会议纪要和培训记录
   - 人员资质证明和授权文件

### 第三步：证据质量审核
1. 检查证据与控制项的对应关系是否正确
2. 验证证据的时效性（评测年度内）
3. 确认证据内容能够充分支撑达标结论
4. 检查证据是否存在矛盾（如制度要求A但配置显示B）
5. 对敏感信息进行适当脱敏（保留证明力的同时保护隐私）

### 第四步：证据组织与归档
1. 按控制域建立证据目录结构
2. 为每份证据编号并建立索引
3. 生成证据清单（含编号/名称/类型/采集时间/对应控制项）
4. 存储到安全的证据库中（访问控制+完整性保护）
5. 标注证据有效期和下次更新提醒

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| assessment_scope | object | 是 | 评测范围和系统清单 |
| control_items | array | 是 | 需要提供证据的控制项清单 |
| evidence_templates | array | 否 | 证据模板（上次评测的材料结构） |
| collection_deadline | date | 是 | 证据收集截止时间 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| evidence_inventory | array | 证据清单索引 |
| evidence_status | object | 各控制域证据就绪状态 |
| missing_evidence | array | 缺失证据列表 |
| quality_issues | array | 证据质量问题 |
| collection_progress | number | 收集完成度百分比 |

## 最佳实践
- 证据收集不应该是年度突击行为——建立日常证据积累机制
- 自动化采集的证据附带时间戳和系统签名，证明力更强
- 准备证据时考虑评测老师的视角——证据要能自说明
- 保留多个时间点的证据，证明合规是持续的而非临时的
- 证据目录结构建议参考等保评测指南的章节结构

## 约束条件
- 证据中不得包含真实密码、密钥等敏感信息
- 证据采集过程不得影响生产系统
- 所有证据保留≥3年（等保评测周期要求）
- 证据库的访问权限限制在安全团队和审计团队
- 评测前30天必须完成全量证据收集
