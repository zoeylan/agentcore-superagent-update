# 异常行为检测 (Anomaly Detection)

## 概述
异常行为检测技能用于分析用户和实体的行为日志，通过建立行为基线识别偏离正常模式的异常活动，是发现内部威胁和已被攻陷账号的关键能力（UEBA - User and Entity Behavior Analytics）。

## 使用场景
- 日常持续性的用户行为监控
- 检测可能被攻陷的合法账号（账号接管）
- 内部威胁检测（数据泄露前兆/恶意操作）
- 特权滥用识别
- 安全事件后的横向移动检测

## 执行步骤

### 第一步：行为基线建立
1. 采集30天以上的历史行为数据建立基线
2. 为每个用户/账号画像：常用登录时间/源IP/访问系统/操作频率
3. 为每个系统建立正常访问模式：典型QPS/数据量/操作类型分布
4. 考虑业务日历因素（月末/季末/大促期间的基线偏移）
5. 基线需定期更新（每7天重新计算）

### 第二步：实时行为分析
1. **时间异常**：非工作时间（22:00-08:00）的操作，特别是特权操作
2. **地理异常**：同一账号短时间内从相距遥远的地点登录（impossible travel）
3. **频率异常**：操作频率显著超出历史基线（如数据查询量突增10倍）
4. **模式异常**：访问了历史上从未访问过的系统或数据
5. **序列异常**：操作序列符合已知攻击模式（侦察→提权→数据外传）

### 第三步：风险评分
1. 单一异常行为给予基础风险分
2. 多个异常行为在时间窗口内叠加，风险分倍增
3. 考虑账号权限级别（特权账号的异常权重更高）
4. 考虑目标数据敏感性（访问敏感数据的异常权重更高）
5. 综合风险分超阈值（≥70分）触发告警

### 第四步：告警与响应
1. 低风险异常（30-50分）：记录并加入观察清单
2. 中风险异常（50-70分）：通知账号Owner确认操作合法性
3. 高风险异常（≥70分）：立即推送给安全事件响应Agent进行威胁研判
4. 极高风险（≥90分且涉及数据外传）：自动触发账号临时锁定

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| behavior_logs | array | 是 | 行为日志数据 |
| user_baseline | object | 否 | 用户行为基线（首次使用时为空） |
| system_baseline | object | 否 | 系统访问基线 |
| privilege_info | object | 否 | 账号权限信息 |
| data_sensitivity | object | 否 | 数据敏感性分级 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| anomalies | array | 检测到的异常行为列表 |
| risk_scores | object | 风险评分详情 |
| alert_level | enum | 告警级别 |
| recommended_action | string | 建议动作 |
| baseline_updates | object | 基线更新建议 |

## 最佳实践
- 异常不等于恶意——可能是业务变更导致的行为变化，需要验证
- 基线需要持续学习和适应，但不能被攻击者"训练"（慢速投毒）
- 关注"低且慢"的异常——APT攻击者会刻意保持低调
- 内部威胁检测需要结合HR数据（即将离职/绩效差评等情境信息）
- False Positive的处理反馈必须用于优化检测模型

## 约束条件
- 行为分析需遵守员工隐私保护规定（明确告知监控范围）
- 自动账号锁定仅限极高风险场景，避免影响正常业务
- 行为基线数据属于敏感信息，需要访问控制
- 检测模型的调整需要安全团队评审
- 分析结果不得用于非安全目的（如员工考勤监控）
