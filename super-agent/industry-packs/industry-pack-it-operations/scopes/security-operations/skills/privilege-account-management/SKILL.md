# 特权账号管理 (Privilege Account Management)

## 概述
特权账号管理技能专注于对root/administrator/DBA等高权限账号的生命周期管控，包括账号清单维护、密码轮换执行、使用监控和异常检测。特权账号是攻击者的首要目标，也是内部威胁的最大风险点。

## 使用场景
- 特权账号90天密码轮换到期需要强制执行
- 新增/删除特权账号需要审批和记录
- 临时提权操作需要管控和自动回收
- 离职/调岗人员的特权账号需要及时回收

## 执行步骤

### 第一步：特权账号清单维护
1. 枚举所有系统中的特权账号（root/admin/DBA/超管/服务账号）
2. 确认每个特权账号的Owner（责任人）和用途说明
3. 验证特权账号是否在PAM系统中注册和管控
4. 标注共享特权账号（需要计划整改为个人特权账号）
5. 定期对比实际系统与清单的一致性（发现未记录的特权账号）

### 第二步：密码策略执行
1. 检查每个特权账号的最后密码修改时间
2. 对于超过90天未修改的账号生成轮换提醒
3. 超过提醒期仍未修改的执行强制轮换
4. 验证新密码符合复杂度要求（≥16位/大小写+数字+特殊字符）
5. 确认密码未在历史12次使用记录中重复

### 第三步：使用监控与异常检测
1. 监控特权账号的登录时间、来源IP和操作内容
2. 标记非工作时间的特权操作（需要确认合法性）
3. 检测同一特权账号的并发登录（可能被共享使用）
4. 监控特权操作中的高危行为（批量删除/全量导出/配置修改）
5. 异常发现推送给安全事件响应Agent研判

### 第四步：生命周期管理
1. 新建特权账号——验证审批流程、记录创建原因和有效期
2. 临时提权——设置自动过期时间（最长不超过72小时）
3. 离职回收——员工离职后24小时内停用所有关联特权账号
4. 调岗处理——旧部门特权权限在调岗生效7天内回收
5. 定期清理——对90天内未使用的特权账号进行确认和清理

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| system_scope | array | 是 | 需要管理的系统范围 |
| pam_data | object | 否 | PAM系统中的账号数据 |
| hr_events | array | 否 | HR人事变动事件（离职/调岗） |
| rotation_policy | object | 否 | 密码轮换策略配置 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| account_inventory | array | 特权账号完整清单 |
| rotation_status | object | 密码轮换状态（过期/即将过期/合规） |
| anomaly_alerts | array | 异常使用告警 |
| lifecycle_actions | array | 生命周期操作记录 |
| compliance_dashboard | object | 合规状态面板 |

## 最佳实践
- 特权账号密码轮换合规率目标为100%，无例外
- 使用PAM系统实现特权账号的集中管控和审计
- 推动"特权账号个人化"——消除共享特权账号
- 临时提权应成为常态，永久特权应成为例外
- 特权账号的操作录屏是重要的审计证据

## 约束条件
- 离职人员账号必须在24小时内完成停用，0容忍
- 密码轮换操作本身需要审批记录
- 特权账号清单属于高度机密信息
- 强制轮换前需确认不影响关联的自动化任务和计划作业
- 所有特权操作日志保留≥6个月
