# 风险优先级排序 (Risk Prioritization)

## 概述
风险优先级排序技能用于在有限的安全资源下，确定漏洞修复和安全措施实施的最优顺序。综合考虑威胁严重性、资产价值、暴露程度和修复成本，为安全团队提供明确的行动优先级指导。

## 使用场景
- 大量漏洞同时发现时确定修复顺序
- 安全预算有限时决定优先投入的安全措施
- 需要向管理层报告最关键的安全风险
- 季度安全工作规划需要确定优先级

## 执行步骤

### 第一步：风险因子采集
1. 收集漏洞/威胁的技术严重性指标（CVSS/CWSS评分）
2. 获取资产重要性分级信息（从CMDB查询）
3. 确认暴露面信息（面向互联网/内网/隔离环境）
4. 查询威胁情报中的在野利用情况
5. 评估修复复杂度和业务影响

### 第二步：风险量化计算
1. 应用多因子风险评分模型：
   - 风险值 = 威胁严重性 × 可利用性 × 资产价值 × 暴露度 / 现有控制措施有效性
2. 计算每个风险项的年化损失预期（ALE = SLE × ARO）
3. 考虑风险的时间紧迫性（在野利用的窗口期）
4. 标注风险之间的关联关系（修复一个可能降低另一个的风险）

### 第三步：优先级排列
1. 按风险评分从高到低排序
2. 考虑修复依赖关系（A必须在B之前修复）
3. 考虑资源约束（同一团队不能同时处理过多高优先级任务）
4. 识别Quick Wins（低成本高收益的修复项）
5. 输出优先级排列结果及理由说明

### 第四步：优先级沟通
1. 为不同受众生成不同粒度的优先级报告
2. 管理层：Top 5风险及业务影响量化
3. 技术团队：完整优先级清单及修复指导
4. 审计合规：合规相关风险及整改时限

## 输入规格
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| risk_items | array | 是 | 待排序的风险项列表 |
| asset_classification | object | 是 | 资产分级信息 |
| threat_intelligence | object | 否 | 威胁情报数据 |
| resource_constraints | object | 否 | 资源约束条件 |
| existing_controls | array | 否 | 现有安全控制措施 |

## 输出规格
| 字段 | 类型 | 说明 |
|------|------|------|
| prioritized_list | array | 优先级排序结果 |
| risk_scores | object | 各项风险评分详情 |
| quick_wins | array | 低成本高收益项 |
| dependency_graph | object | 修复依赖关系图 |
| executive_summary | string | 管理层摘要 |

## 最佳实践
- 优先级不是一成不变的——新的威胁情报可能改变优先级排序
- 不要仅看CVSS评分，业务上下文同样重要
- Quick Wins虽然不是最高风险，但快速解决可以显著降低整体暴露面
- 优先级沟通要用业务语言而非纯技术语言
- 定期回顾优先级模型的准确性（修复后是否真的降低了风险）

## 约束条件
- 高危风险（CVSS≥9.0且在野利用）无论资源约束如何都必须在24小时内处置
- 优先级评估必须基于最新的资产清单（≤7天内更新）
- 风险评分模型的权重调整需要安全团队集体评审
- 优先级变更需要记录变更原因和时间
