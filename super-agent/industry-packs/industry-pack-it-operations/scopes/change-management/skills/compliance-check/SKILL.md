# 合规性检查 (Compliance Check)

## 概述

合规性检查是对变更审批流程中各环节进行规范性和合规性验证的技能。确保每一个变更都按照既定的管理制度执行，审批链完整有效、时间窗口合规、必要的检查点无遗漏。这是变更管理体系合规审计的基础——在等保2.0和内部审计中，变更管理流程的合规性是重点检查项。

## 适用场景

- 变更审批流程各节点的合规校验
- 紧急变更的事后补审合规确认
- 月度/季度变更合规审计报告生成
- 等保2.0相关变更管理控制项检查

## 执行步骤

### 步骤1：审批链完整性验证
- 确认变更的审批流程符合其风险等级要求
- 验证每级审批人的身份和职级是否匹配（VP审批确实由VP级完成）
- 检查审批是否在有效期内（审批通过后超过7天未执行需重新审批）
- 确认没有审批环节被跳过或绕过

### 步骤2：时间合规检查
- 审批时效是否在规定范围内完成
- 执行时间是否在审批通过的窗口内
- 是否遵守了周五/节假日禁令
- 高风险变更是否在维护窗口内执行

### 步骤3：必要文档完整性
- 变更申请表各必填字段是否完整（完整性≥95%）
- 回滚方案是否已通过验证
- 影响面评估是否已完成
- 相关方通知记录是否存在

### 步骤4：紧急变更补审验证
- 紧急变更是否在24小时内完成补审
- 补审材料是否包含紧急原因说明
- 紧急放行人是否具备放行权限
- 是否有关联的故障工单或安全事件作为紧急依据

### 步骤5：违规记录与处理
- 对发现的合规违规进行分类：严重违规/一般违规/轻微违规
- 严重违规：绕过审批执行变更、伪造审批记录
- 一般违规：审批超时未升级、紧急变更超时未补审
- 轻微违规：文档不完整、通知延迟

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| change_id | string | 是 | 变更工单标识 |
| check_type | enum | 是 | single(单变更)/batch(批量审计)/period(周期报告) |
| period | string | 条件必填 | 审计周期(batch/period时必填) |

## 最佳实践

1. **实时检查优于事后审计**：在流程流转的每个环节嵌入实时合规检查，而非仅靠事后审计发现问题
2. **违规零容忍**：严重违规必须触发调查和整改，不可因"结果OK"而忽略过程违规
3. **自动化取证**：所有审批操作自动记录时间戳、操作人、IP等信息，确保审计可追溯
4. **等保对标**：定期将合规检查结果与等保2.0的变更管理相关要求对标
5. **趋势分析**：对违规的类型和频率进行趋势分析，识别系统性合规风险

## 约束与限制

- 合规检查基于已定义的规则集，无法检测规则未覆盖的合规风险
- 审批人身份验证依赖IAM系统的准确性
- 紧急变更的事后合规确认可能因信息不完整而存在盲区
- 合规检查不评价变更本身的技术合理性
