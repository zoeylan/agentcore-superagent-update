# 审批路由 (Approval Routing)

## 概述

审批路由是根据变更的风险等级、类型和业务场景，智能化地将变更申请分发到正确的审批流程的技能。高效的审批路由既要确保风险得到恰当级别的管控，又要避免过度审批造成业务交付瓶颈。核心逻辑是：低风险高频变更走快速通道（标准变更预授权），常规变更走标准CAB审批，高风险变更走强化审批，紧急变更走先执行后补审的绿色通道。

## 适用场景

- 变更评估完成后的审批流程确定
- 紧急变更的快速通道判定
- 标准变更预授权目录匹配
- 审批流程的动态调整（如CAB成员缺席时的替代审批人）

## 执行步骤

### 步骤1：变更类型与紧急程度判定
- 确认变更的风险等级（标准/普通/高风险）
- 确认是否为紧急变更（关联故障修复/安全漏洞修复/业务紧急需求）
- 确认紧急变更的必要性证明（非紧急变更不得走紧急通道）

### 步骤2：审批路径确定

| 风险等级 | 审批路径 | 时效要求 | 执行限制 |
|---------|---------|---------|---------|
| 标准变更 | 预授权目录自动匹配→自动审批 | 即时 | 无特殊限制 |
| 普通变更 | 变更负责人→CAB轮值审批人 | ≤24h(工作日) | 非周五/节假日前 |
| 高风险变更 | 变更负责人→CAB→VP审批 | ≤48h | 仅限维护窗口 |
| 紧急变更 | 值班经理放行→24h内补审(CAB+) | 即时放行 | 无时间限制 |

### 步骤3：标准变更预授权匹配
- 将变更内容与预授权目录逐条匹配
- 匹配条件：变更对象、变更操作、参数范围均在预授权范围内
- 部分匹配不算匹配——必须完全落入预授权范围
- 匹配成功：自动通过，记录匹配的预授权条目编号
- 匹配失败：进入常规审批流程

### 步骤4：审批人确定与通知
- 查询当前CAB轮值表确定审批人
- 确认审批人在线/可用（不在休假/请假状态）
- 如审批人不可用，自动路由到backup审批人
- 发送审批通知（含变更摘要、风险评估、建议审批意见）

### 步骤5：时效监控设置
- 设置审批超时告警（时效的50%时预警，100%时升级）
- 标记审批截止时间
- 超时升级路径：CAB轮值人→CAB组长→运维总监

## 输入规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| change_id | string | 是 | 变更工单标识 |
| risk_level | enum | 是 | 风险等级 |
| is_emergency | boolean | 是 | 是否紧急变更 |
| change_target | string | 是 | 变更对象 |
| change_operation | string | 是 | 变更操作类型 |
| emergency_reason | string | 条件必填 | 紧急原因(紧急变更时必填) |

## 最佳实践

1. **预授权目录定期更新**：每月评审预授权目录，纳入新的成熟标准变更，移除不再适用的条目
2. **审批人轮值透明**：CAB轮值表提前一周发布，确保审批人有准备时间
3. **紧急变更不滥用**：紧急变更占比应控制在≤10%，超过应分析计划性不足的原因
4. **审批意见模板化**：为审批人提供结构化的审批意见模板，降低审批门槛提高效率
5. **自动提醒机制**：审批到达时立即通知，50%时效时提醒，超时自动升级

## 约束与限制

- 标准变更自动审批依赖预授权目录的准确性和时效性
- 紧急变更放行权限严格限制在值班经理及以上级别
- 审批路由不处理审批内容的合理性判断（由审批人决定）
- CAB成员全部不可用时需人工介入确定替代审批人
