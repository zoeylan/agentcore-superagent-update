# 容量评估报告生成

## 技能概述

本技能负责将容量趋势预测和资源优化分析结果整合为月度容量评估报告。报告面向运维管理层和架构团队，为扩缩容决策、预算规划和资源采购提供数据依据。

## 适用场景

- 月度容量评估报告（每月输出）
- 大促前容量评估专项报告
- 季度资源利用率总结与优化报告
- 预算规划参考报告（年度/半年度）

## 执行步骤

### 步骤1：数据汇总
1. 汇总当月资源使用概况（各资源类型平均利用率/峰值/趋势方向）
2. 整合趋势预测模型输出（预计触达水位线时间、增长率）
3. 整合资源优化分析结果（浪费项、扩容需求项）
4. 收集业务增长数据和近期业务计划

### 步骤2：现状概览编写
1. 按资源类型（计算/存储/网络/数据库）输出利用率概况
2. 按业务线维度输出资源消耗排名
3. 对比上月数据，标注显著变化（增长>10%或下降>10%）
4. 输出资源健康度评分（基于利用率合理性）

### 步骤3：趋势预测章节
1. 呈现各核心资源的90天预测曲线描述
2. 标注预计触达预警线的资源和时间点
3. 按紧急程度排序预警项（30天内>60天内>90天内）
4. 附带置信度说明和假设条件

### 步骤4：扩容/缩容建议
1. 对30天内需扩容的资源生成具体扩容方案（规格/数量/时间）
2. 附带扩容成本估算和多方案对比
3. 对缩容建议汇总潜在节省金额
4. 给出优先级排序和建议执行时间表

### 步骤5：成本分析
1. 当前月度基础设施总成本
2. 预估下月成本变化（扩容增加+缩容节省）
3. 成本趋势预测（未来3个月）
4. 对比业务增长率与成本增长率的匹配度

### 步骤6：报告定稿与分发
1. 按标准报告模板格式化
2. 生成关键结论和决策建议摘要
3. 标注需要管理层决策的事项
4. 发送至相关干系人

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| trend_models | array | 是 | 容量趋势预测模型输出 |
| optimization_results | object | 是 | 资源优化分析结果 |
| cost_data | object | 是 | 当前成本数据 |
| business_plans | object | 否 | 近期业务计划/大促日历 |
| previous_report | object | 否 | 上月报告（用于对比） |

## 输出规格


## 最佳实践

1. **数据驱动**：所有结论和建议必须有数据支撑，避免主观判断
2. **决策导向**：报告尾部明确列出需要管理层决策的事项
3. **风险透明**：对不扩容的后果给出明确的风险描述
4. **可执行性**：扩容建议需具体到规格和时间，可直接转为变更申请
5. **趋势对比**：与历史报告对比，展示预测准确性和趋势一致性

## 约束条件

- 报告产出时间：每月第一个工作周内完成
- 成本数据必须基于最新定价，不使用过期价格
- 扩容建议必须包含至少2个备选方案供决策
- 缩容建议需标注"建议"状态，不可表述为确定性决策
