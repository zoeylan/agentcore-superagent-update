# 巡检报告生成

## 技能概述

本技能负责将巡检数据采集结果和异常分析结论整合为结构化、可行动的巡检报告。报告面向运维团队和业务负责人，需在每日09:00前发送，是运维团队了解系统健康状态的核心信息载体。

## 适用场景

- 每日自动巡检报告（标准格式，每日09:00前发送）
- 专项巡检报告（大促前/变更后的针对性评估报告）
- 深度巡检报告（季度全面健康评估报告）
- 异常升级通知（严重异常的即时通知）

## 执行步骤

### 步骤1：数据汇总与分类
1. 汇总巡检原始数据和异常分析结果
2. 按异常严重程度分类：告警项(Critical/Warning)/风险项(Trending)/优化建议项/正常项
3. 按优先级排序（严重程度×影响范围×紧急程度）

### 步骤2：执行摘要生成
1. 生成一句话健康度总评（如"整体健康，2项需立即关注"）
2. 计算系统整体健康评分（0-100分制）
3. 标注与前一日的对比变化（新增异常/已恢复项/持续恶化项）

### 步骤3：告警项详细描述
1. 对每个告警项生成完整的问题描述卡片
2. 包含：指标名称、当前值vs阈值、持续时间、影响范围
3. 附带建议操作和责任人/责任团队
4. 标注是否已自动推送至故障响应Scope

### 步骤4：风险项与趋势分析
1. 对趋势恶化项生成趋势图描述（文字描述7天/30天趋势）
2. 给出预计触及阈值的时间点
3. 标注建议的跟进动作和跟进时间节点

### 步骤5：优化建议项
1. 对资源利用不合理的项给出优化建议
2. 对巡检有效性问题（长期无异常项/频繁告警项）给出调整建议
3. 标注优化的优先级和预期收益

### 步骤6：报告格式化与发送
1. 按标准模板格式化报告（支持Markdown/HTML/PDF）
2. 根据接收人角色调整详略程度（运维团队：完整版；业务负责人：摘要版）
3. 附加巡检统计元数据（完成率、采集耗时、异常数对比）
4. 通过配置的通知渠道发送（邮件/企业微信/钉钉/飞书）

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inspection_data | object | 是 | 巡检原始采集数据 |
| analysis_results | object | 是 | 异常检测分析结果 |
| report_type | string | 是 | 报告类型：daily/special/deep |
| recipients | array | 是 | 报告接收人列表及角色 |
| previous_report | object | 否 | 前次报告（用于对比） |

## 输出规格


## 最佳实践

1. **金字塔原则**：最重要的信息放在最前面，详细信息按需展开
2. **可行动性**：每个异常项必须附带明确的建议操作，避免"仅告知不指导"
3. **对比视角**：与前一日/上周同期对比，突出变化趋势
4. **分层呈现**：为不同角色提供不同详略版本
5. **准时发送**：严格保证09:00前发送，延迟需要生成延迟原因说明

## 约束条件

- 报告生成时间窗口：07:00（初稿生成）至09:00（最终版发送）
- 报告必须包含巡检完成率，不允许隐藏采集失败项
- 告警项数量超过10项时需增加"重点关注Top3"摘要
- 摘要版报告不超过一屏（约500字）
