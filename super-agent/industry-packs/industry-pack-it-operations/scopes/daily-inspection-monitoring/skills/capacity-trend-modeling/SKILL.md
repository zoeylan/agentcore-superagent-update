# 容量趋势建模

## 技能概述

本技能负责基于历史资源使用数据和业务增长预测，构建容量趋势预测模型。通过线性回归、季节性分解和业务因子叠加，预测资源达到容量水位线的时间点，为扩缩容决策提供数据支撑。

## 适用场景

- 月度容量评估中的趋势预测
- 大促前的峰值资源需求预估
- 新业务上线的资源需求建模
- 扩容后的效果验证与模型校准

## 执行步骤

### 步骤1：历史数据采集与预处理
1. 采集目标资源近90天的使用率时序数据（1小时粒度）
2. 数据清洗：处理缺失值（插值法）、剔除异常点（3σ原则）
3. 数据对齐：确保所有资源指标时间轴统一
4. 标记已知事件点（大促、故障、扩容操作）用于分析排除

### 步骤2：趋势分解与建模
1. **趋势分解**：使用STL分解将时序数据拆分为趋势项+季节项+残差项
2. **线性回归**：对趋势项进行线性拟合，计算增长斜率（日增长率/周增长率）
3. **季节性建模**：识别周期性模式（日峰/周峰/月底高峰）
4. **拟合验证**：使用后30天数据验证模型预测精度，R²≥0.7方可采用

### 步骤3：业务因子叠加
1. 获取业务增长计划（新用户增长率、新功能上线计划）
2. 获取大促/活动日历（流量预估倍数）
3. 对趋势预测叠加业务增长因子：`预测值 = 基础趋势 × (1 + 业务增长率) + 季节性波动 + 大促峰值`
4. 对大促场景使用历史大促期间的流量放大系数

### 步骤4：容量预测与预警计算
1. 基于模型预测未来90天的资源使用率曲线
2. 计算资源使用率触达各水位线的预计日期：
   - 黄色预警线（70% CPU / 80% Memory / 85% Disk）
   - 红色告警线（80% CPU / 90% Memory / 90% Disk）
   - 极限线（95% - 留5%安全余量）
3. 输出预警时间窗口：距离触达预警线的剩余天数

### 步骤5：置信区间与风险标注
1. 计算预测的置信区间（95%置信度）
2. 标注预测的不确定性因素（业务增长不确定、是否有大促等）
3. 给出乐观/基准/悲观三个预测场景

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resource_type | string | 是 | 资源类型：cpu/memory/disk/network |
| target_instances | array | 是 | 目标实例/集群列表 |
| history_days | integer | 否 | 历史数据天数（默认90） |
| business_growth_rate | float | 否 | 业务增长率预估（月度） |
| event_calendar | array | 否 | 大促/活动日历 |

## 最佳实践

1. **模型校准**：每月对比预测值与实际值，计算预测偏差，迭代优化模型参数
2. **多维度建模**：CPU/内存/磁盘/网络应分别建模，各自增长模式不同
3. **集群视角**：除了单机维度，还需从集群整体维度进行容量评估
4. **安全余量**：预测值应预留20%安全余量，避免精确到刚好够用
5. **大促特殊处理**：大促期间的数据不纳入常规趋势建模，单独作为峰值参考

## 约束条件

- 预测准确率目标：±20%以内
- 最短预测周期：7天（少于7天的预测不可靠）
- 数据最低要求：至少30天历史数据才能建模
- 大促预测需结合压测数据，纯模型预测不作为唯一依据
