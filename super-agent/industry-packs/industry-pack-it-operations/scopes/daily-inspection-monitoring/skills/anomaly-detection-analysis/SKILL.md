# 异常检测与关联分析

## 技能概述

本技能负责对巡检采集的原始数据进行深度异常检测，包括阈值判定、趋势恶化识别、跨层指标关联分析，并对异常项进行严重程度分级和根因初步推断。这是将原始指标数据转化为可行动洞察的关键分析环节。

## 适用场景

- 每日巡检数据的异常检测和分级
- 跨层指标异常的关联性分析（如应用慢→数据库慢查询→磁盘I/O高）
- 趋势恶化的早期预警（指标虽未超阈值但增长趋势异常）
- 历史模式匹配（当前异常模式是否与历史故障模式相似）

## 执行步骤

### 步骤1：阈值异常检测
1. 对每个指标执行静态阈值判定（与预设告警阈值比对）
2. 对超阈值项判定持续时间是否满足告警条件（如CPU>80%需持续5分钟）
3. 区分瞬时突刺（spike）和持续异常（sustained anomaly）

### 步骤2：动态基线异常检测
1. 计算每个指标的7天同期基线（同一时间段的平均值±标准差）
2. 计算30天趋势基线（移动平均线）
3. 标记偏离7天基线超过2个标准差的指标为"异常"
4. 标记偏离30天趋势超过30%的指标为"趋势恶化"

### 步骤3：趋势恶化分析
1. 对关键资源指标（CPU/内存/磁盘）进行线性回归分析
2. 计算增长斜率，预测到达阈值的剩余天数
3. 对30天内将触及阈值的指标标记为"趋势恶化-紧急"
4. 对60天内将触及阈值的指标标记为"趋势恶化-关注"

### 步骤4：跨层关联分析
1. 当应用层出现异常时，向下追溯检查数据库层和基础设施层
2. 当数据库层出现异常时，检查是否存在磁盘I/O或网络问题
3. 构建异常传播链（如：磁盘IOPS饱和→数据库慢查询增加→应用P99升高）
4. 对关联异常标记为同一问题组，避免重复告警

### 步骤5：异常分级与路由决策
1. **严重异常（Critical）**：核心服务指标超出阈值且持续时间满足条件 → 立即推送故障响应Scope
2. **一般异常（Warning）**：非核心服务异常或核心服务的次要指标异常 → 标记为风险项纳入报告
3. **趋势恶化（Trending）**：指标未超阈值但恶化趋势明显 → 标记为关注项
4. **安全异常（Security）**：异常登录/权限变更/可疑进程 → 同步安全运维Scope

### 步骤6：分析结果输出
1. 输出每个异常项的完整分析结论
2. 对关联异常生成关联图谱描述
3. 对每个异常项生成建议操作（扩容/优化/排查/观察）

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inspection_results | object | 是 | 巡检采集的原始数据结果 |
| historical_baselines | object | 是 | 7天/30天历史基线数据 |
| topology_map | object | 否 | 服务拓扑依赖关系图 |
| known_patterns | array | 否 | 历史故障模式库 |

## 最佳实践

1. **避免告警风暴**：对关联异常进行聚合，一组关联异常只产生一条告警
2. **上下文丰富**：每个异常应附带足够的上下文信息供后续判断
3. **动态阈值优先**：优先使用动态基线检测，静态阈值作为兜底
4. **季节性考量**：分析时考虑业务周期性（周一流量高/月底结算峰值）
5. **误报学习**：记录人工标记的误报，用于迭代优化检测规则

## 约束条件

- 分析时间窗口：06:25-06:30，需在5分钟内完成全部分析
- 不执行任何修复操作
- 严重异常推送需在确认后1分钟内完成
- 关联分析深度限制：最多追溯3层依赖关系
