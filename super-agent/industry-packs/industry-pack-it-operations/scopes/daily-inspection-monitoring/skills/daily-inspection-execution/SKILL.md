# 每日自动巡检执行

## 技能概述

本技能负责按照预定义的巡检模板，定时执行覆盖全栈技术层的系统健康度检查。巡检采集来自多个监控数据源的指标，与基线和阈值进行比对，输出结构化的巡检结果数据供后续分析和报告生成使用。

## 适用场景

- 每日06:00定时触发的常规自动巡检
- 大促前的专项压力评估巡检
- 架构变更后的稳定性验证巡检
- 每季度的深度全面健康评估

## 执行步骤

### 步骤1：巡检模板加载与任务初始化
1. 根据巡检类型（自动/专项/深度）加载对应的巡检模板
2. 确认数据源连接状态（Prometheus/Zabbix/CloudWatch/APM/日志平台）
3. 初始化巡检任务跟踪记录，标记开始时间

### 步骤2：多层指标数据采集
1. **基础设施层**：采集服务器CPU使用率、内存使用率、磁盘使用率及I/O、网络I/O流量及丢包率
2. **数据库层**：采集连接数/连接池使用率、慢查询统计(>1s)、主从复制延迟、锁等待情况
3. **网络层**：采集核心链路带宽利用率、网络延迟、丢包率、路由状态
4. **应用层**：采集服务响应时间(P50/P95/P99)、错误率、QPS/TPS、JVM堆内存及GC状态
5. **业务层**：采集交易成功率、业务转化率、用户活跃度指标

### 步骤3：数据有效性校验
1. 验证每个采集项的数据完整性（非null、时间戳有效、数值在合理范围内）
2. 对采集失败的项标记为"未完成"并记录失败原因（超时/连接拒绝/认证失败等）
3. 计算本次巡检的完成率（成功采集项/总巡检项）

### 步骤4：阈值与基线比对
1. 将采集到的当前值与预设阈值进行比对（CPU>80%、磁盘>85%、慢查询>100条/h、错误率>1%、P99>3s）
2. 将当前值与7天/30天历史基线进行比对，计算偏离百分比
3. 对每个指标标注状态：正常(Green)/关注(Yellow)/告警(Red)

### 步骤5：输出巡检原始结果
1. 输出结构化JSON格式的巡检结果数据
2. 包含每个巡检项的：指标名、当前值、阈值、基线值、偏离度、状态标签
3. 记录巡检完成时间和总耗时

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| inspection_type | string | 是 | 巡检类型：daily/special/deep |
| template_id | string | 是 | 巡检模板ID |
| target_scope | string | 否 | 巡检范围限定（默认全量） |
| data_sources | array | 否 | 指定数据源（默认全部） |

## 最佳实践

1. **并行采集**：对不同数据源使用并行请求，将总采集时间控制在30分钟内
2. **超时容错**：单个数据源超时（默认30秒）不应阻塞整体巡检流程
3. **增量对比**：保留前次巡检结果，支持快速识别新增异常
4. **采集窗口**：避免在业务高峰期（如整点秒杀）采集数据造成指标失真
5. **模板版本控制**：巡检模板变更需记录版本，支持回溯对比

## 约束条件

- 巡检执行时间窗口：06:00-06:30，超时需告警
- 采集频率：每日一次（常规），按需触发（专项）
- 数据源连接失败的重试策略：最多3次，间隔10秒
- 不执行任何修复或变更操作，仅采集和比对
