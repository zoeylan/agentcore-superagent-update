# 资源优化分析

## 技能概述

本技能负责识别资源浪费和利用率不合理的情况，生成缩容/降配/释放建议，同时评估扩容需求并给出成本最优的扩容方案。目标是在保障业务稳定的前提下，最大化资源利用效率、最小化基础设施成本。

## 适用场景

- 月度容量评估中的资源浪费识别
- 成本优化专项分析
- 扩容方案的规格选型和成本对比
- 季度资源利用率回顾与优化

## 执行步骤

### 步骤1：资源利用率画像
1. 对所有实例计算过去30天的资源利用率统计：平均值、P95峰值、最大值、最小值
2. 按利用率水平分组：
   - 极低利用率（CPU<10% 且 内存<20%）→ 释放/缩容候选
   - 低利用率（CPU 10-30% 且 内存 20-40%）→ 降配候选
   - 正常利用率（CPU 40-70% 且 内存 50-80%）→ 维持
   - 高利用率（CPU>70% 或 内存>80%）→ 扩容候选
3. 标注各实例的业务归属、环境类型（生产/预发/测试）

### 步骤2：低利用率实例分析
1. 对极低/低利用率实例进行排查：
   - 是否为灾备/容灾实例（常态低负载为正常）
   - 是否有定时任务峰值（如每日凌晨批处理）
   - 是否为预留buffer（业务明确要求的余量）
   - 是否为新部署尚未接入流量
2. 排除合理场景后，确认真正的资源浪费实例
3. 计算每个浪费实例的月成本和年化成本

### 步骤3：优化方案制定
1. **释放建议**：对30天内无任何访问/使用记录的实例建议释放
2. **降配建议**：对利用率持续低于30%的实例，计算满足P95峰值需求的最小规格，建议降配至该规格
3. **合并建议**：对同一业务的多个低利用率实例，评估是否可合并部署
4. **预留实例转换**：对稳定使用的按需实例建议转为预留实例以降低成本

### 步骤4：扩容方案设计
1. 对需要扩容的资源，确定扩容目标（满足30天/60天/90天增长需求）
2. 制定扩容规格方案（至少2个备选方案）
3. 计算各方案成本：一次性成本 + 月度运行成本
4. 评估扩容的实施复杂度（是否需要停机/迁移/重新部署）

### 步骤5：成本影响评估
1. 汇总优化方案的总预计节省金额（月度/年度）
2. 对扩容方案给出投资回报分析（扩容成本 vs 不扩容的业务风险成本）
3. 生成成本优化优先级排序（按投资回报率从高到低）

## 输入规格

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| instance_list | array | 是 | 实例清单（含规格/业务线/环境） |
| usage_statistics | object | 是 | 30天资源利用率统计数据 |
| cost_data | object | 否 | 实例成本数据（按需/预留价格） |
| business_context | object | 否 | 业务上下文（灾备标记/预留buffer说明） |

## 最佳实践

1. **充分排查**：缩容建议前务必排除灾备、批处理、预留等合理场景
2. **渐进式优化**：建议先降一级规格观察，而非一步到位降到最低
3. **业务确认**：所有缩容/释放建议需标注"需业务方确认"
4. **成本量化**：每条建议必须附带可量化的成本影响
5. **风险评估**：扩容不足的业务影响 >> 多扩容的成本浪费

## 约束条件

- 释放建议需满足：30天无使用 + 非灾备 + 非预留buffer
- 降配建议的目标规格需满足P95峰值+30%余量
- 成本计算基于最新的云服务商定价
- 不直接执行任何缩容/扩容操作，仅输出建议
